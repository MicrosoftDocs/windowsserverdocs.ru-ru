# <a name="add-branding-to-the-dashboard-remote-web-access-and-launchpad"></a>Добавление фирменной символики на панели мониторинга, удаленного доступа и запуска

> Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

Выполнять изменения фирменной символики можно путем добавления записей в реестре. Все записи фирменной символики в реестре для операционной системы находятся в папке `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Server\OEM` .

Для совместной фирменной символики логотип Windows Server Essentials должен иметь минимальную ширину **170 пикселей**, сохраняя правильные пропорции с **96 dpi**.

## <a name="to-add-branding-by-changing-the-registry"></a>Добавление фирменной символики путем изменения реестра

1. На сервере переместите указатель мыши в правый верхний угол экрана и нажмите кнопку **Поиск**.

2. В поле поиска введите **regedit**, затем щелкните приложение **Regedit**.

3. В области навигации разверните разделы **HKEY_LOCAL_MACHINE**, **SOFTWARE**, **Microsoft** и **Windows Server**. Если ключ **OEM** не существует, его можно создать:

    1. Щелкните правой кнопкой мыши **Windows Server**, выберите **Создать**, затем щелкните **Раздел**.

    2. В качестве имени раздела введите **OEM**.

4. Используемых Если вы создаете запись для логотипа, можно создать разные ключи для различения языковых версий логотипа. Например, для английской и немецкой версий эмблемы можно создать разделы en-us и de-de. Поскольку все файлы эмблем сохраняются в одной папке, для каждой языковой версии необходимо предоставить экземпляры файла изображения эмблемы с уникальным именем. В нашем примере создают файлы с именами DashboardLogo_en.png и DashboardLogo_de.png.

5. Щелкните правой кнопкой мыши **OEM** или щелкните правой кнопкой мыши соответствующий ключ языка, выберите пункт **создать**, а затем — **строковое значение**.

6. Введите имя строки, а затем нажмите клавишу ВВОД. Дополнительные сведения о строковых именах и значениях данных см. в разделе " **строки и значения реестра** " этой статьи.

7. Введите имя строки, а затем нажмите клавишу ВВОД. Дополнительные сведения о строковых именах и значениях данных см. в разделе " **строки и значения реестра** " этой статьи.

8. Щелкните новую строку правой кнопкой мыши и выберите команду **Изменить**.

9. Введите значение из таблицы, связанное с данным именем строки, и нажмите кнопку **OK**.

10. При создании записи для изображения логотипа или добавленных ссылок скопируйте файл в `%programFiles%\Windows Server\Bin\OEM` . Если каталог OEM не существует, создайте его.

11. Если вы вносите изменения в удаленный Веб-доступ после того, как клиент выберет сервер, он должен включить удаленный Веб-доступ следующим образом:

    1. Щелкните **Параметры** на панели мониторинга, а затем выберите вкладку **Повсеместный доступ**.

    2. Если **доступ из любого места** включен, нажмите кнопку **настроить**, а затем снимите флажок Удаленный веб-доступ на странице **Выбор компонентов с повсеместным доступом для включения** в мастере **настройки повсеместного доступа** .

    3. Щелкните **Настройка**.

#### <a name="xml-linking-format"></a>Формат связывания XML

Необходимо отформатировать ссылки на **домашней** странице веб-сайта и на странице входа следующим образом:

```xml
<OemLinks>
    <LogonLinks>
        <Link Name=LogonLinkName>
        <Text>LogonLinkDescription</Text>
        <Url>LogonLinkURL</Url>
        <Icon>LinkIcon</Icon>
        </Link>
    </LogonLinks>

    <HomepageLinks>
        <Link Name=HomepageLinkName>
        <Text>HomepageLinkDescription</Text>
        <Url>HomepageLinkURL</Url>
        <Icon>HomepageLinkIcon</Icon>
        </Link>
    </HomepageLinks>
</OemLinks>
```

### <a name="linksxml-elements"></a>Элементы LinksXML

| Элемент LinksXML | Описание |
|--|--|
| LogonLinks | Родительская запись для ссылок входа. |
| Имя связи | Имя ссылки для входа. |
| Text | Текст, отображаемый как ссылка на страницу входа. |
| URL-адрес | URL-адрес, который разрешается в ссылку на страницу входа. |
| Значок | Имя файла значка для ссылки для входа. Этот файл должен располагаться в одной папке с XML-файлом. Изображения значков должны быть 16x16 пикселей и в формате PNG. Если не указать значок, используется изображение значка ссылки по умолчанию. |
| HomepageLinks | Родительская запись для **домашней** страницы. |
| Имя связи | Имя ссылки на **домашнюю** страницу. |
| Text | Текст, отображаемый в ссылке на **домашнюю** страницу. |
| URL-адрес | URL-адрес, который разрешается в ссылку на **домашнюю** страницу. |
| Значок | Имя файла значка для ссылки на **домашнюю** страницу. Этот файл должен располагаться в одной папке с XML-файлом. Изображения значков должны быть 16x16 пикселей и в формате PNG. Если значок не указан, используется изображение значка ссылки на **домашнюю** страницу по умолчанию. |
