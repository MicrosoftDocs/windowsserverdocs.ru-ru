# <a name="add-branding-to-the-dashboard-remote-web-access-and-launchpad"></a>Добавление фирменной символики на панели мониторинга, удаленного доступа и запуска

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

##  <a name="add-branding-to-the-dashboard-remote-web-access-and-launchpad"></a><a name="BKMK_Branding"></a>Добавление фирменной символики на панель мониторинга, удаленный Веб-доступ и панель запуска  
 Выполнять изменения фирменной символики можно путем добавления записей в реестре. Все элементы фирменной символики операционной системы находятся HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Server\OEM.  
  
 Вся совместная фирменная символика должна соответствовать следующим требованиям к эмблемам:  
  
-   Логотип Windows Server Essentials должен иметь минимальную ширину в **170 пикселей**, сохраняя правильное пропорции с **96 dpi**.  
  
#### <a name="to-add-branding-by-changing-the-registry"></a>Добавление фирменной символики путем изменения реестра  
  
1.  На сервере переместите указатель мыши в правый верхний угол экрана и нажмите кнопку **Поиск**.  
  
2.  В поле поиска введите **regedit**, затем щелкните приложение **Regedit**.  
  
3.  В области навигации разверните разделы **HKEY_LOCAL_MACHINE**, **SOFTWARE**, **Microsoft** и **Windows Server**. Если раздел **OEM** не существует, создайте его, выполнив следующие действия.  
  
    1.  Щелкните правой кнопкой мыши **Windows Server**, выберите **Создать**, затем щелкните **Раздел**.  
  
    2.  В качестве имени раздела введите **OEM**.  
  
4.  (Необязательно) При создании записи для эмблемы можно создать отдельные разделы для ее различных языковых версий. Например, для английской и немецкой версий эмблемы можно создать разделы en-us и de-de. Поскольку все файлы эмблем сохраняются в одной папке, для каждой языковой версии необходимо предоставить экземпляры файла изображения эмблемы с уникальным именем. В нашем примере создают файлы с именами DashboardLogo_en.png и DashboardLogo_de.png.  
  
5.  Щелкните правой кнопкой мыши раздел **OEM** или раздел соответствующей языковой версии, выберите команду **Создать** и щелкните пункт **Строковый параметр**.  
  

6.  Введите имя строки, а затем нажмите клавишу ВВОД. Строковые имена и значения параметров см. в таблице [Строки и параметры реестра](Add-Branding-to-the-Dashboard--Remote-Web-Access--and-Launchpad.md#BKMK_RegStrings).  

6.  Введите имя строки, а затем нажмите клавишу ВВОД. Строковые имена и значения параметров см. в таблице [Строки и параметры реестра](../install/Add-Branding-to-the-Dashboard--Remote-Web-Access--and-Launchpad.md#BKMK_RegStrings).  

  
7.  Щелкните новую строку правой кнопкой мыши и выберите команду **Изменить**.  
  
8.  Введите значение из таблицы, связанное с данным именем строки, и нажмите кнопку **OK**.  
  
9. При создании записи для изображения эмблемы или для присоединенных ссылок скопируйте соответствующий файл в папку %programFiles%\Windows Server\Bin\OEM. Если каталог OEM не существует, создайте его.  
  
10. Если внесенные изменения касаются функции удаленного веб-доступа, пользователь, получив владение сервером, должен включить удаленный веб-доступ. Пользователю рекомендуется выполнить следующие действия.  
  
    1.  Щелкните **Параметры** на панели мониторинга, а затем выберите вкладку **Повсеместный доступ**.  
  
    2.  Если Повсеместный доступ включен, щелкните **Настройка** и уберите флажок с пункта "Удаленный веб-доступ" на странице **Выбор функций повсеместного доступа** в мастере установки повсеместного доступа.  
  
    3.  Нажмите кнопку **Настройка**.

### <a name="linksxml-elements"></a>Элементы LinksXML  
  
|Элемент LinksXML|Описание|  
|----------------------|-----------------|  
|**логонлинкс**|  
|LogonLinkName|Имя ссылки на страницу входа.|  
|LogonLinkDescription|Текст, который отображается в виде ссылки на страницу входа.|  
|LogonLinkURL|URL-адрес, в который разрешается ссылка на страницу входа.|  
|LinkIcon|Имя файла значка для ссылки на страницу входа. Этот файл должен располагаться в одной папке с XML-файлом. |HomepageLinkName|Имя ссылки на домашнюю страницу.|  
|HomepageLinkDescription|Текст, который отображается в виде ссылки на домашнюю страницу.|  
|HomepageLinkURL|URL-адрес, который разрешается как ссылка на домашнюю страницу.|  
|HomepageLinkIcon|Имя файла значка для ссылки на домашнюю страницу. Этот файл должен располагаться в одной папке с XML-файлом. При отсутствии файла HomepageLinkIcon используется изображение значка ссылки на домашнюю страницу по умолчанию.|
