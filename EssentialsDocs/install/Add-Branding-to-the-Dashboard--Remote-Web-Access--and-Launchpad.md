# <a name="add-branding-to-the-dashboard-remote-web-access-and-launchpad"></a>Добавление фирменной символики на панели мониторинга, удаленного веб-доступа и панель запуска

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

##  <a name="BKMK_Branding"></a>Добавление фирменной символики на панели мониторинга, удаленного веб-доступа и панель запуска  
 Фирменной символики можно сделать путем добавления записей в реестре. Все фирменной символики в реестре для операционной системы, находятся в разделе HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows Server\OEM.  
  
 Вся Совместная фирменная символика должна соответствовать следующим требованиям к эмблемам:  
  
-   Логотип Windows Server Essentials должен иметь минимальную ширину **170 пикселей**, сохранением правильных пропорций, с **96 точек на ДЮЙМ**.  
  
#### <a name="to-add-branding-by-changing-the-registry"></a>Добавление фирменной символики путем изменения реестра  
  
1.  На сервере, переместите указатель мыши в правый верхний угол экрана, а затем нажмите кнопку **поиска**.  
  
2.  В поле поиска введите **regedit**, а затем нажмите кнопку **Regedit** приложения.  
  
3.  В области навигации разверните **HKEY_LOCAL_MACHINE**, разверните **программного обеспечения**, разверните **Microsoft**, разверните **Windows Server**. Если **OEM** не существует, выполните следующие действия, чтобы создать его:  
  
    1.  Щелкните правой кнопкой мыши **Windows Server**, нажмите кнопку **New**, а затем нажмите кнопку **ключ**.  
  
    2.  Тип **OEM** для имени ключа.  
  
4.  (Необязательно) Если вы создаете запись для логотипа, можно создать отдельные разделы для ее различных языковых версий логотипа. Например, если у вас есть версии на английском и немецком логотипа, можно создать en-us ключа и ключ, de-de. Так как все файлы логотип хранятся в той же папке, необходимо предоставить экземпляров файла с изображением эмблемы уникальное имя для каждого языка. Например вы создадите файл с именем DashboardLogo_en.png и DashboardLogo_de.png.  
  
5.  Либо щелкните правой кнопкой мыши **OEM** или щелкните правой кнопкой мыши раздел соответствующий язык, щелкните, чтобы **New**, а затем нажмите кнопку **строковое значение**.  
  

6.  Введите имя строки и нажмите клавишу ВВОД. Обратитесь к [строки реестра и значения](Add-Branding-to-the-Dashboard--Remote-Web-Access--and-Launchpad.md#BKMK_RegStrings) таблицы для строковых значений имен и данных.  

6.  Введите имя строки и нажмите клавишу ВВОД. Обратитесь к [строки реестра и значения](../install/Add-Branding-to-the-Dashboard--Remote-Web-Access--and-Launchpad.md#BKMK_RegStrings) таблицы для строковых значений имен и данных.  

  
7.  Щелкните правой кнопкой мыши новую строку и выберите **изменить**.  
  
8.  Введите значение из таблицы, который связан с именем строки, а затем нажмите кнопку **ОК**.  
  
9. Если вы создаете запись для изображение логотипа и добавив ссылки, скопируйте файл %programFiles%\Windows Server\Bin\OEM. Если изготовитель Оборудования каталог не существует, создайте ее.  
  
10. Если были внесены изменения, влияющие на удаленный веб-доступ после клиент получит права доступа к серверу, их необходимо включить удаленный веб-доступ. Следует рекомендовать пользователю сделайте следующее:  
  
    1.  На панели мониторинга щелкните **параметры**, а затем нажмите кнопку **повсеместного доступа** вкладку.  
  
    2.  Если включен повсеместного доступа, нажмите кнопку **Настройка**и снимите флажок удаленного веб-доступа на **Выбор функций повсеместного доступа для включения** странице повсеместного доступа мастер настройки.  
  
    3.  Нажмите кнопку **Настройка**.

### <a name="linksxml-elements"></a>Элементов LinksXML  
  
|Элемент LinksXML|Описание|  
|----------------------|-----------------|  
|**LogonLinks**|  
|LogonLinkName|Имя входа ссылку.|  
|LogonLinkDescription|Текст, который отображается в виде ссылки на страницу входа в систему.|  
|LogonLinkURL|URL-адрес, которое разрешается ссылки на страницу входа в систему.|  
|LinkIcon|Имя файла значка для ссылки на вход в систему. Этот файл должен быть в том же расположении папку как XML-файл. Связать значок изображения должны быть 16 x 16 пикселей и должен быть в формате PNG. Если LinkIcon не указано, используется по умолчанию ссылку значок.|  
|**HomepageLinks**|  
|HomepageLinkName|Имя связи домашнюю страницу.|  
|HomepageLinkDescription|Текст, отображаемый в виде ссылки на домашнюю страницу.|  
|HomepageLinkURL|URL-адрес, устраняющее по ссылке, домашнюю страницу.|  
|HomepageLinkIcon|Имя файла значка для ссылки на домашнюю страницу. Этот файл должен быть в том же расположении папку как XML-файл. HomepageLinkIcon изображения должны быть 16 x 16 пикселей и должен быть в формате PNG. Если HomepageLinkIcon не указано, используется изображение значка ссылку Домашняя страница по умолчанию.|
