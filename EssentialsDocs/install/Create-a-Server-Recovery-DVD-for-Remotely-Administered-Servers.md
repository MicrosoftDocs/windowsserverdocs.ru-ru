# <a name="create-a-server-recovery-dvd-for-remotely-administered-servers"></a>Создание DVD-диска восстановления сервера для удаленно администрируемых серверов

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

##  <a name="BKMK_HeadlessRecovery"></a>Создание DVD-диска восстановления сервера для удаленно администрируемых серверов  
 Существует две модели для фабрики сброса и восстановления сервера и зависит от полученного пользователем оборудования.  
  
 **Удаленно администрируемый сервер**  
  
 Данный вариант восстановления сервера требует, что процесс выполняется на клиентском компьютере в той же сети. Поскольку значениям по умолчанию требуется, что серверу быть прилагался образ конкретного оборудования, партнером должен создавать DVD-диска восстановления сервера.  
  
 **Локально администрируемый сервер**  
  
 Это Классическая модель, при которой сервер администрируется с консоли сервера. Для выполнения восстановления используется установочный носитель сервера. Для этого необходимо, чтобы сервер оснащался возможность просмотра видеосигнала устройство чтения DVD-диска. Пользователь загружается с установочного носителя указанного сервера и выбирает соответствующий способ восстановления. Необходимо создать DVD-диска восстановления сервера для локально администрируемых серверов.  
  
> [!NOTE]
>  При использовании модели локально администрируемого сервера пользователь может выполнить сброс к фабричным настройкам при выполнении новой установки. Тем не менее они будут потеряны партнера настройки.  
  
 Существует два типа восстановления сервера.  
  
 **Сброс настроек.**  
  
 Этого восстановления сервер возвращается в исходное состояние, существовавшее на момент отправки сервера с завода-изготовителя. К значениям по умолчанию вам будет предложено выполнить начальной настройки сервера, как вы первый раз, в которой вы включили и все параметры и настройки будут потеряны. Это также называют день 0.? Поскольку значениям по умолчанию требуется, что серверу быть прилагался образ конкретного оборудования, партнером должен создавать DVD-диска восстановления сервера.  
  
 **Восстановление исходного состояния системы**  
  
 Эта операция восстановления предполагается, что пользователь настроил архивацию сервера, что в резервную копию сервера была успешно выполнена хотя бы один раз до выхода сервера из строя. Восстановление исходного состояния системы (BMR) поддерживает восстановление системы и загрузочных дисков только из предыдущей резервной копии сервера.  
  
 После восстановления исходного состояния системы сервер возвращается к состоянию, существовавшему на момент создания архива, который используется для восстановления. Обычно это последней резервной копии, но в некоторых случаях может быть более ранней резервной копии. Восстановление данных выполняется после восстановления системы с помощью восстановления файлов и папок. Восстановление исходного состояния системы является предпочтительным способом восстановления, поскольку возвращаются все параметры и конфигурации, тогда как к значениям по умолчанию сервер возвращается к состоянию на день 0.  
  
### <a name="remotely-administered-server-recovery"></a>Восстановление удаленно администрируемого сервера  
 В этом разделе описаны необходимые настройки, которые должен выполнить партнер, а также конечные носители, которые должны прилагаться к каждому серверу. Прежде чем перейти к сведениях описанию, рассмотрим взаимодействие с пользователем.  
  
 В этом сценарии клиент «больше не работает сервер s отчета. Это может быть вызвано вирус, неисправность жесткого диска или некоторые другие причины. Пользователь вставляет DVD-диска восстановления сервера в клиентский компьютер, расположенный в той же сети, что и сервер. Приложение восстановления сервера выполняет три этапа восстановления:  
  
1.  Создает загрузочный USB-накопитель, используемый для перезагрузки сервера в режиме восстановления. USB-накопителя должна быть 8 ГБ или больше.  
  
2.  Обнаружение сервера, которая теперь находится в режиме восстановления.  
  
3.  Пользователь должен выбрать восстановления исходного состояния системы или к значениям по умолчанию и возвращает их сервер в рабочее состояние.

####  <a name="BKMK_Updating"></a>Шаг 1. (Необязательно) обновление среды предустановки Windows  
 ADK включает настроенную среду Предустановки Windows. При загрузке этого образа автоматически запускается маяк, используемый клиентским приложением восстановления для подключения к серверу в режиме восстановления.  
  
 Необходимо выполнить дальнейшую настройку путем добавления конкретных драйверов оборудования, например сети и драйверов контроллеров дисков среды Предустановки Windows. После загрузки из среды предустановки Windows, жесткие диски системы должны стать распознаваемыми, а сеть должна работать.  
  
####  <a name="BKMK_Collecting"></a>Шаг 2: Сбор фабрика сброса настроек образов и XML-файлов  
 Чтобы сбросить параметры сервера к значениям по умолчанию, необходимо захватить следующие два образа:  
  
-   Образ системного диска  
  
-   Системный раздел  
  
 Для создания этих образов, предоставляется средство GenDiskXML.exe. GenDiskXML.exe также собирает сведения о файле с именем disk.xml, который используется процедурой восстановления для повторного создания конфигурации диска.  
  
1.  После выполнения программы Sysprep перезагрузите систему с использованием 64-разрядной версии среды Предустановки Windows.  
  
2.  Файлы .xml и .wim на внешний носитель выполните `GenDiskXML /outputdir:<path>`для записи файлов .xml и .wim на любой внешний носитель. Файлы добавляются на DVD-диск на следующем шаге.  
  
    > [!NOTE]
    >  WIM-файл системы будет разбит в соответствии с FAT32 на файл, не более 4 ГБ. В ходе процесса емкость целевого устройства, используемого для записи WIM-файлов, должна быть больше 8 ГБ для поддержания процедуры разбиения.  
  
####  <a name="BKMK_Creating"></a>Шаг 3: Создание DVD-диска восстановления сервера  
 Каждый сервер, поставляемый с завода-изготовителя должен комплектоваться DVD-диска восстановления сервера. DVD-диска средства ADK содержит файлы, необходимые для создания DVD-диска.  
  
###### <a name="to-create-the-server-recovery-dvd"></a>Для создания DVD-диска восстановления сервера  
  
1.  Создайте рабочую папку для использования в качестве расположения для хранения конечного образа ISO.  
  
2.  Скопируйте содержимое из партнерском компакт-диске **восстановления сервера** папки в рабочую папку, созданную на шаге 1.  
  
3.  Скопируйте файл disk.xml, созданный при выполнении GenDiskXML.exe для **Сброс** папки.  
  
4.  Скопируйте файлы образов, которые были созданы при выполнении **GenDiskXML.exe** для **Сброс** папки. Это WIM- и SWM файлы, и число файлов, могут отличаться.  
  
5.  Удалите GenDiskXML.exe из папки. Он используется только для целей фабрики и его не следует включать на DVD-ДИСК, поставляемый пользователю.  
  
####  <a name="BKMK_Customizing"></a>Шаг 4: Настройка мастера  
 Приложение восстановления сервера следует настроить, используя образ устройства и текст с описанием способа загрузки конкретного устройства в режим восстановления. На этой странице восстановления файлов и папок определяется оборудованием, этапы загрузки сервера в режиме восстановления могут различаться.  
  
> [!NOTE]
>  Перечисленные имена файлов должны точно совпадать.  
  
1.  Страница мастера снабжается ссылкой на дополнительную справку. Если такой CHM-файл существует, он переопределяет ссылку FWLink на веб-сайт справки. Файл справки находится в папке:  
  
     \RestartHelp.chm \\$OEM$\Customization\\ < имя языка и региональных параметров > < DVD-диска Root\ >  
  
2.  Этот файл содержит текст, который пользователь видит на странице мастера. Текст должен содержать описание процедуры загрузки сервера в режим восстановления. Элемент управления прокрутки, который помещает реальное ограничение на объем текста, который может быть добавлен.  
  
     Следующий файл используется для замены образца рисунка в мастере и касается в основном фирменной символики. Он должен быть PNG-файл. Размер файлов должен составлять 256 пикселей x 256 или оно будет обрезано при отображении в мастере.  
  
     \RestartInstructions.rtf \\$OEM$\Customization\\ < имя языка и региональных параметров > < DVD-диска Root\ >  
  
3.  \ServerImage.png \\$OEM$\Customization\\ < имя языка и региональных параметров > < DVD-диска Root\ >  
  
 При преобразовании DVD-диска для поддержки нескольких языков восстановления сервера, обязательно выполните следующее:  
  
1.  Всегда должен действовать en-us папки. Если приложение восстановления сервера не обнаружит файлы языка, соответствующие работает на клиентском компьютере, оно переключится на язык en-us.  
  
2.  В папке каждого языка и региональных параметров, создания, добавьте три файла настроек (.png, .chm и RTF).  
  
3.  Скопируйте обе папки языка и региональных параметров из языка Packs\\ < CultureName\ > \Server Recovery в корень DVD-диска восстановления сервера. Например: ES и ES-ES папки будут скопированы в корневой каталог DVD для поддержки испанского языка.  
  
4.  Завершите ISO-файл.  
  
 Поддерживаются следующие коды языков включают:  

|-|-|  
|-Cs-CZ<br /><br /> -De-DE<br /><br /> -En-US<br /><br /> -Es-ES<br /><br /> -Fr-FR<br /><br /> -Hu-HU<br /><br /> -It-IT<br /><br /> -Ja-JP<br /><br /> -Ko-KR<br /><br /> -Nl-NL |-pl-PL<br /><br /> -Pt-BR<br /><br /> -Pt-PT<br /><br /> -Ru-RU<br /><br /> -Sv-SE<br /><br /> -Tr-TR<br /><br /> -Zh-CN<br /><br /> -Zh-HK<br /><br /> -Zh-TW
  
####  <a name="BKMK_CreatingISO"></a>Шаг 5: Копирование файла ISO  
 Созданную папку и все содержимое можно записать на DVD-диск. Это DVD-ДИСК будет прилагаться к каждому новому серверу.  
  
####  <a name="BKMK_Testing"></a>Шаг 6: Тестирование DVD-диска восстановления  
 После завершения установки сервера, настройте архивацию, выполнение архивации сервера и затем протестируйте DVD-диска восстановления.  
  
###### <a name="to-configure-and-run-a-server-backup"></a>Настройка и выполнение архивации сервера  
  
1.  Откройте панель мониторинга, нажмите кнопку **устройств**, а затем щелкните **Настройка архивации сервера** в **задачи** области.  
  
2.  Следуйте указаниям мастера для настройки архивации для архивации. Для архива потребуется внешний жесткий диск.  
  
3.  Нажмите кнопку **начать архивацию сервера** в **задачи** области и следуйте инструкциям мастера.  
  
4.  После завершения архивации убедитесь, что она выполнена успешно.  
  
###### <a name="to-restore-a-server"></a>Для восстановления сервера  
  
1.  Вставьте созданный DVD-ДИСК восстановления в клиентский компьютер, который подключен к той же сети, что и сервер через концентратор или коммутатор.  
  
2.  Дважды щелкните **setup.exe**. Мастер восстановления сервера будет предложено пройти такой же процесс, который будет следовать пользователь.  
  
3.  Нажмите кнопку **восстановление сервера из резервной копии**, а затем следуйте инструкциям мастера.
