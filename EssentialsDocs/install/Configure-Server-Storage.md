# <a name="configure-server-storage"></a>Настройка хранилища сервера

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

## <a name="sample-hard-disk-configurations"></a>Примеры конфигураций жестких дисков  
 В таблице ниже приведены примеры конфигураций жесткого диска. Оценки основаны на типичных сценариях использования и функциональных возможностях, но они не учитываются неполадки, влияющие на оптимальную производительность. Можно использовать любой тип, поддерживаемый жестких дисков для этих конфигураций (таких как SATA и SCSI) на основе предпочтений и потребностей клиентов.  
  
> [!IMPORTANT]
>   Windows Server Essentials должен быть установлен как диск c:, а размер тома должен быть не менее 60 ГБ. Рекомендуется создать два раздела на диске с операционной системой и не использовать C: (системный раздел) для хранения деловой информации.  
  
|Уровень сервера|Конфигурация дисков|  
|------------------|------------------------|  
|Запись|-Двух физических дисков<br /><br /> -Настроен как набор RAID 1 Зеркальное отображение, который содержит следующее:<br /><br /> -Тома C:? 60 ГБ<br /><br /> -Том D:? 1000 ГБ|  
|Средний|-Три физических диска<br /><br /> -Настроен как набор RAID 5, который содержит следующее:<br /><br /> -Тома C:? 60 ГБ<br /><br /> -Том D:? 1500 ГБ|  
|Высокий|-Пять или более общее количество физических дисков<br /><br /> -Двух дисков в массиве RAID 1 Зеркальное отображение набор, который содержит тома C:? 100 ГБ<br /><br /> -Все остальные диски объединены в набор RAID 5, который содержит следующее:<br /><br /> -Том D:? 1500 ГБ<br /><br /> -Тома E:? 1500 ГБ|  
  
 Эти рекомендации учитывать размер установленной операционной системы, Средний размер хранилища данных, используемый сервером и роста хранилища ожидаемые данные временем жизни сервера. Тома могут быть разделы на один физический диск, или могут быть размещены на разных физических дисках. На сервере хранятся важные данные клиентов, рекомендуется использовать несколько физических дисков и защищать данные s клиента с помощью аппаратных RAID или дисковых пространств.  
  
## <a name="configuring-your-server-backup"></a>Настройка архивации сервера  
 Помимо Внутренние жесткие диски на сервере клиентам следует использовать внешние жесткие диски USB для резервного копирования. Теоретически клиент должен иметь по крайней мере два внешние жесткие диски с достаточной мощностью, чтобы создать резервную копию всех данных на сервере. Если используются внешние жесткие диски, пользователь может использовать один диск вне вычислительной системы каждую ночь для дополнительной защиты данных.  
  
## <a name="partition-configuration"></a>Конфигурация разделов  
 Во время начальной настройки сервера создается набор папок сервера по умолчанию, включающий общие папки и папку архивации клиентского компьютера создаются в самом большом разделе данных на диске 0.
