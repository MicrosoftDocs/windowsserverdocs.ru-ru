# <a name="preconfiguring-a-router"></a>Предварительная настройка маршрутизатора

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

Как правило новой установке операционной системы требуется Интернет-маршрутизатор и брандмауэр для подключения к внутренней сети клиента в Интернете. Если вы предоставляете маршрутизатор как дополнительное значение комплект предварительно настроенного сервера, можно выполнить дополнительные действия, настроить маршрутизатор для повышения удобства работы пользователя.  
  
 В маршрутизаторе должен быть включен протокол DHCP. Серверу необходимо назначить статический IP-адрес. Для этого, резервирования DHCP IP-адреса или путем присвоения IP-адрес за пределами диапазона адресов DHCP.  
  
 Требуется также настроить параметры перенаправления портов на маршрутизаторе для пересылки определенные порты из внешнего интерфейса маршрутизатора адрес сервера во внутренней сети. В следующей таблице перечислены рекомендуемые конфигурации.  
  
|Параметр конфигурации|Сведения о|  
|---------------------------|-------------|  
|DHCP|На|  
|Перенаправление портов|Следует перенаправлять следующие порты на адрес сервера:<br /><br /> -80 (для размещенной конфигурации используется только 443)<br />-   443|  
|Поддержка UPnP|Необходимо включить поддержку UPnP™ обеспечить настройки маршрутизатора для пользователя и наиболее удобный способ установки.<br /><br /> **Предупреждение:** архитектура UPnP может представлять угрозу безопасности, если она включена.|  
  
 Помимо параметров, предварительной настройки маршрутизатора базовых необходимо выполнить следующие задачи для обеспечения более полного взаимодействия с пользователем для управления маршрутизатором.  
  
-   Расширение панели администрирования путем предоставления надстройки на сервере, который позволяет управлять маршрутизатором через настраиваемый интерфейс пользователя.  
  
-   Расширение оповещений о работоспособности, чтобы все оповещения от маршрутизатора может отображаться в средстве просмотра оповещений.  
  
-   Если маршрутизатор поддерживает несколько подсетей, IP-адрес сервера должен обрабатываться как один DNS-сервер через службу DHCP.  
  
-   Если в маршрутизаторе предусмотрена Интегрированная функция управления доступом для доменных служб Active DirectoryÂ®, можно автоматизировать интеграцию Active Directory во время начальной настройки сервера. Можно также предоставить через маршрутизатор управления надстройки на информационной панели.  
  
> [!NOTE]
>  Дополнительные сведения о настройке беспроводных соединений см. в разделе [Настройка поддержки беспроводных сетей](Configure-Support-for-a-Wireless-Network.md).
