# <a name="rules-used-by-the-windows-server-essentials-best-practices-analyzer-bpa-tool"></a>Правила, используемые с использованием средства Windows Server Essentials лучший анализатор соответствия рекомендациям (BPA)

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

В этой статье описываются правила, используемые в Windows Server Essentials (Анализатор рекомендациям). Анализатор BPA проверяет сервер, под управлением Windows Server Essentials и выводит отчет с описанием проблем и рекомендациями по их устранению. Данные рекомендации разработаны организацией поддержки продукта для Windows Server Essentials.  
  
## <a name="using-the-tool"></a>С помощью средства  
 Это стандартный практический, при миграции на Windows Server Essentials из Windows Server 2011 Essentials, Windows Small Business Server 2011 Essentials или Windows Home Server 2011, для запуска анализатора соответствия Рекомендациям на конечном сервере после завершения миграции параметров и данных. Вы можно запустить из панели мониторинга в любое время.  
  
#### <a name="to-run-the--windows-server-essentials-bpa-on-the-server"></a>Для запуска анализатора соответствия Рекомендациям Windows Server Essentials на сервере  
  
1.  Войдите на сервер с правами администратора и откройте панель мониторинга.  
  
2.  На панели мониторинга щелкните **устройств** вкладку.  
  
3.  На **задачи сервера** панели, щелкните **анализатор соответствия рекомендациям**.  
  
4.  Просмотрите все сообщения Анализатора и следуйте инструкциям для устранения проблем, при необходимости.  
  
## <a name="rules-used-by-the-best-practices-analyzer"></a>Правила, используемые анализатором соответствия рекомендациям  
  
### <a name="disable-ip-filtering"></a>Отключение фильтрации IP-адресов  
 **Проблема:** фильтрация IP-адресов сейчас включена на сервере. Необходимо отключить фильтрацию IP-адресов.  
  
 **Влияние:** Если включена фильтрация IP-адресов, сетевой трафик может быть заблокирован.  
  
 **Решение:**  
  
##### <a name="to-disable-ip-filtering"></a>Отключение фильтрации IP-адресов  
  
1.  Откройте regedit.exe на сервере.  
  
2.  Перейдите к разделу HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters.  
  
3.  Щелкните правой кнопкой мыши **EnableSecurityFilters**, а затем нажмите кнопку **изменить**.  
  
4.  В **изменение параметра DWORD (32-разрядная версия) значение** измените **значение** на ноль и нажмите кнопку **ОК**.  
  
5.  Чтобы применить изменения, перезапустите сервер.  
  
### <a name="the-distributed-transaction-coordinator-msdtc-service-should-be-set-to-start-automatically-by-default"></a>Служба координатора распределенных транзакций (MSDTC) должно быть присвоено автоматический запуск по умолчанию  
 **Проблема:** служба MSDTC не настроена на автоматический запуск  
  
 **Влияние:** служба MSDTC может не запускаться автоматически при запуске сервера. Если служба остановлена, может произойти сбой некоторых функций SQL Server или COM. В результате приложения, использующие функции Microsoft SQL Server или COM, могут работать неправильно.  
  
 **Решение:**  
  
##### <a name="to-configure-the-msdtc-service-to-start-automatically"></a>Настройка автоматического запуска службы MSDTC  
  
1.  Откройте services.msc на сервере.  
  
2.  Щелкните правой кнопкой мыши **координатора распределенных транзакций** службы, а затем нажмите кнопку **свойства**.  
  
3.  На **Общие** измените **тип запуска** для **автоматически (отложенный запуск)**, а затем нажмите кнопку **ОК**.

##### <a name="to-configure-the-block-level-backup-engine-service-for-manual-startup"></a>Настройка ручного запуска для службы модуля архивации уровень блока  
  
1.  Откройте консоль служб.  
  
2.  В списке служб дважды щелкните **служба модуля архивации на уровне блоков**.  
  
3.  Для **тип запуска**выберите **вручную**, а затем нажмите кнопку **применить**.  
  
### <a name="the-logon-account-for-the-block-level-backup-engine-service-is-not-set-to-use-the-local-system-account"></a>Учетная запись входа для службы модуля архивации на уровне блоков не задано использование учетной записи Local System  
 **Проблема:** службы модуля архивации на уровне блоков не задано использование учетной записи Local System для входа в систему по умолчанию.  
  
 **Влияние:** Если служба модуля архивации на уровне блоков не использует Local System в качестве учетной записи для входа по умолчанию, могут возникнуть ошибки, связанные с разрешениями. Эти ошибки могут помешать успешному задания архивации данных Windows Server.  
  
 **Решение:**  
  
##### <a name="to-configure-the-block-level-backup-engine-service-to-use-local-system-as-the-default-logon-account"></a>Настройка службы модуля архивации уровне блоков для использования Local System в качестве учетной записи для входа по умолчанию  
  
1.  Откройте консоль служб.  
  
2.  В списке служб дважды щелкните **служба модуля архивации на уровне блоков**.  
  
3.  На **свойства службы** щелкните **вход в систему** вкладку.  
  
4.  Выберите **локальной системной учетной записи**, а затем щелкните **применить**.  
  
5.  Перезапустите службу.  
  
### <a name="the-common-name-on-the-certificate-that-is-bound-to-the-wss-certificate-web-service-website-does-not-match-the-server-name"></a>Общее имя сертификата, привязанного к веб-сайту веб-служба сертификатов WSS не совпадает с именем сервера  
 **Проблема:** на веб-сайт веб-служба сертификатов WSS в службах IIS привязан недопустимый сертификат. Общее имя в этом сертификате не совпадает с именем сервера.  
  
 **Влияние:** Если привязать недопустимый сертификат к веб-сайту веб-служба сертификатов WSS, мастер подключения может работать неправильно.  
  
 **Решение:**  
  
##### <a name="to-configure-a-valid-certificate-for-the-wss-certificate-web-service"></a>Настройка действительного сертификата для веб-служба сертификатов WSS  
  
1.  Откройте диспетчер Internet Information Services (IIS) на сервере.  
  
2.  В диспетчере служб IIS разверните имя сервера и нажмите кнопку **сайты**.  
  
3.  Щелкните правой кнопкой мыши **веб-служба сертификатов WSS**, а затем нажмите кнопку **изменить привязки**.  
  
4.  В **привязки сайта**, нажмите кнопку **HTTPS**, а затем нажмите кнопку **изменить**.  
  
5.  В **изменение привязки сайта**, для **SSL-сертификат**, выберите сертификат, который имеет то же имя, что и сервер.  
  
6.  Если более чем одной записи сертификата имеет имя, совпадающее с именем сервера, щелкните **представление** чтобы определить, какой сертификат допустим, а затем выберите соответствующий сертификат.  
  
### <a name="there-appears-to-be-a-problem-with-the-certificate-binding-for-the-remote-desktop-gateway-service"></a>Возможно, возникла проблема с привязкой сертификата для службы шлюза удаленных рабочих столов  
 **Проблема:** похоже, сертификат службы шлюза удаленных рабочих столов привязан неправильно.  
  
 **Влияние:** Если сертификат службы шлюза удаленных рабочих столов не настроен правильно, пользователи не смогут подключиться к удаленному веб-доступа.  
  
 **Решение:**  
  
##### <a name="to-fix-the-binding-for-the-remote-desktop-gateway-service"></a>Исправление привязки для службы шлюза удаленных рабочих столов  
  
-   Откройте командную строку от имени администратора и введите следующие команды:  
  
    ```  
    REG ADD HKLM\SYSTEM\CurrentControlSet\services\HTTP\Parameters\SslBindingInfo\0.0.0.0:443  /v DefaultFlags /t REG_DWORD /d 1 /f  
    net stop tsgateway  
    net start tsgateway  
    ```  
  
     Дополнительные сведения см. в разделе [управление службой шлюза удаленных рабочих столов в Windows Server Essentials (статья базы знаний 2472211)](https://support.microsoft.com/kb/2472211) (https://support.microsoft.com/kb/2472211).
