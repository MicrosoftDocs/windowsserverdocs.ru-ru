##  <a name="BKMK_CopyData"></a>Копирование данных на конечный сервер  
 Перед копированием данных с исходного сервера на конечный сервер, выполните следующие задачи.  
  
-   Просмотрите список общих папок на исходном сервере, включая разрешения для каждой папки. Создайте или настройте папки на конечном сервере в соответствии со структурой папок, которую вы перемещаете с исходного сервера.  
  
-   Проверьте размер каждой папки и убедитесь, что целевой сервер имеет достаточно свободного дискового пространства.  
  
-   Сделайте общие папки на исходном сервере только для чтения для всех пользователей, не записи может занять место на диске, при копировании файлов на конечный сервер.  
  
#### <a name="to-copy-data-from-the-source-server-to-the-destination-server"></a>Копирование данных с исходного сервера на конечный сервер  
  
1.  Войдите на целевой сервер как администратор домена.  
  
2.  Нажмите кнопку **запустить**, тип **cmd** в поле поиска и нажмите клавишу ВВОД.  
  
3.  В командной строке введите следующую команду и нажмите клавишу ВВОД:  
  
    `robocopy \\<SourceServerName> \<SharedSourceFolderName> \\<DestinationServerName> \<SharedDestinationFolderName> /E /B /COPY:DATSOU /LOG:C:\Copyresults.txt`  
  
     Где:
     - \ < SourceServerName\ > — имя исходного сервера
     - \ < SharedSourceFolderName\ > — имя общей папки на исходном сервере
     - \ < DestinationServerName\ > — имя конечного сервера
     - \ < SharedDestinationFolderName\ > — Общая папка на конечном сервере, на который будут копироваться данные.  
 
4.  Повторите предыдущий шаг для каждой общей папки, которую вы перемещаете с исходного сервера.  
  
##  <a name="BKMK_ImportADaccounts"></a>Импорт пользовательских учетных записей Active Directory для мониторинга Windows Server Essentials (необязательно)  
 По умолчанию все учетные записи пользователей, созданные на исходном сервере, автоматически переносятся на панель мониторинга в Windows Server Essentials. Тем не менее автоматического переноса учетной записи пользователя в Active Directory завершится сбоем, если некоторые свойства не будут соответствовать требованиям миграции. Следующий командлет Windows PowerShell можно использовать для импорта пользователей Active Directory.  
  
#### <a name="to-import-an-active-directory-user-account-to-the-windows-server-essentials-dashboard"></a>Чтобы импортировать учетную запись пользователя Active Directory на панели мониторинга Windows Server Essentials  
  
1.  Войдите на целевой сервер как администратор домена.  
  
2.  Откройте Windows PowerShell от имени администратора.  
  
3.  Выполните следующий командлет, где `[AD username]` — это имя учетной записи пользователя Active Directory, который требуется импортировать:  
  
     `Import-WssUser SamAccountName [AD username]`  
  
##  <a name="BKMK_RemoveScripts"></a>Удаление старых сценариев входа (необязательно)  
 Для задач, таких как установка программного обеспечения и настройка рабочих столов Windows SBS 2003 используются сценарии входа в систему.  Windows Server Essentials заменяет сценариев входа в систему Windows SBS 2003 используется комбинация сценариев входа в систему и объекты групповой политики.  
  
> [!NOTE]
>  Если вы изменили сценарии входа в систему Windows SBS 2003, необходимо переименовать сценарии для сохранения настроек.  
  
> [!NOTE]
>  Сценарии входа в систему Windows SBS 2003 применяются только к учетных записей пользователей, которые были добавлены с помощью **мастера добавления новых пользователей**.  
  
#### <a name="to-remove-the-windows-sbs-2003-logon-scripts"></a>Удаление сценариев входа в систему Windows SBS 2003  
  
1.  Нажмите кнопку **запустить**, наведите курсор на **Администрирование**, а затем нажмите кнопку **Active Directory-пользователи и компьютеры**.  
  
2.  В **Active Directory-пользователи и компьютеры**, разверните свою сеть и нажмите кнопку **пользователей**.  
  
3.  Щелкните правой кнопкой мыши имя пользователя, нажмите кнопку **свойства**, а затем нажмите кнопку **профиль** вкладку.  
  
4.  Удалите содержимое **сценарий входа** текстовое поле, а затем нажмите кнопку **ОК**.  
  
5.  Повторите шаги 3 и 4 для каждого пользователя.  
  
##  <a name="BKMK_RemoveLegacyADGPO"></a>Удаление устаревших объектов Active Directory групповой политики (необязательно)  
 Объекты групповой политики (GPO) обновляются для Windows Server Essentials. Они являются подмножеством объектов групповой политики Windows SBS 2003. Для Windows Server Essentials ряд фильтров объектов групповой политики для Windows SBS 2003 и инструментария управления Windows (WMI) необходимо вручную удалить для предотвращения конфликтов с групповой политики Windows Server Essentials и фильтрами WMI.  
  
> [!NOTE]
>  Если вы изменили объекты групповой политики исходного Windows SBS 2003, необходимо сохранить их копии в другом месте и затем удалить их из Windows SBS 2003.  
  
#### <a name="to-remove-old-group-policy-objects-from-windows-sbs-2003"></a>Для удаления старых объектов групповой политики из Windows SBS 2003  
  
1.  Войдите на исходный сервер с учетной записью администратора.  
  
2.  Нажмите кнопку **запустить**, а затем нажмите кнопку **управление сервером**.  
  
3.  В области навигации щелкните **расширенного управления**, нажмите кнопку **Управление групповой политикой**, а затем нажмите кнопку **леса:***< YourDomainName\ >*.  
  
4.  Нажмите кнопку **домены**, нажмите кнопку *< YourDomainName\ >*, а затем нажмите кнопку **объектов групповой политики**.  
  
5.  Щелкните правой кнопкой мыши **Small Business Server-политика аудита**, нажмите кнопку **удалить**, а затем нажмите кнопку **ОК**.  
  
6.  Повторите шаг 5, чтобы удалить следующие объекты групповой политики, которые применяются к сети:  
  
    -   Клиентский компьютер Small Business Server  
  
    -   Политика паролей домена Small Business Server  
  
         Мы рекомендуем настроить политику паролей в Windows Server Essentials для применения надежных паролей. Чтобы настроить политику паролей, используйте панель мониторинга, которая записывает конфигурацию в политику домена по умолчанию. Конфигурация политики паролей не записывается в Small Business Server домена объект политики паролей, как это было в Windows SBS 2003.  
  
    -   Брандмауэр подключения к Интернету Small Business Server  
  
    -   Политика блокировки Small Business Server  
  
    -   Политика удаленного помощника Small Business Server  
  
    -   Брандмауэр Windows Small Business Server  
  
    -   Политика компьютера клиента служб обновления Small Business Server  
  
         Этот объект групповой Политики будет присутствовать, если вы проводите миграцию с Windows SBS 2003 R2.  
  
    -   Small Business Server служб политика общих параметров обновления  
  
         Этот объект групповой Политики будет присутствовать, если вы проводите миграцию с Windows SBS 2003 R2.  
  
    -   Политика компьютера сервера служб обновления Small Business Server  
  
         Этот объект групповой Политики будет присутствовать, если вы проводите миграцию с Windows SBS 2003 R2.  
  
7.  Убедитесь, что все объекты групповой политики удалены.  
  
#### <a name="to-remove-wmi-filters-from-windows-sbs-2003"></a>Удаление фильтров WMI из Windows SBS 2003  
  
1.  Войдите на исходный сервер с учетной записью администратора.  
  
2.  Нажмите кнопку **запустить**, а затем нажмите кнопку **управление сервером**.  
  
3.  В области навигации щелкните **расширенного управления**, нажмите кнопку **Управление групповой политикой**, а затем нажмите кнопку **леса:***< YourNetworkDomainName\ >*  
  
4.  Нажмите кнопку **домены**, нажмите кнопку *< YourNetworkDomainName\ >*, а затем нажмите кнопку **фильтры WMI**.  
  
5.  Щелкните правой кнопкой мыши **PostSP2**, нажмите кнопку **удалить**, а затем нажмите кнопку **Да**.  
  
6.  Щелкните правой кнопкой мыши **PreSP2**, нажмите кнопку **удалить**, а затем нажмите кнопку **Да**.  
  
7.  Убедитесь, что эти фильтры WMI удалены.  
  
##  <a name="BKMK_Configure"></a>Настройка сети  
  
#### <a name="to-configure-the-network"></a>Для настройки сети  
  
1.  На конечном сервере откройте панель мониторинга.  
  
2.  На панели мониторинга **Главная** щелкните **установки**, нажмите кнопку **Настройка повсеместного доступа**и выберите **щелкните, чтобы настроить Повсеместный доступ** параметр.  
  
3.  Следуйте инструкциям в **Настройка повсеместного доступа** мастера настройки своего маршрутизатора и имени домена.  
  
 Если маршрутизатор не поддерживает инфраструктуру UPnP, или если UPnP-инфраструктура отключена, рядом с именем маршрутизатора может появиться желтый значок предупреждения. Убедитесь, что открыты следующие порты, и что они будут направляться на IP-адрес конечного сервера:  
  
-   Порт 80: Веб-трафик HTTP  
  
-   Порт 443: Веб-трафик HTTPS  
  
> [!NOTE]
>  При настройке локального сервера Exchange на второй сервер, необходимо убедиться, порт 25 (SMTP) также открыт и перенаправляется на IP-адрес локального сервера Exchange Server.  
  
##  <a name="BKMK_MapPermittedComputers"></a>Сопоставление разрешенных компьютеров с учетными записями пользователей  
 В Windows SBS 2003 при подключении пользователя к удаленного веб-доступа, будут показаны все компьютеры в сети. Это могут быть компьютеры, которые пользователь не имеет разрешения на доступ к. В Windows Server Essentials пользователь должен быть явно назначен компьютеру, чтобы он отображался в удаленном веб-доступа. Каждая учетная запись пользователя, которая переносится из Windows SBS 2003 должна быть сопоставлена с одним или несколькими компьютерами.  
  
#### <a name="to-map-user-accounts-to-computers"></a>Сопоставление учетных записей и компьютеров  
  
1.  На конечном сервере откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей, щелкните правой кнопкой мыши учетную запись пользователя и нажмите кнопку **просмотреть свойства учетной записи**.  
  
4.  Нажмите кнопку **повсеместного доступа** , а затем щелкните **разрешить удаленный веб-доступ и доступ к приложениям веб-служб. **.  
  
5.  Нажмите кнопку **общих папок**, нажмите кнопку **компьютеры**, нажмите кнопку **ссылки главной страницы**, а затем нажмите кнопку **применить**.  
  
6.  Нажмите кнопку **доступ к компьютеру** и щелкните имя компьютера, к которому вы хотите разрешить доступ.  
  
7.  Повторите шаги 3, 4, 5 и 6 для каждой учетной записи пользователя.  
  
> [!NOTE]
>  Необходимо изменить конфигурацию клиентского компьютера. Она настраивается автоматически.  
  
> [!NOTE]
>  По окончании процесса миграции, если вы столкнулись с проблемой при создании первой новой учетной записи пользователя на конечном сервере, удалить учетную запись пользователя, который вы добавили и создайте его заново.
