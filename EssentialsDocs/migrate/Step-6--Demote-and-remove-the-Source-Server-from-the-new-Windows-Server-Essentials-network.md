2.  [Отключите принтеры, которые подключены непосредственно к исходному серверу](Step-6--Demote-and-remove-the-Source-Server-from-the-new-Windows-Server-Essentials-network.md#BKMK_PhysicallyDisconnect)  
  
3.  [Понижение уровня исходного сервера](Step-6--Demote-and-remove-the-Source-Server-from-the-new-Windows-Server-Essentials-network.md#BKMK_DemoteTheSourceServer)  
  
4.  [Удаление и переназначение исходного сервера](Step-6--Demote-and-remove-the-Source-Server-from-the-new-Windows-Server-Essentials-network.md#BKMK_RemoveTheSourceServer)  
  
##  <a name="BKMK_ADCS"></a>Удаление служб сертификатов Active Directory  
 Процедура немного отличается при установке на одном сервере нескольких служб ролей служб сертификатов Active Directory (AD CS). Чтобы удалить службу роли AD CS, сохранив других служб ролей AD CS, можно использовать следующую процедуру.  
  
 Для выполнения этой процедуры необходимо войти в систему с теми же разрешениями, как пользователь, установивший центр сертификации (ЦС). При удалении ЦС предприятия минимальным требованием для выполнения этой процедуры является членство в "Администраторы предприятия" или его эквивалент.  
  
#### <a name="to-remove-ad-cs"></a>Удаление AD CS  
  
1.  Войдите на исходный сервер как администратор домена.  
  
2.  Нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **диспетчера сервера**.  
  
3.  Нажмите кнопку **Продолжить** в **контроля учетных записей** диалоговое окно.  
  
4.  В **Сводка по ролям** щелкните **удаления ролей**.  
  
5.  В мастере удаления ролей щелкните **Далее**.  
  
6.  Очистить **служб сертификатов Active Directory** флажок и нажмите кнопку **Далее**.  
  
7.  На **Подтверждение параметров удаления** страницы, просмотрите сведения и нажмите кнопку **удалить**.  
  
    > [!NOTE]
    >  Если выполняется Internet Information Services (IIS), вам будет предложено остановить их перед продолжением. Нажмите кнопку **ОК**.  
  
    > [!NOTE]
    >  Во-первых, необходимо удалить **Регистрация через Интернет центра сертификации**, если он установлен.  
  
8.  После завершения работы мастера удаления ролей перезагрузите сервер, чтобы завершить процесс удаления.  
  
    > [!IMPORTANT]
    >  Перезапустите сервер, даже если вы не предложено сделать это.  
  
##  <a name="BKMK_PhysicallyDisconnect"></a>Отключите принтеры, которые подключены непосредственно к исходному серверу  
 Прежде чем понизить уровень исходного сервера, физически отключите все принтеры, которые подключены непосредственно к исходному серверу и совместно через исходный сервер. Убедитесь, что осталось никаких объектов Active Directory для принтеров, которые были подключены напрямую к исходному серверу. Принтеры можно напрямую подключены к серверу назначения и совместно с Windows Server Essentials.  
  
##  <a name="BKMK_DemoteTheSourceServer"></a>Понижение уровня исходного сервера  
 Прежде чем понизить уровень исходного сервера с роли контроллера домена AD DS до роли рядового сервера домена, убедитесь, что параметры групповой политики применяются ко всем клиентским компьютерам, как описано в следующей процедуре.  
  
> [!IMPORTANT]
>  Исходный сервер и сервер назначения должны быть подключены к сети, пока на клиентских компьютерах обновляются изменения групповой политики.  
  
#### <a name="to-force-a-group-policy-update-on-a-client-computer"></a>Принудительное обновление групповой политики на клиентском компьютере  
  
1.  Войдите на клиентский компьютер с правами администратора.  
  
2.  Откройте окно командной строки от имени администратора.  
  
3.  В командной строке введите **gpupdate/force**, и нажмите клавишу ВВОД.  
  
4.  Процесс может потребоваться выйти из системы и войти в систему для завершения. Нажмите кнопку **Да** для подтверждения.  
  
 Если вы проводите миграцию с Windows Server Essentials или его более ранних версий, понижения уровня сервера, см. раздел [удаления доменных служб Active Directory](https://technet.microsoft.com/library/hh472163.aspx). После добавления исходного сервера как член рабочей группы и ее необходимо отключить от сети, его необходимо удалить из AD DS на сервере назначения.  
  
 Если вы проводите миграцию с Windows Server Essentials, с помощью диспетчера серверов удалите роль доменных служб Active Directory, что позволит понизить роль контроллера домена на исходном сервере, выполнив следующие действия:  
  
#### <a name="to-remove-the-source-server-from-active-directory"></a>Для удаления исходного сервера из Active Directory  
  
1.  На конечном сервере откройте **Active Directory-пользователи и компьютеры**.  
  
2.  В **Active Directory-пользователи и компьютеры** области навигации разверните имя домена, а затем разверните **компьютеры**.  
  
3.  Если исходный сервер все еще присутствует в списке серверов, щелкните правой кнопкой мыши имя исходного сервера, щелкните **удалить**, а затем нажмите кнопку **Да**.  
  
4.  Убедитесь, что исходный сервер не списке, а затем закройте окно **Active Directory-пользователи и компьютеры**.  
  
##  <a name="BKMK_RemoveTheSourceServer"></a>Удаление и переназначение исходного сервера  
 Отключить исходный сервер и отключите его от сети. Рекомендуется не выполнять переформатирование исходного сервера по крайней мере одну неделю, чтобы убедиться, что все необходимые данные перенесены на конечный сервер. После подтверждения переноса всех данных можно переустановить этот сервер в сети как сервер-получатель для выполнения других задач, при необходимости.  
  
> [!NOTE]
>  После понижения уровня и удаление исходного сервера, перезапустите сервер назначения.  
  
 После понижения роли исходного сервера, не в работоспособном состоянии. Если требуется перепрофилировать исходный сервер, проще всего отформатировать его, установить операционную систему сервера и затем настроить ее для использования в качестве дополнительного сервера.
