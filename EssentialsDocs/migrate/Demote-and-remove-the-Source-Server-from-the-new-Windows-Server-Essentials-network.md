###  <a name="BKMK_UninstallExchangeServer2003"></a>Удаление сервера Exchange Server 2003  
  
> [!IMPORTANT]
>  При добавлении учетных записей пользователей после перемещения почтовых ящиков на целевой сервер, то перед удалением сервера Exchange Server 2003 с исходного сервера почтовые ящики добавляются на исходном сервере. Это сделано намеренно. Почтовые ящики необходимо переместить на конечный сервер для всех учетных записей пользователей, которые добавляются в течение этого времени. Повторите инструкции по почтовых ящиков Exchange Server, перемещение и параметров для миграции Windows Server Essentials, перед удалением сервера Exchange Server 2003.  
  
 Перед понижением роли необходимо удалить Exchange Server 2003 с исходного сервера. Это будут удалены все ссылки в доменных службах Active Directory (AD DS) для Exchange Server на исходном сервере. Необходимо иметь носитель с Windows Small Business Server 2003 для удаления Exchange Server 2003.  
  
##### <a name="to-uninstall-exchange-server-2003-from-the-source-server"></a>Удаление сервера Exchange Server 2003 с исходного сервера  
  
1.  Войдите на исходный сервер от имени администратора  
  
2.  Нажмите кнопку **запустить**, нажмите кнопку **панели управления**, а затем нажмите кнопку **Установка и удаление программ**.  
  
3.  В списке программ выберите **Windows Small Business Server 2003**, а затем нажмите кнопку **заменить или удалить**.  
  
4.  В мастере установки выберите **Далее** до **выбора компонентов** откроется страница.  
  
5.  На странице выбора компонентов разверните **Exchange Server**, а затем выберите **удалить**.  
  
    > [!NOTE]

    >  Чтобы убедиться в том, что не существует почтовые ящики или общие папки на сервере Exchange Server проверяет. Если какие-либо данные еще остались, сообщение об ошибке отображается при выборе **удалить**. Чтобы избежать этой проблемы, убедитесь, что выполнены все процедуры, описанные в разделе [параметров и данных на конечный сервер переместить SBS 2003](Move-Windows-SBS-2003-settings-and-data-to-the-Destination-Server-for-Windows-Server-Essentials-migration.md).  

    >  Чтобы убедиться в том, что не существует почтовые ящики или общие папки на сервере Exchange Server проверяет. Если какие-либо данные еще остались, сообщение об ошибке отображается при выборе **удалить**. Чтобы избежать этой проблемы, убедитесь, что выполнены все процедуры, описанные в разделе [параметров и данных на конечный сервер переместить SBS 2003](../migrate/Move-Windows-SBS-2003-settings-and-data-to-the-Destination-Server-for-Windows-Server-Essentials-migration.md).  

  
6.  Нажмите кнопку **Далее**.  
  
7.  При появлении соответствующего запроса вставьте Windows Small Business Server 2003 CD #3 и следуйте инструкциям на экране.  
  
###  <a name="BKMK_PhysicallyDisconnect"></a>Отключите принтеры, которые подключены непосредственно к исходному серверу  
 Прежде чем понизить уровень исходного сервера, физически отключите все принтеры, которые подключены непосредственно к исходному серверу и совместно через исходный сервер. Убедитесь, что осталось никаких объектов Active Directory для принтеров, которые были подключены напрямую к исходному серверу. Принтеры можно напрямую подключены к серверу назначения и совместно с Windows Server Essentials.  
  
###  <a name="BKMK_DemoteTheSourceServer"></a>Понижение уровня исходного сервера  
 Прежде чем понизить уровень исходного сервера с роли контроллера домена AD DS до роли рядового сервера домена, убедитесь, что параметры групповой политики применяются ко всем клиентским компьютерам, как описано в следующей процедуре.  
  
> [!IMPORTANT]
>  Исходный сервер и сервер назначения должны быть подключены к сети, пока на клиентских компьютерах обновляются изменения групповой политики.  
  
##### <a name="to-force-a-group-policy-update-on-a-client-computer"></a>Принудительное обновление групповой политики на клиентском компьютере  
  
1.  Войдите на клиентский компьютер с правами администратора.  
  
2.  Откройте окно командной строки от имени администратора.  
  
3.  В командной строке введите **gpupdate/force**, и нажмите клавишу ВВОД.  
  
4.  Процесс может потребоваться выйти из системы и войти в систему для завершения. Нажмите кнопку **Да** для подтверждения.  
  
##### <a name="to-demote-the-source-server"></a>Сведения о понижении уровня исходного сервера  
  
1.  На исходном сервере щелкните **запустить**, нажмите кнопку **запуска**, тип **dcpromo**и нажмите кнопку **ОК**.  
  
2.  Нажмите кнопку **Далее** дважды.  
  
    > [!NOTE]
    >  Не устанавливайте **этот сервер — последний контроллер домена в домене**.  
  
3.  Введите пароль для учетной записи администратора на сервере, а затем нажмите кнопку **Далее**.  
  
4.  В **Сводка** диалоговом будет сообщено, что AD DS будет удалена с компьютера, и что сервер станет членом домена. Нажмите кнопку **Далее**.  
  
5.  Нажмите кнопку **Готово **. Исходный сервер будет перезагружен.  
  
6.  После перезагрузки исходного сервера перед отключением от сети добавьте исходный сервер как член рабочей группы.  
  
 После добавления исходного сервера как член рабочей группы и ее необходимо отключить от сети, его необходимо удалить из AD DS на сервере назначения.  
  
##### <a name="to-remove-the-source-server-from-active-directory"></a>Для удаления исходного сервера из Active Directory  
  
1.  На конечном сервере откройте **Active Directory-пользователи и компьютеры**.  
  
2.  В **Active Directory-пользователи и компьютеры** области навигации разверните имя домена, а затем разверните **компьютеры**.  
  
3.  Щелкните правой кнопкой мыши имя исходного сервера, если оно все еще присутствует в списке серверов, нажмите кнопку **удалить**, а затем нажмите кнопку **Да**.  
  
4.  Убедитесь, что исходный сервер не списке, а затем закройте окно **Active Directory-пользователи и компьютеры**.  
  
###  <a name="BKMK_MoveTheDHCPRole"></a>Перемещение роли DHCP-сервера с исходного сервера на маршрутизатор  
  
> [!NOTE]

>  Если вы уже выполнили эту задачу до начала процесса миграции, продолжите с раздела [удаление и переназначение исходного сервера](Demote-and-remove-the-Source-Server-from-the-new-Windows-Server-Essentials-network.md#BKMK_RemoveTheSourceServer).  

>  Если вы уже выполнили эту задачу до начала процесса миграции, продолжите с раздела [удаление и переназначение исходного сервера](../migrate/Demote-and-remove-the-Source-Server-from-the-new-Windows-Server-Essentials-network.md#BKMK_RemoveTheSourceServer).  

  
 Если исходный сервер выполняется роль сервера DHCP, выполните следующие действия, чтобы перемещение роли DHCP на маршрутизатор.  
  
##### <a name="to-move-the-dhcp-role-from-the-source-server-to-the-router"></a>Перемещение роли DHCP с исходного сервера на маршрутизатор  
  
1.  Отключите службу DHCP на исходном сервере следующим образом:  
  
    1.  На исходном сервере нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **службы**.  
  
    2.  В списке текущих выполняемых служб щелкните правой кнопкой мыши **Windows Server**, а затем нажмите кнопку **свойства**.  
  
    3.  Для **типа запуска**выберите **отключено**.  
  
    4.  Остановите службу.  
  
2.  Включение роли DHCP на маршрутизаторе  
  
    1.  Следуйте инструкциям в документации на маршрутизатор, чтобы включить роль DHCP на маршрутизаторе.  
  
    2.  Чтобы убедиться, что IP-адреса, выданные сервером источника не изменяются, следуйте инструкциям в документации на маршрутизатор, чтобы настроить диапазон DHCP на маршрутизаторе должен быть идентичен диапазону DHCP на исходном сервере.  
  
    > [!IMPORTANT]
    >  Если вы не настроили статический IP или резервирования DHCP на маршрутизаторе для целевого сервера, и диапазон DHCP не совпадает с исходным сервером, что маршрутизатор может выдать новый IP-адрес для конечного сервера. В этом случае сбросьте правила маршрутизатор для пересылки на новый IP-адрес конечного сервера перенаправления портов.  
  
###  <a name="BKMK_RemoveTheSourceServer"></a>Удаление и переназначение исходного сервера  
 Отключить исходный сервер и отключите его от сети. Рекомендуется не выполнять переформатирование исходного сервера по крайней мере одну неделю, чтобы убедиться, что все необходимые данные перенесены на конечный сервер. После подтверждения переноса всех данных можно переустановить этот сервер в сети как сервер-получатель для выполнения других задач, при необходимости.  
  
> [!NOTE]
>  После понижения уровня и удаление исходного сервера, перезапустите сервер назначения.  
  
 После понижения роли исходного сервера, не в работоспособном состоянии. Если требуется перепрофилировать исходный сервер, проще всего отформатировать его, установить операционную систему сервера и затем настроить ее для использования в качестве дополнительного сервера.
