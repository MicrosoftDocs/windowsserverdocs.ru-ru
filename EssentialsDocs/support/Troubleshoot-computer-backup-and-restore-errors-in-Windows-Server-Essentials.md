# <a name="troubleshoot-computer-backup-and-restore-errors-in-windows-server-essentials"></a>Устранение неполадок архивации и восстановления данных в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

Используйте эти процедуры для устранения неполадок резервного копирования компьютера в Windows Server Essentials, включая проблемы с настройкой архивации, неполной или неудачной архивации, оповещений о работоспособности архивации и проблемы, связанные с восстановлением всей системы, файлов и папок.  
  
> [!NOTE]
>  Самые последние сведения об устранении неполадок от сообщества пользователей Windows Server Essentials, посетите [форуме Windows Server Essentials](https://social.technet.microsoft.com/Forums//winserveressentials/threads).  
  
##  <a name="BKMK_TroubleshootBackupConfigurationIssues"></a>Устранение неполадок архивации данных подключенных компьютеров  
 Используйте эти процедуры для устранения неполадок с архивацией у компьютеров, которые архивируются на сервере Windows Server Essentials.  
  
### <a name="errors"></a>Ошибки  
  
-   Конфигурация резервного копирования не была успешно завершена  
  
-   Ошибка сбора информации для компьютера  
  
-   Ошибка удаления компьютера из резервной копии  
  
### <a name="resolutions"></a>Решения  
  
##### <a name="to-troubleshoot-errors-that-occur-while-you-configure-backups-for-a-connected-computer"></a>Для устранения ошибок, возникающих во время настройки архивации для подключенных компьютеров  
  
1.  Убедитесь, что компьютер подключен к сети через сетевое устройство.  
  
2.  Убедитесь в том, что сетевое устройство, компьютера, подключенного к также подключен, соединено с сетью и работает правильно.  
  
3.  Убедитесь, что запущена служба архивации клиентских Windows Server и поставщик службы архивации данных Windows Server клиента на сервере.  
  
    ###### <a name="to-start-computer-backup-services-on-the-server"></a>Запуск служб архивации данных компьютера на сервере  
  
    1.  На сервере нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **службы**.  
  
    2.  Прокрутите вниз и нажмите кнопку **поставщик службы архивации данных Windows Server клиента**. Если состояние службы не **запущено**, щелкните правой кнопкой мыши службу и нажмите кнопку **запустить**.  
  
    3.  Нажмите кнопку **Windows Server служба архивации клиентских компьютеров**. Если состояние службы не **запущено**, щелкните правой кнопкой мыши службу и нажмите кнопку **запустить**.  
  
    4.  Закрыть **службы**.  
  
4.  Убедитесь, что поставщик службы архивации данных Windows Server клиента работает на клиентском компьютере.  
  
    ###### <a name="to-start-the-computer-backup-service-on-the-client-computer"></a>Запуск службы архивации данных компьютера на клиентском компьютере  
  
    1.  На клиентском компьютере нажмите кнопку **запустить**, тип **службы** в **поиск программ и файлов** текстовое поле и нажмите клавишу ВВОД.  
  
    2.  Прокрутите вниз и нажмите кнопку **поставщик службы архивации данных Windows Server клиента**. Если состояние службы не **запущено**, щелкните правой кнопкой мыши службу и нажмите кнопку **запустить**.  
  
    3.  Закрыть **службы**.  
  
5.  Проверьте оповещения, чтобы определить наличие ошибок в базе данных архивации. Если есть ошибки, следуйте инструкциям в диалоговом окне оповещения для восстановления базы данных архивации.  
  
6.  Удалите программное обеспечение Windows Server Essentials Connector с компьютера, а затем переустановите его. Дополнительные сведения см. в разделах [удалить программное обеспечение Connector](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_13) и [установите программное обеспечение Connector](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_11).  
  
##  <a name="BKMK_TroubleshootaBackupThatDidNotCompleteProperly"></a>Устранение неполадок резервного копирования, который не был выполнен должным образом  
 Если резервной копии присвоен статус, резервная копия выполнена безуспешно и данные не доступен для восстановления. Тем не менее если резервной копии присвоен статус, не все элементы резервной копии были архивированы, но некоторые данные могут быть доступны для восстановления.  
  
### <a name="errors"></a>Ошибки  
  
-   Резервная копия не была завершена  
  
-   Неудачная Архивация данных  
  
### <a name="resolutions"></a>Решения  
  
##### <a name="to-identify-volumes-that-were-not-backed-up-successfully"></a>Чтобы определить тома, которые не были сохранены успешно  
  
1.  Откройте панель мониторинга Windows Server Essentials, а затем нажмите кнопку **компьютеры и резервное копирование**.  
  
2.  Щелкните имя компьютера, где резервной копии не успешно завершена и нажмите кнопку **просмотреть свойства компьютера** в **задачи** области.  
  
3.  Щелкните резервную копию, которая не была завершена и нажмите кнопку **Просмотр сведений о**.  
  
4.  В **сведения об архивации** диалоговом Зеленый флажок отображается состояние каждого тома, для успешного создания резервной копии.  
  
##### <a name="to-troubleshoot-an-unsuccessful-backup-of-a-volume"></a>Для устранения неполадок с неудачной архивацией тома  
  
1.  Убедитесь в том, что жесткий диск подключен к компьютеру, включен и работает правильно.  
  
2.  Запустите **chkdsk /f /r** для устранения ошибок на жестком диске (**/f**) и восстановления читаемой информации из всех поврежденных секторов (**/r**). Дополнительные сведения о выполнении **chkdsk**, в разделе [CHKDSK](https://go.microsoft.com/fwlink/?LinkId=206562).  
  
3.  Убедитесь, что компьютер не был завершение работы или отключен от сети в процессе выполнения архивации.  
  
4.  Убедитесь, что имеется достаточно свободного места на каждом томе для выполнения архивации. Архивация требует дополнительное место на диске клиентского компьютера для создания моментального снимка VSS. На любом томе, не зарезервированном системой должно быть свободного по крайней мере 10% свободного места. На системном томе Если размер тома составляет менее 500 МБ, VSS требуется 32 МБ свободного места для создания моментального снимка; Если том является 500 МБ или более поздней версии, VSS требуется 320 МБ свободного пространства.  
  
     Если в томе недостаточно свободного места, воспользуйтесь одним из предлагаемых решений:  
  
    -   Расширение тома. Вы можете расширить любой базовый или динамический том за исключением системного тома.  
  
        ###### <a name="to-extend-a-volume"></a>Расширение тома  
  
        1.  В панели управления выберите **система и безопасность**.  
  
        2.  В разделе **Администрирование**, нажмите кнопку **Создание и форматирование разделов жесткого диска**.  
  
        3.  Щелкните правой кнопкой мыши том, который требуется расширить. Если **расширить том** — активен, выберите соответствующий параметр. Если параметр не включен, невозможно расширить том.  
  
        4.  Следуйте указаниям мастера расширения тома для расширения тома.  
  
    -   Удалите содержимое тома, чтобы освободить место на диске.  
  
        > [!NOTE]
        >  Если вам необходимо освободить место на системном томе, можно перенести образ для восстановления системы на другой том. Инструкции см. в разделе [развертывание образа восстановления системы](https://technet.microsoft.com/library/dd744280\(v=ws.10\).aspx).  
  
    -   Исключите том из клиентской резервной копии. Это необходимо сделать, только в том случае, если это не важно, вам необязательно сохранять резервную копию данных на томе.  
  
        > [!WARNING]
        >  Если вы исключите системный том из резервной копии клиента, в системе клиента не будет создаваться резервные копии, и вы не сможете выполнять полное восстановление системы на компьютере.  
  
5.  Проверьте наличие других оповещений на сервере, который может означать, что недостаточно места на диске для успешной архивации данных на сервере. Следуйте инструкциям в диалоговом окне оповещения для устранения проблемы.  
  
6.  Запустите **vssadmin** в командной строке для устранения неполадок службы теневого копирования томов (VSS) неполадок. Сведения о **vssadmin**, в разделе [VSSADMIN](https://go.microsoft.com/fwlink/?LinkID=94332).  
  
##  <a name="BKMK_TroubleshootBackupHealthAlertIssues"></a>Устранение неполадок оповещений о работоспособности архивации  
  
### <a name="errors"></a>Ошибки  
  
-   Поставщик службы архивации данных Windows Server решения перестал работать  
  
-   Служба поставщика архивации Windows Server решения клиентских компьютер перестал работать  
  
### <a name="resolutions"></a>Решения  
  
##### <a name="to-troubleshoot-a-backup-health-alert"></a>Для устранения неполадок оповещения работоспособности архивации  
  
1.  Если в оповещении говорится, что базы данных архивации есть проблемы, следуйте инструкциям в диалоговом окне оповещения для устранения проблемы.  
  
2.  Если в оповещении говорится, что служба резервного копирования не запущена, попробуйте запустить службу на сервере или клиентском компьютере, где вы получили сообщение об ошибке.  
  
    ###### <a name="to-start-backup-services-on-the-server"></a>Запуск служб архивации на сервере  
  
    1.  На сервере нажмите кнопку **запустить**и нажмите кнопку **Администрирование**, а затем нажмите кнопку **службы**.  
  
        > [!NOTE]
        >  При удаленном администрировании сервера необходимо использовать подключение к удаленному рабочему столу для доступа к рабочему столу сервера. Сведения об использовании подключения к удаленному рабочему столу см. в разделе [подключение к другому компьютеру с помощью удаленного рабочего стола](https://windows.microsoft.com/windows-vista/Connect-to-another-computer-using-Remote-Desktop-Connection).  
  
    2.  Прокрутите вниз и нажмите кнопку **поставщик службы архивации данных Windows Server клиента**. Если состояние службы не **запущено**, щелкните правой кнопкой мыши службу и нажмите кнопку **запустить**.  
  
    3.  Нажмите кнопку **Windows Server служба архивации клиентских компьютеров**. Если состояние службы не **запущено**, щелкните правой кнопкой мыши службу и нажмите кнопку **запустить**.  
  
    4.  Закрыть **службы**.  
  
    ###### <a name="to-start-backup-services-on-a-client-computer"></a>Запуск служб архивации на клиентском компьютере  
  
    1.  На клиентском компьютере нажмите кнопку **запустить**, тип **службы** в **поиск программ и файлов** текстовое поле и нажмите клавишу ВВОД.  
  
    2.  Щелкните правой кнопкой мыши **поставщик службы архивации данных Windows Server клиента**, а затем нажмите кнопку **запустить**.  
  
    3.  Закрыть **службы**.  
  
3.  Проверьте журналы событий на клиентском компьютере или сервере вопросы, относящиеся к резервное копирование службы или драйвера.  
  
4.  Перезагрузите сервер или клиентский компьютер, где вы получили сообщение об ошибке.  
  
5.  Проверьте наличие оповещений о других неполадках, которые могут повлиять на архивацию данных клиента.  
  
##  <a name="BKMK_FileAndFolder"></a>Устранение неполадок с восстановлением файла или папки  
  
### <a name="errors"></a>Ошибки  
  
-   Не удалось успешно завершить восстановление файла или папки.  
  
### <a name="resolutions"></a>Решения  
  
##### <a name="to-troubleshoot-an-unsuccessful-file-or-folder-restore"></a>Для устранения неполадок после неудачного восстановления файла или папки  
  
1.  Убедитесь, что компьютер подключен к сети через сетевое устройство.  
  
2.  Убедитесь в том, что сетевое устройство, компьютера, подключенного к также подключен, соединено с сетью и работает правильно.  
  
3.  Проверьте оповещения, чтобы определить наличие ошибок в базе данных архивации. Если есть ошибки, следуйте инструкциям в диалоговом окне оповещения для восстановления базы данных архивации.  
  
4.  Попробуйте восстановить файлы или папки из другой резервной копии.  
  
5.  Убедитесь, что **Windows Server решение Драйвер восстановления данных компьютера** устанавливается и работает надлежащим образом.  
  
    ###### <a name="to-check-the-status-of-the-windows-server-solution-computer-restore-driver"></a>Чтобы проверить состояние Windows Server решение Драйвер восстановления данных компьютера  
  
    1.  Нажмите кнопку **запустить**, тип **диспетчер устройств** в **поиск программ и файлов** текстовое поле и нажмите клавишу ВВОД.  
  
    2.  В диспетчере устройств щелкните **системные устройства**, перейдите к пункту **Windows Server Solutions драйвер восстановления данных компьютера**.  
  
    3.  Если драйвер не отображается:  
  
        1.  Откройте командную строку с правами администратора и выполните следующую команду:  
  
             **%ProgramFiles%\Windows Server\Bin\BackupDriverInstaller.exe?  я**  
  
        2.  Обновите диспетчер устройств. Драйвер должен отобразиться в списке.  
  
    4.  Если отображается значок компьютерного монитора, драйвер установлен и работает правильно. Закройте диспетчер устройств.  
  
    5.  Если не отображается значок в виде компьютерного монитора  
  
        1.  Щелкните правой кнопкой мыши **Windows Server Solutions драйвер восстановления данных компьютера**, а затем нажмите кнопку **свойства**.  
  
        2.  Нажмите кнопку **драйвер**, а затем щелкните **обновить драйвер**.  
  
        3.  Нажмите кнопку **автоматический поиск обновленных драйверов**, а затем следуйте инструкциям, чтобы обновить драйвер.  
  
    6.  Закройте диспетчер устройств.  
  
6.  Удалите программное обеспечение Windows Server Essentials Connector с компьютера, а затем переустановите его. Дополнительные сведения см. в разделах [удалить программное обеспечение Connector](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_13) и [установите программное обеспечение Connector](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_11).  
  
##  <a name="BKMK_Troubleshootfullsystemrestoreissues"></a>Устранение неполадок с полным восстановлением системы  
  
### <a name="errors"></a>Ошибки  
  
-   Не удается войти в систему клиентского компьютера после полного восстановления системы.  
  
### <a name="resolutions"></a>Решения  
 Если изменить имя компьютера, а затем возникла необходимость в восстановлении резервной копии, которая была создана до изменения имени компьютера, после восстановления при попытке войти в систему под учетной записью домена, вы получите этой ошибки: «базы данных безопасности на сервере нет учетной записи компьютера для доверительные отношения с этой рабочей станции.» Чтобы снова получить доступ к компьютеру, удалите программное обеспечение Connector, удалите компьютер из домена Windows и снова подключиться к серверу.  
  
##### <a name="to-regain-network-access-to-a-restored-computer-after-a-computer-name-change"></a>Чтобы получить доступ к сети для восстановленного компьютера после изменения имени компьютера  
  
1.  Войдите на компьютер с правами локального администратора.  
  
2.  Удаление программного обеспечения Connector. Дополнительные сведения см. в разделе [удалить программное обеспечение Connector](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_13).  
  
3.  Удаление компьютера из домена. Дополнительные сведения см. в разделе [удаление компьютера из домена Windows](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_8).  
  
4.  Повторное подключение компьютера к серверу. Дополнительные сведения см. в разделе [подключение компьютеров к серверу](../use/Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).
