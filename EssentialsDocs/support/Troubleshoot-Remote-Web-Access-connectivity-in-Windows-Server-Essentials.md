# <a name="troubleshoot-remote-web-access-connectivity-in-windows-server-essentials"></a>Устранение неполадок подключения удаленного веб-доступа в Windows Server Essentials
 
>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials
  
 Как правило Windows Server Essentials может автоматически настроить маршрутизатор широкополосной связи, если этот маршрутизатор является сертифицированным устройством UPnP и включения параметра UPnP на маршрутизаторе.  
  
## <a name="possible-issues"></a>Возможные проблемы  
 Возможны следующие проблемы с подключением удаленного веб-доступа:  
  
-   Маршрутизатор не включен или не подключен к сети.  
  
-   Параметр UPnP на маршрутизаторе выключен параметр.  
  
-   Маршрутизатор не может полностью поддерживает стандарт UPnP. Корпорация Майкрософт придерживается список маршрутизаторов, работающих с операционными системами Windows. Чтобы просмотреть список маршрутизаторов (включая беспроводные маршрутизаторы), совместимых с Windows Server Essentials, посетите [центре совместимости Windows](https://www.microsoft.com/windows/compatibility/CompatCenter/Home).  
  
## <a name="possible-fixes"></a>Возможные исправления  
 Эти проблемы можно устранить следующие действия:  
  
-   Убедитесь, что маршрутизатор включен и работает надлежащим образом.  
  
-   Убедитесь, что сервер подключен к маршрутизатору напрямую или что он подключен к коммутатору, который подключен к маршрутизатору.  
  
-   Убедитесь, что устройство широкополосной связи, который подключается к поставщику услуг Интернета (ISP) включено и работает правильно, а маршрутизатор подключен к этому устройству.  
  
-   Включите параметр UPnP на маршрутизаторе. Подключиться к веб-странице конфигурации маршрутизатора включить параметр UPnP. Сведения о входе на маршрутизатор и включении параметра UPnP см. в документации по маршрутизатору. После включения параметра UPnP запустите включить на Web мастер удаленного доступа еще раз, чтобы настроить маршрутизатор.  
  
-   Если маршрутизатор не полностью поддерживает стандарт UPnP, его нельзя настроить автоматически. Кроме того, необходимо вручную настроить маршрутизатор или приобрести маршрутизатор, который поддерживает стандарт UPnP.  
  
     Чтобы настроить маршрутизатор вручную, выполните следующие задачи:  
  
    -   Создайте резервирование IP-адреса для сервера Windows Server Essentials.  
  
         Прежде чем вручную настроить маршрутизатор для пересылки необходимых портов в Windows Server Essentials, необходимо задать резервирование протокол динамической настройки узла (DHCP) для сервера, на котором работает Windows Server Essentials на маршрутизаторе. Это гарантирует, что IP-адрес, на который будут пересылаться порты, не изменится.  
  
         Сведения о том, как вручную настроить для сервера на маршрутизаторе резервирования DHCP см. в документации по маршрутизатору s.  
  
    -   Настройте перенаправление портов на маршрутизаторе для следующих портов.  
  
        |Служба или протокол|Порт|  
        |-------------------------|----------|  
        |HTTP|TCP 80|  
        |HTTPS|TCP 443|  
  
     Сведения о том, как вручную настроить перенаправление портов на маршрутизаторе см. в документации изготовителя s.  
  
     Странице конфигурирования маршрутизатора обычно содержит таблицу следующего вида.  
  
    > [!NOTE]
    >  В этой таблице IP-адрес компьютера, на котором работает Windows Server Essentials — 192.168.0.100. Необходимо определить IP-адрес вашего компьютера и заменить этот IP-адрес для IP-адреса, указанные в таблице.  
  
    |IP-адрес|Протокол (TCP/UDP)|Расписание|Входящий фильтр|  
    |----------------|---------------------------|--------------|--------------------|  
    |192.168.0.100|TCP 80|Всегда|Разрешить все|  
    |192.168.0.100|TCP 443|Всегда|Разрешить все|  
  
     После настройки маршрутизатора вручную запустите включить на Web мастер удаленного доступа, с выбранным **пропустить настройку маршрутизатора** вариант на **Приступая к работе** страницы.  
  
-   Приобретите новый маршрутизатор, если маршрутизатор не полностью поддерживает стандарт UPnP.  
  
> [!TIP]
>  Убедитесь, что на маршрутизаторе установлена последняя версия встроенного по BIOS. Часто можно обновить встроенное по BIOS для маршрутизатора из веб-страницы настройки маршрутизатора. Дополнительные сведения см. в документации по маршрутизатору. После обновления маршрутизатора запустите мастера настройки повсеместного доступа.
