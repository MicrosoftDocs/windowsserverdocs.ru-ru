##  <a name="BMRK_Package"></a>Проблема 1  
 **Проблема**  
  
 Пакет установки не удалось получить. Попробуйте снова установить Windows Server Essentials Connector. Если проблема сохраняется, обратитесь к администратору ошибка сети при подключении компьютера к серверу  
  
 **Описание**  
  
 Эта проблема может возникать при подключении компьютера к серверу под управлением Windows Server Essentials, хотя другие обновления Windows или установки приложений находятся в состоянии ожидания и установка Connector отменяется.  
  
 **Решение**  
  
 Завершите все обновления или установки приложений. Если появится соответствующий запрос, перезагрузите компьютер.  
  
##  <a name="BKMK_ConnectorIssue2"></a>Проблема 2  
 **Проблема**  
  
 Не удается присоединить компьютер к Windows Server Essentials  
  
 **Описание**  
  
 Компьютеры, которые содержат символы, отличные от ASCII имя компьютера не может быть присоединен к Windows Server Essentials. Если имя компьютера содержит символы, отличные от ASCII, выводится сообщение об ошибке «Произошла непредвиденная ошибка».  
  
 **Решение**  
  
 Присвойте клиентскому компьютеру имя, содержащее только символы ASCII и повторите попытку добавления компьютера к Windows Server Essentials.  
  
##  <a name="BKMK_ConnectorIssue2a"></a>Проблема 3  
 **Проблема**  
  
 Ошибка разъем установки программного обеспечения отмененные при подключении компьютера к серверу  
  
 **Описание**  
  
 Для подключения компьютера к серверу, СИСТЕМНАЯ учетная запись должна иметь разрешения на полный доступ на серверным папкам, отображаемым на панели мониторинга Windows Server Essentials. Если не предоставлены необходимые разрешения, возникает ошибка «Установка программного обеспечения Connector отменена» сообщения.  
  
 **Решение**  
  
 Предоставьте учетной записи системы **полный доступ к** разрешения для каждой папки на сервере.  
  
#### <a name="to-grant-the-system-account-full-control-permissions-on-a-server-folder"></a>Предоставление системной учетной записи разрешения на полный доступ к папкам сервера  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  Нажмите кнопку **хранилища**, а затем нажмите кнопку **папок сервера**.  
  
3.  Щелкните правой кнопкой мыши папку сервера, а затем нажмите кнопку **откройте папку**. (Если вы не видите **откройте папку** параметр, не нужно задать разрешения на доступ к папке.)  
  
4.  В сетевом пути в верхней части проводника Windows щелкните в папке сервера, чтобы отобразить общие папки на сервере. Например, если путь является **сети > Server01 > архивы журналов файлов**, нажмите кнопку **Server01**.  
  
5.  Щелкните правой кнопкой мыши папку сервера, а затем нажмите кнопку **свойства**.  
  
6.  Нажмите кнопку **безопасности** вкладку.  
  
7.  Если **полный доступ к** — запрещено системной учетной записи, нажмите кнопку **изменить**, а затем нажмите кнопку **системы**. В разделе **разрешения для системы**выберите **Разрешить** флажок рядом с **полный доступ к**.  
  
8.  Нажмите кнопку **ОК** два раза, чтобы обновить разрешения и закрыть **свойства**.  
  
##  <a name="BKMK_ConnectorIssueNetFramework"></a>Проблема 4  
 **Проблема**  
  
 Получить для запуска этого приложения, необходимо установить одну из следующих версий платформы .NET Framework: V4.5.50709» ошибка при подключении компьютера к серверу  
  
 **Описание**  
  
 При подключении компьютера к серверу под управлением Windows Server Essentials или Windows Server Essentials, мастер пытается установить .NET Framework версии 4.5.50709 на компьютере. Тем не менее, если имеется более ранняя версия платформы .NET Framework 4.5, обновленный выпуск установить нельзя и попытка соединения завершается с это сообщение об ошибке: для запуска этого приложения необходимо установить одну из следующих версий платформы .NET Framework: V4.5.50709. Инструкции по получению соответствующей версии платформы .NET Framework, обратитесь к издателю.  
  
 **Решение**  
  
 Удалите .NET Framework 4.5 с компьютера, а затем подключите компьютер к серверу.  
  
#### <a name="to-uninstall-net-framework-45"></a>Удаление .NET Framework 4.5  
  
1.  На **запустить** страницы клиентского компьютера, откройте **панели управления**.  
  
2.  В панели управления в разделе **программы**, нажмите кнопку **удаление программы**.  
  
3.  Щелкните правой кнопкой мыши **Microsoft .NET Framework 4.5**, а затем нажмите кнопку **удалить**.  
  
4.  После успешного удаления .NET Framework 4.5 подключите компьютер к серверу. Нужная версия .NET Framework 4.5 устанавливается вместе с по Connector.  
  
##  <a name="BKMK_Time"></a>Проблема 5  
 **Проблема**  
  
 Получить сервер недоступен. Чтобы устранить эту проблему, обратитесь к сетевому администратору. Ошибка при подключении компьютера к серверу  
  
 **Описание**  
  
 Это может произойти, если дата и время на подключенном компьютере не синхронизированы с датой и временем на сервере.  Windows Server Essentials и Windows Server Essentials используют службу синхронизации времени для синхронизации даты и времени на компьютерах под управлением Windows Server Essentials или сети Windows Server Essentials. Синхронизация времени очень важна, так как протокол проверки подлинности по умолчанию использует время сервера как часть процесса проверки подлинности. Например если часы на клиентском компьютере не синхронизированы с правильными датой и временем, Windows Server Essentials или Windows Server Essentials проверки подлинности может ошибочно интерпретировать запрос на вход как попытка вторжения и запрещать доступ пользователей.  
  
 Это может произойти, если сервер s объем свободной памяти меньше, чем на 5 процентов.  
  
 Это может произойти, если имеется VPN-подключения в Windows Server Essentials и настроить соединитель программного обеспечения во внешней среде с помощью адреса домена.  
  
 **Решение**  
  
1.  Синхронизируйте дату и время на клиентском компьютере с датой на сервере. Затем подключите компьютер к серверу.  
  
2.  Закройте некоторые приложения на стороне сервера, а затем подключите компьютер к серверу.  
  
3.  Закройте VPN-подключения и затем подключите компьютер к серверу.  
  
#### <a name="to-change-the-date-and-time-on-the-client-computer"></a>Чтобы изменить дату и время на клиентском компьютере  
  
1.  На начальной странице клиентского компьютера откройте **панели управления**.  
  
2.  В панели управления выберите **часы, язык и регион**, а затем нажмите кнопку **даты и времени**.  
  
3.  Нажмите кнопку **изменить дату и время**, указать дату и время на правильную дату и время и нажмите кнопку **ОК**.  
  
4.  Нажмите кнопку **ОК**, а затем закройте панель управления.  
  
5.  Повторите попытку подключения клиентского компьютера к серверу. Инструкции см. в разделе подключение компьютеров к серверу.  
  
 Клиентский компьютер по-прежнему не удается подключиться к серверу, убедитесь, что установлены правильная дата и время на сервере. Если дата и время неверны, измените их.  
  
#### <a name="to-change-the-date-and-time-on-the-server"></a>Для изменения даты и времени на сервере  
  
1.  Войдите на сервер, используя пароль, можно настроить во время Windows Server Essentials или Windows Server Essentials установки и настройки.  
  
    > [!NOTE]
    >  При удаленном администрировании сервера вам необходимо войти в систему на сервер с помощью удаленного рабочего стола.  
  
2.  На **запустить** странице откройте **панели управления**.  
  
3.  В панели управления выберите **часы, язык и регион**, а затем нажмите кнопку **даты и времени**.  
  
4.  Нажмите кнопку **изменить дату и время**, указать дату и время на правильную дату и время и нажмите кнопку **ОК**.  
  
5.  Нажмите кнопку **ОК**, а затем закройте панель управления.  
  
6.  На клиентском компьютере повторите попытку подключения клиентского компьютера к серверу. Инструкции см. в разделе подключение компьютеров к серверу.  
  
##  <a name="BKMK_ServiceStopped"></a>Проблема 6  
 **Проблема**  
  
 Получить произошла непредвиденная ошибка. Чтобы устранить эту проблему, обратитесь к сетевому администратору. Ошибка при подключении компьютера к серверу  
  
 **Описание**  
  
 Если сообщение об ошибке, возможно, не запущена веб-служба сертификатов WSS.  
  
 **Решение**  
  
 Запустите веб-служба сертификатов WSS.  
  
#### <a name="to-start-the-wss-certificate-web-service"></a>Чтобы запустить веб-службу сертификатов WSS  
  
1.  На сервере **запустить** странице откройте **Администрирование**.  
  
     В **Администрирование**, щелкните правой кнопкой мыши **диспетчера Internet Information Services (IIS)**, а затем нажмите кнопку **откройте**.  
  
2.  В области навигации щелкните **веб-служба сертификатов WSS**.  
  
3.  В **действия** панели, щелкните **запустить**.  
  
##  <a name="BKMK_ConnectorIssueReconnect"></a>Проблема 7  
 **Проблема**  
  
 При попытке подключения компьютера к серверу после неудачного подключения выводится предупреждение: компьютер с таким именем уже подключен к серверу  
  
 **Описание**  
  
 Если предыдущая попытка подключения компьютера к серверу была отменена или прервана, при попытке повторного подключения может появиться следующее предупреждение: компьютер с таким именем уже подключен к серверу. Это происходит потому, что сертификат был выдан при попытке подключения к серверу в первый раз.  
  
 **Решение** Если вы уверены, что никакой другой компьютер с тем же именем уже подключен к серверу, щелкните **Далее**, а затем следуйте инструкциям для завершения **подключения компьютера к серверу** мастера.  
  
##  <a name="BKMK_JoinWin7"></a>Проблема 8  
 **Проблема**  
  
 При попытке подключения клиентского компьютера, на котором работает Windows 7 Домашняя к серверу открывается веб-страница для запуска по Connector, но клиентский компьютер не может подключиться к серверу  
  
 **Описание**  
  
 Если на маршрутизаторе в сети включена многоадресная рассылка, обмен данными между сервером и клиентский компьютер под управлением Windows 7 Домашняя базовая или Windows 7 Домашняя расширенная работают неправильно.  
  
 **Решение**  
  
 Отключите многоадресную рассылку на маршрутизаторе. На некоторых маршрутизаторах возможно, потребуется отключить протокол маршрутизации RIP - 2M. Дополнительные сведения см. в документации, предоставленной производителем маршрутизатора.  
  
##  <a name="BKMK_ConnectorIssueAutologon"></a>Проблема 9  
 **Проблема**  
  
 После подключения компьютера к серверу перестал работать автоматический вход  
  
 **Описание**  
  
 Если при подключении компьютера к Windows Server Essentials для учетной записи пользователя установлен автоматический вход, эта настройка перезаписывается при установке по Connector на компьютере.  
  
 **Решение** для устранения этой проблемы при подключении компьютера к серверу запишите пароль, который используется для учетной записи пользователя. После установки по Connector настройте автоматический вход для этой учетной записи.  
  
> [!NOTE]
>  Учетная запись домена Windows Server Essentials требуется пароль, отвечающий требованиям политики паролей по умолчанию.  
  
##  <a name="BKMK_ConnectorIssueOldLogs"></a>Проблема 10  
 **Проблема**  
  
 Удаление предварительной версии программного обеспечения connector не приводит к удалению существующих журналов.  
  
 **Описание**  
  
 После обновления с предварительной (бета-версии или версии-Кандидата) версии системы Windows Server Essentials до окончательной версии необходимо удалить программное обеспечение connector с каждого компьютера, который был подключен к серверу и затем подключите компьютер еще раз, чтобы установить окончательную версию по Connector.  
  
 Тем не менее при удалении по Connector с сетевого компьютера существующие файлы журнала в папке %ProgramData%\Microsoft\Windows server\logs\ на этом компьютере не удаляются. Если вы не удалите папку журналов, файлы журнала могут быть повреждены при подключении компьютера к выпущенной версии Windows Server Essentials.  
  
 **Решение** во избежание повреждения файлов журнала вручную следует удалить папку журналов до повторного подключения клиентского компьютера к обновленному серверу.  
  
#### <a name="to-reconnect-a-computer-after-a-server-update-without-corrupting-the-log-files"></a>Повторное подключение компьютера после обновления сервера без повреждения журнала файлов  
  
1.  Удалите по Connector с клиентского компьютера.  
  
2.  Удалите папку журналов из папки %ProgramData%\Microsoft\Windows сервере.  
  
3.  Повторное подключение компьютера к серверу. Что будет установлена окончательная версия по Connector и создана новая папка журналов и файлы журналов.  
  
##  <a name="BKMK_UpgradeClientOS"></a>Проблема 11  
 **Проблема**  
  
 Необходимо обновить операционную систему на клиентском компьютере  
  
 **Описание**  
  
 Во время установки программного обеспечения connector выполняется ряд проверок клиентской операционной системы, чтобы убедиться, что клиент соответствует требованиям connector. Если выполняется обновление операционной системы клиента, после установки программного обеспечения connector, некоторые необходимые компоненты могут отсутствовать и по connector на клиенте может завершиться ошибкой.  
  
 **Решение**  
  
 Перед обновлением клиентской операционной системы до другой версии (например, обновление Windows XP до Windows Vista или Windows Vista до Windows 7), следует удалить программное обеспечение connector. Используйте **Установка и удаление программ** панели управления. После завершения обновления ОС клиента можно переустановить connector клиента, открыв http://< *сервера*> / connect в веб-браузере, где <*сервера*> — имя сервера Windows Server Essentials.  
  
 Если клиент уже был обновлен путем установки программного обеспечения connector, используйте **Установка и удаление программ** или **программы и компоненты** удаление программного обеспечения connector. Затем установите программное обеспечение connector.
