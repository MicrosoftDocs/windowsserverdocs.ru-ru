# <a name="troubleshoot-file-history-in-windows-server-essentials"></a>Устранение неполадок с историей файлов в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials 
  
## <a name="troubleshoot-issues-with-user-file-history-backups"></a>Устранение неполадок с резервными копиями истории файлов  
 При управлении резервными копиями истории файлов для пользователя или компьютера, который был добавлен на сервер под управлением Windows Server Essentials, могут возникнуть следующие проблемы.  
  
### <a name="file-history-data-is-not-automatically-deleted"></a>Данные истории файлов не удаляются автоматически  
 Данные истории файлов могут не удаляться автоматически если:  
  
-   При удалении учетной записи пользователя, решение не удалять данные истории файлов s учетной записи пользователя и выбрать удаление данных вручную.  
  
-   При попытке удаления данных истории файлов, данные истории файлов уже используются другим процессом.  
  
 Чтобы устранить эту проблему, необходимо вручную удалить историю файлов, используя следующую процедуру:  
  
####  <a name="BKMK_manuallyDelete"></a>Удаление резервных копий истории файлов для пользователя или компьютера вручную  
  
1.  Войдите сервер с правами администратора.  
  
2.  Запустите проводник от имени администратора.  
  
3.  Перейдите к папке резервных копий истории файлов. По умолчанию — C:\ServerFolders\File резервные копии журнала.  
  
4.  Удалите общую папку, в которой находится резервная копия истории файлов:  
  
    -   Чтобы удалить историю файлов для пользователя, удалите резервную дочернюю папку истории файлов с именем s пользователя.  
  
    -   Чтобы удалить историю файлов для компьютера, удалите резервную дочернюю папку истории файлов, с именем компьютера. Например, если пользователь снято с учета < MyComputer01\ > после как начал работать на новом ноутбуке < MyComputer02\ >, следует удалить C:\ServerFolders\File Backups\\ журнал < MyAccount\ > \ < MyComputer01\ > после проверки пользователя, что он перенес все файлы и папки на новый ноутбук и ему не нужно будет для истории файлов в будущем.  
  
### <a name="cannot-apply-file-history-setting-to-a-new-user"></a>Не удается применить параметр истории файлов для нового пользователя  
 При добавлении нового пользователя, имя которого совпадает с именем пользователя, пользователю, который был удален из Windows Server Essentials, конфигурирование истории файлов для нового пользователя может завершиться ошибкой из-за конфликта имен при попытке Windows Server Essentials создать папку для хранения истории файлов нового пользователя. Чтобы устранить эту проблему, можно переименовать папку истории файлов для удаленного пользователя.  
  
##### <a name="to-locate-user-file-history-on-the-server"></a>Поиск истории файлов пользователя на сервере  
  
1.  Войдите сервер с правами администратора.  
  
2.  Для мониторинга Windows Server Essentials щелкните **хранилища**.  
  
3.  На **папок сервера** Обратите внимание на расположение папки резервных копий истории файлов. Расположение по умолчанию — %SystemDrive%\ServerFolders\File Backups\\ журнал.  
  
##### <a name="to-resolve-file-history-issues-for-a-new-user-with-a-name-conflict"></a>Для устранения проблем с историей файлов для нового пользователя с конфликтом имен  
  
1.  Войдите сервер с правами администратора.  
  
2.  Запустите проводник от имени администратора.  
  
3.  Перейдите к папке резервных копий истории файлов. По умолчанию — C:\ServerFolders\File резервные копии журнала.  
  
     Папка резервных копий истории файлов содержит вложенную папку для каждой учетной записи пользователя, который был добавлен в Windows Server Essentials. Например История файлов для пользователя John Smith будет храниться во вложенной папке File History Backups\JohnSmith.  
  
4.  Переименуйте вложенную папку для пользователя, который был удален, например, **<*UserName*> _Deleted **. Если История файлов пользователя больше не нужна, можно удалить папку.  
  

5.  Теперь можно добавить нового пользователя. Инструкции см. в разделе Добавление учетной записи пользователя? в [управление учетными записями пользователей](../manage/Manage-User-Accounts-in-Windows-Server-Essentials.md).  
  
### <a name="a-user-account-was-removed-but-the-users-file-history-remains"></a>Учетная запись пользователя удалена, но остается истории файлов пользователя  
 В некоторых случаях сетевой администратор может выбрать для удаления пользователя или компьютера с сервера, но оставить резервную копию, для использования в будущем истории файлов. Если История файлов больше не нужна, удалите папку резервных копий истории файлов для пользователя или компьютера из общих папок на сервере. Чтобы сделать это, в разделе [Удаление резервных копий истории файлов для пользователя или компьютера вручную](Troubleshoot-File-History-in-Windows-Server-Essentials.md#BKMK_manuallyDelete).  

5.  Теперь можно добавить нового пользователя. Инструкции см. в разделе Добавление учетной записи пользователя? в [управление учетными записями пользователей](../manage/Manage-User-Accounts-in-Windows-Server-Essentials.md).  
  
### <a name="a-user-account-was-removed-but-the-users-file-history-remains"></a>Учетная запись пользователя удалена, но остается истории файлов пользователя  
 В некоторых случаях сетевой администратор может выбрать для удаления пользователя или компьютера с сервера, но оставить резервную копию, для использования в будущем истории файлов. Если История файлов больше не нужна, удалите папку резервных копий истории файлов для пользователя или компьютера из общих папок на сервере. Чтобы сделать это, в разделе [Удаление резервных копий истории файлов для пользователя или компьютера вручную](../support/Troubleshoot-File-History-in-Windows-Server-Essentials.md#BKMK_manuallyDelete).
