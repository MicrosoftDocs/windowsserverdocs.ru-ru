# <a name="get-connected-in-windows-server-essentials"></a>Установка подключения в Windows Server Essentials

>Область применения. Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

Этот мастер можно запустить, введя **http://<servername\>/ connect**, где **< servername\>**  — это имя вашего сервера.

###  <a name="BKMK_1"></a> Обзор программного обеспечения Connector  
 Программное обеспечение Connector для операционной системы Windows Server Essentials подключает компьютеры в вашей сети к серверу Windows Server Essentials. Данное программное обеспечение позволяет настраивать сервер Windows Server Essentials и удаленно администрировать его. Программное обеспечение Connector устанавливается при подключении клиентского компьютера к серверу. ###  <a name="BKMK_2"></a> Предварительные требования для подключения компьютера к серверу  
 Перед подключением компьютера к сети необходимо обеспечить выполнение следующих требований:  
  
-   Процесс установки системы Windows Server Essentials полностью выполнен, и сервер работает. Если программное обеспечение Connector не сможет связаться с сервером, оно завершит процесс установки.  
  

-   Клиентский компьютер работает под управлением поддерживаемой операционной системы. Дополнительные сведения см. в разделе [Supported operating systems for client computers](Get-Connected-in-Windows-Server-Essentials.md#BKMK_4).

  
-   Клиентский компьютер должен иметь допустимое подключение к Интернету.  
  
-   Если клиентский компьютер находится в той же сети, что и сервер, то данный компьютер находится в той же IP-подсети, что и сервер, на котором выполняется система Windows Server Essentials.  
  
-   На клиентском компьютере установлена платформа .NET Framework 4.5. Программное обеспечение Connector автоматически устанавливает его на компьютере.  
  
-   Клиентский компьютер соответствует следующим минимальным системным требованиям:  
  
    -   процессор не ниже 1,4 ГГц  
  
    -   ОЗУ емкостью 1 ГБ или больше  
  
    -   1 ГБ свободного места на жестком диске (часть этого пространства будет освобождена после установки)  
  
-   Раздел загрузки (то есть, раздел диска где установлена операционная система Windows) отформатирован с файловой системой NTFS.  
  
-   Имя компьютера содержит не больше 15 символов.  
  
-   Имя компьютера не содержит символ подчеркивания ("_").  
  
-   Значения времени и даты на компьютере совпадают с соответствующими параметрами на сервере.  
  
-   В заданный момент времени клиентский компьютер может подключаться только к одному серверу Windows Server Essentials.  
  
-   Клиентский компьютер, который уже присоединен к другому домену Active Directory, не может быть присоединен к домену Windows Server Essentials.  
  
> [!NOTE]

>  В среде локального клиента для Windows Server Essentials или Windows Server Essentials компьютеры можно подключать к серверу не добавляя их в домен Windows Server Essentials. Этот метод недоступен для всех поддерживаемые клиентских операционных систем, и такие компоненты, например, как групповая политика и виртуальные частные сети (VPN), которым требуется, чтобы компьютер был подключен к домену, недоступны. Соответствующие инструкции см. в разделе [Connect computers to a Windows Server Essentials server without joining the domain](Get-Connected-in-Windows-Server-Essentials.md#BKMK_10).  
  
 Пошаговые инструкции по подключению компьютера к серверу Windows Server Essentials см. в разделе [Connect computers to the server](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).

###  <a name="BKMK_3"></a> Необходимые условия для подключения к сети компьютера Mac  
 Перед подключением компьютера Mac к сети необходимо обеспечить выполнение следующих требований:  
  
-   Процесс установки операционной системы сервера полностью выполнен, и сервер работает. Если программное обеспечение Connector не может связаться с сервером, оно не будет установлено.  
  
-   Компьютер работает под управлением Mac OS X 10.5 (Leopard) или более поздней версии.  
  
-   Компьютер находится в той же IP-подсети, что и сервер.  
  
-   Компьютер должен иметь допустимое подключение к Интернету.  
  
-   Убедитесь, что компьютер соответствует следующим минимальным системным требованиям.  
  
    -   процессор не ниже 1,4 ГГц  
  
    -   ОЗУ емкостью 1 ГБ или больше  
  
    -   1 ГБ свободного места на жестком диске (часть этого пространства будет освобождена после установки)  
  
-   В заданный момент времени клиентский компьютер может подключаться только к одному серверу.  
  
###  <a name="BKMK_4"></a> Поддерживаемые операционные системы для клиентских компьютеров  
 Система Windows Server Essentials предоставляет один и тот же набор функций для всех поддерживаемых клиентских компьютеров. К таким функциям относятся присоединение к домену, панель запуска и уведомления о работоспособности на стороне клиента.  
  
> [!IMPORTANT]
>  Windows Server Essentials не позволяет присоединять к домену компьютеры, работающие под управлением версий Windows Home, Starter или Media Center. Кроме того, для подключения к этим компьютерам нельзя использовать удаленный веб-доступ.

###  <a name="BKMK_5"></a> Изменения, вносимые сервером на клиентском компьютере  
 При подключении компьютера к серверу программное обеспечение Windows Server Essentials вносит некоторые изменения на компьютере, чтобы данный компьютер и сервер могли работать совместно.  
  
 Программное обеспечение выполняет следующие действия:  
  
-   Устанавливает на компьютере программное обеспечение Connector  
  
-   Устанавливает на компьютере платформу Microsoft .NET Framework 4.5, если она еще не установлена  
  
-   Создает ярлыки на рабочем столе компьютера для панели мониторинга и панели запуска  
  
-   Настраивает порты брандмауэра Windows на компьютере, чтобы обеспечить выполнение следующих компонентов:  
  
    -   Основная сеть  
  
    -   Службы удаленных рабочих столов  
  
-   Вносит следующие изменения для ускорения процесса архивации на компьютере:  
  
    -   Создает запланированные задачи для выполнения автоматического резервного копирования  
  
    -   Устанавливает службы, которые управляют операциями резервного копирования с помощью сервера  
  
    -   Устанавливает драйвер виртуального устройства, который используется в процессе восстановления файлов и папок  
  
-   Устанавливает агент работоспособности для выявления проблем и создания соответствующих уведомлений  
  
-   Создает запланированные задачи на компьютере для повторяющихся оценок работоспособности, а также для синхронизации определений оповещений о работоспособности  
  
-   Добавляет службы на компьютере, применяемые для обмена данными с сервером и другими компонентами Windows Server Essentials  
  
-   Открывает TCP-порт 3389 на клиентских компьютерах, на которых выполняются клиенты Windows, обеспечивающие работу служб удаленных рабочих столов  
  
-   Развертывает сеть VPN на клиентском компьютере и обеспечивает взаимодействие одним щелчком, если функция VPN включена в Windows Server Essentials, или предоставляет возможности автоматического соединения, если функция VPN включена в Windows Server Essentials  
  

 Сведения о подключении компьютера к серверу см. в разделе [Connect computers to the server](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).

###  <a name="BKMK_6"></a> Сведения об имени и пароля пользователя сети  
 Имя пользователя сети и пароль можно узнать у пользователя, который управляет вашим сервером. Эти учетные данные можно использовать для подключения компьютера к серверу и доступа к информации на сервере. 

  
 Являясь администратором сервера, вы можете создавать сетевые учетные данные путем добавления учетной записи пользователя на вкладке **Пользователи** панели мониторинга. Дополнительные сведения об учетных записях пользователей см. в подразделе [Управление учетными записями пользователей с помощью панели мониторинга](../manage/Manage-User-Accounts-in-Windows-Server-Essentials.md#BKMK_Manage8).  
  
###  <a name="BKMK_7"></a> Учетной записи администратора сервера  
 Для установки программного обеспечения Connector необходимо указать имя учетной записи администратора сети и пароль. Такая учетная запись позволяет пользователю управлять локальной сетью организации, а также управлять и обслуживать сетевые устройства, например, коммутаторы и маршрутизаторы.  
  
 С помощью учетной записи администратора сети можно выполнять следующие задачи:  
  
-   Установка сетевых приложений и другого программного обеспечения  
  
-   Управление хранением данных на сервере  
  
-   Распространение обновлений программного обеспечения  
  
-   Выполнение регулярного резервного копирования  
  
-   Наблюдение за ежедневными операциями в сети  
  
 В Windows Server Essentials, Windows Server Essentials и Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience, вы можете назначить сетевому администратору уровень доступа для любой учетной записи пользователя. Это позволяет получить необходимые разрешения для выполнения задач администратора сети. При назначении пользователю уровня доступа администратора сети для выполнения любой задачи, для которой требуются права администратора, появляется окно **Управление доступом на уровне пользователей**.  
  
###  <a name="BKMK_8"></a> Удаление компьютера из домена Windows  
 При удалении компьютера из домена система предложит ввести имя пользователя и пароль для учетной записи домена.  
  
##### <a name="to-remove-a-computer-from-a-windows-domain"></a>Удаление компьютера из домена Windows  
  
1.  Щелкните **Запустить**, щелкните правой кнопкой мыши **Компьютер**, затем нажмите кнопку **Свойства**.  
  
2.  В разделе **Имя компьютера, имя домена и параметры рабочей группы**нажмите кнопку **Изменить параметры**.  
  
    > [!NOTE]
    >  При появлении запроса на ввод пароля администратора или его подтверждение введите пароль домена или подтвердите его.  
  
3.  В диалоговом окне **Свойства системы** щелкните **Имя компьютера** , а затем **Изменить**.  
  
4.  В диалоговом окне **Изменение имени компьютера или домена** в области **Член** щелкните **Рабочая группа**, а затем выполните одно из следующих действий:  
  
    1.  Для присоединения к существующей рабочей группе введите имя рабочей группы, к которой вы хотите присоединиться, и нажмите кнопку **ОК**.  
  
    2.  Для создания рабочей группы введите имя рабочей группы, которую требуется создать, и нажмите кнопку **ОК**.  
  
        > [!NOTE]
        >  Компьютер будет удален из домена, а учетная запись компьютера в домене будет отключена.

###  <a name="BKMK_9"></a> Подключение компьютеров к серверу  
 При подключении компьютера к серверу, на котором работает Windows Server Essentials или Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience, убедитесь, что клиентский компьютер имеет допустимое подключение к Интернету.  
  
 Выполните следующую процедуру на всех клиентских компьютерах, чтобы подключить их к серверу.  
  
 Для выполнения данной процедуры необходимы следующие сведения:  
  
-   Имя и пароль пользователя компьютера в новой сети.  
  
-   Имя пользователя и пароль для учетной записи локального администратора данного компьютера.  
  
> [!NOTE]

>  В среде локального клиента для Windows Server Essentials или Windows Server Essentials компьютеры можно подключать к серверу не добавляя их в домен Windows Server Essentials. Этот метод недоступен для всех поддерживаемые клиентских операционных систем, и такие компоненты, например, как групповая политика и виртуальные частные сети (VPN), которым требуется, чтобы компьютер был подключен к домену, недоступны. Соответствующие инструкции см. в разделе [Connect computers to a Windows Server Essentials server without joining the domain](Get-Connected-in-Windows-Server-Essentials.md#BKMK_10).  

>  В среде локального клиента для Windows Server Essentials или Windows Server Essentials компьютеры можно подключать к серверу не добавляя их в домен Windows Server Essentials. Этот метод недоступен для всех поддерживаемые клиентских операционных систем, и такие компоненты, например, как групповая политика и виртуальные частные сети (VPN), которым требуется, чтобы компьютер был подключен к домену, недоступны. Соответствующие инструкции см. в разделе [Connect computers to a Windows Server Essentials server without joining the domain](Get-Connected-in-Windows-Server-Essentials.md#BKMK_10).  

  
##### <a name="to-connect-a-client-computer-to-the-server"></a>Подключение клиентского компьютера к серверу  
  
1.  Выполните вход на компьютере, который требуется подключить к серверу.  
  
    > [!NOTE]
    >  Если на компьютере настроено несколько учетных записей пользователей, войдите в систему с записью пользователя, чьи документы, изображения и личные настройки требуется сохранить после подключения этого компьютера к серверу.  
  
2.  Откройте браузер, например Internet Explorer.  
  
3.  В адресной строке введите **http://<servername\>/Connect**, и нажмите клавишу ВВОД.  
  
    > [!NOTE]
    >  Если компьютер находится в удаленном расположении вне сети Windows Server Essentials, чтобы подключить компьютер к мастеру сервера введите **http://<domainname\>/ connect** в адресной строке браузера к web ( где < домена\> — доменное имя вашей организации). Информацию об имени домена можно получить у администратора сети.  
  
4.  Откроется страница **Подключение компьютера к серверу** . Выполните одно из следующих действий.  
  
    -   Если компьютер работает под управлением ОС Windows, щелкните ссылку **Загрузить программное обеспечение для Windows**.  
  
    -   Если компьютер работает под управлением ОС Mac OS X или более поздней версии, щелкните ссылку **Загрузить программное обеспечение для Mac**.  
  
5.  В окне предупреждения о загрузке файла щелкните **Выполнить**.  
  
6.  Если появится окно контроля учетных записей, нажмите кнопку **Да** или введите имя и пароль локального пользователя (если они запрашиваются).  
  
7.  Откроется мастер подключения компьютера к серверу. Для завершения работы мастера выполните следующие действия.  
  
    1.  Примите лицензионное соглашение.  
  
    2.  На странице **Найти сервер** выполните операцию автоматического обнаружения сервера в локальных сетях и выберите сервер, к которому нужно подключиться. Или, если у вас уже есть нужные сведения, введите вручную имя сервера или адрес домена.  
  
    3.  На странице **Введите новое имя пользователя и пароль для входа в сеть** выполните следующее:  
  
        -   Если это первый компьютер, который подключается к серверу, и он будет использоваться для администрирования сервера, используйте учетную запись администратора, созданную при установке.  
  
        -   Для остальных компьютеров сначала создайте учетные записи на информационной панели. В зависимости от задач, выполняемых пользователем на компьютере, создайте для него учетную запись с правами администратора или учетную запись обычного пользователя.  
  
    4.  Если компьютер работает под управлением Windows 8, Windows 8.1 или Windows 10, пропустите этот шаг. Если компьютер работает под управлением ОС Windows 7 и имеются документы, изображения или личные настройки (например, фоновые изображения рабочего стола, экранные заставки или избранные ссылки Internet Explorer), которые нужно сохранить после присоединения компьютера к новой сети, выберите на странице мастера **Выберите, следует ли перемещать текущие данные и параметры** вариант **Переместить данные и параметры в новую учетную запись пользователя**.  
  
    5.  На странице **Выберите, следует ли выводить компьютер из спящего режима для создания резервной копии** укажите, требуется ли автоматически пробуждать компьютер для создания резервной копии.  
  
    6.  Чтобы присоединить компьютер к сети, следуйте дальнейшим инструкциям мастера.  
  
8.  После присоединения компьютера к сети войдите на него с новым именем пользователя и паролем.  
  
    > [!NOTE]
    >  При первом входе на компьютер с Windows 8 с сетевой учетной записью после подключения к серверу появятся инструкции по миграции файлов и приложений из старой учетной записи. Перенесите все файлы и приложения в сетевую учетную запись, следуя указаниям на странице **Как перенести файлы и приложения из старой учетной записи?**.  
  
9. После успешного подключения компьютера к серверу, ярлыки для Connector TrayApp и панели мониторинга сервера отображаются в меню "Пуск", который можно использовать следующим образом (если компьютер работает под управлением Windows 8, Windows 8.1 или Windows 10, панель мониторинга и Connector TrayApp будут доступны на начальном экране компьютера):  
  
    -   Если компьютер работает под управлением Windows 8, Windows 8.1 или Windows 10, панель мониторинга и Connector trayapp можно будет искать как приложения.  
  
    -   На панели Connector TrayApp можно включить или отключить функцию **Сохранить удаленное подключение** . TrayApp также можно дважды щелкнуть для активации панели запуска. С помощью панели запуска можно получать доступ к ярлыку общих папок, настраивать архивацию компьютера, отправлять оповещения и открывать веб-сайт удаленного доступа.  
  
    -   **Панель мониторинга** позволяет администрировать сервер.  
  
###  <a name="BKMK_10"></a> Подключение компьютеров к серверу Windows Server Essentials без присоединения к домену  
 В этом разделе описывается добавление компьютера Windows 7, Windows 8, Windows 8.1 или Windows 10 к сети Windows Server Essentials без присоединения компьютера к домену Windows Server Essentials в среде локального клиента. Этот способ подключения поддерживается в Windows Server Essentials и Windows Server Essentials.  
  
 Способ является альтернативой обычному методу, при котором требуется присоединение компьютера к домену Windows Server Essentials. С помощью этого метода, если компьютер находится в другом домене, его из данного домена необходимо удалить и только потом добавить к домену Windows Server Essentials.  
  
#### <a name="feature-limits"></a>Ограничения функций  
 Если клиентский компьютер не будет добавлен к домену Windows Server Essentials, то некоторые функции будут ограничены:  
  
-   Все компоненты, требуется, чтобы компьютер был присоединен к домену? включая учетные данные домена, Групповая политика и виртуальные частные сети (VPN)? недоступны.  
  
-   Все надстройки сторонних разработчиков, для которых требуется, чтобы компьютер был присоединен к домену, будут работать неправильно.  
  
-   Этот метод нельзя использовать для подключения локального компьютера к серверу.  
  
#### <a name="prerequisites"></a>предварительные требования  
  
-   Компьютер должен иметь физическое подключение к локальной сети.  
  
-   На компьютере должна быть установлена одна из следующих операционных систем:  
  
    -   Windows 10 Pro, Windows 10 Корпоративная  
  
    -    Windows 8.1 Профессиональная, Корпоративная Windows 8.1, Windows 8 Профессиональная, Windows 8 Корпоративная  
  
    -    Windows 7 Профессиональная (x86 и x64), Windows 7 Корпоративная (x86 и x64), Windows 7 Ultimate (x86 и x64)  
  

-   Компьютер должен удовлетворять всем требованиям для клиентских компьютеров в Windows Server Essentials. Дополнительные сведения см. в разделе [Prerequisites for connecting a computer to the server](Get-Connected-in-Windows-Server-Essentials.md#BKMK_2).  

  
-   Чтобы включить подключение без присоединения к домену, необходимо войти в систему на компьютере с учетной записью, которая является членом локальной группы администраторов.  
  
-   Для подключения компьютера к серверу Windows Server Essentials будут необходимы следующие сведения об учетной записи:  
  
    -   Учетная запись с правами администратора на Windows Server Essentials (ваша учетная запись).  
  
    -   Имя и пароль пользователя для учетной записи домена, который будет использовать компьютер. Учетная запись домена также должна иметь права администратора на сервере Windows Server Essentials.  
  
#### <a name="connect-to-a-windows-server-essentials-network"></a>Подключение к сети Windows Server Essentials  
 После проверки соблюдения всех необходимых условий подключите компьютер к сети Windows Server Essentials.  
  
###### <a name="to-connect-a-computer-in-a-different-domain-to-a-windows-server-essentials-network"></a>Подключение компьютера в другом домене к сети Windows Server Essentials  
  
1.  Войдите на клиентский компьютер с учетной записью, которая является членом локальной группы администраторов.  
  
2.  Откройте командную строку с правами администратора.  
  
    -   В Windows 10, нажмите кнопку **запустить** кнопку, выберите **все приложения** -> **служебные программы Windows** -> **командной**, щелкните правой кнопкой мыши командную строку и нажмите кнопку **Запуск от имени администратора**.  
  
    -   В Windows 8 на **запустить** введите **команду** и нажмите клавишу ВВОД. В списке результатов щелкните правой кнопкой мыши **Командная строка**, а затем щелкните **Запуск от имени администратора**.  
  
    -   В Windows 7 на **запустить** меню введите **команда** в поле поиска, щелкните правой кнопкой мыши **командной**и нажмите кнопку **Запуск от имени администратора**.  
  
3.  В командной строке введите следующую команду и нажмите клавишу ВВОД:  
  
    ```  
    reg add "HKLM\SOFTWARE\Microsoft\Windows Server\ClientDeployment" /v SkipDomainJoin /t REG_DWORD /d 1  
    ```  
  

4.  Выполните задачу из раздела [Connect computers to the server](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).  

  
####  <a name="BKMK_SecondServer"></a> Присоединение второго сервера к сети  
  
###### <a name="to-join-a-second-server-to-the-network"></a>Присоединение второго сервера к сети  
  
1.  Войдите на сервер, который требуется подключить к сети Windows Server Essentials.  
  
2.  Откройте веб-браузер и введите в адресной строке **http://<servername\>/Connect**, где *< servername\>*  — это имя сервера под управлением Windows Server Essentials и нажмите клавишу ВВОД.  
  
3.  Если на сервере, который вы пытаетесь подключить к сети Windows Server Essentials, включена конфигурация усиленной безопасности Internet Explorer, выполните следующую процедуру. В противном случае пропустите этот шаг.  
  
    1.  Чтобы принять сообщение о блокировке, нажмите кнопку **Закрыть**.  
  
    2.  Добавить **http://<servername\>/Connect** веб-сайт в список доверенных сайтов следующим образом:  
  
        1.  В области навигации браузера щелкните **Инструменты**, а затем **Свойства браузера**.  
  
        2.  Перейдите на вкладку **Безопасность** , а затем щелкните **Надежные сайты**.  
  
        3.  Щелкните **Сайты**.  
  
        4.  Веб-сайт должен появиться в поле **Добавить в зону следующий узел** . Нажмите кнопку **Добавить**.  
  
        5.  Щелкните **Закрыть**, а затем нажмите кнопку **ОК**.  
  
    3.  Обновите веб-страницу.  
  

    4.  Для подключения второго сервера к серверу, который работает под управлением ОС Windows Server Essentials, используйте процедуру в подразделе [Подключение компьютеров к серверу](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).  

  
    > [!NOTE]
    >  Между процедурами подключения второго сервера к серверу под управлением Windows Server Essentials и подключения к клиентскому компьютеру существуют следующие различия:  
    >   
    >  -   Миграция профилей пользователей отсутствует. Таким образом, текущий профиль не переносится.  
    > -   С помощью архивной копии клиентского компьютера второй сервер архивировать нельзя. Нельзя принудить компьютер к резервному копированию.  
  
 После присоединения второго сервера к серверу под управлением Windows Server Essentials, подключенному серверу предоставляются следующие возможности:  
  
-   Функции состояния обновлений и оповещений будут работать так же, как на других клиентских компьютерах.  
  
-   Функции "в сети" и "не в сети" будут работать так же, как на других клиентских компьютерах.  
  
-   Ко второму серверу можно подключиться с помощью удаленного веб-доступа.  
  
-   Второй сервер будет включен в отчеты о работоспособности, так как система Windows Server Essentials будет создавать оповещения, связанные с этим сервером.  
  
 Процесс управления вторым сервером с сервера под управлением Windows Server Essentials, будет отличаться от процесса управления другими клиентскими компьютерами следующим образом:  
  
-   Для TrayApp, панели запуска и клиента панели мониторинга точек входа не будет.  
  
-   Второй сервер указывается в группе **Серверы** на вкладке **Устройства** .  
  
-   Так как архивация клиентского компьютера для второго сервера не поддерживается, в качестве состояния резервного копирования будет отображаться значение **Не поддерживается**. Кроме того, если вы выберете второй сервер и щелкните правой кнопкой мыши, то для второго сервера никакие задачи, связанные с резервным копированием и восстановлением, отображаться не будут.  
  
-   Если вы выберите второй сервер и щелкните задачу **Просмотр свойств сервера** , то на странице свойств сервера вкладка **Архивация** будет отсутствовать.  
  
-   Так как в ОС Windows Server нет центра обеспечения безопасности, в качестве состояния безопасности второго сервера будет отображаться значение **Неприменимо**.  
  
-   В качестве состояния групповой политики второго сервера будет отображаться значение **Неприменимо**.  
  
###  <a name="BKMK_11"></a> Установите программное обеспечение Connector  
 Программное обеспечение Connector в Windows Server Essentials устанавливается при подключении компьютера к серверу с помощью мастера подключения компьютера к серверу. Этот мастер можно запустить, введя **http://<ServerName\>/ connect** в адресной строке браузера (где *< ServerName\>*  — это имя вашего сервера).  
  
> [!NOTE]
>  Если компьютер находится в удаленном расположении, чтобы подключить компьютер к мастеру сервера введите **http://<domainname\>/ connect** в адресной строке браузера (где *< домена\>*  — доменное имя вашей организации). Информацию об имени домена можно получить у администратора сети.  
  
 Программное обеспечение Connector выполняет следующие действия:  
  
-   Подключает ваш компьютер к Windows Server Essentials  
  
-   Автоматически создает резервную копию компьютера ночью (если на сервере настроена функция создания резервных копий клиента)  
  
-   Позволяет администратору отслеживать работоспособность вашего компьютера  
  
-   Позволяет настраивать и удаленно администрировать систему Windows Server Essentials с вашего домашнего компьютера  
  

 Пошаговые инструкции по подключению компьютера к серверу Windows Server Essentials см. в подразделе [Connect computers to the server](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).   

  
###  <a name="BKMK_12"></a> Перемещение данных и параметров компьютера вручную  
  Windows Server Essentials и Windows Server Essentials поддерживают перенос профиля пользователя только для клиентских компьютеров, работающих под управлением операционной системы Windows 7. При подключении к серверу компьютера, работающего под управлением ОС Windows 7, мастер подключения компьютера к серверу может автоматически перенести профиль пользователя.  
  
 Профиль пользователя автоматически перенести нельзя при подключении к серверу Windows 8, Windows 8.1 или Windows 10 компьютеров. Однако на компьютере с Windows 8 для переноса данных и параметров исходного локального пользователя на компьютер, присоединенный к домену, можно использовать средство переноса данных Windows. Для этого необходимо иметь права администратора как на исходном компьютере с Windows 8, так и на конечном компьютере Windows 8. Сведения об использовании средства переноса данных Windows для переноса файлов и параметров см. в [статье 2735227](https://support.microsoft.com/kb/2735227) в базе знаний Майкрософт.  
  
###  <a name="BKMK_Transfer"></a> Перенос нескольких профилей пользователей во время развертывания компьютера  
 Перед подключением компьютера под управлением ОС Windows 7 или Windows 7 с пакетом обновления 1 к серверу Windows Server Essentials для передачи несколько профилей локальных пользователей необходимо сначала на сервере создать соответствующие учетные записи пользователей сети. Дополнительные сведения о создании учетных записей пользователей сети см. в подразделе [Добавление учетной записи пользователя](../manage/Manage-User-Accounts-in-Windows-Server-Essentials.md#BKMK_Manage1).  
  
 Миграция профилей пользователей поддерживается только на компьютере под управлением Windows 7 (для Windows Server Essentials) или Windows 7 SP1 (для Windows Server Essentials). При подключении компьютера к серверу Windows Server Essentials с помощью мастера подключения компьютера к серверу данные и параметры пользователя из старых локальных учетных записей пользователей можно перенести в новые учетные записи пользователей сети. Для этого на странице **Перенос данных и параметров существующих пользователей** мастера сопоставьте учетные записи пользователей сети и локальные учетные записи пользователей, существующие на компьютере, чтобы передать профили пользователей, расположенных на клиентском компьютере.  
  
###  <a name="BKMK_13"></a> Удаление программного обеспечения Connector  
 Программное обеспечение Connector можно удалить с компьютера с помощью панели управления. Обычно это необходимо сделать при наличии проблемы с программным обеспечением Connector, или если необходимо установить новую версию программного обеспечения Connector. Для выполнения этой процедуры вы должны войти на компьютер с правами администратора.  
  
> [!IMPORTANT]
>  При обновлении операционной системы на клиентском компьютере программное обеспечение Connector удаляется автоматически. По окончании процедуры обновления программное обеспечение Connector необходимо переустановить. Программное обеспечение Connector рекомендуется удалить до обновления операционной системы. Программное обеспечение Connector можно удалить и по окончании процедуры обновления. Тем не менее, это может привести к тому, что клиентский компьютер будет иметь некорректное состояние для сервера, пока программное обеспечение Connector не будет удалено и переустановлено.  
  
##### <a name="to-uninstall-connector-software-from-a-computer"></a>Удаление программного обеспечения Connector с компьютера  
  
1.  На компьютере под управлением Windows 7, Windows 8, Windows 8.1 или Windows 10, откройте **панели управления**, а затем в **программы** щелкните **Просмотр установленных обновлений**.  
  
2.  В списке установленных программ выберите **Windows Server Essentials Connector**, а затем нажмите кнопку **Удалить**.  
  
3.  На странице предупреждения щелкните **Да**.  
  
4.  Если появляется окно **Контроль учетных записей пользователей** , нажмите кнопку **Разрешить**.  
  
5.  На Windows Server Essentials, при которой предлагается закрыть панель запуска появляется страница Windows Server Essentials Connector, нажмите кнопку **ОК**.  
  
6.  Подождите, пока программа не будет удалена. После удаления программного обеспечения в списке установленных программ или обновлений запись **Windows Server Essentials Connector** больше не отображается. Кроме того, ярлыки панели запуска и панели мониторинга больше не будут отображаться на рабочем столе компьютера.  
  
> [!NOTE]
>  -   Удаление программного обеспечения Connector не приводит к удалению компьютера из списка компьютеров, которые отображаются на вкладке **Устройства** панели мониторинга. Сведения об удалении компьютера с панели мониторинга см. в разделе [Remove a computer from the server](../manage/Manage-Devices-in-Windows-Server-Essentials.md#BKMK_3).  
> -   При удалении программного обеспечения Connector общие папки на клиентском компьютере, которые были сопоставлены с сервером, удалены не будут. Общие папки, сопоставляемые с сервером, необходимо удалить вручную.  

> -   При удалении программного обеспечения Connector компьютер не отсоединяется от исходного домена. Отсоединить компьютер от домена необходимо вручную. Инструкции см. в разделе [Remove a computer from a Windows domain](Get-Connected-in-Windows-Server-Essentials.md#BKMK_8).  

  
###  <a name="BKMK_14"></a> Отключение компьютера или повторное подключение компьютера к серверу  
 Для отключения компьютера от сервера необходимо выполнить следующие действия:  
  

1.  Удалите программное обеспечение Connector на компьютере с помощью панели управления. Пошаговые инструкции см. в разделе [Uninstall the Connector software](Get-Connected-in-Windows-Server-Essentials.md#BKMK_13).   

  
2.  Отсоедините компьютер от домена Windows Server Essentials и присоедините его к рабочей группе. Пошаговые инструкции по присоединению Windows к рабочей группе см. в разделе [Присоединение или создание рабочей группы](https://windows.microsoft.com/windows7/Join-or-create-a-workgroup).  
  
3.  Удалите компьютера с сервера с помощью панели мониторинга. Пошаговые инструкции см. в разделе [Remove a computer from the server](../manage/Manage-Devices-in-Windows-Server-Essentials.md#BKMK_3).  
  
 Для повторного подключения компьютера к серверу, который ранее был отключен от сети сервера Windows Server Essentials, необходимо выполнить следующие действия:  
  

1.  Удалите программное обеспечение Connector на компьютере с помощью панели управления. Пошаговые инструкции см. в разделе [Uninstall the Connector software](Get-Connected-in-Windows-Server-Essentials.md#BKMK_13).  
  
2.  Отсоедините компьютер от домена Windows Server Essentials и присоедините его к рабочей группе. Пошаговые инструкции по присоединению Windows к рабочей группе см. в разделе [Присоединение или создание рабочей группы](https://windows.microsoft.com/windows7/Join-or-create-a-workgroup).  
  
3.  Подключите компьютер к серверу с помощью мастера подключения компьютера. Пошаговые инструкции см. в подразделе [Подключение компьютеров к серверу](Get-Connected-in-Windows-Server-Essentials.md#BKMK_9).  
  
###  <a name="BKMK_Sleep"></a> Как архивации со спящим режимом и режимом гибернации  
 Если при подключении компьютера к серверу установить флажок **Пробудить компьютер для резервного копирования**, то компьютер будет автоматически ежедневно выходить из спящего режима или режима гибернации в соответствии с расписанием резервного копирования, что позволит создавать для него резервные копии. По окончании процедуры резервного копирования компьютер возвращается в спящий режим или режим гибернации в соответствии с параметрами управления электропитанием. Если этот флажок не установлен, сервер не будет создавать резервную копию компьютера, если компьютер находится в спящем режиме или режиме гибернации. Дополнительные сведения см. в разделе [Управление архивацией клиента](../manage/Manage-Client-Computer-Backup-in-Windows-Server-Essentials.md).
