# <a name="use-remote-web-access-in-windows-server-essentials"></a>Использование удаленного веб-доступа в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

При входе в систему удаленного веб-доступа можно подключиться к компьютерам в сети Windows Server Essentials, открыть панель мониторинга для управления сетью Windows Server Essentials и получить доступ ко всем общим папкам и файлам мультимедиа на сервере.

###  <a name="BKMK_1"></a>Войдите в систему удаленного веб-доступа  
 При входе в систему удаленного веб-доступа с локального или удаленного компьютера можно доступ к ресурсам на сервере под управлением Windows Server Essentials и компьютерах в сети.  
  
##### <a name="to-log-on-to-remote-web-access-from-a-network-computer"></a>Вход в удаленный веб-доступ с компьютера в сети  
  
1.  Откройте веб-браузер, введите **https://***< YourServerName\ >***/remote** в адресной строке и нажмите клавишу ВВОД.  
  
    > [!NOTE]
    >  Убедитесь, что ввели букву s протокола HTTPS.  
  
2.  На странице входа в систему удаленного веб-доступа в текстовых полях введите имя пользователя и пароль и нажмите кнопку со стрелкой.  
  
##### <a name="to-log-on-to-remote-web-access-from-a-remote-computer"></a>Вход в удаленный веб-доступ с удаленного компьютера  
  
1.  Откройте веб-браузер, введите **https://***< YourDomainName\ >***/remote** в адресной строке и нажмите клавишу ВВОД.  
  
    > [!NOTE]
    >  Сведения об имени вашего домена можно получить у администратора сети. Убедитесь, что ввели букву s протокола HTTPS.  
  
2.  На странице входа в систему удаленного веб-доступа в текстовых полях введите имя пользователя и пароль и нажмите кнопку со стрелкой.  
  
###  <a name="BKMK_1.5"></a>Удаленный доступ к компьютеру  
 Находясь вне офиса, можно использовать веб-браузер для входа на сайт удаленного веб-доступа для удаленного доступа к вашей панели мониторинга Windows Server Essentials, общих папок и компьютерам в сети.  
  
 При подключении к панели мониторинга Windows Server Essentials можно управлять так же, как если бы вы находились в офисе. Можно выполнить все стандартные задачи администрирования, таких как добавление учетных записей пользователей, Добавление общих папок, Настройка доступа к общей папке и т. д. При подключении к компьютерам в сети, можно доступ к рабочему столу, как если бы вы сидели за ними в офисе.  
  
 **Состояние** столбце показано, можно подключиться к компьютеру в сети и может содержать следующие значения:  
  
-   **Доступные**  
  
     Компьютер включен и доступен для удаленного подключения. Даже если вы видите это состояние, вы по-прежнему может не иметь возможность подключаться к этому компьютеру, если сторонний брандмауэр блокирует подключение.  
  
-   **Автономный или в спящем режиме**  
  
     Компьютер выключен или находится в режиме сна или гибернации. Если компьютер находится в автономном режиме или в спящем режиме, статус обновляется в реальном времени, чтобы знать, когда компьютер станет доступен.  
  
-   **Неподдерживаемая операционная система**  
  
     Операционной системы на компьютере не поддерживает удаленный рабочий стол. Он может занять до 6 часов на обновление на сервере, если изменение этого статуса.  
  
-   **Подключение отключено**  
  
     Подключение к компьютеру заблокировано брандмауэром, либо удаленный рабочий стол отключен на компьютере или с помощью групповой политики. Он может занять до 6 часов на обновление на сервере, если изменение этого статуса.  
  
#### <a name="to-connect-to-a-computer-on-your-network"></a>Для подключения к компьютеру в сети  
 На **устройств** , щелкните имя компьютера. Можно выбрать только компьютеры с **доступно** состояние.  
  
#### <a name="to-connect-to-the-server-dashboard"></a>Для подключения к панели мониторинга сервера  
 На **устройств** , щелкните имя вашего сервера. Можно выбрать только компьютеры с **доступно** состояние. Необходимо указать учетную запись администратора и пароль на сервере, чтобы использовать панели мониторинга.

    > [!NOTE]
    >  Для удаленного веб-доступа можно отправить только файлы и папки нельзя. Если вы хотите иметь ту же иерархию файлов и папок **общих папок** на сервере, что и на компьютере, необходимо создать папки на сервере в удаленного веб-доступа и затем отправить файлы в созданную папку. Сведения о создании папок сервера см. в разделе [Добавление или перемещение папки сервера](../manage/Manage-Server-Folders-in-Windows-Server-Essentials.md#BKMK_5).

-   Создание папки внутри общей папки в Windows Server Essentials.  
  

-   Удаление, перемещение и переименование файлов и папок в Windows Server Essentials. Дополнительные сведения см. в разделе [создание, переименование, перемещение, delete, или копирование файлов и папок в удаленного веб-доступа](Use-Remote-Web-Access-in-Windows-Server-Essentials.md#BKMK_2).

Дополнительные сведения см. в разделе [создание, переименование, перемещение, delete, или копирование файлов и папок в удаленного веб-доступа](../use/Use-Remote-Web-Access-in-Windows-Server-Essentials.md#BKMK_2).  

  
#### <a name="upload-files"></a>Отправка файлов  
  
###### <a name="to-upload-files"></a>Отправка файлов  
  
1.  В удаленный веб-доступ, щелкните **общих папок** вкладку и щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  В списке файлов и папок общей папки щелкните папку, где вы хотите отправить этот файл, а затем нажмите кнопку **Отправка**.  
  
3.  Если стандартное средство отправки еще не загружено, щелкните **используйте стандартный метод отправки**.  
  
4.  Нажмите кнопку **Обзор** найти файл на компьютере.  
  
5.  Просмотрите структуру папок на компьютере, чтобы найти файл, который требуется отправить и нажмите кнопку **откройте**.  
  
6.  Повторите шаги 2 и 3 для каждого файла, который требуется отправить.  
  
7.  После добавления всех файлов, которые требуется отправить, щелкните **Отправка**.  
  
 Средство отправки файлов упрощает процесс передачи файлов на сервер под управлением Windows Server Essentials. Можно добавить требуется средство отправки файлов с помощью функции перетаскивания-файлов и передать их в общие папки на сервере.  
  
> [!NOTE]
>  Возможность отправки нескольких файлов встроена в веб-браузеры, совместимые с HTML5. Это средство необходим, только если веб-браузер не поддерживает HTML5.  
  
###### <a name="to-upload-files-using-the-easy-file-upload-tool"></a>Отправка файлов с помощью средства отправки файлов  
  
1.  В удаленный веб-доступ, щелкните **общих папок** вкладку и щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  В списке файлов и папок общей папки щелкните папку, где вы хотите отправить эти файлы и нажмите кнопку **Отправка**. Если папка, который вы хотите отправить не существует, щелкните **новую папку**, введите имя новой папки в диалоговом окне и нажмите кнопку **ОК**.  
  
3.  Может потребоваться запустить надстройку Windows Server Solutions. В этом случае щелкните желтую полосу в верхней части экрана, нажмите кнопку запуска **надстройки**, а затем нажмите кнопку **запуска** в диалоговом окне.  
  
4.  Если средство отправки файлов еще не загружено, щелкните **используйте средство отправки файлов**.  
  
5.  Вы можете и перетащите файлы из проводника в средство отправки файлов или щелкните **Обзор, чтобы выбрать файлы**.  
  
6.  После завершения добавления файлов, которые требуется отправить в выбранную папку, нажмите кнопку **Отправка**.  
  
7.  После успешной отправки файлов нажмите кнопку **закрыть**.  
  
#### <a name="download-files-or-folders"></a>Загрузка файлов или папок  
  
###### <a name="to-download-a-single-file"></a>Загрузка одного файла  
  
1.  В удаленный веб-доступ, щелкните **общих папок** вкладку и щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  В списке файлов общей папки установите флажок рядом с файлом, который требуется загрузить на домашний компьютер.  
  
3.  Нажмите кнопку **загрузить** чтобы начать загрузку.  
  
4.  На **Загрузка файла** диалоговом нажмите кнопку **Сохранить** для сохранения файла на компьютере.  
  
5.  В **Сохранить как** диалоговом окне выберите расположение для сохранения файла, а затем нажмите кнопку **Сохранить**. Отдельного файла сжатие перед не загружается.  
  
 Существует два варианта загрузки нескольких файлов или папок. Выберите параметр, который соответствует вашим потребностям.  
  
> [!NOTE]
>  Эти параметры доступны только при загрузке нескольких файлов или папок на компьютере.  
  
-   **Самораспаковывающийся исполняемый файл (.exe)**  
  
    > [!NOTE]
    >   Этот раздел относится к серверу под управлением Windows Server Essentials.  
  
     Самораспаковывающийся исполняемый файл — это файл, можно загрузить, сочетает в себе (исполняемую) программу распаковки и сжатые файлы. При запуске исполняемой программы она автоматически распаковывает сжатые файлы (самораспаковывается). Это распространенный способ распространения сжатых данных, не беспокоясь о ли у получателя подходящая программа распаковки.  
  
    > [!NOTE]
    >  Этот параметр поддерживает символы Юникода.  
  
-   **Сжатая папка Windows (ZIP)**  
  
     При сжатии создается версия файла, меньше, чем исходный файл. Сжатая версия файла имеет расширение имени файла .zip. Типы файлов, которые являются уменьшено наиболее при сжатии текст архиватором, например .txt, DOC и XLS, и графические файлы, без сжатия типы использования, например bmp. Некоторые графические файлы, такие как файлы JPG и GIF, уже используют сжатие и уменьшения размера файла архиватора zip почти не. Кроме того документ Word, который содержит множество рисунков не сжимается хуже документа, содержащего практически один текст.  
  
    > [!NOTE]
    >  Этот параметр обеспечивает ограниченную поддержку международных имен файлов в Windows Server Essentials.  
  
 Перед началом фактической загрузки создается файл exe или zip. В зависимости от количества файлов и общий размер файлов для загрузки это может занять несколько минут. После создания загружаемого файла загрузка выполняется в фоновом режиме. Это позволяет продолжить работу во время загрузки.  
  
###### <a name="to-download-multiple-files-or-folders"></a>Для загрузки нескольких файлов или папок  
  
1.  В удаленный веб-доступ, щелкните **общих папок** вкладку и щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  В списке файлов общей папки установите флажок рядом с файлами или папками, которые требуется загрузить на домашний компьютер.  
  
3.  Нажмите кнопку **загрузить** чтобы начать загрузку.  
  
4.  В **выбрать формат загрузки** диалоговое окно, выберите параметр формата загрузки и нажмите кнопку **ОК**. Сжатый файл подготавливается в формате, вы выбрали.  
  
5.  В **Загрузка файла** диалоговом нажмите кнопку **Сохранить** для сохранения файла на компьютере.  
  
6.  В **Сохранить как** диалоговом окне выберите расположение для сохранения файла, а затем нажмите кнопку **Сохранить**.  
  
#### <a name="retrieve-compressed-files-downloaded-to-your-computer"></a>Извлечение сжатых файлов, загруженных на компьютер  
  
> [!NOTE]
>   Этот раздел относится к серверу под управлением Windows Server Essentials.  
  
 Если выбрать несколько файлов или папок для загрузки, можно получить самораспаковывающийся сжатый исполняемый файл (.exe) или сжатый файл (ZIP).  
  
###### <a name="to-retrieve-a-file-from-the-compressed-exe-file"></a>Извлечение файла из сжатого файла (.exe)  
  
1.  На компьютере дважды щелкните сжатый файл, чтобы открыть его.  
  
2.  Следуйте инструкциям, чтобы извлечь файлы в папку на компьютере.  
  
###### <a name="to-retrieve-a-file-from-the-compressed-zip-file"></a>Извлечение файла из сжатого файла (ZIP)  
  
1.  На компьютере дважды щелкните сжатый файл, чтобы открыть его.  
  
2.  Выберите файлы, которые требуется извлечь, а затем перетащите файлы в папку на компьютере место для их хранения.  
  
    > [!NOTE]
    >  При использовании программы сжатия стороннего разработчика выполните процедуры по извлечению файлов из сжатого файла.  
  
###  <a name="BKMK_2"></a>Создание, переименование, перемещения, удалить или скопируйте файлы и папки в удаленного веб-доступа  
 Удаленный веб-доступ можно использовать для создания новых папок в существующей общей папке, переименования файлов и папок, перемещения и копирования файлов и папок, а также удаления файлов и папок на сервере.  
  
> [!NOTE]
>  Чтобы добавить новые общие папки на сервере, на котором работает Windows Server Essentials, необходимо использовать панель мониторинга. Для подключения к консоли сервера из удаленного веб-доступа на **компьютеры** вкладку, щелкните имя сервера, **Connect**и следуйте инструкциям для входа на сервер. Сведения о создании общих папок см. в разделе [Добавление или перемещение папки сервера](../manage/Manage-Server-Folders-in-Windows-Server-Essentials.md#BKMK_5).  
  
##### <a name="to-create-a-new-folder"></a>Чтобы создать новую папку  
  
1.  В удаленный веб-доступ, щелкните **общих папок** , а затем щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  На панели задач щелкните **новую папку**.  
  
3.  Введите имя папки и нажмите кнопку **ОК**.  
  
##### <a name="to-rename-a-file-or-folder"></a>Переименование файла или папки  
  
1.  В удаленный веб-доступ, щелкните **общих папок** , а затем щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  Щелкните правой кнопкой мыши файл или папку, в который требуется переименовать, а затем нажмите кнопку **переименование**.  
  
3.  Введите новое имя в текстовом поле и нажмите кнопку **ОК**.  
  
##### <a name="to-move-files-or-folders"></a>Перемещение файлов и папок  
  
1.  В удаленный веб-доступ, щелкните **общих папок** , а затем щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  Установите флажок рядом с файлами или папками, которые вы хотите переместить, щелкните правой кнопкой мыши один из выбранных файлов или папок и нажмите кнопку **Вырезать**.  
  
3.  Щелкните правой кнопкой мыши папку, в который вы хотите переместить файлы или папки, а затем нажмите кнопку **вставки**.  
  
##### <a name="to-delete-a-file-or-folder"></a>Удаление файла или папки  
  
1.  В удаленный веб-доступ, щелкните **общих папок** , а затем щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  Установите флажок рядом с файлами или папками, которые вы хотите удалить, щелкните правой кнопкой мыши один из выбранных файлов или папок и нажмите кнопку **удалить**.  
  
3.  Чтобы подтвердить удаление выбранных файлов и папок, нажмите кнопку **Да**.  
  
##### <a name="to-copy-files-or-folders"></a>Копирование файлов и папок  
  
1.  В удаленный веб-доступ, щелкните **общих папок** , а затем щелкните ссылку на общую папку. Отображается список файлов и папок в этой общей папке.  
  
2.  Установите флажок рядом с файлами или папками, которые вы хотите скопировать, щелкните правой кнопкой мыши один из выбранных файлов или папок и нажмите кнопку **копирования**.  
  
3.  Щелкните правой кнопкой мыши папку, необходимо скопировать файлы или папки и нажмите кнопку **вставки**.

###  <a name="BKMK_8"></a>Использование удаленного веб-доступа с мобильного устройства  
 Можно выполнить вход в удаленный веб-доступ со смартфона, чтобы просмотреть файлы и папки в общих папках на сервере.  
  
> [!NOTE]
>  Можно также загрузить и использовать приложение My Server для Windows Server Essentials из [Windows Phone Marketplace](http://www.windowsphone.com/apps/6c2f98d5-6fcf-4e1d-b8b1-cde62ea1a94a) для доступа к вашей общей папкам и файлам мультимедиа, сохраненные на сервере.  
  
##### <a name="to-log-on-to-remote-web-access-from-a-mobile-device"></a>Вход в удаленный веб-доступ с мобильного устройства  
  
1.  Откройте веб-браузер и введите **https://***< YourDomainName\ >***/remote** в адресной строке.  Убедитесь, что ввели букву s протокола HTTPS.  
  
2.  На странице входа в систему удаленного веб-доступа в текстовых полях введите имя пользователя и пароль и нажмите кнопку со стрелкой. Войти в систему для мобильной версии удаленного веб-доступа.  
  
##### <a name="to-switch-to-the-desktop-version-of-remote-web-access"></a>Чтобы переключиться в классической версии удаленного веб-доступа  
  
1.  Откройте веб-браузер и введите **https://***< YourDomainName\ >***/remote** в адресной строке.  Убедитесь, что ввели букву s протокола HTTPS.  
  
2.  На странице входа в систему удаленного веб-доступа, введите имя пользователя и пароль в текстовые поля, щелкните **просмотреть версию для рабочего стола**, а затем нажмите кнопку со стрелкой. Войти в систему для классической версии удаленного веб-доступа.  
  
##### <a name="to-return-to-the-mobile-version-of-remote-web-access"></a>Чтобы вернуться к мобильной версии удаленного веб-доступа  
  
1.  Выйдите из системы.  
  
2.  Откройте веб-браузер и введите **https://***< YourDomainName\ >***/remote/m** в адресной строке. Убедитесь, что ввели букву s протокола HTTPS.  
  
3.  Отображается версия мобильной удаленного веб-доступа. На странице входа в систему удаленного веб-доступа в текстовых полях введите имя пользователя и пароль и нажмите кнопку со стрелкой. Войти в систему для мобильной версии удаленного веб-доступа.
