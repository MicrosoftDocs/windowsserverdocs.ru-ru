# <a name="manage-remote-web-access-in-windows-server-essentials"></a>Управление удаленным веб-доступом в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials
 
 Удаленного веб-доступа в Windows Server Essentials или Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience обеспечивает упрощенное использование сенсорного браузера для доступа к приложениям и данным практически из любого места, где есть подключение к Интернету и с помощью практически с любого устройства. Чтобы использовать функции удаленного веб-доступа, необходимо сначала включить ее с помощью мастера настройки повсеместного доступа и затем настроить маршрутизатор и имя домена.

###  <a name="BKMK_Overview"></a>Обзор удаленного веб-доступа  
 Находясь вне офиса, можно открыть веб-браузер и доступ к удаленного веб-доступа из любого места с доступом к Интернету. В удаленный веб-доступ вы можете:  
  
-   Доступ к общим файлам и папкам на сервере.  
  
-   Доступ к серверу и компьютерам в сети. Это означает, что можно доступа к рабочему столу компьютера, подключенного как если бы вы сидели непосредственно перед ним в офисе.  
  
  Удаленный веб-доступ не включен по умолчанию. При запуске мастера настройки повсеместного доступа мастер пытается настроить маршрутизатор и подключение к Интернету. После включения удаленного веб-доступа можно настроить доменное имя сервера и настройка удаленного веб-доступа. Можно также выполнить настройку маршрутизатора снова при изменении маршрутизатор.  
  
 Разрешение на доступ к удаленному веб-доступа не предоставляется автоматически при добавлении учетной записи пользователя. При добавлении учетной записи пользователя, вы можете разрешить доступ к общим папкам, библиотеке мультимедиа, компьютеры, ссылкам на главную страницу и серверной панели мониторинга. Можно также указать, что пользователь не допускается использование удаленного веб-доступа.  
  
 Настройка удаленного веб-доступа отображается для каждой учетной записи пользователя на **пользователей** вкладка панели мониторинга Windows Server Essentials. Чтобы изменить настройку удаленного веб-доступа, щелкните правой кнопкой мыши учетную запись пользователя и нажмите кнопку **просмотреть свойства учетной записи**.  
  
###  <a name="BKMK_TurnOnRWA"></a>Включение удаленного веб-доступа  
 Удаленный веб-доступ можно включить, запустив мастер настройки повсеместного доступа с панели мониторинга сервера.  
  
##### <a name="to-turn-on-remote-web-access"></a>Включение удаленного веб-доступа  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **параметры**, а затем нажмите кнопку **повсеместного доступа** вкладку.  
  
3.  Нажмите кнопку **Настройка**. Появится мастера настройки повсеместного доступа.  
  
4.  На **Выбор функций повсеместного доступа для включения** выберите **удаленного веб-доступа** флажок.  
  
5.  Следуйте инструкциям для завершения работы мастера.  
  
###  <a name="BKMK_Region"></a>Изменить регион  
 Необходимо быть администратором сети, для изменения настройки региона в Windows Server Essentials.  
  
##### <a name="to-change-the-region-setting"></a>Для изменения настройки региона  
  
1.  На компьютере, подключенном к Windows Server Essentials откройте панель мониторинга.  
  
2.  Нажмите кнопку **параметры**.  
  
3.  На **Общие** щелкните раскрывающийся список в **страны или региона расположение сервера** раздела.  
  
4.  В раскрывающемся списке выберите Новый регион и нажмите кнопку **применить** принять новую настройку региона.  
  
###  <a name="BKMK_ManagePerms"></a>Управление разрешениями удаленного веб-доступа  
 При добавлении учетной записи пользователя в Windows Server Essentials новому пользователю разрешен по умолчанию для использования удаленного веб-доступа. Если вы решили не разрешить удаленный веб-доступ для учетной записи пользователя, а затем найдите пользователь должен использовать удаленный веб-доступ, можно обновить свойства s учетной записи пользователя.  
  
##### <a name="to-manage-remote-web-access-permissions-for-a-user-account"></a>Управление разрешениями удаленного веб-доступа для учетной записи пользователя  
  
1.  Выполните вход панель мониторинга и нажмите кнопку **пользователей**.  
  
2.  Выберите учетную запись пользователя, который вы хотите управлять, а затем нажмите кнопку **просмотреть свойства учетной записи** в **задачи** области.  
  
3.  В **свойства** диалоговом нажмите кнопку **повсеместного доступа** вкладку.  
  
4.  На **повсеместного доступа** выберите **разрешить удаленный веб-доступ и доступ к веб-приложениям служб** флажок, чтобы разрешить пользователю подключаться к серверу с помощью удаленного веб-доступа.  
  
5.  Нажмите кнопку **применить**, а затем нажмите кнопку **ОК**.  
  
 Дополнительные сведения см. в разделе [управление учетными записями пользователей](Manage-User-Accounts-in-Windows-Server-Essentials.md).  
  
###  <a name="BKMK_SecureRWA"></a>Безопасная удаленного веб-доступа  
 Windows Server Essentials используется сертификат безопасности помогут защитить данные, передаваемые между по и веб-браузер. При установке по Connector на компьютерах сертификат безопасности для Windows Server Essentials добавляется в список доверенных сертификатов на компьютерах. Для доступа к удаленного веб-доступа, когда они находятся за пределами офиса рекомендуется использовать портативный компьютер с установленным на нем программное обеспечение Connector.  
  
> [!WARNING]
>  Пользователи, которые используют удаленного веб-доступа из общедоступных местоположений или с других недоверенных компьютеров следует убедиться, что они выйти из системы веб-сайта, прежде чем оставлять компьютер без присмотра или по окончании сеанса.  
  
###  <a name="BKMK_ManageRWAVPN"></a>Об управлении пользователями удаленного веб-доступа и VPN  
 Можно воспользоваться VPN для подключения к Windows Server Essentials и доступа к ресурсам, которые хранятся на сервере. Это особенно полезно, если у вас есть клиентский компьютер, который настроен с учетными записями сети, которые могут использоваться для подключения к размещенному серверу Windows Server Essentials через VPN-подключение. Все вновь создаваемые пользовательские учетные записи на размещенном сервере Windows Server Essentials необходимо использовать VPN для входа клиентский компьютер в первый раз.  
  
##### <a name="to-set-vpn-and-remote-web-access-permissions-for-network-users"></a>Настройка разрешений VPN и удаленного веб-доступа для пользователей сети  
  
1.  Откройте панель мониторинга.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, необходимо предоставить разрешения на удаленный доступ к рабочему столу.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **свойства**.  
  
5.  В **< пользователя Account\ > свойства**, нажмите кнопку **повсеместного доступа** вкладку.  
  
6.  На **повсеместного доступа** вкладку, выполните следующие действия:  
  
    1.  Чтобы разрешить пользователю подключаться к серверу с помощью VPN, установите **разрешить виртуальную частную сеть (VPN)** флажок.  
  
    2.  Чтобы разрешить пользователю подключаться к серверу с помощью удаленного веб-доступа, выберите **разрешить удаленный веб-доступ и доступ к веб-приложениям служб** флажок.  
  
7.  Нажмите кнопку **применить**, а затем нажмите кнопку **ОК**.

###  <a name="BKMK_SetUpRouter"></a>Настройка маршрутизатора  
 На этом этапе Windows Server Essentials пытается автоматически настроить маршрутизатор командами UPnP. Для этого маршрутизатор должен поддерживать стандарты UPnP, и должен быть включен параметр UPnP на маршрутизаторе.  
  
> [!NOTE]
>  Конфигурация сети должна отвечать требованиям к поддерживаемым сетям для Windows Server Essentials. В сети должен быть только один маршрутизатор.  
  
 Если маршрутизатор не настроена по домена имя мастер настройки, необходимо вручную отправить порт 443. Сведения о том, как настроить перенаправление портов на маршрутизаторе см. в разделе [Настройка маршрутизатора](https://social.technet.microsoft.com/wiki/contents/articles/windows-small-business-server-2011-essentials-router-setup.aspx).

2.  Нажмите кнопку **повсеместного доступа** вкладку, а затем в **маршрутизатора** статьи, нажмите кнопку **Настройка**. Запускает задать мастер настройки маршрутизатора.  
  
3.  Следуйте инструкциям мастера, чтобы завершить настройку нового маршрутизатора.  
  
###  <a name="BKMK_NetworkLocation"></a>Посвященный определению сетевого размещения  
 Сетевое расположение — это набор сетевых параметров, применяемых Windows при подключении к сети. Параметры варьируются их можно настраивать в зависимости от типа сети, который можно использовать. Параметры сетевого расположения определяют ли иные функции (например, файл и общий доступ к принтерам, обнаружение сети и общий доступ) включен или выключен. Сетевые расположения полезны в тех случаях, когда вам нужно подключаться к разным сетям.  
  
 Например может собственной ноутбук, которым вы пользуетесь дома и на работе. В офисе, вы подключаетесь к корпоративной сети. Однако приходя Домой, вы используете свой ноутбук для воспроизведения видео и музыки с домашнего сервера. При подключении новой сети и указании типа расположения Windows назначает преднастроенный для расположения этого типа сетевой профиль. В следующий раз, при подключении к этой сети Windows распознает сеть и автоматически применяет нужные параметры. Это обеспечивает дополнительный уровень безопасности для защиты данных на компьютере и включаются только сетевые функции, необходимые в соответствующем местоположении.  
  
 Существует четыре типа сетевых расположений:  
  
-   **Домашняя сеть** выбирайте эту сеть для домашних сетей или если вы знаете и доверять пользователей и устройств в сети. Компьютеры в домашней сети могут входить в домашнюю группу. Сетевое обнаружение включено для домашних сетей, что позволяет видеть другие компьютеры и устройства в сети и другие сетевые пользователи могут видеть ваш компьютер.  
  
-   **Рабочая сеть** выберите этот тип сети для небольшого офиса или других рабочих сетей. Обнаружение сетевых ресурсов, который позволяет видеть другие компьютеры и устройства в сети и другие сетевые пользователи могут видеть ваш компьютер, она по умолчанию, но не удается создать или присоединиться к домашней группе.  
  
-   **Общедоступной сети** выбирайте эту сеть для общедоступных мест (например, кофеен и аэропортов). Это местоположение делает ваш компьютер невидимым другим компьютерам и защитить компьютер от вредоносного программного обеспечения из Интернета. Домашняя группа недоступна в общедоступных сетях, и выключить обнаружения сетевых ресурсов. Также следует выбирать этот параметр, если вы подключены непосредственно к Интернету без маршрутизатора или при наличии мобильного широкополосного подключения.  
  
-   **Домен** выбирайте эту сеть для корпоративных рабочих доменов. Этот тип сетевого местоположения контролируется сетевым администратором, и его невозможно выбрать или изменить.  
  
###  <a name="BKMK_ActiveX"></a>Включение элементов управления ActiveX служб удаленных рабочих столов  
 Элементы управления ActiveX служб удаленных рабочих столов позволяет получить доступ к домашнему или рабочему компьютеру, через Интернет с другого компьютера с помощью удаленного веб-доступа.  
  
##### <a name="to-enable-remote-desktop-services-activex-controls"></a>Чтобы включить элементы управления ActiveX служб удаленных рабочих столов  
  
1.  В Internet Explorer, нажмите кнопку **средства**, а затем нажмите кнопку **обозревателя**.  
  
2.  На **безопасности** щелкните **другой**.  
  
3.  В **ActiveX элементы и подключаемые модули** статьи, выполните следующие действия:  
  
    1.  В разделе **Загрузка подписанных элементов ActiveX**, нажмите кнопку **запрос**.  
  
    2.  В разделе **запуск элементов ActiveX и подключаемые модули**, нажмите кнопку **включить**.  
  
4.  Нажмите кнопку **ОК** два раза, чтобы принять изменения и закрыть диалоговое окно.

###  <a name="BKMK_DNOverview"></a>Обзор доменных имен  
 Доменное имя уникально идентифицирует сервер в Интернете. Доменные имена состоят, по крайней мере из двух частей: доменного имени верхнего уровня (TLD) и доменного имени второго уровня. Например в contoso.com, com верхнего и contoso является доменным именем второго уровня.  
  
 Находясь вне офиса, можно использовать имя домена для доступа к общим файлам на сервере или на компьютерах в сети. Можно также управлять сервером, за пределами офиса. Например вы регистрируете contoso.com для сервера. Находясь вне офиса, можно открыть веб-браузер на ноутбуке и ввести **contoso.com** в текстовом поле "адрес" для подключения к экземпляру удаленного веб-доступа, который настроен в Windows Server Essentials.  
  
###  <a name="BKMK_PersonalizedNames"></a>Понимание пользовательских доменных именах Майкрософт  
 Персонализированное имя домена Microsoft включает следующие компоненты:  
  
-   Пользовательское доменное имя для удаленного веб-доступа (например, *имя_хоста*. remotewebaccess.com). Имя домена связан с вашей общедоступный IP-адрес.  
  
-   Динамические DNS обновить службу протокола, удаленного веб-доступа, используя имя домена не будет прерван, если изменяется на общедоступный IP-адрес. Как правило поставщики услуг Интернета (ISP) для вашей организации s широкополосное подключение предоставляют динамические общих IP-адресов, которые можно изменить.  
  
-   Доверенный сертификат, связанный с именем домена.  
  
 Для интеграции персонализированного имени домена Microsoft с сервером, необходимо использовать учетную запись Майкрософт (ранее известной как идентификатор Windows Live ID). Если у вас учетная запись Майкрософт, можно зарегистрировать по адресу [Microsoft Hotmail](https://login.live.com/) веб-сайта.  
  
> [!IMPORTANT]
>  Windows Live позволяет использовать специальные символы в пароле учетной записи Майкрософт, которые сервер не поддерживает. При использовании персонализированного домена Microsoft убедитесь, что пароль учетной записи Microsoft содержит только поддерживаемые сервером символы. Сервер не поддерживает использование символов $, /, ' и %.  
  
###  <a name="BKMK_UseNewName"></a>Использование нового или существующего имени домена  
 Для автоматической настройки доменного имени на сервере Windows Server Essentials, необходимо использовать поставщика службы доменных имен, перечисленных в ваш мастера настройки доменного имени. Вы можете получить новое доменное имя или с помощью существующего имени домена. Выполните одно из следующих действий.  
  
-   Если вы хотите получить новое имя домена из одного домена имя поставщиков служб, перечисленных в мастере, щелкните **требуется настроить новое имя домена**.  
  
-   При наличии существующего имени домена, приобретенного у одного из поддерживаемых поставщиков служб доменных имен домена имя мастер настройки можно использовать для настройки доменного имени сервера. Нажмите кнопку **необходимо использовать имя домена уже есть**и затем введите имя домена в **Настройка домена имени** текстовое поле. Необходимо указать имя пользователя и пароль, который вы использовали для приобретения доменного имени.  
  
-   При наличии существующего имени домена, приобретенного у поставщика службы доменных имен, который не поддерживается в Windows Server Essentials, и вы хотите использовать ваш мастера настройки доменного имени для настройки имени домена для сервера, можно перенести доменное имя одному из поставщиков служб доменных имен, указанных в мастере. Нажмите кнопку **необходимо использовать имя домена уже есть**, введите имя домена в **имя домена** текстовое поле, а затем следуйте инструкциям на сайте поставщика s имя службы домена для переноса доменного имени.  
  
###  <a name="BKMK_SetUpName"></a>Настройка имени домена  
 При включении удаленного веб-доступа можно настроить имя домена Интернета для сервера.  
  
##### <a name="to-set-up-or-manage-an-internet-domain-name"></a>Для настройки или управлять имени домена Интернета  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **параметры сервера**, а затем нажмите кнопку **повсеместного доступа** вкладку.  
  
3.  В **имя домена** щелкните **Настройка**.  
  
4.  Следуйте инструкциям для завершения работы мастера. Если вы еще нет доменного имени и сертификата, мастер поможет вам найти поставщика доменных имен для приобретения доменного имени и сертификата, или можно получить персонализированное доменное имя Microsoft.  
  
###  <a name="BKMK_ChooseProvider"></a>Выберите поставщика службы доменных имен  
 Следует выбирать поставщика служб доменных имен, который поддерживает расширение имени домена. Ваш мастера настройки доменного имени содержит список удовлетворяющих требованиям поставщиков, которые можно использовать со ссылкой на каждом веб-сайте поставщика s. Нажмите кнопку **Подробнее** ссылку рядом с именем каждого поставщика s, чтобы получить сведения об услугах и ценах, предлагаемых поставщиком.  
  
> [!NOTE]
>  Некоторые поставщики служб доменных имен обслуживают крупные международные регионы, другие концентрируются на небольших рынках. По этой причине некоторые поставщики не можете предлагать веб-сайта, переведенного на нужный вам язык предпочтения.  
  
 При приобретении доменного имени, также можно приобрести и службу протоколов динамического обновления доменных имен (DNS) от поставщика службы доменных имен. Протокол динамического обновления DNS — это служба, которая позволяет всем пользователям в Интернете получить доступ к ресурсам в локальной сети в том случае, если IP-адрес этой сети постоянно меняется. Или можно приобрести статический IP-адрес из поставщика услуг Интернета (ISP) для обеспечения неизменность IP-адрес.  
  
###  <a name="BKMK_ChooseDomainName"></a>Выберите доменное имя  
 Выберите имя, уникально идентифицирующее ваш бизнес-сервер. Например если название вашей компании Contoso Ltd, вы можете Contoso для уникальной идентификации домашнего или рабочего сервера в Интернете. Если имя домена, недоступно, попробуйте другой вариант этого же имени или что-то совершенно иное.  
  
 Имя, введенное может содержать следующее:  
  
-   более 63 символов.  
  
-   Буквы (английского и любого другого алфавита), цифры или дефисы (-). Имя должно начинаться и заканчиваться цифрой или буквой.  
  
    > [!NOTE]
    >  Имена доменов не учитывается регистр.  
  
###  <a name="BKMK_Prefixes"></a>Выбор префикса доменного имени  
 Доменное имя состоит из иерархических меток.  
  
 **Расширение домена верхнего уровня** — это самая правая метка доменного имени. Например в www.contoso.com, com — это расширение имени домена верхнего уровня.  
  
 **Имя домена второго уровня** — это метка рядом с расширением имени домена верхнего уровня. Имя домена второго уровня часто создается на основе названия компании, продуктов или служб. Например в www.contoso.com, contoso — имя домена второго уровня и выбранное по названию компании Contoso Pharmaceuticals. Домен второго уровня иногда называют именем хоста, IP-адрес, связанный с ним.  
  
 **Префикс имени домена** обозначает дочерний домен. Имя дочернего домена может использоваться для идентификации услуги, устройства или регионы. Например компания Contoso Pharmaceuticals хочет предоставить удаленным пользователям возможность входа удаленный веб-доступ, но не хочет делать веб-сайт общедоступным, поэтому компания создает дочерний домен, обеспечивающий только пользователям с соответствующими разрешениями на доступ к веб-сайта. Contoso Pharmaceuticals создает дочерний remote.contoso.com домен и удаленных префикс имени домена.  
  
> [!TIP]
>  Рекомендуется использовать по умолчанию **удаленного** как префикс доменного имени.  
  
###  <a name="BKMK_Extension"></a>Выбор расширения доменного имени  
 При выборе имени домена для веб-сайта в Интернете, необходимо также указать расширение имени домена. Расширение обозначается буквами, следующими за последней точкой любого доменного имени. (Формальное название расширения — домена верхнего уровня или TLD).  
  
 Существует два основных типа расширения домена, которые можно использовать: универсальный и по коду страны.  
  
#### <a name="generic-top-level-domains"></a>Универсальные домены верхнего уровня  
 Расширения универсальных доменов состоят из трех и более букв и обычно используются организациями определенных типов.  
  
 **Примеры универсальных доменов верхнего уровня**  
  
|Расширение домена|Описание|  
|----------------------|-----------------|  
|.com|Как правило, используется коммерческими организациями, но может использоваться кем угодно.|  
|.NET|Предназначено для организаций, оказывающих услуги инфраструктуры сети.|  
|.org|Изначально использовался некоммерческими организациями и другими предприятиями, не попадающими в другую категорию универсальных доменов верхнего уровня. Могут быть использованы кем угодно.|  
|.edu|Ограниченное использование образовательными организациями.|  
  
#### <a name="country-code-top-level-domains"></a>Домены верхнего уровня по коду страны  
 Эти доменные расширения состоят из двух букв. Они предназначены для использования организациями в стране или регионе с соответствующим буквенным кодом. Некоторые домены верхнего уровня код страны для использования с предназначены гражданами соответствующей страны или региона. Другие доступны всем.  
  
 **Примеры доменов верхнего уровня с кодом страны**  
  
|Расширение домена|Описание|  
|----------------------|-----------------|  
|.CA|Для использования веб-сайтах в Канаде|  
|.CN|Для использования веб-сайтах в Китае|  
|.de|Для использования веб-сайтах в Германии|  
|. co.uk|Для использования веб-сайтах в Великобритании|  
  
 Чтобы просмотреть полный список доменов верхнего уровня, в разделе [веб-сайта Internet Assigned Numbers Authority](https://go.microsoft.com/fwlink/?LinkId=117438).  
  
#### <a name="if-a-domain-extension-is-not-available-to-select-in-the-set-up-domain-name-wizard"></a>Если расширение домена недоступны для выбора в мастера настройки доменного имени  
 При запуске мастера настройки доменного имени мастер анализирует данные системы, чтобы определить страну или регион. Затем мастер отображает только доменные расширения, которые поддерживают поддерживаются поставщиками-участниками в вашем регионе. Если требуется расширение домена отсутствует в списке, необходимо выбрать другое доменное расширение, чтобы продолжить. Выберите расширение из возвращенного мастером списка.  
  
###  <a name="BKMK_UpdateService"></a>Сведениями об обновлении службы доменных имен  
 Может потребоваться для обновления или обновить службу доменных имен, если вы приобрели имя домена, но организация не приобрела сертификата. Должен иметь сертификат для имени домена из вашего поставщика службы доменных имен.  
  
> [!NOTE]
>  Работать с вашего поставщика службы доменных имен для определения типа сертификата нужно. Сертификат может быть один из недорогих предлагаемых сертификатов. Тем не менее следует ознакомиться с документацией и возможностями выше сертификатов уровень безопасности, если они лучше соответствуют потребностям вашей организации.  
  
###  <a name="BKMK_ExportCert"></a>Об экспорте или импорте сертификата на сервере  
 Если вы хотите создать резервную копию сертификата или использовать его на другом сервере, необходимо экспортировать сертификат. Сведения об экспорте сертификатов см. в разделе [Экспорт сертификата](https://go.microsoft.com/fwlink/p/?LinkId=214362).  
  
###  <a name="BKMK_SetNameManually"></a>Настройка имени домена вручную  
 Если выбрать этот параметр, сервер не отслеживает и не сохраняет ваше имя домена, и он не предупреждает в случае проблемы с конфигурацией. Вы также можете этот параметр, если имеет место одно из следующих условий:  
  
-   Для вашей страны или региона не указаны не поставщиков имя домена партнера.  
  
-   Партнеры — поставщики доменов не поддерживают ваше расширение доменного имени.  
  
-   У вас есть имя существующего домена от поставщика доменных имен, который не является партнером, и вы не хотите переносить это имя домена для поставщика Windows Server Essentials поддерживаемых доменных имен.  
  
-   В мастере не указано нужное расширение имени домена, но оно доступно у поставщика доменных имен, который не является партнером.  
  
 Если вы решили настройки имени домена вручную, работаете с поставщиком служб доменных имен для создания записи A для вашего домена.  
  
##### <a name="to-create-an-a-record"></a>Для создания записи  
  
1.  Выберите имя узла, например remote. Это будет префикс имени домена. Префикс имени домена и имя домена определяют URL для открытия страницы входа в систему удаленного веб-доступа; например **http://remote.contoso.com**.  
  
2.  Панели домена имя службы поставщики конфигурации (обычно на их веб-страницы) создайте запись A для имени узла, выбранного на шаге 1. Убедитесь, что IP-адрес, указанный в записи IP-адрес на стороне WAN вашего маршрутизатора (на стороне сети Интернет). Обратитесь к документации маршрутизатора, чтобы найти ваш IP-адрес WAN.  
  
3.  Рекомендуется у поставщика услуг Интернета (ISP) приобрести статический IP-адрес для сети. Это гарантирует, что IP-адрес не изменится и запись DNS не окажется устаревшей.  
  
     Если нет возможности приобрести статический IP-адрес от поставщика услуг Интернета, может также потребоваться приобрести и службу протоколов динамического обновления доменных имен (DNS) от поставщика службы доменных имен или другого поставщика услуг. Протокол динамического обновления DNS — это служба, которая IP-адреса WAN вашей сети обеспечивает своевременное обновление, чтобы IP-адрес можно разрешить на ваше доменное имя даже в случае изменения IP-адрес.  
  
4.  Запросу мастера импортируйте доверенный сертификат. Если у вас доверенных сертификатов, можно получить от одного из поддерживаемых поставщиков доменных имен отображается в мастере или приобрести у доверенного поставщика по вашему выбору. Дополнительные сведения о доверенном сертификате обратитесь к поставщику доменных имен.  
  
###  <a name="BKMK_Find"></a>Поиск поставщика службы доменных имен  
  
##### <a name="to-find-the-domain-name-service-provider-for-your-domain-name"></a>Поиск поставщика службы доменных имен для имени домена.  
  
1.  Откройте веб-браузер и введите **www.internic.com** в адресной строке, чтобы перейти на домашнюю страницу Internic.  
  
2.  На домашней странице Internic щелкните **Whois**.  
  
3.  В **Whois** введите имя домена (например, contoso.com).  
  
4.  Нажмите кнопку **домена**, а затем щелкните **отправить**.  
  
5.  В результатах поиска имя вашего поставщика службы доменных имен в списке **регистратора**.

###  <a name="BKMK_CustomizeImages"></a>Индивидуальной настройке изображений для фона и эмблем  
 Этот раздел содержит сведения об изображениях, которые можно использовать для настройки удаленного веб-доступа.  
  
#### <a name="image-size"></a>Размер изображения  
 **Эмблем**  
  
 Рекомендуется использовать изображения логотипа размером 32 x 32 пиксела. Более крупные изображения сжимаются до 32 x 32, а более мелкие — растягиваются до 32 x 32, возможны искажения.  
  
 **Фоновые изображения**  
  
 Несмотря на отсутствие размерных ограничений для фоновых изображений для оптимального результата рекомендуется использовать изображения размером примерно 800х500 пикселей. Фоновое изображение размещается в центре (по горизонтали и вертикали) страницы входа в систему. Чтобы сделать текст на странице входа легко читался, центр фонового изображения должен быть светлым.  
  
#### <a name="image-file-types"></a>Типы файлов изображений  
 Для замены логотипа фона и веб-сайт по умолчанию, можно использовать следующие типы файлов изображений.  
  
-   Точечный рисунок (*.bmp, \*.dib, \*.rle)  
  
-   GIF (*.gif)  
  
-   PNG (*.png)  
  
-   JPG (*.jpg)  
  
###  <a name="BKMK_RepairRWA"></a>Восстановление удаленного веб-доступа  
 Мастер восстановления помогает выявить и устранить проблемы с маршрутизатором или именем домена. Существует два способа обнаружения проблем с помощью удаленного веб-доступа:  
  
-   В параметрах сервера, на панели мониторинга на вкладке повсеместного доступа отображается значок с красным крестом и описанием проблемы.  
  
-   Предупреждение в средстве просмотра оповещений.  
  
> [!NOTE]
>  Мастер восстановления не доступен, пока включении удаленного веб-доступа. Сведения о включении удаленного веб-доступа см. в разделе [Включение удаленного веб-доступа](Manage-Remote-Web-Access-in-Windows-Server-Essentials.md#BKMK_TurnOnRWA).  
  
##### <a name="to-repair-remote-web-access"></a>Восстановление удаленного веб-доступа  
  
1.  Войдите в систему панели мониторинга.  
  
2.  Нажмите кнопку **параметры**, а затем нажмите кнопку **повсеместного доступа** вкладку.  
  
3.  Нажмите кнопку **восстановления**. **Восстановление удаленного веб-доступа** запустится мастер.  
  
4.  Нажмите кнопку **Далее**. Мастер анализирует систему удаленного веб-доступа, находит проблему и пытается устранить проблему.  
  
5.  Если вы получаете оповещение, по завершении работы мастера, нажмите кнопку **повторить** чтобы попытаться устранить проблему еще раз. Если вы продолжите получать предупреждение, проверьте предупреждения Дополнительные сведения о проблеме и действия по устранению неполадок.
