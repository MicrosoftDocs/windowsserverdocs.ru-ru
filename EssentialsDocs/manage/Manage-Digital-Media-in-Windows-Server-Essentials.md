##  <a name="BKMK_1"></a>Обзор цифрового мультимедиа  
 Цифрового мультимедиа понимается аудио, видео и фотографий содержимое, прошедшее (цифровое сжатие). Кодирование содержимого подразумевает преобразование аудио и видео в файл цифрового мультимедиа, например файл Windows Media. После кодирования цифрового мультимедиа его легко управлять, распределенных и воспроизводить на компьютерах, а также передавать по компьютерным сетям.  
  
 Примеры типов цифрового мультимедиа: Windows Media Audio (WMA), Windows Media Video (WMV), MP3, JPEG и AVI. Сведения о поддерживаемых проигрывателем Windows Media типах цифрового мультимедиа см. в разделе [типы файлов, поддерживаемые проигрывателем Windows Media](https://support.microsoft.com/kb/316992).  
  
### <a name="why-would-i-want-to-stream-my-digital-media"></a>Зачем выполнять потоковую передачу цифрового мультимедиа?  
 Многие хранят музыку, видео и изображения в общих папках в Windows Server Essentials. Могут возникнуть ситуации, когда нужно сделать следующее:  
  
-   **Смотреть видео**. Сервер можно использовать для хранения и потоковой передачи крупных коллекций видео и записанных ТВ-шоу на компьютер или другие воспроизведения устройств в сети. Можно осуществлять потоковую передачу видео на Xbox 360 или на компьютер с помощью проигрывателя Windows Media.  
  
-   **Воспроизведение музыки**. При включении общего доступа к мультимедиа для **Музыка** общей папки осуществлять доступ к музыке с устройств, поддерживающих Windows Media Connect. Необходимо включить или не настроить все учетные записи пользователей для потоковой передачи из **Музыка** общей папки, после включения общего доступа.  
  
-   **Представить слайдшоу из фото**. Вы можете хранить цифровые фотографии в **фотографии** общей папки на сервере, а затем получить к ним доступ с любого компьютера или с Xbox 360, подключенного к ТВ дома или в офисе. После можно просматривать слайдшоу из фото, а экран ТВ превращается в огромную рамку для фотографий.  
  
###  <a name="BKMK_1.5"></a>Общий доступ к защищенному от копирования мультимедиа  
  Windows Server Essentials не поддерживает общий доступ к защищенному от копирования мультимедиа. Сюда относится музыка, приобретенная через музыкальные онлайн-магазины.  
  
 Copy-protected мультимедиа можно воспроизвести только на компьютере или устройстве, использованном для его приобретения. Защита от копирования не позволяет воспроизводить мультимедиа более чем на одном компьютере или устройстве, даже если копировать мультимедиа на сервер и воспроизводить его оттуда. Тем не менее можно сохранить защищенное от копирования мультимедиа на Windows Server Essentials и продолжить воспроизведение мультимедиа на компьютере или устройстве, использованном для его приобретения.

##  <a name="BKMK_3"></a>Как работает потоковая передача мультимедиа  
 Функция в Windows Server Essentials потоковой передачи мультимедиа делает возможным для компьютеров в сети, а некоторые устройства сетевой мультимедийный для воспроизведения файлов мультимедиа, которые хранятся на сервере.  
  
 При включении сервера мультимедиа содержимое, которое предоставляется общий доступ в библиотеках мультимедиа, будут доступны для воспроизведения на устройствах в вашей сети, которые могут получать потоковое мультимедиа с сервера. Можно осуществлять потоковую передачу большинства типов файлов цифрового мультимедиа. Ниже перечислены некоторые наиболее распространенные типы файлов, которые вы можете выполнять потоковую передачу.  
  
-   Форматы Windows Media (.asf, .wma, .wmv, .wm)  
  
-   Аудио Visual чередования (.avi)  
  
-   Перемещение изображения экспертов группы (.mpeg, .mpg, .mp3)  
  
-   Звук для Windows (.wav)  
  
-   (.cda) звуковую дорожку компакт-диска  
  
 Для воспроизведения файла достаточно найти песню, видео или изображение в общей папке, дважды щелкните файл, и содержимое будет передаваться потоком с сервера на компьютер и воспроизведения. Сведения о поиске и воспроизведении файлов цифрового мультимедиа, которые хранятся на сервере см. в разделе [Воспроизведение цифрового мультимедиа](../use/Play-Digital-Media-in-Windows-Server-Essentials.md).  
  
 Для потоковой передачи мультимедиа, потребуется следующее оборудование:  
  
-   Проводная или беспроводная частной сети  
  
-   Другой компьютер в сети или на устройство, известное как получатель цифрового мультимедиа (иногда называется сетевым мультимедийным проигрывателем). Приемники цифрового мультимедиа — это аппаратные устройства, подключенные к проводной или беспроводной сети, которые можно контролировать с помощью компьютера? даже если компьютер находится в другой комнате.  
  
 Дополнительные сведения см. в разделе [Включение и отключение потоковой передачи мультимедиа](Manage-Digital-Media-in-Windows-Server-Essentials.md#BKMK_4).  
  
##  <a name="BKMK_4"></a>Включение и отключение потоковой передачи мультимедиа  
 Вы можете поделиться музыки, видео и изображениям из Windows Server Essentials, потоковую передачу файлов на любой поддерживаемый приемник цифровых мультимедиа (DMR), таких как компьютеры, мобильные телефоны, телевизор, приемник цифрового мультимедиа, расширением для Windows Media Center (включая Xbox 360) и другие персональные электронные устройства.  
  
 Текущий список цифровых мультимедийных устройств, совместимых с Windows Server Essentials см. в разделе [центре совместимости Windows](https://www.microsoft.com/windows/compatibility/CompatCenter/Home).  
  
### <a name="enabling-media-sharing"></a>Включение общего доступа к мультимедиа  
 Чтобы поделиться носителя, который хранится в Windows Server Essentials, необходимо включить потоковую передачу мультимедиа. Потоковая передача мультимедиа отключена по умолчанию.  
  
####  <a name="BKMK_2.5"></a>Включение и отключение потоковой передачи мультимедиа  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  Нажмите кнопку **параметры**, нажмите кнопку **мультимедиа**, а затем выполните одно из следующих действий:  
  
    -   Нажмите кнопку **включить** запуск общий доступ ко всем файлам, которые хранятся в библиотеке мультимедиа с сервера.  
  
    -   Нажмите кнопку **отключить** Чтобы прекратить общий доступ ко всем файлам, которые хранятся в библиотеке мультимедиа с сервера.  
  
3.  Общий доступ к другим папкам в библиотеке мультимедиа, щелкните **Настройка**, а затем выберите **Да** для каждой общей папки, который вы хотите включить в библиотеку мультимедиа.  
  
4.  Нажмите кнопку **ОК** для сохранения изменений.  
  
 Сведения о поддерживаемых проигрывателем Windows Media типах цифрового мультимедиа см. в разделе [типы файлов, поддерживаемые проигрывателем Windows Media](https://support.microsoft.com/kb/316992).  
  
 Дополнительные сведения см. в разделе [разрешить или ограничить доступ к библиотеке мультимедиа на сервере](Manage-Digital-Media-in-Windows-Server-Essentials.md#BKMK_6).  
  
##  <a name="BKMK_5"></a>Добавление файлов цифрового мультимедиа на сервер  
 Администратор сервера может добавлять цифровое мультимедиа в общие папки в библиотеке мультимедиа, путем доступа к серверу напрямую или используя сайт удаленного веб-доступа для входа на панели мониторинга. Другие пользователи могут добавлять файлы мультимедиа на сервер с помощью **общих папок** подключения на панели запуска с помощью сайта удаленного веб-доступа или с помощью приложения My Server для Windows Phone. Сведения о воспроизведении мультимедиа см. в разделе [Воспроизведение цифрового мультимедиа](../use/Play-Digital-Media-in-Windows-Server-Essentials.md).  
  
> [!NOTE]
>  Можно также отправить файлы мультимедиа на сервер с помощью приложения «Мой сервер» для Windows Phone. Можно загрузить приложение My Server из [магазина Windows Phone](http://www.windowsphone.com/store/app/my-server/6c2f98d5-6fcf-4e1d-b8b1-cde62ea1a94a). Дополнительные сведения о приложении My Server для Windows phone см. в записи блога [приложение My Server для телефона для Windows Server Essentials](http://blogs.technet.com/b/sbs/archive/2012/09/18/my-server-phone-app-for-windows-server-2012-essentials.aspx).  
  
#### <a name="to-add-digital-media-files-to-shared-folders-on-the-server"></a>Добавление файлов цифрового мультимедиа в общие папки на сервере  
  
1.  Выполнить вход на сервер, используйте один из следующих методов:  
  
    1.  Сведения о входе через удаленный веб-доступ см. в разделе [войдите в систему удаленного веб-доступа](../use/Use-Remote-Web-Access-in-Windows-Server-Essentials.md#BKMK_1).

2.  Найдите и щелкните папку добавляемого мультимедиа.  
  
3.  Скопируйте и вставьте или перетащите-файлы мультимедиа, которые вы хотите добавить в соответствующую общую папку на сервере.  
  
##  <a name="BKMK_6"></a>Предоставление и Лишение доступа к библиотеке мультимедиа на сервере  
  
-   При включении общего доступа к мультимедиа создается 4 стандартные папки: музыка, фотографии, видео и записанные Телепередачи. Если любой из этих папок уже существует на сервере, существующая папка повторно используется в качестве общей папки для общего доступа к мультимедиа. Сохраняются все существующие папки s мультимедиа содержимое и пользовательские разрешения, и это предоставляется всем сетевым пользователям.  
  
-   Прежде чем включить общий доступ к библиотеке мультимедиа для общей папки, вы знаете, что общий доступ к библиотеке мультимедиа обходит любые типы доступа учетных записей пользователей, настраиваемые для общей папки. Например, предположим, что включить мультимедиа общий доступ к библиотеке для **фотографии** общей папки, а также задавать **фотографии** общую папку, чтобы **нет доступа** для пользователя учетной записи BOBBY. BOBBY по-прежнему можно осуществлять потоковую передачу любого цифрового мультимедиа из **видео** общей папки на любой поддерживаемый мультимедийный проигрыватель или DMR. Если у вас есть цифрового мультимедиа, вы не хотите выполнять потоковую передачу таким образом, сохраните файлы в папку, которая не поддерживает библиотеки общий доступ к мультимедиа включен.  
  
-   Если включить общий доступ к библиотеке мультимедиа для общей папки, любой поддерживаемый мультимедийный проигрыватель или DMR, имеющий доступ к вашей сети Windows Server Essentials можно также получить доступ к мультимедиа в этой общей папке. Например если у вас есть беспроводной сети и не обеспечена его, любой человек в зоне действия беспроводной сети осуществлять доступ к цифровому мультимедиа в этой папке. Прежде чем включить общий доступ к библиотеке мультимедиа, убедитесь в защищенности беспроводной сети. Дополнительные сведения см. в документации для вашей беспроводной точке доступа.  
  
##  <a name="BKMK_8"></a>Переименование библиотеки мультимедиа  
 Имя библиотеки мультимедиа по умолчанию **сервер цифрового мультимедиа**. Оно создается при включении потоковой передачи мультимедиа в Windows Server Essentials. Дополнительные сведения о включении потоковой передачи мультимедиа см. в разделе [на включение и отключение потоковой передачи мультимедиа](Manage-Digital-Media-in-Windows-Server-Essentials.md#BKMK_2.5). В любое время можно изменить имя библиотеки мультимедиа с помощью панели мониторинга сервера.  
  
#### <a name="to-rename-the-media-library"></a>Переименование библиотеки мультимедиа  
  
1.  Откройте панель мониторинга сервера. Чтобы открыть панель мониторинга с панели запуска, нажмите кнопку **мониторинга**, введите пароль для сервера и нажмите кнопку со стрелкой для входа в систему.  
  
2.  На панели мониторинга сервера, щелкните **параметры**.  
  
3.  На **параметры** щелкните **мультимедиа** вкладку.  
  
4.  В **библиотеки мультимедиа** раздел **параметры мультимедиа** щелкните имя библиотеки мультимедиа.  
  
5.  В **изменить имя библиотеки мультимедиа** диалоговом введите новое имя библиотеки мультимедиа и нажмите кнопку **ОК**.  
  
##  <a name="BKMK_9"></a>Прекращение общего доступа к мультимедиа  
 Администратор сервера может приостановить общий доступ к цифровому мультимедиа, который хранится в общих папках на сервере под управлением Windows Server Essentials.  
  
#### <a name="to-stop-sharing-media-in-shared-folders"></a>Прекращение общего доступа к мультимедиа в общих папках  
  
1.  Откройте панель мониторинга сервера.  
  
2.  На панели мониторинга **Главная** щелкните **установки**, нажмите кнопку **Настройка сервера мультимедиа**, а затем нажмите кнопку **щелкните, чтобы настроить сервер мультимедиа**.  
  
3.  На **мультимедиа** странице параметров вы можете выбрать один из следующих вариантов:  
  
    -   Нажмите кнопку **отключить** Чтобы прекратить общий доступ ко всем файлам, которые хранятся на сервере.  
  
    -   Нажмите кнопку **Настройка**, а затем выберите **нет** для папок, которые вы хотите прекратить общий доступ.  
  
4.  Нажмите кнопку **применить** или **ОК** для сохранения изменений.  
  
##  <a name="BKMK_10"></a>Включение мультимедийных устройств, использующих протокол Server Message Block (SMB) для доступа к общим файлам на сервере  
 Устройства, использующие Server Message Block (SMB) для сетевыми файлами и организации общего доступа вместо DLNA (для потоковой передачи) требуют активации гостевой учетной записи. Это позволяет любому устройству или пользователю в сети просмотреть содержимое общих папок без проверки подлинности.  
  
> [!CAUTION]
>  При включении учетной записи гостя, любой пользователь может доступа к общим ресурсам на сервере по умолчанию.  
  
##  <a name="BKMK_CommonProcessors"></a>Стандартные программы обработки и профили видео, которые они поддерживают  
 Для потоковой передачи мультимедиа с сервера Windows Server Essentials можно воспользоваться компьютером под управлением операционной системы Windows 7 или Windows 8 или другими сетевыми устройствами (например мультимедийными проигрывателями) или расширителями Media Center (например, Xbox 360). Находясь вдали от своей сети, используйте проигрывателем удаленного веб-доступа к мультимедиа для воспроизведения файлов, которые хранятся на сервере.  
  
 Вам потребуется скорость передачи данных между 200 Кбит/с до 10 Мбит/с. Необходимо использовать мультимедийные форматы, которые ваши компьютеры и устройства можно распознать и воспроизвести. Не все устройства поддерживают те же мультимедийные форматы, поэтому необходимо обеспечить возможность на компьютер и устройства воспроизведения файлов мультимедиа, которые у вас есть.  
  
  Windows Server Essentials содержит поддержки перекодирования, которая определяет емкость компьютера или устройства, а затем динамически преобразует неподдерживаемый видеофайл в поддерживаемый. Как правило если проигрыватель Windows Media 12 может воспроизвести содержимое на компьютере под управлением Windows 7 или Windows 8, содержимое на сервере будет воспроизводиться на устройства, подключенного к сети.  
  
 Формат и скорость передачи данных выбранные для перекодирования очень сильно зависят от производительности серверного процессора. Производительность процессора определяется как часть индекса производительности Windows. Для определения производительности сервера, выполните одно из следующих действий.  
  
-   На сетевом компьютере под управлением Windows 7 или Windows 8 с таким же процессором, что сервер, перейдите в меню **панели управления**, нажмите кнопку **счетчики и средства производительности**и затем просмотрите сведения на **скорость и повысить производительность компьютера s** страницы.  
  
-   Обратитесь к изготовителю процессора.  
  
 Для оптимального взаимодействия с пользователем выберите подходящее для серверного процессора качество разрешение потокового видео. Сервер автоматически скорректирует скорость к одному из этих параметров:  
  
-   **Низкая** Если скорость процессора — ниже 3,6.  
  
-   **Средний** Если скорость процессора — от 3,6 до 4,2.  
  
-   **Высокая** Если скорость процессора — от 4,2 до 6,0.  
  
-   **Лучшие** Если скорость процессора — более 6.0.  
  
 Если выбрано разрешение потокового видео, требующее большей вычислительной мощности, нежели на сервере, могут образовываться буферы и прерывания во время потоковой передачи мультимедиа с сервера.  
  
> [!NOTE]
>  Для потоковой передачи высокой четкости видео посредством удаленного веб-доступа потребуется процессор со скоростью не менее 6.0.  
  
##  <a name="BKMK_KnownIssues"></a>Известные проблемы с типы файлов мультимедиа  
 Функция потоковой передачи мультимедиа в системе удаленного веб-доступа использует службу Windows Media Player 12 сети общего доступа. Удаленный веб-доступа к потоковой передачи мультимедиа поддерживает звук, видео и типы файлов изображений, которые поддерживаются Windows Media Player 12 и Silverlight 4.  
  
 В следующей таблице перечислены типы файлов (форматы), поддерживаемые потоковой передачи мультимедиа удаленного веб-доступа. Если media типы файлов на сервере, не включенные в таблице, их невозможно передать через потоковой передачи мультимедиа удаленного веб-доступа.  
  
|Тип файла|Расширение имени файла|  
|---------------|-------------------------|  
|Файлы 3GPP|.3GP, .3gpp, .3g2 и .3gp2|  
|Звуковые файлы звуковых данных транспортного потока (ADTS)|.adts и .adt|  
|Звуковые файлы автоматического обновления|.AU и .snd|  
|Звуковые файлы звука Interchange формат файла (AIFF)|.AIF, .aiff и .aifc|  
|Файлы AVCHD|.m2ts, .mts и .m2t|  
|Компакт-диск с аудио|.CDA|  
|DVD с видео|.vob|  
|Файлы изображений JPEG|.jpg|  
|Звуковые файлы MP3|MP3 и M3U|  
|Файлы MPEG видео|.MPEG, .mpg, .m1v, .mpa, .mpe, .mp4, .mp4v, .m4v, .m4a и .mov|  
|Файлы Windows аудио и видео|.AVI и .wav|  
|Файлы Windows Media звука и видео|.ASF, .asx, .wax, .wm, .wma, .wmd, .wmp, .wmv, .wmx, .wpl и WVX|  
  
> [!NOTE]
>  Если не удается использовать тип файла, перечисленный в этой таблице, файл может зашифровать с помощью кодека, не поддерживается проигрывателем Windows Media.  
  
 Дополнительные сведения о поддерживаемых файловых форматах см. в разделе [типы файлов, поддерживаемые проигрывателем Windows Media](https://go.microsoft.com/fwlink/p/?LinkID=196118) и [поддерживаемые форматы мультимедиа, протоколы и поля журнала](https://go.microsoft.com/fwlink/p/?LinkId=203339) для Silverlight.
