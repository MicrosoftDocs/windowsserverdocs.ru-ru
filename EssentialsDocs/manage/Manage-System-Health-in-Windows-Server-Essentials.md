# <a name="manage-system-health-in-windows-server-essentials"></a>Управление работоспособностью системы в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials
 
 В этом разделе описывается, как просматривать и реагировать на всех оповещений в сети с помощью панели мониторинга.  
  
> [!NOTE]
>  В Windows Server Essentials и Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience оповещения о работоспособности для сервера и клиентских компьютеров в сети больше не отображаются в средстве просмотра оповещений, но вместо этого можно просмотреть на **отчеты о работоспособности** вкладке **Главная** страницы.  
  
 Windows Server Essentials активно отслеживает все компьютеры, который подключен к серверу и оповещает администратора о проблемах, связанных с работоспособностью системы s критическими обновлениями, отсутствием защиты от вредоносных программ, устаревшими определениями вирусов на клиентских компьютерах и других важных аспектах, требующих вмешательства. Эти проблемы отображаются в виде оповещений в средстве просмотра оповещений, которое может быть запущено на сервере мониторинга s или s запуска в Windows Server Essentials, или на клиентском компьютере **отчеты о работоспособности** вкладку в Windows Server Essentials. По умолчанию оповещения обновляются каждые 30 минут, но можно проверить сеть на наличие оповещений в любое время, нажав кнопку **обновить** в средстве просмотра оповещений или на **отчеты о работоспособности** вкладку.  
  
 Следующие разделы помогут понять, просмотра и обработка оповещений в средстве просмотра оповещений и также предоставляют инструкции по настройке сервера, чтобы получать уведомления об оповещениях по электронной почте:  
  
-   [Отчета о работоспособности надстройки](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_AddIn)  
  
-   [Просмотр оповещений с помощью средства просмотра оповещений](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_View)  
  
-   [Упорядочение оповещений в средстве просмотра оповещений](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Organize)  
  
-   [Работа с оповещениями](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Respond)  
  
-   [Настройка уведомлений электронной почты для оповещений](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Email)  
  
-   [Возможные оповещения для компьютера](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Potential)  
  
##  <a name="BKMK_AddIn"></a>Отчета о работоспособности надстройки  
 Надстройка отчета о работоспособности для Windows Server Essentials предоставляет общую информацию о сети Windows Server Essentials и позволяет распространить эту информацию другим пользователям. Эти сведения можно просматривать на **отчеты** панели мониторинга. С помощью **отчеты** вкладку, можно сделать следующее:  
  
-   [Создание отчета по требованию или по расписанию](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Generate)  
  
-   [Настройка содержимого отчета](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Customize)  
  
-   [По электронной почте отчета](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_emailreport)  
  
> [!NOTE]
>  **Windows Server Essentials:** можно загрузить надстройку отчета о работоспособности для Windows Server Essentials из [центра загрузки Майкрософт](https://go.microsoft.com/fwlink/p/?LinkId=266342).  
>   
>  **Windows Server Essentials:** по умолчанию надстройка отчета о работоспособности интегрирована с Windows Server Essentials или Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience и отчеты о работоспособности отображаются на **отчеты о работоспособности** мониторинга s **Главная** страницы.  
  
###  <a name="BKMK_Generate"></a>Создание отчета по требованию или по расписанию  
 После установки надстройки отчета о работоспособности и перезапуска панели мониторинга добавляется новая вкладка **отчеты** добавляется в панель мониторинга. Можно создать отчет о работоспособности по требованию в любое время, щелкнув **создать отчет о работоспособности** задач на **отчеты** вкладку.  
  
 После создания отчета о работоспособности, в области списка, определяемый Дата и время создания отчета создается новый элемент. Чтобы открыть элемент, дважды щелкните его в области списка, или выберите его и нажмите кнопку **откройте отчет о работоспособности** на панели задач. Отчет отображается в новом окне в формате HTML.  
  
 Помимо создания отчета вручную, можно также настроить отчет создаваться автоматически по расписанию ежедневно или каждый час. Для этого в области задач щелкните **настроить параметры отчета о работоспособности**, а затем нажмите кнопку **расписание и электронная почта** вкладку. **Расписание** выключена по умолчанию и включите его, выбрав **создать отчет о работоспособности в его запланированное время** флажок.  
  
###  <a name="BKMK_Customize"></a>Настройка содержимого отчета  
 Отчет о работоспособности содержит следующее:  
  
-   **Критические оповещения и предупреждения** Это согласуется с критическим оповещениям и предупреждениям, отображаемым в средстве просмотра оповещений на панели мониторинга. Информационные оповещения в отчет о работоспособности не включаются.  
  
-   **Критические ошибки в журналах событий** выполняется проверка журналов приложений и служб, а ошибки, зарегистрированные за последние 24 часа, отображаются в **сведений** отчета.  
  
-   **Резервное копирование сервера** сведения о последней архивации сервера отображаются в **сведений** отчета.  
  
-   **Автоматический запуск службы не выполняется** во время создания отчета, если служба автоматического запуска не работает, сведения об этой службе отображаются в **сведений** отчета.  
  
-   **Обновления** можно просмотреть состояние обновления сервера и клиентских компьютеров в **сведений** раздела.  
  
-   **Хранилище** список дисков и их емкость представлены в **сведений** раздела.  
  
 В отчете о работоспособности сначала просмотрите **Сводка**и для этих элементов с красным значком ошибки или желтым значком предупреждения, выберите **сведений** ссылку в той же строке, чтобы просмотреть сведения об элементе.  
  
 Если вас не интересуют некоторые точки данных, которые включены в отчет по умолчанию, можно настроить содержимое отчета, щелкнув **настроить параметры отчета о работоспособности** в панели задач, а затем щелкнув **содержимого**вкладку снимите флажки для содержимого, которое требуется отображать в отчете Дон t. Например, если у вас есть собственный план архивации сервера и Дон t хотите видеть предупреждения об архивации сервера, вам может исключить архивы сервера из отчета, сняв **архивации сервера** флажок.  
  
###  <a name="BKMK_emailreport"></a>По электронной почте отчета  
 При необходимости войдите в панель мониторинга для чтения отчетов по-прежнему доставляет неудобства некоторым администраторам, особенно в том случае, если они имеют более одного сервера для управления. Возможность использования электронной почты отключен, после создания отчета, сообщение электронной почты будут отправляться список указанные адреса электронной почты с содержимым отчета. Администратор может легко просмотреть этот отчет с любого устройства или из любого клиентского приложения и убедитесь, что сервер работает в оптимальном режиме.  
  
 В **настроить параметры отчетов о работоспособности** диалоговом после включения электронной почты, измените параметры SMTP и укажите список получателей сообщения электронной почты, можно заметить, что в области задач отображается новая задача: **отчет о работоспособности по электронной почте,**. Дополнительные сведения о параметрах SMTP см. в разделе [Настройка уведомлений электронной почты для оповещений](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Email).  
  
 Можно выбрать существующий отчет и затем нажмите кнопку **отчет о работоспособности по электронной почте,**. Можно также создать новый отчет и обеспечить его автоматическую отправку в свой почтовый ящик. Если вы настроили расписание для автоматического создания отчета, отчет будет автоматически доставляться в папку "Входящие" после создается каждый день (или каждый час) по расписанию.  
  
##  <a name="BKMK_View"></a>Просмотр оповещений с помощью средства просмотра оповещений  
 В этом разделе описывается использование панели мониторинга или панели запуска для открытия средства просмотра оповещений для просмотра состояния работоспособности всех компьютеров в сети сервера.  
  
#### <a name="to-open-the-alert-viewer-by-using-the-dashboard"></a>Чтобы открыть средство просмотра оповещений с помощью панели мониторинга  
  
1.  Откройте панель мониторинга.  
  
2.  В области навигации щелкните любой из отображаемых значков оповещений (критическое, предупреждение или информационное). Откроется средство просмотра оповещений.  
  
#### <a name="to-open-the-alert-viewer-from-the-launchpad"></a>Чтобы открыть средство просмотра оповещений из панели запуска  
  
1.  Компьютер, подключенный к серверу откройте панель запуска. При получении запроса выполните вход в панель запуска с помощью имени пользователя и пароля.  
  
2.  Щелкните любой из отображаемых значков оповещений (критические, предупреждений и информационных) в нижней части панели запуска, чтобы открыть средство просмотра оповещений, а затем выполните инструкции в области сведений щелкните средство просмотра оповещений для разрешения предупреждения.  
  
##  <a name="BKMK_Organize"></a>Упорядочение оповещений в средстве просмотра оповещений  
 Вы можете упорядочить оповещения в средстве просмотра оповещений и отобразить их по степени серьезности (критическое, предупреждение или информационное) или на основе имени компьютера.  
  
#### <a name="to-organize-alerts-in-the-alert-viewer"></a>Упорядочение оповещений в средстве просмотра оповещений  
  
1.  Откройте панель мониторинга.  
  
2.  В области навигации выберите любой из отображаемых значков оповещений (критическое, предупреждение или информационное). Откроется средство просмотра оповещений.  
  
3.  Разверните **упорядочить** раскрывающегося списка, а затем выполните одно из следующих:  
  
    1.  Выберите **фильтровать по компьютерам**и щелкните имя компьютера, для которого требуется просмотреть оповещения. При этом отображаются оповещения в средстве просмотра оповещений только для выбранного компьютера.  
  
    2.  Выберите **фильтровать по типам оповещений**и выберите тип оповещений (критическое, предупреждение или информационное) для которой вы хотите просмотреть оповещения. При этом в средстве просмотра оповещения отображаются только оповещения выбранного типа.  
  
##  <a name="BKMK_Respond"></a>Работа с оповещениями

-   [Разрешение оповещения](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_Resolve)  
  
-   [Игнорирование оповещения](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_3)  
  
-   [Включение оповещения](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_5)  
  
-   [Удаление оповещения](Manage-System-Health-in-Windows-Server-Essentials.md#BKMK_4)  
  
###  <a name="BKMK_Resolve"></a>Разрешение оповещения  
 Следуйте инструкциям в средстве просмотра оповещений для разрешения оповещения. После разрешения оповещения оно по-прежнему отображается в средстве просмотра оповещений до его обновления.  
  
###  <a name="BKMK_3"></a>Игнорирование оповещения  
 Вы можете игнорировать оповещение, если вы предпочитаете отреагировать на него позднее. Когда оповещение игнорируется, оно по-прежнему указывается в средстве просмотра оповещений и, но отключено и недоступно. Игнорируемое оповещение не включается в общую оценку работоспособности компьютера. Чтобы обработать игнорируемое оповещение, необходимо сначала включить его.  
  
##### <a name="to-ignore-an-alert"></a>Пропуск оповещения  
  
1.  Компьютер, подключенный к серверу Windows Server Essentials откройте панель запуска.  
  
2.  На панели запуска щелкните любой из отображаемых значков оповещений (критические, предупреждений и информационных). Откроется средство просмотра оповещений.  
  
3.  В средстве просмотра оповещений выберите оповещение, которое требуется игнорировать, а затем в **задачи** щелкните **Игнорировать оповещение**.  
  
 Для ответа на отключенное оповещение необходимо сначала включить его.  
  
###  <a name="BKMK_5"></a>Включение оповещения  
 Можно включить оповещение, которое вы решили игнорировать ранее. После включения оповещения можно разрешить его или удалить его по мере необходимости. Предупреждение отображается как отключенное помеченное игнорируются. При включении ранее отключенного оповещения оно становится активным и снова включается в общую оценку работоспособности компьютеров.  
  
##### <a name="to-enable-an-alert"></a>Включение оповещения  
  
1.  Компьютер, подключенный к серверу откройте панель запуска.  
  
2.  На панели запуска щелкните любой из отображаемых значков оповещений (критические предупреждения и информационные), чтобы открыть средство просмотра оповещений.  
  
3.  В средстве просмотра оповещения, щелкните правой кнопкой мыши оповещение, которое вы хотите включить, а затем нажмите кнопку **Включить оповещение**.  
  
###  <a name="BKMK_4"></a>Удаление оповещения  
 Если вы не хотите разрешать или игнорировать оповещение можно удалить. Можно использовать средство просмотра оповещений на панели запуска для удаления оповещения, созданные для компьютера. Если удалить предупреждение и сервер обнаруживает проблему, снова во время следующего цикла оценки работоспособности сети, он создает новые оповещения.  
  
##### <a name="to-delete-an-alert"></a>Удаление оповещения  
  
1.  Компьютер, подключенный к серверу откройте панель запуска.  
  
2.  На панели запуска щелкните любой из отображаемых значков оповещений (критические предупреждения и информационные), чтобы открыть средство просмотра оповещений.  
  
3.  В средстве просмотра оповещения, щелкните правой кнопкой мыши оповещение, которое вы хотите удалить, а затем щелкните **удалить оповещение**.  
  
##  <a name="BKMK_Email"></a>Настройка уведомлений электронной почты для оповещений  
 Можно настроить сервер на отправку уведомлений по электронной почте в случае возникновения оповещений. Уведомления по электронной почте для таких оповещений содержат сведения о проблемах сети и мерах по их, аналогичны информацию, которая отображается в средстве просмотра оповещений. Некоторые оценки работоспособности сети выполняются программно.  
  
 При настройке сервера для отправки уведомлений об оповещениях по электронной почте для оповещений, которые были обнаружены во время оценки работоспособности сети отправляется уведомление по электронной почте. Однако в сообщении электронной почты отражены не все оповещения, зарегистрированные в средстве просмотра оповещений.  
  
 Каждые 30 минут, выполняется задача оценки оповещений электронной почты на сервере, которая проверяет сеть на наличие оповещений. Уведомление по электронной почте отправляется ли оповещение, заданной для происходит уведомление по электронной почте. Если оповещение по-прежнему активно в следующем цикле оценки, чтобы предотвратить переполнение почтового ящика, второе сообщение не отправляется. Тем не менее, при обнаружении нового оповещения в будущем цикле оценки оповещений по электронной почте уведомление будет отправлено, включающее в себя как новые, так и предыдущее оповещение.  
  
###  <a name="BKMK_list"></a>Оповещений, отправляемых по электронной почте  
 Следующие оповещения в средстве просмотра оповещений вызывают уведомления по электронной почте при настройке сервера для отправки уведомлений по электронной почте для оповещений:  
  
-   В архиве клиентского компьютера имеются ошибки.  
  
-   Сервер перезагружен.  
  
-   Один или несколько служб, не запущены.  
  
-   Служба хранилища Windows Server не запущена.  
  
-   Период ознакомления закончен.  
  
-   Активируйте сейчас.  
  
-   Исходный сервер завершает работу.  
  
-   Результаты проверки BPA содержат ошибки.  
  
-   Результаты проверки BPA содержат предупреждения.

###  <a name="BKMK_SMTP"></a>Настройка SMTP на сервере для отправки уведомлений об оповещениях по электронной почте в Windows Server Essentials  
 В этом разделе описывается настройка сервера на отправку электронных уведомлений для оповещений.  
  
> [!NOTE]
>  Можно загрузить надстройку отчета о работоспособности для Windows Server Essentials из [центра загрузки Майкрософт](https://go.microsoft.com/fwlink/p/?LinkId=266342).  
  
##### <a name="to-set-up-email-notification-for-alerts"></a>Чтобы настроить уведомления по электронной почте для оповещений  
  
1.  Из **мониторинга**, откройте **просмотра оповещений**.  
  
2.  В **средство просмотра оповещений**, нажмите кнопку **Настройка уведомлений электронной почты для оповещений**.  
  
3.  В **Настройка электронных уведомлений для оповещений** окно, нажмите кнопку **включить**.  
  
4.  В **параметры SMTP** окна, выполните следующие действия:  
  
    1.  Для **адрес электронной почты отправителя**, введите адрес электронной почты, которого вы хотите отправлять электронные уведомления. Этот адрес электронной почты будет отображаться как адрес отправителя s в уведомлениях об оповещениях.  
  
    2.  Для **имя SMTP-сервера**в **адрес электронной почты отправителя** текст введите имя SMTP-сервера, указанное в шаге 4а. (См. в таблице 1 список несколько имен SMTP-сервера).  
  
    3.  Для **порта SMTP**, введите номер порта, используемый SMTP-сервер для отправки и получения электронной почты. (См. в таблице 1 номера портов, используемые некоторыми SMTP-серверов).  
  
    4.  Выберите **данного сервера требуется безопасное подключение (SSL)** Если SMTP-сервер использует протокол SSL (см. таблицу 1).  
  
    5.  Выберите **этот сервер требует проверки подлинности** Если SMTP-сервер требует указания пользователя имя и пароль (см. таблицу 1). Если флажок установлен, введите имя пользователя и пароль для адреса электронной почты, введенного в **адрес электронной почты отправителя** в шаге 4а, а затем щелкните **ОК**.  
  
    > [!NOTE]
    >  Сведения о имя SMTP-сервера, номер порта и использовании протокола SSL можно получить от поставщика услуг Интернета.  
  
     **В таблице 1** имен примеры SMTP-сервера, проверка подлинности и требования шифрования SSL и номера портов  
  
    |SMTP-сервера|Требуется протокол SSL|Требуется проверка подлинности|Номер порта|Учетная запись имя и имя для входа|  
    |-----------------|------------------|-----------------------------|-----------------|------------------------------|  
    |SMTP.Gmail.com|Да|Да|587|Предоставить полные адреса электронной почты с доменным именем и паролем для проверки подлинности.|  
    |SMTP.Live.com|Да|Да|587|Предоставить полные адреса электронной почты с доменным именем и паролем для проверки подлинности.|  
    |SMTP.comcast.NET|Да|Нет|587|Предоставить полные адреса электронной почты с доменным именем и паролем для проверки подлинности.|  
    |SMTP.Mail.Yahoo.com|Нет|Да|25|Укажите только адрес электронной почты без доменного имени для имени пользователя.|  
  
5.  В **Настройка электронных уведомлений для оповещений**, для **получателям сообщений электронной почты**, введите адреса электронной почты лиц, которые вы хотите получать уведомления об оповещениях по электронной почте. Обязательно отделяйте каждый адрес электронной почты точкой с запятой (;).  
  
6.  Чтобы проверить настройки SMTP-сервера параметры правильно для отправки уведомлений по электронной почте для оповещений, щелкните **применить и отправить по электронной почте**.  
  
    > [!NOTE]
    >  При нажатии кнопки **применить и отправить по электронной почте**, как правило, вы получите тестовое уведомление по электронной почте не содержащие оповещений о работоспособности. Тем не менее если оповещение о работоспособности, который настроен для отправки уведомления по электронной почте, определяется в процессе тестирования, это оповещение включается в тестовое сообщение электронной почты.

> [!NOTE]
>  По умолчанию надстройка отчета о работоспособности интегрирована с Windows Server Essentials или Windows Server 2012 R2 с установленной ролью Windows Server Essentials Experience и отчеты о работоспособности отображаются на **отчеты о работоспособности** мониторинга s **Главная** страницы.  
  
##### <a name="to-set-up-email-notification-for-health-reports"></a>Чтобы настроить уведомление по электронной почте для отчетов о работоспособности

Если флажок установлен, введите имя пользователя и пароль для адреса электронной почты, введенного в **адрес электронной почты отправителя** в шаге 4а, а затем щелкните **ОК**.

###  <a name="BKMK_Protect"></a>Для защиты компьютера  
  
1.  Откройте центр обеспечения безопасности.  
  
2.  Определите состояние защиты от вирусов.  
  
3.  Выполните одно из следующих действий в зависимости от состояния защиты.  
  
    -   Если она не включена, включите его.  
  
    -   Если не обновлена, выполните процесс обновления сигнатур.  
  
    -   Если защита от вирусов не установлена, установите ее.
