# <a name="manage-office-365-in-windows-server-essentials"></a>Управление Office 365 в Windows Server Essentials

>Область применения. Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

При интеграции сервера Windows Server Essentials с Microsoft Office 365, а также к локальным ресурсам из панели мониторинга Windows Server Essentials можно управлять службам Office 365 и учетными записями Интернет. В этом разделе вы сможете узнать, можно получить, интегрировав сервер с Office 365, как это сделать и как управление и устранение неполадок интеграции Office 365.  
  
  
  
> [!IMPORTANT]
>   Интеграция с Office 365 поддерживается только в среде одним контроллером домена. Кроме того мастер интеграции Office 365 необходимо запустить на контроллере домена.

##  <a name="BKMK_IntegrationOverview"></a> Для чего интегрировать Office 365 с моего сервера?  
 Существует множество причин для интеграции Office 365 с сервером Windows Server Essentials. Если управлять некоторыми из внутренних ресурсов, но использует Office 365 для других служб, вы получите иметь возможность управлять службами Office 365 и ресурсы с помощью панели мониторинга, а также локальных ресурсов, а не для работы в двух местах.  
  
-   Управление записями веб-служб, которые предоставляют пользователям доступ к Office 365 вместе с учетными записями пользователей:  
  
    -   Создавайте учетные записи Microsoft Online Services для пользователей с помощью одного простого шага, а также создавайте учетные записи на сервере для уже существующих учетных записей интернет-служб. Также можно добавить учетную запись интернет-службы для новой или уже существующей учетной записи.  
  
         При создании учетных записей с помощью панели мониторинга пользователей войдите в Office 365 с тем же паролем, которые они используют на сервере. Если пользователь изменит пароль для своей учетной записи, то пароль для учетной записи интернет-службы тоже изменится. Как вам известно, пароли учетных записей интернет-служб должны соответствовать требованиям безопасности, установленным вами для пользовательских учетных записей.  
  
    -   Управляйте учетными записями интернет-служб, как и учетными записями пользователей на всем протяжении их существования. При отключении учетной записи пользователя учетная запись интернет-служб тоже будет отключена. Удаление пользовательской учетной записи приведет к удалению учетной записи интернет-служб.  
  
    -   На сервере Windows Server Essentials также управление группами рассылки Exchange Online для электронной почты.  
  
-   Отправлять и получать электронную почту с домена Интернета организации s (например, contoso.com), привязав настраиваемый домен Интернет к подписки Office 365.  
  
-   Управления подпиской и интеграцией Office 365 из панели мониторинга.  
  
-   Если ваша подписка включает библиотеки SharePoint Online, интеграции Office 365 с сервером Windows Server Essentials позволяет:  
  
    -   Создание и управление ими библиотеках SharePoint Online с помощью панели мониторинга.  
  
        > [!NOTE]
        >  Вы также сможете использовать приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online из ноутбуке, мобильном устройстве или Windows phone. Эта функция доступна только для Windows Server Essentials. Дополнительные сведения см. в разделе [использование приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md).  
  
    -   Изменить разрешения для сайта группы SharePoint Online с помощью панели мониторинга или открывать сайт группы с помощью панели мониторинга, чтобы внести другие изменения.  
  
     Дополнительные сведения см. в разделе [управление SharePoint Online](Manage-SharePoint-Online-in-Windows-Server-Essentials.md).  
  
-   Если вы подписаны на Exchange Online, интеграции Office 365 с сервером Windows Server Essentials позволяет управлять мобильными устройствами, которые пользователи будут использовать для подключения к почтовому серверу компании:  
  
    -   Требовать использование пароля при подключении мобильного устройства к почтовому серверу компании. Устанавливать минимальную длину пароля, максимальное количество неудачных попыток входа и минимальный промежуток времени между попытками.  
  
    -   Запрещать подключение мобильного устройства к Exchange Online, если вам известны проблемы с безопасностью данной модели устройства.  
  
    -   В случае утери или кражи устройства очищать данные на нем, чтобы при следующем включении устройства все конфиденциальные данные компании были с него удалены.  
  
-    Интеграция с Office 365 предоставляет новые способы подключения к службам Office 365 и ресурсам.  
  
    -   Откройте службам Office 365 из панели запуска Windows Server Essentials. Сведения см. в разделе [краткое руководство по использованию Microsoft Office 365](../use/Quick-Start-Guide-to-Using-Microsoft-Office-365-with-Windows-Server-Essentials.md).  
  
    -   Используете приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online из ноутбуке, мобильном устройстве или Windows phone. Сведения см. в разделе [использование приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md). Эта функция доступна только в Windows Server Essentials.  
  
##  <a name="BKMK_Configure"></a> Настройка интеграции Office 365  
 Сервер можно интегрировать с Office 365 в любое время после завершения установки сервера. По окончании уже есть подписка Office 365, вы можете приобрести ее или зарегистрироваться для получения бесплатной пробной подписки.  
  
 Необходимо выполнить следующие шаги:  
  
-   [Шаг 1. Проверка выполнения требований интеграции Office 365](#BKMK_StepOne_VERIFY)  
  
-   [Шаг 2. Интеграция сервера с Microsoft Office 365](#BKMK_StepTwo)  
  
-   [Шаг 3. Привязка имени домена Интернета организации s к Office 365 (необязательно)](#BKMK_StepThree)  
  
###  <a name="BKMK_StepOne_VERIFY"></a> Шаг 1. Проверка выполнения требований интеграции Office 365  
 Убедитесь, что сервер соответствует требованиям:  
  
-   Одна из операционных систем:  Windows Server Essentials, Windows Server Essentials или операционной системы Windows Server 2012 R2 Standard или Windows Server 2012 R2 Datacenter с установленной ролью Windows Server Essentials Experience.  
  
-   Среде может иметь только один контроллер домена, а также интеграции Office 365 необходимо выполнить на контроллере домена.  
  
-   У вас должна быть возможность подключения к Интернету через сервер.  
  
-   Все критические и важные обновления должны быть установлены на сервере до начала интеграции Office 365.  
  
-   Если вы хотите использовать домена Интернета организации s адресов электронной почты и для ресурсов SharePoint Online, вы получите необходимо зарегистрировать имя домена, чтобы можно было связать домена к Office 365 во время интеграции. Дополнительные сведения см. в статье [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660).  
  
> [!NOTE]
>  Нигде не t нужно заранее подписываться на Office 365. Вы сможете приобрести подписку или зарегистрироваться для получения бесплатной пробной версии во время интеграции Office 365. Если вы d взгляните на планы и цены на Office 365, [сравнение планов Office 365 для бизнеса](https://office.microsoft.com/compare-office-365-for-business-plans-FX102918419.aspx?CR_CC=200061904&WT.srch=1&WT.mc_ID=PS_bing_O365Comm_subscribe-to-office-365_Text).  
  
###  <a name="BKMK_StepTwo"></a> Шаг 2. Интеграция сервера с Microsoft Office 365  
 Выполните следующую процедуру на контроллере домена для интеграции сервера Windows Server Essentials с Office 365.  
  
> [!NOTE]
>  Процедура s же ли у вас есть Windows Server Essentials или Windows Server Essentials, но вы сможете начать с в другое место на домашней странице. Интеграция сервера с Office 365 и другие Microsoft Online Services в Windows Server Essentials на **служб** вкладки. В Windows Server Essentials, интеграции Office 365 выполняются на **электронной почты** вкладки.  
  
##### <a name="to-integrate-the-server-with-office-365"></a>Интеграция сервера с Office 365  
  
1.  Войдите на сервер с правами администратора и откройте панель мониторинга Windows Server Essentials.  
  
2.  На **Главная** щелкните **служб** (в Windows Server Essentials щелкните **электронной почты**), нажмите кнопку **интеграция с Microsoft Office 365**, а затем нажмите кнопку **Настройка интеграции с Microsoft Office 365**.  
  
     Откроется мастер интеграции с Microsoft Office 365.  
  
3.  На странице **Начало работы** выполните одно из следующих действий:  
  
    -   Если Дон t у вас есть подписка на Office 365, щелкните **Далее**и следуйте инструкциям, чтобы подписаться на Office 365 или зарегистрироваться вверх для получения пробной подписки.  
  
         Вы должны войти Office 365, прежде чем вернуться к мастеру. Но нигде не t необходимость выполнения задач в **начало** раздел на портале Office 365.  
  
    -   Если у вас уже есть подписка на Office 365, который вы хотите интегрировать с сервером, выберите **у меня уже есть подписка для Office 365**, а затем нажмите кнопку **Далее**.  
  
4.  Для завершения работы мастера следуйте инструкциям на экране.  
  
 После успешного завершения работы мастера, вы получите Обратите внимание, что следующие изменения в панели мониторинга:  
  
-   Здесь s новый **Office 365** страницу, которая используется для управления интеграцию и подписки Office 365.  
  
-   На **пользователей** страницы, ll см **учетными записями Интернет** вкладку, где можно создать и управлять учетными записями Microsoft Online Services, которые предоставляют пользователям доступ к Office 365. Если вы используете Exchange Online и сервер Windows Server Essentials, вы также увидите **группы распространения** вкладки.  
  
-   **Хранения** страница на сервере Windows Server Essentials содержит **библиотеки SharePoint** вкладку для изменения разрешений для сайтов групп и управления ими библиотеках SharePoint Online. Все планы Office 365 включает в себя эти ограниченные возможности SharePoint Online.  
  
###  <a name="BKMK_StepThree"></a> Шаг 3. Привязка имени домена Интернета организации s к Office 365 (необязательно)  
 Если вы хотите использовать собственный домен Интернета в адресе электронной почты вашей организации и URL-адресов для ресурсов SharePoint Online, можно связать личный домен к подписке Office 365. При интеграции сервера Windows Server Essentials с Office 365, это можно сделать с помощью панели мониторинга.  
  
 Он проще всего это сделать, прежде чем создавать online s учетные записи для пользователей, чтобы можно было использовать домен при массовом создании учетных записей.  
  
 Получение имени домена, при подписке на Office 365? например, *contoso*. onmicrosoft.com. Если вы d использовать другое доменное имя? скажем, просто contoso.com? вы можете. Вы получите необходимо купить доменное имя по окончании уже принадлежащих и изменить некоторые записи DNS.  
  
 Настройка пользовательского домена для использования с Office 365 состоит из четырех шагов:  
  
1.  **Приобретение имени домена.** Вы можете зарегистрировать доменное имя у регистратора доменных имен или у поставщика услуг размещения DNS.  
  
    -   Выберите доменное имя, которое работает с Office 365. Можно использовать имя домена второго уровня? например, buycontoso.com?, но не имя домена третьего уровня? например, marketing.contoso.com. Дополнительные сведения о выборе домена для использования в Office 365 см. в разделе [домены](https://technet.microsoft.com/library/office-365-domains.aspx).  
  
    -   Купите его у регистратора, который поддерживает сервера (DNS) записи имя домена, необходимые для Office 365. Узнать, какие регистраторы доменных имен поддерживают необходимые записи DNS можно в статье [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660). Вы уже зарегистрировали домен у разных регистратора доменных имен, не беспокоиться; Вы сможете перенести его для различных регистратора при подключении домена к Office 365.  
  
2.  **Настройка записей DNS, позволяющих службам Office 365 использовать доменное имя.** Проще всего предоставить возможность мастеру настроить записи DNS при подключении домена к подписке Office 365 на шаге 3. Если вы d сделать это самостоятельно, см. в разделе [вручную Настройка DNS-записей для интеграции Office 365](#BKMK_ManuallyConfigureDNS).  
  
3.  **Привязка вашего настраиваемого домена Интернета к подписке Office 365.** Используйте ll **привязка домена к Office 365** действие.  
  
4.  **Убедитесь, что ваши службы Office 365 используется имя нового домена.**  
  
 Если вы завершите шаги 1 и 2, прежде чем использовать **привязка домена к Office 365** задач, мастер сможет связать имя домена к Office 365. Кроме того, вы можете использовать мастер для частичного или полного выполнения шагов 1 и 2.  
  
##### <a name="to-link-your-organization-s-internet-domain-to-office-365"></a>Чтобы связать домена Интернета организации s с Office 365  
  
1.  На панели мониторинга откройте страницу **Office 365** и выберите **Привязка домена к Office 365**.  
  
2.  Для завершения работы мастера следуйте инструкциям на экране.  
  
     Мастер поможет выполнить некоторые или все из следующих действий: регистрация, Настройка и привязка нового или существующего доменного имени для использования в Office 365.  
  
     Для получения дополнительных инструкций можно пользоваться ссылкой на справку мастера. Или см. в разделе настройки раздела имени домена [Manage Remote Web Access в Windows Server Essentials](https://technet.microsoft.com/library/jj628152\(d=printer\).aspx) Обзор процесса и требования.  
  
    > [!NOTE]
    >  При использовании мастера для регистрации нового доменного имени необходимо воспользоваться услугами одного из поставщиков, сотрудничающих с Майкрософт. Это обеспечит эффективную интеграцию. Чтобы найти регистратор доменных имен, см. статью [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660).  
  
3.  Если мастер определит, что t является имя вашего домена под управлением сервера, вы получите необходимо вручную настроить необходимые записи DNS, чтобы завершить настройку. Инструкции см. в разделе [Ручная настройка записей DNS для интеграции Office 365](#BKMK_ManuallyConfigureDNS)этой статьи.  
  
4.  Убедитесь, что домен используется в Office 365.  
  
     Здесь s некоторое после завершения работы мастера, пока регистратор доменных имен проверил записи DNS. Это происходит автоматически; Дон t нужно выполнять никаких действий. Обычно это занимает около часа? и иногда немного дольше. Когда верификация домена будет завершена, **Office 365** домена организации s будут отображаться на странице.  
  
####  <a name="BKMK_ManuallyConfigureDNS"></a> Ручная настройка записей DNS для интеграции Office 365  
 Если мастер привязки домена к Office 365 определит, что ваше доменное имя не находится под управлением сервера, то для завершения настройки вам необходимо будет настроить необходимые записи DNS вручную. В этом случае вы найдете список записей DNS, которые необходимо настроить в **%username%\NewDNSRecords_ (n) .txt**, где *(n)* является случайным числом.  
  
 В следующей таблице содержится описание записей DNS, которые необходимо добавить. Методы ввода могут различаться у различных регистраторов доменных имен. Вы можете обращаться за помощью к регистратору доменных имен.  
  
### <a name="required-dns-records-for-linking-a-custom-internet-domain-name-to-office-365"></a>Записи DNS, необходимые для привязки настраиваемого доменного имени Интернет к Office 365  
  
|Обслуживание|Необходимые записи DNS|Цель|  
|-------------|--------------------------|-------------|  
|(Несколько служб)|MX| Office 365 использует эту запись, чтобы убедиться, что вы являетесь владельцем доменного имени. Данная запись MX не мешает маршрутизации сообщений электронной почты.|  
|Exchange Online|MX|Обеспечивает маршрутизацию сообщений электронной почты. **Важно!**  При миграции электронной почты не задавайте в качестве предпочтения новой записи MX значение ноль (**0**). Значение записи должно быть выше, чем значение, заданное текущей записи MX. После завершения миграции электронной почты, и вы готовы приступить к изменению сервера электронной почты Office 365, у регистратора доменных имен сбросил значение предпочтения новой записи MX.|  
|Exchange Online|Псевдоним (CNAME)|Запись автообнаружения, с помощью которой пользователи легко могут устанавливать соединение между Exchange Online и клиентом Outlook для настольного ПК или мобильным почтовым клиентом. **Примечание.**  Если вы хотите получить доступ к Outlook Web Access с помощью собственного доменного имени вашей организации s (например, http://mail.contoso.com) вместо стандартного URL-адреса (https://outlook.com/owa/office365.com), записи псевдонима (CName) можно настроить следующим образом: **Type=CNAME, TTL=01:00:00, HostName=mail, Address=mail.office365.com**|  
|Exchange Online|TXT|Указывает, что то, что outlook.com, домен, используемый серверов электронной почты Office 365, имеет право отправлять электронную почту от имени вашего домена. Создание этой записи обеспечивает то, что ваша исходящая почта не будет отмечена как спам.|  
|Lync Online|SRV|Способствует совместной работе с другими службами мгновенных сообщений, например, с Windows Live или Yahoo!.|  
|Lync Online|SRV|Запись автообнаружения, с помощью которой пользователи легко могут устанавливать соединение между клиентом Lync настольного ПК и Microsoft Lync Online.|  
  
> [!IMPORTANT]
>  Если домен завершена проверку подлинности, не пытайтесь добавить или внести в них изменения записей DNS на портале Office 365.  
  
###  <a name="BKMK_StepFour_ACCOUNTS"></a> Следующий шаг  
  
-   Создание пользовательских учетных записей Microsoft Online Services.  
  
     Для использования служб Office 365, ваши пользователи должны иметь учетную запись Microsoft Online Services, связанную с их сетевыми учетными записями. Это легко сделать при помощи панели мониторинга. Если вы используете новую подписку Office 365, вы можете Массовое создание учетных записей для существующих учетных записей пользователя. Если вы повторно интеграция нового сервера с подпиской Office 365, вы уже используете, вы можете импортировать учетные записи пользователей из существующей сети учетные записи. Процедуры, см. в разделе [управление Online учетных записей пользователей](Manage-Online-Accounts-for-Users.md).  
  
> [!NOTE]
>  На панели мониторинга в Windows Server Essentials учетные записи Microsoft Online Services называются учетными записями Office 365. Это те же самые учетные записи, просто другая терминология.  
  
##  <a name="BKMK_ManageIntegration"></a> Управление интеграцией Office 365  
 После интеграции сервера с Office 365, **Office 365** на странице панели мониторинга отображаются сведения о подписке Office 365 и станут доступны следующие действия:  
  
-   [Управление подпиской Office 365](#BKMK_ManageO365) ? Изменение учетной записи администратора, используемую для управления подпиской. Откройте панель мониторинга администратора Office 365 для управления подпиской.  
  
-   [Привязка домена Интернет организации s к Office 365](#BKMK_StepThree) ? Если вы хотите иметь возможность отправлять и получать электронную почту, адресованную на ваш домен, вы можете связать домен Office 365. (Описано ранее в [Step 3: Привязка вашего домена s организации к Office 365](#BKMK_StepThree).)  
  
-   [Отключение интеграции Office 365](#BKMK_Disable) ? Если Дон t хотите управлять службами Office 365, подписки и учетные записи в Интернете из панели мониторинга, можно отключить интеграции Office 365. Службы по-прежнему доступны на портале Office 365.  
  
###  <a name="BKMK_ManageO365"></a> Управление подпиской Office 365  
 Если вам нужно внести изменения в подписку Office 365 во время работы на сервере, вы можете открыть подписку в Office 365 из **Office 365** страница панели мониторинга. Можно также изменить учетную запись администратора, сервер использует для внесения изменений к службам Office 365.  
  
##### <a name="to-open-your-subscription-on-the-office-365-admin-dashboard"></a>Открытие подписки Office 365 на панели мониторинга администратора  
  
1.  На панели мониторинга Essentials Windows Server откройте **Office 365** страницы.  
  
2.  В разделе **Настройки**выберите **Управление Office 365**.  
  
3.  Войдите в Office 365 с помощью учетной записи Майкрософт, используемом для управления подпиской.  
  
     Открытие панели мониторинга администратора Office 365.  
  
##### <a name="to-change-the-office-365-administrator-account"></a>Изменение учетной записи администратора Office 365  
  
1.  На панели мониторинга выберите **Office 365**.  
  
2.  В разделе **Настройки**выберите **Изменение учетной записи администратора Office 365**. Откроется мастер изменения учетной записи администратора. (В Windows Server Essentials, мастер называется Настройка Office 365 администратор учетной записи).  
  
3.  Введите учетные данные учетной записи, которая будет использоваться для подключения к подписке Office 365 и нажмите кнопку **Далее**.  
  
4.  Нажмите кнопку **Закрыть**. Панель мониторинга перезагрузится.  
  
###  <a name="BKMK_Disable"></a> Отключение интеграции Office 365  
 Если вы решите, что у вас нет t управляемым службам Office 365 и учетными записями Интернет с помощью панели мониторинга, вы можете отключить интеграции Office 365. Подписки Office 365 остается активным, и изменения конфигурации, внесенные с помощью панели мониторинга остаются в силе. Например вы сможете получать электронную почту, адресованную доменному имени, которое вы привязали к подписке Office 365. Ваших потеряйте все сообщения электронной почты и элементы управления, которые вы установили для мобильных устройств по-прежнему использовать Exchange Online.  
  
 В дальнейшем будет управлять подписки Office 365, служб и ресурсов в Office 365, и пользователи также должны управлять паролями для своих учетных записей Интернет в Office 365. Синхронизация паролей больше производиться не будет, а отключение или удаление учетной записи пользователя не повлияет на s online учетной записи пользователя.  
  
 Так как программным обеспечением для интеграции Office 365 устанавливается на локальном сервере, можно отключить функцию, даже если отсутствует соединение службы интеграции с Office 365.  
  
##### <a name="to-disable-office-365-integration-with-the-server"></a>Отключение интеграции Office 365 с сервером  
  
1.  На панели мониторинга выберите **Office 365**.  
  
2.  Нажмите на **Отключение интеграции Office 365**. Откроется мастер отключения Office 365.  
  
3.  Для завершения работы мастера следуйте инструкциям на экране.  
  
> [!NOTE]
>  Чтобы включить снова интеграции Office 365, используйте **интеграция с Office 365** в задач **службы** вкладку панели мониторинга s **Главная** страницы. Инструкции см. в разделе [шаг 2: Интеграция сервера Windows Server Essentials с Microsoft Office 365](#BKMK_StepTwo)ранее в этом разделе.  
  
##  <a name="BKMK_Troubleshoot"></a> Устранение неполадок интеграции Office 365  
 Этот раздел содержит сведения об устранении распространенных проблем, которые могут возникнуть при использовании функций интеграции Office 365 в Windows Server Essentials.  
  
###  <a name="BKMK_AcctsNotCreated"></a> Некоторые учетные записи Microsoft Online Services не были созданы  
 **Описание**  
  
 Предпринята попытка создать один или несколько учетных записей Microsoft Online Services из t панели мониторинга не были успешно.  
  
 **Решение**  
  
1.  Перейдите по ссылке на странице завершения работы мастера, чтобы открыть файл, содержащий детальную информацию о каждой неудачной попытке создания учетной записи. Например, может оказаться, что произошла попытка создать учетную запись Microsoft Online Services с уже существующим именем.  
  
2.  Воспользуйтесь рекомендациями для решения каждой отдельной проблемы.  
  
3.  Если проблема остается, перезагрузите сервер и попробуйте создать учетные записи еще раз.  
  
###  <a name="BKMK_ProblemUninstalling"></a> Возникла проблема с удалением интеграции Office 365  
 **Описание**  
  
 Произошла неизвестная ошибка при попытке отключить интеграцию с Office 365.  
  
 **Решение**  
  
1.  Убедитесь, что компьютер подключен к сети Интернет и попробуйте еще раз.  
  
2.  Если ошибка повторяется, перезагрузите сервер и попробуйте еще раз.
