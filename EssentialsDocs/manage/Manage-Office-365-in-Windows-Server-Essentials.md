# <a name="manage-microsoft-365-in-windows-server-essentials"></a>Управление Microsoft 365 в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

При интеграции сервера Windows Server Essentials с Microsoft 365 вы можете управлять Microsoft 365 службами и учетными записями в Интернете вместе с локальными ресурсами на панели мониторинга Windows Server Essentials. В этом разделе вы узнаете, что можно получить, выполнив интеграцию сервера с Microsoft 365, как это сделать, а также как управлять Microsoft 365 интеграцией и устранять неполадки.



> [!IMPORTANT]
>   Интеграция Microsoft 365 поддерживается только в среде с одним контроллером домена. Кроме того, мастер интеграции Microsoft 365 должен работать на контроллере домена.

##  <a name="why-should-i-integrate-microsoft-365-with-my-server"></a><a name="BKMK_IntegrationOverview"></a> Зачем интегрировать Microsoft 365 с моим сервером?
 Существует множество веских причин для интеграции Microsoft 365 с сервером Windows Server Essentials. Если вы управляете некоторыми ресурсами, но используете Microsoft 365 для других служб, вы сможете управлять Microsoft 365ными службами и ресурсами с панели мониторинга вместе с локальными ресурсами, а не работать в двух местах.

- Управляйте учетными записями в сети, которые предоставляют пользователям доступ к Microsoft 365 вместе с учетными записями пользователей:

  -   Создавайте учетные записи Microsoft Online Services для пользователей с помощью одного простого шага, а также создавайте учетные записи на сервере для уже существующих учетных записей интернет-служб. Также можно добавить учетную запись интернет-службы для новой или уже существующей учетной записи.

       При создании учетных записей в Интернете на панели мониторинга пользователи входят в Microsoft 365 с тем же паролем, который они используют на сервере. Если пользователь изменит пароль для своей учетной записи, то пароль для учетной записи интернет-службы тоже изменится. Как вам известно, пароли учетных записей интернет-служб должны соответствовать требованиям безопасности, установленным вами для пользовательских учетных записей.

  -   Управляйте учетными записями интернет-служб, как и учетными записями пользователей на всем протяжении их существования. При отключении учетной записи пользователя учетная запись интернет-служб тоже будет отключена. Удаление пользовательской учетной записи приведет к удалению учетной записи интернет-служб.

  -   На сервере Windows Server Essentials следует также управлять группами рассылки Exchange Online для электронной почты.

- Отправка и получение электронной почты из домена Интернета вашей организации (например, contoso.com) путем связывания пользовательского Интернет-домена с подпиской Microsoft 365.

- Управляйте подпиской и Microsoft 365 интеграцией из панели мониторинга.

- Если подписка включает библиотеки SharePoint Online, Microsoft 365 интеграция с сервером Windows Server Essentials позволяет:

  - Создание библиотек SharePoint Online и управление ими на панели мониторинга.

    > [!NOTE]
    >  Вы также сможете использовать приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online с ноутбука, мобильного устройства или Windows Phone. Эта функция доступна только для Windows Server Essentials. Дополнительные сведения см. [в разделе Использование приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md).

  - Измените разрешения для сайта группы SharePoint Online на панели мониторинга или откройте сайт группы на панели мониторинга, чтобы внести другие изменения.

    Дополнительные сведения см. в разделе [Управление SharePoint Online](Manage-SharePoint-Online-in-Windows-Server-Essentials.md).

- Если вы подписались на Exchange Online, Microsoft 365 интеграция с сервером Windows Server Essentials позволяет управлять мобильными устройствами, которые пользователи будут использовать для подключения к серверу электронной почты компании:

  -   Требовать использование пароля при подключении мобильного устройства к почтовому серверу компании. Устанавливать минимальную длину пароля, максимальное количество неудачных попыток входа и минимальный промежуток времени между попытками.

  -   Запрещать подключение мобильного устройства к Exchange Online, если вам известны проблемы с безопасностью данной модели устройства.

  -   В случае утери или кражи устройства очищать данные на нем, чтобы при следующем включении устройства все конфиденциальные данные компании были с него удалены.

- Интеграция Microsoft 365 предоставляет новые способы подключения к Microsoft 365 службам и ресурсам.

  -   Откройте Microsoft 365 службы на панели запуска Windows Server Essentials. Дополнительные сведения см. [в разделе Краткое руководство по началу работы to Using Microsoft 365](../use/Quick-Start-Guide-to-Using-Microsoft-Office-365-with-Windows-Server-Essentials.md).

  -   Используйте приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online с ноутбука, мобильного устройства или Windows Phone. Дополнительные сведения см. [в разделе Использование приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md). Эта функция доступна только в Windows Server Essentials.

##  <a name="set-up-microsoft-365-integration"></a><a name="BKMK_Configure"></a> Настройка интеграции Microsoft 365
 Вы можете интегрировать сервер с Microsoft 365 в любое время после завершения установки сервера. Если у вас еще нет подписки Microsoft 365, вы можете приобрести ее или зарегистрироваться для получения бесплатной пробной подписки.

 Необходимо выполнить следующие шаги:

-   [Шаг 1. Проверка требований к интеграции Microsoft 365](#BKMK_StepOne_VERIFY)

-   [Шаг 2. Интеграция сервера с Microsoft 365](#BKMK_StepTwo)

-   [Шаг 3. Связывание доменного имени Организации с Microsoft 365 (необязательно)](#BKMK_StepThree)

###  <a name="step-1-verify-microsoft-365-integration-requirements"></a><a name="BKMK_StepOne_VERIFY"></a> Шаг 1. Проверка требований к интеграции Microsoft 365
 Убедитесь, что сервер соответствует требованиям:

-   Сервер может иметь любую из следующих операционных систем: Windows Server Essentials, Windows Server Essentials или Windows Server 2012 R2 Standard или Windows Server 2012 R2 Datacenter с установленной ролью Windows Server Essentials Experience.

-   В среде может быть только один контроллер домена, и необходимо выполнить интеграцию Microsoft 365 на контроллере домена.

-   У вас должна быть возможность подключения к Интернету через сервер.

-   Перед началом интеграции Microsoft 365 необходимо установить все критические и важные обновления на сервере.

-   Если вы хотите использовать домен Интернета вашей организации в адресах электронной почты и ресурсах SharePoint Online, необходимо зарегистрировать имя домена, чтобы можно было связать домен с Microsoft 365 во время интеграции. Дополнительные сведения см. в статье [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660).

> [!NOTE]
>  Вам не нужно подписываться на Microsoft 365 заранее. Вы сможете приобрести подписку или зарегистрироваться для получения бесплатной пробной версии во время интеграции Microsoft 365. Если вы хотите ознакомиться с планами и ценами на Microsoft 365, [Сравните планы Microsoft 365 для предприятий](https://office.microsoft.com/compare-office-365-for-business-plans-FX102918419.aspx?CR_CC=200061904&WT.srch=1&WT.mc_ID=PS_bing_O365Comm_subscribe-to-office-365_Text).

###  <a name="step-2-integrate-the-server-with-microsoft-365"></a><a name="BKMK_StepTwo"></a> Шаг 2. Интеграция сервера с Microsoft 365
 Выполните следующую процедуру на контроллере домена, чтобы интегрировать сервер Windows Server Essentials с Microsoft 365.

> [!NOTE]
>  Эта процедура та же, что и Windows Server Essentials или Windows Server Essentials, но вы начинаете с другого места на домашней странице. В Windows Server Essentials сервер интегрируется с Microsoft 365 и другими службами Microsoft Online Services на вкладке **службы** . В Windows Server Essentials Microsoft 365 интеграция выполняется на вкладке **Электронная почта** .

##### <a name="to-integrate-the-server-with-microsoft-365"></a>Интеграция сервера с Microsoft 365

1. Войдите на сервер с правами администратора и откройте панель мониторинга Windows Server Essentials.

2. На **домашней** странице щелкните **службы** (в Windows Server Essentials, щелкните **Электронная почта**), щелкните **интегрировать с Microsoft 365**, а затем — **настроить интеграцию Microsoft 365**.

    Откроется мастер интеграции с Microsoft 365.

3. На странице **Начало работы** выполните одно из следующих действий:

   -   Если у вас нет подписки на Microsoft 365, нажмите кнопку **Далее** и следуйте инструкциям, чтобы подписаться на Microsoft 365 или зарегистрироваться для получения пробной подписки.

        Необходимо войти в Microsoft 365, прежде чем вернуться в мастер. Но вам не нужно выполнять какие – либо задачи в разделе **Start** на портале Microsoft 365.

   -   Если у вас уже есть подписка на Microsoft 365, которую необходимо интегрировать с сервером, выберите у **меня уже есть подписка на Microsoft 365** и нажмите кнопку **Далее**.

4. Для завершения работы мастера следуйте инструкциям на экране.

   После успешного завершения работы мастера вы заметите следующие изменения на панели мониторинга:

-   Есть новая **Microsoft 365** страница, которая используется для управления интеграцией и подпиской Microsoft 365.

-   На странице **Пользователи** вы увидите вкладку **учетные записи в Интернете** , где можно создавать учетные записи Microsoft Online Services и управлять ими, которые предоставляют пользователям доступ к Microsoft 365. Если вы используете Exchange Online и используете сервер Windows Server Essentials, вы также увидите вкладку **группы рассылки** .

-   На странице **хранилище** на сервере Windows Server Essentials есть вкладка **библиотеки SharePoint** для управления библиотеками SharePoint Online и изменения разрешений для сайтов рабочих групп. Каждый бизнес-план для Microsoft 365 включает эти базовые возможности SharePoint Online.

###  <a name="step-3-link-your-organizations-internet-domain-name-to-microsoft-365-optional"></a><a name="BKMK_StepThree"></a> Шаг 3. Связывание доменного имени Организации с Microsoft 365 (необязательно)
 Если вы хотите использовать собственный домен Интернета в электронной почте, адресованной вашей организации, и URL-адреса ресурсов SharePoint Online, вы можете связать личный домен с подпиской Microsoft 365. Если вы интегрируете сервер Windows Server Essentials с Microsoft 365, это можно сделать на панели мониторинга.

 Это проще всего сделать перед созданием учетных записей в Интернете для пользователей, чтобы вы могли использовать домен при создании учетных записей в Интернете.

 При регистрации в Microsoft 365 вы получаете доменное имя? например, *contoso*. onmicrosoft.com. Если вы предпочитаете использовать другое доменное имя? скажем, просто contoso.com? можно. Необходимо приобрести имя домена, если вы еще не являетесь его владельцем, и изменить некоторые записи DNS.

 Настройка пользовательского домена для использования с Microsoft 365 состоит из четырех задач:

1. **Приобретение имени домена.** Вы можете зарегистрировать доменное имя у регистратора доменных имен или у поставщика услуг размещения DNS.

   -   Выберите доменное имя, которое работает с Microsoft 365. Вы можете использовать имя домена второго уровня? например, buycontoso.com?, но не имя домена третьего уровня? например, marketing.contoso.com. Дополнительные сведения о выборе домена для использования в Microsoft 365 см. в разделе [домены](/office365/servicedescriptions/office-365-platform-service-description/domains).

   -   Купите его от регистратора доменных имен, который допускает записи DNS, необходимые для Microsoft 365. Узнать, какие регистраторы доменных имен поддерживают необходимые записи DNS можно в статье [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660). Если вы уже зарегистрировали домен с другим регистратором, не беспокойтесь. Вы можете переместить домен в другой регистратор при связывании домена с Microsoft 365.

2. **Настройте записи DNS, которые позволяют службам Microsoft 365 использовать доменное имя.** Самый простой способ — позволить мастеру настроить записи DNS при связывании домена с подпиской Microsoft 365 на шаге 3. Если вы хотите сделать это самостоятельно, см. статью [как вручную настроить записи DNS для интеграции Microsoft 365](#BKMK_ManuallyConfigureDNS).

3. **Свяжите пользовательский домен Интернета с подпиской Microsoft 365.** Для Microsoft 365 действия используется **ссылка на домен** .

4. **Убедитесь, что службы Microsoft 365 используют новое доменное имя.**

   Если вы выполните шаги 1 и 2, прежде чем использовать задачу " **связать домен с Microsoft 365** ", мастер может связать имя домена с Microsoft 365. Кроме того, вы можете использовать мастер для частичного или полного выполнения шагов 1 и 2.

##### <a name="to-link-your-organizations-internet-domain-to-microsoft-365"></a>Связывание домена Интернета Организации с Microsoft 365

1.  На панели мониторинга откройте страницу **Microsoft 365** и щелкните **связать домен с Microsoft 365**.

2.  Для завершения работы мастера следуйте инструкциям на экране.

     Мастер поможет выполнить некоторые или все действия по регистрации, настройке и связыванию нового или существующего доменного имени в Интернете для использования в Microsoft 365.

     Для получения дополнительных инструкций можно пользоваться ссылкой на справку мастера. См. раздел Настройка доменного имени в разделе [Управление удаленными веб-доступ в Windows Server Essentials](https://technet.microsoft.com/library/jj628152\(d=printer\).aspx) для получения общих сведений и требований к процессу.

    > [!NOTE]
    >  При использовании мастера для регистрации нового доменного имени необходимо воспользоваться услугами одного из поставщиков, сотрудничающих с Майкрософт. Это обеспечит эффективную интеграцию. Чтобы найти регистратор доменных имен, см. статью [Покупка доменного имени](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660).

3.  Если мастер обнаружит, что доменное имя не управляется сервером, необходимо вручную настроить необходимые записи DNS для завершения настройки. Инструкции см. в подразделе [Настройка записей DNS вручную для интеграции Microsoft 365](#BKMK_ManuallyConfigureDNS)далее в этой статье.

4.  Убедитесь, что домен используется в Microsoft 365.

     После завершения работы мастера выполняется небольшое ожидание, а регистратор доменных имен проверяет записи DNS. Это происходит автоматически; ничего делать не нужно. Но обычно это занимает около часа? и иногда немного дольше. После завершения проверки домена на странице **Microsoft 365** будет указан домен организации.

####  <a name="how-to-manually-configure-dns-records-for-microsoft-365-integration"></a><a name="BKMK_ManuallyConfigureDNS"></a> Настройка записей DNS вручную для интеграции Microsoft 365
 Если мастер связывания домена с Microsoft 365 обнаруживает, что доменное имя не управляется сервером, для завершения настройки необходимо вручную настроить требуемые записи сервера доменных имен (DNS). В этом случае вы найдете список записей DNS, которые необходимо настроить в **% username% \ NewDNSRecords_ (n). txt**, где *(n)* — это случайное число.

 В следующей таблице содержится описание записей DNS, которые необходимо добавить. Методы ввода могут различаться у различных регистраторов доменных имен. Вы можете обращаться за помощью к регистратору доменных имен.

### <a name="required-dns-records-for-linking-a-custom-internet-domain-name-to-microsoft-365"></a>Необходимые записи DNS для связывания имени пользовательского домена Интернета с Microsoft 365

|Служба|Необходимые записи DNS|Назначение|
|-------------|--------------------------|-------------|
|(Несколько служб)|MX| Microsoft 365 использует эту запись, чтобы убедиться, что вы владеете конкретным доменным именем. Данная запись MX не мешает маршрутизации сообщений электронной почты.|
|Exchange Online|MX|Обеспечивает маршрутизацию сообщений электронной почты. **Важно.**  При переносе электронной почты не присваивайте новой записи MX предпочтение нулю (**0**). Значение записи должно быть выше, чем значение, заданное текущей записи MX. По завершении переноса электронной почты и готовности изменить сервер электронной почты на Microsoft 365, попросите регистратора доменных имен сбросить значение предпочтения новой MX-записи.|
|Exchange Online|Псевдоним (CNAME)|Запись автообнаружения, с помощью которой пользователи легко могут устанавливать соединение между Exchange Online и клиентом Outlook для настольного ПК или мобильным почтовым клиентом. **Примечание.**  Если вы предпочитаете получать доступ к Outlook Веб-доступ с помощью собственного доменного имени Организации (например, http://mail.contoso.com) вместо стандартного URL-адреса ( https://outlook.com/owa/office365.com) ), запись псевдонима (CNAME) можно настроить следующим образом: **Type = CNAME, TTL = 01:00:00, hostname = mail, Address = mail. Office 365. com**|
|Exchange Online|TXT|Указывает, что outlook.com, используемый Microsoft 365 серверами электронной почты, имеет право на отправку электронной почты от имени вашего домена. Создание этой записи обеспечивает то, что ваша исходящая почта не будет отмечена как спам.|
|Lync Online|SRV|Способствует совместной работе с другими службами мгновенных сообщений, например, с Windows Live или Yahoo!.|
|Lync Online|SRV|Запись автообнаружения, с помощью которой пользователи легко могут устанавливать соединение между клиентом Lync настольного ПК и Microsoft Lync Online.|

> [!IMPORTANT]
>  После завершения проверки домена не пытайтесь добавить или внести дальнейшие изменения в записи DNS на портале Microsoft 365.

###  <a name="next-step"></a><a name="BKMK_StepFour_ACCOUNTS"></a> Следующий шаг

-   Создание пользовательских учетных записей Microsoft Online Services.

     Для использования служб Microsoft 365 Services пользователи должны иметь учетную запись Microsoft Online Services, связанную с учетной записью сетевого пользователя. Это легко сделать при помощи панели мониторинга. Если вы используете новую подписку Microsoft 365, можно выполнить групповое создание учетных записей в Интернете для существующих учетных записей пользователей. При интеграции нового сервера с уже используемой подпиской Microsoft 365 можно импортировать учетные записи пользователей из существующих Интернет-учетных записей. Процедуры см. в разделе [Управление учетными записями в сети для пользователей](Manage-Online-Accounts-for-Users.md).

> [!NOTE]
>  На панели мониторинга в Windows Server Essentials учетные записи Microsoft Online Services называются учетными записями Microsoft 365. Это те же самые учетные записи, просто другая терминология.

##  <a name="manage-microsoft-365-integration"></a><a name="BKMK_ManageIntegration"></a> Управление интеграцией Microsoft 365
 После интеграции сервера с Microsoft 365 на странице **Microsoft 365** на панели мониторинга отображаются сведения о вашей подписке Microsoft 365 и становятся доступными следующие задачи:

-   [Управление подпиской Microsoft 365](#BKMK_ManageO365) ? Измените учетную запись администратора, используемую для управления подпиской. Откройте панель мониторинга Microsoft 365 администрирования для управления подпиской.

-   [Связать домен Интернета вашей организации с Microsoft 365](#BKMK_StepThree) ? Если вы хотите иметь возможность отправлять и получать электронную почту, адресованную своему домену, можно связать домен с Microsoft 365. (См. ранее в разделе [Шаг 3. Связывание домена организации с Microsoft 365](#BKMK_StepThree).)

-   [Отключить интеграцию Microsoft 365](#BKMK_Disable) ? Если вы не хотите управлять службами Microsoft 365, подпиской и сетевыми учетными записями на панели мониторинга, можно отключить интеграцию Microsoft 365. Службы по-прежнему доступны на портале Microsoft 365.

###  <a name="manage-your-microsoft-365-subscription"></a><a name="BKMK_ManageO365"></a> Управление подпиской на Microsoft 365
 Если необходимо внести изменения в подписку Microsoft 365 во время работы на сервере, можно открыть подписку в Microsoft 365 на странице **Microsoft 365** панели мониторинга. Также можно изменить учетную запись администратора, которую сервер использует для внесения изменений в службы Microsoft 365.

##### <a name="to-open-your-subscription-on-the-microsoft-365-admin-dashboard"></a>Открытие подписки на панели мониторинга администратора Microsoft 365

1.  На панели мониторинга Windows Server Essentials откройте страницу **Microsoft 365** .

2.  В меню **задачи настройки** выберите пункт **Управление Microsoft 365**.

3.  Войдите в Microsoft 365 с учетной записью Microsoft Online, которая используется для управления подпиской.

     Откроется панель мониторинга Microsoft 365 администрирования.

##### <a name="to-change-the-microsoft-365-administrator-account"></a>Изменение учетной записи администратора Microsoft 365

1.  На панели мониторинга щелкните **Microsoft 365**.

2.  В области **задачи настройки** щелкните **изменить учетную запись администратора Microsoft 365**. Откроется мастер изменения учетной записи администратора. (В Windows Server Essentials мастер называется настройкой учетной записи администратора Microsoft 365.)

3.  Введите учетные данные для учетной записи, которую вы хотите использовать для подключения к Microsoft 365ной подписке, а затем нажмите кнопку **Далее**.

4.  Щелкните **Закрыть**. Панель мониторинга перезагрузится.

###  <a name="disable-microsoft-365-integration"></a><a name="BKMK_Disable"></a> Отключение интеграции Microsoft 365
 Если вы решите, что вы не хотите управлять службами Microsoft 365 и учетными записями в Интернете на панели мониторинга, можно отключить интеграцию с Microsoft 365. Подписка на Microsoft 365 остается активной, и все изменения конфигурации, внесенные из панели мониторинга, остаются в силе. Например, вы получите электронное письмо с именем домена, связанным с подпиской Microsoft 365. Сообщения электронной почты не теряются, а элементы управления, заданные для мобильных устройств, по-прежнему используются в Exchange Online.

 В дальнейшем вы будете управлять Microsoft 365 подпиской, службами и ресурсами в Microsoft 365, и пользователям потребуется управлять паролями для своих учетных записей в Microsoft 365. Синхронизация паролей больше не выполняется, и отключение или удаление учетной записи пользователя не повлияет на учетную запись пользователя в Интернете.

 Поскольку программное обеспечение интеграции Microsoft 365 установлено на локальном сервере, эту функцию можно отключить, даже если службе интеграции не удается подключиться к Microsoft 365.

##### <a name="to-disable-microsoft-365-integration-with-the-server"></a>Отключение интеграции Microsoft 365 с сервером

1.  На панели мониторинга щелкните **Microsoft 365**.

2.  Щелкните **отключить интеграцию с Microsoft 365**. Откроется мастер отключения Microsoft 365.

3.  Для завершения работы мастера следуйте инструкциям на экране.

> [!NOTE]
>  Чтобы снова включить интеграцию Microsoft 365, используйте задачу **Интеграция с Microsoft 365** на вкладке **Служба** **домашней** страницы панели мониторинга. Инструкции см. в разделе [Шаг 2. Интеграция сервера Windows Server Essentials с Microsoft 365](#BKMK_StepTwo)ранее в этой статье.

##  <a name="troubleshoot-microsoft-365-integration"></a><a name="BKMK_Troubleshoot"></a> Устранение неполадок интеграции Microsoft 365
 В этом разделе содержатся сведения, помогающие устранить распространенные проблемы, которые могут возникнуть при использовании функций интеграции Microsoft 365 в Windows Server Essentials.

###  <a name="some-microsoft-online-services-accounts-were-not-created"></a><a name="BKMK_AcctsNotCreated"></a> Некоторые учетные записи Microsoft Online Services не были созданы
 **Описание**

 Попытка создать одну или несколько учетных записей Microsoft Online Services с панели мониторинга не удалась.

 **Решение**

1.  Перейдите по ссылке на странице завершения работы мастера, чтобы открыть файл, содержащий детальную информацию о каждой неудачной попытке создания учетной записи. Например, может оказаться, что произошла попытка создать учетную запись Microsoft Online Services с уже существующим именем.

2.  Воспользуйтесь рекомендациями для решения каждой отдельной проблемы.

3.  Если проблема остается, перезагрузите сервер и попробуйте создать учетные записи еще раз.

###  <a name="there-was-a-problem-uninstalling-microsoft-365-integration"></a><a name="BKMK_ProblemUninstalling"></a> Возникла проблема при удалении интеграции Microsoft 365
 **Описание**

 При попытке отключить интеграцию Microsoft 365 произошла неизвестная ошибка.

 **Решение**

1.  Убедитесь, что компьютер подключен к сети Интернет и попробуйте еще раз.

2.  Если ошибка повторяется, перезагрузите сервер и попробуйте еще раз.
