# <a name="manage-office-365-in-windows-server-essentials"></a>Управление Office 365 в Windows Server Essentials

>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials

При интеграции сервера Windows Server Essentials с Microsoft Office 365, можно управлять службами Office 365 и учетных записей в Интернете вместе с локальными ресурсами, с помощью панели мониторинга Windows Server Essentials. В этом разделе вы получите Узнайте, что можно делать, интегрировав сервер с Office 365, как это сделать и как управления и устранения неполадок интеграции с Office 365.  
  
  
  
> [!IMPORTANT]
>   Интеграция с Office 365 поддерживается только в среде одним контроллером домена. Кроме того мастер интеграции Office 365, необходимо запустить на контроллере домена.

##  <a name="BKMK_IntegrationOverview"></a>Для чего интегрировать Office 365 с сервером  
 Существует множество причин для интеграции Office 365 с сервером Windows Server Essentials. Если для управления некоторыми из ресурсов, но использовать Office 365 для других служб, вы получите иметь возможность управлять службами Office 365 и ресурсы из панели мониторинга, а также своими локальными ресурсами, вместо работает в двух местах.  
  
-   Управлять сетевыми учетными записями, предоставляющих доступа пользователей к Office 365 вместе с учетными записями пользователей:  
  
    -   Создайте учетные записи Microsoft Online Services для пользователей за один шаг или создайте учетные записи пользователей для существующих учетных записей Интернет на сервере. Также можно добавить учетную запись Интернет учетной записи нового или существующего пользователя.  
  
         При создании учетных записей с помощью панели мониторинга пользователей войдите Office 365 с тем же паролем, которые они используют на сервере. Если пользователь изменит пароль для своей учетной записи, Интернет-служб тоже изменится. И вы знаете, что пароли своих учетных записей Интернет должны соответствовать требованиям безопасности, заданные для учетных записей пользователей.  
  
    -   Управление учетной записью Интернет вместе с учетной записи пользователя на протяжении всего существования учетной записи пользователя. При отключении учетной записи пользователя, Интернет-служб тоже будет отключена. Если удалить учетную запись пользователя, эта учетная запись также удаляется.  
  
    -   На сервере Windows Server Essentials также Управляйте группами рассылки Exchange Online для электронной почты.  
  
-   Отправлять и получать электронную почту с домена Интернета организации s (например, contoso.com), привязав настраиваемый домен Интернета к подписке Office 365.  
  
-   Управлять своей подпиской и интеграцией с Office 365 из панели мониторинга.  
  
-   Если ваша подписка включает библиотеки SharePoint Online, интеграция с Office 365 с сервера Windows Server Essentials, вы сможете:  
  
    -   Создание и управление библиотеками SharePoint Online с помощью панели мониторинга.  
  
        > [!NOTE]
        >  Вы все также смогут использовать приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online на ноутбуке, мобильном устройстве или Windows phone. Эта функция доступна только для Windows Server Essentials. Дополнительные сведения см. в разделе [использования приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md).  
  
    -   Изменение разрешений сайта группы SharePoint Online с помощью панели мониторинга или открывать сайт группы с помощью панели мониторинга, чтобы внести другие изменения.  
  
     Дополнительные сведения см. в разделе [управление SharePoint Online](Manage-SharePoint-Online-in-Windows-Server-Essentials.md).  
  
-   Если вы подписаны на Exchange Online, интеграции с Office 365 с сервером Windows Server Essentials позволяет управлять мобильными устройствами, которые пользователи используют для подключения к почтовому серверу компании:  
  
    -   Требовать использование пароля при подключении мобильного устройства к почтовому серверу компании. Установите минимальную длину пароля, количество неудачных попыток входа в систему, чтобы разрешить и минимальный промежуток времени между попытками входа в систему.  
  
    -   Блокировка мобильного устройства, подключение к Exchange Online, если вам известны проблемы с безопасностью модели устройства.  
  
    -   В случае утери или кражи мобильного устройства очищать удаление в следующий раз при включении устройства все конфиденциальные данные компании.  
  
-    Интеграция с Office 365 предоставляет новые возможности для подключения к службам Office 365 и ресурсы:  
  
    -   Откройте службы Office 365 из панели запуска Windows Server Essentials. Подробнее см. в разделе [краткое руководство по использованию Microsoft Office 365](../use/Quick-Start-Guide-to-Using-Microsoft-Office-365-with-Windows-Server-Essentials.md).  
  
    -   Используйте приложение My Server 2012 R2 для работы с документами в библиотеках SharePoint Online на ноутбуке, мобильном устройстве или Windows phone. Подробнее см. в разделе [использования приложения My Server](../use/Use-the-My-Server-App-to-Connect-to-Windows-Server-Essentials.md). Эта функция доступна только в Windows Server Essentials.

###  <a name="BKMK_StepTwo"></a>Шаг 2: Интеграция сервера с Microsoft Office 365  
 Выполните следующую процедуру на контроллере домена для интеграции сервера Windows Server Essentials с Office 365.  
  
> [!NOTE]
>  Процедура s так же, требуется ли Windows Server Essentials или Windows Server Essentials, но вы получите запустите в другом месте на главной странице. Интеграция сервера с Office 365 и другие Microsoft Online Services в Windows Server Essentials на **службы** вкладку. В Windows Server Essentials выполняется интеграции Office 365 на **электронной почты** вкладку.  
  
##### <a name="to-integrate-the-server-with-office-365"></a>Интеграция сервера с Office 365  
  
1.  Войдите на сервер с правами администратора и откройте панель мониторинга Windows Server Essentials.  
  
2.  На **Главная** щелкните **службы** (в Windows Server Essentials щелкните **электронной почты**), нажмите кнопку **интеграция с Microsoft Office 365**, а затем нажмите кнопку **Настройка интеграции Microsoft Office 365**.  
  
     Откроется мастер интеграции с Microsoft Office 365.  
  
3.  На **начать** выполните одно из следующих действий:  
  
    -   Если Дон t подписки на Office 365, щелкните **Далее**и следуйте инструкциям для оформления подписки на Office 365 или зарегистрироваться вверх для использования пробной подписки.  
  
         Вы получите необходимо войти в Office 365, прежде чем вернуться в мастер. Но нигде не t нужно выполнять задачи в **начало** раздел на портал Office 365.  
  
    -   Если уже есть подписка на Office 365, для интеграции с сервером, выберите **меня уже есть подписка для Office 365**, а затем нажмите кнопку **Далее**.  
  
4.  Следуйте инструкциям для завершения работы мастера.  
  
 После успешного завершения работы мастера, вы получите Обратите внимание, описанные ниже изменения в информационной панели:  
  
-   Здесь s создается новый **Office 365** страницы, которая используется для управления интеграции и подписки на Office 365.  
  
-   На **пользователей** страница, все см **учетных записей в Интернете** вкладку, где можно создавать и управлять учетными записями Microsoft Online Services, необходимые пользователям доступ к Office 365. Если вы повторно с помощью Exchange Online и у сервера Windows Server Essentials, вы получите см. также **группы рассылки** вкладку.  
  
-   **Хранилища** страницы на сервере Windows Server Essentials **библиотеки SharePoint** вкладку Управление библиотеками SharePoint Online и изменение разрешений для сайтов групп. Все планы Office 365 включает следующие основные компоненты SharePoint Online.  
  
###  <a name="BKMK_StepThree"></a>Шаг 3: Привязка имени домена Интернета организации s к Office 365 (необязательно)  
 Если вы хотите использовать собственный домен Интернета в адресе электронной почты организации и URL-адреса для ресурсов SharePoint Online, можно связать настраиваемый домен к подписке Office 365. Если интеграция сервера Windows Server Essentials с Office 365, это можно сделать с помощью панели мониторинга.  
  
 Он s, проще всего сделать это перед созданием online учетных записей для пользователей, что позволит использовать домен при массовом создании учетных записей.  
  
 Получить имя домена, когда вы регистрируетесь для Office 365? например, *contoso*. onmicrosoft.com. Если вы d начать использовать другое доменное имя? скажем, просто contoso.com? вы можете. Вы получите необходимо купить имя домена по окончании уже самостоятельно и изменить некоторые записи DNS.  
  
 Установка использования настраиваемого домена с Office 365 состоит из четырех шагов:  
  
1.  **Приобретение имени домена.** То есть зарегистрируйте у регистратора доменных имен или поставщика услуг размещения DNS.  
  
    -   Выберите имя домена, который работает с Office 365. Можно использовать имя домена второго уровня? к примеру, buycontoso.com?, но не именем домена третьего уровня?, например marketing.contoso.com. Дополнительные сведения о выборе домена для использования в Office 365 см. в разделе [домены](https://technet.microsoft.com/library/office-365-domains.aspx).  
  
    -   Купите его у регистратора домена, который поддерживает записи сервера (DNS), имя домена, необходимые для Office 365. Чтобы узнать, какие регистраторы доменных имен поддерживают необходимые записи DNS, в разделе [приобретение имени домена](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660). Вы уже зарегистрировали домен у другого регистратора, не беспокоиться; домен можно передать другого регистратора, при подключении домена к Office 365.  
  
2.  **Настройка записей DNS, позволяющих службам Office 365 использовать доменное имя.** Проще всего предоставить возможность мастеру настроить записи DNS при подключении домена к вашей подписке на Office 365 на шаге 3. Если вы d сделать это самостоятельно, см. раздел [записывает Ручная настройка DNS для интеграции Office 365](#BKMK_ManuallyConfigureDNS).  
  
3.  **Привязка вашего настраиваемого домена Интернета к подписке Office 365.** Используйте все **привязка домена к Office 365** действие.  
  
4.  **Убедитесь, что службам Office 365 с помощью нового доменного имени.**  
  
 Если вы завершите шаги 1 и 2, прежде чем использовать **привязка домена к Office 365** задач, мастер может связать доменное имя в Office 365. Кроме того можно использовать мастер для частичного или полного выполнения шагов 1 и 2.  
  
##### <a name="to-link-your-organization-s-internet-domain-to-office-365"></a>Чтобы привязка домена Интернет вашей организации s к Office 365  
  
1.  На панели мониторинга откройте **Office 365** и нажмите кнопку **привязка домена к Office 365**.

     Щелкните ссылку "Справка" на странице мастера, чтобы получить сведения, необходимые для выполнения задачи. Или см. раздел Настройка раздел имя домена [управление удаленного веб-доступа в Windows Server Essentials](https://technet.microsoft.com/library/jj628152\(d=printer\).aspx) Обзор процесса, а также требования.  
  
    > [!NOTE]
    >  Чтобы использовать мастер для регистрации нового доменного имени, необходимо использовать один из поставщиков служб доменных имен, сотрудничающих с Майкрософт обеспечит эффективную интеграцию с помощью мастера. Чтобы найти регистратор доменных имен, в разделе [приобретение имени домена](https://office.microsoft.com/office365-suite-help/how-to-buy-a-domain-name-HA102819883.aspx?CTT=5&origin=HA102818660).  
  
3.  Если мастер определит, что t это имя вашего домена под управлением сервера, вы получите необходимо вручную настроить необходимые записи DNS, чтобы завершить настройку. Инструкции см. в разделе [записывает Ручная настройка DNS для интеграции Office 365](#BKMK_ManuallyConfigureDNS), далее в этом разделе.  
  
4.  Убедитесь, что домен используется в Office 365.  
  
     Здесь s некоторое после завершения работы мастера, пока регистратора доменных имен проверил записи DNS. Это происходит автоматически. Дон t нужно ничего делать. Обычно это занимает около часа? иногда немного дольше. По завершении проверки домена, **Office 365** домен организации s будут отображаться на странице.  
  
####  <a name="BKMK_ManuallyConfigureDNS"></a>Ручная настройка записей DNS для интеграции Office 365  
 Если привязки домена к Office 365 мастер обнаруживает, что имя домена не под управлением сервера, чтобы завершить настройку, необходимо вручную настроить необходимые записи имени сервера (DNS). В этом случае вы получите список записей DNS, которые необходимо настроить на **%username%\NewDNSRecords_ (n) .txt**, где *(n)* является случайным числом.  
  
 В следующей таблице описаны DNS-записи, которые необходимо добавить. Методы ввода могут различаться у различных регистраторов доменных имен. Если у вас возникли вопросы, обратитесь за помощью регистратора имен доменов.  
  
### <a name="required-dns-records-for-linking-a-custom-internet-domain-name-to-office-365"></a>Необходимые записи DNS для привязки настраиваемого доменного имени Интернет к Office 365  
  
|Службы|Необходимые записи DNS|Назначение|  
|-------------|--------------------------|-------------|  
|(Несколько служб)|MX| Office 365 данная запись используется, чтобы убедиться, что вы владеете определенное имя домена. Этот запись MX не мешает маршрутизации сообщений электронной почты.|  
|Exchange Online|MX|Обеспечивает маршрутизацию сообщений электронной почты. **Важно:** при миграции электронной почты, не задавайте предпочтения, равное нулю (**0**) для новой MX-записи. Убедитесь, что значение для записи больше, чем значение, которое назначается текущей записи MX. После завершения миграции электронной почты, и вы готовы приступить к изменению сервера электронной почты в Office 365, имеют ваш регистратор доменных имен сбросил значение предпочтения новой записи MX.|  
|Exchange Online|Псевдоним (CNAME)|Запись автообнаружения, используемый, чтобы пользователи легко устанавливать соединение между Exchange Online и клиентом Outlook для настольного ПК или мобильным почтовым клиентом. **Примечание:** при желании для доступа к Outlook Web Access с помощью имя собственного домена организации s (например, http://mail.contoso.com), а не стандартный URL (https://outlook.com/owa/office365.com) записи псевдонима (CName) можно настроить следующим образом: **тип = CNAME TTL = 01: 00:00, имя узла = "Почта", "адрес = mail.office365.com**|  
|Exchange Online|TXT|Указывает, что outlook.com, домен, используемый сервером электронной почты Office 365 право отправлять электронную почту от имени вашего домена. Создание этой записи, чтобы предотвратить ваша Исходящая почта не будет отмечена как спам.|  
|Lync Online|SRV|Способствует совместной работе с другими службами мгновенных сообщений, таких как Windows Live или Yahoo!.|  
|Lync Online|SRV|Запись автообнаружения, используемый, чтобы пользователи легко устанавливать соединение между клиентом Lync настольного ПК и Microsoft Lync Online.|  
  
> [!IMPORTANT]
>  После домена проверки завершается, не пытайтесь добавить или внести в них изменения записи DNS на портале Office 365.  
  
###  <a name="BKMK_StepFour_ACCOUNTS"></a>Следующий шаг  
  
-   Создайте учетные записи Microsoft Online Services для пользователей.  
  
     Чтобы использовать службы Office 365, пользователям потребуется учетная запись Microsoft Online Services, связанные с их сетевыми учетными записями. Панель мониторинга это легко сделать. Если вы повторно с помощью новой подписки на Office 365 можно массовом создании учетных записей в Интернете для существующих пользовательских учетных записей. Если вы повторно интеграция нового сервера с подпиской на Office 365, что вы уже используете, вы можете импортировать учетные записи пользователей из существующей сети учетные записи. Процедуры см. в разделе [Управление записями интернет-служб для пользователей](Manage-Online-Accounts-for-Users.md).  
  
> [!NOTE]
>  На панели мониторинга в Windows Server Essentials учетные записи Microsoft Online Services называются учетные записи Office 365. Учетные записи, которые совпадают; только терминологии изменен.  
  
##  <a name="BKMK_ManageIntegration"></a>Управление интеграцией Office 365  
 После интеграции сервера с Office 365 **Office 365** на странице панели мониторинга отображает сведения о подписке Office 365 и станут доступны следующие действия:  
  
-   [Управление подпиской Office 365](#BKMK_ManageO365) ? Измените учетную запись администратора, который используется для управления подпиской. Откройте панель мониторинга администратора Office 365 для управления вашей подпиской.  
  
-   [Привязка домена Интернет вашей организации s к Office 365](#BKMK_StepThree) ? Если вы хотите иметь возможность отправлять и получать электронную почту, адресованную на ваш домен, вы можете привязать домена к Office 365. (Описано ранее в [шаг 3: привязка организации s домена к Office 365](#BKMK_StepThree).)  
  
-   [Отключение интеграции Office 365](#BKMK_Disable) ? Если Дон t хотите управлять службами Office 365, подписка и учетных записей в Интернете с помощью панели мониторинга, можно отключить интеграции Office 365. К службам будет осуществляться через портал Office 365.  
  
###  <a name="BKMK_ManageO365"></a>Управление подпиской Office 365  
 Если требуется внести изменения в подписки на Office 365 во время работы на сервере, вы можете открыть подписку в Office 365 с **Office 365** страницы панели мониторинга. Можно также изменить учетной записи администратора, используемую сервером для внесения изменений в службы Office 365.  
  
##### <a name="to-open-your-subscription-on-the-office-365-admin-dashboard"></a>Открытие подписки на информационной панели администратора Office 365  
  
1.  Для мониторинга Windows Server Essentials, открыть **Office 365** страницы.  
  
2.  В **задачи настройки**, нажмите кнопку **управление Office 365**.  
  
3.  Войдите в Office 365 с online учетной записью Майкрософт, который используется для управления вашей подпиской.  
  
     Откроется панель мониторинга администратора Office 365.  
  
##### <a name="to-change-the-office-365-administrator-account"></a>Изменение учетной записи администратора Office 365  
  
1.  На панели мониторинга щелкните **Office 365**.  
  
2.  В **задачи настройки**, нажмите кнопку **изменение учетной записи администратора Office 365**. Появляется мастер изменения учетной записи администратора. (В Windows Server Essentials, мастер называется установить Office 365 учетной записи администратора).  
  
3.  Введите учетные данные учетной записи, которую вы хотите использовать для подключения к подписке Office 365 и нажмите кнопку **Далее**.  
  
4.  Нажмите кнопку **закрыть**. Панель мониторинга перезагрузится.  
  
###  <a name="BKMK_Disable"></a>Отключение интеграции Office 365  
 Если вы решили, что у вас нет t хотите управлять службами Office 365 и учетных записей в Интернете с помощью панели мониторинга, вы можете отключить интеграцию с Office 365. Подписки на Office 365 останется активной, и изменения конфигурации, сделанные с помощью панели мониторинга в силу. Например вы получите получать электронную почту, адресованную доменному имени, которое вы привязали к подписке Office 365. Вы выиграли потеряйте сообщения электронной почты, и элементы управления, которые вы установили для мобильных устройств будут по-прежнему использовать Exchange Online.  
  
 В дальнейшем будет управлять вашей подписки на Office 365, службами и ресурсами в Office 365, и пользователи должны будут управлять паролями для своих записями интернет-служб в Office 365. Синхронизация паролей больше производиться не будет, а отключение или удаление учетной записи пользователя не повлияет на s учетной записи.  
  
 Так как программным обеспечением для интеграции Office 365 устанавливается на локальный сервер, функцию можно отключить, даже если отсутствует соединение службы интеграции с Office 365.  
  
##### <a name="to-disable-office-365-integration-with-the-server"></a>Отключение интеграции Office 365 с сервером  
  
1.  На панели мониторинга щелкните **Office 365**.  
  
2.  Нажмите кнопку **Отключение интеграции Office 365**. Откроется мастер отключения Office 365.  
  
3.  Следуйте инструкциям для завершения работы мастера.  
  
> [!NOTE]
>  Чтобы включить снова интеграции Office 365, используйте **интеграция с Office 365** задач на **службы** мониторинга s **Главная** страницы. Инструкции см. в разделе [шаг 2: интеграция сервера Windows Server Essentials с Microsoft Office 365](#BKMK_StepTwo)ранее в этом разделе.  
  
##  <a name="BKMK_Troubleshoot"></a>Устранение неполадок с интеграцией Office 365  
 Этот раздел содержит информацию, которая поможет вам справиться с распространенными проблемами, возникающими при использовании функций интеграции Office 365 в Windows Server Essentials.  
  
###  <a name="BKMK_AcctsNotCreated"></a>Некоторые учетные записи Microsoft Online Services не были созданы  
 **Описание**  
  
 Попытка создать одну или несколько учетных записей Microsoft Online Services из t невероятно мониторинга успешно.  
  
 **Решение**  
  
1.  Перейдите по ссылке на странице завершения работы мастера, чтобы открыть файл, содержащий детальную информацию о каждом запрос на создание учетной записи не была успешно завершена. Например может оказаться, что с именем запрошенный учетной записи уже существует учетная запись Microsoft Online Services.  
  
2.  Воспользуйтесь рекомендациями для решения каждой отдельной проблемы.  
  
3.  Если проблема остается, перезагрузите сервер и попробуйте создать учетные записи еще раз.  
  
###  <a name="BKMK_ProblemUninstalling"></a>Возникла проблема с удалением интеграции Office 365  
 **Описание**  
  
 Произошла неизвестная ошибка при попытке отключить интеграцию с Office 365.  
  
 **Решение**  
  
1.  Убедитесь, что компьютер подключен к Интернету, а затем повторите попытку.  
  
2.  Если ошибка повторяется, перезапустите сервер, а затем повторите попытку.
