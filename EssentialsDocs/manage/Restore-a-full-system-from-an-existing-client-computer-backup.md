>Область применения: Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials 
  
 Сбои оборудования и операционной системы редки, но они случаются. Неисправность вентилятора может привести к перегреву системной платы компьютера и вывести ее из строя. Операционной системы может быть поврежден и отказаться от запуска. Ущерб от пожара и воды может привести к неустранимым повреждениям оборудования. Жесткий диск может завершиться с ошибкой или вы можете заменить его размер жесткого диска.

##  <a name="BKMK_WhatIs"></a>Что такое полное восстановление системы компьютера?  
 В том случае, если замены жесткого диска или сбоя компьютера, в точку, где нельзя использовать или запуск окажутся невозможными, можно восстановить систему из предыдущей резервной копии компьютера. Полное восстановление системы возвращается состояние, в котором он находился на момент резервного копирования системы.  
  
> [!IMPORTANT]
>  Не удается выполнить полное восстановление системы на компьютере оборудование, например, системная плата не аналогично оборудования компьютера, который требуется заменить. Установленная операционная система тесно связана с аппаратным обеспечением компьютера. Тем не менее можно выполнить полное восстановление системы на жесткий диск, который больше размера или больше, чем то, что и заменяемый.  
  
 При выполнении полного восстановления системы, можно выбрать конкретную архивную копию для восстановления системы со всеми приложениями, настройками и параметрами, существовавшими на момент сбоя, поломки или кражи. Можно также выбрать тома, которые вы хотите восстановить.  
  
 При планировании или подготовке полного восстановления сетевого компьютера следует учесть следующие:  
  
### <a name="windows-preinstallation-environment"></a>Среда предустановки Windows  
 Среда предустановки Windows (Windows PE) — это минимальная операционная система, предназначенная для подготовки компьютера к установке Windows. Для серверов под управлением Windows Server Essentials среда Предустановки Windows устанавливается автоматически при вставке носителя для восстановления на компьютере, который необходимо восстановить. Для серверов под управлением Windows Server Essentials среды Предустановки Windows устанавливается автоматически при запуске компьютера с службу восстановления клиента или USB-накопителя.  
  
 Windows PE не поддерживает беспроводные соединения. Из-за этого восстанавливаемый компьютер должен быть физически подключен к сети для малого бизнеса.  
  
### <a name="bitlocker"></a>BitLocker  
 Шифрование диска BitLocker (BitLocker) — это функция защиты данных, доступный в некоторых версиях Windows Vista, Windows 7 и Windows 8. BitLocker защищает данные от хищения или несанкционированного просмотра на утерянных или украденных компьютерах, а также обеспечивает более надежное удаление данных при списании компьютеров.  
  
 **Для Windows Server Essentials:** Если компьютер, который необходимо восстановить был зашифрован с помощью BitLocker (будь то только диск с операционной системой или диск с операционной системой и один или несколько других жестких дисков), можно по-прежнему использовать носитель для восстановления полной системы содержащихся на компакт-диске с сервера и мастер полного восстановления системы для повторной установки образа жесткого диска, включая операционную систему из резервной копии и восстановления данных на новый или Отремонтированный компьютер.  
  
 **Для Windows Server Essentials:** Если компьютер, который необходимо восстановить был зашифрован с помощью BitLocker (будь то только диск с операционной системой или диск с операционной системой и один или несколько других жестких дисков), по-прежнему можно использовать мастер полного восстановления системы для повторной установки образа жесткого диска, включая операционную систему из резервной копии и восстановления данных на новый или Отремонтированный компьютер.  
  
 Когда сервер создает резервные копии дисков, папок и файлов, на сервер сохраняется незашифрованная версия. Во время полного восстановления системы незашифрованная версия копируется на компьютер.  
  
> [!NOTE]
>  После успешного полного восстановления системы необходимо повторно активировать BitLocker на компьютере.  
>   
>  Инструкции о том, как включить BitLocker на компьютерах под управлением Windows 8 см. в разделе [BitLocker: включение BitLocker](https://go.microsoft.com/fwlink/p/?LinkID=254918).  
>   
>  Инструкции о том, как включить BitLocker на компьютерах, работающих под управлением Windows 7 см. в разделе [шифрования диска BitLocker Step-by-Step руководство по Windows 7](https://go.microsoft.com/fwlink/p/?LinkId=140225).  
  
 Дополнительные сведения об основах шифрования диска BitLocker см. в разделе [BitLocker часто задаваемые вопросы и ответы](https://go.microsoft.com/fwlink/p/?LinkID=254917).  
  
### <a name="encrypting-file-system-encrypted-files"></a>Шифрованная файловая система-шифрованные файлы  
 Функция шифрованной файловой системы (EFS) в Windows обеспечивает дополнительный файл на основе пользователей шифрование на уровне для различных уровней безопасности для нескольких пользователей на одном компьютере. Важно отметить, что в отличие от шифрование с помощью BitLocker диски, файлы и папки, зашифрованные с помощью EFS, остаются зашифрованными в любой архивной копии компьютера. Шифрование EFS недоступно в Windows XP Home Edition, Windows Vista Starter, Windows Vista Home Basic, Windows Vista Home Premium, Windows 7 Начальная, Windows 7 Home Basic, Windows 7 Домашняя расширенная или Windows 8. Он доступен только в Windows 8 Профессиональная.  
  
> [!WARNING]
>  В отличие от BitLocker доступны только те EFS-файлы, защищенные от в операционной системе, в которой выполнялось шифрование.  
  
### <a name="disk-partitions"></a>Разделы диска  
 В случае размер жесткого диска на новом компьютере такой же или больше, чем, исходный жесткий диск автоматически форматируется, а его разделы организовываются заново. См. на рисунке ниже:  
  
|Исходный компьютер|Восстановленный или новый компьютер|  
|-----------------------|------------------------------|  
|Один диск с несколькими разделами|Один диск с несколькими разделами, остаток пространства включается в последний раздел|  
|Один диск с одним разделом|Один диск с одним разделом, все доступное пространство используется под один раздел|  
  
> [!NOTE]
>  Если диск размер различия и расположения разделов между оригинальным и восстановленным или новым компьютером, необходимо использовать Управление дисками для создания соответствующие разделы на восстановленном или новом компьютере. Это можно сделать в мастер полного восстановления системы.  
  
### <a name="raid-and-dynamic-disks"></a>RAID и динамические диски  
 Архивация избыточного массива из независимых дисков (RAID) и динамических дисков не поддерживается.  
  
##  <a name="BKMK_HowDoes"></a>Как работает среда восстановления системы?  
 Носитель для восстановления системы, предоставляемый с ServerÂ Windows® 2012 Essentials среда предустановки Windows (Windows PE) устанавливает на компьютере. Среда Предустановки Windows заменяет среду MS-DOS и содержит основные программные файлы для Windows. В Windows Server Essentials, существует два способа восстановления системы: с помощью службы восстановления клиента, который используется сеть и не зависит от носителя, или с помощью USB-устройства флэш-памяти.  
  
> [!NOTE]
>  Windows PE не поддерживает беспроводные соединения. Из-за этого восстанавливаемый компьютер должен быть физически подключен к сети для малого бизнеса.  
  
 Среда восстановления система поставляется с (x86) 32-разрядных и 64-разрядных (x64) программными файлами. После вставки системы носителя для восстановления, выберите необходимую версию файлов. 32-разрядные (x86) используется по умолчанию и выбирается автоматически, если вы не выберите другой вариант в течение 30 секунд. В случае обновления для полного восстановления системы программные файлы на сервере, обновленные файлы автоматически загружаются на компьютер.  
  
 После настройки среды предустановки запускается мастер полного восстановления системы. Этот мастер помогает вам восстановить компьютер из предыдущей резервной копии. Можно также использовать среду восстановления системы, чтобы восстановить архив на новый компьютер с таким же оборудованием.  
  
 В большинстве случаев программные файлы и драйверы, включенные в среду восстановления системы — все, что нужно для перезагрузки нового или восстановленного компьютера. В зависимости от аппаратного обеспечения нового или восстановленного компьютера среду восстановления системы не может включать все хранилища и сетевые драйверы адаптеров, которые потребуются после перезапуска нового или восстановленного компьютера. Мастер полного восстановления системы дает возможность установить драйверы, при необходимости. Сведения о поиске драйверов для оборудования см. в разделе [где найти драйверы для моего устройства?](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_FindDrivers). Сведения об использовании носителя для восстановления системы см. в разделе [с помощью мастера полного восстановления системы](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_Using).  
  
##  <a name="BKMK_CreateBootable"></a>Создание загрузочного USB-накопитель флэш-памяти для восстановления клиентского компьютера  
 Если вам требуется восстановить клиентский компьютер из существующей резервной копии, но не можете найти компакт-ДИСК восстановления, поставляемый с сервером (в Windows Server Essentials), или вы не хотите настраивать службу восстановления клиента на сервере (в Windows Server Essentials), можно создать загрузочный USB-накопитель. Затем можно использовать USB-накопитель флэш-памяти для запуска клиентского компьютера и восстановления системы. USB-накопитель флэш-памяти, используемой должен быть по крайней мере 1 ГБ или больше.  
  
#### <a name="to-create-a-bootable-usb-flash-drive"></a>Чтобы создать загрузочное USB-устройство флэш-накопителя  
  
1.  Откройте **мониторинга**.  
  
2.  Нажмите кнопку **устройств** вкладку.  
  
3.  В **задачи** панели, щелкните **параметры настройки архивации компьютеров и журнала файлов**.  
  
    > [!NOTE]
    >  В Windows Server Essentials щелкните **задачи архивации клиентских компьютеров**.  
  
4.  Нажмите кнопку **средства**, а затем щелкните **создать ключ** в **восстановление компьютера** раздела. Откроется мастер создания ключа восстановления компьютера.  
  
5.  Вставьте 1 ГБ или больше USB флэш-накопитель на сервере, а затем выполните инструкции в мастере.  
  
    > [!CAUTION]
    >  Будут удалены все данные на USB-накопителя.  
  
##  <a name="BKMK_Using"></a>С помощью мастера восстановления полной системы  
 После успешного использования восстановления мультимедиа, службы восстановления клиента или USB флэш-запустите компьютер и убедитесь, что все оборудование, драйверы загружаются на новый или восстановленный клиентский компьютер, а мастер полного восстановления системы. Этот мастер позволяет получить доступ к серверу, архивной копии компьютера и исходные тома для восстановления на компьютер и выполняется процедура восстановления.  
  
> [!NOTE]
>   Windows Server Essentials не поддерживает следующие сценарии восстановления:  
>   
>  -   Восстановление диска основной загрузочной записи (MBR) на единый интерфейс (UEFI) на компьютере.  
> -   Восстановление резервной копии UEFI/GPT в системе BIOS.  
>   
>  Если данные восстанавливаются в одном из этих сценариев, вы не сможете перезагрузить систему. Кроме того вам может не иметь возможность использовать жесткие диски, размер которых превышает 2 ТБ, размер.  
  
 **Необходимые условия:**  
  
-   Перед началом полного восстановления системы процесс, с помощью сетевого кабеля (проводное соединение) подключите компьютер к той же сети, что и сервер. Убедитесь, что доступ к все жесткие диски на клиентском компьютере.  
  
    > [!WARNING]
    >  Не следует выполнять полное восстановление системы на компьютере, который использует беспроводного подключения к сети.  
  
-   Если вы знаете, что на компьютере отсутствуют важные сети или драйверы запоминающих устройств, необходимо найти эти драйверы и скопировать на флэш-накопитель до начала процесса восстановления полной системы. Для Windows Server Essentials: При использовании носителя для восстановления полной системы, поставляемое на компакт-диска, данный компакт-ДИСК должен оставаться в устройстве чтения в начале процесса восстановления полной системы. Таким образом не следует копировать недостающие драйверы на компакт-ДИСК или DVD-диска, если у вас есть вторая компакт-диска или DVD-диска. Вместо этого скопируйте отсутствующие драйверы на флэш-накопитель USB.  
  
     Сведения о поиске драйверов для компьютера см. в разделе [где найти драйверы для моего устройства?](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_FindDrivers)  
  
-   Для Windows Server Essentials: Если не удается обнаружить Windows Server Essentials восстановления компакт-диска, можно создать загрузочный USB-накопитель. Дополнительные сведения см. в разделе [создать загрузочный USB-накопитель для восстановления клиентского компьютера](Restore-a-full-system-from-an-existing-client-computer-backup.md#BKMK_CreateBootable).  
  
#### <a name="to-use-the-full-system-restore-wizard"></a>Чтобы использовать мастер полного восстановления системы  
  
1.  Выполните одно из следующих действий.  
  
    -   Windows Server Essentials: Включите компьютер, который требуется восстановить, вставьте носитель для восстановления системы, а затем выключите компьютер.  
  
         Повторно включите компьютер и во время система на Self (POST), нажмите соответствующую функциональную клавишу (F-клавишу) для вызова меню загрузки устройств и затем выберите дисковод CD/DVD. Запускается диспетчер загрузки Windows.  
  
    -   Windows Server Essentials: Если вы используете службу восстановления клиента, перезагрузите компьютер с помощью **загрузка из сети** параметр. В противном случае запустите компьютер, используя USB-накопитель.  
  
         Повторно включите компьютер и во время система на Self (POST), нажмите соответствующую функциональную клавишу (F-клавишу) для вызова меню загрузки устройств и затем выберите **загрузка из сети** (или можно выбрать для загрузки с USB-ключ). Запускается диспетчер загрузки Windows.  
  
    > [!NOTE]
    >  Обратитесь к документации производителя компьютера определите какая из функциональных клавиш служит для вызова меню загрузки устройств.  
  
2.  На носителе для восстановления компьютера содержит (x86) 32-разрядных и 64-разрядный (x 64) загрузочный параметры. Диспетчер загрузки Windows выберите **полное восстановление системы (x86)** или **полное восстановление системы (x64)**. Если драйверы аппаратного обеспечения компьютера 32-разрядные, выберите x86; Если они 64-разрядные, выберите x64. Загрузка файлов Windows и мастер полного восстановления системы выполняет проверку, чтобы обеспечить доступность всех драйверов аппаратного обеспечения.  
  
3.  В **мастер полного восстановления системы** окно, выберите предпочтительный язык и щелкните стрелку.  
  
4.  Выберите нужный **формат времени и денежных единиц**и **клавиатуры или метод ввода** для данного компьютера. Нажмите кнопку **по-прежнему**.  
  
5.  Если драйвера отсутствуют, отображается сообщение, что процесс восстановления не может проверить драйверы. Нажмите кнопку **закрыть**и в диалоговом окне приветствия нажмите кнопку **загрузить драйверы**.  
  
    1.  На **определить оборудование** диалоговом нажмите кнопку **устанавливать драйверы**.  
  
    2.  Вставьте флэш-накопитель USB с драйверами устройств, а затем на **Установка драйверов** диалоговом нажмите кнопку **сканирования**.  
  
    3.  На **Установка драйверов** диалоговом нажмите кнопку **ОК** при обнаружении драйверы.  
  
    4.  На **определить оборудование** диалоговом нажмите кнопку **Продолжить**.  
  
6.  Если все необходимые драйверы обнаружены при первичной проверке или если все важнейшие драйверы установлены, в **полного восстановления системы** окно, нажмите кнопку **Продолжить**.  
  
7.  На **Добро пожаловать в мастер полного восстановления системы** щелкните **Далее**.  
  
8.  Мастер выполняет поиск сервера.  
  
    1.  Если мастер не может найти сервер, вам предоставляется возможность выполнить поиск повторно или ввести IP-адрес сервера.  
  
    2.  Если обнаружены несколько серверов, вам будет предложено выбрать один.  
  
    3.  Если ваш сервер найден, **войдите < YourServerName\ >** страница отображается.  
  
9. На **войдите < YourServerName\ >** введите *< AdministratorAccountName\ >* в **имя пользователя** текстовое поле и пароль учетной записи администратора в **пароль** текстовое поле, а затем нажмите кнопку **Далее**.  
  
    > [!IMPORTANT]
    >  Необходимо использовать учетную запись администратора, созданной на английском языке. Если нет, необходимо создать новую учетную запись администратора. Для этого сначала откройте **пользователей** на панели мониторинга сервера, переключите язык клавиатуру на английский и запустите **добавить учетную запись пользователя** задачи для создания учетной записи администратора. Далее используйте новой учетной записи администратора, чтобы продолжить восстановление клиентского компьютера.  
  
10. На **выберите компьютер для восстановления** выберите компьютер, который нужно восстановить и нажмите кнопку **Далее**. Вы также можете выбрать **< ComputerName\ >: (этот компьютер)** или другой компьютер в сети из **другого компьютера** раскрывающегося списка.  
  
    > [!NOTE]
    >  Если это компьютер неизвестен серверу (например, новый или переназначенный компьютер), **этот компьютер** параметр не отображается.  
  
11. На **выберите архив для восстановления** просмотрите список доступных архивов и выберите тот, который требуется восстановить на компьютере.  
  
    > [!NOTE]
    >  Рекомендуется выбрать успешно резервной копии (отмечена зеленым флажком). Это гарантирует, что все системные файлы и данные будут восстановлены успешно.  
  
12. (*Необязательно*) Выберите архив и нажмите кнопку **сведений** открыть **сведения об архивации** и просмотреть дополнительные сведения об этой архивной копии. Используйте сведения в **сведения об архивации** страницу, чтобы Сравните несколько архивных копий, поможет вам решить, какую архивную копию является наилучшим вариантом. Нажмите кнопку **закрыть** на **сведения об архивации** страницы, чтобы вернуться к **выберите архив для восстановления** страницы.  
  
13. На **выберите архив для восстановления** выберите резервной копии и нажмите кнопку **Далее** кнопки.  
  
14. На **выберите параметр восстановления** страницы, выберите один из следующих действий и нажмите кнопку **Далее**.  
  
    > [!NOTE]
    >  Эта страница не отображается, если автоматическое создание разделов не поддерживается.  
  
    1.  **Позволить мастеру полностью восстановить компьютер (рекомендуется)**. Этот вариант гарантирует восстановление компьютера в состояние, в котором он находился перед время и Дата создания архива, который вы выбрали. Если выбрать этот параметр, перейдите к шагу 15.  
  
    2.  **Я выберу тома для восстановления (Дополнительно)**. Этот параметр позволяет выбирать тома, которые вы хотите восстановить и где вы хотите восстановить их. Можно также создать разделы на жестком диске.  
  
15. На **выберите тома для восстановления** страницы, можно выбрать тома, которые вы хотите восстановить.  
  
    > [!NOTE]
    >  Эта страница отображается, если существует несколько жестких дисков на компьютере резервной копии исходной или целевой диск восстановления имеет меньше дискового пространства, чем резервного копирования исходный диск.  
  
    1.  Мастер пытается сопоставить исходного и конечного тома. Следует проверить правильность сопоставления по умолчанию.  
  
        1.  Чтобы снять выделение тома, щелкните на стрелке меню этого тома и нажмите кнопку **нет**.  
  
        2.  После завершения выбора томов нажмите кнопку **Далее**.  
  
    2.  Если размеры тома-источника и конечного тома имеют одинаковый размер, если размер источника меньше конечного, между двумя томами появляется зеленая стрелка. Если размеры томов не совпадают (том в исходном томе больше конечного тома), между источником и конечным появляется красный значок X.  
  
        > [!NOTE]
        >  Красный значок X может также появиться, если.  
        >   
        >  -   Размеры сектора диска исходного тома не соответствует размеры сектора диска конечного тома. Это может произойти при замене физического диска на диск с отличным размером сектора или при настройке дискового пространства (который может иметь с отличным размером сектора отличалась от физического диска).  
        > -   Достигнут максимальный допустимый размер кластера. Чтобы восстановить исходный том в конечном томе, необходимо отформатировать конечный том размер кластера таким же, как и у исходного тома. Если конечный том слишком велик, а размер кластера слишком мал, может достигнут максимальный допустимый размер кластера.  
  
        1.  Нажмите кнопку **открыть диспетчер дисков (Дополнительно)**и создайте новый том такого же размера, как системный том.  
  
            > [!NOTE]
            >  Если клиентский компьютер находится единый интерфейс (UEFI) на основе, необходимо использовать **diskpart** средство для инициализации системного диска. Чтобы сделать это, откройте окно командной строки (нажмите Ctrl + Alt + Shift в течение 5 секунд в среде WinPE), запустите **diskpart.exe**, а затем запустите следующие команды diskpart:  
            >   
            >  1.  **DISKPART > список дисков**  
            > 2.  **DISKPART > выберите диск #***< диск\ср >*  
            > 3.  **DISKPART > очистки**  
            > 4.  **DISKPART > Преобразовать в gpt**  
            > 5.  **DISKPART > создать размер раздела efi =***100* (где *100* — размер раздела в МБ, должен совпадать с размером исходного раздела)  
            > 6.  **DISKPART > создать размер раздела msr =***128* (где *128* — размер раздела в МБ, должен совпадать с размером исходного раздела)  
            > 7.  **DISKPART > exit**  
  
        2.  *(Необязательно) *Выберите параметр **не назначать буквы диска или пути диска**.  
  
        3.  Отформатируйте том как **NTFS**.  
  
        4.  После форматирования щелкните правой кнопкой мыши новый том и нажмите кнопку **отметить раздел как активный**.  
  
        5.  Если требуются дополнительные тома, чтобы они соответствовали других томов в резервной копии, повторите шаги *ii* через *iv* для создания и активации томов, а затем закройте **Управление дисками**.  
  
        6.  На **выберите тома для восстановления** сопоставьте архивный том архивации источника в том же размер, который вы создали в шаге системы *v*.  
  
        7.  Сопоставьте все исходные тома соответствующим конечным томам.  
  
        8.  Нажмите кнопку **Далее** для продолжения восстановления.  
  
16. На **Подтвердите выбор томов для восстановления** страницы, проверьте сопоставления и нажмите кнопку **Далее**. Если необходимо внести изменения, нажмите кнопку **обратно**, а затем повторите шаг 14.  
  
17. **Восстановление < ComputerName\ > из < Дата и время backup\ >** страница отображается ход процесса восстановления.  
  
18. На **Восстановление успешно завершено**, извлеките носитель для восстановления системы и нажмите кнопку **Готово**. Компьютер будет перезагружен.  
  
    > [!IMPORTANT]
    >  Если на компьютере перед началом восстановления было включено шифрование диска BitLocker, необходимо вручную включить BitLocker, после перезагрузки компьютера.  
  
##  <a name="BKMK_FindDrivers"></a>Где найти драйверы для моего устройства?  
 В зависимости от аппаратного обеспечения нового или восстановленного компьютера на носителе для восстановления может включать все хранилища и не драйверы сетевых адаптеров, потребуются после перезапуска восстановленного компьютера. Необходимо определить, какие драйверы отсутствуют, найдите эти драйверы на имеющемся носителе или на веб-сайте изготовителя s, скопировать их на флэш-накопитель и затем скопировать их с флэш-накопителя на новый или восстановленный компьютер при запуске мастера полного восстановления системы.  
  
 При архивации компьютера драйверы для компьютера сохраняются в архивной копии. Носитель для восстановления не содержит все драйверы, необходимые вам, можно открыть архив данного компьютера и скопировать драйверы на флэш-накопитель USB.  
  
#### <a name="to-copy-drivers-from-a-backup-to-a-usb-flash-drive"></a>Копирование драйверов из резервной копии на флэш-накопитель USB  
  
1.  На другом компьютере откройте панель мониторинга.  
  
2.  Нажмите кнопку **устройств**, а затем выберите компьютер, для которого требуются драйверы.  
  
3.  Нажмите кнопку **восстановить файлы или папки компьютера**. Откроется окно мастера восстановления файлов и папок.  
  
4.  Нажмите кнопку последней успешной архивной копии, а затем нажмите кнопку **Далее**.  
  
5.  Выберите том, чтобы открыть, а затем нажмите кнопку **Далее**. Откроется окно со списком файлов и папок в резервную копию.  
  
6.  Вставьте USB-Накопитель флэш в разъем USB на компьютере и скопируйте драйверы для полного восстановления системы папки на флэш-накопитель USB.  
  
    > [!NOTE]
    >  Вы можете потребоваться щелкнуть **на один уровень вверх** вплоть до корневого системного тома.  
  
7.  Удалите флэш-накопитель и вставьте его в компьютере, на котором вы хотите восстановить.  
  
 Флэш-накопитель USB можно использовать для установки драйверов для компьютера при его восстановлении. Восстановить файлы или папки мастер выполняет поиск дополнительных драйверов на данном флэш-накопителе USB при использовании мастера полного восстановления системы. Драйверы, которые вы, скорее всего, потребуются являются драйвера сетевого адаптера и драйверы устройств хранения.
