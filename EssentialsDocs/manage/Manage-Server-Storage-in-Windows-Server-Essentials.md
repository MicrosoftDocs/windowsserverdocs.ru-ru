##  <a name="BKMK_1"></a>Управление жесткими дисками, с помощью панели мониторинга  
 Windows Server Essentials позволяет управлять все жесткие диски, подключенные к серверу через панель мониторинга. На панели мониторинга **хранилища** вкладке **жестких дисков** отображаются все жесткие диски, доступные на сервере для хранения резервных копий данных и сервера. Сервер отслеживает пространство, доступное на каждом жестком диске и отображает предупреждение, если нехватки места на жестком диске. **Жестких дисков** вкладка содержит следующие сведения:  
  
-   Имя каждого жесткого диска  
  
-   Емкость каждого жесткого диска  
  
-   Объем используемого пространства на каждом жестком диске  
  
-   Объем свободного пространства на каждом жестком диске  
  
-   Состояние каждого жесткого диска; пустое состояние означает, что жесткий диск работает правильно  
  
-   Область сведений, где отображается все информация о стеке хранилища (для пула носителей, дисковом пространстве и жестком диске), если выбранный жесткий диск находится в дисковом пространстве (а не физический диск)  
  
 В следующей таблице перечислены задачи управления жесткими дисками, доступные на панели мониторинга и их описания. Некоторые задачи отображаются только в том случае, если выбран жесткий диск.

##  <a name="BKMK_2"></a>Увеличение емкости хранилища на сервере  
 Чтобы увеличить емкость хранилища на сервере, можно добавить дополнительный внутренний жесткий диск к серверу. Чтобы добавить дополнительный внутренний жесткий диск, необходимо завершить работу сервера, добавить внутренний жесткий диск и перезапустите сервер. Необходимо завершить работу сервера, если жесткий диск подключен к SCSI-контроллеру. В этом случае жесткий диск можно подключить работающем сервере.  
  
 В зависимости от того, отформатирован ли жесткий диск должен быть добавлен выполните одно из следующих действий.  
  
-   **Формат** если внутренний жесткий диск отформатирован в NTFS или ReFS, сервер назначает ему букву диска и жесткий диск отображается на **жестких дисков** вкладку. Теперь можно создавать или перемещать папки сервера на новый жесткий диск.  
  
-   **Не отформатирован** если внутренний жесткий диск не отформатирован, выводится следующее уведомление: один или несколько неформатированных жестких дисков, подключенных к серверу. Используйте следующую процедуру для форматирования жесткого диска.  
  
#### <a name="to-format-the-hard-disk"></a>Для форматирования жесткого диска  
  
1.  Откройте панель мониторинга.  
  
2.  В области навигации щелкните значок предупреждения, чтобы запустить **средство просмотра оповещений** в Windows Server Essentials или **наблюдение за работоспособностью** вкладку в Windows Server Essentials.  
  
3.  В **средство просмотра оповещений** или **наблюдение за работоспособностью** вкладку, щелкните оповещение и затем в области задач щелкните **устранения этой проблемы**.  
  
4.  Следуйте инструкциям для завершения мастера добавления нового жесткого диска.  
  
###  <a name="BKMK_Clean"></a>Очистка жесткого диска  
  
1.  Откройте панель мониторинга.  
  
2.  В области навигации щелкните **хранилища**, а затем нажмите кнопку **жестких дисков**.  
  
3.  В **жестких дисков** статьи, выберите букву диска, которая была назначена добавленному жесткому диску и на панели задач, щелкните **Просмотр свойств жесткого диска**.  
  
4.  В **< Driveletter\ > свойства**на **Общие** щелкните **Очистка диска**.  
  
##  <a name="BKMK_Check"></a>Выполнение проверки и восстановления на жестких дисках  
 Проверки и восстановления жестких дисков процесс проверки работоспособности файловой системы, хранящейся на жестких дисках. Он выполняет **chkdsk** процесса на томе, где хранятся файлы резервной копии. Следующие предупреждения могут быть решены путем выполнения проверки и восстановления жестких дисков:  
  
-   Необходимо проверить один или несколько жестких дисков системы архивации данных сервера.  
  
#### <a name="to-check-and-repair-hard-drives"></a>Проверка и восстановление жестких дисков  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **папок сервера и жесткие диски**, а затем нажмите кнопку **жестких дисков**.  
  
3.  Выберите жесткий диск, который отображает ошибку, а затем выберите **Просмотр свойств жесткого диска**.  
  
4.  На **проверка и восстановление** щелкните **проверка и восстановление**.  
  
##  <a name="BKMK_3"></a>Форматирование жестких дисков  
 При обнаружении неформатированного внутреннего жесткого диска на сервере оповещение о работоспособности руководство пользователя по процессу форматирование. Добавить мастер жесткого диска поможет выполнить форматирование жесткого диска и позволяет настроить жесткий диск одним из следующих способов:  
  
1.  Отформатировать жесткий диск и автоматически создать диск на нем. Если выбран этот вариант, после завершения работы мастера, создается один логический жесткий диск, отформатированный в файловой системе NTFS.  
  
2.  Отформатировать жесткий диск и настроить его для архивации данных сервера. Если выбрать этот параметр, запускается мастер настройки сервера архивации и поможет выполнить настройку архивации сервера.  
  
3.  Если t дисковые пространства существует, используйте новый жесткий диск для создания дискового пространства. Необходимо иметь по крайней мере два жестких дисков для создания дискового пространства.  
  
4.  Если дисковое пространство уже существует, используйте новый жесткий диск для увеличения емкости пула носителей. Этот параметр отображается только в том случае, если на сервере создано дисковое пространство. Если выбран этот вариант, мастер добавит этот жесткий диск в пул носителей.

###  <a name="BKMK_4a"></a>Используйте новый жесткий диск для хранения папок сервера  
 Чтобы использовать новый жесткий диск для хранения папок сервера, можно добавить новую папку сервера на жесткий диск или переместить существующую папку сервера на жесткий диск.  
  
##### <a name="to-store-server-folders"></a>Для хранения папок сервера  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **ХРАНИЛИЩА** , а затем щелкните **папок сервера**.  
  
3.  В **задачи для папок сервера** область, выполните одно из следующих действий:  
  
    1.  Чтобы добавить папку сервера, щелкните **добавить папку**.  
  
    2.  Чтобы переместить папку сервера, выберите папки, которую вы хотите переместить на новый жесткий диск, а затем нажмите кнопку **переместить папку**.  
  
    > [!NOTE]
    >  Если перейти к жесткому диску и выберите его в качестве папки сервера без создания папки, появится следующее сообщение об ошибке: **корневой каталог (например, C:\\ D:\\) невозможно добавить в качестве папки сервера. Создайте новую папку или выберите существующую папку корневого каталога и повторите попытку**. Чтобы устранить эту ошибку, создайте новую папку на добавленном жестком диске и выберите новую папку в качестве расположения для хранения папок сервера.  
  
4.  Следуйте инструкциям для завершения работы мастера.  
  
 Дополнительные сведения о перемещении папок сервера см. в разделе [Добавление или перемещение папки сервера](Manage-Server-Folders-in-Windows-Server-Essentials.md#BKMK_5).  
  
###  <a name="BKMK_4b"></a>Использование нового жесткого диска для хранения резервных копий сервера  
 Новый добавленный жесткий диск можно использовать для хранения резервных копий сервера.  
  
##### <a name="to-store-server-backups"></a>Для хранения резервных копий сервера  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **устройств** вкладку, выберите сервер из панели списка и затем в области задач выполните одно из следующих:  
  
    1.  Если Архивация сервера не настроена на сервере, щелкните **Настройка архивации сервера**.  
  
    2.  Если Архивация сервера настроена на сервере, щелкните **Настройка архивации сервера**.  
  
     Появится мастер настройки сервера архивации.  
  
3.  В **Выбор места назначения архивации** выберите новый жесткий диск в качестве места назначения архивации.  
  
4.  Следуйте инструкциям для завершения работы мастера.  
  
###  <a name="BKMK_4c"></a>Использование нового жесткого диска для увеличения емкости пула носителей  
 При недостаточной емкости пула носителей появится оповещение о том, что может увеличить емкость пула носителей путем добавления нового жесткого диска в пул носителей с помощью мастера увеличения емкости мастер пула носителей.  
  
> [!NOTE]
>  Эту процедуру можно выполнить только в том случае, если вы создали пул носителей на сервере.  
  
##### <a name="to-increase-storage-pool-capacity"></a>Для увеличения емкости пула носителей  
  
1.  Откройте панель мониторинга.  
  
2.  Нажмите кнопку **хранилища** , а затем щелкните **жестких дисков**.  
  
3.  Выберите диск, который недостаточной емкостью.  
  
4.  На панели задач выберите **увеличение емкости пула носителей**. Откроется мастер увеличения емкости хранилища пула.  
  
5.  Следуйте инструкциям для завершения работы мастера.  
  
##  <a name="BKMK_5"></a>Обзор дисковых пространств  
 Дисковые пространства позволяют сгруппировать объединять диски в пуле носителей. Затем емкость пула можно использовать для создания дисковых пространств. Дисковые пространства представляют собой виртуальные диски, которые отображаются на **жестких дисков** панели мониторинга. Можно использовать дисковые пространства аналогично другим дискам, поэтому он s легко работать с файлами на них. В случае нехватки емкости пула, можно создавать большие дисковые пространства и добавлять новые диски в пуле носителей. При наличии двух или нескольких дисков в пуле носителей можно создать дисковые пространства с двусторонним зеркалом, которые не будут затронуты при сбое диска? или даже при сбое двух дисков? Если создать дисковое пространство трехстороннее зеркало.  
  
 Для создания дискового пространства, все, что вам нужно — один или несколько дисков в дополнение к этому на диск, на котором установлена ОС Windows. Эти диски можно внутренние или внешние жесткие диски и твердотельные накопители. Можно использовать различные типы дисков с помощью дисковых пространств, в том числе диски, USB, SATA и SAS.  
  
> [!NOTE]
>  При настройке дискового пространства на сервере под управлением Windows Server Essentials нельзя выполнить сброс параметров с **очистить данные** параметр. Возможным решением этой проблемы является удаление дисковых пространств и последующее выполнение сброса настроек с помощью **очистить данные** параметр.  
  
 Дополнительные сведения о дисковых пространствах см. в разделе [хранилища пространств часто задаваемые вопросы](https://social.technet.microsoft.com/wiki/contents/articles/11382.storage-spaces-frequently-asked-questions-faq.aspx).  
  
##  <a name="BKMK_6"></a>Создание дискового пространства  
 Чтобы начать работу с дисковыми пространствами на сервере, должны выполняться следующие минимальные требования:  
  
-   Сервер под управлением Windows Server Essentials должен быть подключен к дополнительным физическим дискам (не только к загрузочному диску), диски должны быть без томов и иметь минимальный объем 10 ГБ. Необходим один физический диск для создания пула носителей; для создания устойчивого зеркального дискового пространства необходимы как минимум два физических диска.  
  
-   Для создания дискового пространства посредством четности или двустороннего зеркального отображения необходимы как минимум два физических диска.  
  
> [!NOTE]
>  При настройке дискового пространства на сервере под управлением Windows Server Essentials нельзя выполнить сброс параметров с **очистить данные** параметр. Возможным решением этой проблемы является удаление дисковых пространств и последующее выполнение сброса настроек с помощью **очистить данные** параметр.  
  
#### <a name="to-create-a-storage-space-in-windows-server-essentials"></a>Для создания дискового пространства в Windows Server Essentials  
  
1.  Добавьте или подключите все диски, которые вы хотите сгруппировать в дисковом пространстве на сервер под управлением Windows Server Essentials.  
  
2.  С помощью панели мониторинга щелкните **Дополнительно: управление дисковыми пространствами**.  
  
3.  Нажмите кнопку **создать новый пул и дисковое пространство**.  
  
4.  Выберите диски, которые вы хотите добавить в новое дисковое пространство, а затем нажмите кнопку **Создание пула**.  
  
5.  Присвойте имя диску имя и буквы, а затем выберите макет. **Двухстороннее зеркало**, **трехстороннее зеркало**, и **четности** может помочь защитить файлы в места хранения от сбоя диска.  
  
6.  Введите значение максимального размера дискового пространства можно достичь и нажмите кнопку **создать дисковое пространство**.  
  
 В Windows Server Essentials вы можно создать двустороннее зеркальное дисковое пространство с помощью создания мастер пространства хранилища с помощью панели мониторинга.  
  
#### <a name="to-create-a-storage-space-in-windows-server-essentials"></a>Для создания дискового пространства в Windows Server Essentials  
  
1.  Добавьте или подключите все диски, которые вы хотите сгруппировать в дисковом пространстве на сервер под управлением Windows Server Essentials.  
  
2.  С помощью панели мониторинга щелкните **управление дисковыми пространствами**. Откроется мастер пространства хранилища создания.  
  
3.  Следуйте инструкциям для завершения работы мастера.  
  
 Сведения об увеличении емкости пула носителей см. в разделе [использование нового жесткого диска для увеличения емкости пула носителей](Manage-Server-Storage-in-Windows-Server-Essentials.md#BKMK_4c).
