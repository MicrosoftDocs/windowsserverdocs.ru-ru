# <a name="manage-online-backup-in-windows-server-essentials"></a>Управление оперативной архивацией в Windows Server Essentials

>Область применения. Windows Server 2016 Essentials, Windows Server 2012 R2 Essentials, Windows Server 2012 Essentials
  
 После интеграции с Microsoft Azure Backup **оперативной архивации** откроется страница управления в панели мониторинга Windows Server Essentials. Страница **Оперативная архивация** позволяет выполнять стандартные задачи администрирования. На странице оперативной архивации доступны следующие функции.  
  
- Страницы подразделов.  
  
  -   **Оперативная архивация** после регистрации сервера для оперативной архивации, в этом разделе отображается текущий статус архивации, статус хранения и сведения об учетной записи.  
  
  -   **Защищенные папки** в этом разделе перечислены все общие папки и все папки журнала файлов на сервере и все остальные папки, которые вы выбрали для резервного копирования в Azure. Этот список включает имя папки, путь к ней и статус каждой общей папки.  
  
  -   **Журнал резервного копирования** В этом разделе отображается список недавно выполненных оперативных архиваций. В списке указан тип операции, время и статус каждой операции резервного копирования.  
  
- Область сведений с дополнительной информацией о выбранном задании резервного копирования или восстановления.  
  
- Область задач, которая включает набор доступных административных задач.  
  
  Рекомендуется выполнять резервное копирование наиболее важных бизнес-данных и пользовательских данных. Например, необходимо архивировать серверные папки, содержащие файлы с критически важными данными. Кроме того, необходимо выполнять резервное копирование журнала файлов для сетевых компьютеров, содержащих критическую информацию.  
  
  При принятии решения о том, какие данные должны быть оперативно архивированы, необходимо учитывать периодичность архивации и реализуемую политику хранения. Затем необходимо проанализировать финансовые возможности организации и определить, хранилище данных какого объема она может себе позволить. Соотнеся стоимость и объем ресурсов для хранения данных с потребностями организации, вы сможете настроить оперативную архивацию так, чтобы защитить максимальный объем важных данных. Сведения о ценах см. в разделе [Сведения о ценах на архивацию Azure](https://azure.microsoft.com/pricing/details/backup/).  
  
  В следующих разделах описываются различные задачи оперативной архивации, которые могут отображаться на панели мониторинга Windows Server Essentials.

###  <a name="BKMK_1"></a> Отправить сертификат в хранилище службы архивации Azure  
 Прежде чем использовать службу архивации Azure для оперативного резервного копирования в Windows Server Essentials, необходимо передать открытый сертификат для регистрации в хранилище резервных копий. Сертификат используется для проверки подлинности Azure Backup развертывания (агента), действуя от имени владельца подписки Microsoft Online Services для управления ресурсами, связанными с подпиской.  
  
> [!NOTE]
>  Перед загрузкой сертификата необходимо выполнить все процедуры из раздела [Sign up for Azure Backup Service](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_16).  
  
##### <a name="to-upload-a-certificate-to-use-with-the-azure-backup-service"></a>Загрузка сертификата для использования в службе резервного копирования Azure  
  
1. Выполните вход на панель мониторинга Windows Server Essentials с правами администратора.  
  
2. На странице панели мониторинга **Главная** щелкните **ОПЕРАТИВНАЯ АРХИВАЦИЯ**.  
  
3. В области **ОПЕРАТИВНАЯ АРХИВАЦИЯ** щелкните **Отправить сертификат в резервное хранилище**.  
  
    Откроется окно **служб восстановления** на портале управления Azure. Если вы не вошли Azure, нужно будет выполнить вход с использованием учетной записи Майкрософт.  
  
4. Щелкните имя резервного хранилища, которое будет использоваться для оперативной архивации, чтобы открыть страницу **Быстрый запуск** резервного хранилища.  
  
5. Щелкните **Управление сертификатами**.  
  
6. В диалоговом окне **Управление сертификатами** вставьте путь к общему сертификату, созданному Windows Server Essentials. Для получения пути к общему сертификату выполните следующие действия.  
  
   1.  На панели мониторинга Windows Server Essentials щелкните вкладку **Оперативная архивация** .  
  
   2.  На странице **Оперативная архивация** копируйте путь к созданному сертификату.  
  
   3.  Перейдите на портал управления Azure, а затем в **Управление сертификатом** диалоговом окне вставьте путь для загрузки созданного общего сертификата.  
  
   > [!NOTE]
   >  Кроме того, можно использовать собственный общий сертификат. Чтобы узнать, какой нужен сертификат, на странице **Быстрый запуск** щелкните ссылку **Получить сертификат**.  
  
   > [!NOTE]
   >   Azure требует использовать сертификат x.509 v2 с открытым ключом. См. также [Управление сертификатами хранилищ](https://msdn.microsoft.com/library/azure/dn169036.aspx).  
  
7. После выбора сертификата щелкните **ОК** (флажок).  
  
8. После этого вновь отобразится страница **Быстрый запуск** . Щелкните **Панель мониторинга**и убедитесь, что сертификат успешно загружен. После успешной загрузки сертификата на панели мониторинга отобразится эскиз сертификата и срок его действия.  
  
   По окончании данной процедуры выполните следующее.  
  
9. Зарегистрируйте сервер в службе архивации Azure. Дополнительные сведения см. в разделе [Регистрация сервера для резервного копирования](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_5).  
  
10. Настройте оперативную архивацию сервера. Дополнительные сведения см. в разделе [Configure online backup](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_2).  
  
###  <a name="BKMK_2"></a> Настройка оперативного резервного копирования  
 После регистрации сервера в службе Azure Backup, можно настроить параметры оперативной архивации в Windows Server Essentials.  
  
##### <a name="to-configure-online-backup"></a>Настройка оперативной архивации  
  
1.  В мастере регистрации сервера или на странице **Оперативная архивация** панели мониторинга Windows Server Essentials щелкните **Настроить оперативную архивацию**. Отобразится мастер настройки оперативной архивации.  
  
2.  На **Настройка оперативной архивации** странице мастера установите флажок для каждой серверной папки, для резервного копирования в Azure Backup. Очистите флажки для всех серверных папок, которые не требуется копировать. Для добавления не отображенных в списке папок щелкните **Добавить папки**. Завершив выделение, нажмите кнопку **Далее**.  
  
    > [!NOTE]
    >  Невозможно выделить папку, которая не расположена на локальном сервере, или папку на диске, форматированном в ReFS.  
  
3.  На странице **Добавить резервные копии журнала файлов** мастера можно указать, что требуется включать резервные копии журнала файлов для сетевых компьютеров, поддерживающих функцию ведения журнала файлов. Затем щелкните **Далее**, чтобы продолжить.  
  
4.  На странице **Создание расписания архивации** мастера настройте следующие параметры и нажмите кнопку **Далее**, чтобы продолжить.  
  
    -   Выберите или примите требуемое время запуска резервного копирования. Время по умолчанию — 22:00. Можно также указать желаемое время запуска второго копирования.  
  
    -   Выберите или примите дни, когда должна выполняться интерактивная архивация. По умолчанию оперативная архивация выполняется с понедельника по пятницу.  
  
5.  На странице **Создание политики хранения резервных копий** мастера укажите продолжительность хранения оперативных резервных копий в днях, а затем нажмите кнопку **Далее**. По умолчанию копии хранятся 7 дней. Можно также хранить копии 15 или 30 дней.  
  
    > [!NOTE]
    >   Служба Azure Backup всегда сохраняет самой последней резервной копии. Если в месте архивации недостаточно пространства для хранения резервной копии, процедура архивации завершится сбоем. Во избежание этого необходимо приобрести дополнительное пространство для хранения или сократить продолжительность хранения данных. Сведения о ценах см. в разделе [сведения о ценах на](https://azure.microsoft.com/pricing/details/backup/) архивации Microsoft Azure.  
  
6.  На странице **Выбор уровня использования пропускной способности** мастера, если требуется ограничить пропускную способность Интернета, выделяемую на резервное копирование, выберите **Включить использование пропускной способности Интернета**. Используйте параметры на этой странице, чтобы указать, какую часть пропускной способности Интернета можно расходовать на оперативную архивацию в рабочее и нерабочее время. Затем укажите рабочее время и рабочие дни вашей организации.  
  
    > [!NOTE]
    >   Служба архивации Azure позволяет настраивать использование пропускной способности сети, при резервном копировании и восстановлении информации во программным обеспечением для интеграции. При использовании метода, известную как регулирование можно управлять объемом пропускной способности сети, которая доступна для использования в резервной копии и восстановление процессов во время определенного дня и интервалы времени. Установив флажок **Включить использование пропускной способности Интернета** в мастере настройки оперативной архивации, можно указать, в течение каких рабочих дней и рабочих часов нужно применять ограничение пропускной способности для рабочего времени. На протяжении всего остального времени используется ограничение для нерабочего времени. Допустимые диапазоны пропускной способности — от 256 Кбит/с до неограниченного в обоих случаях.  
  
7.  По окончании конфигурации оперативной архивации щелкните **Закрыть**.  
  
    > [!TIP]
    >  После успешной настройки резервного копирования на странице **Оперативная архивация** отобразится статус последней оперативной архивации и объем пространства для хранения, использованный резервным хранилищем. Для просмотра статуса предыдущих операций архивации щелкните **Журнал резервного копирования**.  
  
###  <a name="BKMK_3"></a> Запуск оперативной архивации  
  
> [!NOTE]
>  Прежде чем начать оперативную архивацию, необходимо сначала выполнить операции, описанные в разделе [Register this server for backup](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_5), а затем операции, описанные в разделе [Configure online backup](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_2).  
  
##### <a name="to-start-an-online-backup-immediately"></a>Немедленный запуск оперативной архивации  
  
1.  Выполните вход на панель мониторинга с правами администратора.  
  
2.  На панели навигации щелкните **ОПЕРАТИВНАЯ АРХИВАЦИЯ**.  
  
3.  В области **Задачи оперативной архивации** щелкните **Архивировать сейчас**.  
  
###  <a name="BKMK_4"></a> Восстановление файлов и папок из оперативного архива  
 Мастер восстановления файлов и папок поможет найти, выбрать и восстановить оперативно архивированные файлы и папки. Пользователь выполнит в мастере следующие задачи.  
  
1.  **Выберите оперативный архив, из которого необходимо восстановить файлы и папки**  
  
     При работе с мастером восстановления файлов и папок в первую очередь необходимо указать, нужно ли восстановить файлы из оперативного архива на сервере, на который в настоящее время выполнен вход, либо из архива с другого сервера.  
  
2.  **Выберите сервер, с которого требуется восстановить файлы и папки**  
  
     Если выбрано восстановление файлов и папок с другого сервера, выберите сервер, с которого требуется выполнить восстановление, в списке доступных серверов, а затем нажмите кнопку **Далее**.  
  
3.  **Выберите том, содержащий файлы и папки для восстановления**  
  
     Интерактивные архивы содержат тома, соответствующие томам на исходном сервере резервного копирования. После выбора тома выберите дату и время создания архива, из которого требуется выполнить восстановление, а затем нажмите кнопку **Далее**.  
  
4.  **Выберите файлы и папки для восстановления**  
  
     На странице мастера **Выбор элементов для восстановления** можно открыть разные местоположения папок и отметить флажками все файлы и папки, которые требуется восстановить. Завершив выбор файлов, нажмите кнопку **Далее**.  
  
5.  **Укажите конечное местоположение для восстановленных файлов и папок**  
  
     На странице мастера **Выбор параметров восстановления** можно указать место восстановления файлов и папок. Возможны два варианта.  
  
    -   **Восстановление в исходном местоположении**. Выберите этот вариант, чтобы восстановить файлы и папки именно там, откуда они были изначально копированы во время архивации.  
  
    -   **Восстановление в другом местоположении**.  Выберите этот вариант, чтобы указать другое местоположение на сервере, где требуется восстановить файлы.  
  
         В установке по умолчанию мастер создает копию исходного файла, если файл с таким же именем, что и восстановленный, уже существует в конечном местоположении. Кроме того, список управления доступом обновляется, чтобы отразить текущие разрешения на работу с файлами. Можно щелкнуть **Дополнительно**, чтобы настроить параметры по умолчанию.  
  
6.  **Подтвердите сведения о восстановлении**  
  
     На странице **Подтверждение сведений о восстановлении** обобщены заданные пользователем параметры восстановления. Чтобы продолжить процедуру восстановления файлов, необходимо указать правильную парольную фразу для учетной записи оперативной архивации.  
  
###  <a name="BKMK_5"></a> Регистрация сервера для резервного копирования  
 Для резервного копирования или восстановления файлов, папок и журнала файлов на сервере Windows Server Essentials в службе архивации Azure, необходимо сначала зарегистрировать сервер в службе архивации Microsoft Azure.  
  
> [!NOTE]
>  Перед регистрацией сервера необходимо загрузить сертификат для использования в резервном хранилище, где будут сохраняться оперативные архивные копии. Дополнительные сведения см. в разделе [Отправка сертификата в резервное хранилище Azure](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_1).  
  
##### <a name="to-register-your-server-with-azure-backup"></a>Регистрация сервера в службе резервного копирования Azure  
  
1.  Выполните вход на сервер с правами администратора и откройте панель мониторинга.  
  
2.  На панели навигации щелкните **ОПЕРАТИВНАЯ АРХИВАЦИЯ**.  
  
3.  В подразделе **Оперативная архивация** щелкните **Регистрировать**.  
  
4.  Выберите сертификат, который требуется использовать в резервном хранилище для оперативных резервных копий. Сертификат по умолчанию выбирается по умолчанию; если требуется выбрать другой сертификат, воспользуйтесь кнопкой **Обзор**. Затем нажмите кнопку **Далее**.  
  
5.  Следуйте инструкциям в мастере для создания парольной фразы и завершения регистрации.  
  
###  <a name="BKMK_6"></a> Отмена регистрации сервера  
  
> [!CAUTION]
>  После отмены регистрации сервера Windows Server Essentials с службой Microsoft Azure Backup, Azure Backup не резервное копирование сервера. Кроме того, будут удалены ранее загруженные данные сервера. Чтобы возобновить оперативную архивацию, потребуется снова зарегистрировать сервер.  
  
##### <a name="to-unregister-your-server-with-azure-backup"></a>Отмена регистрации сервера в службе резервного копирования Azure  
  
1.  Войдите [на портал управления Azure](https://manage.windowsazure.com).  
  
2.  Щелкните **Службы восстановления**.  
  
3.  В разделе **Службы восстановления**щелкните имя резервного хранилища.

###  <a name="BKMK_16"></a> Подпишитесь на службу Azure Backup  
 Чтобы подготовиться к интеграции Microsoft Azure Backup с Windows Server Essentials, будет вход на портал управления Azure с помощью учетной записи Microsoft Online Services и затем создать резервное хранилище для хранения оперативных архивных копий в Azure. Затем скачайте модуль интеграции резервного копирования Azure и использовать скачанный файл для установки надстройки резервного копирования Azure на сервере Windows Server Essentials. Если у вас нет учетной записи Майкрософт, можно зарегистрироваться для получения бесплатной пробной версии.  
  
 Для реализации этой настройки выполните следующие задачи.  
  
1.  Зарегистрируйтесь для получения учетной записи в веб-службах Майкрософт и предварительного просмотра резервных копий.  
  
2.  Создайте резервное хранилище для сохранения оперативных архивных копий.  
  
3.  Скачайте агент службы архивации Azure.  
  
4.  Установка надстройки резервного копирования Azure на сервере.  
  
####  <a name="BKMK_SignupforaMicrosoftOnlineServiceAccount"></a> Регистрация учетной записи Microsoft Online Services и предварительного просмотра резервных копий  
  
1.  Выполните вход на панель мониторинга Windows Server Essentials.  
  
2.  На странице **Главная** панели мониторинга щелкните категорию **НАДСТРОЙКИ**, **Интеграция со службой резервного копирования Azure** и выберите команду **Щелкните, чтобы зарегистрироваться в службе резервного копирования Azure**.  
  
3.  В Azure **служб восстановления** странице **резервное копирование (Предварительный просмотр)** , просмотрите сведения.  
  
4.  Если у вас нет подписки Azure, нажмите кнопку **бесплатной пробной версии**, а затем следуйте инструкциям, чтобы получить подписку Azure.  
  
     Если у вас уже есть подписка Azure, нажмите кнопку **портала** в правом верхнем углу веб-страницы, чтобы перейти на портал управления Azure.  
  
5.  На странице портала управления Azure, вы увидите **служб восстановления** в левой области. Это, служащей для управления хранилищами, в которых хранятся оперативные архивные копии из Windows Server Essentials.  
  
####  <a name="BKMK_Createabackupvaulttostoreonlinebackups"></a> Создайте резервное хранилище для хранения оперативных архивных копий  
  
1.  Выполните вход на [портал управления Azure](https://manage.windowsazure.com)из веб-браузера на сервере Windows Server Essentials.  
  
2.  На портале управления Azure щелкните **New**, нажмите кнопку **Data Services**, нажмите кнопку **служб восстановления**, нажмите кнопку **резервное хранилище**, а затем Нажмите кнопку **быстрое создание**.  
  
3.  Введите имя резервного хранилища, выберите область для сохранения резервных копий и щелкните **Быстрое создание**.  
  
     Отобразится область **Службы восстановления** с новым резервным хранилищем.  
  
####  <a name="BKMK_DownloadtheWindowsAzureBackupAgent"></a> Скачайте агент службы архивации Azure  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На странице **Главная** панели мониторинга щелкните вкладку **Начать** , выберите категорию **НАДСТРОЙКИ** , а затем последовательно щелкните **Интеграция со службой резервного копирования Azure**и **Щелкните, чтобы загрузить модуль интеграции со службой резервного копирования Azure**.  
  
####  <a name="BKMK_InstalltheWindowsAzureBackupAddIn"></a> Установка надстройки резервного копирования Azure на сервере  
  
1. Выполните вход на сервер с учетной записью администратора, а затем запустите загруженный в предыдущем шаге файл **OnlineBackupAddin.wssx** .  
  
2. Отобразится страница **Условия лицензионного соглашения на использование ПО**. Если вы согласны с указанными условиями, щелкните **Принять**, чтобы продолжить установку.  
  
3. На странице **Установка надстройки** щелкните **Установить надстройку**.  
  
   > [!NOTE]
   >  Возможно, потребуется перезапустить сервер, чтобы установить требуемое ПО.  
  
    Отобразится страница **Установка**. После начала установки отображается индикатор хода установки. По завершении установки вы получите сообщение, что надстройка Azure Backup установлен успешно.  
  
4. Нажмите кнопку **Готово**.  
  
5. Закройте и снова откройте панель мониторинга.  
  
    На ней появится новая вкладка — **Оперативная архивация**. На этой вкладке можно регистрировать сервер, настраивать параметры резервного копирования и откройте **служб восстановления** на портале управления Azure для управления хранилищами серверов.  
  
   По окончании этих действий выполните следующее.  
  
6. В Windows Server Essentials отправьте сертификат, используемый для оперативного резервного копирования. Дополнительные сведения см. в разделе [Отправка сертификата в резервное хранилище Azure](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_1).  
  
7. Зарегистрируйте сервер в хранилище службы архивации Azure. Дополнительные сведения см. в разделе [Регистрация сервера для резервного копирования](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_5).  
  
8. Настройте оперативную архивацию сервера. Дополнительные сведения см. в разделе [Configure online backup](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_2).  
  
###  <a name="BKMK_17"></a> Интеграция службы архивации Azure с Windows Server Essentials  
 Модуль интеграции службы архивации Microsoft Azure — это новая функция Windows Server Essentials, для шифрования и резервного копирования файлов и папок с вашего сервера в систему хранения, размещенной в Azure, предоставленных корпорацией Майкрософт. С помощью Azure Backup для шифрования и резервного копирования данных на сервере, можно предотвратить катастрофическую потерю важных данных в результате пожара, наводнения, кражи и других чрезвычайных происшествий. При использовании Azure Backup для резервного копирования данных сервера информация шифруется с использованием парольной фразы, затем отправляется в защищенный ЦОД в Интернете. Для доступа к данным из оперативного архива требуются сервер, прошедший аутентификацию с помощью сертификата, и парольная фраза.  
  
 После интеграции и регистрации сервера в службе Azure Backup, можно настроить параметры оперативной архивации для выполнения регулярных плановых архиваций. Можно в любой момент инициировать оперативную архивацию, щелкнув задачу **Архивировать сейчас** на панели мониторинга оперативной архивации.  
  
 Настройка оперативной архивации подразумевает выполнение следующих действий.  
  
1.  [Зарегистрируйтесь для службы архивации Azure](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_16) — после регистрации в службе резервного копирования будет создание резервного хранилища, загрузить и установить модуль интеграции службы архивации Azure.  
  
2.  [Отправить сертификат в хранилище службы архивации Azure](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_1)  
  
3.  [Регистрация сервера для резервного копирования](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_5)  
  
4.  [Настройка оперативного резервного копирования](Manage-Online-Backup-in-Windows-Server-Essentials.md#BKMK_2)  
  
> [!NOTE]
>   Служба архивации Azure использует парольную фразу для шифрования файлов и папок для оперативной архивации. При изменении парольной фразы шифрования меняется и парольная фраза. указанная при регистрации сервера. Парольная фраза может состоять только из символов в кодировке ASCII.

|**Операции:**|Возможны два типа операций: **Резервное копирование** и **Восстановление**.|  
|**Время:**|Время регистрации в журнале наиболее актуального состояния.|  
|**Состояние**|Существует пять состояний - **успех**, **выполняется**, **Canceled**, **предупреждение**, и **сбой**.|
