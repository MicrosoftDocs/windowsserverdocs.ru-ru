###  <a name="BKMK_Manage1"></a>Добавить учетную запись пользователя  
 При добавлении учетной записи пользователя назначенный пользователь может входить в сеть, и можно предоставить пользователю разрешение осуществлять доступ к сетевым ресурсам, например общим папкам и сайту удаленного веб-доступа. Windows Server Essentials содержит мастер учетной записи пользователя, которая позволяет добавить:  
  
-   Укажите имя и пароль для учетной записи пользователя.  
  
-   Определите учетную запись с правами администратора или обычного пользователя.  
  
-   Выберите общие папки, которые могут получить доступ к учетной записи пользователя.  
  
-   Укажите, имеет ли учетная запись пользователя удаленный доступ к сети.  
  
-   Выберите параметры электронной почты, если это применимо.  
  
-   Назначаете учетную запись Microsoft Online Services (так называемой учетной записи Office 365 в Windows Server Essentials), если это применимо.  
  
-   Назначьте группы пользователей (только для Windows Server Essentials).  
  
> [!NOTE]
>  -   Символы не из набора ASCII, не поддерживаются в Microsoft Azure Active Directory (Azure AD). Не используйте в свой пароль, никакие символы не из набора ASCII, если ваш сервер интегрирован с Azure AD.  
> -   Параметры электронной почты доступны только в том случае, если устанавливается надстройка, предоставляющая службу электронной почты.  
  
##### <a name="to-add-a-user-account"></a>Чтобы добавить учетную запись пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В **задачи пользователей** панели, щелкните **добавить учетную запись пользователя**. Откроется мастер создания учетной записи пользователя добавления.  
  
4.  Следуйте инструкциям для завершения работы мастера.  
  
###  <a name="BKMK_Remove"></a>Удаление учетной записи пользователя  
 Если вы решили удалить учетную запись пользователя с сервера, мастер удалит выбранную учетную запись. По этой причине может больше не использовать учетную запись для входа в сеть или для доступ к сетевым ресурсам. В качестве альтернативы можно также удалить файлы для учетной записи пользователя в то же время удаления учетной записи. Если вы не хотите окончательно удалять учетную запись пользователя, можно отключить учетную запись пользователя вместо нее для приостановки доступа к сетевым ресурсам.  
  
> [!IMPORTANT]
>  Если учетная запись пользователя Microsoft online назначить учетной записи при удалении учетной записи пользователя, эта учетная запись также будет удалена из Microsoft Online Services и s данные пользователя, включая сообщения электронной почты, будут подчиняться данных политики хранения в Microsoft Online Services. Если вы хотите сохранить данные пользователя для учетной записи Интернет, отключите учетную запись пользователя, а отключите ее. Дополнительные сведения см. в разделе [Управление записями интернет-служб для пользователей](Manage-Online-Accounts-for-Users.md).  
  
##### <a name="to-remove-a-user-account"></a>Чтобы удалить учетную запись пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, вы хотите удалить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **удалить учетную запись пользователя**. Появляется мастер создания учетной записи пользователя удаления.  
  
5.  На **вы хотите сохранить файлы? ** страницу мастера, вы можете удалить файлы пользователя s, включая резервные копии истории файлов и перенаправленную папку для учетной записи пользователя. Чтобы сохранить файлы s для пользователя, оставьте флажок пустым. После выбора, нажмите кнопку **Далее**.  
  
6.  Нажмите кнопку **удалить учетную запись**.  
  
> [!NOTE]
>  После удаления учетной записи пользователя учетная запись больше не отображается в списке учетных записей пользователей. Если вы решили удалить файлы, сервер окончательно удалит папку пользователя s из **пользователей** папки сервера и из **резервных копий истории файлов** папки сервера.  
>   
>  Если у вас есть поставщик интегрированной электронной почты, будут удалены и учетную запись электронной почты, назначенная данной учетной записи пользователя.  
  
###  <a name="BKMK_Manage3"></a>Просмотр учетных записей пользователей  
 **Пользователей** панели мониторинга Windows Server Essentials отображается список учетных записей пользователей сети. Список также содержит дополнительные сведения о каждой учетной записи.  
  
##### <a name="to-view-a-list-of-user-accounts"></a>Чтобы просмотреть список учетных записей пользователей  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На главной панели навигации, щелкните **пользователей**.  
  
3.  На панели мониторинга отображается текущий список учетных записей пользователей.  
  
##### <a name="to-view-or-change-properties-for-a-user-account"></a>Просмотр или изменение свойств учетной записи пользователя  
  
1.  В списке учетных записей пользователей выберите учетную запись, для которого требуется просмотреть или изменить свойства.  
  
2.  В **задачи < пользователя Account\ >** панели, щелкните **просмотреть свойства учетной записи**. **Свойства** страница учетной записи пользователя.  
  
3.  Откройте вкладку для отображения свойств этого компонента учетной записи.  
  
4.  Чтобы сохранить изменения, внесенные в свойства учетной записи пользователя, нажмите кнопку **применить**.  
  
###  <a name="BKMK_Manage4"></a>Изменение отображаемого имени учетной записи пользователя  
 Отображаемое имя — имя, которое появляется в **имя** столбец **пользователей** страницы панели мониторинга. Изменение отображаемого имени не изменяет входа в систему или учетное имя для учетной записи пользователя.  
  
##### <a name="to-change-the-display-name-for-a-user-account"></a>Чтобы изменить отображаемое имя для учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, вы хотите изменить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **просмотреть свойства учетной записи**. **Свойства** страница учетной записи пользователя.  
  
5.  На **Общие** введите новый **имя** и **Фамилия** учетной записи пользователя, а затем нажмите кнопку **ОК**.  
  
     Новое отображаемое имя появляется в списке учетных записей пользователей.  
  
###  <a name="BKMK_Manage5"></a>Активация учетной записи пользователя  
 При активации учетной записи пользователя назначенный пользователь может войти сети и доступа к сетевым ресурсам, к которым учетная запись имеет разрешения, например к общим папкам и сайту удаленного веб-доступа.  
  
> [!NOTE]
>  Можно активировать только ту учетную запись пользователя, которая отключена. Нельзя активировать учетную запись пользователя, после того как вы удалите с сервера.  
  
##### <a name="to-activate-a-user-account"></a>Для активации учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В представлении списка выберите учетную запись пользователя, который вы хотите активировать.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **активировать учетную запись пользователя**.  
  
5.  В окне подтверждения нажмите кнопку **Да** для подтверждения этого действия.  
  
> [!NOTE]
>  После активации учетной записи пользователя отображаются состояния учетной записи **Active**. Учетная запись пользователя получает те же права доступа, которые были назначены до ее отключения.  
>   
>  Если у вас есть поставщик интегрированной электронной почты, учетную запись электронной почты, назначенная данной учетной записи пользователя также будет активирована.  
  
###  <a name="BKMK_Manage6"></a>Отключение учетной записи пользователя  
 При отключении учетной записи пользователя, учетная запись доступа к серверу временно приостанавливается. Таким образом назначенный пользователь нельзя использовать учетную запись для доступа к сетевым ресурсам, например к общим папкам или сайту удаленного веб-доступа, пока не будет активировать учетную запись.  
  
 Если учетная запись пользователя Microsoft online назначена учетная запись, эта учетная запись также будет отключена. Пользователь не может использовать ресурсы в Office 365 и других сетевых службах, которые вы подписаны, но данные пользователя s, включая сообщения электронной почты, сохраняются в Microsoft Online Services.  
  
> [!NOTE]
>  Можно отключить только ту учетную запись пользователя, в данный момент.  
  
##### <a name="to-deactivate-a-user-account"></a>Для отключения учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В представлении списка выберите учетную запись пользователя, которую необходимо отключить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **отключить учетную запись пользователя**.  
  
5.  В окне подтверждения нажмите кнопку **Да** для подтверждения этого действия.  
  
> [!NOTE]
>  После отключения учетной записи пользователя отображаются состояния учетной записи **неактивно**.  
>   
>  Если у вас есть поставщик интегрированной электронной почты, учетную запись электронной почты, назначенная данной учетной записи пользователя также будет отключена.  
  
###  <a name="BKMK_Manage7"></a>Обзор учетных записей пользователей  
 Учетная запись пользователя предоставляет важные сведения для Windows Server Essentials, которые позволяют пользователям получать доступ к информации, хранящейся на сервере и дает возможность отдельным пользователям создавать и управлять своими файлами и параметрами. Пользователи могут входить любой компьютер в сети, если у них есть учетная запись пользователя Windows Server Essentials, и у них есть разрешения на доступ к компьютеру. Доступ к их учетных записей свое имя пользователя и пароль.  
  
 Существует два основных типа учетных записей пользователей. Каждый тип предоставляет пользователю разные возможности по управлению компьютером:  
  
-   **Стандартный** учетные записи предназначены для выполнения повседневных действий. Стандартная учетная запись позволяет защищать сеть, предотвращая внесение изменений, влияющих на других пользователей, например удаление файлов или изменение параметров сети.  
  
-   **Администратор** учетные записи позволяют полностью управлять компьютерной сетью. Администратору следует назначить учетной записи только при необходимости.  
  
###  <a name="BKMK_Manage8"></a>Управление учетными записями пользователей, с помощью панели мониторинга  
 Windows Server Essentials позволяет выполнять стандартные административные задачи с помощью панели мониторинга Windows Server Essentials. По умолчанию **пользователей** страницы панели мониторинга содержит две вкладки **пользователей** и **пользователи группы**.  
  
> [!NOTE]
>  -   Если интеграции сервера под управлением Windows Server Essentials с Office 365 новую вкладку называются **группы рассылки** добавляется **пользователей** страницы панели мониторинга.  
> -   В Windows Server Essentials **пользователей** страницы панели мониторинга имеется только одна вкладка — **пользователей**.  
  
 **Пользователей** вкладка содержит следующие:  
  
-   Список учетных записей пользователей, который отображает:  
  
    -   Имя пользователя.  
  
    -   Имя входа для учетной записи пользователя.  
  
    -   Имеет ли учетная запись пользователя разрешение на Повсеместный доступ. Разрешение повсеместного доступа для учетной записи пользователя имеет либо **разрешено** или **запрещено**.  
  
    -   Сервер под управлением Windows Server Essentials управляет ли историей файлов для данной учетной записи пользователя. История файлов для учетной записи пользователя имеет состояние **управляемые** или **неуправляемый**.  
  
    -   Уровень доступа, назначаемый учетной записи пользователя. Можно назначить **обычного пользователя** доступа или **администратора** доступ для учетной записи пользователя.  
  
    -   Состояние учетной записи пользователя. Учетная запись пользователя может быть **Active**, **неактивно**, или **неполный**.  
  
    -   В Windows Server Essentials Если сервер интегрирован с Office 365 или Windows Intune, отображается учетной записи Microsoft online.  
  
    -   Если сервер интегрирован с Microsoft Office 365 в Windows Server Essentials отображается состояние учетной записи Office 365 (известных в Windows Server Essentials в качестве учетной записи Microsoft online) для учетной записи пользователя.  
  
-   Область сведений с дополнительной информацией об учетной записи выбранного пользователя.  
  
-   Область задач, которая включает в себя:  
  
    -   Набор задач администрирования учетной записи пользователя, например Просмотр и удаление учетных записей пользователей и изменение паролей.  
  
    -   Задачи, которые позволяют глобально задавать или изменять параметры для всех учетных записей пользователей в сети.  
  
 В следующей таблице описаны различные задачи учетной записи пользователя, доступные в **пользователей** вкладку. Некоторые задачи, конкретной учетной записи пользователя, и они видны только при выборе учетной записи пользователя в списке.  
  
> [!NOTE]
>  При интеграции Office 365 с Windows Server Essentials, будут доступны дополнительные задачи. Дополнительные сведения см. в разделе [Управление записями интернет-служб для пользователей](Manage-Online-Accounts-for-Users.md).

###  <a name="BKMK_Access1"></a>Изменение или сброс пароля для учетной записи пользователя  
 Чтобы изменить или сбросить пароль учетной записи пользователя, выполните следующие действия.  
  
##### <a name="to-reset-the-password-for-a-user-account"></a>Сброс пароля для учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, которого нужно сбросить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **изменить пароль учетной записи пользователя**. Появляется мастер изменения пароля учетной записи пользователя.  
  
5.  Введите новый пароль для учетной записи пользователя, а затем введите пароль еще раз, чтобы подтвердить его.  
  
6.  Нажмите кнопку **Смена пароля**.  
  
7.  Предоставьте новый пароль пользователю.  
  
    > [!IMPORTANT]
    >  -   Вы не сможете изменить пароль, если значение политики паролей для вашей учетной записи **пароль никогда не истекает**.  
    > -   Символы не из набора ASCII, не поддерживаются в Azure AD. Таким образом Если ваш сервер интегрирован с Azure AD, не используйте символы не из набора ASCII в пароле.  
    > -   Если пользователю назначено учетная запись Microsoft online (известных в Windows Server Essentials в качестве учетной записи Office 365), то пароль синхронизируется с паролем учетной записи в сети. Пользователь будет использовать новый пароль для входа в систему на сервере или входа в Office 365. Дополнительные сведения см. в разделе [Управление записями интернет-служб для пользователей](Manage-Online-Accounts-for-Users.md).  
  
###  <a name="BKMK_Access3"></a>Что следует знать о политиках паролей  
 Политика паролей — набор правил, которые определяют, каким образом создаются и используются пароли. Политика позволяет предотвратить несанкционированный доступ к данным пользователя и другие сведения, которые хранятся на сервере. Политика паролей применяется ко всем учетным записям пользователей, доступ к сети.  
  
 Политика паролей Windows Server Essentials содержит следующие три основных элемента:  
  
-   **Длина пароля**.  Чем длиннее пароль, тем он надежнее. Пустые пароли не являются надежными.  
  
-   **Сложный пароль**.  Сложные пароли содержат строчные и прописные буквы (a-z, A-Z), цифры (0 – 9) и неалфавитные символы (например,!, @, #, _,-). Сложные пароли гораздо меньше уязвимы для несанкционированного доступа. Пароли, которые содержат имена пользователей, даты рождения или прочие персональные данные не обеспечивают достаточный уровень безопасности.  
  
-   **Срок действия пароля**.  Windows Server Essentials требуется, чтобы пользователи меняли свои пароли по крайней мере раз в 180 дней. В качестве альтернативы Вы можете иметь пароли, которые никогда не истекает.  
  
 Для упрощения реализации политики паролей в сети в системе Windows Server Essentials имеется простой инструмент, позволяющий устанавливать или изменять политику паролей для всех следующих предварительно определенных профилей политики:  
  
-   **Слабый**.  Можно указать любой непустой пароль.  
  
-   **Средний**.  Эти пароли должны содержать по крайней мере, 5 символов. Сложный пароль не требуется.  
  
-   **Средний или высокий**.  Эти пароли должны содержать по крайней мере, 5 символов и должна содержать буквы, цифры и символы.  
  
-   **Высокий**.  Эти пароли должны содержать менее 7 символов и должна содержать буквы, цифры и символы. Эти пароли являются более надежными, но могут оказаться более сложными для запоминания.  
  
    > [!NOTE]
    >  Пароли не должны содержать адрес имя или адрес электронной почты пользователя.  
    >   
    >  При интеграции с Office 365 принудительно **строгих** политика паролей и обновление политики для соблюдения следующих требований:  
    >   
    >  -   Пароли должны содержать 8 16 символов.  
    > -   Пароли не могут содержать пробел или имя электронной почты Office 365.  
  
 По умолчанию при установке сервера задается политика паролей по умолчанию **строгих** параметр.  
  
###  <a name="BKMK_Access4"></a>Изменение политики паролей  
 Чтобы задать или изменить политику паролей на любой из четырех предварительно определенных профилей политики используйте следующую процедуру.  
  
##### <a name="to-change-the-password-policy"></a>Чтобы изменить политику паролей  
  
1.  Откройте панель мониторинга Windows Server Essentials, а затем нажмите кнопку **пользователей**.  
  
2.  В **задачи пользователей** панели, щелкните **задать политику паролей**.  
  
3.  На **изменить политику паролей** установите уровень надежности паролей с помощью ползунка.  
  
     Корпорация Майкрософт рекомендует настроить надежность паролей **строгих**.  
  
    > [!NOTE]
    >  В качестве альтернативы можно выбрать **пароль никогда не истекает**. Этот уровень является менее надежным, и поэтому не рекомендуется.  
  
4.  Нажмите кнопку **изменить политику**.  
  
###  <a name="BKMK_Access5"></a>Уровень доступа к общим папкам  
 Рекомендуется назначать наиболее строгие доступные разрешения, позволяющие пользователям выполнять необходимые задачи.  
  
 У вас есть три уровня доступа для общих папок на сервере:  
  
-   **Чтение и запись**.  Выберите этот параметр, если вы хотите разрешить учетной записи пользователя разрешение на создание, изменение и удаление всех файлов в общей папке.  
  
-   **Только для чтения**.  Выберите этот параметр, если вы хотите разрешить учетной записи пользователя разрешение только на чтение файлов в общей папке. Учетные записи пользователей с доступом только для чтения не может создать, изменить или удалить все файлы в общей папке.  
  
-   **Нет доступа**.  Выберите этот параметр, если не хотите, чтобы учетная запись пользователя, доступ к файлам в общей папке.  
  
###  <a name="BKMK_Access6"></a>Сохранение и управление доступом к файлам для удаленных учетных записей  
 Администратор сети может удалить учетную запись пользователя и продолжать пользователь s файлы для использования в будущем. В этом сценарии удаленная учетная запись пользователя больше не используются для входа в сеть; Тем не менее файлы для этого пользователя будет сохранен в общей папке, которая может использоваться совместно с другим пользователем.  
  
> [!IMPORTANT]
>  Имейте в виду, что при удалении учетной записи Microsoft online назначена учетная эта учетная запись также будет удалена и данные пользователя, включая сообщения электронной почты, будут подчиняться политикам хранения данных в Microsoft Online Services. Чтобы сохранить данные пользователя для учетной записи Интернет, отключите учетную запись пользователя, а отключите ее. Дополнительные сведения см. в разделе [Управление записями интернет-служб для пользователей](Manage-Online-Accounts-for-Users.md).  
  
##### <a name="to-remove-a-user-account-but-retain-access-to-the-user-s-files"></a>Удаление учетной записи пользователя с сохранением доступа к файлам пользователя s  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, вы хотите удалить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **удалить учетную запись пользователя**. Появляется мастер создания учетной записи пользователя удаления.  
  
5.  На **вы хотите сохранить файлы? ** странице, убедитесь, что **удалить файлы, включая резервные копии истории файлов и перенаправленную папку для данной учетной записи пользователя** снят флажок и нажмите кнопку **Далее**.  
  
     Появляется страница подтверждения, предупреждающая об удалении учетной записи, но сохранении файлов.  
  
6.  Нажмите кнопку **удалить учетную запись** для удаления учетной записи пользователя.  
  
 После удаления учетной записи пользователя администратор может предоставить другой учетной записи доступ пользователя к общей папке.  
  
##### <a name="to-give-a-user-account-permission-to-access-a-shared-folder"></a>Чтобы предоставить учетной записи пользователя разрешения на доступ к общей папке  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **хранилища**, а затем нажмите кнопку **папок сервера** вкладку.  
  
3.  В списке папок выберите **пользователей** папки.  
  
4.  В **задачи пользователей** панели, щелкните **откройте папку**. Проводник Windows открывает и отображает содержимое **пользователей** папки.  
  
5.  Щелкните правой кнопкой мыши папку для учетной записи пользователя, который вы хотите поделиться, а затем нажмите кнопку **свойства**.  
  
6.  В **< пользователя Account\ > свойства**, нажмите кнопку **общим доступом** , а затем щелкните **общего ресурса**.  
  
7.  В **общий доступ к файлам** окна введите или выберите имя учетной записи пользователя, для которой необходимо установить общий доступ к папке, а затем нажмите кнопку **добавить**.  
  
8.  Выберите **уровень разрешений** , требуется учетная запись пользователя, а затем нажмите кнопку **общего ресурса**.  
  
###  <a name="BKMK_Access7"></a>Синхронизация пароля DSRM с паролем администратора сети  
 Режим восстановления служб каталогов (DSRM) — специальный режим загрузки для восстановления Active Directory. Операционная система использует DSRM для входа компьютер, если Active Directory не удается, или необходимо восстановить. Если пароль администратора сети и пароля DSRM не совпадают, то DSRM не будет загружен.  
  
 Во время первой установки Windows Server Essentials программа задает пароль DSRM пароль учетной записи администратора сети, который вводится во время установки или в файле ответов миграции. При изменении пароля администратора сети (рекомендуется менять раз в 60 дней для обеспечения требуемой безопасности сервера), Смена пароля в DSRM не пересылаются. Это приводит к происходит несоответствие паролей. В этом случае можно использовать следующие решения для вручную или автоматически синхронизировать пароль s администратора сети с паролем DSRM.  
  
##### <a name="to-manually-synchronize-the-dsrm-password-to-a-network-administrator-account"></a>Синхронизация вручную пароля DSRM с учетной записью администратора сети  
  
1.  В командной строке выполните `ntdsutil.exe` чтобы открыть инструмент ntdsutil.  
  
2.  Для сброса пароля DSRM введите **пароль dsrm набора**.  
  
3.  Для синхронизации пароля DSRM в контроллере домена с текущей сети s учетной записи администратора, введите:  
  
     **Синхронизация из учетной записи домена** *< текущая_учетная_запись_администратора_сети >*, и нажмите клавишу ВВОД.  
  
 Так как вы будете периодически изменять пароль для учетной записи администратора сети, чтобы убедиться, что пароль DSRM всегда совпадает с текущим паролем администратора сети, рекомендуется создать запланированную задачу для автоматической ежедневной синхронизации пароля DSRM с паролем администратора сети.  
  
##### <a name="to-automatically-synchronize-the-dsrm-password-to-a-network-administrator-account"></a>Автоматическая синхронизация пароля DSRM с учетной записью администратора сети  
  
1.  На сервере, откройте **Администрирование**, а затем дважды щелкните **планировщик**.  
  
2.  В планировщике заданий **действия** панели, щелкните **создать задачу**.  
  
3.  В **имя** текстовом поле введите имя для задачи, например **автоматическая синхронизация пароля DSRM**, а затем выберите **выполнить с наивысшими правами** параметр.  
  
4.  Определите время задачи:  
  
    1.  В **создать задачу** диалоговом нажмите кнопку **триггеры** , а затем щелкните **New**.  
  
    2.  В **новый триггер** диалоговом выберите режим повторения, укажите интервал повторения и установите время начала.  
  
        > [!NOTE]
        >  Рекомендуется следует задать выполнение задачи ежедневно во время в нерабочее время.  
  
    3.  Нажмите кнопку **ОК** для сохранения изменений и возврата к **создать задачу** диалоговое окно.  
  
5.  Определите действия задач:  
  
    1.  Нажмите кнопку **действия** , а затем щелкните **New**. **Новое действие** откроется диалоговое окно.  
  
    2.  В **действие** выберите **запустить программу**и перейдите к **C:\WINDOWS\SYSTEM32\ntdsutil.exe**.  
  
    3.  В **добавить аргументы**(необязательно) введите следующее (необходимо заключить в кавычки): **задание синхронизации пароля dsrm с учетной записи домена SBS_network_administrator_account q q** где *SBS_network_administrator_account* текущее имя учетной записи администратора s сети.  
  
6.  Нажмите кнопку **ОК** два раза, чтобы сохранить задачу и закрыть **создать задачу** диалоговое окно. Новая задача появляется в **активных задач** раздел **расписании задач**.  
  
###  <a name="BKMK_Access8"></a>Предоставьте учетным записям пользователей разрешения для удаленного рабочего стола  
 В установку по умолчанию в Windows Server Essentials пользователи сети не имеют разрешение на удаленное подключение к компьютерам и другим ресурсам в сети.  
  
 Прежде чем пользователи сети смогут установить удаленное подключение к сетевым ресурсам, необходимо сначала разрешить Повсеместный доступ. После настройки повсеместного доступа пользователям файлов, приложений и компьютеров в сети с устройства в любом месте с подключением к Интернету.  
  
 Мастер настройки повсеместного доступа позволяет включить два метода удаленного доступа:  
  
-   Виртуальной частной сети (VPN)  
  
-   Удаленный веб-доступ  
  
 При запуске мастера также можно разрешить Повсеместный доступ для всех текущих и добавленных учетных записей.  
  
 Чтобы настроить Повсеместный доступ, откройте панель мониторинга **Главная** щелкните **установки**, а затем нажмите кнопку **Настройка повсеместного доступа**.  
  
 Дополнительные сведения о повсеместном доступе см. в разделе [управление повсеместным доступом](Manage-Anywhere-Access-in-Windows-Server-Essentials.md).  
  
###  <a name="BKMK_Access9"></a>Разрешить пользователям доступ к ресурсам на сервере  
  Этот раздел относится к серверу под управлением Windows Server Essentials или Windows Server Essentials, или на сервер под управлением Windows Server 2012 R2 Standard или Windows Server 2012 R2 Datacenter с установленной ролью Windows Server Essentials Experience.  
  
 Если требуется, чтобы пользователи, чтобы использовать удаленный доступ, и (или) имели отдельные учетные записи после завершения подключения компьютера к серверу, созданием новой сети учетных записей для пользователей компьютера, подключенного на сервере с помощью панели мониторинга. Дополнительные сведения о создании учетной записи пользователя см. в разделе [добавить учетную запись пользователя](Manage-User-Accounts-in-Windows-Server-Essentials.md#BKMK_Manage1). После создания учетных записей пользователей, сведения имени и пароле пользователя сети необходимо предоставить пользователям клиентских компьютеров, чтобы они могут получить доступ к ресурсам на сервере с помощью панели запуска.  
  
 Для каждой создаваемой учетной записи пользователя можно настроить доступ к следующие через пользовательский свойства учетной записи:  
  
-   **Общие папки**.  По умолчанию администраторы сети имеют **чтения и записи** разрешение для всех общих папок и учетные записи обычных пользователей имеют **только для чтения** разрешения на папку компании. Если включена функция потоковой передачи мультимедиа, можно назначить права доступа к папкам для отдельных стандартных учетных записей пользователей применительно к следующим общим папкам: **Музыка**, **изображения**, **ТВ-записи**, и **видео**. Можно задать разрешения для учетных записей пользователей для доступа к общим папкам на **общих папок** свойств учетной записи пользователя.  
  
-   **Повсеместный доступ**.  По умолчанию администраторы сети могут использовать VPN или удаленный веб-доступ для доступа к ресурсам сервера. Для учетных записей обычного пользователя, необходимо настроить учетную запись пользователя на **повсеместного доступа** вкладку.  
  
-   **Доступ к компьютеру**.  По умолчанию администраторы сети могут доступ ко всем компьютерам в сети. Однако для учетных записей обычного пользователя можно настроить индивидуальные разрешения на доступ к компьютерам в сети на **доступ к компьютеру** свойств учетной записи пользователя.  
  
##### <a name="to-edit-user-account-properties-in-windows-server-essentials-2012-r2"></a>Изменение свойств учетной записи пользователя в Windows Server Essentials 2012 R2  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, которую требуется изменить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **просмотреть свойства учетной записи**.  
  
5.  В **< пользователя Account\ > свойства**, выполните следующие действия:  
  
    1.  На **общих папок** вкладку, при необходимости установите требуемые разрешения для каждой общей папки.  
  
    2.  На **повсеместного доступа** вкладки:  
  
        1.  Чтобы разрешить пользователю подключаться к серверу с помощью VPN, установите **разрешить виртуальную частную сеть (VPN)** флажок.  
  
        2.  Чтобы разрешить пользователю подключаться к серверу с помощью удаленного веб-доступа, выберите **разрешить удаленный веб-доступ и доступ к веб-приложениям служб** флажок.  
  
    3.  На **доступ к компьютеру** выберите сетевые компьютеры, что хотели бы получить пользователя получать доступ к.  
  
##### <a name="to-edit-user-account-properties-in-windows-server-essentials-2012"></a>Изменение свойств учетной записи пользователя в Windows Server Essentials 2012

5.  В **< пользователя Account\ > свойства**, выполните следующие действия:  
  
    1.  На **Общие** выберите **пользователь может просматривать оповещения о работоспособности сети** Если учетной записи пользователя нужен доступ к отчетам о работоспособности сети.  
  
    2.  На **общих папок** вкладку, при необходимости установите требуемые разрешения для каждой общей папки.  
  
    3.  На **повсеместного доступа** вкладки:  
  
        1.  Чтобы разрешить пользователю подключаться к серверу с помощью VPN, установите **разрешить виртуальную частную сеть (VPN)** флажок.  
  
        2.  Чтобы разрешить пользователю подключаться к серверу с помощью удаленного веб-доступа, выберите **разрешить удаленный веб-доступ и доступ к веб-приложениям служб** флажок.  
  
    4.  На **доступ к компьютеру** выберите сетевые компьютеры, что хотели бы получить пользователя получать доступ к.  
  
###  <a name="BKMK_Access10"></a>Измените разрешения удаленного доступа для учетной записи пользователя  
 Пользователь может получить доступ к ресурсам, расположенным на сервере, из удаленного расположения с помощью виртуальной частной сети (VPN), удаленного веб-доступа или других приложений веб-служб. По умолчанию разрешения удаленного доступа включены для пользователей сети при настройке повсеместного доступа в Windows Server Essentials с помощью панели мониторинга.  
  
##### <a name="to-change-remote-access-permissions-for-a-user-account"></a>Чтобы изменить разрешения удаленного доступа для учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, вы хотите изменить.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **просмотреть свойства учетной записи**. **Свойства** страница учетной записи пользователя.  
  
5.  На **повсеместного доступа** вкладку, выполните следующие действия:  
  
    -   Выберите **разрешить виртуальную частную сеть (VPN)** флажок, чтобы разрешить пользователю подключаться к серверу с помощью VPN.  
  
    -   Выберите **разрешить удаленный веб-доступ и доступ к веб-приложениям служб** флажок, чтобы разрешить пользователю подключаться к серверу с помощью удаленного веб-доступа.  
  
6.  Нажмите кнопку **применить**, а затем нажмите кнопку **ОК**.  
  
###  <a name="BKMK_Access11"></a>Изменение разрешений на виртуальную частную сеть для учетной записи пользователя  
 К виртуальной частной сети (VPN) можно использовать для подключения к Windows Server Essentials и доступа к ресурсам, которые хранятся на сервере. Это особенно полезно, если у вас есть клиентский компьютер, который настроен с учетными записями сети, которые могут использоваться для подключения к размещенному серверу Windows Server Essentials через VPN-подключение. Все вновь создаваемые пользовательские учетные записи на размещенном сервере Windows Server Essentials необходимо использовать VPN для входа клиентский компьютер в первый раз.  
  
##### <a name="to-change-vpn-permissions-for-network-users"></a>Изменение разрешений VPN для пользователей сети  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, к которому требуется предоставить разрешения на удаленный доступ к рабочему столу.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **свойства**.  
  
5.  В **< пользователя Account\ > свойства**, нажмите кнопку **повсеместного доступа** вкладку.  
  
6.  На **повсеместного доступа** , чтобы разрешить пользователю подключаться к серверу с помощью VPN, установите **разрешить виртуальную частную сеть (VPN)** флажок.  
  
7.  Нажмите кнопку **применить**, а затем нажмите кнопку **ОК**.  
  
###  <a name="BKMK_Access12"></a>Изменение доступа к внутренним общим папкам для учетной записи пользователя  
 Вы можете управлять доступом к общим папкам на сервере с помощью задач на **папок сервера** панели мониторинга. По умолчанию при установке Windows Server Essentials создаются следующие папки сервера:  
  
-   **Резервные копии клиентских компьютеров**.  Используется для хранения резервных копий клиентских компьютеров, созданных системой архивации данных Windows Server. Эта папка сервера не используется совместно.  
  
-   **Компания**.  Используется для хранения и доступа к документам, относящихся к вашей организации пользователями сети.  
  
-   **Резервные копии истории файлов**.  По умолчанию Windows Server Essentials хранятся резервные копии файлов, созданных с помощью истории файлов. Эта папка сервера не используется совместно.  
  
-   **Перенаправление папок**.  Используется для хранения и доступа к папкам, которые заданы для перенаправления папок пользователей сети. Эта папка сервера не используется совместно.  
  
-   **Музыка**.  Используется для хранения и доступа к музыкальным файлам пользователями сети. Эта папка создается при включении общего доступа к мультимедиа.  
  
-   **Изображения**.  Используется для хранения и доступа пользователей сети к рисункам. Эта папка создается при включении общего доступа к мультимедиа.  
  
-   **ТВ-записи**.  Используется для хранения и доступа к ТВ-записям пользователей сети. Эта папка создается при включении общего доступа к мультимедиа.  
  
-   **Видео**.  Используется для хранения и доступа пользователей сети к видеофайлам. Эта папка создается при включении общего доступа к мультимедиа.  
  
-   **Пользователи**.  Используется для хранения и доступа пользователей сети к файлам. Пользовательская папка создается автоматически в **пользователей** папки сервера для каждой учетной записи пользователя сети, который можно создать.  
  
##### <a name="to-change-access-to-a-shared-folder-for-a-user-account"></a>Изменение доступа к общей папке для учетной записи пользователя  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  Нажмите кнопку **ХРАНИЛИЩА**, а затем нажмите кнопку **папок сервера**.  
  
3.  Перейдите в раздел и выберите папку сервера, для которого требуется изменить разрешения.  
  
4.  В области задач щелкните **просмотреть свойства папки**.  
  
5.  В **< FolderName\ > свойства**, нажмите кнопку **общим доступом**и выберите уровень доступа пользователя для учетных записей пользователей в списке и нажмите кнопку **применить**.  
  
    > [!NOTE]
    >  Не удается изменить разрешения на общий доступ для **резервных копий истории файлов**, **перенаправления папок**, и **пользователей** папок сервера. Следовательно, не следует включать в окнах свойств этих папок сервера **общим доступом** вкладку.  
  
###  <a name="BKMK_Access13"></a>Разрешить учетные записи пользователей установить сеанс удаленного рабочего стола на компьютере  
  Этот раздел относится к серверу под управлением Windows Server Essentials или Windows Server Essentials, или на сервер под управлением Windows Server 2012 R2 Standard или Windows Server 2012 R2 Datacenter с установленной ролью Windows Server Essentials Experience.  
  
 Администратор сети может предоставить разрешения пользователям сети, которые позволяют им получать доступ к своим сетевым компьютерам из удаленного расположения.  
  
##### <a name="to-enable-users-to-access-their-network-computers-from-a-remote-location"></a>Чтобы пользователи могли получить доступ к своим сетевым компьютерам из удаленного расположения  
  
1.  Откройте панель мониторинга Windows Server Essentials.  
  
2.  На панели навигации щелкните **пользователей**.  
  
3.  В списке учетных записей пользователей выберите учетную запись пользователя, необходимо предоставить разрешения на удаленный доступ к рабочему столу.  
  
4.  В **задачи < пользователя Account\ >** панели, щелкните **свойства**.  
  
5.  В **< пользователя Account\ > свойства**, нажмите кнопку **доступ к компьютеру** вкладку.  
  
6.  Выберите компьютеры, которые вы хотите, чтобы эта учетная запись пользователя необходимо разрешить удаленный доступ и нажмите кнопку **ОК**.
