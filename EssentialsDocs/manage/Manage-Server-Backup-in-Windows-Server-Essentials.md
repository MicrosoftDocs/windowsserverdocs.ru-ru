##  <a name="BKMK_WhichBackup"></a>Какую архивную копию выбрать?  
 Выбор архивной копии может быть весьма простой процедурой, если у вас есть недавняя успешно созданная резервная и вы уверены, что этот архив содержит все важные данные. Если вы пытаетесь восстановить сервер или компьютер из более ранней резервной копии, выбора оптимальной резервной копии для восстановления может потребоваться предварительный анализ и, возможно, некоторые компрометации.  
  
#### <a name="to-choose-a-backup"></a>Выбор архивной копии  
  
1.  Узнайте у владельца файлов или папок и запишите даты и время их добавления или изменения их. Используйте эти даты и времени в качестве отправной точки.  
  
2.  На **вариант восстановления** в мастере и восстановления файлов и щелкните **восстановить из выбранной архивной копии (Дополнительно)**.  
  
3.  В зависимости от того, требуется ли восстановить старую или более новые версии файлов или папок выберите резервную копию, которая лучше всего соответствует даты и времени, указанных в шаге 1.  
  
4.  Рекомендуется можно восстановить файлы и папки в альтернативное расположение и позвольте владельцу файлов и папок переместить те из них, которые им необходимы в исходное расположение. После завершения можно удалить файлы и папки, которые останутся в альтернативном расположении.  
  
##  <a name="BKMK_1"></a>Настройка или изменение параметров архивации сервера  
 Архивация сервера не настроена автоматически во время установки. Вам следует защитить сервер и его данные автоматическую ежедневную архивацию по расписанию. Рекомендуется поддерживать ежедневно выполнять план архивации, поскольку большинство организаций не может допустить потери данных, созданных в пределах нескольких дней. Дополнительные сведения см. в разделе [задать Настройка архивации сервера](Set-up-or-customize-server-backup.md).  
  
##  <a name="BKMK_2"></a>Остановка выполняющейся операции архивации сервера  
 Резервной копии сервера запускается по расписанию или Начните архивацию сервера вручную, вы можете остановить архивацию.  
  
#### <a name="to-stop-a-backup-in-progress"></a>Остановка выполняемой архивации  
  
1.  Откройте панель мониторинга.  
  
2.  На панели навигации щелкните **устройств**.  
  
3.  В списке компьютеров выберите сервер и нажмите кнопку **остановить архивацию сервера** в **задачи** области.  
  
4.  Нажмите кнопку **Да** для подтверждения этого действия.  
  
##  <a name="BKMK_3"></a>Удаленное управление резервными копиями  
 Находясь вне офиса, можно использовать Windows Server Essentials удаленного веб-доступа для доступа к панели мониторинга Windows Server Essentials для управления сервером.  
  
#### <a name="to-use-remote-web-access-to-manage-your-server"></a>Использование удаленного веб-доступа для управления сервером  
  
1.  Откройте веб-браузер.  
  
2.  В адресной строке введите имя домена Windows Server Essentials.  
  
3.  При появлении запроса введите имя пользователя и пароль.  
  
4.  Если щелкнуть имя сервера в удаленного веб-доступа, отображается страница входа на панель мониторинга.  
  
5.  Войдите в панель мониторинга как администратор и нажмите кнопку **устройств**.  
  
 Дополнительные сведения об удаленном веб-доступе см. в разделе [Обзор удаленного веб-доступа](Manage-Remote-Web-Access-in-Windows-Server-Essentials.md#BKMK_Overview).  
  
##  <a name="BKMK_4"></a>Отключение архивации сервера  
 Вам следует защитить сервер и его данные автоматическую ежедневную архивацию по расписанию. Рекомендуется поддерживать ежедневно выполнять план архивации, поскольку большинство организаций не может допустить потери данных, созданных в пределах нескольких дней.  
  
 Если вы уже настроили архивацию сервера, а затем решили использовать стороннее приложение для резервного копирования сервера, вы можете отключить архивацию сервера Windows Server Essentials.  
  
#### <a name="to-disable-server-backup"></a>Отключение архивации сервера  
  
1.  Войдите на мониторинга Windows Server Essentials с помощью учетной записи администратора и пароль учетной записи.  
  
2.  Нажмите кнопку **устройств** , а затем щелкните имя сервера.  
  
3.  В области задач щелкните **настроить архивацию сервера**.  
  
    > [!NOTE]
    >  **Настройка архивации сервера** задача отображается после настройки архивации сервера с помощью сервера мастер настройки резервного копирования. Дополнительные сведения о настройке архивации сервера см. в разделе [задать Настройка архивации сервера](Manage-Server-Backup-in-Windows-Server-Essentials.md#BKMK_1).  
  
4.  Откроется мастер настройки архивации сервера.  
  
5.  На **параметры конфигурации** щелкните **отключить архивацию сервера**. Следуйте инструкциям мастера.  
  
##  <a name="BKMK_5"></a>Дополнительные сведения о настройке архивации сервера  
 Архивация сервера не включается во время установки сервера.  
  
> [!NOTE]
>  При настройке архивации сервера следует подключить по крайней мере один внешний жесткий диск, на сервер для использования в качестве целевой диск архивации.  
  
###  <a name="BKMK_Target"></a>Целевой диск архивации  
 Можно использовать несколько внешних дисков хранилища для резервного копирования, и вы можете вращать диски между на предприятии и за его пределами. Это может повысить ваши авариям вы сможете восстановить данные при повреждении оборудования на предприятии.  
  
 При выборе диска хранилища для архивации вашего сервера, учитывайте следующее:  
  
-   Выберите диск, который содержит достаточно места для хранения данных. Объем дисков хранилища должен содержать по крайней мере в 2,5 раз больше объема данных, который требуется создать резервную копию. На дисках также должно хватать для дальнейшего роста данных вашего сервера.  
  
-   Если целевой диск архивации содержит автономные диски, Настройка архивации не будет выполнена. Чтобы завершить настройку, при выборе места назначения архивации, установите флажки для дисков, которые находятся в автономном режиме.  
  
-   Если выбрать диск, содержащий предыдущие резервные копии в качестве места назначения архивации, мастер позволит выбрать, нужно ли сохранить предыдущие резервные копии. Если сохранить резервные копии, мастер не форматировать диск.  
  
-   При повторном использовании внешнего диска, убедитесь, что он пуст или содержит только те данные, которые не требуется.  
  
-   Посетите веб-сайт производителя внешнего диска, чтобы убедиться, что на резервный диск поддерживается на компьютерах под управлением Windows Server Essentials.  
  
> [!CAUTION]
>  Мастер настройки сервера архивации форматирует дисков хранилища, при подготовке к резервному копированию.  
  
### <a name="server-backup-schedule"></a>Расписание архивации сервера  
 Вам следует защитить сервер и его данные автоматическую ежедневную архивацию по расписанию. Рекомендуется поддерживать ежедневно выполнять план архивации, поскольку большинство организаций не может допустить потери данных, созданных в пределах нескольких дней.  
  
 При использовании Windows Server Essentials копии сервера мастера настройки архивации можно архивировать данные сервера несколько раз в течение дня. Так как мастер планирует разностные операции архивации, архивация выполняется быстро и сервере не влияет на производительность. По умолчанию **настройки архивации сервера** планирует архивацию для ежедневного запуска в 00:00 и 23:00:00. Тем не менее можно настроить расписание архивации в соответствии с потребностями вашей организации. Следует периодически оценивать эффективность текущего плана резервного копирования и изменить при необходимости.  
  
> [!NOTE]
>  В установку по умолчанию в Windows Server Essentials сервер настроен для автоматического выполнения дефрагментации один раз в неделю. Это может привести к больше места, чем обычный резервные копии, если вы используете программное обеспечение сторонних разработчиков. Если нет необходимости дефрагментация сервера на регулярной основе, могут выполните следующие действия для отключения расписание дефрагментации.  
>   
>  1.  Нажмите клавиши Windows + W, чтобы открыть **поиска**.  
> 2.  В поле поиска введите **Дефрагментация**.  
> 3.  В разделе результатов щелкните **Дефрагментация и оптимизация дисков**.  
> 4.  В **оптимизация дисков** выберите диск и нажмите кнопку **изменить параметры**.  
> 5.  В **расписание оптимизации** уберите **по расписанию (рекомендуется)** флажок и нажмите кнопку **ОК** чтобы сохранить изменения.  
  
### <a name="items-to-be-backed-up"></a>Элементы для резервного копирования  
 По умолчанию для архивации выбраны все файлы операционной системы и папки. Вы можете архивировать все жесткие диски, файлы и папки на сервере, или выбрать только отдельные жесткие диски, файлы или папки для резервного копирования. Чтобы добавить или удалить элементы для архивации, выполните одно из следующих действий.  
  
-   Чтобы добавить диск с данными в список для архивации, установите флажок рядом с ним.  
  
-   Для исключения диска с данными из резервной копии сервера, снимите флажок рядом с ним.  
  
> [!NOTE]
>  Если вы хотите исключить **операционной системы** элемент из резервной копии, необходимо убрать **архивация системы (рекомендуется)** флажок.  
  
 Чтобы снизить объем жесткого диска архивации, резервными копиями сервера, можно исключить все папки, содержащие не учитывать особо важные файлы.  
  
 Например имеется папка с записями ТВ-программ, занимающая много места на жестком диске. Вы можете не выполнять резервное копирование этих файлов, так как обычно удалите их после их просмотре. Кроме того может существовать папка, содержащая временные файлы, которые не собираетесь хранить.  
  
##  <a name="BKMK_6"></a>Повторное создание разделов жесткого диска на сервере  
 При обнаружении неформатированного внутреннего жесткого диска на сервере Windows Server Essentials возникает оповещение о работоспособности, которое содержит ссылку на мастер установки нового жесткого диска. Мастер установки нового жесткого диска поможет выполнить различные параметры для форматирования жесткого диска. После завершения работы мастера один или несколько, в зависимости от размера диска, логические жесткие диски отформатированы создается на жестком диске, в файловой системе NTFS.  
  
 Если он потребуется повторно создать разделы на жестком диске, выполните следующие действия:  
  
#### <a name="to-repartition-a-hard-disk-drive"></a>Повторное разбиение жесткого диска  
  
1.  На **запустить** нажмите кнопку **Администрирование**, а затем дважды щелкните **Управление компьютером**.  
  
2.  Управление компьютером, нажмите кнопку **хранилища**, а затем дважды щелкните **Управление дисками**.  
  
3.  Щелкните правой кнопкой мыши диск, который требуется разбить на разделы, нажмите кнопку **удалить том**, а затем нажмите кнопку **Да**.  
  
    > [!NOTE]
    >  Повторите этот шаг для каждого раздела на жестком диске.  
  
4.  Щелкните правой кнопкой мыши **нераспределенный** жесткий диск, а затем нажмите кнопку **создать простой том**.  
  
5.  В мастере создания простых томов, создайте и отформатируйте том размером 16 ТБ (16 000 000 МБ) или менее.  
  
    > [!NOTE]
    >  Повторите этот шаг, пока не будет использоваться все незанятое пространство на жестком диске.  
  
##  <a name="BKMK_7"></a>Восстановление файлов и папок из резервной копии сервера  
 Можно просмотреть и восстановить отдельные файлы и папки из резервной копии сервера.  
  
#### <a name="to-restore-files-and-folders-from-a-server-backup"></a>Восстановление файлов и папок из резервной копии сервера  
  
1.  Откройте панель мониторинга, а затем нажмите кнопку **устройств** вкладку.  
  
2.  Щелкните имя сервера, а затем нажмите кнопку **восстановить файлы и папки сервера** в **задачи** области.  
  
3.  Откроется окно мастера восстановления файлов и папок. Следуйте инструкциям в мастере, чтобы восстановить файлы или папки.
