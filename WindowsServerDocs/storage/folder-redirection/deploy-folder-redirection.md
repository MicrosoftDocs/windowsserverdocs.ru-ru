# <a name="deploy-folder-redirection-with-offline-files"></a>Развертывание перенаправления папок с автономными файлами

>Относится к: Windows 10, Windows 7, Windows 8, Windows 8.1, Windows Vista, Windows Server 2019 г., Windows Server 2016, Windows Server (полугодовой канал), Windows Server 2012, Windows Server 2012 R2, Windows Server 2008 R2

В этом разделе описывается, как, в которых будет производиться развертывание перенаправления папок с автономными файлами на клиентских компьютерах Windows в Windows Server.

Список последних изменений раздела см. в разделе [журнал изменений](#change-history).

>[!IMPORTANT]
>Из-за изменений безопасности в [MS16-072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016), мы обновили [Step 3: Создать объект групповой Политики для перенаправления папок](#step-3-create-a-gpo-for-folder-redirection) этого раздела, что Windows правильно применить политику перенаправления папок (а не вернется перенаправленных папок на затронутых компьютерах).

## <a name="prerequisites"></a>Предварительные требования

### <a name="hardware-requirements"></a>Требования к оборудованию

Перенаправление папок требуется компьютер с x64 или x86; не поддерживается Windows® RT.

### <a name="software-requirements"></a>Требования к программному обеспечению

Перенаправление папок имеет следующие требования к программному обеспечению:

- Для управления перенаправлением папок, необходимо войти как член группы администраторов домена, группы администраторов предприятия или группы безопасности Владельцы-создатели групповой политики.
- Клиентские компьютеры должны работать в Windows 10, Windows 8.1, Windows 8, Windows 7, Windows Server 2019, Windows Server 2016, Windows Server (полугодовой канал), Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 или Windows Server 2008.
- Компьютеры клиентов должны быть присоединены к доменным службам Active Directory (AD DS), которыми вы управляете.
- На компьютере должны быть установлены компоненты "Управление групповой политикой" и "Центр администрирования Active Directory".
- Файловый сервер должен быть доступен для размещения перенаправленных папок.
    - Если общая папка использует пространства имен DFS, у папок DFS (ссылок) должен быть один целевой объект, чтобы избежать конфликтов серверов при изменении настроек пользователем.
    - Если общая папка использует репликацию DFS для копирования данных на другой сервер, пользователи должны иметь доступ лишь к исходному серверу, чтобы избежать конфликтов при изменении параметров серверов.
    - При использовании кластеризованного файлового ресурса, отключите непрерывную доступность файлового ресурса, чтобы избежать проблем с производительностью с помощью перенаправления папок и автономных файлов. Кроме того автономные файлы могут не переходить в автономный режим для 3-6 минут после того как пользователь теряет доступ к постоянно доступном файловом ресурсе, что может расстраивать планы пользователей, которые еще не использующих постоянный автономный режим для автономных файлов.

>[!NOTE]
>Некоторые новые функции в перенаправления папок имеют дополнительным клиентским компьютерам и требования к схеме Active Directory. Дополнительные сведения см. в разделе [развертывание основных компьютеров](deploy-primary-computers.md), [Отключение автономных файлов в папках](disable-offline-files-on-folders.md), [Включение постоянного автономного режима](enable-always-offline.md), и [Enable оптимизированных для операций перемещения папки ](enable-optimized-moving.md).

## <a name="step-1-create-a-folder-redirection-security-group"></a>Шаг 1. Создайте группу безопасности перенаправление папки

Если в вашей среде не уже настроен с перенаправлением папок, первым шагом является создание группы безопасности, содержащую всех пользователей, к которым необходимо применить параметры политики перенаправления папок.

Вот как можно создать группу безопасности для перенаправления папок:

1. Откройте диспетчер сервера на компьютере с центром администрирования Active Directory установлен.
2. На **средства** меню, выберите **Центр администрирования Active Directory**. Отобразится центр администрирования Active Directory.
3. Щелкните правой кнопкой мыши соответствующий домен или Подразделение, выберите **New**, а затем выберите **группы**.
4. В окне **Создание группы** в разделе **Группа** укажите следующие параметры.
    - В поле **Имя группы** введите имя группы безопасности, например **Пользователи перенаправление папки**.
    - В **область действия группы**выберите **безопасности**, а затем выберите **Global**.
5. В **члены** выберите **добавить**. Откроется диалоговое окно "Выбор пользователей, контактов, компьютеров, учетных записей служб или групп".
6. Введите имена пользователей или групп, к которым вы хотите развернуть перенаправление папок, выберите **ОК**, а затем выберите **ОК** еще раз.

## <a name="step-2-create-a-file-share-for-redirected-folders"></a>Шаг 2. Создание общей папки для перенаправления папок

Если у вас еще нет общей папки для перенаправляемых папок, используйте следующую процедуру для создания общей папки на сервере под управлением Windows Server 2012.

>[!NOTE]
>Если вы создаете файловый ресурс на сервере под управлением другой версии Windows Server, некоторые функции могут отличаться или быть недоступны.

Вот как можно создать общую папку на Windows Server 2019, Windows Server 2016 и Windows Server 2012:

1. В области навигации диспетчера серверов выберите **файловых служб и служб хранилища**, а затем выберите **общих папок** на страницу общих ресурсов.
2. В **общих папок** плитки, выберите **задачи**, а затем выберите **новую общую папку**. Откроется мастер создания общих ресурсов.
3. На **выберите профиль** выберите **общий ресурс SMB — быстрый**. Если у диспетчера ресурсов файлового сервера и используете свойства управления папками, вместо этого выберите **общий ресурс SMB — дополнительные**.
4. На странице **Расположение общего ресурса** выберите сервер и том, в котором хотите создать общий ресурс.
5. На **имя общего ресурса** введите имя для общей папки (например, **пользователей $** ) в **имя общего ресурса** поле.
    >[!TIP]
    >При создании общего ресурса необходимо скрыть его, поместив ```$``` после имени общего ресурса. Это будет скрыть его от обычных обозревателей.
6. На **другие параметры** странице снимите флажок включить постоянную доступность, при его наличии и при необходимости выберите **включить перечисление на основе доступа** и **шифровать доступ к данным** флажки.
7. На **разрешения** выберите **Настройка разрешений доступа...** . Отобразится диалоговое окно с расширенными параметрами безопасности.
8. Выберите **отключение наследования**, а затем выберите **преобразовать наследуемые разрешения для этого объекта в явные**.
9. Установите разрешения, как описано в таблице 1 и показано на рисунке 1, удаляя разрешения для неуказанных групп и учетных записей и добавьте особые разрешения для группы пользователей перенаправления папок, созданный на шаге 1.
    
    ![Установка разрешений для общей папки перенаправленных папок](media/deploy-folder-redirection/setting-the-permissions-for-the-redirected-folders-share.png)
    
    **Рис. 1** устанавливать разрешения для перенаправленных папок на общую папку
10. Если вы выбрали профиль **Общий ресурс SMB — дополнительные** , на странице **Свойства управления** установите для параметра "Использование папки" значение **Файлы пользователя** .
11. Если вы выбрали профиль **Общий ресурс SMB — дополнительные** , на странице **Квота** вы можете при необходимости выбрать квоту, которая будет распространяться на пользователей общего ресурса.
12. На **Подтверждение** выберите **Create.**

## <a name="step-3-create-a-gpo-for-folder-redirection"></a>Шаг 3. Создать объект групповой Политики для перенаправления папок

Если у вас еще нет объекта групповой Политики для настройки перенаправления папок, используйте следующую процедуру для ее создания.

Вот как можно создать объект групповой Политики для перенаправления папок:

1. Откройте диспетчер сервера на компьютере с установленным компонентом управления групповой политикой.
2. Из **средства** меню, выберите **Управление групповой политикой**.
3. Щелкните правой кнопкой мыши домен или Подразделение, в котором вы хотите настроить перенаправление папок, а затем выберите **создать объект групповой Политики в этом домене и связать его**.
4. В **новый объект групповой Политики** диалоговом окне введите имя объекта групповой политики (например, **параметры перенаправления папок**), а затем выберите **ОК**.
5. Щелкните правой кнопкой мыши по новому объекту групповой политики и уберите флажок **Связь включена** . Это исключит применение GPO до завершения его настройки.
6. Выберите GPO. В **Фильтрация ограничений безопасности** раздел **область** выберите **Authenticated Users**и выберите **удалить** во избежание объект групповой Политики применить для всех пользователей.
7. В **Фильтрация ограничений безопасности** выберите **добавить**.
8. В **Выбор пользователя, компьютер или группу** диалоговом окне имя безопасности группы, которую вы создали на шаге 1 (например, **пользователям перенаправление папки**) и выберите **ОК**.
9. Выберите **делегирования** выберите **добавить**, тип **Authenticated Users**выберите **ОК**, а затем выберите **ОК** снова к примите имя по умолчанию разрешения на чтение.
    
    Этот шаг необходим из-за изменений безопасности в [MS16-072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016).

>[!IMPORTANT]
>Из-за изменений безопасности в [MS16-072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)вы теперь необходимо предоставить разрешения на чтение делегированные группы Authenticated Users в объект групповой политики перенаправления папок — в противном случае объект групповой Политики не применяются к пользователям, или если он уже применен, является объект групповой Политики удалено, перенаправление папок к локальным Компьютером. Дополнительные сведения см. в разделе [развертывание групповой политики безопасности обновления MS16-072](https://blogs.technet.microsoft.com/askds/2016/06/22/deploying-group-policy-security-update-ms16-072-kb3163622/).

## <a name="step-4-configure-folder-redirection-with-offline-files"></a>Шаг 4. Настройка перенаправления папок с автономными файлами

После создания объекта групповой Политики для настройки перенаправления папок, измените параметры групповой политики для включения и настройки перенаправления папок, как описано в следующей процедуре.

>[!NOTE]
>Автономные файлы включены по умолчанию для перенаправляемых папок на клиентских компьютерах Windows и отключен на компьютерах под управлением Windows Server, если не изменено пользователем. Чтобы использовать групповую политику для управления отвечает за Включение автономных файлов, используйте **разрешить или запретить использование автономных файлов** параметр политики.
> Сведения о некоторых других параметров групповой политики автономных файлов, см. в разделе [включить Advanced автономных файлов функциональность](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>), и [Настройка групповой политики для автономных файлов](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>).

Вот как настроить перенаправление папок в групповой политике:

1. В управлении групповыми политиками, щелкните правой кнопкой мыши созданный объект групповой Политики (к примеру, **параметры перенаправления папок**), а затем выберите **изменить**.
2. В окне редактора управления групповыми политиками перейдите к **Конфигурация пользователя**, затем **политики**, затем **параметры Windows**, а затем **папки Перенаправление**.
3. Щелкните правой кнопкой мыши папку, в который вы хотите перенаправить (к примеру, **документов**), а затем выберите **свойства**.
4. В **свойства** диалоговом окне из **параметр** выберите **Basic - перенаправить папки, в то же расположение**.

    > [!NOTE]
    > Чтобы применить перенаправление папок на клиентских компьютерах под управлением Windows XP или Windows Server 2003, выберите **параметры** и выберите **применить политику перенаправления также к Windows 2000, Windows 2000 Server, Windows XP, и Операционные системы Windows Server 2003** флажок.
5. В **целевое расположение папки** выберите **создайте папку для каждого пользователя на корневом пути** и затем в **путь к корневому каталогу** введите путь к общей папки файла хранения Перенаправление папок, например:  **\\ \\fs1.corp.contoso.com\\$ пользователей**
6. Выберите **параметры** вкладке и в **удаление политики** при необходимости выберите **перенаправить папку обратно в расположение локального профиля пользователя при удалении политики** (этот параметр может помочь сделать более предсказуемое поведение adminisitrators и пользователей, перенаправление папок).
7. Выберите **ОК**, а затем выберите **Да** в диалоговом окне Предупреждение.

## <a name="step-5-enable-the-folder-redirection-gpo"></a>Шаг 5. Включение перенаправления папок групповой Политики

После завершения настройки параметров групповой политики перенаправления папок, следующим шагом является возможность GPO, разрешая его для применения к затронутым пользователям.

>[!TIP]
>Если вы планируете задействовать поддержку основного компьютера или другие настройки политики, это следует сделать до включения GPO, чтобы предотвратить копирование данных пользователя на неосновные компьютеры до включения поддержки основного компьютера.

Вот как можно включить объект групповой Политики перенаправления папок:

1. Откройте "Управление групповой политикой".
2. Щелкните правой кнопкой мыши объект групповой Политики, которая создана, а затем выберите **связь включена**. Флажок отображается рядом с элементом меню.

## <a name="step-6-test-folder-redirection"></a>Шаг 6. Проверить перенаправление папок

Чтобы проверить перенаправление папок, войдите на компьютер с учетной записью пользователя, настроенного для перенаправления папок. Убедитесь, что перенаправление папок и профилей.

Вот как можно проверить перенаправление папок:

1. Войдите на основном компьютере (Если вы включили поддержку основного компьютера) с учетной записью пользователя, для которого вы включили перенаправление папок.
2. Если пользователь уже вошел в систему, откройте командную строку с повышенными привилегиями и введите нижеприведенную команду, чтобы убедиться, что на компьютере используются последние настройки групповой политики.
    
    ```PowerShell
    gpupdate /force
    ```
3. Откройте проводник.
4. Щелкните правой кнопкой мыши в перенаправленной папке (например, папка «Мои документы» в библиотеке документов), а затем выберите **свойства**.
5. Выберите **расположение** вкладке и убедитесь, что отобразилась путь общей папки, которые вы указали, а не локальный путь.
