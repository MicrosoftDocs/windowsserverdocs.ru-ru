# Развертывание перенаправления папок с помощью автономных файлов

>Область применения: Windows 10, Windows 7, Windows 8, Windows 8.1, Windows Server 2008 R2, Windows Server 2012, Windows Server 2012 R2, Windows Server 2016, Windows Vista

В этом разделе описывается, как использовать Windows Server для развертывания перенаправления папок с помощью автономных файлов для клиентских компьютеров Windows.

Список последних изменений в этой статье см. в разделе [журнала изменений](#change-history).

>[!IMPORTANT]
>Из-за безопасности изменений, внесенных в [MS16 072](https://support.microsoft.com/en-us/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016), мы обновили [Шаг 3: создание объекта групповой Политики для перенаправления папок](#step-3:-create-a-gpo-for-folder-redirection) этого раздела Windows правильно применить политику перенаправления папок (а не вернется перенаправления папок на затронутых компьютеров).

## Предварительные условия

### Требования к оборудованию

Перенаправление требует x64-разрядных или на основе x86 компьютера; он не поддерживается в Windows® RT.

### Требования к программному обеспечению

Перенаправление имеет следующие требования к программному обеспечению.

- Для управления службой перенаправления папок, необходимо выполнить вход как член группы администраторов домена, группы безопасности администраторов предприятия или группы безопасности создатели групповой политики.
- Клиентские компьютеры должны выполняться Windows 10, Windows 8.1, Windows 8, Windows 7, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 или Windows Server 2008.
- Клиентские компьютеры должны быть присоединены к доменных служб Active Directory (AD DS), вы управляете.
- Компьютер должен быть доступен с помощью управления групповыми политиками и Центр администрирования Active Directory.
- Файловый сервер должен быть доступен для размещения перенаправленные папки.
    - Если в общей используется пространств имен DFS, папок DFS (ссылки) должны иметь один целевой объект, чтобы запретить пользователям вносить изменения конфликтующие на разных серверах.
    - Если в общей используется репликации DFS для репликации содержимого с другого сервера, пользователи должны иметь возможность получить доступ к только сервере источника, чтобы запретить пользователям вносить изменения конфликтующие на разных серверах.
    - Если вы используете кластерный файловый ресурс, отключите непрерывную доступность в общей папке, чтобы избежать проблем с производительностью с помощью автономных файлов и перенаправления папок. Кроме того автономные файлы не перевести в автономный режим для 3-6 минут после пользователь теряет доступ к общему ресурсу непрерывной доступности файла, который можно будет раздражать пользователей, которые еще не используете всегда в автономном режиме автономных файлов.

>[!NOTE]
>Некоторые новые функции перенаправления папок имеют дополнительные клиентский компьютер и требований схемы Active Directory. Дополнительные сведения см. в разделе [Развертывание основной компьютеры](deploy-primary-computers.md), [Отключите автономные файлы в папках](disable-offline-files-on-folders.md), [Включение всегда автономном режиме](enable-always-offline.md)и [Включить перемещение оптимизированного папки](enable-optimized-moving.md).

## Шаг 1: Создание группы безопасности перенаправления папок

Если вашей среды — это не уже настроен с помощью перенаправления папок, первым шагом является создание группы безопасности, содержащий всех пользователей, к которым вы хотите применить параметры политики перенаправления папок.

Вот как можно создать группу безопасности для перенаправления папок:

1. Откройте диспетчер сервера на компьютере с центром администрирования Active Directory установлен.
2. В меню " **Сервис** " выберите **Центр администрирования Active Directory**. Отобразится центр администрирования Active Directory.
3. Щелкните правой кнопкой мыши соответствующий домен или Подразделение, выберите **Создать**и затем выберите **группу**.
4. В окне **Создание группы** в разделе **Группа** укажите следующие параметры.
    - **Имя группы**, введите имя группы безопасности, например: **Пользователям перенаправление папок**.
    - В **область действия группы**выберите « **Безопасность**», а затем выберите **Global**.
5. В разделе " **члены** " выберите " **Добавить**". Откроется диалоговое окно "Выбор пользователей, контактов, компьютеров, учетных записей служб или групп".
6. Введите имена пользователей или групп, к которым вы хотите развернуть перенаправления папок, нажмите кнопку **ОК**и затем нажмите кнопку **ОК** .

## Шаг 2: Создание общего файлового ресурса для перенаправления папок

Если вы уже нет общего файлового ресурса для перенаправления папок, выполните следующие действия, чтобы создать общую папку на сервере под управлением Windows Server 2012.

>[!NOTE]
>Некоторые функциональные возможности могут отличаться или окажется недоступным, если вы создаете файловый ресурс на сервере под управлением другой версии Windows Server.

Ниже приведен пример создания общего файлового ресурса в Windows Server 2012 и Windows Server 2016:

1. В области навигации диспетчера сервера установите **файловые службы и службы хранилища**и выберите **общих папок** для отображения на странице общих ресурсов.
2. В плитке **общих папок** выберите **задачи**, а затем выберите **Новый общий ресурс**. Откроется мастер создания общей папки.
3. На странице **Выберите профиль** выберите **Общей папки SMB — быстрые**. Если у вас установлен диспетчер ресурсов файлового сервера и использовании свойства управления папки, вместо этого выберите **Общей папки SMB - Advanced**.
4. На странице **Совместно использовать расположение** выберите сервер и тома, на которых вы хотите создать общую папку.
5. На странице " **Имя общего ресурса** " введите имя общего ресурса (например, **Пользователи $**) в поле **имя общего ресурса** .
    >[!TIP]
    >При создании общей папки, скрыть общую папку, размещая ```$``` после имя общей папки. Это будет скрывать общей несложных браузеров.
6. На странице " **Общие параметры** " снимите флажок Включить непрерывную доступность означает и (необязательно) установите флажки **Включить перечисление на основе доступа** и **шифровать доступ к данным** .
7. На странице " **разрешения** " выберите **… настроить разрешения**. Откроется диалоговое окно Дополнительные параметры безопасности.
8. Выберите **отключить наследование**, а затем выберите **Преобразование унаследованных разрешений в явных разрешений на этом объекте**.
9. Задайте разрешения, как описано в таблице 1 и показано на рис. 1, удаляя разрешения для учетных записей и групп в списке нет и добавить особые разрешения группе пользователей перенаправление папок, созданный на шаге 1.
    
    ![Настройка разрешений для общей папки перенаправлением](media/deploy-folder-redirection/setting-the-permissions-for-the-redirected-folders-share.png)
    
    **Рисунок 1** Настройка разрешений для общей папки перенаправлением
10. Если вы выбрали профиль **Общей папки SMB - Advanced** , на странице **Управления свойства** выберите значение использования папки **Файлов пользователей** .
11. Если вы выбрали профиль **Поделиться SMB - Advanced** на странице **квоты** (необязательно) выберите квоты для применения к пользователей общего ресурса.
12. На странице **подтверждения** выберите **Создание.**

## Шаг 3: Создание объекта групповой Политики для перенаправления папок

Если вы уже нет объект групповой Политики для настройки перенаправления папок, выполните следующие действия для ее создания.

Вот как можно создать объект групповой Политики для перенаправления папок:

1. Откройте диспетчер сервера на компьютере с помощью управления групповыми политиками установленным.
2. В меню " **Сервис** " выберите **Управление групповой политикой**.
3. Щелкните правой кнопкой мыши домен или Подразделение, в котором требуется для настройки перенаправления папок, а затем выберите **создать объект групповой Политики в этом домене и связать его**.
4. В диалоговом окне **Новый объект групповой Политики** введите имя для объекта групповой Политики (например, **Параметры перенаправления папок**) и нажмите кнопку **ОК**.
5. Щелкните правой кнопкой мыши только что созданного объекта групповой Политики и снимите флажок **Связь включена** . Это предотвращает применяемого до завершения его настройка объект групповой Политики.
6. Выберите объект групповой Политики. В разделе **Фильтры безопасности** вкладке **область** выбрать **Проверку подлинности пользователей**и выберите **Удалить** , чтобы предотвратить объект групповой Политики применяются для всех пользователей.
7. В разделе **Фильтры безопасности** установите флажок " **Добавить**".
8. В диалоговом окне **Выбор пользователей, компьютеров или группы** введите имя группы безопасности, созданной на шаге 1 (например, **Пользователям перенаправление папки**) и нажмите кнопку **ОК**.
9. Выберите вкладку **делегирования** , выберите **Добавить**, введите **Прошедшие проверку**, нажмите кнопку **ОК**и затем нажмите кнопку **ОК** еще раз, чтобы принять разрешения на чтение по умолчанию.
    
    Этот шаг является необходимым из-за безопасности изменения, внесенные в [MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016).

>[!IMPORTANT]
>Из-за изменения, внесенные в [MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016), теперь вы должны дать безопасности группы Прошедшие делегированные разрешения на чтение для объекта групповой Политики перенаправления папок - в противном случае объект групповой Политики не применяются к пользователям или если он уже применен, удаляется объект групповой Политики, перенаправление папки обратно на локальный компьютер. Дополнительные сведения см. в разделе [Развертывание групповой политики безопасности обновление MS16-072](https://blogs.technet.microsoft.com/askds/2016/06/22/deploying-group-policy-security-update-ms16-072-kb3163622/).

## Шаг 4: Настройка перенаправления папок с помощью автономных файлов

Создав объект групповой Политики для настройки перенаправления папок, измените параметры групповой политики для включения и настройки перенаправления папок, как описано в следующей процедуре.

>[!NOTE]
>Автономные файлы включено по умолчанию для перенаправления папок на клиентских компьютерах Windows и отключены на компьютерах под управлением Windows Server, если изменено пользователем. Чтобы использовать групповую политику для управления Включение автономных файлов, используйте **Разрешить или запретить использование автономных файлов** параметр политики.
> Сведения о некоторых других параметров автономных файлов групповой политики см. в разделе [Включить дополнительные автономных файлов функциональные возможности](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>)и [Настройка групповой политики для автономных файлов](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>).

Ниже показано, как настроить перенаправления папок в групповой политике:

1. В управлении групповой политикой щелкните правой кнопкой мыши объект групповой Политики, вы создали (например, **Параметры перенаправления папок**), затем выберите **Изменить**.
2. В окне Редактор управления групповыми политиками перейдите к **Конфигурации пользователя**, а затем **политик**, затем **Параметры Windows**и **Перенаправления папок**.
3. Щелкните правой кнопкой мыши папку, которая требуется перенаправить (например, **документы**), а затем выберите **Свойства**.
4. В диалоговом окне **Свойства** в поле **параметра** выберите **Basic - перенаправить папки в той же папке**.

    > [!NOTE]
    > Чтобы применить перенаправление на клиентских компьютерах под управлением Windows XP или Windows Server 2003, выберите вкладку " **Параметры** " и выберите, **Применить политику перенаправления для Windows 2000, Windows 2000 Server, Windows XP и Windows Server 2003 эксплуатация систем** флажок.
5. В разделе " **расположение папки** " выберите **Создать папку для каждого пользователя на корневом пути** и введите путь к общей хранения перенаправления папок, например в **Корневом каталоге** : **\\\fs1.corp.contoso.com\\ Пользователи $**
6. Выберите вкладку " **Параметры** " и в разделе " **Удаление политики** " при необходимости **перенаправить папку обратно в расположение локального профиля пользователя при удалении политики** (этот параметр может помочь сделать перенаправление ведут себя больше предсказуемость для adminisitrators и пользователей).
7. Нажмите кнопку **ОК**, а затем выберите **Да** в диалоговом окне предупреждения.

## Шаг 5: Включение перенаправления папок объект групповой Политики

После завершения настройки параметров групповой политики перенаправления папок, следующим шагом является возможность объект групповой Политики, принципам, оно будет применяться для соответствующих пользователей.

>[!TIP]
>Если вы планируете реализовать поддержку основных компьютеров или другие параметры политики, сделайте это прямо сейчас, прежде чем включать объект групповой Политики. Это предотвращает копируются на компьютеры неосновных до включения поддержки основного компьютера данные пользователя.

Ниже показано, как включить объект групповой Политики перенаправления папок:

1. Откройте Управление групповой политикой.
2. Щелкните правой кнопкой мыши объект групповой Политики, который вы создали, а затем выберите **Связь включена**. Флажок будет отображаться рядом с элементом меню.

## Шаг 6: Проверка перенаправления папок

Чтобы протестировать перенаправления папок, вход на компьютер с учетной записью пользователя, настроенных для перенаправления папок. Затем убедитесь, что перенаправления папок и профилей.

Ниже показано, как проверить перенаправления папок:

1. Войдите в главный компьютер (Если вы включили поддержку основных компьютеров) с учетной записью пользователя, для которого вы включили перенаправления папок.
2. Если пользователь ранее выполнил вход на компьютер, откройте командную строку с повышенными привилегиями и введите следующую команду, чтобы убедиться, что последние параметры групповой политики применяются к на клиентском компьютере:
    
    ```PowerShell
    gpupdate /force
    ```
3. Откройте проводник.
4. Щелкните правой кнопкой мыши перенаправления папок (например, Мои документы папка в библиотеке документов), а затем выберите **Свойства**.
5. Перейдите на вкладку **расположение** и убедитесь, что путь отображает файловый ресурс, указанный вместо локального пути.
