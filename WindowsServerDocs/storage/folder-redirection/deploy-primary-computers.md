# Развертывание основной компьютеры для перенаправления папок и перемещаемые профили пользователей

>Область применения: Windows 10, Windows 8, Windows 8.1, Windows Server 2012, Windows Server 2012 R2, Windows Server 2016

В этом разделе описывается, как включить поддержку основных компьютеров и назначить основной компьютеров для пользователей. Это позволяет контролировать какие компьютеры использовать перенаправление папок и перемещаемые профили пользователей.

>[!IMPORTANT]
>При включении основного компьютера поддержка перемещаемые профили пользователей, всегда включите поддержку основного компьютера для перенаправления папок также. Это позволяет сохранять документы и другие файлы пользователей за пределы профили пользователей, позволяет профили остаются небольшой и вход в систему раз оставаться быстро.

## Предварительные условия

## Требования к программному обеспечению

Поддержка основной компьютер имеет следующие требования:

- Необходимо обновить схему доменных служб Active Directory (AD DS) для включения дополнения схемы Windows Server 2012 (Установка контроллера домена Windows Server 2012 автоматически обновляет схему). Сведения об обновлении схемы AD DS см. в разделе [Adprep.exe интеграции](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh472161(v=ws.11)#adprepexe-integration>) и [Выполнение Adprep.exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd464018(v=ws.10)>).
- Клиентские компьютеры должны выполняться Windows 10, Windows 8.1, Windows 8, Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.

>[!TIP]
>Несмотря на то, что основной компьютер поддержки требуется перенаправление папок и перемещаемые профили пользователей при развертывании этих технологий в первый раз, лучше всего поддерживать основной компьютер перед включением объектов групповой политики, которые настроить перенаправление папок и Перемещаемых профилей пользователей. Это предотвращает копируются на компьютеры неосновных до включения поддержки основного компьютера данные пользователя. Сведения о конфигурации см. [Развертывание перенаправление](deploy-folder-redirection.md) папок и [Развертывание перемещаемых профилей пользователей](deploy-roaming-user-profiles.md).

## Шаг 1: Назначить основной компьютеров для пользователей

Первым шагом в развертывании основной компьютеров поддержка назначение основной компьютеры для каждого пользователя. Чтобы сделать это, используйте Центр администрирования Active Directory для получения различающееся имя соответствующих компьютеров и затем задать атрибут **msDs-PrimaryComputer** .

>[!TIP]
>Чтобы использовать Windows PowerShell для работы с основной компьютеров, см. в записи блога [копнем немного глубже в основной компьютер Windows 8](<https://blogs.technet.microsoft.com/askds/2012/10/23/digging-a-little-deeper-into-windows-8-primary-computer/>).

Вот как задать основной компьютеров для пользователей.

1. Откройте диспетчер сервера на компьютере с установленными средствами администрирования Active Directory.
2. В меню " **Сервис** " выберите **Центр администрирования Active Directory**. Отобразится центр администрирования Active Directory.
3. Перейдите к контейнеру **компьютеров** в соответствующем домене.
4. Щелкните правой кнопкой мыши компьютер, который вы хотите назначить в качестве основного компьютера, а затем выберите **Свойства**.
5. В области навигации выберите **расширения**.
6. Выберите вкладку **Редактор атрибутов** , прокрутите до **distinguishedName**, выберите **представление**, щелкните правой кнопкой мыши значение в списке, выберите **Копировать**, нажмите **кнопку ОК**и нажмите кнопку **Отмена**.
7. Перейдите к контейнеру **пользователей** в соответствующем домене, щелкните правой кнопкой мыши пользователя, к которому требуется назначить компьютер и затем выберите **Свойства**.
8. В области навигации выберите **расширения**.
9. Выберите вкладку **Редактор атрибутов** , выберите **msDs-PrimaryComputer** , а затем выберите **Изменить**. Отобразится диалоговое окно "Редактор многозначных строк".
10. Щелкните правой кнопкой мыши текстовое поле, выберите **Вставить**, выберите **Добавить**, нажмите **кнопку ОК**и затем снова нажмите **кнопку ОК** .

## Шаг 2: При необходимости включите основной компьютеры для перенаправления папок в групповой политике

Следующий шаг — Настройка групповой политики для включения поддержки основного компьютера для перенаправления папок при необходимости. Это позволяет папки пользователей для перенаправления на компьютерах, в качестве основного компьютеры пользователя, но не на других компьютерах. Вы можете контролировать основной компьютеры для перенаправления папок для отдельного компьютера, или отдельно для каждого пользователя.

Ниже показано, как включить основной компьютеры для перенаправления папок:

1. Управление групповой политикой, щелкните правой кнопкой мыши объект групповой Политики, созданного при выполнении начальной конфигурации перенаправления папок и/или профили пользователей роуминга (например, **Параметры перенаправления папок** или **Перемещаемых профилей параметры пользователя**), а затем Выберите **Изменить**.
2. Чтобы включить поддержку основных компьютеров с помощью групповой политики на компьютере, перейдите к **Конфигурации компьютера**. Для пользователя групповой политики перейдите к **Конфигурации пользователя**.
    - На компьютере Групповая политика применяется обработки основной компьютер все компьютеры, к которым применяется объект групповой Политики, влияют на всех пользователей компьютеров.
    - На основе пользователя групповой политики применяется к все учетные записи пользователей, к которым применяется объект групповой Политики, влияющие на всех компьютерах, к которым пользователи выполняли вход на обработку основного компьютера.
3. В разделе **Конфигурация компьютера** или **Конфигурация пользователя**перейдите к **политик**, затем **Административные шаблоны**, то **системы**, а затем **Перенаправления папок**.
4. Щелкните правой кнопкой мыши **перенаправления папок основной только на компьютерах**, а затем выберите **Изменить**.
5. Выберите **включено**и затем нажмите **кнопку ОК**.

## Шаг 3: При необходимости включите основной компьютеры для перемещаемых профилей пользователей в групповой политике

Следующий шаг — можно дополнительно настроить групповой политики для включения поддержки основного компьютера для перемещаемых профилей пользователей. Это позволяет профиля пользователя для перемещения на компьютерах, в качестве основного компьютеры пользователя, но не на других компьютерах.

Вот как включить основной компьютеры для перемещаемых профилей пользователей.

1. Включите поддержку основного компьютера для перенаправления папок, если вы еще не сделали этого.
    * Это позволяет сохранять документы и другие файлы пользователей за пределы профили пользователей, позволяет профили остаются небольшой и вход в систему раз оставаться быстро.
2. В управлении групповой политикой щелкните правой кнопкой мыши объект групповой Политики, вы создали (например, **Перенаправление папок и перемещаемые профили параметры пользователя**), а затем выберите **Изменить**.
3. Перейдите к **Конфигурация компьютера**, а затем **политик**, затем **Административные шаблоны**, а затем **системы**и **Профили пользователей**.
4. **Загрузите перемещаемых профилей основной только на компьютерах,** щелкните правой кнопкой мыши и выберите **Изменить**.
5. Выберите **включено**и затем нажмите **кнопку ОК**.

## Шаг 4: Включение объект групповой Политики

После завершения настройки перенаправления папок и перемещаемые профили пользователей, включите объект групповой Политики, если это еще не сделано. Это позволяет применять для соответствующих пользователей и компьютеров.

Ниже показано, как включить перенаправление папок и перемещаемые профили GPO пользователя:

1. Откройте Управление групповой политикой
2. Щелкните правой кнопкой мыши объекты групповой политики, которые вы создали, а затем выберите **Связь включена**. Флажок должен отображаться рядом с элементом меню.

## Шаг 5: Проверка функции основного компьютера

Чтобы проверить поддержку основных компьютеров, войдите на главный компьютер, убедитесь, что перенаправления папок и профилей, затем войдите на компьютер неосновных и подтвердите не перенаправление папок и профили.

Ниже показано, как проверить функциональность основного компьютера:

1. Войдите в назначенное главный компьютер с учетной записью пользователя, для которого вы включили перенаправление папок и перемещаемые профили пользователей.
2. Если учетная запись пользователя выполнил вход на компьютер ранее, откройте сеанс Windows PowerShell или окно командной строки от имени администратора, введите следующую команду и выйти из при появлении запроса, чтобы убедиться, что последние параметры групповой политики применяются к клиентский компьютер:
    ```PowerShell
    Gpupdate /force
    ```
3. Откройте проводник.
4. Щелкните правой кнопкой мыши перенаправленной папки (например, к папке Мои документы библиотеки документов), а затем выберите **Свойства**.
5. Выберите вкладку " **расположение** " и убедитесь, что путь отображает файлового ресурса, указанного вместо локальный путь. Чтобы убедиться, что профиль пользователя находится в роуминге, откройте **Панель управления**, выберите **Система и безопасность**, выбор **системы**, выберите **Дополнительные параметры системы**, выберите **Параметры** в разделе "Профили пользователей" и найдите ** Роуминг** в столбце " **Тип** ".
6. Войдите в систему под учетной записью пользователя на компьютере, который не определен как основной компьютер пользователя.
7. Повторите шаги 2 – 5, вместо этого ищете локального пути и тип **локального** профиля.

>[!NOTE]
>Если папки были перенаправлены на компьютере перед включением компьютера основной поддержки, папки будет оставаться перенаправления, если следующий параметр не настроен в параметре политики перенаправления папок каждую папку: **перенаправить папку в локальную размещение профиля пользователя при удалении политики**. Аналогичным образом профили, которые ранее роуминга на конкретном компьютере будет отображаться **роуминга** в столбцах **типа** ; Тем не менее будет отображаться **локального**столбце " **состояние** ".
