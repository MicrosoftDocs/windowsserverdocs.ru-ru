# <a name="deploy-primary-computers-for-folder-redirection-and-roaming-user-profiles"></a>Развертывание основных компьютеров для перенаправления папок и перемещаемых профилей пользователей

>Относится к: Windows 10, Windows 8, Windows 8.1, Windows Server 2019 г., Windows Server 2016, Windows Server 2012, Windows Server 2012 R2

В этом разделе описывается включение поддержки основного компьютера и назначение основных компьютеров для пользователей. Таким образом можно контролировать использование перенаправления папок и перемещаемых профилей пользователей.

> [!IMPORTANT]
> При включении поддержки основного компьютера для перемещаемых профилей пользователей, всегда Включение поддержки основного компьютера для перенаправления папок также. Это позволяет сохранять документы и другие файлы пользователя из профилей пользователей, которое помогает профили оставаться небольшими и входить в раз оставаться fast.

## <a name="prerequisites"></a>Предварительные требования

## <a name="software-requirements"></a>Требования к программному обеспечению

Поддержка основного компьютера необходимо соблюдать следующие требования:

- Необходимо обновить схему доменных служб Active Directory (AD DS), чтобы включать дополнения схемы Windows Server 2012 (при установке контроллера домена Windows Server 2012 автоматически обновляет схему). Сведения об обновлении схемы AD DS, см. в разделе [интеграция Adprep.exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh472161(v=ws.11)#adprepexe-integration>) и [Выполнение Adprep.exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd464018(v=ws.10)>).
- Компьютеры клиентов должны управляться Windows 10, Windows 8.1, Windows 8, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.

> [!TIP]
> Несмотря на то, что поддержки основного компьютера требует перенаправления папок и перемещаемые профили пользователей, при развертывании этих технологий в первый раз, рекомендуется настроить поддержку основного компьютера перед включением объекты групповой политики для настройки перенаправления папок и Перемещаемые профили пользователей. предотвратить копирование данных пользователя на неосновные компьютеры до включения поддержки основного компьютера. Сведения о конфигурации, см. в разделе [развертывание перенаправления папок](deploy-folder-redirection.md) и [развертывание перемещаемых профилей пользователей](deploy-roaming-user-profiles.md).

## <a name="step-1-designate-primary-computers-for-users"></a>Шаг 1. Назначение основных компьютеров для пользователей

Первый шаг в развертывании поддержку основных компьютеров назначение основных компьютеров для каждого пользователя. Для этого нужно использовать Центр администрирования Active Directory для получения различающееся имя из соответствующих компьютеров, а затем задайте **msDs-PrimaryComputer** атрибута.

> [!TIP]
> Чтобы использовать Windows PowerShell для работы с основных компьютеров, см. в записи блога [вглубь немного основного компьютера Windows 8](<https://blogs.technet.microsoft.com/askds/2012/10/23/digging-a-little-deeper-into-windows-8-primary-computer/>).

Вот как можно указать основных компьютеров для пользователей:

1. Откройте диспетчер сервера на компьютере с установленными средствами администрирования Active Directory.
2. На **средства** меню, выберите **Центр администрирования Active Directory**. Отобразится центр администрирования Active Directory.
3. Перейдите к **компьютеров** контейнера в соответствующем домене.
4. Щелкните правой кнопкой мыши компьютер, который вы хотите назначить в качестве основного компьютера, а затем выберите **свойства**.
5. В области навигации выберите **расширения**.
6. Выберите **редактор атрибутов** вкладке, прокрутите до **distinguishedName**выберите **представление**, щелкните правой кнопкой мыши значение в списке, выберите **копирования**, Выберите **ОК**, а затем выберите **отменить**.
7. Перейдите к **пользователей** контейнера в соответствующем домене, щелкните правой кнопкой мыши пользователя, к которому вы хотите назначить компьютер, а затем выберите **свойства**.
8. В области навигации выберите **расширения**.
9. Выберите **редактор атрибутов** выберите **msDs-PrimaryComputer** , а затем выберите **изменить**. Отобразится диалоговое окно "Редактор многозначных строк".
10. Щелкните текстовое поле правой кнопкой мыши **вставить**выберите **добавить**выберите **ОК**, а затем выберите **ОК** еще раз.

## <a name="step-2-optionally-enable-primary-computers-for-folder-redirection-in-group-policy"></a>Шаг 2. При необходимости включите основных компьютеров для перенаправления папок в групповой политике

Далее необходимо дополнительно настроить групповую политику для включения поддержки основного компьютера для перенаправления папок. Это позволяет папки пользователей, перенаправление на компьютерах, обозначенных как основные компьютеры пользователя, но не на других компьютерах. Вы можете управлять основных компьютеров для перенаправления папок на отдельного компьютера или каждого пользователя.

Вот как можно включить основных компьютеров для перенаправления папок:

1. В управлении групповыми политиками, щелкните правой кнопкой мыши объект групповой Политики, созданной при выполнении начальной конфигурации перенаправления папок и перемещаемых профилей пользователей (например, **параметры перенаправления папок** или **перемещаемых пользователя Настройки профилей**), а затем выберите **изменить**.
2. Чтобы включить поддержку основных компьютеров, с помощью групповой политики на компьютере, перейдите к **Конфигурация компьютера**. Для пользователя на основе групповой политики, перейдите к **Конфигурация пользователя**.
    - На компьютере Групповая политика применяется обработки основного компьютера для всех компьютеров, к которым применяется объект групповой Политики, влияющие на всех пользователей, компьютеров.
    - Пользователя на основе групповой политики к применяется обработки основного компьютера всем учетным записям пользователей, к которым применяется объект групповой Политики, влияющие на все компьютеры, к которым пользователи будут входить на.
3. В разделе **Конфигурация компьютера** или **Конфигурация пользователя**, перейдите к **политики**, затем **административные шаблоны**, а затем  **Система**, затем **перенаправления папок**.
4. Щелкните правой кнопкой мыши **перенаправить папки только на основных компьютерах**, а затем выберите **изменить**.
5. Выберите **включено**, а затем выберите **ОК**.

## <a name="step-3-optionally-enable-primary-computers-for-roaming-user-profiles-in-group-policy"></a>Шаг 3. При необходимости включите основные компьютеры для перемещаемых профилей пользователей в групповой политике

Далее необходимо дополнительно настроить групповую политику для включения поддержки основного компьютера для перемещаемых профилей пользователей. Это позволяет профиля пользователя, чтобы можно было использовать на компьютерах, обозначенных как основные компьютеры пользователя, но не на других компьютерах.

Вот как можно включить основные компьютеры для перемещаемых профилей пользователей:

1. Включение поддержки основного компьютера для перенаправления папок, если у вас еще нет.<br>Это позволяет сохранять документы и другие файлы пользователя из профилей пользователей, которое помогает профили оставаться небольшими и входить в раз оставаться fast.
2. В управлении групповыми политиками, щелкните правой кнопкой мыши созданный объект групповой Политики (к примеру, **настройки перемещаемых профилей пользователей и перенаправления папок**), а затем выберите **изменить**.
3. Перейдите к **Конфигурация компьютера**, затем **политики**, затем **административные шаблоны**, затем **системы**, а затем **Профили пользователей**.
4. Щелкните правой кнопкой мыши **загрузить перемещаемые профили только, на основных компьютерах** , а затем выберите **изменить**.
5. Выберите **включено**, а затем выберите **ОК**.

## <a name="step-4-enable-the-gpo"></a>Шаг 4. Включить объект групповой Политики

После завершения настройки перенаправления папок и перемещаемых профилей пользователей, включить объект групповой Политики, если вы еще этого не сделали. Это позволяет применять для затронутых пользователей и компьютеров.

Вот как можно включить перенаправление папок и перемещаемые профили групповой политики пользователя:

1. Откройте Управление групповой политикой
2. Щелкните правой кнопкой мыши объекты групповой политики, которые созданы и затем выберите **связь включена**. Рядом с элементом меню появится флажок.

## <a name="step-5-test-primary-computer-function"></a>Шаг 5. Тестирование функции основного компьютера

Чтобы проверить поддержку основного компьютера, выполните вход на основном компьютере, убедитесь, что перенаправление папок и профилей, затем войдите на неосновной компьютер и подтверждение не перенаправление папок и профилей.

Вот как можно проверить функциональность основного компьютера:

1. Войдите на указанный главный компьютер с учетной записью пользователя, для которого вы включили перенаправление папок и перемещаемых профилей пользователей.
2. Если учетной записи пользователя был направлен через ранее на компьютере, откройте сеанс Windows PowerShell или окно командной строки с правами администратора, введите следующую команду и затем выход при появлении запроса, чтобы убедиться, что последние настройки групповой политики применяются к клиентский компьютер:

    ```PowerShell
    Gpupdate /force
    ```

3. Откройте проводник.
1. Щелкните правой кнопкой мыши в перенаправленной папке (например, папка «Мои документы» в библиотеке документов), а затем выберите **свойства**.
1. Выберите **расположение** вкладке и убедитесь, что отобразилась путь общей папки, которые вы указали, а не локальный путь. Чтобы убедиться, что профиль пользователя перемещается, откройте **панели управления**выберите **система и безопасность**выберите **системы**выберите **Дополнительные параметры системы** выберите **параметры** в профили пользователей разделе и выполните поиск **перемещаемых** в **тип** столбца.
1. Войдите с помощью той же учетной записью на компьютере, которые не указаны в качестве основного компьютера пользователя.
1. Повторите шаги 2 – 5, вместо этого поиска для локальных путей и **локального** тип профиля.

> [!NOTE]
> Если папки были перенаправлены на компьютере, прежде чем вы включили поддержку основного компьютера, папки, остаются, поэтому перенаправленный следующий параметр задан в параметре политики перенаправления папок каждой папки: **После удаления политики перенаправить папку обратно в расположение локального профиля пользователя**. Аналогичным образом, профили, которые ранее перемещаемых на конкретном компьютере будет отображаться **перемещаемых** в **тип** столбцов, однако **состояние** столбец будет содержать **Локального**.
