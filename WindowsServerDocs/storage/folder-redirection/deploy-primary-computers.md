# <a name="deploy-primary-computers-for-folder-redirection-and-roaming-user-profiles"></a>Развертывание основных компьютеров для перенаправления папок и перемещаемых профилей пользователей

>Относится к: Windows 10, Windows 8, Windows 8.1, Windows Server 2019, Windows Server 2016, Windows Server 2012, Windows Server 2012 R2

В этом разделе описывается включение поддержки основного компьютера и назначение пользователям основных компьютеров. Это позволяет контролировать, какие компьютеры используют перенаправление папок и перемещаемые профили пользователей.

> [!IMPORTANT]
> При включении поддержки основного компьютера для перемещаемых профилей пользователей всегда включайте поддержку основного компьютера для перенаправления папок. Это позволяет хранить в профилях пользователей документы и другие файлы пользователей, что помогает сохранять небольшую длину и время входа.

## <a name="prerequisites"></a>Предварительные требования

## <a name="software-requirements"></a>Требования к программному обеспечению

Поддержка основного компьютера имеет следующие требования.

- Схема домен Active Directory Services (AD DS) должна быть обновлена для включения дополнений схемы Windows Server 2012 (Установка контроллера домена Windows Server 2012 автоматически обновляет схему). Сведения об обновлении схемы AD DS см. в разделе [Интеграция Adprep. exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh472161(v=ws.11)#adprepexe-integration>) и [выполнение программы Adprep. exe](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/dd464018(v=ws.10)>).
- Клиентские компьютеры должны работать под управлением Windows 10, Windows 8.1, Windows 8, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.

> [!TIP]
> Несмотря на то, что поддержка основного компьютера требует перенаправления папок и (или) перемещаемых профилей пользователей, при первом развертывании этих технологий рекомендуется настроить поддержку основного компьютера перед включением объектов групповой политики, которые настраивают перенаправление папок, и Перемещаемые профили пользователей. предотвратить копирование данных пользователя на неосновные компьютеры до включения поддержки основного компьютера. Сведения о конфигурации см. в разделе [Развертывание перенаправления папок](deploy-folder-redirection.md) и [развертывание перемещаемых профилей пользователей](deploy-roaming-user-profiles.md).

## <a name="step-1-designate-primary-computers-for-users"></a>Шаг 1. Назначение основных компьютеров для пользователей

Первым шагом при развертывании основных компьютеров является назначение основных компьютеров каждому пользователю. Для этого используйте центр администрирования Active Directory, чтобы получить различающееся имя соответствующих компьютеров, а затем установите атрибут **msDs-примарикомпутер** .

> [!TIP]
> Чтобы использовать Windows PowerShell для работы с основными компьютерами, см. запись блога, [продвигаясь вглубь чуть глубже в основной компьютер Windows 8](<https://blogs.technet.microsoft.com/askds/2012/10/23/digging-a-little-deeper-into-windows-8-primary-computer/>).

Вот как можно указать основные компьютеры для пользователей:

1. Откройте диспетчер сервера на компьютере с установленными средствами администрирования Active Directory.
2. В меню **Сервис** выберите **Active Directory центр администрирования**. Отобразится центр администрирования Active Directory.
3. Перейдите к контейнеру **Компьютеры** в соответствующем домене.
4. Щелкните правой кнопкой мыши компьютер, который необходимо назначить основным компьютером, и выберите пункт **Свойства**.
5. В области навигации выберите **расширения**.
6. Перейдите на **вкладку Редактор атрибутов** , прокрутите до **distinguishedName**, выберите **вид**, щелкните правой кнопкой мыши значение в списке, выберите **Копировать**, нажмите кнопку **ОК**, а затем нажмите **кнопку Отмена**.
7. Перейдите к контейнеру **Пользователи** в соответствующем домене, щелкните правой кнопкой мыши пользователя, которому требуется назначить компьютер, и выберите пункт **свойства**.
8. В области навигации выберите **расширения**.
9. Перейдите на вкладку **Редактор атрибутов** , выберите **msDs-примарикомпутер** и нажмите кнопку **изменить**. Отобразится диалоговое окно "Редактор многозначных строк".
10. Щелкните правой кнопкой мыши текстовое поле, выберите пункт **Вставить**, выберите **Добавить**, нажмите кнопку **ОК**, а затем еще раз нажмите кнопку **ОК** .

## <a name="step-2-optionally-enable-primary-computers-for-folder-redirection-in-group-policy"></a>Шаг 2. При необходимости включите основные компьютеры для перенаправления папок в групповая политика

Следующий шаг — при необходимости настройте групповая политика, чтобы обеспечить поддержку основного компьютера для перенаправления папок. Это позволяет перенаправлять папки пользователя на компьютерах, определенных как основные компьютеры пользователя, но не на других компьютерах. Можно управлять основными компьютерами для перенаправления папок отдельно для каждого компьютера или для каждого пользователя.

Вот как можно включить основные компьютеры для перенаправления папок:

1. В групповая политика управления щелкните правой кнопкой мыши объект групповой политики, созданный при выполнении начальной настройки перенаправления папок и (или) перемещаемых профилей пользователей (например, **Параметры перенаправления папок** или **параметры перемещаемых профилей пользователей**), а затем Выберите **изменить**.
2. Чтобы включить поддержку основных компьютеров с помощью групповая политика на основе компьютеров, перейдите в раздел **Конфигурация компьютера**. Для групповая политика на основе пользователей перейдите в раздел **Конфигурация пользователя**.
    - Групповая политика на основе компьютеров применяет обработку основного компьютера ко всем компьютерам, к которым применяется объект групповой политики, влияя на всех пользователей компьютеров.
    - Групповая политика на основе пользователей для применения обработки основного компьютера ко всем учетным записям пользователей, к которым применяется объект групповой политики, что влияет на все компьютеры, на которых они входят.
3. В разделе **Конфигурация компьютера** или **Конфигурация пользователя**перейдите к разделу **политики**, затем **Административные шаблоны**, затем **система**и **Перенаправление папок**.
4. Щелкните правой кнопкой мыши **папку перенаправить только на основных компьютерах**, а затем выберите **изменить**.
5. Выберите **включено**и нажмите кнопку **ОК**.

## <a name="step-3-optionally-enable-primary-computers-for-roaming-user-profiles-in-group-policy"></a>Шаг 3. При необходимости включите основные компьютеры для перемещаемых профилей пользователей в групповая политика

Следующий шаг — при необходимости настройте групповая политика, чтобы включить поддержку основного компьютера для перемещаемых профилей пользователей. Это позволяет профилю пользователя перемещаться по компьютерам, назначенным основным компьютерам пользователя, но не на других компьютерах.

Вот как можно включить основные компьютеры для перемещаемых профилей пользователей:

1. Включите поддержку основного компьютера для перенаправления папок, если это еще не сделано.<br>Это позволяет хранить в профилях пользователей документы и другие файлы пользователей, что помогает сохранять небольшую длину и время входа.
2. В групповая политика управления щелкните правой кнопкой мыши созданный объект групповой политики (например, **Перенаправление папок и параметры перемещаемых профилей пользователей**), а затем выберите **изменить**.
3. Перейдите в раздел **Конфигурация компьютера**, затем **политики**, затем **Административные шаблоны**, **система**и **Профили пользователей**.
4. Щелкните правой кнопкой мыши пункт **загрузить перемещаемые профили только на основных компьютерах,** а затем выберите **изменить**.
5. Выберите **включено**и нажмите кнопку **ОК**.

## <a name="step-4-enable-the-gpo"></a>Шаг 4. Включение объекта групповой политики

После завершения настройки перенаправления папок и перемещаемых профилей пользователей включите объект групповой политики, если он еще не создан. Это позволяет применить его к затронутым пользователям и компьютерам.

Вот как можно включить объекты групповой политики для перенаправления папок и (или) перемещаемых профилей пользователей:

1. Открыть управление групповая политика
2. Щелкните правой кнопкой мыши созданные объекты групповой политики и выберите пункт **связь включена**. Рядом с пунктом меню должен появиться флажок.

## <a name="step-5-test-primary-computer-function"></a>Шаг 5. Проверка функции основного компьютера

Чтобы проверить поддержку основного компьютера, войдите на основной компьютер, убедитесь, что папки и профили перенаправлены, затем войдите на компьютер, не являющийся первичным, и убедитесь, что папки и профили не перенаправляются.

Вот как можно протестировать функциональность основного компьютера:

1. Войдите на назначенный основной компьютер с учетной записью пользователя, для которой включены перенаправление папок и/или перемещаемые профили пользователей.
2. Если учетная запись пользователя уже выполнила вход на компьютере, откройте сеанс Windows PowerShell или окно командной строки от имени администратора, введите следующую команду и выйдите из системы, чтобы убедиться, что последние параметры групповая политика применены к клиентский компьютер:

    ```PowerShell
    Gpupdate /force
    ```

3. Откройте проводник.
1. Щелкните правой кнопкой мыши перенаправленную папку (например, папку Мои документы в библиотеке документов) и выберите пункт **Свойства**.
1. Перейдите на вкладку **Расположение** и убедитесь, что в пути отображается указанная общая папка вместо локального пути. Чтобы убедиться, что профиль пользователя является перемещаемым, откройте **Панель управления**, **выберите система и безопасность**, выберите **система**, выберите **Дополнительные параметры системы**, в разделе Профили пользователей выберите **Параметры** , а затем найдите  **Перемещение** в столбце **тип** .
1. Войдите с помощью той же учетной записи пользователя на компьютер, который не обозначен как основной компьютер пользователя.
1. Повторите шаги 2 – 5, чтобы найти локальные пути и тип **локального** профиля.

> [!NOTE]
> Если папки были перенаправлены на компьютер до включения поддержки основного компьютера, папки останутся перенаправлены, если для каждой папки не будет настроен следующий параметр политики перенаправления папки: **При удалении политики Перенаправление папки обратно в локальную папку профиля**пользователя. Аналогичным образом, профили, которые ранее были перемещены на определенном компьютере, будут показывать **роуминг** в столбцах **типа** . Однако в столбце **состояние** будет отображаться **локальный**.
