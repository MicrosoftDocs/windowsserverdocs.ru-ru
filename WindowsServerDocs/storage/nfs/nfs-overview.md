# Обзор NFS

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В этом разделе описываются службы роли сетевой файловой системы и компонентов с ролью сервера файловые службы и службы хранилища в Windows Server. Сетевой файловой системы (NFS) предоставляет решение для предприятий с разнородными средами, которые включают Windows и включающими компьютеры к файлам.

## Описание компонента

С помощью протокола NFS можно перенести файлы между компьютерами под управлением Windows и других операционных систем без Windows, например Linux или UNIX.

NFS в Windows Server включает в себя сервер для NFS и клиент для NFS. Компьютер под управлением Windows Server можно использовать сервер для NFS в качестве NFS файловый сервер для других компьютеров без Windows клиента. Клиент для NFS позволяет компьютеру под управлением Windows под управлением Windows Server для доступа к файлам, хранящимся на сервере NFS, отличных от Windows.

## Версии Windows и Windows Server

Windows поддерживает несколько версий NFS клиент и сервер, в зависимости от версии операционной системы и семейства. 

| Операционные системы | Серверные версии NFS |Версии клиента NFS|
| ----------------- | ------------------- | ----------------- |
| Windows 7, Windows 8.1, Windows 10 | Н/Д | NFSv2, NFSv3 |
| Windows Server 2008, Windows Server 2008 R2 | NFSv2, NFSv3 | NFSv2, NFSv3 |
| Windows Server 2012, Windows Server 2012 R2, Windows Server 2016, Windows Server 2019 г. | NFSv2, NFSv3, NFSv4.1  | NFSv2, NFSv3 |

## Практическое применение

Ниже приведены некоторые способы, вы можете использовать NFS.

- Используйте файловый сервер Windows NFS для предоставления доступа нескольких протоколов в том же общую через протоколы SMB и NFS от клиентов в нескольких платформ.
- Развертывание Windows NFS файлового сервера в среде операционной системы преимущественно Windows для предоставления включающими клиентских компьютеров доступа к файловые ресурсы NFS.
- Перенос приложений за счет хранения данных на файловые ресурсы доступны через протоколы SMB и NFS из одной операционной системы в другой.

## NFS версии 4.1

NFS версии 4.1 реализует все необходимые аспекты, помимо некоторые аспекты необязательно, [RFC 5661](https://tools.ietf.org/html/rfc5661):

- **Псевдо файловой системы**, файловая система, которая отделяет физическим и логическим пространства имен и совместим с NFS версии 3 и NFS версии 2. Псевдоним предоставляется системы экспортированному файлу, который является частью псевдо файловой системы.
- **Составные RPC** объединить различные операции и уменьшить за разговорчивости.
- **Сеансы и магистральных сеанса** позволяет только один семантика и непрерывной доступности, а также более высокую производительность при использовании нескольких сетей между клиентами NFS 4.1 и сервер NFS.

## Инфраструктура NFS

Ниже подробно описаны усовершенствования в общую инфраструктуру NFS в Windows Server 2012:

- **Удаленного вызова процедур (RPC) и внешних представления данных (XDR)** инфраструктура транспорта, компонентам сетевой протокол WinSock, доступен для обоих сервер для NFS и клиент для NFS. Это заменяет интерфейс устройства транспорта (TDI), лучше поддерживает и обеспечивает более высокую масштабируемость и масштабирование получать стороне (RSS).
- Эта функция **порт RPC мультиплексор** брандмауэром (меньше портов для управления) и упрощает развертывание NFS.
- **Кэши автоматической настройки и пулы потоков** , возможности управления ресурсами новой инфраструктуры RPC/XDR, являются динамическими, автоматически настройки кэши и группы, в зависимости от рабочей нагрузки потоков. Это полностью отменяет участвующих догадки, при настройке параметров, предоставляя оптимальную производительность сразу же после развертывания NFS.
- **Параметры конфиденциальности реализации и проверки подлинности Kerberos новый** с добавлением поддержки Kerberos конфиденциальности (Krb5p), а также существующих krb5 и krb5i параметры проверки подлинности.
- **Модуль Windows PowerShell сопоставления удостоверений** командлеты упростить управление удостоверений сопоставления, настройки служб Active Directory облегченного доступа (AD LDS) и настроить UNIX и Linux паролей и плоские файлы.
- **Точки подключения тома** позволяет выполнять доступ к тома, подключенные в общую папку NFS с NFS версии 4.1.
- Функция **Мультиплексирования порт** поддерживает RPC порт мультиплексор (2049), который брандмауэрами и упрощает развертывание NFS.

## Непрерывную доступность NFS версии 3

NFS версии 3 клиенты могут иметь быстрый и прозрачные запланированных переходы дополнительную доступность и сокращение простоев. Процесс перехода на другой ресурс связано быстрее для клиентов NFS версии 3:

- Инфраструктуре кластеризации теперь позволяет один ресурс по имени сети вместо одного ресурса на общий ресурс, который значительно сокращает время отработки отказа ресурсов.
- Пути отработки отказа в NFS-сервера настроены для повышения производительности.
- Подстановочный знак регистрации на сервере NFS больше не требуется, а переходы более точного.
- Отправки уведомления сети монитора состояния (NSM) после отработки отказа процесса и клиенты больше не нужно ожидать время ожидания TCP для неудачных подключения через сервер.

Обратите внимание, что сервер для NFS поддерживает прозрачные отказоустойчивости только в том случае, когда инициируется вручную, обычно во время запланированного обслуживания. В случае незапланированную отработку отказа клиентам NFS теряется их подключения. Сервер для NFS также не имеет любой интеграции с фильтром ключ возобновления. Это означает, что локального приложения или сеанса SMB пытается получить доступ к тот же файл, клиент NFS обращается к сразу же после запланированного отработки отказа, клиент NFS может привести к потере его (прозрачный отработки отказа не успешного) подключения.

## Развертывание и более широкие возможности

Развертывание и управление ими NFS обладает улучшенными следующим образом:

- Более чем 40 новых командлетов Windows PowerShell упростить для настройки и администрирования NFS файловые ресурсы. Дополнительные сведения см. в разделе [NFS командлеты Windows PowerShell](https://docs.microsoft.com/powershell/module/nfs/?view=win10-ps).
- Сопоставление удостоверения повышена с локального плоский файл сопоставления магазина и новых командлетов Windows PowerShell для настройки сопоставления удостоверений.
- Диспетчер серверов, графический пользовательский интерфейс проще в использовании.
- Новый поставщик WMI версии 2 доступна для упрощения управления.
- Порт мультиплексор (порт RPC 2049) брандмауэрами и упрощает развертывание NFS.

## Сведения о диспетчере серверов

Диспетчер серверов - или более поздней версии [Windows Admin Center](../../manage/windows-admin-center/understand/windows-admin-center.md) — использовать функции мастера добавления ролей и добавить сервер для NFS роль службы (в файле и iSCSI роль служб). Общие сведения об установке компонентов см. в разделе [установки или удаление ролей, служб ролей и компонентов](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/hh831809(v=ws.11)>). Сервер для NFS средства входят службы для сетевых файлов системы оснастки управления на сервер для NFS и клиент для NFS компонентов. С помощью оснастки, вы можете управлять на сервер для NFS компоненты, установленные на компьютере. Сервер для NFS также содержит несколько средств администрирования из командной строки Windows:

- **Подключите** локально присоединение удаленной общей папке NFS (также известной как экспортируемый) и сопоставляет его с локальный диск на клиентском компьютере Windows.
- **Nfsadmin** позволяет управлять параметрами конфигурации сервера для NFS и клиент для NFS компонентов.
- **Nfsshare** настраивает параметры NFS общий ресурс для папки, которые передаются через сервер для NFS.
- **Nfsstat** отображает или сброс статистика полученных сервером для NFS вызовов.
- **Showmount** отображает подключенный файловыми системами, экспортированного из сервер для NFS.
- **Umount** удаляет NFS подключенные диски.

NFS в Windows Server 2012 представляет модуль NFS для Windows PowerShell с помощью несколько новых командлетов, специально для NFS. Эти командлеты предоставляют простой способ для автоматизации задач управления NFS. Дополнительные сведения см. в разделе [NFS командлеты Windows PowerShell](https://docs.microsoft.com/powershell/module/nfs/?view=win10-ps).
