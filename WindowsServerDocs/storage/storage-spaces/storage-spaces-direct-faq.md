# <a name="storage-spaces-direct---frequently-asked-questions-faq"></a>Дисковые пространства — часто задаваемые вопросы (FAQ)

В этой статье перечислены некоторые наиболее распространенные и часто задаваемые вопросы, связанные с [Storage Spaces Direct](storage-spaces-direct-overview.md).

## <a name="when-you-use-storage-spaces-direct-with-3-nodes-can-you-get-both-performance-and-capacity-tiers"></a>При использовании дисковых пространств с 3 узлами, вы получаете уровни емкости и производительности?

Да, можно получить уровень емкости и производительности в узел 2 или 3-узловой конфигурации дисковых пространств. Тем не менее необходимо убедиться в том, что у вас есть 2 емкость устройства. Это означает, что необходимо использовать все три типа устройств: NVME SSD и HDD.
 
## <a name="refs-file-system-provides-real-time-tiaring-with-storage-spaces-direct-does-refs-provides-the-same-functionality-with-shared-storage-spaces-in-2016"></a>Файловая система Refs предоставляет в реальном времени tiaring с дисковыми пространствами. REFS предоставляет те же функциональные возможности с помощью общих дисковых пространств в 2016?

Нет, вы не получите в реальном времени, распределение по уровням с помощью общих дисковых пространств с 2016 г. Это необходимо только для дисковых пространств. 
 
## <a name="can-i-use-an-ntfs-file-system-with-storage-spaces-direct"></a>Можно ли использовать файловую систему NTFS с дисковыми пространствами
  
Да, можно использовать в файловой системе NTFS с дисковыми пространствами. Тем не менее рекомендуется REFS. NTFS не предоставляет в реальном времени распределение по уровням. 
 
## <a name="i-have-configured-2-node-storage-spaces-direct-clusters-where-the-virtual-disk-is-configured-as-2-way-mirror-resiliency-if-i-add-a-new-fault-domain-will-the-resiliency-of-the-existing-virtual-disk-change"></a>Я настроил 2 дисковых кластеры, которой задан виртуальный диск как устойчивость двустороннее зеркало. Если добавить новый домен сбоя, изменится на устойчивость в существующий виртуальный диск

После добавления нового домена сбоя, новые виртуальные диски, создаваемые перейдет к трехстороннего зеркала. Однако существующий виртуальный диск будет оставаться двунаправленного зеркальный диск. Данные можно скопировать в новые виртуальные диски из существующих томов, чтобы воспользоваться преимуществами нового устойчивости.
 
## <a name="the-storage-spaces-direct-was-created-using-the-autoconfig0-switch-and-the-pool-created-manually-when-i-try-to-query-the-storage-spaces-direct-pool-to-create-a-new-volume-i-get-a-message-that-says-enable-clusters2d-again-what-should-i-do"></a>Storage Spaces Direct был создан с помощью autoconfig:0 коммутатор и пуле, созданных вручную. При попытке запросить пул дисковых, чтобы создать новый том, я получаю сообщение об ошибке: «Enable-ClusterS2D еще раз.» Что мне делать?

По умолчанию при настройке дисковых пространств с помощью командлета enable-S2D, командлет выполняет все автоматически. Он создает пул и уровни. При использовании autoconfig:0, все должны выполняться вручную. Если вы создали только пул, уровень не обязательно создается. Если у вас либо вообще не создан уровни или не создан уровни в виде соответствующих устройств, подключенных вы получите сообщение об ошибке «Снова Enable-ClusterS2D». Мы рекомендуем не использовать параметр autoconfig в рабочей среде. 
 
## <a name="is-it-possible-to-add-a-spinning-disk-hdd-to-the-storage-spaces-direct-pool-after-you-have-created-storage-spaces-direct-with-ssd-devices"></a>Это можно добавить в пул дисковых вращающийся диск (HDD) созданного дисковыми пространствами с помощью устройств SSD?

Нет. По умолчанию при использовании типа одно устройство для создания пула, его не следует настроить диски кэша, и все диски, которые будут использоваться для емкости. Можно добавить в конфигурацию дисков NVME и диски NVME будет настроена для кэша.
 
## <a name="i-have-configured-a-2-rack-fault-domain-rack-1-has-2-fault-domains-rack-2-has-1-fault-domain-each-server-has-4-capacity-100-gb-devices-can-i-use-all-1200-gb-of-space-from-the-pool"></a>Домен сбоя 2-rack настройки: СТОЙКА 1 имеет в двух доменах сбоя, СТОЙКИ 2 — 1 домен сбоя. Каждый сервер имеет 4 устройств емкостью 100 ГБ. Можно ли использовать все 1200 ГБ пространства из пула?

Нет, можно использовать только емкостью 800 ГБ. В домене сбоя стойки необходимо убедиться в том, наличие конфигурации двустороннее зеркало, так что каждый избавиться и его land повторяющиеся в различных стойку.
 
## <a name="what-should-the-cache-size-be-when-i-am-configuring-storage-spaces-direct"></a>Размер кэша выполняемые при я настройке дисковых пространств?

Изменения размеров кэша для размещения рабочего набора (данные, которые активно чтения или записи в любой момент времени) рабочих нагрузок и приложений.

## <a name="how-can-i-determine-the-size-of-cache-that-is-being-used-by-storage-spaces-direct"></a>Как определить размер кэша, который используется дисковыми пространствами

Используйте встроенную служебную программу системного монитора для изучения промахов кэша. Просмотрите кэша промах операций чтения в секунду от счетчика диска гибридного хранилища кластера. Помните, что если слишком много операций чтения, что отсутствуют кэша недостаточного размера кэша, и вы можете развернуть его. 
 
## <a name="is-there-a-calculator-that-shows-the-exact-size-of-the-disks-that-are-being-set-aside-for-cache-capacity-and-resiliency-that-would-enable-me-to-plan-better"></a>Есть ли калькулятора, показывающий точный размер дисков, который резервируется для кэша, емкость и устойчивость, которая позволила бы мне более эффективное планирование?

Чтобы помочь в планировании можно использовать калькулятор размера хранилища пробелы. Оно доступно в http://aka.ms/s2dcalc.
 
## <a name="what-is-the-best-configuration-that-you-would-recommend-when-configuring-6-servers-and-3-racks"></a>Что такое наилучшую конфигурацию, что вы порекомендуете при настройке шести серверов и 3 стойки?

Используйте два сервера на каждом из стойки, чтобы получить устойчивость виртуальных дисков для трехстороннего зеркала. Помните, что стойки конфигурации будет работать правильно только в том случае, если нужно предоставить конфигурацию ОС, так же, как разместить его в стойку. 
 
## <a name="can-i-enable-maintenance-mode-for-a-specific-disk-on-a-specific-server-in-storage-spaces-direct-cluster"></a>Можно включить режим обслуживания для определенного диска на конкретном сервере в кластере Storage Spaces Direct?

Да, можно включить режим обслуживания хранилища на определенном диске и с определенным доменом сбоя. Команды Enable-StorageMaintenanceMode вызывается автоматически, когда узел приостановлен. Вы можете включить для определенного диска, выполнив следующую команду:

```powershell
Get-PhysicalDisk -SerialNumber <SerialNumber> | Enable-StorageMaintenanceMode
```

## <a name="is-storage-spaces-direct-supported-on-my-hardware"></a>Storage Spaces Direct поддерживается на моего устройства?

Рекомендуется связаться с поставщиком оборудования, чтобы проверить поддержку. Поставщики оборудования тестирование решения на их оборудования и комментарий о его поддержке или нет. Например, во время написания статьи, серверов, таких как R730 / R730xd / R630, имеют более чем 8 разъемов для дисков может поддерживать SES и совместимы с дисковыми пространствами. Dell поддерживает только HBA330 с дисковыми пространствами. R620 не поддерживает SES и не совместим с дисковыми пространствами.

Для большего количества оборудования сведения о поддержке, перейдите на следующий веб-сайт: Каталог Windows Server
 
## <a name="how-does-storage-spaces-direct-make-use-of-ses"></a>Как дисковыми пространствами, чтобы использовать SES?

Storage Spaces Direct использует сопоставление SCSI корпуса Services (SES) чтобы убедиться в том, что слоев данных и метаданных распределены между доменами сбоя в виде устойчивой. Если оборудование не поддерживает SES, отсутствует сопоставление корпусов и расположение данных не становится отказоустойчивым.
 
## <a name="what-command-can-you-use-to-check-the-physical-extent-for-a-virtual-disk"></a>Какие команды можно использовать для проверки области физической памяти для виртуального диска?
  
Этот:

```powershell
get-virtualdisk -friendlyname “xyz” | get-physicalextent
```
