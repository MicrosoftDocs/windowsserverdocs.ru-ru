# Локальные дисковые пространства-часто задаваемые вопросы и ответы

В этой статье перечислены некоторые распространенные и часто задаваемых вопросов, связанных с [Локальных дисковых пространств](storage-spaces-direct-overview.md).

## При использовании локальных дисковых пространств с 3 узлами, можно получить производительности и емкости уровней?

Да, вы можете получить производительность и уровень емкости в узел 2 или 3-узел конфигурации локальных дисковых пространств. Тем не менее необходимо убедиться, что у вас есть 2 устройств хранения. Это означает, что необходимо использовать три типа устройства: NVME, SSD и HDD.
 
## Файловая система Refs обеспечивает tiaring в режиме реального времени с помощью локальных дисковых пространств. REFS предоставляет те же функции общего дисковых пространств в 2016?

Нет, вы не получите таким общего дисковых пространств с 2016 в реальном времени. Он используется только для локальных дисковых пространств. 
 
## Можно использовать в файловой системе NTFS с локальными дисковыми пространствами
  
Да, с помощью локальных дисковых пространств можно использовать в файловой системе NTFS. Тем не менее мы рекомендуем REFS. NTFS не предоставляет в режиме реального времени, обеспечивающий простое подключение. 
 
## Я настроили 2 кластеров локальных дисковых пространств узел, где виртуальный диск, который настроен как двухстороннее зеркало устойчивости. Если добавить новый домен сбоя, изменится устойчивости существующего виртуального диска

После добавления новых домен сбоя новые виртуальные диски, которые вы создаете перемещаются трехстороннего зеркального. Тем не менее существующий виртуальный диск будет оставаться зеркальный диск способ 2. Вы можете скопировать новые виртуальные диски данных из существующего тома, чтобы воспользоваться преимуществами новых устойчивости.
 
## Локальные дисковые была создана с помощью переключателя autoconfig:0 и пула, созданного вручную. При попытке запросить локальных дисковых пространств пула, чтобы создать новый том, появляется сообщение о том, «Enable-ClusterS2D еще раз.» Что мне делать?

По умолчанию при настройке локальных дисковых пространств с помощью командлета enable-S2D командлет выполняет все. Он создает пул и уровни. При использовании autoconfig:0, все, что необходимо сделать вручную. Если вы создали только пула, уровень не обязательно создается. Если вы используете не создаются уровни вообще или не создан уровней таким образом, соответствующий с устройствами, подключенными вы получите сообщение об ошибке «Снова Enable-ClusterS2D». Мы рекомендуем не использовать параметр автоматической настройки в рабочей среде. 
 
## Имеется возможность добавления вращающийся диск (HDD) в пул локальных дисковых пространств, создав локальных дисковых пространств с устройствами SSD?

Нет. По умолчанию при использовании типа одно устройство для создания пула его не следует настроить кэш дисков и все диски, которые будут использоваться для хранения. Можно добавить диски NVME в конфигурации и диски NVME будет настроена для кэша.
 
## Я настроили домен сбоя стоек 2: 1 СТОЕК имеет 2 доменов сбоя, СТОЕК 2 — 1 домен сбоя. Каждый сервер имеет 4 устройств хранения 100 ГБ. Можно использовать все 1 200 ГБ доступного из пула?

Нет, можно использовать только 800 ГБ. В домен сбоя стоек необходимо убедиться, наличие двухстороннее зеркало конфигурации, таким образом, в которых избавиться и его повторяющихся Земли в разных стойку.
 
## Что размер кэша следует настраивая я локальных дисковых пространствах?

Размер кэша должен могла рабочий набор (данных, которое активно для чтения или записи в любой момент времени) ваших приложений и рабочих нагрузок.

## Как определить размер кэша, который используется с локальных дисковых пространств

Для проверки показателя промахов в кэше используйте встроенную служебную программу системного монитора. Просмотрите операции чтения/sec неправильно кэш счетчика гибридный диск кластера хранилища. Помните, что если слишком много операций чтения не попадают в кэш, кэш недостаточным и может потребоваться развернуть его. 
 
## Существует ли калькулятора, который показывает точный размер диски, на которых выполняется резервируется для кэша, емкости и устойчивости, которое необходимо предпринять, чтобы лучше запланировать?

Калькулятор локальных дисковых пространств можно использовать для упрощения планирования. Он будет доступен на момент http://aka.ms/s2dcalc.
 
## Что такое лучше конфигурацию, которая будет рекомендуем при настройке 6 серверов и 3 этажах?

Используйте 2 сервера на каждом из стойках для получения устойчивости виртуального диска трехстороннего зеркального. Помните, что конфигурация стоек будет работать правильно только в том случае, если вы предоставляете конфигурации для операционной системы, таким образом, который будет располагаться на стойке. 
 
## Можно включить режим обслуживания для конкретного диска на конкретном сервере в кластере локальных дисковых пространств

Да, вы можете включить режим обслуживания хранилища на определенный диск и сбоям на уровне домена. Команды Enable-StorageMaintenanceMode вызывается автоматически при приостановке узла. Его можно включить для конкретного диска, выполнив следующую команду:

```powershell
Get-PhysicalDisk -SerialNumber <SerialNumber> | Enable-StorageMaintenanceMode
```

## Локальные дисковые пространства поддерживается на оборудование?

Мы рекомендуем связаться с поставщиком оборудования, чтобы проверить поддержку. Поставщики оборудования проверьте решение на своих оборудования и комментарий о том, поддерживается ли или нет. Например, во время этого письмо, серверы, такие как R730 / R730xd / R630, которые имеют более 8 гнезда диска может поддерживать SES и совместимы с локальными дисковыми пространствами. Dell поддерживает только HBA330 с локальными дисковыми пространствами. R620 не поддерживает SES и не совместим с локальными дисковыми пространствами.

Для оборудования, Дополнительные сведения о поддержке, перейдите на следующий веб-сайт: каталог Windows Server
 
## Как локальных дисковых пространств, чтобы использовать SES?

Локальные дисковые пространства использует сопоставление SCSI Enclosure Services (SES) убедитесь, что бетонные плиты; данных и метаданные распределены по доменов сбоя устойчивыми образом. Если оборудование не поддерживает SES, отсутствует сопоставление корпусов и размещение данных не resilient.
 
## Какие команды можно использовать для проверки физическая область виртуального диска?
  
Этот:

```powershell
get-virtualdisk -friendlyname “xyz” | get-physicalextent
```
