# <a name="troubleshoot-storage-spaces-direct-health-and-operational-states"></a>Устранение неполадок с дисковыми пространствами работоспособности и состояния работоспособности

> Относится к: Windows Server 2019 г., Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server (полугодовой канал), Windows 10, Windows 8.1

В этом разделе описываются работоспособности и состояния работоспособности пулов носителей, виртуальных дисков (которые располагаются под тома в дисковых пространствах) и дисков в [Storage Spaces Direct](storage-spaces-direct-overview.md) и [дисковых](overview.md). Эти состояния может быть очень полезным при попытке устранения различных неполадок, например, почему не удается удалить виртуальный диск из-за конфигурации только для чтения. В нем также описывается, почему диска не может быть добавлено в пул (CannotPoolReason).

Дисковых пространств имеет три основных объектов - *физических дисков* (жесткие диски, SSDs, и т.д.), добавляемое к *пула носителей*, виртуализация хранилища, которые можно создавать *виртуальные диски* из свободного пространства в пуле, как показано ниже. Метаданные пула записывается для каждого диска в пуле. Тома создаются на основе виртуальных дисков и хранения файлов, но мы не будем говорить о томах здесь.

![Физические диски добавляются в пул носителей, а затем создать виртуальные диски из пространство пула](media/storage-spaces-states/storage-spaces-object-model.png)

В диспетчере серверов или с помощью PowerShell можно просмотреть работоспособности и состояния работоспособности. Вот пример различных (главным образом плохое) работоспособности и состояния работоспособности в кластере дисковыми пространствами, отсутствует большая часть его узлы кластера (щелкните правой кнопкой мыши заголовок столбца, чтобы добавить **оперативное состояние**). Это не удачного кластера.

![Отобразить результаты для двух отсутствующие узлы в кластере Storage Spaces Direct - много отсутствующих физических дисков и виртуальных дисков в неработоспособном состоянии диспетчера серверов](media/storage-spaces-states/unhealthy-disks-in-server-manager.png)

## <a name="storage-pool-states"></a>Состояния пула хранилища

Каждый пул носителей находится в состоянии работоспособности - **Исправен**, **предупреждение**, или **Неизвестный**/**Unhealthy**, а также как один или несколько следующие состояния работоспособности.

Чтобы узнать, какие состояния пула, используйте следующие команды PowerShell:

```PowerShell
Get-StoragePool -IsPrimordial $False | Select-Object HealthStatus, OperationalStatus, ReadOnlyReason
```

Ниже приведен пример выходных данных, отображающая пул хранения в неизвестное состояние с оперативное состояние только для чтения.

```
FriendlyName                OperationalStatus HealthStatus IsPrimordial IsReadOnly
------------                ----------------- ------------ ------------ ----------
S2D on StorageSpacesDirect1 Read-only         Unknown      False        True
```

В следующих разделах перечислены работоспособности и состояния работоспособности.

### <a name="pool-health-state-healthy"></a>Состояние работоспособности пула: Работоспособное

|Рабочее состояние    |Описание|
|---------            |---------  |
|ОК|Пул носителей — работоспособное.|

### <a name="pool-health-state-warning"></a>Состояние работоспособности пула: Предупреждение

Если пул носителей находится в **предупреждение** состояние работоспособности, это означает, что доступен пул, но один или несколько дисков не удалось, или отсутствуют. Таким образом пул носителей может снижена устойчивости.

|Рабочее состояние    |Описание|
|---------            |---------  |
|Ухудшенное состояние|Существуют неудачных или отсутствующих дисков в пуле носителей. Такая ситуация возникает только с дисками, размещение пула метаданных. <br><br>**Действие**: Проверьте состояние дисков и замените любой неисправных дисков, прежде чем существуют другие ошибки.|

### <a name="pool-health-state-unknown-or-unhealthy"></a>Состояние работоспособности пула: Неизвестный или неработоспособное

Если пул носителей находится в **Неизвестный** или **Unhealthy** состояние работоспособности, он означает, что пул носителей доступен только для чтения и нельзя изменять, пока возвращается пула **предупреждение**или **ОК** состояний работоспособности.

|Рабочее состояние    |Причина только для чтения |Описание|
|---------            |---------       |--------   |
|Только чтение|Неполные|Это может произойти, если пул носителей теряет его [кворума](understand-quorum.md), который означает, что большинство дисков в пуле произошел сбой или находятся в автономном режиме для какой-либо причине. При пулу теряет кворум, дисковые пространства автоматически устанавливает конфигурацию пула только для чтения до недостаточно дисков снова не станет доступной.<br><br>**Действие:** <br>1. Заново все отсутствующие диски, а если вы используете дисковые пространства, подключите все серверы в сети. <br>2. Задайте пул чтения и записи, открыв сеанс PowerShell с правами администратора и введите:<br><br> <code>Get-StoragePool <PoolName> -IsPrimordial $False \| Set-StoragePool -IsReadOnly $false</code>|
||Политика|Администратор присвоено только для чтения в пул носителей.<br><br>**Действие:** Чтобы задать кластерного пула носителей для чтения и записи доступа в диспетчере отказоустойчивости кластеров, перейдите на **пулы**, щелкните правой кнопкой мыши пул и выберите **перевести в оперативный режим**.<br><br>Для других серверов и компьютеров откройте сеанс PowerShell с правами администратора и введите:<br><br><code>Get-StoragePool <PoolName> \| Set-StoragePool -IsReadOnly $false</code><br><br> |
||Starting|Дисковые пространства запускается или Ожидание дисков должен быть подключен в пуле. Это должно быть это временное состояние. После полного запуска пула должен перейти в другой рабочее состояние.<br><br>**Действие:** Если пул, остается в состоянии *запуск* состоянии, убедитесь, что все диски в пуле подключены правильно.|

См. также [изменение пула носителей с конфигурацией только для чтения](https://social.technet.microsoft.com/wiki/contents/articles/14861.modifying-a-storage-pool-that-has-a-read-only-configuration.aspx).

## <a name="virtual-disk-states"></a>Состояния виртуального диска

В дисковых пространствах тома будут размещены на виртуальных дисков (дисковых), которые выделены из свободного пространства в пуле. Каждый виртуальный диск находится в состоянии работоспособности - **Исправен**, **предупреждение**, **Unhealthy**, или **Неизвестный** а также один или несколько следующие состояния работоспособности.

Чтобы узнать, находятся в состоянии виртуальных дисков, используйте следующие команды PowerShell:

```PowerShell
Get-VirtualDisk | Select-Object FriendlyName,HealthStatus, OperationalStatus, DetachedReason
```

Ниже приведен пример выходных данных, отсоединить виртуальный диск и снижение или неполные виртуального диска:

```
FriendlyName HealthStatus OperationalStatus      DetachedReason
------------ ------------ -----------------      --------------
Volume1      Unknown      Detached               By Policy
Volume2      Warning      {Degraded, Incomplete} None
```

В следующих разделах перечислены работоспособности и состояния работоспособности.

### <a name="virtual-disk-health-state-healthy"></a>Состояние работоспособности виртуального диска: Работоспособное

|Рабочее состояние    |Описание|
|---------            |---------          |
|ОК    |Виртуальный диск работоспособен.|
|Неоптимальный    |Данные не записываются равномерно на дисках. <br><br>**Действие**: Оптимизация использования дисков в пуле носителей, выполнив [Optimize-StoragePool](https://technet.microsoft.com/itpro/powershell/windows/storage/optimize-storagepool) командлета.|

### <a name="virtual-disk-health-state-warning"></a>Состояние работоспособности виртуального диска: Предупреждение

Если виртуальный диск находится в **предупреждение** состояние работоспособности, это означает, что один или несколько копий данных недоступны, но дисковых пространств можно считывать данные по крайней мере одну копию данных.

Если вы не используете дисковые пространства, затем восстановите виртуальный диск с помощью [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk) командлета.<br> Дисковые пространства автоматически запускает восстановление при необходимости после повторного подключения или замены диска.|

Если вы не используете дисковые пространства, затем восстановите виртуальный диск с помощью [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk) командлета. <br>Дисковые пространства автоматически запускает восстановление при необходимости после повторного подключения или замены диска.|

## <a name="drive-physical-disk-states"></a>Состояния диска (физический диск)

Данные о состоянии работоспособности диск может находиться в следующих разделах. Диски в пуле, представлены в PowerShell с правами *физического диска* объектов.

### <a name="drive-health-state-healthy"></a>Состояние работоспособности диска: Работоспособное

|Рабочее состояние    |Описание|
|---------            |---------          |
|ОК|Диск работоспособен.|
|В службе|Диск выполняет некоторые операции действий по внутреннему обслуживанию. После завершения действия, диск следует вернуться к *ОК* состояние работоспособности.|

### <a name="drive-health-state-warning"></a>Состояние работоспособности диска: Предупреждение

Диск в состояние предупреждения могут читать и записывать данные успешно, но возникла проблема.

|Рабочее состояние    |Описание|
|---------            |---------          |
|Потеря связи|Диск отсутствует. Если вы используете дисковые пространства, возможно, сервер не работает.<br><br>**Действие**: Если вы используете дисковые пространства, подключите все серверы обратно в оперативный режим. Если она не устранена, снова подключите диск, замените его или пытаемся получить подробные диагностические сведения о диске, выполнив действия, описанные в Устранение неполадок с помощью отчетов об ошибках Windows > [истекло время ожидания физического диска](../../failover-clustering/troubleshooting-using-wer-reports.md#physical-disk-timed-out).|
|Удаление из пула|Дисковые пространства — процессе извлечения диска из пула хранения. <br><br> Это временное состояние. По завершении удаления, если диск подключен к системе, диск переходит другой рабочее состояние (обычно "ОК") в исходном пуле.|
|Начальный режим обслуживания|Дисковые пространства — создает диск в режиме обслуживания после администратора перевести диск в режим обслуживания. Это временное состояние — диск должен быть вскоре в *в режиме обслуживания* состояния.|
|В режиме обслуживания|Администратор переводе диск в режим обслуживания, остановка работы операций чтения и записи с диска. Обычно это делается, перед обновлением встроенного по диска или при тестировании сбоев.<br><br>**Действие**: Чтобы воспользоваться диск из режима обслуживания, используйте [Disable StorageMaintenanceMode](https://technet.microsoft.com/itpro/powershell/windows/storage/disable-storagemaintenancemode) командлета.|
|Остановка режима обслуживания|Администратор выполненные диск из режима обслуживания, и дисковые пространства — процессе диск обратно в оперативный режим. Это временное состояние – диск вскоре в другое состояние — в идеале *Исправен*.|
|Прогнозируемый сбой|Дисковод сообщил, что оно находится рядом сбоя.<br><br>**Действие**: Замените диск.|
|Ошибка ввода-ВЫВОДА|Произошла временная ошибка, доступ к диску.<br><br>**Действие**: <br>1. Если диск не переход обратно в **ОК** state, попробуйте использовать [Reset-PhysicalDisk](https://docs.microsoft.com/powershell/module/storage/reset-physicaldisk) командлет, чтобы очистить диск. <br> 2. Используйте [Repair-VirtualDisk](https://docs.microsoft.com/powershell/module/storage/repair-virtualdisk) для восстановления устойчивости затронутых виртуальных дисков. <br>3. Если это будет повторяться, замените диск.|
|Временная ошибка|Произошла временная ошибка с диском. Обычно это означает, диск не ответил, но это может также означать, что дисковых пространств, раздел, защищенный ошибочно был удален с диска. <br><br>**Действие**: <br>1. Если диск не переход обратно в **ОК** state, попробуйте использовать [Reset-PhysicalDisk](https://docs.microsoft.com/powershell/module/storage/reset-physicaldisk) командлет, чтобы очистить диск. <br> 2. Используйте [Repair-VirtualDisk](https://docs.microsoft.com/powershell/module/storage/repair-virtualdisk) для восстановления устойчивости затронутых виртуальных дисков. <br>3. Если это будет повторяться, замените диск или пытаемся получить подробные диагностические сведения о диске, выполнив действия, описанные в Устранение неполадок с помощью отчетов об ошибках Windows > [не удалось подключить физический диск](../../failover-clustering/troubleshooting-using-wer-reports.md#physical-disk-failed-to-come-online).|
|Аномальные задержки|Диск работает медленно, согласно замерам службы работоспособности в Storage Spaces Direct.<br><br>**Действие**: Если это будет повторяться, замените диск, поэтому она не уменьшает производительность дисковых пространств в целом.

### <a name="drive-health-state-unhealthy"></a>Состояние работоспособности диска: Unhealthy

В настоящее время диска в неработоспособное состояние не может записываться или получить доступ к.

|Рабочее состояние    |Описание|
|---------            |---------          |
|Не может использоваться|Этот диск не может использоваться с дисковыми пространствами. Дополнительные сведения см. в разделе [дисковые требования к оборудованию](storage-spaces-direct-hardware-requirements.md); Если вы не используете дисковыми пространствами, см. в разделе [обзор дисковых пространств](https://technet.microsoft.com/library/hh831739(v=ws.11).aspx#Requirements).|
|Разбиение|Диск отделены от пула.<br><br>**Действие**: Сбросьте диск, удаление всех данных с диска и его добавления в пул как пустой диск. Чтобы сделать это, откройте сеанс PowerShell с правами администратора, запустите [Reset-PhysicalDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/reset-physicaldisk) командлета, а затем запустите [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk). <br><br>Чтобы получить подробные диагностические сведения о диске, выполните действия, описанные в Устранение неполадок с помощью отчетов об ошибках Windows > [не удалось подключить физический диск](../../failover-clustering/troubleshooting-using-wer-reports.md#physical-disk-failed-to-come-online).|
|Устаревшие метаданные|Дисковые пространства обнаружил старые метаданные на диске.<br><br>**Действие**: Это должно быть это временное состояние. Если диск не переход обратно в OK, вы можете запустить [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk) для запуска операции восстановления на затронутые виртуальные диски. Если это не решит проблему, можно сбросить на диск с [Reset-PhysicalDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/reset-physicaldisk) командлета, очистка всех данных с диска, а затем выполните [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk).|
|Нераспознанные метаданные|Дисковые пространства найти нераспознанные метаданные на диске, что обычно означает, что диск имеет метаданных из другого пула на нем.<br><br>**Действие**: Очистка диска и добавить его в текущий пул, сброс диска. Чтобы сбросить на диск, откройте сеанс PowerShell с правами администратора, запустите [Reset-PhysicalDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/reset-physicaldisk) командлета, а затем запустите [Repair-VirtualDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/repair-virtualdisk).|
|Неисправный носитель|Диск не удалось и не будет пригоден по дисковых пространств.<br><br>**Действие**: Замените диск. <br><br>Чтобы получить подробные диагностические сведения о диске, выполните действия, описанные в Устранение неполадок с помощью отчетов об ошибках Windows > [не удалось подключить физический диск](../../failover-clustering/troubleshooting-using-wer-reports.md#physical-disk-failed-to-come-online).|
|Сбой оборудования устройства|Произошел сбой оборудования на этом диске. <br><br>**Действие**: Замените диск.|
|Обновление встроенного ПО|Windows обновление встроенного по на диске. Это временное состояние, обычно исчезают не более минуты, и во время которого времени других дисков в пуле обработки всех операций чтения и записи. Дополнительные сведения см. в разделе [обновления встроенного по дисков](../update-firmware.md).|
|Starting|Диск подготавливается для операции. Это должно быть это временное состояние - после завершения диск должен перейти в другой рабочее состояние.|

## <a name="reasons-a-drive-cant-be-pooled"></a>Причин, по которым диск не может быть помещен в пул

Только некоторые диски не готовы в пул носителей. Можно выяснить, почему устройство не подходят для пула, просмотрев `CannotPoolReason` свойство физического диска. Ниже приведен пример сценария PowerShell для отображения свойства CannotPoolReason:

```PowerShell
Get-PhysicalDisk | Format-Table FriendlyName,MediaType,Size,CanPool,CannotPoolReason
```

Ниже приведен пример вывода:

```
FriendlyName          MediaType          Size CanPool CannotPoolReason
------------          ---------          ---- ------- ----------------
ATA MZ7LM120HCFD00D3  SSD        120034123776   False Insufficient Capacity
Msft Virtual Disk     SSD         10737418240    True
Generic Physical Disk SSD        119990648832   False In a Pool
```

В следующей таблице приведены немного более подробно на каждой из причин.

|Причина|Описание|
|---|---|
|В пуле|Диск уже принадлежит к пулу носителей. <br><br>Диски могут принадлежать только один пул носителей за раз. Чтобы использовать этот диск в другом пуле носителей, сначала удалите диск из существующего пула, который сообщает дисковых пространств для перемещения данных на диске на других дисках в пуле. Или сбросить диск в том случае, если диск был отключен из пула без уведомления дисковых пространств. <br><br>Чтобы безопасно удалить диск из пула носителей, используйте [Remove-PhysicalDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/remove-physicaldisk), или перейдите в диспетчер серверов > **файловых служб и служб хранилища** > **пулы носителей**, > **Физических дисков**, щелкните правой кнопкой мыши диск, а затем выберите **удалить диск**.<br><br>Чтобы сбросить на диск, используйте [Reset-PhysicalDisk](https://technet.microsoft.com/itpro/powershell/windows/storage/reset-physicaldisk).|
|Неработоспособны|Диск не находится в работоспособном состоянии и может потребоваться заменить.|
|Съемный носитель|Диск классифицируется как съемный диск. <br><br>Дисковые пространства не поддерживает мультимедиа, которые распознаются в Windows, являющихся съемными, таких как диски Blu-Ray. Несмотря на то, что многие фиксированной диски находятся на съемных слотов, вообще говоря, мультимедиа, которые *классифицировать* с Windows, являющихся съемными не подходит для использования с дисковыми пространствами.|
|Используется кластер|Диск в данный момент используется отказоустойчивый кластер.|
|Вне сети|Диск находится в автономном режиме. <br><br>Чтобы объединить все автономные диски через Интернет и set для чтения и записи, откройте сеанс PowerShell с правами администратора и использовать следующие сценарии:<br><br><code>Get-Disk \| Where-Object -Property OperationalStatus -EQ "Offline" \| Set-Disk -IsOffline $false</code><br><br><code>Get-Disk \| Where-Object -Property IsReadOnly -EQ $true \| Set-Disk -IsReadOnly $false</code>|
|Недостаточно свободных ресурсов|Обычно это происходит, когда имеются разделы, занимающих свободного места на диске. <br><br>**Действие**: Удалите все тома на диске, удаление всех данных на диске. Один из способов сделать это является использование [Clear-Disk](https://docs.microsoft.com/powershell/module/storage/clear-disk?view=win10-ps) командлета PowerShell.|
|Идет проверка|[Служба работоспособности](../../failover-clustering/health-service-overview.md#supported-components-document) проверка, если путь к диску или встроенного по на диске утверждения для использования администратором сервера.|
|Не удалось выполнить проверку|[Служба работоспособности](../../failover-clustering/health-service-overview.md#supported-components-document) не удалось проверить, если путь к диску или встроенного по на диске утверждения для использования администратором сервера.|
|Встроенное по не соответствующих политике|Встроенное по на физическом диске, не находится в список утвержденных микропрограммы, указанного администратором сервера с помощью [служба работоспособности](../../failover-clustering/health-service-overview.md#supported-components-document). |
|Не соответствует оборудованию|Диск отсутствует в списке утвержденных хранилище моделей, указанного администратором сервера с помощью [служба работоспособности](../../failover-clustering/health-service-overview.md#supported-components-document).|
