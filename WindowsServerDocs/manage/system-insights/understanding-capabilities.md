# <a name="understanding-capabilities"></a>Общие сведения о возможностях

>Область применения. Windows Server 2019

В этом разделе определяет понятие возможности в Insights системы и предоставляет возможности по умолчанию, доступные в Windows Server 2019. 

В этом разделе также описываются источники данных, прогнозирования временных шкал и прогнозирования состояний, используемый для возможности по умолчанию. 

## <a name="capability-overview"></a>Общие сведения о возможностях
Возможность Insights системы — машинного обучения или статистики модель, которая анализирует системные данные, чтобы дать возможность увеличить понимания работы развертывания. System Insights вводит начальный набор возможностей по умолчанию, а также вы можете добавить новые возможности динамически, без необходимости обновления операционной системы. 

>[!NOTE]
>Подробную документацию о том, как создать, добавить, и возможностей обновления доступен [здесь](adding-and-developing-capabilities.md), и [управление документа возможности](managing-capabilities.md) предоставляет более высокоуровневые сведения об этом функциональные возможности.

Кроме того каждая возможность выполняется локально в экземпляре Windows Server, и каждая возможность может управляться как индивидуально.

### <a name="capability-outputs"></a>Выходные данные функции
Если функция вызывается, он предоставляет выход объясняющее результат его анализа или прогнозирования. Каждый выход должен содержать **состояние** и **описание состояния** для описания прогноза и каждый результат может дополнительно содержать конкретных возможностей данные, связанные с прогнозом. **Описание состояния** помогает обеспечить контекстную объяснение для **состояние**, и сообщает о возможности либо **ОК**, **предупреждение**, или **критическое** состояния. Кроме того, можно использовать функцию **ошибка** или **None** состояние, если прогноз не была выполнена. Вместе Вот состояния возможностей и их основные значения. 

- **ОК** — все в порядке.
- **Предупреждение** — необходимые без немедленного вмешательства, но вы должны взглянуть. 
- **Критические** -скоро вы Обратите внимание. 
- **Ошибка** — Неизвестная ошибка вызвала функцию переход на другой. 
- **Нет** -прогноз не была сделана. Это может быть из-за отсутствия данных или любой другой причине конкретных возможностей для не формирования прогноза. 

Кроме того, все возможности конкретных данные, содержащиеся в результате будут помещены в файл JSON недоступны для пользователей и путь к файлу [можно найти с помощью PowerShell](https://docs.microsoft.com/windows-server/manage/system-insights/managing-capabilities#retrieving-capability-results). 

## <a name="default-capabilities"></a>Возможности по умолчанию
В Windows Server 2019 системы Insights представляет четыре возможности по умолчанию, уделено прогнозирования емкости:

- **Прогнозирование емкость ЦП** -прогнозов ЦП. 
- **Сетевые подключения прогнозирования емкость** -использование для каждого сетевого адаптера сети прогнозов. 
- **Прогнозирование потребления общий объем хранилища** -прогнозов общий объем занятого объема хранилища на всех локальных дисках. 
- **Прогнозирование потребления тома** -прогнозирует потребление хранилища для каждого тома.

Каждая возможность анализирует исторических данных для прогнозирования будущих нужд в прошлом и **все возможности прогнозирования предназначены для прогнозирования долгосрочных тенденций, а не краткосрочной поведение**, помочь администраторам правильно Подготовка оборудования и настраивать их рабочие нагрузки, чтобы избежать конфликта ресурсов в будущем. Так как эти возможности сосредоточиться на долгосрочного использования, эти возможности анализировать ежедневной данные. 

### <a name="forecasting-model"></a>Модели прогнозирования
Возможности по умолчанию использовать модели прогнозирования для прогнозирования будущих нужд, а для каждого прогноза, модель обучается локально на вашем компьютере данных. Эта модель предназначена для определения тенденций больше термин и повторного обучения на каждом экземпляре Windows Server у вас есть возможность адаптироваться к особое поведение и особенности использования каждого компьютера.

>[!NOTE]
>Определяет, какой тип модели для использования требуется тестирование множество моделей, используя набор данных, содержащий десятки тысяч компьютеров. После анализа и настройки этих моделей, мы решили использовать авторегрессией модели прогнозирования, как он выдает прогнозы относительно попадания высокоточные и улучшения визуального при не требует слишком много времени для обучения. Этой модели, однако, требуется три недели обучающих данных, поэтому каждая возможность использует основные линейным трендом, пока не доступны три недели данных.

### <a name="forecasting-timelines"></a>Прогнозирование временных шкал
Возможности по умолчанию прогноз определенного числа дней в будущем, в зависимости от количества дней, для которых были собраны данные. В следующей таблице показаны прогнозирования временных шкал этих возможностей:

| Размер входных данных | Длина прогноза |
| --------------- | --------------- |
| 0 – 5 дней | Прогноз не выполняется. |
| 6 – 180 дней | 1/3 * размер входных данных |
| 180-365 дней | через 60 дней | 

### <a name="forecasting-data"></a>Прогнозирование данных
Каждая возможность анализирует ежедневные данные для прогнозирования будущих нужд. Использование хранилища по ЦП, сети и даже, тем не менее, можно часто меняться в течение суток, динамически приспособиться к рабочим нагрузкам, на компьютере. Так как использование не является неизменным на протяжении всего дня, очень важно правильно представляют данные о ежедневном использовании в одну точку данных. В таблице ниже подробно определенных точках данных и как обрабатываются данные:


| Имя возможности | Источники данных | Логику фильтрации |
| --------------- | -------------- | ---------------- |
 Прогнозирование потребления тома          | Размер тома                    | Максимальное ежедневное использование              
 Прогнозирование потребления общий объем хранилища   | Суммарный размер тома, суммарный размер диска              | Максимальное ежедневное использование             
 Прогнозирование емкость ЦП                | % загруженности процессора  | Максимальное среднее значение 2 часа в день   
 Сетевые подключения прогнозирования емкости         | Всего байт/с         | Максимальное среднее значение 2 часа в день  

При оценке логику фильтрации выше, его следует отметить, что каждая возможность стремится информируют администраторов, когда дальнейшее использование значимо превышает доступной емкости — несмотря на то, что ЦП моментально попаданий 100% использованию, загрузка ЦП может не иметь Причиной конкуренции значимые производительности снижение или ресурсов. За ресурсы ЦП и сети затем следует устойчивый интенсивное использование, а не небольшой пики. Вычисление среднего значения ЦП и сети использования на протяжении всего дня, тем не менее, будет утеряна важные сведения по использованию, как несколько часов высокую ЦП или использования сети значимо может повлиять на производительность важных рабочих нагрузок. Максимальное среднее значение 2 часа за каждый день и позволяет избежать этими экстремальными случаями и по-прежнему создает значимых данных для каждой возможности для анализа.

Для тома и использования общего хранилища, тем не менее, использование хранилища не может превышать доступной емкости, даже кратковременно, поэтому для этих возможностей используется максимальное ежедневное использование. 

### <a name="forecasting-statuses"></a>Прогнозирование состояния
Все возможности системы аналитики должны выходные состояние, связанное с каждого прогноза. Каждая возможность по умолчанию используется следующая логика для определения состояния каждого прогноза:
- **ОК**: Прогноз не превышают доступную емкость.
- **Предупреждение**: Прогноз превышает доступный объем в течение 30 дней. 
- **Ошибка**. Прогноз превышает доступный объем в течение 7 дней. 
- **Ошибка**. Возможность запуска произошла непредвиденная ошибка. 
- **Нет**. Не достаточно данных для создания прогноза. Это может быть из-за отсутствия данных или из-за недавно обнаруженную нет данных.

>[!NOTE]
>Если возможность прогнозы на нескольких экземплярах - например несколько томов или сетевых адаптеров — состояние отражает наиболее серьезных состояние для всех экземпляров. Отдельные состояния для каждого тома или сетевым адаптером видимы, в Windows Admin Center или в данных, содержащихся в выходных данных каждой возможности. Инструкции о том, как проанализировать возможности по умолчанию выходные данные JSON можно найти [этот блог](https://aka.ms/systeminsights-mitigationscripts).
