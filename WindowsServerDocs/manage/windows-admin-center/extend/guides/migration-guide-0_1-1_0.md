# Перенос из пакета SDK Windows Admin Center 0,1 до 1,0

>Область применения: Ознакомительная версия Windows Admin Center

Это руководство поможет вам переход с Windows Admin Center SDK версии 0,1 до 1,0.  

## 1. Узнайте о новых элементов управления с расширением руководство по разработки

Windows Admin Center версии 1902 и более поздних версий содержит расширений **Разработки руководство** , которое можно использовать для поиска примеров элементов управления (включая новые элементы управления) и сценарии, которые помогут вам создавать собственные расширения.  Руководство по разработки заменяет расширение **Средства разработчика** с предыдущих версий пакета SDK.

### Следуйте руководству разработки в Windows Admin Center

Руководство по разработки доступны в виде решения для Windows Admin Center версии 1902 и более поздних версиях.  Руководство по разработки предварительно установлены, но он должен быть включен в настройках.

**Включите руководство разработки в Windows Admin Center:**

* Откройте Windows Admin Center (версия 1902 и более поздние версии)
* Щелкните значок « **Параметры** » в правом верхнем углу окна
* Выберите вкладку " **Дополнительно** "
* В разделе *Ключи эксперимента*нажмите кнопку **Добавить**
* Введите новое значение ```msft.sme.shell.devguide``` в поле пустым, созданной на предыдущем этапе
* Нажмите кнопку **Сохранить и перезагрузить**

**Руководство по Open разработки в Windows Admin Center:**

* Откройте Windows Admin Center (версия 1902 и более поздние версии)
* Щелкните раскрывающийся список в верхнем левом углу для отображения всех типов решение
* Выберите решение **Руководство по разработки** 
    * Если вы не видите решения в списке, убедитесь, что вы включили в руководстве по разработки (см. выше) и перезагружен Windows Admin Center.
* Просматривать содержимое руководства разработки, выбрав один из вкладок
    * **Целевые:** Содержит примеры кода для сценариев *Manage As* и *уведомления*
    * **Элементов управления:** Содержит примеры каждого элемента управления, доступные в SDK
    * **Каналы:** Содержит примеры доступные функции преобразования и форматирования
    * **Стили:** Содержит примеры стилей CSS, доступные в SDK
    * **MsftSme:** Содержит примеры и руководства для более сложных сценариев 

### Обзор исходный код руководства разработки на GitHub

Можно просматривать [исходный код](https://github.com/Microsoft/windows-admin-center-sdk/) руководства разработки на GitHub, чтобы найти пример примеры кода HTML, CSS и TypeScript.

## 2. Подготовка среды разработки для использования последней версии SDK

Установите или обновите версии node.js [10.15.1 Результатов или более поздней версии](https://nodejs.org/en/).

Обновление до последней версии Windows Admin Center CLI:

[//]: # "npm удаление windows-admin-center-cli@next -g"

``` cmd
npm uninstall -g windows-admin-center-cli
npm install -g windows-admin-center-cli
```

Обновление глобальный зависимости для этих версий:

``` cmd
npm install npm@6.4.1 -g
npm install @angular/cli@7.1.2 -g
npm install gulp -g
npm install typescript@3.1.6 -g
npm install tslint@5.11.0 -g
```

## 3. Создание нового проекта с помощью последнюю версию SDK

Использовать интерфейс командной строки Windows Admin Center для создания нового проекта предназначенных для ```next``` версии (SDK 1.0):

[//]: # "Создание WAC--компании «Contoso Inc»--средство «Управление Foo работает»--экспериментальные версии"

``` cmd
wac create --company "Contoso Inc" --tool "Manage Foo Works" --version next
```

Затем измените каталог в папке только что создали, а затем установите необходимые локальные зависимости, выполнив ```npm install ```.

## 4. Изменение существующего проекта для использования последней версии SDK

ВНИМАНИЕ: Создайте резервную копию проекта перед продолжением.

Измените следующую строку в ```package.json``` целевой объект ```next``` версии (SDK 1.0):

[//]: # "«@microsoft/windows-admin-center-sdk»: «экспериментальных»"

``` json
"@microsoft/windows-admin-center-sdk": "next",
```

Затем запустите ```npm install``` обновить ссылки во всем проекте.

## 5. Используйте SDK CLI для устранения распространенных проблем миграции

ВНИМАНИЕ: Создайте резервную копию проекта перед продолжением.

В корневой папке проекта выполните следующую команду командной строки на проект, чтобы устранить распространенные проблемы миграции автоматически:

``` cmd
wac updateSeven --update
```

Эта команда CLI автоматически решает следующие проблемы.

* Повторное создание ```package-lock.json```
* Файлы обновления в среде angular компиляции:
    - ```.gitignore```
    - ```tslint.json```
    - ```tsconfig.json```
    - ```tsconfig.inline.json```
    - ```ng-package.json```
    - ```angular.json```
    - ```src\tsconfig.app.json```
    - ```src\tsconfig.lib.json```
    - ```src\tsconfig.spec.json```

## 7. понимания и устранения проблем с наблюдаемые объекты

### Обновление ```rxjs``` функция используется для отслеживаемую объектов

Ниже перечислены некоторые распространенные имена функций, которые были изменены, могут существовать другие в свой проект.

* Обновление ```Observable.empty()``` для ```empty()```
* Обновление ```Observable.of()``` для ```of()```
* Обновление ```.switchMap()``` для ```.pipe(switchMap())```
* Обновление ```.map()``` для ```.pipe(map())```
* Обновление ```flatMap()``` для ```mergeMap()```


### Устранить неполадки, среда выполнения с ```.map()``` и ```.filter()``` функции на наблюдаемые объекты

Если компилятор не может определить надлежащим образом ```observable``` тип объекта, ```.map()``` и ```.filter()``` функции из ```array``` объект может быть сопоставлен вместо вашего объекта, благодаря чему ошибки во время выполнения.  Убедитесь, что ваш функции возвращают ```observable``` указывающий явный тип данных для устранения данной проблемы.

```any``` и не возвращаемый тип можно вызвать эту проблему, найдите код с помощью этих шаблонов:

``` ts
public getMyObservable(): any { //any return type can cause issues
 ...
}

public getMyObservable() { //no return type can cause issues
...
}
```

## 10. Включение Темная тема в Windows Admin Center

Включение Темная тема в Windows Admin Center версии 1902 и более поздних версиях, выполните следующие действия.

* Откройте Windows Admin Center (версия 1902 и более поздние версии)
* Щелкните значок « **Параметры** » в правом верхнем углу окна
* Выберите вкладку " **Дополнительно** "
* В разделе *Ключи эксперимента*нажмите кнопку **Добавить**
* Введите новое значение ```msft.sme.shell.personalization``` в поле пустым, созданной на предыдущем этапе
* Нажмите кнопку **Сохранить и перезагрузить**
* Параметры должны новой вкладке **персонализации**.  Выберите на этой вкладке
* Изменить **цвета** в **темном режиме (ознакомительная версия)**
