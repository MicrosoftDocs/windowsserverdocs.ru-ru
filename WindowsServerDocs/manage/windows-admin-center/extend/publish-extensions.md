# Публикация расширений

>Относится к: Windows Admin Center, ознакомительная версия Windows Admin Center

Как только вы разрабатывали расширения, необходимо опубликовать его и сделать его доступным для других пользователей для проверки или использовать. В зависимости от вашей аудитории и цель публикации существует несколько вариантов, которые мы будем Представляем ниже, а также действия и требования к публикации.

## Параметры публикации

Существует три основных варианта источников настраиваемых пакетов, которые поддерживает Windows Admin Center.
* Корпорации Майкрософт открытых Windows Admin Center NuGet веб-каналов
* Собственный закрытый канал NuGet
* Локальный или общей сетевой папке

### Публикация веб-каналов расширения Windows Admin Center

По умолчанию Windows Admin Center подключен к NuGet канала независимыми группой продукта Windows Admin Center в корпорации Майкрософт. Ранний предварительных версий библиотек новые расширения, разработанные корпорацией Майкрософт может быть опубликованы на веб-канал и доступны для пользователей Windows Admin Center. Планирование для построения и освободить расширения открыто сторонним разработчикам может также [Отправить запрос](#publishing-your-extension-to-the-windows-admin-center-feed) к публикации веб-канал.

### Канал публикации для разных NuGet

Вы также можете создавать собственные NuGet канал для публикации расширений с помощью одного из многих [разные параметры для настройки источника частной или с помощью NuGet размещения службы](https://docs.microsoft.com/nuget/hosting-packages/overview). Канал NuGet должен поддерживать NuGet v2 API. Так как Windows Admin Center в настоящее время не поддерживает проверку подлинности веб-каналов, веб-канала необходимо настроить, чтобы разрешить доступ на чтение к любым пользователем.

### Публикация в общую папку

Чтобы ограничить доступ расширения для вашей организации или ограниченной группой пользователей, можно использовать общий файловый ресурс SMB как расширение веб-каналов. В этом случае файл права доступа к общей папке будут применяться для предоставления доступа к веб-канала.

## Создание пакета расширения

Windows Admin Center использует современные веб-каналы для распространения и загрузки расширений и пакетов NuGet.  В порядке для пакета поставляться необходимо создать пакет NuGet, содержащий подключаемые модули и расширения.  Один пакет может содержать как расширение пользовательского интерфейса так и подключаемый модуль шлюза и со следующим разделом пошаговый процесс.

### 1. Создание расширения

Как только вы будете готовы начнет упаковки расширения, создайте новый каталог в файловой системе, откройте консоль и компакт-диска в него.  Это будет корневой каталог, который мы будем использовать для хранения всех nuspec и содержимое каталоги, которые составляют нашего пакета.  Мы будет ссылаться на эту папку как «Пакет NuGet» в течение срока действия данного документа.

#### Расширения пользовательского интерфейса

Чтобы начать процедуру на сбор все содержимое, необходимое для расширения пользовательского интерфейса, запустите «gulp сборки» на ваше средство и убедитесь, что построение выполнено успешно.  Этот процесс пакеты, все компоненты друг с другом в папку с именем «bundle», расположенный в корневом каталоге расширения (на том же уровне каталога src).  Скопируйте этот каталог и все приложения содержимое в папку «Пакет NuGet».

#### Подключаемые модули шлюза

Использование инфраструктуры сборки (это может быть так же просто, как открыть Visual Studio и нажать кнопку сборка), скомпилировать и создать ваш подключаемый модуль.  Откройте каталог выходных данных сборки и скопируйте Dll(s), представляющие ваш подключаемый модуль и поместить их в новую папку в каталоге «Пакет NuGet» под названием» пакет «.  Скопируйте dll FeatureInterface, просто Dll(s), представляющие ваш код не нужно.

### 2. Создайте файл .nuspec

Чтобы создать пакет NuGet, необходимо сначала создать файл .nuspec. Файл .nuspec является манифест XML, который содержит метаданные пакета NuGet. Этот манифест используется для создания пакета и содержат сведения для потребителей.  Поместите этот файл в корневой папке «Пакет NuGet».

Ниже приведен пример файла .nuspec и список требуемые или рекомендуемые свойства. Полная схема см. в разделе [.nuspec ссылку](https://docs.microsoft.com/nuget/reference/nuspec). Сохраните файл .nuspec корневой папки вашего проекта с именем файла по вашему выбору.

> [!IMPORTANT]
> ```<id>``` Должен соответствовать значению в файле .nuspec ```"name"``` значение в своем проекте ```manifest.json``` файл, в противном случае опубликованного расширения не загружается успешно в Windows Admin Center.

```xml
<?xml version="1.0" encoding="utf-8"?>
<package xmlns="http://schemas.microsoft.com/packaging/2011/08/nuspec.xsd">
  <metadata>
    <packageTypes>
      <packageType name="WindowsAdminCenterExtension" />
    </packageTypes>  
    <id>contoso.project.extension</id>
    <version>1.0.0</version>
    <title>Contoso Hello Extension</title>
    <authors>Contoso</authors>
    <owners>Contoso</owners>
    <requireLicenseAcceptance>false</requireLicenseAcceptance>
    <projectUrl>https://msft-sme.myget.org/feed/windows-admin-center-feed/package/nuget/contoso.sme.hello-extension</projectUrl>
    <licenseUrl>http://YourLicenseLink</licenseUrl>
    <iconUrl>http://YourLogoLink</iconUrl>
    <description>Hello World extension by Contoso</description>
    <copyright>(c) Contoso. All rights reserved.</copyright> 
    <tags></tags>
  </metadata>
  <files>
    <file src="bundle\**\*.*" target="ux" />
    <file src="package\**\*.*" target="gateway" />
  </files>
</package>
```

| ---- | ---- | ---- |

| id | Обязательный | Уникальный идентификатор пакета в потоке данных. Это значение должно совпадать со значением «имя» в файл manifest.json вашего проекта.  Инструкции см. [Выбор пакета уникальный идентификатор](https://docs.microsoft.com/nuget/create-packages/creating-a-package#choosing-a-unique-package-identifier-and-setting-the-version-number) . |

### 3. Выполните сборку пакета NuGet расширения

С помощью файла .nuspec, созданный выше, теперь создадим файл .nupkg пакета NuGet, который можно передать и опубликовать NuGet веб-каналов.

1. Загрузите средство командной строки nuget.exe с [веб-сайт средства клиента NuGet](https://docs.microsoft.com/nuget/install-nuget-client-tools).
2. Запустите «пакет nuget.exe [имя файла .nuspec]», чтобы создать файл .nupkg.

### 4. подписывается пакет NuGet расширения

Все DLL-файлы, включенные в расширении обязательно должны быть подписаны с помощью сертификата из доверенного сертификации (ЦС). По умолчанию без знака DLL-файлы будут блокироваться для предотвращения выполнения при запуске Windows Admin Center в рабочей среде режиме.

Настоятельно рекомендуется подписать пакет NuGet расширения для обеспечения целостности пакета, что это не обязательный этап.

### 5. Протестируйте пакет NuGet расширения

Пакет расширения готов для тестирования! Загрузите файл .nupkg канал NuGet или скопировать его в общую папку. Просмотреть и загрузить пакеты на файловом ресурсе или другой веб-канала, будет необходимо [изменить конфигурацию веб-каналов](../configure/using-extensions.md#installing-extensions-from-a-different-feed) на укажите файл или веб-каналов NuGet для совместного использования. При тестировании, убедитесь, что свойства отображаются правильно в "диспетчере расширений" и может успешной установки и удаления расширения.

## Публикация расширения Windows Admin Center веб-каналов

Публикации веб-каналов Windows Admin Center, вы можете сделать расширение доступных любому пользователю Windows Admin Center. Так как пакет SDK Windows Admin Center по-прежнему в режиме предварительного просмотра, мы хотели тесно сотрудничать с вами, чтобы помочь устранить неполадки разработки, убедитесь, что можно доставлять качество продукта и опыта пользователей.

Перед выпуском начальной версии расширения, рекомендуется отправить запрос отзыва расширения в корпорацию Майкрософт по крайней мере 2 – 3 недель до выпуска, чтобы убедиться, что у нас есть достаточно времени для просмотра и позволяет вносить изменения в расширении при необходимости. После расширения Готово к публикации, вам потребуется отправить ее нам для просмотра и если утвержден, мы опубликуем ее на канал для вас.

Затем Если вы хотите предложить обновление расширения, необходимо будет отправить другой запрос на проверку. Хотя в зависимости от объема изменений, время обработки обновлений отзывы о обычно должны содержать гораздо.

### Отправка запроса расширения отзыв в корпорацию Майкрософт

Чтобы отправить запрос отзыва расширения, введите следующие сведения и отправьте электронное письмо для [wacextensionrequest@microsoft.com](mailto:wacextensionrequest@microsoft.com?subject=Windows%20Admin%20Center%20Extension%20Review%20Request). Мы ответит к своей электронной почте в течение недели.

```
Windows Admin Center Extension Review Request
1. Name and email address of extension owner/developer (up to 3 users). If you will be releasing an extension on behalf of your company, provide your company email address.
2. Company name (Only required if you are releasing an extension on behalf of your company):
3. Extension name:
4. Release target date (estimate):
5. For new extension submission - Extension description (early design wire frames, screen mockups or product screenshots are highly recommended):
6. For extension update review – Description of changes (include product screenshots if UI has been significantly changed):
```

### Отправьте пакет расширения для проверки и публикации

Убедитесь, что, следуйте инструкциям выше для [создания пакета расширения](#creating-an-extension-package) и файл .nuspec определяется надлежащим образом и файлы подписаны. Мы также рекомендуем, что у вас есть веб-сайт проекта, включая следующие:

- Подробное описание вашего расширения, включая снимки экрана или видео
- Функция адрес или веб-сайт по электронной почте, для получения отзывов или вопросы

Когда вы будете готовы публикация расширения, отправить сообщение электронной почты [wacextensionrequest@microsoft.com](mailto:wacextensionrequest@microsoft.com?subject=Windows%20Admin%20Center%20Extension%20Package%20Review) и мы предоставят инструкции о том, как отправить нам пакета расширения. Как только мы получаем пакет, мы просмотрите и если утвержден, опубликовать веб-каналов Windows Admin Center.
