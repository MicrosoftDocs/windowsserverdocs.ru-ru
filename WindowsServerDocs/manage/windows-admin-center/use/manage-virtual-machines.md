# Управление виртуальными машинами с помощью Windows Admin Center

>Относится к: Windows Admin Center, ознакомительная версия Windows Admin Center

Средство виртуальных машин доступна в [сервера](manage-servers.md), [Отказоустойчивого кластера](manage-failover-clusters.md) или [Гиперконвергированном кластере](manage-hyper-converged.md) подключений включена роль Hyper-V на сервере или кластере. Можно использовать средство виртуальных машин для управления узлы Hyper-V под управлением Windows Server 2012 или более поздней версии, либо установить с возможностями рабочего стола или в виде Server Core. Hyper-V Server 2012, также поддерживаются 2016 и 2019 г.

## Ключевые функции

Особенности средства виртуальных машин в Windows Admin Center:

- **Высокоуровневый Hyper-V узла отслеживания ресурсов.** Просмотрите общее ЦП и памяти, метрик производительности операций ввода-ВЫВОДА, виртуальная машина работоспособности оповещений и событий для сервера узла Hyper-V или всего кластера в одной панели мониторинга.
- **Единый интерфейс начались возможности диспетчера Hyper-V и диспетчера отказоустойчивости кластеров.** Просмотр всех виртуальных машин всему кластеру и переходить к одной виртуальной машины для расширенного управления и устранения неполадок.
- **Упрощенный и мощное рабочие процессы для управления виртуальной машины.** Новые возможности пользовательского интерфейса позволяет IT-сценарии администрирования для создания, управления и репликацию виртуальных машин.

## Монитор ресурсов узла Hyper-V и производительности

![Сводка виртуальных машин](../media/manage-virtual-machines/virtual-machines-summary.png)

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. Существует две вкладки в верхней части средства **виртуальных машин** , вкладку " **Сводка** " и вкладку " **Инвентаризация** ". Вкладку " **Сводка** " позволяет централизованно контролировать ресурсы узла Hyper-V и производительности для текущего сервера или всего кластера, включая следующие:
    - Число виртуальных машин, сгруппированных по состоянию - запущена, отключено, приостановки и сохранения
    - Недавние предупреждения работоспособности или Hyper-V журналов событий (оповещения — только для гиперконвергентными кластерами под управлением Windows Server 2016 или более поздней версии)
    - Использования ЦП и памяти с помощью узла и гостевой детализация
    - Основные виртуальные машины, используются наиболее ресурсов ЦП и памяти
    - Live данные и данные журнала графики для пропускная способность операций ввода-ВЫВОДА и операций ввода-ВЫВОДА (графиков производительности хранилища подходят только для гиперконвергентными кластерами под управлением Windows Server 2016 или более поздней версии. Данные журнала доступна только для гиперконвергентными кластерами под управлением Windows Server 2019)

## Просмотр запасов виртуальной машины

![Экран инвентаризации виртуальных машин](../media/manage-virtual-machines/virtual-machines-inventory.png)

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. Существует две вкладки в верхней части средства **виртуальных машин** , вкладку " **Сводка** " и вкладку " **Инвентаризация** ". Вкладке **инвентаризации** перечислены виртуальные машины, доступных для текущего сервера или всего кластера и предоставляет команды для управления отдельных виртуальных машин. Можно сделать следующее.
    - Просмотрите список виртуальных машин, работающих на текущем сервере или кластере.
    - Просмотр состояния и узел сервера виртуальной машины при просмотре виртуальных машин для кластера. Также просмотрите использование ЦП и памяти с точки зрения узла, в том числе нехватка памяти, требования к памяти и назначенного объема памяти и виртуальной машины время бесперебойной работы, состояние пульса и состояние защиты с помощью Azure Site Recovery.
    - [Создание новой виртуальной машины](#create-a-new-virtual-machine).
    - Удалить, начать, отключить, завершение работы, приостановить, возобновить, сброс или переименовать виртуальной машины. Также сохранить виртуальную машину, удалить сохраненные состояния или создайте контрольную точку.
    - [Изменение параметров для виртуальной машины](#change-virtual-machine-settings).
    - Подключение к консоли виртуальной машины с помощью VMConnect через узел Hyper-V.
    - [Репликация на виртуальной машине с помощью Azure Site Recovery](#protect-virtual-machines-with-azure-site-recovery).
    - Для операций, которые можно выполнять на несколько виртуальных машин, например в меню "Пуск", завершение работы, сохранить, приостановка удалить, сброс, можно выбрать несколько виртуальных машин и выполнить операцию за один раз.

Примечание: При подключении к кластеру, средство виртуальной машины будет отображаться только виртуальных машин. Мы планируем также Показывать некластеризованные виртуальных машин.

## Создание новой виртуальной машины

![Создание новой виртуальной машины экрана](../media/manage-virtual-machines/new-vm.png)

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин выберите вкладку " **Инвентаризация** ", а затем щелкните **Создать** , чтобы создать новую виртуальную машину.
3. Введите имя виртуальной машины и выбрать один из виртуальных машин поколения 1 и 2.
4. При создании виртуальной машины в кластере, вы можете изначально создать виртуальную машину в узел. Если вы используете Windows Server 2016 или более поздней версии, средство предоставит рекомендации узла для вас.
5. Выберите путь для файлов виртуальной машины. В раскрывающемся списке выберите том или нажмите кнопку **Обзор** , чтобы выбрать папку с помощью средства выбора папок. Файлы конфигурации виртуальной машины и файл виртуального жесткого диска будут сохранены в одной папке `\Hyper-V\\[virtual machine name]` путь к выбранный том или путь.

>[!Tip]
> В средстве выбора папки, можно просмотреть все доступные общей папки SMB в сети, указав путь в поле **имени папки** в виде ```\\server\share```. С помощью общего сетевого ресурса для хранилища виртуальных Машин потребуется [CredSSP](../understand/faq.md#does-windows-admin-center-use-credssp).

6. Укажите, сколько виртуальных процессоров ли вы хотите включенной вложенной виртуализацией, настройте параметры памяти, сетевые адаптеры, виртуальные жесткие диски и выберите, следует ли установить операционную систему из ISO-файл образа или по сети.
7. Нажмите кнопку **Создать** для создания виртуальной машины.
8. После ее создается и отображается в списке виртуальной машины, можно запустить на виртуальной машине.
9. После запуска на виртуальной машине, вы можете подключиться к виртуальной машине консоли через VMConnect для установки операционной системы. Выберите виртуальную машину в списке, нажмите кнопку **Дополнительные** > **подключение** для скачивания RDP-файл. Откройте RDP-файл в приложении удаленного рабочего стола. Так как это подключение к консоли виртуальной машины, необходимо будет ввести учетные данные администратора узла Hyper-V.

## Изменение параметров виртуальной машины

![Экран параметров виртуальной машины](../media/manage-virtual-machines/vm-settings.png)

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин, выбрать вкладку **инвентаризации** выберите виртуальную машину в списке и нажмите кнопку **больше** > **Параметры**.
3. Переключаться между вкладке **Общие**, **безопасности**, **памяти**, **процессоры**, **диски**, **сетей**, **порядок загрузки** и **контрольных точек** , настроить необходимые параметры, а затем нажмите кнопку **Сохранить** для сохранения параметры текущей вкладки. Доступные параметры будут различаться в зависимости от поколения виртуальной машины. Кроме того некоторые параметры нельзя изменить для запуска виртуальных машин, вам потребуется сначала остановите работу виртуальной машины.

## Динамическая миграция виртуальной машины на другой узел кластера

При подключении к кластеру, можно Динамическая миграция виртуальной машины на другой узел кластера.

1. Отказоустойчивый кластер или подключения гиперконвергентных кластеров выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин, выбрать вкладку **инвентаризации** выберите виртуальную машину в списке и нажмите кнопку **больше** > **перемещения**.
3. Выберите сервер из списка доступных узлов и нажмите кнопку **перемещения**.
4. Уведомления для перемещения хода выполнения отображается в правом верхнем углу Windows Admin Center. Если перемещение выполнено успешно, вы увидите имя хост-сервера, изменения в списке виртуальной машины.

## Расширенное управление и устранение неполадок в одной виртуальной машины

![Экран подробных сведений о одной виртуальной машины](../media/manage-virtual-machines/vm-details.png)

Вы можете просмотреть подробные сведения и диаграммы производительности для одной виртуальной машины на странице одной виртуальной машины.

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин выберите вкладку **инвентаризации** щелкните на имени виртуальной машины в списке виртуальной машины.
3. На странице одной виртуальной машине можно сделать следующее.
    - Просмотр подробных сведений для виртуальной машины.
    - Просмотр Live накопленных статистических данных диаграммы и графики для ЦП, память, сеть, пропускная способность операций ввода-ВЫВОДА и операций ввода-ВЫВОДА (накопленных статистических данных доступна только для гиперконвергентными кластерами под управлением Windows Server 2019)
    - Просмотр, создание, применяются, переименовать и удаление контрольные точки.
    - Просмотр подробных сведений для виртуальной машины виртуальный жесткий диск (VHD), сетевые адаптеры и хост-сервера.
    - Удалить, начать, отключить, завершение работы, приостановить, возобновить, сброс или переименовать виртуальной машины. Также сохранить виртуальную машину, удалить сохраненные состояния или создайте контрольную точку.
    - [Изменение параметров виртуальной машины](#change-virtual-machine-settings).
    - Подключение к консоли виртуальной машины, с помощью VMConnect через узел Hyper-V.
    - [Репликация виртуальной машины с помощью Azure Site Recovery](#protect-virtual-machines-with-azure-site-recovery).

## Управление виртуальной машины через узел Hyper-V (VMConnect)

![Виртуальная машина устанавливать подключение через веб-браузере](../media/manage-virtual-machines/vm-connect.png)

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин, выбрать вкладку **инвентаризации** выберите виртуальную машину в списке и нажмите кнопку **больше** > **Connect** или **загрузить RDP-файл**. **Подключение** позволяют взаимодействовать с гостевой виртуальной Машины через веб-консоль удаленного рабочего стола, интегрирован в Windows Admin Center. **Скачать RDP-файл** загружается RDP-файл, открываемый с помощью приложения удаленного рабочего стола (mstsc.exe). Оба варианта будет использовать VMConnect для подключения к гостевой виртуальной Машины через узел Hyper-V и потребуется ввести учетные данные администратора для сервера узла Hyper-V.

## Изменение параметров узла Hyper-V

![Экран параметров узла Hyper-V](../media/manage-virtual-machines/host-settings.png)

1. Подключение с сервера, кластера Hyper или отказоустойчивый кластер щелкните меню " **Параметры** " в нижней части левой панели навигации.
2. На сервере узла Hyper-V или в кластере вы увидите группу **Параметров узла Hyper-V** с помощью следующих разделах:
    - Общие: Изменение виртуальных жестких дисков и виртуальных машин путь к файлу и гипервизор запланировать тип (если поддерживается)
    - Режим расширенного сеанса
    - Эта опция NUMA
    - Динамическая миграция
    - Миграция хранилищ
3. В случае внесения любой узел Hyper-V, параметр изменения в соединение гиперконвергентных кластеров или отказоустойчивого кластера, изменение будет применяться ко всем узлам кластера.

## Просмотрите журналы событий Hyper-V

Вы можете просмотреть журналы событий Hyper-V непосредственно из средства виртуальных машин.

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин выберите вкладку " **Сводка** ". В верхнем правом разделе события щелкните **ПРОСМОТРЕТЬ все СОБЫТИЯ**.
3. Средство просмотра событий будет отображаться каналы событие Hyper-V на левой панели. Выберите канал для просмотра событий в правой области. Если вы управляете отказоустойчивого кластера или кластера, журналы событий будет отображаться события для всех узлов кластера, отображая хост-сервера в столбце "компьютер".
