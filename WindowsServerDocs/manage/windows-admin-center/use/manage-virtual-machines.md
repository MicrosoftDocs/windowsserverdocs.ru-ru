# <a name="managing-virtual-machines-with-windows-admin-center"></a>Управление виртуальными машинами с помощью Windows Admin Center

>Область применения. Windows Admin Center, предварительная версия Windows Admin Center

Виртуальные машины средство доступно в [Server](manage-servers.md), [отказоустойчивого кластера](manage-failover-clusters.md) или [Hyper-Converged кластера](manage-hyper-converged.md) подключений, если роль Hyper-V включена на сервере или кластере. Можно использовать средство виртуальных машин для управления узлами Hyper-V под управлением Windows Server 2012 или более поздней версии, либо с возможностями рабочего стола или основных серверных компонентов. Также поддерживаются Hyper-V Server 2012 и 2016.

## <a name="key-features"></a>Основные особенности

Вот некоторые средства виртуальных машин в Windows Admin Center:

- **Высокоуровневые Hyper-V узла отслеживания ресурсов.** Просмотрите общее использование ЦП и памяти, метрики производительности операций ввода-ВЫВОДА, оповещения о работоспособности виртуальной Машины и события для сервера узла Hyper-V или кластера в одной панели мониторинга.
- **Унифицированный процесс, объединяя возможности диспетчера Hyper-V и диспетчера отказоустойчивости кластеров.** Просмотр всех виртуальных машин в кластере и детализировать одну виртуальную машину для расширенного управления и устранения неполадок.
- **Упрощенное и мощное рабочие процессы для управления виртуальными машинами.** Новые возможности пользовательского интерфейса, специально для сценариев администрирования ИТ для создания, управления и репликации виртуальных машин.

## <a name="monitor-hyper-v-host-resources-and-performance"></a>Отслеживать ресурсы узла Hyper-V и производительность

![Сводка виртуальных машин](../media/manage-virtual-machines/virtual-machines-summary.png)

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. Есть две вкладки в верхней части **виртуальных машин** средство, **Сводка** вкладку и **инвентаризации** вкладки. **Сводка** вкладка предоставляет целостное представление ресурсов узла Hyper-V и производительности для текущего сервера или всего кластера, включая следующие:
    - Число виртуальных машин, сгруппированных по состоянию — под управлением, задано значение off, приостановки и сохранен
    - Последние оповещения о работоспособности или события журнала событий Hyper-V (оповещений доступны только для гиперконвергентной кластеров под управлением Windows Server 2016 или более поздней версии)
    - Использование ЦП и памяти с разбивкой гостевого узла vs
    - TOP виртуальных машин, потребляющие больше всего ресурсов ЦП и памяти
    - Динамические данные и данные журнала графики для пропускной способности операций ввода-ВЫВОДА и операций ввода-ВЫВОДА (графиков производительности хранилища доступны только для гиперконвергентной кластеров под управлением Windows Server 2016 или более поздней версии. Исторических данных доступен только для гиперконвергентной кластеров под управлением Windows Server 2019)

## <a name="view-virtual-machine-inventory"></a>Просмотр данных инвентаризации виртуальной машины

![Экран инвентаризации виртуальных машин](../media/manage-virtual-machines/virtual-machines-inventory.png)

2. Есть две вкладки в верхней части **виртуальных машин** средство, **Сводка** вкладку и **инвентаризации** вкладки. **Инвентаризации** вкладке перечислены виртуальные машины, доступных на текущем сервере или весь кластер и предоставляет команды для управления отдельными виртуальными машинами. Можно выполнить следующие действия: 
    - Просмотрите список виртуальных машин, работающих на текущем сервере или кластере.
    - Просмотр состояния и сервера виртуальной машины, при просмотре виртуальных машин для кластера. Также просмотрите сведения об использовании ЦП и памяти с точки зрения узла, включая нехватки памяти, требуемый объем памяти и объем выделенной памяти и виртуальной машины время работы, пульса состояния и состояния защиты с помощью Azure Site Recovery.
    - [Создание новой виртуальной машины](#create-a-new-virtual-machine).
    - Удалить, запустить, отключить, завершение работы, приостановить, возобновить, сброс или переименования виртуальной машины. Сохранить виртуальную машину, удалить сохраненное состояние или создать контрольную точку.
    - [Изменить параметры для виртуальной машины](#change-virtual-machine-settings).
    - Подключитесь к консоли виртуальной машины, с помощью VMConnect через узел Hyper-V.
    - [Репликация виртуальной машины с помощью Azure Site Recovery](#protect-virtual-machines-with-azure-site-recovery).

ПРИМЕЧАНИЕ. При подключении к кластеру, средство виртуальной машины будут отображаться только виртуальных машин. Мы планируем также Показывать некластеризованным виртуальных машин.

## <a name="create-a-new-virtual-machine"></a>Создание новой виртуальной машины

![Создание нового экрана виртуальной машины](../media/manage-virtual-machines/new-vm.png)

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **инвентаризации** , а затем нажмите кнопку **New** для создания новой виртуальной машины.
3. Введите имя виртуальной машины и выберите между виртуальными машинами поколения 1 и 2.
4. При создании виртуальной машины в кластере, вы можете изначально Создание виртуальной машины на узел. Если вы используете Windows Server 2016 или более поздней версии, средство предложит рекомендации узла для вас.
5. Выберите путь для файлов виртуальной машины. Выберите том из раскрывающегося списка или нажмите кнопку **Обзор** чтобы выбрать папку, используя средство выбора папки. Файлы конфигурации виртуальной машины и файл виртуального жесткого диска будут сохранены в одной папке `\Hyper-V\\[virtual machine name]` путь выбранного тома или путь.
6. Выберите число виртуальных процессоров, будь то требуется вложенной виртуализацией, настроить параметры памяти, сетевых адаптеров, виртуальные жесткие диски и выберите, следует ли установить операционную систему из ISO-файла образа или от сети.
7. Нажмите кнопку **создать** для создания виртуальной машины.
8. После того как виртуальная машина создается и отображается в списке виртуальных машин, можно запустить виртуальную машину.
9. После запуска виртуальной машины можно подключиться к консоли виртуальной машины с помощью VMConnect для установки операционной системы. Выберите виртуальную машину из списка, щелкните **дополнительные** > **Connect** загрузить RDP-файл. Откройте RDP-файл в приложении подключение удаленного рабочего стола. Так как это подключается к консоли виртуальной машины, необходимо будет ввести учетные данные администратора на узле Hyper-V.

## <a name="change-virtual-machine-settings"></a>Изменение параметров виртуальной машины

![Экран параметров виртуальной машины](../media/manage-virtual-machines/vm-settings.png)

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **инвентаризации** вкладки. Выберите виртуальную машину из списка и нажмите кнопку **дополнительные** > **параметры**.
3. Переключаться между **Общие**, **памяти**, **процессоров**, **дисков**, **сетей**, **Порядке загрузки** и **контрольные точки** , настроить необходимые параметры, а затем нажмите кнопку **Сохранить** сохранить параметры текущей вкладки. Доступные параметры зависят от создания виртуальной машины. Кроме того некоторые параметры нельзя изменить для работающих виртуальных машин, и вам потребуется сначала остановите виртуальную машину.

## <a name="live-migrate-a-virtual-machine-to-another-cluster-node"></a>Динамическая миграция виртуальной машины на другой узел кластера

При подключении к кластеру, можно Динамическая миграция виртуальной машины на другой узел кластера.

1. Из отказоустойчивого кластера или подключение к кластеру Hyper со схождением, нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **инвентаризации** вкладки. Выберите виртуальную машину из списка и нажмите кнопку **дополнительные** > **переместить**.
3. Выберите сервер из списка доступных узлов и нажмите кнопку **переместить**.
4. Уведомления для перемещения ход выполнения отображается в правом верхнем углу Windows Admin Center. Если переход выполнен успешно, вы увидите имя узла сервера, измененный в списке виртуальных машин.

## <a name="advanced-management-and-troubleshooting-for-a-single-virtual-machine"></a>Расширенное управление и устранение неполадок для одной виртуальной машины

![Экран сведений об одной виртуальной машины](../media/manage-virtual-machines/vm-details.png)

Можно просмотреть подробные сведения и диаграммы производительности для отдельной виртуальной машины на странице одной виртуальной машины.

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **инвентаризации** вкладки. Щелкните имя виртуальной машины из списка виртуальных машин.
3. На странице одну виртуальную машину вы можете:
    - Просмотрите подробные сведения для виртуальной машины.
    - Просмотреть в прямом эфире или исторических данных графики для ЦП, памяти, сети, пропускная способность операций ввода-ВЫВОДА и операций ввода-ВЫВОДА (исторических данных доступен только для гиперконвергентной кластеров под управлением Windows Server 2019)
    - Просмотра, создания, применения, переименование и удаление контрольных точек.
    - Просмотр сведений для виртуальной машины файлы виртуального жесткого диска (VHD), сетевые адаптеры и сервера узла.
    - Удалить, запустить, отключить, завершение работы, приостановить, возобновить, сброс или переименования виртуальной машины. Сохранить виртуальную машину, удалить сохраненное состояние или создать контрольную точку.
    - [Изменить параметры для виртуальной машины](#change-virtual-machine-settings).
    - Подключитесь к консоли виртуальной машины, с помощью VMConnect через узел Hyper-V.
    - [Реплицируйте виртуальную машину с помощью Azure Site Recovery](#protect-virtual-machines-with-azure-site-recovery).

## <a name="view-hyper-v-event-logs"></a>Просмотреть журналы событий Hyper-V

Можно просмотреть журналы событий Hyper-V непосредственно из средства виртуальных машин.

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **Сводка** вкладки. В правом верхнем разделе события, щелкните **Просмотр всех СОБЫТИЙ**.
3. Средство просмотра событий отобразится каналы событий Hyper-V в левой области. Выберите канал для просмотра событий в области справа. Если Управление отказоустойчивым кластером или гиперконвергированного кластера, журналы событий для отображения событий для всех узлов кластера, отображается в столбце компьютера сервера узла.
