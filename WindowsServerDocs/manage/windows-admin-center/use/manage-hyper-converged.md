# Управление Гиперконвергентной инфраструктурой с помощью Windows Admin Center

>Относится к: Windows Admin Center, ознакомительная версия Windows Admin Center

## Что такое гиперконвергированном инфраструктуры

Гиперконвергентная инфраструктура объединяет программно определяемого вычислений, хранения и сети в один кластер для обеспечения высокой производительности, экономичные и без труда масштабируемости виртуализации. Эта возможность появилась в Windows Server 2016 с помощью [Локальных дисковых пространств](https://docs.microsoft.com/windows-server/storage/storage-spaces/storage-spaces-direct-overview), [Программно-конфигурируемой сети](https://docs.microsoft.com/en-us/windows-server/networking/sdn/software-defined-networking) и [Hyper-V](https://docs.microsoft.com/windows-server/virtualization/hyper-v/hyper-v-on-windows-server).

> [!Tip]
> Необходимо приобрести инфраструктуры гиперконвергированном? Корпорация Майкрософт рекомендует эти решения [Windows Server Software-Defined](https://microsoft.com/wssd) от наших партнеров. Они разработаны, собраны и проверяется наших Эталонная архитектура для обеспечения совместимости и надежности, чтобы получить и быстро.

> [!IMPORTANT]
> Некоторые функции, описанные в этой статье доступны только в ознакомительной версии Windows Admin Center. [Как получить эту версию?](http://aka.ms/windowsadmincenter)

## Что такое Windows Admin Center

[Windows Admin Center](../understand/windows-admin-center.md) — это средство управления следующего поколения, предназначенную для Windows Server, пришла традиционных средств «в поле» как диспетчер серверов. Это бесплатное и можно установить и использовать без подключения к Интернету. Windows Admin Center можно использовать для управления и отслеживания инфраструктуры гиперконвергированном под управлением Windows Server 2016 или Windows Server 2019.

![Информационная панель гиперконвергентных кластеров](../media/manage-hyper-converged/hci-dashboard-v1809.png)

## Ключевые функции

Особенности Windows Admin Center для инфраструктуры гиперконвергированном:

- **Единый один панели из-консоль управления для вычислений, хранения и скоро сети.** Просмотр виртуальных машин, хост-серверы, тома, дисков и сведения в пределах одного основная файловая система, стабильное и взаимосвязанных взаимодействия.
- **Создание и управление виртуальными машинами Hyper-V и дисковые пространства.** Существенно простой рабочие процессы для создания, открытия, изменение размера и удаления тома; Создание, запуск, подключения к и перемещения виртуальных машин; и многое другое.
- **Эффективный мониторинг всего кластера.** Панель мониторинга диаграмм памяти и ЦП, памяти, операций ввода-ВЫВОДА, пропускной способности и задержки в режиме реального времени на каждом сервере в кластере, с помощью очистить оповещения, когда что-то не подходит.
- **Поддержка программно конфигурируемой сети (SDN).** Управление и мониторинг виртуальных сетей, подсетей, подключения виртуальных машин к виртуальным сетям и отслеживания инфраструктуры SDN.

Windows Admin Center для инфраструктуры гиперконвергированном активно разработанные корпорацией Майкрософт. Он получает частых обновлений, которые улучшения существующих функций и добавлять новые компоненты.

## Перед началом работы

Для управления кластер как инфраструктура гиперконвергированном в Windows Admin Center, он должен работать под управлением Windows Server 2016 или Windows Server 2019 и Hyper-V и локальными дисковыми пространствами включили. Кроме того он также может иметь программно-конфигурируемой сети включен и управлять с помощью Windows Admin Center.

> [!Tip]
> Windows Admin Center также предоставляет возможность управления взаимодействия кластеров поддержка любой рабочей нагрузкой, доступные для Windows Server 2012 и более поздних версиях. Это может показаться лучше расположить, при добавлении кластера в Windows Admin Center, установите [**Отказоустойчивый кластер**](manage-failover-clusters.md) вместо **Гиперконвергированном кластере**.

### Подготовка кластера Windows Server 2016 для Windows Admin Center

Windows Admin Center для инфраструктуры гиперконвергированном зависит от управления, которые добавлены API после выпуска Windows Server 2016. Прежде чем можно будет управлять кластер Windows Server 2016 с помощью Windows Admin Center, необходимо выполнить следующие два действия:

1. Убедитесь, что каждый сервер в кластере установлено [2018 г. — 05 накопительное обновление для Windows Server 2016 (KB4103723)](https://support.microsoft.com/help/4103723/windows-10-update-kb4103723) или более поздней версии. Чтобы загрузить и установить это обновление, перейдите к **параметрам** > **обновление & безопасности** > **Центра обновления Windows** и выберите **online проверить наличие обновлений из центра обновления Майкрософт**.
2. Выполните следующий командлет PowerShell от имени администратора в кластере:

```powershell
    Add-ClusterResourceType -Name "SDDC Management" -dll "$env:SystemRoot\Cluster\sddcres.dll" -DisplayName "SDDC Management"
```

> [!Tip]
> Необходимо только один раз, выполните командлет на любом сервере в кластере. Можно запустить его локально в Windows PowerShell или использовать поставщик службы безопасности учетных данных (CredSSP) для удаленного запуска. В зависимости от конфигурации вы не сможете для выполнения этого командлета из в Windows Admin Center.

### Подготовка Windows Server 2019 кластер для Windows Admin Center

Если кластер работает Windows Server 2019, описанные выше действия не требуются. Просто добавьте кластера в Windows Admin Center, как описано в следующем разделе, и вы достаточно просто!

### Настройка программно-определяемой сети (необязательно) ###

Можно настроить инфраструктуру гиперконвергированном под управлением Windows Server 2016 или 2019 г., чтобы использовать программно-конфигурируемой сети (SDN), выполнив следующие действия:

1. Подготовьте виртуальный жесткий ДИСК операционной системы, что же ОС, установленных на узлах гиперконвергентной инфраструктурой. Этот виртуальный жесткий ДИСК будет использоваться для всех виртуальных машин контекста Именования/SLB/GW.
2. Скачать все папки и файлы в SDN Express из [https://github.com/Microsoft/SDN/tree/master/SDNExpress](https://github.com/Microsoft/SDN/tree/master/SDNExpress).
3. Подготовка разных виртуальных Машин с помощью консоли развертывания. Эту виртуальную Машину должны иметь возможность доступа к SDN узлов. Кроме того виртуальная машина должен иметь средства удаленного администрирования сервера Hyper-V установлена.
4. Скопируйте все, что вы загрузили для SDN Express на консоль развертывания виртуальной Машины. Это **SDNExpress** папки и. Убедитесь, что каждый узел может получить доступ к общей папке **SDNExpress** согласно определению в строке файла конфигурации 8:
```
    \\$env:Computername\SDNExpress
```
5. Скопируйте виртуальный жесткий ДИСК операционной системы в папке **изображений** в папке **SDNExpress** на консоль развертывания виртуальной Машины.
6. Изменение конфигурации SDN Express с Настройка вашей среды. Завершите следующие два действия после изменения конфигурации SDN Express, на основе данных вашей среды.
7. Запустите PowerShell с правами администратора для развертывания SDN:

```powershell
    .\SDNExpress.ps1 -ConfigurationDataFile .\your_fabricconfig.PSD1 -verbose 
```

Развертывание займет около 30-45 минут.

## Начало работы

После развертывания инфраструктуры гиперконвергированном можно управлять с помощью Windows Admin Center.

### Установка Windows Admin Center

Если вы еще не сделали этого, скачайте и установите Windows Admin Center. Самый быстрый способ получения и выполнении — установить его на компьютере Windows 10 и удаленного управления серверами. Это занимает меньше 5 минут. [Загрузить](https://aka.ms/windowsadmincenter) или [более ознакомьтесь с другими вариантами установки](../deploy/install.md).

### Добавление кластера

Чтобы добавить кластера Windows Admin Center:

1. Выберите вариант **+ Добавить** всех подключений.
2. При необходимости добавьте **Гиперконвергированных кластеров подключения**.
3. Введите имя кластера и, если появится запрос, учетные данные для использования.
4. Нажмите кнопку **Добавить** для завершения.

Кластер будет добавлен в список подключений. Щелкните его, чтобы запустить на информационной панели.

![Добавление подключения гиперконвергентного кластера](../media/manage-hyper-converged/add-hyper-converged-cluster-connection.gif)

### Добавление SDN с поддержкой кластера (ознакомительная версия Windows Admin Center)

Последние просмотра центра администрирования Windows поддерживает управление программно-конфигурируемой сети для гиперконвергированном инфраструктуры. Добавляя URI REST сетевого контроллера подключения Гиперконвергентных кластеров, можно использовать диспетчер гиперконвергентных кластеров управлять ресурсами SDN и отслеживать SDN инфраструктуры.

1. Выберите вариант **+ Добавить** всех подключений.
2. При необходимости добавьте **Гиперконвергированных кластеров подключения**.
3. Введите имя кластера и, если появится запрос, учетные данные для использования.
4. Проверьте **Настройка сетевого контроллера** , чтобы продолжить.
5. Введите **URI сетевого контроллера** и щелкните **Проверить**.
6. Нажмите кнопку **Добавить** для завершения.

Кластер будет добавлен в список подключений. Щелкните его, чтобы запустить на информационной панели.

![Добавить подключение SDN с поддержкой кластера](../media/manage-hyper-converged/add-snd-enabled-hci-connection.png)

> [!Important]
> SDN средах с проверкой подлинности Kerberos для Northbound связи, в настоящее время не поддерживаются.

## Вопросы и ответы

### Существуют различия между управление Windows Server 2016 и Windows Server 2019?

Да. Windows Admin Center для инфраструктуры гиперконвергированном получает частых обновлений, которые улучшают взаимодействие с Windows Server 2016 и Windows Server 2019. Тем не менее некоторые новые возможности доступны только для Windows Server 2019 — например, элемент управления toggle switch для дедупликации и сжатия.

### Можно ли использовать Windows Admin Center для управления локальными дисковыми пространствами для других вариантов использования (не гиперконвергентной), например стойке масштабируемый файловый сервер (SoFS) или Microsoft SQL Server?

Windows Admin Center для инфраструктуры гиперконвергированном не поддерживает управление или средства наблюдения специально для других вариантов использования локальных дисковых пространств — например, он не позволяет создавать общие папки. Тем не менее панели мониторинга и основные функции, как создание томов или замена дисков, работают для любого кластера локальных дисковых пространств.

### Что такое разница между отказоустойчивом кластере и гиперконвергированном кластере?

Как правило термин «гиперконвергентной» относится к под управлением Hyper-V и локальные дисковые пространства на том же кластерные серверы для виртуализации ресурсы вычислений и хранения данных. В контексте Windows Admin Center при нажатии кнопки **+ Добавить** из списка подключений, вы можете выбрать между добавления **подключения отказоустойчивого кластера** или **подключения гиперконвергированном кластере**:

- **Подключения отказоустойчивого кластера** призван классического приложения диспетчера отказоустойчивости кластеров. Он предоставляет знакомым, позволяющей управления вкладками для кластеров поддержка любой рабочей нагрузкой, в том числе Microsoft SQL Server. Это для Windows Server 2012 и более поздних версиях.

- **Подключение гиперконвергированном кластере** — это совершенно новый, предназначенное для локальных дисковых пространств и Hyper-V. В его состав входит информационная панель с диаграммами и оповещениями для мониторинга. Эта функция доступна для Windows Server 2016 и Windows Server 2019.

### Зачем нужна последний накопительный пакет обновления для Windows Server 2016?

Windows Admin Center для инфраструктуры гиперконвергированном зависит от управления API-интерфейсы разрабатываются с момента выпуска Windows Server 2016. Эти API-интерфейсы добавляются в [2018 г. — 05 накопительное обновление для Windows Server 2016 (KB4103723)](https://support.microsoft.com/help/4103723/windows-10-update-kb4103723), включенная в 8 мая 2018 г.

### Сколько стоит использование Windows Admin Center?

Пользоваться Windows Admin Center можно бесплатно в рамках оплаченной лицензии на Windows.

Windows Admin Center (доступна в виде отдельной загрузки) можно использовать с действующими лицензиями Windows Server или Windows 10 без дополнительной платы — он предоставляется по лицензии Windows регламентируется дополнительным ЛИЦЕНЗИОННЫМ соглашением.

### Требуется ли для работы с Windows Admin Center пакет продуктов System Center?

Нет.

### Он требует подключения к Интернету?

Нет.

Несмотря на то, что Windows Admin Center обеспечивает мощные и удобные интеграции с в облако Microsoft Azure, core управления и мониторинга возможности для инфраструктуры гиперконвергированном будет полностью локального. Его можно установить и использовать без подключения к Интернету.

### Создание новой виртуальной машины

1. Выберите средство **виртуальных машин** на левой навигационной панели.
2. В верхней части средства виртуальных машин выберите вкладку " **Инвентаризация** ", а затем щелкните **Создать** , чтобы создать новую виртуальную машину.
3. Введите имя виртуальной машины и выбрать один из виртуальных машин поколения 1 и 2.
4. Закомментированный могут узел первоначального создания виртуальной машины на или использовать рекомендуемые узла.
5. Выберите путь для файлов виртуальной машины. В раскрывающемся списке выберите том или нажмите кнопку **Обзор** , чтобы выбрать папку с помощью средства выбора папок. Файлы конфигурации виртуальной машины и файл виртуального жесткого диска будут сохранены в одной папке `\Hyper-V\[virtual machine name]` путь к выбранный том или путь.
6. Укажите, сколько виртуальных процессоров ли вы хотите включенной вложенной виртуализацией, настройте параметры памяти, сетевые адаптеры, виртуальные жесткие диски и выберите, следует ли установить операционную систему из ISO-файл образа или по сети.
7. Нажмите кнопку **Создать** для создания виртуальной машины.
8. После ее создается и отображается в списке виртуальной машины, можно запустить на виртуальной машине.
9. После запуска на виртуальной машине, вы можете подключиться к виртуальной машине консоли через VMConnect для установки операционной системы. Выберите виртуальную машину в списке, нажмите кнопку **Дополнительные** > **подключение** для скачивания RDP-файл. Откройте RDP-файл в приложении удаленного рабочего стола. Так как это подключение к консоли виртуальной машины, необходимо будет ввести учетные данные администратора узла Hyper-V.

[Подробнее об управлении виртуальной машины с помощью Windows Admin Center](manage-virtual-machines.md).

### Приостановить и безопасно перезапустить сервер

1. На панели **мониторинга**выберите **серверы** навигации слева или, перейдя по ссылке **> Просмотр СЕРВЕРОВ** на плитке в правом нижнем углу панели мониторинга.
2. В верхней переход с **Сводка** на вкладке **инвентаризации** .
3. Выберите сервер, щелкнув его имя, чтобы открыть страницу сведений **сервера** .
4. Щелкните **приостановки сервера для обслуживания**. Если это безопасно, перемещение виртуальных машин на другие серверы в кластере. Сервер будет иметь состояние очистка при этом процесс. Если вы хотите, посмотрите виртуальных машин перемещения на странице **> виртуальных машин инвентаризации** , где их сервера узла отображается четко в сетке. Когда все виртуальные машины были перемещены, состояние сервера будет **приостановлено**.
5. Выберите **сервер управление** для доступа к все средства управления каждого сервера в Windows Admin Center.
6. Нажмите кнопку **перезагрузки**, нажмите **Да**. Вы будете открывает возможность обратно в список подключений.
7. Кнопка возврата на панели **мониторинга**сервера красного цвета во время его работы.
8. Установив его, резервное копирование снова перейдите на страницу **сервера** и выберите **Возобновить сервер от обслуживания** в состояние сервера снова установить просто вверх. Во время виртуальные машины будут перемещаться назад — не требуется никаких действий пользователя.

### Руководства по замене диска

1. Когда диск не удается, предупреждение отображается в левой **оповещения** верхней панели **мониторинга**.
2. Можно также выбрать **диски** при навигации с левой стороны или **> ПРЕДСТАВЛЕНИЕ ДИСКИ** по ссылке на плитке в правом нижнем углу для просмотра дисков и узнайте их состояние. На вкладке **инвентаризации** сетки поддерживает сортировку, группирование и поиск по ключевым словам.
3. На панели **мониторинга**выберите оповещение, чтобы просмотреть подробные сведения, такие как физическое расположение этого диска.
4. Дополнительные сведения, щелкните ярлык **Перейти на диск** , на страницу сведений **диска** .
5. Если устройство поддерживает его, можно щелкнуть **Включить свет Вкл. / выкл.** для управления индикатор этого диска.
6. Локальные дисковые пространства автоматически завершение и из ноутбука вместе с неисправных дисков. Если это произошло, действовать состояние диска, а отрезка емкость хранилища будет пустым.
7. Неисправные диски и вставьте их замене.
8. В **диски с данными > инвентаризации**появится новый диск. Во время оповещения будут очищены, томов будет восстановить обратно в состояние OK и хранилища будет заново сбалансировать на новый диск — не требуется никаких действий пользователя.
