# <a name="manage-hyper-converged-infrastructure-with-windows-admin-center"></a>Управление Гиперконвергентной инфраструктуры с помощью Windows Admin Center

>Область применения. Windows Admin Center, предварительная версия Windows Admin Center

## <a name="what-is-hyper-converged-infrastructure"></a>Что такое Hyper-Converged инфраструктуры

Гиперконвергентное инфраструктуры объединяет программно определяемых вычислений, хранилища и сети в один кластер для обеспечения высокой производительности, экономичные и легко масштабируются виртуализации. Эта возможность появилась в Windows Server 2016 с [Storage Spaces Direct](https://docs.microsoft.com/windows-server/storage/storage-spaces/storage-spaces-direct-overview) и [Hyper-V](https://docs.microsoft.com/windows-server/virtualization/hyper-v/hyper-v-on-windows-server).

> [!Tip]
> Необходимы для инфраструктуры Hyper-Converged? Корпорация Майкрософт рекомендует эти [определяемые серверное программное обеспечение Windows](https://microsoft.com/wssd) решения от наших партнеров. Они разработаны, собраны и проверены в нашей эталонной архитектуры для обеспечения совместимости и надежности, чтобы оперативно настраивайте и быстрый запуск.

> [!IMPORTANT]
> Некоторые функции, описанные в этой статье доступны только в предварительной версии Windows Admin Center. [Как получить эту версию?](http://aka.ms/windowsadmincenter)

## <a name="what-is-windows-admin-center"></a>Что такое Windows Admin Center

[Windows Admin Center](../understand/windows-admin-center.md) является средством управления следующего поколения для Windows Server, преемник традиционные средства «коробочных» как диспетчер серверов. Она бесплатна и может установить и использовать без подключения к Интернету. Windows Admin Center можно использовать для управления и мониторинга инфраструктуры Hyper-Converged под управлением Windows Server 2016 или более построение Windows Server 2019 Insider Preview.

![Панель мониторинга гиперконвергентного кластера](../media/manage-hyper-converged/hci-dashboard-v1809.png)

## <a name="key-features"></a>Основные особенности

Особенности Windows Admin Center Hyper-Converged инфраструктуры:

- **Единой одним панели объекта-аварийного доступа для вычислений, хранения и скоро сети.** Просмотр виртуальных машин, серверы узлов, тома, дисков и многое другое в одну процедуру специализированное, согласованных и взаимосвязанных.
- **Создание и управление дисковыми пространствами и Hyper-V виртуальные машины.** Радикально простых рабочих процессов для создания, открытия, изменения размера и удаление томов; и создания, запуска, подключения и перемещать виртуальные машины; и многое другое.
- **Мощные мониторинг на уровне кластера.** Панель диаграмм, памяти и ЦП, объем хранилища, операций ввода-ВЫВОДА, пропускной способности и задержки в реальном времени, на каждом сервере кластера и очистить создает предупреждение, если что-то не обладает требуемыми правами.
- **Поддержка программного обеспечения конфигурируемой сети (SDN). (Новое в предварительной версии Windows Admin Center)**  Управление и мониторинг виртуальных сетей, подсетей, подключать виртуальные машины для виртуальных сетей и мониторинг инфраструктуры SDN.

Windows Admin Center для инфраструктуры Hyper-Converged активно разрабатывается корпорацией Майкрософт. Он получает частые обновления, улучшения существующих функций и добавления новых функций.

## <a name="before-you-start"></a>Прежде чем начать

Для управления кластером, как инфраструктура Hyper-Converged в Windows Admin Center, он должен работать под управлением Windows Server 2016 или предварительной версии сборки Windows Server 2019 и Hyper-V и дисковые пространства включены.

> [!Tip]
> Windows Admin Center также предлагает возможности общего назначения управления для любой кластер, поддерживающий любой рабочей нагрузки, доступные для Windows Server 2012 и более поздних версий. Если это может показаться нехитрой более эффективными при добавлении кластера Windows Admin Center, выберите [ **отказоустойчивого кластера** ](manage-failover-clusters.md) вместо **Hyper-Converged кластера**.

### <a name="prepare-your-windows-server-2016-cluster-for-windows-admin-center"></a>Подготовка к Windows Admin Center вашего кластера Windows Server 2016

Windows Admin Center для инфраструктуры Hyper-Converged зависит от управления, которые добавлены интерфейсы API, после выпуска Windows Server 2016. Прежде чем можно будет управлять кластера Windows Server 2016 с помощью Windows Admin Center, необходимо выполнить следующие два действия:

1. Убедитесь, что каждый сервер в кластере установлен [2018-05 накопительный пакет обновления для Windows Server 2016 (KB4103723)](https://support.microsoft.com/help/4103723/windows-10-update-kb4103723) или более поздней версии. Чтобы скачать и установить это обновление, перейдите к **параметры** > **обновление и безопасность** > **обновление Windows** и выберите  **Проверять наличие обновлений из центра обновления Майкрософт**.
2. Выполните следующий командлет PowerShell от имени администратора в кластере:

```powershell
    Add-ClusterResourceType -Name "SDDC Management" -dll "$env:SystemRoot\Cluster\sddcres.dll" -DisplayName "SDDC Management"
```

> [!Tip]
> Необходимо только один раз, выполните командлет на любом сервере в кластере. Запустить его локально в Windows PowerShell также можно использовать для удаленного запуска поставщика службы безопасности учетных данных (CredSSP). В зависимости от конфигурации может не появиться возможность выполнения этого командлета в Windows Admin Center.

> [!Important]
> Для развертываний в неанглийских языковых стандартов имеется известная проблема в версии 1804 Windows Admin Center, который предотвращает загрузку (только в первый раз) панели мониторинга. Решение заключается в запуске `Add-ClusterResource -Name 'SDDC Management' -Group 'Cluster Group' -ResourceType 'SDDC Management'` замена *«Группы кластера»* с локализованное имя, например, *«Группы кластера du»* на французском языке. Эта проблема будет устранена в следующем обновлении.
>
> **ОБНОВЛЕНИЕ:** Теперь эта проблема устранена в версии Preview Windows Admin Center 1806.

### <a name="prepare-your-windows-server-2019-cluster-for-windows-admin-center"></a>Подготовка к Windows Admin Center вашего кластера Windows Server 2019

Если ваш кластер выполняет сборку Insider Preview Windows Server 2019, описанные выше действия не требуются. Просто добавьте в кластер Windows Admin Center, как описано в следующем разделе, и вы готовы к работе! [Скачать последнюю сборку предварительной версии Windows Server 2019](https://www.microsoft.com/en-us/software-download/windowsinsiderpreviewserver).

### <a name="configure-software-defined-networking-optional"></a>Настроить программно-определяемой сети (необязательно) ###

Можно настроить инфраструктуру Hyper-Converged под управлением Windows Server 2016 или 2019 г., чтобы использовать программно определяемой сети (SDN), выполнив следующие действия:

1. Подготовка виртуального жесткого диска операционной системы, который является одной операционной системе установлены на узлах гиперконвергентного инфраструктуры. Этот виртуальный жесткий ДИСК будет использоваться для всех виртуальных машин сетевого Контроллера и SLB или ШЛЮЗ.
2. Скачать все папки и файлы в SDN Express из [ https://github.com/Microsoft/SDN/tree/master/SDNExpress ](https://github.com/Microsoft/SDN/tree/master/SDNExpress).
3. Подготовьте другой виртуальной Машины с помощью консоли развертывания. Эта виртуальная машина должна иметь возможность доступа к узлы SDN. Кроме того виртуальная машина должна иметь установлено средство средства удаленного администрирования сервера Hyper-V.
4. Скопируйте все, что вы скачали ранее для SDN Express консоль развертывания виртуальной Машины. И поделиться **SDNExpress** папки. Убедитесь, что каждый узел имеет доступ к **SDNExpress** общую папку, в том случае, как определено в строке файла конфигурации, 8:
```
    \\$env:Computername\SDNExpress
```
5. Скопируйте виртуальный жесткий ДИСК операционной системы, что **образы** папке **SDNExpress** папку на консоль развертывания виртуальной Машины.
6. Измените конфигурацию SDN Express с помощью настройки вашей среды. Завершите следующие два шага, после изменения конфигурации SDN Express, исходя из сведений о вашей среде.
7. Запустите PowerShell с правами администратора для развертывания SDN:

```powershell
    .\SDNExpress.ps1 -ConfigurationDataFile .\your_fabricconfig.PSD1 -verbose 
```

Развертывание займет около 30 – 45 минут.

## <a name="get-started"></a>Начало работы

После развертывания инфраструктуры Hyper-Converged можно управлять с помощью Windows Admin Center.

### <a name="install-windows-admin-center"></a>Установка Windows Admin Center

Если вы еще этого не сделали, скачайте и установите Windows Admin Center. Самый быстрый способ начать работу и запущена — установить его на компьютере Windows 10 и удаленное управление серверами. Это занимает менее чем за пять минут. [Загрузить](https://aka.ms/windowsadmincenter) или [Дополнительные сведения о других вариантах установки](../deploy/install.md).

### <a name="add-hyper-converged-cluster"></a>Добавление Гиперконвергированного кластера

Чтобы добавить кластер Windows Admin Center:

1. Нажмите кнопку **+ добавить** под все подключения.
2. Выберите добавление **подключение к кластеру Hyper-Converged**.
3. Введите имя кластера и, если будет предложено, учетные данные, используемые.
4. Нажмите кнопку **добавить** до конца.

Кластер будет добавляться в список подключений. Щелкните его, чтобы запустить панель мониторинга.

![Добавление подключения гиперконвергированного кластера](../media/manage-hyper-converged/add-hyper-converged-cluster-connection.gif)

### <a name="add-sdn-enabled-hyper-converged-cluster-windows-admin-center-preview"></a>Добавление SDN с поддержкой Гиперконвергированного кластера (Предварительная версия Windows Admin Center)

Последнюю предварительную версию Windows Admin Center поддерживает Hyper-Converged инфраструктуры программно-Конфигурируемую сеть управления. Добавляя URI REST сетевого контроллера для подключения к Гиперконвергентного кластера, можно использовать гиперконвергентное диспетчер кластеров для управления ресурсами SDN и мониторинга инфраструктуры SDN.

1. Нажмите кнопку **+ добавить** под все подключения.
2. Выберите добавление **подключение к кластеру Hyper-Converged**.
3. Введите имя кластера и, если будет предложено, учетные данные, используемые.
4. Проверьте **настройку сетевого контроллера** для продолжения.
5. Введите **URI сетевого контроллера** и нажмите кнопку **Validate**.
6. Нажмите кнопку **добавить** до конца.

Кластер будет добавляться в список подключений. Щелкните его, чтобы запустить панель мониторинга.

![Добавление подключения SDN с поддержкой гиперконвергированного кластера](../media/manage-hyper-converged/add-snd-enabled-hci-connection.png)

> [!Important]
> В настоящее время среды SDN с проверкой подлинности Kerberos для Северный связи не поддерживаются.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="are-there-differences-between-managing-windows-server-2016-and-windows-server-2019-insider-preview"></a>Существуют ли различия между управлением Windows Server 2016 и Windows Server 2019 Insider Preview?

Да. Windows Admin Center для инфраструктуры Hyper-Converged получает частые обновления, которые повышают производительность Windows Server 2016 и Windows Server 2019 Insider Preview. Тем не менее некоторые новые возможности доступны только для Insider Preview — например, переключатель для дедупликации и сжатия.

### <a name="can-i-use-windows-admin-center-to-manage-storage-spaces-direct-for-other-use-cases-not-hyper-converged-such-as-converged-scale-out-file-server-sofs-or-microsoft-sql-server"></a>Можно ли использовать Windows Admin Center для управления дисковыми пространствами для других вариантов использования (не гиперконвергентной), например со схождением масштабируемого файлового сервера (SoFS) или Microsoft SQL Server?

Windows Admin Center для инфраструктуры Hyper-Converged не поддерживает управление или параметров мониторинга специально для других вариантов использования дисковых — например, нельзя создавать общие папки. Тем не менее панели мониторинга и основные функции, такие как создание томов или замена дисков, работают для любых дисковых кластера.

### <a name="whats-the-difference-between-a-failover-cluster-and-a-hyper-converged-cluster"></a>Какова разница между отказоустойчивый кластер и кластер Hyper-Converged?

Как правило термин «гиперконвергентной» относится к под управлением Hyper-V и дисковыми пространствами на том же кластеризованные серверы виртуализации ресурсов вычислений и хранения. В контексте Windows Admin Center, если щелкнуть **+ добавить** из списка подключений, можно выбрать между добавлением **подключение кластера отработки отказа** или **Hyper-Converged кластера подключение**:

- **Подключение кластера отработки отказа** является преемником классическое приложение Диспетчер отказоустойчивости кластеров. Предоставляет возможности управления знакомы, общего назначения для любых кластера, которые поддерживают любой рабочей нагрузки, включая Microsoft SQL Server. Это, предназначенные для Windows Server 2012 и более поздних версий.

- **Подключение к кластеру Hyper-Converged** новый интерфейс разработанных специально для дисковых и Hyper-V. В его состав входит информационная панель с диаграммами и оповещениями для мониторинга. Эта возможность доступна для Windows Server 2016 и предварительной версии сборок Windows Server 2019.

### <a name="why-do-i-need-the-latest-cumulative-update-for-windows-server-2016"></a>Почему нужно устанавливать последний накопительный пакет обновления для Windows Server 2016?

Windows Admin Center для инфраструктуры Hyper-Converged зависит от управления, которые разработали API-интерфейсы с момента выпуска Windows Server 2016. Эти API-интерфейсы добавляются в [2018-05 накопительный пакет обновления для Windows Server 2016 (KB4103723)](https://support.microsoft.com/help/4103723/windows-10-update-kb4103723), которые доступны начиная с 8 мая 2018 г.

### <a name="how-much-does-it-cost-to-use-windows-admin-center"></a>Сколько стоит использование Windows Admin Center?

Пользоваться Windows Admin Center можно бесплатно в рамках оплаченной лицензии на Windows.

Вы можете использовать Windows Admin Center (доступно в качестве отдельной загрузки) с действительных лицензий Windows Server или Windows 10 без дополнительных затрат — она предоставляется по лицензии Windows дополнительное лицензионное соглашение.

### <a name="does-windows-admin-center-require-system-center"></a>Требуется ли для работы с Windows Admin Center пакет продуктов System Center?

Нет.

### <a name="does-it-require-an-internet-connection"></a>Требуется подключение к Интернету?

Нет.

Несмотря на то, что Windows Admin Center предлагает мощный и удобный интегрируются с облако Microsoft Azure, управление ядром и возможности мониторинга для инфраструктуры Hyper-Converged является полностью в локальной. Его можно установить и использовать без подключения к Интернету.

### <a name="create-a-new-virtual-machine"></a>Создание новой виртуальной машины

1. Нажмите кнопку **виртуальных машин** средство из левой части области навигации.
2. В верхней части инструмента виртуальные машины, выберите **инвентаризации** , а затем нажмите кнопку **New** для создания новой виртуальной машины.
3. Введите имя виртуальной машины и выберите между виртуальными машинами поколения 1 и 2.
4. С ним можно выбрать узел изначально создать виртуальную машину на или использовать рекомендуемый узел.
5. Выберите путь для файлов виртуальной машины. Выберите том из раскрывающегося списка или нажмите кнопку **Обзор** чтобы выбрать папку, используя средство выбора папки. Файлы конфигурации виртуальной машины и файл виртуального жесткого диска будут сохранены в одной папке `\Hyper-V\[virtual machine name]` путь выбранного тома или путь.
6. Выберите число виртуальных процессоров, будь то требуется вложенной виртуализацией, настроить параметры памяти, сетевых адаптеров, виртуальные жесткие диски и выберите, следует ли установить операционную систему из ISO-файла образа или от сети.
7. Нажмите кнопку **создать** для создания виртуальной машины.
8. После того как виртуальная машина создается и отображается в списке виртуальных машин, можно запустить виртуальную машину.
9. После запуска виртуальной машины можно подключиться к консоли виртуальной машины с помощью VMConnect для установки операционной системы. Выберите виртуальную машину из списка, щелкните **дополнительные** > **Connect** загрузить RDP-файл. Откройте RDP-файл в приложении подключение удаленного рабочего стола. Так как это подключается к консоли виртуальной машины, необходимо будет ввести учетные данные администратора на узле Hyper-V.

[Дополнительные сведения об управлении виртуальной машины с помощью Windows Admin Center](manage-virtual-machines.md).

### <a name="pause-and-safely-restart-a-server"></a>Приостановка и перезапуск сервера, безопасно

1. Из **панели мониторинга**выберите **серверы** в области навигации слева или щелкнув **Просмотр СЕРВЕРОВ >** ссылку на плитке в правом нижнем углу панели мониторинга .
2. В верхней, переключитесь из **Сводка** для **инвентаризации** вкладки.
3. Выберите сервер, щелкнув его имя, чтобы открыть **Server** страницу сведений.
4. Нажмите кнопку **приостановить сервер для обслуживания**. В противном случае можно продолжить перемещение виртуальных машин к другим серверам в кластере. Сервер будет иметь состояние, расходуя при этом процесс. При желании вы можете ознакомиться с виртуальным машинам перейдем **виртуальные машины > инвентаризации** страницы, где четко показан их сервер узла в сетке. Если все виртуальные машины были перемещены, состояние сервера будет **приостановлено**.
5. Нажмите кнопку **server управление** для доступа к все средства управления для каждого сервера в Windows Admin Center.
6. Нажмите кнопку **перезапустите**, затем **Да**. Вы будете запускать обратно в список подключений.
7. Вернитесь на **панели мониторинга**, сервер выделяется красным, когда он недоступен.
8. После резервного копирования, еще раз перейдите **Server** и нажмите кнопку **возобновить сервер от обслуживания** присвоено состояние сервера обратно просто вверх. В случае виртуальных машин вернет — вмешательство пользователя не требуется.

### <a name="replace-a-failed-drive"></a>Замена неисправного диска

1. При сбое диска, а в левом верхнем углу появится предупреждение **оповещения** область **панели мониторинга**.
2. Можно также выбрать **дисков** в области навигации слева или щелкните **ДИСКОВ ПРЕДСТАВЛЕНИЕ >** ссылку на плитке в правом нижнем углу для просмотра дисков и посмотрите сами их состояние. В **инвентаризации** вкладке сетки поддерживает сортировку, группирование и поиск по ключевому слову.
3. Из **панели мониторинга**, щелкните оповещение, чтобы просмотреть сведения, такие как расположение физического диска.
4. Дополнительные сведения, нажмите кнопку **Go диск** ярлык для **диск** страницу сведений.
5. Если позволяет оборудование, можно щелкнуть **включить свет/выключить** для управления световой индикатор этого диска.
6. Дисковые пространства автоматически удаляет и из ноутбука вместе с неисправных дисков. Когда это произошло, состояние диска прекращается, и ее строки емкость хранилища является пустой.
7. Удалите неисправный диск и вставить его замены.
8. В **дисков > инвентаризации**, появится новый диск. Вовремя очистит оповещение, тома исправит к нормального состояния и хранения выполнит повторную балансировку на новый диск — вмешательство пользователя не требуется.
