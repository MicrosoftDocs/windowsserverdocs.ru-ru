## <a name="managing-windows-server-machines"></a>Управление компьютерами Windows Server

Можно добавить отдельные серверы под управлением Windows Server 2012 или более поздней версии для Windows Admin Center для управления сервером с полный набор средств, включая сертификаты, устройств, события, процессы, роли и компоненты, обновления, виртуальные машины и больше.

![Экран обзора подключения сервера](../media/manage-servers/server-overview.png)

## <a name="adding-a-server-to-windows-admin-center"></a>Добавление сервера Windows Admin Center

Чтобы добавить сервер Windows Admin Center:

1. Нажмите кнопку **+ добавить** под все подключения.
2. Выберите добавление **соединение с сервером**.
3. Введите имя сервера и, при появлении соответствующего запроса, учетные данные, используемые.
4. Нажмите кнопку **отправить** до конца.

Сервер будет добавлен в список соединений на странице "Обзор". Щелкните его, чтобы подключиться к серверу.

> [!NOTE]
> Можно также добавить [отказоустойчивых кластеров](manage-failover-clusters.md) или [гиперконвергентной кластеров](manage-hyper-converged.md) как отдельное подключение в Windows Admin Center.

## <a name="virtual-switches"></a>Виртуальные коммутаторы

**Виртуальные коммутаторы** позволяет управлять виртуальными коммутаторами Hyper-V на компьютере или сервере.

### <a name="features"></a>Компоненты

Следующие функции поддерживаются в виртуальных коммутаторов:

- Просмотр и поиск виртуальные коммутаторы на сервере
- Создать новый виртуальный коммутатор
- Переименование виртуального коммутатора
- Удалить существующий виртуальный коммутатор
- Изменение свойств виртуального коммутатора
