# <a name="configuring-cluster-accounts-in-active-directory"></a>Настройка учетных записей кластеров в Active Directory


Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 и Windows Server 2008

При создании отказоустойчивого кластера и настроить кластеризованных служб или приложений, мастеров отказоустойчивого кластера в Windows Server, создайте необходимые учетные записи компьютера Active Directory (также называемый объектов-компьютеров) и предоставить им определенные разрешения. Мастера создания учетной записи компьютера для самого кластера (эта учетная запись также называется объектом имени кластера или CNO) и учетной записи компьютера для большинства типов кластеризованных служб и приложений, за исключением на виртуальной машине Hyper-V. Мастеров отказоустойчивого кластера автоматически задаются разрешения для этих учетных записей. Если изменяются разрешения, они должны быть изменен для соответствия требования к кластеру. В этом руководстве описываются эти учетные записи Active Directory и разрешения, базовые сведения о, почему они важны и описываются шаги для настройки и управления учетными записями.
      

## <a name="overview-of-active-directory-accounts-needed-by-a-failover-cluster"></a>Обзор учетных записей Active Directory, необходимые для отказоустойчивого кластера

В этом разделе описывается Active Directory учетных записей компьютеров (также называемый объектов-компьютеров Active Directory), которые важны для отказоустойчивого кластера. Ниже приведены эти учетные записи.

  - **Учетная запись пользователя, используемая для создания кластера.** Это учетная запись пользователя, используемая для запуска мастера создания кластера. Учетная запись важно, так как он предоставляет основу, из которого создается учетная запись компьютера для самого кластера.  
      
  - **Учетная запись имени кластера.** (учетная запись компьютера сам кластер также называется объектом имени кластера или CNO). Эта учетная запись создается автоматически с помощью мастера создания кластеров и имеет имя, совпадающее с именем кластера. Учетная запись имени кластера очень важно, так как через эту учетную запись, другие учетные записи создаются автоматически при настройке новых служб и приложений в кластере. Если удаляется учетная запись имени кластера или разрешения являются позаимствована у, другие учетные записи не могут создаваться как требуется в кластере, пока не будет восстановлен учетная запись имени кластера или восстанавливаются правильные разрешения.  
      
    Например, при создании кластера с именем Cluster1 и затем повторите настройку кластеризованного сервера печати, вызывается PrintServer1 в кластере, Cluster1 учетной записи в Active Directory будет нужно хранить необходимые разрешения, чтобы его можно использовать для создания компьютера Учетная запись называется PrintServer1.  
      
    В контейнере по умолчанию для учетных записей компьютеров в Active Directory создается учетная запись имени кластера. По умолчанию это контейнер «Компьютеры», но администратор домена можно перенаправить его в другой контейнер или подразделение (OU).  
      
  - **Учетная запись компьютера (объект-компьютер) кластеризованной службы или приложения.** Эти учетные записи создаются автоматически с помощью мастера высокой доступности, как часть процесса создания большинства типов кластеризованные службы или приложения, за исключением на виртуальной машине Hyper-V. Учетная запись имени кластера предоставляется необходимые разрешения для управления этим учетным записям.  
      
    Например при наличии кластера с именем Cluster1, и затем создать кластерный файловый сервер, называется FileServer1, мастер высокой доступности создает учетную запись компьютера Active Directory, называется FileServer1. Мастер высокой доступности также предоставляет этой учетной записи Cluster1 необходимые разрешения для управления FileServer1 учетной записи.

## <a name="requirements-related-to-failover-clusters-active-directory-domains-and-accounts"></a>Требования, связанные с отказоустойчивых кластеров, доменов Active Directory и учетных записей

Как описано в предыдущих трех разделах, определенные требования должны быть удовлетворены кластеризованные службы и приложения могут быть успешно настроены в отказоустойчивом кластере. Самый простой требования относятся к расположение узлов кластера (в пределах одного домена) и уровень разрешений учетной записи пользователя, который устанавливает кластер. Если эти требования выполняются, другие учетные записи, необходимые для работы кластера могут создаваться автоматически с мастеров отказоустойчивого кластера. Ниже приведены сведения об этих основных требований.

  - **узлы:** Все узлы должны находиться в одном домене Active Directory. (Домен не может быть на основе Windows NT 4.0, которая не поддерживает Active Directory.)  
      
  - **Учетная запись пользователя, который устанавливает кластер:** Лицо, устанавливающее кластера необходимо использовать учетную запись со следующими характеристиками:  
      
      - Учетная запись должна быть учетной записью домена. Его не нужно быть учетную запись администратора домена. Он может быть учетная запись пользователя домена, если он соответствует требованиям, в этом списке.  
          
      - Учетная запись должна иметь права администратора на серверах, которые станут узлами кластера. Для предоставления это проще всего создать учетную запись пользователя домена, а затем добавьте эту учетную запись в локальную группу администраторов на всех серверах, которые станут узлами кластера. Дополнительные сведения см. в разделе [действия по настройке учетной записи для пользователя, который устанавливает кластер](#steps-for-configuring-the-account-for-the-person-who-installs-the-cluster)далее в этом руководстве.  
          
      - Необходимо предоставить учетной записи (или группе, в которой находится учетная запись) **создание объектов-компьютеров** и **чтение всех свойств** разрешений в контейнере, который используется для учетных записей компьютеров в домене. Другой вариант — сделать учетную запись, учетную запись администратора домена. Дополнительные сведения см. в разделе [действия по настройке учетной записи для пользователя, который устанавливает кластер](#steps-for-configuring-the-account-for-the-person-who-installs-the-cluster)далее в этом руководстве.  
          
      - Если ваша организация решает предварительно настроить учетную запись имени кластера (учетной записи компьютера с тем же именем, что и кластер), учетная запись имени предварительно подготовленного кластера нужно предоставить разрешение «Полный доступ» к учетной записи пользователя, который устанавливает кластер. Другие важные сведения о предварительной подготовке учетная запись имени кластера, см. в разделе [учетной записи имени действия по предварительной настройке кластера](#steps-for-prestaging-the-cluster-name-account)далее в этом руководстве.  
          

### <a name="planning-ahead-for-password-resets-and-other-account-maintenance"></a>Планирование для сброса пароля и другое обслуживание учетной записи

Администраторы отказоустойчивых кластеров, иногда может потребоваться сбросить пароль учетной записи имени кластера. Это действие требует определенного разрешения, **сброс пароля** разрешение. Таким образом, рекомендуется изменить разрешения учетной записи имени кластера (используя оснастку «Active Directory — пользователи и компьютеры») чтобы дать администраторам кластера **сброс пароля** разрешение для кластера Имя учетной записи. Дополнительные сведения см. в разделе [действия для устранения неполадок пароль с кластером учетной записи имени](#steps_for_troubleshooting_password_problems_with_the_cluster_name_account)далее в этом руководстве.

## <a name="steps-for-configuring-the-account-for-the-person-who-installs-the-cluster"></a>Действия по настройке учетной записи для пользователя, который устанавливает кластер

Учетная запись пользователя, который устанавливает кластер важно, так как он предоставляет основу, из которого создается учетная запись компьютера для самого кластера.

Минимальное членство в группе для выполнения следующей процедуры требуется зависит от того, создание учетной записи домена и назначив ей необходимые разрешения в домене или только размещаете учетной записи (кем) в локальный **Администраторы** группы на серверах, которые будут узлами в отказоустойчивом кластере. Если бывший, членство в **операторы** или **"Администраторы домена"**, или эквивалентной является минимальным требованием для выполнения этой процедуры. Если последний, членство в локальной **Администраторы** группе на серверах, которые будут узлами в отказоустойчивом кластере, либо эквивалентной, является все, что является обязательным. Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [ http://go.microsoft.com/fwlink/?LinkId=83477 ](http://go.microsoft.com/fwlink/?linkid=83477).

#### <a name="to-configure-the-account-for-the-person-who-installs-the-cluster"></a>Настройка учетной записи для пользователя, который устанавливает кластер

1.  Создайте или получите учетную запись домена для пользователя, который устанавливает кластер. Эта учетная запись может быть учетная запись пользователя домена или учетную запись администратора домена (в **"Администраторы домена"** или эквивалентные группы).

2.  Если учетной записи, который был создан или получен на шаге 1 имеет учетную запись пользователя домена или учетные записи администратора домена в домене не включаются автоматически в локальной **Администраторы** группу на компьютерах в домене, добавить Учетная запись в локальную **Администраторы** группы на серверах, которые будут узлами в отказоустойчивом кластере:
    
    1.  Щелкните **Пуск**, затем **Администрирование**и **Диспетчер серверов**.  
          
    2.  В дереве консоли разверните **конфигурации**, разверните **локальные пользователи и группы**, а затем разверните **группы**.  
          
    3.  В центральной области щелкните правой кнопкой мыши **Администраторы**, нажмите кнопку **добавить группу**, а затем нажмите кнопку **добавить**.  
          
    4.  В разделе **введите имена выбираемых объектов**, введите имя учетной записи пользователя, который был создан или получен на шаге 1. При появлении соответствующего запроса, введите имя учетной записи и пароль с достаточными разрешениями для этого действия. Затем нажмите кнопку **ОК**.  
          
    5.  Повторите эти действия на каждом сервере, который будет узлом в отказоустойчивом кластере.  
          
    

> [!IMPORTANT]
> Эти действия необходимо повторить на всех серверах, которые будут узлами в кластере. 
<br>


3.  Если учетная запись, который был создан или получен на шаге 1 учетную запись администратора домена, пропустите оставшуюся часть этой процедуры. В противном случае присвойте учетной записи **создание объектов-компьютеров** и **чтение всех свойств** разрешений в контейнере, который используется для учетных записей компьютеров в домене:
    
    1.  На контроллере домена нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **Active Directory — пользователи и компьютеры**. При появлении диалогового окна **Контроль учетных записей** подтвердите отображаемое в нем действие и нажмите кнопку **Продолжить**.  
          
    2.  На **представление** меню, убедитесь, что **дополнительные функции** выбран.

## <a name="steps-for-prestaging-the-cluster-name-account"></a>Действия по предварительной настройке учетной записи имени кластера

Тем не менее если это необходимо для предварительной подготовки учетная запись имени кластера из-за требований в вашей организации, используйте следующую процедуру.

#### <a name="to-prestage-a-cluster-name-account"></a>Предварительная настройка учетной записи имени кластера

1.  Убедитесь, что вы знаете имя, которое кластер будет иметь и имя учетной записи пользователя, который будет использоваться тем, кто создает кластер. (Обратите внимание, что можно использовать эту учетную запись для выполнения этой процедуры).

7.  На **представление** меню, убедитесь, что **дополнительные функции** выбран.
    
    Когда **дополнительные функции** установлен, вы увидите **безопасности** вкладке свойств учетных записей (объекты) в **Active Directory — пользователи и компьютеры**.

9.  На вкладке **Безопасность** нажмите кнопку **Дополнительно**.

10. Нажмите кнопку **добавить**, нажмите кнопку **типы объектов** и убедитесь, что **компьютеров** выбран, а затем нажмите кнопку **ОК**. Затем в разделе **введите имена выбираемых объектов**, введите имя учетной записи компьютера, только что создан и нажмите кнопку **ОК**. Если появится сообщение, о том, что вы собираетесь добавить отключенный объект, нажмите кнопку **ОК**.

11. В **запись разрешения** диалоговом окне найдите **создание объектов-компьютеров** и **чтение всех свойств** разрешения и убедитесь, что **Разрешить** флажок для каждого из них.
    
    ![](media/configure-ad-accounts/Cc731002.f5977c4d-a62e-4b17-81e3-8c19ddca2078(WS.10).gif)

12. Нажмите кнопку **ОК** вернуться к **Active Directory — пользователи и компьютеры** оснастки.

13. Если вы используете ту же учетную запись для выполнения этой процедуры, так как будет использоваться для создания кластера, пропустите оставшиеся шаги. В противном случае необходимо настроить разрешения, чтобы учетная запись пользователя, который будет использоваться для создания кластера имеет полный доступ, только что созданной учетной записи компьютера:
    
    1.  На **представление** меню, убедитесь, что **дополнительные функции** выбран.  
          
    2.  Щелкните правой кнопкой мыши учетную запись компьютера, только что создан и нажмите кнопку **свойства**.  
          
    3.  На вкладке **Безопасность** нажмите кнопку **Добавить**. При появлении диалогового окна **Контроль учетных записей** подтвердите отображаемое в нем действие и нажмите кнопку **Продолжить**.  
          
    4.  Используйте **Выбор пользователей, компьютеров или групп** диалоговое окно, чтобы указать учетную запись пользователя, который будет использоваться при создании кластера. Затем нажмите кнопку **ОК**.  
          
    5.  Убедитесь, что выбран учетной записи пользователя, который вы только что добавили, а затем, в поле **полный контроль**выберите **Разрешить** "флажок".

Если требуется учетная запись будет удалена, или изменяются необходимые разрешения, могут возникнуть неполадки. В следующих подразделах приводятся действия по устранению этих неполадок.

### <a name="steps-for-troubleshooting-password-problems-with-the-cluster-name-account"></a>Действия для устранения неполадок пароль с помощью учетной записи имени кластера

Эта процедура используется в том случае, если сообщение о событии о объектов-компьютеров или удостоверение кластера, который содержит следующий текст. Обратите внимание, что этот текст в сообщении о событии, не в начале сообщения о событии:

`Logon failure: unknown user name or bad password.`

Сообщения о событиях, которые помещаются предыдущего описания указывают, что пароль для учетной записи имени кластера и соответствующий пароль кластерное программное обеспечение больше не совпадают.

Убедитесь, что администраторы кластера необходимые разрешения для выполнения следующей процедуры, при необходимости, узнать о планирование вперед для сброса пароля и другое обслуживание учетной записи, ранее в этом руководстве.

Чтобы выполнить эту процедуру, вы должны быть членом локальной группы **Администраторы** или аналогичной. Кроме того, необходимо предоставить учетной записи **сброс пароля** разрешение для учетной записи имени кластера (если ваша учетная запись является **"Администраторы домена"** учетной записи или является Создатель-владелец учетной записи имени кластера). Учетная запись, используемую установившего кластера можно использовать для выполнения этой процедуры. Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [ http://go.microsoft.com/fwlink/?LinkId=83477 ](http://go.microsoft.com/fwlink/?linkid=83477).

#### <a name="to-troubleshoot-password-problems-with-the-cluster-name-account"></a>Для устранения неполадок пароль учетной записи имени кластера

1.  Чтобы открыть оснастку отказоустойчивого кластера, нажмите кнопку **Пуск**, выберите **Администрирование**, а затем щелкните **Управление отказоустойчивого кластера**. (Если **контроль учетных записей пользователей** откроется диалоговое окно, убедитесь, что в нем действие и нажмите кнопку **Продолжить**.)

2.  Если кластер, который вы хотите настроить, не отображается в оснастке управления отказоустойчивым кластером, в дереве консоли щелкните правой кнопкой мыши **Диспетчер отказоустойчивости кластеров**, щелкните **Управление кластером** и выберите или укажите нужный кластер.

3.  В центральной области разверните **основные ресурсы кластера**.

4.  В разделе **имя кластера**, щелкните правой кнопкой мыши **имя** элемента, выберите пункт **дополнительные действия**, а затем нажмите кнопку **восстановления объекта Active Directory**.

### <a name="steps-for-troubleshooting-problems-caused-by-changes-in-cluster-related-active-directory-accounts"></a>Действия для устранения неполадок, из-за изменений в учетные записи Active Directory, относящиеся к кластеру

Если удаляется учетная запись имени кластера или разрешения являются позаимствована у, проблем во время при попытке настройки новой кластеризованной службы или приложения. Устранение неполадок, где, это может стать причиной, используйте Active Directory-пользователи и компьютеры – – оснастка для просмотра или изменения кластера имя учетной записи и другие связанные учетные записи. Сведения о событиях, которые были зарегистрированы (событие 1193 1194, 1206 или 1207), см. в случае проблемы такого типа [ http://go.microsoft.com/fwlink/?LinkId=118271 ](http://go.microsoft.com/fwlink/?linkid=118271).

Необходимым минимальным требованием для выполнения этой процедуры является членство в группе **Администраторы домена** или эквивалентной. Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [ http://go.microsoft.com/fwlink/?LinkId=83477 ](http://go.microsoft.com/fwlink/?linkid=83477).

#### <a name="to-troubleshoot-problems-caused-by-changes-in-cluster-related-active-directory-accounts"></a>Для устранения неполадок, из-за изменений в учетные записи Active Directory, относящиеся к кластеру

1.  На контроллере домена нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **Active Directory — пользователи и компьютеры**. При появлении диалогового окна **Контроль учетных записей** подтвердите отображаемое в нем действие и нажмите кнопку **Продолжить**.

2.  Разверните узел по умолчанию **компьютеров** контейнер или папку, в котором находится учетная запись имени кластера (учетная запись компьютера для кластера). **Компьютеры** находится в <b>Active Directory — пользователи и компьютеры /</b><i>узел домена</i><b>/Computers</b>.

3.  Изучите значок для учетной записи имени кластера. Она не должна иметь стрелку вниз на нем, то есть, учетная запись не должна отключаться. Если она отключена, щелкните его правой кнопкой мыши и найдите команду **включить учетную запись**. Если команда отображается, щелкните его.

4.  На **представление** меню, убедитесь, что **дополнительные функции** выбран.
    
    Когда **дополнительные функции** установлен, вы увидите **безопасности** вкладке свойств учетных записей (объекты) в **Active Directory — пользователи и компьютеры**.

5.  Щелкните правой кнопкой мыши по умолчанию **компьютеров** контейнер или папку, в котором находится учетная запись имени кластера.

6.  Выберите пункт **Свойства**.

7.  На вкладке **Безопасность** нажмите кнопку **Дополнительно**.

8.  В списке учетных записей с разрешениями, щелкните учетную запись имени кластера и нажмите кнопку **изменить**.
    

> [!NOTE]
> Если учетная запись имени кластера не перечислен, щелкните <STRONG>добавить</STRONG> и добавьте его в список. 
<br>


9.  Учетная запись имени кластера (также называется объектом имени кластера или CNO), убедитесь, что **Разрешить** выбран для **создание объектов-компьютеров** и **чтение всех свойств** разрешения.
    
    ![](media/configure-ad-accounts/Cc731002.f5977c4d-a62e-4b17-81e3-8c19ddca2078(WS.10).gif)

10. Нажмите кнопку **ОК** вернуться к **Active Directory — пользователи и компьютеры** оснастки.

11. Политики домена проверки (консультаций с администратором домена, если применимо) связанных с созданием учетных записей компьютеров (объектов). Убедитесь, что учетная запись имени кластера можно создать учетную запись компьютера, каждый раз, настройкой кластеризованной службы или приложения. Например, если администратор домена настроил параметры, которые вызывают все новые учетные записи компьютеров в специализированных контейнера, а не значение по умолчанию **компьютеров** контейнера, убедитесь, что эти параметры позволяют Учетная запись имени кластера также создание новых учетных записей компьютеров в этом контейнере.

12. Разверните узел по умолчанию **компьютеров** или контейнер, в котором находится учетная запись компьютера для одной из кластеризованных служб и приложений.

13. Щелкните правой кнопкой мыши учетную запись компьютера для одной из кластеризованных служб или приложений, а затем нажмите кнопку **свойства**.

14. На **безопасности** вкладке, убедитесь, что учетная запись имени кластера указан среди всех учетных записей, имеющих разрешения и выберите его. Убедитесь, что учетная запись имени кластера **полный контроль** разрешение ( **Разрешить** флажок). Если этого не произошло, добавьте учетную запись имени кластера в список и присвойте ему **полный контроль** разрешение.
    
    ![](media/configure-ad-accounts/Cc731002.2e614376-87a6-453a-81ba-90ff7ebc3fa2(WS.10).gif)

15. Повторите шаги 13 – 14 для каждой кластерной службы и приложения, настроенного в кластере.

16. Проверьте, что домена для создания объектов-компьютеров (по умолчанию — 10) не достигнута квота (консультаций с администратором домена если применимо). Если предыдущие элементы в этой процедуре все будут проверены и исправлены, и достигнута квота, рассмотрите возможность увеличения квоты. Изменение квоты.
    
    1.  Откройте командную строку от имени администратора и выполните **ADSIEdit.msc**.  
          
    2.  Щелкните правой кнопкой мыши **"Редактирование ADSI"**, нажмите кнопку **подключиться к**, а затем нажмите кнопку **ОК**. **Контекст именования по умолчанию** добавляется в дерево консоли.  
          
    3.  Дважды щелкните **контекст именования по умолчанию**, щелкните правой кнопкой мыши объект домена, под ним и нажмите кнопку **свойства**.  
          
    4.  Прокрутите страницу до **ms-DS-MachineAccountQuota**, выберите его, нажмите кнопку **изменить**, измените значение и нажмите кнопку **ОК**.
