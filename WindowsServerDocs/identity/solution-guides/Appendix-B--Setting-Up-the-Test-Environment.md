# <a name="appendix-b-setting-up-the-test-environment"></a>Приложение б. Настройка тестовой среды

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В этом разделе описано, как создать лабораторию для тестирования динамического контроля доступа. Данные инструкции предназначены для последовательного применения, так как многие компоненты, имеющие зависимости.  
  
## <a name="prerequisites"></a>Предварительные требования  
**Требования к оборудованию и программному обеспечению**  
  
Требования к настройке лаборатории тестирования:  
  
-   Хост-сервер под управлением Windows Server 2008 R2 с SP1 и Hyper-V  
  
-   Копию Windows Server 2012 ISO  
  
-   Копию ISO-ОБРАЗА Windows 8  
  
-   Microsoft Office 2010  
  
-   Сервер под управлением Microsoft Exchange Server 2003 или более поздней версии  
  
Необходимо создать следующие виртуальные машины для тестирования динамического контроля доступа:  
  
-   DC1 (контроллер домена)  
  
-   DC2 (контроллер домена)  
  
-   File1 (файловый сервер и службы управления правами Active Directory)  
  
-   SRV1 (POP3- и SMTP-сервер)  
  
-   CLIENT1 (клиентский компьютер с Microsoft Outlook)  
  
Для виртуальных машин должны использоваться следующие пароли:  
  
-   BUILTIN\Administrator:pass@word1  
  
-   Contoso\Administrator:pass@word1  
  
-   Все прочие учетные записи:pass@word1  
  
## <a name="build-the-test-lab-virtual-machines"></a>Создание виртуальных машин лаборатории тестирования  
  
### <a name="install-the-hyper-v-role"></a>Установка роли Hyper-V  
Необходимо установить роль Hyper-V на компьютере под управлением Windows Server 2008 R2 с пакетом обновления 1.  
  
##### <a name="to-install-the-hyper-v-role"></a>Чтобы установить роль Hyper-V  
  
1.  Нажмите кнопку **запустить**и нажмите кнопку диспетчера сервера.  
  
2.  В области "Сводка по ролям" окна диспетчера серверов щелкните **добавления ролей**.  
  
3.  На **Выбор ролей сервера** щелкните **Hyper-V**.  
  
4.  На **Создание виртуальных сетей** щелкните один или несколько сетевых адаптеров, если вы хотите, чтобы их сетевое подключение для виртуальных машин.  
  
5.  На **Подтвердите выбранные элементы** щелкните **установить**.  
  
6.  Для завершения установки необходимо перезагрузить компьютер. Нажмите кнопку **закрыть** завершения работы мастера, а затем нажмите кнопку **Да** для перезагрузки компьютера.  
  
7.  После перезагрузки компьютера войдите с использованием той же учетной записью, которая использовалась для установки роли. После завершения установки мастером нажмите кнопку **закрыть** для завершения работы мастера.  
  
### <a name="create-an-internal-virtual-network"></a>Создание внутренней виртуальной сети  
Теперь необходимо создать внутреннюю виртуальную сеть с именем ID_AD_Network.  
  
##### <a name="to-create-a-virtual-network"></a>Чтобы создать виртуальную сеть  
  
1.  Откройте диспетчер Hyper-V.  
  
2.  Из **действия** меню, нажмите кнопку **диспетчер виртуальных сетей**.  
  
3.  В разделе **Создание виртуальной сети**выберите **внутренняя**.  
  
4.  Нажмите кнопку **добавить**. **Новая виртуальная сеть** откроется страница.  
  
5.  Тип **ID_AD_Network** как имя для новой сети. Просмотрите другие свойства и измените их при необходимости.  
  
6.  Нажмите кнопку **ОК** Чтобы создать виртуальную сеть и закрыть диспетчер виртуальных сетей либо нажмите **применить** Чтобы создать виртуальную сеть и продолжить работу с диспетчером виртуальных сетей.  
  
### <a name="BKMK_Build"></a>Создание контроллера домена  
Создание виртуальной машины для использования в качестве контроллера домена (DC1). Установите виртуальную машину с помощью Windows Server 2012 ISO и имени укажите DC1.  
  
##### <a name="to-install-active-directory-domain-services"></a>Для установки доменных служб Active Directory  
  
1.  Подключите виртуальную машину к сети id_ad_network. Войдите в контроллер домена DC1 как администратором с паролем ** pass@word1 **.  
  
2.  В диспетчере серверов щелкните **управление**, а затем нажмите кнопку **Добавить роли и компоненты**.  
  
3.  На **перед началом** щелкните **Далее**.  
  
4.  На **Выбор типа установки** щелкните **Установка на основе ролей или компонентов**, а затем нажмите кнопку **Далее**.  
  
5.  На **Выбор целевого сервера** щелкните **Далее**.  
  
6.  На **Выбор ролей сервера** щелкните **доменных служб Active Directory**. В **мастера добавления ролей и компонентов** диалоговом нажмите кнопку **добавить компоненты**, а затем нажмите кнопку **Далее**.  
  
7.  На **выберите компоненты** щелкните **Далее**.  
  
8.  На **доменных служб Active Directory** страницы, просмотрите сведения и нажмите кнопку **Далее**.  
  
9. На **подтверждение выбранных элементов для установки** щелкните **установить**. Строка хода выполнения установки компонентов на странице результатов означает, что выполняется установка роли.  
  
10. На **результатов** убедитесь, что установка успешно завершена и нажмите кнопку **закрыть**. В диспетчере серверов щелкните значок предупреждения с восклицательным знаком в правом верхнем углу экрана, рядом с пунктом **управление**. В списке задач выберите **повысить роль этого сервера до уровня контроллера домена** ссылку.  
  
11. На **конфигурации развертывания** щелкните **добавить новый лес**, введите имя корневого домена, **contoso.com**, а затем нажмите кнопку **Далее**.  
  
12. На **параметры контроллера домена** выберите режимы работы домена и леса как Windows Server 2012, укажите пароль DSRM ** pass@word1 **, а затем нажмите кнопку **Далее**.  
  
13. На **параметры DNS** щелкните **Далее**.  
  
14. На **Дополнительные параметры** щелкните **Далее**.  
  
15. На **пути** страницы введите расположения для базы данных Active Directory, файлы журналов и папки SYSVOL (либо примите расположение по умолчанию), а затем нажмите кнопку **Далее**.  
  
16. На **просмотреть параметры** страницы, проверьте выбранные параметры и нажмите кнопку **Далее**.  
  
17. На **проверка предварительных требований** убедитесь, что проверка предварительных требований завершена и нажмите кнопку **установить**.  
  
18. На **результатов** , убедитесь, что сервер успешно настроен в качестве контроллера домена и щелкните **закрыть**.  
  
19. Перезапустите сервер, чтобы завершить установку AD DS. (По умолчанию это происходит автоматически.)

#### <a name="install-the-microsoft-office-filter-packs-on-the-file-server"></a>Установите пакеты фильтров Microsoft Office на файловом сервере  
Пакеты фильтров Microsoft Office следует установить в Windows Server 2012, чтобы включить фильтры IFilter для большего числа файлов Office, чем предоставлено по умолчанию.  Windows Server 2012 не фильтры IFilter для файлов Microsoft Office, устанавливается по умолчанию, и инфраструктура классификации файлов использует фильтры IFilter для анализа контента.  
  
Загрузке и установке фильтров IFilter см. в разделе [пакеты фильтров Microsoft Office 2010](https://go.microsoft.com/fwlink/?LinkID=234122).  
  
#### <a name="configure-email-notifications-on-file1"></a>Настройка уведомлений по электронной почте на сервере FILE1  
При создании квот и фильтров блокировки файлов, вы можете отправлять уведомления по электронной почте для пользователей, если приближается к пределу квоты или после попытки сохранить файлы, которые были заблокированы. Если требуется, чтобы периодически уведомлять определенных администраторов о квоты и блокировки события файлов, можно настроить одного или нескольких получателей. Для отправки этих уведомлений необходимо указать SMTP-сервер, будет использоваться для переадресации сообщений электронной почты.  
  
###### <a name="to-configure-email-options-in-file-server-resource-manager"></a>Чтобы настроить параметры электронной почты в диспетчер ресурсов файлового сервера  
  
1.  Откройте диспетчер ресурсов файлового сервера. Чтобы открыть диспетчер ресурсов файлового сервера, нажмите кнопку **запустить**, тип **диспетчера ресурсов файлового сервера**, а затем нажмите кнопку **диспетчера ресурсов файлового сервера**.  
  
2.  В интерфейсе диспетчера ресурсов файлового сервера, щелкните правой кнопкой мыши **диспетчера ресурсов файлового сервера**, а затем нажмите кнопку **настроить параметры**. **Параметры диспетчера ресурсов файлового сервера** откроется диалоговое окно.  
  
3.  На **уведомлений по электронной почте** в поле имя SMTP-сервера или IP-адрес, введите имя узла или IP-адрес SMTP-сервера, который будет пересылать уведомления по электронной почте.  
  
4.  Если вы хотите, чтобы периодически уведомлять определенных администраторов о квоты событиях и блокировки файлов, в разделе **Администраторы-получатели по умолчанию**, введите адрес электронной почты, например fileadmin@contoso.com. Используйте формат account@domain, а несколько учетных записей разделяйте точкой с запятой.

#### <a name="BKMK_CS1"></a>Установка служб управления правами Active Directory  
Добавьте прав службы управления Active Directory (AD RMS) и все необходимые компоненты с помощью диспетчера серверов. Выберите все параметры по умолчанию.

###### <a name="to-install-and-configure-an-ad-rms-cluster-in-windows-server-2012-using-windows-powershell"></a>Установка и настройка кластера AD RMS в Windows Server 2012 с помощью Windows PowerShell  
  
1.  Войдите как пользователь CONTOSO\Administrator и паролем: ** pass@word1 **.  
  
    > [!IMPORTANT]  
    > Чтобы установить роль сервера AD RMS установщик будет учетной записи (в данном случае — CONTOSO\Administrator) необходимо предоставить членство в локальной группе администраторов на компьютере сервера, где AD RMS должна быть установлена, а также членство в группе "Администраторы предприятия" в Active Directory.  
  
2.  На рабочем столе сервера щелкните правой кнопкой мыши значок Windows PowerShell на панели задач и выберите **Запуск от имени администратора** , чтобы открыть командную строку Windows PowerShell с правами администратора.  
  
3.  Чтобы использовать командлеты диспетчера серверов для установки роли сервера AD RMS, введите следующую команду:  
  
    ```  
    Add-WindowsFeature ADRMS '"IncludeAllSubFeature '"IncludeManagementTools  
    ```  
  
4.  Создайте диск Windows PowerShell для представления, устанавливаемый сервер AD RMS.  
  
    Например чтобы создать диск Windows PowerShell с именем RC для установки и настройки первого сервера в корневом кластере AD RMS, введите следующую команду:  
  
    ```  
    Import-Module ADRMS  
    New-PSDrive -PSProvider ADRMSInstall -Name RC -Root RootCluster  
    ```  
  
5.  Настройте свойства объектов в пространстве имен диска, представляющие необходимые параметры конфигурации.  
  
    Например чтобы настроить учетную запись службы AD RMS, в командной строке Windows PowerShell, введите следующую команду:  
  
    ```  
    $svcacct = Get-Credential  
    ```  
  
    Когда откроется диалоговое окно безопасности Windows, введите имя пользователя домена учетной записи службы AD RMS CONTOSO\RMS и назначенный пароль.  
  
    Затем Чтобы назначить учетную запись службы AD RMS параметрам кластера AD RMS, введите:  
  
    ```  
    Set-ItemProperty -Path RC:\ -Name ServiceAccount -Value $svcacct  
    ```  
  
    Затем Чтобы настроить сервер AD RMS для использования внутренней базы данных Windows, в командной строке Windows PowerShell введите:  
  
    ```  
    Set-ItemProperty -Path RC:\ClusterDatabase -Name UseWindowsInternalDatabase -Value $true  
    ```  
  
    Затем чтобы надежно сохранить пароль ключа кластера в переменной, в командной строке Windows PowerShell введите:  
  
    ```  
    $password = Read-Host -AsSecureString -Prompt "Password:"  
    ```  
  
    Введите пароль ключа кластера и нажмите клавишу ВВОД.  
  
    Чтобы назначить пароль ключа для установки служб AD RMS, в командной строке Windows PowerShell введите:  
  
    ```  
    Set-ItemProperty -Path RC:\ClusterKey -Name CentrallyManagedPassword -Value $password  
    ```  
  
    Затем Чтобы настроить адрес кластера AD RMS, в командной строке Windows PowerShell, введите следующую команду:  
  
    ```  
    Set-ItemProperty -Path RC:\ -Name ClusterURL -Value "http://file1.contoso.com:80"  
    ```  
  
    Затем Чтобы назначить имя SLC для установки служб AD RMS, в командной строке Windows PowerShell введите:  
  
    ```  
    Set-ItemProperty -Path RC:\ -Name SLCName -Value "FILE1"  
    ```  
  
    Затем Чтобы задать точку подключения службы (SCP) для кластера AD RMS, в командной строке Windows PowerShell, введите следующую команду:  
  
    ```  
    Set-ItemProperty -Path RC:\ -Name RegisterSCP -Value $true  
    ```  
  
6.  Запустите **Install-ADRMS** командлета. Кроме установки роли сервера AD RMS и настройки сервера, этот командлет также устанавливает другие компоненты, если они требуются для служб AD RMS.  
  
    Например чтобы изменить диск Windows PowerShell, с именем RC и установить и настроить службы AD RMS, введите следующую команду:  
  
    ```  
    Set-Location RC:\  
    Install-ADRMS -Path.  
    ```  
  
    Введите «Y» по запросу командлета убедитесь, что требуется для запуска установки.  
  
7.  Выйдите из учетной записи CONTOSO\Administrator и войдите как CONTOSO\RMS с использованием заданного пароля («pass@word1»).  
  
    > [!IMPORTANT]  
    > Для управления сервером AD RMS будет учетной записи вы вошли в систему и использовать для управления сервером (в данном случае — CONTOSO\RMS), необходимо предоставить членство в локальной группе администраторов на компьютере сервера AD RMS, а также членство в группе "Администраторы предприятия" в Active Directory.  
  
8.  На рабочем столе сервера щелкните правой кнопкой мыши значок Windows PowerShell на панели задач и выберите **Запуск от имени администратора** , чтобы открыть командную строку Windows PowerShell с правами администратора.  
  
9. Создайте диск Windows PowerShell для представления сервера AD RMS, который вы настраиваете.  
  
    Например чтобы создать диск Windows PowerShell с именем RC для настройки корневого кластера AD RMS, введите следующую команду:  
  
    ```  
    Import-Module ADRMSAdmin `  
    New-PSDrive -PSProvider ADRMSAdmin -Name RC -Root http://localhost -Force -Scope Global  
    ```  
  
10. Чтобы создать шаблон прав для финансового администратора Contoso и назначить ему права полного доступа в вашей установке AD RMS, в командной строке Windows PowerShell введите:  
  
    ```  
    New-Item -Path RC:\RightsPolicyTemplate '"LocaleName en-us -DisplayName "Contoso Finance Admin Only" -Description "Contoso Finance Admin Only" -UserGroup financeadmin@contoso.com  -Right ('FullControl')  
    ```  
  
11. Чтобы убедиться, что вы можете видеть новый шаблон прав для финансового администратора Contoso, в командной строке Windows PowerShell:  
  
    ```  
    Get-FsrmRmsTemplate  
    ```  
  
    Просмотрите выходные данные командлета и убедитесь, что шаблон RMS, созданный на предыдущем шаге присутствует.

Дополнительные сведения см. в разделе [Настройка DNS-клиента для статического IP-адреса](https://go.microsoft.com/fwlink/?LinkId=150952).  
  
### <a name="BKMK_2.2"></a>Настройка нового леса с именем adatum.com  
  
##### <a name="to-install-active-directory-domain-services"></a>Для установки доменных служб Active Directory

6.  На **Выбор ролей сервера** щелкните **доменных служб Active Directory**. В **мастера добавления ролей и компонентов** диалоговом нажмите кнопку **добавить компоненты**, а затем нажмите кнопку **Далее**.

11. На **конфигурации развертывания** щелкните **добавить новый лес**, введите имя корневого домена, **adatum.com**, а затем нажмите кнопку **Далее**.  
  
12. На **параметры контроллера домена** выберите режимы работы домена и леса как Windows Server 2012, укажите пароль DSRM ** pass@word1 **, а затем нажмите кнопку **Далее**.  
  
13. На **параметры DNS** щелкните **Далее**.  
  
14. На **Дополнительные параметры** щелкните **Далее**.  
  
15. На **пути** страницы введите расположения для базы данных Active Directory, файлы журналов и папки SYSVOL (либо примите расположение по умолчанию), а затем нажмите кнопку **Далее**.  
  
16. На **просмотреть параметры** страницы, проверьте выбранные параметры и нажмите кнопку **Далее**.  
  
17. На **проверка предварительных требований** убедитесь, что проверка предварительных требований завершена и нажмите кнопку **установить**.  
  
18. На **результатов** , убедитесь, что сервер успешно настроен в качестве контроллера домена и щелкните **закрыть**.  
  
19. Перезапустите сервер, чтобы завершить установку AD DS. (По умолчанию это происходит автоматически.)  
  
> [!IMPORTANT]  
> Чтобы убедиться, что сеть настроена правильно, после настройки лесов, необходимо выполнить следующие действия.  
>   
> -   Войдите в домен adatum.com как adatum\administrator. Откройте окно командной строки введите **nslookup contoso.com**, и нажмите клавишу ВВОД.  
> -   Войдите домен contoso.com как contoso\administrator. Откройте окно командной строки введите **nslookup adatum.com**, и нажмите клавишу ВВОД.  
>   
> Если эти команды выполняются без ошибок, леса могут взаимодействовать друг с другом. Дополнительные сведения об ошибках nslookup см. раздел об устранении неполадок в разделе [использование NSlookup.exe](https://support.microsoft.com/kb/200525)

### <a name="BKMK_2.4"></a>Создание дополнительных пользователей в лесу Adatum  
Создайте пользователя Jeff Low с паролем ** pass@word1 **и задайте для атрибута компании значение **Adatum**.  
  
##### <a name="to-create-a-user-with-the-company-attribute"></a>Создание пользователя с атрибутом компании  
  
1.  В Windows PowerShell откройте командную строку с повышенными привилегиями и вставьте следующий код:  
  
    ```  
    New-ADUser `  
    -SamAccountName jlow `  
    -Name "Jeff Low" `  
    -UserPrincipalName jlow@adatum.com `  
    -AccountPassword (ConvertTo-SecureString `  
    -AsPlainText "pass@word1" -Force) `  
    -Enabled $true `  
    -PasswordNeverExpires $true `  
    -Path 'CN=Users,DC=adatum,DC=com' `  
    -Company Adatum`  
  
    ```  
  
### <a name="BKMK_2.5"></a>Создание типа утверждения компании adataum.com  
  
##### <a name="to-create-a-claim-type-by-using-windows-powershell"></a>Создание типа утверждения с помощью Windows PowerShell  
  
1.  Войдите в домен adatum.com как администратор.  
  
2.  В Windows PowerShell откройте командную строку с повышенными привилегиями и введите следующий код:  
  
    ```  
    New-ADClaimType `  
    -AppliesToClasses:@('user') `  
    -Description:"Company" `  
    -DisplayName:"Company" `  
    -ID:"ad://ext/Company:ContosoAdatum" `  
    -IsSingleValued:$true `  
    -Server:"adatum.com" `  
    -SourceAttribute:Company `  
    -SuggestedValues:@((New-Object Microsoft.ActiveDirectory.Management.ADSuggestedValueEntry("Contoso", "Contoso", "")), (New-Object Microsoft.ActiveDirectory.Management.ADSuggestedValueEntry("Adatum", "Adatum", ""))) `  
  
    ```  
  
### <a name="BKMK_2.55"></a>Включение свойства ресурса "Компания" для contoso.com

4.  Выберите **компании** из **свойства ресурса** , щелкните правой кнопкой мыши и выберите **свойства**. В **предложенные значения** статьи, щелкните **добавить** Чтобы добавить предложенные значения: Contoso и Adatum, а затем нажмите кнопку **ОК** дважды.  
  
5.  Выберите **компании** из **свойства ресурса** , щелкните правой кнопкой мыши и выберите **включить**.  
  
### <a name="BKMK_2.6"></a>Включение динамического контроля доступа для adatum.com  
  
##### <a name="to-enable-dynamic-access-control-for-adatumcom"></a>Включение динамического контроля доступа для adatum.com  
  
1.  Войдите в домен adatum.com как администратор.  
  
2.  Откройте консоль управления групповыми политиками, щелкните **adatum.com**, а затем дважды щелкните **контроллеры домена**.  
  
3.  Щелкните правой кнопкой мыши **политика контроллеров домена по умолчанию**и выберите **изменить**.  
  
4.  В окне редактора управления групповыми политиками дважды щелкните **Конфигурация компьютера**, дважды щелкните **политики**, дважды щелкните **административные шаблоны**, дважды щелкните **системы**, а затем дважды щелкните **KDC**.  
  
5.  Дважды щелкните **поддержки центра распространения КЛЮЧЕЙ для утверждений, комплексной проверки подлинности и защиты Kerberos** и установите флажок рядом с пунктом **включено**. Необходимо включить этот параметр, чтобы использовать централизованные политики доступа.

### <a name="BKMK_2.9"></a>Создание правила централизованного доступа  
  
##### <a name="to-create-a-central-access-rule"></a>Создание правила централизованного доступа  
  
1.  В левой области центра администрирования Active Directory, нажмите кнопку **представление в виде дерева**. В левой области щелкните **динамический контроль доступа**, а затем нажмите кнопку **централизованные правила доступа**.  
  
2.  Щелкните правой кнопкой мыши **централизованные правила доступа**, нажмите кнопку **New**, а затем **централизованное правило доступа**.  
  
3.  В **имя** введите **AdatumEmployeeAccessRule**.  
  
4.  В **разрешения** выберите **использовать следующие разрешения как текущие** , нажмите кнопку **изменить**, а затем нажмите кнопку **добавить**. Нажмите кнопку **выберите субъект** , введите **прошедшие**, а затем нажмите кнопку **ОК**.  
  
5.  В **запись для разрешений** диалоговом нажмите кнопку **добавить условие**и введите следующие условия: [**пользователя**] [**компании**] [**равняется**] [**значение**] [**Adatum**]. Разрешения должны быть **изменение, чтение и выполнение, чтение, запись**.  
  
6.  Нажмите кнопку **ОК**.  
  
7.  Нажмите кнопку **ОК** три раза, чтобы завершить операцию и вернуться в центр администрирования Active Directory.  
  
    ![руководства по решениям](media/Appendix-B--Setting-Up-the-Test-Environment/PowerShellLogoSmall.gif)Windows PowerShell эквивалентные команды ***  
  
    Следующий командлет Windows PowerShell или командлеты выполняют ту же функцию, что и предыдущая процедура. Введите каждый командлет в отдельной строке, несмотря на то, что они могут отображаться по здесь несколько строк словам из-за ограничений форматирования.  
  
    ```  
    New-ADCentralAccessRule `  
    -CurrentAcl:"O:SYG:SYD:AR(A;;FA;;;OW)(A;;FA;;;BA)(A;;FA;;;SY)(XA;;0x1301bf;;;AU;(@USER.ad://ext/Company:ContosoAdatum == `"Adatum`"))" `  
    -Name:"AdatumEmployeeAccessRule" `  
    -ProposedAcl:$null `  
    -ProtectedFromAccidentalDeletion:$true `  
    -Server:"contoso.com" `  
    ```  
  
### <a name="BKMK_2.10"></a>Создание централизованной политики доступа  
  
##### <a name="to-create-a-central-access-policy"></a>Создание централизованной политики доступа

### <a name="BKMK_2.11"></a>Публикация новой политики с помощью групповой политики  
  
##### <a name="to-apply-the-central-access-policy-across-file-servers-through-group-policy"></a>Применение централизованной политики доступа на файловых серверах с помощью групповой политики  
  
1.  На **запустить** введите **Администрирование**и в **поиска** щелкните **параметры**. В **параметры** результатов, нажмите кнопку **Администрирование**. Откройте консоль управления групповыми политиками из **Администрирование** папки.  
  
    > [!TIP]  
    > Если **Показать средства администрирования** отключен, папка средств администрирования и ее содержимое не будет отображаться в **параметры** результаты.  
  
2.  Щелкните правой кнопкой мыши домен contoso.com, нажмите кнопку **создать объект групповой Политики в этом домене и связать его**  
  
3.  Введите описательное имя для объекта групповой Политики, такие как **AdatumAccessGPO**, а затем нажмите кнопку **ОК**.  
  
##### <a name="to-apply-the-central-access-policy-to-the-file-server-through-group-policy"></a>Чтобы применить централизованную политику доступа на файловом сервере с помощью групповой политики  
  
1.  На **запустить** введите **Управление групповой политикой**в **поиска** поле. Откройте **Управление групповой политикой** из папки "Администрирование".  
  
    > [!TIP]  
    > Если **Показать средства администрирования** отключен, папка средств администрирования и ее содержимое не появятся в результатах параметры.  
  
2.  Найдите и выберите Contoso следующим образом: групповой политикой\forest: contoso.com\Domains\contoso.com.  
  
3.  Щелкните правой кнопкой мыши **AdatumAccessGPO** политики и выберите **изменить**.  
  
4.  В редакторе управления групповой политикой щелкните **Конфигурация компьютера**, разверните **политики**, разверните **параметры Windows**, а затем нажмите кнопку **параметры безопасности**.  
  
5.  Разверните **файловой системы**, щелкните правой кнопкой мыши **централизованная политика доступа**, а затем нажмите кнопку **управление централизованными политиками доступа**.  
  
6.  В **Настройка централизованных политик доступа** диалоговом нажмите кнопку **добавить**выберите **только политика доступа Adatum**, а затем нажмите кнопку **ОК**.  
  
7.  Закройте редактор управления групповыми политиками. Теперь вы добавили централизованную политику доступа для групповой политики.

### <a name="BKMK_2.13"></a>Настройка классификации и применение централизованной политики доступа к папке Earnings  
  
##### <a name="to-assign-the-central-access-policy-on-the-file-server"></a>Назначение централизованной политики доступа на файловом сервере

8.  Нажмите кнопку **ОК** чтобы вернуться в проводник.
