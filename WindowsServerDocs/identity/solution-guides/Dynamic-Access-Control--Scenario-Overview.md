# <a name="dynamic-access-control-scenario-overview"></a>Динамического контроля доступа: Обзор сценария

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В Windows Server 2012 можно применить управление данными на файловых серверах контролировать, кто имеет доступ к информации и кто обращался к информации. Динамический контроль доступа, вы сможете:  
  
-   Идентифицировать данные с помощью автоматической и ручной классификации файлов. Например можно снабдить тегами данные на файловых серверах в организации.  
  
-   Управлять доступом к файлам, применяя политики сетевой безопасности с использованием централизованных политик доступа. Например можно определить, кто имеет доступ к информации о состоянии в пределах организации.  
  
-   Аудит доступа к файлам, используя централизованные политики аудита для отчетов и криминалистического анализа соответствия. Например можно определить, кто обращался к самой конфиденциальной информации.  
  
-   Примените защиту служб управления правами (RMS), используя автоматическое шифрование RMS для конфиденциальных документов Microsoft Office. Например можно настроить службы RMS на шифрование всех документов, содержащих сведения о медицинского страхования и учета HIPAA (HIPAA).  
  
Набор компонентов динамического контроля доступа основан на инфраструктуру, которые можно использовать дополнительные партнеры и бизнес-приложений, а компоненты могут обеспечить значительные преимущества организациям, использующим Active Directory. Эта инфраструктура включает в себя:  
  
-   Новый механизм авторизации и аудита для Windows, способный обрабатывать условные выражения и централизованные политики.  
  
-   Поддержка проверки подлинности Kerberos для утверждения пользователей и устройств.  
  
-   Улучшения инфраструктуры классификации файлов (FCI).  
  
-   Поддержка расширяемости RMS, чтобы партнеры могли предоставлять решения, шифрующие файлы сторонних разработчиков.
