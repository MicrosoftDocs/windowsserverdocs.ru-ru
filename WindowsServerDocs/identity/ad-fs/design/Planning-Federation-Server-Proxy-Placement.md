# <a name="planning-federation-server-proxy-placement"></a>Планирование размещения прокси-сервера федерации

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

После сбора всех сведений, который будет использоваться для разработки инфраструктуры \(AD FS\) служб федерации Active Directory и после планирования сервера федерации и стратегии веб-серверов можно запланировать время и место размещения прокси-серверов федерации в новой среде. Сведения в следующих разделах, помогут определить, когда и где разместить федерации прокси-сервера и нужно ли настроить его для учетной записи партнера роль или роли партнера по ресурсам.  
  
-   [Сведения о роли сервера федерации в партнера по учетным записям](Review-the-Role-of-the-Federation-Server-in-the-Account-Partner.md)  
  
-   [Сведения о роли прокси-сервера федерации в организации партнера по ресурсам](Review-the-Role-of-the-Federation-Server-Proxy-in-the-Resource-Partner.md)  
  
-   [Когда следует создавать прокси-сервера федерации](When-to-Create-a-Federation-Server-Proxy.md)  
  
-   [Место размещения прокси-сервера федерации](Where-to-Place-a-Federation-Server-Proxy.md)  
  
-   [Когда следует создавать ферму прокси-сервера федерации](When-to-Create-a-Federation-Server-Proxy-Farm.md)  
  
-   [Требования к сертификатам для прокси-серверов федерации](Certificate-Requirements-for-Federation-Server-Proxies.md)  
  
-   [Требования к разрешению имен для прокси-сервера федерации](Name-Resolution-Requirements-for-Federation-Server-Proxies.md)  
  
> [!NOTE]  
> Несмотря на то, что эти сведения могут помочь при планировании прокси-серверов федерации размещения, они не объясняют, как определить нужное число прокси-серверов и требования к оборудованию прокси-сервера для каждого дизайна служб AD FS.  
  
Примеры того, как разместить прокси-сервера федерации в одной из двух основных AD FS сценариев разработки, см. в разделе [сопоставление Your целей развертывания с разработкой AD FS](Mapping-Your-Deployment-Goals-to-an-AD-FS-Design.md).
