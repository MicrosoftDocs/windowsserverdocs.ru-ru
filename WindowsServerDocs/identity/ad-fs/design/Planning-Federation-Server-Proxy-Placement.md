# <a name="planning-federation-server-proxy-placement"></a>Планирование размещения прокси-серверов федерации

После сбора всех сведений, которые будут использоваться для проектирования инфраструктуры службы федерации Active Directory (AD FS) AD FS, \( \) а также после планирования стратегии сервера федерации и веб-сервера, можно запланировать, когда и где следует размещать прокси сервера федерации в новой структуре. Сведения в следующих разделах помогут определить, когда и где следует поместить прокси-сервер федерации, а также нужно ли настроить его для роли партнера по учетным записям или роли партнера по ресурсам.

-   [Сведения о роли сервера федерации в организации партнера по учетным записям](Review-the-Role-of-the-Federation-Server-in-the-Account-Partner.md)

-   [Сведения о роли прокси-сервера федерации в организации партнера по ресурсам](Review-the-Role-of-the-Federation-Server-Proxy-in-the-Resource-Partner.md)

-   [Когда следует создавать прокси-сервер федерации](When-to-Create-a-Federation-Server-Proxy.md)

-   [Размещение прокси-сервера федерации](Where-to-Place-a-Federation-Server-Proxy.md)

-   [Когда следует создавать ферму прокси-сервера федерации](When-to-Create-a-Federation-Server-Proxy-Farm.md)

-   [Требования к сертификатам для прокси-серверов федерации](Certificate-Requirements-for-Federation-Server-Proxies.md)

-   [Требования к разрешению имен для прокси-серверов федерации](Name-Resolution-Requirements-for-Federation-Server-Proxies.md)
