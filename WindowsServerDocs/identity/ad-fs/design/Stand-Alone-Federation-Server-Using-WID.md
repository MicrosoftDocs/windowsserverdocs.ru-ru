# <a name="stand-alone-federation-server-using-wid"></a>Независимый сервер федерации с использованием WID

Автономный\-сервера только федерации в службах федерации Active Directory \(AD FS\) состоит из одного сервера, на котором размещена служба федерации, настроена для использования внутренней базы данных Windows \(WID\). Эта топология AD FS — для тестовых лабораторных условий. Не рекомендуется для рабочей среды, так как она ограничена только одним сервером федерации и не может использоваться для увеличения масштаба на дополнительные серверы.  
  
Если вы хотите добавить дополнительные серверы федерации в лабораторию тестирования, развернув другие топологии, которая описывается ниже в этом разделе необходимо перестроить службы федерации с нуля. Таким образом, мы рекомендуем использовать эту топологию для лаборатории тестирования или проверки\-из\-тестовую среду в вашей частной сети тестирования, в котором одного сервера федерации подходит, как показано на следующем рисунке.

## <a name="test-lab-considerations"></a>Рекомендации по лаборатории тестирования  
В этом разделе описывает различные вопросы о целевая аудитория, преимущества и ограничения, связанные с этой топологии для тестовых лабораторных сред.  
  
### <a name="who-should-use-this-topology"></a>Для кого предназначена эта топология?  
  
-   Информационные технологии \(ИТ\) специалистов или ИТ-архитекторы, которые хотят оценить или подтверждения концепции для этой технологии разработки  
  
### <a name="what-are-the-benefits-of-using-this-topology"></a>Каковы преимущества использования этой топологии?  
  
-   Простота настройки в тестовой лабораторной среде  
  
### <a name="what-are-the-limitations-of-using-this-topology"></a>Что такое ограничение на использование этой топологии  
  
-   Только одним сервером федерации на службу федерации \(отсутствует возможность, чтобы увеличить масштаб в ферму\)  
  
-   Нет избыточности \(существует только один экземпляр базы данных конфигурации AD FS\)
