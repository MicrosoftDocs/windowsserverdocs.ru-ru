# <a name="name-resolution-requirements-for-federation-servers"></a>Требования к разрешению имен для серверов федерации

Когда клиентские компьютеры в корпоративной сети пытаются получить доступ к приложению или веб-службе, защищенной службы федерации Active Directory (AD FS) \(AD FS @ no__t-1, они должны сначала пройти проверку подлинности на сервере федерации. Одним из способов проверки подлинности является наличие у клиентов корпоративной сети доступа к локальному серверу федерации через встроенную проверку подлинности Windows.  
  
## <a name="configure-corporate-dns"></a>Настройка корпоративной DNS  
Чтобы успешно выполнить разрешение имен через встроенную проверку подлинности Windows на локальных серверах федерации, необходимо настроить доменное имя System \(DNS @ no__t-1 в корпоративной сети партнера по учетным записям для нового узла \(A @ no__t-3. запись ресурса, которая будет разрешать полное доменное имя сервера федерации \(FQDN @ no__t-5 на IP-адрес кластера сервера федерации.
