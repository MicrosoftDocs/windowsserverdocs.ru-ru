# <a name="certificate-requirements-for-federation-server-proxies"></a>Требования к сертификатам для прокси-серверов федерации

Серверы, работающие в роли прокси-сервера федерации в службы федерации Active Directory (AD FS) \(AD FS @ no__t-1, должны использовать сертификаты проверки подлинности сервера SSL \(SSL @ no__t-3. Прокси-серверы федерации используют SSL-сертификаты проверки подлинности сервера для защиты обмена трафиком между веб-сервером и веб-клиентами.  
  
Прокси-серверы федерации обычно предоставляются компьютерам в Интернете, не включенным в инфраструктуру открытого ключа предприятия \(PKI @ no__t-1. Поэтому используйте сертификат проверки подлинности сервера, выданный открытым центром сертификации \(third @ no__t-1party @ no__t-2 \(CA @ no__t-4, например VeriSign.  
  
При наличии фермы прокси-серверов федерации все компьютеры прокси-сервера федерации должны использовать один и тот же сертификат проверки подлинности сервера. Дополнительные сведения см. в разделе [Когда следует создавать ферму прокси-серверов федерации](When-to-Create-a-Federation-Server-Proxy-Farm.md).  
  
Важно убедиться в том, что имя субъекта в сертификате проверки подлинности сервера соответствует значению служба федерации имени, указанному в AD FS оснастки управления @ no__t-0in. Чтобы найти это значение, откройте **службу**"привязка @ no__t-0in, правую @ no__t-1Click", щелкните **изменить служба федерации свойства**, а затем найдите значение в текстовом поле **имя служба федерации** .

> [!NOTE]  
> Сертификаты проверки подлинности клиента не требуются для AD FS прокси-серверов федерации.  
  
Если любой сертификат, который вы используете, имеет списки отзыва сертификатов \(CRLs @ no__t-1, сервер с настроенным сертификатом должен иметь возможность связаться с сервером, который распространяет списки отзыва сертификатов. Тип CRL определяет используемые порты.
