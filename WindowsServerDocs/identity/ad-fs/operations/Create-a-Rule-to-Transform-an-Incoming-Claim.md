# <a name="create-a-rule-to-transform-an-incoming-claim"></a>Создание правила для преобразования входящего утверждения

>Область применения: Windows Server 2016, Windows Server 2012 R2

С помощью **преобразование входящего утверждения** шаблона правила в \(AD FS\) служб федерации Active Directory, можно выбрать входящее утверждение, изменить свой тип утверждения и изменить его значение. Например можно использовать этот шаблон правила, чтобы создать правило, которое отправляет заявку роли с помощью того же значение входящего утверждения о группе. Это правило можно также использовать для отправки утверждения о группе со значением утверждения лиц, приобретающих при наличии входящего утверждения о группе со значением «Администраторы», или вы можете отправить только имя участника-пользователя \(UPN\) утверждений, заканчивающиеся @fabrikam.  
  
Можно использовать следующую процедуру для создания правила утверждения с управления AD FS оснастка.  
  
Членство в группе **Администраторы**, или в эквивалентной группе на локальном компьютере, является минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477). 

## <a name="to-create-a-rule-to-transform-an-incoming-claim-on-a-relying-party-trust-in-windows-server-2016"></a>Создание правила для преобразования входящего утверждения на доверия с проверяющей стороной в Windows Server 2016 

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение политики выдачи утверждений **.
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule10.PNG)   
  
4.  В **изменение политики выдачи утверждений** диалогового окна **правила преобразования выдачи** щелкните **добавить правило** для запуска мастера создания правила. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule11.PNG)    

5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **преобразования входящего утверждения** из списка и нажмите кнопку **Далее **.  
![Создание правила](media/Create-a-Rule-to-Transform-an-Incoming-Claim/transform3.PNG)      

6.  На **Настройка правила** в разделе **имя правила утверждений**, введите отображаемое имя для данного правила. В **тип входящего утверждения**выберите утверждения введите список. В **исходящие утверждения и тип**, выберите в списке тип утверждения и затем выберите один из следующих параметров, которые зависит от требований вашей организации:  
  
    -   **Пропуск всех значений утверждений**  
  
    -   **Замените значение входящего утверждения на значение другого исходящего утверждения**  
  
    -   **Замена суффикса e\ почты входящих утверждений новым суффиксом электронной почты e\**  
![Создание правила](media/Create-a-Rule-to-Transform-an-Incoming-Claim/transform4.PNG)   

7.  Нажмите кнопку **Готово** кнопки.  
  
8.  В **изменение правил для утверждений** диалоговом нажмите кнопку **ОК** чтобы сохранить правило.
  
> [!NOTE]  
> При настройке сценария динамического контроля доступа, использующее утверждения, выданные AD FS\, создать правила преобразования в отношении доверия поставщика утверждений, а также в **тип входящего утверждения**, введите имя для входящего утверждения или, если ранее была создана описания утверждения, выберите его из списка. Во-вторых, в **исходящие утверждения и тип**, выберите утверждений URL-адрес, который будет, а затем создать правила преобразования на доверия с проверяющей стороной для выдачи утверждения устройства.  
>   
> Дополнительные сведения о сценариях динамического контроля доступа см. в разделе [динамического управления план содержимого по доступа](../../solution-guides/dynamic-access-control--scenario-overview.md) или [использование утверждений AD DS в AD FS ](https://technet.microsoft.com/library/hh831504.aspx).
