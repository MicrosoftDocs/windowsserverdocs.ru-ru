# <a name="create-a-rule-to-send-an-authentication-method-claim"></a>Создание правила для отправки утверждения методов проверки подлинности

>Область применения: Windows Server 2016, Windows Server 2012 R2

Вы можете использовать **отправлять членство в группе как утверждения** шаблона правила или **преобразование входящего утверждения** шаблон правила для отправки утверждения методов проверки подлинности. Проверяющая сторона может использовать утверждения методов проверки подлинности для определения входа механизм, который пользователь использует для проверки подлинности и получение утверждений из \(AD FS\) служб федерации Active Directory. Также можно использовать функцию контроля механизма проверки подлинности служб федерации Active Directory \(AD FS\) в Windows Server 2012 R2 в качестве входных данных для формирования утверждений метод проверки подлинности для ситуаций, в которых проверяющей стороны хочет определяют уровень доступа, который основан на вход в систему смарт-карты. Например разработчик можно назначить разные уровни доступа федеративных пользователей приложения проверяющей стороной. Уровни доступа основаны на ли входе пользователей в систему с помощью своих имени и пароля учетные данные пользователя, в отличие от смарт-карты.  
  
В зависимости от требований вашей организации используйте одну из следующих процедур:  
  
-   Создать это правило с помощью **отправлять членство в группе как утверждения** шаблона правила \-этот шаблон правила можно использовать, когда требуется группе, что вы указали в этот шаблон в конечном итоге определить, какой метод проверки подлинности утверждений для выдачи.  
  
-   Создать это правило с помощью **преобразования входящего утверждения** шаблона правила \-этот шаблон правила можно использовать, если вы хотите изменить текущий метод проверки подлинности на новый метод проверки подлинности, который работает с продуктом, не распознает стандартных утверждений метод проверки подлинности Служб федерации Active Directory.  
  


## <a name="to-create-by-using-the-send-group-membership-as-claims-rule-template-on-a-relying-party-trust-in-windows-server-2016"></a>Чтобы создать с помощью отправлять членство в группе как шаблон правила утверждения на доверия с проверяющей стороной в Windows Server 2016 

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение политики выдачи утверждений **.
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule10.PNG)   
  
4.  В **изменение политики выдачи утверждений** диалогового окна **правила преобразования выдачи** щелкните **добавить правило** для запуска мастера создания правила. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule11.PNG)    

5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **отправлять членство в группе как утверждение** из списка и нажмите кнопку **Далее**.  
![Создание правила](media/Create-a-Rule-to-Send-Group-Membership-as-a-Claim/group3.PNG)      

6.  На **Настройка правила** введите имя правила утверждений.  
  
7.  Нажмите кнопку **Обзор**, выберите группу, члены которой должны получать это утверждение метод проверки подлинности и нажмите кнопку **ОК**.  
  
8.  В **исходящие утверждения и тип**выберите **метод проверки подлинности** в списке.  
  
9. В **исходящие утверждения на значение**, введите одно из значений по умолчанию универсальный код ресурса идентификатор \(URI\) в следующей таблице, в зависимости от вашей предпочтительным методом проверки подлинности, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК** чтобы сохранить правило.  
  
|Фактические метод проверки подлинности|Соответствующий код URI|  
|--------------------------------|---------------------|  
|Проверка подлинности имени и пароля пользователя|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/password|  
|Проверка подлинности Windows|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/windows|  
|Протокол безопасности транспортного уровня \(TLS\) взаимной проверки подлинности, которая использует сертификаты X.509 транспорта|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/tlsclient|  
|Проверка подлинности на основе X.509\, которая не использует протокол TLS|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/x509|  
![Создание правила](media/Create-a-Rule-to-Send-an-Authentication-Method-Claim/auth2.PNG)

## <a name="to-create-this-rule-by-using-the-transform-an-incoming-claim-rule-template-in-windows-server-2012-r2"></a>Чтобы создать это правило с помощью преобразования шаблона правила входящего утверждения в Windows Server 2012 R2  
   
  
  
1.  В диспетчере серверов щелкните **средства**, а затем нажмите кнопку **управления AD FS **.  
  
2.  В дереве консоли в разделе **отношения FS\\Trust AD**, выбрать пункт **отношения доверия поставщиков утверждений** или **доверия с проверяющей стороной**и нажмите кнопку определенных отношений доверия в списке, где вы хотите создать это правило.  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение правил для утверждений **.  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule6.PNG) 
 
4.  В **изменение правил для утверждений** диалоговое окно, выберите один из следующих вкладок, в которых зависит от доверия редактируете и в какие наборе правил вы хотите создать это правило, а затем нажмите кнопку **добавить правило** для запуска мастера правил, который связан с этого набора правил:  
  
    -   **Правила преобразования принятия**  
  
    -   **Правила преобразования выдачи**  
  
    -   **Правила авторизации выдачи**  
  
    -   **Правила авторизации делегирования**  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall5.PNG)
  
5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **преобразования входящего утверждения** из списка и нажмите кнопку **Далее **.  
![Создание правила](media/Create-a-Rule-to-Transform-an-Incoming-Claim/transform1.PNG)    
  
6.  На **Настройка правила** введите имя правила утверждений.  
  
7.  В **тип входящего утверждения**выберите **метод проверки подлинности** в списке.  
  
8.  В **исходящие утверждения и тип**выберите **метод проверки подлинности** в списке.  
  
9. Выберите **заменить значение входящего утверждения на значение другого исходящего утверждения**, а затем выполните следующие действия:  
  
    1.  В **значение входящего утверждения**, введите одну из следующих значений URI, основанные на сам процесс проверки подлинности метод URI, который изначально был использован, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК** чтобы сохранить правило.  
  
    2.  В **исходящие утверждения на значение**, введите одно из значений по умолчанию URI в следующей таблице, в которых зависит от выбранного метода проверки подлинности новый, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК** чтобы сохранить правило.  
  
|Метод сам процесс проверки подлинности|Соответствующий код URI|  
|--------------------------------|---------------------|  
|Проверка подлинности имени и пароля пользователя|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/password|  
|Проверка подлинности Windows|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/windows|  
|Взаимная проверка подлинности TLS, которая использует сертификаты X.509|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/tlsclient|  
|Проверка подлинности на основе X.509\, которая не использует протокол TLS|https://schemas.microsoft.com/ws/2008/06/identity/authenticationmethod/x509|  
![Создание правила](media/Create-a-Rule-to-Send-an-Authentication-Method-Claim/auth3.PNG)
  
> [!NOTE]  
> Помимо значений в таблице можно использовать другие значения URI. Значения URI, которые отображаются ионный предыдущей таблице отражают URI, на который проверяющей стороны принимает по умолчанию.
