# <a name="create-a-rule-to-permit-all-users"></a>Создание правила для разрешения всем пользователям

В Windows Server 2016, можно использовать **политику контроля доступа** создать правило, которое предоставит всем пользователям доступ к проверяющей стороне.  В Windows Server 2012 R2 с помощью **разрешить всем пользователям** шаблон правила в службах федерации Active Directory \(AD FS\), можно создать правило авторизации, которое предоставит всем пользователям доступ к проверяющей стороны субъект. 

Дополнительные правила авторизации можно использовать для ограничения доступа. Пользователи, которым разрешен доступ к проверяющей стороне из службы федерации, все еще могут получить отказ в обслуживании проверяющей стороной.  
  
Можно использовать следующие процедуры для создания правила утверждения с помощью оснастки управления AD FS\-в.  
  
Для выполнения этой процедуры минимальным требованием является членство в группе **Администраторы** или эквивалентные права на локальном компьютере.  Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [локальные и доменные группы по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477). 

## <a name="to-create-a-rule-to-permit-all-users-in-windows-server-2016"></a>Чтобы создать правило, чтобы разрешить всем пользователям в Windows Server 2016

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS**.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия проверяющей стороны**. 
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall1.PNG)

3.  Щелкните правой кнопкой мыши **доверия с проверяющей стороной** , вы хотите разрешить доступ к и выберите **изменить политику контроля доступа**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall2.PNG)

4. При обращении контролировать политики выберите **разрешение для каждого** и нажмите кнопку **применить** и **ОК**.
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall3.PNG)
  
## <a name="to-create-a-rule-to-permit-all-users-in-windows-server-2012-r2"></a>Чтобы создать правило, чтобы разрешить всем пользователям в Windows Server 2012 R2 
  
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS**.  
  
2.  В дереве консоли в разделе **AD FS\\отношения доверия\\доверия проверяющей стороны**, щелкните определенные доверие в списке, где вы хотите создать это правило.  

3.  Справа\-выберите выбранного отношения доверия и нажмите кнопку **изменение правил для утверждений**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall4.PNG)  

4.  В **изменение правил для утверждений** диалоговом окне щелкните **правила авторизации выдачи** вкладку или **правила авторизации делегирования** вкладке \(исходя из типа правила авторизации, требуется\), а затем нажмите кнопку **добавить правило** запустить **утверждения мастера добавления правила авторизации**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall5.PNG)  
5.  На **Выбор шаблона правила** раздела **шаблон правила утверждения**выберите **разрешить всем пользователям** в списке и нажмите кнопку **Далее**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall6.PNG)    
6.  На **Настройка правила** щелкните **Готово**.  
  
7.  В **изменение правил для утверждений** диалоговом окне щелкните **ОК** сохранить правило.
