# <a name="create-a-rule-to-permit-all-users"></a>Создание правила для разрешения всем пользователям

>Область применения: Windows Server 2016, Windows Server 2012 R2

В Windows Server 2016, можно использовать **политики управления доступом** для создания правила, которое будет предоставить всем пользователям доступ к проверяющей стороне.  В Windows Server 2012 R2 с помощью **разрешить всем пользователям** шаблона правила в \(AD FS\) служб федерации Active Directory, можно создать правило авторизации, которое будет предоставить всем пользователям доступ к проверяющей стороне. 

Дополнительные правила авторизации можно использовать для ограничения доступа. Пользователи, которым разрешен доступ к проверяющей стороне из службы федерации могут по-прежнему будет отказ в обслуживании проверяющей стороной.  
  
Следующие процедуры используются для создания правила утверждения с помощью управления AD FS оснастка.  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477). 

## <a name="to-create-a-rule-to-permit-all-users-in-windows-server-2016"></a>Чтобы создать правило для разрешения всем пользователям в Windows Server 2016

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall1.PNG)

3.  Щелкните правой кнопкой мыши **доверия с проверяющей стороной**, вы хотите разрешить доступ к и выберите **изменить политики управления доступом **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall2.PNG)

4. На доступ управления выберите политики **разрешить всем пользователям** и нажмите кнопку **применить** и **ОК **.
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall3.PNG)
  
## <a name="to-create-a-rule-to-permit-all-users-in-windows-server-2012-r2"></a>Чтобы создать правило для разрешения всем пользователям в Windows Server 2012 R2 
  
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **доверия стороны AD FS\\Trust Relationships\\Relying**, нажмите кнопку определенных отношений доверия в списке, где вы хотите создать это правило.  

3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение правил для утверждений **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall4.PNG)  

4.  В **изменение правил для утверждений** диалоговом нажмите кнопку **правила авторизации выдачи** вкладка или **правила авторизации делегирования** вкладку \ (на основе типа правила авторизации можно require\) и нажмите кнопку **добавить правило** запуск **утверждения мастера добавления правила авторизации **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall5.PNG)  
5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **разрешить всем пользователям** из списка и нажмите кнопку **Далее **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall6.PNG)    
6.  На **Настройка правила** щелкните **Готово **.  
  
7.  В **изменение правил для утверждений** диалоговом нажмите кнопку **ОК** чтобы сохранить правило.
