# <a name="create-a-rule-to-pass-through-or-filter-an-incoming-claim"></a>Создание правила для пропуска или фильтрации входящего утверждения

>Область применения: Windows Server 2016, Windows Server 2012 R2

С помощью пропуск или Фильтрация входящего утверждения шаблона правила в \(AD FS\) служб федерации Active Directory, можно передать через все входящие утверждения с помощью выбранного типа утверждения. Можно также отфильтровать значения входящих утверждений с помощью выбранного типа утверждения. Например этот шаблон правила можно использовать для создания правила, которое будет отправлять все входящие утверждения о группе. Можно также использовать это правило для отправки только пользователь имя участника \(UPN\) утверждений, которые заканчиваются @fabrikam.  
  
Можно использовать следующую процедуру для создания правила утверждения с управления AD FS оснастка.  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477).   

## <a name="to-create-a-rule-to-pass-through-or-filter-an-incoming-claim-on-a-relying-party-trust-in-windows-server-2016"></a>Чтобы создать правило для пропуска или фильтрации входящего утверждения на доверия с проверяющей стороной в Windows Server 2016 

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение политики выдачи утверждений **.
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule10.PNG)   
  
4.  В **изменение политики выдачи утверждений** диалогового окна **правила преобразования выдачи** щелкните **добавить правило** для запуска мастера создания правила. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule11.PNG)    

5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **пропуск или Фильтрация входящего утверждения** из списка и нажмите кнопку **Далее**.  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule4.PNG)    

6.  На **Настройка правила** в разделе **имя правила утверждений** введите отображаемое имя для этого правила в **тип входящего утверждения** Выбор типа утверждения списка, а затем выберите один из следующих вариантов, в зависимости от потребностей вашей организации:  
  
    -   **Пропуск всех значений утверждений**  
  
    -   **Пропускать только значения утверждения определенной**  
  
    -   **Пропускать только значения утверждения, соответствующие указанному значению суффикса электронной почты**  
  
    -   **Пропускать только значения утверждения, начинающиеся с указанного значения**  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule5.PNG)    

7.  Нажмите кнопку **Готово** кнопки.  
  
8.  В **изменение правил для утверждений** диалоговом нажмите кнопку **ОК** чтобы сохранить правило.

    -   **Пропускать только значения утверждения, начинающиеся с указанного значения**  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule5.PNG)

8.  В **изменение правил для утверждений** диалоговом нажмите кнопку **ОК** чтобы сохранить правило.
