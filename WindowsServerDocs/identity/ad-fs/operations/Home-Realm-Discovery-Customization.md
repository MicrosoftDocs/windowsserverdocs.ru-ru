# <a name="home-realm-discovery-customization"></a>Настройка обнаружения домашней области

>Область применения: Windows Server 2016, Windows Server 2012 R2

Когда клиента AD FS первом запросе ресурса, сервер федерации ресурсов не обладает информацией об области клиента. Сервер федерации ресурсов отвечает клиенту AD FS с **обнаружение области клиента** страницы, где пользователь выбирает домашней области в списке. Список заполняется значениями из свойства отображаемого имени в отношения доверия поставщиков утверждений. Используйте следующие командлеты Windows PowerShell для изменения и настройки AD FS функции обнаружения домашней области.  
  
![Домашняя область](media/AD-FS-user-sign-in-customization/ADFS_Blue_Custom4.png)  
  
> [!WARNING]  
> Имейте в виду, что имя поставщика утверждений, отображаемое для локальной службы Active Directory — отображаемое имя службы федерации.  
  
## <a name="configure-identity-provider-to-use-certain-email-suffixes"></a>Настройка поставщика удостоверений для использования определенных суффиксов электронной почты  
Организация может состоять в федеративных отношениях с несколькими поставщиками утверждений. Службы федерации Active Directory теперь предоставляет количества поле для администраторов вывести список суффиксов, например, @us.contoso.com, @eu.contoso.com, поддерживаемые поставщиком утверждений и включения обнаружения на основе suffix\. Эта конфигурация конечные пользователи могут ввести свою учетную запись в организации и AD FS автоматически выберет соответствующего поставщика утверждений.  
  
Чтобы настроить поставщика удостоверений \(IDP\), таких как `fabrikam`, для использования определенных суффиксов электронной почты, используйте следующий командлет Windows PowerShell и синтаксисом.  
  

`Set-AdfsClaimsProviderTrust -TargetName fabrikam -OrganizationalAccountSuffix @("fabrikam.com";"fabrikam2.com") ` 
 
  
## <a name="configure-an-identity-provider-list-per-relying-party"></a>Настройка списка поставщиков удостоверений по проверяющей стороной  
В некоторых случаях организации может возникнуть конечным пользователям видеть только поставщиков утверждений, относящихся к приложению, то на странице обнаружения домашней области отображалось только определенное подмножество поставщиков утверждений.  
  
Чтобы настроить список поставщиков Удостоверений по проверяющей стороне \(RP\), используйте следующий командлет Windows PowerShell и синтаксисом.  
  
 
`Set-AdfsRelyingPartyTrust -TargetName claimapp -ClaimsProviderName @("Fabrikam","Active Directory") ` 

  
## <a name="bypass-home-realm-discovery-for-the-intranet"></a>Отключение обнаружения домашней области для интрасети  
В большинстве организаций поддерживают только их локальной службы Active Directory для любого пользователя, доступ в пределах брандмауэра. В таком случае администраторы могут настроить AD FS для обнаружение домашней области для интрасети.  
  
Чтобы отключить обнаружение домашней области для интрасети, используйте следующий командлет Windows PowerShell и синтаксисом.  
  

`Set-AdfsProperties -IntranetUseLocalClaimsProvider $true ` 
 
  
> [!IMPORTANT]  
> Пожалуйста Обратите внимание, что если список поставщиков удостоверений для проверяющей стороны настроен, даже если предыдущая настройка включена и пользователь имеет доступ из интрасети, службы федерации Active Directory по-прежнему отображается \(HRD\) страница обнаружения домашней области. Чтобы отключить обнаружение домашней области в этом случае, необходимо убедиться, что «Active Directory» также добавлен в список поставщиков Удостоверений для данной проверяющей стороны.
