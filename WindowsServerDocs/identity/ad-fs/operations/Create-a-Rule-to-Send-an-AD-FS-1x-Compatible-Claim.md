# <a name="create-a-rule-to-send-an-ad-fs-1x-compatible-claim"></a>Создание правила для отправки AD FS 1.x утверждения, совместимые

>Область применения: Windows Server 2016, Windows Server 2012 R2


В ситуациях, в которых вы используете службы федерации Active Directory \(AD FS\) для выдачи утверждений, который будет получать серверами федерации под управлением AD FS 1.0 \ (Windows Server 2003 R2\) или AD FS 1.1 \ (Windows Server 2008 или Windows Server 2008 R2\), необходимо выполнить следующие действия:  
  
-   Создание правила, которое будет отправлять тип утверждения идентификатор имени в формате UPN, электронная почта или общее имя.  
  
-   Все остальные утверждения, отправляемые должен иметь одно из следующих типов утверждений:  
  
    -   AD FS 1. *x* адрес электронной почты  
  
    -   AD FS 1. *x* UPN  
  
    -   Общее имя  
  
    -   Группы  
  
    -   Любого другого типа утверждений, который начинается с https://schemas.xmlsoap.org/claims/, например https://schemas.xmlsoap.org/claims/EmployeeID  
  
В зависимости от потребностей вашей организации используйте одну из следующих процедур, чтобы создать AD FS 1. *x* утверждения, совместимые NameID:  
  
-   Создание правила для проблемы с AD FS 1.x имя Идентификатору утверждений с помощью **пропуск или Фильтрация входящего утверждения шаблона правила**  
  
-   Создание правила для проблемы с AD FS 1.x имя Идентификатору утверждений с помощью **преобразования шаблона правила входящего утверждения**. Этот шаблон правила можно использовать в ситуациях, в которых вы хотите изменить существующий тип утверждения новый тип утверждения, которые будут работать с AD FS 1. *x* утверждений.  
  
> [!NOTE]  
> Для этого правила для правильной работы убедитесь, что доверия с проверяющей стороной или доверия поставщика утверждений, где создается правило была настроена для использования **профиль AD FS 1.0 и 1.1**. 




## <a name="to-create-a-rule-to-issue-an-ad-fs-1x-name-id-claim-using-the-pass-through-or-filter-an-incoming-claim-rule-template-on-a-relying-party-trust-in-windows-server-2016"></a>Создание правила для выдачи AD FS 1. *x* идентификатор имени утверждений с помощью пропуск или Фильтрация входящего утверждения шаблона правила на доверия с проверяющей стороной в Windows Server 2016 

1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение политики выдачи утверждений **.
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule10.PNG)   
  
4.  В **изменение политики выдачи утверждений** диалогового окна **правила преобразования выдачи** щелкните **добавить правило** для запуска мастера создания правила. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule11.PNG)    

5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **пропуск или Фильтрация входящего утверждения** из списка и нажмите кнопку **Далее**.  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule4.PNG)    

6.  На **Настройка правила** введите имя правила утверждений.  
  
7.  В **тип входящего утверждения**выберите **идентификатор имени** в списке.  
  
8.  В **формат идентификатора имени входящего**, выберите один из следующих AD FS 1. *x*\-compatible утверждений форматов в списке:  
  
    -   **ИМЯ УЧАСТНИКА-ПОЛЬЗОВАТЕЛЯ**  
  
    -   **E\ почта**  
  
    -   **Общее имя**  
  
9. Выберите один из следующих вариантов, в зависимости от потребностей вашей организации.  
  
    -   **Пропуск всех значений утверждений**  
  
    -   **Пропускать только значения утверждения определенной**  
  
    -   **Пропускать только значения утверждения, соответствующие указанному значению суффикса электронной почты**  
  
    -   **Пропускать только значения утверждения, начинающиеся с указанного значения**  
![Создание правила](media/Create-a-Rule-to-Send-an-AD-FS-1x-Compatible-Claim/adfs3.PNG)   

10. Нажмите кнопку **Готово**, а затем нажмите кнопку **ОК** чтобы сохранить правило.

7.  В **тип входящего утверждения**, выберите тип входящего утверждения, который требуется преобразовать в списке.  
  
8.  В **исходящие утверждения и тип**выберите **идентификатор имени** в списке.  
  
9. В **формата идентификатора исходящего имени**, выберите один из следующих AD FS 1. *x*\-compatible утверждений форматов в списке:  
  
    -   **ИМЯ УЧАСТНИКА-ПОЛЬЗОВАТЕЛЯ**  
  
    -   **E\ почта**  
  
    -   **Общее имя**  
  
10. Выберите один из следующих вариантов, в зависимости от потребностей вашей организации.  
  
    -   **Пропуск всех значений утверждений**  
  
    -   **Замените значение входящего утверждения на значение другого исходящего утверждения**  
  
    -   **Замена суффикса e\ почты входящих утверждений новым суффиксом электронной почты e\**  
![Создание правила](media/Create-a-Rule-to-Send-an-AD-FS-1x-Compatible-Claim/adfs2.PNG)    

11. Нажмите кнопку **Готово**, а затем нажмите кнопку **ОК** чтобы сохранить правило.
