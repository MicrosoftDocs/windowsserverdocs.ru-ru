# <a name="create-a-rule-to-permit-or-deny-users-based-on-an-incoming-claim"></a>Создание правила для разрешения или запрета пользователям на основании входящего утверждения 

>Область применения: Windows Server 2016, Windows Server 2012 R2

В Windows Server 2016, можно использовать **политики управления доступом** для создания правила, которое разрешает из запрещать пользователям на основании входящего утверждения.  В Windows Server 2012 R2 с помощью **разрешать или запрещать пользователям на основании входящего утверждения** шаблона правила в \(AD FS\) служб федерации Active Directory, можно создать правило авторизации, которое будет предоставить или запретить доступ конкретного пользователя к проверяющей стороне на основе типа и значения входящего утверждения. 

Например это можно использовать для создания правила, которое разрешает доступ только пользователям, имеющим утверждение о группе со значением "Администраторы домена" доступ к проверяющей стороне. Если вы хотите разрешить всем пользователям доступ к проверяющей стороне, используйте **разрешить все** политики управления доступом или **разрешить всем пользователям** шаблон правила, в зависимости от используемой версии Windows Server. Пользователи, которым разрешен доступ к проверяющей стороне из службы федерации могут по-прежнему будет отказ в обслуживании проверяющей стороной.  
  
Можно использовать следующую процедуру для создания правила утверждения с управления AD FS оснастка.  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477).  

## <a name="to-create-a-rule-to-permit-users-based-on-an-incoming-claim-on-windows-server-2016"></a>Создание правила для разрешения пользователям на основании входящего утверждения на Windows Server 2016
 
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **политики управления доступом **. 
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny3.PNG)

3. Щелкните правой кнопкой мыши и выберите **добавить политику управления доступом к **.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny4.PNG)

4. В поле "имя" введите имя политики, описание и нажмите кнопку **добавить **.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny5.PNG)

5. На **редактор правил**, в группе пользователей, установите флажок в **с утверждениями, определенных в запросе** и щелкните подчеркнутое **определенных** в нижней.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny6.PNG)

6. На **выберите утверждений** нажмите кнопку **утверждений** переключатель, выберите **тип утверждения**, **оператор**и **значение утверждения** щелкните **ОК **.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny7.PNG)

7.  На **редактор правил** щелкните **ОК **.  На **добавить политику управления доступом** нажмите кнопку **ОК **.

8. В **управления AD FS** дереве консоли **AD FS**, нажмите кнопку **доверия с проверяющей стороной **. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)

9.  Щелкните правой кнопкой мыши **доверия с проверяющей стороной**, вы хотите разрешить доступ к и выберите **изменить политики управления доступом **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall2.PNG)

10. В политике управления доступом, выберите политику и нажмите кнопку **применить** и **ОК **.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny8.PNG)

## <a name="to-create-a-rule-to-permit-or-deny-users-based-on-an-incoming-claim-on-windows-server-2012-r2"></a>Чтобы создать правило для разрешения или запрета пользователям на основании входящего утверждения на Windows Server 2012 R2
  
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS **.    
  
2.  В дереве консоли в разделе **доверия стороны AD FS\\Trust Relationships\\Relying**, нажмите кнопку определенных отношений доверия в списке, где вы хотите создать это правило.  
  
3.  Щелкните правой кнопкой мыши выбранные доверия, а затем нажмите кнопку **изменение правил для утверждений **.  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule6.PNG)   

4.  В **изменение правил для утверждений** диалоговом нажмите кнопку **правила авторизации выдачи** вкладка или **правила авторизации делегирования** вкладку \ (на основе типа правила авторизации можно require\) и нажмите кнопку **добавить правило** запуск **утверждения мастера добавления правила авторизации **.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall5.PNG)

5.  На **Выбор шаблона правила** в разделе **шаблон правила утверждения**выберите **разрешать или запрещать пользователям на основании входящего утверждения** из списка и нажмите кнопку **Далее **.  
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny1.PNG)

6.  На **Настройка правила** в разделе **имя правила утверждений** введите отображаемое имя для этого правила в **тип входящего утверждения** Выбор типа утверждения список, в разделе **значение входящего утверждения** введите значение, или нажмите кнопку Обзор, \ (если он не available\) и выберите значение и затем выберите один из следующих вариантов, в зависимости от потребностей вашей организации:  
  
    -   **Разрешить доступ пользователям с данным входящим утверждением**  
  
    -   **Запретить доступ пользователям с данным входящим утверждением**  
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny2.PNG)  
7.  Нажмите кнопку **Готово **.  
  
8.  В **изменение правил для утверждений** диалоговом нажмите кнопку **ОК** чтобы сохранить правило.
