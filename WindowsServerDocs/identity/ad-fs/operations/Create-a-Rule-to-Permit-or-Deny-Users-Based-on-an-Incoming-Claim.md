# <a name="create-a-rule-to-permit-or-deny-users-based-on-an-incoming-claim"></a>Создание правила для разрешения или запрета пользователям на основании входящего утверждения 


В Windows Server 2016, можно использовать **политику контроля доступа** создать правило, которое будет разрешать или запрещать пользователям на основании входящего утверждения.  В Windows Server 2012 R2 с помощью **разрешать или запрещать пользователям на основании входящего утверждения** шаблон правила в службах федерации Active Directory \(AD FS\), можно создать правило авторизации, которая будет предоставлена или Запретите доступ конкретного пользователя к проверяющей стороне на основе типа и значения входящего утверждения. 

Например это можно использовать для создания правила, которое разрешает доступ только пользователям, имеющим утверждение со значением "Администраторы домена", чтобы получить доступ к проверяющей стороне о группе. Если вы хотите разрешить всем пользователям доступ к проверяющей стороне, используйте **разрешить всем пользователям** политики управления доступом или **разрешить всем пользователям** шаблон правила в зависимости от используемой версии Windows Server. Пользователи, которым разрешен доступ к проверяющей стороне из службы федерации, все еще могут получить отказ в обслуживании проверяющей стороной.  
  
Можно использовать следующую процедуру для создания правила утверждения с помощью оснастки управления AD FS\-в.  
  
Для выполнения этой процедуры минимальным требованием является членство в группе **Администраторы** или эквивалентные права на локальном компьютере.  Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [локальные и доменные группы по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477).  

## <a name="to-create-a-rule-to-permit-users-based-on-an-incoming-claim-on-windows-server-2016"></a>Чтобы создать правило, чтобы разрешить пользователям на основании входящего утверждения в Windows Server 2016
 
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS**.  
  
2.  В дереве консоли в разделе **AD FS**, нажмите кнопку **политики управления доступом**. 
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny3.PNG)

3. Щелкните правой кнопкой мыши и выберите **добавить политику контроля доступа**.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny4.PNG)

4. В поле «Имя» введите имя политики, описание и нажмите кнопку **добавить**.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny5.PNG)

5. На **редактор правил**, в списке пользователей, установите флажок **с утверждениями, определенных в запросе** и щелкните подчеркнутое слово **конкретных** внизу.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny6.PNG)

6. На **выберите утверждения** щелкните **утверждений** "переключатель", выберите **тип утверждения**, **оператор**и  **Значение утверждения** щелкните **ОК**.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny7.PNG)

7.  На **редактор правил** щелкните **ОК**.  На **добавить политику контроля доступа** щелкните **ОК**.

8. В **управления AD FS** консоли в узле дерева, **AD FS**, нажмите кнопку **доверия проверяющей стороны**. 
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule9.PNG)

9.  Щелкните правой кнопкой мыши **доверия с проверяющей стороной** , вы хотите разрешить доступ к и выберите **изменить политику контроля доступа**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall2.PNG)

10. В политике управления доступом, выберите политику и нажмите кнопку **применить** и **ОК**.
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny8.PNG)

## <a name="to-create-a-rule-to-permit-or-deny-users-based-on-an-incoming-claim-on-windows-server-2012-r2"></a>Чтобы создать правило, чтобы разрешать или запрещать пользователям на основании входящего утверждения на Windows Server 2012 R2
  
1.  В диспетчере серверов щелкните **средства**, а затем выберите **управления AD FS**.    
  
2.  В дереве консоли в разделе **AD FS\\отношения доверия\\доверия проверяющей стороны**, щелкните определенные доверие в списке, где вы хотите создать это правило.  
  
3.  Справа\-выберите выбранного отношения доверия и нажмите кнопку **изменение правил для утверждений**.  
![Создание правила](media/Create-a-Rule-to-Pass-Through-or-Filter-an-Incoming-Claim/claimrule6.PNG)   

4.  В **изменение правил для утверждений** диалоговом окне щелкните **правила авторизации выдачи** вкладку или **правила авторизации делегирования** вкладке \(исходя из типа правила авторизации, требуется\), а затем нажмите кнопку **добавить правило** запустить **утверждения мастера добавления правила авторизации**.  
![Создание правила](media/Create-a-Rule-to-Permit-All-Users/permitall5.PNG)

5.  На **Выбор шаблона правила** раздела **шаблон правила утверждения**выберите **разрешать или запрещать пользователям на основании входящего утверждения** в списке и нажмите кнопку  **Далее**.  
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny1.PNG)

6.  На **Настройка правила** странице в разделе **имя правила утверждения** введите отображаемое имя для этого правила в **тип входящего утверждения** выберите тип утверждения в списке в разделе  **Значение входящего утверждения** введите значение или нажмите кнопку Обзор \(если таковой имеется\) выберите значение и затем выберите один из следующих вариантов, в зависимости от потребностей вашей организации:  
  
    -   **Разрешить доступ пользователям с этим входящим утверждением**  
  
    -   **Запретить доступ пользователям с этим входящим утверждением**  
![Создание правила](media/Create-a-Rule-to-Permit-or-Deny-Users-Based-on-an-Incoming-Claim/permitdeny2.PNG)  
7.  Нажмите кнопку **Готово**.  
  
8.  В **изменение правил для утверждений** диалоговом окне щелкните **ОК** сохранить правило.
