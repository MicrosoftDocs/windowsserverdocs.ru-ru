# <a name="access-control-policies-in-windows-server-2016-ad-fs"></a>Политики управления доступом в Windows Server 2016 AD FS

>Область применения: Windows Server 2016

  
## <a name="access-control-policy-templates-in-ad-fs"></a>Шаблоны политики управления доступом в AD FS  
Службы федерации Active Directory теперь поддерживает использование шаблонов политики управления доступом.  С помощью шаблонов политики управления доступом, администратор может применить параметры политики, назначив шаблон политики к группе проверяющих сторон (RPs). Администратор также может сделать обновлений шаблон политики и изменения будут применяться к проверяющих сторон автоматически в случае не требуется никаких действий пользователя.  
  
## <a name="what-are-access-control-policy-templates"></a>Что такое шаблоны политики управления доступом?  
Конвейер основных Служб федерации Active Directory для обработки политики содержит три этапа: выдачи проверки подлинности, авторизации и утверждения. В настоящее время службы федерации Active Directory администраторы должны отдельно настроить политику для каждого из этих этапов.  Это также включает в себя основные сведения о последствиях этих политик, а если эти политики между зависимостей. Кроме того Администраторы имеют понять утверждения правило языка и автор настраиваемые правила для включения некоторые простой/Общие политики (например) блокировать внешний доступ).  
  
Какие политики управления доступом, шаблоны, замените этот старая модель, где администраторам необходимо настроить с помощью правил авторизации выдачи утверждения языка.  Старый командлеты PowerShell правил авторизации выдачи по-прежнему применяются, однако это взаимно новую модель. Администраторы могут выбрать либо использовать новую модель или старая модель.  Новая модель позволяет администраторам контролировать, для предоставления доступа, включая осуществления многофакторной проверки подлинности.  
  
Шаблоны политики управления доступом используется модель разрешения.  Это означает, по умолчанию, никто не имеет доступа и необходимо явным образом предоставлен доступ.  Однако это не просто все или ничего не разрешать.  Администраторы могут добавлять исключений для правила разрешения.  Например администратор может потребоваться предоставить доступ, в зависимости от конкретной сети, если выбрать этот параметр и укажите диапазон IP-адресов.  Однако администратор может добавлять и исключения, например, администратор может добавить исключение из определенной сети и указывать, диапазон IP-адресов.

Это может быть изменен после создания и изменения шаблон настраиваемые политики применяются ко всем RPs, которые управляются эти шаблоны политики. ## <a name="parameterized-policy-template-vs-non-parameterized-policy-template"></a>Без параметров политики vs параметрами политики шаблона  
Может быть политики управления доступом  
  
Шаблон политики с параметрами является шаблон политики с параметрами. Администратор должен ввести значение для этих параметров, при назначении этого шаблона RPs.An администратора нельзя было изменить шаблон политики с параметрами, после его создания.  Пример параметрами политики — встроенные политики, разрешающего определенной группы.  Каждый раз, когда эта политика применяется к RP, этот параметр должен быть указан.  
  
![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP5.PNG)  
  
Шаблон без параметров политики — это шаблон политики, который не имеет параметров. Администратор можно назначать этот шаблон RPs без ввода, необходимого и внести изменения в шаблон без параметров политики, после его создания.  Примером этого встроенные политики, все разрешения и требует многофакторной проверки Подлинности.  
  
![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP4.PNG)  
  
## <a name="how-to-create-a-non-parameterized-access-control-policy"></a>Создание политику управления без параметров доступа  
Чтобы создать доступа без параметров политики управления используйте следующую процедуру  
  
#### <a name="to-create-a-non-parameterized-access-control-policy"></a>Чтобы создать политику управления без параметров доступа  
  
1.  Выберите политики управления доступом из управления AD FS слева и справа щелкните Добавить политику управления доступом.  
  
2.  Введите имя и описание.  Например: разрешить пользователям с устройствами с проверкой подлинности.  
  
3.  В разделе **предоставление доступа в случае, если удовлетворены любые из следующих правил**, нажмите кнопку **добавить**.  
  
4.  В списке разрешения, установите флажок в поле рядом с пунктом **с устройств с уровнем определенных отношений доверия**  
  
5.  Выбрать подчеркнутый внизу **определенных**  
  
6.  В окне, щелчкам вверх, выберите **с проверкой подлинности** из раскрывающегося списка.  Нажмите кнопку **ОК**.  
  
    ![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP6.PNG)  
  
7.  Нажмите кнопку **ОК**. Нажмите кнопку **ОК**.  
  
    ![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP7.PNG)  
  
## <a name="how-to-create-a-parameterized-access-control-policy"></a>Создание политику управления параметрами доступа  
Чтобы создать элемент управления параметрами доступа политики используйте следующую процедуру  
  
#### <a name="to-create-a-parameterized-access-control-policy"></a>Чтобы создать политику управления параметрами доступа  
  
1.  Выберите политики управления доступом из управления AD FS слева и справа щелкните Добавить политику управления доступом.  
  
2.  Введите имя и описание.  Например: разрешить пользователям с конкретной заявки.  
  
3.  В разделе **предоставление доступа в случае, если удовлетворены любые из следующих правил**, нажмите кнопку **добавить**.  
  
4.  В списке разрешения, установите флажок в поле рядом с пунктом **с утверждениями, определенных в запросе**  
  
5.  Выбрать подчеркнутый внизу **определенных**  
  
6.  В окне, щелчкам вверх, выберите **параметр указан, при назначении политики управления доступом**.  Нажмите кнопку **ОК**.  
  
    ![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP8.PNG)  
  
7.  Нажмите кнопку **ОК**. Нажмите кнопку **ОК**.  
  
    ![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP9.PNG)  
  
## <a name="how-to-create-a-custom-access-control-policy-with-an-exception"></a>Как создать политики управления настраиваемого доступа из-за исключения  
Чтобы настроить управление доступом политики из-за исключения используйте следующую процедуру.  
  
#### <a name="to-create-a-custom-access-control-policy-with-an-exception"></a>Чтобы создать политику управления настраиваемого доступа из-за исключения  
  
1.  Выберите политики управления доступом из управления AD FS слева и справа щелкните Добавить политику управления доступом.  
  
2.  Введите имя и описание.  Например: разрешить пользователям проверку подлинности устройств, но не управление.  
  
3.  В разделе **предоставление доступа в случае, если удовлетворены любые из следующих правил**, нажмите кнопку **добавить**.  
  
4.  В списке разрешения, установите флажок в поле рядом с пунктом **с устройств с уровнем определенных отношений доверия**  
  
5.  Выбрать подчеркнутый внизу **определенных**  
  
6.  В окне, щелчкам вверх, выберите **с проверкой подлинности** из раскрывающегося списка.  Нажмите кнопку **ОК**.  
  
7.  В разделе, за исключением, установите флажок в поле рядом с пунктом **с устройств с уровнем определенных отношений доверия**  
  
8.  В нижней в разделе, за исключением щелкните подчеркнутое **определенных**  
  
9. В окне, щелчкам вверх, выберите **управляемых** из раскрывающегося списка.  Нажмите кнопку **ОК**.  
  
10. Нажмите кнопку **ОК**. Нажмите кнопку **ОК**.  
  
    ![политики управления доступом](media/Access-Control-Policies-in-AD-FS/ADFSACP10.PNG)  
  
## <a name="how-to-create-a-custom-access-control-policy-with-multiple-permit-conditions"></a>Как создать политики управления настраиваемого доступа с несколькими условиями разрешения  
Чтобы создать политики управления доступом с нескольких разрешения условия используйте следующую процедуру  
  
#### <a name="to-create-a-parameterized-access-control-policy"></a>Чтобы создать политику управления параметрами доступа  
  
1.  Выберите политики управления доступом из управления AD FS слева и справа щелкните Добавить политику управления доступом.  
  
2.  Введите имя и описание.  Например: разрешить пользователям с конкретной заявки и из конкретной группы.  
  
3.  В разделе **предоставление доступа в случае, если удовлетворены любые из следующих правил**, нажмите кнопку **добавить**.  
  
4.  В списке разрешения, установите флажок в поле рядом с пунктом **из определенной группы** и **с утверждениями, определенных в запросе**  
  
5.  Внизу выбрать Подчеркнутый **определенных** для первого условия, рядом с группами

7.  Выбрать подчеркнутый внизу **определенных** для второе условие, рядом с утверждений  
  
8.  В окне, щелчкам вверх, выберите **параметр указан, при назначении политики управления доступом**.  Нажмите кнопку **ОК**.  
  
9. Нажмите кнопку **ОК**. Нажмите кнопку **ОК**.
