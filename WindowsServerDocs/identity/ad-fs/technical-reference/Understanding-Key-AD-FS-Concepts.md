## <a name="ad-fs-terminology-used-in-this-guide"></a>Термины AD FS, употребляемые в этом руководстве  
  
|Термин AD FS|Определение|  
|--------------|--------------|  
|Партнерской организации по учетной записи|Партнерской организации федерации, представленного отношения доверия с поставщиком утверждений в службе федерации. Организации партнера по учетным записям содержит пользователей, которые будут обращаться к веб-приложений в организации партнера по ресурсам.|  
|Сервер федерации учетных записей|Сервер федерации в организации партнера по учетным записям. Сервер федерации учетных записей издает маркеры безопасности пользователям на основе аутентификации пользователей. Сервер проверяет подлинность пользователя, извлекает соответствующие атрибуты и сведения о членстве в группах из хранилища атрибутов, упаковывает эту информацию в утверждения, создает и подписывает маркер безопасности \ (содержит claims\) для возвращения пользователю, либо для использования в его собственной организации или отправки в партнерскую организацию.|  
|База данных конфигурации AD FS|База данных для хранения всех данных конфигурации, представляющие отдельный экземпляр AD FS или службу федерации. Эти данные могут храниться в базе SQL Server или с помощью функции внутренней базы данных Windows включенных в Windows Server 2016, Windows Server 2012 и 2012 R2 и Windows Server 2008 и 2008 R2. </br></br>Можно создать базу данных конфигурации AD FS для SQL Server, с помощью средства командной строки Fsconfig.exe и для внутренней базы данных Windows с помощью мастера конфигурации сервера AD FS федерации.|  
|Поставщик утверждений|Организация, поставляющая утверждения своим пользователям. В разделе партнерской организации по учетной записи.|  
|Отношения доверия с поставщиком утверждений|В оснастка управления AD FS отношения доверия с поставщиком утверждений — это объекты доверия, которые обычно создаются в партнерских организациях по ресурсам для представления организации во взаимоотношениях доверия, учетные записи которых будут осуществлять доступ к ресурсам в партнерской организации по ресурсам. Объект доверия поставщика утверждений состоит из разных идентификаторов, имен и правил, идентифицирующих этого партнера в локальной службе федерации.|  
|Отношения доверия с поставщиком утверждений локального|Объект доверия, представляющий AD LDS или third\ стороной каталоги на основе LDAP\ в ферму AD FS. Локальный доверия поставщика утверждений объект состоит из разных идентификаторов, имен и правил, идентифицирующих этого на основе LDAP\ каталога в локальной службе федерации.|  
|Метаданные федерации|Формат данных для передачи сведений о конфигурации между поставщиком утверждений и проверяющей стороной с целью правильной настройки отношений доверия поставщика утверждений и отношений доверия с проверяющей стороной. Формат данных определяется в \(SAML\) Security Assertion Markup Language 2.0 и расширяется в наличия WS-Federation.|  
|Сервера федерации|Windows Server, настроенный с использованием мастера конфигурации сервера AD FS федерации выступать в роли сервера федерации. Сервер федерации издает маркеры и функционирует как часть службы федерации.|  
|Прокси-сервера федерации|Windows Server, которая была настроена с помощью мастер настройки прокси-сервера AD FS федерации сервера в качестве промежуточной прокси-сервер службы между Интернет-клиентом и службой федерации, расположенной за пределами брандмауэра в корпоративной сети.|  
|Основной сервер федерации|Windows Server, настроенный в роли сервера федерации, с помощью мастера конфигурации сервера AD FS федерации и имеет для чтения и записи копия базы данных конфигурации AD FS. </br></br> Основной сервер федерации создается при использовании мастера конфигурации сервера AD FS федерации и выберите параметр для создания новой службы федерации и сделать этот компьютер первого сервера федерации в ферме. Все остальные серверы федерации на этой ферме должны реплицировать изменения, внесенные на основном сервере федерации для только для чтения копия базы данных конфигурации AD FS, которая хранится локально. Термин «основной сервер федерации» неприменим, если база данных конфигурации AD FS хранится в базе данных SQL как все серверы федерации могут одинаково чтение и запись к базе данных конфигурации, хранимой на SQL Server.|  
|Проверяющая сторона|Организация, получающая и обрабатывающая утверждения. В разделе партнерской организации по ресурсам.|  
|Проверяющей стороной доверия|В оснастка управления AD FS отношения доверия проверяющей стороны объекты доверия, которые обычно создаются в:<br /><br />-В партнерских организациях учетной записи для представления организации во взаимоотношениях доверия, учетные записи которых будут осуществлять доступ к ресурсам в партнерской организации по ресурсам.<br />-Партнерских организациях по ресурсам для представления отношения доверия между службы федерации и отдельным веб-приложением.<br /><br />Объект доверия проверяющей стороны состоит из разных идентификаторов, имен и правил, идентифицирующих этого партнера или веб приложение в локальной службе федерации.|  
|Сервер федерации ресурсов|Сервер федерации в организации партнера по ресурсам. Как правило, сервер федерации ресурсов издает маркеры безопасности пользователям на основании маркера безопасности, выданные сервером федерации учетных записей. Сервер получает маркер безопасности, проверяет подпись, применяет логику правил утверждений к нераспакованным утверждениям, чтобы получить нужные исходящие утверждения, создает новый маркер безопасности \ (с исходящих claims\) на основе информации из входящего маркера безопасности и подписывает новый маркер для возвращения пользователю и в конечном счете веб-приложению.|  
|Партнерская организация по ресурсам|Партнер по федерации, представленного доверия с проверяющей стороной в службе федерации. Партнер по ресурсам издает маркеры безопасности на основе claims\, которые содержат опубликованные веб-приложения, доступные пользователям партнера по учетным записям.|  
  
## <a name="overview-of-ad-fs"></a>Обзор Служб федерации Active Directory  
AD FS является это решение доступа удостоверений, обеспечивающее для клиентских компьютеров \ (внутренних или внешних по отношению к вашей сетевой) беспроблемный доступ единого входа к защищенным модулю-приложениям или службам, даже если учетные записи пользователей и приложения размещены в совершенно разных сетях или организациях.  
  
Если приложение или служба находятся в одной сети и учетную запись пользователя находится в другой сети, обычно пользователю предлагается ввести дополнительные учетные данные при попытке получить доступ к приложению или службе. Эти дополнительные учетные данные представляют удостоверение пользователя в области нахождения приложения или службы. Их ввода обычно требует веб-сервер, на котором размещается служба или приложение, чтобы она могла принимать наиболее взвешенного решения об авторизации.  
  
С AD FS организации могут обходить запросы ввода дополнительных учетных данных, предоставляя \(federation trusts\) отношения доверия, могут использоваться этими организациями для проецирования пользователя цифрового удостоверения и права доступа доверенным партнерам. В федерированной среде каждая организация продолжает управлять своими собственными удостоверениями, но каждой организации может безопасно проецировать и принимать удостоверения от других организаций.
