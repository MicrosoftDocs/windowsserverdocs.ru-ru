# <a name="the-role-of-claims"></a>Роль утверждений
В модели идентификации на базе claims\ утверждения играют жизненно важную роль в процессе федерации — это ключевой компонент, с помощью которого определяются в результате выполнения всех веб-проверки подлинности и авторизации запросов. Эта модель позволяет организациям безопасно проецировать цифрового удостоверения, а также прав, или *утверждений*, невзирая на безопасности и границ стандартных механизмов.  
  
## <a name="what-are-claims"></a>Что такое утверждения?  
Проще говоря, утверждения — это просто *инструкций* \ (например, имя, удостоверение, group\), относящиеся к пользователям, которые используются в основном для авторизации доступа к приложениям на базе claims\, расположенных в Интернете. Каждое заявление соответствует *значение*, хранится в утверждении.  
  
### <a name="how-claims-are-sourced"></a>Источники утверждений  
Службы федерации в службах федерации Active Directory \(AD FS\) определяет, какие утверждения передаются между партнерами федерации. Тем не менее до этого он должен сначала заполнить или утверждение с полученное значение или значение, вычисленное. Каждое значение утверждения представляет значение пользователя, группы или сущности и источники одним из двух способов:  
  
1.  Если значение, составляющее утверждение извлекается из хранилища атрибутов, например, когда значение атрибута отдела продаж извлекается из свойств учетной записи пользователя в Active Directory. Дополнительные сведения см. в разделе [роль хранилищ атрибутов](The-Role-of-Attribute-Stores.md).  
  
2.  Если значение входящего утверждения трансформируется в другое значение на основании логики, реализованной в правиле. Например, если входящее утверждение со значением "Администраторы домена" преобразуется в новое значение Администраторы перед отправкой в качестве исходящего утверждения. Дополнительные сведения см. в разделе [The Role of Claim Rules](The-Role-of-Claim-Rules.md).  
  
Правила могут включать значения, такие как адрес электронной почты e\, \(UPN\) имя участника-пользователя, членство в группе и другие атрибуты учетных записей.  
  
### <a name="how-claims-flow"></a>Принципы потока утверждений  
Другие стороны используют значения утверждений для выполнения задач авторизации для веб-приложений, которые в них размещены. Эти стороны называются *проверяющих сторон* в управления AD FS оснастка. Служба федерации несет ответственность за установку отношений доверия между многочисленными разрозненными сторонами. Она призвана обрабатывать и стимулировать доверенный обмен утверждениями между организацией, которая изначально ищет утверждения, также называется *Поставщики утверждений* в управления AD FS оснастка, и проверяющей стороной. Затем проверяющая сторона использует эти утверждения для принятия решений об авторизации.  
  
Поток утверждений, осуществляемый по вышеописанной схеме, известен как *конвейер утверждений *. Поток утверждений через конвейер утверждений состоит из трех этапов:  
  
1.  Утверждения, полученные от поставщика утверждений, обрабатываются правилами преобразования принятия в отношении доверия поставщика утверждений. Эти правила определяют, какие утверждения принимаются от поставщика утверждений.  
  
2.  Вывод правил преобразования приемки используется как ввод для правил авторизации выдачи. Эти правила определяют, разрешен ли пользователю доступ к проверяющей стороне.  
  
3.  Вывод правил преобразования приемки используется как ввод для правил преобразования выдачи. Эти правила определяют утверждений, которые будут отправляться проверяющей стороны.  
  
Дополнительные сведения см. в разделе [роль канала утверждений](The-Role-of-the-Claims-Pipeline.md)  
  
### <a name="how-claims-are-issued"></a>Выпуск утверждений  
При создании правил утверждений источник входящих утверждений для правил утверждений варьируется в зависимости от того пишутся ли правила для отношения доверия с поставщиком утверждений или отношений доверия с проверяющей стороной. При создании правил для утверждений для отношений доверия с поставщиком утверждений входящие утверждения — это утверждения, отправляемые от доверенного поставщика утверждений в службу федерации. При создании правил для отношений доверия с проверяющей стороной входящие утверждения — это утверждения, выводимые правилами утверждений отношений доверия с поставщиком утверждений применимо. Дополнительные сведения о входящих и исходящих утверждениях см. в разделе [The Role of the Claims Pipeline](The-Role-of-the-Claims-Pipeline.md) и [The Role of the Claims Engine ](The-Role-of-the-Claims-Engine.md).

## <a name="what-are-claim-descriptions"></a>Что такое описания утверждений  
Описания утверждений представляют список типов утверждений, что поддерживает AD FS, который можно опубликовать в метаданных федерации. Упомянутые в предыдущей таблице типы утверждений настраиваются как описания утверждений в управления AD FS оснастка.  
  
Коллекция описаний утверждений, которые будут опубликованы в метаданных федерации, хранится в базе данных конфигурации AD FS. Эти утверждений описания, используются разными компонентами службы федерации.  
  
Каждое описание утверждения включает тип утверждения URI, имя, статус публикации и описание. Можно управлять коллекцией определений утверждений с помощью **описания утверждений** узел управления AD FS оснастка. Можно изменить статус публикации описания утверждения, воспользовавшись оснастка. Доступны следующие параметры:  
  
-   **Опубликовать это утверждение в метаданных федерации как тип утверждения, принимаемый этой службой федерации** \(Publish as Accepted\) — обозначает типы утверждений, которые будут приняты от других поставщиков утверждений этой службой федерации.  
  
-   **Опубликовать это утверждение в метаданных федерации как тип утверждения, отправляемый этой службой федерации** \(Publish as Sent\) — обозначает типы утверждений, предлагаемые этой службой федерации. Это типы утверждений, публикуемые службой федерации для других пользователей как для отправки. Типы утверждений, фактически отправляемые поставщиком утверждений, часто представляют собой подмножество этого списка.  
  
Дополнительные сведения о настройке состояния публикации типа утверждения см. в разделе [Добавление описания утверждения](https://technet.microsoft.com/library/dd807051.aspx) в руководстве по развертыванию AD FS.  
  
### <a name="when-generating-federation-metadata"></a>При создании федерации метаданных  
Метаданные федерации включают все описания утверждений, помеченные для публикации.  
  
### <a name="when-claims-rules-are-processed"></a>При обработке правил утверждений  
Если сведения конфигурации об описаниях утверждений сохранены, настроить правила об утверждениях проще. Дополнительные сведения о правилах утверждений, которые могут использоваться в организации поставщика утверждений см. в разделе [роль правил утверждений ](The-Role-of-Claim-Rules.md).
