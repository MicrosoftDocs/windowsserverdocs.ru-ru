# <a name="the-role-of-attribute-stores"></a>Роль хранилищ атрибутов
Термин «хранилище атрибутов» используется служб федерации Active Directory для обозначения каталогов или баз данных, используемые для хранения учетных записей пользователей и связанных значений атрибутов. После его настройки в организации поставщика удостоверений, службы федерации Active Directory, получает значения этих атрибутов из хранилища и создает утверждения на основе этих данных, чтобы веб-приложение или служба, размещенная в организации проверяющей стороны могла принимать соответствующие решения об авторизации всякий раз, когда Федеративный пользователь \ (пользователь, чья учетная запись хранится в organization\ поставщика удостоверений) пытается получить доступ к приложению или службе.  
  
Дополнительные сведения о создании утверждений см. в разделе [The Role of Claims](The-Role-of-Claims.md).  
  
## <a name="how-attribute-stores-fit-in-with-your-ad-fs-deployment-goals"></a>Соответствие хранилищ атрибутов целям AD FS развертывания  
Расположение хранилища атрибутов пользователей и расположение, из которого пользователи проходят проверку подлинности, определяют способ разработки AD FS для поддержки удостоверений пользователей. В зависимости от того, где находится хранилище атрибутов и откуда пользователи получают доступ к приложения \ (в интрасети или через модулю), можно использовать один из следующих целей развертывания:  
  
-   [Предоставить Your Active Directory Users Access Your Claims-Aware Applications and Services](https://technet.microsoft.com/library/dd807071.aspx)— в этом случае пользователи в вашей организации доступ к AD FS защищаемому приложение или служба \ (собственные приложения или службы или партнера приложения или служба\текущих) при входе в Active Directory в корпоративной интрасети.  
  
-   [Предоставление Your Active Directory Users Access к приложениям и службам других организаций](https://technet.microsoft.com/library/dd807123.aspx)— в этом случае пользователи в вашей организации доступ к AD FS защищаемому приложение или служба \ (собственные приложения или службы или партнера приложения или служба\текущих) при входе в хранилище атрибутов в корпоративной интрасети и при удаленном входе из Интернета.  
  
-   [Предоставление пользователям другой организации доступа к Your Claims-Aware Applications and Services](https://technet.microsoft.com/library/dd807099.aspx)— в этом случае учетные записи пользователей в другой организации, расположенные в хранилище атрибутов в корпоративной интрасети этой организации необходимо получить доступ к защищенное AD FS — приложение в вашей организации. Эта цель также актуальна, когда учетные записи пользователей на основе consumer\, находящихся в хранилище атрибутов в сети периметра вашей организации должен быть предоставлен доступ к AD FS защищенного приложения в вашей организации.  
  
В зависимости от расположения хранилища атрибутов и других требований вашей организации можно объединить несколько этих целей развертывания для завершения проекта развертывания Служб федерации Active Directory.  
  
## <a name="attribute-stores-that-are-supported-by-ad-fs"></a>Хранилища атрибутов, которые поддерживаются службами AD FS  
Службы федерации Active Directory поддерживает широкий спектр каталогов и базы данных сохраняет, можно использовать для извлечения определенных administrator\ значений атрибутов и заполнения утверждений этими значениями. Службы федерации Active Directory поддерживает следующие каталоги или базы данных в качестве хранилищ атрибутов:  
  
-   Active Directory в Windows Server 2003, доменные службы Active Directory \(AD DS\) в Windows Server 2008, AD DS в Windows Server 2012 и 2012 R2 и Windows Server 2016. 
  
-   Все выпуски Microsoft SQL Server 2005, SQL Server 2008, SQL Server 2012, SQL Server 2014 и SQL Server 2016  
  
-   Хранилища настраиваемых атрибутов
