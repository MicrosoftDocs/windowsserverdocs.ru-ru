# <a name="export-the-private-key-portion-of-a-server-authentication-certificate"></a>Экспорт части сертификата аутентификации сервера с закрытым ключом

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Каждый сервер федерации в ферме служб федерации Active Directory \(AD FS\) должны иметь доступ к закрытому ключу сертификата проверки подлинности сервера. Если вы реализуете фермы серверов федерации или веб-серверов, необходимо один сертификат аутентификации. Этот сертификат должен быть выдан центром сертификации предприятия \(CA\) и он должен иметь экспортируемый закрытый ключ. Закрытый ключ сертификата аутентификации сервера должен быть экспортируемым, чтобы его можно было доступным для всех серверов в ферме.  
  
То же самое справедливо фермах серверов прокси-сервера федерации, в том смысле, что все прокси-серверов федерации в ферме должны совместно использовать часть закрытым ключом один и тот же сертификат проверки подлинности сервера.  
  
> [!NOTE]  
> Управление AD FS оснастка расценивает сертификаты проверки подлинности для серверов федерации как сертификаты связи служб.  
  
В зависимости от роли, которую этот компьютер будет воспроизводить, выполните эту процедуру на компьютере сервера федерации или прокси-серверов федерации, где установлен сертификат проверки подлинности сервера с закрытым ключом. По окончании процедуры можно затем импортировать этот сертификат на веб-сайте по умолчанию каждого сервера в ферме. Дополнительные сведения см. в разделе [Импорт сертификата проверки подлинности сервера на веб-сайт по умолчанию](Import-a-Server-Authentication-Certificate-to-the-Default-Web-Site.md).  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477).   
  
### <a name="to-export-the-private-key-portion-of-a-server-authentication-certificate"></a>Экспорт закрытого ключа части сертификат проверки подлинности сервера  
  
1.  На **запустить** введите**\(IIS\) службы IIS Manager**, и нажмите клавишу ВВОД.  
  
2.  В дереве консоли щелкните **ComputerName**.  
  
3.  В центральной области двойном щелчке **сертификаты сервера**.  
  
4.  В центральной области щелкните правой кнопкой мыши сертификат, который вы хотите экспортировать, а затем нажмите кнопку **Экспорт**.  
  
5.  В **экспорт сертификатов** диалоговом нажмите кнопку **...** Нажатие кнопки.  
  
6.  В **имя файла**, тип **C:\\***Имя_сертификата*, а затем нажмите кнопку **откройте**.  
  
7.  Введите пароль для сертификата, подтвердите его и нажмите кнопку **ОК**.  
  
8.  Проверьте успешно выполнен экспорт, убедившись, что указанный файл создается в указанном расположении.  
  
    > [!IMPORTANT]  
    > Чтобы этот сертификат можно импортировать в локальное хранилище сертификатов на новом сервере, необходимо переместить файл на физический носитель и обеспечить его безопасность во время переноса на новый сервер. Очень важно обеспечить безопасность закрытого ключа. Если этого ключа нарушена, безопасность всего развертывания Служб федерации Active Directory \ нарушена (включая ресурсы в вашей организации и в organizations\ партнера по ресурсам).  
  
9. Импортируйте экспортированный сертификат аутентификации сервера в хранилище сертификатов на новом сервере до установки службы федерации. Сведения об импорте сертификата см. в разделе Импорт сертификата сервера \ ([http:///\/go.microsoft.com\/fwlink\/? LinkId\ = 108283](https://go.microsoft.com/fwlink/?LinkId=108283)\).
