# <a name="set-a-service-communications-certificate"></a>Задание сертификата связи со службой

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Серверы федерации в \(AD FS\) служб федерации Active Directory используют сертификата взаимодействия служб для обеспечения безопасности трафика веб-служб для \(SSL\) Secure Sockets Layer обмена данными с веб-клиентами или прокси-серверами федерации. Это тот же сертификат, который сервер федерации использует в качестве сертификата SSL в службах IIS \(IIS\).  
  
Можно использовать следующую процедуру для изменения сертификата взаимодействия служб с помощью управления AD FS оснастка.  
  
> [!NOTE]  
> Управление AD FS оснастка расценивает сертификаты проверки подлинности для серверов федерации как сертификаты связи служб.  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477) \ (http:///\/ go.microsoft.com\/fwlink\ /? LinkId\ = 83477\).   
  
### <a name="to-set-a-service-communications-certificate"></a>Для установки сертификата связи со службой  
  
1.  На **запустить** введите**управления AD FS**, и нажмите клавишу ВВОД.  
  
2.  В дереве консоли двойном щелчке **службы**, а затем нажмите кнопку **сертификаты**.  
  
3.  В **действия** панели, щелкните **Настройка сертификата взаимодействия служб** ссылку.  
  
4.  В **выберите сертификата связи со службой** диалоговом окне перейдите к файлу сертификата, который требуется установить в качестве сертификата взаимодействия служб, выберите файл сертификата и нажмите кнопку **откройте**.
