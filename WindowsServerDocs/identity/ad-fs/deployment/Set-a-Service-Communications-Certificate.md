# <a name="set-a-service-communications-certificate"></a>Задание сертификата связи со службой


Серверы федерации в службы федерации Active Directory (AD FS) \( AD FS \) используют сертификат связи службы для защиты трафика веб-служб для SSL \( \) связи SSL с веб-клиентами или с прокси сервера федерации.

> [!NOTE]
> Сертификат связи служб отличается от сертификата SSL. Чтобы изменить AD FS SSL-сертификат, потребуется использовать PowerShell. Следуйте указаниям в этой [статье](../operations/manage-ssl-certificates-ad-fs-wap.md).


Чтобы изменить сертификат связи службы с помощью оснастки управления AD FS, можно использовать следующую процедуру \- .

> [!NOTE]
> Оснастка управления AD FS \- в относится к сертификатам проверки подлинности сервера для серверов федерации в качестве сертификатов связи служб.

Для выполнения этой процедуры требуется членство в группе **Администраторы** или в эквивалентной группе на локальном компьютере.  Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [локальной среде и группах домена по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477) \( http: \/ \/ go.Microsoft.com \/ fwlink \/ ? LinkId \= 83477 \) .

### <a name="to-set-a-service-communications-certificate"></a>Настройка сертификата связи службы

1.  На **начальном** экране введите **AD FS управление** и нажмите клавишу ВВОД.

2.  В дереве консоли дважды \- щелкните **Служба**, а затем щелкните **Сертификаты**.

3.  На панели **действия** щелкните ссылку **задать сертификат связи служб** .

4.  В диалоговом окне **Выбор сертификата взаимодействия служб** перейдите к файлу сертификата, который необходимо задать в качестве сертификата связи службы, выберите файл сертификата и нажмите кнопку **Открыть**.
