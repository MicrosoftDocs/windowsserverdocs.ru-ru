# <a name="set-a-service-communications-certificate"></a>Задание сертификата связи со службой


Серверы федерации в службах федерации Active Directory \(AD FS\) использовать сертификата взаимодействия служб для защиты трафика веб-служб для Secure Sockets Layer \(SSL\) обмен данными с веб- Клиенты или прокси-серверами федерации.

> [!NOTE]  
> Сертификат связи со службой, не является таким же, как SSL-сертификат. Чтобы изменить сертификат AD FS SSL, необходимо использовать Powershell. Следуйте указаниям в этом [статье](https://docs.microsoft.com/windows-server/identity/ad-fs/operations/manage-ssl-certificates-ad-fs-wap).


Можно использовать следующую процедуру для изменения сертификата взаимодействия служб с помощью оснастки управления AD FS\-в.  

> [!NOTE]  
> Оснастка управления AD FS\-в расценивает сертификаты проверки подлинности серверов для серверов федерации как сертификаты взаимодействия служб.  

Для выполнения этой процедуры минимальным требованием является членство в группе **Администраторы** или эквивалентные права на локальном компьютере.  Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [локальные и доменные группы по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477) \(http:\/\/go.microsoft.com\/fwlink\/? LinkId\=83477\).   

### <a name="to-set-a-service-communications-certificate"></a>Для установки сертификата связи со службой  

1.  На **запустить** введите**управления AD FS**, и нажмите клавишу ВВОД.  

2.  В дереве консоли дважды\-щелкните **службы**, а затем нажмите кнопку **сертификаты**.  

3.  В **действия** панели щелкните **Настройка сертификата взаимодействия служб** ссылку.  

4.  В **выберите сертификата связи со службой** диалоговом окне перейдите к файлу сертификата, который требуется задать в качестве сертификата взаимодействия служб, выберите файл сертификата и нажмите кнопку **откройте**.
