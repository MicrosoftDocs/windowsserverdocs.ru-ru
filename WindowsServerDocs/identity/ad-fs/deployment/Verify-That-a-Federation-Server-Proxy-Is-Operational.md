# <a name="verify-that-a-federation-server-proxy-is-operational"></a>Проверка работоспособности прокси-сервера федерации

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Можно использовать следующую процедуру, чтобы убедиться, что прокси-сервер федерации могут обмениваться данными со службой федерации в \(AD FS\) служб федерации Active Directory. Выполнить эту процедуру после запуска **мастера конфигурации прокси-сервера федерации AD FS** Настройка компьютера для работы в роли прокси-сервера федерации. Дополнительные сведения о запуске этого мастера см. в разделе [Настройка компьютера для роли прокси-сервера федерации](Configure-a-Computer-for-the-Federation-Server-Proxy-Role.md).  
  
> [!IMPORTANT]  
> Этот тест образом успешного создания конкретного события в средстве просмотра событий на компьютере прокси-сервера федерации.  
  
Членство в группе **Администраторы**, или эквивалентной группе на локальном компьютере минимальным требованием для выполнения этой процедуры.  Подробные сведения об использовании соответствующих учетных записей и членстве в группах в [локальные и доменные группы по умолчанию ](https://go.microsoft.com/fwlink/?LinkId=83477).   
  
### <a name="to-verify-that-a-federation-server-proxy-is-operational"></a>Чтобы Проверка работоспособности прокси-сервера федерации  
  
1.  Выполните вход на прокси-сервера федерации от имени администратора.  
  
2.  На **запустить** введите**средство просмотра событий**, и нажмите клавишу ВВОД.  
  
3.  В области сведений щелкните двойном щелчке **журналы приложений и служб**, двойном щелчке **события AD FS**, а затем нажмите кнопку **администратора**.  
  
4.  В **событие с Идентификатором** столбец, найдите событие с кодом 198.  
  
    Если прокси-сервер федерации настроен правильно, вы увидите новое событие в журнале приложений средства просмотра событий, в событие с кодом 198. Это событие свидетельствует о том, что служба прокси-сервера федерации была успешно запущена и теперь находится в оперативном режиме.
