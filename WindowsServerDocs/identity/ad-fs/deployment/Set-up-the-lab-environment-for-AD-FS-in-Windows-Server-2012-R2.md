## <a name="BKMK_1"></a>Шаг 1: Настройка контроллера домена (DC1)
Для целей данной тестовой среды можно назвать корневой домен Active Directory **contoso.com** и укажите ** pass@word1 ** в качестве пароля администратора.

-   Установка службы роли AD DS и установки доменных служб Active Directory (AD DS), чтобы сделать компьютер контроллером домена в Windows Server 2012 R2. Это действие обновляет схему AD DS в рамках создания контроллера домена. Дополнительные сведения и пошаговые инструкции см. в разделе[https://technet.microsoft.com/library/hh472162.aspx ](https://technet.microsoft.com/library/hh472162.aspx).

### <a name="BKMK_2"></a>Создание тестовых учетных записей Active Directory
После контроллер домена станет функциональным, можно создать тестовую группу и тестовые пользователей учетных записей в этом домене и добавить учетную запись пользователя в учетную запись группы. Эти учетные записи используются для выполнения пошаговых инструкций в соответствующих руководствах, которые перечислены ранее в этом разделе.

Создайте следующие учетные записи:

-   Пользователь: **Robert Hatley** со следующими учетными данными: имя пользователя: **RobertH** и пароль:**P@ssword**

-   Группа: **Finance**

Сведения о том, как создание пользователей и групп в Active Directory (AD) см. в разделе [https://technet.microsoft.com/library/cc783323%28v.aspx ](https://technet.microsoft.com/library/cc783323%28v=ws.10%29.aspx).

Добавить **Robert Hatley** учетную запись **процент** группы. Сведения о том, как добавить пользователя в группу в Active Directory см. в разделе [https://technet.microsoft.com/library/cc737130%28v=ws.10%29.aspx ](https://technet.microsoft.com/library/cc737130%28v=ws.10%29.aspx).

### <a name="create-a-gmsa-account"></a>Создание групповой Управляемой учетной записи
Групповая управляемая учетная запись службы (GMSA) необходима во время установки служб федерации Active Directory (AD FS) и настройки.

##### <a name="to-create-a-gmsa-account"></a>Для создания групповой Управляемой учетной записи

1.  Откройте окно команд Windows PowerShell и введите:

    ```
    Add-KdsRootKey -EffectiveTime (Get-Date).AddHours(-10)
    New-ADServiceAccount FsGmsa -DNSHostName adfs1.contoso.com -ServicePrincipalNames http/adfs1.contoso.com

    ```

### <a name="install-a-server-ssl-certificate"></a>Установка SSL-сертификат сервера
Необходимо установить сертификат сервера Secure Socket Layer (SSL) на сервере ADFS1 в хранилище локального компьютера. Сертификат должен иметь следующие атрибуты:

-   Имя субъекта (CN): adfs1.contoso.com

-   Альтернативное имя субъекта (DNS): adfs1.contoso.com

-   Альтернативное имя субъекта (DNS): enterpriseregistration.contoso.com

Дополнительные сведения о настройке SSL-сертификатов см. в разделе [Настройка SSL/TLS на веб-сайте в домене с помощью ЦС предприятия](https://social.technet.microsoft.com/wiki/contents/articles/12485.configure-ssltls-on-a-web-site-in-the-domain-with-an-enterprise-ca.aspx).

[Серии видеороликов с инструкциями службами федерации Active Directory: Обновление сертификатов](https://technet.microsoft.com/video/adfs-updating-certificates).

### <a name="install-the-ad-fs-server-role"></a>Установка роли сервера AD FS

##### <a name="to-install-the-federation-service-role-service"></a>Для установки службы роли службы федерации

1.  Войдите на сервер с помощью учетной записи администратора домена administrator@contoso.com.

2.  Запустите диспетчер сервера. Для запуска диспетчера серверов нажмите кнопку **диспетчера сервера** на Windows **запустить** экрана или щелкните **диспетчера сервера** на панели задач Windows, на рабочем столе Windows. На **быстрый запуск** вкладке **приветствия** плитку **мониторинга** щелкните **Добавить роли и компоненты**. Кроме того, можно нажать кнопку **Добавить роли и компоненты** на **управление** меню.

3.  На **перед началом** щелкните **Далее**.

4.  На **Выбор типа установки** щелкните **Установка на основе ролей или компонентов**, а затем нажмите кнопку **Далее**.

5.  На **Выбор целевого сервера** щелкните **выберите сервер из пула серверов**, убедитесь, что выбран целевой компьютер и нажмите кнопку **Далее**.

6.  На **Выбор ролей сервера** щелкните **служб федерации Active Directory**, а затем нажмите кнопку **Далее**.

7.  На **выберите компоненты** щелкните **Далее**.

8.  На **службы федерации Active Directory (AD FS)** щелкните **Далее**.

9. После проверки информации на **подтверждение выбранных элементов для установки** выберите **автоматический перезапуск конечного сервера, если требуется** флажок и нажмите кнопку **установить**.

10. На **ход выполнения установки** , убедитесь, что все установлено верно и щелкните **закрыть**.

### <a name="configure-the-federation-server"></a>Настройка сервера федерации
Следующий шаг — Настройка сервера федерации.

##### <a name="to-configure-the-federation-server"></a>Настройка сервера федерации

1.  В диспетчере сервера **мониторинга** щелкните **уведомления**, а затем щелкните **настроить службу федерации на сервере**.

    **Мастер конфигурации служб федерации сертификации Active Directory** открывается.

2.  На **приветствия** выберите **создание первого сервера федерации в ферме серверов федерации**, а затем нажмите кнопку **Далее**.

3.  На **подключиться к AD DS** укажите учетную запись с правами администратора домена для **contoso.com** этот компьютер присоединен к домену Active Directory, а затем нажмите кнопку **Далее**.

4.  На **укажите свойства службы** страницы, выполните следующие действия и нажмите кнопку **Далее**:

    -   Импортируйте сертификат SSL, который вы получили более ранних версий. Этот сертификат является необходимым сертификатом аутентификации службы. Перейдите в расположение SSL-сертификат.

    -   Чтобы предоставить имя для вашей службы федерации, введите **adfs1.contoso.com**. Это значение имеет то же значение, что и предоставленное при регистрации сертификата SSL в службах сертификатов Active Directory (AD CS).

    -   Чтобы указать отображаемое имя для вашей службы федерации, введите **Contoso Corporation**.

5.  На **укажите учетную запись службы** выберите **использовать существующую учетную запись пользователя домена или групповую управляемую учетную запись службы**, а затем укажите учетную запись GMSA **fsgmsa** , созданную при создании контроллера домена.

6.  На **укажите базу данных конфигурации** выберите **создать базу данных на этом сервере с использованием внутренней базы данных Windows**, а затем нажмите кнопку **Далее**.

7.  На **просмотреть параметры**, проверьте выбранные параметры конфигурации и нажмите кнопку **Далее**.

8.  На **предварительные проверки** убедитесь, что все предварительные проверки успешно пройдены и нажмите кнопку **Настройка**.

9. На **результатов** , просмотрите результаты, проверьте, является ли конфигурация успешно завершена и щелкните **далее действия, необходимые для развертывания службы федерации**.

### <a name="configure-device-registration-service"></a>Настройка службы регистрации устройств
Следующий шаг — Настройка службы регистрации устройств на сервере ADFS1. Видеокурс, [федерации служб руководства серии видеороликов Active Directory: Включение службы регистрации устройств](https://technet.microsoft.com/video/adfs-how-to-enabling-the-device-registration-service).

##### <a name="to-configure-device-registration-service-for-windows-server-2012-rtm"></a>Чтобы настроить устройство регистрации службы для Windows Server 2012 RTM

1.  > [!IMPORTANT]
    > **Дальнейший шаг актуален для сборки Windows Server 2012 R2 RTM.**

    Откройте окно команд Windows PowerShell и введите:

    ```
    Initialize-ADDeviceRegistration
    ```

    При появлении запроса учетной записи службы введите **contoso\fsgmsa$**.

    Теперь запустите командлет Windows PowerShell.

    ```
    Enable-AdfsDeviceRegistration
    ```

2.  На сервере ADFS1 в **управления AD FS** консоли, перейдите в каталог **политики проверки подлинности**. Выберите **редактировать глобальную первичную аутентификацию**. Установите флажок рядом с пунктом **включить аутентификацию устройств**, а затем нажмите кнопку **ОК**.

### <a name="add-host-a-and-alias-cname-resource-records-to-dns"></a>Добавление узла (A) и записи ресурсов псевдонима (CNAME) в DNS
В DC1 необходимо убедиться, что для службы регистрации устройств созданы следующие записи доменных имен (DNS).

|Запись|Тип|Адрес|
|---------|--------|-----------|
|adfs1|Узел (A)|IP-адрес сервера AD FS|
|enterpriseregistration|Псевдоним (CNAME)|adfs1.contoso.com|

Можно использовать следующую процедуру для добавления записи ресурса узла (A) на корпоративные серверы имен DNS для сервера федерации и службы регистрации устройств.

Минимальным требованием для выполнения этой процедуры является членство в группе администраторов или эквивалентной ей. Подробные сведения об использовании соответствующих учетных записей и членства в группах гиперссылка «https://go.microsoft.com/fwlink/?LinkId=83477 «локальные и доменные группы по умолчанию (https://go.microsoft.com/fwlink/p/?LinkId=83477).

##### <a name="to-add-a-host-a-and-alias-cname-resource-records-to-dns-for-your-federation-server"></a>Добавление узла (A) и записи ресурсов псевдонима (CNAME) в DNS для сервера федерации

1.  На компьютере DC1 из диспетчера сервера на **средства** меню, нажмите кнопку **DNS** чтобы открыть оснастку DNS.

2.  В дереве консоли разверните DC1, затем **зоны прямого просмотра**, щелкните правой кнопкой мыши **contoso.com**, а затем нажмите кнопку **новый узел (A или AAAA)**.

3.  В **имя,** введите имя фермы AD FS. Для этого пошагового руководства введите **adfs1**.

4.  В **IP-адрес**, введите IP-адрес сервера ADFS1. Нажмите кнопку **добавления узла**.

5.  Щелкните правой кнопкой мыши **contoso.com**, а затем нажмите кнопку **нового псевдонима (CNAME)**.

6.  В **новую запись ресурса** диалоговое окно, введите **enterpriseregistration** в **псевдоним** поле.

7.  Введите в полное доменное имя (FQDN) целевого узла поля, **adfs1.contoso.com**, а затем нажмите кнопку **ОК**.

    > [!IMPORTANT]
    > В реальном развертывании Если у компании несколько суффиксов имени участника (UPN) пользователя, необходимо создать несколько записей CNAME, по одному для каждой из этих суффиксов в DNS.

### <a name="BKMK_9"></a>Настройка примера приложения утверждений в IIS

1.  Установите действительный сертификат SSL в хранилище сертификатов на компьютере. Сертификат должен содержать имя веб-сервера, **webserv1.contoso.com**.

2.  Скопируйте содержимое C:\Program Files (x86) \Windows Identity Foundation SDK\v3.5\Samples\Quick Start\Web Application\PassiveRedirectBasedClaimsAwareWebApp C:\Inetpub\Claimapp.

3.  Изменение **Default.aspx.cs** файл, чтобы Фильтрация утверждений не осуществлялась. Этот шаг выполняется, чтобы убедиться, что в примере приложения отображались все утверждения, выданные сервером федерации. Выполните следующие действия.

    1.  Откройте **Default.aspx.cs** в текстовом редакторе.

    2.  Найдите файл для второго экземпляра `ExpectedClaims`.

    3.  Закомментируйте `IF` заявление, включая круглые скобки. Обозначьте комментарии знаком «/ /» (без кавычек) в начале строки.

    4.  Ваш `FOREACH` теперь должно выглядеть как в следующем примере кода.

        ```
        Foreach (claim claim in claimsIdentity.Claims)
        {
           //Before showing the claims validate that this is an expected claim
           //If it is not in the expected claims list then don't show it
           //if (ExpectedClaims.Contains( claim.ClaimType ) )
           // {
              writeClaim( claim, table );
           //}
        }

        ```

    5.  Сохраните и закройте **Default.aspx.cs**.

    6.  Откройте **web.config** в текстовом редакторе.

    7.  Полностью удалите `<microsoft.identityModel>` раздела. Удалите все, начиная от `including <microsoft.identityModel>` и до и включая `</microsoft.identityModel>`.

    8.  Сохраните и закройте **web.config**.

4.  **Настройка диспетчера IIS Manager**

    1.  Откройте **Internet Information Services (IIS) диспетчера**.

    2.  Перейдите в раздел **пулы приложений**, щелкните правой кнопкой мыши **DefaultAppPool** для выбора **Дополнительные параметры**. Задайте **загрузить профиль пользователя** для **True**, а затем нажмите кнопку **ОК**.

    3.  Щелкните правой кнопкой мыши **DefaultAppPool** для выбора **основные параметры**. Изменение **версия .NET CLR** для **.NET CLR версии v2.0.50727**.

    4.  Щелкните правой кнопкой мыши **веб-сайт по умолчанию** для выбора **изменить привязки**.

    5.  Добавить **HTTPS** привязку к порту **443** с SSL-сертификата, на которых было установлено.

    6.  Щелкните правой кнопкой мыши **веб-сайт по умолчанию** для выбора **добавить приложение**.

    7.  Задайте псевдоним для **claimapp** и физический путь к **c:\inetpub\claimapp**.

5.  Чтобы настроить **claimapp** для работы с сервером федерации, выполните следующие действия:

    1.  Запустите FedUtil.exe, расположенный в **C:\Program Files (x86) \Windows Identity Foundation SDK\v3.5**.

    2.  Задайте расположение конфигурации приложений **C:\inetput\claimapp\web.config** и установить приложение URI в URL-адрес вашего сайта **https://webserv1.contoso.com /claimapp/**. Нажмите кнопку **Далее**.

    3.  Выберите **использовать существующий STS** и перейдите к URL-АДРЕСУ метаданных сервера AD FS **https://adfs1.contoso.com/federationmetadata/2007-06/federationmetadata.xml**. Нажмите кнопку **Далее**.

    4.  Выберите **отключить проверку цепочки сертификатов**, а затем нажмите кнопку **Далее**.

    5.  Выберите **без шифрования**, а затем нажмите кнопку **Далее**. На **предлагаемых утверждений** щелкните **Далее**.

    6.  Установите флажок рядом с пунктом **запланировать задание для выполнения ежедневных обновлений метаданных WS-Federation**. Нажмите кнопку **Готово **.

    7.  Пример приложения настроен. Если вы проверяете URL-адрес приложения **https://webserv1.contoso.com/claimapp**, он должен перенаправить вас на сервер федерации. На сервере федерации должна отображаться страница ошибки, потому что отношения доверия с проверяющей стороной еще не настроены. Другими словами вы не обеспечена этого тестового приложения с помощью AD FS.

Теперь необходимо обеспечить безопасность примера приложения, веб-сервер, с помощью AD FS. Это можно сделать с помощью добавления отношения доверия с проверяющей стороной на сервере федерации (ADFS1). Видеокурс, [Active Directory руководства серии видеороликов службами федерации: Добавление доверия с проверяющей стороной](https://technet.microsoft.com/video/adfs-how-to-add-a-relying-party-trust).

### <a name="BKMK_11"></a>Создание отношения доверия с проверяющей стороной на сервере федерации

1.  На сервере федерации (ADFS1) в **консоль управления AD FS**, перейдите в раздел **доверия с проверяющей стороной**, а затем нажмите кнопку **добавить доверия с проверяющей стороной**.

2.  На **Выбор источника данных** выберите **Импорт данных о проверяющей стороне опубликованных в Интернете или локальной сети**, введите URL-адрес метаданных для **claimapp**, а затем нажмите кнопку **Далее**. Запустите FedUtil.exe созданный XML-файл метаданных. Он расположен в **https://webserv1.contoso.com/claimapp/federationmetadata/2007-06/federationmetadata.xml**.

3.  На **укажите отображаемое имя** укажите **отображаемое имя** отношений доверия с проверяющей стороной, **claimapp**, а затем нажмите кнопку **Далее**.

4.  На **Настройка многофакторной проверки подлинности теперь? ** выберите **требуется указать параметр многофакторной проверки подлинности для этого отношения доверия с проверяющей стороной в данный момент не**, а затем нажмите кнопку **Далее**.

5.  На **Выбор правил авторизации выдачи** выберите **разрешить всем пользователям доступ к этой проверяющей стороне**, а затем нажмите кнопку **Далее**.

6.  На **Готово к добавлению отношения доверия** щелкните **Далее**.

7.  На **изменение правил для утверждений** диалоговом нажмите кнопку **добавить правило**.

8.  На **выберите тип правила** выберите **Отправка утверждений с помощью настраиваемого правила**, а затем нажмите кнопку **Далее**.

9. На **Настройка правил утверждений** страницы в **имя правила утверждений** введите **все утверждения**. В **настраиваемого правила** введите следующее правило утверждений.

    ```
    c:[ ]
    => issue(claim = c);

    ```

10. Нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.

## <a name="BKMK_10"></a>Шаг 4: Настройка клиентского компьютера (КЛИЕНТ1)
Настройте другую виртуальную машину и установите Windows 8.1. Эта виртуальная машина должны находиться в той же виртуальной сети, что и другие машины. Этой машины не должна быть объединена в домен Contoso.

Клиент должен доверять SSL-сертификат, используемый для сервера федерации (ADFS1), который настроен в [шаг 2: Настройка сервера федерации (ADFS1) with Device Registration Service](../../ad-fs/deployment/Set-up-the-lab-environment-for-AD-FS-in-Windows-Server-2012-R2.md#BKMK_4). Также должна быть возможность проверить информацию отзыва сертификатов для сертификата.

Вам также необходимо установить и использовать учетную запись Майкрософт для входа на КЛИЕНТ1.
