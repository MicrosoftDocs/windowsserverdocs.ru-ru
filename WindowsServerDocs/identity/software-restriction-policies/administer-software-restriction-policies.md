# <a name="administer-software-restriction-policies"></a>Администрирование политик ограниченного использования программ

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В этом разделе для ИТ-специалистов содержит процедуры для администрирования политик управления приложениями с помощью политик ограниченного использования программ (SRP), начиная с Windows Server 2008 и Windows Vista.

## <a name="introduction"></a>Введение
Политики ограниченного использования программ (SRP) — компонент на основе групповой политики, который идентифицирует программы, работающие на компьютерах в домене и управляет возможностью запуска этих программ. Политики ограниченного использования программ позволяют создать конфигурацию со строгими ограничениями для компьютеров, в которых разрешается запуск только определенных приложений. Они интегрированы с доменные службы Microsoft Active Directory и групповой политикой, но также могут настраиваться на изолированных компьютерах. Дополнительные сведения о политиках ограниченного использования см. в разделе [политик ограниченного использования программ](software-restriction-policies.md).

Начиная с Windows Server 2008 R2 и Windows 7, Windows AppLocker можно использовать вместо или вместе с SRP в части стратегии управления приложениями.

## <a name="BKMK_Open_SRP"></a>Чтобы открыть политики ограниченного использования программ

-   [Для локального компьютера](#BKMK_1)

-   [Для домена сайта, или подразделение, вы находитесь на рядовом сервере или рабочей станции, присоединенной к домену](#BKMK_2)

-   [Для сайта, вы находитесь на контроллере домена или на рабочей станции с установленными средствами удаленного администрирования сервера](#BKMK_4)

### <a name="BKMK_1"></a>Для локального компьютера

1.  Откройте локальные параметры безопасности.

2.  В дереве консоли щелкните **политик ограниченного использования программ**.

    **Где?**

    -   Параметры и программным обеспечением политик ограниченного использования программ

> [!NOTE]
> Для выполнения этой процедуры необходимо быть членом группы администраторов на локальном компьютере, либо вам должны быть делегированы соответствующие полномочия.

### <a name="BKMK_2"></a>Для домена сайта, или подразделение, вы находитесь на рядовом сервере или рабочей станции, присоединенной к домену

1.  Откройте консоль управления (MMC).

2.  На **файл** меню, нажмите кнопку **добавить или удалить оснастку**, а затем нажмите кнопку **добавить**.

3.  Нажмите кнопку **редактор объектов групповой политики**, а затем нажмите кнопку **добавить**.

4.  В **Выбор объекта групповой политики**, нажмите кнопку **Обзор**.

5.  В **поиск объекта групповой политики**, выберите объект групповой политики (GPO) в соответствующем домене, сайта или подразделения- или создать новый и нажмите кнопку **Готово**.

6.  Нажмите кнопку **закрыть**, а затем нажмите кнопку **ОК**.

7.  В дереве консоли щелкните **политик ограниченного использования программ**.

    **Где?**

    -   *Объект групповой политики* [*ComputerName*] Policy/Computer Configuration или

        Политики ограниченного использования программ параметры или программным обеспечением безопасности Конфигурация Windows пользователя

> [!NOTE]
> Для выполнения этой процедуры необходимо быть членом группы администраторов домена.

### <a name="BKMK_3"></a>Для домена или подразделения: вы находитесь на контроллере домена или на рабочей станции с установленными средствами удаленного администрирования сервера

1.  Откройте консоль управления групповыми политиками.

2.  В дереве консоли щелкните правой кнопкой мыши группу объектов политики (GPO), который требуется открыть политики ограниченного использования программ для.

3.  Нажмите кнопку **изменить** открыть объект групповой Политики, который требуется изменить. Можно также щелкнуть **New** создать новый объект групповой Политики, а затем нажмите кнопку **изменить**.

4.  В дереве консоли щелкните **политик ограниченного использования программ**.

    **Где?**

    -   *Объект групповой политики* [*ComputerName*] Policy/Computer Configuration или

        Политики ограниченного использования программ параметры или программным обеспечением безопасности Конфигурация Windows пользователя

> [!NOTE]
> Для выполнения этой процедуры необходимо быть членом группы администраторов домена.

### <a name="BKMK_4"></a>Для сайта, вы находитесь на контроллере домена или на рабочей станции с установленными средствами удаленного администрирования сервера

1.  Откройте консоль управления групповыми политиками.

2.  В дереве консоли щелкните правой кнопкой мыши сайт, который требуется задать групповую политику.

    **Где?**

    -   Active Directory — сайты и службы [*имя_контроллера_домена.имя_домена*] / Sites/Site

3.  Щелкните запись в **ссылки объекта групповой политики** выберите существующий объект групповой политики (GPO), а затем нажмите кнопку **изменить**. Можно также щелкнуть **New** создать новый объект групповой Политики, а затем нажмите кнопку **изменить**.

4.  В дереве консоли щелкните **политик ограниченного использования программ**.

    **Где**

    -   *Объект групповой политики* [*ComputerName*] Policy/Computer Configuration или

        Политики ограниченного использования программ параметры или программным обеспечением безопасности Конфигурация Windows пользователя

> [!NOTE]
> -   Для выполнения этой процедуры необходимо быть членом группы администраторов на локальном компьютере, либо вам должны быть делегированы соответствующие полномочия. Если компьютер присоединен к домену, члены группы администраторов домена смогут для выполнения этой процедуры.
> -   Чтобы задать параметры политики, которые будут применяться к компьютерам независимо от пользователей, входящих в систему, щелкните **Конфигурация компьютера**.
> -   Чтобы задать параметры политики, которые будут применяться к пользователям независимо от того, какой компьютер они входят в систему, щелкните **Конфигурация пользователя**.

## <a name="BKMK_Create_SRP"></a>Чтобы создать новые политики ограниченного использования программ

1.  Открыть политики ограниченного использования программ.

2.  На **действие** меню, нажмите кнопку **новых политик ограниченного использования программ**.

> [!WARNING]
> -   Для выполнения этой процедуры, в зависимости от среды требуются различные административные учетные данные:
> 
>     -   При создании новых политик ограниченного использования программ для локального компьютера: членство в локальной **Администраторы** группы, либо эквивалентной, является минимальным требованием для выполнения этой процедуры.
>     -   При создании новых политик ограниченного использования программ для компьютера, присоединенного к домену, члены группы администраторов домена могут выполнять эту процедуру.
> -   Если политики ограниченного использования программ уже созданы для объект групповой политики (GPO), **новых политик ограниченного использования программ** не отображается команда **действие** меню. Для удаления политик ограниченного использования программ, которые применяются к объекту групповой Политики в дереве консоли щелкните правой кнопкой мыши **политик ограниченного использования программ**, а затем нажмите кнопку **удалить политики ограниченного использования программ**. При удалении политик ограниченного использования программ для объекта групповой Политики также удаляются все правила политик ограниченного использования программ для этого объекта групповой Политики. После удаления политик ограниченного использования программ можно создать новые политики ограниченного использования программ для этого объекта групповой Политики.

## <a name="BKMK_Add_Del"></a>Добавление или удаление назначенного типа файлов

1.  Открыть политики ограниченного использования программ.

2.  В области сведений дважды щелкните **назначенные типы файлов**.

3.  Выполните одно из следующих действий.

    -   Чтобы добавить тип файла в **расширение имени файла**, введите расширение имени файла и нажмите кнопку **добавить**.

    -   Чтобы удалить тип файла в **назначенных типов файлов**, выберите тип файла и нажмите кнопку **удалить**.

> [!NOTE]
> -   Для выполнения этой процедуры, в зависимости от среды, в которой добавляется или удаляется назначенный тип файлов требуются различные административные учетные данные:
> 
>     -   Если добавляется или удаляется назначенный тип файлов для локального компьютера: членство в локальной **Администраторы** группы, либо эквивалентной, является минимальным требованием для выполнения этой процедуры.
>     -   При создании новых политик ограниченного использования программ для компьютера, присоединенного к домену, члены группы администраторов домена могут выполнять эту процедуру.
> -   Может потребоваться создать новый параметр политики ограниченного использования программ для объекта групповой политики (GPO), если вы еще не выполнена.
> -   Список назначенных типов файлов является общим для всех правил для конфигурации компьютера и конфигурации пользователя для объекта групповой Политики.

## <a name="BKMK_Prevent_Admin"></a>Запрет применения к локальным администраторам политик ограниченного использования программ

1.  Открыть политики ограниченного использования программ.

2.  В области сведений дважды щелкните **принудительного применения**.

3.  В разделе **применения политик ограниченного использования программ к следующим пользователям**, нажмите кнопку **всех пользователей, кроме локальных администраторов**.

> [!WARNING]
> -   Членство в локальной **Администраторы** группы, либо эквивалентной, является минимальным требованием для выполнения этой процедуры.
> -   Может потребоваться создать новый параметр политики ограниченного использования программ для объекта групповой политики (GPO), если вы еще не выполнена.
> -   Если многие пользователи будут членами локальной группы администраторов на своих компьютерах в вашей организации, вы не хотите включать этот параметр.
> -   При задании политики ограниченного использования программ для локального компьютера, используйте данную процедуру, чтобы политики ограниченного использования программ применяются к их локальных администраторов. При задании политики ограниченного использования программ для сети, отфильтруйте параметры политики пользователя, на основе членства в группах безопасности с помощью групповой политики.

## <a name="BKMK_Sec_Lvl"></a>Изменение уровня безопасности по умолчанию для политик ограниченного использования программ

1.  Открыть политики ограниченного использования программ.

2.  В области сведений дважды щелкните **уровни безопасности**.

3.  Щелкните правой кнопкой мыши уровень безопасности, который вы хотите использовать по умолчанию, а затем нажмите кнопку **по умолчанию**.

> [!CAUTION]
> В некоторых каталогах уровень безопасности по умолчанию для **запрещено** может отрицательно сказаться на вашей операционной системы.

> [!NOTE]
> -   Для выполнения этой процедуры, в зависимости от среды, в которой изменяется уровень безопасности по умолчанию политики ограниченного использования программ требуются различные административные учетные данные.
> -   Может потребоваться создать новый параметр политики ограниченного использования программ для этой групповой политики объект (GPO), если вы еще не выполнена.
> -   В области сведений текущий уровень безопасности по умолчанию обозначается черным кружком с отметкой. Если щелкнуть правой кнопкой мыши текущий уровень безопасности по умолчанию **по умолчанию** команда отсутствует в меню.
> -   Чтобы задать исключения для уровня безопасности по умолчанию, создаются правила политики ограниченного использования программ. Если уровень безопасности по умолчанию присвоено **Unrestricted**, правилах можно указывать программное обеспечение, которое не разрешено запускать. Если уровень безопасности по умолчанию присвоено **запрещено**, правилах можно указывать программное обеспечение, которое разрешено запускать.
> -   Во время установки, значение уровня безопасности по умолчанию для политик ограниченного использования программ для всех файлов в системе **Unrestricted**.

## <a name="BKMK_Apply_SRP_DLLs"></a>Применение политик ограниченного использования программ к библиотекам DLL

1.  Открыть политики ограниченного использования программ.

2.  В области сведений дважды щелкните **принудительного применения**.

3.  В разделе **применения политик ограниченного использования программ для следующих**, нажмите кнопку **ко всем файлам программ**.

> [!NOTE]
> -   Для выполнения этой процедуры необходимо быть членом группы администраторов на локальном компьютере, либо вам должны быть делегированы соответствующие полномочия. Если компьютер присоединен к домену, члены группы администраторов домена смогут для выполнения этой процедуры.
> -   По умолчанию политики ограниченного использования программ не устанавливайте библиотек динамической компоновки (DLL). Проверка библиотек DLL может понизить производительность системы, так как политики ограниченного использования программ должны вычисляться при каждой загрузке библиотеки DLL. Тем не менее Иногда целесообразно проверку библиотек DLL, если есть опасность заражения вирусом, затрагивающим библиотеки DLL. Если уровень безопасности по умолчанию имеет значение **запрещено**, и включена проверка библиотек DLL, необходимо создать правила политики, которые разрешают выполнение каждой библиотеки DLL, ограниченного использования программ.
