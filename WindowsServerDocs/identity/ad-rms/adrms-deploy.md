# <a name="upgrading-ad-rms-to-windows-server-2016"></a>Обновление до Windows Server 2016 AD RMS

## <a name="introduction"></a>Введение

Службы управления правами Active Directory (AD RMS) — это служба Майкрософт, которая защищает конфиденциальные документы и электронные письма. В отличие от традиционных защиты методов, таких как брандмауэры и списки управления доступом, шифрование AD RMS и защита закрепляются независимо от того, где выходит из файла или как их передаче. 

В этом документе приводятся инструкции по миграции Windows Server 2012 R2 с SQL Server 2012 для Windows Server 2016 и SQL Server 2016. Тот же процесс используется для переноса из старый, но поддерживаемых версий AD RMS.
Обратите внимание, что службы управления правами Active Directory больше не находится в активной разработке, и для новейшие возможности клиентам стоит рассмотреть переход к [Azure Information Protection](https://azure.microsoft.com/services/information-protection/), который включает многое другое комплексный набор функций с более полный устройства и поддержку для приложения. 

Сведения о миграции из AD RMS на Azure Information Protection без необходимости повторно защитить содержимое см. в разделе [документации по миграции Azure Information Protection](https://docs.microsoft.com/azure/information-protection/migrate-from-ad-rms-to-azure-rms).

## <a name="about-the-environment-used-in-this-guide"></a>О среде, используемые в данном руководстве

AD FS является дополнительный компонент установки AD RMS. В этом руководстве подразумевается использование служб федерации Active Directory. Если служб федерации Active Directory не использовался в вашей среде для поддержки пользователей AD RMS, можно пропустить все шаги, которые ссылаются на служб федерации Active Directory.

В этом руководстве SQL Server обновляется до SQL Server 2016, выполняя параллельную установку и перемещение баз данных с помощью резервной копии. Кроме того Если службы AD RMS и AD FS серверах баз данных можно обновить до SQL Server 2016 на месте, можно переместить к следующему разделу, в этом документе после выполнения, без необходимости выполните действия, описанные в этом разделе.  

## <a name="installation"></a>Установка

### <a name="configuring-sql-server-2016"></a>Настройка SQL Server 2016

Следующие задачи по реализации сведения о разделе, связанных непосредственно с конфигурацией SQL Server 2016. В этом руководстве рассматривается использование диспетчера сервера и SQL Server Management Studio для выполнения этих задач.

Эти действия необходимо выполнить для установки SQL Server 2016. Установите SQL Server 2016 на оборудование согласно стандартные методы вашей организации и политик. 

#### <a name="preparing-the-sql-server"></a>Подготовка SQL Server

В следующем разделе описаны как подготовить SQL Server, таким образом, чтобы его можно обновить до SQL Server 2016 перед обновлением других служб на платформе службы AD RMS для использования Windows Server 2016.

##### <a name="adding-cname-for-sql-server-2016-to-dns"></a>Добавление DNS запись CNAME для SQL Server 2016

Запись CNAME позволяет гарантировать, что установки Windows Server 2016 получите соответствующие данные, так как он будет указывать на новый SQL Server 2016. **Примечание. Если уже используется запись CNAME для службы служб федерации Active Directory и AD RMS, можно перейти к дальнейшим действиям.**


**Чтобы добавить запись CNAME для SQL Server 2016 в DNS**

1.  Войдите на контроллере Windows Server 2012 R2 домен с учетными данными администратора домена.

2.  Откройте диспетчер сервера.

3.  Нажмите кнопку **средства** и выберите **DNS** чтобы открыть диспетчер DNS.

4.  На панели навигации слева разверните контроллер домена и открою **зоны прямого просмотра**.

5.  Открыть ресурсы соответствующего домена, а затем щелкните правой кнопкой мыши в области просмотра справа и выберите **новый псевдоним (CNAME)** , чтобы создать запись CNAME.

6.  Введите имя псевдонима в логическое имя, чтобы отличать его от других, могут присутствовать (например: SQLADRMS или SQLADFS)

7.  После ввода имени, укажите полное доменное имя целевого узла, которое будет иметь на новый сервер SQL Server 2016. (например) SQL2016.contoso.com)

8.  После ввода всех сведений щелкните **ОК**.

##### <a name="backup-the-ad-rms-and-adfs-databases"></a>Резервное копирование баз данных служб федерации Active Directory и AD RMS

Базы данных AD RMS и AD FS содержаться критически важные сведения, необходимые для AD RMS, например, открытый ключ сертификата лицензиара сервера, шаблоны политики прав, данные конфигурации служб федерации Active Directory и сведения о ведении журнала. Без этих баз данных клиенты не могут выдавать лицензии для использования защищенного контента среди других проблем.

Баз данных, база данных конфигурации AD RMS считается наиболее важным, так как он сохраняет сертификат лицензиара сервера, права шаблоны политики пользователей ключи и сведения о конфигурации. Таким образом хотя следует позаботиться для резервного копирования всех баз данных AD RMS и AD FS, необходимо регулярно создавайте резервные копии базы данных конфигурации.

Ведение журнала базы данных хранятся сведения о запросы пользователей в кластер AD RMS для сертификатов и лицензий на использование. Стратегии резервного копирования этой базы данных должно быть основано на политику компании для сохранения данных этого типа.

Базы данных службы каталогов не влияет на функциональность AD RMS и, если последние данные в случае утери базы данных будет повторно заполнить сведения, как сервер AD RMS получает запросы на сертификаты и лицензии на использование. Необходимо создать резервную копию этой базы данных, регулярно, но необходимо иметь по крайней мере копию базы данных, так как она была изначально настроена после развертывания службы AD RMS.

**Чтобы создать резервную копию базы данных AD RMS и/или служб федерации Active Directory с Microsoft SQL Server**

1.  Войдите на сервер базы данных Windows Server 2012 R2 AD RMS с SQL 2012.

2.  Нажмите кнопку **запустить**, нажмите кнопку **все программы**, нажмите кнопку **Microsoft SQL Server**и нажмите кнопку **SQL Server Management Studio**.

3.  В **соединение с сервером** окне подтверждения сервер размещения базы данных AD RMS в **имя сервера** поле и нажмите кнопку **Connect**.

4.  Разверните **баз данных**. Щелкните правой кнопкой мыши соответствующую базу данных (**системы DRM** и **Adfs**), пункты **задачи**и выберите **резервного копирования**.

5.  Повторите шаг 4 для оставшихся баз данных.

6.  Убедитесь, что резервная копия базы данных может осуществляться с других компьютеров в сети или с помощью запоминающего устройства, как они потребуются для выполнения дальнейших действий во время миграции.

Теперь вы можете хранить копии базы данных в безопасном месте. Не забудьте часто создавать резервные копии баз данных.

##### <a name="adding-domain-admin-sql-ad-rms-andor-adfs-service-account-to-sql-server-2016"></a>Добавление администратора домена, SQL, AD RMS и/или службы ADFS учетную запись SQL Server 2016

Следующие действия будут показаны способы добавления различных учетных записей служб SQL Server 2016 для облегчения переноса данных из среды Windows Server 2012 R2. Это позволит получить соответствующие разрешения, при попытке доступа к содержимому и обработки данных.

**Чтобы добавить администратора домена, SQL, AD RMS и служб федерации Active Directory учетной записи службы SQL Server**

1.  Войдите на сервер с SQL Server 2016 как учетная запись локального администратора.

2.  Нажмите кнопку **запустить**, нажмите кнопку **все программы**, нажмите кнопку **Microsoft SQL Server**и нажмите кнопку **SQL Server Management Studio**.

3.  В **соединение с сервером** окне подтверждения сервер размещения базы данных AD RMS в **имя сервера** поле, а затем для проверки подлинности щелкните раскрывающееся меню и выберите **SQL Server Проверка подлинности**.

4.  В **входа** поле введите имя учетной записи локального администратора (например) localadmin) и затем укажите соответствующий пароль и нажмите кнопку **Connect**.

5.  Разверните **безопасности** и щелкните правой кнопкой мыши **имена входа** и выберите **новое имя входа** в контекстном меню, появившемся.

6.  Когда появится окно, введите в учетной записи администратора домена в **имя входа** поля (например) Contoso\\ContosoAdmin)

7.  На панели навигации слева, выберите **ролей сервера**.

8.  Затем установите флажок для **sysadmin** в группе ролей сервера и нажмите кнопку **ОК**.

9.  Перезапустите **управления SQL Server**.

10. В **соединение с сервером** окне подтверждения сервер размещения базы данных AD RMS в **имя сервера** поле, а затем для проверки подлинности щелкните раскрывающееся меню и выберите **Windows Проверка подлинности** и нажмите кнопку **Connect**.

##### <a name="restoring-the-ad-rms-and-adfs-databases-to-sql-server-2016"></a>Восстановление AD RMS и баз данных служб федерации Active Directory для SQL Server 2016

Следующие действия будут показаны как восстановить данные из предыдущего экземпляра SQL Server к новому экземпляру 2016. Таким образом, текст SQL для использования конфигураций, связанных данных из предыдущих баз данных AD RMS и AD FS.

**Чтобы восстановить данные из предыдущих SQL Server на новом сервере SQL**

1.  Войдите на сервер с SQL Server 2016, с помощью соответствующей учетной записи.

2.  На панели навигации слева щелкните правой кнопкой мыши **баз данных** и выберите **Restore Database** чтобы начать процесс восстановления.

3.  В разделе **источника** выберите **устройства** и затем перейдите к расположению, где были сохранены файлы базы данных на предыдущих этапах.

4.  После выбора файлов нажмите кнопку **ОК**.

5.  Убедитесь, что все файлы базы данных были добавлены и завершить процесс, нажав кнопку **ОК**.

### <a name="configuring-windows-server-2016-active-directory-federation-services-ad-fs"></a>Настройка служб федерации Active Directory Windows Server 2016 (AD FS)

Службы федерации Active Directory для предоставления единого входа (SSO) для доступа к AD RMS как приложение будет развернута. Она также настроена с AD RMS мобильного устройства расширения (MDE), позволяющая Mac и поддержка мобильных устройств для конечных пользователей.

Следующие разделы содержат руководство по на рабочие задачи, которые могут потребоваться для выполнения развертывания AD FS.

#### <a name="adding-a-2016-ad-fs-server-to-the-farm"></a>Добавление сервера 2016 AD FS в ферме

Вы можете развернуть дополнительные серверы AD FS для поддержки развертывания службы AD RMS. Вы можете выполнить это действие в случае большего объема трафика к серверам службы AD RMS или других приложениях, или если требуется снять с учета, один из серверов, используемые для служб AD FS.

**Чтобы добавить в ферму сервера 2016 AD FS**

1.  С сервера Azure AD Connect, дважды щелкните **Azure AD Connect** значок, чтобы запустить мастер Azure AD Connect.

2.  На странице приветствия щелкните **Настройка**.

3.  На странице Дополнительные задачи, щелкните **развернуть дополнительный сервер федерации** и нажмите кнопку **Далее**.

4.  На странице подключение к Azure AD, введите имя пользователя и пароль учетной записи с разрешениями глобального администратора и нажмите кнопку **Далее**.

5.  На странице учетных данных администратора домена, введите имя пользователя и пароль учетной записи с разрешениями администратора домена и нажмите кнопку **Далее**.

6.  Нажмите кнопку **Обзор** и выберите файл сертификата, который используется при настройке фермы AD FS, с помощью Azure AD Connect.

7.  Нажмите кнопку **ввод пароля** чтобы открыть диалоговое окно Пароль сертификата.

8.  Введите пароль сертификата в соответствующее поле и нажмите кнопку **ОК**.

9.  Нажмите кнопку **Далее**.

10. На странице серверов AD FS, введите имя или IP-адрес нового сервера AD FS и нажмите кнопку **добавить**.

11. Готовы к странице "Настройка", нажмите кнопку **установить**.

12. На странице «Установка завершена» щелкните **выхода**.

#### <a name="raising-the-adfs-farm-behavior-level"></a>Повышение уровня поведение фермы служб федерации Active Directory

При развертывании это сервер служб федерации Active Directory, который превышает текущий уровень среды, такие как наличие служб федерации Active Directory на Windows Server 2012 R2 и затем Добавление служб федерации Active Directory Windows Server 2016, на уровне поведения фермы будет необходимо увеличить. Это необходимо, чтобы убедиться, что в окружении используется наиболее актуальные сведения и функции.

**Чтобы повысить режим поведения фермы служб федерации Active Directory**

1.  Перейдите к Windows Server 2016 ADFS.

2.  Откройте сеанс администратора PowerShell.

3.  Введите следующую команду:  **\$cred = Get-Credential**

4.  Откроется окно с запросом учетных данных, введите учетные данные администратора домена.

5.  Затем введите следующую команду: **Вызвать AdfsFarmBehaviorLevelRaise-Credential \$cred**

6.  Появится запрос, запрос **вы хотите продолжить эту операцию?** Затем введите **** чтобы принять приглашение.

7.  После завершения команды на уровне поведения фермы будет установки и Готово.

#### <a name="enabling-mobile-device-extension-logging"></a>Включение ведения журнала расширение мобильных устройств

Расширение для мобильных устройств может регистрировать запросы, получаемые с устройств конечных пользователей. Ведение журнала отключено по умолчанию и рекомендуется только Включение ведения журналов для сценария устранения неполадок. Все запросы от мобильных устройств и настольных компьютеров, чтобы приобрести лицензии на использование или начальной загрузки записываются в базу данных журналов AD RMS или учетной записи хранения Azure. Ведение журнала MDE создаст два дополнительных таблиц для SQL Server, используемый службой AD RMS: клиент отладки таблицы журнала и таблицу журнала производительности клиента.

**Чтобы включить ведение журнала расширение для мобильных устройств**

1.  С сервера AD RMS откройте Windows PowerShell с правами администратора.

2.  Введите следующую команду и нажмите клавишу **ввод**: **Import-Module AdRmsAdmin**

3.  Введите следующую команду и нажмите клавишу **ввод**: **New-PSDrive-Name AdRmsAdmin - PsProvider AdrmsCluster-корневой https://localhost**

4.  Введите следующую команду и нажмите клавишу **ввод**: **Командлет Set-ItemProperty-Path AdrmsCluster:\\ -IsLoggingEnabled имя-значение \$true**

Если вы используете MDE ведения журнала для устранения неполадок, рекомендуется отключить его после решения проблемы.

**Чтобы отключить ведение журнала расширение для мобильных устройств**

1.  С сервера AD RMS откройте Windows PowerShell с правами администратора.

2.  Введите следующую команду и нажмите клавишу **ввод**: **Import-Module AdRmsAdmin**

3.  Введите следующую команду и нажмите клавишу **ввод**: **New-PSDrive-Name AdRmsAdmin - PsProvider AdrmsCluster-корневой https://localhost**

4.  Введите следующую команду и нажмите клавишу **ввод**: **Командлет Set-ItemProperty-Path AdrmsCluster:\\ -IsLoggingEnabled имя-значение \$false**

### <a name="upgrading-ad-rms-to-windows-server-2016"></a>Обновление до Windows Server 2016 AD RMS

Следующие разделы содержат рекомендации о том, как добавить сервер на базе Windows Server 2016 AD RMS в текущем кластере Windows Server 2012 R2. Сервер будет добавлен в кластер, и данные будут реплицированы на его таким образом, чтобы предыдущего сервера AD RMS может считаться устаревшей для высвобождения ресурсов.

После добавления, AD RMS на основе Windows Server 2016 сервер был добавлен в кластер AD RMS, все узлы, в зависимости от более старых версиях Windows становится неактивным. После этого можно отозвать эти серверы (например завершение работы, повторного использования или переустановить с Windows Server 2016 будет добавлен в кластер AD RMS). 

Можно развернуть дополнительные серверы AD RMS в кластер для поддержки нагрузки по развертыванию AD RMS. Можно также выполнить это действие в случае большего объема трафика на серверы AD RMS.

В данном руководстве не рассматривается шаги, необходимые для изменения механизма, который может использоваться в вашей среде для исключения серверов, которые стали неподдерживаемыми, так и для включения тех, которые вы добавляете в кластер балансировки нагрузки. 

#### <a name="adding-a-2016-ad-rms-server"></a>Добавление сервера 2016 AD RMS

Если ваш кластер AD RMS использует аппаратный модуль безопасности вместо централизованно управляемого ключа для сертификата лицензиара сервера, необходимо установить программное обеспечение и другие артефакты аппаратного модуля безопасности (например ключ и конфигурации, файлы) на сервере перед установкой AD RMS. Также необходимо будет подключиться к серверу, в аппаратный модуль безопасности, либо физически, либо с помощью соответствующих сетевых конфигураций. Следуйте инструкциям в HSM следующие действия. 

**Чтобы добавить сервер 2016 AD RMS**

1.  Установите роль AD RMS на требуемое развертывание Windows Server 2016.

2.  После завершения установки щелкните ссылку, чтобы **выполнить дополнительную настройку**.

3.  Выберите **Добавление в существующий кластер AD RMS** и нажмите кнопку **Далее**.

4.  На **Выбор базы данных конфигурации** введите запись CNAME, указанный в DNS для SQL server 2016 (FQDN).

5.  Нажмите кнопку **списка** на второй строке и выберите **DefaultInstance** из раскрывающегося списка.

6.  В разделе **имя базы данных конфигурации**выберите раскрывающееся меню и выберите конфигурацию системы DRM, отображается. Затем нажмите кнопку **Далее**.

7.  На **сведения о базе данных** введите пароль ключа кластера в соответствующем поле. После этого нажмите кнопку **Далее**.

28. После добавления другого сервера, установите флажки и задайте серверу-получателю, чтобы быть доступной для чтения получателя.

29. Перейдите к **конечные точки** вкладку и нажмите кнопку **обновить** параметр. Во время также здесь при прокрутке и убедитесь, что одну и ту же учетную запись службы на узле первичной и вторичной.

30. Теперь выберите **настройки резервного копирования** и выберите **предпочитать-получателя** параметр.

31. Перейти к **прослушивателя** вкладки.

32. Укажите имя (например) SQLListener) и убедитесь, что порт **1433** и нажмите кнопку **Далее**.

33. В **Выбор начальной синхронизации данных** странице мастера выберите **полный** и укажите сетевую папку, доступную для всех серверов SQL и нажмите кнопку **Далее**.

34. Наконец, нажмите кнопку **Готово** и процесс будет завершен.

### <a name="decommission-windows-server-2012-r2-nodes"></a>Списание узлов Windows Server 2012 R2

Следующие разделы содержат руководство по на рабочие задачи, может потребоваться удалить серверы Windows Server 2012 R2 после успешного обновления кластер AD RMS для Windows Server 2016.

#### <a name="removing-a-windows-server-2012-r2-ad-rms-server"></a>Удаление сервера Windows Server 2012 R2 AD RMS

После обновления можно удалить ненужные серверов AD RMS. Вы можете на выполнение этого действия, когда он становится необходимо списание серверов AD RMS.

**Чтобы удалить** **сервера Windows Server 2012 R2 AD RMS**

1.  На сервере Windows Server 2012 R2 AD RMS в диспетчере серверов выберите **управление** в верхней правой кнопкой мыши меню, а затем выберите **удалить роли и компоненты**.

2.  **Мастера удаления ролей и компонентов** откроется вверх и на **перед началом работы** щелкните **Далее**.

3.  На **Выбор сервера** экрана, нажмите кнопку **Далее**.

4.  На **ролей сервера** экрана, снимите флажок рядом с полем **службы управления правами Active Directory** и нажмите кнопку **Далее**.

5.  На **функции** экрана, нажмите кнопку **Далее**.

6.  На **Подтверждение** экрана, нажмите кнопку **удалить**.

7.  По завершении перезапустите сервер.

8.  Теперь можно завершить работу этого сервера и перераспределении ресурсов при необходимости.
