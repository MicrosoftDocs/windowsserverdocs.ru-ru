# <a name="forest-and-domain-functional-levels"></a>Режимы работы леса и домена

>Область применения. Windows Server

Функциональные уровни определяют доступные возможности домена или леса домен Active Directory Services (AD DS). Они также определяют, какие операционные системы Windows Server можно запускать на контроллерах домена в домене или лесу. Однако функциональные уровни не влияют на то, какие операционные системы можно запускать на рабочих станциях и рядовых серверах, присоединенных к домену или лесу.

При развертывании AD DS задайте для функциональных уровней домена и леса наибольшее значение, которое может поддерживать среда. Таким образом можно использовать как можно больше функций AD DS. При развертывании нового леса вам будет предложено задать функциональный уровень леса, а затем задать режим работы домена. Для режима работы домена можно задать значение, превышающее функциональный уровень леса, но нельзя задать для режима работы домена значение ниже режима работы леса.

После окончания жизненного цикла Windows 2003 контроллеры домена Windows 2003 необходимо обновить до Windows Server 2008, 2008R2, 2012, 2012 R2, 2016 или 2019. В результате все контроллеры домена под управлением Windows Server 2003 должны быть удалены из домена.

На режимах работы домена Windows Server 2008 и более поздних версий репликация распределенных файловых служб (DFS) используется для репликации содержимого папки SYSVOL между контроллерами домена. Если вы создаете новый домен на функциональном уровне домена Windows Server 2008 или выше, репликация DFS автоматически используется для репликации SYSVOL. Если вы создали домен на более низком функциональном уровне, вам потребуется выполнить миграцию с репликации FRS на репликацию DFS для SYSVOL. Чтобы выполнить миграцию, выполните процедуры, описанные [на сайте TechNet](https://technet.microsoft.com/library/dd640019(v=WS.10).aspx) , или воспользуйтесь [упрощенным набором действий в блоге о хранилище файлов группы хранения](http://blogs.technet.com/b/filecab/archive/2014/06/25/streamlined-migration-of-frs-to-dfsr-sysvol.aspx).

## <a name="windows-server-2019"></a>Windows Server 2019

В этом выпуске не Добавлено новых функциональных уровней леса или домена.

Минимальным требованием для добавления контроллера домена Windows Server 2019 является функциональный уровень Windows Server 2008. Домен также должен использовать DFS-R в качестве подсистемы репликации SYSVOL.

     * Время последней неудачной попытки входа на рабочую станцию Windows Server 2008 или Windows Vista
     * Время последней успешной попытки входа на сервер Windows Server 2008 или на рабочую станцию Windows Vista

[в разделе Пошаговое руководств по настройке политики блокировки учетных записей и паролей](https://go.microsoft.com/fwlink/?LinkID=91477).

      - Репликация связанных значений позволяет изменять членство в группах для хранения и репликации значений отдельных членов вместо репликации всего членства в виде единого блока. Хранение и репликация значений отдельных членов использует меньше пропускной способности сети и уменьшает количество циклов процессора во время репликации, а также предотвращает потерю обновлений при одновременном добавлении или удалении нескольких членов на разных контроллерах домена.
   * Возможность развертывания контроллера домена только для чтения (RODC)
   * Улучшенные алгоритмы и масштабируемость средства проверки согласованности знаний (KCC)
      - Генератор межсайтовых топологий (ISTG) использует улучшенные алгоритмы, масштабируемые для поддержки лесов с большим количеством сайтов, чем AD DS может поддерживаться на функциональном уровне леса Windows 2000. Улучшенный алгоритм выборов ISTG является менее агрессивным механизмом выбора ISTG на функциональном уровне леса Windows 2000.
   * Возможность создания экземпляров динамического вспомогательного класса с именем **DynamicObject** в разделе каталога домена
   * Возможность преобразования экземпляра объекта **InetOrgPerson** в экземпляр объекта **пользователя** и для завершения преобразования в обратном направлении.
   * Возможность создавать экземпляры новых типов групп для поддержки авторизации на основе ролей. 
      - Эти типы называются группами приложений Basic и группами запросов LDAP.
   * деактивация и переопределение атрибутов и классов схемы. Следующие атрибуты можно использовать повторно: ldapDisplayName, schemaIdGuid, OID и mapiID.
   * Пространства имен DFS на основе домена, работающие в режиме Windows Server 2008, включая поддержку перечисления на основе доступа и повышенную масштабируемость. Дополнительные сведения см. [в разделе Выбор типа пространства имен](https://go.microsoft.com/fwlink/?LinkId=180400).

### <a name="windows-server-2003-domain-functional-level-features"></a>Функции функционального уровня домена Windows Server 2003

* Все функции AD DS по умолчанию, все функции, доступные на функциональном уровне собственного домена Windows 2000, и доступны следующие функции:
   * Средство управления доменами Netdom. exe, позволяющее переименовывать контроллеры домена.
   * Обновления метки времени входа
      * Атрибут **lastLogonTimestamp** обновляется с учетом времени последнего входа пользователя или компьютера. Этот атрибут реплицируется внутри домена.
   * Возможность задать атрибут **userpassword** в качестве действующего пароля для **InetOrgPerson** и объектов User
   * Возможность перенаправлять контейнеры пользователей и компьютеров
      * По умолчанию используются два хорошо известных контейнера для размещения учетных записей компьютеров и пользователей, а именно, CN = Computers<domain root> и CN =<domain root>Users. Эта функция позволяет определить новое, хорошо известное расположение этих учетных записей.
   * Возможность диспетчера авторизации сохранять свои политики авторизации в AD DS
   * Ограниченное делегирование
      * Ограниченное делегирование позволяет приложениям использовать преимущества безопасного делегирования учетных данных пользователя с помощью проверки подлинности на основе Kerberos.
      * Делегирование можно ограничить только конкретными целевыми службами.
   * Выборочная проверка подлинности
      * Выборочная проверка подлинности позволяет указать пользователей и группы из доверенного леса, которым разрешено проходить проверку подлинности на серверах ресурсов в доверяющем лесу.
