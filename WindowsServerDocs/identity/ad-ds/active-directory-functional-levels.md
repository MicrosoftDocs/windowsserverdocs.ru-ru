# <a name="forest-and-domain-functional-levels"></a>Функциональные уровни леса и домена

>Область применения. Windows Server

Функциональные уровни определяют доступные возможности домена или леса доменных служб Active Directory (AD DS). Они также определяют, какие операционные системы Windows Server, можно запустить на контроллерах домена в домене или лесу. Тем не менее функциональные уровни не влияют на каких операционных систем, можно запустить на рабочих станциях и серверах, входящих в домен или лес.

При развертывании AD DS, равным наибольшее значение, которое может поддерживать среду режимы работы домена и леса. Таким образом, можно использовать столько функций AD DS максимально. При развертывании нового леса, приглашение задать режим работы леса и задайте режим работы домена. Режим работы домена может быть присвоено значение, больше, чем режим работы леса, но режим работы домена не может быть присвоено значение, которое меньше, чем режим работы леса.

Конец жизненного цикла Windows 2003, нужно обновить до Windows Server 2008, 2008 R2, 2012, Windows 2003-контроллеров доменов (DC) 2012 R2, 2016 или 2019 г. Таким образом любой контроллер домена под управлением Windows Server 2003 должны удаляться из домена.

В Windows Server 2008 и более поздней версии режимы работы домена репликации распределенной файловой службы (DFS) используется для репликации содержимого папки SYSVOL между контроллерами домена. При создании нового домена в режиме работы домена Windows Server 2008 или более поздней версии, служба репликации DFS автоматически используется для репликации SYSVOL. Если вы создали домен на более низком функциональном уровне, необходимо будет переход от средств FRS на DFS для SYSVOL. Этапы миграции, вы можете либо выполните [процедуры на сайте TechNet](https://technet.microsoft.com/library/dd640019(v=WS.10).aspx) или можно ссылаться на [упрощенная схема набор шагов в блоге CAB-файла группы хранения](http://blogs.technet.com/b/filecab/archive/2014/06/25/streamlined-migration-of-frs-to-dfsr-sysvol.aspx).

## <a name="windows-server-2019"></a>Windows Server 2019

Не существует новый лес или режимы работы домена, добавленных в данном выпуске.

Минимальное требование для добавления контроллера домена Windows Server 2019 — на режим работы Windows Server 2008. Домен должен также использовать DFS-R в качестве подсистемы для репликации SYSVOL.

   * Средства управления доменом Netdom.exe, которое позволяет переименовать контроллеров домена
   * Обновляет отметку времени входа в систему
      * Атрибут **lastLogonTimestamp** обновляется с учетом времени последнего входа пользователя или компьютера. Этот атрибут реплицируется внутри домена.

      * Вы можете ограничить делегирование только определенных целевых служб.
   * Выборочная проверка подлинности
      * Выборочная проверка подлинности упрощает позволяет указать пользователей и группы из доверенного леса, которым разрешена проверка подлинности на серверах ресурсов в доверяющем лесу.
