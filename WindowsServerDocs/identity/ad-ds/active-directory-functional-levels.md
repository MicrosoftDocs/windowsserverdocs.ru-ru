# <a name="forest-and-domain-functional-levels"></a>Режимы работы леса и домена

>Область применения: Windows Server

В конце жизненного цикла Windows 2003 Windows 2003 контроллеры домена (DC) должны быть обновлены до Windows Server 2008, 2012 или 2016. В результате любой контроллер домена под управлением Windows Server 2003 следует удалить из домена. Режим работы домена и леса должен быть повышен до по крайней мере Windows Server 2008 на контроллере домена под управлением более ранней версии Windows Server будут добавляться в среде.

Рекомендуется, чтобы обновлять свои режима работы домена (DFL) и режим работы леса (FFL) в рамках этого клиентов, поскольку 2003 DFL и FFL отмечены как устаревшие в Windows Server 2016, и они больше не будет поддерживаться в будущих выпусках.

Для клиентов, которым требуется дополнительное время для оценки перемещение их DFL & FFL из Windows Server 2003, 2003 DFL и FFL будет поддерживаться в Windows 10 и Windows Server 2016 предоставляются все контроллеры домена в домене и лесу, либо на Windows Server 2008, 2008 R2, 2012, 2012 R2, или 2016.

В Windows Server 2008 и более высокие функциональные уровни домена репликации распределенной файловой службы (DFS) используется для репликации между контроллерами домена содержимое папки SYSVOL. При создании нового домена в режим работы домена Windows Server 2008 или более поздней версии, автоматически используется репликация DFS для репликации SYSVOL. При создании домена на более низком уровне режима работы, необходимо перенести с помощью FRS для репликации DFS для SYSVOL. Для шагов миграции, вы можете либо выполните [процедуры на сайте TechNet](https://technet.microsoft.com/library/dd640019(v=WS.10).aspx) или можно сослаться на [упрощен набор действий в блоге CAB-файла группы хранения](http://blogs.technet.com/b/filecab/archive/2014/06/25/streamlined-migration-of-frs-to-dfsr-sysvol.aspx).

Windows Server 2003 леса режимы работы домена и дальше поддерживаться, но организации должны повысить режим работы Windows Server 2008 (или более поздней версии, если возможно) для обеспечения совместимости репликации SYSVOL и поддержки в будущем. Кроме того доступны многие другие преимущества и возможности на более высоких функциональных уровнях выше. Ознакомьтесь со следующими ресурсами, Дополнительные сведения:

    * [Управления привилегированным доступом (PAM) с помощью Microsoft Identity Manager (MIM)] (https://docs.microsoft.com/windows-server/identity/whats-new-active-directory-domain-services#a-namebkmkpamaprivileged-access-management)

### <a name="windows-server-2016-domain-functional-level-features"></a>Компоненты функционального уровня домена Windows Server 2016

    * Контроллеры домена могут поддерживать последовательное секреты открытого ключа только пользователя NTLM.

    * Клиентов Kerberos, успешной проверки подлинности с расширением актуальности PKInit будут получать новые открытого ключа удостоверения SID.

    Дополнительные сведения см. [новые возможности проверки подлинности Kerberos](https://docs.microsoft.com/windows-server/security/kerberos/whats-new-in-kerberos-authentication) и [новые возможности защиты учетных данных](https://docs.microsoft.com/windows-server/security/credentials-protection-and-management/whats-new-in-credential-protection)

## <a name="windows-server-2012r2"></a>Windows Server 2012 R2

Поддерживаемые операционную систему контроллера домена:

* В Windows Server 2016
* Windows Server 2012 R2

### <a name="windows-server-2012r2-forest-functional-level-features"></a>Windows Server 2012 R2 леса возможности режимов работы

* Все функции, доступные в Windows Server 2012 леса функционального уровня, но не дополнительные возможности.

### <a name="windows-server-2012r2-domain-functional-level-features"></a>Windows Server 2012 R2 домена возможности режимов работы

* Все стандартные возможности Active Directory, все возможности режима работы домена Windows Server 2012, а также следующие возможности:
    * Со стороны контроллера домена меры защиты для защищенных пользователей. Защищенные аутентификации пользователей в Windows Server 2012 R2 домена больше не могут:
        * Проверки подлинности с помощью проверки подлинности NTLM
        * Используйте DES или наборы шифров RC4 в предварительной проверке подлинности Kerberos
        * Делегироваться в рамках неограниченного или ограниченного делегирования
        * Выполнять продление пользовательских билетов (TGT) за время существования начальной 4 часа
    * Политики проверки подлинности
        * Политики на основе нового леса Active Directory, которые могут применяться к учетным записям в доменах Windows Server 2012 R2, чтобы управлять тем, какие узлы учетную запись можно входа и применить условия контроля доступа для проверки подлинности для службы, запущенные от имени учетной записи.
    * Приемники команд политик проверки подлинности
        * На основе нового леса Active Directory объект, который можно создать связь между пользователей, управляемых служб и компьютеров, учетных записей, используемых для классификации учетных записей политики проверки подлинности или для проверки подлинности изоляции.

    * Автоматическое управление имя участника-службы для служб, запущенных на конкретном компьютере в контексте управляемой учетной записи службы, если имя или имя узла DNS изменения учетной записи компьютера. ### <a name="windows-server-2008-domain-functional-level-features"></a>Компоненты функционального уровня домена Windows Server 2008

    * Поддержка репликации распределенной файловой системы (DFS) для Windows Server 2003 системный том (SYSVOL)
        * Поддержка репликации DFS обеспечивает более надежную и управляемую репликацию содержимого SYSVOL.
        [!NOTE]>
        >Начиная с Windows Server 2012 R2, служба репликации файлов (FRS) устарела. Новый домен, созданный на контроллере домена под управлением по крайней мере Windows Server 2012 R2 необходимо установить для режима работы домена Windows Server 2008 или более поздней версии.

    * Доменных пространств имен DFS в Windows Server 2008 режим, который включает поддержку перечисление на основе доступа и повышенную масштабируемость. Доменные пространства имен в режиме Windows Server 2008 также требуют леса, чтобы использовать режим работы леса Windows Server 2003. Дополнительные сведения см. в разделе [выбрать тип пространства имен](https://go.microsoft.com/fwlink/?LinkId=180400).
    * Поддержка расширенного стандарта шифрования (AES 128 и AES-256) для протокола Kerberos. Чтобы TGT для выдачи с помощью AES режим работы домена должен быть Windows Server 2008 или более поздней версии и пароль домена должен быть изменен. 
        * Дополнительные сведения см. в разделе [дополнительные возможности Kerberos](https://technet.microsoft.com/library/cc749438(ws.10).aspx).
        [!NOTE]>
        >Ошибки проверки подлинности могут возникнуть на контроллере домена при повышении режима работы домена до Windows Server 2008 или более поздней версии Если контроллер домена уже реплицированы изменения режиме работы ЛЕСА, но еще не обновили пароля krbtgt. В этом случае перезапуск службы KDC на контроллере домена будет запустить обновление в памяти нового пароля krbtgt и устраните ошибки связанных проверки подлинности.

    * [Интерактивный вход в систему последним](https://go.microsoft.com/fwlink/?LinkId=180387) сведения отображаются следующие сведения:
        * Общее число неудачных попыток входа на сервер присоединен к домену Windows Server 2008 или Windows Vista рабочей станции
        * Общее число неудачных попыток входа после успешного входа на сервер Windows Server 2008 или Windows Vista рабочей станции
        * Время последней попытки ошибки входа в Windows Server 2008 или Windows Vista рабочей станции
        * Попытка время последнего успешного входа на сервер Windows Server 2008 или Windows Vista рабочей станции
    * Детальные политики паролей позволяют указать паролей и политики блокирования учетных для пользователей и глобальных групп безопасности в домене. Дополнительные сведения см. в разделе [пошагового руководства по конфигурации политики блокирования учетных записей и детально регулируемых паролей](https://go.microsoft.com/fwlink/?LinkID=91477).
    * Личные виртуальные рабочие столы
        * Чтобы использовать добавлены функции, предоставляемые вкладке «Личный виртуальный рабочий стол» в диалоговом окне свойств учетной записи пользователя в Active Directory-пользователи и компьютеры, необходимо расширить схему AD DS для Windows Server 2008 R2 (версия объект схемы = 47). Дополнительные сведения см. в разделе [развертывание личных виртуальных рабочих столов путем подключения к удаленным рабочим столам и приложениям RemoteApp с помощью Step-by-Step руководство](https://go.microsoft.com/fwlink/?LinkId=183552).

### <a name="windows-server-2003-forest-functional-level-features"></a>Возможности работы уровня леса Windows Server 2003

* Доступны все возможности по умолчанию AD DS и следующие возможности:
    * Доверие между лесами
    * Переименование домена
    * Репликация связанного значения
        - Репликация связанного значения позволяет изменить членство в группах для хранения и репликация значений отдельных членов вместо репликации всего членства как отдельного элемента. Хранение и репликация значений отдельных членов использует меньше пропускной способности сети и меньшее количество циклов процессора во время репликации и предотвращает потери обновлений при добавлении или удалении членов нескольких одновременно на разных контроллерах домена.
    * Возможность развертывания контроллера домена только для чтения (RODC)
    * Улучшенные алгоритмы проверки согласованности знаний (KCC) и масштабируемость
        - Генератор межсайтовой топологии (ISTG) использует улучшенные алгоритмы, масштабируемые для поддержки лесов с увеличенным количеством сайтов, чем может поддерживать AD DS при режиме работы леса Windows 2000. Улучшенный алгоритм выборов генератор межсайтовой Топологии — это слишком высокий менее механизм по выбору ISTG при режиме работы леса Windows 2000.
    * Возможность создания экземпляров динамического вспомогательного класса с именем **Динамическийобъект** в разделе каталога домена
    * Возможность преобразования **inetOrgPerson** экземпляр объекта в **пользователя** экземпляр объекта и для выполнения преобразования в обратном направлении
    * Возможность создания экземпляров новых типов групп, для поддержки авторизации на основе ролей. 
        - Таких типов называются основными группами приложений и группами запросов LDAP.
    * Деактивация и переопределение атрибутов и классов схемы. Можно повторно использовать следующие атрибуты: ldapDisplayName schemaIdGuid OID и mapiID.
    * Доменных пространств имен DFS в Windows Server 2008 режим, который включает поддержку перечисление на основе доступа и повышенную масштабируемость. Дополнительные сведения см. в разделе [выбрать тип пространства имен](https://go.microsoft.com/fwlink/?LinkId=180400).

### <a name="windows-server-2003-domain-functional-level-features"></a>Компоненты функционального уровня домена Windows Server 2003

* Доступны все возможности по умолчанию AD DS, все возможности, доступные в Windows 2000 основного режима работы домена и следующие возможности:
    * Средства управления доменом Netdom.exe, которое позволяет переименовать контроллеры домена
    * Обновление метки времени входа
        * **LastLogonTimestamp** атрибут обновляется со времени последнего входа пользователя или компьютера. Этот атрибут реплицируется внутри домена.
    * Возможность устанавливать **userPassword** атрибута в качестве эффективного пароля на **inetOrgPerson** и объектов пользователей
    * Возможность перенаправления пользователей и компьютеров контейнеров
        * По умолчанию предоставляются два известных контейнера для размещения учетных записей компьютеров и пользователей, а именно, cn = Computers,<domain root> и cn = Users,<domain root>. Эта функция позволяет определение новых известных расположении для этих учетных записей.
    * Возможность для диспетчера авторизации, чтобы хранить свои политики авторизации в Доменных службах Active Directory
    * Ограниченное делегирование
        * Ограниченное делегирование позволяет приложениям использовать безопасное делегирование учетных данных пользователя с помощью проверки подлинности на основе Kerberos.
        * Вы можете ограничить делегирование для отдельных служб назначения только.
    * Выборочная проверка подлинности
        * Выборочная проверка подлинности упрощает можно задать пользователей и группы из доверенного леса, которым разрешена проверка подлинности на серверах ресурсов в доверяющем лесу.
