# <a name="introduction"></a>Введение

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Атаки, направленные на вычислительные инфраструктуры, как простые, так и сложные, существуют до тех пор, пока компьютеры. Тем не менее за последнее десятилетие растет число организаций всех размеров во всех частях мира были атакована и скомпрометирована способами, которые существенно изменили характер угроз. Электронные войны и киберпреступность увеличиться рекордными темпами. «Хактивизм», в котором основе атак лежит активная социальная позиция, был заявлен как мотивация для ряда нарушений безопасности предназначен для предоставления раскрытие секретных сведений организаций, для создания отказов службы, или даже уничтожение инфраструктуры. Атаки, направленные на и частные учреждения, целью которых является общественные интеллектуальной собственности организации (IP) участились.  
  
Нет организаций, инфраструктура информационных технологий (ИТ) которых полностью защищена от атак, но если для защиты сегментов ключей вычислительной инфраструктуры реализованы соответствующие политики, процессы и элементы управления, эскалацию атак — от проникновения до полной компрометации можно предотвратить. Поскольку количество и масштаб атак за пределами организации число внутренних угроз в последние годы, в этом документе обсуждаются часто внешних атак, а не неправильное использование среду, авторизованных пользователей. Тем не менее принципы и рекомендации, приведенные в этом документе, предназначены для обеспечения защиты среды от внешних атак, а также ошибочных или злонамеренных действий участников программы предварительной оценки.  
  
Сведения и рекомендации, приведенные в этом документе получены из ряда источников и методик, разработанных для защиты установок Active Directory от взлома. Несмотря на то, что не невозможно предотвратить атаки, можно уменьшить уязвимость Active Directory и реализации элементов управления, которые делают компрометации каталога значительно усложняет для злоумышленников. В этом документе представлены наиболее распространенных типов уязвимостей, что мы должны зафиксированное в скомпрометированных средах и самые Общие рекомендации, что мы внесли пользователей для повышения безопасности установки Active Directory.

## <a name="about-this-document"></a>Содержание документа  
Информационной безопасности корпорации Майкрософт и организации управления рисками (ISRM), который входит в состав из технологии сведения корпорации Майкрософт (MSIT), работает с внутренние бизнес-подразделениям, внешних клиентов и партнеров отрасли сбора, распространять и определения политик, рекомендации и элементов управления. Эти сведения можно использовать корпорацией Майкрософт и наших клиентов для повышения безопасности и снизить вероятность атак ИТ-инфраструктур. Рекомендации, приведенные в этом документе основаны на несколько источников данных и методик, используемых в MSIT и ISRM. В следующих разделах представлены дополнительные сведения о происхождение этого документа.  
  
### <a name="microsoft-it-and-isrm"></a>ИТ и ISRM  
Количество рекомендации и элементы управления были разработаны внутри MSIT и ISRM для обеспечения безопасности Майкрософт Доменные службы Active Directory леса и домены. Эти элементы управления широкой областью, где они были объединены в этом документе. БЕЗОПАСНЫЙ-T (ускорители решений для новых технологий) является группу в ISRM, задачей которой является для определения новых технологий и определите требования к безопасности и элементы управления для ускорения их внедрения.  
  
### <a name="active-directory-security-assessments"></a>Оценка безопасности Active Directory  
В Microsoft ISRM, оценки, консультирование и группы Engineering (ACE) работает с внутренние бизнес-подразделениям корпорации Майкрософт и внешних клиентов для оценки приложений и безопасности инфраструктуры и тактический характер и стратегические рекомендаций для повышения уровня безопасности в организации. Предложение по службе одну запись ACE является Active Directory безопасности оценки (ADSA), являющийся целостное оценки Доменных службах Active Directory организации, которая оценивает людей, процессы и технологии дает рекомендации конкретных клиентов. Пользователям будет предоставлен рекомендации, основанные на организации уникальных характеристик, методик и позиции в отношении рисков. ADSAs были выполнены для установки Active Directory в корпорации Майкрософт, помимо наших клиентов. Со временем были обнаружены ряд рекомендаций для применения по клиентам разных размеров и отраслях.

### <a name="planning-for-compromise"></a>Планирование при компрометации  
Этот раздел начинается с «переход назад» от технических подробностей, чтобы сосредоточиться на принципы и процессы, которые можно использовать для идентификации пользователей, приложений и систем, которые наиболее важны не только для ИТ-инфраструктурой, но для бизнеса. После идентификации, что наиболее важно обеспечить стабильность и операций вашей организации, можно сосредоточиться на разделения и защиты таких ресурсов являются ли они права интеллектуальной собственности, пользователей и систем. В некоторых случаях может быть выполнено разделения и защиты активов в существующей среде AD DS, в других случаях вы должны учитывать при реализации small, отдельный «ячейки», позволяющих установить безопасного границу вокруг критически важные ресурсы и более строго, чем менее важных компонентов наблюдения за этими ресурсами. Рассматривается концепция, называемая «creative уничтожения», который — это механизм, с помощью которого устаревших приложений и систем, можно устранить путем создания новых решений и разделе заканчивается рекомендации, которые могут помочь для создания более безопасной среды путем объединения бизнеса и ИТ-службу для создания подробной картины того, что нормальное рабочее состояние. Зная, что является нормальным для организации, нарушений, которые могут указывать атак и взлом можно легче идентифицировать.
