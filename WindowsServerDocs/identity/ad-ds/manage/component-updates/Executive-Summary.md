# <a name="executive-summary"></a>Краткий обзор

>Область применения. Windows Server 2012

>[!IMPORTANT] 
>Следующая документация была записана в 2013 и предоставляется только для справки.  В настоящее время рассмотрения этой документации, и это может быть изменена.  Он может не отражать текущие рекомендации.

Нет организаций инфраструктуры информационных технологий (ИТ) защищена от хакерских атак, но если для защиты сегментов ключей вычислительной инфраструктуры организации реализованы соответствующие политики, процессы и элементы управления, могут быть доступны для предотвратить событие нарушения рост оптовая под угрозу безопасность вычислительной среды.  
  
Этот краткий обзор предназначен будут полезны как документ автономных суммирования содержимое документа, который содержит рекомендации, которые помогут организациям в повышение уровня безопасности установки Active Directory. Путем реализации этих рекомендаций, организации будут иметь возможность идентифицировать и определять их приоритеты действия безопасности, защиты сегментов ключей вычислительной инфраструктуры своей организации и создавать элементы управления, что значительно сокращает вероятность успешных атак против важных компонентов ИТ-среду.  
  
Несмотря на то, что в этом документе рассматриваются наиболее распространенные атаки на Active Directory и меры противодействия, чтобы снизить вероятность атак, он также содержит рекомендации для восстановления в случае полной компрометации. Единственный надежный способ восстановления в случае полной компрометации Active Directory — Будьте готовы к компрометации до ее выполнения.  
  
Ниже приведены основных разделов этого документа.

## <a name="avenues-to-compromise"></a>Способы решения проблем  
В этом разделе представлены сведения о некоторых наиболее часто возможную уязвимостей, использованы злоумышленниками для нарушения инфраструктур заказчиков. Он содержит общие категории уязвимостей и их использования изначально проникновение инфраструктур заказчиков, распространять компрометации за дополнительных систем, и в конечном итоге Active Directory и контроллерами домена для получения полного элемент управления лесов организации. Она предоставляет подробные рекомендации об адресации каждый тип уязвимостей, особенно в областях, в которых эти уязвимости не используются для непосредственно указывают на Active Directory. Тем не менее для каждого типа уязвимости представлены ссылки на дополнительную информацию, в которых будет производиться разрабатывать меры противодействия и уменьшение уязвимой зоны организации.  
  
Сюда входят следующие темы:  
  
-   **Первоначальный цели нарушения** -большинство нарушений безопасности сведения начинаться с компрометация небольшие части инфраструктуры часто одного или двух систем организации за раз. Эти события начальной или точки входа в сеть, часто уязвимостями, которые были исправлены может, но не были. Ниже приведены наиболее распространенных уязвимостей.  
  
    -   Пропуски в антивирусные и антивредоносные программы развертывания  
  
    -   Незавершенные установки исправлений  
  
    -   Устаревших приложений и операционных систем  
  
    -   Неправильной конфигурации  
  
    -   Отсутствие по разработке безопасных приложений  
  
-   **Учетные записи, подверженные риску кражи учетных данных** -атак кражи учетных данных являются те, в которых он изначально получает привилегированный доступ к компьютеру в сети, а затем использует свободно доступные средства для извлечения из сеансов других учетных данных учетные записи вошедшего в систему.   
    В этом разделе, включены следующие:  
  
    -   **Действия, позволяющих повысить вероятность нарушения** — так как целью кражи учетных данных обычно является учетные записи домена с высоким уровнем привилегий и «очень важным person» учетных записей (VIP), очень важно администраторам необходимо действия, которые увеличивают вероятность успеха атаки кражи учетных данных. Эти действия являются:  
  
        -   Вход в незащищенные компьютеры с привилегированных учетных записей  
  
        -   Поиск в Интернете с учетной записью с высоким уровнем привилегий  
  
        -   Настройка локальных привилегированных учетных записей с теми же учетными данными во всех системах  
  
        -   Overpopulation и чрезмерного использования групп домена привилегированными пользователями  
  
        -   Недостаточно Управление безопасностью контроллеров домена.  
  
    -   **Право доступа, прав и распространение** -конкретным учетным записям, серверов и компонентов инфраструктуры обычно являются основной целевыми объектами атак с помощью Active Directory. Эти учетные записи являются:  
  
        -   Окончательно привилегированных учетных записей  
  
        -   Учетные записи виртуального IP-адреса  
  
        -   «Подключенные привилегий» учетные записи Active Directory  
  
        -   Контроллеры доменов  
  
        -   Другие службы инфраструктуры, которые влияют на удостоверение, доступа и конфигурации управления, такими как серверы инфраструктуры открытых ключей (PKI) и серверы управления системами  
  
## <a name="reducing-the-active-directory-attack-surface"></a>Снижение уязвимостей Active Directory  
Этот раздел посвящен технические средства, чтобы снизить вероятность атак установки Active Directory. В этом разделе включены следующие темы:  
  
-   **Привилегированных учетных записей и групп в Active Directory** разделе обсуждаются наивысший привилегированных учетных записей и групп в Active Directory и механизмы, которые защищены привилегированных учетных записей. В Active Directory три встроенные группы, которые наивысший группы привилегий в каталоге ("Администраторы предприятия", "Администраторы домена" и Администраторы), несмотря на то, что ряд дополнительных групп и учетных записей также должны быть защищены.  
  
-   **Реализация административных моделей минимальными** раздел посвящен определение риска, который представляет использование высокопривилегированные учетные записи для повседневного администрирования, помимо рекомендаций по снижения этого риска.  
  
Чрезмерное привилегий только не найдены в Active Directory в скомпрометированных средах. Когда организация разработала привычку предоставления более высоким уровнем привилегий, чем требуется, обычно находится во всей инфраструктуре:  
  
-   В Active Directory  
  
-   На рядовых серверах  
  
-   На рабочих станциях  
  
-   В приложениях  
  
-   В репозитории данных  
  
-   **Реализации защищенные административные узлы** разделе описываются защищенные административные узлы, являющиеся компьютеров, настроенных для поддержки администрирования Active Directory и подключенных систем. Эти узлы выделяются административные функции и не запускайте программного обеспечения, такие как почтовые приложения, веб-браузеров или офисных приложений (например, Microsoft Office).  
  
В этом разделе, включены следующие:  
  
-   **Принципы для создания безопасного администрирования узлов** -являются общие принципы, имейте в виду:  
  
    -   Никогда не администрировать доверенной системы из менее доверенного узла.  
  
    -   Не следует полагаться на одного фактора проверки подлинности, при выполнении привилегированных действий.  
  
    -   Не забудьте указать физической безопасности при разработке и реализации защищенные административные узлы.  
  
-   **Защита контроллеров домена от атак** -Если злонамеренный пользователь получает привилегированный доступ к контроллеру домена, можно изменить этого пользователя, поврежден и разрушает базу данных Active Directory, а следовательно, все системы и учетных записей управляются с помощью Active Directory.  
  
В этом разделе включены следующие темы:  
  
-   **Физическая безопасность для контроллеров домена** — содержит рекомендации для обеспечения физической безопасности для контроллеров доменов в центрах обработки данных, филиалов и удаленных расположениях.  
  
-   **Операционные системы контроллеров домена** — содержит рекомендации по защите операционные системы контроллеров домена.  
  
-   **Защита контроллеров домена конфигурации** -конфигурации собственного и свободно доступные средства и параметры можно использовать для создания шаблонов базовой конфигурации для контроллеров домена, которые может впоследствии применяться (объекты групповой политики безопасности Объекты групповой политики).  
  
## <a name="monitoring-active-directory-for-signs-of-compromise"></a>Мониторинг Active Directory для обнаружения признаков компрометации  
Этот раздел содержит сведения об устаревших аудита категории и подкатегории политики аудита (которые впервые появились в Windows Vista и Windows Server 2008) и расширенной политики аудита (которая была введена в Windows Server 2008 R2). Также предоставляются сведения о событиях и объектов для отслеживания, можно указать компрометации среды и некоторые дополнительные ссылки, которые могут использоваться для создания всеохватывающей политики аудита для Active Directory.  
  
В этом разделе включены следующие темы:  
  
-   **Политика аудита Windows** — журналы событий безопасности Windows имеют категории и подкатегории, которые определяют, какие события безопасности отслеживаются и записываются.  
  
-   **Рекомендации по политике аудита** -в этом разделе описываются параметры политики аудита Windows по умолчанию, параметры политики, которые рекомендуются корпорацией Майкрософт и более строгими требованиями и рекомендации для организаций с помощью аудита критически важными серверами аудита и рабочие станции.  
  
## <a name="planning-for-compromise"></a>Планирование при компрометации  
Этот раздел содержит рекомендации, которые помогут организациям подготовиться к компрометации до ее выполнения, внедрите элементы управления, которые можно обнаружить событие компрометации прежде, чем полный брешей в системе, а также предоставить рекомендации по реагированию и восстановлению для случаев, когда полной компрометации каталога достигается за счет злоумышленники. В этом разделе включены следующие темы:  
  
-   **Переосмысление подход** -содержит принципы и рекомендации для создания безопасных сред, в которых организации могут помещаться наиболее важных активов. Эти рекомендации являются следующим образом:  
  
    -   Определение принципов для разделения и защиты важных активов  
  
    -   Определение плана миграции ограниченный, на основе рисков  
  
    -   Использование «nonmigratory» миграции, при необходимости  
  
    -   Реализация «creative уничтожение»  
  
    -   Изоляция устаревших систем и приложений  
  
    -   Упрощение безопасности для конечных пользователей  
  
-   **Поддержка более безопасной среде** -содержит общие рекомендации, которые предназначены для использования в качестве рекомендации для разработки не только эффективную защиту, но эффективный жизненный цикл управления. В этом разделе включены следующие темы:  
  
    -   **Создание деловых по обеспечению безопасности для Active Directory** - Чтобы эффективно управлять жизненным циклом пользователей, данных, приложений и систем, которые управляются с помощью Active Directory, следовать следующим принципам.  
  
        -   **Назначить владельца бизнеса данные Active Directory** -во владение компонентов инфраструктуры ИТ; для данных, который добавляется в доменных службах Active Directory (AD DS) для поддержки бизнеса, например, новые сотрудники, новые приложения и новые сведения о репозитории, назначенный подразделения или пользователь должен быть связан с данными.  
  
        -   **Реализовать управление жизненным циклом Business-Driven** -управление жизненным циклом, которые должны быть реализованы для данных в Active Directory.  
  
        -   **Классифицировать все данные Active Directory** -владельцы предприятий должен предоставить классификации для данных в Active Directory. В рамках модели классификации данных классификации для следующих данных Active Directory должны быть включены:  
  
            -   **Систем** -классификации сервера заполнений, операционную систему их роли приложений, выполняющихся на них и ИТ и владельцы предприятий записи.  
  
            -   **Приложения** -Классификация приложений, функциональные возможности, базу пользователей и их операционной системе.  
  
            -   **Пользователи** -учетных записей в Active Directory установок, которые, скорее всего, стали целью злоумышленников следует с тегами и отслеживается.
