# <a name="resetting-a-trust-password-on-one-side-of-the-trust"></a>Сброс пароля доверия с одной стороны отношения доверия  

>Область применения: Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 Если восстановления леса, связанных с брешь в системе безопасности, используйте следующую процедуру, чтобы сбросить пароль доверия с одной стороны отношения доверия. Сюда входят неявные отношения доверия между дочерних и родительских доменов, а также явные доверительные отношения между этим доменом (доверяющего домена) и другого домена (домен).  
  
 Сброс пароля доверяющего домена стороны отношения доверия, также известные как входящее доверие (сторона, к которой принадлежит этот домен). Затем можно используйте тот же пароль доверия, также известные как исходящее доверие со стороны доверенного домена. Сброс пароля исходящего доверия, при восстановлении первого контроллера домена в каждом из других доменов (доверенных).  
  
 Сброс пароля доверия гарантирует, что контроллер домена не реплицирует потенциально поврежденных контроллеров домена вне своего домена. Установив один и тот же пароль доверия при восстановлении первого контроллера домена в каждом из доменов, убедитесь, что данный контроллер домена реплицирует с каждым восстановленные контроллеры домена. Последующие контроллеры домена в домене, которые восстанавливаются, установив AD DS автоматически реплицирует эти новые пароли во время установки.  
  
## <a name="to-reset-a-trust-password-on-one-side-of-the-trust"></a>Чтобы сбросить пароль доверия с одной стороны отношения доверия  
  
1.  В командной строке введите следующую команду и нажмите клавишу ВВОД:  
  
    ```  
    netdom experthelp trust  
    ```  
  
2.  Используйте синтаксис, что эта команда предоставляет для средства NetDom сброс пароля доверия.  
  
     Например, если существует двумя доменами в лесу — родительской и дочерней — и выполните следующую команду на восстановленного контроллера домена в родительском домене, используйте следующий синтаксис команды:  
  
    ```  
    netdom trust parent domain name /domain:child domain name /resetOneSide /passwordT:password /userO:administrator /passwordO:*  
    ```  
  
     При запуске этой команды в дочернем домене, используйте следующий синтаксис команды:  
  
    ```  
    netdom trust child domain name /domain:parent domain name /resetOneSide /password:password /userO:administrator /passwordO:*  
    ```  
  
    > [!NOTE]
    >  **passwordT** должно совпадать со значением по обе стороны отношения доверия. Только один раз выполнить эту команду (в отличие от **netdom resetpwd** команды), так как он автоматически сбрасывает пароль два раза.
