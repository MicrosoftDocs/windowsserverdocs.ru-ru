# <a name="resetting-a-trust-password-on-one-side-of-the-trust"></a>Сброс пароля доверия на одной стороне отношения доверия  

>Область применения. Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 Если восстановление леса имеет отношение к нарушению безопасности, используйте следующую процедуру для сброса пароля доверия на одной стороне отношения доверия. Сюда входят неявные отношения доверия между дочерних и родительских доменов, а также явные отношения доверия между данным доменом (доверяющего домена) и другой домен (домен). 
  
 Сброс пароля со стороны только доверяющем домене доверия, также известный как входящее отношение доверия (сторона, которой принадлежит этот домен). Затем используйте тот же пароль доверия, также известный как исходящее доверие со стороны доверенного домена. Сброс пароля исходящее доверие при восстановлении первого контроллера домена в каждом из других доменов (доверенной). 
  
 Сброс пароля доверия гарантирует, что контроллер домена не реплицируется с потенциально плохих контроллеров домена за пределами своего домена. Установив один и тот же пароль доверия при восстановлении первого контроллера домена в каждом из доменов, убедитесь, что этот контроллер домена реплицирует с каждым из восстановленные контроллеры домена. Последующие контроллеры домена в домене, которые восстанавливаются, установив AD DS автоматически реплицирует эти новые пароли во время установки. 
  
## <a name="to-reset-a-trust-password-on-one-side-of-the-trust"></a>Чтобы сбросить пароль доверия на одной стороне отношения доверия  
  
1. В командной строке введите следующую команду и нажмите клавишу ВВОД:  

   ```  
   netdom experthelp trust  
   ```  
  
2. Используйте синтаксис, что эта команда предоставляет для использования средства NetDom, чтобы сбросить пароль доверия.
   Например, при наличии двух доменов в лесу — родительским и дочерним и при выполнении этой команды на восстановленном контроллере домена в родительском домене, используйте следующий синтаксис команды:  

   ```  
   netdom trust parent domain name /domain:child domain name /resetOneSide /passwordT:password /userO:administrator /passwordO:*  
   ```  

   При выполнении этой команды в дочерних доменах, используйте следующий синтаксис команды:  

   ```  
   netdom trust child domain name /domain:parent domain name /resetOneSide /passwordT:password /userO:administrator /passwordO:*  
   ```  

   > [!NOTE]
   > **passwordT** должно быть одинаковым для обеих сторон доверия. Только один раз выполните следующую команду (в отличие от **netdom resetpwd** команды), так как она автоматически сбрасывает пароль дважды.
