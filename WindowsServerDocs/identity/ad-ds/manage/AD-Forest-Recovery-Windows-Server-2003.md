# <a name="ad-forest-recovery---windows-server-2003-recovery"></a>Восстановление леса AD - восстановления Windows Server 2003

>Область применения. Windows Server 2003

Этот раздел содержит процедуры восстановления леса для контроллеров домена (DC) под управлением Windows Server 2003. Общая процедура восстановления леса то же самое контроллеров домена Windows Server 2003, но конкретных процедур могут отличаться из-за различных средств. Например можно использовать Ntdsutil.exe резервного копирования и восстановления контроллеров домена под управлением контроллеров домена Windows Server 2003, в то время как система архивации данных Windows Server или Wbadmin.exe, используемых для контроллеров домена под управлением Windows Server 2008 или более поздней версии.  
  
- [Резервное копирование данных состояния системы](#backing-up-the-system-state-data)  
- [Выполнение непринудительное восстановление](#performing-a-nonauthoritative-restore)  
- [Установка и настройка службы DNS-сервера](#install-and-configure-the-dns-server-service)

## <a name="backing-up-the-system-state-data"></a>Резервное копирование данных состояния системы
Используйте следующую процедуру для резервного копирования данных состояния системы, а также любые другие данные, выбранный для текущей операции резервного копирования, контроллера домена под управлением Windows Server 2003. Windows Server 2003 включает в себя средство Ntbackup, которое можно использовать для резервного копирования данных состояния системы.  
  
Членство в группе **Администраторы** или **операторы архива**, или эквивалентной является минимальным требованием для резервного копирования файлов и папок.   
  
Если при резервном копировании данных состояния системы на ленту и программе архивации данных указывает, что нет доступных неиспользуемые носителя, может потребоваться использовать съемные носители. Поэтому эту резервную копию можно использовать это добавляет ленты в пул свободного носителя.  
  
Только можно выполнять архивацию данных состояния системы на локальном компьютере. Он не может архивировать на удаленном компьютере.  
  
### <a name="to-back-up-the-system-state-data-on-a-domain-controller-that-runs-windows-server-2003"></a>Для резервного копирования данных состояния системы на контроллере домена, ОС Windows Server 2003  
  
1. Нажмите кнопку **запустить**, пункты **все программы**, пункты **стандартные**, пункты **служебные программы**, а затем нажмите кнопку **резервного копирования** .  
2. На **приветствия** щелкните **расширенный режим**.  
3. На **резервного копирования** вкладке, установите флажок для любой диск, папку или файл, который вы хотите создать резервную копию.  
4. Выберите **состояния системы** "флажок".  
5. Нажмите кнопку **запуск резервного копирования**.  
  
## <a name="performing-a-nonauthoritative-restore"></a>Выполнение непринудительное восстановление  

Используйте следующую процедуру для выполнения непринудительное восстановление контроллера домена под управлением Windows Server 2003. Выполняя непринудительное восстановление в Active Directory в Windows Server 2003, автоматически выполняет принудительное восстановление SYSVOL. Никакие дополнительные действия не требуются.  
  
> [!NOTE]
> Также при переустановке операционной системы Windows Server 2003, может или не может присоединить компьютер к домену, и ему можно присвоить любое имя на компьютер во время установки операционной системы. Не устанавливайте Active Directory. После переустановки операционной системы, перейдите к шагу 4.  
  
На контроллерах домена Windows Server 2003, где восстановлена только данные о состоянии системы необходимо также повторно установить любые приложения, которые были запущены на контроллерах домена, перед восстановлением. Также восстановление AD DS на первого контроллера домена в домене восстановление реестра, поскольку они оба являются частью данных состояния системы. Имейте это в виду, если у вас есть все приложения, работающего на соответствующие контроллеры доменов, и если у него есть все данные, хранимые в реестре.  
  
Чтобы сохранить время, необходимое для повторной установки программного обеспечения, определите, совместимы ли приложения, которые должны быть установлены на контроллерах домена с клонирование виртуального контроллера домена. Такие приложения могут устанавливаться на исходный контроллер домена до клонирования, чтобы сэкономить время и усилия, необходимые для их установки на клонированный виртуальных контроллеров домена.  
  
### <a name="to-perform-a-nonauthoritative-restore"></a>Для выполнения непринудительное восстановление
  
1. После запуска контроллера домена, нажмите клавишу F8, чтобы перезагрузить компьютер в режиме восстановления служб каталогов (DSRM).  
2. Выберите **режима восстановления служб каталогов (только для контроллеров домена Windows)** .  
3. Выберите операционную систему, необходимо запустить в режиме восстановления.  
4. Войдите в систему с правами администратора (вы можете использовать учетную запись локального компьютера, не вариант входа домена доступен только).  
5. В командной строке введите **ntbackup**, и нажмите клавишу ВВОД.  
6. На **приветствия** щелкните **расширенный режим**, а затем выберите **восстановление и управление носителем** вкладки. (Не выбирайте **мастер восстановления**.)  
7. Выберите соответствующий файл резервной копии для восстановления из и убедитесь, что **системный диск** и **состояния системы** флажки.  
8. Нажмите кнопку **начать восстановление**.  
9. После завершения операции восстановления, перезагрузите компьютер.  
  
Используйте следующую процедуру для выполнения полномочного восстановления (также известный как основная) SYSVOL на Контроллере домена под управлением Windows Server 2003. Выполните эту процедуру только на первом Windows Server 2003 контроллера домена, который восстанавливается в домене.  
  
### <a name="to-perform-an-authoritative-restore-of-sysvol"></a>Для выполнения полномочного восстановления SYSVOL  
  
1. Выполните шаги 1 – 8 предыдущей процедуры.  
2. В **Подтверждение восстановления** диалоговом окне щелкните **Дополнительно**.  
3. Чтобы выполнить Полномочное восстановление SYSVOL, установите флажок **при восстановлении реплицируемых наборов данных, помечать восстановленные данные как основные для всех реплик**.  

   > [!NOTE]
   > Пометка восстановленные данные, как основные данные в резервной копии эквивалентно заданию **BurFlags** запись D4 в следующем подразделе реестра:  
   >   
   > **Наборов реплик HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NtFrs\Parameters\Cumulative\\**  *GUID*  

4. После завершения операции восстановления, перезагрузите компьютер.  
  
## <a name="install-and-configure-the-dns-server-service"></a>Установка и настройка службы DNS-сервера

Если контроллера домена, который был восстановлен из резервной копии работает под управлением Windows Server 2003, можно установить DNS-сервер без подключения контроллера домена, к какой сети.  
  
### <a name="to-install-and-configure-the-dns-server-service"></a>Установка и настройка службы DNS-сервера  
  
1. Откройте мастер компонентов Windows. Чтобы открыть мастер:  

   - Щелкните **Пуск**, нажмите **Панель управления**, а затем **Установка и удаление программ**.  
   - Нажмите кнопку **Добавление и удаление компонентов Windows**.  

2. В **компоненты**выберите **сетевые службы** флажок и нажмите кнопку **сведения**.  
3. В **сетевые службы-состав**выберите **доменных имен (DNS)** флажок, нажмите кнопку **ОК**, а затем нажмите кнопку **Далее**.  
4. Если появится запрос, в **скопируйте файлы из**, введите полный путь к установочным файлам и нажмите кнопку **ОК**.  

   После установки выполните следующие действия, чтобы настроить DNS-сервер.  

5. Нажмите кнопку **запустить**, пункты **все программы**, пункты **Администрирование**, а затем нажмите кнопку **DNS**.  
6. Создание зоны DNS для же DNS-имена домена, которые размещались на DNS-серверах перед критических неисправности. Дополнительные сведения см. в разделе Add зоны прямого просмотра ([https://go.microsoft.com/fwlink/?LinkId=74574](https://go.microsoft.com/fwlink/?LinkId=74574)).  
7. Настройте данные DNS, как он существовал до критических неисправности. Пример:  

   - Настройка зоны DNS для сохранения в Доменных службах Active Directory. Дополнительные сведения см. в разделе Изменение типа зоны ([https://go.microsoft.com/fwlink/?LinkId=74579](https://go.microsoft.com/fwlink/?LinkId=74579)).  
   - Настройка зоны DNS, которая является доверенной для записей ресурсов локатора контроллера домена контроллера домена Разрешить безопасное динамическое обновление. Дополнительные сведения см. в разделе только Secure динамическое обновление ([https://go.microsoft.com/fwlink/?LinkId=74580](https://go.microsoft.com/fwlink/?LinkId=74580)).  

8. Убедитесь, что в зону DNS содержит ресурс записи делегирования (имя сервера (NS) и клей ресурсов узлов (A) записи) для дочерней зоны, размещенной на этот DNS-сервер. Дополнительные сведения см. в разделе создать делегирование зоны ([https://go.microsoft.com/fwlink/?LinkId=74562](https://go.microsoft.com/fwlink/?LinkId=74562)).  
9. После настройки DNS, в командной строке введите следующую команду и нажмите клавишу ВВОД:  

   **net stop netlogon**

10. Введите следующую команду и нажмите клавишу ВВОД.  

    **net start netlogon**

    > [!NOTE]
    > Сетевого входа в систему будет зарегистрировать в DNS записи ресурсов локатора Контроллеров домена для данного контроллера домена. При установке службы DNS-сервера на сервере в дочерний домен, этот контроллер домена не сможете регистрировать свои записи немедленно. Это, поскольку он изолирован в настоящее время как часть процесса восстановления и его основной DNS-сервер является корневой сервер DNS леса. Настройте этот компьютер с тем же IP-адрес, как оно было до сбоя, чтобы избежать ошибок при поиске контроллера домена службы.
