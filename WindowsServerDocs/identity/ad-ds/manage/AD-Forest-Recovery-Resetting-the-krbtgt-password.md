# <a name="ad-forest-recovery---resetting-the-krbtgt-password"></a>Восстановление леса AD — сброс пароля krbtgt

>Область применения. Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

Используйте следующую процедуру для сброса пароля krbtgt для домена. Следующая процедура применяется для записи контроллеров домена, но контроллеры домена не только для чтения (RODC).
  
> [!IMPORTANT]
> Если вы планируете восстановить RODC в интерактивном режиме во время восстановления леса, не удаляйте учетные записи krbtgt для RODC. Учетная запись krbtgt для RODC указана в формате krbtgt_*номер*.
>
> Если вы используете фильтр измененный пароль (например, устанавливаемые файлом passfilt.dll) на Контроллере домена, затем может появиться ошибка при попытке сбросить пароль krbtgt. Дополнительные сведения, включая обойти это ограничение, см. в статье базы знаний Майкрософт [статьи 2549833](https://support.microsoft.com/kb/2549833) (https://support.microsoft.com/kb/2549833).
  
## <a name="to-reset-the-krbtgt-password"></a>Чтобы сбросить пароль krbtgt  
  
1. Нажмите кнопку **запустить**, пункты **панели управления**, пункты **Администрирование**, а затем нажмите кнопку **Active Directory — пользователи и компьютеры**.
2. Выберите **Вид** и щелкните **Дополнительные параметры**.
3. В дереве консоли дважды щелкните контейнер домена и нажмите кнопку **пользователей**.
4. В области сведений щелкните правой кнопкой мыши **krbtgt** учетной записи пользователя, а затем щелкните **сброс пароля**.
   ![Сброс пароля](media/AD-Forest-Recovery-Resetting-the-krbtgt-password/resetpass1.png)
5. В **новый пароль**, введите новый пароль, подтвердите пароль в **подтверждение пароля**, а затем нажмите кнопку **ОК**. Пароль не имеет значения, так как система создаст надежный пароль, автоматически независимо от вами пароль.
  
> [!NOTE]
> Вы должны выполнить эту операцию дважды. Журнал паролей учетной записи krbtgt равно двум, это означает, что он включает два последних пароля. Сбросив пароль дважды эффективно очистить все старые пароли из журнала, поэтому нет способа другого контроллера домена будут реплицированы с этот контроллер домена с помощью старого пароля.
