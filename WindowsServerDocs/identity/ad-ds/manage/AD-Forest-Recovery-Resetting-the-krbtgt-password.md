# <a name="ad-forest-recovery---resetting-the-krbtgt-password"></a>Восстановление леса AD — Смена пароля krbtgt 

>Область применения: Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 Используйте следующую процедуру для сброса пароля krbtgt для домена. Следующая процедура применяется для записи контроллеров домена, но контроллеры домена не только для чтения (RODC).  
  
> [!IMPORTANT]
>  Если вы планируете восстановить RODC сети во время восстановления леса, не удаляйте учетную запись krbtgt для RODC. Учетной записи krbtgt для RODC указан в формате krbtgt_*номер*.  
>   
>  Если вы используете фильтр измененный пароль (например, passfilt.dll) на Контроллере домена, может возникнуть ошибка при попытке выполнить сброс пароля krbtgt. Дополнительные сведения, включая решения, см. в базы знаний Майкрософт [статья 2549833](https://support.microsoft.com/kb/2549833) (https://support.microsoft.com/kb/2549833).  
  
## <a name="to-reset-the-krbtgt-password"></a>Для сброса пароля krbtgt  
  
1.  Нажмите кнопку **запустить**, наведите курсор на **панели управления**, наведите курсор на **Администрирование**и нажмите кнопку **Active Directory-пользователи и компьютеры**.  
2.  2.  Нажмите кнопку **представление**, а затем нажмите кнопку **дополнительные функции**.  
3.  В дереве консоли дважды щелкните контейнер домена и нажмите кнопку **пользователей**.  
4.  В области сведений щелкните правой кнопкой мыши **krbtgt** учетной записи пользователя, а затем щелкните **Смена пароля**.  
![Сброс пароля](media/AD-Forest-Recovery-Resetting-the-krbtgt-password/resetpass1.png)
5.  В **новый пароль**, введите новый пароль, повторно введите пароль в **подтверждение пароля**, а затем нажмите кнопку **ОК**. Пароль, указанный не имеет значения, так как система будет создавать автоматически независимо от пароль, который указывается надежный пароль.  
  
    > [!NOTE]
    >  Следует выполнять эту операцию дважды. Журнал паролей для учетной записи krbtgt должен содержать две, это означает, что он включает в себя два последних пароли. Смена пароля для дважды эффективно снятия старых паролей из журнала, не существует способа другого контроллера домена будут реплицированы с этого контроллера домена с помощью старого пароля.
