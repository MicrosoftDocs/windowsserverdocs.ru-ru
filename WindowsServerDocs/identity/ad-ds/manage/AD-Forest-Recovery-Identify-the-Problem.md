# <a name="identify-the-problem"></a>Определить проблему

>Область применения. Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2
  
Когда появятся симптомы сбоя леса, например в журналы событий или других решений для мониторинга, обратитесь в службу поддержки Microsoft, чтобы определить причину сбоя и оценить все возможные способы их устранения.  

## <a name="examples-of-forest-wide-failures"></a>Примеры сбоев на уровне леса

- Все контроллеры домена были логически или физически поврежден в точку, что непрерывности бизнес-процессов невозможна; Например все бизнес-приложений, которые зависят от AD DS, неработоспособны.  
- Администратор-злоумышленник скомпрометирует среды Active Directory.  
- Злоумышленник намеренно, или администратор случайно — запускает сценарий, который распространяется повреждения данных в пределах леса.  
- Злоумышленник намеренно, или администратор случайно — расширяет схему Active Directory с вредоносными или конфликтующие изменения.  
- Злоумышленник управления для установки вредоносных программ на контроллерах домена, а вы было известно службой поддержки Майкрософт для восстановления леса.  
  
   > [!IMPORTANT]
   >  В этом документе рассматриваются рекомендации по безопасности о том, как восстановление леса, если хакерской атаки или скомпрометированы. В общем случае рекомендуется выполните Pass--Hash приемы снижения рисков для усиления защиты среды. Дополнительные сведения см. в разделе [атак защита от атак Pass--Hash (PtH) и других способов кражи учетных данных](https://www.microsoft.com/download/details.aspx?id=36036).
  
- Ни один из контроллеров домена можно реплицировать с помощью своих партнеров репликации.  
- AD DS на любом контроллере домена не может быть внесены изменения.  
- Новый контроллер домена не может устанавливаться в любом домене.
