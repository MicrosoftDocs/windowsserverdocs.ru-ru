# <a name="ad-forest-recovery---cleanup"></a>Восстановление леса AD — очистки 

>Область применения: Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 Выполните следующие действия по восстановлению post при необходимости.  
  
-   После восстановления всего леса, можно вернуться к исходной конфигурации DNS, включая настройки основного и дополнительного DNS-серверов, на каждом из контроллеров домена. После настройки DNS-серверы, как они находились до неисправности их предыдущей возможности разрешения имени будут восстановлены. Удалите все записи DNS для контроллеров домена, которые не были восстановлены.  
  
-   Удаление записей службы Windows Internet Name службы (WINS) для всех контроллеров домена, которые не были восстановлены.  
  
-   Можно перенести роли хозяина операций на другие контроллеры домена в домене или лесу и добавить серверы более глобального каталога, в зависимости от конфигурации до сбоя.  
  
-   Так как весь лес восстанавливается предыдущее состояние объектов (таких как пользователи и компьютеры), которые были добавлены и все обновления (например, изменение пароля), которые были внесены в существующих объектах после данной точки будут утеряны. Таким образом необходимо заново создать эти объекты отсутствует, а затем снова примените отсутствующих обновлений соответствующим образом.  
  
-   Может также необходимо восстановить исх. отношения доверия с внешними доменами и лесами, так как эти внешние доверительные отношения, не восстанавливаются автоматически из резервных копий.
