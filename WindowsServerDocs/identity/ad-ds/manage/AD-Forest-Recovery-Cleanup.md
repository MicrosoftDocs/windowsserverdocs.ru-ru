# <a name="ad-forest-recovery---cleanup"></a>Восстановление леса AD - очистки

>Область применения. Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 При необходимости, выполните следующие действия после восстановления.  
  
- После восстановления всего леса, можно вернуться к исходной конфигурации DNS, включая настройку основного и дополнительного DNS-серверов на каждом из контроллеров домена. После того как DNS-серверы настроены как перед неисправности, будут восстановлены их предыдущих возможности разрешения имени. Удалите все DNS-записи для контроллеров домена, которые не были восстановлены.  
- Удалите записи службы имя Windows Internet (WINS) для всех контроллеров домена, которые не были восстановлены.  
- Можно перенос ролей хозяина операций на другие контроллеры домена в домене или лесу и добавить серверы, использующих более глобального каталога, в зависимости от конфигурации до сбоя.  
- Так как весь лес восстанавливается в предыдущее состояние, любые объекты (например, пользователи и компьютеры), которые были добавлены и все обновления (например, изменения пароля), которые были внесены в существующие объекты с этого момента будут потеряны. Таким образом необходимо заново создать эти отсутствующие объекты и повторно применить отсутствующие обновления соответствующим образом.  
- Может также потребоваться восстановить исх. отношения доверия с внешними доменами и лесами, так как эти внешние доверительные отношения не восстанавливаются автоматически из резервных копий.
