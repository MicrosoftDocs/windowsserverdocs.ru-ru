# <a name="ad-forest-recovery---cleanup"></a>Восстановление леса AD — очистка

>Область применения: Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

 При необходимости выполните следующие действия после восстановления:

- После восстановления всего леса можно вернуться к исходной конфигурации DNS, включая настройку предпочитаемых и альтернативных DNS-серверов на каждом контроллере домена. После настройки DNS-серверов так, как они находились до сбоя, будут восстановлены предыдущие возможности разрешения имен. Удалите все записи DNS для контроллеров домена, которые не были восстановлены.
- Удаление записей службы Windows Internet Name Service (WINS) для всех контроллеров домена, которые не были восстановлены.
- Роли хозяина операций можно переносить в другие контроллеры домена или леса и добавить другие серверы глобального каталога на основе конфигурации до сбоя.
- Так как весь лес восстанавливается до предыдущего состояния, все добавленные объекты (например, пользователи и компьютеры) и все обновления (например, изменения паролей), внесенные в существующие объекты после этой точки, теряются. Поэтому следует повторно создать эти отсутствующие объекты и снова применить недостающие обновления.
- Также может потребоваться восстановить исходящие доверия с внешними доменами и лесами, так как эти внешние отношения доверия не восстанавливаются автоматически из резервных копий.
