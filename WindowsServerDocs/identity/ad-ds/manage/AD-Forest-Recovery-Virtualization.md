# <a name="active-directory-forest-recovery-virtualization"></a>Виртуализация восстановления леса Active Directory

>Область применения: Windows Server 2016, Windows Server 2012 и 2012 R2, Windows Server 2008 и 2008 R2

В этом разделе описаны компонента виртуализированных контроллеров домена клонирования в Windows Server 2016, 2012 и 2012 R2.  
 
## <a name="using-virtualized-domain-controller-cloning-to-expedite-forest-recovery"></a>Для ускорения восстановления леса с помощью клонирования виртуализированного контроллера домена  
 Клонирования виртуализированного контроллера (DC) домена упрощает и ускоряет установку дополнительных виртуализированных контроллеров домена в домене, особенно в централизованного расположения, такие как центров обработки данных, некоторые контроллеры домена работают в низкоуровневых оболочках, процесс. После восстановления из резервной копии одного виртуального контроллера домена в каждом домене, дополнительные контроллеры домена в каждом домене, можно быстро перевести в оперативный режим с помощью процесса клонирования виртуализированного контроллера домена. Вы можете подготовить первый виртуализованного контроллера домена, восстановление, завершить работу компьютера и затем скопировать этого виртуального жесткого диска, как много раз, необходимых для создания клонированного виртуализированных контроллерах домена, можно строить домена.  
  
 Ниже перечислены требования для клонирования виртуализированного контроллера домена  
  
-   Низкоуровневая оболочка должна поддерживать VM-GenerationID. Hyper-V в Windows Server 2016, 2012 и Windows 8 является примером гипервизоре, поддерживающем VM-GenerationID. Свяжитесь с поставщиком гипервизора, если поддерживается VM-GenerationID.  
  
-   Виртуализованного контроллера домена, который используется в качестве источника для клонирования необходимо запустить Windows Server 2016 или 2012 и быть членом группы клонируемые контроллеры домена.  
  
-   Эмулятор основного контроллера домена необходимо запустить Windows Server 2016 или 2012. Если виртуализованный, можно клонировать эмулятор основного контроллера домена.  
  
 Пошаговые инструкции о том, как выполнить виртуализированных клонирование контроллера домена, см. в разделе [Знакомство с виртуализацией доменных служб Active Directory (AD DS) (уровень 100)](../Introduction-to-Active-Directory-Domain-Services-AD-DS-Virtualization-Level-100.md).
