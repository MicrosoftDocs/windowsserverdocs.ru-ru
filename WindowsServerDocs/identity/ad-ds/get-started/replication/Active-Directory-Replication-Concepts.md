## <a name="BKMK_1"></a>Объект подключения  
Объект подключения является объект Active Directory, который представляет подключение репликации с исходного контроллера домена на конечный контроллер домена. Контроллер домена является членом один сайт и представлен в узле объект-сервер в доменных службах Active Directory (AD DS). Каждый объект сервера содержит дочерний объект параметров NTDS, представляющий репликацию контроллера домена на сайте.  
  
Объект подключения является дочерним элементом объекта параметров NTDS на конечном сервере. Для репликации между контроллерами домена объект сервера одного должен иметь объект подключения, представляющий входящая репликация с другой. Всех подключений репликации для контроллера домена, хранятся в виде объектов соединения в объекте параметров NTDS. Объект подключения определяет сервер источника репликации, содержит расписание репликации и указывает транспорт репликации.  
  
Средство проверки согласованности знаний (KCC) объекты соединения создаются автоматически, но могут быть созданы и вручную. Объекты подключения, созданные при проверке согласованности ЗНАНИЙ отображаются в Active Directory — сайты и оснастка служб в качестве **<automatically generated>**и считаются достаточно при нормальных рабочих условиях. Объекты соединения, созданным администратором вручную создаются объекты подключения. Объект подключения, созданных вручную, определяется имя, присвоенное администратором, если он был создан. При изменении **<automatically generated>**объектов подключения, можно преобразовать в объект административно измененные подключения и объект отображается в форме GUID. Средство проверки согласованности ЗНАНИЙ не вносить изменения в объекты соединения или измененные вручную.  
  
## <a name="BKMK_2"></a>ПРОВЕРКУ СОГЛАСОВАННОСТИ ЗНАНИЙ.  
Средство проверки согласованности ЗНАНИЙ является встроенной процесс, который запускается на всех контроллерах домена и создает топологию репликации для леса Active Directory. При проверке согласованности ЗНАНИЙ топологий отдельной репликации, в зависимости от того, выполняется ли репликация в пределах сайта (внутрисайтовая) или между сайтами (межсайтовых). Средство проверки согласованности ЗНАНИЙ также динамически регулирует топологию для реализации возможности добавления новых контроллеров домена, удаления существующих контроллеров домена, движение, контроллеры домена из сайтов, изменение затраты и расписания и контроллеры домена, которые временно недоступны или в состоянии ошибки.  
  
Внутри сайта соединения между контроллерами домена с возможностью записи всегда организованы в кольцевой двунаправленное с подключений Дополнительные сочетания позволяет сократить задержку в больших веб-узлов. С другой стороны межсайтовой топологии является расположение охват деревьев, то есть одно межсайтовое подключение между любыми двумя сайтами, для каждого раздела каталога и обычно не содержит ярлык подключения. Дополнительные сведения об охвате деревья и топологию репликации Active Directory см. в разделе Active Directory технической справке по топологии репликации ([https://go.microsoft.com/fwlink/?LinkID=93578](https://go.microsoft.com/fwlink/?LinkID=93578)).  
  
На каждом контроллере домена при проверке согласованности ЗНАНИЙ маршруты репликации, создав одностороннее входящее подключение объектами, которые определяют соединений с контроллерами домена. Для контроллеров домена в том же сайте проверки Согласованности создает объекты соединения, автоматически без вмешательства администратора. При наличии более одного сайта, настройки сайта связи между сайтами и одной проверки согласованности ЗНАНИЙ на каждом сайте автоматически создает подключения между сайтами также.  
  
### <a name="kcc-improvements-for-windows-server-2008-rodcs"></a>При проверке согласованности ЗНАНИЙ улучшения для Windows Server 2008 RODC  

Существует ряд улучшений проверки Согласованности в соответствии с контроллером нового домена только для чтения (RODC) в Windows Server 2008. Типичное развертывание сценарий для RODC — в филиале. Наиболее часто развертывается в этом сценарии топологии репликации Active Directory основана на разработки и звезда, где филиала контроллеры домена на нескольких узлах осуществлять репликацию с небольшим количеством серверы-плацдармы на сайте концентратора.  
  
Одним из преимуществ развертывании RODC см. в этом сценарии является однонаправленным репликации. Серверы-плацдармы не являются обязательными для репликации с RODC, уменьшая администрирования и использования сети.  
  
Тем не менее одна из административных проблем, выделенный топологии "звезда" в предыдущих версиях операционной системы Windows Server, что после добавления нового контроллера домена сервера-плацдарма в концентраторе, автоматическое способ распространения подключений репликации между контроллерами домена филиала и центральных контроллеров домена, чтобы воспользоваться преимуществами нового контроллера домена концентратора.  
  
Для контроллеров домена Windows Server 2003, можно балансировки нагрузки с помощью такого средства, как Adlb.exe из руководства по Branch Office развертывания Windows Server 2003 ([https://go.microsoft.com/fwlink/?LinkID=28523](https://go.microsoft.com/fwlink/?LinkID=28523)).  
  
Для Windows Server 2008 RODC работы средства проверки согласованности ЗНАНИЙ предоставляет некоторые перераспределения, что устраняет необходимость использования дополнительного средства, такие как Adlb.exe. Новые функциональные возможности включены по умолчанию. Вы можете отключить его, добавив следующий раздел реестра на RODC:  
  
**HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\NTDS\Parameters**  
  
**«Случайные BH Loadbalancing разрешено»**  
**1 = включено (по умолчанию), 0 = отключено**  
  
Дополнительные сведения о принципах работы этих улучшений проверки Согласованности см. в разделе Планирование и развертывание доменных служб Active Directory для филиалов ([https://go.microsoft.com/fwlink/?LinkId=107114](https://go.microsoft.com/fwlink/?LinkId=107114)).  
  
## <a name="BKMK_3"></a>Функции отработки отказа  
Сайты убедитесь, что репликация направляется вокруг сетевые сбои, а отключенные контроллеры домена. Средство проверки согласованности ЗНАНИЙ выполняется через заданные промежутки времени, чтобы настроить топологию репликации изменений, произошедших в Доменных службах Active Directory, например при добавлении новых контроллеров домена и новые сайты создаются. Средство проверки согласованности ЗНАНИЙ просматривает состояние репликации существующих подключений, чтобы определить, если все подключения не работают. Если подключение не работает из-за неисправного контроллера домена, проверки Согласованности автоматически сборки временного подключения для других партнеров по репликации (если доступно) убедитесь, что репликация могла произойти. Если все контроллеры домена в сайте не будут доступны, средство проверки согласованности ЗНАНИЙ автоматически создает подключений репликации между контроллерами домена с другого сайта.  
  
## <a name="BKMK_4"></a>Подсети  
Подсеть является сегментов сети TCP/IP, в который назначены набор логических IP-адресов. Подсети группирования компьютеров образом определяет их близости физической сети. Объекты подсети в Доменных службах Active Directory указывают сетевых адресов, используемые для сопоставления компьютеров сайтам.  
  
## <a name="BKMK_5"></a>Сайт  
Сайты являются объектами Active Directory, которые представляют один или несколько подсетей TCP/IP с надежной и быстрой сетевых подключений. Сведения о сайте позволяет администраторам настроить доступ к Active Directory и репликации для оптимизации использования физической сети. Объекты сайтов связаны с набором подсетей и связан с сайтом Active Directory в соответствии с его IP-адрес каждого контроллера домена в лесу. Узлы можно разместить контроллеры домена из одного домена и домена может быть представлена в более чем одного сайта.  
  
## <a name="BKMK_6"></a>Связи сайтов  
Связи сайтов являются объектами Active Directory, которые представляют логические пути, используемые проверки Согласованности для подключения для репликации Active Directory. Объект связи сайтов представляет набор узлов, которые могут взаимодействовать с одинаковыми затратами через указанный межсайтового транспорта.  
  
Все сайты, содержащихся в связь сайтов, считаются подключаться с помощью одного типа сети. Узлы должны быть вручную связаны с другими сайтами с помощью связей сайтов, чтобы контроллеры домена в одном сайте могут реплицировать изменения каталога с контроллерами домена в другой сайт. Так как фактический путь, по сетевых пакетов в физической сети во время репликации не соответствуют связей сайтов, создавать связи сайтов избыточных для повышения эффективности репликации Active Directory не нужно.  
  
При подключении двух сайтов посредством связи сайтов, в системе репликации автоматически создает подключений между контроллерами домена на каждом сайте, называемые серверы-плацдармы. В Windows Server 2008 всех контроллеров домена на сайте с одного раздела каталога подходят для выбранной как серверы-плацдармы. Подключений репликации, созданные при проверке согласованности ЗНАНИЙ случайным образом распространяются среди всех серверов-плацдармов кандидатов на сайте, чтобы поделиться репликация рабочей нагрузки. По умолчанию случайного выбора процесс выполняется только один раз, когда объекты соединения сначала добавляются на сайт.  
  
## <a name="BKMK_7"></a>Мост связей сайтов.  
Мост связей сайтов является объектом Active Directory, представляющий набор связей сайтов, все сайты, могут обмениваться данными с помощью общий транспорт. Мостов связей сайтов включить контроллеры домена, которые не соединены напрямую посредством связи репликации друг с другом. Как правило мост связей сайтов соответствует маршрутизатор (или набор маршрутизаторы) на IP-сети.  
  
По умолчанию проверки Согласованности формирования транзитивное маршрута по какой-либо сайта ссылки, которые имеют некоторые узлы Общие. Если это поведение будет отключен, каждой связи сайтов представляет собственную отдельную и изолированную сеть. Наборы связей сайтов, которые воспринимаются как единый маршрут выражаются через мост связей сайтов. Каждый мост представляет среде изолированной связи для сетевого трафика.  
  
Мостов связей сайтов — это механизм для логического представления транзитивное физическое подключение между сайтами. Мост связей сайтов позволяет использовать любую комбинацию связей сайтов, включенные для определения маршрута наименее затратный для соединения разделов каталога, которые содержатся в этих сайтов средства проверки согласованности ЗНАНИЙ. Этот мост связей сайтов не поддерживает фактические подключение к контроллерам домена. Если мост связей сайтов будет удален, репликация по связи сайтов комбинированные будет продолжаться до проверки Согласованности удаляет ссылки.  
  
Мостов связей сайтов необходимы, только если сайт содержит контроллер домена, содержащий раздел каталога, также не размещается на контроллере домена в соседних сайта, но размещение этого раздела каталога контроллера домена находится в один или несколько других сайтов в лесу. Соседних сайтах определяются как два или несколько сайтов, включенных в отдельной связи сайтов.  
  
Мост связей сайтов создает логическое соединение между две связи сайтов, предоставляя транзитивное путь между двумя сайтами отключенных через промежуточный узел. В целях генератор межсайтовой топологии (ISTG) мост подразумевает физическое подключение через промежуточный узел. Мост не означает, что контроллер домена в промежуточном сайте будет предоставлять пути репликации. Тем не менее это может случиться, если промежуточный узел содержит контроллер домена, которое размещено реплицировать данный раздел каталога, в этом случае мост связей сайтов не требуется.  
  
Добавляется стоимость каждой связи сайтов, создания суммарном затраты для получившийся путь. Мост связей сайтов будет использоваться, если промежуточный узел не содержит контроллер домена, содержащий раздел каталога и ниже стоимости связи не существует. Если промежуточный узел содержит контроллер домена, который содержит раздел каталога, два отключенных сайтов будет настроить репликации подключения к контроллеру домена промежуточные и не использовать мост.  
  
## <a name="BKMK_8"></a>Транзитивность связи сайтов  
По умолчанию все связи сайтов, транзитивное или «мостом». Если связи сайтов соединены и расписания перекрывают друг друга, подключений репликации, определяющие партнеров репликации контроллера домена между сайтами, когда сайты не соединены напрямую посредством связи сайтов, но транзитивно подключены через набор общих сайтов при проверке согласованности ЗНАНИЙ. Это означает, что любой сайт может подключиться к другой сайт, используя сочетание связей сайтов.  
  
Как правило для полностью маршрутизируемой сети, не необходимо создавать любой сайт мостов связей сайтов, если вы хотите контролировать процесс репликации изменения. Если не удалось полностью направить вашей сети, чтобы избежать попыток невозможно репликации должны быть созданы мостов связей сайтов. Все связи сайтов для конкретных транспорта неявно принадлежат мостов связей сайтов для указанного транспорта. Мост по умолчанию для связей сайтов происходит автоматически, а не объект Active Directory представляет этот мост. **Установить мост для всех связей сайтов** параметр, находятся в свойствах контейнеры межсайтового транспорта IP и передачи протокола SMTP (Simple Mail), реализующий автоматического мост.  
  
> [!NOTE]  
> SMTP-репликации не будет поддерживаться в будущих версиях AD DS; Таким образом создание объектов связей сайтов в контейнере SMTP не рекомендуется.  
  
## <a name="BKMK_9"></a>Сервер глобального каталога  
Сервер глобального каталога является контроллера домена, в которой хранится информация о всех объектов в лесу, чтобы приложения можно найти AD DS без обращения в определенные контроллеры домена, хранящие запрошенные данные. Как и все контроллеры домена сервер глобального каталога хранит полный, доступный для записи реплик разделов каталога схемы и конфигурации и полной, доступный для записи реплики раздела каталога домена для домена, на котором он размещается. Кроме того сервер глобального каталога хранит частичные, доступные только для чтения реплики всех других доменов в лесу. Частичные, доступные только для чтения домена реплики содержат каждого объекта в домен, но только ограниченный набор атрибутов (атрибутов, наиболее часто используемые для поиска объекта).  
  
## <a name="BKMK_10"></a>Кэширование членства в универсальных группах  
Кэширование членства в универсальных группах контроллеру домена разрешено сведения кэша членства в универсальных группах для пользователей. Вы можете включить контроллеров домена, работающих под управлением Windows Server 2008 для кэша в универсальных группах с помощью Active Directory — сайты и службы.  
  
Включение кэширование членства в универсальных группах исключает необходимость в сервере глобального каталога на каждом сайте в домене, что сводит к минимуму использование пропускной способности сети, так как контроллер домена не требуется реплицировать все объекты, расположенные в лесу. Также уменьшает время входа, так как проверка подлинности контроллеры домена не всегда необходимо получить доступ к глобальному каталогу для получения сведений о членстве в универсальных группах. Дополнительные сведения о том, когда использовать кэширование членства в универсальных группах см. в разделе [планирование размещения сервера глобального каталога](../../../ad-ds/plan/Planning-Global-Catalog-Server-Placement.md).
