# <a name="selecting-the-forest-root-domain"></a>Выбор корневого домена леса

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Первый домен, который развертывается в лесу Active Directory называется корневым доменом леса. Этот домен остается корневого домена леса в течение жизненного цикла развертывания AD DS.  
  
Корневой домен леса содержит группы "Администраторы предприятия" и "Администраторы схемы". Эти группы администратора службы используются для управления операциями уровня леса, такие как добавление и удаление доменов и реализация изменений схемы.  
  
Выбор корневого домена леса подразумевает определение того, если один из доменов Active Directory при проектировании домена может функционировать в качестве корневого домена леса, или если необходимо развернуть корневой домен выделенный лес.  
  
Сведения о развертывании корневого домена леса, см. в разделе [развертывание корневого домена леса Windows Server 2008](https://technet.microsoft.com/library/cc731174.aspx).  
  
## <a name="choosing-a-regional-or-dedicated-forest-root-domain"></a>Выбор корневого домена леса региональные или выделенный

В случае применения одну модель предметной области, отдельного домена функционирует как корневого домена леса. В случае применения нескольких модель предметной области, вы можете развернуть корневой домен выделенный лес или выбрать регионального домена в качестве корневого домена леса.  
  
### <a name="dedicated-forest-root-domain"></a>Выделенный лес корневого домена

Выделенный лес корневой домен является доменом, специально созданный в качестве корня леса. Он не содержит все учетные записи пользователей, отличных от учетных записей администратора службы для корневого домена леса. Кроме того он не представляет любом географическом регионе, в структуру доменов. Все домены в лесу являются дочерними элементами корневой домен выделенный лес.  

С помощью выделенным корнем леса обеспечивает следующие преимущества:  

- Operational отделение администраторам леса службы от администраторов служб домена. В среде с одним домена члены групп "Администраторы домена" и встроенные группы администраторов можно использовать стандартные средства и процедуры для становиться членами группы "Администраторы предприятия" и "Администраторы схемы". В лесу, который использует корневой домен выделенный лес члены групп "Администраторы домена" и встроенные группы администраторов в региональных доменов не может становиться членами групп администраторов уровня леса службы с помощью стандартных средств и процедур.  
- Защита от оперативных изменений в других доменах. Корневой домен выделенный лес не представляет конкретного географического региона в структуру доменов. По этой причине он не зависит от реорганизации или другие изменения, которые приводят к переименование, либо путем реструктуризации доменов.  
- Служит в качестве корня нейтральный так, чтобы быть подчинено в другой регион не страны или региона. В некоторых организациях может предпочесть избежать внешний одной страны или региона является подчиненным для другой страны или региона, в пространстве имен. При использовании с корневым доменом выделенный лес, все региональных доменов может быть одноранговых узлов в иерархии домена.  

В среде нескольких язык домена, в котором используется выделенным корнем леса репликации корневого домена леса оказывает минимальное влияние на сетевой инфраструктуры. Это обусловлено корень леса только узлы, расположенные учетных записей администраторов служб. Большинство учетных записей пользователей в лесу и другие данные, зависящие от домена, хранятся в региональных доменов.  
  
Один недостаток: дополнительная используется выделенный лес корневой домен создается дополнительных затрат на управление для поддержки дополнительного домена.  
  
### <a name="regional-domain-as-a-forest-root-domain"></a>Регионального домена в качестве корневого домена леса

Если вы не захотите развернуть корневой домен выделенный лес, необходимо выбрать регионального домена в качестве корневого домена леса. Этот домен является родительский домен всех региональных доменов и будет первый домен, который развертывается. Корневого домена леса содержит учетные записи пользователей и управляется таким же образом, что ведется региональных доменов. Основное различие заключается в том, что он также включает группы "Администраторы предприятия" и "Администраторы схемы".  
  
Создает преимущество выбора регионального домена функции, как что он не создает дополнительных затрат на управление, обслуживание дополнительного домена корневого домена леса. Выберите соответствующий региональный домен для быть корнем леса, например домена, который представляет ваш главный офис или области, которая имеет самый быстрый сетевых подключений. Если это сложно для вашей организации выбрать регионального домена корневого домена леса, можно вместо этого используйте выделенный лес корневой модели.  
  
## <a name="assigning-the-forest-root-domain-name"></a>Назначение имя корневого домена леса

Имя корневого домена леса совпадает с именем леса. Имя корня леса — это имя системы доменных имен (DNS), которая состоит из префикса и суффикса в виде prefix.suffix. Например организация может использовать corp.contoso.com имя корень леса. В этом примере corp является префиксом, а contoso.com — суффикс.  
  
Выберите суффикс из списка существующих имен в сети. Для префикса выберите новое имя, которое не используется в сети, ранее. Присоединив новый префикс для существующего суффикса, создайте уникальное пространство имен. Создание нового пространства имен для доменных служб Active Directory (AD DS) гарантирует, что существующую инфраструктуру DNS не нужно изменить в соответствии с AD DS.  
  
### <a name="selecting-a-suffix"></a>Выбрав суффикс

Чтобы выбрать суффикса для корневого домена леса:  
  
1. Обратитесь к владельцу DNS для организации для получения списка зарегистрированных DNS-суффиксов, которые используются в сети, где будет размещаться AD DS. Обратите внимание, что суффиксы, используемые во внутренней сети может отличаться от суффиксы, используемые извне. Например организация может использовать contosopharma.com в Интернете и contoso.com во внутренней корпоративной сети.  
  
2. Обратитесь к владельцу DNS выберите суффикс для использования с AD DS. Если нет подходящего суффиксы существует, зарегистрируйте новое имя в именования центра Интернета.  
  
Мы рекомендуем использовать DNS-имена, которые зарегистрированы с центром Интернета в пространстве имен Active Directory. Гарантируется, что только зарегистрированные имена быть глобально уникальным. Если впоследствии другую организацию регистрирует имя домена DNS (или если ваша организация объединения с или приобретается другой компанией, которая использует то же имя DNS), инфраструктуры, в которой два не может взаимодействовать друг с другом.  
  
> [!CAUTION]  
> Не следует использовать однокомпонентное DNS-имена. Дополнительные сведения см. сведения о настройке Windows для доменов с однокомпонентными DNS-имена ([https://go.microsoft.com/fwlink/?LinkId=106631](https://go.microsoft.com/fwlink/?LinkId=106631)). Кроме того с помощью незарегистрированные суффиксов, например .local не рекомендуется.  
  
### <a name="selecting-a-prefix"></a>Выбрав префикс

Если вы выбрали зарегистрированных суффикс, который уже используется в сети, выберите префикс для имя корневого домена леса с помощью правил префикс в следующей таблице. Добавьте префикс, который сейчас не используется для создания нового имени подчиненных. Например если корневой DNS-имя — contoso.com, можно создать concorp.contoso.com имя для корневого домена Active Directory леса, если concorp.contoso.com пространство имен уже не используется в сети. Это новая ветвь, пространства имен будет выделен для AD DS и можно легко интегрировать с существующей реализации DNS.  
  
Если вы выбрали регионального домена в качестве корневого домена леса, может потребоваться выбрать новый префикс для домена. Так как имя корневого домена леса влияет на все другие имена доменов в лесу, географически на основе имени может не подходить. Если вы используете новый суффикс, который сейчас не используется в сети, можно использовать как имя корневого домена леса без выбора дополнительных префикс.  
  
Ниже перечислены правила выбора префикс для зарегистрированного DNS-имени.  
  
|Правило|Объяснение|  
|--------|---------------|  
|Выберите префикс, скорее всего, окажется устаревшей.|Избегайте имен, например линию продукта или операционной системы, которые могут измениться в будущем. Мы рекомендуем использовать универсальными именами, например corp или доменных служб Active Directory.|  
|Выберите префикс, который включает только стандартные символы Интернета.|A-Z, a – z, 0-9 и (-), но не полностью числовой.|  
|Включить 15 символов или меньше префикс.|Если длина префикса равна 15 символов или меньше, NetBIOS-имя является таким же, как префикс.|  
  
Очень важно для владельца Active Directory DNS для работы с владельца DNS организации, чтобы получить права владельца объекта имя, которое будет использоваться для пространства имен Active Directory. Дополнительные сведения о проектировании инфраструктуры DNS, чтобы обеспечить поддержку AD DS см. в разделе [Создание проекта инфраструктуры DNS](../../ad-ds/plan/Creating-a-DNS-Infrastructure-Design.md).  
  
## <a name="documenting-the-forest-root-domain-name"></a>Документирование имя корневого домена леса

Запишите DNS-префикс и суффикс, который выбран для корневого домена леса. На этом этапе определите, какой домен будет корень леса. Сведения об имени для корневого домена леса можно добавить в лист «Планирование домена», созданный для документирования план для новых и обновленных доменов и доменных имен. Чтобы открыть его, загрузите Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip из [задания вспомогательные средства для Windows Server 2003 Deployment Kit](https://go.microsoft.com/fwlink/?LinkID=102558) и откройте «Домена планирование» (DSSLOGI_5.doc).
