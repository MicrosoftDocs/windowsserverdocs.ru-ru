# <a name="selecting-the-forest-root-domain"></a>Выбор корневого домена леса

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Первый домен, который можно развернуть в лесу Active Directory, называется корневым доменом леса. Этот домен остается корневого домена леса для жизненного цикла развертывания служб AD DS.  
  
Корневой домен леса содержит группы "Администраторы предприятия" и "Администраторы схемы". Эти группы администратора служб используются для управления леса операций, таких как добавление и удаление доменов и реализацию изменений в схему.  
  
Выбор корневого домена леса включает в себя определение Если одного из доменов Active Directory в конструкции своего домена может работать в качестве корневого домена леса, или если необходимо выполнить развертывание корневого домена леса выделенный.  
  
Сведения о развертывании корневого домена леса см. в разделе [развертывание корневого домена леса Windows Server 2008](https://technet.microsoft.com/library/cc731174.aspx).  
  
## <a name="choosing-a-regional-or-dedicated-forest-root-domain"></a>Выбор корневого домена леса региональные или выделенный  
Если вы применяете модель с одним доменом, один домен работает в качестве корневого домена леса. Если вы применяете несколько модели домена, можно развернуть корневой домен леса выделенный или выберите регионального домена в качестве корневого домена леса.  
  
### <a name="dedicated-forest-root-domain"></a>Выделенный лес корневого домена  
Выделенный лес корневой домен является доменом, созданный специально в качестве корня леса. Он не содержит все учетные записи пользователей, отличные от учетных записей администратора службы для корневого домена леса. Кроме того он не представляет любой географический регион, в структуру домена. Все домены в лесу являются дочерними для корневого домена выделенный лес.  
  
С помощью выделенным корнем леса обеспечивает следующие преимущества:  
  
-   Операционный разделение администраторов служб леса из администраторов служб домена. В среде с одним домена члены групп "Администраторы домена" и встроенной группы администраторов можно использовать стандартные средства и процедуры для обеспечения членов групп "Администраторы предприятия" и "Администраторы схемы". В лесу, использует выделенный лес корневого домена, члены групп "Администраторы домена" и встроенной группы администраторов в региональных доменов не могут принять сами члены группы администраторов уровня леса службы с помощью стандартных средств и процедур.  
  
-   Защита от рабочей изменений в других доменах. Выделенный лес корневого домена не представляет определенного географического региона в структуру домена. По этой причине не затронуты преобразования доменов или другие изменения, которые привести к переименованием и реструктуризация доменов.  
  
-   Служит в качестве корня нейтральный, чтобы не страну или регион, вероятно, подчиненный в другой регион. В некоторых организациях отдают предпочтение во избежание внешний одной страны или региона подчинен в другую страну или регион, в пространстве имен. При использовании корневой домен леса выделенный всех региональных доменов может быть одноранговые элементы в иерархии доменов.  
  
В среде с несколькими региональные домена, в котором используется выделенным корнем леса репликация корневого домена леса будет оказывать минимальное влияние на сетевую инфраструктуру. Это вызвано корня леса размещается только учетные записи администратора. Большинство учетных записей пользователей в лесу и другие данные зависят от домена, хранятся в региональных доменов.  
  
Недостаток с помощью корневого домена леса выделенный создается расходы на управление дополнительных для поддержки дополнительных домена.  
  
### <a name="regional-domain-as-a-forest-root-domain"></a>Регионального домена в качестве корневого домена леса  
Если вы не планируете развертывание корневого домена леса выделенный, необходимо выбрать регионального домена в качестве корневого домена леса. Этот домен является родительский домен всех других региональных доменов и будет первый домен, который развертывается. Корневой домен леса содержит учетные записи пользователей и осуществляется в так же, что управляются других региональных доменов. Основным различием является то, что он включает групп "Администраторы предприятия" и "Администраторы схемы".  
  
Создает преимуществами выбора регионального домена функции, как том, что он не создает расходы на управление дополнительные поддержка дополнительного домена корневого домена леса. Выберите соответствующий регионального домена, для корня леса, например домена, который представляет ваш главный офис или области, которая имеет быстрым сетевых подключений. Если он также труден для вашей организации выбрать регионального домена для корневого домена леса, можно использовать вместо него модель корневого выделенный лес.  
  
## <a name="assigning-the-forest-root-domain-name"></a>Назначение имя корневого домена леса  
Имя корневого домена леса также является имя леса. Имя корня леса — это имя системы доменных имен (DNS), которая состоит из префикса и суффиксов в виде prefix.suffix. Например организация может иметь имя корня леса corp.contoso.com. В этом примере corp — это префикс, и contoso.com является суффикс.  
  
Выберите суффикс список существующих имен в сети. В качестве префикса выберите новое имя, которое не используется в вашей сети, ранее. Можно подключить новый префикс к существующего суффикса, создайте уникальное пространство имен. Создание нового пространства имен для доменных служб Active Directory (AD DS) гарантирует, что любой имеющуюся инфраструктуру DNS не требуется изменены в соответствии с AD DS.  
  
### <a name="selecting-a-suffix"></a>Выбрав суффикс  
Чтобы выбрать суффикса для корневого домена леса:  
  
1.  Обратитесь к владельцу DNS для организации список зарегистрированных DNS-суффиксы, которые используются в сети, которые будут размещаться в Доменных службах Active Directory. Обратите внимание, что суффиксы, используемые во внутренней сети, может отличаться от суффиксы, используемые извне. Например организация может использовать contosopharma.com в Интернете и contoso.com во внутренней корпоративной сети.  
  
2.  Обратитесь к владельцу DNS-сервера, выберите суффикс, для использования в Доменных службах Active Directory. Если нет подходящих суффиксы существует, регистрации нового имени с именования центра через Интернет.  
  
Мы рекомендуем использовать DNS-имена, которые зарегистрированы с центром Интернета в пространстве имен Active Directory. Только зарегистрированные имена гарантированно глобально уникальным. Если позже в другой организации регистрирует то же имя домена DNS (или вашей организации объединения с или приобрести другой компанией, использует то же имя DNS), две инфраструктуры не могут взаимодействовать друг с другом.  
  
> [!CAUTION]  
> Не следует использовать однокомпонентное DNS-имена. Дополнительные сведения см. сведения о настройке Windows для доменов с однокомпонентное DNS-имена ([https://go.microsoft.com/fwlink/?LinkId=106631](https://go.microsoft.com/fwlink/?LinkId=106631)). Кроме того не рекомендуется использовать Незарегистрированное суффиксов, например .local.  
  
### <a name="selecting-a-prefix"></a>Выбор префикса  
Если вы выбрали зарегистрированного суффикса, который уже находится в сети, выберите префикс для имя корневого домена леса с помощью правил префикс в таблице ниже. Добавьте префикс, который не используется для создания нового имени подчиненного в настоящее время. Например если DNS-имя корневого contoso.com, можно создать concorp.contoso.com имя для корневого домена Active Directory леса, если пространство имен concorp.contoso.com уже не используется в сети. Этот новый branch пространство имен будет выделен для AD DS и можно легко интегрировать с существующей реализации DNS-сервера.  
  
Если вы выбрали регионального домена в качестве корневого домена леса, может потребоваться выбрать новый префикс для домена. Поскольку имя корневого домена леса влияет на другие имена доменов в лесу, на региональном уровнях на основе имени могут не подходить. Если вы используете новый суффикс, который в настоящее время не используется в сети, можно использовать как имя корневого домена леса без выбора дополнительных префикс.  
  
В следующей таблице перечислены правила для выбора префикс для зарегистрированного DNS-имени.  
  
|Правило|Объяснение|  
|--------|---------------|  
|Выберите это префикс, который не окажется устаревшей.|Избегайте названий, таких как семейство продуктов или операционную систему, может измениться в будущих. Мы рекомендуем использовать универсальными именами, например, предприятие или доменных служб Active Directory.|  
|Выберите это префикс, который включает только стандартные символы Интернета.|0 – 9, A-Z, a – z и (-), но не полностью числовой.|  
|Содержит 15 символов или меньше префикса.|Если вы выберите длину префикса 15 символов или меньше, имя NetBIOS — так же, как префикс.|  
  
Очень важно для владельца Active Directory DNS для работы с DNS владельца организации, чтобы стать владельцем этого имя, которое будет использоваться для пространства имен Active Directory. Дополнительные сведения о разработке инфраструктуры DNS для поддержки Доменных службах Active Directory см. в разделе [Создание проекта инфраструктуры DNS](../../ad-ds/plan/Creating-a-DNS-Infrastructure-Design.md).  
  
## <a name="documenting-the-forest-root-domain-name"></a>Документирование имя корневого домена леса  
Запишите префикс DNS и суффикс, который выбран для корневого домена леса. На этом этапе определите, какой домен будет корня леса. Сведения об имени корневого домена леса можно добавить в журнал «Планирование домена», созданный для документирования план для новых и обновленных доменов и имен доменов. Чтобы открыть его, скачайте Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip из задания вспомогательные средства для Windows Server 2003 Deployment Kit ([https://go.microsoft.com/fwlink/?LinkID=102558](https://go.microsoft.com/fwlink/?LinkID=102558)) и откройте «Домена планирования» (DSSLOGI_5.doc).
