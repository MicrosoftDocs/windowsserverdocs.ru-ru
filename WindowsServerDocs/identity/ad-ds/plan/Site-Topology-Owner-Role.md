# <a name="site-topology-owner-role"></a>Роль владельца топологии сайта

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Администратора, управляющего топологии сайтов называется владельца топологии сайта. Владелец топологии сайтов понятны условия сети между сайтами и имеет полномочия на изменение параметров в доменных службах Active Directory (AD DS) для реализации изменений в топологии сайта. Изменение топологии сайта влияет на изменения в топологии репликации. Владелец узла топологии обязанности входит:  
  
-   Управление изменениями для топологии сайтов, при изменении подключения к сети.  
  
-   Получение и обслуживание сведения о сетевых подключений и маршрутизаторы в группе сети. Владелец топологии сайтов необходимо поддерживать список адресов подсети, маски подсети и расположение, к которой принадлежит каждый. Владельца топологии сайта необходимо также понимать проблемы о скорости сетевого подключения и их производительности, которые влияют на топологию сайт эффективной настройке стоимость связи сайтов.  
  
-   Перемещение объекты сервера Active Directory, представляющие контроллеры домена между сайтами, если изменения IP-адреса контроллера домена в другую подсеть, в другом сайте, или если подсеть назначается другому сайту. В любом случае владельца топологии сайта необходимо вручную переместить объект сервера контроллера домена Active Directory на новый сайт.
