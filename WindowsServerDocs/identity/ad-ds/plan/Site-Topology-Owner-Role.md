# <a name="site-topology-owner-role"></a>Роль владельца топологии сайта

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Администратор, который управляет топологии сайта называется владельца топологии сайта. Владельца топологии сайта понимает условий сети между сайтами и прав для изменения параметров в доменных службах Active Directory (AD DS) для реализации изменений в топологии сайта. Изменение топологии сайта влияет на изменения в топологии репликации. Ниже перечислены обязанности владельца топологии сайта.  
  
-   Управление изменения топологии сайта при изменении сетевого подключения.  
  
-   Получение и обслуживание сведения о сетевых подключений и маршрутизаторы сетевой группы. Владельца топологии сайта необходимо поддерживать список адреса подсети, маски подсети и расположения, к которому принадлежит каждого. Владельца топологии сайта также должны понимать все проблемы о скорости сети и их производительности, которые влияют на топологию сайта для эффективной настройке затраты для связи сайтов.  
  
-   Перемещение объектов Active Directory сервера, представляющий контроллеры домена между сайтами, если изменения IP-адреса контроллера домена в другой подсети, в другом сайте, или если подсеть назначается другому сайту. В любом случае владельца топологии сайта необходимо вручную переместить Active Directory объект сервера контроллера домена для нового сайта.
