# <a name="reviewing-ou-design-concepts"></a>Общие сведения о понятиях проекта подразделения

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Структура подразделения (OU) для домена включает в себя следующее:  
  
-   Схема иерархии Подразделений  
  
-   Список подразделений  
  
-   Для каждого Подразделения:  
  
    -   Назначение для Подразделения  
  
    -   Список пользователей или групп, которые могут контролировать, Подразделения или объекты в Подразделении  
  
    -   Тип элемента управления, пользователи и группы имеют над объектами в Подразделении  
  
Иерархия Подразделений не требуется отразить иерархию подразделений организации или группы. Подразделения создаются для определенной цели, например делегирования администрирования, приложение групповой политики или чтобы ограничить видимость этих объектов.  
  
Можно разработать структуру Подразделений и делегировать администрирование для отдельных пользователей или групп в вашей организации, которым требуется автономность для управления свои собственные данные и ресурсы. Подразделения представляют административные границы и позволяют контролировать объем полномочий администраторов данных.  
  
Например можно создать Подразделение, вызывается ResourceOU и использовать для хранения всех учетных записей компьютеров, принадлежащих к файлу и серверов печати под управлением группы. Затем можно настроить безопасности в Подразделении, таким образом, чтобы только администраторы данных в группе имеют доступ к Подразделению. Благодаря этому администраторы данных в других группах незаконное изменение файла и учетных записей сервера печати.  
  
Структура Подразделений можно уточнить, создав поддеревьев подразделений для определенных целей, например приложение групповой политики или чтобы ограничить видимость этих защищенных объектов, чтобы только определенные пользователи могут видеть их. Например если вам нужно применить групповую политику, выберите группу пользователей или ресурсов, можно добавить этих пользователей или ресурсы в Подразделение и затем применить групповую политику к этому Подразделению. Иерархия Подразделений также можно дополнительно включить делегирование административного управления.  
  
Хотя нет технических ограничений на количество уровней в вашем структурном подразделении, управляемость рекомендуется ограничить структуру Подразделений и глубину не более 10 уровней. Нет технических ограничений на число подразделений на каждом уровне. Обратите внимание, службе (AD DS) — приложения, могут иметь ограничения на число символов, используемых в различающееся имя (то есть полный Lightweight Directory Access Protocol (LDAP) путь к объекту в каталоге) или на Подразделение глубина в иерархии.  
  
Структуры Подразделений в AD DS не должен отображаться для конечных пользователей. Структуры Подразделений является средством администрирования для администраторов служб и для администраторов данных и легко изменить. По-прежнему просматривать и обновлять проектирования структуры Подразделений для отражения изменений в структуру администрирования и для поддержки администрирования на основе политик.
