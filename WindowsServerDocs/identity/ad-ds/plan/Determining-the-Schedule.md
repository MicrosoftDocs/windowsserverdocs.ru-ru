# <a name="determining-the-schedule"></a>Определение расписания

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Доступность связи сайтов можно управлять, задав расписание для связи сайтов. При репликации между двумя сайтами проходит через несколько связей сайтов, пересечение расписания репликации на все соответствующие ссылки определяет расписание подключения между двумя сайтами.  
  
Для планирования для установки по расписанию связи сайтов, создайте два пересечения временных интервалов между связей сайтов, которые содержат контроллеры домена, которые непосредственно реплицировать друг с другом. Используйте расписание по умолчанию (100 - процентную доступных) для этих связей, только если вы хотите заблокировать трафик репликации в часы пик. Блокировки репликацией приоритетное другой трафик, но можно также повысить задержку репликации.  
  
Контроллеры домена хранения времени в формате UTC (UTC). Параметры времени в расписания объекта связей сайтов соответствует местного времени сайта и компьютера, на котором устанавливается расписание. Когда контроллер домена обращается компьютера, находящегося в другой сайт и часового пояса, расписание на контроллере домена отображает параметр времени в соответствии с местным временем для узла компьютера.
