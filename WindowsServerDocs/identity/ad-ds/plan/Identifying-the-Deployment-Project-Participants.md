# <a name="identifying-the-deployment-project-participants"></a>Определение участников проекта развертывания

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Является первым этапом установки проекта развертывания для доменной службы Active Directory (AD DS) для создания проекта и в цикле группы проекта развертывания, которые будете нести ответственность за управление этапе проектирования и этапа развертывания проекта Active Directory. Кроме того необходимо определить отдельных пользователей и групп, которые будут отвечать за владельца и обслуживание каталоге, после завершения развертывания.  
  
-   [Определение ролей для конкретного проекта](#BKMK_1)  
  
-   [Установка владельцев и администраторов](#BKMK_2)  
  
-   [Создание групп проекта](#BKMK_3)  
  
## <a name="BKMK_1"></a>Определение ролей для конкретного проекта  
Для идентификации пользователей, которые являются для конкретного проекта ролями является важным шагом для установления групп проекта. К ним относятся заказчиком проекта, архитектор проекта и руководитель проекта. Эти пользователи ответственны за под управлением проекта развертывания Active Directory.  
  
После назначенный архитектора проекта и руководитель проекта пользователями устанавливать каналов обмена данными в организации, построение проекта расписания и определить пользователей, которые будут членами групп проекта, начиная с различных владельцев.  
  
### <a name="executive-sponsor"></a>Заказчиком  
Развертывание инфраструктуры, таких как AD DS могут оказать влияние разнообразных в организации. По этой причине важно иметь заказчиком, понимают ценность для бизнеса развертывания, поддерживающие проект на уровне руководства и может устранить конфликты в организации.  
  
### <a name="project-architect"></a>Архитектор проекта  
Каждый проекта развертывания Active Directory требуется архитектором проекта для управления Active Directory разработке и развертыванию процесс принятия решений. Архитектор предоставляет техническую поддержку для упрощения процесса проектирования и развертывания AD DS.  
  
> [!NOTE]  
> Если не существующие в вашей организации проектирования directory взаимодействие, может потребоваться привлекать внешний консультант опытного в Active Directory проектирования и развертывания.  
  
Ниже перечислены обязанности архитектора проекта Active Directory.  
  
-   Владелец структуры Active Directory  
  
-   Общие сведения о воспроизведении и записи основой для ключевых проектных решениях  
  
-   Убедившись, что проект соответствует потребностям организации  
  
-   Установление согласования между проектирования, развертывания и рабочими группами  
  
-   Общее представление о потребностей AD DS, интегрированный приложений  
  
В окончательном проекте Active Directory должно отражать сочетание бизнес-целей и технические решения. Таким образом архитектор проекта необходимо просмотреть проектные решения, чтобы убедиться, что они соответствуют бизнес-целей.  
  
### <a name="project-manager"></a>Руководитель проекта  
Руководитель проекта облегчает взаимодействие между бизнес-единиц и между группами управления технологии. В идеале руководитель проекта развертывания Active Directory является кому-либо в организации, хорошо знакомый с действующие политики ИТ-группа и требования к разработке для групп, Подготовка к развертыванию Доменных службах Active Directory. Руководитель проекта отвечает за весь процесс развертывания проекта, начиная с разработки и заканчивая реализации и гарантирует, что проект остается в соответствии с расписанием и бюджетом. Ниже перечислены обязанности руководитель проекта.  
  
-   Предоставление базового проекта, планирования, как планирование и бюджета  
  
-   Ход выполнения, влияющие на проектирования и развертывания проекта Active Directory  
  
-   Убедитесь, что участвующие в каждый этап этого процесса разработки соответствующих лиц  
  
-   Обслуживает как единая точка контакта для проекта развертывания Active Directory  
  
-   Установление связи между проектирования, развертывания и рабочими группами  
  
-   Установка и обслуживание связи с заказчиком на протяжении всего процесса развертывания  
  
## <a name="BKMK_2"></a>Установка владельцев и администраторов  
В проекта развертывания Active Directory тех, кто владельцев занятые систем, отвечающие управления обязательно развертывания, выполнения задач и этой структуры Active Directory спецификации соответствуют потребностям организации. Владельцы не обязательно иметь доступ к или напрямую управлять инфраструктуре каталогов. Администраторы могут сотрудников, ответственных за выполнение задач обязательного развертывания. Администраторы имеют доступ к сети и разрешения, необходимые для управления каталоге и ее инфраструктурой.  
  
Роль владельца, стратегические и отдельного. Владельцы отвечают за взаимодействие администраторам задачи, необходимые для реализации структуры Active Directory, таких как создание новых контроллеров домена в лесу. Администраторы несут ответственность за реализации проекте в сети согласно спецификации проектирования.  
  
В крупных организациях разным пользователям fill роли владельца и администратора. Однако в некоторых организациях малого же лицо может выступать в качестве владельца и администратора.  
  
### <a name="service-and-data-owners"></a>Владельцы служб и данных  
Управление AD DS на ежедневной основе включает в себя два типа владельцев:  
  
-   Владельцы служб, отвечающих для планирования и долгосрочного обслуживания инфраструктуры Active Directory и обеспечения каталоге продолжает работать и сохранением целей, установленным для соглашения об уровне обслуживания  
  
-   Владельцы данных, несущие ответственность за обслуживание данные, хранящиеся в каталоге. Это включает в себя и управление учетной записью компьютера управления пользователями и локальных ресурсов, например, рядовые серверы и рабочие станции.  
  
Важно раньше определить владельцев служб и данных Active Directory, чтобы они могут принимать участие в столько процесса проектирования максимально. Так как владельцы служб и данных несете ответственность за обслуживание каталога после завершения проекта развертывания, важно для этих пользователей, для предоставления входных данных, касающихся потребностей организации и быть знакомы с выполняются как и почему некоторые проектные решения. Служба владельцам относятся Владелец леса Active Directory доменных имен (DNS) и владельца владельца топологии сайта. Данные владельцам относятся владельцев подразделение (OU).  
  
### <a name="service-and-data-administrators"></a>Администраторы служб и данных  
Операции AD DS включает два типа администраторов: Администраторы и Администраторы данных службы. Администраторы служб реализации политики решений, принятых владельцами службы и обрабатывать повседневные задачи, связанные с обслуживанием службы каталогов и инфраструктуры. Это включает в себя управление контроллеры домена, на которых размещены службы каталогов, управление другие сетевые службы, такие как DNS, которые требуются для служб AD DS, управление конфигурацией параметров на уровне леса и убедившись, что каталог доступен.  
  
Администраторы служб также несете ответственность за завершения текущих задач развертывания Active Directory, которые необходимы после завершения начальной процесса развертывания Windows Server 2008 Active Directory. Например как нагрузку на увеличение каталог администраторов служб создания дополнительных контроллеров домена и устанавливать или удалять отношения доверия между доменами, при необходимости. По этой причине группа развертывания Active Directory должна включать администраторов служб.  
  
Будьте внимательны, чтобы назначать роли администратора службы только доверенных людей в организации. Поскольку эти пользователи имеют возможность изменять системные файлы на контроллерах домена, они могут изменить поведение AD DS. Необходимо убедиться, что администраторы служб в вашей организации, пользователей, которые знакомы с операционные и политик безопасности, в сети и кто понять необходимость соблюдения этих политик.  
  
Администраторы данных, пользователи в домене, несущие ответственность за сохранение данных, хранящихся в Доменных службах Active Directory, такие как учетные записи пользователей и групп и поддержка компьютеров, являющихся членами домена. Администраторы данных управления подмножества объекты в каталоге и контролировать установку или конфигурацией службы каталогов.  
  
Данные учетные записи администратора не предоставляются по умолчанию. После того как специалисты по разработке определит, как ресурсы должны управляться для организации, владельцы домена необходимо создания учетных записей администратора данных и делегировать их соответствующие разрешения на основе набора объектов, для которых администраторы должны быть ответственность.  
  
Рекомендуется ограничить количество администраторов служб в организации, минимальное количество необходимых для обеспечения инфраструктуры продолжает работать. Большинство административных рабочих может выполняться администраторами данных. Администраторы служб требуется гораздо шире навык настроено, поскольку они несете ответственность за поддержание каталога и инфраструктуре, которая поддерживает его. Администраторы данных требуется только навыки, необходимые для управления своей частью каталога. Разделение назначения таким образом приводит экономии для организации, так как небольшое количество администраторы должны научиться работы и поддержки всего каталога и ее инфраструктурой.  
  
Например администратор службы необходимо понять, как добавить домен в лесу. Сюда входят как установить программное обеспечение для преобразования сервера в контроллер домена и как управлять среда DNS, чтобы контроллер домена могут быть объединены в среде Active Directory без проблем. Администратор данных только необходимо знать, как управлять конкретные данные, в том, что он отвечает за, таких как создание новых учетных записей пользователей для новые сотрудники отдела их.  
  
Развертывание AD DS требует координации и связь между множество различных групп, участвующих в данной операции сетевой инфраструктуры. Эти группы следует назначать владельцев служб и данных, отвечающих за представляющий различных групп во время процесса проектирования и развертывания.  
  
После завершения проекта развертывания владельцев эти службы и данные по-прежнему нести ответственность за часть инфраструктуры управляется их группе. В среде Active Directory указанных владельцев являются Владелец леса, DNS для владельца AD DS, владельца топологии сайта и владелец Подразделение. В следующих разделах описаны роли владельцев этих служб и данных.  
  
#### <a name="forest-owner"></a>Владелец леса  
Владелец леса обычно — это диспетчер технологий (ИТ) старший сведения в организации, кто отвечает за процесс развертывания Active Directory и в конечном счете кто отвечает за поддержание предоставление услуг в пределах леса, после завершения развертывания. Владелец леса назначает лиц для заполнения других ролей владения путем определения ключа персонала в организации, получающих contribute необходимые сведения о сетевой инфраструктуре и потребностями администрирования. Владелец леса отвечает за следующее:  
  
-   Развертывание корневого домена леса, чтобы создать лес  
  
-   Развертывание первого контроллера домена в каждом домене, для создания доменов, необходимых для леса  
  
-   Членство группы администраторов службы во всех доменах леса  
  
-   Создание проекта структуры Подразделений для каждого домена в лесу  
  
-   Делегирование административных прав владельцев Подразделения  
  
-   Изменения схемы  
  
-   Изменения параметров конфигурации на уровне леса  
  
-   Реализация определенных параметров групповой политики, включая политики учетной записи пользователя домена, например детально настроенной политики паролей и блокировки  
  
-   Бизнес-параметры политики, применяемые к контроллерам домена  
  
-   Другие параметры групповой политики, которые применяются на уровне домена  
  
Владелец леса имеет центр по всему лесу. Именно владелец леса для настройки групповой политики и бизнес-политик и выберите пользователей, которые являются администраторами служб. Владелец леса является владельцем службы.  
  
#### <a name="dns-for-ad-ds-owner"></a>DNS-сервер для владельца AD DS  
DNS-сервер для владельца AD DS является лицо, которое имеет глубокого понимания имеющуюся инфраструктуру DNS и существующим пространством имен организации.  
  
DNS-сервер для AD DS владельца отвечает за следующее:  
  
-   Служащий в качестве посредника между специалистов по разработке и ИТ-группа, которому принадлежит инфраструктуры DNS  
  
-   Предоставление сведений о существующим пространством имен DNS организации, чтобы помочь в создании новое пространство имен Active Directory  
  
-   Работа с группы развертывания, чтобы убедиться в том, что новую инфраструктуру DNS развертывается согласно спецификации специалистов по разработке и что он работает должным образом  
  
-   Управление DNS для инфраструктуры AD DS, включая службы DNS-сервера и данных DNS  
  
DNS-сервер для AD DS владельца является владельцем службы.  
  
#### <a name="site-topology-owner"></a>Владельца топологии сайта  
Владельца топологии сайта хорошо знаком с физическую структуру сети организации, включая сопоставление отдельных подсети и маршрутизаторы, областях сети, подключенные с помощью медленных подключений. Владельца топологии сайта отвечает за следующее:  
  
-   Общее представление о топологии физической сети и как это влияет на Доменных службах Active Directory  
  
-   Общее представление о влияние сети развертывания Active Directory  
  
-   Определение логических сайтов Active Directory, которые должны быть созданы  
  
-   Обновление сайта объектов контроллеров домена при добавлении подсети, изменить или удалить  
  
-   Создание связи сайтов, мостов связей сайтов и объектов подключения вручную  
  
Владельца топологии сайта является владельцем службы.  
  
#### <a name="ou-owner"></a>Владелец Подразделения  
Владелец Подразделение отвечает за управление данные, хранящиеся в каталоге. Данного пользователя должен быть знаком с операционные и политик безопасности, которые находятся в месте сети. Владельцы Подразделения можно выполнить только те задачи, которые были делегированы их администраторами службы, и могут выполнять только те задачи, для подразделений, которым они назначены. Следующие задачи, которые могут быть назначены владельца Подразделение:  
  
-   Выполнение всех задач управления учетной записью в рамках назначенных Подразделения  
  
-   Управление рабочих станций и рядовых серверах, которые являются членами их назначенного подразделения  
  
-   Делегирование полномочий для локальных администраторов в рамках назначенных Подразделения  
  
Владелец Подразделение — владельца данных.  
  
## <a name="BKMK_3"></a>Создание групп проекта  
Проектные группы Active Directory являются временные группами, которые отвечают за выполнение задач проектирования и развертывания службы каталогов Active Directory. После завершения проекта развертывания Active Directory владельцев принимать ответственность за каталоге и можно расформировать групп проекта.  
  
Размер групп проекта зависит от размера организации. В небольшой организации один человек относятся к нескольким областям ответственности группы проекта и участвует в нескольких этапа развертывания. Крупные организации могут потребовать больших команд с разным пользователям или даже разными рабочими группами, охватывающих области ответственности. Размер группы не важен, как назначаются всем областям ответственности, а достигнуты разработки цели организации.  
  
### <a name="identifying-potential-forest-owners"></a>Определение возможных владельцев леса  
Определите группы в вашей организации, владеющие и управлять ресурсами, необходимых для предоставления служб каталогов для пользователей в сети. Эти группы считаются потенциальных владельцев леса.  
  
Разделение данных и служебных администрирования в Доменных службах Active Directory позволяет инфраструктуры ИТ группы (или группы) организации для управления службы каталогов, пока локальных администраторов в каждой группе управления данными, к которой принадлежит собственные группы. Возможные владельцы леса имеют достаточных прав через сетевой инфраструктуры для развертывания и поддержки AD DS.  
  
Для организаций, имеющих один централизации инфраструктуры ИТ-группы группы ИТ обычно является владельцем леса и, следовательно, возможных владельца леса для любой будущих развертываний. Организации, которые включают несколько независимых ИТ-инфраструктуры группы имеют ряд потенциальных владельцев леса. Если в вашей организации уже имеется инфраструктура Active Directory, любой текущий владелец леса, также возможных владельцев леса для новых развертываний.  
  
Выберите один из возможных владельцев леса в качестве владельца леса для каждого леса, который вы собираетесь для развертывания. Для работы с командой разработки, чтобы определить ли своего леса выполняется фактическое развертывание или если альтернативный набор действий (например, присоединение другого существующего леса) лучше использовать доступные ресурсы и по-прежнему соответствует их потребностям отвечают указанных потенциальных владельцев леса. Владелец леса (или владельцы) в организации являются членами группы специалистов по разработке Active Directory.  
  
### <a name="establishing-a-design-team"></a>Установка группы разработки  
Специалисты по разработке Active Directory отвечает за сбор всех сведений, необходимые для принятия решений о проектирования логической структуры Active Directory.  
  
Ниже перечислены обязанности специалистов по разработке.  
  
-   Определение количества лесов и доменов являются обязательными и каковы отношения между лесами и доменами  
  
-   Работа с владельцев данных, чтобы обеспечить соответствие их безопасность и административных требований проекта  
  
-   Работа с текущей администраторам сети убедитесь, что текущей сетевой инфраструктуры поддерживает разработку и что проекта не скажется отрицательно на существующих приложений, развертываемых в сети  
  
-   Работа с представителями группы безопасности организации, чтобы обеспечить соответствие настроенные политики безопасности проекта  
  
-   Проектирование структуры Подразделений, обеспечивающих необходимого уровня защиты и правильной делегирование полномочий владельцам данных  
  
-   Работа с специалисты по развертыванию для тестирования проекта в лаборатории среде, чтобы убедиться, что он работает как запланировать и изменения конструкции как необходимые для устранения любых возможных неполадок  
  
-   Создание проектирование топологии сайта, соответствует требованиям репликации леса а перегрузку доступной пропускной способности. Дополнительные сведения о проектировании топологии сайта см. в разделе [проектирование сайта топологии для Windows Server 2008 службы AD DS](https://technet.microsoft.com/library/cc772013.aspx).  
  
-   Работа с специалисты по развертыванию для обеспечения разработки реализовано правильно  
  
Специалисты по разработке включает следующие члены:  
  
-   Возможные владельцы леса  
  
-   Архитектор проекта  
  
-   Руководитель проекта  
  
-   Пользователи, которые отвечают за установление и поддержке политик безопасности в сети  
  
В процессе проектирования логической структуры специалистов по разработке определяет других владельцев. Эти пользователи должны запустить участие в процессе проектирования, как только они определены. После передан в проект развертывания для развертывания группы специалистов по разработке отвечает за надзора за все процесс развертывания, чтобы правильно реализован проекта. Специалисты по разработке также вносит изменения в проект на основе отзывов тестирования.  
  
### <a name="establishing-a-deployment-team"></a>Установка группы развертывания  
Для тестирования и реализации проектирования логической структуры Active Directory работает над развертывания Active Directory. Включает в себя следующие задачи:  
  
-   Установка тестовой среды, которую достаточно эмулирует рабочей среде  
  
-   Тестирование проекта посредством реализации предполагаемой структуры леса и домена в лабораторной среде, чтобы убедиться, что он соответствует целей каждый владелец роли  
  
-   Разработка и любые сценарии миграции, предложенный разработки в среде лаборатории тестирования  
  
-   Убедитесь, что каждый владелец выхода из системы на процесс тестирования для обеспечения тестируемой возможности правильный разработки  
  
-   Тестирование операции развертывания в среде пилотного развертывания  
  
После завершения разработки и тестирования задачи, специалисты по развертыванию выполняет следующие задачи:  
  
-   Создает лесов и доменов согласно проектирования логической структуры Active Directory  
  
-   Создает сайтов и объектов связи сайтов при необходимости на основе на проектирование топологии сайта  
  
-   Гарантирует, что инфраструктуры DNS настроен для поддержки службы AD DS и что все новые пространства имен интегрированы в существующее пространство имен организации  
  
Специалисты по развертыванию Active Directory включает следующие члены:  
  
-   Владелец леса  
  
-   DNS-сервер для владельца AD DS  
  
-   Владельца топологии сайта  
  
-   Владельцы Подразделения  
  
Специалисты по развертыванию работает с администраторами служб и данных на этапе развертывания убедитесь, что участники группы операций знакомы с обновленной. Это позволяет гарантировать плавный переход владения, после завершения операции развертывания. По завершении процесса развертывания ответственность за поддержание новой среды Active Directory передает группа эксплуатации.  
  
### <a name="documenting-the-design-and-deployment-teams"></a>Документирование группами разработки и развертывания  
Документ имена и контактные данные пользователей, которые будут участвовать в проектирования и развертывания службы AD DS. Определите, кто будет отвечать за каждой роли на группами разработки и развертывания. Изначально этот список содержит возможные владельцы леса, руководитель проекта и архитектор проекта. При определении количества лесов, которые будут развертываться может потребоваться создать новый специалисты по разработке для дополнительных лесов. Обратите внимание, что необходимо обновить документации по мере изменения членства в группы и как определить различных владельцев Active Directory в процессе разработки. Лист для документирования группами разработки и развертывания для каждого из лесов можно скачайте Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip из задания вспомогательные средства для Windows Server 2003 Deployment Kit ([https://go.microsoft.com/fwlink/?LinkID=102558](https://go.microsoft.com/fwlink/?LinkID=102558)) и откройте «Разработки и развертывания группы информация» (DSSLOGI_1.doc).
