# <a name="planning-forest-root-domain-controller-placement"></a>Планирование размещения корневого контроллера домена в лесу

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Контроллеры корневого домена леса необходимые для создания путей доверия для клиентов, которым требуется доступ к ресурсам в доменах, отличных от свои собственные. Установите контроллеры корневого домена леса в узловых офисах и в местах, на которых размещаются в центрах обработки данных. Если пользователи в нужном расположении необходим доступ к ресурсам из других доменов в одном расположении и доступности сети между центра обработки данных и расположением пользователя является ненадежным, можно добавить контроллера корневого домена леса в расположении или создать Установленное напрямую доверие между двумя доменами. Это дополнительные средства для создания установленного напрямую доверия между доменами, если у вас нет других причин размещать контроллера корневого домена леса в этом расположении.  
  
Установленное напрямую доверие помогут оптимизировать запросы на проверку подлинности пользователей, расположенных в любом домене. Дополнительные сведения о установленное напрямую доверие между доменами см. в статье [представление о необходимости создания установленного напрямую доверия](https://go.microsoft.com/fwlink/?LinkId=107061).  
  
Лист, чтобы помочь в документировании вашего леса размещения корневого контроллера домена, см. в разделе [задания вспомогательные средства для Windows Server 2003 Deployment Kit](https://go.microsoft.com/fwlink/?LinkID=102558), скачайте Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip и откройте «Размещение контроллера домена» (DSSTOPO_4.doc).  
  
Необходимо будет использовать эту информацию при создании корневого домена леса. Дополнительные сведения о развертывании корневого домена леса, см. в разделе [развертывание корневого домена леса Windows Server 2008](https://technet.microsoft.com/library/cc731174.aspx).
