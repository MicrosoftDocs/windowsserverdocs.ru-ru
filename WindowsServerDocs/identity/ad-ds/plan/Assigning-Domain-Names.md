# <a name="assigning-domain-names"></a>Присвоение имен доменов

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Необходимо назначить имя для каждого домена в плане. Доменов Active Directory доменных служб (AD DS) имеет два типа имен: Доменных имен (DNS) имена и NetBIOS-имена. Как правило имена обоих видимы конечным пользователям. DNS-имена доменов Active Directory включают состоит из двух частей, префикс и суффикс. При создании доменных имен, необходимо сначала Определите префикс DNS. Это первая метка DNS-имя домена. Суффикс определяется при выборе имени корневого домена леса. В следующей таблице перечислены правила именования для DNS-имен префикс.  
  
|Правило|Объяснение|  
|--------|---------------|  
|Выберите префикс, скорее всего, окажется устаревшей.|Избегайте имен, например линию продукта или операционной системы, которые могут измениться в будущем. Мы рекомендуем использовать географические имена.|  
|Выберите префикс, который включает только стандартные символы Интернета.|A-Z, a – z, 0-9 и (-), но не полностью числовой.|  
|Включить 15 символов или меньше префикс.|Если длина префикса равна 15 символов или меньше, NetBIOS-имя является таким же, как префикс.|  
  
Дополнительные сведения см. в разделе соглашения об именовании в Active Directory для компьютеров, доменов, сайтов и подразделений ([https://go.microsoft.com/fwlink/?LinkId=106629](https://go.microsoft.com/fwlink/?LinkId=106629)).  
  
> [!NOTE]  
>  Хотя Dcpromo.exe в Windows Server 2008 и Windows Server 2003 позволяет создавать однокомпонентное DNS-имя домена, не следует использовать однокомпонентное DNS-имя для домена по нескольким причинам. В Windows Server 2008 R2 Dcpromo.exe не позволяет создавать однокомпонентное DNS-имя для домена. Дополнительные сведения см. в разделе [ https://go.microsoft.com/fwlink/?LinkId=92467.](https://go.microsoft.com/fwlink/?LinkId=92467)   
  
Если текущее NetBIOS-имя домена не может использоваться для представления области или не удовлетворяют правилам именования префикс, выберите новый префикс. В этом случае имя NetBIOS домена отличается от префикса DNS домена.  
  
Для каждого нового домена, развертывании выберите префикс, подходит для региона и который удовлетворяет правилам именования префикс. Мы рекомендуем использовать NetBIOS-имя домена, так же, как префикс DNS.  
  
Запишите префикс DNS и NetBIOS-имена, которые выбраны для каждого домена в лесу. Сведения об имени NetBIOS и DNS можно добавить в лист «Планирование домена», созданный для документирования план для новых и обновленных доменов. Чтобы открыть его, загрузить Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip задания вспомогательные средства для Windows Server 2003 Deployment Kit ([https://go.microsoft.com/fwlink/?LinkID=102558](https://go.microsoft.com/fwlink/?LinkID=102558)) и откройте «Домена планирование» (DSSLOGI_5.doc).
