# <a name="determining-the-number-of-domains-required"></a>Определение требуемого количества доменов

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Каждый лес начинается с одного домена. Максимальное число пользователей, которое может содержать один лес домена, зависит от самой медленной ссылки, которая должна поддерживать репликацию между контроллерами домена и доступной пропускной способностью, которую нужно выделить для домен Active Directory служб (AD DS). В следующей таблице указано максимальное рекомендуемое число пользователей, которые могут содержаться в домене, на основе одного леса домена, скорости медленного канала и доли пропускной способности, которую требуется зарезервировать для репликации. Эта информация относится к лесам, которые содержат не более 100 000 пользователей и имеют подключение 28,8 килобит в секунду (кбит/с) или выше. Для получения рекомендаций, относящихся к лесам, которые содержат более 100 000 пользователей или возможность подключения менее 28,8 кбит/с, ознакомьтесь с опытным Active Directoryм конструктором. Значения в следующей таблице основаны на трафике репликации, созданном в среде со следующими характеристиками.  
  
- Новые пользователи присоединяются к лесу с частотой 20% в год.  
- Пользователи оставляют лес с частотой 15% в год.  
- Каждый пользователь является членом пяти глобальных групп и пяти универсальных групп.  
- Отношение пользователей к компьютерам — 1:1.  
- Используется интегрированная система доменных имен (DNS) Active Directory.  
- Используется очистка DNS.  

> [!NOTE]  
> Цифры, перечисленные в следующей таблице, являются приближениями. Количество трафика репликации в основном зависит от количества изменений, внесенных в каталог за определенный период времени. Убедитесь, что сеть может разместить трафик репликации, проверив оценочное количество и частоту изменений в структуре в лаборатории перед развертыванием доменов.  
  
|Медленная связь с подключением контроллера домена (кбит/с)|Максимальное число пользователей, если доступна 1-процентная пропускная способность|Максимальное число пользователей, если доступна 5-процентная пропускная способность|Максимальное число пользователей, если доступна 10-процентная пропускная способность|  
| --- | --- | --- | --- |  
|28,8|10 000|25 000|40 000|  
|32|10 000|25 000|50 000|  
|56|10 000|50 000|100 000|  
|64|10 000|50 000|100 000|  
|128|25 000|100 000|100 000|  
|256|50 000|100 000|100 000|  
|512|80 000|100 000|100 000|  
|1,500|100 000|100 000|100 000|  

Для использования этой таблицы:  

1. На самой **медленной линии, соединяющей столбец контроллер домена** , выберите значение, которое соответствует скорости медленного канала, по которому AD DS будет реплицироваться в вашем домене.  

2. В строке, которая соответствует скорости канала, выберите столбец, представляющий пропускную способность в процентах, которую необходимо выделить для AD DS. Значение в этом расположении — максимальное число пользователей, которое может содержать домен в одном лесу домена.  

Если вы определили, что общее число пользователей в лесу меньше максимального числа пользователей, которое может содержать ваш домен, можно использовать один домен. Не забудьте учесть запланированный будущий рост, когда вы выполните это определение. Если общее число пользователей в лесу превышает максимальное число пользователей, которое может содержать ваш домен, необходимо зарезервировать более высокий процент пропускной способности для репликации, увеличить скорость связи или разделить организацию на региональные домены.  
  
## <a name="dividing-the-organization-into-regional-domains"></a>Разделение Организации на региональные домены

Если не удается разместить всех пользователей в одном домене, необходимо выбрать модель "региональный домен". Разделите свою организацию на регионы так, чтобы иметь смысл в Организации и в существующей сети. Например, вы можете создавать регионы на основе границ континентальной части.  
  
Обратите внимание, что поскольку необходимо создать домен Active Directory для каждого установленного региона, рекомендуется максимально сокращать количество регионов, определенных для AD DS. Хотя в лесу можно включить неограниченное количество доменов, для управляемости рекомендуется, чтобы лес включал не более 10 доменов. Необходимо установить соответствующий баланс между оптимизацией пропускной способности репликации и свести к минимуму сложность администрирования при разделении Организации на региональные домены.  
  
Сначала определите максимальное число пользователей, которое может разместить лес. Это можно сделать на основе самой медленной ссылки в лесу, в рамках которой реплицируются контроллеры домена, и среднего объема пропускной способности, которую нужно выделить для Active Directory репликации. В следующей таблице представлено максимальное рекомендуемое число пользователей, которое может содержать лес. Это зависит от скорости медленного канала и доли пропускной способности, которую необходимо зарезервировать для репликации. Эти сведения относятся к лесам, которые содержат не более 100 000 пользователей и имеют возможность подключения 28,8 кбит/с или выше. Значения в следующей таблице основаны на следующих допущениях.  

- Все контроллеры домена являются серверами глобального каталога.

Если максимальное число пользователей, на которых может размещаться лес, больше числа пользователей, которые необходимо разместить, то для вашего проекта будет работать один лес. Если необходимо разместить больше пользователей, чем указано в описании максимального числа, необходимо увеличить минимальную скорость связи, выделить более высокий процент пропускной способности для AD DS или развернуть дополнительные леса.  

Если вы определили, что один лес будет соответствовать количеству пользователей, которые необходимо разместить, то следующим шагом будет определение максимального количества пользователей, поддерживаемых каждым регионом, в зависимости от самой медленной ссылки, расположенной в этом регионе. Разделите свой лес по регионам, которые имеют смысл. Убедитесь, что ваше решение основано на что-то, что вряд ли изменится. Например, используйте континенты вместо регионов продаж. Эти регионы будут являться основанием вашей структуры домена, если вы определили максимальное число пользователей.  

Определите число пользователей, которые должны размещаться в каждом регионе, а затем убедитесь, что они не превышают максимально допустимое значение в зависимости от скорости канала и пропускной способности, выделенной для AD DS в этом регионе. Оцените каждый предлагаемый регион и определите, меньше ли максимальное число пользователей в каждом регионе, чем рекомендованное максимальное число пользователей, которое может содержать домен. Если вы определили, что в регионе может размещаться требуемое количество пользователей, можно создать домен для этого региона. Если вы определили, что вы не можете разместить большое количество пользователей, рассмотрите возможность разделения структуры на небольшие регионы и повторное вычисление максимального числа пользователей, которые могут размещаться в каждом регионе. Другими альтернативами является выделение дополнительной пропускной способности или увеличение скорости связи.

Меньшее количество пользователей на домен в лесу с несколькими доменами соответствует дополнительным затратам на репликацию, созданным с помощью поддержки глобального каталога в этой среде. ## <a name="documenting-the-regions-identified"></a>Документирование указанных регионов

После разделения Организации на региональные домены необходимо задокументировать нужные регионы и количество пользователей, которые будут существовать в каждом регионе. Кроме того, обратите внимание на скорость самых медленных ссылок в каждом регионе, который будет использоваться для Active Directory репликации. Эти сведения используются для определения необходимости в дополнительных доменах или лесах.  

Чтобы получить помощь в документировании указанных вами регионов, скачайте Job_Aids_Designing_and_Deploying_Directory_and_Security_Services. zip из [комплекта вспомогательных средств для Windows Server 2003 Deployment Kit](https://go.microsoft.com/fwlink/?LinkID=102558) и откройте "идентификационные регионы" (дсслоги _4. doc).
