# <a name="determining-the-number-of-domains-required"></a>Определение требуемого количества доменов

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Запускает каждый лес с одним доменом. Максимальное число пользователей, которые могут содержать один лес основана на медленных ссылки, должны включать репликации между контроллерами домена и доступной пропускной способности, которое нужно выделить в доменных службах Active Directory (AD DS). В следующей таблице перечислены максимальное рекомендуемое число пользователей, домен может включать в зависимости от одного домена леса, скорости медленного подключения и процент пропускной способности, который требуется зарезервировать для репликации. Эти сведения относятся к лесам, содержащие не более 100 000 пользователей и, имеющих подключения 28,8 килобит в секунду (Кбит/с) или выше. Рекомендации, которые применяются к лесах, содержащих более чем 100 000 пользователей или подключения меньше 28,8 Кбит/с обратитесь к опытным конструктора Active Directory. Значения в приведенной ниже таблице основаны на трафик репликации, созданные в среде, которая имеет следующие характеристики:  
  
-   Новые пользователи присоединиться к лесу скорость 20% в год.  
  
-   Пользователи покидают лес со скоростью 15% в год.  
  
-   Каждый пользователь является членом пять глобальные группы и пять универсальные группы.  
  
-   На компьютерах пользователей приходится 1:1.  
  
-   Используется Active Directory интегрированы доменных имен (DNS).  
  
-   Используется DNS-очистку.  
  
> [!NOTE]  
> Рисунки, перечисленные в следующей таблице будут приблизительными. Количество трафика репликации во многом зависит от числа изменений, внесенных в каталог, в определенный период времени. Убедитесь, что сети можно использовать для размещения трафик репликации, тестирования предполагаемое количество и частота изменений на разработку в лаборатории перед развертыванием доменов.  
  
|Медленная ссылку подключение контроллера домена (Кбит/с)|Максимальное число пользователей, если пропускная способность % 1|Максимальное число пользователей, если пропускная способность 5 процентов|Максимальное число пользователей, если пропускная способность 10%|  
|--------------------------------------------------------|----------------------------------------------------------------|----------------------------------------------------------------|-----------------------------------------------------------------|  
|28.8|10,000|25,000|40,000|  
|32|10,000|25,000|50,000|  
|56|10,000|50,000|100,000|  
|64|10,000|50,000|100,000|  
|128|25,000|100,000|100,000|  
|256|50,000|100,000|100,000|  
|512|80,000|100,000|100,000|  
|1,500|100,000|100,000|100,000|  
  
Для использования в этой таблице:  
  
1.  В **медленных ссылку подключение контроллера домена** столбец, найдите значение, соответствующее скорости медленных подключения, через который будет репликации AD DS в вашем домене.  
  
2.  В строке, соответствующий вашей Низкая скорость связи найдите столбец, представляющий процент пропускной способности, которую вы хотите выделить в AD DS. В этом месте значение максимальное число пользователей, которые могут содержать домена в лесу один домен.  
  
Если выяснится, что общее число пользователей в лесу меньше максимальное число пользователей, которые могут содержать вашего домена, можно использовать один домен. Убедитесь, что в соответствии с запланированной дальнейшего развития при внесении этого определения. Если выяснится, что общее число пользователей в лесу превышает максимальное количество пользователей, которые могут содержать вашего домена, необходимо зарезервировать высокий процент пропускной способности для репликации, увеличить скорость связи или разделить вашей организации в региональных доменов.  
  
## <a name="dividing-the-organization-into-regional-domains"></a>Разделение организации в региональных доменов  
Если вы не может вставить всех пользователей в одном домене, необходимо выбрать модель регионального домена. Разделите вашей организации в области способом, имеет смысл для вашей организации и в существующей сети. Например можно создать на основе границ внутриконтинентальная регионах.  
  
Обратите внимание, что необходимо создать домен Active Directory для каждого региона, в который вы устанавливаете, поэтому мы рекомендуем свести к минимуму число областей, которые можно определить для AD DS. Несмотря на то, что можно включить неограниченное количество доменов в лесу, управляемость рекомендуется наличие не более 10 доменов леса. Необходимо установить соответствующий баланс оптимизации пропускной способности репликации и свести к минимуму вашей сложностей при делении вашей организации в региональных доменов.  
  
Во-первых определите максимальное число пользователей, которых могут размещаться в лесу. Это базовый медленных ссылку в лесу между реплицирует контроллеров домена и средний объем пропускной способности, которые нужно выделить для репликации Active Directory. В следующей таблице перечислены максимальное рекомендуемое число пользователей, которые могут содержать леса. Это на основе скорости медленных ссылки, а также процент пропускной способности, который требуется зарезервировать для репликации. Эти сведения относятся к лесам, содержащие не более 100 000 пользователей и который подключен к 28,8 Кбит/с или выше. Значения в приведенной ниже таблице основаны на следующих допущениях.  
  
-   Все контроллеры домена либо серверами глобального каталога.

2.  В строке, соответствующий вашей Низкая скорость связи найдите столбец, представляющий процент пропускной способности, нужно выделить в AD DS. В этом месте значение максимальное число пользователей, которых могут размещаться в лесу.  
  
Если максимальное число пользователей, которых могут размещаться лесу больше, чем число пользователей, которые необходимы для размещения, один лес будет работать для проекта. Если требуется разместить дополнительные пользователей превышает максимальное количество, которые вы определили, необходимо для увеличения скорости минимальное ссылку выделить больше процент пропускной способности для AD DS или развертывание дополнительных лесов.  
  
Если вы обнаружите, что одного леса в нем будут размещаться число пользователей, которые необходимы для размещения, следующим шагом является определение максимальное число пользователей, которые может поддерживать в каждом регионе, основанные на медленных ссылку, расположенную в этом регионе. Разделите своего леса на областей, которые имеют смысл для вас. Убедитесь в том, основанных на принятие решения о том, что-то, не изменится. Например используйте континенты вместо регионы продаж. Эти области будут основой структуру домена, если вы определили максимальное количество пользователей.  
  
Определите число пользователей, которые должны размещаться в каждом регионе, а затем убедитесь, что они не превышают максимальную на основе низкая скорость связи и пропускной способности, выделенного для AD DS в этом регионе. Если вы обнаружите, что область может размещаться число пользователей, которые требуется, можно создать домен для этого региона. Вы определить, не может размещать, многие пользователи считают разделить проекта на более мелкие регионов и пересчета максимальное количество пользователей, которые могут быть размещены в каждом регионе. Меньшее число пользователей для каждого домена в лесу с несколькими доменами согласует издержек дополнительной репликации, поддерживая глобального каталога в этой среде. Кроме того Обратите внимание, скорость медленных ссылки в каждом регионе, который будет использоваться для репликации Active Directory. Эта информация используется для определения того, если требуются дополнительные домены или леса.
