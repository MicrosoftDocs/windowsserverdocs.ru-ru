# <a name="identifying-your-functional-level-upgrade"></a>Определение обновления режимов работы

>Область применения. Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Прежде чем можно повысить функциональные уровни домена и леса, в наличии для оценки текущей среды и определение функциональных требований к режиму, который лучше всего соответствует потребностям вашей организации. Оценить текущую среду с определения доменов в лесу, контроллеры домена, которые находятся в каждый домен, операционной системы и пакеты обновления, которые каждый контроллер домена работает и даты, который планируется обновить домен контроллеры. Если планируется снять с учета контроллером домена, убедитесь, что вам полностью понять, это окажет на среду.  
  
Следующих обстоятельствах может препятствовать обновлению более ранней версии операционной системы Windows Server до функционального уровня Windows Server 2008 или Windows Server 2008 R2:  
  
-   Недостаток аппаратных средств  
  
-   Контроллер домена под управлением антивирусной программы, несовместим с Windows Server 2008 или Windows Server 2008 R2   
  
-   Использование конкретной версии программы, которая работает в Windows Server 2008 или Windows Server 2008 R2   
  
-   Необходимость обновления программы с последним пакетом  
  
Документирование эта информация может помочь выявить действия необходимо предпринять для убедитесь, что в полнофункциональной среде Windows Server 2008 или Windows Server 2008 R2.  
  
После оценки текущей среды необходимо определение обновления режимов работы, которая применяется к вашей организации. Эти параметры доступны:  
  
-   Среды с собственным режимом Windows 2000 на Windows Server 2008 или Windows Server 2008 R2   
  
-   Лес Windows Server 2003, Windows Server 2008 или Windows Server 2008 R2   
  
-   Новый лес Windows Server 2008  
  
-   Новый лес Windows Server 2008 R2  
  
## <a name="upgrading-functional-levels-in-a-native-windows-2000-active-directory-forest"></a>Обновление режимы работы в собственном лесу Windows 2000 Active Directory  
В изначальной среде Windows 2000, состоящий только из контроллеров домена под управлением Windows 2000 по умолчанию для следующих уровней заданы режимы работы, и они остаются на этих уровнях, пока не вызвать их вручную:  
  
-   Windows 2000 собственный режим работы домена  
  
-   Режим работы леса Windows 2000  
  
Чтобы использовать все возможности уровня домена и леса в Windows Server 2008 или Windows Server 2008 R2, необходимо обновить этой среде Windows 2000 до Windows Server 2008 или Windows Server 2008 R2. Это обновление можно выполнить одним из следующих способов:  
  
-   Внедряют установленным Windows Server 2008 — на основе или Windows Server 2008 R2-на основе контроллеров домена в лесу, а затем выполните списание все контроллеры домена под управлением Windows 2000.  
  
-   Выполните обновление на месте всех контроллеров домена под управлением Windows 2000 в лесу, к контроллерам домена под управлением Windows Server 2003. Затем выполните обновление на месте этих контроллеров домена до Windows Server 2008 или Windows Server 2008 R2. Дополнительные сведения см. в разделе [обновления доменов Active Directory до доменов доменных Служб Windows Server 2008 \[LH\]](assetId:///9c91be5f-df14-40b2-b176-2b1852a51e61).  
  
    > [!IMPORTANT]  
    >  Windows Server 2008 R2 — это x64 операционная система. Если сервер работает под управлением x64-разрядной версией Windows Server 2003, можно успешно выполнить обновление на месте операционной системы на этом компьютере до Windows Server 2008 R2. Если сервер работает под управлением x86 версию Windows Server 2003, нельзя обновить этот компьютер до Windows Server 2008 R2.  
  
Чтобы использовать функции уровня домена Windows Server 2008 или Windows Server 2008 R2 без обновления всей леса Windows 2000 до Windows Server 2008 или Windows Server 2008 R2, вызывать только режим работы домена до Windows Server 2008 или Windows Server 2008 R2.  
  
> [!NOTE]  
> Прежде чем повысить режим работы домена, необходимо обновить все контроллеры домена под управлением Windows 2000 в домене Windows Server 2008 или Windows Server 2008 R2.  
  
После замены всех контроллеров домена под управлением Windows 2000 в лес контроллеры домена под управлением Windows Server 2008 или Windows Server 2008 R2, можно повысить режим работы леса до Windows Server 2008 или Windows Server 2008 R2. Это автоматически вызывает режим работы всех доменов в лесу, для которых установлен Windows 2000 или выше для Windows Server 2008 или Windows Server 2008 R2.  
  
Дополнительные сведения о вызове функциональные уровни леса и домена и процедур для выполнения этих задач, см. в разделе [развертывание корневого домена леса Windows Server 2008 \[LH\]](assetId:///92406e8d-dc1c-4740-a00a-2c4032896dd1).
