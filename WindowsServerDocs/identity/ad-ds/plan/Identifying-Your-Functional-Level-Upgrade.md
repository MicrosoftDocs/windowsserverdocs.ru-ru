# <a name="identifying-your-functional-level-upgrade"></a>Определение обновления режимов работы

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Чтобы может повысить функциональные уровни домена и леса, необходимо выполнить оценку вашей текущей среды и определить функционального уровня требование, что наилучшим образом соответствует потребностям вашей организации. Оцените текущую среду путем определения доменов в лесу, контроллеры домена, которые находятся в каждом домене, операционной системы и пакетов обновления, каждый контроллер домена работает под управлением и даты, который вы планируете обновление контроллеров домена. Если вы планируете учета контроллера домена, убедитесь, что вы полностью понять влияние, это будет предоставлен в вашей среде.  
  
Следующих условий, которые помогут предотвратить обновления более ранней версии операционной системы Windows Server до Windows Server 2008 или Windows Server 2008 R2 режим работы:  
  
-   Недостаточно оборудования  
  
-   Контроллер домена работает антивирусная программа, не совместим с Windows Server 2008 или Windows Server 2008 R2   
  
-   Использование конкретных версий программа, которая работает на Windows Server 2008 или Windows Server 2008 R2   
  
-   Необходимо просто обновить программу с последним пакетом обновления  
  
Документирование эти сведения помогут вам понять шаги необходимо выполнить, чтобы обеспечить полнофункциональные среды Windows Server 2008 или Windows Server 2008 R2.  
  
После оценки текущей среды, необходимо определить обновления режимов работы, которая применяется к вашей организации. Доступны следующие параметры:  
  
-   Среды основном режиме Windows 2000, Windows Server 2008 или Windows Server 2008 R2   
  
-   Леса Windows Server 2003, Windows Server 2008 или Windows Server 2008 R2   
  
-   Новый лес Windows Server 2008  
  
-   Новый лес Windows Server 2008 R2  
  
## <a name="upgrading-functional-levels-in-a-native-windows-2000-active-directory-forest"></a>Обновление режимы работы в основном лесу Windows 2000 Active Directory  
В среде собственный Windows 2000, которая включает в себя только контроллеры домена под управлением Windows 2000 по умолчанию для следующих уровней заданы режимы работы и остаются на этих уровнях, пока вы повышаете вручную:  
  
-   Windows 2000 основного режима работы домена  
  
-   Режим работы леса Windows 2000  
  
Для использования всех возможностей уровня домена и леса в Windows Server 2008 или Windows Server 2008 R2, необходимо выполнить обновление до Windows Server 2008 или Windows Server 2008 R2 эта среда Windows 2000. Это обновление можно выполнить одно из следующих действий:  
  
-   Вносит только что установленной Windows Server 2008-на основе или Windows Server 2008 R2 — на основе контроллеров домена в лесу, а затем выполните списание все контроллеры домена под управлением Windows 2000.  
  
-   Выполните обновление на месте все существующие контроллеры домена под управлением Windows 2000 в лес контроллеры домена под управлением Windows Server 2003. Затем выполните обновление на месте контроллеры домена Windows Server 2008 или Windows Server 2008 R2. Дополнительные сведения см. в разделе [обновления доменов Active Directory до доменов Windows Server 2008 AD DS \[LH\]](assetId:///9c91be5f-df14-40b2-b176-2b1852a51e61).  
  
    > [!IMPORTANT]  
    >  Windows Server 2008 R2 является x64-разрядной операционной системы. Если сервер работает под управлением x64-разрядной операционной системой Windows Server 2003, можно успешно выполнить обновление на месте операционной системы компьютера до Windows Server 2008 R2. Если сервер работает под управлением более x86-разрядной версии Windows Server 2003, нельзя обновить этот компьютер до Windows Server 2008 R2.  
  
Чтобы использовать возможности уровня домена Windows Server 2008 или Windows Server 2008 R2 без обновления всего леса Windows 2000 до Windows Server 2008 или Windows Server 2008 R2, повышайте только режим работы домена Windows Server 2008 или Windows Server 2008 R2.  
  
> [!NOTE]  
> Прежде чем повысить режим работы домена, необходимо обновить все контроллеры домена под управлением Windows 2000 в домене Windows Server 2008 или Windows Server 2008 R2.  
  
После замены все контроллеры домена под управлением Windows 2000 в лесу с контроллерами домена под управлением Windows Server 2008 или Windows Server 2008 R2, можно повысить режим работы леса до Windows Server 2008 или Windows Server 2008 R2. Поэтому автоматически выполнив поднимает функциональный уровень всех доменов в лесу, настроенные для Windows 2000 в основном или более поздней версии Windows Server 2008 или Windows Server 2008 R2.  
  
Дополнительные сведения о повышении режимов работы леса и домена и процедуры для выполнения этих задач см. в разделе [развертывание Windows Server 2008 леса корневого домена \[LH\]](assetId:///92406e8d-dc1c-4740-a00a-2c4032896dd1).  
  
## <a name="upgrading-functional-levels-in-a-windows-server-2003-active-directory-forest"></a>Обновление режимы работы леса Windows Server 2003 Active Directory  
В среде Windows Server 2003, которая включает в себя только контроллеры домена под управлением Windows Server 2003 по умолчанию для следующих уровней заданы режимы работы и остаются на этих уровнях, пока вы повышаете вручную:  
  
-   Windows 2000 основного режима работы домена

Для использования всех возможностей уровня домена и леса в Windows Server 2008 или Windows Server 2008 R2, необходимо выполнить обновление до Windows Server 2008 или Windows Server 2008 R2 эта среда Windows Server 2003. Это обновление можно выполнить одно из следующих действий:  
  
-   Внедрения только что установленной Windows Server 2008 — на основе или Windows Server 2008 R2 — на базе контроллера домена в лесу и снять с учета все контроллеры домена под управлением Windows Server 2003 или обновить до Windows Server 2008 или Windows Server 2008 R2.  
  
-   Выполните обновление на месте все существующие контроллеры домена под управлением Windows Server 2003 контроллеры домена под управлением Windows Server 2008 или Windows Server 2008 R2. Дополнительные сведения см. в разделе [обновления доменов Active Directory до доменов Windows Server 2008 AD DS \[LH\]](assetId:///9c91be5f-df14-40b2-b176-2b1852a51e61).  
  
> [!IMPORTANT]

Если сервер работает под управлением x64-разрядной операционной системой Windows Server 2003, можно успешно выполнить обновление на месте операционной системы компьютера до Windows Server 2008 R2. Если сервер работает под управлением более x86-разрядной версии Windows Server 2003, не может обновить этот компьютер для запуска Windows Server 2008 R2.  
  
Чтобы использовать возможности уровня домена Windows Server 2008 или Windows Server 2008 R2 без обновления всего леса Windows Server 2003 до Windows Server 2008 или Windows Server 2008 R2, поднимите только режим работы домена Windows Server 2008 или Windows Server 2008 R2.  
  
> [!NOTE]  
> Прежде чем повысить режим работы домена, необходимо обновить все контроллеры домена под управлением Windows Server 2003, в том же домене Windows Server 2008 или Windows Server 2008 R2.  
  
После обновления до Windows Server 2008 или Windows Server 2008 R2 контроллеры домена под управлением Windows Server 2003 в лесу, можно повысить режим работы леса до Windows Server 2008 или Windows Server 2008 R2. Поэтому автоматически выполнив поднимает функциональный уровень всех доменов в лесу, настроенные для Windows Server 2003 до Windows Server 2008 или Windows Server 2008 R2.  
  
Дополнительные сведения о повышении режимов работы леса и домена и процедуры для выполнения этих задач см. в разделе [развертывание Windows Server 2008 леса корневого домена \[LH\]](assetId:///92406e8d-dc1c-4740-a00a-2c4032896dd1).  
  
## <a name="upgrading-functional-levels-in-a-new-windows-server-2008-forest"></a>Обновление режимы работы в новом лесу Windows Server 2008  
При установке первого контроллера домена в новом лесу Windows Server 2008, заданы режимы работы по умолчанию для следующих уровней и остаются на этих уровнях, пока вы повышаете вручную:  
  
-   Windows 2000 основного режима работы домена  
  
-   Режим работы леса Windows 2000  
  
Чтобы дать вам возможность добавления контроллеров домена на базе Windows Server 2003 или Windows 2000 в новый лес Windows Server 2008 на этих уровнях по умолчанию заданы режимы работы. После создания корневого домена леса, режим работы домена для каждого домена, добавляемые в лес Windows Server 2008 имеет значение Windows 2000 собственный. Тем не менее если требуется, чтобы все контроллеры домена в среде Windows Server 2008 для запуска Windows Server 2008, задайте режим работы леса и режим работы домена до Windows Server 2008 при установке первого контроллера домена в лесу. Это экономит время и включает все возможности уровня домена и леса Windows Server 2008.  
  
> [!IMPORTANT]  
> Если лес действует на режим работы Windows Server 2008 уровня и вы попытаетесь установить Active Directory на сервере под управлением Windows Server 2003 или под управлением Windows 2000 рядового сервера, установка завершается сбоем.  
  
Дополнительные сведения о повышении режимов работы леса и домена и процедуры для выполнения этих задач см. в разделе [развертывание Windows Server 2008 леса корневого домена \[LH\]](assetId:///92406e8d-dc1c-4740-a00a-2c4032896dd1).  
  
## <a name="upgrading-functional-levels-in-a-new-windows-server-2008-r2-forest"></a>Обновление режимы работы в новом лесу Windows Server 2008 R2  
При установке первого контроллера домена в новом лесу Windows Server 2008 R2 по умолчанию для следующих уровней заданы режимы работы и остаются на этих уровнях, пока вы повышаете вручную:  
  
-   Режим работы домена Windows Server 2003  
  
-   Режим работы леса Windows Server 2003  
  
На этих уровнях по умолчанию заданы режимы работы, чтобы дать вам возможность добавления контроллерах домена под управлением Windows Server 2003 в новый лес Windows Server 2008 R2. После создания корневого домена леса Windows Server 2003 значение режима работы домена для каждого домена, добавляемые в лес Windows Server 2008 R2. Тем не менее если требуется, чтобы все контроллеры домена в среде Windows Server 2008 R2 для запуска Windows Server 2008 R2, значение режима работы леса и режим работы домена Windows Server 2008 R2 при установке первого контроллера домена в лесу. Это экономит время и включает все возможности уровня домена и леса, в Windows Server 2008 R2.  
  
> [!IMPORTANT]  
> Если лес действует на режим работы Windows Server 2008 R2, и при попытке установки Active Directory в Windows Server 2008 — на основе под управлением Windows Server 2003 или рядовом сервере или на сервере под управлением Windows 2000, происходит сбой установки.  
  
Дополнительные сведения о повышении режимов работы леса и домена и процедуры для выполнения этих задач см. в разделе [развертывание Windows Server 2008 леса корневого домена \[LH\]](assetId:///92406e8d-dc1c-4740-a00a-2c4032896dd1).  
  
> [!NOTE]  
> Несмотря на то, что ADMT v3.1 должен быть установлен на Windows Server 2008, можно использовать для переноса объектов домена, который размещается в один или несколько контроллеров домена Windows Server 2008 R2 ADMT v3.1. Дополнительные сведения см. в разделе [статья 976659](https://go.microsoft.com/fwlink/?LinkId=180398) в базе знаний Майкрософт (https://go.microsoft.com/fwlink/?LinkId=180398).
