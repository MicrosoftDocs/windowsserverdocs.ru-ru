# <a name="appendix-l-events-to-monitor"></a>Приложение м: События для мониторинга

>Область применения. Windows Server

В следующей таблице перечислены события, которые необходимо отслеживать в вашей среде, в соответствии с рекомендациям, приведенным в [мониторинг Active Directory для знаки нарушения](../../ad-ds/plan/security-best-practices/Monitoring-Active-Directory-for-Signs-of-Compromise.md). В следующей таблице в столбце «Текущий код события Windows» перечислены идентификатор события, так как она реализована в версиях Windows и Windows Server, находящиеся в основной фазе поддержки.  
  
В столбце «Код события предыдущих версий Windows» перечислены идентификатор соответствующего события в устаревших версий Windows, такие как клиентские компьютеры под управлением Windows XP или более ранней версии и серверы под управлением Windows Server 2003 или более ранней версии. «Потенциальный уровень важности», столбец указывает, следует ли рассматривать как события низкий, средний или высокий уровень уровнем важности при обнаружении атак, а столбец «Сводка событий» содержит краткое описание события.  
  
Высокий потенциальный уровень важности означает, что одно возникновение события следует изучить. Потенциальный уровень важности, средний или низкий означает, что эти события следует изучить, только если они встречаются неожиданно или номера, значительно превышает ожидаемое базовое значение за измеряемый период времени. Всем организациям следует проверить эти рекомендации в своих средах перед созданием оповещений, которые требуют обязательного расследования. Каждая среда имеет свои, и некоторые события с потенциально высоким уровнем важности может возникать из-за других безвредных событий.
