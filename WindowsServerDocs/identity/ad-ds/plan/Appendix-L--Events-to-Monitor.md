# <a name="appendix-l-events-to-monitor"></a>Приложение м. события для мониторинга

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

  
## <a name="appendix-l-events-to-monitor"></a>Приложение м. события для мониторинга  
В следующей таблице перечислены события, которые необходимо контролировать в вашей среде, в соответствии с рекомендации, приведенные в [наблюдения за Active Directory для признаков компрометации](../../ad-ds/plan/security-best-practices/Monitoring-Active-Directory-for-Signs-of-Compromise.md). В следующей таблице столбце «Текущий код события Windows» перечислены КОДЫ событий, реализованные в версиях Windows и Windows Server, которые в настоящее время основной фазы поддержки.  
  
В столбце «Устаревший код события Windows» перечислены соответствующий код события в предыдущих версий Windows, таких как клиентские компьютеры под управлением Windows XP или более ранних версий и серверов под управлением Windows Server 2003 или более ранней версии. «Потенциальный уровень важности», столбец определяет события следует ли рассматривать низкие, средним или высоким уровнем важности при обнаружении атаки, а в столбце «Сводка событий» содержит краткое описание события.  
  
Высокий потенциальный уровень важности означает, что одно возникновение события должны быть изучены. Потенциальный уровень важности средний или низкий означает, что эти события следует изучить, только если они происходят неожиданно или число значительно превышает ожидаемое базовое значение за измеряемый период времени. Всем организациям следует проверить эти рекомендации в своих средах перед созданием оповещений, которые требуют обязательного расследования. Все среды отличается, а некоторые события с высокий потенциальный уровень важности могут возникнуть из-за других безвредных событий.
