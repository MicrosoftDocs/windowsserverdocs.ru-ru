# <a name="attractive-accounts-for-credential-theft"></a>Хороший учетные записи для кражи учетных данных

>Область применения: Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Атак кражи учетных данных являются те, в которых злоумышленник получает изначально высоким уровнем привилегий (корень, администратор или системы, в зависимости от операционной системы используется) доступ к компьютеру в сети, а затем использует бесплатно получить набор средств для извлечения учетные данные из сеансы другим учетным записям вошедшего в систему. В зависимости от конфигурации системы эти учетные данные могут быть извлечены в виде хэш-коды, билеты или даже обычные текстовые пароли. Если какие-либо полученного учетных данных для локальных учетных записей, которые, скорее всего, существует на других компьютерах в сети (например, учетных записей администраторов в Windows или корневой учетных записей в OSX, ОС UNIX или Linux), злоумышленник представляется учетные данные на других компьютерах в сети для распространения компрометации на дополнительных компьютерах и попытаться получить учетные данные два определенных типов учетных записей :  

1.  Привилегированных учетных записей домена с привилегиями обширных и глубоких (то есть учетные записи, имеющие прав администратора на нескольких компьютерах и в Active Directory). Эти учетные записи не могут быть членами группы высоким уровнем прав в Active Directory, но они могут предоставлены права администратора между множеством серверов и рабочих станций в домене или лесу, что делает их эффективно мощные, как членами привилегированных групп в Active Directory. В большинстве случаев учетные записи, которые были предоставлены высокий уровень привилегий через широкая объема инфраструктуры Windows, учетные записи служб, поэтому учетные записи служб должны всегда оцениваться Широта и уровень привилегий.  

2.  «Очень важно лицо» учетные записи домена (VIP). В контексте этого документа учетной записи виртуальных IP-адресов является любой учетной записи, которая имеет доступ к информации, злоумышленник желает (интеллектуальной собственности и другие конфиденциальные сведения) или любая учетная запись, можно использовать для предоставления злоумышленнику получить доступ к этой информации. Ниже приведены примеры этих учетных записей пользователей:  

    1.  Руководители, чьи учетные записи имеют доступ к конфиденциальной корпоративной информации  

    2.  Учетные записи для персонала службы технической поддержки, который несете ответственность за поддержание компьютеров и приложений, используемых руководителей  

    3.  Учетные записи юридического отдела, имеющих доступ к документов заявка и контракт организации, являются ли документы для своих собственных организации или клиента организации  

    4.  Конвейер разработчики идей, имеющих доступ к планы и спецификаций продуктов в разработке компании, независимо от типы продуктов не делает компании  

    5.  Исследователи, чьи учетные записи используются для доступа к пример данных, формулировки продукта или другие справочные материалы интерес для злоумышленников  

Поскольку привилегированных учетных записей в Active Directory можно использовать для компрометации распространения и для управления учетными записями виртуальных IP-адресов или данные, которые они могут получить доступ, наиболее полезна для атак кражи учетных данных представляют учетные записи, являющиеся членами группы "Администраторы предприятия", "Администраторы домена" и "Администраторы" в Active Directory.  

Поскольку контроллеры домена работают репозитории для базы данных AD DS и контроллеры домена имеют полный доступ ко всем данным в Active Directory, контроллеров домена также предназначены для нарушения безопасности ли параллельно атак кражи учетных данных или после один или несколько привилегированных Active Directory был скомпрометирован учетных записей. Несмотря на то, что многочисленные публикации (и многие злоумышленники) сосредоточиться на членства в группах "Администраторы домена" при описании кражи атак pass--hash и других учетных данных (как описано в [снижение уязвимостей Active Directory](../../../ad-ds/plan/security-best-practices/Reducing-the-Active-Directory-Attack-Surface.md)), злоумышленнику весь процесс установки AD DS можно использовать учетную запись, которая является членом любой из групп, перечисленных здесь.

## <a name="activities-that-increase-the-likelihood-of-compromise"></a>Действия, которые повышают вероятность взлома  
Так как целью кражи учетных данных обычно является высокой привилегированных учетных записей домена и учетные записи виртуальных IP-адресов, важно администраторам необходимо осознавать действий, которые повышают вероятность успешных атак кражи учетных данных. Несмотря на то, что злоумышленники делают своей целью также ВАЖНЫХ учетных записей, если виртуальные IP-адреса, не получают высокий уровень привилегий в системах или в домене, в случае кражи учетных данных требуется других типов атак, например социальных проектирования виртуальных IP-адресов для предоставления секретных сведений. Или злоумышленнику необходимо получить привилегированный доступ к системе, на каких виртуальных IP-адресов кэшированные учетные данные. По этой причине действий, которые повышают вероятность кражи учетных данных, описанных здесь сосредоточены главным образом на предотвращение приобретения привилегированных учетных данных администратора. Эти действия — стандартных механизма, с помощью которых злоумышленники могут для нарушения безопасности системы, чтобы получить привилегированных учетных данных.  

### <a name="logging-on-to-unsecured-computers-with-privileged-accounts"></a>Вход в систему к незащищенным компьютерам с привилегированных учетных записей  
Уязвимости основных компонентов, что позволяет атак кражи учетных данных — это вход на компьютеры, которые не являются безопасным, используя учетные записи, которые широко и глубоко привилегированных во всей среде. Эти вход в систему может быть результатом различных неверные настройки, описанные ниже.  

#### <a name="not-maintaining-separate-administrative-credentials"></a>Без поддержки отдельные учетные данные администратора  
Несмотря на то, что это довольно редко в оценки различных установок AD DS, мы выяснили, что ИТ-сотрудников с помощью одной учетной записи для всех свою работу. Учетная запись является членом по крайней мере одной из наиболее привилегированных групп в Active Directory и находится в той же учетной записью, используйте сотрудники для входа на свои рабочие станции утром, ознакомьтесь со своей электронной почты, просмотр веб-сайтов и загружать содержимое на своих компьютерах. Когда пользователи работают с учетными записями, которым предоставляются разрешения и права локального администратора, они предоставляют локального компьютера, чтобы завершить компрометации. При этих учетных записей также являются членами наиболее привилегированных групп в Active Directory, они предоставляют всего леса, чтобы взломать, облегчая просто злоумышленник может получить полный контроль над средой Active Directory и Windows.  

Аналогично в некоторых средах, мы обнаружили, что же имена пользователей и пароли используются для корневого учетных записей на компьютерах, отличных от Windows как используемые в среде Windows, что позволяет злоумышленникам расширить компрометации из систем UNIX или Linux, в системах Windows и наоборот.

#### <a name="logons-to-compromised-workstations-or-member-servers-with-privileged-accounts"></a>Вход в систему для скомпрометированного рабочих станциях или рядовых серверах с привилегированных учетных записей  
При использовании учетной записи домена с высоким уровнем привилегий для интерактивного входа в скомпрометированных рабочей станции или рядового сервера скомпрометированных компьютера может получить учетные данные из любой учетной записи, на которые входит в систему.  

#### <a name="unsecured-administrative-workstations"></a>Небезопасную рабочих станций администрирования.  
Во многих организациях ИТ-специалистам использовать несколько учетных записей. Одна учетная запись используется для входа в систему рабочей станции сотрудника, а потому, что их ИТ-специалисты, они часто иметь права локального администратора на их рабочих станциях. В некоторых случаях контроля учетных Записей — слева, включен, чтобы пользователь получает по крайней мере доступа split маркер при входе в систему и необходимо повысить уровень необходимы права. Когда эти пользователи выполняются действия по обслуживанию, они обычно с помощью средств управления, установленных локально и введите учетные данные для своих учетных записей домена уровнем привилегий, выбрав **Запуск от имени администратора** вариант или путем предоставления учетных данных, при появлении соответствующего запроса. Несмотря на то, что эта конфигурация может не подходят, он предоставляет среду для нарушения безопасности, так как:  

-   «Обычный» учетная запись, сотрудник использует для входа своей рабочей станции имеет права локального администратора, компьютер уязвимым для [скрытой загрузки](https://www.microsoft.com/security/sir/glossary/drive-by-download-sites.aspx) атак, в которых пользователь является убедились в том, чтобы установить вредоносные программы.  

-   Вредоносные программы устанавливается в контексте учетной записи администратора, компьютер теперь можно использовать для записи нажатий клавиш, содержимое буфера обмена, снимки экрана и учетные данные в памяти, которые может привести к раскрытию учетные данные учетной записи домена мощные.  

Проблемы в этом сценарии есть две причины. Во-первых несмотря на то, что отдельные учетные записи используются для администрирования локальных и домена, компьютер незащищенных и не обеспечивает защиту от хищения учетных записей. Во-вторых предоставлены учетной записью пользователя и учетная запись администратора чрезмерного права и разрешения.  

### <a name="browsing-the-internet-with-a-highly-privileged-account"></a>В Интернете с высоким уровнем привилегий учетной записи  
Пользователей, войдите на компьютер с учетными записями, которые являются членами локальной группы администраторов на компьютере или членов привилегированных групп в Active Directory и, и просмотрите Интернета (или скомпрометированных интрасети) предоставляют локального компьютера и каталога для нарушения безопасности.  

Доступ к специальный веб-сайт в браузере, с правами администратора может дать злоумышленнику перевести вредоносного кода на локальном компьютере в контексте привилегированных пользователей. Если у пользователя есть права локального администратора на компьютере, злоумышленники могут задачу обмана пользователя в загрузки вредоносного кода или открытие вложения электронной почты, которые используют уязвимостей приложений и использовать прав пользователя, чтобы извлечь локально кэшированных учетных данных для всех активных пользователей на компьютере. Если у пользователя есть права администратора в каталоге, членство в группе "Администраторы предприятия", "Администраторы домена" или группы администраторов в Active Directory, злоумышленник можно извлечь учетные данные домена и использовать их для компрометации всего домена AD DS или в лесу, без необходимости взлома других компьютера в лесу.  

### <a name="configuring-local-privileged-accounts-with-the-same-credentials-across-systems"></a>Настройка локальных привилегированных учетных записей с теми же учетными данными между системами  
Настройка именем учетной записи локального администратора и пароль на многих или всех компьютеров включает учетные данные кражи из базы данных SAM на одном компьютере, который будет использоваться для компрометации всех остальных компьютерах, использующих те же учетные данные. Как минимум следует использовать разные пароли для локальных учетных записей администратора в каждой системе присоединенных к домену. Учетные записи локального администратора может также быть уникальность имен, но использовать разные пароли для привилегированных локальных учетных записей каждой системы достаточно, чтобы убедиться, что учетные данные нельзя использовать в других системах.  

### <a name="overpopulation-and-overuse-of-privileged-domain-groups"></a>Overpopulation и чрезмерного использования групп привилегированных домена  
Предоставление членства в группах EA, DA или BA домена создает целевой объект для злоумышленников. Чем больше число членам этих групп, тем выше вероятность привилегированного пользователя может случайно без авторизации использовать учетные данные и предоставлять их для кражи учетных данных атак. Каждой рабочей станции или сервере, к которому привилегированных домен входа в систему представляет возможные механизм, с помощью которого привилегированные учетные данные пользователя могут собираемого и использоваться для нарушения безопасности в Доменных службах Active Directory домена и леса.  

### <a name="poorly-secured-domain-controllers"></a>Контроллеры домена плохо защищенном  
Контроллеры домена размещения реплики базы данных домена AD DS. В случае контроллеров домена только для чтения локальной реплики базы данных содержит учетные данные только для подмножества учетные записи в каталоге нет привилегированных учетных записей домена по умолчанию. На контроллерах домена для чтения и записи каждый контроллер домена поддерживает полной реплики базы данных AD DS, включая учетные данные не только для привилегированных пользователей, таких как Администраторы домена, но привилегированных учетных записей, например учетных записей контроллера домена или учетной записи Krbtgt домена, является учетной записью, которая связана с KDC службы на контроллерах домена. Если установлены дополнительные приложения, которые не являются необходимыми для работы контроллера домена, на контроллерах домена, если контроллеры домена не строго установленным, а защищенный, злоумышленники могут скомпрометировать их через незакрытых уязвимостей, или они могут использовать другие атак, для установки вредоносных программ непосредственно на них.  

## <a name="privilege-elevation-and-propagation"></a>Повышение уровня прав доступа и распространение  
Независимо от методов атак, используемых Active Directory всегда предназначен при атаке среде Windows, так как в конечном счете элементы управления доступом с для того, что требуется злоумышленники. Это не означает целевых всего каталога, однако. Определенные учетные записи, серверы и компоненты инфраструктуры обычно являются основных целей атак Active Directory. Эти учетные записи как описано ниже.  

### <a name="permanent-privileged-accounts"></a>Постоянные привилегированных учетных записей  
Так как введение Active Directory, можно было использовать высокий уровень привилегий учетные записи для создания леса Active Directory, а затем к делегировать права и разрешения, необходимые для выполнения повседневных администрирования меньше привилегий учетным записям. Членство в группе "Администраторы предприятия", "Администраторы домена" или "Администраторы" в Active Directory не требуется только временно и редко в среде, в которой реализован подход наименьшим количеством прав повседневные задачи администрирования.  

Постоянные привилегированные учетные записи являются учетные записи, которые были помещены в привилегированных групп и оставить в нем за каждый день. Если организации размещаются пять учетных записей в группе администраторов домена в домене, эти пять учетных записей можно целевых 24 часов дня, 7 дней в неделю. Однако фактические потребность в использовании учетных записей с правами администраторов домена обычно является только для определенной конфигурации на уровне домена, а также на короткое время.  

### <a name="vip-accounts"></a>Учетные записи виртуальных IP-адресов  
Часто упускают из виду такой конечный объект в Active Directory нарушений — учетные записи «очень важно лиц» (или виртуальные IP-адреса) в организации. Так как эти учетные записи можно предоставить доступ для злоумышленников, что позволяет решать компрометации или даже уничтожение целевых систем, как описано ранее в этом разделе, предназначены привилегированных учетных записей.  

### <a name="privilege-attached-active-directory-accounts"></a>Учетные записи Active Directory «Подключенные привилегий»  
«Подключенные привилегий» учетные записи Active Directory являются учетные записи домена, которые не были внесены членов группы, к которым имеют более высокий уровень привилегий в Active Directory, но вместо предоставлены высокий уровень привилегий на несколько серверов и рабочих станций в среде. Эти учетные записи являются большинство часто на основе домена учетных записей, настроенных для запуска служб в системах, присоединенных к домену, как правило, для приложений, работающих на больших разделах инфраструктуры. Несмотря на то, что эти учетные записи привилегии нет в Active Directory, если им предоставлены высоким уровнем привилегий, по большому количеству систем, их можно использовать для компрометации или даже уничтожение больших сегментов инфраструктуры, добиться того же эффекта, что компрометация привилегированной учетной записью Active Directory.
