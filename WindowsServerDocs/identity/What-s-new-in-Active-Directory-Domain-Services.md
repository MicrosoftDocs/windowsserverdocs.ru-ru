## <a name="BKMK_PAM"></a>Управление привилегированным доступом  
Привилегированным доступом управления (PAM) помогает свести безопасности проблемы для сред Active Directory, которые могут быть вызваны credential кражу такие pass--hash, целевой фишинг и похожих типов атак. Он предоставляет новое решение административного доступа, настроенной с помощью Microsoft Identity Manager (MIM). PAM предоставляет:  
  
-   Нового леса-бастиона Active Directory, что подготовлены MIM. Лесу-бастионе имеет специальные доверия PAM в существующем лесу. Он предоставляет новой среды Active Directory, которая свободного любой действий злоумышленников и изоляции из существующего леса для использования привилегированных учетных записей.  
  
-   Новые процессы в MIM для запроса прав администратора, а также новые рабочие процессы на утверждение запросов на основе.  
  
-   Новые тени субъекты безопасности (группы), подготовленных в лесу-бастионе службой MIM в ответ на запрос прав администратора. Субъекты безопасности тени имеют атрибут, который ссылается идентификатор безопасности административной группы в существующем лесу. Это позволяет теневую группу, для доступа к ресурсам в существующем лесу без изменения все списки управления доступом (ACL).  
  
-   Срок действия ссылки функцию, которая позволяет ограничивать их по времени членство в группе тени. Пользователь может добавить в группу для просто недостаточно времени, необходимые для выполнения административных задач. Ограниченные по времени членство выражается значение срока жизни (TTL), распространяется на время жизни билета Kerberos.  
  
    > [!NOTE]  
    > Срок действия ссылки доступны на всех связанных атрибутов. Но связанного атрибута члена или memberOf отношения между группу и пользователь-только пример, где полное решение, например PAM заранее настроенный для использования функции срок действия ссылки.  
  
-   Усовершенствования KDC встроены в контроллеры домена Active Directory, чтобы ограничить время жизни билета Kerberos минимальное возможное срока жизни (TTL) значение в случаях, когда пользователь имеет несколько ограниченные по времени членство в административных групп. Например при добавлении группе ограниченные по времени a, то при входе в систему срок жизни билета предоставления билета (TGT) Kerberos будет равно времени у вас есть оставшееся группы A. Если вы также является членом другого ограниченные по времени группы B, который имеет меньшее значение TTL, чем группы A, затем время жизни TGT равно времени у вас осталось в группе б.  
  
-   Новые возможности мониторинга помогут вам легко определить, кто запрошен доступ, какие доступа и какие действия были выполнены.  
  
**Требования к**  
  
-   Microsoft Identity Manager  
  
-   Active Directory режим работы леса Windows Server 2012 R2 или более поздней.  
  
## <a name="BKMK_AzureADJoin"></a>Присоединение к Azure AD  
Azure Active Directory присоединиться к улучшает взаимодействие удостоверений для предприятия, организаций и EDU клиентов - с улучшенные возможности для корпоративных и персональных устройств.  
  
Преимущества:  
  
-   **Доступность современные параметры** на принадлежащих компании устройствах с Windows. Службы кислород больше не требуют личную учетную запись Майкрософт: они выполняются отключение существующих рабочих учетных записей пользователей для обеспечения соответствия. Службы кислород будут работать на ПК, которые присоединены к домену Windows локальной и компьютеров и устройств, «присоединенных» к вашему клиенту Azure AD («домен облака»). Эти параметры включают:  
  
    -   Роуминг или персонализации, параметры специальных возможностей и учетные данные  
  
    -   Резервное копирование и восстановление  
  
    -   Доступ к магазину Майкрософт с рабочей учетной записи  
  
    -   Живые плитки и уведомления  
  
-   **Доступ к ресурсам организации** на мобильных устройств (телефонов, фаблетов), которые нельзя присоединить к домену Windows, являются ли они принадлежащие компании или BYOD  
  
-   **Единый вход на** Office 365 и другие организации приложения, веб-сайтов и ресурсов.  
  
-   **На ЛИЧНЫХ устройствах**, добавить рабочую учетную запись (из локального домена или Azure AD) на устройство с персональными и наслаждайтесь единого входа к рабочим ресурсам, через приложения и в Интернете, таким образом, что помогает обеспечить соблюдение новые возможности, такие как Аттестация условного контроля учетных записей и работоспособности устройства.  
  
-   **Интеграция MDM** позволяет автоматически зарегистрировать устройства в системе MDM (Intune или сторонних производителей)  
  
-   **Настройка режима «полного экрана» и общие устройства** для нескольких пользователей в организации  
  
-   **Возможности разработчика** позволяет создавать приложения, предназначенных для корпоративных и личных контекстов в стеке общего программирования.  
  
-   **Образов** можно выбрать между образов и позволяет пользователям для настройки устройств, принадлежащих компании непосредственно во время первого запуска.  
  
Дополнительные сведения см. в разделе [Windows 10 на предприятии: способов использования устройств для работы](https://azure.microsoft.com/en-us/documentation/articles/active-directory-azureadjoin-windows10-devices-overview/?rnd=1).  
  
## <a name="BKMK_IDLocker"></a>Microsoft Passport  
Microsoft Passport — это новый проверки подлинности на основе ключей подход организации и пользователей, которые выходит за рамки пароли. Эта форма проверки подлинности зависит от взлома, кражи и воздействиям ловят учетные данные.  
  
Пользователь входит в систему на устройстве биометрические или ПИН-код для входа, ссылка на сертификате или асимметричной паре ключей. Поставщики удостоверений (IDP) проверки пользователя путем сопоставления IDLocker открытый ключ пользователя и предоставляет журнала на информацию через один раз пароля (OTP), Phonefactor или механизм различных уведомлений.  
  
Дополнительные сведения см. в разделе [проверка подлинности удостоверений без паролей с помощью Microsoft Passport](https://azure.microsoft.com/en-us/documentation/articles/active-directory-azureadjoin-passport/)  
  
## <a name="BKMK_FRSDeprecation"></a>Не рекомендуется к использованию режимы работы службы репликации файлов (FRS) и Windows Server 2003  
Несмотря на то, что служба репликации файлов (FRS) и режимы работы Windows Server 2003 были устаревшими в предыдущих версиях Windows Server, следует повторить, повторяющиеся, что операционная система Windows Server 2003 больше не поддерживаются. В результате любой контроллер домена под управлением Windows Server 2003 следует удалить из домена. Режим работы домена и леса должен быть повышен до по крайней мере Windows Server 2008 на контроллере домена под управлением более ранней версии Windows Server будут добавляться в среде.  
  
В Windows Server 2008 и более высокие функциональные уровни домена репликации распределенной файловой службы (DFS) используется для репликации между контроллерами домена содержимое папки SYSVOL. При создании нового домена в режим работы домена Windows Server 2008 или более поздней версии, автоматически используется репликация DFS для репликации SYSVOL. При создании домена на более низком уровне режима работы, необходимо перенести с помощью FRS для репликации DFS для SYSVOL. Для шагов миграции, вы можете либо выполните [процедуры на сайте TechNet](https://technet.microsoft.com/library/dd640019(v=WS.10).aspx) или можно сослаться на [упрощен набор действий в блоге CAB-файла группы хранения](http://blogs.technet.com/b/filecab/archive/2014/06/25/streamlined-migration-of-frs-to-dfsr-sysvol.aspx).  
  
Windows Server 2003 леса режимы работы домена и дальше поддерживаться, но организации должны повысить режим работы Windows Server 2008 (или более поздней версии, если возможно) для обеспечения совместимости репликации SYSVOL и поддержки в будущем. Кроме того доступны многие другие преимущества и возможности на более высоких функциональных уровнях выше. Ознакомьтесь со следующими ресурсами, Дополнительные сведения:
