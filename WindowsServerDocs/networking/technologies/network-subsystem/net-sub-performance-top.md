# <a name="network-subsystem-performance-tuning"></a>Производительности сетевой подсистемы настройки

Производительности сетевой подсистемы, особенно для рабочих нагрузок интенсивных сетевых, могут включать каждого уровня архитектура сети, которая также называется сетевой стек. Эти уровни широко делятся на следующие разделы.

1. **Сетевой интерфейс**. Это самый низкий уровень сетевой стек и содержит сети драйвер, который взаимодействует непосредственно с сетевого адаптера.

2. **Сетевой интерфейс спецификацию драйвера (NDIS)**. NDIS предоставляет интерфейсы для драйвера под ней и слои выше, такие как стек протокола.
  
3. **Стек протокола**. Стек протокола реализует протоколов, таких как TCP/IP и UDP и IP-адреса. Эти уровни предоставлять интерфейс уровня транспорта для слои выше их.
  
4. **Драйверы системы**. Обычно это клиентов, использующих расширение транспортный данных (TDX) или интерфейса Winsock Kernel (WSK) для предоставления интерфейсов для приложений пользовательского режима. Интерфейс WSK впервые появился в Windows Server 2008 и Windows&reg; Vista и он предоставляемыми AFD.sys. Интерфейс повышает производительность за счет исключения переключение между режимами пользователя и режима ядра.
  
5. **Приложения пользовательского режима**. Обычно это решение Microsoft или пользовательских приложений.
