# <a name="data-center-bridging-dcb"></a>Мост для центра обработки данных \(мост центра обработки данных\)

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Вводные сведения о мост для центра обработки данных можно использовать в этом разделе \(DCB\).

DCB — это набор института эта функция \(IEEE\) стандартов, включающих Конвергентные в центре обработки данных, когда служба хранилища, сети, данные кластера Inter\-связи процесса \(IPC\), и все трафик управления совместно использовать сетевую инфраструктуру Ethernet.

>[!NOTE]
>Дополнение к данному разделу доступна следующая документация DCB
>
>- [Установить мост центра обработки данных в Windows Server 2016 или Windows 10](dcb-install.md)
>- [Управление мост центра обработки данных](dcb-manage.md)

DCB предоставляет оборудование\-на основе выделения пропускной способности для определенного типа сетевого трафика и повышает надежность передачи Ethernet с использованием приоритет\-потока управления на основе.

Оборудование\-распределение пропускной способности на основе крайне важно, если трафик обходит операционную систему и разгружается в конвергентном сетевом адаптере, который может поддерживать Интернет интерфейс малых вычислительных систем \(iSCSI\), Удаленный прямой доступ к памяти \(RDMA\) over Ethernet или оптоволоконный канал через Ethernet \(FCoE\).

Приоритет\-управления потоками на основе важно, если протокол верхнего уровня, такой как Fiber Channel, предполагает базовую транспортировку.

## <a name="dcb-protocols-and-management-options"></a>DCB протоколы и параметры управления

Мост центра обработки данных состоит из следующих набор протоколов. 

- Расширенный службы передачи \(ETS\) — IEEE 802.1Qaz, основывающуюся на 802.1 P и 802.1Q стандартов
- Управление потоком приоритет \(PFS\), IEEE 802.1Qbb 
- Протокол обмена DCB \(DCBX\), IEEE 802.1AB, как расширенные в 802.1Qaz standard.

Протокол DCBX позволяет настроить мост центра обработки данных на коммутаторе, которые можно автоматически настроить устройства, например компьютер под управлением Windows Server 2016.

Если вы предпочитаете управлять DCB от коммутатора, не нужно установить мост центра обработки данных как компонент в Windows Server 2016, однако этот подход включает некоторые ограничения.

Поскольку DCBX может сообщать только сетевой адаптер узла размеров класс ETS и включение PFC, тем не менее, узлы Windows Server обычно требуют установку DCB, чтобы трафик сопоставляется классы ETS.

Обычно приложения Windows не предназначены для участия в обмене DCBX. По этой причине узел должен быть настроен отдельно от сетевые коммутаторы, но с идентичными параметрами.

Если вы решили управлять DCB от коммутатора, можно по-прежнему просматривать конфигураций в Windows Server 2016 с помощью команд Windows PowerShell.

##  <a name="important-dcb-functionality"></a>Важные функции DCB

Ниже представлен список функций, выполняемых мостом для центра обработки данных.

1. Обеспечивает взаимодействие между DCB\-сетевыми адаптерами, поддерживающими и DCB\-коммутаторами.

2. Предоставляет имеющего потерь транспорта Ethernet между компьютером, использующим Windows Server 2016 и его соседним коммутатором, включив приоритет\-на основе управления потоком на сетевом адаптере.

3. Предоставляет возможность выделить объем пропускной способности управления трафиком \(TC\) в процентах, где ТРАФИКОМ может содержать один или несколько классов трафика, которые различаются по класс трафика 802.1p \(приоритет\) индикаторы.

4. Предоставление администраторам сервера или сетевым администраторам возможности относить приложение к определенному классу трафика или на основе приоритета относить его к известным протоколам, известным портам TCP/UDP или NetworkDirect, используемым этим приложением.

5. Обеспечение управления мост центра обработки данных с помощью Windows Server 2016 Windows Management Instrumentation \(WMI\) и Windows PowerShell. Дополнительные сведения см. в разделе [команды Windows PowerShell для DCB](#bkmk_wps) далее в этой статье, а также следующие разделы.
    - [DCB системных компонентов](https://msdn.microsoft.com/windows/hardware/drivers/network/system-provided-dcb-components)
    - [Требования к качеству обслуживания NDIS для мост для центра обработки данных](https://msdn.microsoft.com/windows/hardware/drivers/network/ndis-qos-requirements-for-data-center-bridging)

6. Обеспечение управления мост центра обработки данных с помощью групповой политики Windows Server 2016.

7. Поддерживает сосуществование Windows Server 2016 качества обслуживания \(QoS\) решения.

>Прежде чем использовать любой RDMA over Converged Ethernet \(RoCE\) версии из RDMA, включите DCB. Хотя не требуется для протокола RDMA ширину области \(iWARP\) определила сетей, тестирования, что все Ethernet\-основе работы технологии RDMA, более эффективное программирование с DCB. По этой причине следует использовать DCB для iWARP RDMA развертываний. Дополнительные сведения см. в разделе [удаленный доступ к памяти (RDMA) и коммутатора внедренных коммутаторов (SET)](../../../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).

##  <a name="practical-applications-of-dcb"></a>Практическое применение блока управления Каталогами

Во многих организациях больших Fiber Channel \(FC\) сети хранения данных \(SAN\) установок для службы хранилища. Оптоволоконной сети хранения данных требуются специальные сетевые адаптеры на серверах и оптоволоконных коммутаторах в сети. Эти организации обычно используются сетевые адаптеры Ethernet и коммутаторы.

В большинстве случаев Оптоволоконного оборудования намного дороже развертывание, чем оборудование Ethernet, что приводит к повышению капитальных затрат. Кроме того потребность в отдельном Ethernet и сеть SAN FC сетевой адаптер и аппаратный коммутатор требуется дополнительное пространство, питания и охлаждения в центре обработки данных, что приводит к дополнительной постоянным эксплуатационным затратам.

С точки зрения затрат, бывает полезно во многих организациях объединить Оптоволоконные технологии с их Ethernet\-основе аппаратное решение для хранения данных и данных, сетевых служб.

### <a name="using-dcb-for-an-ethernet-based-converged-fabric-for-storage-and-data-networking"></a>С помощью DCB для Ethernet\-конвергентную структуру для хранилища и сетевые данные на основе

Для организаций, которые уже имеют большой Оптоволоконной сети хранения данных, но желающих избежать дополнительных инвестиций в Оптоволоконную технологию, мост центра обработки данных позволяет создавать на основе Ethernet конвергентные структуры для хранилища и сети данных. Мост центра обработки данных конвергентные структуры можно уменьшить будущих совокупную стоимость владения \(совокупной стоимости владения\) и упростить управление.

Для поставщиков услуг размещения, которые уже внедрили или планируют внедрить, iSCSI в качестве решения хранилища, DCB может предоставляют оборудование\-assisted резервирование пропускной способности для трафика iSCSI, чтобы обеспечить максимальную производительность. И в отличие от других коммерческих решений мост центра обработки данных является стандартов\-зависимости — и таким образом, сравнительно легко развертывать и управлять ими в разнородных сетевых средах.

Windows Server 2016\-реализация блока управления Каталогами устраняет многие проблемы, которые могут возникнуть при конвергентных предоставляются несколькими изготовителями оборудования \(сборщикам систем\).

Реализации коммерческих решений, предоставляемые несколькими изготовители оборудования могут плохо взаимодействовать друг с другом, может быть трудно управлять и обычно будет иметь расписания обслуживания в другом программном обеспечении. 

Напротив, Windows Server 2016 DCB является стандартов\-зависимости, и вы можете развернуть и управлять DCB в разнородной сети.
