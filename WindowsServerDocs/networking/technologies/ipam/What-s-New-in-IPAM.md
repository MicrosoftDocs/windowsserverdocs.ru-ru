# <a name="whats-new-in-ipam"></a>Новые возможности IPAM

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

В этом разделе описываются возможности управления IP-адресами (IPAM), которые являются новыми или измененными в Windows Server 2016.  
  
IPAM предоставляет широко настраиваемые возможности администрирования и мониторинга инфраструктуры DNS и IP-адрес в сети предприятия или поставщика облачных служб (CSP). Можно отслеживать, аудита и управлении серверами конфигурации протокола DHCP (Dynamic Host) и доменных имен (DNS) с помощью IPAM.

### <a name="EIP"></a>Улучшенное управление IP-адрес  
Следующие возможности улучшения функций управления адрес IPAM.  
>[!NOTE]
>Справочник по командам IPAM Windows PowerShell, см. в разделе [командлеты сервера управления IP-адресами (IPAM) в Windows PowerShell](https://technet.microsoft.com/library/jj553807.aspx).  
  
#### <a name="support-for-31-32-and-128-subnets"></a>Поддержка /31 /32 и /128 подсетей  
IPAM в Windows Server 2016 теперь поддерживает /31 /32 и /128 подсетей. Например, два адреса подсети (/ 31 IPv4) могут потребоваться точка-точка между коммутаторами. Кроме того, некоторыми параметрами может потребоваться один замыкания на себя адресов (/ 32 для IPv4, / 128 для IPv6).  
  
#### <a name="find-free-subnets-with-find-ipamfreesubnet"></a>**Найти бесплатные подсети, в которых IpamFreeSubnet поиска**  
  
Эта команда возвращает подсети, доступные для выделения, заданного блока, длины префикса и количество запрошенного подсетей IP-адрес.   
  
Если количество доступных подсетей меньше, чем количество запрошенной подсети, доступные подсети возвращаются с предупреждение о том, что число доступных меньше запрошенного числа.  
  
>[!NOTE]
>Эта функция фактически не выделить подсетей, он только выводит их доступности. Тем не менее, можно передать выходные данные командлета в **IpamSubnet добавить** команду, чтобы создать подсеть.  
  
Дополнительные сведения см. в разделе [IpamFreeSubnet найти](https://technet.microsoft.com/library/mt712782.aspx).  
  
#### <a name="find-free-address-ranges-with-find-ipamfreerange"></a>**Найти свободный адрес диапазоны с IpamFreeRange поиска**  
  
Этот новый команда возвращает доступные диапазоны IP-адресов заданного IP-подсеть, количество адресов, которые необходимы в диапазоне и число диапазонов в запросе.   
  
Команда ищет серия нераспределенных IP-адресов, которые соответствуют номер запрошенного адреса. Этот процесс повторяется, пока не запрошенное число диапазонов, или пока больше нет доступных диапазоны доступных адресов.  
  
> [!NOTE]
> Эта функция фактически не выделить диапазоны, он только выводит их доступности. Тем не менее, можно передать выходные данные командлета в **Add-IpamRange** команду, чтобы создать диапазон.  
  
Дополнительные сведения см. в разделе [IpamFreeRange найти](https://technet.microsoft.com/library/mt712772.aspx).  
  
### <a name="EDNS"></a>Улучшенное управление службы DNS  
IPAM в Windows Server 2016 теперь поддерживает обнаружение DNS-серверов на основе файла, присоединенных к домену в лесу Active Directory, в котором работает IPAM.  
  
Кроме того были добавлены следующие функции DNS:  
  
-   Зоны DNS и ресурсов записи коллекции (кроме тех, которые относятся к DNSSEC) с DNS-серверы под управлением Windows Server 2008 или более поздней версии.  
  
-   Настройка (Создание, изменение и удаление) свойства и операции для всех типов записей ресурсов (кроме тех, которые относятся к DNSSEC).  
  
-   Настройка (Создание, изменение и удаление) свойства и операции для всех типов зон DNS, включая дополнительный первичный и заглушках).  
  
-   Независимо от того активированные задачи на сервере-получателе и заглушках, если они вперед или зоны обратного просмотра. Например, задачи, такие как **передачи из главной** или **передача новую копию зоны с главного**.  
  
-   Управление доступом на основе ролей для поддерживаемой конфигурации DNS (DNS-записи и зон DNS).  
  
-   Конфигурации и коллекции серверы условной пересылки (Создание, удаление, редактирование).  
  
### <a name="DDI"></a>Встроенная DNS, DHCP и IP-адресов управления (DDI)  
При просмотре IP-адресом в перечень IP-адресов, вы можете в представлении сведений для просмотра всех записей ресурсов DNS, связанные с IP-адрес.  
  
Как часть набора записей ресурсов DNS IPAM собирает записи типа PTR для зон обратного поиска DNS. Все зоны обратного просмотра связанных с любой диапазон IP-адресов IPAM создает записи IP-адресов для PTR-записей, принадлежащих этой зоны в соответствующий сопоставленный диапазон IP-адресов. Если IP-адрес уже существует, запись PTR, просто связан с этим IP-адресом. IP-адреса не создаются автоматически, если зона обратного просмотра не сопоставлен с любой диапазон IP-адресов.  
  
При создании записи типа PTR в зону обратного просмотра с помощью IPAM, перечень IP-адресов обновляется так же, как описано выше. Во время последующих сбора так как IP-адрес уже будет существовать в системе, PTR-запись будет просто сопоставлен с этим IP-адресом.  
  
### <a name="bkmk_ad"></a>Поддержка нескольких лесах Active Directory  
В Windows Server 2012 R2 для обнаружения и управления DNS и DHCP-сервера, принадлежащие одному лесу Active Directory как ipam-сервер мог IPAM. Теперь вы можете управлять DNS и DHCP-сервера, принадлежащие к отличном от леса AD, когда в нем двустороннее отношение доверия с лесом, где установлен ipam-сервера. Вы можете перейти к **настроить обнаружение серверов** диалоговое окно и добавить домены от другого доверенного леса, которые вы хотите управлять. После обнаружения серверов в среду управления такое же, как серверы, которые принадлежат к тому же лесу, где установлен IPAM.  
  
Дополнительные сведения см. в разделе [управление ресурсами в нескольких лесах Active Directory](../../technologies/ipam/Manage-Resources-in-Multiple-Active-Directory-Forests.md)  
  
### <a name="bkmk_purge"></a>Очистка данных об использовании  
Очистка данных об использовании позволяет уменьшить размер базы данных IPAM, удалив старые данные об использовании IP-адресов. Чтобы выполнить удаление данных, необходимо указать дату и IPAM удаляет все записи базы данных, которые существуют дольше или равна дате предоставления.   
  
Дополнительные сведения см. в разделе [очистить данные об использовании](../../technologies/ipam/Purge-Utilization-Data.md).  
  
### <a name="bkmk_ps"></a>Поддержка Windows PowerShell для управления доступом на основе ролей  
Теперь можно использовать Windows PowerShell для настройки управления доступом на основе ролей. Можно использовать команды Windows PowerShell для извлечения объектов DNS и DHCP в IPAM и измените их области доступа. По этой причине можно написать скрипты Windows PowerShell для назначения области доступа к следующим объектам.  
  
-   Пространство IP-адресов  
  
-   Блок IP-адресов  
  
-   Подсети IP-адресов  
  
-   Диапазоны IP-адресов  
  
-   DNS-серверы  
  
-   Зоны DNS  
  
-   Серверы условной пересылки  
  
-   Записи ресурсов DNS  
  
-   DHCP-серверы  
  
-   суперобласти DHCP;  
  
-   Области DHCP  
  
Дополнительные сведения см. в разделе [управление доступом на основе ролей с помощью Windows PowerShell](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Windows-PowerShell.md) и [командлеты сервера управления IP-адресами (IPAM) в Windows PowerShell](https://technet.microsoft.com/library/jj553807.aspx).
