# <a name="whats-new-in-ipam"></a>Новые возможности IPAM

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе описываются функции управления IP-адресами (IPAM), добавленные или измененные в Windows Server 2016.  
  
IPAM предоставляет широкие возможности администрирования и мониторинга инфраструктуры DNS и IP-адрес в сети предприятия или поставщика облачных услуг (CSP). Отслеживание, аудит и управлять серверами под управлением протокол динамической настройки узла (DHCP) и доменных имен (DNS) с помощью IPAM.

### <a name="EIP"></a>Улучшенное управление IP-адрес  
Следующие функции улучшения функций управления адрес IPAM.  
>[!NOTE]
>Справочник по командам IPAM Windows PowerShell в разделе [командлеты сервер управления IP-адресами (IPAM) в Windows PowerShell](https://technet.microsoft.com/library/jj553807.aspx).  
  
#### <a name="support-for-31-32-and-128-subnets"></a>Поддержка /31, /32 и /128 подсетей  
IPAM в Windows Server 2016 теперь поддерживает /31, /32 и /128 подсетей. Например, два адреса подсети (или 31 IPv4) может быть обязательным для точка-точка между коммутаторами. Кроме того, некоторые ключи требуется один замыкания на себя адреса (/ 32 для IPv4, / 128 для IPv6).  
  
#### **<a name="find-free-subnets-with-find-ipamfreesubnet"></a>Найти бесплатные подсетях с IpamFreeSubnet поиска**  
  
Эта команда возвращает подсети, доступные для выделения, заданным блок, длина префикса и количество запрошенный подсети IP.   
  
Если количество доступных подсетей меньше, чем номер запрошенного подсетей, с предупреждение о том, что число доступных меньше, чем номер, запрашиваемый возвращаются доступные подсетей.  
  
>[!NOTE]
>Эта функция не фактически выделить подсетей, сообщается только их доступности. Тем не менее, можно передать выходные данные командлета в **добавить IpamSubnet** команду, чтобы создать подсеть.  
  
Дополнительные сведения см. в разделе [найти IpamFreeSubnet](https://technet.microsoft.com/library/mt712782.aspx).  
  
#### **<a name="find-free-address-ranges-with-find-ipamfreerange"></a>Найти свободный адрес диапазоны с IpamFreeRange поиска**  
  
Этот новый команда возвращает доступные диапазоны IP-адресов заданным IP-подсеть, число адресов, которые необходимы в диапазоне и количества диапазонов запрашивается.   
  
Команда ищет непрерывные ряды нераспределенного IP-адресов, соответствующих число запрошенный адресов. Этот процесс повторяется, пока не запрошенного количества диапазонов, или диапазоны пока доступно больше нет доступных IP-адресов.  
  
> [!NOTE]
> Эта функция не фактически выделить диапазоны, сообщается только их доступности. Тем не менее, можно передать выходные данные командлета в **Add-IpamRange** команду, чтобы создать диапазон.  
  
Дополнительные сведения см. в разделе [найти IpamFreeRange](https://technet.microsoft.com/library/mt712772.aspx).  
  
### <a name="EDNS"></a>Улучшенное управление службы DNS  
IPAM в Windows Server 2016 теперь поддерживает обнаружение DNS-серверов на основе файлов, присоединенных к домену в лесу Active Directory, в котором работает IPAM.  
  
Кроме того были добавлены следующие функции DNS-сервера:  
  
-   Зоны DNS-сервера и ресурс записывает коллекции (кроме тех, относящиеся к DNSSEC) от DNS-серверов под управлением Windows Server 2008 или более поздней версии.  
  
-   Настройка (создания, изменения и удаления) свойства и операции для всех типов записей ресурсов (кроме тех, относящиеся к DNSSEC).  
  
-   Настройка (Создание, изменение и удаление) свойства и операции на всех типах зон DNS, включая основной сервер-получатель и зонах-заглушках).  
  
-   Независимо от активации задачи на дополнительной и зонах-заглушках, если они вперед или зоны обратного просмотра. Например, задач, таких как **передачи с основного сервера** или **передачи новую копию зону с основного сервера**.  
  
-   Управление доступом на основе ролей для поддерживаемой конфигурации DNS-сервера (DNS-записи и зон DNS).  
  
-   Серверы условной пересылки сбора данных и конфигурации (создания, удаления, изменения).  
  
### <a name="DDI"></a>Встроенная DNS, DHCP и IP-адреса управления (DDI)  
При просмотре IP-адрес в перечне IP-адресов, вы можете в представлении подробностей, для просмотра всех записей ресурсов DNS, связанные с IP-адрес.  
  
Как часть коллекции записи ресурсов DNS IPAM собирает PTR-записей для зон обратного поиска DNS. Для все зоны обратного просмотра, сопоставленных любого диапазона IP-адресов IPAM создает записи IP-адресов для всех записей PTR, принадлежащие этой зоны, в соответствующие сопоставленные диапазон IP-адресов. Если IP-адреса уже существует, просто связанные с этого IP-адреса PTR-запись. Если зона обратного просмотра не сопоставляются с любого диапазона IP-адресов IP-адреса не создается автоматически.  
  
При создании PTR-запись в зоне обратного просмотра через IPAM перечень IP-адресов обновляется так же, как описано выше. Во время последующих коллекции так как IP-адреса уже существует в системе, PTR-запись будет просто сопоставлено с этого IP-адреса.  
  
### <a name="bkmk_ad"></a>Поддержка нескольких лесах Active Directory  
В Windows Server 2012 R2 IPAM удалось обнаруживать и управлять DNS и DHCP-серверы, принадлежащие одному лесу Active Directory, как IPAM-сервера. Теперь вы можете управлять DNS и DHCP-серверы, принадлежащие к лесу AD, когда в нем двустороннее отношение доверия с лесом, где установлен IPAM-сервера. Вы можете перейти на **настроить обнаружение серверов** диалоговое окно и добавить доверенные домены из других лесов, которым требуется управлять. После обнаружения серверов управление ничем не такая же, как серверы, которые принадлежат тому же лесу, где установлена IPAM.  
  
Дополнительные сведения см. в разделе [управление ресурсами в нескольких лесах Active Directory](../../technologies/ipam/Manage-Resources-in-Multiple-Active-Directory-Forests.md)  
  
### <a name="bkmk_purge"></a>Очистка данных об использовании  
Данные об использовании очистки позволяет уменьшить размер базы данных IPAM, удалив старые данные об использовании IP-адресов. Чтобы выполнить удаление данных, укажите дату и IPAM удаляет все записи в базе данных, возраст которых превышает или даты вами.   
  
Дополнительные сведения см. в разделе [Очистка данных об использовании](../../technologies/ipam/Purge-Utilization-Data.md).  
  
### <a name="bkmk_ps"></a>Поддержка Windows PowerShell для управления доступом на основе ролей  
Теперь можно использовать Windows PowerShell для настройки управления доступом на основе ролей. Для получения объектов DNS и DHCP в IPAM и измените их областей доступа можно использовать команды Windows PowerShell. Таким образом можно создавать сценарии Windows PowerShell для назначения области доступа для следующих объектов.  
  
-   Пространство IP-адресов  
  
-   Блок IP-адресов  
  
-   Подсети IP-адресов  
  
-   Диапазоны IP-адресов  
  
-   DNS-серверы  
  
-   Зоны DNS  
  
-   Серверы условной пересылки  
  
-   Записи ресурсов DNS  
  
-   DHCP-серверы  
  
-   Суперобласти DHCP  
  
-   Области DHCP  
  
Дополнительные сведения см. в разделе [управление Role Based Access Control с помощью Windows PowerShell](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Windows-PowerShell.md) и [командлеты сервер управления IP-адресами (IPAM) в Windows PowerShell](https://technet.microsoft.com/library/jj553807.aspx).
