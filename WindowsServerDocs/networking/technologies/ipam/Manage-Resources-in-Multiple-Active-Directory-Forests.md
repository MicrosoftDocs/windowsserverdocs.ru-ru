# <a name="manage-resources-in-multiple-active-directory-forests"></a>Управление ресурсами в нескольких лесах Active Directory

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе можно использовать, чтобы узнать, как с помощью IPAM для управления контроллерами домена, DHCP-серверы и DNS-серверы в нескольких лесах Active Directory.  
  
С помощью IPAM для управления ресурсами в удаленных лесах Active Directory, каждый лес, который вы хотите управлять необходимы два способом отношений доверия с лесом, в котором установлен IPAM.  
  
Для запуска процесса обнаружения для разных лесах Active Directory, откройте диспетчер сервера и нажмите кнопку IPAM. В консоли IPAM-клиента, щелкните **настроить обнаружение серверов**, а затем нажмите кнопку **получить лесов**. При этом инициируется фоновую задачу, которая обнаруживает доверенных лесов и доменов. После завершения процесса обнаружения, нажмите кнопку **настроить обнаружение серверов**, где открывается следующее диалоговое окно.  
  
![Настройка обнаружения серверов](../../media/Manage-Resources-in-Multiple-Active-Directory-Forests/ipam_serverdiscovery.jpg)  

>[!NOTE]
>Для подготовки на основе группы вступил сценария между лесами Active Directory, убедитесь, что следующий командлет Windows PowerShell на сервере IPAM, а не на доверяющего домена контроллеры домена. Например если IPAM-сервер присоединен к лесу corp.contoso.com и доверяющий лес — fabrikam.com, можно выполнить следующий командлет Windows PowerShell на сервере IPAM в corp.contoso.com для подготовки с помощью вступил группы в лесу fabrikam.com. Для выполнения этого командлета, необходимо быть членом группы администраторов домена в лесу fabrikam.com.

    
    Invoke-IpamGpoProvisioning -Domain fabrikam.COM -GpoPrefixName IPAMSERVER -IpamServerFqdn IPAM.CORP.CONTOSO.COM
    

В **настроить обнаружение серверов** диалоговом нажмите кнопку **выберите леса**, а затем выберите леса, которым требуется управлять с помощью IPAM. Также выбрать домены, которые вы хотите управлять, а затем нажмите кнопку **добавить**.

В **Выбор ролей сервера, чтобы обнаружить**, для каждого домена, которым требуется управлять, укажите тип серверов для обнаружения. Параметры **контроллер домена**, **DHCP-сервера**, и **DNS-сервер**.

По умолчанию контроллеры домена, DHCP-серверы и DNS-серверы обнаружены — поэтому если вы не хотите обнаружить одного из этих типов серверов, убедитесь, отмените выбор флажок для этого варианта.

В приведенном выше примере рисунке IPAM-сервер устанавливается contoso.com леса и корневого домена леса fabrikam.com добавляется для управления IPAM. Ролей выбранным сервером разрешить IPAM для обнаружения и управления контроллерами домена, DHCP-серверы и DNS-серверы в корневом домене fabrikam.com и contoso.com корневого домена.

После указания леса, домены и роли сервера, щелкните **ОК**. IPAM выполняет обнаружение, и после завершения обнаружения управлять ресурсами в локальных и удаленных леса.
