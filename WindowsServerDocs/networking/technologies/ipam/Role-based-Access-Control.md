# <a name="role-based-access-control"></a>Управление доступом на основе ролей

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

Этот раздел содержит сведения об использовании управления доступом на основе ролей в IPAM.  
  
> [!NOTE]  
> Помимо этого раздела IPAM доступа управления доступна следующая документация в этом разделе.  
>   
> -   [Ролевое управление доступом с помощью диспетчера серверов](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Server-Manager.md)  
> -   [Ролевое управление доступом с помощью Windows PowerShell](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Windows-PowerShell.md)  
  
Управление доступом на основе ролей позволяет указать права доступа на различных уровнях, включая DNS-сервера, DNS-зоны и уровни записей ресурсов DNS.  
С помощью управления доступом на основе ролей, можно указать, кто имеет контроль над операций на создание, изменение и удаление различных типов записей ресурсов DNS.  
  
Можно настроить управление доступом, чтобы пользователи не ограничивается следующие разрешения.  
  
-   Пользователи могут изменить только определенные записи ресурсов DNS  
  
-   Пользователи могут изменить записей ресурсов DNS для конкретного типа, например PTR или MX  
  
-   Пользователи могут изменять записи ресурсов DNS для конкретных зон
