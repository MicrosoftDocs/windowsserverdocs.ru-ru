# <a name="manage-qos-policy"></a>Управлять политиками качества обслуживания

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Дополнительные сведения об использовании мастера политики качества обслуживания, чтобы создать, изменить или удалить политику качества обслуживания можно использовать в этом разделе.

>[!NOTE]
>  Дополнение к данному разделу доступна следующая документация по управления политики качества обслуживания.
> 
>  - [Ошибки и события политики качества обслуживания](qos-policy-errors.md)

В операционных системах Windows политики QoS сочетает в себе функции качества обслуживания на основе стандартов с управляемостью групповой политики. Конфигурации это сочетание позволяет упростить применение политик качества обслуживания для объектов групповой политики. Windows включает в себя Мастер политики QoS, которые помогут выполнить следующие задачи.

-  [Создание политики качества обслуживания](#bkmk_createpolicy)

-  [Просмотреть, изменить или удалить политику качества обслуживания](#bkmk_editpolicy)

##  <a name="bkmk_createpolicy"></a>Создание политики качества обслуживания

Перед созданием политики QoS, важно понимать два ключевых качества обслуживания на уровне элемента управления, которые используются для управления сетевым трафиком.

- Значение поля DSCP

-   Ограничение скорости

### <a name="prioritizing-traffic-with-dscp"></a>Назначение приоритетов трафика с помощью DSCP

Как отмечалось в предыдущем примере бизнес-приложения, можно определить приоритет для исходящего сетевого трафика с помощью **укажите значение DSCP** настроить политику QoS с определенным значением поля DSCP. 

Как описано в RFC 2474, DSCP позволяет значения от 0 до 63 указывать внутри поля TOS пакетов IPv4, так и в поле класса трафика в IPv6. Сетевые маршрутизаторы используют значение DSCP для классификации сетевых пакетов и их в очередь соответствующим образом.
  
> [!NOTE]
>  По умолчанию трафик Windows имеет значение DSCP 0.
  
В рамках стратегии обеспечения качества обслуживания вашей организации необходимо предусмотреть количество очередей и поведение для пакетов с разными приоритетами. Например, организации, можно иметь пять очередей: чувствительных к задержкам трафика, трафик управления, критически важных бизнес-трафик, трафик усилия и Массовая передача данных трафика.  
  
### <a name="throttling-traffic"></a>Регулируемый трафик

Наряду со значениями поля DSCP регулирования — другого ключа элемента управления для управления пропускной способностью сети. Как упоминалось ранее, можно использовать **укажите частоту глушителя** параметр, чтобы настроить политику качества обслуживания конкретного ограничение скорости для исходящего трафика. С помощью политики QoS ограничивает исходящий сетевой трафик на указанной скорости. Для эффективного управления трафиком можно одновременно использовать указание значений DSCP и регулирование количества запросов.

>[!NOTE]
>По умолчанию флажок **Укажите частоту глушителя** не установлен.

Чтобы создать политику качества обслуживания, измените параметры объекта групповой политики объект (GPO) из этой программы либо на консоли управления групповыми политиками (GPMC). Консоль управления групповыми Политиками затем открывает редактор объектов групповой политики.

Имя политики QoS должно быть уникальным. Применение политик к серверам и конечных пользователей зависит от хранения политики QoS в редакторе объектов групповой политики:

- Политика QoS в Конфигурация компьютера\Параметры Settings\QoS политика применяется к компьютерам независимо от пользователя, вошедшего в систему. Как правило, политики QoS для компьютеров используются для серверов.

- Политика QoS в политике Конфигурация компьютера\Параметры Settings\QoS применяется к пользователям после вошедшим в систему, независимо от того, какой компьютер они вошли в систему.

#### <a name="to-create-a-new-qos-policy-with-the-qos-policy-wizard"></a>Чтобы создать политику QoS с помощью мастера политики QoS

-   В редакторе объектов групповой политики, щелкните правой кнопкой мыши любой из **политику качества обслуживания** узлов, а затем щелкните **создайте новую политику**.

### <a name="wizard-page-1---policy-profile"></a>Страница мастера 1 - профиля политики

На первой странице мастера политики качества обслуживания можно указать имя политики и настроить, каким образом QoS контролирует исходящего сетевого трафика.

#### <a name="to-configure-the-policy-profile-page-of-the-qos-based-policy-wizard"></a>Настройка параметров на странице "Профиль политики" мастера QoS на основе политики

1. В поле **Имя политики** введите имя политики QoS. Имя должно однозначно идентифицировать политику.

2. При необходимости использовать **укажите значение DSCP** включить назначение DSCP и затем задать значение DSCP в диапазоне от 0 до 63.

3. Также при желании можно установить флажок **Укажите частоту глушителя** для включения регулировки трафика и настройки ограничения скорости. Значение частоты передачи должно быть больше 1, и можно указать единицы, в килобайтах в секунду \(Кбит/с\) или мегабайт в секунду \(Мбит/с\).

4. Нажмите кнопку **Далее**.

### <a name="wizard-page-2---application-name"></a>Страница мастера 2 — имя приложения

На второй странице мастера QoS политики можно применить политику ко всем приложениям, для определенного приложения, определенные в зависимости от его имя исполняемого файла, путь и имя приложения или приложения сервера HTTP, которые обрабатывают запросы для определенного URL-адреса.

- **Все приложения** указывает, что параметры управления трафиком на первой странице мастера QoS политики применяются ко всем приложениям.

- **Только приложения с именем исполняемого файла** указывает, что параметры управления трафиком на первой странице мастера политик QoS для конкретного приложения. Имя исполняемого файла должно иметь расширение EXE.

- **Только приложения сервера HTTP, реагирование на запросы для этого URL-адреса** указывает, что параметры управления трафиком на первой странице мастера QoS политики применяются для определенных HTTP-сервера приложений только.

Дополнительно можно указать путь к каталогу приложения. Путь к приложению указывается вместе с именем приложения. Этот путь может содержать переменные среды. Например, %ProgramFiles%\Путь к моему приложению\MyApp.exe или c:\program files\путь к моему приложению\myapp.exe.

>[!NOTE]
>Путь к приложению, не может содержать путь, который разрешается в символьную ссылку.

URL-адрес должен соответствовать [приведено](https://tools.ietf.org/html/rfc1738), в виде `http[s]://<hostname\>:<port\>/<url-path>`. Можно использовать подстановочные знаки, `‘*’`, для `<hostname>` и/или `<port>`, например `https://training.\*/, https://\*.\*`, но подстановочный знак не может указать подстроку `<hostname>` или `<port>`.

Другими словами, ни `https://my\*site/` , ни `https://\*training\*/` является допустимым. 

При желании вы можете проверить **включить подкаталоги и файлы** для соответствия все подкаталоги и файлы с URL-адрес. Например, если этот флажок установлен, и URL-адрес `https://training`, политику качества обслуживания будет рассматривать запросы` https://training/video` хорошего соответствия.

#### <a name="to-configure-the-application-name-page-of-the-qos-policy-wizard"></a>Настройка мастера политики качества обслуживания на уровне страницы имя приложения

1. В **применение политики к**, выберите пункт **все приложения** или **только к приложениям с именем исполняемого файла**.

2. Если вы выберете параметр **Только к приложениям с именем исполняемого файла**, укажите имя исполняемого файла с расширением .EXE.

3. Нажмите кнопку **Далее**.

### <a name="wizard-page-3---ip-addresses"></a>Страница мастера 3 — IP-адресов

На третьей странице мастера политики QoS можно задать условия адрес IP-адрес для политики QoS, включая следующие:

- ко всем исходным IPv4- или IPv6-адресам либо к определенным исходным IPv4- или IPv6-адресам;

- Все назначения IPv4 или IPv6-адреса или адреса IPv4 или IPv6 в определенное место назначения

При выборе вариантов **Только к следующему IP-адресу источника** или **Только к следующему конечному IP-адресу** необходимо указать одно из следующих значений:

- Адрес IPv4, такие как `192.168.1.1`

- Префикс адреса IPv4, с помощью нотации длины префикса сети, например `192.168.1.0/24`

- Адрес IPv6, таких как `3ffe:ffff::1`

- Например, префикс, адреса IPv6 `3ffe:ffff::/48`

При выборе вариантов **только для следующих IP-адрес источника** и **только для следующих IP-адрес назначения**, адреса или префиксы адресов должен быть либо IPv4 - или IPv6-зависимости.

Если указан URL-адрес HTTP-сервера приложений на предыдущей странице мастера, вы заметите, что исходный IP-адрес для политики качества обслуживания на этой странице мастера отображается серым. 

Это верно, поскольку исходный IP-адрес — адрес HTTP, а не настраивается здесь. С другой стороны по-прежнему можно настроить политику, указав IP-адрес назначения. Это позволяет создавать различные политики для разных клиентов, используя те же приложения сервера HTTP.

#### <a name="to-configure-the-ip-addresses-page-of-the-qos-policy-wizard"></a>Настройка IP-адреса страницы мастера политики QoS

1. В **применение политики к** (источник), выберите **любой источник, IP-адрес** или **только для следующих IP-адрес источника**.

2. Если вы выбрали **только следующие IP-адреса источника**, укажите адрес IPv4 или IPv6 или префикс.

3. В **применение политики к** (конечные) выберите **любым конечным адресам** или **только для следующих IP-адрес назначения.**

4. Если вы выбрали **только для следующих IP-адрес назначения**, укажите IPv4 или IPv6-адрес или префикс, соответствующий типу адреса или префикса, указанного для исходного адреса.

5.  Нажмите кнопку **Далее**.  

### <a name="wizard-page-4---protocols-and-ports"></a>Страница мастера 4 - протоколы и порты

На четвертой странице мастера политики качества обслуживания можно указать типы трафика и порты, которые управляются с помощью параметров на первой странице мастера. Можно указать:  
-   трафик TCP, трафик UDP или оба типа трафика;  

-   все исходные порты, диапазон исходных портов или определенный исходный порт;

-   Все конечные порты, диапазон портов назначения или определенный конечный порт, выполните следующие действия.  

#### <a name="to-configure-the-protocols-and-ports-page-of-the-qos-policy-wizard"></a>Чтобы настроить протоколы и порты страницы мастера политики QoS

1. В разделе **Выберите протокол, к которому применяется политика QoS:** выберите параметр **TCP**, **UDP** или **TCP и UDP**.

2. В разделе **Укажите номер исходного порта** выберите параметр **Любой исходный порт** или **Номер исходного порта**.

3. Если вы выбрали **номер исходного порта**, введите номер порта в диапазоне от 1 до 65535.

     При необходимости можно указать диапазон портов, в формате "*низким*:*высокой*,» где *низким* и *высокой* представляют нижней границы и верхние границы порта в диапазоне, включительно. *Низкий* и *высокой* каждый должен быть числом от 1 до 65535. Между символом двоеточия (:) и числами не должно быть пробела.

4. В разделе **Укажите номер порта назначения** выберите параметр **Любой порт назначения** или **Номер целевого порта**.

5. Если на предыдущем этапе был выбран параметр **Номер целевого порта**, введите номер порта в диапазоне от 1 до 65535.

Чтобы завершить создание новой политики QoS, нажмите кнопку **Готово** на **протоколы и порты** мастера политик качества обслуживания. После завершения нового политика QoS появится в области сведений редактора объектов групповой политики.  
  
Чтобы применить параметры политики QoS к пользователям или компьютерам, свяжите объект групповой Политики, в котором размещены политики QoS в контейнер доменных служб Active Directory, например домена, сайта или подразделения (OU).  
  
##  <a name="bkmk_editpolicy"></a>Просмотреть, изменить или удалить политику качества обслуживания

Страницы политики QoS, мастер описанные ранее соответствуют страницам свойств, которые отображаются при просмотре или изменение свойств политики.  
  
### <a name="to-view-the-properties-of-a-qos-policy"></a>Просмотр свойств политики QoS  
  
-   Щелкните правой кнопкой мыши имя политики в области сведений редактора объектов групповой политики, а затем нажмите кнопку **свойства**.  
  
     Редактор объектов групповой политики будет выведена страница свойств со следующими вкладками:  
  
    -   "Профиль политики";  
  
    -   "Имя приложения";  
  
    -   "IP-адреса";  
  
    -   Протоколы и порты  
  
### <a name="to-edit-a-qos-policy"></a>Изменение политики QoS  
  
-   Щелкните правой кнопкой мыши имя политики в области сведений редактора объектов групповой политики, а затем нажмите кнопку **изменить существующую политику**.  
  
     Отображает редактор объектов групповой политики **изменение существующей политики QoS** диалоговое окно.  
  
### <a name="to-delete-a-qos-policy"></a>Удаление политики QoS  
  
-   Щелкните правой кнопкой мыши имя политики в области сведений редактора объектов групповой политики, а затем нажмите кнопку **удалить политику**.  
  
### <a name="qos-policy-gpmc-reporting"></a>Политики качества обслуживания отчетов консоли управления групповыми Политиками 

После применения ряд политик качества обслуживания в организации, часто бывает удобно или необходимо периодически просматривать применение политик. Можно просмотреть сводку политики качества обслуживания для конкретного пользователя или компьютера с помощью отчетов консоли управления групповыми Политиками.  
  
#### <a name="to-run-the-group-policy-results-wizard-for-a-report-of-qos-policies"></a>Чтобы запустить мастер результатов групповой политики для отчета политик качества обслуживания  
  
-   В консоли управления групповыми Политиками, щелкните правой кнопкой мыши **результаты групповой политики** и выберите узел для параметра меню **мастер результатов групповой политики.**  
  
После создания результатов групповой политики, щелкните **параметры** вкладки. На **параметры** вкладке политики качества обслуживания можно найти в узлах «Конфигурация компьютера\Параметры Settings\QoS политика» и «Политика Settings\QoS Конфигурация компьютера\Параметры пользователей».  
  
На **параметры** вкладке политики качества обслуживания перечисляются по имени политики QoS с их значение DSCP, ограничение скорости, условия политики, а результирующий объект групповой Политики в той же строке...  
  
Просмотр результатов групповой политики однозначно определяет конечный GPO. Если несколько объектов групповой политики имеют политики качества обслуживания с тем же именем политики QoS, применяется объект групповой Политики с самым высоким приоритетом объекта групповой Политики. Это конечный GPO. Конфликтующие политики качества обслуживания (определяется по имени политики), подключенных к более низким приоритетом, объект групповой Политики не применяются. Обратите внимание, что определить приоритеты GPO политики QoS, развернутые в узел, домен или Подразделение, соответствующим образом. После развертывания на уровне пользователя или компьютера [правил приоритета политики QoS](#BKMK_precedencerules) определить, какой трафик разрешенные и заблокированные.  
  
Политики QoS значение DSCP ограничение скорости и условия политики доступны в редактор объектов групповой политики (GPOE)  
  
### <a name="advanced-settings-for-roaming-and-remote-users"></a>Дополнительные параметры для роуминга и удаленным пользователям  
С помощью политики QoS предназначена для управления трафиком в сети предприятия. В мобильных ПК пользователи могут отправлять трафик или отключить корпоративной сети. Так как политики качества обслуживания не относятся, даже работая вне корпоративной сети, только на сетевых интерфейсов, которые подключены к организации для Windows 8, Windows 7 или Windows Vista включены политики качества обслуживания.  
  
Например пользователь может подключиться своего портативного компьютера к своей корпоративной сети через виртуальную частную сеть (VPN) из кафе. Для VPN физический сетевой интерфейс (например, беспроводные сети) не применяются политики качества обслуживания. Тем не менее интерфейса VPN будет иметь политик качества обслуживания, применить, поскольку он подключается к организации. Если пользователь введет позже сети другой организации, не имеет отношение доверия AD DS, политики качества обслуживания не будет включен.  
  
Обратите внимание, что эти сценариев для мобильных устройств не применяются для рабочих нагрузок сервера. Например сервер с несколькими сетевыми адаптерами могут некоторое границы сети предприятия. ИТ-отдел может выбрать трафик регулирования политики качества обслуживания, который egresses предприятия; Тем не менее этот сетевой адаптер, который отправляет этот исходящий трафик не обязательно обратно к корпоративной сети. По этой причине политики качества обслуживания всегда включена на всех сетевых интерфейсов на компьютере под управлением Windows Server 2012.  
  
> [!NOTE]
>  Выборочная Включение применяется только в том случае, для политики качества обслуживания, а не параметры качества обслуживания на Advanced, рассматривается далее в этом документе.  
  
### <a name="advanced-qos-settings"></a>Дополнительные параметры качества обслуживания

Дополнительные параметры качества обслуживания предоставляют дополнительные элементы управления для ИТ-администраторам управлять потреблением сети компьютера и пометок DSCP. Дополнительные параметры качества обслуживания применяются только на уровне компьютера, тогда как политики QoS можно применять на уровне компьютера и пользователя.

#### <a name="to-configure-advanced-qos-settings"></a>Чтобы настроить дополнительные параметры качества обслуживания

1.  Нажмите кнопку **Конфигурация компьютера**, а затем нажмите кнопку **параметры Windows в групповой политике**.
  
2.  Щелкните правой кнопкой мыши **политику качества обслуживания**, а затем нажмите кнопку **Дополнительные параметры качества обслуживания**.

     На следующем рисунке показана два дополнительных вкладок параметров качества обслуживания: **Входящий трафик TCP** и **Переопределение пометки DSCP**.
  
> [!NOTE]
>  Дополнительные параметры качества обслуживания и параметры групповой политики уровня компьютера.
  
#### <a name="advanced-qos-settings-inbound-tcp-traffic"></a>Дополнительные параметры качества обслуживания: входящий трафик TCP

**Входящий трафик TCP** элементы управления TCP использование пропускной способности на стороне получателя, а политики качества обслуживания влияют на исходящий трафик TCP и UDP. 

Установив уровень на меньшей пропускной способности **входящий трафик TCP** вкладке TCP ограничит его объявленной TCP окна приема. Влияние этой настройки будет повышенную пропускную способность и использование канала для TCP-подключения с более высокой пропускной способности и задержки (произведения задержки, пропускной способности). По умолчанию компьютеры под управлением Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 и Windows Vista присваивается уровень максимальной пропускной способности.
  
Приема TCP окно изменилось в Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 и Windows Vista от предыдущих версий Windows. Предыдущие версии Windows ограничение окна на стороне приема TCP максимум 64 килобайт (КБ), Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 и Windows Vista динамически размер окна на стороне приема — до 16 мегабайтам (МБ ). В элементе управления входящего трафика TCP можно управлять уровнем входящей пропускной способности, задав максимальное значение, до которого может расти окно приема TCP. Уровни соответствуют следующие максимальные значения. 
  
|Уровень входящей пропускной способности|Максимум|  
|------------------------|-------|  
|0|64 КБ|
|1|256 КБ|
|2|1 MБ|
|3|16 МБ|

Размер окна может быть значение равным или меньшим, чем максимальное значение, в зависимости от состояния сети.

###### <a name="to-set-the-tcp-receive-side-window"></a>Чтобы задать период на стороне приема TCP

1. Щелкните в редакторе объектов групповой политики **политика локального компьютера**, нажмите кнопку **параметры Windows**, щелкните правой кнопкой мыши **политику качества обслуживания**и нажмите кнопку **Advanced QoS Параметры**.
  
2. В **пропускную способность приема TCP**выберите **Настройка пропускной способности TCP получение**и затем выберите нужный уровень пропускной способности, который вы хотите.

3.  Свяжите объект групповой Политики для Подразделения.

#### <a name="advanced-qos-settings-dscp-marking-override"></a>Дополнительные параметры качества обслуживания: Переопределение пометки DSCP

Переопределение пометки DSCP ограничивает возможности приложения, чтобы указать, или «отметки» — DSCP значения, отличные от указанных политик качества обслуживания. Указав, что приложения могут задавать значения DSCP, приложение может задать ненулевые значения DSCP. 

Путем указания **пропустить**, приложения, использующие API-интерфейсы QoS будет иметь значения установлено в ноль, и только политики QoS можно задавать значения DSCP. 

По умолчанию компьютеры под управлением Windows Server 2016, Windows 10, Windows Server 2012 R2, Windows 8.1, Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 и Windows Vista позволяют приложениям указывать значения DSCP; приложения и устройства, которые не используют API-интерфейсы QoS не переопределяются.

##### <a name="wireless-multimedia-and-dscp-values"></a>Значения DSCP и беспроводного мультимедиа

[Wi-Fi Alliance](https://go.microsoft.com/fwlink/?LinkId=160769) установил сертификации для беспроводного мультимедиа \(WMM\) , определяет четыре категории доступа \(WMM_AC\) назначении приоритетов сетевого трафика передано по Wi\-Fi беспроводной сети. Категории доступа включают \(в соответствии с наибольшим к-самый низкий приоритет\): голос, видео, усилий и фона; соответственно обозначается аббревиатурой VO, VI, BE и BK. Спецификация WMM определяет, какие поля DSCP значения соответствуют каждой из категорий четыре доступа:
  
|Значение поля DSCP|Категории доступа к WMM|
|----------|-------------------|
|48-63|Голоса (VO)|
|32-47|Видео (VI)|
|24-31, 0-7|По возможности (BE)|
|8-23|Фон (КБ)|

Можно создать политики QoS, использующих эти значения DSCP, чтобы убедиться, что переносимая компьютеры с Wi\-Fi Certified™ для адаптеров беспроводной сети WMM получать приоритет обработки, когда связанный с Wi\-Fi Сертифицировано для WMM точек доступа.
  
### <a name="BKMK_precedencerules"></a>Правила приоритета политики качества обслуживания

Как и приоритеты объектов групповой Политики, политики качества обслуживания имеют правила приоритета для разрешения конфликтов, когда несколько политик QoS применяются к определенному набору трафика. Для исходящего трафика TCP или UDP только одна политика качества обслуживания могут применяться одновременно, это означает, что политики качества обслуживания не имеют общее влияние, например, где будет суммироваться регулирование скорости.

Как правило политики QoS с наиболее соответствующие условия wins. Если применить несколько политик качества обслуживания, правила делятся на три категории: уровня пользователя и уровня компьютера; приложения и quintuple сети; и среди сети quintuple.

По *сети пятиэлементный*, мы имеем в виду, исходный IP-адрес, конечный IP-адрес, исходный порт, конечный порт и протокол \(TCP/UDP\).  

 **Политики качества обслуживания на уровне пользователя имеет приоритет над политики качества обслуживания на уровне компьютера**

Это правило существенно облегчает управление сетевым администраторам объектов групповой политики качества обслуживания, особенно для пользователя на основе групповой политики. Например если сетевой администратор хочет определить политику качества обслуживания для группы пользователей, их можно просто создать и распространить объект групповой Политики в эту группу. Они не нужно беспокоиться о компьютеры, которые эти пользователи выполнили вход и ли этих компьютерах будут указаны конфликтующие политики качества обслуживания, которые определены, так как при возникновении конфликта приоритет всегда получает политика уровня пользователя.

> [!NOTE]
>  Политику качества обслуживания уровня пользователя применяется только к трафику, который создается этим пользователем. Другим пользователям определенного компьютера и на самом компьютере, не будут применяться все политики QoS, которые определены для этого пользователя.

 **Приложение специфичность и имеют приоритет над quintuple сети**

Если определенный трафик соответствует несколько политик качества обслуживания, применяется более конкретная политика. Среди политик, определить приложения политику, которая содержит путь к файлу отправляющее приложение считается более специфичен, чем другой политики, которая только определяет имя приложения (без пути). Если несколько политик с приложениями по-прежнему, правила приоритета используйте пятиэлементный сети, чтобы найти наилучшее соответствие.

Кроме того несколько политик QoS может применяться тот же трафик, задав условия перекрываться. Между условия приложений и quintuple сети политику, которая задает приложение, считается более конкретным и применяется. 

Например policy_A только указывает имя приложения (app.exe), с указанием policy_B 192.168.1.0/24 адрес IP-адрес назначения. При этих политик QoS конфликтов \(app.exe начинает отправлять трафик к IP-адрес в диапазоне от 192.168.4.0/24\), policy_A применяется.

 **Более конкретный имеет более высокий приоритет, в сети пятиэлементный**

Для конфликтов политик в quintuple сети приоритет имеет политика, наиболее соответствующий условиям. Например предположим, что policy_C указывает исходный IP-адрес, «любое», конечный IP-адрес 10.0.0.1, исходного порта «любой», «любой» порт назначения и протокол «TCP». 

Затем Предположим, что policy_D указывает исходный IP-адрес, «любой», конечный IP-адрес, адрес 10.0.0.1, исходный порт «любая», конечный порт 80, а также протокол «TCP». Затем policy_C и policy_D соответствует подключений к 10.0.0.1:80 назначения. Поскольку политику с наиболее конкретные условия сопоставления применяется политика QoS, policy_D приоритет в этом примере.  
  
Однако политики качества обслуживания могут иметь одинаковое число условий. Например, несколько политик может каждый из которых задает только один (но не совпадают) часть quintuple сети. Среди quintuple сети указанном ниже порядке — от вышестоящего уровня на более низкий приоритет:

- Исходный IP-адрес

- IP-адрес назначения

- Исходный порт

- Порт назначения

- Протокол (TCP или UDP).

В определенное условие, например IP-адрес рассматривается более конкретные IP-адрес с более высоким приоритетом; например IP-адресом 192.168.4.1 точнее, чем 192.168.4.0/24.

Разработка политик качества обслуживания как можно точнее для упрощения организации возможность понять, какие политики вступают в силу.
