# <a name="qos-policy-frequently-asked-questions"></a>Вопросы и ответы по политики QoS

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

Следующие часто задаваемые вопросы — эти вопросы и ответы на эти вопросы — для политики QoS.
  
1.  **Какая операционная система контроллер домена должна работать использовать политики качества обслуживания?**
  
     В Windows Server 2016, Windows Server 2012 R2, Windows Server 2012, Windows Server 2008 R2 или Windows Server 2008

2.  **Какие операционные системы поддерживают применения политики QoS к пользователю или компьютеру?**

     Можно применить политики QoS к пользователям или компьютерам под управлением Windows Server 2016, Windows 10, Windows Server 2012 R2, Windows 8.1, Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows Server 2008 и Windows Vista.

3.  **Применяются ли политики качества обслуживания для отправителя и получателя трафика?**

     Необходимо применить политики качества обслуживания на исходном компьютере повлияет на его исходящего трафика. Чтобы повлиять на двунаправленный трафик двух компьютеров, политики QoS должны применяться для обоих компьютерах.

4.  **Что произойдет, если конфликтующих политик QoS развертываются на одном компьютере?**  
  
     Если применить несколько политик, конкретная политика QoS имеет приоритет. Например, политики, состояний узлом адресов (192.168.4.12) применяется вместо менее конкретным сетевой адрес (192.168.0.0/16). Если же спецификой политику уровня пользователя и компьютера, вместо политики QoS на уровне компьютера применяется политика QoS уровне пользователя. 

5.  **Политики качества обслуживания, включенный по умолчанию?**

     Нет, политики QoS не включена по умолчанию. Необходимо создать политики качества обслуживания вручную, чтобы настроить службу качества обслуживания.  Дополнительные сведения см. в разделе [управление политики QoS](qos-policy-manage.md).

Первый раздел в данном руководстве, см. [политики качества обслуживания (QoS)](qos-policy-top.md).
