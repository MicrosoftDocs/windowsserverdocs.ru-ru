# <a name="configure-firewalls-for-radius-traffic"></a>Настройка брандмауэров для RADIUS-трафика

>Применяется к: Windows Server (Semi-Annual Channel), Windows Server 2016

Брандмауэры можно настроить для разрешения или блокировки типов IP-трафика на компьютере или устройстве, на котором работает брандмауэр. Если брандмауэры не настроены должным образом, чтобы разрешить трафик RADIUS между клиентами RADIUS, RADIUS-прокси и серверами RADIUS, проверка подлинности доступа к сети может завершиться сбоем, предотвращая доступ пользователей к сетевым ресурсам.

Может потребоваться настроить два типа брандмауэров, чтобы разрешить трафик RADIUS.

- Брандмауэр защитника Windows в повышенной безопасности на локальном сервере, на котором выполняется сервер политики сети (NPS).
- Брандмауэры, работающие на других компьютерах или устройствах.

## <a name="windows-firewall-on-the-local-nps"></a>Брандмауэр Windows в локальном NPS

По умолчанию сервер политики сети отправляет и получает трафик RADIUS с помощью \( UDP- \) портов 1812, 1813, 1645 и 1646 протокола Datagram. Брандмауэр защитника Windows на сервере политики сети автоматически настраивается с исключениями во время установки NPS, чтобы разрешить отправку и получение трафика RADIUS.

Поэтому, если используются UDP-порты по умолчанию, не нужно изменять конфигурацию брандмауэра защитника Windows, чтобы разрешить трафик RADIUS в НПСС и обратно.

В некоторых случаях может потребоваться изменить порты, используемые NPS для трафика RADIUS. Если сервер политики сети и серверы доступа к сети настроены для отправки и получения RADIUS-трафика на портах, отличных от параметров по умолчанию, необходимо выполнить следующие действия.

- Удалите исключения, которые разрешают трафик RADIUS на портах по умолчанию.
- Создайте новые исключения, разрешающие трафик RADIUS на новых портах.

Дополнительные сведения см. в разделе [Настройка данных UDP-портов NPS](nps-udp-ports-configure.md).

## <a name="other-firewalls"></a>Другие брандмауэры

В наиболее распространенной конфигурации брандмауэр подключен к Интернету, а NPS — это ресурс интрасети, подключенный к сети периметра.

Для доступа к контроллеру домена в интрасети сервер политики сети может иметь следующее:

- Интерфейс в сети периметра и интерфейс в интрасети (IP-маршрутизация не включена).
- Единый интерфейс в сети периметра. В этой конфигурации NPS взаимодействует с контроллерами домена через другой брандмауэр, который подключает сеть периметра к интрасети.

## <a name="configuring-the-internet-firewall"></a>Настройка брандмауэра Интернета

Брандмауэр, подключенный к Интернету, должен быть настроен с помощью входных и выходных фильтров в интерфейсе Интернета \( и (необязательно) сетевого периметра сети \) , чтобы разрешить ПЕРЕНАПРАВЛЕНИЕ RADIUS-сообщений между сервером политики сети и клиентами RADIUS или прокси-серверами в Интернете. Можно использовать дополнительные фильтры, чтобы разрешить передачу трафика на веб-серверы, VPN-серверы и другие типы серверов в сети периметра.

Отдельные фильтры входящих и исходящих пакетов можно настроить для интерфейса Интернета и интерфейса сети периметра.

### <a name="configure-input-filters-on-the-internet-interface"></a>Настройка фильтров входа для интерфейса Интернета

Настройте следующие фильтры входящих пакетов в интерфейсе Интернета брандмауэра, чтобы разрешить следующие типы трафика:

- Конечный IP-адрес интерфейса сети периметра и порт назначения UDP 1812 (0x714) NPS.  Этот фильтр разрешает трафик проверки подлинности RADIUS от клиентов RADIUS на основе Интернета к NPS. Это UDP-порт по умолчанию, используемый NPS, как определено в RFC 2865. Если вы используете другой порт, замените номер этого порта 1812.
- Конечный IP-адрес интерфейса сети периметра и порт назначения UDP 1813 (0x715) NPS. Этот фильтр разрешает RADIUS-трафик учета от клиентов RADIUS на основе Интернета к NPS. Это UDP-порт по умолчанию, используемый NPS, как определено в RFC 2866. Если вы используете другой порт, замените номер этого порта 1813.
- \(Необязательный \) конечный IP-адрес интерфейса периметра сети и порт назначения UDP 1645 \( 0x66D \) сервера политики сети. Этот фильтр разрешает трафик проверки подлинности RADIUS от клиентов RADIUS на основе Интернета к NPS. Это UDP-порт, используемый старыми клиентами RADIUS.
- \(Необязательный \) конечный IP-адрес интерфейса периметра сети и порт назначения UDP 1646 \( 0x66E \) сервера политики сети. Этот фильтр разрешает RADIUS-трафик учета от клиентов RADIUS на основе Интернета к NPS. Это UDP-порт, используемый старыми клиентами RADIUS.

Настройте следующие фильтры вывода в интерфейсе Интернета брандмауэра, чтобы разрешить следующие типы трафика:

- \(Необязательный \) Исходный IP-адрес интерфейса периметра сети и UDP-порт источника 1645 \( 0x66D \) сервера политики сети. Этот фильтр разрешает трафик проверки подлинности RADIUS от NPS к клиентам RADIUS, основанным на Интернете. Это UDP-порт, используемый старыми клиентами RADIUS.
- \(Необязательный \) Исходный IP-адрес интерфейса периметра сети и UDP-порт источника 1646 \( 0x66E \) сервера политики сети. Этот фильтр разрешает RADIUS-трафик с сервера политики сети к клиентам RADIUS на основе Интернета. Это UDP-порт, используемый старыми клиентами RADIUS.
