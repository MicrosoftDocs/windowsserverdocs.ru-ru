# <a name="configure-firewalls-for-radius-traffic"></a>Настройка брандмауэров для RADIUS-трафика

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Брандмауэры можно настроить для разрешения или запрета типы IP-трафика в и из компьютера или устройства, на котором работает брандмауэр. Если брандмауэры настроены правильно для разрешения трафика RADIUS между клиентами RADIUS, прокси-серверы RADIUS и серверами RADIUS, проверка подлинности сетевого доступа может завершиться ошибкой, предотвращая доступ к сетевым ресурсам. 

Может потребоваться настроить брандмауэры так, чтобы разрешить трафик RADIUS два типа:

- Брандмауэр Защитника Windows с повышенной безопасности на локальном сервере под управлением сервера политики сети (NPS).
- Брандмауэры, выполняемые на других компьютерах или устройствах.

## <a name="windows-firewall-on-the-local-nps"></a>Брандмауэр Windows на локальном сервере политики сети

По умолчанию NPS, отправляет и получает RADIUS-трафика с помощью протокол UDP \(UDP\) портах 1812, 1813, 1645 и 1646. Брандмауэр Защитника Windows на NPS автоматически настраивается с исключениями, во время установки сервера политики сети, чтобы разрешить этот трафик RADIUS для отправки и получения.

Таким образом Если вы используете UDP-порты по умолчанию, вы не обязательно должны изменить конфигурацию брандмауэра Защитника Windows, чтобы разрешить трафик RADIUS от NPSs.

В некоторых случаях может потребоваться изменить порты, используемые сервером политики сети для RADIUS-трафика. Если настроить сервер политики сети и серверы доступа к сети для отправки и получения RADIUS-трафик на порты, отличные от значения по умолчанию, выполните следующие действия.

- Удалите исключения, которые позволяют RADIUS-трафик на порты по умолчанию.
- Создаете новые исключения, разрешающие трафик RADIUS на новых портов.

Дополнительные сведения см. в разделе [сведения о порте UDP NPS Настройка](nps-udp-ports-configure.md).

## <a name="other-firewalls"></a>Другие брандмауэры

В наиболее распространенные конфигурации брандмауэр подключения к Интернету и сервером политики сети — это ресурс интрасети, который подключен к сети периметра.

Чтобы получить доступ к контроллеру домена в интрасети, возможно, сервер политики сети:

- Интерфейс в сети периметра и интерфейс в интрасети (IP-маршрутизация не включена). 
- Единый интерфейс локальной сети. В этой конфигурации сервер политики сети взаимодействует с контроллерами домена через другой брандмауэр, который подключается к интрасети сети периметра.

## <a name="configuring-the-internet-firewall"></a>Настройка брандмауэра Интернета

Брандмауэр, подключенный к Интернету должны быть настроены входные и выходные фильтры для интерфейса Интернета \(и, при необходимости интерфейсом локальной сети\), чтобы разрешить перенаправление RADIUS между сервером политики сети и RADIUS-клиентами или прокси-серверов в Интернете. Чтобы разрешить передачу трафика на веб-серверы, VPN-серверов и других типов серверов в сети периметра можно использовать дополнительные фильтры.

Отдельные входные и выходные пакетов можно настроить фильтры на веб-интерфейс и интерфейс сети периметра.

### <a name="configure-input-filters-on-the-internet-interface"></a>Настройка фильтров входа на веб-интерфейс

Настройте следующие фильтры входящих пакетов на веб-интерфейсу брандмауэр так, чтобы разрешить следующие типы трафика.

- IP-адрес назначения интерфейса локальной сети и UDP-порт назначения 1812 (0x714) сервера политики сети.  Этот фильтр допускает проверку подлинности RADIUS-трафика от клиентов RADIUS, Интернет клиентов в службу NPS. Это UDP-порт по умолчанию, который используется сервером политики сети, как определено в RFC 2865. Если вы используете другой порт, замените этот номер порта 1812.
- IP-адрес назначения интерфейса локальной сети и конечный порт UDP 1813 (0x715) сервера политики сети. Этот фильтр допускает учета RADIUS-трафика от клиентов RADIUS, Интернет клиентов в службу NPS. Это UDP-порт по умолчанию, который используется сервером политики сети, как определено в RFC 2866. Если вы используете другой порт, замените этот номер порта для 1813.
- \(Необязательный\) конечный IP-адрес интерфейса локальной сети и UDP-порт назначения 1645 \(0x66D\) сервера политики сети. Этот фильтр допускает проверку подлинности RADIUS-трафика от клиентов RADIUS, Интернет клиентов в службу NPS. Это UDP-порт, используемый для старых клиентов RADIUS.
- \(Необязательный\) конечный IP-адрес интерфейса локальной сети и UDP-порт назначения 1646 \(0x66E\) сервера политики сети. Этот фильтр допускает учета RADIUS-трафика от клиентов RADIUS, Интернет клиентов в службу NPS. Это UDP-порт, используемый для старых клиентов RADIUS.

- Исходный IP-адрес интерфейса локальной сети и исходный порт UDP 1813 (0x715) сервера политики сети. Этот фильтр допускает передачу трафика учета RADIUS с сервером политики сети для Интернет клиентов RADIUS-клиенты. - \(Необязательный\) исходный IP-адрес интерфейса локальной сети и UDP-порт источника 1646 \(0x66E\) сервера политики сети. Этот фильтр допускает передачу трафика учета RADIUS с сервером политики сети для Интернет клиентов RADIUS-клиенты. Это UDP-порт, используемый для старых клиентов RADIUS.
