# <a name="network-policy-server-management-with-administration-tools"></a>Управление сервером политики сети с помощью средств администрирования

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

В этом разделе можно использовать, чтобы узнать о средствах, которые можно использовать для управления вашей NPSs.

После установки сервера политики сети, могут администрировать NPSs:

- Локально, с помощью консоли управления MMC NPS \(MMC\) оснастки, статические консоли сервера политики сети в инструментах администрирования, команды Windows PowerShell или Network Shell \(Netsh\) команд для сервера политики сети.
- От удаленного сервера политики сети, с помощью оснастки NPS MMC, команды Netsh для сервера политики сети, команды Windows PowerShell для сервера политики сети или подключения удаленного рабочего стола.
- Из удаленной рабочей станции, с помощью удаленного рабочего стола в сочетании с другими средствами, например NPS MMC или Windows PowerShell.

>[!NOTE]
>В Windows Server 2016 локальный NPS можно управлять с помощью консоли сервера политики сети. Для управления удаленных и локальных NPSs, необходимо использовать оснастку NPS MMC\-в.

Следующие разделы содержат инструкции по управлению NPSs ваши локальные и удаленные.

## <a name="configure-the-local-nps-by-using-the-nps-console"></a>Настройка локального сервера политики сети с помощью консоли сервера политики сети

После установки сервера политики сети, эту процедуру можно использовать для управления локальной политики сети с помощью NPS MMC.

**Учетные данные администратора** 

Для выполнения этой процедуры необходимо быть членом группы "Администраторы".

### <a name="to-configure-the-local-nps-by-using-the-nps-console"></a>Чтобы настроить локальный сервер политики сети с помощью консоли сервера политики сети

1. В диспетчере серверов щелкните **средства**, а затем нажмите кнопку **сервера политики сети**. Откроется консоль NPS.

2. В консоли сервера политики сети щелкните сервер политики сети \(локального\). В области сведений выберите **стандартной конфигурации** или **Расширенная конфигурация**, а затем выполните одно из указанных ниже, в зависимости от выбора:
    - Если вы выберете **стандартной конфигурации**, выберите сценарий из списка и следуйте инструкциям, чтобы запустить мастер настройки.
    - При выборе **Расширенная конфигурация**, щелкните стрелку, чтобы развернуть **параметров конфигурацией**, а затем просмотреть и настроить доступные параметры в зависимости от функциональности сервера политики сети — Сервер RADIUS, прокси-сервера RADIUS или оба.

## <a name="manage-multiple-npss-by-using-the-nps-mmc-snap-in"></a>Управление несколькими NPSs с помощью оснастки NPS MMC\-в

Эту процедуру можно использовать для управления с помощью оснастки NPS MMC локальный NPS и несколько удаленных NPSs\-в.

Прежде чем выполнять описанную ниже процедуру, необходимо установить сервер политики сети на локальном компьютере и на удаленных компьютерах.

В зависимости от состояния сети и количество NPSs вы управляете с помощью оснастки NPS MMC\-в ответ на оснастки MMC\-в может снижаться. Кроме того, сервер политики сети конфигурации трафик отправляется по сети во время сеанса удаленного управления с помощью оснастки NPS\-в. Убедитесь, что сети физически защищен и что пользователи-злоумышленники не имеют доступа к этой сетевой трафик.

**Учетные данные администратора** 

Для выполнения этой процедуры необходимо быть членом группы "Администраторы".

### <a name="to-manage-multiple-npss-by-using-the-nps-snap-in"></a>Для управления несколькими NPSs с помощью оснастки NPS\-в

1. Чтобы открыть консоль Управления, запустите Windows PowerShell с правами администратора. В Windows PowerShell, введите **mmc**, и нажмите клавишу ВВОД. Откроется консоль управления (MMC).
2. В MMC на **файл** меню, щелкните **привязать добавить или удалить\-в**. **Добавление или удаление оснасток\-ins** откроется диалоговое окно.
3. В **Добавление или удаление оснасток\-модули**в **Доступные оснастки\-модули**, прокрутите список вниз, щелкните **сервера политики сети**, а затем нажмите кнопку  **Добавление**. Откроется **Выбор компьютера**.
4. В **Выбор компьютера**, убедитесь, что **локального компьютера \(компьютера, на котором запущена эта консоль\)**  выбран, а затем нажмите кнопку **ОК**. Привязать\-в для локального сервера политики сети добавляется в список в **Выбранные оснастки\-ins**.
5. В **Добавление или удаление оснасток\-ins**в **Доступные оснастки\-ins**, убедитесь, что **сервера политики сети** по-прежнему выбран, а затем нажмите кнопку  **Добавление**. **Выбор компьютера** снова откроется диалоговое окно.
6. В **Выбор компьютера**, нажмите кнопку **другой компьютер**, а затем введите IP-адрес или полное доменное имя \(полное доменное имя\) удаленного сервера политики сети, который вы хотите управлять с помощью сервера политики сети привязать\-в. При необходимости можно щелкнуть **Обзор** для выскажите свое каталоге компьютера, на котором вы хотите добавить. Нажмите кнопку **ОК**.
7. Повторите шаги 5 и 6, чтобы добавить дополнительные NPSs оснастки NPS\-в. После добавления всех NPSs, вы хотите управлять, щелкните **ОК**.
8. Чтобы сохранить оснастку сервера политики сети для последующего использования, нажмите кнопку **файл**, а затем нажмите кнопку **Сохранить**. В **Сохранить как** диалоговом окне перейдите в папку на жестком диске, где требуется сохранить файл, введите имя для вашей консоли управления \(.msc\) файла и нажмите кнопку **Сохранить**. 

## <a name="manage-an-nps-by-using-remote-desktop-connection"></a>Управлять сервер политики сети с помощью удаленного рабочего стола

Эту процедуру можно использовать для управления удаленный сервер политики сети с помощью удаленного рабочего стола.

С помощью удаленного рабочего стола, можно удаленно управлять вашей NPSs под управлением Windows Server 2016. Можно также удаленно управлять NPSs на компьютере под управлением Windows 10 или более ранних операционных систем Windows клиента.

Можно использовать подключения удаленного рабочего стола для управления несколькими NPSs с помощью одного из двух методов.

1. Создайте подключение удаленного рабочего стола к каждому из вашей NPSs по отдельности.
2. Использование удаленного рабочего стола для подключения к один сервер политики сети и затем управлять другими удаленными серверами с помощью консоли Управления сервера политики сети на этом сервере. Дополнительные сведения см. в предыдущем разделе **управление несколькими NPSs с помощью MMC NPS, привязать\-в**.

**Учетные данные администратора** 

Для выполнения этой процедуры необходимо быть членом группы администраторов на сервере политики сети.

### <a name="to-manage-an-nps-by-using-remote-desktop-connection"></a>Для управления сервер политики сети с помощью удаленного рабочего стола

1. Для каждого сервера политики сети, который вы хотите управлять удаленно, в диспетчере серверов выберите **локального сервера**. В области сведений диспетчера серверов, просмотр **удаленного рабочего стола** параметр и выполните одно из следующих действий. 
    1. Если значение **удаленного рабочего стола** параметр **включено**, необходимо выполнить некоторые действия, описанные в этой процедуре. Перейдите к шагу 4, чтобы приступить к настройке разрешений пользователя удаленного рабочего стола.
    2. Если **удаленного рабочего стола** параметр **отключено**, щелкните слово **отключено**. **Системные свойства** откроется диалоговое окно на **удаленного** вкладки.
2. В **удаленного рабочего стола**, нажмите кнопку **разрешить удаленные подключения к этому компьютеру**. **Удаленный рабочий стол** откроется диалоговое окно. Выполните одно из указанных ниже действий.
    1. Чтобы настроить сетевые подключения, которые разрешено, нажмите кнопку **брандмауэр Windows в режиме повышенной безопасности**, а затем настройте параметры, которые вы хотите разрешить. 
    2. Чтобы включить удаленный рабочий стол для всех сетевых подключений на компьютере, щелкните **ОК**.
3. В **системные свойства**в **удаленного рабочего стола**, решить, следует ли включить **разрешить подключения только от компьютеров с удаленным рабочим столом с сетевой проверкой подлинности**и выбрать нужный.
4. Щелкните **Выбрать пользователей**. **Пользователи удаленного рабочего стола** откроется диалоговое окно.
5. В **пользователи удаленного рабочего стола**, чтобы предоставить разрешение пользователю для удаленного подключения в NPS, щелкните **добавить**, а затем введите имя пользователя для учетной записи пользователя. Нажмите кнопку **ОК**.
6. Повторите шаг 5 для каждого пользователя, для которого вы хотите предоставить разрешение удаленного доступа в службу NPS. По завершении добавления пользователей нажмите кнопку **ОК** закрыть **пользователи удаленного рабочего стола** диалоговое окно и **ОК** еще раз, чтобы закрыть **системные свойства**диалоговое окно.
7. Чтобы подключиться к удаленный сервер политики сети, настроенные с помощью ранее, нажмите кнопку **запустить**, прокрутите вниз список и нажмите кнопку **стандартные Windows**и нажмите кнопку **удаленного Рабочий стол**. **Удаленный рабочий стол** откроется диалоговое окно.
8. В **удаленный рабочий стол** отображаемое в диалоговом окне **компьютера**, введите имя сервера политики сети или IP-адрес. Если вы предпочитаете, нажмите кнопку **параметры**, настройте дополнительные параметры соединения и нажмите кнопку **Сохранить** сохранить соединение для повторного использования.
9. Нажмите кнопку **Connect**и при появлении запроса укажите учетные данные пользователя для учетной записи с разрешениями, которые входят в систему и настроить сервер политики сети.

## <a name="use-netsh-nps-commands-to-manage-an-nps"></a>Использовать команды Netsh NPS для управления сервер политики сети

Команды в контексте Netsh NPS используются для отображения и изменения параметров проверки подлинности, авторизации, учет и аудит базы данных, используемых средствами NPS и службы удаленного доступа. Использование команд в контексте Netsh NPS, чтобы:

- Для настройки или изменения сервер политики сети, включая все аспекты NPS, которые также доступны для конфигурации с помощью консоли сервера политики сети в интерфейсе Windows.
- Экспорт конфигурации одного сервера политики сети (исходного сервера), включая разделы реестра и конфигурацию сервера политики сети хранилища, как сценарий Netsh.
- Импортируйте конфигурацию на другой сервер политики сети с помощью сценария Netsh и файл конфигурации из источника NPS.

Эти команды запускаются из Windows Server 2016 командной строки или с помощью Windows PowerShell. Можно также запустить команды netsh nps в сценарии и пакетные файлы.

**Учетные данные администратора** 

Выполнить эту процедуру может только член группы "Администраторы" на локальном компьютере.

### <a name="to-enter-the-netsh-nps-context-on-an-nps"></a>Чтобы задать контекст Netsh NPS на сервер политики сети

1. Откройте командную строку или Windows PowerShell.
2. Тип **netsh**, и нажмите клавишу ВВОД.
3. Тип **nps**, и нажмите клавишу ВВОД.
4. Чтобы просмотреть список доступных команд, введите знак вопроса \(?\) и нажмите клавишу ВВОД.


Дополнительные сведения о командах Netsh NPS см. в разделе [команды Netsh для сервера политики сети в Windows Server 2008](https://technet.microsoft.com/library/cc754428(v=ws.10).aspx), или загрузить весь [Технический справочник по Netsh](https://gallery.technet.microsoft.com/Netsh-Technical-Reference-c46523dc?redir=0) из коллекции TechNet. Данный файл является полной сетевой оболочки Технический справочник по для Windows Server 2008 и Windows Server 2008 R2. Справка Windows имеет формат \(*.chm\) в ZIP-файл. Эти команды по-прежнему присутствуют в Windows Server 2016 и Windows 10, чтобы можно было использовать netsh в этих средах, несмотря на то, что рекомендуется использовать Windows PowerShell.
