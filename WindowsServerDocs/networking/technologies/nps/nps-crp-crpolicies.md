# <a name="connection-request-policies"></a>Политики запросов на подключение

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе можно использовать, чтобы узнать, как использовать политики запросов на подключение сервера политики сети для настройки сервера политики сети как RADIUS-сервер, RADIUS-прокси или оба.

>[!NOTE]
>Дополнение к данному разделу доступна следующая документация политики запроса подключения.
> - [Настройка политик запросов на подключение](nps-crp-configure.md)
> - [Настройка групп внешних RADIUS-сервера](nps-crp-rrsg-configure.md)

Политики запросов на подключение — это наборы условий и параметров, которые позволяют администраторам сети назначать серверы службы проверки подлинности удаленного пользователя (RADIUS) для выполнения проверки подлинности и авторизации запросов на подключение, сервер под управлением сервера политики сети (NPS), получает от клиентов RADIUS. Чтобы назначить RADIUS-серверы используются для RADIUS-учет можно настроить политики запросов на подключение.

Вы можете создать подключение политики запросов, чтобы локально обрабатываются часть запросов RADIUS, полученных от клиентов RADIUS (сервер политики сети как RADIUS-сервер используется) и другие типы сообщений будут перенаправлены на другой сервер RADIUS (сервер политики сети используется в качестве RADIUS-прокси).

Политики запросов на подключение можно использовать сервер политики сети как RADIUS-сервер или как RADIUS-прокси, в зависимости от следующих факторов: 

- Время дня и дня недели
- Имя области в запросе на подключение
- Тип запрашиваемой подключения
- IP-адрес клиента RADIUS

Запросов доступа RADIUS обрабатываются или перенаправляются сервером политики сети только в том случае, если параметры входящего сообщения соответствовать хотя бы один из политики запросов на подключение, настроенных на сервере политики сети. 

Если параметры политики и политики требует, что сервер NPS обрабатывать сообщения, сервер политики сети выступает в качестве сервера RADIUS, проверки подлинности и авторизацию запроса на подключение. Если параметры политики и политики требует, что на NPS-сервер перенаправляет сообщение, сервер политики сети выступает в качестве RADIUS-прокси и переадресует запрос на подключение к удаленному серверу RADIUS для обработки.

Если параметры входящего сообщения запроса доступа RADIUS не совпадают по крайней мере один из политики запросов на подключение, отправляется сообщение отказа в доступе RADIUS-клиента и пользователя или компьютера, пытающегося подключиться к сети отказано в доступе.

## <a name="configuration-examples"></a>Примеры конфигурации

В следующих примерах конфигурации показано, как можно использовать политики запросов на подключение.

### <a name="nps-as-a-radius-server"></a>Сервер политики сети как RADIUS-сервер

Политики запросов на подключение по умолчанию является единственной настроенной политикой. В этом примере сервер политики сети настроен как RADIUS-сервер и все запросы на подключение обрабатываются локальным сервером политики сети. Сервер политики сети можно проверки подлинности и авторизации пользователей, учетные записи в домене сервера, а также в доверенных доменах.


### <a name="nps-as-a-radius-proxy"></a>Сервер политики сети как RADIUS-прокси

Удалить политики запросов на подключение по умолчанию, и для пересылки запросов в два других домена создаются две новые политики запросов на подключение. В этом примере сервер политики сети настроен в качестве RADIUS-прокси. Сервер политики сети не обрабатывает запросы на подключение на локальном сервере. Вместо этого он перенаправляет запросы на подключение сервера политики сети или другие RADIUS-серверы, настроенные в качестве членов удаленных групп серверов RADIUS.


### <a name="nps-as-both-radius-server-and-radius-proxy"></a>Сервер политики сети в качестве сервера RADIUS и прокси-сервера RADIUS

Помимо политики запроса подключения по умолчанию создается политики запросов на подключение, который перенаправляет запросы на подключение, сервер политики сети или другого сервера RADIUS в недоверенном домене. В этом примере политика прокси отображается первой в списке политик. Если запрос на подключение соответствует политике прокси, запрос на подключение перенаправляется RADIUS-сервер в группе удаленных RADIUS-серверов. Если запрос на подключение не соответствует политике прокси, но соответствует политики запросов на подключение по умолчанию, сервер политики сети обрабатывает запросы на подключение на локальном сервере. Если запрос на подключение не соответствует либо политики, они удаляются.

### <a name="nps-as-radius-server-with-remote-accounting-servers"></a>Сервер политики сети как RADIUS-сервера с удаленными серверами учета

В этом примере локальный сервер политики сети не настроен на ведение учета и политики запросов на подключение по умолчанию изменена таким образом, чтобы сервер политики сети или другие RADIUS-сервер в группе удаленных RADIUS-серверов пересылки сообщений учета RADIUS. Несмотря на то, что сообщения учета перенаправляются, не будут перенаправлены сообщения проверки подлинности и авторизации и локальный сервер политики сети выполняет следующие функции для локального домена и всех доверенных доменов.


### <a name="nps-with-remote-radius-to-windows-user-mapping"></a>Сервер политики сети с использованием удаленного RADIUS для сопоставления пользователя Windows

В этом примере NPS действует как сервер RADIUS, а как RADIUS-прокси для каждого отдельного запроса на подключение, перенаправляя запрос на проверку подлинности на удаленном сервере RADIUS, используя локальную учетную запись пользователя Windows для авторизации. Эта конфигурация реализуется путем настройки удаленных RADIUS для сопоставления пользователей Windows атрибута в качестве условия политики запросов на подключение. (Кроме того, учетной записи пользователя должны быть созданы локально, совпадает с именем учетной записи удаленного пользователя, по которой будет выполняться проверка подлинности удаленных RADIUS-сервером.)

## <a name="connection-request-policy-conditions"></a>Условия политики запросов на подключение

Условия политики запросов на подключение, один или несколько атрибутов RADIUS, которые сравниваются с атрибутами входящие сообщения запроса доступа RADIUS. Если несколько условий, все условия в подключении запросить сообщение и в запрос на подключение политика должна соответствовать политика принудительного применения на сервере политики сети.

Ниже перечислены доступные атрибуты условий, можно настроить политики запросов на подключение.

### <a name="connection-properties-attribute-group"></a>Группа атрибутов свойства подключения

Группа атрибутов свойства подключения содержит следующие атрибуты.

- **Протокол кадрирования**. Используется для назначения типа кадрирования входящих пакетов. Примерами являются протокол PPP (Point-to-Point), последовательного строки Internet протокол SLIP (), Frame Relay и X.25.
- **Тип службы**. Используется для назначения тип запрашиваемой службы. Примеры рамке (например, подключений по протоколу PPP) и войдите в систему (например, Telnet-подключения). Дополнительные сведения о типах служб RADIUS см «Удаленного служба проверки подлинности удаленного пользователя (RADIUS).»
- **Тип туннеля**. Используется для назначения типа туннеля, который создается клиентом. Типы туннелирования включить протокол PPTP (Point-to-Point Tunneling) и протокол (L2TP).

### <a name="day-and-time-restrictions-attribute-group"></a>Группа атрибутов день и ограничения по времени 

Группа атрибутов день и ограничения по времени содержит атрибут день и ограничения по времени. С помощью этого атрибута можно назначить день недели и время суток, попытки подключения. День и время задается относительно день и время сервера политики сети.

### <a name="gateway-attribute-group"></a>Группа атрибутов шлюза

Группа атрибутов шлюза содержит следующие атрибуты.

- **Идентификатор вызываемой станции**. Используется для назначения номер телефона сервера доступа к сети. Этот атрибут представляет собой строку символов. Для задания области кодов можно использовать синтаксис соответствия шаблону.
- **Идентификатор NAS**. Используется для назначения имени сервера доступа к сети. Этот атрибут представляет собой строку символов. Для задания кодов NAS можно использовать синтаксис соответствия шаблону.
- **IPv4-адрес NAS**. Для обозначения протокола IP версии 4 (IPv4) адрес сервера доступа к сети (клиент RADIUS). Этот атрибут представляет собой строку символов. Чтобы указать IP-сетей, можно использовать синтаксис соответствия шаблону.
- **IPv6-адрес NAS**. Для обозначения протокола IP версии 6 (IPv6) адрес сервера доступа к сети (клиент RADIUS). Этот атрибут представляет собой строку символов. Чтобы указать IP-сетей, можно использовать синтаксис соответствия шаблону.
- **Тип порта NAS**. Используется для назначения тип носителя, используемого клиентом доступа. Примерами являются аналоговые телефонные линии (известных как async), цифровой сети ISDN (Integrated Services), туннели или виртуальные частные сети (VPN), беспроводные сети, IEEE 802.11 и коммутаторы Ethernet.

### <a name="machine-identity-attribute-group"></a>Группа атрибутов удостоверение компьютера

Группа атрибутов идентификатор машины содержит идентификатор машины атрибута. С помощью этого атрибута, можно указать метод, с которой клиенты идентифицируются в политике.

### <a name="radius-client-properties-attribute-group"></a>Группа атрибутов свойства клиента RADIUS

Группа атрибутов свойства RADIUS-клиента содержит следующие атрибуты.

- **Код вызывающей станции**. Используется для назначения номер телефона, используемые вызывающим (клиента). Этот атрибут представляет собой строку символов. Для задания области кодов можно использовать синтаксис соответствия шаблону.
- **Понятное имя клиента**. Используется для назначения имени компьютера RADIUS-клиента, запрашивающего проверку подлинности. Этот атрибут представляет собой строку символов. Для указания имен клиентов можно использовать синтаксис соответствия шаблону.
- **IPv4-адрес клиента**. Используется для назначения IPv4-адрес сервера доступа к сети (клиент RADIUS). Этот атрибут представляет собой строку символов. Чтобы указать IP-сетей, можно использовать синтаксис соответствия шаблону.
- **IPv6-адрес клиента**. Используется для назначения IPv6-адрес сервера доступа к сети (клиент RADIUS). Этот атрибут представляет собой строку символов. Чтобы указать IP-сетей, можно использовать синтаксис соответствия шаблону.
- **Клиент поставщика**. Используется для назначения поставщика сервера доступа к сети, который запрашивает проверку подлинности. На компьютере под управлением службы маршрутизации и удаленного доступа является поставщиком Microsoft NAS. Этот атрибут можно использовать для настройки отдельных политик для различных поставщиков NAS. Этот атрибут представляет собой строку символов. Можно использовать синтаксис соответствия шаблону.

### <a name="user-name-attribute-group"></a>Группа атрибутов имени пользователя

Группа атрибутов имя пользователя содержит атрибут имени пользователя. С помощью этого атрибута, можно назначить имя пользователя или часть имени пользователя, которое должно совпадать с именем пользователя, предоставленный клиентом доступа, в сообщении RADIUS. Этот атрибут представляет строку символов, который обычно содержит имя сферы и имя учетной записи пользователя. Чтобы указать имена пользователей, можно использовать синтаксис соответствия шаблону.

## <a name="connection-request-policy-settings"></a>Параметры политики запросов на подключение

Параметры политики запросов на подключение представляют собой набор свойств, которые применяются с входящим сообщением RADIUS. Параметры состоят из следующих групп параметров.

- Проверка подлинности
- Бухгалтерский учет
- Управление атрибутами
- Перенаправление запроса
- Дополнительно

В следующих разделах приводятся дополнительные сведения об этих параметрах.

### <a name="authentication"></a>Проверка подлинности

С помощью этого параметра, можно переопределить параметры проверки подлинности, настроенные в все сетевые политики и можно назначить методы проверки подлинности и типов, которые необходимы для подключения к сети.

>[!IMPORTANT]
>При настройке метода проверки подлинности в политике запросов на подключение, которое менее надежна, чем метод проверки подлинности, которые можно настроить в политике сети переопределяется более безопасный метод проверки подлинности, можно настроить в политике сети. Например, при наличии одной политики сети, требуется использование защищенных расширяемой проверки подлинности протокол Microsoft запрос проверки подлинности протоколом MSCHAP версии 2 \ (PEAP-MS-CHAP v2\), способ проверки подлинности на основе пароля для защищенных беспроводных сетей, а также настроить политики запросов на подключение, чтобы разрешить доступ без проверки подлинности получается, что клиенты не требуются для проверки подлинности с помощью протокола PEAP-MS-CHAP v2. В этом примере все клиенты, подключающиеся к сети предоставлен доступ без проверки подлинности.

### <a name="accounting"></a>Бухгалтерский учет

С помощью этого параметра, можно настроить политики запросов на подключение для пересылки данных учета в сервер политики сети или другие RADIUS-сервер в группе удаленных RADIUS-серверов, чтобы учета выполняет группы внешних серверов RADIUS.

>[!NOTE]
>Если имеется несколько серверов RADIUS и учет сведения для всех серверов, которые хранятся в одной централизованной базе данных учета RADIUS требуется, можно использовать параметр учета политики запроса подключения в политике, на каждом сервере RADIUS переадресацию данных учета из всех серверов на один сервер политики сети или другого сервера RADIUS, которые указаны в качестве сервера учетных записей.

Параметры учета политики запросов на подключение работать независимо от конфигурации учета локального сервера политики сети. Другими словами при настройке локального сервера политики сети в журнал данных учета RADIUS в локальный файл или базы данных Microsoft SQL Server будет поступать независимо от того, является ли настроить политики запросов на подключение на пересылку сообщений учета группы внешних серверов RADIUS.

Если вы хотите войти в систему удаленно, но не локально сведения об учетных данных, необходимо настроить локальный сервер политики сети не вести учет, также в при настройке учета политики запросов на подключение к переадресацию данных учета для группы внешних серверов RADIUS.

### <a name="attribute-manipulation"></a>Управление атрибутами

Можно настроить набор правил поиска и замены, работы с текстовыми строками одного из следующих атрибутов.

- Имя пользователя
- Идентификатор вызываемой станции
- Код вызывающей станции

Обработка правил поиска и замены возникает по одной из предыдущей атрибуты перед RADIUS сообщения зависит от параметров проверки подлинности и учета. Правила работы с атрибутами применяются только для одного атрибута. Не удается настроить правила работы с атрибутами для каждого атрибута. Кроме того список атрибутов, которыми можно управлять — статический список; не удается добавить к списку атрибуты для обработки.

>[!NOTE]
>При использовании протокола проверки подлинности MS-CHAP v2 могут управлять атрибут имени пользователя, если политики запросов на подключение используется для перенаправления RADIUS-сообщения. Единственное исключение при использовании символа обратной косой черты (\) и изменяют только данные слева от него. Обратной косой черты обычно используется для указания имени домена (слева от знака) и имя учетной записи пользователя в домене (сведения справа от обратной косой черты). В этом случае допускаются только правила работы с атрибутами, изменить или заменить имя домена.

Примеры того, как управлять имя области в атрибуте имя пользователя, см. раздел «Примеры для взаимодействия с неким имя области в атрибуте имя пользователя» в статье [использование регулярных выражений в NPS](nps-crp-reg-expressions.md).

### <a name="forwarding-request"></a>Перенаправление запроса

Можно задать следующие параметры перенаправления запросов, используемых для RADIUS-сообщений запроса доступа:

- **Проверку подлинности запросов на этом сервере**. С помощью этого параметра, сервер политики сети использует домен Windows NT 4.0, Active Directory или локальной базы данных учетных записей диспетчера учетных записей безопасности (SAM) пользователя для проверки подлинности запроса на подключение. Этот параметр также указывает, что соответствующей политики сети настроен на сервере политики сети, а также параметры входящих свойств учетной записи пользователя, используются сервером политики сети для авторизации запроса на подключение. В этом случае сервер политики сети настроен для выполнения в качестве RADIUS-сервера.

- **Пересылки запросов в следующие группы внешних серверов RADIUS**. С помощью этого параметра, сервер политики сети перенаправляет запросы на подключение группы внешних RADIUS-серверов, указать. Если сервер политики сети получает допустимое сообщение разрешения доступа, соответствующее сообщение запроса доступа, попытка подключения считается прошедшим проверку подлинности и авторизацию. В этом случае сервер политики сети действует как RADIUS-прокси.

- **Принимать пользователей без проверки учетных данных**. С помощью этого параметра, сервер политики сети не проверяет удостоверение пользователя, пытающегося подключиться к сети и сервера политики сети не будет пытаться убедитесь, что пользователь или компьютер имеет право подключения к сети. Если сервер политики сети настроен так, чтобы разрешить доступ без проверки подлинности, и он получает запрос на подключение, политики сети немедленно отправляет сообщение предоставления доступа к RADIUS клиента и пользователя или компьютера, предоставляется доступ к сети. Этот параметр используется для некоторых типов принудительного туннелирования, где туннелирование доступа клиента выполняется до проверки подлинности учетных данных пользователя.

>[!NOTE]
>Этот параметр проверки подлинности не может использоваться, если протокол проверки подлинности клиента доступа MS-CHAP v2 или расширяемой проверки подлинности протокол-TLS (EAP-TLS), обеспечивающие взаимной проверки подлинности. Взаимной проверки подлинности клиент доступа доказывает является действительным клиентом для проверки подлинности сервера (сервер политики сети), что сервер проверки подлинности удостоверяет, что является допустимым сервером проверки подлинности для доступа клиента. Если используется этот параметр проверки подлинности, возвращается сообщение разрешения доступа. Тем не менее сервер проверки подлинности не будет предоставлять проверки клиентского доступа и взаимная проверка подлинности завершилась ошибкой.

Примеры того, как использовать регулярные выражения для создания правила маршрутизации для пересылки сообщений RADIUS с именем указанной областью для группы внешних серверов RADIUS, см. раздел «Образец для переадресации сообщений RADIUS, прокси-сервера» в статье [использование регулярных выражений в NPS](nps-crp-reg-expressions.md).

### <a name="advanced"></a>Дополнительно

Можно задать дополнительные параметры для задания набора атрибутов RADIUS, которые являются:

- Добавить для ответа RADIUS, если сервер политики сети, используется как проверка подлинности RADIUS и сервера учетных записей. При наличии атрибутов, указанные на сетевой политики и политики запросов на подключение, атрибуты, которые отправляются в сообщении ответа RADIUS представляют собой сочетание двух наборов атрибутов.
- Добавить RADIUS-сообщение, если сервер политики сети будет использоваться в качестве RADIUS-проверки подлинности или учета прокси-сервера. Если в сообщении, который передается атрибут уже существует, он заменяется значением атрибута, указанного в политике запросов на подключение. 

Кроме того, некоторые атрибуты, доступные для настройки соединения запрашивают политику **параметры** вкладке **Дополнительно** категории предоставляет специальные возможности. Например, можно настроить **удаленных RADIUS для сопоставления пользователей Windows** атрибута, если нужно разделить проверки подлинности и авторизации запроса на подключение между двумя пользователей учетных записей для базы данных.

**Удаленных RADIUS для сопоставления пользователей Windows** атрибут указывает, что авторизация Windows для пользователей, которые прошли проверку подлинности удаленных RADIUS-сервером. Другими словами удаленных RADIUS-сервер выполняет проверку подлинности от учетной записи пользователя в базе данных учетных записей удаленного пользователя, но локальный сервер политики сети разрешает запроса на подключение по учетной записи пользователя в базе данных учетных записей локальных пользователей. Это полезно в том случае, если вы хотите разрешить доступ к вашей сети посетителей.

Например посетителей из партнерских организациях могут проходить проверку подлинности RADIUS-сервер организации собственные партнеров и затем использовать учетную запись пользователя Windows в вашей организации для доступа к виртуальной машине локальной сети (LAN) в сети.

Ниже перечислены другие атрибуты, которые предоставляет специальные возможности

- **MS-карантина IPFilter и MS-карантина--время ожидания сеанса**. Эти атрибуты используются при развертывании управления карантином сетевого доступа (NAQC) с развертыванием маршрутизации и удаленного доступа VPN.
- **Passport пользователя сопоставления-имя участника-пользователя суффикса**. Этот атрибут можно выполнять проверку подлинности запросов на подключение с учетными данными учетной записи Windows Live™ ID пользователя.
- **Туннель тег**. Этот атрибут обозначает номер идентификатор виртуальной локальной сети, что подключение должно быть назначено сервером NAS при развертывании виртуальные локальные сети (VLAN).

## <a name="default-connection-request-policy"></a>Политики запросов на подключение по умолчанию

Политики запросов на подключение по умолчанию создается при установке сервера политики сети. Эта политика имеет следующие конфигурации.

- Проверка подлинности не настроена.
- Учет не настроен для пересылки данных учета группы внешних серверов RADIUS.
- Атрибут не настроены правила работы с атрибутами, пересылки запросов подключения для удаленных групп серверов RADIUS.
- Перенаправление запроса настроен, чтобы запросы на подключение с проверкой подлинности и авторизации на локальном сервере политики сети.
- Дополнительные атрибуты не настроены.

Политики запросов на подключение по умолчанию использует сервер политики сети как RADIUS-сервер. Настройка сервера политики сети в качестве RADIUS-прокси, необходимо также настроить группы внешних серверов RADIUS. Группы внешних RADIUS-серверов можно создать при создании новой политики запросов на подключение с помощью мастера создания политики запроса подключения. Можно удалить политики запросов на подключение по умолчанию или убедитесь, что политики запросов на подключение по умолчанию последние политики, обрабатываемых NPS, поместив его последнего в списке политик.

>[!NOTE]
>Если сервер политики сети и удаленного доступа службы установлены на одном компьютере и удаленного доступа служба настроена для проверки подлинности Windows и учет, существует возможность проверки подлинности удаленного доступа и запросы учета пересылать на сервер RADIUS. Это может произойти, когда учета запросов проверки подлинности удаленного доступа и соответствует политики запросов на подключение, который настроен для пересылки их в группы внешних серверов RADIUS.
