# <a name="connection-request-policies"></a>Политики запросов на подключение

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

В этом разделе можно использовать, чтобы узнать, как с помощью политик запросов на подключение NPS можно настроить сервер политики сети как RADIUS-сервера и RADIUS-прокси.

>[!NOTE]
>Дополнение к данному разделу доступна следующая документация по политики запроса подключения.
> - [Настройка политики запросов на подключение](nps-crp-configure.md)
> - [Настройка групп удаленных RADIUS-серверов](nps-crp-rrsg-configure.md)

Политики запросов на подключение — это набор условий и параметров, позволяющих администраторам сети назначать какие серверы службы проверки подлинности удаленного пользователя (RADIUS) выполнения проверки подлинности и авторизации подключения запрашивает сервер под управлением сервера политики сети (NPS) получает от клиентов RADIUS. Чтобы указать, какие RADIUS-серверы используются для учета RADIUS можно настроить политики запросов на подключение.

Можно создать подключение политики запросов, чтобы некоторые запросы RADIUS, полученных от клиентов RADIUS, обрабатываются локально (сервер политики сети используется в качестве сервера RADIUS) и другие типы сообщений, перенаправляются на другой сервер RADIUS (NPS используется в качестве RADIUS-прокси).

С помощью политик запросов на подключение можно использовать сервер политики сети как RADIUS-сервер или как RADIUS-прокси, в зависимости от следующих факторов: 

- Время суток и дни недели
- Имя области в запрос на подключение
- Тип запрашиваемого подключения
- IP-адрес клиента RADIUS

Сообщения Access-Request RADIUS обрабатываются или перенаправляются на сервере политики сети, только в том случае, если параметры входящего сообщения соответствуют хотя бы одной политики запроса подключения, настроенной на сервере политики сети. 

Если параметры политики совпадают, и политика требует, что NPS обработать сообщение, сервер политики сети выступает в качестве сервера RADIUS, проверки подлинности и авторизации запрос на подключение. Если параметры политики совпадают, и политика требует, что сервер политики сети перенаправляет сообщения, сервер политики сети выступает в качестве RADIUS-прокси и перенаправляет запрос на подключение к удаленному серверу RADIUS для обработки.

Если параметры входящее сообщение RADIUS Access-Request не соответствуют по крайней мере один из политик запросов на подключение, сообщение Access-Reject отправляется клиенту RADIUS и пытается подключиться к сети компьютера или пользователя, отказано в доступе.

## <a name="configuration-examples"></a>Примеры конфигурации

В следующих примерах конфигурации показано, как можно использовать политики запросов на подключение.

### <a name="nps-as-a-radius-server"></a>Сервер политики сети в качестве RADIUS-сервера

Политика запросов на подключение по умолчанию является единственной настроенной политикой. В этом примере сервер политики сети настроен как RADIUS-сервер и все запросы на подключение обрабатываются на локальном сервере политики сети. Сервер политики сети могут выполнять аутентификацию и авторизацию пользователей, учетные записи которых находятся в домене домена сервера политики сети и в доверенных доменах.


### <a name="nps-as-a-radius-proxy"></a>Сервер политики сети в качестве прокси-сервера RADIUS

Политика запросов на подключение по умолчанию удаляется, а два новых политик запросов подключения создаются для перенаправления запросов в двух разных доменах. В этом примере сервер политики сети настроен в качестве RADIUS-прокси. Сервер политики сети не обрабатывает запросы на подключение на локальном сервере. Вместо этого он перенаправляет запросы на подключение NPS или другие RADIUS-серверы, настроенные в качестве членов удаленных групп серверов RADIUS.


### <a name="nps-as-both-radius-server-and-radius-proxy"></a>Сервер политики сети как RADIUS-сервера и прокси-сервера RADIUS

Помимо политика запросов на подключение по умолчанию создается политики запросов на подключение, который перенаправляет запросы подключения сервер политики сети или другом сервере RADIUS в домене без доверия. В этом примере политика прокси отображается первым в списке политик. Если запрос на подключение не соответствует политике прокси-сервера, запрос на соединение перенаправляется на сервер RADIUS в группе удаленных RADIUS-серверов. Если запрос на подключение не соответствует политике прокси-сервера, но соответствует по умолчанию политика запросов на подключение, политики сети обрабатывает запрос на подключение на локальном сервере. Если запрос на подключение не соответствует либо политики, они удаляются.

### <a name="nps-as-radius-server-with-remote-accounting-servers"></a>Сервер политики сети как RADIUS-сервера с удаленными серверами учета

В этом примере локальный сервер политики сети не настроен на ведение учета и политика запросов на подключение по умолчанию изменен таким образом, чтобы сервер политики сети или другом сервере RADIUS в группе удаленных RADIUS-сервера пересылки сообщений учета RADIUS. Несмотря на то, что пересылать сообщения учета, не перенаправляются сообщения проверки подлинности и авторизации и локальный NPS выполняет эти функции для локального домена и всех доверенных доменов.


### <a name="nps-with-remote-radius-to-windows-user-mapping"></a>Сервер политики сети с удаленного RADIUS сопоставление пользователя Windows

В этом примере сервер политики сети действует как сервер RADIUS и как RADIUS-прокси для каждого отдельного запроса на подключение, перенаправляя запрос проверки подлинности на удаленном сервере RADIUS при использовании учетной записи локального пользователя Windows для авторизации. Эта конфигурация реализуется путем настройки удаленного RADIUS для сопоставления пользователей Windows атрибут в условии Политика запросов на подключение. (Кроме того, учетной записи пользователя необходимо создать локально, имеющий имя, совпадающее с именем учетной записи удаленного пользователя, по которой будет выполняться проверка подлинности удаленного RADIUS-сервером.)

## <a name="connection-request-policy-conditions"></a>Условия политики запросов на подключение

Условия политики запросов на подключение, один или несколько атрибутов RADIUS, которые сравниваются с атрибуты входящее сообщение RADIUS Access-Request. При наличии нескольких условий, все условия в соединении сообщение запроса, и в запрос на подключение политика должна соответствовать политика принудительно выполняться на сервере политики сети.


Ниже перечислены доступные атрибуты условий, которые можно настроить политики запросов на подключение.

### <a name="connection-properties-attribute-group"></a>Группа атрибутов свойства подключения

Группа атрибутов свойства соединения содержит следующие атрибуты.

- **Протокол кадрирования**. Используется для назначения типа кадрирования входящих пакетов. Примерами являются Point-to-Point Protocol (PPP), последовательной линии Internet протокол SLIP (), Frame Relay и X.25.
- **Тип службы**. Позволяет определить тип запрашиваемой службы. Примеры включают заключенных (например, подключений по протоколу PPP) и войдите в систему (например, Telnet-подключения). Дополнительные сведения о типах служб RADIUS см «Удаленной проверки подлинности службой удаленного доступа пользователя (RADIUS).»
- **Тип туннеля**. Используется для назначения типа туннеля, который создается отправляющим запрос клиентом. Типы туннелей включают Point-to-Point Tunneling Protocol (PPTP) и Layer Two Tunneling Protocol (L2TP).

### <a name="day-and-time-restrictions-attribute-group"></a>День и время ограничения группы атрибутов 

Группы атрибутов день и ограничения по времени содержит атрибут день и ограничения по времени. С помощью этого атрибута можно назначить день недели и время попытки подключения. День и время задается относительно день и время сервера политики сети.

### <a name="gateway-attribute-group"></a>Группы атрибутов шлюза

Группа атрибутов шлюз содержит следующие атрибуты.

- **Идентификатор вызываемой станции**. Используется для назначения номер телефона сервера доступа к сети. Этот атрибут — строка символов. Чтобы указать коды городов, можно использовать синтаксис соответствия шаблону.
- **Идентификатор NAS**. Используется для назначения имени сервера доступа к сети. Этот атрибут — строка символов. Чтобы указать идентификаторы NAS, можно использовать синтаксис соответствия шаблону.
- **IPv4-адрес NAS**. Используется для назначения протокол IP версии 4 (IPv4) адрес сервера доступа к сети (клиент RADIUS). Этот атрибут — строка символов. Чтобы указать IP-сетей можно использовать синтаксис соответствия шаблону.
- **IPv6-адрес NAS**. Используется для назначения протокол IP версии 6 (IPv6) адрес сервера доступа к сети (клиент RADIUS). Этот атрибут — строка символов. Чтобы указать IP-сетей можно использовать синтаксис соответствия шаблону.
- **Тип порта NAS**. Позволяет определить тип носителя, используемого клиентом доступа. Примерами являются аналоговые телефонные линии (это называется async), Digital сеть ISDN (Integrated Services), туннели или виртуальные частные сети (VPN), беспроводные сети, IEEE 802.11 и коммутаторы Ethernet.

### <a name="machine-identity-attribute-group"></a>Группа атрибутов удостоверение компьютера

Группы атрибутов удостоверения машины содержит атрибут удостоверения машины. С помощью этого атрибута, можно указать метод, с помощью которого клиенты определяются в политике.

### <a name="radius-client-properties-attribute-group"></a>Группы атрибутов свойства клиента RADIUS

Группа атрибутов свойства клиента RADIUS содержит следующие атрибуты.

- **Код вызывающей станции**. Используется для назначения номер телефона, используемый вызывающим объектом (клиента). Этот атрибут — строка символов. Чтобы указать коды городов, можно использовать синтаксис соответствия шаблону.  В проверки подлинности 802.1 x MAC-адрес обычно заполняется и могут быть сопоставлены с клиента.  Это поле обычно используется для обхода адрес Mac сценариев, при настройке политики запросов на подключение для «Принять пользователей без проверки учетных данных».  
- **Понятное имя клиента**. Используется для назначения имени компьютера RADIUS-клиента, который запрашивает проверку подлинности. Этот атрибут — строка символов. Чтобы указать имена клиентов, можно использовать синтаксис соответствия шаблону.
- **IPv4-адрес клиента**. Используется для назначения IPv4-адрес сервера доступа к сети (клиент RADIUS). Этот атрибут — строка символов. Чтобы указать IP-сетей можно использовать синтаксис соответствия шаблону.
- **IPv6-адрес клиента**. Используется для назначения IPv6-адрес сервера доступа к сети (клиент RADIUS). Этот атрибут — строка символов. Чтобы указать IP-сетей можно использовать синтаксис соответствия шаблону.
- **Поставщик клиента**. Используется для назначения поставщика сервера доступа к сети, который запрашивает проверку подлинности. На компьютере под управлением службы маршрутизации и удаленного доступа является поставщиком Microsoft NAS. Этот атрибут можно использовать для настройки отдельных политик для различных поставщиков NAS. Этот атрибут — строка символов. Можно использовать синтаксис соответствия шаблону.

### <a name="user-name-attribute-group"></a>Группы атрибутов имени пользователя

Группы атрибутов имя пользователя содержит атрибут имени пользователя. С помощью этого атрибута, можно указать имя пользователя или часть имени пользователя, которое должно совпадать с именем пользователя, предоставленный клиентом доступа в сообщения RADIUS. Этот атрибут — строка символов, которое обычно содержит имя и имя учетной записи пользователя. Чтобы указать имена пользователей, можно использовать синтаксис соответствия шаблону.

## <a name="connection-request-policy-settings"></a>Параметры политики запросов на подключение

Параметры политики запросов на подключение — это набор свойств, которые применяются к входящего сообщения RADIUS. Параметры состоят из следующих групп параметров.

- Проверка подлинности
- Учет
- Управление атрибутами
- Перенаправление запроса
- Дополнительно

Ниже приведены дополнительные сведения об этих параметрах.

### <a name="authentication"></a>Проверка подлинности

При использовании этого параметра, можно переопределить параметры проверки подлинности, настроенные в все сетевые политики и можно указать методы проверки подлинности и типы, необходимые для подключения к сети.

>[!IMPORTANT]
>Если настроить метод проверки подлинности в политике запросов на подключение, который является менее безопасной, чем метод проверки подлинности, настроенного в политике сети, более безопасный метод проверки подлинности, заданной в политике сети переопределяется. Например, при наличии одной политики сети, требует использования защищенного расширяемой проверки подлинности протокола проверки подлинности протоколом MSCHAP версии 2 \(PEAP-MS-CHAP v2\), являющийся с использованием пароля метод проверки подлинности для защищенных беспроводных сетей, а также настроить политики запросов на подключение, чтобы разрешить доступ без проверки подлинности, результатом является то, что клиенты не являются обязательными для проверки подлинности с помощью протокола PEAP-MS-CHAP v2. В этом примере предоставляются все клиенты, подключающиеся к сети доступ без проверки подлинности.

### <a name="accounting"></a>Учет

При использовании этого параметра, можно настроить политики запросов на подключение для пересылки данных учета сервер политики сети или другом сервере RADIUS в группе удаленных RADIUS-сервера таким образом, чтобы группе удаленных RADIUS-сервер выполняет учета.

>[!NOTE]
>Если имеется несколько серверов RADIUS и вам нужны сведения учета для всех серверов, которые хранятся в одной центральной базе данных учета RADIUS, можно использовать параметр учета политики запроса подключения политики на каждом сервере RADIUS на переадресацию данных учета из все серверы, на один сервер политики сети или другом сервере RADIUS, который используется в качестве сервера учетных записей.

Запрос на подключение, параметры учета политики действуют независимо от конфигурации учета локального сервера политики сети. Другими словами Если задан локальный NPS в журнал сведений об учете RADIUS в локальный файл или в базу данных Microsoft SQL Server, он сделает это, независимо от того, ли настраивать политики запросов на подключение для пересылки сообщений учета удаленного RADIUS группы серверов.

Если требуется учета сведений, регистрируемых удаленно, но не в локальной среде, необходимо настроить локальный NPS не вести учет, также в при настройке учета политики запросов на подключение для отправки данных учета для группы удаленных серверов RADIUS.

### <a name="attribute-manipulation"></a>Управление атрибутами

Можно настроить набор правил поиска и замены, которые управляют строками текста одного из следующих атрибутов.

- Имя пользователя
- Идентификатор вызываемой станции
- Код вызывающей станции

Обработка правил поиска и замены возникает по одной из выше атрибуты, прежде чем сообщение RADIUS регулируется параметры проверки подлинности и учета. Правила работы с атрибутами применяются только для одного атрибута. Нельзя настроить правила работы с атрибутами для каждого атрибута. Кроме того список атрибутов, которыми можно управлять — статический список; Невозможно добавить в список атрибуты для обработки.

>[!NOTE]
>При использовании протокола проверки подлинности MS-CHAP v2, не может управлять атрибут имени пользователя, если политика запросов на подключение используется для пересылки сообщения RADIUS. Единственное исключение при обратной косой черты (\) используется символ и манипуляции затрагивает только слева от него. Символ обратной косой черты обычно используется для указания имени домена (слева от знака) и имя учетной записи пользователя в домене (сведения справа от символа обратной косой черты). В этом случае допускаются только правила работы с атрибутами, изменить или заменить имя домена.

Примеры того, как управлять имени области в атрибуте имени пользователя, см. в подразделе «Примеры для работы со имени области в атрибуте имени пользователя» в разделе [использовать регулярные выражения в NPS](nps-crp-reg-expressions.md).

### <a name="forwarding-request"></a>Перенаправление запроса

Можно задать следующие параметры перенаправления запросов, которые используются для сообщения запроса доступа RADIUS:

- **Проверка подлинности запросов на этом сервере**. При использовании этого параметра, сервер политики сети использует для проверки подлинности запроса на подключение в домен Windows NT 4.0, Active Directory или локальной базы данных учетных записей диспетчера учетных записей безопасности (SAM) пользователя. Этот параметр также указывает, что соответствующей сетевой политики настроены на сервере политики сети, вместе со свойствами удаленного доступа в учетной записи пользователя, используются сервером политики сети для авторизации запроса подключения. В этом случае сервер политики сети настроен для выполнения в качестве сервера RADIUS.

- **Перенаправление запросов в следующую группу удаленных RADIUS-сервера**. При использовании этого параметра, сервер политики сети перенаправляет запросы соединения к группе удаленных серверов RADIUS вами. Если сервер политики сети получает корректное сообщение Access-Accept, соответствующее сообщение запроса доступа, попытка подключения считается проверка подлинности и авторизация. В этом случае сервер политики сети действует как RADIUS-прокси.

- **Принимать пользователей без проверки учетных данных**. При использовании этого параметра, NPS не проверять подлинность пользователей, пытающихся подключиться к сети и сервера политики сети не будет пытаться убедитесь, что пользователь или компьютер имеет право на подключение к сети. Если сервер политики сети настроен так, чтобы разрешить доступ без проверки подлинности, он получает запрос на подключение NPS немедленно отправляет сообщение Access-Accept RADIUS клиента и пользователя или компьютера, предоставляется доступ к сети. Этот параметр используется для некоторых типов принудительного туннелирования, где клиент доступа туннелируется перед проверяются учетные данные пользователя.

>[!NOTE]
>Этот вариант проверки подлинности не может использоваться, если протокол проверки подлинности клиента доступа MS-CHAP v2 или расширяемой проверки подлинности протокол-TLS (EAP-TLS), обеспечивающие взаимной проверки подлинности. Взаимной проверки подлинности клиент доступа удостоверяет, он является действительным клиентом для проверки подлинности сервера (NPS), что сервер проверки подлинности удостоверяет, что является действительным сервером проверки подлинности для этого клиента. При использовании этого параметра проверки подлинности, возвращается сообщение Access-Accept. Тем не менее сервер проверки подлинности не предоставляет подтверждение клиенту доступа, и произойдет сбой взаимной проверки подлинности.

Примеры того, как использовать регулярные выражения, чтобы создать правила маршрутизации, которые пересылают сообщения RADIUS с именем указанной областью для группы удаленных серверов RADIUS, см. в разделе «Пример для пересылки сообщений RADIUS через прокси-сервер» в разделе [использовать Регулярные выражения в NPS](nps-crp-reg-expressions.md).

### <a name="advanced"></a>Дополнительно

Можно задать дополнительные свойства для задания набора атрибутов RADIUS, которые являются:

- Добавить ответное сообщение RADIUS, если используется сервер политики сети в качестве проверки подлинности RADIUS и сервера учетных записей. Если атрибуты, указанные на сетевой политики и политики запросов на подключение, атрибуты, которые отправляются в сообщении ответа RADIUS представляют собой сочетание двух наборов атрибутов.
- Добавить сообщение RADIUS, если используется сервер политики сети в качестве проверки подлинности RADIUS или учета прокси-сервера. Если атрибут уже существует в сообщении, которое перенаправляется, он заменяется значением атрибута, заданного в политике запросов на подключение. 

Кроме того, некоторые атрибуты, которые можно настраивать на подключение запрашивают политику **параметры** вкладке **Дополнительно** категории предоставляют специальные возможности. Например, можно настроить **удаленного RADIUS для сопоставления пользователей Windows** атрибут, если вы хотите разделить проверки подлинности и авторизации запроса на подключение между двумя учетными записями баз данных.

**Удаленного RADIUS для сопоставления пользователей Windows** атрибут указывает, что Windows Авторизация происходит для пользователей, проверка подлинности выполняется с удаленного RADIUS-сервера. Другими словами удаленного RADIUS-сервера выполняет проверку подлинности по учетной записи пользователя в базу данных учетных записей удаленного пользователя, но локальный NPS авторизует запрос на подключение к учетной записи пользователя в базе данных учетных записей локальных пользователей. Это полезно в том случае, если вы хотите разрешить доступ к сети посетителей.

Например посетители из партнерских организаций могут проходить проверку подлинности, свои собственные RADIUS-сервер партнерской организации и затем учетной записи пользователя Windows в вашей организации используется для доступа к гостевой локальной сети (LAN) в сети.

Ниже приведены другие атрибуты, которые предоставляют специализированные функции.

- **MS-карантина IPFilter и MS-карантина-времени ожидания сеанса**. Эти атрибуты используются при развертывании управления карантина сетевого доступа (NAQC) при развертывании службы маршрутизации и удаленного доступа VPN.
- **Passport пользователь сопоставление UPN-суффиксами**. Этот атрибут позволяет осуществлять проверку подлинности запросов на подключение с учетными данными учетной записи Windows Live™ ID пользователя.
- **Тег туннель**. Этот атрибут определяет номер идентификатора виртуальной локальной сети, который подключение должно быть назначено сервером NAS при развертывании виртуальных локальных сетей (VLAN).

## <a name="default-connection-request-policy"></a>Политика запросов на подключение по умолчанию

Политики запросов на подключение по умолчанию создается при установке сервера политики сети. Эта политика имеет следующую конфигурацию.

- Не настроена проверка подлинности.
- Учет не настроен на перенаправление учетных сведений для группы удаленных серверов RADIUS.
- Атрибут не настроен правила работы с атрибутами, которые переадресовывать запросы на подключение к удаленными группами серверов RADIUS.
- Запрос пересылки настраивается, чтобы запросы на подключение проверку подлинности и авторизацию на локальном сервере политики сети.
- Дополнительные атрибуты не настроены.

Политика запросов на подключение по умолчанию использует сервер политики сети как RADIUS-сервер. Для настройки сервера политики сети в качестве RADIUS-прокси, необходимо также настроить группу удаленных серверов RADIUS. При создании новой политики запросов на подключение с помощью мастера создания политики запроса подключения, можно создать новую группу удаленных серверов RADIUS. Можно удалить политики запросов на подключение по умолчанию или убедитесь, что политика запросов на подключение по умолчанию политика последнего, обработанных сервером политики сети, поместив его последнего в списке политик.

>[!NOTE]
>Если сервер политики сети и службы удаленного доступа установлены на одном компьютере и удаленного доступа служба настроена для проверки подлинности Windows и accounting, существует возможность удаленного доступа и соответствующих запросов пересылать на сервер RADIUS . Это может произойти, когда удаленный доступ и соответствующих запросов соответствует политики запросов на подключение, перенаправляет их в группу удаленных серверов RADIUS.
