>Относится к: Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2019 г.

Сервер сетевых политик (NPS) позволяет создавать и применять политики доступа к сети на уровне организации для проверки подлинности и авторизации сетевых подключений.

Можно также настроить сервер политики сети как прокси службы проверки подлинности удаленного пользователя (RADIUS) для пересылки запросов подключения на удаленный сервер политики сети или другой сервер RADIUS, так что можно балансировать нагрузку по запросам подключения и пересылать их в нужный домен, для проверки подлинности и авторизации.

Сервер политики сети позволяет централизованно настраивать и управлять ими, проверка подлинности сетевого доступа, авторизации и учета с помощью следующих средств:

- **Сервер RADIUS**. Сервер политики сети выполняет централизованную проверку подлинности, авторизацию и учет для беспроводных сетей, проверки подлинности коммутатора, удаленного доступа и подключений виртуальной частной сети (VPN). Если сервер политики сети используется в качестве RADIUS-сервера, серверы доступа к сети, например точки беспроводного доступа и VPN-серверы, настраиваются как RADIUS-клиенты на сервере политики сети. Кроме того, можно настроить политики сети, используемые сервером политики сети для авторизации запросов на подключение. Также можно настроить RADIUS-учет, чтобы сервер политики сети сохранял информацию в файлах журнала, хранящихся на локальном жестком диске или в базе данных Microsoft SQL Server. Дополнительные сведения см. в разделе [сервер RADIUS](#radius-server).
- **Прокси-сервера RADIUS**. При использовании сервера политики сети как RADIUS-прокси, необходимо настроить политики запросов на подключение, которые сообщают NPS, какие запросы на пересылка на другие серверы RADIUS и на какие RADIUS-серверы требуется переадресовывать запросы на подключение подключение. На сервере политики сети можно также настроить переадресацию данных учета для их хранения на одном или нескольких компьютерах в группе удаленных RADIUS-серверов. Чтобы настроить сервер политики сети в качестве прокси-сервер RADIUS, в следующих разделах. Дополнительные сведения см. в разделе [прокси-сервера RADIUS](#radius-proxy).
    - [Настройка политики запросов на подключение](nps-crp-configure.md)
- **RADIUS-учет**. Можно сконфигурировать NPS так регистрировать события в локальный файл журнала или для создания локального или удаленного экземпляра Microsoft SQL Server. Дополнительные сведения см. в разделе [ведения журнала NPS](#nps-logging).

>[!IMPORTANT]
>Защита доступа к сети \(NAP\), центра регистрации работоспособности \(HRA\)и протокол авторизации учетных данных узла \(HCAP\) являлись нерекомендуемыми в Windows Server 2012 R2 они также не доступны в Windows Server 2016. Если у вас есть развертывание защиты доступа к сети с помощью операционных систем более ранних, чем Windows Server 2016, нельзя перенести развертывания защиты доступа к сети Windows Server 2016.

Можно сконфигурировать NPS с любой комбинацией этих функций. Например можно настроить один сервер политики сети как RADIUS-сервер для VPN-подключений, а также как RADIUS-прокси для пересылки запросов на подключение к членам группы удаленных серверов RADIUS для аутентификации и авторизации в другом домене.

## <a name="radius-server-and-proxy"></a>Сервер RADIUS и прокси-сервера

Можно использовать сервер политики сети как RADIUS-сервера и RADIUS-прокси.

### <a name="radius-server"></a>RADIUS-сервер

Сервер политики сети является реализацией стандарта RADIUS, указанный по Internet Engineering Task Force \(IETF\) в документах RFC 2865 и 2866. Как RADIUS-сервер, сервер политики сети выполняет централизованную проверку подлинности, авторизации и учета для многих типов доступа к сети, включая коммутатор беспроводной сети, выполняя проверку подлинности, удаленного доступа и виртуальной частной сети \(VPN\) удаленного доступ и подключений маршрутизатор маршрутизатор.

>[!NOTE]
>Сведения о развертывании сервера политики сети как RADIUS-сервер, см. в разделе [развертывание сервера политики сети](nps-deploy.md).

Сервер политики сети позволяет использовать разнородных набор беспроводной связи, коммутатора, удаленного доступа или VPN оборудования. Сервер политики сети можно использовать со службой удаленного доступа, который доступен в Windows Server 2016.

Сервер политики сети использует доменные службы Active Directory \(AD DS\) домена или локальных учетных записей пользователя диспетчера учетных записей безопасности (SAM) базы данных для проверки подлинности учетных данных пользователя для попытки подключения. При сервера политики сети входит в домен AD DS, сервер политики сети использует службу каталогов в качестве своей базы данных учетной записи пользователя и входит в единое решение единого входа. Тот же набор учетных данных используется для управления доступом к сети \(аутентификация и авторизация доступа к сети\) и для входа в домен AD DS.

>[!NOTE]
>Сервер политики сети использует свойства коммутируемый пользователя учетной записи и политик сети для авторизации подключения.

Поставщики услуг Интернета \(поставщики услуг Интернета\) и организациях, имеющих доступ к сети имеют повышенную задачу управления всеми типами доступа к сети из одной точки администрирования, независимо от типа доступа к сети оборудования. Стандарт RADIUS поддерживает данную функцию в однородных и разнородных средах. RADIUS — это протокол клиент сервер, который позволяет оборудования для доступа к сети (используется в качестве RADIUS-клиентов) для отправки запросов на сервер RADIUS.

RADIUS-сервер имеет доступ к учетной записи пользователя и можно проверить учетные данные проверки подлинности доступа к сети. Если учетные данные проходят проверку подлинности и попытка подключения авторизован, сервер RADIUS авторизует доступ пользователя на основе заданных условий и затем доступ к сети в журнал учета. Использование RADIUS позволяет проверке подлинности, авторизации и учета данных быть сохраняются и обрабатываются в центральном расположении, а не на каждом сервере доступа.

#### <a name="using-nps-as-a-radius-server"></a>С помощью сервера политики сети как RADIUS-сервер

Можно использовать сервер политики сети как RADIUS-сервер при:

- Вы используете домен AD доменных служб Active Directory или локальной базы данных учетных записей SAM пользователя как учетную запись пользователя базы данных для доступа клиентов. 
- При использовании удаленного доступа на нескольких серверах удаленного доступа, VPN-серверов или маршрутизаторы вызова по требованию и хотите, чтобы централизовать настройку политик сети и подключения ведения журнала и учета. 
- Выполняется сторонними исполнителями вашей удаленного доступа, VPN или беспроводной доступ к поставщику службы. Серверы доступа используют RADIUS для аутентификации и авторизации подключения, выполненные членов вашей организации.
- Вы хотите централизовать проверки подлинности, авторизации и учета для разнородного набора серверов доступа.

### <a name="radius-proxy"></a>RADIUS-прокси

RADIUS-прокси сервер политики сети перенаправляет проверки подлинности сообщения и учета для сервера политики сети и других серверов RADIUS. Можно использовать сервер политики сети как RADIUS-прокси для маршрутизации RADIUS сообщения между клиентами RADIUS \(также называется серверы сетевого доступа\) и RADIUS-серверы, выполняющие проверку подлинности пользователей, авторизации и учета для Попытка подключения. 

При использовании в качестве RADIUS-прокси, NPS является центральной точкой коммутации или маршрутизации через поток сообщения доступа и учета RADIUS. NPS информация записывается в журнал учета о пересылаемых сообщениях.

#### <a name="using-nps-as-a-radius-proxy"></a>С помощью сервера политики сети как RADIUS-прокси

Сервер политики сети можно использовать в качестве RADIUS-прокси при:

- Вы являетесь поставщиком услуг, предлагающего внешнего подключения удаленного доступа, VPN или службы доступа к беспроводной сети для нескольких клиентов. Серверы доступа к сети отправлять запросы на подключение для RADIUS-прокси. Исходя из сферы, являющееся частью имени пользователя в запрос на подключение, RADIUS-прокси перенаправляет запрос на подключение на сервер RADIUS, который ведется с клиента и может проверки подлинности и авторизации попытки подключения.
- Вы хотите предоставить проверку подлинности и авторизацию для учетных записей пользователей, которые не являются членами домена, членом которого является сервер политики сети или другом домене, имеющем двустороннее отношение доверия с доменом, членом которого является сервер NPS. Сюда входят учетные записи в недоверенных доменах, доменов с односторонним доверием и других лесах. Вместо того чтобы настраивать серверы доступа для отправки запросов на подключение к серверу NPS RADIUS, можно настроить их для отправки запросов на подключение к прокси-сервер NPS RADIUS. RADIUS-прокси использует часть имени области имя пользователя и перенаправляет их на сервер политики сети в правильный домен или лес. Попытки подключения для учетных записей из одного домена или леса могут проходить проверку подлинности для серверов NAS в другом домене или лесу.
- Вы хотите выполнять аутентификацию и авторизацию с помощью базы данных, которая не является базой данных учетной записи Windows. В этом случае запросы на подключение, которые соответствуют указанному имени области, перенаправляются на сервер RADIUS, который имеет право на другую базу данных учетных записей пользователей и данных авторизации. Другие пользовательские базы данных Примеры баз данных Novell Directory Services (NDS) и Structured Query Language (SQL).
- Вы хотите обрабатывать большое количество запросов на подключение. Таким образом вместо того чтобы настраивать клиенты RADIUS для равномерного распределения их запросов подключения и учета между несколькими серверами RADIUS, можно настроить их для отправки своих запросов подключения и учета RADIUS-прокси. RADIUS-прокси динамически распределяет нагрузку подключения, а учета запросы между несколькими серверами RADIUS и обработки большого количества клиентов RADIUS и проверок подлинности в секунду.
- Вы хотите обеспечить проверку подлинности RADIUS и авторизацию для поставщиков внешних служб и свести к минимуму конфигурации брандмауэра интрасети. Брандмауэр подключения к интрасети — между вашей сетью периметра (сеть между интрасетью и Интернетом) и интрасети. Поместив сервер политики сети в сети периметра, брандмауэр между вашей сетью периметра и интрасети должен разрешать обмен данными между сервером политики сети и несколько контроллеров домена. Заменив сервером политики сети с помощью прокси-сервер политики сети, брандмауэр должен разрешать только RADIUS-трафик между прокси-сервер политики сети и одного или нескольких NPSs в интрасети.

Конфигурации сервера политики сети могут создаваться для следующих сценариев:

- Беспроводной доступ
- Организации удаленного доступа или виртуальной частной сети (VPN) удаленного доступа
- Внешнего подключения удаленного доступа или доступа к беспроводной сети
- Доступ к Интернету
- Доступ с проверкой подлинности к ресурсам экстрасети для деловых партнеров

## <a name="radius-server-and-radius-proxy-configuration-examples"></a>Примеры настройки прокси-сервера RADIUS и сервера RADIUS

В следующих примерах конфигурации показано, как можно настроить политики сети как RADIUS-сервера и RADIUS-прокси.

**Сервер политики сети как RADIUS-сервер**. В этом примере сервер политики сети настроен как сервер RADIUS, политики запросов на подключение по умолчанию является только настроенные параметры политики и обработки всех запросов на подключение на локальном сервере политики сети. Сервер политики сети могут выполнять аутентификацию и авторизацию пользователей, учетные записи которых находятся в домене сервера политики сети и в доверенных доменах.

**Сервер политики сети как RADIUS-прокси**. В этом примере сервер политики сети настроен как RADIUS-прокси, который перенаправляет запросы соединения для удаленных групп серверов RADIUS в двух доменах без доверия. Удаляется политика запросов на подключение по умолчанию, и перенаправлять запросы к каждой из два недоверенных домена создаются две новые политики запросов на подключение. В этом примере сервер политики сети не обрабатывает запросы на подключение на локальном сервере. 

**Сервер политики сети как RADIUS-сервера и прокси-сервера RADIUS**. Помимо политика запросов на подключение по умолчанию, который указывает, что запросы на подключение обрабатываются локально, создается политики запросов на подключение, который перенаправляет запросы подключения сервер политики сети или другом сервере RADIUS в домене без доверия. Эта вторая политика имеет имя политики прокси-сервера. В этом примере политика прокси отображается первым в списке политик. Если запрос на подключение не соответствует политике прокси-сервера, запрос на соединение перенаправляется на сервер RADIUS в группе удаленных RADIUS-серверов. Если запрос на подключение не соответствует политике прокси-сервера, но соответствует по умолчанию политика запросов на подключение, политики сети обрабатывает запрос на подключение на локальном сервере. Если запрос на подключение не соответствует либо политики, они удаляются.

**Сервер политики сети как RADIUS-сервер с удаленными серверами учета**. В этом примере локальный сервер политики сети не настроен на ведение учета и политика запросов на подключение по умолчанию изменен таким образом, чтобы сервер политики сети или другом сервере RADIUS в группе удаленных RADIUS-сервера пересылки сообщений учета RADIUS. Несмотря на то, что пересылать сообщения учета, не перенаправляются сообщения проверки подлинности и авторизации и локальный NPS выполняет эти функции для локального домена и всех доверенных доменов.

**Сервер политики сети с удаленного RADIUS с пользователем Windows**. В этом примере сервер политики сети действует как сервер RADIUS и как RADIUS-прокси для каждого отдельного запроса на подключение, перенаправляя запрос проверки подлинности на удаленном сервере RADIUS при использовании учетной записи локального пользователя Windows для авторизации. Эта конфигурация реализуется путем настройки удаленного RADIUS для сопоставления пользователей Windows атрибут в условии Политика запросов на подключение. \(Кроме того учетной записи пользователя создаются локально на сервере RADIUS, который имеет имя, совпадающее с именем учетной записи удаленного пользователя, по которой будет выполняться проверка подлинности на удаленном сервере RADIUS.\)

## <a name="configuration"></a>Конфигурация

Чтобы настроить сервер политики сети как RADIUS-сервер, можно использовать стандартной или расширенной конфигурации в консоли сервера политики сети или в диспетчере сервера. Чтобы настроить сервер политики сети как RADIUS-прокси, необходимо использовать расширенную конфигурацию.

### <a name="standard-configuration"></a>Стандартная конфигурация

С помощью стандартной конфигурации мастеров предоставляются для настройки сервера политики сети для следующих сценариев:

- RADIUS-сервер для удаленного доступа или VPN-подключений
- RADIUS-сервер для беспроводных или проводных подключений 802.1 X

Чтобы настроить сервер политики сети с помощью мастера, откройте консоль сервера политики сети, выберите один из вышеперечисленных вариантов и щелкните ссылку, чтобы открыть мастер.

### <a name="advanced-configuration"></a>Дополнительная настройка

При использовании расширенной конфигурации вручную настроить сервер политики сети как RADIUS-сервер или прокси-сервера RADIUS. 

Настройка сервера политики сети с помощью расширенной конфигурации, откройте консоль сервера политики сети и щелкните стрелку рядом с полем **расширенной конфигурации** чтобы развернуть этот раздел.

Предоставляются следующие дополнительные настройки.

#### <a name="configure-radius-server"></a>Настройка сервера RADIUS

Чтобы настроить сервер политики сети как RADIUS-сервер, необходимо настроить клиенты RADIUS, политики сети и RADIUS-учет.

Инструкции по обеспечению надежности эти конфигурации в следующих разделах.

- [Настройка клиентов RADIUS](nps-radius-clients-configure.md)
- [Настройка политик сети](nps-np-configure.md)
- [Настройка учета сервера политики сети](nps-accounting-configure.md)

#### <a name="configure-radius-proxy"></a>Настройка прокси-сервера RADIUS

Чтобы настроить сервер политики сети как RADIUS-прокси, необходимо настроить клиенты RADIUS, удаленными группами серверов RADIUS и политик запросов на подключение.

Инструкции по обеспечению надежности эти конфигурации в следующих разделах.

- [Настройка клиентов RADIUS](nps-radius-clients-configure.md)

## <a name="nps-logging"></a>Ведение журнала NPS

Ведение журнала NPS также называется учета RADIUS. Настройка учета в соответствии со своими требованиями ли сервер политики сети используется в качестве сервера RADIUS, прокси-сервера или любое сочетание этих конфигураций.

Чтобы настроить ведение журнала NPS, необходимо настроить, какие события в журнал и просмотра в окне просмотра событий, а затем определить другие данные будут использоваться для входа. Кроме того необходимо решить, хотите ли вы входа проверки подлинности пользователей и сведения об учетных данных, текстовые файлы журнала, хранящиеся на локальном компьютере или в базу данных SQL Server на локальном компьютере или удаленном компьютере.

Дополнительные сведения см. в разделе [настроить сетевой политики сервера Accounting](nps-accounting-configure.md).
