# <a name="configure-network-policies"></a>Настройка политик сети

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе можно использовать для настройки политик сети на сервере политики сети.

## <a name="add-a-network-policy"></a>Добавление политики сети

\(NPS\) сервера политики сети использует сетевые политики и свойств удаленного доступа учетных записей пользователей, чтобы определить, является ли запрос на подключение разрешено подключаться к сети.

Эту процедуру можно использовать для настройки новой политики сети в консоли сервера политики сети или консоль удаленного доступа.

### <a name="performing-authorization"></a>Выполнение авторизации

Когда сервер политики сети выполняет авторизацию запросов на подключение, она сравнивает запрос с политиками сети в списке политик, начиная с первой политики, а затем перенос вниз по списку настроенных политик. Если сервер политики сети выполняет поиск политики, состояние которых соответствует запрос на подключение, сервер политики сети использует соответствующей политики свойств удаленного доступа учетной записи пользователя для выполнения авторизации. Если свойств удаленного доступа учетной записи пользователя, настроенные для предоставления доступа или управления доступом с помощью политики сети и авторизацию запроса на подключение, сервер политики сети применяет параметры, настроенные в политике сети для подключения.

Если сервер политики сети не удается найти сетевой политики, которая соответствует запрос на подключение, запрос на подключение отклоняется, пока не будут заданы свойства удаленного доступа к учетной записи пользователя, для предоставления доступа.

Если запретить доступ свойств удаленного доступа учетной записи пользователя, запрос на подключение отклоняется сервером политики сети.

### <a name="key-settings"></a>Параметры раздела

При использовании мастера создания сетевых политик для создания политики сети, значение, указанное в **метод подключения сетевых** используется для автоматической настройки **тип политики** условие: 

- Если оставить значение по умолчанию не определен, политики сети, созданные оценивается сервера политики сети для всех типов сетевых подключений, использующих какой-либо сервера доступа к сети (NAS).
- Если указать способ подключения к сети, сервер политики сети оценивает политики сети только в том случае, если запрос на подключение происхождения тип сервера доступа к сети, указанном вами.

На **разрешение на доступ** страницы, необходимо выбрать **доступом, предоставленным** Если вы хотите политику, чтобы разрешить пользователям подключаться к сети. Если вы хотите политику, чтобы предотвратить подключение к сети, выберите **отказано в доступе**. 

Если вы хотите, чтобы разрешения на доступ к определяться свойств учетной записи пользователя в dial в Active Directory&reg; \(AD DS\) службы домена, можно выбрать **доступа определяется свойствами пользователем Dial-in** флажок.

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

### <a name="to-add-a-network-policy"></a>Добавление политики сети 

1. Откройте консоль сервера политики сети, а затем дважды щелкните **политики**.

2. В дереве консоли щелкните правой кнопкой мыши **политик сети**и нажмите кнопку **New**. Откроется мастер создания сетевых политик.

3. Мастер новой политики сети используется для создания политики.

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>Создание политики сети для удаленного доступа или VPN с помощью мастера

Эту процедуру можно использовать для создания политики запросов на подключение и политики сети, необходимые для развертывания серверов удаленного доступа или серверы виртуальных частных сетей \(VPN\) \(RADIUS\) удаленной службы проверки подлинности удаленного пользователя клиентами NPS RADIUS-сервера.

>[!NOTE]
>Клиентские компьютеры, например ноутбуков и другие компьютеры с клиентскими операционными системами, не являются клиентами RADIUS. Клиенты RADIUS — это серверы доступа к сети — например точки беспроводного доступа, коммутаторы с проверкой подлинности 802.1 X, серверы виртуальных частных сетей \(VPN\) и серверы удаленного доступа, так как эти устройства используют протокол RADIUS для взаимодействия с серверами RADIUS, например серверами политики сети.

Эта процедура описывается открыть мастер удаленного доступа подключений или виртуальной частной сети на сервере политики сети.

После запуска мастера создаются следующие политики:

- Политики запросов на подключение
- Один сетевой политики

Можно запустить мастер создания удаленного подключений или виртуальной частной сети, каждый раз, необходимо создать новые политики для серверов удаленного доступа и VPN-серверы.

Мастер удаленного доступа подключений или виртуальной частной сети не остается только действие, необходимое для развертывания удаленного доступа или VPN-серверы в качестве RADIUS-клиентов на сервере политики сети. Оба метода доступа к сети требуется развернуть дополнительные компоненты оборудования и программного обеспечения.

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>Для создания политики удаленного доступа или VPN с помощью мастера

3. Нажмите кнопку **Настройка VPN или удаленный доступ с помощью мастера**. Откроется мастер создания удаленного подключений или виртуальной частной сети.

4. Следуйте инструкциям мастера, чтобы завершить создание новой политики.

Запуск мастера новых IEEE 802.1 X защищенных проводных и беспроводных подключений не только действие, необходимое для развертывания 802.1 X с проверкой подлинности коммутаторы и точки беспроводного доступа в качестве RADIUS-клиентов на сервере политики сети. ## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>Настройка сервера политики сети, чтобы игнорировать свойства Dial в учетной записи пользователя

Используйте данную процедуру, чтобы настроить политики сети игнорировать свойств удаленного доступа учетных записей пользователей в Active Directory в процессе авторизации. Свойства удаленного доступа, сервер политики сети оценивает процессе авторизации, если не имеют учетных записей пользователей в Active Directory-пользователи и компьютеры **права доступа к сети** учетной записи пользователя свойству **управление на основе политики сети**. 

Существуют два условия, где может потребоваться настройка сервера политики сети, чтобы игнорировать свойств удаленного доступа учетных записей пользователей в Active Directory:

- Когда вы хотите упростить авторизации сервера политики сети с помощью политики сети, но не все учетные записи пользователей имеют **права доступа к сети** свойство значение **управление на основе политики сети**. Например, некоторые учетные записи пользователей, возможно, **права доступа к сети** свойства учетной записи пользователя **запрещать доступ** или **разрешить доступ**.

- Когда другие свойства удаленного доступа учетных записей пользователей не применимы к тип подключения, указанные в политике сети. Например, свойств, отличающихся от **права доступа к сети** параметр применимы только к коммутируемых или VPN-подключений, но политики сети, вы создаете предназначен для подключения к беспроводной сети или проверки подлинности.

Эта процедура позволяет настроить сервер политики сети, чтобы игнорировать свойства dial в учетной записи пользователя. Если запрос на подключение соответствует политики сети, где этот флажок установлен, сервер политики сети не использует свойств удаленного доступа учетной записи пользователя для определения пользователя или компьютера, разрешено ли доступ к сети; для определения авторизации используются только параметры в политике сети.

Членство в группе **Администраторы**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

1. На сервере политики сети, в диспетчере серверов щелкните **средства**, а затем нажмите кнопку **сервера политики сети**. Откроется консоль NPS.

2. Дважды щелкните **политики**, нажмите кнопку **политик сети**и в области сведений дважды щелкните политику, которую требуется настроить.

3. В политике **свойства** диалогового **Обзор** вкладке **разрешение на доступ**выберите **игнорировать свойства учетной записи пользователя dial** флажок и нажмите кнопку **ОК**.

## <a name="configure-nps-for-vlans"></a>Настройка сервера политики сети для виртуальных ЛС

С помощью поддержкой VLAN серверы доступа к сети и сервера политики сети в Windows Server 2016, можно предоставить доступ только к сетевым ресурсам, которые подходят для их разрешения безопасности групп пользователей. Например можно предоставить посетителей беспроводной доступ к Интернету без предоставления им доступа к сети вашей организации. 

Кроме того виртуальные локальные сети позволяют логически группы сетевых ресурсов, которые существуют в разных физических местоположениях или на разных физических подсетей. Например члены вашей отдела продаж и их сетевым ресурсам, например, клиентские компьютеры, серверы и принтеры, может находиться в несколько разных зданиях в вашей организации, но все эти ресурсы можно разместить на одной виртуальной локальной сети, использующий же диапазон IP-адресов. Виртуальная ЛС затем функции, с точки зрения конечного пользователя, как одной подсети.

Также можно использовать виртуальные ЛС для разделения сети между разных групп пользователей. После определения того, каким образом требуется определить группы, можно создать группы безопасности в Active Directory-пользователи и компьютеры оснастки и добавлять членов в группы.

### <a name="configure-a-network-policy-for-vlans"></a>Настройка политики сети для виртуальных ЛС

Эту процедуру можно использовать для настройки политики сети, которая назначает пользователям виртуальной локальной сети. При использовании поддержкой виртуальной ЛС сетевого оборудования, например маршрутизаторы, коммутаторы и контроллеры доступа, можно настроить политики сети, чтобы указать серверы доступа для размещения участниками определенных групп Active Directory на определенных виртуальных локальных сетей. Эта возможность группы сетевых ресурсов логически с использованием виртуальных ЛС обеспечивает гибкость при разработке и реализации решения по сети.

При настройке параметров политики сети для использования с виртуальными локальными сетями, необходимо настроить атрибуты **Тип туннеля Средний**, **туннеля-Pvt-Group-ID**, **типа туннеля**, и **туннеля тег**. 

Эта процедура предоставляется в качестве рекомендации; Конфигурация сети может потребоваться различных параметров, чем описано ниже.

Членство в группе **Администраторы**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

### <a name="to-configure-a-network-policy-for-vlans"></a>Чтобы настроить политики сети для виртуальных ЛС

1. На сервере политики сети, в диспетчере серверов щелкните **средства**, а затем нажмите кнопку **сервера политики сети**. Откроется консоль NPS.

2. Дважды щелкните **политики**, нажмите кнопку **политик сети**и в области сведений дважды щелкните политику, которую требуется настроить.

3. В политике **свойства** диалоговом нажмите кнопку **параметры** вкладку.

4. В политике **свойства**в **параметры**в **атрибутов RADIUS**, убедитесь, что **стандартных** выбран.

5. В области сведений в **атрибуты**, **тип службы** атрибут настроен со значением по умолчанию **Framed**. По умолчанию для политик и методов доступа VPN и удаленного доступа **протокола** атрибут настроен со значением **PPP**. Чтобы указать дополнительное подключение атрибуты, необходимые для виртуальных ЛС, щелкните **добавить**. **Добавить атрибут RADIUS Standard** откроется диалоговое окно.

6. В **добавить атрибут RADIUS Standard**, атрибутов, прокрутите вниз и добавьте следующие атрибуты:

    - **Тип туннеля Средний**. Выберите значение, соответствующее предыдущей нужные изменения, внесенные для политики. Например, если политики сети, вы настраиваете политику беспроводной сети, выберите **значение: 802 (включает все 802 мультимедиа, а также канонический формат Ethernet)**.

    - **Туннель Pvt-Group-ID**. Введите целое число, представляющее номер виртуальной локальной сети, к которой будет назначен членов группы. 

    - **Тип туннеля**. Выберите **виртуальные локальные сети (VLAN)**.


7. В **добавить атрибут RADIUS Standard**, нажмите кнопку **закрыть**. 

8. Если сервер доступа к сети (NAS) требует использования **туннеля тег** атрибут, выполните следующие действия, чтобы добавить **туннеля тег** атрибут политики сети. Если этот атрибут не упомянуть в документации NAS, не добавляйте его в политику. При необходимости добавьте атрибуты следующим образом.

    - В политике **свойства**в **параметры**в **атрибутов RADIUS**, нажмите кнопку **конкретного поставщика**. 

    - В области сведений щелкните **добавить**. **Добавить поставщика конкретным атрибутом** откроется диалоговое окно.

    - В **атрибуты**, прокрутите вниз и выберите **туннеля тег**, а затем нажмите кнопку **добавить**. **Сведения об атрибутах** откроется диалоговое окно. 

    - В **значение атрибута**, введите значение, полученное из документации по оборудованию.

## <a name="configure-the-eap-payload-size"></a>Настроить размер полезных данных EAP

В некоторых случаях маршрутизаторы и брандмауэры удалить пакеты, так как они настроены для отмены пакетов, которые требуют фрагментации.

При развертывании сервера политики сети с политиками сети, используйте \(EAP\) расширяемого протокола проверки подлинности с \(TLS\) безопасности уровня транспорта TLS или EAP-TLS в качестве метода проверки подлинности по умолчанию \(MTU\), сервер политики сети использует для полезных данных EAP представляет максимальный 1500 байтов. 

Это максимальный размер полезных данных EAP можно создать сообщений RADIUS, которые требуют фрагментации, маршрутизатор или брандмауэр между клиентом RADIUS и сервера политики сети. Если это так, маршрутизатор или брандмауэр, расположенного между клиентом RADIUS и сервером политики сети без вмешательства пользователя может отменить некоторые фрагментов, из-за которой сбой проверки подлинности и невозможность доступа клиента подключения к сети.

Используйте следующую процедуру для снижения максимальный размер, который сервер политики сети использует для EAP полезных данных с помощью атрибута Framed MTU в политике сети не должен превышать 1344 значение.

Членство в группе **Администраторы**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

### <a name="to-configure-the-framed-mtu-attribute"></a>Настройка атрибутов Framed MTU

1. На сервере политики сети, в диспетчере серверов щелкните **средства**, а затем нажмите кнопку **сервера политики сети**. Откроется консоль NPS.
 
2. Дважды щелкните **политики**, нажмите кнопку **политик сети**и в области сведений дважды щелкните политику, которую требуется настроить.

3. В политике **свойства** диалоговом нажмите кнопку **параметры** вкладку.

4. В **параметры**в **атрибутов RADIUS**, нажмите кнопку **стандартных**. В области сведений щелкните **добавить**. **Добавить атрибут RADIUS Standard** откроется диалоговое окно.

5. В **атрибуты**, прокрутите вниз и нажмите кнопку **Framed MTU**, а затем нажмите кнопку **добавить**. **Сведения об атрибутах** откроется диалоговое окно.

6. В **значение атрибута**, введите значение меньше или равно чем **1344**. Нажмите кнопку **ОК**, нажмите кнопку **закрыть**, а затем нажмите кнопку **ОК**.
