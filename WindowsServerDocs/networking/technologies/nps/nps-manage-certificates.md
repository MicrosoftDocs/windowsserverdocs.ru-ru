# <a name="manage-certificates-used-with-nps"></a>Управление сертификатами, используемыми с сервером политики сети

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

Если выполнено развертывание метода проверки подлинности на основе сертификатов, такие как расширяемой проверки подлинности Protocol\ Transport Layer Security \(EAP\-TLS\) и \(PEAP\-TLS\) защищенных расширяемой проверки подлинности Protocol\-TLS, Майкрософт PEAP\ запрос проверки подлинности протоколом MSCHAP версии 2 \ (MS\ CHAP v2\), необходимо зарегистрировать сертификат сервера для всех ваших серверов политики сети. Сертификат сервера выполните следующие действия.

- Требованиям к минимальной серверной сертификат как описано в [Настройка шаблонов сертификатов для требований PEAP и EAP](nps-manage-cert-requirements.md)

- Выдан центром сертификации, которому доверяют клиентские компьютеры \(CA\). ЦС является доверенным, если его сертификат существует в хранилище сертификатов доверенных корневых центров сертификации для текущего пользователя и локальный компьютер.

Приведенные ниже инструкции помочь в управлении сертификатами сервера политики сети в развертываниях, где доверенный корневой ЦС сторонним центром сертификации, например Verisign, или ЦС, который развертывания для инфраструктуры открытого ключа \(PKI\) с помощью \(AD CS\) служб сертификатов Active Directory.

## <a name="change-the-cached-tls-handle-expiry"></a>Изменение срока действия маркера кэшированные TLS

Во время процессов начальной проверки подлинности EAP\ TLS, PEAP\ TLS и PEAP\-MS\-CHAP v2 сервер политики сети кэширует часть свойства подключения клиента TLS-подключения. Клиент кэширует часть свойства TLS-подключения сервера политики сети.

Каждой отдельной коллекции эти свойства TLS-подключения, называется дескриптор TLS.

Клиентские компьютеры можно кэш TLS-дескрипторы несколько средств проверки подлинности, пока серверов политики сети может кэшировать TLS-дескрипторы многих клиентских компьютеров.

Кэшированные дескрипторы TLS на клиенте и сервере разрешить процесс повторная проверка подлинности происходит быстрее. Например при выполняемой беспроводного компьютера с сервером политики сети, сервер политики сети можно проверить дескриптор TLS для беспроводного клиентского и можно быстро определить, что подключение клиента переподключения. Сервер политики сети разрешает подключение без выполнения полной проверки подлинности.

Соответственно клиент проверяет дескриптор TLS для сервера политики сети, определяет, что он является переподключения и не требуется выполнение проверки подлинности сервера.

На компьютерах под управлением Windows 10 и Windows Server 2016 срока действия маркера TLS по умолчанию — 10 часов.

В некоторых случаях может потребоваться увеличить или уменьшить время истечения срока действия маркера TLS.

Например можно уменьшить время истечения срока действия дескриптор TLS в случаях, где — отозван сертификат пользователя с правами администратора и сертификата истек. В этом сценарии пользователь может по-прежнему подключаться к сети сервер политики сети содержит кэшированный дескриптор TLS, не истек. Уменьшение TLS истечения срока действия маркера может предотвратить таких пользователей с отозванные сертификаты повторное подключение.

>[!NOTE]
>В этом случае лучше отключить учетную запись пользователя в Active Directory или удалить учетную запись пользователя из группы Active Directory, которому предоставлено разрешение на подключение к сети в политике сети. Распространение этих изменений на все контроллеры домена может также быть отложен, однако из-за задержки репликации. 

## <a name="configure-the-tls-handle-expiry-time-on-client-computers"></a>Настройте время истечения срока действия маркера TLS на клиентских компьютерах

Эта процедура позволяет изменять количество времени, что клиентские компьютеры кэширует дескриптор TLS сервер политики сети. После успешной проверки подлинности сервера политики сети, клиентские компьютеры кэша свойства TLS-подключения на сервере политики сети как дескриптор TLS. По умолчанию дескриптор TLS длительностью \(36,000,000 milliseconds\) 10 часов. Можно увеличить или уменьшить время истечения срока действия маркера TLS с помощью следующей процедуры.

Членство в группе **Администраторы**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

>[!IMPORTANT]
>Эту процедуру необходимо выполнить на сервере политики сети, а не на клиентском компьютере.

### <a name="to-configure-the-tls-handle-expiry-time-on-client-computers"></a>Чтобы настроить TLS обрабатывать время истечения срока действия на клиентских компьютерах

1. На NPS-сервере откройте редактор реестра.

2. Перейдите в раздел реестра **HKEY\_LOCAL\_MACHINE\System\CurrentControlSet\Control\SecurityProviders\SCHANNEL**

3. На **изменить** меню, нажмите кнопку **New**, а затем нажмите кнопку **ключ**.

4. Тип **ClientCacheTime**, и нажмите клавишу ВВОД.

5. Щелкните правой кнопкой мыши **ClientCacheTime**, нажмите кнопку **New**, а затем нажмите кнопку **значение DWORD (32-разрядная версия)**.

6. Введите количество времени в миллисекундах, в том, что необходимо, чтобы компьютеры на кэширование TLS NPS-сервера после попытки первой успешной проверки подлинности сервером политики сети.

## <a name="configure-the-tls-handle-expiry-time-on-nps-servers"></a>Настройте время истечения срока действия маркера TLS на серверах политики сети

Используйте данную процедуру, чтобы изменить время, что NPS-серверы кэширование дескрипторов TLS клиентских компьютеров. После успешной проверки подлинности клиента, серверов политики сети кэширование свойства TLS-подключения клиентского компьютера в виде дескриптор TLS. По умолчанию дескриптор TLS длительностью \(36,000,000 milliseconds\) 10 часов. Можно увеличить или уменьшить время истечения срока действия маркера TLS с помощью следующей процедуры.

Членство в группе **Администраторы**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

>[!IMPORTANT]
>Эту процедуру необходимо выполнить на сервере политики сети, а не на клиентском компьютере.

### <a name="to-configure-the-tls-handle-expiry-time-on-nps-servers"></a>Для настройки TLS обрабатывать времени истечения срока действия на серверах политики сети

1. На NPS-сервере откройте редактор реестра.

2. Перейдите в раздел реестра **HKEY\_LOCAL\_MACHINE\System\CurrentControlSet\Control\SecurityProviders\SCHANNEL**

3. На **изменить** меню, нажмите кнопку **New**, а затем нажмите кнопку **ключ**.

4. Тип **ServerCacheTime**, и нажмите клавишу ВВОД.

5. Щелкните правой кнопкой мыши **ServerCacheTime**, нажмите кнопку **New**, а затем нажмите кнопку **значение DWORD (32-разрядная версия)**.

6. Введите количество времени, в миллисекундах, что требуется NPS-серверов на кэширование TLS клиентского компьютера после попытки первой успешной проверки подлинности клиента.

## <a name="obtain-the-sha-1-hash-of-a-trusted-root-ca-certificate"></a>Получить SHA-1 хэш сертификата доверенного корневого ЦС

Эта процедура используется для получения безопасного хэш-алгоритм (SHA-1) хэша доверенного корневого центра сертификации (ЦС) от сертификат, установленный на локальном компьютере. В некоторых случаях например при развертывании групповой политики, необходимо указать сертификат с помощью хэш SHA-1 сертификата.

При использовании групповой политики, можно назначить один или несколько сертификатов доверенного корневого ЦС, которые клиенты должны использовать для проверки подлинности сервера политики сети в процессе взаимной проверки подлинности EAP или PEAP. Чтобы назначить сертификат доверенного корневого ЦС, который клиенты должны использовать для проверки сертификата сервера, можно ввести SHA-1 хэш сертификата.

Показано, как получить хэш SHA-1 сертификат доверенного корневого ЦС с помощью сертификатов Microsoft Management Console (MMC). 

Для выполнения этой процедуры необходимо быть членом **пользователей** на локальном компьютере.

### <a name="to-obtain-the-sha-1-hash-of-a-trusted-root-ca-certificate"></a>Для получения SHA-1 хэш сертификата доверенного корневого центра сертификации

1. В диалоговом окне Запуск или Windows PowerShell, введите **mmc**, и нажмите клавишу ВВОД. Открытие \(MMC\) Microsoft Management Console. В консоли MMC щелкните **файл**, нажмите кнопку **Snap\in добавить или удалить**. **Добавить или удалить оснастку** откроется диалоговое окно.

2. В **добавить или удалить оснастку**в **Доступные оснастки**, дважды щелкните **сертификаты**. Откроется мастер оснастки сертификатов. Нажмите кнопку **учетной записи компьютера**, а затем нажмите кнопку **Далее**.

3. В **Выбор компьютера**, убедитесь, что **локальный компьютер (компьютер, на которой запущена эта консоль)** —, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.

4. В левой области дважды щелкните **сертификаты (локальный компьютер)**, а затем дважды щелкните **доверенные корневые центры сертификации** папки.

5. **Сертификаты** папка вложена **доверенные корневые центры сертификации** папки. Нажмите кнопку **сертификаты** папки.

6. В области сведений перейдите к сертификату для вашей доверенным корневым ЦС. Дважды щелкните сертификат. **Сертификат** откроется диалоговое окно.

7. В **сертификат** диалоговом нажмите кнопку **сведений** вкладку.

8. В списке поля, прокрутите список и выберите **отпечаток**.

9. В нижней области отображается шестнадцатеричную строку, которая является хэшем SHA-1 сертификата. Выберите SHA-1 хэш и затем нажмите сочетание клавиш Windows для копирования команд \(CTRL\+C\) для копирования хэш в буфер обмена Windows.

10. Откройте папку, для которой нужно вставить хэш SHA-1 правильно найдите курсор и нажмите сочетание клавиш Windows для вставки команд \(CTRL\+V\). 

Дополнительные сведения о сертификатах и сервера политики сети см. в разделе [Настройка шаблонов сертификатов для требований PEAP и EAP](nps-manage-cert-requirements.md).

Дополнительные сведения о сервере политики сети см. в разделе [сервера политики сети (NPS)](nps-top.md).
