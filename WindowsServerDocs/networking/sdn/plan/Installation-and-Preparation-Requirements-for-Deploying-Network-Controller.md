# <a name="requirements-for-deploying-network-controller"></a>Требования для развертывания сетевого контроллера

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Подготовка вашего центра обработки данных для развертывания сетевого контроллера, который требует один или несколько компьютеров или виртуальных машин и на одном компьютере или виртуальной Машины. Перед развертыванием сетевого контроллера, необходимо настроить группы безопасности, расположение файлов журнала (при необходимости) и динамическая регистрация DNS.


## <a name="network-controller-requirements"></a>Требования для сетевого контроллера

Развертывание сетевого контроллера требуется один или несколько компьютеров или виртуальных машин, которые служат в качестве сетевого контроллера и один компьютер или виртуальную Машину для использования в качестве клиента управления для сетевого контроллера. 

- Все виртуальные машины и компьютеры, которые планируется как узлы сетевого контроллера должна быть запущена Windows Server 2016 Datacenter edition. 
- Любого компьютера или виртуальной машины (VM), на котором установки сетевого контроллера должна быть запущена Datacenter edition, Windows Server 2016. 
- Управления клиентского компьютера или виртуальной Машины для сетевого контроллера должна быть запущена Windows 10. 

  
## <a name="configuration-requirements"></a>Требования к конфигурации

Перед развертыванием сетевого контроллера, необходимо настроить группы безопасности, расположение файлов журнала (при необходимости) и динамическая регистрация DNS.
  
### <a name="step-1-configure-your-security-groups"></a>Шаг 1. Настройка групп безопасности
  
Первое, что необходимо сделать — создать две группы безопасности для проверки подлинности Kerberos. 

Создайте группы для пользователей, имеющих разрешение на: 

1. Настройка сетевого контроллера<p>Можно задать имя этой группы "Администраторы" сетевого контроллера, к примеру. 
2.  Настройка и управление сетью с помощью сетевого контроллера<p>Можно задать имя этой группы Пользователи сетевого контроллера, к примеру. Используйте Representational State Transfer (REST) для настройки и управления сетевым контроллером.

>[!NOTE]
>Все пользователи, добавляемые должны быть членами группы пользователей домена в Active Directory — пользователи и компьютеры.

### <a name="step-2-configure-log-file-locations-if-needed"></a>Шаг 2. Настройка расположение файлов журнала, при необходимости

Следующее, что необходимо сделать — это настройка расположение файлов для хранения журналов отладки сетевого контроллера на сетевой контроллер компьютера или виртуальной Машине или на удаленном файловом ресурсе. 

>[!NOTE]
>Если журналы хранятся в удаленной общей папке, убедитесь, что общий ресурс доступен с сетевого контроллера.


### <a name="step-3-configure-dynamic-dns-registration-for-network-controller"></a>Шаг 3. Настроить динамическую регистрацию DNS для сетевого контроллера
  
Наконец следующее, что необходимо сделать — развертывание сетевого контроллера узлов кластера, на той же подсети или разных подсетях. 

|Если...  |То...  |
|---------|---------|
|В той же подсети, |Необходимо указать сетевой контроллер IP-адрес REST. |
|В разных подсетях, |Необходимо указать сетевой контроллер DNS-имя REST, который создается во время развертывания. Также необходимо выполнить следующее:<ul><li>Настройка динамических обновлений DNS для имени сетевого контроллера DNS на DNS-сервере.</li><li>Ограничьте динамические обновления DNS только узлы сетевого контроллера.</li></ul> |
---

> [!NOTE]
> Членство в группе **"Администраторы домена"**, или эквивалентной является минимальным требованием для выполнения этих процедур.
  
1. Разрешены динамические обновления DNS для зоны.

   1. Откройте диспетчер DNS, в дереве консоли щелкните правой кнопкой мыши соответствующую зону и выберите команду **свойства**. 
      
   2. На **Общие** убедитесь, что тип зоны — **основной** или **интегрированный с Active Directory**.

   В. В **динамические обновления**, убедитесь, что **только безопасные** выбран, а затем нажмите кнопку **ОК**.

2. Настройка разрешений безопасности зоны DNS для узлов сетевого контроллера

   1.  Откройте вкладку **Безопасность** и выберите **Дополнительно**. 

   2. В **Дополнительные параметры безопасности**, нажмите кнопку **добавить**. 
  
   В. Щелкните **Выбрать субъект**. 

   Г. В **Выбор пользователя, компьютера, учетной записи службы или группы** диалоговом окне щелкните **типы объектов**. 

   Д. В **типы объектов**выберите **компьютеров**, а затем нажмите кнопку **ОК**.

   f. В **Выбор пользователя, компьютера, учетной записи службы или группы** диалоговом окне введите NetBIOS-имя одного из узлов сетевого контроллера в развертывании и нажмите кнопку **ОК**.

   ж. В **запись разрешения**, проверьте следующие значения:

      - **Тип** = разрешить
      - **Применяется к** = этот объект и все дочерние объекты
  
   з) В **разрешения**выберите **запись всех свойств** и **удалить**, а затем нажмите кнопку **ОК**.

3. Повторите для всех компьютеров и виртуальных машин в кластере сетевого контроллера.

### <a name="step-4-configure-service-principal-name-if-using-kerberos-based-authentication"></a>Шаг 4. Настройка имени участника-службы, если проверка подлинности с помощью Kerberos на основе

Если сетевой контроллер использует проверку подлинности Kerberos для взаимодействия с клиентами для управления, необходимо настроить имя участника-службы (SPN) для сетевого контроллера в Active Directory. Сетевой контроллер автоматически настраивает имя участника-службы. Вам нужно всего лишь для предоставления разрешений для машин сетевого контроллера для регистрации и изменить имя участника-службы. Дополнительные сведения см. в разделе [настроить основной имена служб (SPN)](https://docs.microsoft.com/windows-server/networking/sdn/security/kerberos-with-spn#configure-service-principal-names-spn).
