## <a name="bkmk_sdn"></a>Программно-конфигурируемой Обзор работы в сети

Определенные сетевые \(SDN\) программного обеспечения позволяет централизованно настраивать и управлять физическими и виртуальными сетевыми устройствами, такими как маршрутизаторы, коммутаторы и шлюзы в центре обработки данных. Элементы виртуальной сети, такие как виртуальный коммутатор Hyper-V, виртуализация сети Hyper-V и шлюз RAS-сервера предназначены для как интегральные части SDN инфраструктуры.

>[!NOTE]
>Для узлов Hyper-V и виртуальные машины \(VMs\) выполнения SDN серверы инфраструктуры, такие как сетевой контроллер и балансировку нагрузки программного обеспечения узлов необходимо установить Windows Server 2016 Datacenter edition. Для узлов Hyper-V, содержащие только нагрузку клиентов виртуальных машин, которые подключены к сетям, контроль SDN\ можно запустить Windows Server 2016 Standard edition.

Хотя по-прежнему можно использовать существующих физических коммутаторов, маршрутизаторов и других устройств, можно добиться более глубокого уровня интеграции между виртуальной и физической сети, если эти устройства предназначены для обеспечения совместимости с программно-конфигурируемая сеть.  
  
SDN возможна, так как планы сети — управление, контроль и данные — больше не привязаны к самим сетевым устройствам, а абстрагированы другими объектами, такими как управляющее программное обеспечение как System Center 2016.  
  
SDN позволяет динамически управлять сетью центра обработки данных для обеспечения автоматического, централизованного обеспечения соответствия требованиям приложений и рабочих нагрузок. Программно-конфигурируемая сеть предоставляет следующие возможности.  
  
-   Возможность абстрагирования приложений и рабочих нагрузок из базовой физической сети, осуществляемая за счет виртуализации сети. Как и при использовании Hyper-V для виртуализации серверов, эти абстракции являются согласованными и работают с приложениями и рабочими нагрузками без нарушения их функционирования. Например программно-конфигурируемая сеть предоставляет виртуальные абстракции для элементов физической сети, такие как IP-адреса, коммутаторы и подсистемы балансировки нагрузки.  
  
-   Возможность централизованного определения и контроля политик, которые управляют физическими и виртуальными сетями, включая поток трафика между этими двумя типами сети.  
  
-   Возможность реализации политики сети в согласованного масштабирования, даже при развертывании новых рабочих нагрузок или перемещении рабочих нагрузок в виртуальных или физических сетях.  
  
## <a name="bkmk_ws"></a>Технологии Windows Server для программно-конфигурируемой сети  
Windows Server включает следующие технологии программно-конфигурируемых сетей.  

### <a name="bkmk_nc"></a>Сетевой контроллер

Новое в Windows Server 2016 сетевой контроллер обеспечивает централизованную программируемую точку автоматизации для управления, настройки, отслеживания и устранения неисправностей в виртуальной и физической сетевой инфраструктуре центра обработки данных. С помощью сетевого контроллера можно автоматизировать настройки сетевой инфраструктуры и отказаться от ручной настройки сетевых устройств и служб.  
  
Сетевой контроллер — это роль сервера, высокодоступная, масштабируемая и имеет единый программный интерфейс (API) - Южный API - которого сетевой контроллер осуществляет связь с сетью, а также второй API - Северный API -, который позволяет вам обмениваться данными с сетевым контроллером приложения.  
  
С помощью Windows PowerShell, API Representational State Transfer (REST) или приложения управления, можно использовать сетевого контроллера управлять следующими физическими и виртуальными сетевыми инфраструктурами.  
  
-   Виртуальные машины Hyper-V и виртуальных коммутаторов  
  
-   Физические сетевые коммутаторы  
  
-   Физические сетевые маршрутизаторы  
  
-   Брандмауэр  
  
-   Шлюзы VPN, включая шлюзы Мультитенантного службы удаленного доступа (RAS)  
  
-   Подсистемы балансировки нагрузки.  
  
Дополнительные сведения см. в разделе [сетевой контроллер](../sdn/technologies/network-controller/Network-Controller.md).  
  
### <a name="bkmk_hv"></a>Виртуализация сети Hyper-V

Виртуализация сети Hyper-V помогает абстрагирования приложений и рабочих нагрузок из физической сети с помощью виртуальных сетей. Виртуальные сети обеспечивают необходимую многопользовательскую изоляцию во время работы в совместно используемой физической сетевой инфраструктуре, повышая тем самым эффективность использования ресурсов. Чтобы убедиться, что вы можно перенести существующие инвестиции в инфраструктуру, можно настроить виртуальные сети на имеющемся сетевом оборудовании. Кроме того виртуальные сети, поддерживающих виртуальные локальные сети (VLAN).  
  
Дополнительные сведения см. в разделе [виртуализации сети Hyper-V](../sdn/technologies/hyper-v-network-virtualization/Hyper-V-Network-Virtualization.md).  
  
### <a name="bkmk_switch"></a>Виртуальный коммутатор Hyper-V

Hyper-V виртуальный коммутатор — это программный уровня 2 сетевой коммутатор Ethernet, доступное в диспетчере Hyper-V, после установки роли сервера Hyper-V. Коммутатор предоставляет программно управляемые и расширяемые возможности для подключения виртуальных машин к виртуальным сетям и к физической сети. Кроме того виртуальный коммутатор Hyper-V обеспечивает применение политик для безопасности, изоляции и уровней обслуживания.  
  
В виртуальный коммутатор Hyper-V в Windows Server 2016 также можно развернуть коммутатор объединение ВНЕДРЕННЫХ и удаленный прямой доступ к памяти (RDMA). Дополнительные сведения см. раздел [удаленный прямой доступ к памяти (RDMA) и коммутатора объединение ВНЕДРЕННЫХ](#bkmk_rdma) в этом разделе.  
  
Дополнительные сведения о виртуальном коммутаторе Hyper-V см. в разделе [виртуального коммутатора Hyper-V](../../virtualization/hyper-v-virtual-switch/Hyper-V-Virtual-Switch.md).

### <a name="bkmk_idns"></a>Внутренняя служба DNS & #40; имена IDN. & #41;

\(VMs\) размещенные виртуальные машины и приложения, требующие DNS для обмена данными в их собственных сетей и внешние ресурсы в Интернете. С имена IDN вы можете предоставить клиентам службы разрешения имен DNS для их пространство имен изолированной, локальный и для Интернет-ресурсов.

Дополнительные сведения см. в разделе [внутренняя служба DNS & #40; имена IDN. & #41; для SDN](technologies/Idns-for-Sdn.md).
  
### <a name="bkmk_nfv"></a>Виртуализация сетевых функций

В современное программное обеспечение определенных центрах обработки данных сетевые функции, выполняемые аппаратными устройствами (такими как подсистемы балансировки нагрузки, брандмауэры, маршрутизаторы, коммутаторы и т. д) все чаще виртуализируются в виде виртуальных устройств. Такая «виртуализация сетевых функций» является естественным этапом развития виртуализации серверов и сети. Виртуальных приборов быстро развивающихся и создание нового рынка. Они по-прежнему вызвать интерес и получить импульс в обеих платформах виртуализации и облачных служб.  
  
Доступны следующие технологии виртуализация сетевых функций.  
  
-   **Балансировка нагрузки программного обеспечения (SLB) и преобразование сетевых адресов (NAT)**. Север Юг и Восток Запад уровня 4 подсистемы балансировки нагрузки и NAT повышает пропускную способность с помощью поддержки прямой возвращает сервера, с которым возврата сетевого трафика, может обойти балансировки нагрузки мультиплексор. Дополнительные сведения см. в разделе [программной балансировки нагрузки (SLB) для SDN](technologies/network-function-virtualization/software-load-balancing-for-sdn.md).  
  
-   **Брандмауэр центра обработки данных**. Брандмауэр распределенного предоставляет детальное управления доступом (ACL), которая позволяет применять политики брандмауэра на уровне интерфейса виртуальной Машины или на уровне подсети.  
  
    Дополнительные сведения см. в разделе [Обзор брандмауэра центра обработки данных](../sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).  
  
-   **Шлюз RAS**. Шлюзы можно использовать для ретрансляции трафика между виртуальными сетями и без виртуализации сети; в частности можно развернуть шлюзах VPN ТИПА сайта для сайта, шлюзы переадресации и шлюзов Generic Routing Encapsulation (GRE). Кроме того, + N, M избыточности шлюзов поддерживается. Дополнительные сведения см. в разделе [шлюза RAS-сервера для SDN](technologies/network-function-virtualization/RAS-Gateway-for-SDN.md).
  
Дополнительные сведения см. в разделе [виртуализация сетевых функций](technologies/network-function-virtualization/Network-Function-Virtualization.md).  
  
### <a name="bkmk_rdma"></a>Удаленный прямой доступ к памяти (RDMA) и внедренных коммутаторов объединения)  
В Windows Server 2016 можно включить RDMA на сетевых адаптеров, привязанных к виртуальному коммутатору Hyper-V с или без объединения (ВНЕДРЕННЫХ коммутаторов). Это позволяет использовать сетевые адаптеры, меньше, если вы хотите использовать RDMA и SET в то же время.  
  
НАБОР является альтернативным решением объединение сетевых КАРТ, можно использовать в средах Hyper-V и стека программно-конфигурируемые сети (SDN) в Windows Server 2016. НАБОР объединяет некоторые функции объединения сетевых КАРТ в виртуальный коммутатор Hyper-V.  
  
НАБОР позволяет вам группировать от одного до восьми физические сетевые адаптеры Ethernet в один или несколько программных виртуальных сетевых адаптеров. Эти виртуальные сетевые адаптеры обеспечивают высокую производительность и отказоустойчивость в случае сбоя сетевого адаптера.  
НАБОР элементов сетевых адаптеров необходимо установить в одном физическом узле Hyper-V, необходимо поместить в группу.  
  
Кроме того можно использовать команды Windows PowerShell для включения мост для центра данных (DCB), создайте виртуальный коммутатор Hyper-V с RDMA виртуальный сетевой Адаптер (vNIC) и создания виртуального коммутатора Hyper-V с помощью SET и RDMA.  
  
Дополнительные сведения см. в разделе [удаленный прямой доступ к памяти (RDMA) и коммутатора объединение ВНЕДРЕННЫХ](https://technet.microsoft.com/library/mt403349.aspx).  
  
### <a name="bkmk_rras"></a>Шлюз RAS-сервера для SDN  
Шлюз RAS-сервера является программного обеспечения, мультитенантном, протокол пограничного шлюза (BGP) маршрутизатор с поддержкой в Windows Server 2016, которая предназначена для поставщиков облачных служб (CSP) и предприятий, которые размещают несколько виртуальных сетей клиента с помощью виртуализации сети Hyper-V.  
  
Шлюз RAS-сервера предоставляет возможности пулов шлюза, избыточность + N, M, нескольких типов site to site VPN-подключения и рефлектор маршрутов BGP в предоставляют гибкие варианты для инфраструктуры шлюза.  
  
Дополнительные сведения см. в разделе [шлюза RAS-сервера для SDN](technologies/network-function-virtualization/RAS-Gateway-for-SDN.md)  
  
### <a name="bkmk_slb"></a>Балансировка нагрузки программного обеспечения  
Поставщики облачных служб (CSP) и предприятия, развертывающие программно определенные сети (SDN) в Windows Server 2016 можно использовать программную балансировку нагрузки (SLB), чтобы равномерно распределить сетевой трафик клиента между ресурсами виртуальной сети и клиента. Windows Server SLB позволяет размещать же рабочую нагрузку на нескольких серверах, обеспечивает высокой доступности и масштабируемости.  
  
Дополнительные сведения см. в разделе [балансировку нагрузки программного обеспечения & #40; SLB & #41; для SDN](../sdn/technologies/network-function-virtualization/software-load-balancing-for-sdn.md).

Для этого в каждом контейнера имеется свое представление операционной системы, процессов, файловой системы, реестра и IP-адресов. С Windows Server 2016 теперь можно подключаться к виртуальным сетям контейнеры Windows Server. Дополнительные сведения см. в разделе [контейнеры Windows Server](technologies/containers/Container-networking-overview.md).
