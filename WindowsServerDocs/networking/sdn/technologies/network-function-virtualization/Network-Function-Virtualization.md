# <a name="network-function-virtualization"></a>Виртуализация сетевых функций

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе можно использовать для изучения функции виртуализации сети, которая позволяет развертывать виртуальные сети приборов, таких как брандмауэр центра обработки данных, мультитенантного шлюза RAS-сервера и балансировку нагрузки программного обеспечения \(SLB\) мультиплексор \(MUX\).

>Дополнение к данному разделу доступна следующая документация виртуализация сетевых функций.  
> - [Обзор брандмауэра центра обработки данных](../../../sdn/technologies/network-function-virtualization/../../../sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md)  
> - [Шлюз RAS-сервера для SDN](../../../sdn/technologies/network-function-virtualization/RAS-Gateway-for-SDN.md)  
> - [Программное обеспечение балансировки нагрузки (SLB) для SDN](../../../sdn/technologies/network-function-virtualization/Software-Load-Balancing--SLB--for-SDN.md)  
  
В современное программное обеспечение определенных центрах обработки данных сетевые функции, выполняемые аппаратными устройствами (такими как подсистемы балансировки нагрузки, брандмауэры, маршрутизаторы, коммутаторы и т. д) все чаще виртуализируются в виде виртуальных устройств. Такая «виртуализация сетевых функций» является естественным этапом развития виртуализации серверов и сети. Виртуальных приборов быстро развивающихся и создание нового рынка. Они по-прежнему вызвать интерес и получить импульс в обеих платформах виртуализации и облачных служб.  
  
Корпорация Майкрософт и включила шлюза автономного виртуального устройства, начиная с Windows Server 2012 R2. Дополнительные сведения см. в разделе [шлюз Windows Server](https://technet.microsoft.com/library/dn313101.aspx). Теперь с помощью Windows Server 2016, корпорация Майкрософт продолжает разверните и инвестиций в рынок виртуализации сети функции.  
  
## <a name="virtual-appliance-benefits"></a>Преимущества виртуальное устройство  
Виртуальное устройство динамических и легко изменить, так как он предварительно созданные, специализированных виртуальной машины. Это может быть один или несколько виртуальных машин упакованных, обновления и как единое целое. Вместе с программным обеспечением конфигурируемая сеть (SDN), вы получите оперативность и гибкость, необходимые в современных облачной инфраструктуры. Например:  
  
-   SDN предоставляет сети в виде динамического и в составе пула ресурсов.  
  
-   SDN облегчает изоляции клиентов.  
  
-   SDN позволяет увеличить масштаб и производительность.  
  
-   Виртуальных приборов включить безупречное емкости мобильности расширения и рабочей нагрузки.  
  
-   Виртуальных приборов свести к минимуму сложности.  
  
-   Виртуальных приборов позволяют пользователям легко получить, развертывания и управления предварительно встроенных решения.  
  
    -   Клиентов можно легко переместить виртуальное устройство в любом месте в облаке.  
  
    -   Клиентов можно масштабировать виртуальных устройств или вниз динамически на основе по требованию.  
  
Дополнительные сведения о Microsoft SDN см [программного обеспечения определены сети](https://technet.microsoft.com/windows-server-docs/networking/sdn/software-defined-networking--sdn-).  
  
### <a name="what-network-functions-are-being-virtualized"></a>Какие сетевые функции, виртуализируются?  
Быстро увеличивается marketplace для функций виртуализированной сети. Следующие функции сети, виртуализируются:  
  
-   **Безопасность**  
  
    -   Брандмауэр  
  
    -   Антивирусная программа  
  
    -   От атак DDoS (распределенных отказ в обслуживании)  
  
    -   IP-адресов или Идентификаторов (Предотвращение системы или вторжения система обнаружения атак)  
  
-   **Оптимизаторы приложения или глобальной сети**  
  
-   **Edge**  
  
    -   Шлюз сайта для сайта  
  
    -   Шлюзы уровня 3  
  
    -   Маршрутизаторы  
  
    -   Переключатели  
  
    -   NAT  
  
    -   Подсистемы балансировки нагрузки (не обязательно на границе)  
  
    -   HTTP-прокси

## <a name="network-function-virtualization-in-windows-server-2016"></a>Виртуализация сетевых функций в Windows Server 2016  
  
### <a name="virtual-appliances-functions-provided-by-microsoft"></a>Функции виртуальных приборов, предоставляемые корпорацией Майкрософт  
С Windows Server 2016 представлены следующие виртуальных устройств.  
  
**Балансировка нагрузки программного обеспечения**  
  
Подсистема балансировки нагрузки уровня 4 операционной в масштабе центра обработки данных. Это аналогично версия подсистемы балансировки нагрузки в Azure, которая была развернута в масштабе в среде Azure. Дополнительные сведения о подсистеме балансировки нагрузки программного обеспечения корпорации Майкрософт см. в разделе [программной балансировки нагрузки (SLB) для SDN](https://technet.microsoft.com/library/mt632286.aspx). Дополнительные сведения о службы балансировки нагрузки Microsoft Azure см. в разделе [службы балансировки нагрузки Microsoft Azure](https://azure.microsoft.com/blog/2014/04/08/microsoft-azure-load-balancing-services/).  
  
**Шлюз**. Шлюз RAS-сервера предоставляет все комбинации следующие функции шлюза.  
  
-   **Шлюз сайта для сайта**  
  
    Протокол пограничного шлюза (BGP) обеспечивает шлюза RAS-управляются и поддержкой шлюз, который позволит вашим клиентам получать доступ и управлять их ресурсами через VPN-подключение для узла с удаленных сайтов и, позволяющая потоки сетевого трафика между виртуальными ресурсами в физических сетях облака и клиента. Дополнительные сведения о шлюза RAS-сервера см. в разделе [высокая доступность шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS-сервера](https://technet.microsoft.com/library/mt626650.aspx).  
  
-   **Шлюз переадресации**  
  
    Шлюз удаленного доступа перенаправляет трафик между виртуальными сетями и физической сети поставщика услуг размещения. Например если клиентам создать один или несколько виртуальных сетей и требуется доступ к общим ресурсам на сайте поставщика размещения физической сети, шлюз перенаправления может осуществлять маршрутизацию трафика между виртуальной и физической сети для предоставления пользователям работать в виртуальной сети со службами, которые им необходимы. Дополнительные сведения см. в разделе [высокая доступность шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS-сервера](https://technet.microsoft.com/library/mt626650.aspx).  
  
-   **Шлюзы туннеля GRE**  
  
    На основе GRE, туннели обеспечивают подключение между виртуальными сетями и внешними сетями. Так как протокол GRE является упрощенным и поддерживается GRE доступно на большинстве устройств в сети, он становится идеальным выбором для туннелирования которых шифрование данных не требуется. Поддержка GRE в межсайтовых (S2S) туннелях решает проблему перенаправления между виртуальными сетями и внешними сетями клиента с помощью мультитенантного шлюза. Дополнительные сведения о туннели GRE см. в разделе [туннелирование GRE в Windows Server 2016](https://technet.microsoft.com/library/dn765485.aspx).  
  
**Маршрутизации плоскости управления с протоколом BGP**  
  
Управления маршрутизации виртуализации сети Hyper-V (HNV) — это логические, централизованного сущность в плоскости элемента управления, который выполняет все маршруты плоскости адрес клиента и динамически определяет и обновляет распределенных маршрутизаторы шлюза RAS-сервера в виртуальной сети. Дополнительные сведения см. в разделе [высокая доступность шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS-сервера](https://technet.microsoft.com/library/mt626650.aspx).  
  
**Брандмауэр распределенного нескольких клиентов**  
  
Брандмауэр защищает сетевого уровня виртуальных сетей. Политики применяются в порт SDN vSwitch каждого из них виртуальной Машины. Защищает все потоки трафика: Восток Запад "и" Север Юг. Политики передаются на портал клиентов, а сетевой контроллер распространяет их для всех применимых узлов. Дополнительные сведения о распределенных брандмауэра нескольких клиентов см. в разделе [Обзор брандмауэра центра обработки данных](../../../sdn/technologies/network-function-virtualization/../../../sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).
