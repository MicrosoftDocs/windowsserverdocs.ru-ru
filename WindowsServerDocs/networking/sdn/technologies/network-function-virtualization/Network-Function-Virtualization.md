В программном обеспечении текущей определенных центрах обработки данных, сетевые функции, выполняемые аппаратными устройствами (например, подсистемы балансировки нагрузки, брандмауэры, маршрутизаторы, коммутаторы и т. д.), все чаще виртуализируются виде виртуальных устройств. Такая "виртуализация сетевых функций" является естественным этапом развития виртуализации серверов и сетей. Виртуальные устройства быстро развивающихся и создании совершенно новом рынке. Они по-прежнему вызвать интерес и получать импульс на обеих платформах виртуализации и облачных служб.  
  
Microsoft включил шлюза автономный как виртуальное устройство, начиная с Windows Server 2012 R2. Дополнительные сведения см. в разделе [Шлюз Windows Server](https://technet.microsoft.com/library/dn313101.aspx). Теперь с помощью Windows Server 2016, корпорация Майкрософт продолжает разверните и инвестировать в рынке функция виртуализации сети.  
  
## <a name="virtual-appliance-benefits"></a>Преимущества виртуального устройства  
Виртуальный модуль является динамической и легко изменить, так как он представляет собой готовые, настроенный виртуальную машину. Это может быть один или несколько виртуальных машин упакованы, обслуживается и обновляется как единое. Вместе с программным обеспечением конфигурируемой сети (SDN), вы получаете гибкость и динамичность, необходимые в современных облачной инфраструктуры. Пример:  
  
-   SDN представляет сети в виде в составе пула и динамического ресурса.  
  
-   SDN обеспечивает изоляцию клиентов.  
  
-   SDN повышает масштабируемость и производительность.  
  
-   Виртуальные устройства Включение простого емкости мобильности расширения и рабочей нагрузки.  
  
-   Виртуальные устройства свести к минимуму сложность эксплуатации.  
  
-   Виртуальные устройства позволяют пользователям легко получить, развертывания и управления предварительно интегрированные решения.  
  
    -   Клиенты могут легко перейти виртуальное устройство в любом месте в облаке.  
  
    -   Пользователи могут выполнять масштабирование виртуальные устройства или вниз динамически по запросу.  
  
Дополнительные сведения о Microsoft SDN см. в разделе [программно-Конфигурируемую сеть](https://technet.microsoft.com/windows-server-docs/networking/sdn/software-defined-networking--sdn-).  
  
### <a name="what-network-functions-are-being-virtualized"></a>Какие функции сети виртуализированы?  
Marketplace для функций виртуализированной сети быстро растет. Следующие функции сети виртуализированы:  
  
-   **безопасность**  
  
    -   Брандмауэр  
  
    -   Антивирусная программа  
  
    -   От атак DDoS (распределенных отказ в обслуживании)  
  
    -   IPS/IDS (систему обнаружения системы/вторжений предотвращения вторжений)  
  
-   **Оптимизаторы приложения или глобальной сети**  
  
-   **Edge**  
  
    -   Сеть сеть шлюза  
  
    -   Шлюзы L3  
  
    -   Маршрутизаторы  
  
    -   Коммутаторы  
  
    -   NAT  
  
    -   Подсистемы балансировки нагрузки (не обязательно на границе)  
  
    -   Прокси-сервер HTTP

## <a name="network-function-virtualization-in-windows-server-2016"></a>Виртуализация сетевых функций в Windows Server 2016  
  
### <a name="virtual-appliances-functions-provided-by-microsoft"></a>Виртуальные устройства функции, предоставляемые корпорацией Майкрософт  
Следующие виртуальные устройства входящие в состав Windows Server 2016:  
  
**Программная Подсистема балансировки нагрузки**  
  
Подсистема балансировки нагрузки уровня 4 крупномасштабных центра обработки данных. Это аналогичная версия балансировщик нагрузки Azure, которая была развернута в масштабе, в среде Azure. Дополнительные сведения о программной подсистемы балансировки нагрузки Майкрософт см. в разделе [программного обеспечения нагрузки балансировки (SLB) для SDN](https://technet.microsoft.com/library/mt632286.aspx). Дополнительные сведения о службы балансировки нагрузки Microsoft Azure, см. в разделе [службы балансировки нагрузки Microsoft Azure](https://azure.microsoft.com/blog/2014/04/08/microsoft-azure-load-balancing-services/).  
  
**Шлюз**. Шлюз RAS обеспечивает все комбинации следующие функции шлюза.  
  
-   **Сеть-сеть шлюза**  
  
    Шлюз RAS обеспечивает протокол пограничного шлюза (BGP)-шлюз поддерживает и поддержкой нескольких клиентов, позволяющий клиентам получать доступ к ресурсам и управлять ими через сеть сеть VPN-подключений с удаленных сайтов и позволяет потоком сетевого трафика между виртуальными ресурсами в облачных и физических сетей клиентов. Дополнительные сведения об использовании шлюза RAS см. в разделе [высокий уровень доступности шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS](https://technet.microsoft.com/library/mt626650.aspx).  
  
-   **Шлюз перенаправления**  
  
    Шлюз удаленного доступа направляет трафик между виртуальными сетями и физической сети поставщика услуг размещения. Например если клиенты создать один или несколько виртуальных сетей, а также требуется доступ к общим ресурсам в физической сети у поставщика услуг размещения, шлюз перенаправления может осуществлять маршрутизацию трафика между виртуальной и физической сети для предоставления пользователям, работающим на Виртуальная сеть со службами, которые им необходимы. Дополнительные сведения см. в разделе [высокий уровень доступности шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS](https://technet.microsoft.com/library/mt626650.aspx).  
  
-   **Шлюзы туннель GRE**  
  
    Туннели на основе GRE обеспечивают подключение между виртуальными сетями клиентов и внешними сетями. Так как протокол GRE является упрощенным и поддерживается для GRE доступна в большинстве сетевых устройств, он становится идеальным выбором для туннелирования когда шифрование данных не требуется. Поддержка GRE в межсайтовых (S2S) туннелях решает проблему перенаправления между виртуальными сетями и внешними сетями клиента с использованием мультитенантного шлюза. Дополнительные сведения о туннели GRE, см. в разделе [туннелирование GRE в Windows Server 2016](https://technet.microsoft.com/library/dn765485.aspx).  
  
**Маршрутизации плоскости управления с использованием BGP**  
  
Элемент управления маршрутизации виртуализации сети Hyper-V (HNV) — это логический, централизованное сущность в плоскости управления, которая содержит все маршруты плоскости адрес клиента и динамически изучает и обновляет распределенного шлюза RAS маршрутизаторы в виртуальной сети. Дополнительные сведения см. в разделе [высокий уровень доступности шлюза RAS](https://technet.microsoft.com/library/mt631692.aspx) и [шлюза RAS](https://technet.microsoft.com/library/mt626650.aspx).  
  
**Распределенный брандмауэр несколькими клиентами**  
  
Брандмауэр защищает сетевого уровня для виртуальных сетей. Политики применяются на порт SDN vSwitch каждого клиента виртуальной Машины. Она обеспечивает защиту всех потоков трафика: Восток Запад "и" Север Юг. Политики отправляются через службу на портале клиента и распределяет их сетевого контроллера для всех подходящих узлов. Дополнительные сведения о распределенном брандмауэре несколькими клиентами, см. в разделе [Обзор брандмауэра центра обработки данных](../../../sdn/technologies/network-function-virtualization/../../../sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).
