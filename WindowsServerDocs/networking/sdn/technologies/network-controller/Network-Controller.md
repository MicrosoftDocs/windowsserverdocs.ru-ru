# <a name="network-controller"></a>Сетевой контроллер

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

Новое в Windows Server 2016 сетевой контроллер обеспечивает централизованную программируемую точку автоматизации для управления, настройки, мониторинга и устранения неисправностей в виртуальной и физической сетевой инфраструктуре центра обработки данных. 

С помощью сетевого контроллера можно автоматизировать настройки сетевой инфраструктуры и отказаться от ручной настройки сетевых устройств и служб.

## <a name="bkmk_overview"></a>Сетевой контроллер: Обзор

Сетевой контроллер — это роль сервера, высокодоступная, масштабируемая и имеет один программирования приложений интерфейса \(API\) которого сетевой контроллер осуществляет связь с сетью и второй API, который позволяет вам обмениваться данными с сетевым контроллером.

Вы можете развернуть сетевого контроллера в домене и не доменной среде. В доменных средах сетевой контроллер выполняет проверку подлинности пользователей и устройств в сети по протоколу Kerberos; в средах, не входящих в домен необходимо развернуть сертификаты для проверки подлинности.

>[!IMPORTANT]
>Не выполняйте развертывание роли сервера сетевого контроллера на физических узлах. Для развертывания сетевого контроллера, необходимо установить роль сервера сетевого контроллера на виртуальной машине Hyper-V \(VM\), установленной на узле Hyper-V. После установки сетевого контроллера на виртуальные машины на трех разных узлах Hyper\-V, необходимо включить узлов Hyper\-V для программного обеспечения определены сети \(SDN\) путем добавления узлов для сетевого контроллера с помощью команды Windows PowerShell **New-NetworkControllerServer**. Таким образом, включается Балансировка нагрузки программного обеспечения SDN функции. Дополнительные сведения см. в разделе [New-NetworkControllerServer](https://technet.microsoft.com/itpro/powershell/windows/network-controller/new-networkcontrollerserver).

Сетевой контроллер взаимодействует с сетевыми устройствами, служб и компонентов с помощью Южный API. С его помощью сетевого контроллера можно находит сетевые устройства, отслеживает конфигурации служб и собирает необходимые сведения о сети. Кроме того Южный API-Интерфейс обеспечивает для сетевого контроллера маршрут для отправки данных в сетевую инфраструктуру, таких как изменения конфигурации, которые были внесены.

Северный API-Интерфейс контроллера предоставляет возможность извлекать данные из сетевого контроллера и использовать его для мониторинга и настройки сети.

Северный API-Интерфейс контроллера дает возможность настройки, отслеживания, устранение неполадок и развертывание новых устройств в сети с помощью Windows PowerShell, API Representational State Transfer \(REST\) или приложения управления с графическим пользовательским интерфейсом, например System Center Virtual Machine Manager.

>[!NOTE]
>Северный API-Интерфейс контроллера реализован как интерфейс REST.

Можно управлять сетью центра обработки данных с помощью сетевого контроллера с помощью приложений управления, таких как System Center Virtual Machine Manager \(SCVMM\) и \(SCOM\) System Center Operations Manager, так как сетевой контроллер позволяет настроить, мониторинг, программирование и устранение неполадок сетевой инфраструктуры, который находится под его управлением.

С помощью Windows PowerShell, API-интерфейса REST или приложения управления, можно использовать сетевого контроллера управлять следующими физическими и виртуальными сетевыми инфраструктурами:

- Виртуальные машины Hyper-V и виртуальных коммутаторов

- Брандмауэр центра обработки данных

- Шлюзы Мультитенантного \(RAS\) удаленной службы доступа к, шлюзы виртуальных и пулы шлюза

- Подсистемы балансировки нагрузки программного обеспечения

На следующем рисунке администратор использует средство управления, взаимодействующее напрямую с сетевым контроллером. Сетевой контроллер предоставляет сведения о сетевой инфраструктуре, включая виртуальной и физической инфраструктуры для средства управления и вносит изменения в конфигурацию в соответствии с действиями администратора при использовании средства.  

![Сетевой контроллер: Обзор](../../../media/Network-Controller/NetController_overview.png)  

При развертывании сетевого контроллера в проверочной лабораторной среде, роли сервера сетевого контроллера можно запустить на виртуальной машине Hyper-V \(VM\), установленной на узле Hyper-V.

Для обеспечения высокой доступности в крупных центрах данных можно развернуть кластер с помощью трех ВМ, установленных на трех или несколько узлов Hyper-V. Дополнительные сведения см. в разделе [высокая доступность сетевого контроллера](network-controller-high-availability.md).

## <a name="bkmk_features"></a>Функции сетевого контроллера

Следующие функции сетевого контроллера позволяют настраивать и управлять виртуальных и физических сетевых устройств и служб.  
  
-   [Управление брандмауэром](#bkmk_firewall)  
  
-   [Программное управление подсистемой балансировки нагрузки](#bkmk_slb)  
  
-   [Управление виртуальной сети](#bkmk_virtual)  
  
-   [Управление шлюза RAS-сервера](#bkmk_gateway)

>[!IMPORTANT]
>Сетевой контроллер Архивация и восстановление в данный момент недоступен в Windows Server 2016.
  
### <a name="bkmk_firewall"></a>Управление брандмауэром

Эта функция сетевого контроллера позволяет настраивать и управлять разрешить или запретить доступ правила брандмауэра для рабочих нагрузок виртуальных машин для Восток-Запад "и" Север-Юг сетевого трафика в центре обработки данных. Правила брандмауэра встроены в порт vSwitch рабочих нагрузок виртуальных машин, и поэтому они распределяются всех рабочих нагрузок в центре обработки данных. С помощью северного API-интерфейса, можно задать правила брандмауэра для входящего и исходящего трафика от рабочих нагрузок виртуальных Машин. Можно также настроить каждого правила брандмауэра, журнал всех данных, пропущенных или отклоненных в соответствии с правилом.  

Дополнительные сведения см. в разделе [Обзор брандмауэра центра обработки данных](../../../sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).

### <a name="bkmk_slb"></a>Программное управление подсистемой балансировки нагрузки

Эта функция сетевого контроллера позволяет включить несколько серверов для размещения ту же рабочую нагрузку, обеспечивая высокий уровень доступности и масштабируемости.  
  
Дополнительные сведения см. в разделе [балансировку нагрузки программного обеспечения & #40; SLB & #41; для SDN](../../../sdn/technologies/network-function-virtualization/Software-Load-Balancing--SLB--for-SDN.md).  
  
### <a name="bkmk_virtual"></a>Управление виртуальной сети

Эта функция сетевого контроллера позволяет производить развертывание и настройку виртуализации сети Hyper-V, включая виртуальный коммутатор Hyper-V и виртуальные сетевые адаптеры на отдельных ВМ и хранить и распространять политики виртуальной сети.

Сетевой контроллер поддерживает сетевой виртуализации универсального Routing Encapsulation (NVGRE) и виртуальные расширяемые локальной сети (VXLAN).

### <a name="bkmk_gateway"></a>Управление шлюза RAS-сервера

Эта функция сетевого контроллера позволяет развертывать, настраивать и управлять виртуальных машин (ВМ), входящих в пул шлюза RAS-сервера, предоставлять пользователям услуги шлюза. Сетевой контроллер дает возможность автоматически развертывать виртуальные машины под управлением шлюза RAS-сервера со следующими функциями:

> [!NOTE]
> В System Center Virtual Machine Manager, шлюза RAS-сервера, называется шлюз Windows Server.

- Добавления и удаления шлюзу ВМ из кластера, а затем укажите требуемого уровня резервного копирования.

- Узел узел связи виртуальной частной сети (VPN) шлюз между удаленными сетями клиентов и центром данных с помощью IPsec.

- Сеть сеть VPN-подключение шлюз между удаленными сетями клиентов и центром данных с помощью Generic Routing Encapsulation (GRE).

- Перенаправление для уровня 3.

- Протокол пограничного шлюза (BGP) маршрутизации, которые позволяют управлять маршрутизацией сетевого трафика между сетями виртуальных Машин и их удаленными сайтами.

Сетевой контроллер можно разместить другие подключения клиента на отдельных шлюзах. Можно использовать один открытый IP-адрес для подключения к шлюзу или другой общих IP-адресов для подмножества подключений. Сетевой контроллер регистрирует все изменения состояния, которые можно использовать для аудита и устранения неполадок и конфигурация шлюза.

Дополнительные сведения о протоколе BGP см. в разделе [протокола BGP & #40; BGP & #41; ](../../../../remote/remote-access/bgp/Border-Gateway-Protocol-BGP.md).

Дополнительные сведения о шлюза RAS-сервера см. в разделе [шлюза RAS-сервера для SDN](../../../sdn/technologies/network-function-virtualization/RAS-Gateway-for-SDN.md).
