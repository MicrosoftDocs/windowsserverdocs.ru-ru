# <a name="configure-quality-of-service-qos-for-a-tenant-vm-network-adapter"></a>Настройка качества обслуживания (QoS) для сетевого адаптера виртуальной Машины клиента

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

При настройке качества обслуживания для сетевого адаптера виртуальной Машины клиента, у вас есть выбор между мост для центра обработки данных \(DCB\)или программно-Конфигурируемую сеть \(SDN\) QoS.

1.  **DCB**. Можно настроить мост центра обработки данных с помощью командлетов Windows PowerShell NetQoS. Например, см. в подразделе «Включение данных мост для центра обработки» раздела [удаленный доступ к памяти (RDMA) и коммутатора внедренных коммутаторов (SET)](../../../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).

2.  **Качества обслуживания SDN**. Вы можете включить качества обслуживания SDN с помощью сетевой контроллер, который может быть присвоено ограничить пропускную способность для виртуального интерфейса для предотвращения блокировки других пользователей виртуальной Машины с большим объемом трафика.  Можно также настроить качества обслуживания SDN, чтобы зарезервировать определенным объемом пропускной способности для виртуальной Машины, чтобы убедиться, что виртуальная машина будет доступна вне зависимости от объема сетевого трафика.  

Примените все параметры качества обслуживания SDN с помощью параметров портов свойств сетевого интерфейса. См. таблицу ниже для получения дополнительных сведений.

|Имя элемента|Описание|
|------------|-----------| 
|macSpoofing| Позволяет виртуальным машинам изменить исходный аппаратный \(MAC\) адрес в исходящих пакетах на MAC-адрес не назначен виртуальной Машине.<p>Допустимые значения:<ul><li>Включено — используйте другой MAC-адрес.</li><li>Отключена — используйте только MAC-адрес назначается.</li></ul>|
|arpGuard| Разрешает только адреса, указанного в ArpFilter проходить через порт guard ARP.<p>Допустимые значения:<ul><li>Включено — разрешено</li><li>Отключено — не допускается</li></ul>|
|dhcpGuard| Разрешает или удаляет все сообщения DHCP от виртуальной Машины, которая выдает DHCP-сервера. <p>Допустимые значения:<ul><li>Включено — удаляет сообщения DHCP, так как виртуализированный сервер DHCP считается при отсутствии доверия.</li><li>Отключено — позволяет сообщение должно быть получено, так как виртуализированный сервер DHCP считается trustworthy.</li></ul>|
|stormLimit| Число пакетов (широковещательных, многоадресных и неизвестных одноадресных) в секунду виртуальной Машины может отправлять через виртуальный сетевой адаптер. Получите удаленные пакеты превысить предел в течение этого интервала в одну секунду. Нулевое значение \(0\) означает неограниченное...|
|portFlowLimit| Максимальное число потоков, может выполняться для порта. Пустое значение или ноль \(0\) означает, что не ограничено. |
|vmqWeight| Относительный вес задает степень сходства виртуального сетевого адаптера использование очереди виртуальных машин (VMQ). Допустимые значения — от 0 до 100.<p>Допустимые значения:<ul><li>0 — отключает VMQ на виртуальном сетевом адаптере.</li><li>Включает 1 – 100 – VMQ на виртуальном сетевом адаптере.</li></ul>|
|iovWeight| Относительный вес задает степень сходства адаптера виртуальной сети для единой корневой виртуализации ввода-вывода, назначенных \(SR-IOV\) виртуальной функции. <p>Допустимые значения:<ul><li>0 — отключает SR-IOV на виртуальном сетевом адаптере.</li><li>1 – 100 – позволяет SR-IOV на виртуальном сетевом адаптере.</li></ul>|
|iovInterruptModeration|<p>Допустимые значения:<ul><li>по умолчанию — настройками производителя физического сетевого адаптера определяет значение.</li><li>Адаптивная- </li><li>отключить </li><li>Низкий</li><li>средний;</li><li>Высокий уровень</li></ul><p>Если вы выберете **по умолчанию**, значение определяется настройками производителя физического сетевого адаптера.  При желании **адаптивной**, трафик среды выполнения шаблона determins частота контроля прерываний.|
|iovQueuePairsRequested| Число пар очередей оборудования, выделенных виртуальной функции SR-IOV. Если на стороне приема масштабирование \(RSS\) является обязательным и если физический сетевой адаптер, которая выполняет привязку к виртуальному коммутатору, поддерживает RSS на виртуальной функции SR-IOV, а затем более одной пары очередей является обязательным. <p>Допустимые значения: 1 до 4294967295.|
|QosSettings| Настройте следующие параметры качества обслуживания, все из которых являются необязательными: <ul><li>**outboundReservedValue** — Если outboundReservedMode «абсолютный», то значение указывает на пропускную способность в Мбит/с, гарантированно виртуального порта для передачи данных (исходящих). Если outboundReservedMode является «вес» значение указывает взвешенную долю пропускной способности гарантируется.</li><li>**outboundMaximumMbps** -указывает максимальное допустимое пропускной способности на стороне отправки, в Мбит/с, для виртуального порта (исходящий трафик).</li><li>**InboundMaximumMbps** -указывает максимальное допустимое пропускной способности на стороне приема для виртуального порта Мбит/с (входящий трафик).</li></ul> |
