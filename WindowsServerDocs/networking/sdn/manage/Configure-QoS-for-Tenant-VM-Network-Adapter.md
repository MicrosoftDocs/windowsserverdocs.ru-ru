# <a name="configure-quality-of-service-qos-for-a-tenant-vm-network-adapter"></a>Настройка качества обслуживания (QoS) для сетевого адаптера виртуальной машины клиента

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016

При настройке QoS для сетевого адаптера виртуальной машины клиента можно выбрать между мостами центра обработки данных \(DCB\)или программно-определяемой сети \(SDN\) QoS.

1.  **DCB**. Можно настроить DCB с помощью командлетов NetQoS Windows PowerShell. Пример см. в разделе "Включение моста центра обработки данных" статьи [Удаленный прямой доступ к памяти (RDMA) и включение объединения внедренных функций (Set)](../../../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).

2.  **Качество обслуживания Sdn**. Службу SDN QoS можно включить с помощью сетевого контроллера, который можно настроить для ограничения пропускной способности виртуального интерфейса, чтобы предотвратить блокирование другими пользователями виртуальной машины с большим трафиком.  Можно также настроить SDN QoS для резервирования определенного объема пропускной способности виртуальной машины, чтобы обеспечить доступность виртуальной машины независимо от объема сетевого трафика.  

Примените все параметры службы SDN QoS с помощью параметров порта в свойствах сетевого интерфейса. Дополнительные сведения см. в таблице ниже.

|Имя элемента|Описание|
|------------|-----------| 
|макспуфинг| Позволяет виртуальным машинам изменять исходный контроль доступа к носителю \(MAC\) адрес в исходящих пакетах на MAC-адрес, не назначенный виртуальной машине.<p>Допустимые значения:<ul><li>Включено — используйте другой MAC-адрес.</li><li>Отключено — используйте только назначенный ей MAC-адрес.</li></ul>|
|арпгуард| Разрешает протоколу ARP только адреса, указанные в Арпфилтер, для передачи через порт.<p>Допустимые значения:<ul><li>Enabled — разрешено</li><li>Disabled — запрещено</li></ul>|
|дхкпгуард| Разрешает или удаляет все сообщения DHCP с виртуальной машины, которая заявляет DHCP-сервер. <p>Допустимые значения:<ul><li>Включено — удаляет сообщения DHCP, так как виртуализированный DHCP-сервер считается ненадежным.</li><li>Disabled — разрешает получение сообщения, так как виртуализированный DHCP-сервер считается надежным.</li></ul>|
|стормлимит| Число пакетов (широковещательной, многоадресной рассылки и неизвестной одноадресной рассылки) в секунду, в течение которого виртуальная машина может отправить через виртуальный сетевой адаптер. Пакеты, не превышающие предельное значение, в течение этого интервала в 1 секунду. Нулевое значение \(0\) означает отсутствие ограничения.|
|портфловлимит| Максимальное число потоков, разрешенных для выполнения порта. Пустое или нулевое значение \(0\) означает отсутствие ограничения. |
|вмквеигхт| Относительный вес описывает сходство виртуального сетевого адаптера с использованием очереди виртуальных машин (VMQ). Диапазон значений — от 0 до 100.<p>Допустимые значения:<ul><li>0 — отключает VMQ на виртуальном сетевом адаптере.</li><li>1-100 — включает VMQ на виртуальном сетевом адаптере.</li></ul>|
|иоввеигхт| Относительный вес задает сходство виртуального сетевого адаптера с назначенной виртуализацией \(SR-IOV\) виртуальной функции. <p>Допустимые значения:<ul><li>0 — отключает SR-IOV на виртуальном сетевом адаптере.</li><li>1-100 — включает SR-IOV на виртуальном сетевом адаптере.</li></ul>|
|иовинтерруптмодератион|<p>Допустимые значения:<ul><li>по умолчанию — значение определяется поставщиком физического сетевого адаптера.</li><li>Отключение </li><li>отключить </li><li>Низшую</li><li>средний;</li><li>Высоком</li></ul><p>Если выбран параметр **по умолчанию**, значение определяется поставщиком физического сетевого адаптера.  Если выбран параметр, **адаптивный**шаблон трафика среды выполнения обоценивает частоту контроля прерываний.|
|иовкуеуепаирсрекуестед| Число пар очередей оборудования, выделенных для виртуальной функции SR-IOV. Если требуется масштабирование на стороне приема \(RSS\), и если физический сетевой адаптер, привязанный к виртуальному коммутатору, поддерживает RSS в виртуальных функциях SR-IOV, то требуется более одной пары очередей. <p>Допустимые значения: от 1 до 4294967295.|
|коссеттингс| Настройте следующие параметры QoS, все из которых являются необязательными. <ul><li>**аутбаундресерведвалуе** — если аутбаундресерведмоде — "Absolute", то значение указывает пропускную способность (в Мбит/с), которую гарантированно передается виртуальному порту для передачи (исходящего трафика). Если Аутбаундресерведмоде имеет значение "Weight", то оно указывает взвешенную часть гарантии пропускной способности.</li><li>**аутбаундмаксимуммбпс** — указывает максимально допустимую пропускную способность на стороне отправки (в Мбит/с) для виртуального порта (исходящего трафика).</li><li>**Инбаундмаксимуммбпс** — указывает максимально допустимую пропускную способность приема для виртуального порта (входящий трафик) в Мбит/с.</li></ul> |
