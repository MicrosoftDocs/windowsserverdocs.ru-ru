# <a name="whats-new-in-dns-server-in-windows-server"></a>Новые возможности DNS-сервера в Windows Server

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе описываются функции сервера доменных имен (DNS), добавленные или измененные в Windows Server 2016.  
  
В Windows Server 2016 DNS-сервер обеспечивает расширенную поддержку в следующих областях.  
  
|Функциональные возможности|Новые и совершенствуют существующие|Описание|  
|-----------------|-------------------|---------------|  
|Политики DNS|Новые функции|Можно настроить политики DNS, чтобы указать, как DNS-сервер отвечает на запросы DNS. Ответов DNS может быть основан на IP-адрес клиента (местоположение), времени суток и несколько других параметров. Политики DNS включить DNS, учитывающих местоположение, трафика управления, балансировки нагрузки, разделенная система DNS и других сценариев.|  
|Скорость отклика, ограничивая (RRL)|Новые функции|Вы можете включить ограничения скорости ответа на DNS-серверы. Таким образом, можно избежать возможность вредоносных систем с помощью DNS-серверы для инициирования атаки отказа в обслуживании на DNS-клиента.|  
|Проверка подлинности на основе DNS именованных объектов (DANE)|Новые функции|Записывает TLSA (проверка подлинности безопасности уровня транспорта) можно использовать для предоставления информации DNS-клиенты, которые состояния какие ЦС, они должны ожидать сертификат для имени домена. Это предотвращает атаки в середине где кто-то повреждены кэш DNS для указания собственные веб-сайта и предоставить сертификат, который они выпущены с другой ЦС.|  
|Поддержка Неизвестный записи|Новые функции|Вы можете добавить записи, которые явно не поддерживаются сервером DNS-сервера Windows, с помощью функции неизвестной записи.|  
|IPv6 корневых ссылок|Новые функции|Можно использовать собственный протокол IPV6, поддерживать корневые ссылки, чтобы выполнять разрешение имен Интернета с помощью корневых серверов IPV6.|  
|Поддержка Windows PowerShell|Улучшена|Новые командлеты Windows PowerShell доступны для DNS-сервера.|  
  
## <a name="dns-policies"></a>Политики DNS

Политики DNS можно использовать для управления трафиком на основе географического расположения, интеллектуальных ответов DNS на основе времени дня, для управления один DNS-сервер настроен для развертывания вычислительных мощностей split\ применение фильтров на запросы DNS и многое другое. Следующие элементы содержат более подробные сведения об этих возможностях.

-   **Балансировки нагрузки приложений.** При развертывании нескольких экземпляров одного приложения в разных местах можно использовать политики DNS для балансировки нагрузки трафика между экземплярами другого приложения, динамическое распределение нагрузки трафика для приложения.

-   **Расположение Geo\ на основе трафика управления.** Чтобы разрешить основного и дополнительного DNS-серверов, отвечать на запросы DNS-клиента на основе географического расположения для клиента и ресурсов, к которой клиент пытается подключиться, предоставление ближайший ресурс IP-адрес клиента, можно использовать политики DNS. 

-   **Разделите систему DNS.** С помощью DNS вычислительных мощностей split\ DNS-записи разбиты на разных областей зоны на том же сервере DNS, и DNS-клиенты получают ответ, в зависимости от того, являются ли клиенты внутренних или внешних клиентов. На автономных серверах DNS можно настроить вычислительных мощностей split\ DNS для зон, интегрированных в Active Directory или для зон.

-   **Фильтрация.** Можно настроить политики DNS для создания фильтрах запросов, которые основываются на критериях, предоставленных вами. Фильтры запроса в политике DNS позволяют настроить DNS-сервер для ответа особым образом на основе запроса DNS и DNS-клиента, который отправляет запросы DNS. 
-   **Расследований.** Можно использовать политики DNS для перенаправления вредоносных DNS-клиентов на несуществующие отличных IP-адрес вместо направив их компьютера, к которому они пытаются получить доступ.

-   **Время дня, на основе перенаправления.** Можно использовать политику DNS для распределения трафика приложения по географически различных экземпляров приложения с помощью политики DNS, которые основаны на время дня. 
  
Можно также использовать политики DNS для интегрированных с Active Directory DNS зоны.

Дополнительные сведения см. в разделе [руководство по сценарию политики DNS](deploy/DNS-Policies-Overview.md).

## <a name="response-rate-limiting"></a>Ограничение скорости ответа

Можно настроить параметры RRL, определяющие, как реагировать на запросы к DNS-клиента, когда сервер получает несколько запросов, предназначенные для того же клиента. Таким образом, можно запретить пользователю отправки атаки отказа в обслуживании (Dos), с использованием DNS-серверов. Например роботе net разрешено отправлять запросы на DNS-сервер, используя IP-адрес третий компьютер в качестве запрашивающей стороны. Без RRL DNS-серверы могут отвечать на все запросы, переполнение третий компьютер. При использовании RRL, можно настроить следующие параметры:  
  
-   **Откликов в секунду**. Это максимальное число попыток один ответ будет выдано клиента в пределах одной секунды.  
  
-   **Ошибок в секунду**. Это максимальное число попыток ответ на ошибку будет отправки того же клиента в пределах одной секунды.  
  
-   **Окно**. Это время в секундах, для которых ответы на клиенте будет приостановлено, если выполняются слишком много запросов.  
  
-   **Скорость утечки**. Это, как часто DNS-сервер будет отвечать на запрос во время приостановки ответы. Например если сервер приостанавливает ответы клиента на 10 секунд, а скорость утечки — 5, сервера по-прежнему будет отвечать на один запрос каждые 5 запросов, отправляемых. Это позволяет клиентам, зарегистрированным для получения ответов даже в том случае, если DNS-сервер — применение скорость ответа, ограничивая их подсети или полное доменное имя.  
  
-   **Скорость TC**. Это позволяет сообщить клиенту попробуйте подключить при использовании протокола TCP, когда приостанавливаются ответа для клиента. Так если показатели TC 3, и сервер приостанавливает ответы на заданном клиенте, сервер будет выдавать запрос подключения TCP для каждые 3 запросов, полученных. Убедитесь, что значение TC скорость меньше скорости утечка, чтобы предоставить возможность подключения через TCP перед утечка ответы клиента.  
  
-   **Максимальное количество ответов**. Это максимальное число ответы, сервер будет выдавать клиенту во время приостановки ответы.  
  
-   **Белый список доменов**. Это список доменов, которые должны быть исключены из RRL параметры.  
  
-   **Белый список подсетей**. Это список подсетей должны быть исключены из RRL параметры.  
  
-   **Белый список интерфейсов сервера**. Это список интерфейсов сервера DNS должны быть исключены из RRL параметры.  
  
## <a name="dane-support"></a>Поддержка DANE

Можно использовать поддержку DANE \ (RFC 6394 и 6698\), чтобы указать для клиентов DNS размещенного ЦС, они должны ожидать сертификаты, должен быть выдан на имена доменов в DNS-сервера. Это предотвращает формы в середине атаки, когда кто-то может привести к повреждению кэш DNS и указывать собственные IP-адрес DNS-имя.  
  
Например представьте узла безопасный веб-сайт, который использует протокол SSL в www.contoso.com с помощью сертификата из известного центра с именем CA1. Кто-то по-прежнему смогут получить сертификат для www.contoso.com из разных, не так известной, сертификат центра с именем CA2. Затем сущности, размещения фальшивых www.contoso.com веб-сайта может быть повреждения кэш DNS для клиента или сервера для точки www.contoto.com их фальшивых сайту. Пользователю предлагается сертификат из CA2 и может просто подтвердить его и подключитесь к фальшивых сайта. DANE клиент будет сделать запрос на DNS-сервер для домена contoso.com, запрашивает запись TLSA и узнайте, что сертификат к www.contoso.com, проблемы по CA1. Если предъявляется сертификат от другого ЦС, подключение будет прервано.  
  
## <a name="unknown-record-support"></a>Поддержка Неизвестный записи

«Запись Неизвестный» является записи Ресурса, формат которых RDATA известен на DNS-сервер. Вновь добавлена поддержка типы неизвестной записи (RFC 3597) означает, что вы можете добавить неподдерживаемые типы записей в зонах Windows DNS server в двоичном формате на лету. Windows кэширования Сопоставитель уже есть возможность обработки типов записей неизвестно. DNS-сервер Windows будет выполнять запись определенных обработку для неизвестных записей, но будет отправить его обратно в ответы поступление запросы для него.  
  
## <a name="ipv6-root-hints"></a>IPv6 корневых ссылок

Корневые ссылки IPV6 как публикует IANA, были добавлены на DNS-сервер windows. Запросы имен Интернета теперь можно использовать IPv6 корневых серверов для выполнения разрешения имен.

## <a name="windows-powershell-support"></a>Поддержка Windows PowerShell

В Windows Server 2016 представлены следующие новые командлеты Windows PowerShell и параметры.
  
-   **Добавить DnsServerRecursionScope**. Этот командлет создает новую область рекурсию на DNS-сервер. Области рекурсии используются с помощью политик DNS-сервера можно указать список серверов пересылки для использования в запрос DNS.  
  
-   **Remove-DnsServerRecursionScope**. Этот командлет удаляет существующий области рекурсию.  
  
-   **SET-DnsServerRecursionScope**. Этот командлет изменяет параметры существующей области рекурсию.  
  
-   **Get-DnsServerRecursionScope**. Этот командлет возвращает сведения об областях существующих рекурсию.  
  
-   **Добавить DnsServerClientSubnet**. Этот командлет создает подсеть клиента DNS. Подсети используются DNS политиками, чтобы определить, где находится DNS-клиента.  
  
-   **Remove-DnsServerClientSubnet**. Этот командлет удаляет существующий подсети клиента DNS.  
  
-   **SET-DnsServerClientSubnet**. Этот командлет изменяет параметры существующей подсети клиента DNS.  
  
-   **Get-DnsServerClientSubnet**. Этот командлет извлекает сведения о существующих подсети клиента DNS.  
  
-   **Добавить DnsServerQueryResolutionPolicy**. Этот командлет создает новую политику разрешения запросов DNS. Политики разрешения запросов DNS используются для указания как это сделать, или если запрос ответа, на основе различных критериев.  
  
-   **Remove-DnsServerQueryResolutionPolicy**. Этот командлет удаляет существующих политик DNS.  
  
-   **SET-DnsServerQueryResolutionPolicy**. Этот командлет изменяет параметры существующей политики DNS.  
  
-   **Get-DnsServerQueryResolutionPolicy**. Этот командлет возвращает информацию об имеющихся политик DNS.  
  
-   **Enable-DnsServerPolicy**. Этот командлет позволяет существующих политик DNS.  
  
-   **Отключение DnsServerPolicy**. Этот командлет отключает существующих политик DNS.  
  
-   **Добавить DnsServerZoneTransferPolicy**. Этот командлет создает новый передачи политика зоны DNS-серверов. Политики передачи зоны DNS укажите, следует ли запретить или игнорировать передачу зоны на основе различных критериев.  
  
-   **Remove-DnsServerZoneTransferPolicy**. Этот командлет удаляет существующие политики передачи зоны DNS сервера.  
  
-   **SET-DnsServerZoneTransferPolicy**. Этот командлет изменяет параметры существующей политики передачи зоны DNS сервера.  
  
-   **Get-DnsServerResponseRateLimiting**. Этот командлет извлекает параметры RRL.  
  
-   **SET-DnsServerResponseRateLimiting**. Этот командлет изменяет RRL settigns.  
  
-   **Добавить DnsServerResponseRateLimitingExceptionlist**. Этот командлет создает список исключений RRL на DNS-сервер.  
  
-   **Get-DnsServerResponseRateLimitingExceptionlist**. Этот командлет извлекает списки excception RRL.  
  
-   **Remove-DnsServerResponseRateLimitingExceptionlist**. Этот командлет удаляет существующий RRL список исключений.  
  
-   **SET-DnsServerResponseRateLimitingExceptionlist**. Этот командлет изменяет RRL списки исключений.  
  
-   **Добавить DnsServerResourceRecord**. Этот командлет был обновлен для поддержки неизвестных запись типа.  
  
-   **Get-DnsServerResourceRecord**. Этот командлет был обновлен для поддержки неизвестных запись типа.  
  
-   **Remove-DnsServerResourceRecord**. Этот командлет был обновлен для поддержки неизвестных запись типа.  
  
-   **SET-DnsServerResourceRecord**. Этот командлет был обновлен для поддержки неизвестных запись типа
