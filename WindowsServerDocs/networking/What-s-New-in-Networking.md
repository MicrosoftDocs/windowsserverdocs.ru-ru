## <a name="bkmk_features"></a>Новые сетевые средства и технологии

Сеть — это Основополагающая часть платформы программного обеспечения определенный центр обработки данных (SDDC) и Windows Server 2016 предоставляет новые и улучшенные технологии программно-определяемой сети (SDN), которые помогут перейти на полнофункциональное решение SDDC для вашей организации.  
  
При управлении сетями как программно-определяемым ресурсом, описывающие требования к инфраструктуре приложения один раз и выберите где выполняется приложение — локально или в облаке.  Эта согласованность означает, что стали упрощает масштабирование приложений и позволяет легко запускать приложения, где угодно, с уверенностью в безопасности, производительность, качество обслуживания и доступности.  
  
Следующие разделы содержат сведения об этих новые сетевые возможности и технологии.  
  
### <a name="software-defined-networking-infrastructure"></a>Программно-конфигурируемой сети инфраструктуры

Ниже приведены новых и улучшенных технологий инфраструктуры SDN.  
  
-   **Сетевой контроллер**. Новое в Windows Server 2016, сетевой контроллер обеспечивает централизованную программируемую точку автоматизации для управления, настройки, мониторинга и устранения неисправностей в виртуальной и физической сетевой инфраструктуре центра обработки данных. С помощью сетевого контроллера можно автоматизировать настройки сетевой инфраструктуры и отказаться от ручной настройки сетевых устройств и служб. Дополнительные сведения см. в разделе [сетевого контроллера](sdn/technologies/network-controller/Network-Controller.md) и [развертывание программного обеспечения определены сетей с помощью скриптов](https://technet.microsoft.com/library/mt427380.aspx).  
  
-   **Виртуальный коммутатор Hyper-V**. Виртуальный коммутатор Hyper-V выполняется на узлах Hyper-V и позволяет создавать распределенные переключения и маршрутизации, и слой принудительного применения политики, это выровненных, так и совместимый с Microsoft Azure. Дополнительные сведения см. в разделе [Виртуальный коммутатор Hyper-V](../virtualization/hyper-v-virtual-switch/Hyper-V-Virtual-Switch.md).  
  
-   **Функция виртуализации (NFV) сети**. В программном обеспечении текущей определенных центрах обработки данных, сетевые функции, выполняемые аппаратными устройствами (например, подсистемы балансировки нагрузки, брандмауэры, маршрутизаторы, коммутаторы и т. д.) все чаще разворачиваются как виртуальные устройства. Такая "виртуализация сетевых функций" является естественным этапом развития виртуализации серверов и сетей. Виртуальные устройства быстро развивающихся и создании совершенно новом рынке. Они по-прежнему вызвать интерес и получать импульс на обеих платформах виртуализации и облачных служб. Следующие технологии NFV доступны в Windows Server 2016.  
  
    -   **Брандмауэр центра обработки данных**. Это распределенный брандмауэр предоставляет детального управления доступом (ACL), что позволяет применять политики брандмауэра на уровне интерфейса виртуальной Машины или на уровне подсети.  
  
        Дополнительные сведения см. в разделе [Обзор брандмауэра центра обработки данных](sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).  
  
    -   **Шлюз RAS**. Шлюз удаленного доступа можно использовать для маршрутизации трафика между виртуальными сетями и физическими сетями, включая VPN-подключения сайт сайт из облачного центра обработки данных к удаленным сайтам ваших клиентов. В частности, можно развернуть Internet Key Exchange версии 2 (IKEv2) сеть сеть виртуальные частные сети (VPN), VPN уровня 3 (L3) и шлюзов протоколом (GRE). Кроме того теперь поддерживается пулов шлюзов и избыточностью M + N шлюзов; и границы шлюза протокола BGP с возможностями рефлектор маршрутов обеспечивает динамическую маршрутизацию между сетями для всех сценариев шлюза (IKEv2 для VPN, GRE VPN и VPN уровня 3).  
  
        Дополнительные сведения см. в разделе [новые возможности шлюза RAS](sdn/technologies/network-function-virtualization/What-s-New-in-RAS-Gateway.md) и [шлюз RAS](sdn/technologies/network-function-virtualization/RAS-Gateway-for-SDN.md).  
          
    - **Программная Подсистема балансировки нагрузки (SLB) и преобразование сетевых адресов (NAT)**. Север Юг и запад Восток слоя 4 подсистемы балансировки нагрузки и NAT повышает пропускную способность, поддерживая прямым ответом от сервера, с которым обратный сетевой трафик можно обойти, балансировка нагрузки мультиплексор.  
       Дополнительные сведения см. в разделе [программная Балансировка нагрузки &#40;SLB&#41; для SDN](sdn/technologies/network-function-virtualization/Software-Load-Balancing--SLB--for-SDN.md).  
  
    Дополнительные сведения см. в разделе [виртуализация сетевых функций](sdn/technologies/network-function-virtualization/Network-Function-Virtualization.md).  
  
-   **Стандартизованные протоколы**. Сетевой контроллер использует Representational State Transfer (REST) Северный интерфейс с полезными данными JavaScript Object Notation (JSON). Сетевой контроллер южного интерфейс использует Open коммутатор vSwitch протокола управления базы данных (OVSDB).  
  
-   **Технологии гибкой инкапсуляции**. Эти технологии работают в плоскости данных и поддержки виртуальной Extensible LAN (VxLAN) и сети Virtualization Generic Routing Encapsulation (NVGRE). Дополнительные сведения см. в разделе [туннелирование GRE в Windows Server 2016](../remote/remote-access/ras-gateway/gre-tunneling-windows-server.md).  
  
Дополнительные сведения о SDN, см. в разделе [программно-Конфигурируемую сеть &#40;SDN&#41;](sdn/software-defined-networking.md).  
  
### <a name="cloud-scale-fundamentals"></a>Основы масштабирования облака
 
Теперь доступны следующие Основы масштабирования облака.  
  
-   **Конвергентный сетевой адаптер (NIC)**. Конвергентную сетевую КАРТУ можно использовать один сетевой адаптер для управления, хранения с поддержкой удаленного доступа к памяти RDMA и трафик клиентов. Это уменьшает капитальных затрат, которые связаны с каждого сервера в центре обработки данных, так как требуется меньшее количество сетевых адаптеров, чтобы управлять различными типами трафика на сервере.  
  
-   **Packet Direct**.  Packet Direct предоставляет пропускную способность трафика высокой нагрузки на сеть и инфраструктура обработки пакетов с низкой задержкой.  
  
-   **Switch Embedded Teaming (SET)**.        НАБОР — это решение, объединение Сетевых карт, который интегрирован в виртуальном коммутаторе Hyper-V. НАБОР позволяет карт до восьми физических сетевых Адаптеров в одной команде SET, что повышает доступность и обеспечивает отработку отказа. В Windows Server 2016 можно создать НАБОР команд, которые доступны только для использование Server Message Block (SMB) и RDMA. Кроме того можно использовать НАБОР команд для распределения сетевого трафика для виртуализации сети Hyper-V. Дополнительные сведения см. в разделе [удаленный доступ к памяти &#40;RDMA&#41; и Switch Embedded Teaming &#40;ЗАДАТЬ&#41;](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).  
  
## <a name="bkmk_existing"></a>Новые возможности для дополнительных сетевых технологий

Этот раздел содержит сведения о новых возможностях для знакомых сетевых технологий.
  
## <a name="bkmk_dhcp"></a>DHCP  
Протокол DHCP является стандартом IETF, разработанным для облегчения управления и упрощения настройки узлов сети, основанной на протоколе TCP/IP, например частной интрасети. С использованием службы DHCP-сервера процесс настройки протокола TCP/IP на DHCP-клиентах происходит автоматически.  
  
Дополнительные сведения см. в разделе [новые возможности DHCP](technologies/dhcp/What-s-New-in-DHCP.md).  
  
## <a name="bkmk_dns"></a>DNS  
Служба DNS представляет собой систему, используемую в сетях TCP/IP для именования компьютеров и сетевых служб. DNS-имена позволяют находить компьютеры и службы с помощью понятных имен. Когда пользователь вводит DNS-имя в приложении, службы DNS могут сопоставить имя с другими сведениями, например с IP-адресом.  
  
Ниже приведены сведения о DNS-клиент и DNS-сервер.  
  
### <a name="bkmk_dnsc"></a>DNS-клиента  
Ниже перечислены новые или улучшенные клиентские технологии DNS.  
  
-   **Привязка службы DNS-клиент**. В Windows 10 служба DNS-клиента обеспечивает расширенную поддержку для компьютеров с более чем один сетевой интерфейс.  
  
Дополнительные сведения см. в разделе [новые возможности DNS-клиента в Windows Server 2016](dns/What-s-New-in-DNS-Client.md)  
  
### <a name="bkmk_dnss"></a>DNS-сервер  
Ниже перечислены новые или улучшенные технологии сервера DNS.  
  
-   **Политики DNS**.  Можно настроить политики DNS, чтобы указать, как DNS-сервер отвечает на запросы DNS. Ответы DNS, которые могут быть основаны на IP-адрес клиента (расположение), время дня и несколько других параметров. Политики DNS включить DNS, привязанный к местонахождению, управление трафиком, балансировка нагрузки, разделенная система DNS и других сценариев.  
  
-   **Поддержка Nano Server для файла на основе DNS**, можно развернуть сервер DNS в Windows Server 2016 на основе образа Nano Server. Этот параметр развертывания доступен для вас, если вы используете файл на основе DNS. Работающей DNS-сервером на основе образа Nano Server можно запустить DNS-серверы с сокращенный быстрого загрузки и свернутое установки исправлений.  
  
    > [!NOTE]   
    > Встроенная DNS Active Directory не поддерживается на сервере Nano Server.  
  
-   **Ответ оценить ограничивающим (RRL)**.  Вы можете включить ограничение скорости ответа на DNS-серверы. Таким образом, вы защититься от вредоносных систем, с помощью DNS-серверы для инициирования атаки отказа в обслуживании на DNS-клиента.  
  
-   **Проверка подлинности на основе DNS именованных сущностей (DANE)**.   Записи TLSA (проверка подлинности безопасности уровня транспорта) можно использовать для предоставления информации о DNS-клиенты, сообщающие некий того, какой центр сертификации (ЦС) они должны ожидать сертификат для доменного имени. Это предотвращает атаки man-in--middle где кто-то поврежденный кэш DNS для указания собственных веб-сайтов и укажите сертификат, который они выданных разные ЦС.  
  
-   **Поддержка неизвестных записи**.   
     Вы можете добавить записи, которые явно не поддерживаются Windows DNS-сервером, используя функцию неизвестной записи.  
  
-   **Корневые ссылки IPv6**.   
     Можно использовать собственный протокол IPV6, поддерживают корневые ссылки для разрешения имен Интернета, с помощью корневые серверы IPV6.  
  
-   **Улучшенная поддержка Windows PowerShell**.   
      Новые командлеты Windows PowerShell доступны для DNS-сервера.  
  
Дополнительные сведения см. в разделе [новые возможности DNS-сервера в Windows Server 2016](dns/What-s-New-in-DNS-Server.md)  
  
## <a name="bkmk_GRE"></a>Туннелирование GRE  
Шлюз RAS теперь поддерживает туннели протоколом (GRE) высокий уровень доступности для подключения сеть-сеть и избыточностью M + N шлюзов. GRE — это упрощенный протокол туннелирования, который может инкапсулировать разнообразные протоколы сетевого уровня внутри виртуальных каналов "точка-точка" в IP-сети.  
  
Дополнительные сведения см. в разделе [туннелирование GRE в Windows Server 2016](../remote/remote-access/ras-gateway/gre-tunneling-windows-server.md).  
  
## <a name="HNV"></a>Виртуализация сети Hyper-V  
Представленный в Windows Server 2012, виртуализация сети Hyper-V (HNV) обеспечивает виртуализацию сетей клиентов на основе общей физической сетевой инфраструктуры. С минимальными изменениями, необходимых на физической сетевой инфраструктуре, HNV дают поставщикам услуг необходимую гибкость для развертывания и перехода клиентских рабочих нагрузок в любом месте в трех облаках: облако поставщика службы, частное облако или общедоступное облако Microsoft Azure.  
  
Дополнительные сведения см. в разделе [новые возможности виртуализации сети Hyper-V в Windows Server 2016](sdn/technologies/hyper-v-network-virtualization/whats-new-hyperv-network-virtualization-windows-server.md)  
  
## <a name="bkmk_ipam"></a>IPAM  
IPAM предоставляет широко настраиваемые возможности администрирования и мониторинга инфраструктуры DNS и IP-адрес в сети организации. С помощью IPAM, можно отслеживать, аудита и управление серверами под управлением конфигурации протокола DHCP (Dynamic Host) и доменных имен (DNS).  
  
-   **Расширенное управление IP-адресами**.  
     Расширенные возможности IPAM для сценариев, таких как обработка IPv4 /32 и IPv6 /128 подсети и поиск свободного IP-адрес подсети и диапазоны в блок IP-адресов.  
  
-   **Расширенное управление DNS**.  
     IPAM поддерживает запись ресурса DNS, сервер условной пересылки и Управление зонами DNS для присоединенных к домену интегрированный с Active Directory и файловой поддержкой DNS-серверов.  
  
-   **Интегрированное DNS, DHCP и IP-адрес (DDI) Управление**.  
     Несколько новых возможностей и интеграции жизненного цикла управления включено, таких как визуализация всех ресурсов DNS адресов записей, которые относятся к IP-адрес, автоматического учета IP-адресов на основе записей ресурсов DNS и управление жизненным циклом IP-адресов для операций DNS и DHCP.  
  
-   **Поддержка нескольких лесах Active Directory**.  
     IPAM можно использовать для управления серверами DNS и DHCP несколько лесов Active Directory, если имеется двустороннее отношение доверия между лесом, где установлен IPAM и каждого из удаленных лесах.  
  
-   **Поддержка Windows PowerShell для управления доступом на основе ролей**.  
     Windows PowerShell можно использовать для задания области доступа IPAM объектов.  
  
Дополнительные сведения см. в разделе [новые возможности IPAM](technologies/ipam/What-s-New-in-IPAM.md) и [управление IPAM](technologies/ipam/Manage-IPAM.md).
