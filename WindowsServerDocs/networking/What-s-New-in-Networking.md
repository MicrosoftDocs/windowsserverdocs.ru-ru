## <a name="bkmk_features"></a>Новые сетевые средства и технологии

Сетевое взаимодействие — это Основополагающая часть платформы программно определенные центра обработки данных (SDDC), и Windows Server 2016 предоставляет новые и улучшенные технологии программно-конфигурируемые сети (SDN), помогающие вашей организации перейти на полнофункциональное решение SDDC.  
  
При управлении сетями как программно-определяемым ресурсом, можно однократно описать требования к инфраструктуре приложения и затем выбрать которой работает приложение — на локальном компьютере или в облаке.  Такая согласованность означает, что стали упрощает масштабирование приложений и позволяет легко запускать приложения, в любом месте с полной уверенностью в безопасности, производительности, качестве обслуживания и доступности.  
  
Следующие разделы содержат сведения об этих новые сетевые возможности и технологии.  
  
### <a name="software-defined-networking-infrastructure"></a>Программно-конфигурируемой сети инфраструктуры

Ниже перечислены новые и улучшенные технологии SDN инфраструктуры.  
  
-   **Сетевой контроллер**. Новое в Windows Server 2016 сетевой контроллер обеспечивает централизованную программируемую точку автоматизации для управления, настройки, мониторинга и устранения неисправностей в виртуальной и физической сетевой инфраструктуре центра обработки данных. С помощью сетевого контроллера можно автоматизировать настройки сетевой инфраструктуры и отказаться от ручной настройки сетевых устройств и служб. Дополнительные сведения см. в разделе [сетевой контроллер](sdn/technologies/network-controller/Network-Controller.md) и [развертывание программного обеспечения определены сетей с помощью сценариев](https://technet.microsoft.com/library/mt427380.aspx).  
  
-   **Виртуальный коммутатор Hyper-V**. Виртуальный коммутатор Hyper-V выполняется на узлах Hyper-V и позволяет создавать распределенных переключение и маршрутизации, и уровень принудительного применения политики, ориентированный и совместимые с Microsoft Azure. Дополнительные сведения см. в разделе [виртуального коммутатора Hyper-V](../virtualization/hyper-v-virtual-switch/Hyper-V-Virtual-Switch.md).  
  
-   **Функция виртуализации (NFV) сети**. В современных программного обеспечения определенным центрах обработки данных сетевые функции, выполняемые аппаратными устройствами (такими как подсистемы балансировки нагрузки, брандмауэры, маршрутизаторы, коммутаторы и т. д) все чаще развернутым в качестве виртуальных устройств. Такая «виртуализация сетевых функций» является естественным этапом развития виртуализации серверов и сети. Виртуальных приборов быстро развивающихся и создание нового рынка. Они по-прежнему вызвать интерес и получить импульс в обеих платформах виртуализации и облачных служб. В Windows Server 2016 доступны следующие технологии NFV.  
  
    -   **Брандмауэр центра обработки данных**. Брандмауэр распределенного предоставляет детальное управления доступом (ACL), которая позволяет применять политики брандмауэра на уровне интерфейса виртуальной Машины или на уровне подсети.  
  
        Дополнительные сведения см. в разделе [Обзор брандмауэра центра обработки данных](sdn/technologies/network-function-virtualization/Datacenter-Firewall-Overview.md).  
  
    -   **Шлюз RAS**. Шлюз RAS-сервера можно использовать для маршрутизации трафика между виртуальными сетями и физическими сетями, включая сеть сеть VPN-подключений в облаке центра обработки данных к удаленным сайтам ваших клиентов. В частности, можно развернуть Internet Key Exchange версии 2 (IKEv2) для узла виртуальные частные сети (VPN), VPN уровня 3 (уровня 3) и шлюзов Generic Routing Encapsulation (GRE). Кроме того теперь поддерживается пулы шлюза и избыточности + N, M шлюзов; и протокол пограничного шлюза (BGP) с возможностями рефлектор маршрутов динамической маршрутизации между сетями для всех сценариев шлюза (IKEv2 VPN, GRE VPN и уровня 3 VPN).  
  
        Дополнительные сведения см. в разделе [новые возможности шлюза RAS-сервера](sdn/technologies/network-function-virtualization/What-s-New-in-RAS-Gateway.md) и [шлюза RAS-сервера для SDN](sdn/technologies/network-function-virtualization/RAS-Gateway-for-SDN.md).  
          
    - **Балансировка нагрузки программного обеспечения (SLB) и преобразование сетевых адресов (NAT)**. Север Юг и Восток Запад уровня 4 подсистемы балансировки нагрузки и NAT повышает пропускную способность с помощью поддержки прямой возвращает сервера, с которым возврата сетевого трафика, может обойти балансировки нагрузки мультиплексор.  
       Дополнительные сведения см. в разделе [балансировку нагрузки программного обеспечения & #40; SLB & #41; для SDN](sdn/technologies/network-function-virtualization/Software-Load-Balancing--SLB--for-SDN.md).  
  
    Дополнительные сведения см. в разделе [виртуализация сетевых функций](sdn/technologies/network-function-virtualization/Network-Function-Virtualization.md).  
  
-   **Стандартизованы протоколы**. Сетевой контроллер использует Representational State Transfer (REST) Северный интерфейс с помощью нотации объектов JavaScript (JSON) полезных данных. Южный интерфейс сетевого контроллера использует открыть виртуальный коммутатор протокола управления базы данных (OVSDB).  
  
-   **Технологии гибкие инкапсуляции**. Эти технологии работают на информационной панели и поддерживают виртуальной Extensible LAN (VxLAN) и сетевой виртуализации универсального Routing Encapsulation (NVGRE). Дополнительные сведения см. в разделе [туннелирование GRE в Windows Server 2016](../remote/remote-access/ras-gateway/gre-tunneling-windows-server.md).  
  
Дополнительные сведения о SDN см. в разделе [программного обеспечения определены сети & #40; SDN & #41; ](sdn/software-defined-networking.md).  
  
### <a name="cloud-scale-fundamentals"></a>Основы масштабирования облака
 
Теперь доступны следующие Основы масштабирования облака.  
  
-   **Схождение выполнено сетевой адаптер (NIC)**. Схождением сетевых КАРТ позволяет использовать один сетевой адаптер для управления, хранения с поддержкой удаленного доступа к памяти и клиентского трафика. Это снижает капитальных затрат, которые связаны с каждого сервера в центре обработки данных, поскольку требуется меньшее количество сетевых адаптеров для управления различными типами трафика на каждом сервере.  
  
-   **Пакет Direct**.  Пакет Direct предоставляет пропускной способности высокой сетевого трафика и инфраструктура обработки пакетов с низкой задержкой.  
  
-   **Коммутатор Embedded объединение (SET)**.        НАБОР — это объединение сетевых КАРТ решение, которое интегрируется в виртуальном коммутаторе Hyper-V. НАБОР позволяет объединение до восьми физических сетевых Адаптеров в одной команды SET, что повышает доступность и обеспечивает отработку отказа. В Windows Server 2016 можно создать НАБОР команд, ограничения на использование Server Message Block (SMB) и RDMA. Кроме того можно использовать НАБОР групп для распределения сетевого трафика для виртуализации сети Hyper-V. Дополнительные сведения см. в разделе [удаленного прямого доступа к памяти & #40; RDMA & #41; и переключения объединение внедренных & #40; ЗАДАТЬ & #41; ](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).  
  
## <a name="bkmk_existing"></a>Новые функции для дополнительных технологии сетевого взаимодействия

## <a name="bkmk_dhcp"></a>DHCP  
Протокол DHCP является стандартом IETF Internet Engineering Task Force (), разработанным для облегчения управления и упрощения настройки узлов на основе IP сети, например частной интрасети. С использованием службы DHCP-сервера процесс настройки протокола TCP/IP на DHCP-клиентах происходит автоматически.  
  
Дополнительные сведения см. в разделе [новые возможности DHCP](technologies/dhcp/What-s-New-in-DHCP.md).  
  
## <a name="bkmk_dns"></a>DNS-СЕРВЕРА  
DNS — это система, используемая в сетях TCP/IP для именования компьютеров и сетевых служб. DNS-имена позволяют находить компьютеры и службы с помощью понятных имен. Когда пользователь вводит DNS-имя в приложении, службы DNS могут сопоставить имя с другими сведениями, которая связана с именем, например IP-адрес.  
  
Ниже приведены сведения о DNS-клиент и DNS-сервера.  
  
### <a name="bkmk_dnsc"></a>DNS-клиента  
Ниже перечислены новые и улучшенные технологии клиента DNS.  
  
-   **Привязка службы DNS-клиента**. В Windows 10 служба DNS-клиента обеспечивает расширенную поддержку для компьютеров с более чем одного сетевого интерфейса.  
  
Дополнительные сведения см. в разделе [новые возможности DNS-клиента в Windows Server 2016](dns/What-s-New-in-DNS-Client.md)  
  
### <a name="bkmk_dnss"></a>DNS-сервера  
Ниже перечислены новые и улучшенные технологии сервера DNS.  
  
-   **Политики DNS**.  Можно настроить политики DNS, чтобы указать, как DNS-сервер отвечает на запросы DNS. Ответов DNS может быть основан на IP-адрес клиента (местоположение), времени суток и несколько других параметров. Политики DNS включить DNS, учитывающих местоположение, трафика управления, балансировки нагрузки, разделенная система DNS и других сценариев.  
  
-   **Поддержка файлов в Nano Server на основе DNS**, можно развернуть DNS-сервера в Windows Server 2016 в образе Nano Server. Этот параметр развертывания доступно, если вы используете DNS на основе файлов. Выполнение DNS-сервером в образе Nano Server можно запускать DNS-серверы с сокращенных Быстрая загрузка вверх и свернутые исправления.  
  
    > [!NOTE]   
    > Интегрированная в Active Directory DNS-сервера не поддерживается на сервере Nano Server.  
  
-   **Ответ оценить ограничивающей (RRL)**.  Вы можете включить ограничения скорости ответа на DNS-серверы. Таким образом, можно избежать возможность вредоносных систем с помощью DNS-серверы для инициирования атаки отказа в обслуживании на DNS-клиента.  
  
-   **Проверка подлинности на основе DNS именованных объектов (DANE)**.   Записи TLSA (проверка подлинности безопасности уровня транспорта) можно использовать для предоставления информации DNS-клиенты с сообщением, какой центр сертификации (ЦС) они должны ожидать сертификат для имени вашего домена. Это предотвращает атаки в середине где кто-то повреждены кэш DNS для указания реализованных веб-сайтах и предоставить сертификат, который они выпущены с другой ЦС.  
  
-   **Поддержка Неизвестный записи**.   
     Вы можете добавить записи, которые явно не поддерживаются сервером DNS-сервера Windows, с помощью функции неизвестной записи.  
  
-   **Корневые ссылки IPv6**.   
     Можно использовать собственный протокол IPV6, поддерживать корневые ссылки, чтобы выполнять разрешение имен Интернета с помощью корневых серверов IPV6.  
  
-   **Улучшенная поддержка Windows PowerShell**.   
      Новые командлеты Windows PowerShell доступны для DNS-сервера.  
  
Дополнительные сведения см. в разделе [новые возможности DNS-сервера в Windows Server 2016](dns/What-s-New-in-DNS-Server.md)  
  
## <a name="bkmk_GRE"></a>Туннелирование GRE  
Шлюз RAS-сервера теперь поддерживает туннели Generic Routing Encapsulation (GRE) высокой доступности для межсайтовых подключений и избыточности + N, M шлюзов. GRE является упрощенный протокол туннелирования, который может инкапсулировать разнообразные протоколы сетевого уровня внутри виртуальных каналов IP-сети.  
  
Дополнительные сведения см. в разделе [туннелирование GRE в Windows Server 2016](../remote/remote-access/ras-gateway/gre-tunneling-windows-server.md).  
  
## <a name="HNV"></a>Виртуализация сети Hyper-V  
Представленный в Windows Server 2012, виртуализация сети Hyper-V (HNV) обеспечивает виртуализацию сетей клиентов на основе общей физической сетевой инфраструктуры. С минимальными изменениями необходимые физической сетевой инфраструктуре, HNV позволяет поставщикам услуг гибкость, развертывать и перемещать рабочие нагрузки клиентов в любом месте в облаках три: облако поставщика службы, частное облако или общедоступное облако Microsoft Azure.  
  
Дополнительные сведения см. в разделе [новые возможности виртуализации сети Hyper-V в Windows Server 2016](sdn/technologies/hyper-v-network-virtualization/whats-new-hyperv-network-virtualization-windows-server.md)  
  
## <a name="bkmk_ipam"></a>IPAM  
IPAM предоставляет широкие возможности администрирования и мониторинга инфраструктуры DNS и IP-адрес в сети организации. IPAM можно отслеживание, аудит и управление серверами, на которых выполняется протокол динамической настройки узла (DHCP) и доменных имен (DNS).  
  
-   **Расширенное управление IP-адресами**.  
     Улучшение возможности IPAM для сценариев, таких как обработка IPv4 /32 и IPv6 /128 подсетей и поиск свободного IP-адрес подсети и диапазоны в блоком IP-адресов.  
  
-   **Расширенное управление DNS**.  
     IPAM поддерживает запись ресурса DNS, условного сервера пересылки и Управление зоной DNS для присоединенных к домену интегрированный с Active Directory и файловые DNS-серверов.  
  
-   **Интегрированное DNS, DHCP и IP-адрес (DDI) Управление**.  
     Несколько новых возможностей и управления жизненным циклом интегрированной включено, например, чтобы создать наглядное представление всех записей ресурсов DNS, которые относятся к IP-адрес автоматической инвентаризации IP-адресов на основе DNS-записи ресурсов и управление жизненным циклом IP-адресов для операций DNS и DHCP.  
  
-   **Поддержка нескольких лесах Active Directory**.  
     IPAM можно использовать для управления DNS и DHCP-серверами в нескольких лесах Active Directory, когда двустороннее отношение доверия между лесом, где установлена IPAM и каждого из удаленных лесах.  
  
-   **Поддержка Windows PowerShell для управления доступом на основе ролей**.  
     Задание области доступа на объектов IPAM, можно использовать Windows PowerShell.  
  
Дополнительные сведения см. в разделе [новые возможности IPAM](technologies/ipam/What-s-New-in-IPAM.md) и [управление IPAM](technologies/ipam/Manage-IPAM.md).
