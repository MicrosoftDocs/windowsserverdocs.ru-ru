#### <a name="bkmk_netc"></a>Сценарии сетевого контроллера

В сценариях сетевого контроллера позволяют:  
  
-   Развертывание и управление ими в экземпляре несколько узлов сетевого контроллера. Дополнительные сведения см. в разделе [развертывание сетевого контроллера с помощью Windows PowerShell](sdn/deploy/Deploy-Network-Controller-using-Windows-PowerShell.md).  
  
-   Сетевой контроллер используйте программным способом определить политики сети с помощью северного API-интерфейса REST.  
  
-   Сетевой контроллер можно используйте для создания и управления виртуальными сетями с Hyper-V виртуализация сетей — инкапсуляция NVGRE или VXLAN с помощью.  
  
Дополнительные сведения см. в разделе [сетевой контроллер](sdn/technologies/network-controller/Network-Controller.md).  
  
#### <a name="bkmk_netf"></a>Функция виртуализации (NFV) сетях.  
В сценариях NFV позволяют:  
  
-   Развертывание и использовать для распределения трафика Северный и Южный Балансировка нагрузки программного обеспечения.  
  
-   Развертывание и использовать подсистему балансировки нагрузки программного обеспечения для распределения трафика eastbound и westbound виртуальных сетей, созданного с помощью виртуализации сети Hyper-V.  
  
-   Развертывание и балансировка нагрузки программного обеспечения NAT для виртуальных сетей, созданные с помощью виртуализации сети Hyper-V.  
  
-   Развертывание и использование шлюза переадресации третьего уровня  
  
-   Развертывание и использовать шлюз виртуальной частной сети (VPN) для туннелей IPsec (IKEv2) для узла  
  
-   Развертывание и использовать шлюз Generic Routing Encapsulation (GRE).  
  
-   Развертывание и настройка динамической маршрутизации и Транзитная маршрутизация между сайтами, используя протокол пограничного шлюза (BGP).  
  
-   Настройте избыточности + N, M для уровня 3 и шлюзы сайта для сайта, а для маршрутизации BGP.  
  
-   Сетевой контроллер используется для указания списки управления доступом на виртуальных сетей и сетевых интерфейсов.  
  
Дополнительные сведения см. в разделе [виртуализация сетевых функций](sdn/technologies/network-function-virtualization/Network-Function-Virtualization.md).  
  
### <a name="bkmk_netp"></a>Сценарии платформы сети

Для сценариев в этом разделе сети Windows Server команды поддерживает использование любой сертифицированных драйверов Windows Server 2016. Проверьте у изготовителя сетевого интерфейса карты \(NIC\) чтобы убедиться, что у вас установлены самые последние обновления драйверов.
  
Сценарии сетевой платформы позволяют:  
  
-   Используйте схождением сетевых КАРТ для объединения трафика RDMA и Ethernet, с помощью одного сетевого адаптера.  
  
-   Создайте путь низкой задержкой данных с помощью прямого пакетов, включенная в виртуальный коммутатор Hyper-V и один сетевой адаптер.  
  
-   Настройка SET распространения SMB Direct и RDMA потоки трафика между двумя сетевыми адаптерами.  
  
Дополнительные сведения см. в разделе [удаленного прямого доступа к памяти & #40; RDMA & #41; и переключения объединение внедренных & #40; ЗАДАТЬ & #41; ](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).

В сценариях виртуального коммутатора Hyper-V позволяет:

-   Создание группы НАБОР в виртуальный коммутатор Hyper-V  
  
-   Управление группой НАБОР с помощью команд Windows PowerShell  
  
Дополнительные сведения см. в разделе [удаленного прямого доступа к памяти & #40; RDMA & #41; и переключения объединение внедренных & #40; ЗАДАТЬ & #41;](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md)  
  
### <a name="bkmk_dns"></a>Сценарии DNS-сервера

DNS-сервер сценариев позволяют:  
  
-   Укажите, что трафик управления, с помощью политики DNS на основе географического расположения  
  
-   Настройка разделенной DNS, с помощью политики DNS  
  
-   Применение фильтров на запросы DNS, с помощью политики DNS  
  
-   Настройка балансировки нагрузки приложений с помощью политики DNS  
  
-   Укажите, что интеллектуальных ответов DNS на основе времени дня  
  
-   Настройка политик передачи зоны DNS  
  
-   Настройка политики в доменных службах Active Directory (AD DS) интегрировать зон DNS-сервера  
  
-   Настройки скорости ответа ограничение  
  
-   Укажите проверку подлинности на основе DNS именованных объектов (DANE)  
  
-   Настройка поддержки неизвестных записей в DNS  
  
Дополнительные сведения см. в разделах [новые возможности DNS-клиента в Windows Server 2016](dns/What-s-New-in-DNS-Client.md) и [новые возможности DNS-сервера в Windows Server 2016](dns/What-s-New-in-DNS-Server.md).  
  
### <a name="bkmk_ipam"></a>IPAM сценариев в DHCP и DNS

В сценариях IPAM позволяют:  
  
-   Обнаружение и управление DNS и DHCP-серверов и IP-адресов в нескольких лесах Active Directory для федеративных  
  
-   Используйте IPAM для централизованного управления свойства DNS-сервера, включая зоны и записи ресурсов.  
  
-   Определения политик управления детализации доступа на основе ролей и делегировать IPAM пользователей или групп пользователей для управления набор свойств DNS-сервера, указанных вами.  
  
-   Используйте команды Windows PowerShell для IPAM для автоматизации настройки управления доступа для DHCP и DNS.  
  
    Дополнительные сведения см. в разделе [управление IPAM](technologies/ipam/Manage-IPAM.md).  
  
### <a name="bkmk_nicteam"></a>Объединение сетевых КАРТ сценариев

В сценариях, объединение сетевых КАРТ позволяют:  
  
-   Создание группы сетевых КАРТ в поддерживаемой конфигурации  
  
-   Удаление группы сетевых КАРТ  
  
-   Добавление сетевых адаптеров к объединению Сетевых карт в поддерживаемой конфигурации  
  
-   Удаление сетевых адаптеров из группы сетевых КАРТ  
  
> [!NOTE]  
> В Windows Server 2016 используется объединение сетевых КАРТ в Hyper-V, однако в некоторых случаях очереди виртуальных машин (VMQ) может не включать автоматически на основную сетевых адаптеров при создании группы сетевых КАРТ. В этом случае можно использовать следующую команду Windows PowerShell, чтобы убедиться, что VMQ включена на адаптерах члена группы сетевых КАРТ: `Set-NetAdapterVmq -Name <NetworkAdapterName> -Enable`  

Дополнительные сведения см. в разделе [объединение сетевых КАРТ](technologies/nic-teaming/NIC-Teaming.md). 

### <a name="bkmk_set"></a>Объединение внедренных \(SET\) сценариев коммутатора

НАБОР является альтернативным решением объединение сетевых КАРТ, можно использовать в средах Hyper-V и стека программно-конфигурируемые сети (SDN) в Windows Server 2016. НАБОР объединяет некоторые функции объединения сетевых КАРТ в виртуальный коммутатор Hyper-V. 

Дополнительные сведения см. в разделе [удаленный прямой доступ к памяти (RDMA) и коммутатора объединение внедренных (SET)](https://technet.microsoft.com/windows-server-docs/networking/technologies/hyper-v-virtual-switch/rdma-and-switch-embedded-teaming)
  
 
  
## <a name="bkmk_unsupp"></a>Неподдерживаемые сценарии сетевого взаимодействия  
В Windows Server 2016 не поддерживаются следующие сценарии работы в сети.  
  
-   На основе виртуальной ЛС виртуальными сетями.  
  
-   IPv6 не поддерживается в underlay или наложения.
