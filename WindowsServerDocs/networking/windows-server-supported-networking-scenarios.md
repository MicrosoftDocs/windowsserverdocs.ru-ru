#### <a name="bkmk_netc"></a>Сценарии сетевой контроллер

В сценариях сетевого контроллера позволяет делать следующее:  
  
-   Развертывание и управление экземпляром несколько узлов сетевого контроллера. Дополнительные сведения см. в разделе [развертывание сетевого контроллера с помощью Windows PowerShell](sdn/deploy/Deploy-Network-Controller-using-Windows-PowerShell.md).  
  
-   Используйте сетевой контроллер программными средствами определить политики сети с помощью Северный API-Интерфейс REST.  
  
-   Использование сетевого контроллера для создания и управления виртуальными сетями с использованием виртуализации сети Hyper-V — с помощью инкапсуляции NVGRE или VXLAN.  
  
Дополнительные сведения см. в разделе [Сетевой контроллер](sdn/technologies/network-controller/Network-Controller.md).  
  
#### <a name="bkmk_netf"></a>Сетевые сценарии функция виртуализации (NFV)  
В сценариях NFV позволяют:  
  
-   Развернуть и использовать подсистему балансировки нагрузки для распределения трафика Северный и подсистемамми.  
  
-   Развернуть и использовать подсистему балансировки нагрузки для распределения трафика eastbound и westbound для виртуальных сетей, созданных с помощью виртуализации сети Hyper-V.  
  
-   Развертывание и использование NAT подсистемы балансировки нагрузки для виртуальных сетей, созданных с помощью виртуализации сети Hyper-V.  
  
-   Развертывание и использование шлюза переадресации третьего уровня  
  
-   Развернуть и использовать шлюз виртуальной частной сети (VPN) для туннелей IPsec (IKEv2) сеть сеть  
  
-   Развертывание и использование шлюза протоколом (GRE).  
  
-   Развертывание и настройка динамической маршрутизации и Транзитная маршрутизация между сайтами, с помощью протокола пограничного шлюза (BGP).  
  
-   Настройте избыточностью M + N для третьего уровня и сайтами шлюзов, а также для маршрутизации BGP.  
  
-   Позволяет указать списки управления доступом на виртуальные сети и сетевые интерфейсы сетевого контроллера.  
  
Дополнительные сведения см. в разделе [виртуализация сетевых функций](sdn/technologies/network-function-virtualization/Network-Function-Virtualization.md).  
  
### <a name="bkmk_netp"></a>Сценарии сетевой платформы

Для сценариев в этом разделе, Windows Server Networking team позволяет использовать любой сертифицированных драйверов Windows Server 2016. Пожалуйста, свяжитесь с вашей сетевой карты \(сетевой Адаптер\) производителя, чтобы можно было самые последние обновления драйвера.
  
Сценарии сетевой платформы позволяют:  
  
-   Используйте конвергентную сетевую КАРТУ для объединения трафика RDMA и Ethernet, с помощью одного сетевого адаптера.  
  
-   Создайте путь к данным с малым временем задержки с помощью Packet Direct, включена в виртуальный коммутатор Hyper-V и один сетевой адаптер.  
  
-   Настройка НАБОРА для распределения потоков трафика SMB Direct и RDMA между двумя сетевыми адаптерами.  
  
Дополнительные сведения см. в разделе [удаленный доступ к памяти &#40;RDMA&#41; и Switch Embedded Teaming &#40;ЗАДАТЬ&#41;](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md).  
  
#### <a name="bkmk_switch"></a>Сценарии виртуального коммутатора Hyper-V

В сценариях виртуального коммутатора Hyper-V позволяет делать следующее:  
  
-   Создание виртуального коммутатора Hyper-V с помощью виртуального сетевого адаптера удаленный доступ к памяти (RDMA)  
  
-   Создание виртуального коммутатора Hyper-V с помощью коммутатора внедренных коммутаторов (SET) и RDMA  
  
-   Создать команду SET в виртуальный коммутатор Hyper-V  
  
-   Управление группой НАБОРА с помощью команд Windows PowerShell  
  
Дополнительные сведения см. в разделе [удаленный доступ к памяти &#40;RDMA&#41; и Switch Embedded Teaming &#40;значение&#41;](../virtualization/hyper-v-virtual-switch/RDMA-and-Switch-Embedded-Teaming.md)  
  
### <a name="bkmk_dns"></a>DNS-сервер сценариев

DNS-сервер сценариев позволяют:  
  
-   Укажите географическое расположение на основе управление трафиком с помощью политики DNS  
  
-   Настройка разделенная система DNS с помощью политики DNS  
  
-   Применить фильтры на запросы DNS, с помощью политики DNS  
  
-   Настройка балансировки нагрузки приложения с помощью политики DNS  
  
-   Укажите, что интеллектуальных ответов DNS на основе времени дня  
  
-   Настройка политик передачи зоны DNS  
  
-   Настройка политики в доменных службах Active Directory (AD DS) интегрировать зон DNS-сервера  
  
-   Настройка процента ответивших ограничение  
  
-   Проверку подлинности на основе DNS именованных сущностей (DANE)  
  
-   Настройка поддержки за неизвестными записями в DNS  
  
Дополнительные сведения см. в разделах [новые возможности DNS-клиента в Windows Server 2016](dns/What-s-New-in-DNS-Client.md) и [новые возможности DNS-сервера в Windows Server 2016](dns/What-s-New-in-DNS-Server.md).  
  
### <a name="bkmk_ipam"></a>Сценарии IPAM с DHCP и DNS

В сценариях IPAM позволяет делать следующее:  
  
-   Обнаружение и администрирования DNS и DHCP-серверов и IP-адресов в нескольких лесах Active Directory для федеративной  
  
-   Использование IPAM для централизованного управления свойства DNS, включая зон и записей ресурсов.  
  
-   Определить политики управления на основе ролей доступ и делегировать IPAM пользователям или группам пользователей для управления набором заданных пользователем свойств DNS.  
  
-   Используйте команды Windows PowerShell для IPAM для автоматизации конфигурации контроля доступа для DHCP и DNS.  
  
    Дополнительные сведения см. в разделе [управление IPAM](technologies/ipam/Manage-IPAM.md).  
  
### <a name="bkmk_nicteam"></a>Объединение Сетевых карт сценариев

В сценариях, объединение Сетевых карт позволяет делать следующее:  
  
-   Создание группы сетевых КАРТ в поддерживаемые конфигурации  
  
-   Удаление группы сетевых КАРТ  
  
-   Добавление сетевых адаптеров в группу Сетевых карт в поддерживаемой конфигурации  
  
-   Удалите из Сетевых карт сетевые адаптеры  
  
> [!NOTE]  
> В Windows Server 2016 можно использовать объединение Сетевых карт в Hyper-V, однако в некоторых случаях очереди виртуальной машины (VMQ) может не включать автоматически на базовые сетевые адаптеры при создании группы сетевых КАРТ. В этом случае можно использовать следующую команду Windows PowerShell, чтобы убедиться, что VMQ включена на адаптеры членов группы сетевых Адаптеров: `Set-NetAdapterVmq -Name <NetworkAdapterName> -Enable`  

Дополнительные сведения см. в разделе [объединение Сетевых карт](technologies/nic-teaming/NIC-Teaming.md). 

### <a name="bkmk_set"></a>Объединение внедренных коммутаторов \(ЗАДАТЬ\) сценариев

НАБОР представляют собой альтернативное решение объединение Сетевых карт, который можно использовать в средах Hyper-V и стек программно определяемой сети (SDN) в Windows Server 2016. НАБОР объединяет некоторые функции объединения Сетевых карт в виртуальный коммутатор Hyper-V. 

Дополнительные сведения см. в разделе [удаленный доступ к памяти (RDMA) и коммутатора внедренных коммутаторов (SET)](https://technet.microsoft.com/windows-server-docs/networking/technologies/hyper-v-virtual-switch/rdma-and-switch-embedded-teaming)
  
 
  
## <a name="bkmk_unsupp"></a>Неподдерживаемые сценарии сети  
Следующие сетевые сценарии не поддерживаются в Windows Server 2016.  
  
-   Виртуальных сетей клиентов на основе виртуальной ЛС.  
  
-   IPv6 не поддерживается в лежащие или наложения.
