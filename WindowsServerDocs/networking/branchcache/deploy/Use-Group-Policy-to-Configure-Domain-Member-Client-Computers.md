# <a name="use-group-policy-to-configure-domain-member-client-computers"></a>Чтобы настроить домен клиентских компьютеров с помощью групповой политики

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

В этом разделе описано можно создать объект групповой политики для всех компьютеров в вашей организации, настроить домен клиентских компьютеров в режиме распределенного кэша или в режиме размещенного кэша и настройки брандмауэра Windows с повышенной безопасности, чтобы разрешить BranchCache трафик.  
  
Этот раздел содержит следующие процедуры.  
  
1.  [Для создания объекта групповой политики и настройки режима BranchCache](#bkmk_gp)  
  
2.  [Чтобы настроить брандмауэр Windows в Advanced Security трафика правила для входящих подключений](#bkmk_inbound)  
  
3.  [Чтобы настроить брандмауэр Windows в Advanced Security трафика правила для исходящих подключений](#bkmk_outbound)  
  
> [!TIP]  
> В следующей процедуре будет предложено создать объект групповой политики в политику домена по умолчанию, однако можно создать объект в подразделение (OU) или другой контейнер, который подходит для развертывания.  
  
Необходимо быть членом **"Администраторы домена"**, или эквивалентных для выполнения этих процедур.  
  
## <a name="bkmk_gp"></a>Для создания объекта групповой политики и настройки режима BranchCache  
  
1.  На компьютере, на котором установлена роль сервера доменных служб Active Directory, в диспетчере серверов щелкните **средства**, а затем нажмите кнопку **Управление групповой политикой**. Откроется консоль управления групповыми политиками.  
  
2.  В консоли управления групповыми политиками разверните следующий путь: **Лес:** *example.com*, **домены**, *example.com*, **объекты групповой политики**, где  *example.com* — это имя домена, где находятся учетные записи компьютера клиента BranchCache, которые вы хотите настроить.  
  
3.  Щелкните правой кнопкой мыши **объектов групповой политики**, а затем нажмите кнопку **New**. **Новый объект групповой Политики** откроется диалоговое окно. В **имя**, введите имя для новой политики объекта Групповой политики. Например, если вы хотите присвоить имя объекту клиентских компьютеров BranchCache, введите **клиентских компьютеров BranchCache**. Нажмите кнопку **ОК**.  
  
4.  Убедитесь, что в консоли управления групповыми политиками, **объектов групповой политики** выбран и в области сведений щелкните правой кнопкой мыши только что созданный объект групповой Политики. Например, если с именем "компьютеры-клиенты групповой Политики BranchCache", щелкните правой кнопкой мыши **клиентских компьютеров BranchCache**. Щелкните **Изменить**. Откроется консоль управления групповыми политиками.  
  
5.  В консоли управления групповыми политиками разверните следующий путь: **Конфигурация компьютера**, **политики**, **административные шаблоны: Определения политик (файлы ADMX), полученные из локального компьютера**, **сети**, **BranchCache**.  
  
6.  Нажмите кнопку **BranchCache**, а затем в области сведений дважды щелкните **включить BranchCache**. Откроется диалоговое окно параметра политики.  
  
7.  В **включить BranchCache** диалоговом окне щелкните **включено**, а затем нажмите кнопку **ОК**.  
  
8.  Чтобы включить режим распределенного кэша BranchCache, в области сведений дважды щелкните **режим распределенного кэша BranchCache задать**. Откроется диалоговое окно параметра политики.  
  
9. В **режим задать распределенного кэша BranchCache** диалоговом окне нажмите кнопку **включено**, а затем нажмите кнопку **ОК**.  
  
10. Если у вас есть один или несколько филиалов где вы развертываете BranchCache в режиме размещенного кэша и развертывания серверов размещенного кэша в этих филиалов, дважды щелкните **Включение автоматического размещенного кэша обнаружения с использованием точки подключения службы**. Откроется диалоговое окно параметра политики.  
  
11. В **Включение автоматического размещенного кэша обнаружения с использованием точки подключения службы** диалоговом окне щелкните **включено**, а затем нажмите кнопку **ОК**.  
  
    > [!NOTE]  
    > При включении обоих **режим задать распределенного кэша BranchCache** и **Включение автоматического размещенного кэша обнаружения с использованием точки подключения службы** параметры политики, клиентские компьютеры работают в BranchCache распределенные режим кэша, если только они находят сервер размещенного кэша в филиале, после чего они работают в режиме размещенного кэша.  
  
12. Используйте приведенные ниже процедуры для настройки параметров брандмауэра на клиентских компьютерах с помощью групповой политики.  
  
## <a name="bkmk_inbound"></a>Чтобы настроить брандмауэр Windows в Advanced Security трафика правила для входящих подключений  
  
1.  В консоли управления групповыми политиками разверните следующий путь: **Лес:** *example.com*, **домены**, *example.com*, **объекты групповой политики**, где  *example.com* — это имя домена, где находятся учетные записи компьютера клиента BranchCache, которые вы хотите настроить.  
  
2.  Убедитесь, что в консоли управления групповыми политиками, **объектов групповой политики** выбран и в области сведений щелкните правой кнопкой мыши объект групповой Политики, который вы создали ранее клиентских компьютеров BranchCache. Например, если с именем "компьютеры-клиенты групповой Политики BranchCache", щелкните правой кнопкой мыши **клиентских компьютеров BranchCache**. Щелкните **Изменить**. Откроется консоль управления групповыми политиками.  
  
3.  В консоли управления групповыми политиками разверните следующий путь: **Конфигурация компьютера**, **политики**, **параметры Windows**, **параметры безопасности**, **брандмауэр Windows в режиме повышенной безопасности**, **Брандмауэр Windows в режиме повышенной безопасности — LDAP**, **правила для входящего трафика**.  
  
4.  Правой кнопкой мыши щелкните **Правила для входящих подключений**, а затем щелкните **Создать правило**. Откроется мастер создания правила для нового входящего подключения.  
  
5.  В **тип правила**, нажмите кнопку **предопределенное**, разверните список вариантов и нажмите кнопку **BranchCache - получение содержимого (использует HTTP)**. Нажмите кнопку **Далее**.  
  
6.  В **предварительно определенные правила**, нажмите кнопку **Далее**.  
  
7.  В **действие**, убедитесь, что **разрешить подключение** выбран, а затем нажмите кнопку **Готово**.  
  
    > [!IMPORTANT]  
    > Необходимо выбрать **разрешить подключение** для клиента BranchCache, чтобы иметь возможность получать трафик через этот порт.  
  
8.  Чтобы создать исключение брандмауэра WS-Discovery, снова щелкните правой кнопкой мыши **правила для входящих подключений**, а затем нажмите кнопку **новое правило**. Откроется мастер создания правила для нового входящего подключения.  
  
9. В **тип правила**, нажмите кнопку **предопределенное**, разверните список вариантов и нажмите кнопку **BranchCache - обнаружение кэширующих узлов (использует WSD)**. Нажмите кнопку **Далее**.  
  
10. В **предварительно определенные правила**, нажмите кнопку **Далее**.  
  
11. В **действие**, убедитесь, что **разрешить подключение** выбран, а затем нажмите кнопку **Готово**.  
  
    > [!IMPORTANT]  
    > Необходимо выбрать **разрешить подключение** для клиента BranchCache, чтобы иметь возможность получать трафик через этот порт.  
  
## <a name="bkmk_outbound"></a>Чтобы настроить брандмауэр Windows в Advanced Security трафика правила для исходящих подключений  
  
1.  В консоли управления групповыми политиками, щелкните правой кнопкой мыши **правила для исходящих подключений**, а затем нажмите кнопку **новое правило**. Откроется мастер создания правила исходящего трафика.  
  
2.  В **тип правила**, нажмите кнопку **предопределенное**, разверните список вариантов и нажмите кнопку **BranchCache - получение содержимого (использует HTTP)**. Нажмите кнопку **Далее**.  
  
3.  В **предварительно определенные правила**, нажмите кнопку **Далее**.  
  
4.  В **действие**, убедитесь, что **разрешить подключение** выбран, а затем нажмите кнопку **Готово**.  
  
    > [!IMPORTANT]  
    > Необходимо выбрать **разрешить подключение** для клиента BranchCache, чтобы иметь возможность отправлять трафик через этот порт.  
  
5.  Чтобы создать исключение брандмауэра WS-Discovery, снова щелкните правой кнопкой мыши **правила для исходящих подключений**, а затем нажмите кнопку **новое правило**. Откроется мастер создания правила исходящего трафика.  
  
6.  В **тип правила**, нажмите кнопку **предопределенное**, разверните список вариантов и нажмите кнопку **BranchCache - обнаружение кэширующих узлов (использует WSD)**. Нажмите кнопку **Далее**.  
  
7.  В **предварительно определенные правила**, нажмите кнопку **Далее**.  
  
8.  В **действие**, убедитесь, что **разрешить подключение** выбран, а затем нажмите кнопку **Готово**.  
  
    > [!IMPORTANT]  
    > Необходимо выбрать **разрешить подключение** для клиента BranchCache, чтобы иметь возможность отправлять трафик через этот порт.
