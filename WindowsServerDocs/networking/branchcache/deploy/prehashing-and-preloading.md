# <a name="prehashing-and-preloading-content-on-hosted-cache-servers-optional"></a>Предварительное хэширование и предварительная загрузка содержимого на серверах размещенного кэша (необязательно)

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Эту процедуру можно использовать для принудительного создания сведений о содержимом - также называется хэши - на серверах веб-приложений и файлов, с поддержкой BranchCache. Вы также можете собирать данные в файл и веб-серверы в пакеты, которые можно перенести на серверы удаленного размещенного кэша.  Это дает возможность предварительной загрузки содержимого на серверах удаленного размещенного кэша, чтобы данные будут доступны для первого клиентского доступа.  
  
Необходимо быть членом **Администраторы**, или эквивалентных для выполнения этой процедуры.  
  
### <a name="to-prehash-content-and-preload-the-content-on-hosted-cache-servers"></a>Хэширование содержимого и предварительная загрузка содержимого на серверах размещенного кэша  
  
1.  Войдите в систему в файл или веб-сервер, который содержит данные, которые вы хотите выполнить предварительную загрузку и определения папок и файлов, которые требуется загрузить в один или несколько серверов удаленного размещенного кэша.  
  
2.  Запустите Windows PowerShell от имени администратора. Для каждой папки и файла запуска `Publish-BCFileContent` команды или `Publish-BCWebContent` команду, в зависимости от типа сервера содержимого, триггера создания хэшей и добавление данных в пакет данных.  
  
3.  После добавления всех данных в пакет данных, экспортировать его с помощью `Export-BCCachePackage` команду, чтобы создать файл пакета данных.  
  
4.  Перемещение файла пакета данных на серверах размещенного кэша, удаленных с помощью Выбор технологии передачи файла.  FTP, SMB, HTTP, DVD-диска и переносимых жесткие диски являются все приемлемого транспортов.  
  
5.  Импортировать файл пакета данных на серверах размещенного кэша, удаленных с помощью `Import-BCCachePackage` команды.
