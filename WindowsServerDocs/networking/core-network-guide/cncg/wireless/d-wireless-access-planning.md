# <a name="wireless-access-deployment-planning"></a>Беспроводного доступа Планирование развертывания

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

Перед развертыванием беспроводного доступа, необходимо спланировать перечисленные ниже элементы:

- Установка беспроводного доступа \(APs\) точек в сети

- Конфигурация клиент беспроводной сети и доступа

В следующих разделах сведения на эти действия по планированию.

## <a name="planning-wireless-ap-installations"></a>Планирование установок беспроводной точки доступа
При разработке решения беспроводного доступа, необходимо выполнить следующие действия.

1. Определите, какие стандарты беспроводных точках доступа должны поддерживать
2. Определение областей покрытия, где вы хотите предоставить услуг беспроводной связи
3. Укажите, где найти беспроводные точки доступа

Кроме того необходимо спланировать схемы IP-адресов беспроводной точкой доступа и беспроводных клиентов. См. в разделе **Планирование конфигурации беспроводной точке доступа в NPS** ниже for связанные сведения.

### <a name="verify-wireless-ap-support-for-standards"></a>Проверка поддержки беспроводных сетей AP стандартов
Для целей согласованности и упрощения развертывания и управления AP рекомендуется развертывать беспроводные точки доступа же торговой марки и модели.

Эти беспроводные точки доступа, которые развертываются должна поддерживать следующее:

- **IEEE 802.1 X**

- **Проверка подлинности RADIUS**

- **Беспроводной проверки подлинности и шифрования.** Перечисленные в порядке большинство менее предпочитаемого:

    1.  WPA2\-Enterprise с AES

    2.  WPA2\-Enterprise с TKIP

    3.  WPA\-Enterprise с AES

    4.  WPA\-Enterprise с TKIP

>[!NOTE]
>Чтобы развернуть WPA2, необходимо использовать адаптеры беспроводной сети и беспроводные точки доступа, которые также поддерживают WPA2. В противном случае — используйте WPA\ предприятия.

Кроме того для повышения безопасности сети, беспроводные точки доступа должны поддерживать следующие параметры безопасности:

- **Фильтрация DHCP.** Беспроводной точке доступа необходимо использовать фильтрацию по IP-портам для предотвращения передачи широковещательных сообщений DHCP в тех случаях, в которых беспроводной клиент настроен как DHCP-сервер. Беспроводной точке доступа необходимо заблокировать отправку IP-пакетов с UDP-порт 68 сети клиента.

- **Фильтрация DNS.** Беспроводной точкой доступа необходимо использовать фильтрацию по IP-портам, чтобы предотвратить выполнение являются DNS-сервер клиента. Беспроводной точке доступа необходимо заблокировать отправке IP-пакетов TCP или UDP-порт 53 к сети клиента.

- **Изоляция клиента** Если ваш беспроводной точке доступа предоставляет возможности изоляции клиентов, следует включить функцию предотвратить возможные \(ARP\) протокола разрешения адресов спуфинг эксплойтов.

### <a name="identify-areas-of-coverage-for-wireless-users"></a>Определение области покрытия для беспроводных пользователей
Используйте архитектуры рисунки каждого этаж для каждого построения для определения областей, где вы хотите предоставить беспроводной покрытия. Например можно определите соответствующие офисов, комнаты на конференциях, фойе, кафетерии или courtyards.

В рисунки укажите все устройства, которые создавать помехи для сигналов беспроводной сети, таких как медицинское оборудование беспроводных видеокамеры, беспроводные телефоны, работа которых в 2.4 диапазоне \(ISM\) 2.5 промышленной ГГц, научные и медицинских и устройства с поддержкой Bluetooth\.

На панели инструментов Рисование пометьте аспектов сборки, могут создавать помехи для сигналов беспроводной сети; исходного объекты, используемые при создании здания может повлиять на беспроводного сигнала. Например, следующие общие объекты могут мешать распространения сигнала: эскалаторах, отопления и кондиционирования air\ трубопроводы и конкретные поддержки мостовые фермы.

Обратитесь к производителю AP сведения об источниках, которые могут привести к затухания радиочастотной беспроводной точкой доступа. Большинство AP предоставляют тестирования программного обеспечения, которое можно использовать для проверки сигнала, частота ошибок и пропускная способность.

### <a name="determine-where-to-install-wireless-aps"></a>Выбор места установки беспроводные точки доступа
На архитектуре рисунки найдите закрыть беспроводных точках доступа достаточно вместе для обеспечения достаточно беспроводной покрытия, но при этом гораздо достаточно друг от друга, они не мешали работать друг с другом.

Необходимые расстояние между AP зависит от типа AP и AP антенны, аспекты сборки, которые блокируют Беспроводные сигналы и других источников помех. Можно помечать размещения беспроводной точкой доступа для каждой точки доступа не более чем 300 метров из смежных беспроводной точке доступа. В разделе беспроводной точкой доступа изготовителем документации по AP спецификации и рекомендации для размещения.

Временно установите беспроводные точки доступа в расположениях, указанных на архитектуре рисунков. После этого на ноутбуке оснащаются 802.11 беспроводной адаптер, а также программное обеспечение опроса узла, часто поставляются с адаптерами беспроводной сети, определите, уровень сигнала в каждой области покрытия.

В покрытии областях уровень сигнала низкий поместите AP повысить уровень сигнала для области покрытия, установить дополнительные беспроводные точки доступа предоставляют необходимые покрытия, переместить или удалить источники сигнала помех.  

Обновление вашей архитектуры рисунки, чтобы указать конечное положение все беспроводные точки доступа. Наличие точных карту размещения AP поможет вам позднее, во время устранения неполадок операций или если вы хотите обновить или заменить AP.

### <a name="plan-wireless-ap-and-nps-radius-client-configuration"></a>Планирование беспроводной точкой доступа и клиента RADIUS сервера политики сети конфигурации
Чтобы настроить беспроводные точки доступа по отдельности или в группах, можно использовать сервер политики сети.

При развертывании большой беспроводную сеть, которая включает в себя много AP гораздо проще для настройки точек доступа в группах. Добавление точки доступа в качестве группы клиентов RADIUS в сервере политики сети, необходимо настроить точки доступа с этими свойствами.

- Эти беспроводные точки доступа настроены с IP-адреса из того же диапазон IP-адресов.

- Эти беспроводные точки доступа настроены с тот же общий секрет.

### <a name="plan-the-use-of-peap-fast-reconnect"></a>Планирование использования быстрое переподключение PEAP
В инфраструктуре 802.1 X точки беспроводного доступа настраиваются как RADIUS-клиенты RADIUS-серверов. Когда быстрое переподключение PEAP развертывается, клиент беспроводной сети, перемещаются между двумя или более точек доступа не требуется для проверки подлинности в каждой новой связи.

Быстрое переподключение PEAP сокращает время ответа для проверки подлинности между клиентом и сервером, поскольку запрос на проверку подлинности перенаправляется из новой точки доступа на сервере политики сети, которая изначально выполнена проверки подлинности и авторизации для запроса на подключение клиентов.

Так как оба варианта используют ранее NPS-сервер и клиент PEAP кэшируются свойства подключения Transport Layer Security \(TLS\) \ (набор которых называется TLS handle\), сервер политики сети можно быстро определить, что клиент авторизован для переподключения.

>[!IMPORTANT]
>Для быстрого переподключения для правильной работы, эти точки доступа должны быть настроены как клиенты RADIUS того же сервера политики сети.

Если исходный сервер политики сети не становится доступным, если клиент перемещается к точке доступа, которая настроена как RADIUS-клиента на другой сервер RADIUS, полноценную проверку подлинности между клиентом и новые средства проверки подлинности.

### <a name="wireless-ap-configuration"></a>Настройки беспроводной точки доступа
В следующем списке перечислены элементы, которые обычно настраиваются на 802.1X\-поддержкой беспроводных точек доступа:

> [!NOTE]
> Имена элементов могут быть разные торговой марки и модели и может отличаться от тех, в следующем списке. Содержатся в документации беспроводной точкой доступа для определенных конфигурации.

- **Идентификатор набора служб \(SSID\)**. Это имя беспроводной сети \ (например, ExampleWlan\), а имя, объявленных для беспроводных клиентов. Чтобы снизить путаницу, SSID, выбранного для объявления не должны совпадать с SSID, они передаются по любой беспроводных сетей, находящихся в пределах диапазона приема беспроводной сети.

    В случаях, в которых развернуты несколько беспроводных точек доступа в рамках той же беспроводной сети Настройка каждого беспроводного доступа с же SSID. В случаях, в которых развернуты несколько беспроводных точек доступа в рамках той же беспроводной сети Настройка каждого беспроводного доступа с же SSID.  

    В случаях, когда у вас есть необходимость развертывания разных беспроводных сетей, в соответствии с потребностями бизнеса точки беспроводного доступа в одной сети следует широковещательных разных SSID, чем SSID вашей других network\(s\). Например, если требуется отдельный беспроводной сети для сотрудников и гостевых виртуальных машинах, вы можете назначить беспроводных точках доступа для бизнес-сети SSID, задайте трансляции **ExampleWLAN**. Для сети виртуальной машине, можно затем задать SSID каждого беспроводного доступа в режиме вещания передать **GuestWLAN**. Таким образом сотрудники и гостевые системы могут подключаться к соответствующей сети без лишних путаницы.  

    > [!TIP]  
    > Некоторые беспроводной точкой доступа могут в режиме вещания передать несколько SSID для размещения развертывания в сети используется. Беспроводной точке доступа, можно транслировать несколько SSID можно уменьшить, развертывания и затрат на обслуживание рабочей.  

- **Беспроводной проверки подлинности и шифрования**.

    Проверка подлинности беспроводной сети является проверки безопасности, который используется, когда беспроводной клиент обращается беспроводной точки доступа.  

    Шифрование беспроводной сети — шифра безопасности шифрования, который используется с помощью проверки подлинности беспроводной сети для защиты сообщений, которые передаются между беспроводной точкой доступа и клиент беспроводной сети.  

- **Беспроводной AP IP адрес \(static\)**. На каждой точке беспроводного доступа настройте уникальный статический IP-адрес. Если подсети обслуживается DHCP-сервера, убедитесь, что все AP IP-адреса могут находиться в пределах диапазона исключения DHCP, чтобы DHCP-сервер не пытайтесь выдавать одного IP-адреса других компьютеров и устройств. Диапазоны исключения подробно описаны в разделе «Создание и активация новой области DHCP» [руководство по основной сети](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide). Если вы планируете настроить точках доступа в качестве RADIUS-клиенты по группам в сервере политики сети, каждый AP в группе должен иметь IP-адрес из того же диапазон IP-адресов.

- **DNS-имя**. Некоторые беспроводные точки доступа можно настроить с помощью DNS-имя. Настройте каждый беспроводного доступа с уникальное имя. Например при наличии развернутого беспроводных точек доступа в здании несколькими историю может имя первые три беспроводные точки доступа, развернутых на третий этаж AP3\-01, AP3\-02 и AP3\-03.

- **Маска подсети беспроводной точкой доступа**. Настройте маску для обозначения какие часть IP-адресов идентификатор сети и какие части IP-адрес — это узел.

- **Служба AP DHCP**. Если ваш беспроводной точке доступа встроенных службу DHCP, отключите его.

- **Общий секрет RADIUS**. Используйте уникальный RADIUS общий секрет для каждой точки доступа, если не планируется настроить сервер политики сети RADIUS-клиентов в группы — в каких обстоятельствах необходимо настроить все AP в группе тот же общий секрет. Общие секреты следует случайных последовательность по крайней мере 22 символов, с помощью и прописные и строчные буквы, числа и знаки препинания. Чтобы обеспечить случайность, можно использовать программу создания случайных символов для создания вашего общие секреты. Рекомендуется записывать общий секрет для каждой точки доступа и сохранить его в безопасном расположении, например безопасный office. При настройке RADIUS-клиентов в консоли сервера политики сети вы создадите виртуальную версию каждого AP. Общий секрет, который настраивается для каждого виртуального точки доступа на сервере политики сети должна соответствовать общий секрет на AP фактические, физических.

- **IP-адрес сервера RADIUS**. Введите IP-адрес сервера политики сети, который вы хотите использовать для проверки подлинности и авторизации запросов на подключение к этой точке доступа.

- **UDP port\(s\)**. По умолчанию сервер политики сети использует UDP-порты 1812 и 1645 для сообщений проверки подлинности RADIUS и UDP-порты 1813 и 1646 для сообщений учета RADIUS. Рекомендуется не изменять параметры по умолчанию для портов RADIUS UDP.

- **VSA**. Некоторые беспроводные точки доступа требуют \(VSAs\) vendor\ определенные атрибуты для предоставления функций полной беспроводной точкой доступа.

- **Фильтрация DHCP**. Настройка беспроводных точек доступа для блокирования беспроводные клиенты из отправку IP-пакетов с UDP-порт 68 сети. См. в документации для вашей беспроводной точке доступа, настроить фильтрацию DHCP.

- **DNS-фильтрация**. Настройка беспроводных точек доступа для блокирования беспроводные клиенты из отправкой IP-пакетов TCP или UDP-порт 53 к сети. См. в документации для вашей беспроводной точке доступа, настроить фильтрацию, DNS-сервера.

## <a name="planning-wireless-client-configuration-and-access"></a>Планирование конфигурации клиент беспроводной сети и доступа

При планировании развертывания 802.1X\-беспроводного доступа с проверкой подлинности необходимо учесть некоторые факторы, зависящие отведенный период:

- **Планирование нескольких стандартов**.

    Определите, ли компьютеры вашей беспроводной сети используют ту же версию Windows или того, являются ли они сочетание компьютеров под управлением различных операционных систем. Если они различаются, убедитесь, что понимаете любое различие в стандартах, поддерживаемых операционными системами.

    Определение того, поддерживают ли все адаптеры беспроводной сети на всех беспроводных клиентских компьютеров тем же стандартам беспроводной или, требуется ли для поддержки различных стандартов. Например определите, поддерживает ли некоторые драйверы оборудования сетевого адаптера WPA2\-Enterprise и AES, тогда как другие поддерживают только WPA\-Enterprise и TKIP.

- **Планирования режим проверки подлинности клиента**. Режимы проверки подлинности определяют, как клиенты Windows обрабатывают учетные данные домена. Можно выбрать следующие режимы проверки подлинности три сети в политиках беспроводной сети.  

    1. **Проверка подлинности пользователя повторно**. Этот режим указывает, что проверка подлинности всегда проводится с использованием учетных данных безопасности, основанные на текущем состоянии компьютера. Когда пользователи не вошли компьютер, проверка подлинности проводится с использованием учетных данных компьютера. Когда пользователь вошел в систему на компьютере, проверки подлинности всегда проводится с использованием учетных данных пользователя.  

    2. **Только на компьютере**. Компьютер только режим указывает, что проверка подлинности всегда проводится с использованием только учетных данных компьютера.  

    3.  **Проверка подлинности пользователя**. Режим проверки подлинности пользователь указывает, что проверка подлинности выполняется только в том случае, если пользователь вошел в систему на компьютере. Если нет пользователей войти в систему на компьютере, попыток проверки подлинности не выполняется.  

- **Планирование беспроводной ограничения**. Определите ли вы хотите предоставить всем пользователям беспроводной тот же уровень доступа к беспроводной сети, или если вы хотите ограничить доступ для некоторых пользователей беспроводной сети. Вы можете применить ограничения на сервере политики сети для определенных групп пользователей беспроводных.  Например можно определить конкретные дни и часы, определенных групп разрешен доступ к беспроводной сети.  

- **Планирование методы для добавления новых компьютеров беспроводной**. Для компьютеров с поддержкой wireless\, присоединенных к домену перед развертыванием беспроводной сети, если компьютер подключен к сегменту проводной сети, который не защищен 802. 1 X, параметры конфигурации беспроводной связи автоматически применяются после настройки беспроводной сети \ (IEEE 802.11\) политики на контроллере домена, и после обновления групповой политики на клиент беспроводной сети.  

    Для компьютеров, не присоединенных к домену, однако необходимо учесть способ применения параметров, которые требуются для 802.1X\-доступ с проверкой подлинности. Например определите, следует ли присоединить компьютер к домену с помощью одного из следующих методов.

    1.  Подключите компьютер к сегменту проводной сети, который не защищен 802. 1 X, а затем присоединить компьютер к домену.

    2.  Предоставьте пользователям беспроводной действия и параметры, которые требуется добавить собственные профиля беспроводной связи начальной загрузки, что позволяет присоединить компьютер к домену.

    3.  Назначьте ИТ-специалистам присоединение беспроводных клиентов к домену.

### <a name="planning-support-for-multiple-standards"></a>Планирование нескольких стандартов

Беспроводной сети \ (IEEE 802.11\) расширения политик в групповой политике предоставляет широкий набор параметров конфигурации для поддержки различных вариантов развертывания.

Можно развертывать беспроводные точки доступа, настроенных с помощью стандартов, которые требуется поддерживать и затем настроить несколько профилей беспроводной связи в беспроводной сети \ (IEEE 802.11\) политики, с каждым профилем указание один набор стандартов, которые требуется.

Например если в сети беспроводные компьютеры, которые поддерживают WPA2\-Enterprise и AES, других компьютеров, поддерживающих WPA\-Enterprise и AES, а также другие компьютеры, которые поддерживают только WPA\-Enterprise и TKIP, необходимо определить, нужно ли.

- Настройте один профиль для поддержки все беспроводные компьютеры, используя метод шифрования слабая что все компьютеры поддерживают — в этом случае WPA\ Enterprise и TKIP.  
- Настройка двух профилей, чтобы обеспечить оптимальную безопасность, поддерживаемый каждый компьютер. В этом случае следует настроить один профиль, который задает надежного шифрования \ (WPA2\ предприятия и AES\) и один профиль, с использованием слабого шифрования WPA\ Enterprise и TKIP. В этом примере важно поместить профиль, который использует WPA2\-Enterprise и AES наивысший в порядке предпочтения. Компьютеры, которые не поддерживают использование WPA2\-Enterprise и AES будет автоматически перейдите к следующему профиля в порядке предпочтения и обрабатывать профиль, который указывает WPA\ Enterprise и TKIP.

> [!IMPORTANT]
> Профиль наиболее безопасный высоким стандартам необходимо поместить в упорядоченный список профилей, поскольку первый профиль, что они способны использовать используют подключении компьютеров.

### <a name="planning-restricted-access-to-the-wireless-network"></a>Планирование ограниченный доступ к беспроводной сети

Во многих случаях может потребоваться предоставить пользователям различные уровни доступа к беспроводной сети. Например может потребоваться разрешить некоторые пользователи неограниченный доступ любого часа каждый день недели. Для других пользователей можно только для разрешения доступа часы core, с понедельника по пятницу и запретить доступ на суббота и воскресенье.

В этом руководстве приведены инструкции по созданию доступа к среде, в которой помещает всех беспроводных пользователей в группу с общий доступ к ресурсам беспроводной сети. Можно создать одну группу безопасности беспроводной пользователей в Active Directory-пользователи и компьютеры оснастка, а затем сделайте каждого пользователя, для которого вы хотите предоставить беспроводного доступа, является членом этой группы.

При настройке политики сети сервер политики сети, можно указать группу безопасности пользователей беспроводных как объект, который обрабатывает NPS при определении авторизации.

Тем не менее если развертывание требует поддержки различные уровни доступа нужно только необходимо следующее:  

1. Создание более одной беспроводной группы безопасности пользователей для создания дополнительной безопасности беспроводной сети групп в Active Directory-пользователи и компьютеры. Например можно создать группу, содержащую пользователей, которые имеют полный доступ, в группу для тех, кто доступны только во время обычных рабочих часов и других групп, которые соответствуют другие условия, которые соответствуют вашим требованиям.

2. Добавьте пользователей в нужные группы безопасности, созданные.

3. Настройка дополнительных политик сети сервер политики сети для каждой группы дополнительной безопасности беспроводной сети и настроить политики для применения условия и ограничения, которые требуются для каждой группы.

### <a name="planning-methods-for-adding-new-wireless-computers"></a>Планирование методы для добавления новых компьютеров беспроводной сети

С помощью проводного подключения к сегменту сети, имеет доступ к контроллерам домена, который не защищен 802.1 X с проверкой подлинности коммутатор Ethernet является предпочтительным методом для присоединения нового беспроводные компьютеры к домену и вход в домен.

В некоторых случаях Однако, не возможно неудобно использовать проводное подключение присоединение компьютеров к домену, или пользователь может использовать проводное подключение для первого входа при попытке с помощью компьютеров, которые уже присоединены к домену.

Чтобы присоединить компьютер к домену с помощью беспроводного подключения или пользователям выполнять вход в домен первый раз с помощью компьютера, присоединенного к domain\ и по беспроводному соединению, беспроводные клиенты сначала должен установить подключение к беспроводной сети в сегменте, имеет доступ к контроллерам домена сети с помощью одного из следующих методов.

1. **Членом ИТ-специалисты присоединяет компьютер к домену и затем настраивает единого входа загрузочный профиль беспроводной связи.** С помощью этого метода ИТ-администратор беспроводной компьютер подключается к проводной сети Ethernet и затем компьютер присоединяется к домену. Затем администратор распространяет компьютера пользователю. При запуске компьютера, учетные данные домена, указываемые им для процесса входа пользователя используются для подключения к беспроводной сети и войдите в домен.  

2. **Пользователь вручную настраивает на компьютере с загрузочного профиля беспроводной сети и затем присоединения к домену.** С помощью этого метода пользователи вручную настроить свои компьютеры беспроводной сети с загрузочного профиля беспроводной сети, в зависимости от инструкции от ИТ-администратора. Загрузочный профиль беспроводной сети позволяет пользователям установить беспроводное подключение, а затем присоединить компьютер к домену. После присоединения компьютера к домену и перезагрузки компьютера, пользователь может войти домен с помощью беспроводного подключения и учетные записи домена.

Для развертывания беспроводного доступа, в разделе [развертывания беспроводного доступа](e-wireless-access-deployment.md).
