# <a name="wireless-access-deployment-planning"></a>Планирование развертывания беспроводного доступа

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Перед развертыванием беспроводного доступа, необходимо спланировать перечисленные ниже:

- Установка точки беспроводного доступа \(APs\) в сети

- Конфигурация клиента беспроводной сети и доступа

Следующие разделы содержат сведения на эти действия по планированию.

## <a name="planning-wireless-ap-installations"></a>Планирование установки беспроводных AP
При разработке решения доступа беспроводной сети, выполните следующие действия.

1. Определить, какие стандарты должны поддерживать беспроводных точках доступа
2. Определение областей покрытия, где вы хотите предоставить услуг беспроводной связи
3. Определить, где будет располагаться беспроводные точки доступа

Кроме того необходимо спланировать схему IP-адресов вашей беспроводной точкой доступа и беспроводных клиентов. См. в разделе **конфигурацию из беспроводной точке доступа на сервере политики сети** ниже дополнительные сведения.

### <a name="verify-wireless-ap-support-for-standards"></a>Проверьте беспроводных AP поддержка стандартов
В рамках согласованность и простота развертывания и управления AP рекомендуется развертывать беспроводные точки доступа же торговой марки и модели.

Эти беспроводные точки доступа, которые можно развернуть, должно поддерживать следующее:

- **IEEE 802.1 X**

- **Проверка подлинности RADIUS**

- **Проверка подлинности и шифрования.** В списке в порядке от наиболее к наименее предпочитаемый:

    1.  WPA2\-предприятия с помощью AES

    2.  WPA2\-Enterprise с TKIP

    3.  WPA\-предприятия с помощью AES

    4.  WPA\-Enterprise с TKIP

>[!NOTE]
>Чтобы развернуть WPA2, необходимо использовать беспроводных сетевых адаптерах и беспроводные точки доступа, которые также поддерживают WPA2. В противном случае используйте WPA\-Enterprise.

Кроме того обеспечивает повышенную безопасность для сети, беспроводные точки доступа должны поддерживать следующие параметры безопасности:

- **Фильтрация DHCP.** Беспроводной точке доступа необходимо выполнить фильтрацию по IP-портам для предотвращения передачи широковещательных сообщений DHCP в тех случаях, когда беспроводной клиент работает в режиме DHCP-сервера. Беспроводной точке доступа необходимо заблокировать отправку IP-пакетов из порт UDP с номером 68 к сети клиентами.

- **Фильтрация DNS.** Беспроводной точке доступа необходимо выполнить фильтрацию по IP-портам, чтобы предотвратить выполнение в качестве DNS-сервера клиента. Беспроводной точке доступа необходимо заблокировать отправку, что IP-пакетов от TCP или UDP-порт 53 к сети клиентами.

- **Изоляцию клиентов** Если вашей беспроводной точке доступа предоставляет клиент возможностей изоляции, следует включить функцию предотвратить возможные протокол разрешения адресов \(ARP\) спуфинг эксплойтов.

### <a name="identify-areas-of-coverage-for-wireless-users"></a>Определение областей покрытия для пользователей беспроводных сетей
Использование архитектуры рисунки этажа для каждого здания для определения областей, где вы хотите предоставить беспроводной покрытия. Например определите соответствующие офисы, комнаты конференциях, вестибюль, cafeterias или courtyards.

На рисунки, указания всех устройств, которые мешают беспроводных сигналов, например медицинского оборудования, беспроводные камеры видео, беспроводные телефоны, которые работают в 2.4 по технологических 2,5 ГГц, экспоненциальном и медицинских \(ISM\) диапазон и Bluetooth\-устройства с поддержкой.

На панели инструментов Рисование пометьте аспекты здания, в котором может повлиять на работу беспроводных сигналов; Metal объекты, используемые при построении здания может повлиять на беспроводного сигнала. Например следующие общие объекты могут мешать сигнала распространения: Лифты, отопления и air\-сглаживание трубы и конкретные поддержки мостовые фермы.

Обратитесь к производителю "AP" сведения об источниках, которые могут вызвать затухания радиочастотной беспроводных AP. Большинство APs предоставляют тестирования программного обеспечения, которое можно использовать для проверки для сигнала, частота ошибок и пропускная способность.

### <a name="determine-where-to-install-wireless-aps"></a>Выбор места установки беспроводных точках доступа
На архитектурные рисунки найдите закройте беспроводных точках доступа достаточно вместе для предоставления достаточно беспроводной покрытие, но достаточно далеко друг от друга, что они не конфликтуют друг с другом.

Необходимые расстояние между точками доступа зависит от типа из Тихоокеанского Региона и антенны AP, аспекты здания, которые блокируют беспроводных сигналов и других источников помех. Можно пометить беспроводных AP размещения, чтобы каждый беспроводной точке доступа не более чем 300 метров от смежные беспроводной точке доступа. См. в разделе, беспроводных AP документации по AP спецификации и рекомендации для размещения.

Временной установки беспроводных точках доступа в расположениях, указанных на архитектурные рисунков. На ноутбуке, оснащенных беспроводной адаптер 802.11 и Обзорное программное обеспечение, часто предоставляются с помощью адаптеров беспроводной сети, а также определите уровень сигнала в пределах каждой зоны покрытия.

В области покрытия, где сила сигнала мала поместите AP, чтобы повысить уровень сигнала для зоны покрытия, установите дополнительные беспроводные точки доступа содержат необходимые сведения, переместить или удалить источники помех сигнала.  

Обновление вашей архитектуры рисунков, чтобы указать конечное положение всех беспроводных точек доступа. Наличие карту размещения точных AP поможет позднее, во время устранения неполадок операций, или если вы хотите обновить или заменить APs.

### <a name="plan-wireless-ap-and-nps-radius-client-configuration"></a>Планирование беспроводной точкой доступа и клиента RADIUS NPS конфигурации
Сервер политики сети можно использовать для настройки беспроводных точках доступа, по отдельности или в группах.

При развертывании больших беспроводную сеть, которая включает в себя множество APs, гораздо проще настроить APs в группах. Добавление точки доступа в качестве групп клиента RADIUS на сервере политики сети, необходимо настроить точки доступа со следующими свойствами.

- Эти беспроводные точки доступа настроены с IP-адреса из диапазона адресов.

- Эти беспроводные точки доступа настраиваются с тот же общий секрет.

### <a name="plan-the-use-of-peap-fast-reconnect"></a>Планируйте использование быстрое переподключение PEAP
В инфраструктуре 802.1 X точки беспроводного доступа настраиваются как RADIUS-клиенты RADIUS-серверов. Развертывается при быстрое переподключение PEAP, беспроводной клиент, который перемещается между двух или нескольких точек доступа не требуется проходить проверку подлинности в каждой новой связи.

Быстрое переподключение PEAP уменьшает время отклика при проверке подлинности между клиентом и сервером, так как запрос на проверку подлинности пересылаются из новой точки доступа на сервер политики сети, изначально создана для проверки подлинности и авторизации для клиента запрос на подключение.

Так как PEAP клиента и сервера политики сети, как использовать ранее кэшированных Transport Layer Security \(TLS\) свойства соединения \(набор которых называется TLS дескриптор\), сервер политики сети можно быстро определить, клиент получает полномочия для переподключения.

>[!IMPORTANT]
>Для быстро повторно подключиться к правильной, точки доступа должны быть настроены как RADIUS-клиенты же сервера политики сети.

Если исходный сервер политики сети становится недоступным или если клиент переместится к точке доступа, настроенный в качестве клиента RADIUS на другой сервер RADIUS, полноценную проверку подлинности между клиентом и новые средства проверки подлинности.

### <a name="wireless-ap-configuration"></a>Конфигурация беспроводных AP
В следующем списке перечислены элементы, которые обычно настраиваются на 802.1 X\-поддержкой беспроводных точках доступа:

> [!NOTE]
> Имена элементов могут зависеть от торговой марки и модели и могут отличаться от показанных в следующем списке. См. в документации беспроводной точкой доступа для конфигурации\-конкретные сведения.

- **Идентификатор набора служб \(SSID\)**. Это имя беспроводной сети \(к примеру, ExampleWlan\)и имя, которое объявляется беспроводных клиентов. Чтобы избежать путаницы, SSID, выбранная для объявления не должны совпадать с SSID, передавать, любые беспроводные сети, которые находятся в пределах диапазона приема беспроводной сети.

    В случаях, в которых несколько беспроводные точки доступа развернуты как часть одной беспроводной сети настройте каждый беспроводной точке доступа с тем же SSID. В случаях, в которых несколько беспроводные точки доступа развернуты как часть одной беспроводной сети настройте каждый беспроводной точке доступа с тем же SSID.  

    В случаях, когда имеется необходимо развертывать беспроводные сети в соответствии с потребностями конкретной организации, беспроводной точки доступа в одной сети должен широковещательных разных SSID идентификатор SSID другие сети\(s\). Например, если вам требуется отдельный беспроводной сети для сотрудников и гостей, можно настроить беспроводных точках доступа для бизнес-сети с SSID равным широковещательных **ExampleWLAN**. Гостевой сети, при этом возможно установить SSID каждого беспроводной точкой доступа для рассылки **GuestWLAN**. Таким образом сотрудники и гостей можно подключиться к соответствующей сети позволяет избежать ненужных путаницы.  

    > [!TIP]  
    > Некоторые беспроводной точке доступа имеют рассылать несколько SSID в соответствии с несколькими\-развертывание по сети. Беспроводной точке доступа, могут производить широковещательную рассылку несколько SSID можно снизить, развертывания и производственных потребностей затраты.  

- **Беспроводные проверки подлинности и шифрования**.

    Проверки подлинности беспроводной сети используется проверка подлинности безопасности, который используется, когда беспроводной клиент связывается с точкой беспроводного доступа.  

    Беспроводной шифрование — шифр шифрования безопасности, который используется с помощью проверки подлинности беспроводной сети для защиты обмена информацией, отправляемых между беспроводной точкой доступа и клиент беспроводной связи.  

- **Беспроводных AP IP-адрес \(статический\)**. На каждой точке беспроводного доступа настройте уникальный статический IP-адрес. Если подсети обслуживается DHCP-сервер, убедитесь, что все AP IP-адреса не попадает в пределах диапазона исключения DHCP, чтобы DHCP-сервер не будет пытаться выполнить тот же адрес на другом компьютере или устройстве. Диапазоны исключения, описанные в процедуре «Создание и активация новой области DHCP» в [руководство по основной сети](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide). Если вы планируете настроить APs в качестве RADIUS-клиенты группой на сервере политики сети, каждой точки беспроводного доступа в группе должен иметь IP-адрес и том же диапазон IP-адресов.

- **DNS-имя**. Некоторые беспроводные точки доступа можно настроить с помощью DNS-имя. Настройка каждой точке беспроводного доступа с уникальным именем. Например, если у вас есть развернутые беспроводные точки доступа в нескольких\-истории построения, можно назвать первые три беспроводных точках доступа, которые развертываются на третьем этаже AP3\-01, AP3\-02 и AP3\-03.

- **Маска подсети беспроводных AP**. Настройте маску, чтобы указать, какая часть IP-адрес — это идентификатор и какая часть IP-адреса является узлом.

- **Служба AP DHCP**. Если вашей беспроводной точкой доступа имеет встроенный\-в службе DHCP, отключите его.

- **Общий секрет RADIUS**. Если вы планируете настроить клиенты NPS RADIUS в группы — в какие обстоятельствах необходимо будет настроить все точки доступа в группе, указав тот же общий секрет, используйте уникальный RADIUS общий секрет для каждой точки доступа. Общие секреты должны быть случайную последовательность по крайней мере 22 символов и включать и прописные и строчные буквы, цифры и знаки препинания. Чтобы случайность, можно использовать для создания общих секретов программы создания случайных символов. Рекомендуется записывать общий секрет для каждого беспроводной точкой доступа и сохраните его в безопасном месте, например office safe. При настройке RADIUS-клиентов в консоли сервера политики сети вы создадите виртуальную версию каждой ТД. Общий секрет, который настраивается для каждой виртуальной точки доступа на сервере политики сети должен соответствовать общие секреты AP действительной, физический.

- **IP-адрес сервера RADIUS**. Введите IP-адрес сервера политики сети, который вы хотите использовать для проверки подлинности и авторизации запросов на подключение к этой точке доступа.

- **UDP-порт\(s\)**. По умолчанию сервер политики сети использует UDP-порты 1812 и 1645 для сообщений проверки подлинности RADIUS и порты UDP 1813 и 1646 для сообщений учета RADIUS. Рекомендуется не изменять параметры по умолчанию порты RADIUS UDP.

- **VSA**. Некоторые беспроводные точки доступа требуют поставщика\-определенные атрибуты \(VSA\) для обеспечения полной беспроводных AP функциональности.

- **Фильтрация DHCP**. Настройка беспроводных точках доступа для блокировки беспроводные клиенты из отправки IP-пакетов из порт UDP с номером 68 к сети. См. в документации для вашей беспроводной точкой доступа настроить фильтрацию DHCP.

- **Фильтрация DNS**. Настройка беспроводных точках доступа для блокировки беспроводные клиенты из отправки IP-пакетов из TCP или UDP-порт 53 к сети. См. в документации для вашей беспроводной точкой доступа настроить фильтрацию DNS.

## <a name="planning-wireless-client-configuration-and-access"></a>Планирование конфигурации клиента беспроводной сети и доступа

При планировании развертывания 802.1 X\-беспроводной доступ с проверкой подлинности необходимо учитывать ряд клиентских\-определенных факторов:

- **Планирование нескольких стандартов**.

    Определите ли беспроводные компьютеры используют ту же версию Windows или того, являются ли они сочетание компьютеров под управлением разных операционных систем. Если они отличаются, убедитесь, что вы понимаете все различия в стандартов, поддерживаемых операционными системами.

    Определите, поддерживают ли все адаптеры беспроводной сети на всех компьютерах беспроводного клиента те же стандарты беспроводной или необходима для поддержки различных стандартов. Например, определить, поддерживает ли некоторые драйверы оборудования сетевого адаптера WPA2\-Enterprise и AES, хотя другие поддерживают только WPA\-Enterprise и TKIP.

- **Планирования режим проверки подлинности клиента**. Режимы проверки подлинности определяют, каким образом клиенты Windows обрабатывать учетные данные домена. Можно выбрать один из следующих режимов проверки подлинности три сети в политиках беспроводной сети.  

    1. **Пользователь повторно\-проверки подлинности**. Этот режим указывает, что проверка подлинности всегда проводится с использованием учетных данных безопасности на основе текущего состояния компьютера. Если никто из пользователей не вошел на компьютер, проверка подлинности выполняется с помощью учетных данных компьютера. Когда пользователь вошел в систему на компьютере, проверка подлинности всегда проводится с помощью учетных данных пользователя.  

    2. **Только компьютер**. Компьютер только режим указывает, что проверка подлинности всегда выполняется с использованием только учетных данных компьютера.  

    3.  **Проверка подлинности пользователя**. Указывает режим проверки подлинности пользователя, проверку подлинности выполняется только в том случае, когда пользователь выполнил вход на компьютер. Если нет пользователей, выполнивших вход на компьютер, не выполняются попытки проверки подлинности.  

- **Планирование беспроводной ограничения**. Определите, требуется ли предоставить всех беспроводных пользователей тот же уровень доступа к беспроводной сети, или того, требуется ли ограничить доступ для некоторых пользователей беспроводных. Два способа применения ограничений на сервере политики сети к определенным группам пользователей беспроводных сетей.  Например можно определить определенные дни и часы, определенные группы будет разрешен доступ к беспроводной сети.

### <a name="planning-support-for-multiple-standards"></a>Планирование нескольких стандартов

Беспроводной сети \(IEEE 802.11\) расширение политики в групповой политике предоставляет широкий набор параметров конфигурации для поддержки различных вариантов развертывания.

Можно развертывать беспроводные точки доступа, для которых настроена стандарты, которые требуется поддерживать и затем настройте несколько профилей беспроводной связи в беспроводной сети \(IEEE 802.11\) политики, с каждым профилем, указав один набор стандартов что вам требуется.

Например, если в сети имеется беспроводные компьютеры, которые поддерживают WPA2\-Enterprise и AES, других компьютеров, поддерживающих WPA\-Enterprise и AES и другие компьютеры, которые поддерживают только WPA\-Enterprise и TKIP, необходимо Определите, хотите ли вы:

- Настройте один профиль для обеспечения поддержки беспроводных компьютеров с помощью слабым метод шифрования, все компьютеры поддерживают - в этом случае WPA\-Enterprise и TKIP.  
- Настройте два профиля для предоставления обеспечения наибольшей безопасности, поддерживаемый каждый компьютер. В этот экземпляр необходимо настроить профиль, который указывает стойкое шифрование \(WPA2\-Enterprise и AES\)и один профиль, который использует слабый WPA\-Enterprise и TKIP шифрования. В этом примере важно размещать профиль, который использует WPA2\-Enterprise и AES, наивысшая в порядке предпочтения. Компьютеры, которые не поддерживают использования WPA2\-Enterprise и AES автоматически пропустить следующий профиль в порядке предпочтения и обработать профиль, который указывает WPA\-Enterprise и TKIP.

> [!IMPORTANT]
> Профиль наиболее безопасный высоким стандартам необходимо разместить в упорядоченный список профилей, так как при подключении компьютеров используется первый профиль, что они способны использовать.

### <a name="planning-restricted-access-to-the-wireless-network"></a>Планирование ограниченный доступ к беспроводной сети

Для других пользователей можно только разрешить доступ в течение дня, с понедельника по пятницу, и запретить доступ в субботу и воскресенье.

Создайте одну группу безопасности пользователей беспроводных сетей в Active Directory-пользователи и компьютеры привязать\-in и затем каждый пользователь, для которого требуется предоставить беспроводного доступа является членом этой группы.

При можно настроить политики сети сервер политики сети, необходимо указать группу безопасности пользователей беспроводных сетей, как объект, который обрабатывает NPS, при определении авторизации.

Тем не менее если развертывание требует поддержки для различных уровней доступа требуется только необходимо следующее:  

1. Создание более одной беспроводной группы безопасности пользователей для создания дополнительных безопасности беспроводных сетей групп в Active Directory — пользователи и компьютеры. Например можно создать группу, содержащую пользователей, которые имеют полный доступ, группу для тех, кто будет доступен только во время обычных рабочих часов и других групп, которые соответствуют другим критериям, в соответствии со своими требованиями.

2. Добавьте пользователей в соответствующие группы безопасности, которые вы создали.

3. Настройка дополнительных политик сети сервер политики сети для каждой группы дополнительной безопасности беспроводной сети и настроить политики для применения условия и ограничения, которые необходимы для каждой группы.

### <a name="planning-methods-for-adding-new-wireless-computers"></a>Методы планирования для добавления новых беспроводных компьютеров

Для присоединения новых беспроводных компьютеров к домену и вход в домен рекомендуется с помощью проводного подключения с сегментом в локальной сети, имеет доступ к контроллерам домена и не защищены 802.1 X с проверкой подлинности коммутатор Ethernet.

В некоторых случаях Однако он может оказаться нецелесообразным использовать проводного подключения для присоединения компьютеров к домену, или пользователь может использовать проводного соединения для первого входа при попытке с помощью компьютеров, которые уже присоединены к домену.

Чтобы присоединить компьютер к домену, с помощью беспроводного соединения или для пользователей для входа на время первого домена с помощью домена\-компьютере, присоединенном к и по беспроводному соединению, беспроводные клиенты сначала необходимо установить подключение к беспроводной сети на Сегмент, который имеет доступ к контроллерам домена сети с помощью одного из следующих методов.

1. **Членом ИТ-специалистам присоединяют компьютер к домену и затем настраивает Single Sign On загрузочного профиля беспроводной сети.** В этом случае ИТ-администратор подключает компьютер к проводной сети Ethernet и затем присоединяет компьютер к домену. Затем администратор передает компьютер пользователю. Когда пользователь запускает компьютер, учетные данные домена, указываемые им для процесса входа пользователя используются для подключения к беспроводной сети и войдите в домен.  

2. **Пользователь вручную настраивает на компьютере загрузочный профиль для беспроводной сети и затем присоединяется к домену.** С помощью этого метода пользователи вручную настроить свои беспроводные компьютеры с загрузочного профиля беспроводной сети на основе указаний из ИТ-администратору. Загрузочного профиля беспроводной сети позволяет пользователю установить беспроводное подключение, а затем присоединить компьютер к домену. После присоединения компьютера к домену и перезагрузки компьютера, пользователь может войти в домен с помощью беспроводного соединения и учетные данные учетной записи домена.

Развертывание беспроводного доступа, см. в разделе [развертывания беспроводного доступа](e-wireless-access-deployment.md).
