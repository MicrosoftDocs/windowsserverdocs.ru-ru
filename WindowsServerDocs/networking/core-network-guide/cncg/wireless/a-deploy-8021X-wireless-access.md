# <a name="deploy-password-based-8021x-authenticated-wireless-access"></a>Развертывание пароль\-на основе 802.1 X беспроводной доступ с проверкой подлинности

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

Это дополнительное руководство по серверу Windows&reg; 2016 руководство по основной сети. Руководство по основной сети предоставляет инструкции по планированию и развертыванию компонентов, необходимых для создания полнофункциональной сети и нового домена Active Directory® в новом лесу.

В этом руководстве рассматривается построение основной сети приводятся инструкции по развертыванию института эта функция \(IEEE\) 802.1 X\-использование прошедшего проверку подлинности беспроводного доступа IEEE 802.11 Protected Extensible Authentication Protocol — протокол проверки пароля Microsoft версии 2 \(PEAP\-MS\-CHAP v2\).

Так как PEAP\-MS\-CHAP v2 пользователи должны предоставлять пароль\-на основе учетных данных, а не сертификат во время процесса проверки подлинности, это обычно проще и дешевле, чем EAPразвертывание\-TLS или PEAP\-TLS.

>[!NOTE]
>В этом руководстве IEEE 802.1 X беспроводной доступ с проверкой в PEAP\-MS\-CHAP версии 2 сокращается до «беспроводной доступ» и «Доступ по Wi-Fi».

В этом руководстве, в сочетании с направляющими готовности к установке, описанные ниже, содержит инструкции о том, как развертывание следующей инфраструктуры доступа Wi-Fi.

- Сертификаты сервера для компьютеров под управлением NPS

- Беспроводные клиентские компьютеры под управлением Windows&reg; 10, Windows 8.1 или Windows 8.

## <a name="technology-overviews"></a>Обзор технологий
Ниже приведены Обзор технологий для развертывания беспроводного доступа:

### <a name="ieee-8021x"></a>IEEE 802.1X
Стандарт IEEE 802.1 X определяет порт\-на основе управления доступом к сети, который используется для предоставления доступа к сети с проверкой подлинности к Ethernet-сетям. Этот порт\-управление доступом на основе сети используются физические характеристики инфраструктуры коммутируемой локальной сети для проверки подлинности устройств, подключенных к Сетевому порту. В случае непрохождения проверки подлинности доступ к данному порту может быть запрещен. Несмотря на то, что этот стандарт, предназначенный для проводных Ethernet-сетей, он был адаптирован для использования в беспроводных локальных сетях 802.11.

### <a name="8021x-capable-wireless-access-points-aps"></a>802.1 X\-точек беспроводного доступа с поддержкой \(APs\)
В этом сценарии требуется развертывание одного или нескольких 802.1 X\-поддержкой беспроводных точках доступа, совместимых с удаленной проверки подлинности удаленного\-пользователя службы \(RADIUS\) протокола.

802.1 X и RADIUS\-совместимые точки доступа, при развертывании в инфраструктуре протокола RADIUS с RADIUS-сервером, таких как сервер политики сети называются *RADIUS-клиенты*.

### <a name="wireless-clients"></a>Беспроводные клиенты
Это руководство содержит сведения о комплексное конфигурации для предоставления 802.1 X доступ с проверкой подлинности для домена\-пользователей, которые подключаются к сети с помощью беспроводных клиентских компьютеров под управлением Windows 10, Windows 8.1 и Windows 8. Компьютеры должны быть присоединены к домену, чтобы успешно установить доступ с проверкой подлинности.

>[!NOTE]
>Также можно использовать компьютеры под управлением Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012, как клиенты беспроводной связи.

### <a name="support-for-ieee-80211-standards"></a>Поддержка IEEE 802.11 стандартов
Поддерживаемые операционные системы Windows и Windows Server предоставляют встроенные\-поддержка 802.11 беспроводной сети. В этих операционных систем установленных адаптер беспроводной сети 802.11 отображается как беспроводного сетевого подключения в сети и общим доступом. 

Несмотря на то, что существует построена\-поддержка беспроводные сети 802.11, компоненты беспроводной связи Windows, зависят от следующих:

- Возможности беспроводного сетевого адаптера. Установленные беспроводного сетевого адаптера должен поддерживать беспроводной локальной сети или стандартов безопасности беспроводных сетей, которые необходимы. Например, если адаптер беспроводной сети не поддерживает Wi\-защищенный доступ Fi \(WPA\), нельзя включить или настроить параметры безопасности WPA.

- Возможности беспроводного сетевого адаптера. Для дают возможность настраивать параметры беспроводной сети, драйвер беспроводного сетевого адаптера должен поддерживать создание отчетов по все его возможности для Windows. Убедитесь, что драйвер беспроводного сетевого адаптера написан для использования возможностей операционной системы. Также убедитесь, что драйвер самой последней версии путем проверки центра обновления Майкрософт или веб-сайта поставщика адаптера беспроводной сети.

### <a name="wireless-network-security-methods"></a>Методы безопасности беспроводной сети
**Беспроводные сети методы безопасности** представляет собой неофициальные группировку проверки подлинности беспроводной сети \(иногда называют безопасности беспроводных сетей\) и шифрование безопасности беспроводных сетей. Проверка подлинности и шифрования используются в пары для предотвращения несанкционированного доступа к беспроводной сети и защиты беспроводной передаче данных. 

При настройке параметров безопасности беспроводной сети в беспроводной сети из групповой политики, существует несколько сочетаний для выбора. Тем не менее только WPA2\-Enterprise, WPA\-Enterprise и открытая с 802.1 X стандарты аутентификации поддерживаются для 802.1 X с проверкой подлинности беспроводных сред.

>[!NOTE]
>При настройке политик беспроводной сети, необходимо выбрать **WPA2\-Enterprise**, **WPA\-Enterprise**, или **открыть с 802.1 X** в порядок, чтобы получить доступ к параметрам EAP, которые требуются для подключений 802.1 X с проверкой подлинности беспроводных сред.  

#### <a name="wireless-authentication"></a>Проверка подлинности беспроводной сети
Данное руководство рекомендует использовать следующие стандарты проверки подлинности беспроводной сети для 802.1 X с проверкой подлинности беспроводных сред.  

**Wi\-Fi защищенного доступа — Enterprise \(WPA\-Enterprise\)**  WPA-Шифрование является внутренним стандартом разработанный Wi-Fi Alliance, в соответствии с протоколом беспроводной связи 802.11. Протокол WPA был разработан в ответ на ряд серьезные недостатки, которые были обнаружены в выше Wired Equivalent Privacy \(WEP\) протокола.

WPA\-Enterprise обеспечивает улучшенную безопасность WEP-Шифрование с:  

1. Проверка подлинности, использующее платформу 802.1 X EAP как часть инфраструктуры, обеспечивающий централизованное взаимной проверки подлинности и динамического управления ключами  

2. Улучшение значение проверки целостности \(значение проверки Подлинности\) с проверки целостности сообщения \(MIC\), чтобы защитить заголовок и полезные данные  

3. Реализация счетчик кадров, чтобы не допустить атак с повторением  

**Wi\-шифрования WPA2-Enterprise \(WPA2\-Enterprise\)**  как WPA\-Enterprise standard, WPA2\-предприятия использует 802.1 X и EAP framework. WPA2\-Enterprise обеспечивает более надежную защиту данных для нескольких пользователей и крупных управляемыми сетями. WPA2\-Enterprise — это надежный протокол, который предназначен для предотвращения несанкционированного сетевого доступа, проверяя пользователей через сервер проверки подлинности.  

#### <a name="wireless-security-encryption"></a>Шифрование безопасности беспроводных сетей
Шифрование безопасности беспроводной сети используется для защиты беспроводной передаче данных, которыми обмениваются клиент беспроводной связи и беспроводной точке доступа. Шифрование безопасности беспроводной сети используется в сочетании с методом проверки подлинности для безопасности выбранной сети. По умолчанию компьютеры под управлением Windows 10, Windows 8.1 и Windows 8 поддерживают два стандартов шифрования:

1. **Временной ключ целостности протокола** \(TKIP\) — это старый протокол шифрования, которая была первоначально разработана для обеспечения более безопасные беспроводные шифрования, отличный от предложенного по по своей природе слабые Wired Equivalent Privacy \(WEP\) протокола. TKIP был разработан IEEE 802.11i задач группы и Wi\-Fi Alliance для замены WEP-Шифрования без необходимости замены устаревшего оборудования. TKIP — это набор алгоритмов, который инкапсулирует WEP-Шифрование полезных данных, а также позволяет пользователям устаревшее оборудование Wi-Fi для обновления до TKIP без замены оборудования. Как и WEP TKIP использует алгоритм шифрования RC4 потока в основу. Новый протокол, однако шифрует каждый пакет данных с помощью уникальный ключ шифрования, а ключи являются намного надежнее, чем с WEP-Шифрование. Несмотря на то, что TKIP полезен для обновление системы безопасности на старых устройств, которые были разработаны для использования только WEP, он не устраняет все проблемы безопасности, с которыми сталкиваются беспроводных локальных сетей и в большинстве случаев не достаточно надежна для защиты конфиденциальных для государственных организаций или корпоративные данные передачи данных.  

2. **Advanced Encryption Standard** \(AES\) предпочтительный шифрования для шифрования данных, коммерческих и государственных протокол. AES обеспечивает более высокий уровень безопасности беспроводной передачи данных, чем TKIP или WEP. В отличие от TKIP и WEP AES требует оборудования для беспроводных сетей, поддерживающий стандарт AES. AES — это симметричный\-ключа стандарту шифрования, использующий три блочные шифры, AES\-128, AES\-192 и AES\-256.

В Windows Server 2016 следующие AES\-методов шифрования в беспроводных сетях на основе доступных для настройки свойства профиля беспроводной связи, при выборе метода проверки подлинности WPA2\-Enterprise, (рекомендуется).

1. **AES\-CCMP**. Счетчик режим шифрования блока цепочки сообщения протокола проверки подлинности кода \(CCMP\) реализует стандартный стандарта 802.11i предназначен для более поздней версии шифрование, чем WEP и использует 128-битных ключей шифрования AES.
2. **AES\-GCMP**. Протокол режим счетчика Galois \(GCMP\) поддерживает стандарт 802.11ac, более эффективно, чем AES\-CCMP и обеспечивает более высокую производительность для беспроводных клиентов. GCMP используются ключи шифрования AES 256-разрядный.

> [!IMPORTANT]
> Проводной конфиденциальности эквивалентности \(WEP\) был исходная Стандартная безопасности беспроводных сетей, который использовался для шифрования трафика. Не следует развертывать WEP-Шифрования в сети, так как существуют хорошо\-известные уязвимости в этой устаревшей форме безопасности.

### <a name="active-directorydoman-services-adds"></a>Доменных служб Active Directory \(AD DS\)
AD DS предоставляют распределенную базу данных, который хранит и управляет сведениями о сетевых ресурсах и приложения\-определенные данные из каталога\-приложений с поддержкой. Администраторы могут использовать AD DS для упорядочения в иерархическую вложенную структуру таких элементов сети, как пользователи, компьютеры и другие устройства. Иерархическая вложенная структура включает лес Active Directory, домены в лесу и организационные подразделения \(подразделения\) в каждом домене. Сервер, на котором выполняется AD DS, называется *контроллер домена*.  

AD DS содержит учетные записи пользователей, учетные записи компьютеров и свойства учетной записи, которые требуются IEEE 802.1 X и PEAP\-MS\-CHAP версии 2 для проверки подлинности учетных данных пользователя и оценки авторизации для беспроводных подключений.

### <a name="active-directory-users-and-computers"></a>Active Directory — пользователи и компьютеры
Active Directory — пользователи и компьютеры — это компонент службы AD DS, который содержит учетные записи, которые представляют физические объекты, например компьютер, пользователя или группу безопасности. Объект *группы безопасности* является коллекцией учетных записей пользователя или компьютера, которые администраторы могут управлять как единое целое. Учетные записи пользователей и компьютеров, принадлежащих к определенной группе, называются *членов группы*.  

### <a name="group-policy-management"></a>Управление групповой политикой
Управление групповой политикой позволяет directory\-на основе управления изменениями и настройкой параметров пользователя и компьютера, включая сведения о безопасности и пользователей. Использовать групповую политику для определения конфигурации групп пользователей и компьютеров. С помощью групповой политики можно указать параметры для записи реестра, безопасности, установки программного обеспечения, сценарии, перенаправление папок, службы удаленной установки и обслуживания Internet Explorer. Создаваемые параметры групповой политики содержатся в объекте групповой политики \(GPO\). Связав объект групповой Политики с выбранными контейнерами системы Active Directory — сайты, домены и подразделения — параметры объектов групповой Политики можно применить к пользователям и компьютерам в эти контейнеры Active Directory. Для управления объектами групповой политики на предприятии, можно использовать консоль управления групповыми политиками управления \(MMC\).  

В этом руководстве содержатся подробные инструкции о том, как параметры беспроводной сети \(IEEE 802.11\) расширения политик управления групповыми политиками. Беспроводной сети \(IEEE 802.11\) политики настраивают домена\-беспроводных клиентских компьютеров с подключение и параметры беспроводной связи для подключений 802.1 X беспроводной доступ с проверкой подлинности.

### <a name="server-certificates"></a>Сертификаты сервера
В этом сценарии развертывания требуются сертификаты каждого NPS, который выполняет проверку подлинности 802.1 X.  

Сертификат сервера — это электронный документ, который широко применяется для проверки подлинности и защиты данных в общедоступных сетях. Сертификат безопасно привязывает открытый ключ к объекту, обладающему закрытым ключом. Сертификаты утверждаются цифровой подписью ЦС, и они могут быть выданы для пользователя, компьютера или службы.  

Центр сертификации \(ЦС\) — это сущность, ответственность за установку и документальное подтверждение подлинности открытых ключей, принадлежащих субъектам \(обычно пользователи или компьютеры\) или другим ЦС. Действия центра сертификации можно включить привязки открытые ключи к различающимся именам посредством подписанных сертификатов, управлять серийными номерами сертификатов и отзыв сертификатов.  

Службы сертификатов Active Directory \(AD CS\) — это роль сервера, который выдает сертификаты выдаче. AD CS сертификатов инфраструктуры, также известный как *инфраструктуры открытых ключей \(PKI\)*, предоставляет настраиваемые службы выдачи и управления сертификатов для предприятия.

### <a name="eap-peap-and-peap-ms-chap-v2"></a>EAP, PEAP и PEAP\-MS\-CHAP v2
Протокол расширенной проверки подлинности \(EAP\) расширяет точки\-для\-протокол \(PPP\) , позволяя дополнительные методы проверки подлинности, которые используют учетные данные и сведения обмен произвольной длины. При использовании проверки подлинности EAP как в сети доступ к клиента и средством проверки подлинности \(например NPS\) должен поддерживать тот же тип EAP, для успешной проверки подлинности и выполнение. Windows Server 2016 включает инфраструктуру EAP, поддерживает два типа EAP и возможность передавать сообщения EAP NPSs. С помощью EAP, может поддерживать дополнительных схем проверки подлинности, известный как *типы EAP*. Ниже приведены типы EAP, поддерживаемых Windows Server 2016.  

- Безопасность на транспортном уровне \(TLS\)

- Майкрософт протокола CHAP версии 2 \(MS\-CHAP v2\)

>[!IMPORTANT]
>Строгие типы EAP \(например те, которые основаны на сертификатах\) обеспечивают лучшую защиту от грубой\-принудительно атаки, атак перебором по словарю и атаки, чем подбор паролей\-зависимости протоколы проверки подлинности \(например CHAP или MS\-CHAP версии 1\).

Защищенный EAP \(PEAP\) использует протокол TLS для создания защищенного канала между PEAP-клиентом, например беспроводного компьютера и проверки подлинности PEAP, таких как сервер политики сети или другие серверы RADIUS. PEAP указывает метод проверки подлинности, но он обеспечивает дополнительную безопасность для других протоколов проверки подлинности EAP \(такие как EAP\-MS\-CHAP v2\) , могут работать по зашифрованному каналу TLS предоставляемые PEAP. PEAP используется в качестве метода проверки подлинности для доступа клиентов, подключающихся к сети организации через следующие серверы сетевого доступа \(NAS\):

- 802.1 X\-поддерживает беспроводные точки доступа

- 802.1 X\-поддержкой коммутаторов с проверкой подлинности

- Компьютеры под управлением Windows Server 2016 и службы удаленного доступа \(RAS\) , настроенные в качестве виртуальной частной сети \(VPN\) серверов и серверов DirectAccess

- Компьютеры под управлением Windows Server 2016 и служб удаленных рабочих столов

PEAP\-MS\-CHAP v2 проще в развертывании чем EAP\-TLS так, как проверка подлинности пользователей выполняется с помощью пароля\-учетные данные на основе \(имя пользователя и пароль\), а не сертификаты или смарт-карты. Только сервер политики сети или другие серверы RADIUS должны иметь сертификат. Сертификат сервера политики сети используется сервером политики сети в процессе проверки подлинности для подтверждения его подлинности для PEAP-клиентов.

Это руководство содержит инструкции по настройке беспроводных клиентов и вашей NPS\(s\) для использования протокола PEAP\-MS\-доступ с проверкой подлинности CHAP версии 2 для 802. 1 X.

### <a name="network-policy-server"></a>Сервер политики сети
Сервер политики сети \(NPS\) позволяет централизованно настраивать и контролировать политики сети с помощью удаленного вызова проверки подлинности\-пользователя службы \(RADIUS\) сервера и прокси-сервера RADIUS. NPS-сервер необходим, если развертывание беспроводного доступа 802.1 X.

При настройке точки беспроводного доступа 802.1 X в качестве RADIUS-клиентов на сервере политики сети, сервер политики сети обрабатывает запросы на подключение, отправленных точки доступа. Во время обработки запросов на подключение, выполняет проверку подлинности и авторизацию. Проверки подлинности определяет, ли клиент представлены действительные учетные данные. Если сервер политики сети успешно проходит проверку подлинности клиента, отправившего запрос, NPS определяет соответствие клиента авторизован для запрошенного подключения и либо разрешает или запрещает подключение. Это объясняется более подробно следующим образом:

#### <a name="authentication"></a>Проверка подлинности

Успешно взаимной PEAP\-MS\-проверку подлинности CHAP версии 2 состоит из двух основных частей:

1.  Клиент выполняет проверку подлинности сервером политики сети.  На этом этапе взаимной проверки подлинности сервером политики сети отправляет свой сертификат сервера на клиентский компьютер, чтобы клиент мог проверить удостоверение сервера политики сети с помощью сертификата. Для успешной проверки подлинности сервером политики сети, клиентский компьютер должен доверять ЦС, выдавшего сертификат сервера политики сети. Клиент доверяет этим ЦС, если сертификат ЦС в хранилище сертификатов доверенных корневых центров сертификации на клиентском компьютере.

    При развертывании собственного частного ЦС, сертификат ЦС автоматически устанавливается в хранилище сертификатов доверенных корневых центров сертификации для текущего пользователя и для локального компьютера, при обновлении групповой политики на клиентском компьютере член домена. Если вы решили развернуть сертификаты сервера из общедоступного центра сертификации, убедитесь, что открытый сертификат ЦС уже находится в хранилище сертификатов доверенных корневых центров сертификации.  

2.  NPS выполняет проверку подлинности пользователя. После клиент успешно проходит проверку подлинности сервером политики сети, клиент отправляет пароль\-учетные данные для сервера политики сети, который проверяет учетные данные пользователя для учетных записей пользователей базы данных доменных служб Active Directory на основе \(AD DS\).

Если учетные данные допустимы, и проверка подлинности завершается успешно, сервер политики сети начинается на этапе авторизации обработки запроса на подключение. Если учетные данные не являются допустимыми и не пройдет проверку подлинности, сервер политики сети отправляет сообщение, отклонить доступ и запрос на подключение отклоняется.  

#### <a name="authorization"></a>Authorization

Сервер политики сети выполняет авторизацию следующим образом:  

1. NPS проверяет ограничения в удаленный учетной записи пользователя или компьютера\-в свойствах в Доменных службах Active Directory. Каждая учетная запись пользователя и компьютера в Active Directory — пользователи и компьютеры включает в себя несколько свойств, в том числе на **Dial\-в** вкладки. На этой вкладке в **права доступа к сети**, если значение равно **доступ**, пользователю или компьютеру разрешено подключаться к сети. Если значение равно **запретить доступ**, пользователь или компьютер не авторизован для подключения к сети. Если значение равно **управлять доступом с помощью политики сети**, сервер политики сети оценивает настроенные сетевые политики, чтобы определить, авторизован ли пользователь или компьютер для подключения к сети. 

2. NPS обрабатывает его сетевые политики, чтобы найти политику, которая соответствует запрос на подключение. При обнаружении соответствующей политики NPS предоставляет или запрещает подключение, в зависимости от конфигурации этой политики.  

Если проверка подлинности и авторизация выполняются успешно, и если соответствующую политику сети предоставил доступ, сервер политики сети предоставляет доступ к сети, и пользователь и компьютер может подключаться к сетевым ресурсам, для которых у них есть разрешения.  

>[!NOTE]
>Развертывание беспроводного доступа, необходимо настроить политики NPS. Это руководство содержит инструкции для использования **мастера настройки 802.1 X** на сервере политики сети, чтобы создать политики сервера политики сети для беспроводного доступа с проверкой подлинности 802. 1 X.  

### <a name="bootstrap-profiles"></a>Профили начальной загрузки
В 802.1 X\-беспроводных сетях с проверкой беспроводные клиенты должны предоставлять учетные данные безопасности, которые проверяются RADIUS-сервером для подключения к сети. Для защищенного EAP \[PEAP\]\-Майкрософт протокола CHAP версии 2 \[MS\-CHAP v2\], указаны учетные данные безопасности, имя пользователя и пароль. Для EAP\-Transport Layer Security \[TLS\] или PEAP\-TLS, сертификатов, например, сертификаты пользователей и компьютеров клиентов или смарт-карты используются учетные данные безопасности.

При подключении к сети, которая настроена для выполнения PEAP\-MS\-CHAP v2, PEAP\-TLS или EAP\-по умолчанию, Windows, клиентам необходимо также проверить сертификат компьютера, который является проверка подлинности TLS отправленный сервером RADIUS. Сертификат компьютера, отправленный сервером RADIUS, для каждого сеанса проверки подлинности часто называют сертификатом сервера.

Как упоминалось ранее, можно выдать серверы RADIUS их сертификата сервера в одном из двух способов: в коммерческом центре сертификации \(например VeriSign, Inc.,\), или частным центром сертификации, развертываемыми в вашей сети. Если сервер RADIUS отправляет сертификат компьютера, выданный в коммерческом центре сертификации, который уже установлен корневой сертификат в хранилище сертификатов доверенных корневых центров сертификации для клиента, беспроводной клиент может проверить RADIUS сервера сертификат компьютера независимо от того, является ли клиент беспроводной связи присоединением к домену Active Directory. В этом случае беспроводной клиент может подключиться к беспроводной сети, и затем присоединить компьютер к домену.

>[!NOTE]
>Поведение, требуя от клиента для проверки сертификата сервера можно отключить, но отключать проверку сертификата сервера в производственных средах не рекомендуется.

Профили беспроводных сетей начальной загрузки являются временными профилями, которые настроены таким образом, чтобы пользователи клиента беспроводной сети для подключения к 802.1 X\-проверку подлинности беспроводной сети, прежде чем компьютер присоединен к домену, и\/или перед пользователь успешно вошел в домен с помощью беспроводных компьютере в первый раз.  В этом разделе обобщаются какие проблема возникает при попытке присоединить беспроводной сети компьютер к домену, или пользователь может использовать домен\-беспроводных компьютеров, входящих в первый раз войти в домен.

Для развертываний, в которых пользователь или ИТ-администратору не удается подключиться физически компьютера к проводной сети Ethernet для присоединения компьютера к домену, и компьютер не имеет необходимых выдающего корневой сертификат ЦС, в его  **Доверенные корневые центры сертификации** хранилище сертификатов, можно настроить беспроводные клиенты с профиль временного беспроводного подключения, который называется *начальной загрузки профиля*для подключения к беспроводной сети.

Объект *начальной загрузки профиля* исключает необходимость проверить сертификат RADIUS сервера компьютера. Этот временный конфигурация позволяет пользователя беспроводной сети для присоединения компьютера к домену, после чего беспроводной сети \(IEEE 802.11\) применяются политики, и соответствующий корневой ЦС сертификат автоматически устанавливается на компьютер.

Если применяется групповая политика, один или несколько профилей беспроводного соединения, которые обеспечивают соблюдение требования для взаимной проверки подлинности применяются на компьютере; загрузочный профиль больше не требуется, а также удаляется. После присоединения компьютера к домену и перезагрузки компьютера, пользователь может использовать беспроводное подключение для входа в домен.

Обзор процесса развертывания беспроводного доступа с помощью этих технологий, см. в разделе [Обзор развертывания беспроводного доступа](b-wireless-access-deploy-overview.md).
