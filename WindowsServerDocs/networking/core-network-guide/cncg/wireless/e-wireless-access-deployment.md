### <a name="bkmk_channel"></a>Укажите частоту канал беспроводной точкой доступа

При развертывании беспроводные точки доступа на одном сайте географических, необходимо настроить беспроводные точки доступа, которые имеют перекрывающиеся сигналы использовать уникальный канал частоты передачи данных для уменьшения помех между беспроводные точки доступа.

Можно использовать приведенные ниже рекомендации помогут вам в выборе канала частоты, которые не конфликтуют с других беспроводных сетей, в географическом расположении беспроводной сети.

- Если имеются другие организации с офисами в непосредственной близости или в одном здании как организации, определить ли любой беспроводных сетей, принадлежащих организации. Узнать о размещении и назначенного канала частоту их беспроводного доступа в, так как вам понадобится назначить другой канал частоты передачи данных в вашей AP и вам необходимо определить самое лучшее местоположение, чтобы установить ваш AP

- Определите перекрывающихся Беспроводные сигналы на соседних этажах, в вашей организации. После идентификации перекрывающихся зонах покрытия за пределами и в вашей организации назначить частоты канала для беспроводных точках доступа, гарантий, что любые два беспроводных точек доступа с одними назначаются частоты другой канал.

### <a name="bkmk_wirelessaps"></a>Настройка беспроводных точек доступа

Используйте следующие сведения, а также в документации, предоставленной производителем беспроводной точкой доступа для настройки беспроводных точках доступа.

Эта процедура перечисляет элементов, обычно настраиваются на беспроводной точке доступа. Имена элементов могут быть разные торговой марки и модели и может отличаться от тех, в следующем списке. Дополнительные сведения см. в разделе документации по беспроводной точке доступа.

#### <a name="to-configure-your-wireless-aps"></a>Для настройки беспроводных точках доступа  

- **SSID**. Укажите имя беспроводной network\(s\) \ (например, ExampleWLAN\). Это имя, которое объявленных для беспроводных клиентов.

- **Шифрование**. Укажите \(preferred\) WPA2\ предприятия или WPA\-Enterprise и AES \(preferred\) или TKIP шифрования шифров, в зависимости от того, что поддерживаются версии сетевых адаптеров компьютера клиент беспроводной сети.

- **Беспроводной AP IP адрес \(static\)**. Для каждой точки доступа настройте уникальный статический IP-адрес, находящийся в диапазоне исключения для области DHCP для подсети. Использование адреса, которые исключены из назначение DHCP-сервером запрещает DHCP-сервера назначение одного IP-адреса на компьютере или другом устройстве.

- **Маска подсети**. Настройте среду восстановления в соответствии с параметрами маска подсети локальной сети, к которому вы подключены беспроводной точке доступа.  

- **DNS-имя**. Некоторые беспроводные точки доступа можно настроить с помощью DNS-имя. Служба DNS в сети может разрешить DNS-имена в IP-адрес. На каждой точке беспроводного доступа, поддерживающей эту функцию введите уникальное имя для разрешения DNS.  

- **Служба DHCP**. Если ваш беспроводной точке доступа встроенных службу DHCP, отключите его.  

- **Общий секрет RADIUS**. Используйте уникальный общий секрет RADIUS для каждой точки доступа, если вы планируете настроить точках доступа в качестве RADIUS-клиентов на сервере политики сети по группам. Если вы планируете настраивать AP по группам в сервере политики сети, общий секрет должен совпадать для каждого члена группы. Кроме того каждый общий секрет, который используется следует случайных последовательность по крайней мере 22 символов, которые одновременно применяются прописные и строчные буквы, цифры и знаки препинания. Чтобы обеспечить случайность, можно использовать генератор случайных символов, например генератор случайных символов, в NPS **Настройка 802.1 X** мастера, чтобы создавать общие секреты.

>[!TIP]
>Запишите общий секрет для каждой точки доступа и сохранить его в безопасном расположении, например безопасный office. При настройке клиентов RADIUS в сервере политики сети, необходимо знать общий секрет для каждой точки доступа.  

- **IP-адрес сервера RADIUS**. Введите IP-адрес NPS-сервера.

- **UDP port\(s\)**. По умолчанию сервер политики сети использует UDP-порты 1812 и 1645 для проверки подлинности сообщения и UDP-порты 1813 и 1646 для сообщений учета. Рекомендуется использовать эти UDP-порты на точках доступа, но если у вас есть причины для использования разных портов убедитесь не только Настройка точки доступа с использованием новых номеров портов, но также перенастроить всех ваших серверов политики сети, чтобы использовать те же номера портов в качестве точки доступа. Если точки доступа и NPS-серверы не настроены одинаковые порты UDP, сервер политики сети не может получать или обрабатывать запросы на подключение от точки доступа и беспроводные подключения в сети будет невозможно.

- **VSA**. Некоторые беспроводные точки доступа требуют \(VSAs\) vendor\ определенные атрибуты для предоставления функций полной беспроводной точкой доступа. VSA добавляются в политики сети.

- **Фильтрация DHCP**. Настройка беспроводных точек доступа, чтобы заблокировать беспроводные клиенты из отправку IP-пакетов с UDP-порт 68 сети, задокументированного производителем беспроводной точкой доступа.

- **DNS-фильтрация**. Настройка беспроводных точек доступа для блокирования беспроводные клиенты из отправкой IP-пакетов TCP или UDP-порт 53 к сети, задокументированного производителем беспроводной точкой доступа.

### <a name="bkmk_groups"></a>Создайте группу безопасности беспроводных пользователей

Эту процедуру можно использовать для создания группы безопасности беспроводной сети в Active Directory-пользователи и компьютеры Microsoft Management Console \(MMC\) snap\-в.  

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

#### <a name="to-create-a-wireless-users-security-group"></a>Чтобы создать группу безопасности беспроводных пользователей

1. Нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **Active Directory-пользователи и компьютеры**. Active Directory — пользователи и компьютеры оснастка открывает. Если он еще не выбран, щелкните узел домена. Например, если ваш домен называется example.com, щелкните **example.com**.

2. В области сведений щелкните правой кнопкой мыши папку, в которой требуется добавить новую группу \ (например, щелкните правой кнопкой мыши **пользователей**\), наведите курсор на **New**и нажмите кнопку **группы**.

3. В **новый объект — группа**в **имя группы**, введите имя новой группы. Например, введите **беспроводной группы**.

4. В **группы область**, выберите один из следующих вариантов:

    - **Локальная в домене**

    - **Глобальные**

    - **Универсальная**

5. В **тип группы**выберите **безопасности**.

6. Нажмите кнопку **ОК**.

Если вам требуется более одной группы безопасности для беспроводной сети пользователей, повторите эти действия для создания групп пользователей дополнительные беспроводной сети. Позже можно создать отдельные сетевые политики на сервере политики сети для применения различных условий и ограничений для каждой группы, предоставляя им разрешения на доступ к другой и подключения к правилам.

### <a name="bkmk_addusers"></a>Добавление пользователей в группу безопасности беспроводных пользователей

Эту процедуру можно использовать для добавления пользователя, компьютера или группы в группу безопасности беспроводной сети в Active Directory-пользователи и компьютеры Microsoft Management Console \(MMC\) snap\-в.

Членство в группе **"Администраторы домена"**, или эквивалентной группе минимальным требованием для выполнения этой процедуры.

#### <a name="to-add-users-to-the-wireless-security-group"></a>Чтобы добавить пользователей в группу безопасности беспроводной сети

1. Нажмите кнопку **запустить**, нажмите кнопку **Администрирование**, а затем нажмите кнопку **Active Directory-пользователи и компьютеры**. Откроется окно MMC Active Directory-пользователи и компьютеры. Если он еще не выбран, щелкните узел домена. Например, если ваш домен называется example.com, щелкните **example.com**.

2. В области сведений дважды щелкните папку, содержащую вашей группе безопасности беспроводной сети.

3. В области сведений, щелкните правой кнопкой мыши группу безопасности беспроводной сети и нажмите кнопку **свойства**. **Свойства** откроется диалоговое окно для группы безопасности.

4. На **члены** щелкните **добавить**, а затем выполните одно из следующих процедур, чтобы добавить компьютер или добавить пользователя или группу.

##### <a name="to-add-a-user-or-group"></a>Чтобы добавить пользователя или группы

1. В **введите имена выбираемых объектов**введите имя пользователя или группы, которые нужно добавить, а затем нажмите кнопку **ОК**.

2. Чтобы назначить членство в группе другим пользователям или группам, повторите шаг 1 этой процедуры.  

##### <a name="to-add-a-computer"></a>Добавление компьютера

1. Нажмите кнопку **типов объектов**. **Типы объектов** откроется диалоговое окно.

2. В **типов объектов**выберите **компьютеры**, а затем нажмите кнопку **ОК**.

3. В **введите имена выбираемых объектов**, введите имя компьютера, на котором вы хотите добавить и нажмите кнопку **ОК**.

4. Чтобы назначить членство в группе другим компьютерам, повторите шаги 1\ 3 этой процедуры.

### <a name="bkmk_opengpme"></a>Открытие или добавление и открытие объекта групповой политики

По умолчанию компонент управления групповой политики установлен на компьютерах под управлением Windows Server 2016 при установке роли сервера доменных служб Active Directory \(AD DS\) и сервер настроен в качестве контроллера домена. Описывает Открытие \(GPMC\) консоли управления групповыми политиками на контроллере домена следующую процедуру. Затем описано, как либо открыть существующую групповую политику уровня domain\ объект \(GPO\) для редактирования, или создать новый объект групповой Политики домена и откройте его для изменения.

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

#### <a name="to-open-or-add-and-open-a-group-policy-object"></a>Чтобы открыть или добавить и открыть объект групповой политики

1. На контроллере домена нажмите кнопку **запустить**, нажмите кнопку **средства администрирования Windows**, а затем нажмите кнопку **Управление групповой политикой**. Откроется консоль управления групповыми политиками.  

2. В левой области дважды щелкните лес. Например, двойном щелчке **леса: example.com**.  

3. В левой области двойном щелчке **домены**и затем двойном щелчке домена, для которого вы хотите управлять объект групповой политики. Например, двойном щелчке **example.com**.  

4. Выполните одно из следующих действий.

    -   **Чтобы открыть существующий объект групповой Политики уровня domain\ для редактирования**, дважды щелкните домен, содержащий объект групповой политики, которым требуется управлять, щелкните правой кнопкой мыши политику домена, вы хотите управлять, например политики домена по умолчанию, а затем нажмите кнопку **изменить**. **Редактор управления групповыми политиками** открывается.

    -   **Чтобы создать новый объект групповой политики и открыть для редактирования**, щелкните правой кнопкой мыши домен, для которого требуется создать объект групповой политики, а затем нажмите кнопку **создать объект групповой Политики в этом домене и связать его**.

        В **новый объект групповой Политики**в **имя**, введите имя нового объекта групповой политики и нажмите кнопку **ОК**.

        Щелкните правой кнопкой мыши созданный объект групповой политики, а затем нажмите кнопку **изменить**. **Редактор управления групповыми политиками** открывается.

В следующем разделе редактора управления групповыми политиками будет использовать для создания политики беспроводного доступа.

### <a name="bkmk_activate"></a>Активировать беспроводной сети по умолчанию \ (IEEE 802.11\) политики

Здесь описано, как активировать по умолчанию беспроводной сети \ (IEEE 802.11\) с помощью \(GPME\) редактор управления групповыми политиками.

>[!NOTE]
>После активации **Windows Vista и более поздних версий** версии беспроводной сети \ (IEEE 802.11\) политики или **Windows XP** версии, версия автоматически удаляется из список параметров, когда вы щелкните правой кнопкой мыши **беспроводной сети \ (IEEE 802.11\) политики**. Это происходит потому, что политика добавляется в области сведений Редактора после выбора версии политики, при выборе **беспроводной сети \ (IEEE 802.11\) политики** узла. Это состояние сохраняется, пока не будут удалены политики беспроводных сетей, в какое время версия политики беспроводных сетей, возвращает меню правой кнопкой мыши для **беспроводной сети \ (IEEE 802.11\) политики** в Управление групповыми Политиками. Кроме того, политики беспроводной сети отображаются в списке только в области сведений щелкните GPME При **беспроводной сети \ (IEEE 802.11\) политики** выбирается узел.

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

#### <a name="to-activate-default-wireless-network-ieee-80211-policies"></a>Для активации по умолчанию беспроводной сети \ (IEEE 802.11\) политики  

1. Выполните предыдущую процедуру **для открытия или добавления и открыть объект групповой политики** открыть GPME.

2. В Управление групповыми Политиками, в области слева двойном щелчке **Конфигурация компьютера**, двойном щелчке **политики**, двойном щелчке **параметры Windows**, а затем дважды щелкните **параметры безопасности**.

![802.1 X Групповая политика беспроводной](../../../media/Wireless-GP/Wireless-GP.jpg)

3. В **параметры безопасности**, щелкните правой кнопкой мыши **беспроводной сети \ (IEEE 802.11\) политики**, а затем нажмите кнопку **создать новый беспроводной политики для Windows Vista и более поздних версий**. 

![802.1 X политики беспроводного доступа](../../../media/Wireless-Policy/Wireless-Policy.jpg)

##### <a name="to-configure-a-wireless-connection-profile-for-peap-ms-chap-v2"></a>Для настройки профиля беспроводного соединения для PEAP\-MS\-CHAP v2

1. В Управление групповыми Политиками, в диалоговом окне свойств беспроводной сети для политики, который вы только что создали, на **Общие** tab и в **описание**, введите краткое описание для политики.

2. Чтобы использовать службу автонастройки беспроводной сети для настройки параметров адаптера беспроводной сети, убедитесь, что **службу автонастройки Беспроводной сети Windows используется для клиентов** выбран.  

3. В **подключаться к доступным сетям в порядке перечисления профилей**, нажмите кнопку **добавить**, а затем выберите **инфраструктуры**. **Свойства нового профиля** откроется диалоговое окно.

4. В**свойства нового профиля** диалогового **подключения** вкладке **имя профиля** введите новое имя для профиля. Например, введите **Example.com профиль беспроводной сети для Windows 10**.

5. В **\(SSID\) Name\(s\) сети**, введите SSID, соответствующий SSID, заданному на беспроводных точках доступа и нажмите кнопку **добавить**.

    При развертывании используется несколько идентификаторов SSID, и каждый беспроводной точкой доступа используется один набор параметров безопасности беспроводной сети, повторите Добавление SSID для каждой беспроводной точки доступа, к которому должен применяться этот профиль.

    Если в развертывании используется несколько идентификаторов SSID, и параметры безопасности для разных SSID не совпадают, настройте отдельный профиль для каждой группы идентификаторов SSID, которые используют те же параметры безопасности. Например если имеется одна группа беспроводных точек доступа настроены для использования WPA2\ Enterprise и AES и другая группа беспроводных точек доступа WPA\-Enterprise и TKIP, настройте профиль для каждой группы беспроводных точек доступа.

6. Если текст по умолчанию **новый код SSID** представления, выберите его и нажмите кнопку **удалить**.

7. Если у вас развернуты беспроводные точки доступа, широковещательная рассылка маяков, выберите **подключаться, даже если эта сеть не ведет вещание**.

    > [!NOTE]
    > Включение этого параметра может создавать угрозу безопасности, поскольку беспроводные клиенты будут пытаться подключиться к любой беспроводной сети. По умолчанию этот параметр не включен.  

8. Нажмите кнопку **безопасности** щелкните **Дополнительно**и настройте следующие параметры:

    1. Чтобы настроить дополнительные параметры 802. 1 X, в **IEEE 802.1 X**выберите **применить дополнительные параметры 802. 1 X**.

        При расширенного стандарта 802.1 X применяются параметры, значений по умолчанию для **Eapol\ Start Макс сообщений:**, **период задержки**, **период запуска**, и **период проверки** достаточно для типовой беспроводной среды. По этой причине изменения параметров по умолчанию, если нет особых причин для этого не нужно.

    2. Чтобы включить единый вход, выберите **включить единую регистрацию для сети **.

    3. Остальных значений по умолчанию в **единого входа** достаточно для типовой беспроводной среды.

    4. В **быстрого перемещения**, если ваш беспроводной Точки задана предварительно-проверка подлинности, выберите **эта сеть использует проверку подлинности предварительно **.

9. Чтобы указать, что беспроводная связь соответствие стандартам FIPS 140\-2, установите **выполнять шифрование в режиме 140\ 2 FIPS **.

10. Нажмите кнопку **ОК** для возврата **безопасности** вкладку. В **выберите методы безопасности для данной сети**в **проверки подлинности**выберите **WPA2\ предприятия** Если поддерживается вашей беспроводной точкой доступа и адаптерами беспроводной клиентской сети. В противном случае, выберите **WPA\ предприятия **.

11. В **шифрования**, если поддерживается вашей беспроводной точкой доступа и адаптерами беспроводной клиентской сети, выберите **AES-CCMP **. Если вы используете точек доступа и адаптерами беспроводной сети, поддерживающие стандарт 802.11ac, выберите **AES GCMP **. В противном случае, выберите **TKIP **.

    > [!NOTE]  
    > Параметры для обоих **проверки подлинности** и **шифрования** должны соответствовать значениям, заданным в беспроводных точках доступа. Параметры по умолчанию для **режим проверки подлинности**, **Макс. число ошибок проверки подлинности**, и **кэшировать пользовательские данные для следующих подключений к этой сети** достаточно для типовой беспроводной среды.  

12. В **выберите метод проверки подлинности сети**выберите **защищенного EAP \(PEAP\)**, а затем нажмите кнопку **свойства **. **Свойства защищенного EAP** откроется диалоговое окно.

13. В **Свойства защищенного EAP**, убедитесь, что **проверить удостоверение сервера с помощью проверки сертификата** выбран.

14. В **доверенные корневые центры сертификации**выберите доверенный корневой центр сертификации, выдавшего сервера \(CA\) сертификатов NPS-сервер.

    > [!NOTE]  
    > Этот параметр ограничивает число корневых ЦС, которым доверяет клиент, выбранными значениями. Если доверенные корневые ЦС не выбраны, затем клиенты будут доверять всем корневым ЦС, указанным в их хранилище сертификатов доверенных корневых центров сертификации.  

15. В **выберите метод проверки подлинности** выберите **защищенный пароль \ (v2\ EAP\-MS\-CHAP)**.

16. Нажмите кнопку **Настройка**. В **свойства EAP MSCHAPv2** диалогового окна убедитесь, **автоматически использовать имя входа в систему Windows и пароль \ (и домене, если any\)** установлен и нажмите кнопку **ОК **.

17. Чтобы включить быстрое переподключение PEAP, убедитесь, что **включить быстрое переподключение** выбран.

18. Чтобы требовать сервера криптографическое значение привязки TLV попыток подключения, выберите **отключиться, если сервер не представляет криптографическое значение привязки TLV **.

19. Чтобы указать, что удостоверение пользователя скрывается на первом этапе проверки подлинности, выберите **Включить удостоверение конфиденциальности**и в текстовом поле введите имя анонимного удостоверения, или оставьте поле пустым.

    >[! ЗАМЕТКИ]
    >- Необходимо создать политики для беспроводных сетей 802.1 X с помощью сервера политики сети **политика запросов на подключение **. Если политики NPS, созданные с помощью сервера политики сети **политики сети**, а затем удостоверение конфиденциальности не будет работать.
    >- Там, где пустое или анонимного удостоверения EAP удостоверение конфиденциальности предоставляется некоторые методы EAP \ (отличный от фактического identity\) отправляется в ответ на запрос идентификатора EAP. PEAP отправляет удостоверение дважды во время проверки подлинности. На первом этапе удостоверение отправляется в виде обычного текста, и этот идентификатор используется для маршрутизации, не для проверки подлинности клиента. Реальное удостоверение, используемый для проверки подлинности, отправляется на втором этапе проверки подлинности, внутри безопасный туннель, которое устанавливается на первом этапе. Если **Включить удостоверение конфиденциальности** флажок установлен, имя пользователя будет заменен запись, указанную в текстовом поле. Предположим, например, **Включить удостоверение конфиденциальности** выбран и псевдоним удостоверение конфиденциальности **анонимный** указан в текстовом поле. Для пользователя с псевдонимом реальное удостоверение **jdoe@example.com**, будет изменен удостоверение, отправляются на первом этапе проверки подлинности на **anonymous@example.com**. Области часть 1 этап удостоверения не изменяется, как он используется для маршрутизации.  

20. Нажмите кнопку **ОК** закрыть **Свойства защищенного EAP** диалоговое окно.
21. Нажмите кнопку **ОК** закрыть **безопасности** вкладку.
22. Если вы хотите создать дополнительные профили, щелкните **добавить**, а затем повторите предыдущие шаги, выбора различных параметров для настройки каждого профиля для беспроводных клиентов и сети, к которому будет применен профиль. По завершении добавления профилей, нажмите кнопку **ОК** чтобы закрыть диалоговое окно свойств политики беспроводной сети.

#### <a name="bkmk_preferenceorder"></a>Задать порядок предпочтения профилей беспроводного подключения
Эту процедуру можно использовать, если вы создали несколько профилей беспроводной связи в политике беспроводной сети, и вы хотите заказать профили для оптимальной эффективности и безопасности.

Чтобы убедиться, что беспроводные клиенты будут подключаться с высоким уровнем безопасности, которые они могут поддерживаться, поместите наиболее строгие политики в верхней части списка.

Например при наличии двух профилей, один для клиентов, которые поддерживают WPA2 и один для клиентов, которые поддерживают WPA, разместите профиле WPA2 выше в списке. Это гарантирует, что клиенты, которые поддерживают WPA2 будет использовать этот метод для подключения, а не опаснее WPA.

В этой процедуре приводятся действия, чтобы определить порядок, в котором профили беспроводных подключений используются для подключения беспроводных клиентов домена к беспроводным сетям.

Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

##### <a name="to-set-the-preference-order-for-wireless-connection-profiles"></a>Чтобы задать порядок предпочтения профилей беспроводного подключения

1. В Управление групповыми Политиками, в диалоговом окне свойств для беспроводной сети, политики, которая только что настроена щелкните **Общие** вкладку.

2. На **Общие** вкладке **подключаться к доступным сетям в порядке перечисления профилей**, выберите профиль, который вы хотите переместить в списке, а затем нажмите кнопку «вниз» или кнопки либо «стрелка вверх», чтобы переместить профиль в нужное место, в списке.

3.  Повторите шаг 2 для каждого профиля, который требуется переместить в списке.  

4.  Нажмите кнопку **ОК** сохранить все изменения.

Эту процедуру можно использовать для регистрации сервера под управлением \(NPS\) сервера политики сети в доменных службах Active Directory \(AD DS\) в домене, членом которого является сервер политики сети. Регистрация NPS-сервер добавляется в **серверы RAS и IAS** группы безопасности в Доменных службах Active Directory.

>[!NOTE]

    Install-WindowsFeature NPAS -IncludeManagementTools
    
Членство в группе **"Администраторы домена"**, или в эквивалентной минимальным требованием для выполнения этой процедуры.

#### <a name="to-register-an-nps-server-in-its-default-domain"></a>Чтобы зарегистрировать сервер политики сети в домене по умолчанию

1. На NPS-сервере в **диспетчера сервера**, нажмите кнопку **средства**, а затем нажмите кнопку **сервера политики сети**. Откроется оснастка NPS.

## <a name="bkmk_domain"></a>Присоединение новые беспроводные компьютеры к домену
Простой способ для присоединения к домену новые беспроводные компьютеры — физически присоединить компьютер к сегменту проводной локальной сети \ (сегмент, не управляются объектом 802.1 X switch\) до присоединения компьютера к домену. Это самый простой, так как беспроводной применяются параметры групповой политики автоматически и немедленно, и, если вы развернули собственной инфраструктуры PKI, компьютер получает сертификат ЦС и помещает его в хранилище сертификатов доверенных корневых центров сертификации, после чего клиент беспроводной доверять серверов политики сети с сервера сертификаты, выдаваемые ЦС.

Аналогичным образом после новый компьютер присоединен к домену, для пользователям выполнять вход в домен рекомендуется выполнить журнала с помощью проводного подключения к сети.

### <a name="other-domain-join-methods"></a>Другие методы для присоединения к домену.
В тех случаях, когда он не практические присоединение компьютеров к домену, с помощью проводного подключения Ethernet или в случаях, когда пользователь не может войти в домен, в первый раз с помощью проводного подключения, воспользуйтесь альтернативным способом.

- **Конфигурация компьютера специалисты ИТ**. Членом ИТ-специалисты присоединяет компьютер к домену и настраивает единого входа загрузочный профиль беспроводной связи. С помощью этого метода ИТ-администратор беспроводной компьютер подключается к проводной сети Ethernet и присоединяет компьютер к домену. Затем администратор распространяет компьютера пользователю. Когда пользователя запускается, компьютер с помощью проводного подключения, учетные данные домена, указываемые им для входа пользователя используются для обоих установить подключение к беспроводной сети и для входа в домен.

Дополнительные сведения см. раздел [присоединения к домену и вход в систему, используя метод конфигурации компьютера специалисты ИТ](#bkmk_itstaff)

-   **Загрузки для настройки профиля беспроводной связи пользователей**. Пользователь вручную настраивает на компьютере с загрузочного профиля беспроводной сети и присоединен к домену, в соответствии с инструкциями, полученный из ИТ-администратор. Загрузочный профиль беспроводной сети позволяет пользователю установить беспроводное подключение, а затем присоединитесь к домену. После присоединения компьютера к домену и перезагрузки компьютера, пользователь может войти домен с помощью беспроводного подключения и учетные записи домена.

Дополнительные сведения см. раздел [присоединение к домену и вход в систему с использованием настройки профиля беспроводной связи Bootstrap пользователями](#bkmk_userbootstrap).

### <a name="bkmk_itstaff"></a>Присоединение к домену и вход в систему, используя метод конфигурации компьютера специалисты ИТ
Пользователей-членов домена с помощью присоединенных domain\ беспроводных клиентских компьютеров можно использовать временный профиль беспроводной сети для подключения к 802.1X\-проверку подлинности беспроводной сети без первого подключения к проводной локальной сети. Вызывается временного профиля *загрузки профиля беспроводной связи*.

Загрузочного профиля беспроводной сети, пользователь должен будет вручную указать свои учетные данные учетной записи пользователя домена и не проверяет сертификат сервера удаленного пользователя службы проверки подлинности Dial\-In \(RADIUS\) \(NPS\) сервера политики сети.

После установления подключения по беспроводной сети будет применена групповая политика беспроводной сети клиентского компьютера и профиля беспроводной связи, выданного автоматически. Новая политика использует учетные данные учетной записи компьютера и пользователя для проверки подлинности клиента. 

Кроме того, как часть PEAP\-MS\-CHAP v2 взаимной проверки подлинности с помощью нового профиля вместо загрузочный профиль клиента проверяет учетные данные для сервера RADIUS.

После присоединения компьютера к домену, используйте данную процедуру для настройки единого входа загрузочный профиль беспроводной связи, перед отправкой беспроводной компьютера пользователю domain\-member.

#### <a name="to-configure-a-single-sign-on-bootstrap-wireless-profile"></a>Для настройки единого входа загрузочного профиля беспроводной сети

1. Создание загрузочного профиля с помощью процедуры в этом руководстве с именем [настройки профиля беспроводного подключения для PEAP\-MS\-CHAP v2](#bkmk_configureprofile)и используйте следующие параметры:

    - Проверка подлинности PEAP\-MS\-CHAP v2

    - Проверка отключено сертификат сервера RADIUS

    - Единый вход включен

2. В свойствах политики беспроводной сети, в котором вы создали новый загрузочный профиль на **Общие** выберите загрузочный профиль и нажмите кнопку **Экспорт** Экспорт профиля на общий сетевой ресурс, USB-накопитель, или других легкодоступном расположении. Профиль сохраняется как файл *.xml в указанном вами месте.

3. Присоединить компьютер к домену \ (например, с помощью Ethernet-подключение, который не требует IEEE 802. 1 X authentication\) и добавить загрузочный профиль беспроводной сети на компьютере с помощью **netsh wlan добавить профиль** команды.

    >[!NOTE]
    >Дополнительные сведения см. в разделе команды Netsh для беспроводной локальной сети \(WLAN\) в [http:///\/technet.microsoft.com\/library\/dd744890.aspx](https://technet.microsoft.com/library/dd744890).

4. Распространить новый компьютер для пользователя с помощью процедуры «Входить в домен с компьютеров под управлением Windows 10.»

При запуске компьютера, Windows предлагает пользователю вводить имя учетной записи домена пользователя и пароль. Так как единый вход включен, компьютер использует учетные данные учетной записи пользователя домена, чтобы сначала установить подключение к беспроводной сети, а затем войдите в домен.

#### <a name="bkmk_w10"></a>Войдите в домен с компьютеров под управлением Windows 10

1. Выйдите из системы компьютера или перезагрузить компьютер.

2. Нажмите любую клавишу на клавиатуре или щелкните на рабочем столе. С помощью имени учетной записи локального пользователя, который отображается и поле ввода пароля, под именем появления экрана входа. Не войти в систему с учетной записью локального пользователя.

3. В левом нижнем углу экрана щелкните **другой пользователь**. Журнал другого пользователя на экране появится с двумя полями: для имени пользователя и пароля. Ниже пароль поле — это текст **вход в систему:** и затем имя домена, где данный компьютер. Например, если ваш домен называется example.com, следующий текст: **вход в систему: пример**.

4. В **имя пользователя**, введите имя пользователя домена.

5. В **пароль**, введите пароль для домена и затем щелкните стрелку или нажмите клавишу ВВОД.

>[!NOTE]
>Если **другой пользователь** экрана не включает текст **вход в систему:** и имя домена, следует ввести имя пользователя в формате *домен\пользователь *. Например, для входа в домен example.com с именем учетной записи **определяемые 01**, тип **example\\User\-01**.

### <a name="bkmk_userbootstrap"></a>Присоединение к домену и вход в систему с использованием настройки профиля беспроводной связи Bootstrap пользователями
С помощью этого метода выполните действия, описанные в разделе Общие действия, а затем предоставить пользователям domain\ член с инструкциями о том, как вручную настроить беспроводной компьютер с загрузочного профиля беспроводной сети. Загрузочный профиль беспроводной сети позволяет пользователю установить беспроводное подключение, а затем присоединитесь к домену. После компьютер присоединен к домену и перезапуска, пользователь может войти в домен через беспроводное подключение.

#### <a name="general-steps"></a>Общие действия

1. Настройка учетной записи администратора локального компьютера, в **панели управления**, для пользователя.

    >[!IMPORTANT]
    >Присоединение компьютера к домену, пользователь должен войти в компьютер с учетной записи локального администратора. Кроме того пользователь должен предоставить учетные данные для учетной записи локального администратора в процессе присоединения компьютера к домену. Кроме того пользователь должен иметь учетную запись пользователя в домене, к которому пользователь хочет присоединить компьютер. В процессе присоединения компьютера к домену, пользователю будет запрашиваться учетные данные учетной записи домена \ (имя пользователя и password\).

2. Предоставьте пользователям домена инструкции по настройке загрузочного профиля беспроводной сети, как описано в следующей процедуре **для настройки загрузочного профиля беспроводной сети **.
3. Кроме того, предоставить пользователям учетные данные локального компьютера \ (имя пользователя и password\) и учетные данные домена \ (имя учетной записи пользователя домена и password\) в виде *Имя_домена\\имя_пользователя*, и процедуры «Присоединить компьютер к домену» и «Вход в домен,» как указывалось в Windows Server 2016 [руководство по основной сети ](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide).

#### <a name="to-configure-a-bootstrap-wireless-profile"></a>Для настройки загрузочного профиля беспроводной сети

1. Использовать учетные данные, предоставленные администратором сети, или ИТ-поддержки специалистов Войдите на компьютер с учетной записью администратора на локальном компьютере.

2. Щелкните правой кнопкой мыши значок сети на рабочем столе и нажмите кнопку **открыть управления сетями и общим доступом **. **Центр управления сетями** открывается. В **изменение сетевых параметров**, нажмите кнопку **Настройка нового подключения или сети **. **Настройки подключения или сети** откроется диалоговое окно.

3. Нажмите кнопку **вручную подключиться к беспроводной сети**, а затем нажмите кнопку **Далее **.

4. В **вручную подключиться к беспроводной сети**в **сетевое имя**, введите имя SSID точки доступа.  

5. В **тип безопасности**, выберите параметр, предоставленный администратором.

6. В **тип шифрования** и **ключ безопасности**выберите или укажите параметры, предоставляемые администратором.

7. Выберите **запускать это подключение автоматически**, а затем нажмите кнопку **Далее **.

8. В **успешно добавлено *** вашей сети SSID*, нажмите кнопку **изменять параметры подключения**.

9. Нажмите кнопку **изменять параметры подключения **. *Вашей сети SSID* откроется диалоговое окно свойств беспроводной сети.

10. Нажмите кнопку **безопасности** вкладки, а затем в **выбрать метод проверки подлинности сети**выберите **\(PEAP\) защищенного EAP **.

11. Нажмите кнопку **параметры**. **Свойства защищенного EAP \(PEAP\)** откроется страница.

12. В **\(PEAP\) свойства защищенного EAP** убедитесь, что **проверять сертификат сервера** — не выбран, щелкните **ОК** дважды, а затем нажмите кнопку **закрыть **.

13. Windows пытается подключиться к беспроводной сети. Параметры загрузочного профиля укажите, что должен предоставить учетные данные домена. Когда Windows запрашивает имя учетной записи и пароль, введите свои учетные данные учетной записи домена следующим образом: *имя домена Name\\User*, *пароль домена *.

##### <a name="to-join-a-computer-to-the-domain"></a>Чтобы присоединить компьютер к домену

1. Войдите компьютер с учетной записи локального администратора.

2. В поле поиска введите **PowerShell **. В результатах поиска, щелкните правой кнопкой мыши **Windows PowerShell**, а затем нажмите кнопку **Запуск от имени администратора **. Windows PowerShell открывается с строки с повышенными привилегиями.

3. В Windows PowerShell введите следующую команду и нажмите клавишу ВВОД. Убедитесь, заменить DomainName переменной с именем домена, который вы хотите вступить.
    
    Add-Computer Имя_домена
    
4. При появлении запроса введите имя пользователя домена и пароль и нажмите кнопку **ОК **.
5. Перезагрузите компьютер.
6. Следуйте инструкциям в предыдущем разделе [вход в домен с компьютеров под управлением Windows 10](#bkmk_w10).
