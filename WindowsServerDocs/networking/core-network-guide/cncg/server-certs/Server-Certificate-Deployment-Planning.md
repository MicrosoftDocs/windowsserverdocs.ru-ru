## <a name="bkmk_basic"></a>Планирование конфигурации базового сервера  
После установки Windows Server 2016 на компьютерах, которые вы планируете использовать в качестве центра сертификации и веб-сервер, необходимо переименовать компьютер и назначить и настроить статический IP-адрес для локального компьютера.  
  
Дополнительные сведения см. в разделе Windows Server 2016 [руководство по основной сети](../../../core-network-guide/Core-Network-Guide.md).  
  
## <a name="bkmk_domain"></a>Доступ к домену плана  
Чтобы войти в домен, компьютер должен быть членом домена и учетной записи пользователя должны быть созданы в Доменных службах Active Directory, прежде чем попытки входа в систему. Кроме того большинство процедуры из этого руководства требуют учетной записи пользователя является членом группы "Администраторы предприятия" или "Администраторы домена" в Active Directory — пользователи и компьютеры, поэтому необходимо войти в систему в ЦС под учетной записью, членом соответствующей группы.  
  
Дополнительные сведения см. в разделе Windows Server 2016 [руководство по основной сети](../../../core-network-guide/Core-Network-Guide.md).  
  
## <a name="bkmk_virtual"></a>Планирование расположение и имя виртуального каталога веб-сервера  
Чтобы предоставить доступ для списка отзыва Сертификатов и сертификатов ЦС на другие компьютеры, необходимо хранить эти элементы в виртуальный каталог веб-сервера. В этом руководстве виртуальный каталог находится на веб-сервере WEB1. Эта папка находится на диске «C:» и именем «pki.» Виртуальный каталог можно найти на веб-сервере в любом расположении папки, соответствующие вашему развертыванию.  
  
## <a name="bkmk_cname"></a>Планирование запись DNS-псевдонима (CNAME) для веб-сервера  
Записи ресурсов псевдонима (CNAME) иногда называются каноническое имя записи ресурсов. С помощью этих записей можно использовать несколько имен для одного узла, облегчая процесс для выполнения таких операций, как узел, сервера протокола передачи файлов (FTP) и веб-сервера на одном компьютере. Например хорошо известные имена серверов (ftp, www) регистрируются с помощью записи ресурсов псевдонима (CNAME), которые сопоставляются доменных имен (DNS) имя узла, например WEB1 для сервера, на котором размещены эти службы.  
  
Это руководство содержит инструкции по настройке веб-сервера для размещения списка отзыва сертификатов (CRL) для центра сертификации (ЦС). Поскольку можно также использовать веб-сервера для других целей, например для размещения FTP или веб-сайта, рекомендуется создать в DNS записи ресурса псевдонима для веб-сервера. В этом руководстве записи CNAME с именем «pki», но можно выбрать имя, которое подходит для вашего развертывания.  
  
## <a name="bkmk_capolicy"></a>Планирование конфигурации файла CAPolicy.inf  
Перед установкой AD CS, необходимо настроить файл CAPolicy.inf в центре сертификации с информацией для развертывания. Файл CAPolicy.inf содержит следующие сведения:  
  
```  
[Version]  
Signature="$Windows NT$"  
[PolicyStatementExtension]  
Policies=InternalPolicy  
[InternalPolicy]  
OID=1.2.3.4.1455.67.89.5  
Notice="Legal Policy Statement"  
URL=https://pki.corp.contoso.com/pki/cps.txt  
[Certsrv_Server]  
RenewalKeyLength=2048  
RenewalValidityPeriod=Years  
RenewalValidityPeriodUnits=5  
CRLPeriod=weeks  
CRLPeriodUnits=1  
LoadDefaultTemplates=0  
AlternateSignatureAlgorithm=1  
```  
Необходимо спланировать перечисленные ниже для этого файла:  
  
-   **URL-АДРЕС**. В примере файла CAPolicy.inf имеет значение URL-адрес **https://pki.corp.contoso.com/pki/cps.txt**. Это обусловлено веб-сервера в этом руководстве именем WEB1 и содержит запись ресурса CNAME в DNS инфраструктуры PKI. Веб-сервер также присоединить к домену corp.contoso.com. Кроме того есть виртуальный каталог на веб-сервере, с именем «pki», где хранится список отзыва сертификатов. Убедитесь, что значение, задаваемое для URL-адрес в файл CAPolicy.inf очков в виртуальный каталог веб-сервера в домене.  
  
-   **RenewalKeyLength**. Длина ключа обновления по умолчанию для AD CS в Windows Server 2012 — 2048. Длина ключа, который выбран должно быть условии, что можно по-прежнему обеспечивая совместимость с приложениями, которые вы собираетесь использовать.  
  
-   **RenewalValidityPeriodUnits**. В примере файла CAPolicy.inf имеет значение RenewalValidityPeriodUnits 5 лет. Это обусловлено ожидаемый срок службы ЦС — около десяти лет. Значение RenewalValidityPeriodUnits должны отражать общий срок действия ЦС или наибольшее число лет, для которых требуется для регистрации.  
  
-   **CRLPeriodUnits**. В примере файла CAPolicy.inf имеет CRLPeriodUnits значение 1. Это обусловлено примере интервал обновления для списка отзыва сертификатов в этом руководстве — 1 час. Значение интервала, указанных в этот параметр необходимо опубликовать список отзыва Сертификатов в центре сертификации виртуальный каталог веб-сервера, где хранить список отзыва Сертификатов и обеспечить доступ к нему для компьютеров, находящихся в процессе проверки подлинности.  
  
-   **AlternateSignatureAlgorithm**. Этот файл CAPolicy.inf реализует механизм повышения безопасности путем реализации альтернативных форматов подписи. Этот параметр не следует реализовывать, если у вас есть клиенты Windows XP, которым требуются сертификаты из этого ЦС.  
  
Если вы не планируете добавлять все подчиненные ЦС для инфраструктуры открытых ключей в дальнейшем, а если вы хотите добавлять все подчиненные ЦС, можно добавить PathLength ключ в файл CAPolicy.inf со значением 0. Чтобы добавить этот ключ, скопируйте и вставьте следующий код в файл:  
  
```  
[BasicConstraintsExtension]  
PathLength=0  
Critical=Yes  
```  
  
> [!IMPORTANT]  
> Измените любые другие параметры, в файл CAPolicy.inf, если у вас нет особых причин для этого не рекомендуется.  
  
## <a name="bkmk_cdp"></a>Планирование конфигурации расширений CDP и AIA на CA1  
При настройке точки распространения списка отзыва сертификатов (CRL) (CDP) и параметры доступа сведения о центрах сертификации (AIA) на CA1, требуется указать имя веб-сервера и имя вашего домена. Также требуется указать имя виртуального каталога, созданный на вашей веб-сервере, где хранятся список отзыва сертификатов (CRL) и сертификат центра сертификации.  
  
Расположения CDP, который необходимо ввести на этом этапе развертывания имеет следующий формат:  
      
    `http:\/\/*DNSAlias\(CNAME\)RecordName*.*Domain*.com\/*VirtualDirectoryName*\/<CaName><CRLNameSuffix><DeltaCRLAllowed>.crl.`  
      
Например если веб-сервера называется WEB1 и DNS псевдоним CNAME для веб-сервера является «pki», ваш домен называется corp.contoso.com и именем виртуального каталога pki, расположения CDP:  
      
    `http:\/\/pki.corp.contoso.com\/pki\/<CaName><CRLNameSuffix><DeltaCRLAllowed>.crl`  
      
Необходимо ввести расположение AIA имеет следующий формат:  
      
    `http:\/\/*DNSAlias\(CNAME\)RecordName*.*Domain*.com\/*VirtualDirectoryName*\/<ServerDNSName>\_<CaName><CertificateName>.crt.`  
      
Например если веб-сервера называется WEB1 и DNS псевдоним CNAME для веб-сервера является «pki», ваш домен называется corp.contoso.com и именем виртуального каталога pki, AIA расположение:  
      
    `http:\/\/pki.corp.contoso.com\/pki\/<ServerDNSName>\_<CaName><CertificateName>.crt`  
      
## <a name="bkmk_copy"></a>Планирование операции копирования между ЦС и веб-сервера  
Чтобы опубликовать список отзыва Сертификатов и ЦС сертификата из ЦС в виртуальный каталог веб-сервера, запуском команды certutil - crl после настройки расположений CDP и AIA в центре сертификации. Обязательно Настройте правильные пути на свойств ЦС **расширения** вкладке перед выполнением этой команды, с помощью инструкций в этом руководстве. Кроме того Скопируйте сертификат ЦС предприятия на веб-сервер, необходимо уже создал виртуальный каталог на веб-сервере и настроить папку в виде общей папки.  
  
## <a name="bkmk_template"></a>Планирование конфигурации шаблона сертификата сервера в центре сертификации  
Чтобы развернуть сертификаты сервера автоматически подающие, необходимо скопировать шаблон сертификата с именем **RAS и IAS-сервер**. По умолчанию, эта копия называется **копия RAS и IAS-сервер**. Если вы хотите переименовать эту копию шаблона, необходимо запланировать имя, которое будет использоваться при выполнении этого шага развертывания.  
  
> [!NOTE]  
> Последние три развертывания разделы в этом руководстве, — которые позволяют Настройка автоматической регистрации сертификатов сервера, обновление групповой политики на серверах и убедитесь, что серверы получили действительный сертификат сервера из ЦС - не требуют дополнительного планирования действия.
