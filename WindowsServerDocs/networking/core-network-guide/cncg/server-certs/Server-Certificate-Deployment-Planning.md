## <a name="bkmk_basic"></a>Планирование базовой конфигурации сервера  
После установки Windows Server 2016 на компьютерах, которые планируется использовать в качестве центра сертификации и веб-сервер, необходимо переименовать компьютер и назначить и настроить статический IP-адрес для локального компьютера.  
  
Дополнительные сведения см. в разделе Windows Server 2016 [руководство по основной сети](../../../core-network-guide/Core-Network-Guide.md).  
  
## <a name="bkmk_domain"></a>Планирование доступа к домену  
Чтобы войти в домен, то компьютеру необходима компьютера входит в домен и учетная запись пользователя должна создаваться в Доменных службах Active Directory, до попытки входа в систему. Кроме того большинство процедур в данном руководстве требуют, чтобы учетная запись пользователя является членом группы "Администраторы предприятия" или "Администраторы домена" в Active Directory-пользователи и компьютеры, поэтому необходимо войти в систему в ЦС с учетной записью, являющегося членом соответствующей группы.  
  
Дополнительные сведения см. в разделе Windows Server 2016 [руководство по основной сети](../../../core-network-guide/Core-Network-Guide.md).  
  
## <a name="bkmk_virtual"></a>Спланируйте расположение и имя виртуального каталога веб-сервера  
Для предоставления доступа к списку отзыва Сертификатов и сертификат ЦС на другие компьютеры, необходимо сохранить эти элементы в виртуальный каталог на веб-сервере. В этом руководстве виртуальный каталог расположен на веб-сервера WEB1. Эта папка находится на диске «C:» и называется «pki». Виртуальный каталог можно найти на веб-сервере, в любую папку, которая подходит для вашего развертывания.  
  
## <a name="bkmk_cname"></a>Планирование записи псевдонима (CNAME) DNS для веб-сервера  
Записи ресурсов псевдонима (CNAME) иногда называются каноническое имя записи ресурсов. С помощью этих записей можно использовать более одного имени, чтобы они указывали на один узел, что упрощает решение таких задач, в качестве узла сервера протокол передачи файлов (FTP) и веб-сервер на том же компьютере. Например хорошо известные имена серверов (ftp, www) зарегистрированы с помощью записей ресурсов псевдонимов (CNAME), сопоставления доменных имен (DNS) имя узла, например WEB1 для компьютера сервера, на котором размещены эти службы.  
  
Это руководство содержит инструкции по настройке веб-сервера для размещения списка отзыва сертификатов (CRL) для центра сертификации (ЦС). Поскольку можно также использовать веб-сервер для других целей, например для размещения FTP или веб-сайта, рекомендуется создать записи ресурсов псевдонима в DNS для веб-сервера. В этом руководстве запись CNAME с именем «pki», но вы можете выбрать имя, который подходит для вашего развертывания.  
  
## <a name="bkmk_capolicy"></a>Планирование конфигурации файла CAPolicy.inf  
Перед установкой AD CS, необходимо настроить CAPolicy.inf ЦС со сведениями, является правильным для развертывания. Файл CAPolicy.inf содержит следующие сведения:  
  
```  
[Version]  
Signature="$Windows NT$"  
[PolicyStatementExtension]  
Policies=InternalPolicy  
[InternalPolicy]  
OID=1.2.3.4.1455.67.89.5  
Notice="Legal Policy Statement"  
URL=http://pki.corp.contoso.com/pki/cps.txt  
[Certsrv_Server]  
RenewalKeyLength=2048  
RenewalValidityPeriod=Years  
RenewalValidityPeriodUnits=5  
CRLPeriod=weeks  
CRLPeriodUnits=1  
LoadDefaultTemplates=0  
AlternateSignatureAlgorithm=1  
```  
Необходимо спланировать перечисленные ниже элементы, для этого файла:  
  
-   **URL-адрес**. Пример файла CAPolicy.inf имеет значение URL-адрес **http://pki.corp.contoso.com/pki/cps.txt**. Это так, как веб-сервера в этом руководстве имеет имя WEB1 и запись ресурса DNS-запись CNAME инфраструктуры открытых ключей. Веб-сервер также присоединить к домену corp.contoso.com. Кроме того существует виртуальный каталог на веб-сервере, с именем «pki», где хранится список отзыва сертификатов. Убедитесь, что значение, задаваемое для URL-адрес в файл CAPolicy.inf баллы в виртуальный каталог на веб-сервере, в вашем домене.  
  
-   **RenewalKeyLength**. Длина ключа возобновления по умолчанию для служб AD CS в Windows Server 2012 является 2048. Длина ключа, который вы установите должен быть максимально возможное обеспечивая совместимости с приложениями, которые вы планируете использовать.  
  
-   **RenewalValidityPeriodUnits**. Пример файла CAPolicy.inf имеет значение RenewalValidityPeriodUnits в течение 5 лет. Это вызвано ожидаемое время существования ЦС — около 10 лет. Значение RenewalValidityPeriodUnits должно отражать общий срок действия ЦС или к максимальному числу лет, для которых требуется обеспечить регистрацию.  
  
-   **Параметров CRLPeriodUnits**. Пример файла CAPolicy.inf есть параметров CRLPeriodUnits значение 1. Это вызвано примере интервал обновления списка отзыва сертификатов в этом руководстве — 1 неделя. В указанном вами с помощью этого параметра значение интервала необходимо опубликовать список отзыва Сертификатов на ЦС в виртуальный каталог веб-сервера, где можно хранить список отзыва Сертификатов и предоставить доступ к нему для компьютеров, которые находятся в процессе проверки подлинности.  
  
-   **AlternateSignatureAlgorithm**. Это CAPolicy.inf реализует механизм обеспечения повышенной безопасности путем реализации альтернативных форматов подписи. Этот параметр не следует реализовывать, если клиенты Windows XP, для которых требуются сертификаты ЦС.  
  
Если вы не планируете Добавление все подчиненные ЦС в инфраструктуру открытых ключей на более позднее время, и если вы хотите запретить добавление все подчиненные ЦС, можно добавить ключ PathLength в файл CAPolicy.inf со значением 0. Для добавления этого раздела, скопируйте и вставьте следующий код в файл:  
  
```  
[BasicConstraintsExtension]  
PathLength=0  
Critical=Yes  
```  
  
> [!IMPORTANT]  
> Изменить другие параметры в файле CAPolicy.inf, если нет особых причин для этого не рекомендуется.  
  
## <a name="bkmk_cdp"></a>Планирование конфигурации расширений CDP и AIA в CA1  
При настройке точки распространения списка отзыва сертификатов (CRL) (CDP) и параметры доступа сведения о центрах сертификации (AIA) в CA1 требуется указать имя веб-сервера и имя домена. Также требуется указать имя виртуального каталога, созданный на вашей веб-сервере, где хранятся списка отзыва сертификатов (CRL) и центра сертификации.  
  
Расположения CDP, который необходимо ввести на этом этапе развертывания имеет формат:  
      
    `http:\/\/*DNSAlias\(CNAME\)RecordName*.*Domain*.com\/*VirtualDirectoryName*\/<CaName><CRLNameSuffix><DeltaCRLAllowed>.crl.`  
      
Например если веб-сервер имеет имя WEB1 и ваша служба DNS псевдоним CNAME для веб-сервера «pki», ваш домен называется corp.contoso.com и именем виртуального каталога инфраструктуры открытых ключей, расположения CDP:  
      
    `http:\/\/pki.corp.contoso.com\/pki\/<CaName><CRLNameSuffix><DeltaCRLAllowed>.crl`  
      
Расположение AIA, необходимо ввести имеет формат:  
      
    `http:\/\/*DNSAlias\(CNAME\)RecordName*.*Domain*.com\/*VirtualDirectoryName*\/<ServerDNSName>\_<CaName><CertificateName>.crt.`  
      
Например если веб-сервер имеет имя WEB1 и ваша служба DNS псевдоним CNAME для веб-сервера «pki», ваш домен называется corp.contoso.com и именем виртуального каталога инфраструктуры открытых ключей, AIA расположение:  
      
    `http:\/\/pki.corp.contoso.com\/pki\/<ServerDNSName>\_<CaName><CertificateName>.crt`  
      
## <a name="bkmk_copy"></a>Планирование операции копирования между ЦС и веб-сервера  
Для публикации списка отзыва Сертификатов и ЦС сертификата от ЦС для виртуальный каталог веб-сервера, можно выполнить команды certutil - crl после расположений CDP и AIA настройки ЦС. Убедитесь, что на свойства ЦС настроен правильный пути **расширения** вкладке перед выполнением этой команды, следуя инструкциям данного руководства. Кроме того на веб-сервер, скопируйте сертификат ЦС предприятия, вы должны уже создан виртуальный каталог на веб-сервере и настроен папку в качестве общей папки.  
  
## <a name="bkmk_template"></a>Планирование конфигурации сервера шаблона сертификата в ЦС  
Чтобы развернуть сертификаты сервера пользователь, необходимо скопировать шаблон сертификата, указанный **RAS и IAS-сервер**. По умолчанию, такая копия называется **копия RAS и IAS-сервер**. Если вы хотите переименовать эту копию шаблона, необходимо запланировать имя, которое вы хотите использовать на этом этапе развертывания.
