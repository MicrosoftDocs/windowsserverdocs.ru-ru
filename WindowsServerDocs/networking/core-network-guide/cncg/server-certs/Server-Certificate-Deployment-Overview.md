## <a name="bkmk_components"></a>Компоненты развертывания сертификатов сервера
В этом руководстве можно использовать для установки служб сертификатов Active Directory (AD CS) в качестве корневого центра сертификации (ЦС) предприятия и для регистрации сертификатов сервера для серверов, на которых выполняется служба сервера политики сети (NPS), службы маршрутизации и удаленного доступа (RRAS), или как сервер политики сети и RRAS.


Если вы развертываете SDN с проверкой подлинности на основе сертификатов, серверы должны использовать сертификат сервера для подтверждения удостоверения для других серверов, чтобы они обеспечить безопасную связь.

### <a name="bkmk_ca1"></a>CA1 выполняется роль сервера AD CS  
В этом сценарии предприятия корневых центров сертификации (ЦС) также является выдающий ЦС. ЦС выдает сертификаты для компьютеров сервера разрешения безопасности, чтобы зарегистрировать сертификат. Службы сертификации Active Directory (AD CS) устанавливается в CA1.  
  
Для более крупных сетей или если вопросы безопасности обеспечивают обоснования можно разделить роли корневого ЦС и выдающих ЦС, а затем развернуть подчиненные ЦС выдающий ЦС.  
  
В наиболее безопасного развертывания в сети и физически защищенным берется корпоративного корневого ЦС.   
  
#### <a name="capolicyinf"></a>Файла CAPolicy.inf  
Перед установкой AD CS, Настройка файла CAPolicy.inf с определенными параметрами, для развертывания.  
  
#### <a name="copy-of-the-ras-and-ias-servers-certificate-template"></a>Копирование **серверы RAS и IAS** шаблона сертификата  
При развертывании сертификатов сервера, можно сделать одну копию **серверы RAS и IAS** шаблон сертификата, а затем настройте шаблон в соответствии с требованиями и инструкциям этого руководства.   
  
Использовать копию шаблона, а не исходный шаблон, так что конфигурации исходного шаблона сохраняется для возможности использования в будущем. Настройка копию **серверы RAS и IAS** шаблона, чтобы ЦС можно создавать сертификаты сервера, выдаваемых им группы в Active Directory-пользователи и компьютеры, указанных вами.  
  
#### <a name="additional-ca1-configuration"></a>Дополнительная настройка CA1  
ЦС публикация списка отзыва сертификатов (CRL), компьютеры необходимо проверить, убедитесь, что сертификаты, которые предоставляются для их как доказательство подлинности действительных сертификатов и не был отозван. Необходимо настроить ЦС с правильное расположение списков отзыва сертификатов, чтобы компьютеры знать, где найти список отзыва Сертификатов во время процесса проверки подлинности.  
  
### <a name="bkmk_web1"></a>WEB1 ролью сервера веб-служб (IIS)  
На компьютере, на котором выполняется роль сервера веб-сервер (IIS), WEB1, необходимо создать папку в проводнике Windows для использования в качестве расположения для списка отзыва Сертификатов и AIA.  
  
#### <a name="virtual-directory-for-the-crl-and-aia"></a>Виртуальный каталог для AIA и списков отзыва Сертификатов  
После создания папки в проводнике Windows, необходимо настроить папку как виртуальный каталог в Диспетчер Internet Information Services (IIS), а также настройка управления доступом для виртуального каталога разрешить компьютерам доступ к AIA и списков отзыва Сертификатов после их публикации существует.  
  
### <a name="bkmk_dc1"></a>DC1 под управлением роли сервера AD DS и DNS  
DC1 — контроллер домена и DNS-сервер в сети.  
  
#### <a name="group-policy-default-domain-policy"></a>Групповая политика домена по умолчанию для политики  
После настройки шаблона сертификата в ЦС, можно настроить политику домена по умолчанию в групповой политике, чтобы сертификаты — это пользователь, для серверов политики сети и удаленного доступа. Групповая политика настроена в AD DS на сервере DC1.  
  
#### <a name="dns-alias-cname-resource-record"></a>DNS запись ресурса псевдонима (CNAME)  
Необходимо создать запись ресурса псевдонима (CNAME) для веб-сервера, чтобы убедиться, что другие компьютеры могут найти сервер, а также AIA и списков отзыва Сертификатов, которые хранятся на сервере. Кроме того с помощью записи ресурса CNAME псевдонимов обеспечивает гибкость, чтобы веб-сервера можно использовать для других целей, например FTP и веб-узлов.  
  
### <a name="bkmk_nps1"></a>NPS1 работает служба роли сервера политики сети роли сервера служб политики сети и доступа  
На NPS-сервер устанавливается при выполнении задач в Windows Server 2016 руководства по основной сети, поэтому перед выполнением задачи в данном руководстве, необходимо иметь один или несколько NPS-серверов, установленных в сети.  
  
#### <a name="group-policy-applied-and-certificate-enrolled-to-servers"></a>Групповая политика применяется и сертификатов, выдаваемых серверам  
После настройки шаблона сертификата сервера и автоматической регистрации можно обновить групповую политику на всех целевых серверах. В настоящее время серверы зарегистрировать сертификат сервера из CA1.  
  
### <a name="bkmk_process"></a>Обзор процесса развертывания сертификата сервера  
  
> [!NOTE]  
> В разделе представлены сведения о том, как выполнить эти действия [развертывание сертификатов сервера](../../../core-network-guide/cncg/server-certs/Server-Certificate-Deployment.md).  
  
Процесс настройки регистрации сертификатов сервера выполняется в следующих этапов:  
  
1.  На WEB1 установите роль веб-сервер (IIS).  
  
2.  На DC1 Создание записи псевдонима (CNAME) веб-сервера WEB1.  
  
3.  Настройка веб-сервера для размещения списка отзыва Сертификатов из ЦС, а затем опубликовать список отзыва Сертификатов и скопируйте сертификат корневого ЦС в новый виртуальный каталог.  
  
4.  На компьютере, где вы планируете установки служб AD CS назначить статический IP-адрес компьютера, переименования компьютера, присоединить компьютер к домену и вход компьютер с учетной записью пользователя, которая является членом группы "Администраторы домена" и "Администраторы предприятия".  
  
5.  На компьютере, где вы планируете установки служб AD CS необходимо настройте файл CAPolicy.inf с параметрами, которые относятся к развертыванию.  
  
6.  Установите роль сервера AD CS и выполнить дополнительную настройку центра сертификации.  
  
7.  Скопируйте сертификат списка отзыва Сертификатов и ЦС из CA1 к общему ресурсу на веб-сервера WEB1.  
  
8.  В ЦС настройте копию шаблона сертификата серверы RAS и IAS. ЦС выдает сертификаты на основе шаблона сертификата, поэтому необходимо настроить шаблон для сертификата сервера, прежде чем ЦС для выдачи сертификата.  
  
9.  Настройка автоматической регистрации сертификатов сервера в групповой политике. При настройке автоматической подачи заявок все серверы, которые вы указали в группах Active Directory автоматически получать сертификат сервера при обновлении групповой политики на каждом сервере. Если позднее добавить дополнительные серверы, они будут автоматически получать сертификат сервера слишком.  
  
10. Обновление групповой политики на серверах. При обновлении групповой политики серверов получить сертификат сервера, который основан на шаблоне, настроенную в предыдущем шаге. Этот сертификат используется сервером для подтверждения его подлинности для клиентских компьютеров и других серверов в процессе проверки подлинности.  
  
    > [!NOTE]  
    > Все члены домена автоматически получают сертификат корневого ЦС без настройки автоматической подачи заявок. Этот сертификат отличается от сертификата сервера, настраивать и распространять с использованием автоматической подачи заявок. Сертификат ЦС автоматически устанавливается в хранилище сертификатов доверенных корневых центров сертификации для всех компьютеров домена, чтобы они будут доверять сертификатам, выдаваемым этим ЦС.   
  
10. Убедитесь, что все серверы были зарегистрированы действительный сертификат сервера.
