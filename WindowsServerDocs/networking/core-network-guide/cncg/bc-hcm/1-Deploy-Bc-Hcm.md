# <a name="deploy-branchcache-hosted-cache-mode"></a>Развертывание режима размещенного кэша BranchCache

>Область применения: Windows Server (канал точками годовой), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Руководство по сети основных компонентов Windows Server 2016 содержит инструкции по планированию и развертыванию основных компонентов, необходимых для создания полнофункциональной сети и нового Active Directory&reg; домена в новом лесу.

В этом руководстве рассматривается построение основной сети приводятся инструкции по развертыванию BranchCache в режиме размещенного кэша в офисах филиалов с контроллером домена Read\-Only, где клиентские компьютеры под управлением Windows&reg; 10, Windows 8.1 или Windows 8, которые присоединены к домену.

>[!IMPORTANT]
>Не используйте это руководство, если вы планируете развертывание или уже развернули сервер размещенного кэша BranchCache под управлением Windows Server 2008 R2. Это руководство содержит инструкции по развертыванию режима размещенного кэша с помощью сервера размещенного кэша под управлением Windows Server&reg; 2016, Windows Server 2012 R2 или Windows Server 2012.

## <a name="bkmk_pre"></a>Необходимые условия для использования в этом руководстве

Это дополнительное руководство по для Windows Server 2016 руководства по основной сети. Чтобы развернуть BranchCache в режиме размещенного кэша с помощью этого руководства, вы должны сначала выполните следующие действия.

- Разверните основную сеть в главном офисе с помощью руководства по основной сети или уже имеют технологии, представленные в руководстве по основной сети установлены и функционируют в вашей сети должным образом. Эти технологии включают \(AD DS\) доменных служб Active Directory протоколом TCP/IP v4, DHCP и DNS.

    > [!NOTE]
    > Windows Server 2016 [руководство по основной сети](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide) доступно в технической библиотеке Windows Server 2016.  

- Развертывание серверов содержимого BranchCache под управлением Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 в главном офисе или в облачном центре обработки данных. Сведения о развертывании серверов содержимого BranchCache см. в разделе [дополнительные ресурсы](11-Bc-Hcm-additional-resources.md).

- Установите соединения \(WAN\) глобальной сети между филиалом, главным офисом и при необходимости облачными ресурсами, с помощью виртуальной частной сети \(VPN\), DirectAccess или другой метод подключения.

- Разверните в филиале клиентские компьютеры, работающих под управлением одной из следующих операционных систем, предоставляющих BranchCache с поддержкой фоновой интеллектуальной службы передачи (BITS), Hyper протокола передачи текста (HTTP) и Server Message Block (SMB).
    - Windows 10 Корпоративная
    - Windows 10 для образовательных учреждений
    - Windows 8.1 Корпоративная
    - Windows 8 Корпоративная

>[!NOTE]
>В следующих операционных систем BranchCache не поддерживает функции HTTP и SMB, но поддерживает функциональность BranchCache BITS.
>     - Windows 10 Pro, BITS поддерживает только
>     - Windows 8.1 Профессиональная, BITS поддерживает только
>     - Windows 8 Профессиональная, BITS поддерживает только

## <a name="bkmk_about"></a>Об этом руководстве

Данное руководство предназначено для администраторов сети и системы, которые следовали инструкциям, приведенным в руководство по сети Windows Server 2016 Core или руководство по сети основных компонентов Windows Server 2012, чтобы развернуть основную сеть, или для тех, кто уже развернута технологий, включенных в руководстве по основной сети, включая \(AD DS\) доменных служб Active Directory, службы доменных имен \(DNS\), \(DHCP\) Dynamic Host Configuration Protocol и протокол TCP/IP v4.

Рекомендуется ознакомиться с руководствами по проектированию и развертыванию для каждой из технологий, используемых в данном сценарии развертывания. Эти руководства помогут вам определить, предоставляет ли данный сценарий развертывания службы и конфигурации, необходимые для сети вашей организации.

## <a name="bkmk_not"></a>Что это руководство не содержит

Это руководство не содержит общие сведения о службе BranchCache, включая сведения о режимах и возможностях BranchCache.  

В этом руководстве не содержатся сведения о развертывании подключений к глобальной сети и других технологиях в вашем филиале, таких как DHCP, RODC или VPN-сервера.

Кроме того в этом руководстве даются рекомендации по оборудованию, которое следует использовать при развертывании сервера размещенного кэша. Это возможно запускать другие службы и приложения на сервере размещенного кэша, однако его следует исходя из, рабочей нагрузки, возможностей оборудования и размера филиала, следует ли устанавливать сервер размещенного кэша BranchCache на конкретном компьютере и объем дискового пространства, выделенного для кэша.  
Это руководство не содержит инструкции по настройке компьютеров, работающих под управлением Windows 7. При наличии клиентских компьютеров, работающих под управлением Windows 7 в филиалах, их необходимо настроить с помощью процедур, отличающихся от процедур в данном руководстве для клиентских компьютеров, работающих под управлением Windows 10, Windows 8.1 и Windows 8.
  
Кроме того при наличии компьютеров под управлением Windows 7, необходимо настроить сервер размещенного кэша с сертификатом сервера, выданный центром сертификации, которому доверяют клиентские компьютеры. \ (Если все клиентские компьютеры работают под управлением Windows 10, Windows 8.1 или Windows 8, необходимо настроить сервер размещенного кэша с сертификатом сервера. \) 
> [!IMPORTANT]
> Если серверы размещенного кэша работают под управлением Windows Server 2008 R2, используйте Windows Server 2008 R2 [руководство по развертыванию BranchCache](https://technet.microsoft.com/library/ee649232(v=ws.10).aspx) вместо в этом руководстве по развертыванию BranchCache в режиме размещенного кэша. Примените параметры групповой политики, описанные в этом руководстве, чтобы все клиенты BranchCache, работающие под управлением версий Windows из Windows 7 до Windows 10. Компьютеры под управлением Windows Server 2008 R2 нельзя настроить с помощью действия в данном руководстве.

## <a name="bkmk_tech"></a>Обзор технологий

В этом дополнительном руководстве единственную технологию, которую необходимо установить и настроить — BranchCache. Вы должны будете выполнять команды Windows PowerShell BranchCache на серверах содержимого, таких как веб-сервер и файловых серверов, однако вы не должны изменять или перенастраивать серверы содержимого каким-либо другим образом. Кроме того необходимо настроить клиентские компьютеры с помощью групповой политики на контроллерах домена, работающих под управлением AD DS на Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.

### <a name="branchcache"></a>BranchCache

BranchCache — это технология оптимизации глобальной сети (WAN) пропускной способности, включенная в некоторые выпуски операционных систем Windows Server 2016 и Windows 10, а также в некоторые выпуски Windows Server 2012 R2, Windows 8.1, Windows Server 2012, Windows 8, Windows Server 2008 R2 и Windows 7.

Для оптимизации пропускной способности глобальной сети, когда пользователи обращаются к содержимому на удаленных серверах, BranchCache загружает содержимое запроса клиентом в главном офисе или облаке серверов содержимого и кэширует это содержимое в офисах филиалов, позволяя других клиентских компьютерах в филиалах, для доступа к то же содержимое локально, а не через глобальную сеть.

При развертывании BranchCache в режиме размещенного кэша необходимо настроить клиентские компьютеры в филиале как клиенты в режиме размещенного кэша, и затем развернуть в филиале сервер размещенного кэша. В этом руководстве показано, как развернуть сервер размещенного кэша с предварительно хэшированным и загруженным содержимым с веб-сервера и файлового содержимого серверам на основе на сервере.

### <a name="group-policy"></a>Групповая политика

Групповая политика в Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012 — это инфраструктура, используемая для доставки и применение требуемой конфигурации или параметры политики для нескольких целевых пользователей и компьютеров в среде Active Directory. 

Эта инфраструктура состоит из обработчика групповых политик и нескольких \(CSEs\) отведенный период расширениям, отвечающих за чтение параметров политики на целевых клиентских компьютерах.

Групповая политика используется в этом сценарии для настройки домен клиентских компьютеров в режиме размещенного кэша BranchCache.

Чтобы продолжить работу в этом руководстве, см. [режим размещенного кэша BranchCache развертывания Обзор](2-Bc-Hcm-Deploy-Overview.md).
