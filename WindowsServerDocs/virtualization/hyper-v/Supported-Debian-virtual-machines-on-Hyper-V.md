## <a name="BKMK_notes"></a>Примечания

1. Создание файловых систем на виртуальных жестких дисков размером более 2 ТБ не поддерживается.

2. В Windows Server 2008 R2 SCSI дисков создайте 8 различные записи в/dev/sd *.

3. Windows Server 2012 R2 виртуальной Машины с 8 ядер, или несколькими будет иметь все прерывания, направляются в один виртуальный ЦП.

4. Начиная с Debian 8.3 вручную установить пакет Debian «Hyper-v-управляющими программами» содержит пары "ключ значение", fcopy и управляющие программы VSS. Debian 7.x и 8.0 8.2 пакет управляющие программы Hyper-v должны поступать из [Debian бэкпортированным](https://wiki.debian.org/Backports).

5. Резервное копирование виртуальной машины не будет работать с файловую систему ext2 файловыми системами. Макет по умолчанию, созданный установщиком Debian включает файловую систему ext2 файловые системы, необходимо настроить макет, чтобы не создавать этот тип файловой системы.

6. Debian 7.x выходит за пределы поддержки и использует старые ядра, ядро включены в [Debian бэкпортированным](https://wiki.debian.org/Backports) для Debian 7.x повысила возможности Hyper-V.

7. В Windows Server 2012 R2 поколения 2 безопасная загрузка включена по умолчанию у виртуальных машин и некоторых виртуальных машин Linux не загрузится, если отключен параметр безопасной загрузки. Вы можете отключить безопасную загрузку в **встроенного по** разделе параметров для виртуальной машины в **диспетчера Hyper-V** или отключить с помощью Windows Powershell:

   ```Powershell
   Set-VMFirmware -VMName "VMname" -EnableSecureBoot Off

   ```
8. Новейшие возможности Основное ядро доступны только при использовании ядра включены [Debian бэкпортированным](https://wiki.debian.org/Backports).
