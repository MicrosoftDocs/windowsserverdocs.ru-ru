# <a name="a-replica-server-must-be-configured-to-accept-replication-requests"></a>Сервер реплики должен быть настроен для приема запросов репликации.

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и сканировании см. в разделе [Запуск сканирования анализатором соответствия рекомендациям и управление результатами сканирования](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|Свойство|Сведения|
|-|-|
|**Операционная система**|Windows Server 2016|
|**Продукт или компонент**|Hyper-V|
|**Уровень серьезности**|Ошибка|
|**Категория**|Конфигурация|

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.

## <a name="issue"></a>Проблема
*Этот компьютер обозначен как сервер реплики Hyper-V, но не настроен на прием входящих данных репликации с основных серверов.*

## <a name="impact"></a>Влияние
*Этот сервер не может принимать трафик репликации с основных серверов.*

## <a name="resolution"></a>Решение
*Используйте диспетчер Hyper-V, чтобы указать первичные серверы, от которых этот сервер реплики должен принимать данные репликации.*

#### <a name="create-authorization-entries-using-hyper-v-manager"></a>Создание записей авторизации с помощью диспетчера Hyper-V

1.  Откройте диспетчер Hyper-V. (В диспетчер сервера щелкните **инструменты**  >  . **Диспетчер Hyper-V**.)

2.  В списке узлов щелкните правой кнопкой мыши нужный, а затем выберите пункт **Параметры Hyper-V**.

3.  В области навигации щелкните **Конфигурация репликации**.

4.  В разделе **авторизация и хранилище**щелкните **Разрешить репликацию с указанных серверов**.

5.  Под списком серверов нажмите кнопку **Добавить**.

6.  В разделе **Добавление записи авторизации**:

    -   Введите полное имя первого сервера.

    -   Укажите выделенное расположение для хранения только файлов этого сервера.

7.  Нажмите кнопку **ОК**.

8.  Повторите эти действия для каждого основного сервера.

9. Нажмите кнопку **ОК** еще раз, чтобы завершить работу и закрыть окно.

### <a name="create-authorization-entries-using-windows-powershell"></a>Создание записей авторизации с помощью Windows PowerShell

1.  Откройте Windows PowerShell. (На рабочем столе нажмите кнопку Пуск и начните вводить **Windows PowerShell**.)

2.  Щелкните правой кнопкой мыши **Windows PowerShell** и выберите команду **Запуск от имени администратора**.

3.  Выполните команду, аналогичную приведенной ниже, заменив:

    -   Имя сервера источника server01.domain01.contoso.com с полным доменным именем сервера.

    -   Расположение Д:\репликавмстораже с вашим расположением.

    -   Группа доверия с именем по УМОЛЧАНИю с именем группы, если она создана. В противном случае используйте значение по умолчанию.

```
New-VMReplicationAuthorizationEntry server01.domain01.contoso.com D:\ReplicaVMStorage DEFAULT
```
