# <a name="resynchronization-of-replication-should-be-scheduled-for-off-peak-hours"></a>Повторная синхронизация репликации должна быть запланирована на часы наименьшей нагрузки

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>Проблемы  
*Повторная синхронизация репликации для основных виртуальных машин не запланирована в часы наименьшей нагрузки.*  
  
## <a name="impact"></a>Влияние  
*The дольше виртуальная машина находится в состоянии, которое требует повторной синхронизации, тем дольше файлы журнала репликации увеличиваются, а на основных виртуальных машинах происходят более нереплицированные изменения. Это влияет на следующие виртуальные машины:* .  
  
@no__t 0list виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*С помощью диспетчера Hyper-V измените параметры репликации для виртуальной машины, чтобы она автоматически выполняла синхронизацию в часы наименьшей нагрузки.*
