# <a name="resynchronization-of-replication-should-be-scheduled-for-off-peak-hours"></a>Повторная синхронизация репликации следует планировать на часы наименьшей нагрузки

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>Проблемы  
*Повторная синхронизация репликации для основных виртуальных машинах не запланировано на часы наименьшей нагрузки.*  
  
## <a name="impact"></a>Влияние  
*Больше времени, виртуальная машина находится в состоянии требуется повторная синхронизация, тем больше репликации файлы журнала разрастаются, и происходят более нереплицированные изменения на основных виртуальных машинах. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Используйте диспетчер Hyper-V, чтобы изменить параметры репликации для виртуальной машины для выполнения повторной синхронизации автоматически в часы наименьшей нагрузки.*
