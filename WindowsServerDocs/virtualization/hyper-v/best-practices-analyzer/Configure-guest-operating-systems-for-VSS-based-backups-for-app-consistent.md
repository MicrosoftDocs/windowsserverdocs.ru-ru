# <a name="configure-guest-operating-systems-for-vss-based-backups-to-enable-application-consistent-snapshots-for-hyper-v-replica"></a>Настройка гостевых операционных систем для резервного копирования на основе VSS для включения моментальных снимков с согласованием приложений для реплики Hyper-V

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>Проблемы  
*Снимки приложений требует, что службы теневого копирования томов (VSS) включена и настроена в гостевых операционных систем виртуальных машин, участвующих в репликации.*  
  
## <a name="impact"></a>Влияние  
*Даже если моментальных снимков с согласованием приложений указаны в конфигурации репликации, Hyper-V не будет использовать их только если VSS настроена. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Используйте подключение к виртуальной машине для установки служб integration services на виртуальной машине.*
