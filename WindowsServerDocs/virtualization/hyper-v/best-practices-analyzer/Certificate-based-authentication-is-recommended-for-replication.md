# <a name="certificate-based-authentication-is-recommended-for-replication"></a>Рекомендуется использовать проверку подлинности на основе сертификатов для репликации

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).  
  
|Свойство|Подробности|  
|-|-|  
|**Операционная система**|Windows Server 2016|  
|**Продукта или компонента**|Hyper-V|  
|**Уровень серьезности**|Предупреждение|  
|**Категория**|Конфигурация|  
  
В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>**Проблема**  
*Один или несколько виртуальных машин, выбранных для репликации настраиваются для проверки подлинности Kerberos.*  
  
## <a name="impact"></a>**Влияние**  
*Сетевой трафик репликации с сервера-источника на сервер репликации не шифруется. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>**Решение**  
*Если используется другой метод для выполнения шифрования, его можно игнорировать. В противном случае измените параметры виртуальной машины, чтобы выбрать проверку подлинности на основе сертификата.*
