# <a name="recovery-snapshots-should-be-removed-after-failover"></a>Следует удалить моментальные снимки, восстановление после отработки отказа

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>**Проблема**  
*Объект, отработки отказа виртуальной машины имеет один или несколько моментальных снимков восстановления.*  
  
## <a name="impact"></a>**Влияние**  
*Доступное пространство может израсходовано на физическом диске, в котором хранятся файлы моментальных снимков. В этом случае могут выполняться никакие операции дополнительный диск в физическом хранилище. Может повлиять на любой виртуальной машины, которая зависит от физического хранилища. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>**Решение**  
*Для каждой отработке отказа виртуальной машины используйте командлет Complete-VMFailover в Windows PowerShell отобразить выполнение отработки отказа и восстановления моментальных снимков.*
