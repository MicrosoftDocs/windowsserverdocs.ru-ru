# <a name="authorization-entries-should-have-distinct-trust-group-names-for-primary-servers-with-virtual-machines-that-are-not-part-of-the-same-trust-group"></a>Записи авторизации должны иметь разные имена групп доверия для основных серверов с виртуальными машинами, которые не входят в одну и ту же группу доверия.

>Область применения: Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|**Продукт или функция**|Hyper-V|  
|**Серьезности**|Предупреждение|  
|**Категория**|Конфигурация|  
  
В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>**Проблема**  
*Сервер будет принимать запросы на репликацию для виртуальной машины реплики с любого сервера в списке авторизации, связанного с тем же тегом репликации, что и виртуальная машина.*  
  
## <a name="impact"></a>**Благоприятн**  
*Возможно, существуют проблемы конфиденциальности и безопасности виртуальной машины, принимающей репликацию с основных серверов, принадлежащих к разным записям авторизации. Это влияет на следующие записи авторизации: \<список записей авторизации >*  
  
## <a name="resolution"></a>**Решение**  
*Используйте разные теги в записях авторизации для серверов-источника с виртуальными машинами, которые не входят в одну группу безопасности. Измените параметры Hyper-V, чтобы настроить теги репликации.*
