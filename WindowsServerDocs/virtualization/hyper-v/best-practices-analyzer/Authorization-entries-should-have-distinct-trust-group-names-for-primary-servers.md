# <a name="authorization-entries-should-have-distinct-trust-group-names-for-primary-servers-with-virtual-machines-that-are-not-part-of-the-same-trust-group"></a>Записи авторизации должны иметь имена группы distinct доверия для серверов-источников с виртуальными машинами, которые не являются частью одной и той же группе доверия

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|**Продукта или компонента**|Hyper-V|  
|**Уровень серьезности**|Предупреждение|  
|**Категория**|Конфигурация|  
  
В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>**Проблема**  
*Сервер будет принимать запросы на репликацию для виртуальной машины реплики из любого из серверов в список авторизации, связанный с тем же тегом репликации, что и виртуальная машина.*  
  
## <a name="impact"></a>**Влияние**  
*Возможно, конфиденциальности и обеспечения безопасности с виртуальной машиной принимать репликацию с основного сервера, принадлежащие к записи разных авторизации. Это влияет на следующие записи авторизации: \<список записи авторизации >*  
  
## <a name="resolution"></a>**Решение**  
*Используйте разные теги в записи авторизации серверов-источников с виртуальными машинами, которые не являются частью одной и той же группе безопасности. Измените параметры Hyper-V для настройки репликации теги.*
