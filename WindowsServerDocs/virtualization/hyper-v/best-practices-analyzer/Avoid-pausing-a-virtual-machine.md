# <a name="avoid-pausing-a-virtual-machine"></a>Избегайте Приостановка виртуальной машины

## <a name="issue"></a>Проблемы  
  
*Этот сервер имеет один или несколько виртуальных машин в приостановленном состоянии.*  
  
## <a name="impact"></a>Влияние  
  
*В зависимости от объема доступной памяти может не появиться возможность запуска дополнительных виртуальных машин.*  
  
Приостановленных виртуальных машин, не стоит выпускать их выделенной памяти, что означает, что память не доступна для запуска других виртуальных машин.  
  
## <a name="resolution"></a>Разрешение  
  
*Если это сделано намеренно, никаких дополнительных действий не требуется. В противном случае рассмотрите возможность возобновления эти виртуальные машины или завершения их работы.*  
  
#### <a name="use-hyper-v-manager-to-resume-the-virtual-machine"></a>Возобновление виртуальной машины с помощью диспетчера Hyper-V  
  
1.  Откройте диспетчер Hyper-V. (Из **средства** меню диспетчера серверов щелкните **диспетчера Hyper-V**.)  
  
2.  Из **виртуальных машин** список, найдите виртуальные машины с состоянием **приостановлено**.  
  
    > [!IMPORTANT]  
    > Состояние **приостановлено критическое** возникает, когда очень мало свободного места, оставшегося на физическое хранилище для этой виртуальной машине. Прежде чем возобновить работу виртуальной машины в этом состоянии, чтобы Освободите занятую доступное пространство в физическом хранилище.  
  
3.  Щелкните правой кнопкой мыши имя каждой виртуальной машины, а затем щелкните **Resume**. Это Возвращает виртуальную машину в рабочее состояние. После этого, если вы хотите завершить работу виртуальной машины, щелкните правой кнопкой мыши его еще раз и выберите **завершение работы**.  
  
#### <a name="use-windows-powershell-to-resume-the-virtual-machine"></a>Возобновить работу виртуальной машины с помощью Windows PowerShell  
  
Это можно сделать в одну команду с помощью фильтрации и конвейера после его получения всех виртуальных машин на узле. Введите команду:  
  
```  
get-vm | where state -eq 'paused' | resume-vm  
```
