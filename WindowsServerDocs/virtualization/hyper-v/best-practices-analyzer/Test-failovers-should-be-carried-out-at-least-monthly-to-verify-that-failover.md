# <a name="test-failovers-should-be-carried-out-at-least-monthly-to-verify-that-failover-will-succeed-and-that-virtual-machine-workloads-will-operate-as-expected-after-failover"></a>Тестовые отработки отказа должны выполняться по крайней мере ежемесячно для убедитесь, что отработка отказа будет выполнена успешно, и что рабочих нагрузок виртуальных машин будет работать как ожидалось после отработки отказа

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|**Уровень серьезности**|Предупреждение|  
|**Категория**|Операции|

## <a name="issue"></a>Проблемы  
*Произошел тестовая отработка отказа не в хотя бы один месяц.*  
  
## <a name="impact"></a>Влияние  
*Нет подтверждение плановой или незапланированной отработки отказа будет выполнена успешно, или должным образом после отработки отказа будут продолжены операции рабочей нагрузки. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Тестовую отработку отказа с помощью диспетчера Hyper-V.*
