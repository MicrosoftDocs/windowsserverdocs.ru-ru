# <a name="the-hyper-v-virtual-machine-management-service-should-be-configured-to-start-automatically"></a>Службы управления виртуальными машинами Hyper-V должен быть настроен автоматический запуск

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и сканировании см. в разделе [Анализатор соответствия рекомендациям](https://go.microsoft.com/fwlink/?LinkId=122786).

## <a name="issue"></a>Проблемы  
  
*Служба управления виртуальными машинами Hyper-V не настроен автоматический запуск.*  
  
## <a name="impact"></a>Влияние  
  
*Виртуальные машины невозможно, пока не будет запущена служба.*  
  
Виртуальных машин, работающих под управлением продолжит работу. Тем не менее нельзя будет управлять виртуальными машинами, или создать или удалить их, пока служба запущена.  
  
## <a name="resolution"></a>Разрешение  
  
*Позволяет изменить конфигурацию службы для автоматического запуска службы оснастки или sc config средство командной строки.*  
  
> [!TIP]  
> Если не удается найти службу в классическое приложение или средство командной строки сообщает, что служба не существует, средства управления Hyper-V, возможно, не установлены. Чтобы установить их:  
>   
> - В Windows Server откройте диспетчер серверов, а затем с помощью мастера добавления ролей и компонентов. Дополнительные сведения см. в разделе [установить роль Hyper-V в Windows Server 2016](../get-started/Install-the-Hyper-V-role-on-Windows-Server.md).  
> - В Windows, на рабочем столе, начните вводить **программы**, нажмите кнопку **программы и компоненты** (панель управления) > **или отключение компонентов Windows включить**  >   **Hyper-V** > **средства управления Hyper-V**. Затем нажмите кнопку **ОК**.  
  
#### <a name="to-reconfigure-the-service-to-start-automatically-using-the-services-desktop-app"></a>Чтобы изменить конфигурацию службы для запуска, автоматически с помощью классического приложения службы  
  
1.  Откройте приложение службы рабочего стола. (Щелкните **запустить**, щелкните в поле поиска, начните вводить **служб**и выберите службы в списке результатов.  
  
2.  В области сведений щелкните правой кнопкой мыши **управления виртуальными машинами Hyper-V**, а затем нажмите кнопку **свойства**.  
  
3.  На **Общие** на вкладке **запуска** введите, нажмите кнопку **автоматического**.  
  
#### <a name="to-reconfigure-the-service-to-start-automatically-using-the-sc-config-command"></a>Чтобы изменить конфигурацию службы для запуска, автоматически с помощью команды SC Config  
  
1.  Откройте Windows PowerShell.  
  
2.  Введите команду:  
  
    ```  
    set-service  vmms -startuptype automatic  
    ```  
  
3.  Если служба еще не запущена, введите:  
  
    ```  
    start-service -name vmms  
    ```
