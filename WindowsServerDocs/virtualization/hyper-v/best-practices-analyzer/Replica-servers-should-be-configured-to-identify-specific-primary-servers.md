# <a name="replica-servers-should-be-configured-to-identify-specific-primary-servers-authorized-to-send-replication-traffic"></a>Серверы-реплики следует настроить для идентификации заданных серверов-источников, право отправлять трафик репликации

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|**Продукта или компонента**|Hyper-V|

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>Проблемы  
*Настроенный таким образом этот сервер-реплику принимает трафик репликации из всех серверов-источников и сохраняет их в одном месте.*  
  
### <a name="impact"></a>Влияние  
*Все репликации из всех основных серверов хранятся в одном месте, что может привести к конфиденциальности или проблемам с безопасностью.*  
  
## <a name="resolution"></a>Разрешение  
*Создание новой записи авторизации для заданных серверов-источников и указывать отдельные места хранения для каждого из них с помощью диспетчера Hyper-V. Для группирования серверов-источников на наборы для каждой записи авторизации можно использовать подстановочные знаки.*  
  
#### <a name="create-authorization-entries-using-hyper-v-manager"></a>Создание записи авторизации с помощью диспетчера Hyper-V  
  
1.  Откройте диспетчер Hyper-V. (В диспетчере сервера щелкните **средства** > **диспетчера Hyper-V**.)  
  
2.  Из списка узлов, щелкните правой кнопкой мыши и нажмите щелкните **параметры Hyper-V**.  
  
3.  В области навигации щелкните **конфигурация репликации**.  
  
4.  В разделе **авторизация и хранилище**, нажмите кнопку **разрешается репликация с указанным серверам**.  
  
5.  Под списком серверов, выберите **добавить**.  
  
6.  В разделе **добавить запись авторизации**:  
  
    -   Введите полное имя первого сервера.  
  
    -   Укажите выделенную расположение для хранения только файлов с этого сервера.  
  
7.  Нажмите кнопку **ОК**.  
  
8.  Повторите для каждого сервера-источника.  
  
9. Нажмите кнопку **ОК** еще раз, чтобы завершить и закрыть окно.  
  
### <a name="create-authorization-entries-using-windows-powershell"></a>Создание записи авторизации с помощью Windows PowerShell  
  
1.  Откройте Windows PowerShell. (На рабочем столе нажмите кнопку Пуск и начните вводить **Windows PowerShell**.)  
  
2.  Щелкните правой кнопкой мыши **Windows PowerShell** и нажмите кнопку **Запуск от имени администратора**.  
  
3.  Выполните команду, аналогичную приведенной ниже, заменив:  
  
    -   Имя сервера-источника server01.domain01.contoso.com с полное доменное имя сервера.  
  
    -   Расположение D:\ReplicaVMStorage с местоположением.  
  
    -   Группа доверия с именем по умолчанию с именем своей группы, если вы создали. В противном случае используется значение по умолчанию.  
  
```  
New-VMReplicationAuthorizationEntry server01.domain01.contoso.com D:\ReplicaVMStorage DEFAULT  
```
