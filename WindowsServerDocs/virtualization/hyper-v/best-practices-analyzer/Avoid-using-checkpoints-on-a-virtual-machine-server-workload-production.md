# <a name="avoid-using-checkpoints-on-a-virtual-machine-that-runs-a-server-workload-in-a-production-environment"></a>Старайтесь не использовать контрольные точки на виртуальной машине выполнение рабочих нагрузок сервера в рабочей среде

>Область применения. Windows Server 2016

## <a name="issue"></a>Проблемы  
  
*Найдена виртуальной машины с контрольными точками один или несколько.*  
  
## <a name="impact"></a>Влияние  
  
*Доступное пространство может израсходовано на физическом диске, в котором хранятся файлы контрольных точек. В этом случае могут выполняться никакие операции дополнительный диск в физическом хранилище. Может повлиять на любой виртуальной машины, которая зависит от физического хранилища.*  
  
Если места на физическом диске заканчивается, работающей виртуальной машины с контрольными точками или виртуальных жестких дисков, хранящихся на нем может автоматически приостановлена. Диспетчер Hyper-V отображается состояние этих виртуальных машин как «приостановлено критическое».  
  
## <a name="resolution"></a>Разрешение  
  
*Если рабочая нагрузка сервера операционной системы виртуальной машины в рабочей среде, отключите виртуальную машину и с помощью диспетчера Hyper-V в применение и удаление контрольных точек. Чтобы удалить контрольные точки, необходимо завершить работу виртуальной машины, чтобы завершить процесс.*  
  
> [!NOTE]  
> Рабочие контрольные точки теперь доступны в качестве альтернативы стандартных контрольных точек. Дополнительные сведения см. в разделе [сравнение стандартных или рабочих контрольных точек](../manage/Choose-between-standard-or-production-checkpoints-in-Hyper-V.md).
