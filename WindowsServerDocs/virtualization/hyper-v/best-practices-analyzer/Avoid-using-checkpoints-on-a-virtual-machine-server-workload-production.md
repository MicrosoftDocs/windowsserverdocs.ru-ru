## <a name="issue"></a>Проблема

*Обнаружена виртуальная машина с одной или несколькими контрольными точками.*

## <a name="impact"></a>Влияние

*Доступное место на физическом диске, где хранятся файлы контрольных точек, может быть исчерпано. В этом случае дополнительные операции с диском не могут быть выполнены для физического хранилища. Это может повлиять на любую виртуальную машину, основанную на физическом хранилище.*

Если место на физическом диске заканчивается, любая работающая виртуальная машина с контрольными точками или виртуальными жесткими дисками, хранящимися на этом диске, может быть приостановлена автоматически. Диспетчер Hyper-V отображает состояние этих виртуальных машин как приостановку, критическое.

## <a name="resolution"></a>Решение

*Если виртуальная машина выполняет рабочую нагрузку сервера в рабочей среде, переведите виртуальную машину в автономный режим, а затем с помощью диспетчера Hyper-V примените или удалите контрольные точки. Чтобы удалить контрольные точки, необходимо завершить работу виртуальной машины, чтобы завершить процесс.*

> [!NOTE]
> Рабочие контрольные точки теперь доступны в качестве альтернативы стандартным контрольным точкам. Дополнительные сведения см. в разделе [Выбор контрольных точек](../manage/Choose-between-standard-or-production-checkpoints-in-Hyper-V.md)уровня "Стандартный" или "Рабочая".
