# <a name="virtual-machines-configured-with-a-virtual-fibre-channel-adapter-should-be-configured-for-high-availability-to-the-fibre-channel-based-storage"></a>Виртуальные машины, настроенные с помощью виртуального адаптера Fibre Channel, которые должны быть настроены для высокого уровня доступности в хранилище на базе Fibre Channel

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>**Проблема**  
*Один или несколько виртуальных машин не хватает высокодоступные подключения к хранилищам на базе Fibre Channel, так как эти виртуальные машины настроены с помощью виртуального адаптера Fibre Channel, подключенный к только один адаптер шины (HBA).*  
  
## <a name="impact"></a>**Влияние**  
*Сбой адаптера шины узла может блокировать подключение Fibre Channel между хранилища и виртуальных машин. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>**Решение**  
*Добавить еще одно соединение от виртуальной машины для адаптера шины и настройка многопутевого ввода-вывода (MPIO) в гостевой операционной системы для установления избыточные подключения Fibre Channel.*
