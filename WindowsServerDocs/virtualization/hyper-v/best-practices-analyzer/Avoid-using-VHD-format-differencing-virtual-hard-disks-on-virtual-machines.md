# <a name="avoid-using-vhd-format-differencing-virtual-hard-disks-on-virtual-machines-that-run-server-workloads-in-a-production-environment"></a>Старайтесь не использовать формат VHD разностных виртуальных жестких дисков на виртуальных машинах под управлением серверных рабочих нагрузок в рабочей среде

## <a name="issue"></a>**Проблема**  
*Один или несколько виртуальных машин используйте разностных виртуальных жестких дисков формата VHD.*  
  
## <a name="impact"></a>**Влияние**  
*Формат VHD разностных виртуальных жестких дисков будет испытывать проблемы согласованности при сбое питания. Проблемы согласованности может произойти, если физический диск выполняет обновление неполных или неверных в сектор в VHD-файла, которая изменяется при возникновении сбоя питания. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>**Решение**  
*Завершите работу виртуальной машины и преобразовать цепочку разностных виртуальных жестких дисков в формат VHDX формат VHD или слияния цепочке фиксированный виртуальный жесткий диск. (Механизмами для обеспечения надежности, направленных на защиту от поврежденных из-за сбоев питания диск имеет формат VHDX.) Тем не менее не преобразовывать виртуальный жесткий диск, если он может быть подключен к более ранней версии Windows в определенный момент. Windows освобождает раньше, чем Windows Server 2012 не поддерживают формат VHDX.*
