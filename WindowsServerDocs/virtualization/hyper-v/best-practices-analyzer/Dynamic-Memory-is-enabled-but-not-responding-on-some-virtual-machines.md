# <a name="dynamic-memory-is-enabled-but-not-responding-on-some-virtual-machines"></a>Динамическая память включена, но не отвечает на некоторые виртуальные машины

## <a name="issue"></a>Проблемы  
*Один или несколько виртуальных машин при возникновении проблем с драйвером, необходимых для динамической памяти в гостевой операционной системы.*  
  
## <a name="impact"></a>Влияние  
*Гостевой операционной системы в следующих виртуальных машин может не запуститься или может привести работать так, как Hyper-V не удается настроить память динамически реагировать на изменения в требуемый объем памяти. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Это ожидаемое поведение при загрузке виртуальной машины. Если виртуальная машина не загружается, убедитесь, что службы integration services будут обновлены до последней версии, и что гостевая операционная система поддерживает динамическую память.*  
  
Начиная с Windows Server 2016 службы integration services доставляются через Центр обновления Windows. Убедитесь, что виртуальные машины настроены для получения обновлений, чтобы получить последнюю версию служб integration services.  
  
Динамическая память работает с конкретным версиям поддерживаемых гостевых компьютеров. См. в разделе [Hyper-V Dynamic Memory Overview](https://technet.microsoft.com/library/hh831766.aspx) для версии, предшествующей Windows Server 2016 и Windows 10.
