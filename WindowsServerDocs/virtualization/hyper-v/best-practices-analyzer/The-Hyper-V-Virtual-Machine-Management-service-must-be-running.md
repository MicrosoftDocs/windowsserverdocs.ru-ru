# <a name="the-hyper-v-virtual-machine-management-service-must-be-running"></a>Должна быть запущена служба управления виртуальными машинами Hyper-V

>Область применения. Windows Server 2016
  
Дополнительные сведения о рекомендациях и сканировании см. в разделе [Анализатор соответствия рекомендациям](https://go.microsoft.com/fwlink/?LinkId=122786).  
  
|Свойство|Подробности|  
|-|-|  
|**Операционная система**|Windows Server 2016|  
|**Продукта или компонента**|Hyper-V|  
|**Уровень серьезности**|Ошибка|  
|**Категория**|предварительные требования|  

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.

## <a name="issue"></a>Проблемы  
  
*Не запущена служба, необходимая для управления виртуальными машинами.*  
  
## <a name="impact"></a>Влияние  
  
*Могут быть выполнены никакие операции управления виртуальной машины.*  
  
Виртуальных машин, работающих под управлением продолжит работу. Тем не менее нельзя будет управлять виртуальными машинами, или создать или удалить их, пока служба запущена.  
  
## <a name="resolution"></a>Разрешение  
  
*Чтобы изменить конфигурацию службы для автоматического запуска используйте оснастку служб или средство командной строки Sc config.*  
  
> [!TIP]  
> Если не удается найти службу в классическое приложение или средство командной строки сообщает, что служба не существует, средства управления Hyper-V, возможно, не установлены. И если вы не сможете см. в разделе консоли Hyper-V из меню "Пуск", следует установить средства управления Hyper-V.

Чтобы установить средства управления Hyper-V.  
>   
> - В Windows Server откройте диспетчер серверов, а затем с помощью мастера добавления ролей и компонентов. Дополнительные сведения см. в разделе [установить роль Hyper-V в Windows Server 2016](../get-started/Install-the-Hyper-V-role-on-Windows-Server.md).  Можно также использовать PowerShell для установки инструментов (`Install-WindowsFeature -Name Hyper-V-Tools, Hyper-V-PowerShell`) 
> - В Windows, на рабочем столе, начните вводить **программы**, нажмите кнопку **программы и компоненты** (панель управления) > **или отключение компонентов Windows включить**  >   **Hyper-V** > **средства управления Hyper-V**. Затем нажмите кнопку **ОК**.  
  
### <a name="to-reconfigure-the-service-to-start-automatically-using-the-services-desktop-app"></a>Чтобы изменить конфигурацию службы для запуска, автоматически с помощью классического приложения службы  
  
1.  Откройте приложение службы рабочего стола. (Щелкните **запустить**, щелкните в **начать поиск** введите **services.msc**, и нажмите клавишу ВВОД.)  
  
2.  В области сведений щелкните правой кнопкой мыши **управления виртуальными машинами Hyper-V**, а затем нажмите кнопку **свойства**.  
  
3.  На **Общие** на вкладке **запуска** введите, нажмите кнопку **автоматического**.  
  
4.  Чтобы запустить службу, нажмите кнопку **запустить**.  
  
### <a name="to-reconfigure-the-service-to-start-automatically-using-sc-config"></a>Чтобы изменить конфигурацию службы для запуска, автоматически с помощью SC Config  
  
1.  Откройте Windows PowerShell. (На рабочем столе щелкните **запустить** и начните вводить **Windows PowerShell**.)  
  
2.  Щелкните правой кнопкой мыши **Windows PowerShell** и нажмите кнопку **Запуск от имени администратора**.  
  
3.  Чтобы изменить конфигурацию службы, введите:  
  
    ```  
    sc config vmms start=auto  
    ```  
  
4.  Чтобы запустить службу, введите:  
  
    ```  
    sc start vmms  
    ```  
  
Если служба уже настроена на автоматический запуск, необходимо перезапустить службу, можно сделать с помощью диспетчера Hyper-V или с помощью команды «sc start vmms», показанный выше.  
  
#### <a name="to-restart-the-service-from-hyper-v-manager"></a>Чтобы перезапустить службу с помощью диспетчера Hyper-V  
  
1.  Откройте диспетчер Hyper-V. Нажмите кнопку **запустить**, пункты **Администрирование**, а затем нажмите кнопку **диспетчера Hyper-V**.  
  
2.  В области навигации щелкните имя сервера, если он еще не установлен.  
  
3.  В **действия** панели щелкните **запустить службу**.
