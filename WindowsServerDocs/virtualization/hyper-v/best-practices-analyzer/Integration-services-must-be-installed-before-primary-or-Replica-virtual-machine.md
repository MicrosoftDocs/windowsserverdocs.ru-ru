# <a name="integration-services-must-be-installed-before-primary-or-replica-virtual-machines-can-use-an-alternate-ip-address-after-a-failover"></a>Службы Integration services должен быть установлен до первичного или реплик виртуальных машин можно использовать альтернативный IP-адрес после отработки отказа

>Область применения. Windows Server 2016

## <a name="issue"></a>Проблемы  
*Виртуальные машины, участвующие в репликации может быть настроен на использование определенного IP-адреса, в случае отработки отказа, но только в том случае, если установлены службы интеграции в операционной системе виртуальной машины.*  
  
## <a name="impact"></a>Влияние  
*В случае отработки отказа (плановой, незапланированной или) реплика виртуальной машины подключится к сети, используя тот же адрес в качестве основной виртуальной машины. Эта конфигурация может вызвать проблемы с подключением. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Используйте подключение к виртуальной машине для установки служб integration services на виртуальной машине.*  
  
Начиная с Windows Server 2016 integration services для виртуальных машин Windows предоставляются через Центр обновления Windows. Убедитесь, что эти виртуальные машины настроены для получения обновлений Windows, чтобы получить последнюю версию служб integration services. Ядро Linux теперь включает в себя службы интеграции Linux (LIS) и обновляется для новых выпусков, но дистрибутивов Linux, в зависимости от более старых версиях ядер отсутствуют последние улучшения или исправления. Дополнительные сведения см. в разделе [виртуальных машин поддерживается Linux и FreeBSD для Hyper-V в Windows](../Supported-Linux-and-FreeBSD-virtual-machines-for-Hyper-V-on-Windows.md).
