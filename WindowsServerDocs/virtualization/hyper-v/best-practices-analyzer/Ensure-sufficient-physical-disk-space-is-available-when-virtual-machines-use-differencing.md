# <a name="ensure-sufficient-physical-disk-space-is-available-when-virtual-machines-use-differencing-virtual-hard-disks"></a>Обеспечение доступности достаточного места на физическом диске, если виртуальные машины используют разностные виртуальные жесткие диски

>Область применения. Windows Server 2016

## <a name="issue"></a>Проблема
*Одна или несколько виртуальных машин используют разностные виртуальные жесткие диски.*

## <a name="impact"></a>Влияние
*Для разностных виртуальных жестких дисков требуется доступное место на томе размещения, чтобы можно было выделить место при записи на виртуальные жесткие диски. Если доступное место исчерпано, это может повлиять на все виртуальные машины, использующие физическое хранилище. Это влияет на следующие виртуальные машины:*

\<list of virtual machines>

## <a name="resolution"></a>Решение
*Мониторинг свободного места на диске для обеспечения достаточного объема свободного места для расширения виртуального жесткого диска. Рассмотрите возможность объединения разностных виртуальных жестких дисков в их родительский узел. В диспетчере Hyper-V проверьте разностный диск, чтобы определить родительский виртуальный жесткий диск. Если разностный диск объединяется с родительским диском, который совместно используется другими разностными дисками, это действие приведет к повреждению связи между другими разностными дисками и родительским диском, что сделает их непригодными для использования. Убедившись, что родительский виртуальный жесткий диск не является общим, можно с помощью мастера изменения дисков выполнить слияние разностного диска к родительскому виртуальному жесткому диску.*
