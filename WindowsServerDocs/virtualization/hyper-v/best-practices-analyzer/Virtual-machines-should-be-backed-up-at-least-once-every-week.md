# <a name="virtual-machines-should-be-backed-up-at-least-once-every-week"></a>Виртуальные машины необходимо архивировать хотя бы один раз в неделю

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>Проблемы  
*Один или несколько виртуальных машин не будут заархивированы за последнюю неделю.*  
  
## <a name="impact"></a>Влияние  
*Может произойти потеря значительного объема данных, если виртуальная машина сталкивается с проблемой и последней резервной копии не существует. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Запланируйте резервное копирование виртуальных машин для выполнения по крайней мере один раз в неделю. Это правило можно пропустить, если эта виртуальная машина — это реплика, и его основной виртуальной машины выполняется резервное копирование, или если выполняется основной виртуальной машины, а ее реплика выполняется резервное копирование.*
