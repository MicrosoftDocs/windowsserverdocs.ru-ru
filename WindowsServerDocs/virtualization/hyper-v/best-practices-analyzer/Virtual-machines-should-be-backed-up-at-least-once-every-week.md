# <a name="virtual-machines-should-be-backed-up-at-least-once-every-week"></a>Виртуальные машины должны создаваться по крайней мере раз в неделю.

>Область применения: Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|-|-|  
|**Операционная система**|Windows Server 2016|  
|**Продукт или функция**|Hyper-V|  
|**Серьезности**|Ошибка|  
|**Категория**|Конфигурация|  
  
В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>Проблема  
*На прошлой неделе не была создана резервная копия одной или нескольких виртуальных машин.*  
  
## <a name="impact"></a>Влияние  
*Значительная потери данных может возникать, если на виртуальной машине возникла проблема, а последняя резервная копия не существует. Это влияет на следующие виртуальные машины:*  
  
\<список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Запланируйте запуск резервной копии виртуальных машин по крайней мере раз в неделю. Это правило можно проигнорировать, если эта виртуальная машина является репликой и выполняется резервное копирование основной виртуальной машины, или если это основная виртуальная машина и выполняется резервное копирование ее реплики.*
