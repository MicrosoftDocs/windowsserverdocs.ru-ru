# <a name="use-at-least-smb-protocol-version-30-configured-for-continuous-availability-on-file-shares-that-store-files-for-virtual-machines"></a>Используйте по меньшей мере версия протокола SMB 3.0, настроен для постоянной доступности в общих папках, в которых хранятся файлы для виртуальных машин

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

## <a name="issue"></a>**Проблема**  
*Файлы виртуальной машины или виртуальные жесткие диски хранятся в общей сетевой папки, в которой не настроены функции непрерывной доступности версии протокола SMB 3.0.*  
  
## <a name="impact"></a>**Влияние**  
*Майкрософт не рекомендует эту конфигурацию, так как она может влиять на доступность виртуальных машин с помощью сервера. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>**Решение**  
Выполните одно из следующих действий.  
  
-   Переместите файлы в файловый ресурс SMB 3.0, который настроен для обеспечения непрерывной доступности.  
  
-   Перенастройте текущий файловый ресурс для обеспечения непрерывной доступности.
