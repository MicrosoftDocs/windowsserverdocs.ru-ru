# <a name="to-participate-in-replication-servers-in-failover-clusters-must-have-a-hyper-v-replica-broker-configured"></a>Чтобы участвовать в репликации, серверы в отказоустойчивых кластерах необходимо иметь настроен брокер реплики Hyper-V

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и проверках см. в разделе [запуска наиболее проверок анализатором соответствия рекомендациям и Управление результатами проверок](https://go.microsoft.com/fwlink/p/?LinkID=223177).

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>Проблемы  
*Для кластеров отработки отказа реплики Hyper-V требует использования имя брокера реплики Hyper-V, а не имя отдельного сервера.*  
  
## <a name="impact"></a>Влияние  
*Если виртуальная машина перемещается на другой узел, не удается продолжить репликацию.*  
  
## <a name="resolution"></a>Разрешение  
*Используйте диспетчер отказоустойчивости кластеров для настройки брокера реплики Hyper-V. В диспетчере Hyper-V убедитесь, что конфигурация репликации использует имя брокера реплики Hyper-V в качестве имени сервера.*
