# <a name="hyper-v-should-be-the-only-enabled-role"></a>Hyper-V должен быть единственной ролью включена

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и сканировании см. в разделе [Анализатор соответствия рекомендациям](https://go.microsoft.com/fwlink/?LinkId=122786).  
  
|Свойство|Подробности|  
|-|-|  
|**Операционная система**|Windows Server 2016|  
|**Продукта или компонента**|Hyper-V|  
|**Уровень серьезности**|Предупреждение|  
|**Категория**|Конфигурация|  
  
В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>Проблемы  
  
*На этом сервере включены роли, отличные от Hyper-V.*  
  
В большинстве случаев это не рекомендуется для установки других ролей на сервере под управлением роли Hyper-V. Удаленной службы роли узла виртуализации рабочих столов — это исключение, так как он является частью роли служб удаленных рабочих столов и требуется Hyper-V, устанавливаемые на одном сервере.  
  
## <a name="impact"></a>Влияние  
  
*Роль Hyper-V должен быть единственной ролью включена на сервере.*  
  
Эта рекомендация позволяет обеспечить операционной системы узла, ролей, компонентов и приложений, которые не требуются для запуска Hyper-V. Следование этой рекомендации и под управлением Hyper-V на сервере Nano Server помогает снизить число обновлений, которые понадобятся, так как только Nano Server, компоненты службы Hyper-V и гипервизор Windows будет от обновлений программного обеспечения.  
  
## <a name="resolution"></a>Разрешение  
  
*Чтобы удалить все роли, за исключением Hyper-V с помощью диспетчера серверов.*  
  
Диспетчер сервера включает в себя Мастер удаления ролей. Этот мастер позволяет удалить сразу несколько ролей. Прежде чем удалить роли, мастер удаления ролей проверяет зависимости снизить риск удаления программного обеспечения, которая зависит от других ролей. Если зависимости есть, мастер запросит подтверждение удаления других ролей, служб ролей или программное обеспечение, необходимое для установленных ролей.   
  
Чтобы использовать диспетчер сервера, необходимо войти в систему компьютера с правами администратора.  
  
#### <a name="to-remove-a-role"></a>Чтобы удалить роль  
  
1.  Откройте диспетчер серверов с помощью сочетания клавиш на **запустить** меню на панели задач Windows, или «Администрирование».  
2.   В **Сводка по ролям** области главного окна диспетчера серверов щелкните **удаления ролей**. Следуйте инструкциям мастера для удаления роли.
