# <a name="configure-a-policy-to-throttle-the-replication-traffic-on-the-network"></a>Настройка политики для регулирования трафика репликации в сети

## <a name="issue"></a>Проблема
*Возможно, не будет ограничений на объем пропускной способности сети, которую может использовать репликация.*

## <a name="impact"></a>Влияние
*Пропускная способность сети может быть полностью обработана трафиком репликации, что влияет на другие критические сетевые операции. Это влияет на следующие порты:*

\<list of virtual machines>

## <a name="resolution"></a>Решение
*Если для регулирования сетевого трафика используется другой метод, это можно пропустить. В противном случае используйте редактор групповая политика, чтобы настроить политику, которая будет регулировать сетевой трафик до соответствующего порта сервера реплики.*
