Дополнительные сведения о рекомендациях и сканировании см. в разделе [Анализатор соответствия рекомендациям](https://go.microsoft.com/fwlink/?LinkId=122786).

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.  
  
## <a name="issue"></a>Проблемы  
  
*Один или несколько служб integration services, отключено или не работает на виртуальной машине.*  
  
## <a name="impact"></a>Влияние  
  
*Компонент службы или интеграции может завершаться ошибкой для следующих виртуальных машин:*  
  
\<Список имен виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
  
*Позволяет убедиться, что служба настроена на автоматический запуск и не остановить службы оснастки или sc config средство командной строки.*  
  
#### <a name="to-configure-how-a-service-is-started-using-the-services-snap-in"></a>Чтобы настроить службы запускается через оснастку «службы»  
  
1.  Использование служб удаленных рабочих столов или подключение к виртуальной машине для подключения к виртуальной машине и войдите на гостевую ОС.  
  
2.  Откройте компонент "Службы". (Щелкните **запустить**, щелкните в **начать поиск** введите **services.msc**, и нажмите клавишу ВВОД.)  
  
3.  В области сведений щелкните правой кнопкой мыши службу, который требуется настроить, а затем нажмите кнопку **свойства**.  
  
4.  На **Общие** на вкладке **запуска** введите, нажмите кнопку **автоматического**.  
  
#### <a name="to-configure-how-a-service-is-started-using-sc-config"></a>Чтобы настроить службы запускается с помощью SC Config  
  
1.  Откройте Windows PowerShell. (На рабочем столе щелкните **запустить** и начните вводить **Windows PowerShell**.)  
  
2.  Щелкните правой кнопкой мыши **Windows PowerShell** и нажмите кнопку **Запуск от имени администратора**.  
  
3.  Замените < service-name > именем службы, введите:  
  
    ```  
    sc config <service-name> start=auto  
    ```
