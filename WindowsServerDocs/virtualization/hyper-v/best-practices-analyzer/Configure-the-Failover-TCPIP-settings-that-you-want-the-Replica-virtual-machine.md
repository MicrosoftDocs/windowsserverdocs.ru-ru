# <a name="configure-the-failover-tcpip-settings-that-you-want-the-replica-virtual-machine-to-use-in-the-event-of-a-failover"></a>Настройка параметров TCP/IP отработки отказа, которые должна использовать виртуальная машина реплики в случае отработки отказа

>Область применения. Windows Server 2016

Дополнительные сведения о рекомендациях и сканировании см. в разделе [Запуск сканирования анализатором соответствия рекомендациям и управление результатами сканирования](https://go.microsoft.com/fwlink/p/?LinkID=223177).

|**Продукт или компонент**|Hyper-V|
|**Уровень серьезности**|Предупреждение|
|**Категория**|Конфигурация|

В следующих разделах курсив указывает текст пользовательского Интерфейса, который отображается в анализатор соответствия рекомендациям для этой проблемы.

## <a name="issue"></a>Проблема
*Реплики виртуальных машин, настроенных со статическим IP-адресом, должны быть настроены на использование другого IP-адреса из основной виртуальной машины в случае отработки отказа.*

## <a name="impact"></a>Влияние
*Клиенты, использующие рабочую нагрузку, поддерживаемую основной виртуальной машиной, не смогут подключиться к реплике виртуальной машины после отработки отказа. Кроме того, исходный IP-адрес основной виртуальной машины не будет действителен в топологии сети виртуальной машины реплики. Это влияет на следующие виртуальные машины:*

\<list of virtual machines>

## <a name="resolution"></a>Решение
*С помощью диспетчера Hyper-V настройте IP-адрес, который виртуальная машина реплики должна использовать в случае отработки отказа.*
