# <a name="configure-the-failover-tcpip-settings-that-you-want-the-replica-virtual-machine-to-use-in-the-event-of-a-failover"></a>Настройте параметры TCP/IP отработки отказа, которые должны реплики виртуальной машины для использования при отработке отказа

## <a name="issue"></a>Проблемы  
*Реплики виртуальных машин настроены статические IP-адрес должен использовать другой IP-адрес, из какой их основной виртуальной машины при отработке отказа.*  
  
## <a name="impact"></a>Влияние  
*Клиенты, с помощью рабочей нагрузки, поддерживаемый основной виртуальной машины не может быть возможность подключиться к виртуальной машине реплики после отработки отказа. Кроме того исходный IP-адрес основной виртуальной машины не будет допустимым топологии сети виртуальной машины реплики. Это влияет на следующие виртуальные машины:*  
  
\<Список виртуальных машин >  
  
## <a name="resolution"></a>Разрешение  
*Использование диспетчера Hyper-V для настройки IP-адрес, реплика виртуальной машины следует использовать в случае отработки отказа.*
