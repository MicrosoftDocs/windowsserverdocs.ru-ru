# <a name="hyper-v-technology-overview"></a>Общие сведения о технологии Hyper-V

>Область применения. Windows Server 2016, Microsoft Hyper-V Server 2016, Windows Server 2019 г., Microsoft Hyper-V Server 2019 г.

Hyper-V — программный продукт для виртуализации оборудования корпорации Майкрософт. Он позволяет создавать и использовать версию программного обеспечения компьютер, называемый *виртуальной машины*. Каждая виртуальная машина работает как полный компьютер, под управлением операционной системы и программы. При необходимости вычислительных ресурсов, виртуальные машины предоставляют большую гибкость, экономить время и деньги они более эффективный способ использовать оборудование, чем просто под управлением одной операционной системы на физическом оборудовании.

Hyper-V выполняется каждая виртуальная машина в собственной изолированной области, это означает, что более чем одну виртуальную машину можно запустить на том же оборудовании в то же время. Может потребоваться сделать это во избежание проблем, таких как сбой влияет на другие рабочие нагрузки, или для предоставления различных пользователей, групп или службы доступа к разных систем.

## <a name="some-ways-hyper-v-can-help-you"></a>Некоторые способы для Hyper-V может помочь

Hyper-V может помочь:

- **Создание или расширение частной облачной среде.** Предоставлять более гибкий и по требованию ИТ-служб, перемещение или расширение использования общих ресурсов и регулировать их использование в мере изменения потребностей.

- **Более эффективное использование оборудования.** Консолидация серверов и рабочих нагрузок на мощных физических компьютеров, чтобы использовать меньше электроэнергии и физического пространства.

- **Непрерывные деловые операции.** Минимизации влияния запланированных и незапланированных простоев на рабочие нагрузки.

- **Создание или расширение инфраструктуры виртуальных рабочих столов (VDI).** Использование стратегия централизованных рабочих столов с помощью VDI может помочь вам повысить гибкость бизнеса и защиты данных, а также соответствие нормативным требованиям и управление настольных операционных систем и приложений. Развертывание Hyper-V и узел виртуализации удаленных рабочих столов (RD Virtualization Host) на том же сервере, чтобы сделать личные виртуальные рабочие столы или пулы виртуальных рабочих столов доступными для пользователей.

- **Упростить разработку и тестирование более эффективным.** Воспроизведения реальных компьютерных сред без необходимости приобретения и обслуживания все оборудование, которое потребовалось бы при использовании физических систем.

## <a name="hyper-v-and-other-virtualization-products"></a>Hyper-V и другими средствами виртуализации

Hyper-V в Windows и Windows Server заменяет старые продукты виртуализации оборудования, например Microsoft Virtual PC, Microsoft Virtual Server и Windows Virtual PC. Hyper-V предоставляет функции сети, производительности, хранения и безопасности, недоступные в этих старых продуктов.

Hyper-V и большинство виртуализации сторонних приложений, требующих тех же функций процессора не совместимы. Это, так как функции процессора, известный как расширения аппаратной виртуализации, не будут передаваться. Дополнительные сведения см. в разделе [виртуализации приложения не работают вместе с Hyper-V, Device Guard и Credential Guard](https://support.microsoft.com/kb/3204980).

## <a name="what-features-does-hyper-v-have"></a>Какие возможности есть Hyper-V?

Hyper-V предлагает множество функций. Это Обзор, сгруппированные по новые функции предоставляют или помочь.

**Вычислительную среду** -виртуальной машины Hyper-V включает в себя же основные элементы в виде физического компьютера, таких как память, процессор, хранилища и сети. Все эти части имеют компоненты и параметры, которые можно настроить различные способы достижения различных целей. Хранилища и сети можно каждого считать свои собственные категории, из-за множества способов их можно настроить.

**Аварийное восстановление и резервное копирование** -для аварийного восстановления, реплики Hyper-V создает копии виртуальных машин, предназначенный для сохранения в другом физическом расположении, чтобы восстановить виртуальную машину из копии. При резервном копировании Hyper-V предоставляет два типа. Одна использует сохраненные состояния и другой использует службу теневого копирования томов (VSS), поэтому можно сделать резервные копии с согласованием приложений для программ, поддерживающих VSS.

**Оптимизация** -каждая поддерживаемая операционная система имеет настраиваемый набор служб и драйверов, вызывается *служб integration services*, упрощающие процесс для использования операционной системы на виртуальной машине Hyper-V.

**Переносимость** - компонентами, такими как Динамическая миграция, перенос хранилища и импорта и экспорта облегчают перемещение и не распространяйте виртуальной машины.

**Возможности удаленного подключения** -Hyper-V включает подключение к виртуальной машине, это средство удаленного подключения для использования с Windows и Linux. В отличие от удаленного рабочего стола, это средство дает консоли доступа, поэтому увидеть, что происходит в гостевой даже в том случае, если еще не загружен операционной системы.

**Безопасность** -безопасной загрузки и экранированных виртуальных машин защиты от вредоносных программ и других несанкционированного доступа для виртуальной машины и ее данных.

Сводка функций, реализованных в этой версии, см. в разделе [новые возможности в Hyper-V в Windows Server](What-s-new-in-Hyper-V-on-Windows.md). Некоторые компоненты или компоненты ограничен могут быть настроены. Дополнительные сведения см. в разделе [планирование масштабируемость Hyper-V в Windows Server 2016](plan/Plan-for-Hyper-V-scalability-in-Windows-Server-2016.md).

## <a name="how-to-get-hyper-v"></a>Как получить Hyper-V

Hyper-V доступен в Windows Server и Windows, как роль сервера, доступных для x64 версий Windows Server. Инструкции по сервера, см. в разделе [установить роль Hyper-V в Windows Server](get-started/Install-the-Hyper-V-role-on-Windows-Server.md). В Windows, он доступен как [функция](https://docs.microsoft.com/virtualization/hyper-v-on-windows/index) в некоторых 64-разрядных версиях Windows. Эта схема также доступна как загружаемый, сервер автономным продуктом, [Microsoft Hyper-V Server](https://www.microsoft.com/evalcenter/evaluate-hyper-v-server-2019).

## <a name="how-hyper-v-works"></a>Как работает Hyper-V

Hyper-V — это технология виртуализации на основе низкоуровневой оболочки. Hyper-V использует гипервизор Windows, который требует физический процессор с определенными функциями. Сведения об оборудовании, см. в разделе [требования к системе для Hyper-V в Windows Server](System-requirements-for-Hyper-V-on-Windows.md).

В большинстве случаев гипервизор управляет взаимодействием между оборудованием и виртуальные машины. Этот управлением низкоуровневой оболочки, доступ к оборудованию предоставляет виртуальные машины изолированную среду, в котором они выполняются. В некоторых конфигурациях виртуальной машины или операционной системы на виртуальной машине имеет прямой доступ к оборудованию графики, сети или хранилища.

## <a name="what-does-hyper-v-consist-of"></a>Что состоят из Hyper-V?

Необходимые компоненты, которые работают совместно, чтобы вы могли создавать и запускать виртуальные машины Hyper-V. Вместе эти части называются платформы виртуализации. Они установлены как набор при установке роли Hyper-V. Обязательные части включают низкоуровневой оболочки Windows, служба управления виртуальными машинами Hyper-V, поставщик виртуализации WMI, шины виртуальной машины (VMbus), поставщик службы виртуализации (VSP) и инфраструктуры виртуальных драйвера (VID).

Hyper-V также входят инструменты для управления и подключения. Их можно установить на том же компьютере, что роль Hyper-V установлена и на компьютерах без установленной ролью Hyper-V. Эти средства являются:

- В диспетчере Hyper-V
- [Модуль Hyper-V для Windows PowerShell](https://docs.microsoft.com/powershell/module/hyper-v/index)
- [Подключение к виртуальной машине](https://docs.microsoft.com/windows-server/virtualization/hyper-v/learn-more/hyper-v-virtual-machine-connect) \(иногда называют VMConnect\)
- [Windows PowerShell Direct](manage/Manage-Windows-virtual-machines-with-PowerShell-Direct.md)
