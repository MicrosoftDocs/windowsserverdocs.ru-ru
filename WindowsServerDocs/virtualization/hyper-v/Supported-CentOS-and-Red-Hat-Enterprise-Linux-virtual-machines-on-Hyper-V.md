# <a name="supported-centos-and-red-hat-enterprise-linux-virtual-machines-on-hyper-v"></a>Поддерживается CentOS и Red Hat Enterprise Linux виртуальных машин в Hyper-V

>Область применения. Windows Server 2016, Hyper-V Server 2016, Windows Server 2012 R2, Hyper-V Server 2012 R2, Windows Server 2012, Hyper-V Server 2012, Windows Server 2008 R2, Windows 10, Windows 8.1, Windows 8, Windows 7.1, Windows 7

Следующие карт распространения функций указывают функции, присутствующие во встроенных и загружаемых версиях Linux Integration Services. После таблицы перечислены известные проблемы и обходные пути для каждого распределения.

Встроенные драйверы Red Hat Enterprise Linux Integration Services для Hyper-V (доступно с Red Hat Enterprise Linux 6.4) достаточны для гостевых систем Red Hat Enterprise Linux для запуска с помощью синтетических устройств высокой производительности на узлах Hyper-V. Эти встроенные драйверы также сертифицированы Red Hat для этих целей. Сертифицированные конфигурации можно просмотреть на этой веб-странице Red Hat: [Red Hat сертификации каталога](https://access.redhat.com/ecosystem/search/#/ecosystem/Red%20Hat%20Enterprise%20Linux?sort=sortTitle%20asc&vendors=Microsoft&category=Server). Это необязательно скачать и установить пакеты служб интеграции Linux из центра загрузки Майкрософт, и это таким образом, могут ограничить технической поддержки Red Hat как описано в статье базы знаний Red Hat 1067: [Базы знаний Red Hat 1067](https://access.redhat.com/articles/1067).

Из-за возможных конфликтов между встроенной поддержки LIS и загружаемые поддержки LIS при обновлении ядра, отключить автоматическое обновление, удаление на LIS загружаемые пакеты, обновления ядра, перезагрузки и затем установить последнюю версию LIS выпуске, и снова выполнить перезагрузку.

> [!NOTE]
> Официальные данные о сертификации Red Hat Enterprise Linux доступна через [клиентский портал Red Hat](https://access.redhat.com/ecosystem/search/#/category/Server?sort=sortTitle%20asc&query=windows%20server&ecosystem=Red%20Hat%20Enterprise%20Linux).

## <a name="notes"></a>Примечания

1. Для этого выпуска RHEL или CentOS, виртуальной локальной сети, теги works, но коммутации виртуальной локальной сети — нет.

2. Статический IP-адрес внедрения может не работать, если диспетчер сети был настроен для заданного синтетический сетевой адаптер на виртуальной машине. Для бесперебойной работы статический IP-адрес внедрения убедитесь, что диспетчера сети, либо выключена полностью, или отключены для конкретного сетевого адаптера через свой файл ifcfg-ethX.

3. Windows Server 2012 R2 при использовании устройства виртуального адаптера fibre channel убедитесь в том, что заполнения логического номера (устройства LUN 0) 0. Если LUN 0 не заполнена, виртуальной машины Linux может не удается подключить устройствам fibre channel в собственном коде.

4. Для встроенных LIS «Hyper-v-управляющие программы» пакет должен быть установлен для использования этой функции.

5. Если во время операции резервного копирования виртуальной машины, а затем в некоторых случаях углу открытые дескрипторы файлов, сохраненный в резервной копии виртуальных жестких дисков может потребоваться пройти проверку подлинности, проверку согласованности системы (fsck) файла при восстановлении. Активных операций резервного копирования может завершиться ошибкой без уведомления, если виртуальная машина имеет вложенные iSCSI-устройству или непосредственно подключенное хранилище (также известный как транзитный диск).

6. Во время загрузки служб интеграции Linux является предпочтительной, поддержку резервного копирования для RHEL и CentOS 5.9 live - 5.11/6.4/6.5 также доступна через [основы резервного копирования Hyper-V для Linux](https://github.com/LIS/backupessentials/tree/1.0).

7. Поддержка динамической памяти доступна только на 64-разрядные виртуальные машины.

8. По умолчанию в этом распространения не включена поддержка «горячей» заменой. Чтобы включить поддержку горячей, необходимо добавить правила udev под /etc/udev/rules.d/ следующим образом:

   1. Создайте файл **/etc/udev/rules.d/100-balloon.rules**. Вы можете использовать любое нужное имя для файла.

   2. Добавьте в файл следующее содержимое: `SUBSYSTEM=="memory", ACTION=="add", ATTR{state}="online"`

   3. Перезагрузите систему, чтобы включить поддержку горячей.

   Во время загрузки служб интеграции Linux это правило создает в установке, правило также удаляется при удалении LIS, поэтому правила должны быть созданы заново динамической памяти при необходимости после удаления.

9. Операции динамической памяти может завершиться ошибкой, если гостевая операционная система работает под управлением слишком мало памяти. Ниже приведены некоторые рекомендации.

   * Равно или больше, чем объем памяти, рекомендует поставщика распространения следует память при запуске и минимальный объем памяти.

   * Приложения, которые стремятся использовать всю доступную память в системе ограничиваются использование до 80 процентов доступной оперативной памяти.

10. Если вы используете динамическую память в операционной системе Windows Server 2016 или Windows Server 2012 R2, укажите **память при запуске**, **минимальный объем памяти**, и **максимальный размер памяти** Параметры кратно 128 мегабайт (МБ). Невыполнение этого требования может привести к «горячей» заменой сбоев и увеличить в гостевой операционной системе памяти могут отсутствовать.

11. Только некоторые дистрибутивы, в том числе с помощью LIS 4.0 и 4.1, обеспечивают поддержку воздушным шаром и не поддерживают горячей. В этом случае можно использовать компонент динамической памяти, задав параметр запуска памяти в значение, равное параметру максимум памяти. Это приводит все необходимые памяти, "Горячий" добавлен на виртуальную машину во время загрузки и а позже — в зависимости от требований к памяти узла Hyper-V можно свободно выделить или памяти из гостей с помощью воздушного шага. Выполните настройку **память при запуске** и **минимальный объем памяти** равна или превышает рекомендуемое значение для распределения.

12. Чтобы включить инфраструктуры (KVP) пары ключ/значение, установите пакет rpm hypervkvpd или управляющие программы Hyper-v из ISO-образ RHEL. В качестве альтернативы можно установить пакет непосредственно из репозиториев RHEL.

13. Без обновления программного обеспечения Linux инфраструктуры (KVP) пары ключ/значение может работать неправильно. Обратитесь к поставщику распространения для получения обновления программного обеспечения, в случае, если вы видите возникшие проблемы.

14. В Windows Server 2012 R2 поколения 2 безопасная загрузка включена по умолчанию у виртуальных машин и некоторых виртуальных машин Linux не загрузится, если отключен параметр безопасной загрузки. Вы можете отключить безопасную загрузку в **встроенного по** разделе параметров для виртуальной машины в **диспетчера Hyper-V** или отключить с помощью Windows Powershell:

    ```Powershell
    Set-VMFirmware -VMName "VMname" -EnableSecureBoot Off
    ```

    Загрузки служб интеграции Linux могут применяться к существующим виртуальным машинам поколения 2, но не издавать возможности поколения 2.

15. В Red Hat Enterprise Linux или CentOS 5.2 5.3, 5.4 файловой системы замораживание возможности не доступны, поэтому Live резервного копирования виртуальных машин не доступна.
