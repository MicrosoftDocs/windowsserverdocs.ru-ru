## <a name="BKMK_notes"></a>Примечания

1. В этом выпуске Oracle Linux не поддерживает теги работает, но коммутации виртуальной локальной сети VLAN.

2. При использовании устройства виртуального адаптера fibre channel, убедитесь, что логический номер (устройства LUN 0) 0 был заполнен. Если LUN 0 не заполнена, виртуальной машины Linux может не удается подключить устройствам fibre channel в собственном коде.

3. Если во время операции резервного копирования виртуальной машины, а затем в некоторых случаях углу открытые дескрипторы файлов, сохраненный в резервной копии виртуальных жестких дисков может потребоваться пройти проверку подлинности, проверку согласованности системы (fsck) файла при восстановлении.

4. Активных операций резервного копирования может завершиться ошибкой без уведомления, если виртуальная машина имеет вложенные iSCSI-устройству или непосредственно подключенное хранилище (также известный как транзитный диск).

5. Live поддержки резервного копирования для Oracle Linux 6.4/6.5/UEKR3 QU2 и QU3 доступна через [основы резервного копирования Hyper-V для Linux](https://github.com/LIS/backupessentials/tree/1.0).

6. Поддержка динамической памяти доступна только на 64-разрядные виртуальные машины.

7. По умолчанию в этом распространения не включена поддержка «горячей» заменой. Чтобы включить поддержку горячей, необходимо добавить правила udev под /etc/udev/rules.d/ следующим образом:

   1. Создайте файл **/etc/udev/rules.d/100-balloon.rules**. Вы можете использовать любое нужное имя для файла.

   2. Добавьте в файл следующее содержимое: `SUBSYSTEM=="memory", ACTION=="add", ATTR{state}="online"`

   3. Перезагрузите систему, чтобы включить поддержку горячей.

   Во время загрузки служб интеграции Linux это правило создает в установке, правило также удаляется при удалении LIS, поэтому правила должны быть созданы заново динамической памяти при необходимости после удаления.

8. Операции динамической памяти может завершиться ошибкой, если гостевая операционная система работает под управлением слишком мало памяти. Ниже приведены некоторые рекомендации.

   * Равно или больше, чем объем памяти, рекомендует поставщика распространения следует память при запуске и минимальный объем памяти.

   * Приложения, которые стремятся использовать всю доступную память в системе ограничиваются использование до 80 процентов доступной оперативной памяти.

9. Если вы используете динамическую память в операционной системе Windows Server 2016 или Windows Server 2012 R2, укажите **память при запуске**, **минимальный объем памяти**, и **максимальный размер памяти** Параметры кратно 128 мегабайт (МБ). Невыполнение этого требования может привести к «горячей» заменой сбоев и увеличить в гостевой операционной системе памяти могут отсутствовать.

10. Некоторые дистрибутивы, в том числе с помощью LIS 3.5 или LIS 4.0 только обеспечивают поддержку воздушным шаром и не поддерживают горячей. В этом случае можно использовать компонент динамической памяти, задав параметр запуска памяти в значение, равное параметру максимум памяти. Это приводит все необходимые памяти, "Горячий" добавлен на виртуальную машину во время загрузки и а позже — в зависимости от требований к памяти узла Hyper-V можно свободно выделить или памяти из гостей с помощью воздушного шага. Выполните настройку **память при запуске** и **минимальный объем памяти** равна или превышает рекомендуемое значение для распределения.

11. Управляющие программы Oracle Linux Hyper-V не установлены по умолчанию. Чтобы использовать эти управляющие программы, установите пакет управляющие программы Hyper-v. Этот пакет конфликтует с загрузки служб интеграции Linux и не должен устанавливаться в системах с Скачанный LIS.

12. Без обновления программного обеспечения Linux инфраструктуры (KVP) пары ключ/значение может работать неправильно. Обратитесь к поставщику распространения для получения обновления программного обеспечения, в случае, если вы видите возникшие проблемы.

13. Виртуальные машины с платформ Windows Server 2012 R2Generation 2 имеют безопасная загрузка включена по умолчанию и некоторых виртуальных машин Linux не загрузится, если отключен параметр безопасной загрузки. Вы можете отключить безопасную загрузку в **встроенного по** разделе параметров для виртуальной машины в **диспетчера Hyper-V** или отключить с помощью Windows Powershell:

   ```Powershell
   Set-VMFirmware -VMName "VMname" -EnableSecureBoot Off

   ```

    Загрузки служб интеграции Linux могут применяться к существующим виртуальным машинам поколения 2, но не издавать возможности поколения 2.

14. Статический IP-адрес внедрения может не работать, если диспетчер сети был настроен для заданного синтетический сетевой адаптер на виртуальной машине. Для бесперебойной работы статический IP-адрес внедрения убедитесь, что диспетчера сети, либо выключена полностью, или отключены для конкретного сетевого адаптера через свой файл ifcfg-ethX.
