# <a name="hyper-v-virtual-machine-connection"></a>Подключение к виртуальной машине Hyper-V

>Область применения. Windows Server 2016, Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2012, Windows 8

Подключение к виртуальной машине \(VMConnect\) — это средство, которое используется для подключения к виртуальной машине, таким образом, вы можете установить или взаимодействия с гостевой операционной системы на виртуальной машине. Ниже перечислены некоторые задачи, которые можно выполнять с помощью VMConnect.  
  
-   Запуск и завершение работы виртуальной машины  
  
-   Подключение к DVD-образ \(ISO-файл\) или флэш-накопитель  
  
-   Создание контрольной точки  
  
-   Изменение параметров виртуальной машины  
    
## <a name="tips-for-using-vmconnect"></a>Советы по использованию VMConnect  
Следующие сведения могут оказаться полезными для с помощью VMConnect.  
  
|Для этого...|Выполняемые действия|  
|---------------|------------|  
|Отправить щелчок мыши или ввод с клавиатуры на виртуальную машину|Щелкните в любом месте окна виртуальной машины. Указатель мыши может выглядеть как маленькой точки, при подключении к работающей виртуальной машины.|  
|Возвращать щелчок мыши или клавиатуры на физический компьютер|Нажмите клавиши CTRL\+ALT\+СЛЕВА стрелку, а затем переместите указатель мыши за пределами окна виртуальной машины. Это сочетание клавиш мыши выпуска могут быть изменены в Hyper\-параметры Hyper V\-V Manager.|  
|Отправить CTRL\+ALT\+сочетание клавиш DELETE к виртуальной машине|Выберите **действие** > **Ctrl\+Alt\+удалить** или используйте сочетание клавиш CTRL\+ALT\+END.|  
|Переключение из режима окна полной\-полноэкранный режим|Выберите **представление** > **полноэкранный режим**. Чтобы переключиться обратно на окно, нажмите клавиши CTRL\+ALT\+ПРЕРВАТЬ.|  
|Создание контрольной точки для записи текущего состояния машины для устранения неполадок|Выберите **действие** > **контрольной точки** или используйте сочетание клавиш CTRL\+N.|  
|Изменить параметры виртуальной машины|Выберите **файл** > **параметры**.|  
|Подключение к DVD-образ \(ISO-файл\) или виртуального гибкого диска \(VFD-файла\)|Выберите **мультимедиа**.<br /><br />Виртуальные гибкие диски не поддерживаются для виртуальных машин поколения 2. Дополнительные сведения см. в разделе [следует ли создавать виртуальные машины поколения 1 или 2 в Hyper-V?](../plan/Should-I-create-a-generation-1-or-2-virtual-machine-in-Hyper-V.md).|  
|Использовать локальные ресурсы узла на Hyper\-V виртуальной машины, такие как флэш-накопитель|Включение режима расширенного сеанса на узле Hyper-V, с помощью VMConnect подключитесь к виртуальной машине и перед подключением выберите нужный локальный ресурс, который вы хотите использовать. Конкретная последовательность действий, см. в разделе [использовать локальные ресурсы на Hyper\-V виртуальной машины с вложенной виртуализацией](Use-local-resources-on-Hyper-V-virtual-machine-with-VMConnect.md).|  
|Сохранить изменение параметров VMConnect для виртуальной машины|В Windows PowerShell или командной строки, выполните следующую команду:<br /><br />`VMConnect.exe <ServerName> <VMName> \/edit`|  
|Запрет VMConnect от захвата контроля над сеансом другого пользователя VMConnect|[Включение режима расширенного сеанса на узле Hyper-V](Use-local-resources-on-Hyper-V-virtual-machine-with-VMConnect.md#BKMK_OVER).<br /><br />Не необходимости режим расширенного сеанса включено может представлять угрозу безопасности и конфиденциальности. Если пользователь подключен и вход в систему виртуальной машины с помощью VMConnect и другим авторизованным пользователям подключается к той же виртуальной машине, второй пользователь перейдет сеанса и первый пользователь потеряет подключение. Второй пользователь будет иметь возможность просматривать рабочий стол первого пользователя, документов и приложений.|
|Управление служб integration services или компоненты, которые позволяют ЕЙ обмениваться данными с узлом Hyper-V| На узлах Hyper-V под управлением Windows 10 или Windows Server 2016 нельзя управлять служб integration services с вложенной виртуализацией. См. в статьях: <br />- [Включить / отключить службы integration services на узле Hyper-V](https://msdn.microsoft.com/virtualization/hyperv_on_windows/user_guide/managing_ics) <br />- [Включить / отключить службы интеграции из виртуальной машины Windows](https://msdn.microsoft.com/virtualization/hyperv_on_windows/user_guide/managing_ics#manage-integration-services-from-guest-os-windows)<br />- [Включить / отключить службы интеграции из виртуальной машины Linux](https://msdn.microsoft.com/virtualization/hyperv_on_windows/user_guide/managing_ics#manage-integration-services-from-guest-os-linux) <br />- [Обновление служб интеграции, обновлены для виртуальной машины](https://msdn.microsoft.com/virtualization/hyperv_on_windows/user_guide/managing_ics#integration-service-maintenance)  <br />Для узлов под управлением Windows Server 2012 или Windows Server 2012 R2, см. в разделе [служб Integration Services](https://technet.microsoft.com/library/dn798297(v=ws.11).aspx).|
|Изменение размера окна VMConnect|Можно изменить размер окна VMConnect для виртуальных машин поколения 2 под управлением операционной системы Windows. Чтобы сделать это, необходимо включить режим расширенного сеанса на узле Hyper-V. Дополнительные сведения см. в разделе [включить режим расширенного сеанса на узле Hyper-V](Use-local-resources-on-Hyper-V-virtual-machine-with-VMConnect.md#BKMK_OVER). Виртуальные машины под управлением Ubuntu, см. в разделе [изменения разрешения экрана Ubuntu на виртуальной Машине Hyper-V](https://blogs.msdn.microsoft.com/virtual_pc_guy/2014/09/19/changing-ubuntu-screen-resolution-in-a-hyper-v-vm/).|


## <a name="keyboard-shortcuts"></a>Сочетания клавиш  
По умолчанию щелчков мыши и входные данные клавиатуры отправляются к виртуальной машине. Поэтому вам может потребоваться нажать CTRL + ALT + стрелка влево стрелку, прежде чем использовать следующие сочетания клавиш. 

|Сочетание клавиш|Описание|  
|-------------------|---------------|  
|CTRL\+ALT\+стрелка влево|Отпускание кнопки мыши|  
|CTRL\+ALT\+END|Эквивалент CTRL\+ALT\+удаление виртуальной машины|  
|CTRL\+ALT\+BREAK|Переключение от полного\-полноэкранный режим в оконный режим|  
|CTRL\+O|Открытие параметров виртуальной машины|  
|CTRL\+S|Запускает виртуальную машину|  
|CTRL\+N|Создание контрольной точки|  
|CTRL\+E|Вернуться к контрольной точки|  
|CTRL\+C|Сделать снимок экрана|
