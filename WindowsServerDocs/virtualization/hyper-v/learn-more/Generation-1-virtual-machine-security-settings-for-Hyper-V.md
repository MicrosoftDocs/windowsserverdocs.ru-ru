# <a name="generation-1-virtual-machine-security-settings"></a>Параметры безопасности виртуальных машин версии 1

>Область применения. Windows Server 2016, Microsoft Hyper-V Server 2016, Windows Server 2019, Microsoft Hyper-V Server 2019

Используйте параметры безопасности виртуальной машины поколения 1 в диспетчере Hyper-V для защиты данных и состояния виртуальной машины.

## <a name="encryption-support-settings-in-hyper-v-manager"></a>Параметры поддержки шифрования в диспетчере Hyper-V

Чтобы защитить данные и состояние виртуальной машины, выберите следующий параметр поддержки шифрования.

- **Шифрование трафика миграции состояния и виртуальной машины** — шифрует сохраненное состояние виртуальной машины при его запись на диск и передачу трафика динамической миграции.

Чтобы включить этот параметр, необходимо добавить диск хранилища ключей для виртуальной машины.

## <a name="key-storage-drive-in-hyper-v-manager"></a>Диск хранилища ключей в диспетчере Hyper-V

Диск хранилища ключей предоставляет виртуальным машинам небольшой диск для хранения ключа BitLocker. Это позволяет виртуальной машине шифровать диск операционной системы, не требуя использования виртуальной микросхемы доверенный платформенный модуль (TPM) (TPM). Содержимое диска хранилища ключей шифруется с помощью предохранителя ключа. Предохранитель ключа разрабатывает узел Hyper-V для запуска виртуальной машины. Как содержимое диска хранилища ключей, так и предохранитель ключа хранятся как часть состояния среды выполнения виртуальной машины.

Чтобы расшифровать содержимое диска хранилища ключей и запустить виртуальную машину, узел Hyper-V должен быть либо:

- Часть проверенной защищенной структуры для этой виртуальной машины или
- Получите закрытый ключ от одного из хранителей виртуальной машины.

Дополнительные сведения о защищенных структурах см. в разделе Введение в экранированные виртуальные машины статьи [безопасность и надежность](../../../security/Security-and-Assurance.md).

Диск хранилища ключей можно добавить в пустой слот на одном из контроллеров IDE виртуальной машины. Для этого щелкните **Добавить диск хранилища ключей** , чтобы добавить диск хранилища ключей в первый свободный слот контроллера IDE этой виртуальной машины.
