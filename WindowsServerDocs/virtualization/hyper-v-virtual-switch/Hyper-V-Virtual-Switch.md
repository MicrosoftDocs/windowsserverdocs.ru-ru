# <a name="hyper-v-virtual-switch"></a>Виртуальный коммутатор Hyper-V

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В этом разделе содержится обзор виртуального коммутатора Hyper-V, который предоставляет возможность подключения виртуальных машин \(виртуальных машин\) к сетям, которые являются внешними по отношению к Hyper\-узел V, включая интрасеть вашей организации и Интернетом. 

Можно также подключиться к виртуальным сетям на сервере, на котором выполняется Hyper\-V при развертывании программно-Конфигурируемую сеть \(SDN\).

Hyper\-виртуальный коммутатор V — это программный уровень-2 Ethernet сетевой коммутатор, доступный в Hyper\-V Manager при установке Hyper\-V роли сервера.

Виртуальный коммутатор Hyper-V включает программно управляемые и расширяемые возможности для подключения виртуальных машин к виртуальным сетям и физической сети. Виртуальный коммутатор Hyper-V также обеспечивает применение политик в области безопасности, изоляции и уровней обслуживания.  
  
> [!NOTE]  
> Виртуальный коммутатор Hyper-V поддерживает только Ethernet и не поддерживает другие технологии проводной сети, такие как Infiniband и Fibre Channel.  
  
Виртуальный коммутатор Hyper-V включает в себя возможности изоляции клиента, управления скоростью обработки трафика, защиты от вредоносных виртуальных машин и простого устранения неполадок. 

Со встроенной поддержкой спецификации интерфейса сетевого устройства \(NDIS\) фильтровать драйверы и платформа фильтрации Windows \(WFP\) драйверов внешнего вызова виртуального коммутатора Hyper-V позволяет независимым программного обеспечения Поставщики \(независимые поставщики программного обеспечения\) чтобы создавать расширяемые подключаемые модули, вызывается расширения виртуального коммутатора, который может предоставить улучшенными возможностями сетевые подключения и безопасность. Добавленные расширения виртуального коммутатора отображаются в компоненте "Диспетчер виртуальных коммутаторов" в составе диспетчера Hyper-V.
  
На следующем рисунке на виртуальной Машине установлен виртуальный сетевой Адаптер, подключенный к Hyper-V Virtual Switch через порт коммутатора.  
  
![Подключения виртуальный коммутатор](../media/Hyper-V-Virtual-Switch/Vswitch_01.jpg)  
  
Возможности виртуального коммутатора Hyper-V дают дополнительные возможности для изоляции клиентов, управлению скоростью обработки сетевого трафика и принятию защитных мер против вредоносных виртуальных машин.

>[!NOTE]
> В Windows Server 2016 виртуальную Машину с помощью виртуального сетевого Адаптера точно отображает максимальную пропускную способность для виртуального сетевого адаптера. Для просмотра виртуального скорость сетевого Адаптера в **сетевых подключений**, щелкните правой кнопкой мыши нужный значок виртуального сетевого Адаптера и нажмите кнопку **состояние**. Виртуальный сетевой Адаптер **состояние** откроется диалоговое окно. В **подключения**, значение **скорость** соответствующую физический сетевой Адаптер, установленного на сервере.
  
## <a name="bkmk_apps"></a>Использует для виртуального коммутатора Hyper-V

Ниже приведены некоторые сценарии вариантов использования для виртуального коммутатора Hyper-V.

**Отображение статистики**: Разработчик в облачной среде создает реализует пакет управления, который отображает текущее состояние виртуального коммутатора Hyper-V. Пакет управления может запрашивать текущие возможности коммутатора, параметры конфигурации и сетевую статистику по отдельным портам при помощи инструментария WMI. После этого для администраторов отображается состояние коммутатора.  
  
**Отслеживание ресурсов**: Хостинг-компания продает служб размещения, стоимость которых зависит от уровня участия. Каждый уровень участия имеет свои характеристики сети. Администратор выделяет ресурсы таким образом, чтобы одновременно обеспечивать соответствие соглашениям об уровне обслуживания и доступность сети. Администратор программным способом отслеживает информацию, например текущее использование выделенной полосы пропускания и номер виртуальной машины (VM) назначены очередь виртуальной машины (VMQ) или каналы IOV. Та же программа периодически записывает в журнал используемые ресурсы, а также ресурсы, выделенные для отслеживания двойных записей или ресурсов на каждой виртуальной машине.  
  
**Управление порядком расширений коммутатора**: Предприятие установило расширения на своем узле Hyper-V для мониторинга трафика и обнаружения вторжений. Во время технического обслуживания некоторые расширения могут быть обновлены, что приведет к изменению их порядка. Выполняется простой сценарий для переупорядочения расширений после обновления.  
  
**Расширение переадресации управляет VLAN ID**: Крупная компания коммутации создает расширение переадресации, которое применяет все политики для работы в сети. Один из управляемых элементов — идентификаторы виртуальной локальной сети (VLAN). Виртуальный коммутатор передает управление виртуальной локальной сетью расширению переадресации. Установка компании программным способом вызывает это инструментария управления Windows (WMI) прикладной программный интерфейс (API), включает прозрачность, предписывая Hyper-V виртуальному коммутатору для передачи и теги VLAN не выполнять никаких действий.  
  
## <a name="bkmk_func"></a>Функциональные возможности виртуального коммутатора Hyper-V
 
Виртуальный коммутатор Hyper-V включает следующие важные компоненты.  
  
-   **Защита ARP/ND Poisoning (спуфинга)**: Обеспечивает защиту от вредоносных виртуальных Машин, использующих протокол (ARP) спуфинг для кражи IP-адресов с других виртуальных машин. Защищает от атак узлов IPv6 путем спуфинга протокола обнаружения соседей (ND).  
  
-   **Защита DHCP**: Обеспечивает защиту от вредоносных виртуальных Машин, выдающих себя как сервер конфигурации протокола DHCP (Dynamic Host) man-in--middle атак.  
  
-   **Списки ACL для портов**: Предоставляет фильтрацию трафика на основе управления доступом мультимедиа (MAC) или протокола Интернета (IP) адресов или диапазонов, что дает возможность настроить изоляцию виртуальной сети.  
  
-   **Режим магистрали для виртуальной Машины**: Позволяет администраторам настроить определенную виртуальную Машину в качестве виртуального устройства, а затем направлять трафик из разных виртуальных локальных сетей на эту виртуальную Машину.  
  
-   **Мониторинг сетевого трафика**: Позволяет администраторам анализировать трафик, проходящий через сетевой коммутатор.  
  
-   **Изолированная (частная) виртуальная локальная сеть**: Позволяет администраторам разделять трафик из нескольких виртуальных локальных сетей, чтобы более легко Создание изолированных сообществ клиентов.  
  
Ниже приведен перечень возможностей, которые улучшают использование виртуального коммутатора Hyper-V.  
  
-   **Ограничение полосы пропускания и поддержка пакетного режима**: Минимум полосы пропускания — это гарантированный резерв. Максимум полосы пропускания — это объем полосы пропускания, который может использовать виртуальная машина.  
  
-   **Explicit Congestion Notification (ECN) поддержка маркировки**:  ECN пометки, также называемая Data CenterTCP (DCTCP), позволяет физическому коммутатору и операционной системе управлять потоком трафика, буферных ресурсов коммутатора не являются избыточными, пропускную способность сети большего объема трафика.  
  
-   **Диагностика**: Диагностика обеспечивает простое отслеживание и мониторинг событий и пакетов, передаваемых через виртуальный коммутатор.
