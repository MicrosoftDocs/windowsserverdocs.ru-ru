# <a name="group-managed-service-accounts-overview"></a>Group Managed Service Accounts Overview

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В этом разделе для ИТ-специалистов представлены групповые управляемые учетные записи служб описаны варианты практического применения изменений в реализации корпорации Майкрософт и требования к оборудованию и программному обеспечению.


## <a name="BKMK_OVER"></a>Описание компонента
Автономной управляемой учетной записи службы (sMSA) является управляемой учетной записью домена, обеспечивающий автоматическое управление паролями, упрощенная служба управления имя участника (SPN) и возможность делегирования управления другим администраторам. Этот тип учетной записи управляемой службы (MSA) был введен в Windows Server 2008 R2 и Windows 7.

Группа управляемую учетную запись службы (gMSA) предоставляет те же функциональные возможности в пределах домена, но также расширяет функциональность на несколько серверов. При подключении к службе, размещенной на ферме серверов, таких как решения кластера балансировки сетевой нагрузки, для протоколов взаимной проверки подлинности требует, что все экземпляры службы используют тот же субъект. При использовании управляемой как субъекты-службы, операционная система Windows управляет пароль для учетной записи, а не полагаться на администратора для управления паролями.

Служба распространения ключей Майкрософт \(kdssvc.dll\) предоставляет механизм для безопасного получения последнего ключа или определенного ключа с помощью идентификатора ключа для учетной записи Active Directory. Служба распространения ключей использует общий секрет для создания ключей учетной записи. Эти ключи периодически изменяются. Для управляемой контроллер домена вычисляет пароль для ключа, предоставленного службами распространения ключей, дополнение к прочим атрибутам эта учетная запись.  Участвующие узлы могут получить текущий и предыдущий пароль значения, обратившись к контроллеру домена.

## <a name="BKMK_APP"></a>Практическое применение
gMSAs предоставляют решение единого удостоверения для служб, выполняемых на ферме серверов или в системах с подсистемой балансировки нагрузки на сеть. Предоставляя решение групповой управляемой учетной записи, службы могут быть настроены для новой групповой управляемой учетной записи субъекта, а управление паролями обрабатывается Windows.

С помощью управляемой, службам или администраторам служб не нужно синхронизировать пароли между экземплярами служб. Эта учетная запись поддерживает узлы, которые хранятся вне сети в течение продолжительного периода времени, а также управление участвующими узлами для всех экземпляров службы. Это означает, что можно развертывать ферму серверов, поддерживающую единое удостоверение, по которому существующие клиентские компьютеры могут проверять подлинность без идентификации экземпляра службы, к которому они подключаются.

Отказоустойчивые кластеры не поддерживают групповые управляемые учетные записи служб. При этом групповую или автономную учетную запись службы могут использовать службы, работающие поверх службы кластеров и представляющие собой службу Windows, пул приложений или назначенную задачу либо поддерживающие такие учетные записи изначально.

## <a name="BKMK_SOFT"></a>Требования к программному обеспечению

64\-для выполнения команд Windows PowerShell, которые используются для администрирования gMSAs необходима разрядная архитектура.

Управляемая учетная запись службы зависит от поддерживаемых протоколом Kerberos типов шифрования. Если проверка подлинности клиентского компьютера на сервере выполняется с помощью Kerberos, контроллер домена создает билет службы Kerberos, защищенный шифром, который поддерживают и контроллер, и сервер. Контроллер домена использует учетную запись msDS\-SupportedEncryptionTypes атрибут, чтобы определить, какие типы шифрования поддерживает сервер, и, если не существует атрибута, предполагается, клиентский компьютер не поддерживает более надежные типы шифрования. Если настройка узла не поддерживает RC4, проверка подлинности всегда завершится ошибкой. Поэтому стандарт AES всегда должен быть явным образом настроен для поддержки MSA.

> [!NOTE]
> Начиная с версии Windows Server 2008 R2, стандарт DES отключен по умолчанию. Дополнительные сведения о поддерживаемых типах шифрования см. в статье [Изменения в проверке подлинности Kerberos](https://technet.microsoft.com/library/dd560670(WS.10).aspx).

gMSAs не применимы к операционным системам Windows до Windows Server 2012.

## <a name="server-manager-information"></a>Сведения о диспетчере сервера
