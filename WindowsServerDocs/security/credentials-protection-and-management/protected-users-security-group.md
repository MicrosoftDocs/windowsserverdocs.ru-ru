# <a name="protected-users-security-group"></a>Защищенная группа безопасности пользователей

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом разделе для ИТ-специалистов описывается группа безопасности Active Directory защищенные пользователи и объясняются принципы ее работы. Эта группа появилась в Windows Server 2012 R2 контроллеры домена.

## <a name="BKMK_ProtectedUsers"></a>Обзор

Эта группа безопасности предназначен как часть стратегии для управления раскрытия учетных данных на предприятии. Члены этой группы имеют автоматически применяются к учетным записям ненастраиваемые функции защиты. Членство в группе защищенных пользователей предназначен для ограничения и обеспечивает профилактическую защиту по умолчанию. Единственный способ изменить вышеуказанные функции защиты для учетной записи необходимо удалить учетную запись из группы безопасности.

> [!WARNING]
> Учетные записи служб и компьютеров не должны быть членами группы защищенных пользователей. Эта группа будет в любом случае обеспечивает защиту была завершена, так как пароль или сертификат всегда доступен на узле. Проверка подлинности завершится с именем ошибка \"the пользователя или пароль — incorrect\» для любой службы или компьютера, который добавляется в группу защищенных пользователей.

Этот связанная с доменом глобальная группа предусматривает ненастраиваемую защиту для устройств и главных компьютеров под управлением Windows Server 2012 R2 и Windows 8.1 или более поздней версии для пользователей в доменах с основного контроллера домена под управлением Windows Server 2012 R2. Это существенно сокращает объем памяти по умолчанию учетные данные, когда пользователи для входа на компьютеры с эти средства защиты.

Дополнительные сведения см. в разделе [как защищенные группы "пользователи" работает](#BKMK_HowItWorks) в этом разделе.



## <a name="BKMK_Requirements"></a>Требования группы защищенных пользователей
Действуют следующие требования для предоставления устройства меры защиты для членов группы защищенных пользователей.

- Защищенные пользователи глобальной группы безопасности реплицируются на все контроллеры домена в домене учетной записи.

- Windows 8.1 и Windows Server 2012 R2 была добавлена поддержка по умолчанию. [Советы по безопасности Майкрософт 2871997](https://technet.microsoft.com/library/security/2871997) добавляет поддержку Windows 7, Windows Server 2008 R2 и Windows Server 2012.

Ниже перечислены требования к защите контроллеров домена для членов группы защищенных пользователей.

- Пользователи должны находиться в доменах Windows Server 2012 R2 или высокий функциональный уровень домена.

### <a name="adding-protected-user-global-security-group-to-down-level-domains"></a>Добавление защищенных пользователей глобальной группы безопасности к доменам нижнего уровня

Контроллеры домена под управлением операционной системы более ранних, чем Windows Server 2012 R2 можно поддерживает добавление элементов в новую группу безопасности защищенных пользователей. Это позволяет пользователям воспользоваться преимуществами функции защиты устройства до обновления домена. 

> [!Note]
> Контроллеры домена не будет поддерживать функции защиты домена. 

Можно создать группу защищенных пользователей [передачи роли эмулятора основного домена (PDC) контроллеру](https://technet.microsoft.com/library/cc816944(v=ws.10).aspx) до уровня контроллера домена под управлением Windows Server 2012 R2. После этого объект группы реплицируется на другие контроллеры домена, роль эмулятора основного контроллера домена может размещаться на контроллере домена под управлением более ранней версии Windows Server.

## <a name="BKMK_HowItWorks"></a>Принцип работы группы защищенных пользователей
В этом разделе описывается принцип работы группы защищенных пользователей, когда:

-   вход устройств с Windows

-   Домен учетной записи пользователя находится в Windows Server 2012 R2 или высокий функциональный уровень домена

### <a name="device-protections-for-signed-in-protected-users"></a>Защиту устройства регистрируются в защищенных пользователей
Если пользователь входит в группу защищенных пользователей применяются следующие меры защиты:

-   Учетных данных будет делегирование (CredSSP) не кэша обычного текста учетные данные пользователя, даже когда **разрешить передачу учетных данных по умолчанию** включить параметр групповой политики.

-   Начиная с Windows 8.1 и Windows Server 2012 R2, дайджест Windows не кэширует учетные данные пользователя обычный текст даже в том случае, когда включен дайджест Windows.

> [!Note]
> После установки [советы по безопасности Майкрософт 2871997](https://technet.microsoft.com/library/security/2871997) дайджест Windows продолжит кэширование учетных данных, пока не будет настроен раздел реестра. В разделе [советы по безопасности Майкрософт: обновления для улучшения Защита учетных данных и управления: 13 мая 2014](https://support.microsoft.com/en-us/help/2871997/microsoft-security-advisory-update-to-improve-credentials-protection-a) инструкции.

-   NTLM не кэширует учетные данные пользователя обычный текст или необратимой функции NT (NTOWF).

-   Kerberos больше не будет создавать ключи DES или RC4. Также он не кэширует учетные данные в виде обычного текста или долговременных ключей пользователя после начальной TGT получается.

-   Кэшированного средства проверки не создается при входе в систему или разблокирования, поэтому входа в больше не поддерживается.

После добавления учетной записи пользователя в группу защищенных пользователей защита начнется при его входе в систему на устройстве.

### <a name="domain-controller-protections-for-protected-users"></a>Средства защиты контроллера домена для защищенных пользователей
Учетные записи, которые являются членами группы защищенных пользователей, проверку подлинности в домене Windows Server 2012 R2 не смогут:

-   Проверка подлинности с помощью проверки подлинности NTLM.

-   Используйте тип шифрования DES и RC4 в предварительной проверке подлинности Kerberos.

-   Делегироваться в рамках неограниченного или ограниченного делегирования.

-   Обновите TGT Kerberos свыше исходного четырехчасового срока действия.

Ненастраиваемые параметры истечения срока действия TGT задаются для каждой учетной записи в группу защищенных пользователей. Обычно контроллер домена устанавливает время жизни TGT и обновления, основываясь на политиках домена **максимальный срок жизни билета пользователя** и **максимальный срок жизни для возобновления билета пользователя**. Для группы защищенных пользователей для этих политик домена задается значение 600 минут.

Дополнительные сведения см. в разделе [Настройка защищенных учетных записей](how-to-configure-protected-accounts.md).

## <a name="troubleshooting"></a>Устранение неполадок
Для диагностики неполадок событий, связанных с защищенными пользователями, доступны два операционных административных журнала. Эти новые журналы расположены в средстве просмотра событий по умолчанию отключены и находятся в папке **приложений и служб\microsoft\windows\microsoft\проверка**.

|Идентификатор события и журнал|Описание|
|----------|--------|
|104<br /><br />**ProtectedUser клиента**|Причина: Пакет безопасности на клиенте не содержит учетных данных.<br /><br />Эта ошибка регистрируется в журнал клиентского компьютера, если учетная запись является членом группы безопасности "защищенные пользователи". Это событие означает, что пакет безопасности не кэширует учетные данные, которые необходимы для проверки подлинности на сервере.<br /><br />Отображает имя пакета, имя пользователя, имя домена и имя сервера.|
|304<br /><br />**ProtectedUser клиента**|Причина: Пакет безопасности не учетных данных защищенного пользователя.<br /><br />Информационное событие регистрируется в клиента, указывая, что пакет безопасности не кэширует пользователя учетных данных для входа. Ожидается, что дайджест (WDigest), делегирование учетных данных (CredSSP) и NTLM, не могут иметь учетные данные для входа для защищенных пользователей. Приложения могут выводить запрос на ввод учетных данных.<br /><br />Отображает имя пакета, имя пользователя и имя домена.|
|100<br /><br />**ProtectedUserFailures-DomainController.**|Причина:, Возникает ошибка входа NTLM для учетной записи, которая входит в группу безопасности защищенных пользователей.<br /><br />Ошибка регистрируется в журнале контроллера домена, чтобы указать, что проверка подлинности NTLM не удалось, так как учетная запись является членом группы безопасности "защищенные пользователи".<br /><br />Отображает имя учетной записи и имя устройства.|
|104<br /><br />**ProtectedUserFailures-DomainController.**|Причина: Типы шифрования DES и RC4 используются для проверки подлинности Kerberos, и возникает ошибка входа пользователя в группу защищенных пользователей.<br /><br />Ошибка предварительной проверки подлинности Kerberos, поскольку, если учетная запись является членом группы безопасности "защищенные пользователи" не удается использовать типы шифрования DES и RC4.<br /><br />(AES допускается.)|
|303<br /><br />**ProtectedUserSuccesses-DomainController.**|Причина: Kerberos предоставления билета (TGT) успешно выдан для члена группы защищенных пользователей.|
