## <a name="gdpr-and-its-implications"></a>Его влияние и GDPR
GDPR является сложной нормативных требований, которые могут потребовать значительных изменений в сбор, использование и управление персональные данные. Корпорация Майкрософт уже давно помочь нашим клиентам нормам сложных и речь идет о подготовке к GDPR, мы при партнером на работе.

GDPR применяет правила для организаций, которые предлагают товары и служб для пользователей в Европейского Союза (ЕС), или, сбор и анализ данных, привязанных к жителей ЕС, независимо от того, где расположены этих предприятий. Основные элементы GDPR являются следующие:

- **Улучшенные права на конфиденциальность личных.** Убедитесь, что они имеют право на доступ к личным данным, чтобы правильно неточности в эти данные для стирания данных, объект для обработки их личных данных и для его перемещения усиления защиты данных для лиц, проживающих ЕС.

- **Повышенная пошлины для защиты личных данных.** Усиленная учета организаций, которые обрабатывают персональные данные, обеспечивая повышенную четкость ответственность за обеспечение соответствия.

- **Отчеты для связи с нарушением обязательные персональные данные.** Организаций, которые управляют персональные данные необходимы для нарушений персональные данные отчетов, которые представлять риск для прав и freedoms лиц их контролирующий органы без вызывали задержки и, где возможно, не более чем на один раз 72 часа они получают сведения о нарушении.

Как вы предполагаете, GDPR может оказать значительное влияние на бизнес, потенциально требуется обновить политики конфиденциальности, реализации и Усиление защиты элементы управления данными и нарушения процедуры уведомления, развертывание высокой прозрачный политик и дальнейшего инвестиций в ИТ и обучения. Microsoft Windows 10 может эффективно помочь и эффективно решить некоторые из этих требований.

## <a name="personal-and-sensitive-data"></a>Личных и конфиденциальных данных
В рамках усилий, чтобы обеспечить соответствие требованиям GDPR необходимо понять, как правила определяет личных и конфиденциальных данных и как связаны эти определения данных, находящихся в вашей организации. На основании что факт, что вы сможете обнаружить, где создается данных, обработанных в управлении и хранится.

GDPR рассматривает все сведения, связанные с указанных или личные физического лица личных данных. Который может включать прямой идентификации (например, имя юридические) и косвенные идентификации (такие как конкретная информация, которую делает снимите — это вы ссылки на данные). GDPR также понятно, что понятие персональные данные включает идентификаторы сети (например, IP-адреса, идентификаторы мобильных устройств) и данные о местоположении.

GDPR представляет конкретного определения для генетическую данных (например, последовательности основные физического лица) и биометрических данных. Генетического и биометрические данные, наряду с другими sub категории персональные данные (персональные данные, отображая расовых или этнического происхождения, политических мнения, религиозной или философским положения или объединение коммерческих членства: данные о работоспособности; или данные о Пол жизни человека или сексуальной ориентации) рассматриваются как секретные личные данные в разделе GDPR. Секретные личные данные предоставляемых улучшенную защиту и обычно требует явного согласия частное лицо, где эти данные должны обрабатываться.

Защита потока управления предотвращает атаки JOP, размещение плотной ограничения на какой код приложения может быть выполняемый — особенно косвенный вызов инструкции. Он добавляет проверки для определения набора функций в приложении, допустимые целевых объектов для косвенных вызовов упрощенных безопасности. При запуске приложения, он проверяет правильность целевых непрямой вызов.

Если проверка Защита потока управления во время выполнения, Windows Server 2016 немедленно завершается программы, последние любым способом, пытается косвенно вызвать недопустимый адрес. При компрометации приложения, перечисленные в белый его будет запускаться этот флажок снят, Device Guard, поскольку Device Guard, блокировки увидят, что приложение имеет цифровую подпись и считается доверенным.

Но так как защита потока управления можно определить, является ли приложение выполняется в порядке, предварительно не определены, неработоспособные, атаки завершится ошибкой, препятствующие запуску скомпрометированных приложения. Вместе эти средства защиты затруднить очень злоумышленникам внедрить вредоносные программы в программного обеспечения, работающими под управлением Windows Server 2016.

Разработчики приложений, где будет обрабатываться персональные данные, рекомендуется включить потока управления (CFG) в своих приложениях. Кроме того CFG детально включить работает в «CFG учитывающие» версиях Windows и поэтому полностью совместим с ними.

#### <a name="windows-defender-antivirus"></a>Антивирусная программа для Защитника Windows
Windows Server 2016 включает отрасли начальные, активно возможности обнаружения Защитника Windows для блокировки известных вредоносных программ. Антивирусная программа Windows Defender (AV) используется совместно с Device Guard Защитника Windows и защита потока управления, чтобы предотвратить установку на серверах вредоносный код какой-либо. Она включена по умолчанию — администратор не требуется никаких дополнительных действий для того, чтобы начать работу. AV Защитник Windows также оптимизирован для поддержки различных ролей сервера в Windows Server 2016. В прошлом злоумышленники использовать оболочках, например PowerShell для запуска вредоносных двоичного кода. В Windows Server 2016 PowerShell теперь интегрирован с AV Защитника Windows на наличие вредоносных программ перед запуском кода.

AV Защитник Windows — это решение встроенной защиты от вредоносных программ, обеспечивающий безопасности и защиты от вредоносных программ управления настольные компьютеры, портативные компьютеры и серверы. AV Защитник Windows существенно улучшена с момента его появления в Windows 8. Антивирусная программа Защитника Windows в Windows Server использует многоуровневые алгоритмы для улучшения защиты от вредоносных программ:

- **Доставлено облачной защиты** помогает обнаруживать и блокировать новые вредоносные программы за считанные секунды, даже если вредоносная программа никогда не были раньше.

- **Локальный Контекст разнообразные** улучшает как идентифицируется вредоносных программ. Windows Server информирует AV Защитник Windows не только о содержимом файлах и процессах, но и источнике этого содержимого, где он был сохранен и многое другое. 

- **Всеобъемлющим глобальным датчикам** обеспечивать AV Защитника Windows, текущий и учитывать даже новых вредоносных программ. Это достигается двумя способами: по сбору данных об объемном локальном контексте с конечных точек и централизованным анализом этих данных.

- **Взлома проверки правописания** помогает защитить Защитник Windows AV, сам от атак вредоносных программ. Например AV Защитник Windows использует защищенные процессы, не дает недоверенным процессам при попытке незаконно компоненты AV Защитника Windows, ее ключи реестра и т. д.

- **Корпоративные функции** предоставить ИТ-специалистов, средства и параметры конфигурации, необходимо сделать AV Защитник Windows антивредоносного решения корпоративного уровня.

#### <a name="enhanced-security-auditing"></a>Аудит безопасности 
Windows Server 2016 активно предупреждая администраторов о потенциальных попытки связи с нарушением с усиленной безопасности аудита, который предоставляет более подробные сведения, которые можно использовать для быстрее атаки обнаружения и криминалистического анализа. Он регистрирует события из Защита потока управления, Device Guard Защитника Windows и других функций безопасности в одном месте, что облегчает администраторам определять системы могут быть под угрозой.

Новые категории событий включают:

- **Аудит активности PnP.** Позволяет отслеживать обнаружение внешнего устройства, которые могут содержать вредоносные программы plug and play. События PnP можно использовать для отслеживания изменений в системном оборудовании. Список поставщиков оборудования идентификаторы, включенных в событии.

Windows Server 2016 интегрирует легко с помощью системы управления (SIEM) инцидента событий безопасности, например Microsoft Operations Management Suite (OMS), который могут быть включены в отчеты аналитики о возможных нарушений. Глубина информация, предоставленная расширенного аудита позволяет группы безопасности определить и реагировать на потенциальные нарушений быстро и эффективно.

### <a name="secure-virtualization"></a>Безопасная виртуализация
Сегодня предприятиям виртуализации все, что можно, из SQL Server для SharePoint на контроллеры домена Active Directory. Виртуальные машины (VM) просто упростить развертывание и управление ими, службы автоматизации инфраструктуры. Но если речь идет о безопасности, структура скомпрометированных виртуализации стали новый вектором атаки, что сложно защититься — до данного момента. С точки зрения GDPR следует подумать о защите виртуальных машин, что бы Защита физических серверов, включая использование технологии TPM виртуальной Машины.

Windows Server 2016 фундаментально изменяет уровень как предприятий безопасной виртуализации, в том числе несколько технологий, которые позволяют создавать виртуальные машины, которые будут выполняться только на свои собственные структуры; обеспечивает защиту от хранилища, сети и узла устройств, которые выполняются на.

#### <a name="shielded-virtual-machines"></a>Экранированные виртуальные машины
То же, что делает виртуальных машин так легко перенести, резервное копирование и репликацию, также их легче изменить и скопировать. Виртуальная машина является просто файл, поэтому он не защищен в сети, в хранилище, в резервных копий, или в другом месте. Еще одна проблема является то, что администраторов структуры, являются ли они администратор хранилища или администратор сети — доступ к виртуальным машинам.

Скомпрометированных администратора в структуре легко может привести к компрометации данных между виртуальными машинами. Все, что необходимо сделать злоумышленник — использовать скомпрометированных учетных данных для копирования файлов любой виртуальной Машины, как на USB-накопитель и пройти его из организации, где эти файлы виртуальных Машин может осуществляться из другой системы. Если какой-либо из этих украденных виртуальных машин с контроллером домена Active Directory, например, злоумышленник может легко просмотреть содержимое и использовать рукой подбора способы взлома паролей в базе данных Active Directory, в конечном счете предоставления им доступа все остальные элементы в пределах инфраструктуры.

Windows Server 2016 предоставляет экранированные виртуальные машины (экранированных виртуальных машин) для защиты от сценариев описанное из них. Экранированные виртуальные машины включают виртуальное устройство TPM позволяет организациям применить шифрование BitLocker на виртуальные машины и убедитесь, что они выполняются только на доверенных узлах для защиты от скомпрометированного хранилища, сети и администраторов узла. Экранированные виртуальные машины создаются с помощью виртуальные машины поколения 2, который поддерживают единый интерфейс (UEFI) встроенного по и виртуальным доверенным платформенным Модулем.

#### <a name="host-guardian-service"></a>Служба защиты узла
Вместе с экранированных виртуальных машин службе защиты узла является это необходимый компонент для создания структуры безопасной виртуализации. Его является аттестацию на работоспособность узла Hyper-V, прежде чем он позволит экранированной виртуальной Машины на загрузку или перейти на этом узле. Он содержит разделы для экранированных виртуальных машин и не будет освобождать их пока не может быть уверен, индекса безопасности. Существует два способа, можно потребовать узлов Hyper-V для проверки работоспособности для службы защиты узла.

Первой и наиболее безопасным является Аттестация по группе доверенного оборудования. Это решение требует запуск экранированных виртуальных машин на узлах, которые имеют микросхемы TPM 2.0 и UEFI 2.3.1. Для поддержки измеряемой загрузки требуется это оборудование и сведения о целостности ядра операционной системы необходимо службой защиты узла для узла Hyper-V не были незаконно.

ИТ-организации имеют использование Аттестация с доверием администратора, который может быть удобно, если оборудования TPM 2.0 не используется в вашей организации. Эта модель аттестации отличается простотой в развертывании, так как узлы просто помещаются в группу безопасности и служба защиты узла разрешено экранированные виртуальные машины работают на узлах, которые являются членами группы безопасности. С помощью этого метода существует измерения отсутствуют сложные, чтобы убедиться, что хост-компьютера не было изменено. Тем не менее исключить возможность незашифрованные диски дверцу USB начинаете виртуальные машины или запуска виртуальной Машины на несанкционированного узла. Это так, как файлы виртуальной Машины не будут запускаться на любом компьютере, кроме входящих в указанной группе. Если вы еще не имеет оборудования TPM 2.0, можно начать с Аттестация с доверием администратора и переключитесь на аттестацию по ключу доверенного оборудования, при обновлении оборудования.

#### <a name="virtual-machine-trusted-platform-module"></a>Виртуальная машина доверенный платформенный модуль
Windows Server 2016 поддерживает платформенный модуль для виртуальных машин, который позволяет поддерживает технологии повышенной безопасности, такие как шифрование диска BitLocker® в виртуальных машинах. Виртуальный доверенный платформенный модуль (vTPM) можно защитить с помощью локального криптографические ключи хранятся на узле или в службе защиты узла. Таким образом хотя служба защиты узла требует большего объема инфраструктуры, он также обеспечивает дополнительную защиту.

#### <a name="distributed-network-firewall-using-software-defined-networking"></a>Брандмауэр распределенной сети, с помощью программно определяемых сетей
Для сегментации сети в виртуальных машинах обмениваться данными только с определенными системами, необходимые для работы таким образом, является одним из способов улучшить защиту в виртуализированных средах. Например если приложение не требуется подключение к Интернету, можно разделить его отключали, устраняя этих систем целевые от внешних атак. Программно определяемая сеть (SDN) в Windows Server 2016 включает брандмауэр распределенной сети, который дает возможность динамического создания политик безопасности, которые можно защитить приложения от атак из внутри или вне сети. Брандмауэр распределенной сети добавляет уровни безопасности, позволяя вам для изоляции приложений в сети. Политики могут применяться в любом масштабе вашей инфраструктуре виртуальной сети, изоляции VM-to-VM трафика, трафика VM-to-host или VM-to-Internet трафика, при необходимости — для отдельных систем, был скомпрометирован или программными средствами в нескольких подсетях. Программно определяемого сетевыми возможностями Windows Server 2016 также позволяют маршрут или зеркального входящего трафика для виртуальных устройств сторонних разработчиков. Это позволяет легко слой в дополнительную безопасность локально или в облаке.

- **Обнаружение, которое не удается обнаружить.** Датчики, встроенные в ядро операционной системы, эксперты по вопросам безопасности Windows и уникальный оптика более 1 миллиарда машин и сигналы во всех службах Microsoft.

- **Встроенная, не болтовое на.** Без агента с высокой производительностью и минимальное влияние, облачных мощностей; упрощенное управление с помощью без развертывания. 

- **Унифицированного для обеспечения безопасности Windows.** Изучите за последние 6 месяцев функционально насыщенных и временной шкале объединении события безопасности из ATP в Защитнике Windows, антивирусной программы Защитником Windows и Device Guard Защитника Windows.

- **Питание Microsoft graph.** Использует график безопасности аналитики Майкрософт для интеграции с Office 365 ATP подписки, чтобы обратно отслеживать и реагировать на атаки обнаружения и исследования.

Дополнительные сведения в [новые возможности обновления Creators ATP в Защитнике Windows preview](https://blogs.microsoft.com/microsoftsecure/2017/03/13/whats-new-in-the-windows-defender-atp-creators-update-preview/).

ATA — локальный продукт, позволяющий обнаружения нарушений безопасности удостоверений в организации. ATA можно записать и проанализировать сетевой трафик для проверки подлинности, авторизации и сведения, сбор протоколы (например, Kerberos, DNS, RPC, NTLM и другие протоколы). ATA использует эти данные для создания поведения профиля о пользователях и других сущностей в сети, чтобы он мог определить аномалий и шаблоны известных атак. В следующей таблице перечислены типы атак, обнаруженных ATA.


|Тип атаки |Описание |
|---------|---------|
|Вредоносные атаки |Эти атаки определяются поиск атак, в списке известных типов атак, включая:<ul><li>PASS--Ticket (доверенной)</li><li>PASS--Hash (PtH)</li><li>Overpass хэша</li><li>Неправильным PAC (MS14-068)</li><li>Золотой билета</li><li>Вредоносный репликациями</li><li>Reconnaissance</li><li>Подбора</li><li>Удаленное выполнение</li></ul>Полный список вредоносных атак, которые могут быть обнаружены и их описание см. в разделе [что подозрительных действий может обнаружить ATA?](https://docs.microsoft.com/en-us/advanced-threat-analytics/understand-explore/ata-threats).|
|Чрезмерное поведение |Эти атаки распознаются средствами анализа поведения и используйте машинного обучения для идентификации сомнительные действий, включая:<ul><li>Аномальное попыток входа</li><li>Неизвестных угроз</li><li>Совместное использование пароля</li><li>Бокового смещения</li></ul>|
|Проблемы безопасности и рисков |Эти атаки обнаруживаются, просмотрев текущей сети и конфигурации системы, включая:<ul><li>Неработающие доверия</li><li>Слабый протоколы</li><li>Протокол известных уязвимостей</li></ul>|

ATA можно использовать для обнаружения злоумышленников, пытающихся для компрометации привилегированного удостоверений. Дополнительные сведения о развертывании ATA см. в разделах Планирование, проектирование и развертывание в [документации Advanced Threat Analytics](https://docs.microsoft.com/en-us/advanced-threat-analytics/).
