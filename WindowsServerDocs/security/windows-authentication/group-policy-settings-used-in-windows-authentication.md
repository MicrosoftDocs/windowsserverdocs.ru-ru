# <a name="group-policy-settings-used-in-windows-authentication"></a>Параметры групповой политики в проверке подлинности Windows

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В этом справочном разделе для ИТ-специалистов описывается использование и влияние параметров групповой политики в процессе проверки подлинности.

Вы можете управлять проверки подлинности в операционных системах Windows, путем добавления в группы пользователей, компьютеров и учетных записей служб, а затем, применяя политики проверки подлинности в эти группы. Эти политики определяются как локальные политики безопасности и как административные шаблоны, также известный как параметры групповой политики. Оба набора можно настраивать и распространять своей организации с помощью групповой политики.

> [!NOTE]
> Функций, появившихся в Windows Server 2012 R2 позволяют настраивать политики проверки подлинности для целевой службы или приложения, которые обычно называются приемники команд проверки подлинности, с помощью защиты учетных записей. Сведения о том, как это сделать в Active Directory, см. в разделе [How to Configure Protected Accounts](how-to-configure-protected-accounts.md).

Например следующие политики можно применить к группам, в зависимости от их функции в организации.

-   Войдите в систему локально или в домен

-   Войдите в сети

-   Сброс учетных записей

-   Создание учетных записей

В следующей таблице перечислены группы политики, связанные с проверкой подлинности и предоставляет ссылки на документацию, которые помогут вам настроить эти политики.

|--------|------|--------|
|**Политика паролей**|Локальный компьютер \Конфигурация Configuration\Windows Settings\Security безопасности\Политики политики|Политики паролей влияет на характеристики и поведение паролей. Политики паролей используются для учетных записей домена или учетные записи локальных пользователей. Они определяют параметры паролей, таких как принудительного применения и время существования.<br /><br />Сведения о конкретных параметрах см. в разделе [ןאנמכוי](https://technet.microsoft.com/itpro/windows/keep-secure/password-policy).|
|**Политики блокировки учетных записей**|Локальный компьютер \Конфигурация Configuration\Windows Settings\Security безопасности\Политики политики|Параметры политики блокировки учетной записи отключения учетных записей после нескольких неудачных попыток. С помощью этих параметров может помочь обнаружить и заблокировать попыток взлома паролей.<br /><br />Сведения о параметрах политики блокировки учетной записи, см. в разделе [политики блокировки учетных записей](https://technet.microsoft.com/itpro/windows/keep-secure/account-lockout-policy).|
|**Политика Kerberos**|Локальный компьютер \Конфигурация Configuration\Windows Settings\Security безопасности\Политики политики|Параметры Kerberos включить запрос в службу времени существования и принудительного применения правила. Политика Kerberos не применяется к локальной учетной записи базы данных, так как протокол проверки подлинности Kerberos не используется для проверки подлинности локальных учетных записей. Таким образом параметры политики Kerberos можно настроить только с помощью области по умолчанию политики объекта Групповой политики, где оно влияет на входов в домен.<br /><br />Сведения о параметрах политики Kerberos для контроллера домена, см. в разделе [политика Kerberos](https://technet.microsoft.com/itpro/windows/keep-secure/kerberos-policy).|
|**Политика аудита**|Локальный компьютер \Конфигурация компьютера\Конфигурация Windows\Параметры безопасности\Локальные Политики\политика|Политика аудита позволяет вам контролировать и понять, доступ к объектам, таких как файлы и папки, а также управлять учетными записями пользователей и групп и вход пользователей в систему и выходов из системы. Политики аудита можно указать категории событий, которые требуется аудит, задайте размер и параметры журнала безопасности и определить, какие объекты, которые требуется отслеживать доступ и тип доступа, вы хотите отслеживать.<br /><br />|
|**Назначение прав пользователя**|Локальный компьютер \Конфигурация Configuration\Windows Settings\Security Settings\Local Policies\User Rights Assignment|Обычно права пользователя назначаются на основе групп безопасности, к которым принадлежит пользователь, например Администраторы, Опытные пользователи или пользователи. Параметры политики в этой категории обычно используются предоставлены или запрещены разрешения на доступ к компьютеру, на основе метода доступа и безопасности членства в группе.|
|**Параметры безопасности**|Локальный компьютер \Конфигурация компьютера\Конфигурация Windows\Параметры безопасности\Локальные Политики\параметры|Политики, связанные с проверкой подлинности включают в себя:<br /><br />-Устройств<br />-Контроллер домена<br />-Член домена<br />-Интерактивный вход в систему<br />-Сетевой сервер Майкрософт<br />-Сетевой доступ<br />-Сетевой безопасности<br />-Консоль<br />— Завершение работы<br /><br />|
