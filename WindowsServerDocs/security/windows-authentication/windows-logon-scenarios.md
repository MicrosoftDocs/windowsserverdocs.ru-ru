# <a name="windows-logon-scenarios"></a>Сценарии входа в систему Windows

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом справочном разделе для ИТ-специалистов представлены сценарии входа и распространенные входа в Windows.

Операционная система Windows требует всех пользователей, войдите на компьютер с действительной учетной записью для доступа к локальным и сетевым ресурсам. Компьютеры под управлением Windows безопасной ресурсы, реализуя процесс входа в систему, в котором пользователи проходят проверку подлинности. После проверки подлинности пользователя, авторизации технологии и управления доступом реализовать вторую фазу защиты ресурсов: определение, разрешено ли доступ к ресурсу с проверкой подлинности пользователя.

В этом разделе применяются к версиям Windows, указанных в **относится к** в начале этой статьи.

Кроме того приложения и службы, может требоваться выполнить вход для доступа к этим ресурсам, предоставляемые приложением или службой. Процесс входа в похож на процесс входа в систему, действительную учетную запись и правильные учетные данные являются обязательными, но учетные данные хранятся в базе данных диспетчера учетных записей (SAM) на локальном компьютере и в Active Directory, где это возможно. Вход учетной записи и учетные данные, который управляется приложением или службой и при необходимости можно хранить локально в хранилище учетных данных.

Чтобы понять, как работает проверка подлинности, в разделе [понятия проверки подлинности Windows](windows-authentication-concepts.md).

В этом разделе описаны следующие сценарии:

## <a name="BKMK_InteractiveLogon"></a>Интерактивный вход в систему
Процесс входа в систему начинается, когда пользователь вводит учетные данные в диалоговом окне ввода учетных данных или если пользователь вставляет смарт-карту в устройство чтения смарт-карт, или при взаимодействии пользователя с помощью биометрического устройства. Пользователи могут выполнять интерактивный вход в систему с помощью учетной записи локального пользователя или учетной записи домена для входа на компьютер.

### <a name="BKMK_LocaDomainLogon"></a>Вход в систему локального компьютера или домена
Учетные данные для входа в домен, пользователь содержат все элементы, необходимые для локального входа, такие как имя учетной записи и пароль или сертификат и сведения о домене Active Directory. Процесс подтверждает подлинность пользователя в базу данных системы безопасности на локальном компьютере или к домену Active Directory. Нельзя отключить этот процесс обязательного входа для пользователей в домене.

Пользователи могут выполнять интерактивный вход в систему на компьютере в любой из двух способов:

-   Локально, если у пользователя есть прямой физический доступ к компьютеру или если компьютер является частью сети компьютеров.

    При локальном входе предоставляет пользователю разрешение осуществлять доступ к ресурсам Windows на локальном компьютере. При локальном входе наличие у пользователя учетной записи пользователя в диспетчере учетных записей безопасности (SAM) на локальном компьютере. SAM защищает и управляет пользователе и групп в форме учетные записи безопасности, хранящиеся в реестре локального компьютера. Компьютер может иметь доступ к сети, но не является обязательным. Сведения о членстве пользователя локальной учетной записи и группы используется для управления доступом к локальным ресурсам.

    Вход в сеть предоставляет пользователю разрешение осуществлять доступ к ресурсам Windows на локальном компьютере, а также все ресурсы на сетевых компьютерах, в соответствии с маркер доступа учетных данных. Локального входа и входа в сеть требуется, что пользователь имеет учетной записи пользователя в диспетчере учетных записей безопасности (SAM) на локальном компьютере. Сведения о членстве пользователя локальной учетной записи и группы используется для управления доступом к локальным ресурсам, и маркер доступа для пользователя определяет, какие ресурсы доступны на сетевых компьютерах.

    Локального входа и входа в сеть, недостаточно для разрешения пользователя и компьютера для доступа к и доменным ресурсам.

-   Удаленно, с помощью служб терминалов или Remote Desktop Services (RDS), в котором находится дальше случае вход в систему полным как удаленный интерактивный.

После интерактивного входа в систему Windows запускает приложения, от имени пользователя, и пользователь может взаимодействовать с этими приложениями.

При локальном входе предоставляет пользователю разрешение осуществлять доступ к ресурсам на локальном компьютере или ресурсов на сетевых компьютерах. Если компьютер присоединен к домену, функции Winlogon пытается войти в систему для этого домена.

Вход в домен предоставляет пользователю разрешение осуществлять доступ к локальной и доменным ресурсам. Вход в домен наличие у пользователя учетной записи пользователя в Active Directory. Компьютер должен иметь учетную запись в домене Active Directory и быть физически подключен к сети. Пользователи также должны иметь права пользователя для входа на локальный компьютер или домен. Сведения об учетной записи пользователя домена и членства в группах используются для управления доступом к домену и локальных ресурсов.

### <a name="BKMK_RemoteLogon"></a>Удаленного входа в систему
В Windows обращение к другому компьютеру с помощью удаленного входа полагается на протокол удаленного рабочего стола (RDP). Поскольку пользователь должен уже успешно выполнил вход на клиентский компьютер перед попыткой удаленного подключения, интерактивный вход в систему процессы успешно завершены.

Протокол удаленного рабочего СТОЛА управляет учетные данные, вводимые пользователем с помощью клиента удаленного рабочего стола. Эти учетные данные, предназначенные для целевого компьютера, и пользователь должен иметь учетную запись на этом компьютере. Кроме того целевой компьютер необходимо настроить для принятия удаленных подключений. Учетные данные для целевого компьютера отправляются при попытке выполнить процесс проверки подлинности. Если проверка подлинности успешна, к которым подключен пользователь для локальных и сетевых ресурсов, доступных с помощью предоставленных учетных данных.

## <a name="BKMK_NetworkLogon"></a>Вход в сеть
Вход в сеть можно использовать только после выполнения проверки подлинности пользователя, службы или компьютера. Во время входа в сеть процесс не использует диалоговые окна ввода учетных данных для сбора данных. Вместо этого ранее созданные учетные данные или использовать другой метод для сбора учетных данных. Этот процесс подтверждает удостоверение пользователя, любой пользователь пытается получить доступ к сетевым ресурсам. Этот процесс не обычно невидимы для пользователя, пока альтернативные учетные данные должны указываться.

Чтобы указать этот тип аутентификации, система безопасности включает эти механизмы проверки подлинности:

-   Протокол Kerberos версии 5

-   Сертификаты открытого ключа

-   Secure Sockets Layer/транспортный протокол безопасности транспортного уровня (SSL/TLS)

-   Дайджест

-   NTLM для обеспечения совместимости с системами на основе Microsoft Windows NT 4.0

Дополнительные сведения об элементах и процессов см. на интерактивный вход в систему схеме выше.

## <a name="BKMK_SmartCardLogon"></a>Вход по смарт-карт
Вход только в учетные записи домена, а не в локальные учетные записи могут использовать смарт-карты. Проверка подлинности смарт-карты требует использования проверки подлинности Kerberos. Появился в Windows 2000 Server, в операционных системах Windows расширение открытого ключа для реализации запроса начальную проверку подлинности по протоколу Kerberos. В отличие от общего секретного ключа шифрования асимметричного шифрования с открытым ключом, то есть нужны два разных ключа: один для шифрования, другую для расшифровки. Вместе ключи, необходимые для выполнения обоих операций составляют пары закрытого и открытого ключей.

Для запуска сеанса типичные входа в систему, пользователь должен подтвердить личности, предоставляя данные, известные только пользователь и базовой инфраструктуры протокола Kerberos. Секретные сведения является шифрования общим ключом на основе пароля. Общего секретного ключа — симметричного, это означает, что тот же ключ используется для шифрования и расшифровки.

**Архитектура поставщика учетных данных смарт-карты**

При использовании смарт-карт вместо пароля закрытого и открытого пары ключей, хранящихся на смарт-карт пользователя заменяется общий секретный ключ, который является производным от пароль пользователя. Закрытый ключ хранится только на смарт-карты. Открытый ключ можно было предоставить всем кому владелец хочет exchange конфиденциальной информации.

## <a name="BKMK_BioLogon"></a>Биометрический вход
Устройство используется для записи и создавать цифровые характеристикой артефакт, такие как отпечаток пальца. Это представление цифровой затем по сравнению с примером же артефактов и может произойти при сравниваются два проверки подлинности. Компьютеры под управлением любой из операционных систем, указанных в **относится к** в начале этого раздела можно настроить на прием эту форму входа в систему. Тем не менее если биометрической настроен только на локальный вход в систему, пользователь должен предоставить учетные данные домена при доступе к домену Active Directory.
