# <a name="windows-logon-scenarios"></a>Сценарии входа в систему Windows

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В этом справочном разделе для ИТ-специалистов перечислены распространенные входа Windows и сценариев входа в систему.

Операционные системы Windows требуют всех пользователей, войдите на компьютер с действительной учетной записи для доступа к локальным и сетевым ресурсам. Компьютеры на базе Windows защитить ресурсы путем реализации процесс входа в систему, в котором пользователи проходят проверку подлинности. После проверки подлинности пользователя, технологии управления авторизацией и доступом к реализовать вторую фазу защиты ресурсов: определение, авторизован ли прошедшего проверку подлинности пользователя для доступа к ресурсу.

Содержимое этого раздела применяются к версиям Windows, указанных в **применяется к** списка в начале этого раздела.

Кроме того приложения и службы может потребоваться пользователи выполняли вход для доступа к этим ресурсам, предлагаемых приложением или службой. Процесс входа в систему похож на процесс входа в систему, в том, что действующей учетной записи и правильные учетные данные являются обязательными, но учетные данные хранятся в базе данных диспетчера учетных записей безопасности (SAM) на локальном компьютере и в Active Directory, где это возможно. Входа в учетную запись и учетные данные управляются приложением или службой и при необходимости могут храниться локально в хранилище учетных данных.

Чтобы понять, как работает проверка подлинности, см. в разделе [понятия проверки подлинности Windows](windows-authentication-concepts.md).

В этом разделе описаны следующие сценарии:

## <a name="BKMK_InteractiveLogon"></a>Интерактивный вход в систему
Процесс входа в систему начинается, когда пользователь вводит учетные данные в диалоговом окне входа учетные данные и когда пользователь вставляет смарт-карту в устройство чтения смарт-карт, либо когда пользователь взаимодействует с биометрическим устройством. Пользователи могут выполнять интерактивный вход в систему, используя учетную запись локального пользователя или учетную запись домена для входа на компьютер.

### <a name="BKMK_LocaDomainLogon"></a>Вход в систему локального компьютера или домена
Учетные данные для входа в домен, он содержал все элементы, необходимые для локального входа, такие как имя учетной записи, пароля или сертификата и сведения о домене Active Directory. Процесс подтверждает подлинность пользователя в базу данных безопасности на локальном компьютере пользователя или к домену Active Directory. Этот процесс входа в систему обязательно нельзя отключить для пользователей в домене.

Пользователи могут выполнять интерактивный вход в систему на компьютере одним из двух способов:

-   Локально, когда пользователь имеет прямой физический доступ к компьютеру, или если компьютер находится в сети компьютеров.

    Локальный вход в систему предоставляет пользователю разрешение на доступ к ресурсам Windows на локальном компьютере. Локальный вход в систему требуется наличие у пользователя учетной записи пользователя в диспетчере учетных записей безопасности (SAM) на локальном компьютере. SAM защищает и управляет пользовательской и групповой информации в виде учетных записей безопасности, хранящихся в реестре локального компьютера. Компьютер может иметь доступ к сети, но это не обязательно. Локальный пользователь учетной записи и группы сведений о членстве используется для управления доступом к локальным ресурсам.

    Вход в сеть предоставляет пользователю разрешение на доступ к ресурсам Windows на локальном компьютере, а также любые материалы на сетевых компьютерах, согласно маркер доступа для учетных данных. Локальный вход в систему и вход в сеть требует, что у пользователя есть учетная запись пользователя в диспетчере учетных записей безопасности (SAM) на локальном компьютере. Локальный пользователь учетной записи и группы сведений о членстве используется для управления доступом к локальным ресурсам и маркер доступа для пользователя определяет, какие ресурсы доступны на сетевых компьютерах.

    Локальный вход в систему и вход в сеть, недостаточно для предоставления разрешения пользователя и компьютера для доступа к и использовать ресурсы домена.

-   Удаленно через службы терминалов или Remote Desktop Services (RDS), в котором дальнейших случае вход в систему, заданный в формате удаленный интерактивный.

После интерактивного входа Windows запускает приложения от имени пользователя, и пользователь может взаимодействовать с этими приложениями.

Локальный вход в систему предоставляет пользователю разрешение на доступ к ресурсам на локальном компьютере или ресурсов на компьютерах сети. Если компьютер присоединен к домену, функцию Winlogon пытается войти в систему для этого домена.

Войти в домен предоставляет пользователю разрешение на доступ к локальной и доменным ресурсам. Войти в домен наличие у пользователя учетной записи пользователя в Active Directory. Компьютер, необходимо иметь учетную запись в домене Active Directory и быть физически подключен к сети. Пользователи также должны иметь права пользователя для входа на локальный компьютер или домен. Сведения об учетной записи пользователя домена и сведения о членстве группы используются для управления доступом к домену и локальным ресурсам.

### <a name="BKMK_RemoteLogon"></a>Удаленного входа
В Windows доступ к другому компьютеру с помощью удаленного входа полагается на протокол удаленного рабочего стола (RDP). Поскольку пользователь должен уже успешно выполнил вход на клиентский компьютер перед попыткой подключения к удаленному, интерактивный вход в систему процессы успешно завершены.

RDP управляет учетными данными, которые пользователь вводит с помощью клиента удаленного рабочего стола. Эти учетные данные, предназначенные для конечного компьютера, и пользователь должен иметь учетную запись на этом компьютере. Кроме того чтобы принять подключение к удаленному настраиваются конечного компьютера. Учетные данные для целевого компьютера отправляются при попытке выполнить процесс проверки подлинности. При успешном выполнении проверки подключен локальный пользователь, и сетевых ресурсов, доступных с помощью предоставленных учетных данных.

## <a name="BKMK_NetworkLogon"></a>Вход в сеть
Вход в сеть может использоваться только после выполнения проверки подлинности пользователя, службы или компьютера. При входе в систему процесс не использует диалоговые окна ввода учетных данных для сбора данных. Вместо этого, установленное ранее учетные данные или использовать другой метод для получения учетных данных. Этот процесс проверяет удостоверение пользователя, любой пользователь пытается получить доступ к сетевым ресурсам. Этот процесс обычно невидимой для пользователя, если не нужно указать альтернативные учетные данные.

Для предоставления этого типа проверки подлинности, система безопасности включает в себя эти механизмы проверки подлинности:

-   Протокол Kerberos версии 5

-   Сертификаты открытого ключа

-   Защита безопасности уровня/транспортного уровня сокетов (SSL/TLS)

-   Digest

-   NTLM, для совместимости с системами Microsoft Windows NT 4.0

Сведения об элементах и процессов см. в разделе приведенной выше схеме интерактивный вход в систему.

## <a name="BKMK_SmartCardLogon"></a>Вход смарт-картой
Смарт-карты можно использовать для входа только учетные записи домена, не локальные учетные записи. Проверка подлинности смарт-карт требует использования протокола проверки подлинности Kerberos. Появился в Windows 2000 Server, в операционных системах Windows расширение открытого ключа для реализации протокола первоначальный запрос проверки подлинности Kerberos. В отличие от общего секретного ключа шифрования, является асимметричной криптографии с открытым ключом, то есть необходимы два различных ключа: один для шифрования, другой для расшифровки. Ключи, которые требуются для выполнения обеих операций составляют пары закрытого и открытого ключей.

Создание типичных входа сеанса, пользователь должны подтвердить свою подлинность, предоставляя информацию, известного только пользователь и базовой инфраструктуры протокола Kerberos. Секретные сведения, шифрования общего ключа, полученного из пароля пользователя. Общий секретный ключ симметричен, это означает, что тот же ключ используется для шифрования и расшифровки.

Закрытый ключ хранится только на смарт-карты. Открытый ключ могут быть сделаны доступными всем, кому владелец требуется выполнить обмен конфиденциальной информации.

## <a name="BKMK_BioLogon"></a>Биометрической
Устройство используется для снятия и создания цифровых характеристикой артефакта, например отпечатка пальца. Это представление digital затем сравнивается к выборке одного артефакта и может произойти при сравнении двух успешно проверки подлинности. Компьютеры под управлением одной из операционных систем, указанных в **применяется к** списка в начале этого раздела может быть настроена на принятие такая форма входа в систему. Тем не менее при биометрической настроен только для локального входа, пользователь должен предоставить учетные данные домена при доступе к домену Active Directory.
