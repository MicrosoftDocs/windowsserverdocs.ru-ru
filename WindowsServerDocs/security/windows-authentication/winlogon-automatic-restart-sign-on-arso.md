## <a name="overview"></a>Обзор  
Windows 8 была представлена приложения на экране блокировки.  Это приложения, которые выполняются и отображать уведомления, если сеанс заблокирован (календаря встреч, электронной почты и сообщения, и т. д.).  Не отображать эти уведомления на экране блокировки после перезапуска устройства, которые будут перезапущены, из-за процесса обновления Windows.  Некоторые пользователи зависят от этих приложений экрана блокировки.  
  
## <a name="whats-changed"></a>Что изменилось?  
Когда пользователь входит на устройстве Windows 8.1, LSA будет сохранить учетные данные пользователя в зашифрованном объем доступной памяти только lsass.exe. Если Центр обновления Windows инициирует автоматической перезагрузки без присутствия пользователя, эти учетные данные будет использоваться для настройки автоматического входа для пользователя. Центр обновления Windows используется в качестве системы с привилегии TCB инициирует вызов RPC для этого.  
  
Для перезагрузки, пользователь будет автоматически вошел в систему через механизм автоматического входа в систему и затем Кроме того заблокировано для защиты сеанс пользователя. Блокировка инициируется через Winlogon в то время как управление учетными данными осуществляется LSA.  Автоматически входе и блокировка пользователя на консоли перезапуск и доступность будет приложения экрана блокировки пользователя.  
  
> [!NOTE]  
> После перезагрузки вносимое центра обновления Windows, последний интерактивный пользователь автоматически входит в систему и сеанс блокируется, поэтому можно запустить приложения на экране блокировки пользователя.

## <a name="group-policy-sign-in-last-interactive-user-automatically-after-a-system-initiated-restart"></a>Групповая политика: Вход последнего текущего пользователя автоматически после перезапуска, инициированного системой  
В Windows 8.1 и Windows Server 2012 R2 автоматический вход в систему пользователя экрана блокировки после перезапуска Windows Update согласие на сервере SKU и отказ для клиентских SKU.  
  
**Размещение политики:** Конфигурация компьютера > политики > Административные шаблоны > компоненты Windows > параметр входа в систему Windows  
  
**Имя политики:** вход последнего текущего пользователя автоматически после перезапуска, инициированного системой  
  
**Поддерживается в:** по крайней мере Windows Server 2012 R2, Windows 8.1 или Windows RT 8.1  
  
**Описание и справки:**  
  
Этот параметр политики контролирует ли устройство будет автоматически вход последнего текущего пользователя после обновления Windows перезагрузка системы.  
  
Если вы включите или не настраивать этот параметр политики, устройство безопасно сохраняет учетные данные пользователя (включая имя пользователя, домен и зашифрованный пароль) для настройки автоматического входа в систему после перезагрузки компьютера центра обновления Windows. После перезагрузки центра обновления Windows пользователь автоматически вход в систему, а сеанс блокируется автоматически всеми приложениями экрана блокировки, настроенные для этого пользователя.  
  
Если вы отключите этот параметр политики, устройство не сохраняет учетные данные пользователя для автоматического входа в систему после перезапуска Windows Update. После перезапуска системы, приложения на экране блокировки пользователей не перезапускаются.  
  
**Редактор реестра**  
  
|Имя значения|Тип|Данные|  
|-------|----|----|  
|DisableAutomaticRestartSignOn|DWORD|0<br /><br />**Пример:**<br /><br />0 (включено)<br /><br />1 (отключено)|  
  
**Политика реестра:** HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System  
  
**Тип:** DWORD  
  
**Имя реестра:** DisableAutomaticRestartSignOn  
  
Значение: 0 или 1  
  
0 = включено  
  
1 = отключено

### <a name="reasons-why-autologon-might-fail"></a>Причин сбоя автоматический вход в систему может  
Существует несколько вариантов, в которых нельзя достичь автоматический вход пользователя.  Этот раздел предназначен для записи известных сценариев, в которых это может произойти.  
  
### <a name="user-must-change-password-at-next-login"></a>Пользователь должен изменить пароль при следующем входе в систему  
Имя входа пользователя можно ввести состояние блокировки, при необходимости изменения пароля при следующем входе в систему.  Это может быть обнаружены до перезапуска в большинстве случаев, но не все (например, можно получить истечения срока действия пароля между завершение работы и следующем входе в систему.  
  
### <a name="user-account-disabled"></a>Учетная запись пользователя отключена  
Можно поддерживать существующего сеанса пользователя, даже если он отключен.  Перезапуск учетной записи, которая отключена могут быть обнаружены локально в большинстве случаев заранее, в зависимости от групповой политики, не может быть для учетных записей домена (некоторые домена кэшированных рабочих сценариев входа даже при отключении учетной записи на контроллере домена).  
  
### <a name="logon-hours-and-parental-controls"></a>Разрешенные часы доступа и родительского контроля  
Время входа и родительского контроля можно запретить создается новый сеанс пользователя.  В случае перезагрузки в течение этого периода, пользователю будет запрещено для входа в систему.  Есть дополнительные политики, при котором блокировки или выхода из системы, как действие соответствия.  Это может быть проблематично для многих дочерних случаях, когда блокировки учетной записи может между временем кровати и пробуждения, особенно в том случае, если окна обслуживания — обычно это время.
