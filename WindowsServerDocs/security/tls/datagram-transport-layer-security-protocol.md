# Протокола передачи датаграмм TLS.

>Область применения: Windows Server (канал точками годовой), Windows Server 2016

В этом справочном разделе для ИТ-специалистов описываются протокол безопасности уровня транспорта датаграмм (DTLS), который входит в состав из поставщика поддержки безопасности Schannel (SSP).

## <a name="BKMK_DTLS"></a>
Представленный в Schannel SSP в Windows Server 2012 и Windows 8, протокол DTLS обеспечивает конфиденциальность связи для протоколов датаграмм. Сведения о которых DTLS версия поддерживается в версиях Windows см. в разделе [протоколов TLS/SSL (Schannel SSP)](https://msdn.microsoft.com/en-us/library/windows/desktop/mt808159(v=vs.85).aspx). Данный протокол позволяет клиентским и серверным приложениям взаимодействовать способом, предназначенную для предотвращения перехвата, взлома или подделки сообщений. Протокол DTLS основывается на протокол безопасности транспортного уровня (TLS) и предоставляет аналогичные гарантии безопасности, что сокращает потребность в использовании IPsec или разработке пользовательского протокола безопасности прикладного уровня.

Датаграммы являются обычным объектом в потоковой передачи, такой как игры или безопасные видеоконференции. Разработчики могут создавать приложения для использования протокола DTLS в контексте модели интерфейс поставщика поддержки безопасности (SSPI) для проверки подлинности Windows для обеспечения безопасного обмена данными между клиентами и серверами. Протокол DTLS основывается на основе протокола передачи датаграмм пользователя (UDP). Чтобы быть максимально похожим на протокол TLS, что позволяет чтобы свести к минимуму новый изобретения безопасности и увеличить объем повторно используемых элементов инфраструктуры и кода предназначен DTLS.

Наборы шифров, доступные для настройки настроенная в те, которые можно настроить для протокола TLS. RC4 не поддерживается. Schannel продолжается использование криптографии следующего поколения (CNG). После этого сертификацией FIPS 140, которая была введена в Windows Vista.
