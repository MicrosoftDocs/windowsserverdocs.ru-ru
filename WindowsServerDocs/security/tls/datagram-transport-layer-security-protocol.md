# <a name="datagram-transport-layer-security-protocol"></a>Протокол DTLS

Windows Server (полугодовой канал), Windows Server 2016, Windows 10

В этом справочном разделе для ИТ-специалистов описывается протокол безопасности уровня транспорта датаграмм (DTLS), входящий в состав из поставщика поддержки безопасности Schannel (SSP).

## <a name="BKMK_DTLS"></a>
Представленный в Schannel SSP в Windows Server 2012 и Windows 8, протокол DTLS обеспечивает конфиденциальность связи для протоколов датаграмм. Сведения о какой протокол DTLS версии поддерживается в версиях Windows, см. в разделе [протоколы TLS/SSL (Schannel SSP)](https://msdn.microsoft.com/en-us/library/windows/desktop/mt808159(v=vs.85).aspx). Данный протокол позволяет клиентским и серверным приложениям взаимодействовать способом, обеспечивающим защиту от перехвата, взлома или подделки сообщений. Протокол DTLS основывается на протоколе TLS и предоставляет аналогичные гарантии безопасности, которые сокращают потребность в использовании IPsec или разработке пользовательского протокола безопасности на уровне приложений.

Датаграммы являются обычным в потоковой передачи мультимедиа, такие как игры или безопасные видеоконференции. Разработчики могут создавать приложения для использования протокола DTLS в контексте модели интерфейса поставщика поддержки безопасности (SSPI) проверки подлинности Windows для безопасного взаимодействия между клиентами и серверами. Протокол DTLS построена на основе протокола UDP (User Datagram). Протокол DTLS должна быть как на аналогичную TLS можно свести к минимуму изобретения новых безопасности и увеличить объем повторного использования кода и инфраструктуры.

Наборы шифров, доступных для конфигурации создаются в соответствии с после фильтров, которые можно настроить для TLS. Версия-кандидат 4 не допускается. Schannel продолжается использование криптографии следующего поколения (CNG). Это позволяет пользоваться преимуществами сертификации FIPS 140, которая была введена в Windows Vista.
