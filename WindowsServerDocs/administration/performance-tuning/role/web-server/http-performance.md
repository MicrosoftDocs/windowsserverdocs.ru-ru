# <a name="performance-tuning-http-112"></a>Настройка HTTP 1.1/2 производительности

HTTP/2 предназначен для повышения производительности на стороне клиента (например, время загрузки страницы в браузере). На сервере он может представлять небольшое увеличение в ЦП. В то время как сервер больше не требует одно подключение TCP для каждого запроса, некоторые из этого состояния теперь будут храниться на уровне HTTP. Кроме того HTTP/2 имеет сжатие заголовков, который представляет дополнительную нагрузку на ЦП.

В некоторых ситуациях необходимо HTTP/1.1 резервный (сброса соединения HTTP/2 и вместо создания нового подключения для использования HTTP/1.1). В частности повторного согласования TLS и проверки подлинности HTTP (кроме Basic и хэш-кода) требуется резервная HTTP/1.1. Несмотря на то, что это увеличивает затраты, эти операции уже подразумевают некоторая задержка и поэтому не особенно чувствительных к производительности.
