# <a name="hyper-v-architecture"></a>Архитектура Hyper-V

Для функции типа 1 архитектура на основе низкоуровневой оболочки Hyper-V. Низкоуровневая оболочка виртуализирует процессоры и память и предоставляет механизмы для стека виртуализации в корневом разделе для управления дочерних разделов (виртуальных машин) и предоставления доступа к службам, например, устройства ввода-вывода к виртуальным машинам.

Корневой раздел владеет и имеет прямой доступ к физическим устройствам ввода-вывода. Стек виртуализации в корневом разделе предоставляет диспетчер памяти для виртуальных машин, API-интерфейсов управления и виртуализированных устройств ввода-вывода. Он также реализует эмулированные устройства, такие как контроллера дисков интегрированных устройств электроники (IDE) и порт устройства ввода PS/2 и поддерживает синтетических устройств Hyper v, появляющихся для повышения производительности и сокращение издержек.

![архитектура на основе гипервизора Hyper-v](../../media/perftune-guide-hyperv-arch.png)

Архитектура Hyper-V-конкретных операций ввода-вывода состоит из поставщиков служб виртуализации (VSP) в корневой раздел и виртуализации клиентов служб (VSC) в дочернем разделе. Каждая служба предоставляется как устройство по VMBus, которая выступает в качестве шине ввода-вывода и предоставляет высокопроизводительного обмена данными между виртуальными машинами, использовать механизмы, такие как общей памяти. Гостевой операционной системы диспетчера Plug and Play перечисляет этих устройств, включая VMBus и загружает соответствующие драйверы устройств (клиентов виртуальных служб). Службами, отличными от ввода-вывода также предоставляются посредством этой архитектуры.

Начиная с Windows Server 2008, просветления компоненты операционной системы для оптимизации его поведение при выполнении на виртуальных машинах. К преимуществам относятся, снижая затраты средств виртуализации памяти, улучшая масштабируемость многоядерных и снижения фонового использования ЦП гостевой операционной системы.

Следующие разделы описывают рекомендации, которые дают повышения производительности на серверах под управлением роли Hyper-V.
