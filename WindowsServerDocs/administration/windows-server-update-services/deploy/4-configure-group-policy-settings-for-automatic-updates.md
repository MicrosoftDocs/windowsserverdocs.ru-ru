# <a name="step-4-configure-group-policy-settings-for-automatic-updates"></a>Шаг 4. Настройка параметров групповая политика для автоматическое обновление

Этот раздел содержит два основных раздела.

[Параметры групповая политика для обновлений клиента WSUS](#group-policy-settings-for-wsus-client-updates), которые предоставляют рекомендации и подробные сведения о параметрах планировщика центр обновления Windows и обслуживания групповая политика, которые управляют взаимодействием клиентов wsus с центр обновления Windows получение автоматических обновлений.

#### <a name="allow-automatic-updates-immediate-installation"></a>Разрешить немедленную установку автоматических обновлений
Указывает, будут ли автоматическое обновление автоматически устанавливать обновления, не прерывающие работу служб Windows или перезапускать Windows.

|---------|-------|
|Операционные системы Windows, которые по-прежнему находятся в [жизненном цикле поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|null|

> [!NOTE]
> Если параметр политики "настроить автоматическое обновление" имеет значение " **отключено**", эта политика не действует.

|||
|-|-|
|**Состояние параметра политики**|**Неполадок**|
|**Не настроено**|Указывает, что обновления не будут устанавливаться немедленно. Локальные администраторы могут изменить этот параметр с помощью локального редактора групповая политика.|
|**Включен**|Указывает, что автоматическое обновление немедленно устанавливает обновления после их скачивания и готовности к установке.|
|**Отключен**|Указывает, что обновления не будут устанавливаться немедленно.|

#### <a name="always-automatically-restart-at-the-scheduled-time"></a>Всегда автоматически перезагружаться в запланированное время
Указывает, будет ли таймер перезагрузки запускаться немедленно после Центр обновления Windows установки важных обновлений, вместо того чтобы уведомлять пользователей на экране входа в течение как минимум двух дней.

|Поддерживается в:|Исключение|
|---------|-------|
|Операционные системы Windows, которые по-прежнему находятся в [жизненном цикле поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|null|

> [!NOTE]
> Если включен параметр политики «не выполнять автоматический перезапуск с вошедшими в систему пользователями для запланированных автоматических обновлений», эта политика не действует.

|||
|-|-|
|**Состояние параметра политики**|**Неполадок**|
|**Не настроено**|Указывает, что Центр обновления Windows не изменит поведение компьютера при перезапуске.|
|**Включен**|Указывает, что таймер перезапуска всегда запускается сразу после Центр обновления Windows устанавливает важные обновления, а не уведомляет пользователей на экране входа в течение не менее двух дней.<br /><br />Таймер перезапуска можно настроить для запуска с любым значением от 15 до 180 минут. При запуске таймера перезагрузка будет продолжена даже в том случае, если компьютер вошел в систему пользователей.|
|**Отключен**|Указывает, что Центр обновления Windows не изменит поведение компьютера при перезапуске.|

**Параметры:** если этот параметр включен, можно указать время, которое должно пройти после установки обновлений, прежде чем произойдет Принудительная перезагрузка компьютера.

#### <a name="automatic-updates-detection-frequency"></a>Частота поиска автоматических обновлений
Указывает промежуток времени в часах между поисками доступных обновлений. Истинное время ожидания определяется путем вычитания от 0 до 20 процентов от указанного времени. Например, если эта политика используется для задания частоты обнаружения в 20 часов, все клиенты, к которым применяется эта политика, будут проверять наличие обновлений в любом месте от 16 до 20 часов.

|Поддерживается в:|Исключение|
|---------|-------|
|Операционные системы Windows, которые по-прежнему находятся в [жизненном цикле поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|Windows RT|

> [!NOTE]
> Чтобы эта политика действовала, необходимо включить параметр "указать расположение службы обновлений Майкрософт в интрасети".
>
> Если параметр политики "настроить автоматическое обновление" отключен, эта политика не действует.

> [!NOTE]
> Эта политика не поддерживается в Windows RT. Включение этой политики не оказывает никакого влияния на компьютеры, на которых выполняется Windows RT.

|||
|-|-|
|**Состояние параметра политики**|**Неполадок**|
|**Не настроено**|Указывает, что Windows будет проверять наличие доступных обновлений с интервалом по умолчанию (22 часа).|
|**Включен**|Указывает, что Windows будет проверять наличие доступных обновлений через указанный интервал.|
|**Отключен**|Указывает, что Windows будет проверять наличие доступных обновлений с интервалом по умолчанию (22 часа).|

**Параметры:** если этот параметр включен, можно указать интервал времени (в часах), центр обновления Windows ожидания перед проверкой наличия обновлений.

#### <a name="configure-automatic-updates"></a>Настройка автоматического обновления

Если параметр включен, необходимо выбрать один из четырех параметров, указанных в этой групповая политика.

Чтобы использовать этот параметр, выберите **включено**, а затем в разделе **Параметры** раздела **Настройка автоматического обновления**выберите один из параметров (2, 3, 4 или 5).

|||
|-|-|
|**Состояние параметра политики**|**Неполадок**|
|**Не настроено**|Указывает, что использование автоматического обновления не указано на уровне групповая политика. Однако администратор компьютера по-прежнему может настроить автоматические обновления на панели управления.|

### <a name="accessing-the-windows-update-settings-in-group-policy"></a>Доступ к параметрам Центр обновления Windows в групповая политика
В следующей процедуре описывается, как открыть консоль управления групповыми политиками на контроллере домена. В этой процедуре описывается, как либо открыть существующий объект групповая политика уровня домена (GPO) для редактирования, либо создать новый объект групповой политики на уровне домена и открыть его для редактирования.

> [!NOTE]
> Для выполнения этой процедуры необходимо быть членом группы **"Администраторы домена"** или эквивалентными.

##### <a name="to-open-or-add-and-open-a-group-policy-object"></a>Открытие или добавление и открытие объекта групповая политика

1.  На контроллере домена перейдите в раздел **Диспетчер сервера**, > **инструменты**, > **Групповая политика управления**. Откроется консоль управления групповыми политиками.

2.  В левой области разверните свой лес. Например, дважды щелкните **лес: example.com**.

3.  В левой области дважды щелкните **домены**, а затем дважды щелкните домен, для которого требуется управлять объектом Групповая политика. Например, дважды щелкните **example.com**.

4.  Выполните одно из следующих действий.

    -  **Чтобы открыть существующий объект групповой политики на уровне домена для редактирования**, дважды щелкните домен, содержащий объект Групповая политика, которым требуется управлять, щелкните правой кнопкой мыши политику домена, которой требуется управлять, и выберите команду **изменить**. Откроется редактор групповая политика Management (GPME).

    -  **Чтобы создать новый объект Групповая политика и открыть его для редактирования,** сделайте следующее:
        1.  Щелкните правой кнопкой мыши домен, для которого нужно создать новый объект групповая политика, а затем выберите **в этом домене пункт Создать объект групповой политики и связать его**.

        2.  В поле **имя** **нового объекта групповой политики**введите имя нового объекта Групповая политика и нажмите кнопку **ОК**.

        3.  Щелкните правой кнопкой мыши новый объект групповая политика и выберите пункт **изменить**. Откроется GPME.

##### <a name="to-open-the-windows-update-or-maintenance-scheduler-extensions-of-group-policy"></a>Чтобы открыть Центр обновления Windows или расширения планировщика обслуживания групповая политика

1.  В групповая политика редакторе управления выполните одно из следующих действий.

    -   **Откройте > Конфигурация компьютера центр обновления Windows расширение групповая политика**. Перейдите к: *PolicyName* > **Computer Configuration** > **политики** / **Административные шаблоны** > **компонентов Windows** > **Центр обновления Windows**.

    -   **Откройте Центр обновления Windows > "Конфигурация пользователя" Групповая политика**. Перейдите к: *PolicyName* > **Конфигурация пользователя** > **политики** > **Административные шаблоны** > **компонентов Windows** > **Центр обновления Windows**.

    -   **Откройте окно Конфигурация компьютера > расширение планировщика обслуживания групповая политика**. В GPOE перейдите в раздел *PolicyName* > **Конфигурация компьютера** > **политики** > **административные шаблоны** > ,**компоненты Windows** > **планировщика обслуживания**.

Дополнительные сведения о групповая политика см. в разделе [Общие](https://technet.microsoft.com/library/hh831791.aspx(v=ws.12))сведения о групповая политика.

> [!TIP]
> После открытия расширения групповая политика необходимо выполнить следующие действия для включения, отключения или перехода между параметрами.

##### <a name="to-configure-group-policy-settings"></a>Настройка параметров групповой политики

1.  В *екстенсионофграупполици*дважды щелкните параметр, который необходимо просмотреть или изменить.

2.  Чтобы настроить этот параметр, выполните одно из следующих действий.

    -   Чтобы не задавать неопределенное состояние параметра по умолчанию, выберите значение **не настроено**.

    -   Чтобы включить этот параметр, выберите **включено**.

    -   Чтобы отключить этот параметр, выберите **отключено**.

3.  Если в списке **параметров**указаны какие либо параметры, оставьте значения по умолчанию или измените их по мере необходимости.

4.  Выполните одно из следующих действий.

    -   Чтобы сохранить изменения и перейти к следующему параметру, нажмите кнопку **Применить**, а затем нажмите кнопку **Далее**.

    -   Чтобы сохранить изменения и закрыть диалоговое окно, нажмите кнопку " **ОК**".

    -   Чтобы отменить все несохраненные изменения и закрыть диалоговое окно, нажмите кнопку **"Отмена"** .
