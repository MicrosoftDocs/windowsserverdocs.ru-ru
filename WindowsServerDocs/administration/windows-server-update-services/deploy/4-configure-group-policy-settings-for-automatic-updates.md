# <a name="step-4-configure-group-policy-settings-for-automatic-updates"></a>Шаг 4. Настройка параметров групповой политики для автоматического обновления

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

#### <a name="allow-automatic-updates-immediate-installation"></a>Разрешить немедленную установку автоматических обновлений
Указывает ли автоматическое обновление будет автоматически устанавливать обновления, которые не прерывания работы служб Windows или перезапуска Windows.

|Поддерживается в:|Исключение:|
|---------|-------|
|Операционные системы Windows, не выходя за рамки их [жизненный цикл поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|null|

> [!NOTE]

|||

|**Не настроено**|Указывает, что обновления не устанавливаются немедленно. Локальные администраторы могут изменить этот параметр, с помощью редактора локальных групповых политик.|
|**включен**|Указывает, что автоматическое обновление немедленно выполняется после загружены и готовы к установке.|
|**Отключено**|Указывает, что обновления не устанавливаются немедленно.|

#### <a name="always-automatically-restart-at-the-scheduled-time"></a>Всегда автоматически перезагружаться в запланированное время

|---------|-------|
|Операционные системы Windows, не выходя за рамки их [жизненный цикл поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|null|

> [!NOTE]
> Если включен параметр политики «не выполнять автоматическую перезагрузку with logged on users для запланированного автоматического обновления установки», эта политика не влияет.

|||
|-|-|
|**Перевод в состояние политики**|**Поведение**|

|**включен**|Указывает, что таймер перезапуска, всегда будет начинаться сразу после установки обновления Windows важные обновления, вместо первого уведомление пользователей на экране входа в систему по крайней мере два дня.<br /><br />Таймер перезапуска могут быть настроены на запуск с любое значение от 15 до 180 минут. #### <a name="automatic-updates-detection-frequency"></a>Частота поиска автоматических обновлений
Указывает промежуток времени в часах между поисками доступных обновлений. Истинное время ожидания определяется путем вычитания от 0 до 20 процентов от указанного времени. Например если эта политика позволяет задать частоту обнаружения 20 часов, все клиенты, к которым применяется эта политика будет искать обновления в любом от 16 до 20 часов.

|Поддерживается в:|Исключение:|
|---------|-------|
|Операционные системы Windows, не выходя за рамки их [жизненный цикл поддержки продуктов Майкрософт](https://support.microsoft.com/gp/lifeselect).|Windows RT|

> [!NOTE]
> Для этой политики действовала должен быть включен параметр «Указать размещение службы обновлений Майкрософт в интрасети».
>
> Если отключить параметр «Настройка автоматического обновления», эта политика не действует.

> [!NOTE]
> Эта политика не поддерживается в Windows RT. При включении этой политики не влияет на компьютеры под управлением Windows RT.

|||
|-|-|
|**Перевод в состояние политики**|**Поведение**|
|**Не настроено**|Указывает, что Windows будет проверять наличие доступных обновлений с интервалом 22 часа.|
|**включен**|Указывает, что Windows будет проверять наличие доступных обновлений с указанным интервалом.|
|**Отключено**|Указывает, что Windows будет проверять наличие доступных обновлений с интервалом 22 часа.|

**Параметры:** Если этот параметр включен, можно указать интервал времени (в часах), Windows Update ожидает перед проверкой наличия обновлений.

### <a name="accessing-the-windows-update-settings-in-group-policy"></a>Доступ к параметрам обновления Windows в групповой политике
Приведенной ниже процедуре описывается, как открыть консоль управления групповыми Политиками на контроллере домена. Затем описано, как открыть существующий режим домена политики объекта Групповой политики для редактирования, или создать новый объект групповой Политики доменного уровня и открыть его для редактирования.

> [!NOTE]
> Необходимо быть членом **"Администраторы домена"** группы или наличие эквивалентных прав для выполнения этой процедуры.

##### <a name="to-open-or-add-and-open-a-group-policy-object"></a>Чтобы открыть или добавить и открыть объект групповой политики

1.  На контроллере домена, перейдите в каталог **диспетчера серверов**, > **средства**, > **Управление групповой политикой**. Откроется консоль управления групповыми политиками.

2.  В левой области разверните леса. Например, дважды щелкните **леса: example.com**.

3.  В левой области дважды щелкните **домены**, а затем дважды щелкните домен, для которого вы хотите управлять объект групповой политики. Например, дважды щелкните **example.com**.

4.  Выполните одно из следующих действий.

    -  **Чтобы открыть существующий объект групповой Политики доменного уровня для редактирования**, дважды щелкните домен, содержащий объект групповой политики, который вы хотите управлять, щелкните правой кнопкой мыши политику домена, вы хотите управлять и нажмите кнопку **изменить**. Откроется редактор групповой политики управления (GPME).

    -  **Чтобы создать объект групповой политики и откройте для редактирования**:
        1.  Щелкните правой кнопкой мыши домен, для которого вы хотите создать объект групповой политики и нажмите кнопку **создать объект групповой Политики в этом домене и связать его**.

        2.  В **новый объект групповой Политики**в **имя**, введите имя для нового объекта групповой политики и нажмите кнопку **ОК**.

        3.  Щелкните правой кнопкой мыши новый объект групповой политики, а затем нажмите кнопку **изменить**. Открывается GPME.

##### <a name="to-open-the-windows-update-or-maintenance-scheduler-extensions-of-group-policy"></a>Чтобы открыть Windows Update или планировщик заданий обслуживания расширения групповой политики

1.  В редакторе управления групповыми политиками выполните одно из следующих действий.

    -   **Откройте Конфигурация компьютера > расширения групповой политики Windows Update**. Перейдите к: *PolicyName* > **компьютере конфигурация** > **политики** / **административные шаблоны**  >  **Компоненты Windows** > **Windows Update**.

    -   **Открыть конфигурацию пользователя > расширения групповой политики Windows Update**. Перейдите к: *PolicyName* > **Конфигурация пользователя** > **политики** > **административные шаблоны**  >  **Компоненты Windows** > **Windows Update**.

    -   **Откройте Конфигурация компьютера > планировщик заданий обслуживания расширения групповой политики**. В GPOE, перейдите к *PolicyName* > **компьютере конфигурация** > **политики** > **администрирования Шаблоны** > **компоненты Windows** > **планировщик заданий обслуживания**.

Дополнительные сведения о групповой политике см. в разделе [Group Policy Overview](https://technet.microsoft.com/library/hh831791.aspx(v=ws.12)).

> [!TIP]
> После открытия расширение групповой политики, которые вы хотите, можно использовать следующие действия для включения, отключения или перемещаться между параметры:

##### <a name="to-configure-group-policy-settings"></a>Настройка параметров групповой политики

1.  В *ExtensionOfGroupPolicy*, дважды щелкните параметр, который вы хотите просмотреть или изменить.

2.  Настройка параметров, выполните одно из следующих действий.

    -   Чтобы сохранить значение по умолчанию не указано состояние параметра, выберите **не настроен**.

    -   Чтобы включить этот параметр, выберите **включено**.

    -   Чтобы отключить этот параметр, выберите **отключено**.

3.  В **параметры**, если все параметры в списке, оставьте значения по умолчанию или изменить их при необходимости.

4.  Выполните одно из следующих действий.

    -   Чтобы сохранить изменения и перейти на следующую настройку, нажмите кнопку **применить**, а затем нажмите кнопку **следующий параметр**.

    -   Чтобы сохранить изменения и закрыть диалоговое окно, щелкните **ОК**.

    -   Чтобы отменить все несохраненные изменения и закрыть диалоговое окно, щелкните **отменить**.
