# <a name="running-wsus-replica-mode"></a>Работа в режиме реплики служб WSUS

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

На сервере WSUS, работающем в режиме реплики наследуют утверждения обновлений и группы компьютеров, созданные на сервере администрирования. В сценарии, который использует режим реплики, у вас обычно есть один сервер администрирования и один или несколько серверов WSUS подчиненных реплики распределены по всему в организации, в зависимости от особенностей ее структуры. Вы утверждаете обновления и создаете группы компьютеров на сервере администрирования, а затем отразит серверы-реплики. Серверы-реплики можно настроить только во время установки служб WSUS, и если вы реализовали этот сценарий, вполне вероятно, так как очень важно в вашей организации, утверждения обновлений и групп компьютеров осуществляется централизованно.

Если сервер WSUS работает в режиме реплики, можно будет выполнять только ограниченное Администрирование на сервере, на который в основном состоит из:

-   Добавление и удаление компьютеров из группы компьютеров. Членство в группе компьютеров не распространяется на серверы-реплики, группирует только на компьютере, сами. Таким образом в режиме сервер-реплика будет наследовать группы компьютеров, которые созданы на сервере администрирования. Однако группы компьютеров будет пустым. Затем необходимо назначить клиент компьютеров, подключающихся к серверу реплики для группы компьютеров.

-   Настройка расписания синхронизации

-   Настройка параметров прокси сервера

-   Настройка источника обновления. Это может быть сервере, отличном от сервера администрирования

-   Просмотр доступных обновлений

-   Обновление мониторинга, синхронизации, состояние компьютера и параметров служб WSUS на сервере

-   Под управлением WSUS всех стандартных отчетов, доступных на серверы-реплики
