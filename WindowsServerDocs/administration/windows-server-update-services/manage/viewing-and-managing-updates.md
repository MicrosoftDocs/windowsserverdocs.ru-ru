# <a name="viewing-and-managing-updates"></a>Просмотр обновлений и управление ими

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Для просмотра и управления обновлениями, можно использовать консоль WSUS.

## <a name="viewing-updates"></a>Просмотр обновлений
На **обновления** страницы, можно сделать следующее:

-   Просмотр обновлений. Обзор обновления отображаются обновления, которые были синхронизированы из источника обновлений с сервером WSUS и доступные для утверждения.

-   Фильтрация обновлений. В представлении по умолчанию может фильтровать обновления, статус утверждения и состояние установки. По умолчанию задается для неутвержденных обновлений, требуются некоторые клиенты или который были сбои установки на некоторых клиентах. В этом представлении можно изменить, изменив фильтры утверждения состояние и установки состояния, а затем щелкнув **обновить**.

-   Создайте новые представления обновления. В **действия** панели щелкните **новое представление обновления**. Вы можете фильтровать обновления по классификации, продукта, группы, для которого они были утверждены и дату синхронизации. Список можно отсортировать, щелкнув соответствующий заголовок столбца в строке заголовка.

-   Поиск обновлений. Можно поиск отдельного обновления или набор обновлений, заголовок, описание, статье базы знаний или номер Microsoft Security Response Center для обновления.

-   Просмотр сведений о состоянии и журнал изменений для каждого обновления.

-   Утверждение и отклонение обновлений.

#### <a name="to-view-updates"></a>Просмотр обновлений

1.  В консоли администрирования WSUS разверните узел обновления и выберите пункт все обновлений.

2.  По умолчанию обновления отображаются их title, классификации, установлен или не применимо процент и состояние утверждения. Если вы хотите отобразить обновление дополнительных или других свойств, щелкните правой кнопкой мыши заголовок столбца и выберите соответствующие столбцы.

3.  Чтобы отсортировать по различным критериям, например состояние загрузки, title, классификации, Дата выпуска или состояние утверждения, щелкните соответствующий заголовок столбца.

#### <a name="to-filter-the-list-of-updates-displayed-on-the-updates-page"></a>Чтобы отфильтровать список обновлений, которые отображаются на странице "обновления"

1.  В консоли администрирования WSUS разверните узел **обновления**, а затем нажмите кнопку **все обновления**.

2.  В центральной области рядом с полем **утверждения**, выберите состояние нужные утверждения и рядом с полем **состояние** выберите состояние требуемой установки. Нажмите кнопку **Refresh**(Обновить).

#### <a name="to-create-a-new-update-view-on-wsus"></a>Чтобы создать новое представление обновления в WSUS

1.  В консоли администрирования WSUS разверните узел **обновления**, а затем нажмите кнопку **все обновления**.

2.  В **действия** панели щелкните **новое представление обновления**.

3.  В **добавить представление обновить** окна в разделе **шаг 1: выберите свойства**, выберите свойства, необходимые для фильтрации представления обновления:

    -   Выбранные обновления затрагивает конкретный класс для фильтрации по принадлежащих обновлений для одного или нескольких классификаций обновлений.

    -   Выберите обновления выполняются для конкретного продукта, следует выполнить фильтрацию обновлений для одного или нескольких продуктов и семейств продуктов.

    -   Выберите обновления, утвержденные для определенной группы отфильтровать обновления, утвержденные для одного или нескольких групп компьютеров.

    -   Выберите обновления были синхронизированы в течение конкретного периода времени для фильтрации обновлений, синхронизация в определенное время.

    -   Выберите обновления — это обновления WSUS следует выполнить фильтрацию обновлений WSUS.

4.  В разделе **шаг 2: изменение свойств**, выберите подчеркнутые слова, чтобы выбрать нужные значения.

5.  В разделе **шаг 3: Укажите имя**, присвойте новому представлению имя.

6.  Нажмите кнопку **ОК**.

Новое представление будет отображаться в области представления дерева в группе обновлений. Он отображается, как и стандартные представления, в центральной области во время выбора.

#### <a name="to-search-for-an-update"></a>Поиск обновления

1.  Выберите **обновления** узла (или любой узел).

2.  В **действия** панели щелкните **поиска**.

3.  В **поиска** окна на **обновления** введите условия поиска. Вы можете использовать текст из **Title**, **описание**, и **номер статьи базы знаний Майкрософт (KB)** поля. Каждый из этих элементов — это свойство, перечисленные на **сведения** вкладку в свойствах обновления.

#### <a name="to-view-the-properties-for-an-update"></a>Чтобы просмотреть свойства для обновления

1.  В консоли администрирования WSUS разверните узел **обновления**, а затем нажмите кнопку **все обновления**.

2.  В списке обновлений выберите обновление, которое вы хотите просмотреть.

3.  В нижней области вы увидите в другое свойство разделах:

    -   В строке заголовка отображается название обновления. Например, безопасности обновления для Windows Media Player 9 (KB911565).

    -   В разделе состояния отображается состояние установки обновления (компьютеры, на которых должен быть установлен, компьютеры, на которых он был установлен с ошибками, компьютеры, на которых установлен или не применяется и компьютеров, которые не были получены Состояние обновления), а также общие сведения (Дата выпуска номера КБ и MSRC, и т.д.).

    -   В разделе описания отображается краткое описание обновления.

    -   Дополнительные сведения о разделе отображаются следующие сведения:

        -   Режим установки обновления (ли его удалить, запрашивает перезагрузку, требует ввода данных пользователем или необходимо установить монопольно).

        -   Имеет ли обновление условия лицензионного соглашения на использование программного обеспечения Майкрософт

        -   Продукты, к которым применяется обновление

        -   Обновления, заменяющие данное обновление

        -   Обновления, заменяемые этим обновлением

        -   Языки, поддерживаемые обновления

        -   Идентификатор обновления

Обратите внимание на то, что эту процедуру можно выполнить только один обновления за раз. Если выбрать несколько обновлений, только первого обновления в списке будет отображаться в **свойства** области.

## <a name="managing-updates-with-wsus"></a>Управление обновлениями с помощью служб WSUS
Обновления используются для обновления или предоставляя замена всего файла для программного обеспечения, установленного на компьютере. Каждое обновление, которое доступно в центре обновления Майкрософт состоит из двух компонентов:

-   Метаданные: Предоставляет сведения об обновлении. Например метаданные содержат информацию о свойствах обновления, тем самым давая возможность узнать о возможностях обновления полезно. В состав метаданных входят также условия лицензионного соглашения на использование программного обеспечения Майкрософт. Пакет метаданных, загружаемых для обновления, обычно намного меньше, чем файловый пакет фактического обновления.

-   Файлы обновления: Фактические файлы, необходимые для установки обновления на компьютере.

Когда обновления синхронизируются на сервере WSUS, файлы метаданных и файлы обновлений хранятся в двух разных расположениях. Метаданные хранятся в базе данных WSUS. Файлы обновлений могут храниться на сервере WSUS или на серверах центра обновления Майкрософт, в зависимости от того, как вы настроили параметры синхронизации. Если вы решили хранить файлы обновлений на серверах центра обновления Майкрософт, только метаданные загружается во время синхронизации; утверждения обновлений через консоль WSUS, а затем клиентские компьютеры файлы обновления непосредственно из центра обновления Майкрософт во время установки. Дополнительные сведения о возможностях хранения обновлений, см. разделе [1.3. Выбор стратегии хранения данных WSUS](../plan/plan-your-wsus-deployment.md#BKMK_1.3.) шаг 1: Подготовка к развертыванию WSUS, в руководстве по развертыванию WSUS.

Вы хотите Настройка и запуск синхронизации, добавление компьютеров и групп компьютеров и развертывания обновлений на регулярной основе. Ниже представлены примеры общих задач, которые может предпринять при обновлении компьютеров с помощью служб WSUS.

-   Определение общего обновления плана управления на основе топологии сети и пропускной способности, потребностей компании и организационной структуры.

    -   Нужно ли настраивать иерархии серверов WSUS и структуру иерархии.

    -   Группирует компьютер для создания и назначение компьютеров к ним (серверный или клиентский нацеливания).

    -   Какие базы данных для обновления метаданных (например, Внутренняя база данных Windows, SQL Server).

    -   Ли обновления должны быть синхронизированы, автоматически и в какое время.

-   Установка параметров синхронизации, такие как источник обновлений, продукта и обновления классификации, язык, параметры подключения, место хранения и расписание синхронизации.

-   Получите обновления и связанные метаданные на сервере WSUS через синхронизацию из центра обновления Microsoft или восходящего WSUS сервера.

-   Утверждение или отклонение обновлений. У вас есть возможность предоставления пользователям установить сами обновления (если они являются локальными администраторами на своих клиентских компьютерах).

-   Настройка автоматических утверждений. Можно также настроить, хотите ли вы включить автоматического утверждения изменения существующих обновлений или утверждать изменения вручную. Если вы решили вручную утверждать изменения, затем сервер WSUS будет продолжена, с помощью более старой версии, пока вы вручную утвердить новой редакции.

-   Проверьте состояние обновлений. Можно просмотреть состояние обновления, распечатать отчет о состоянии или Настройка электронной почты для регулярного отчеты.

## <a name="update-products-and-classifications"></a>Обновления продуктов и классификаций
Обновления, доступные в центре обновления Майкрософт различаются по продукту (или семейства продуктов) и классификации.

Продукт представляет собой определенную версию операционной системы или приложения, например, Windows Server 2012. Семейство продуктов — это базовая операционная система или приложение, производными которого являются отдельные продукты. Пример семейство продуктов — Microsoft Windows, членом которого является Windows Server 2012. Можно выбрать продуктов и семейств продуктов, для которых требуется настроить сервер для синхронизации обновлений. Можно указать семейство продуктов или отдельные продукты в семействе. Выбрав любой продукт или семейство продуктов, получите обновления для текущих и будущих версий продукта.

Классификации обновлений представляют тип обновления. Для каждого конкретного продукта или семейства продуктов обновления может быть доступны из различных классификаций обновлений (например, Windows 7 семейства критические обновления и обновления для системы безопасности). В следующей таблице перечислены категории обновлений.

| Классы обновлений  | Описание   |
|--|--|
|Критические обновления|Исправления, выпускаемые для устранения важных, не связанный с безопасностью связанные ошибки.|
|Обновления определений|Обновления определений вирусов или других файлов определений.|
|Драйверы|Компоненты программного обеспечения, предназначенных для поддержки нового оборудования.|
|Пакеты дополнительных компонентов|Новые выпуски компонентов, обычно свертываются продуктов со следующим выпуском.|
|Обновления для системы безопасности|Исправления, выпускаемые для конкретных продуктов, проблемы безопасности.|
|Пакеты обновления|Накопительные наборы, все исправления, обновления для системы безопасности, критические обновления и обновления, созданные с момента выпуска продукта. Пакеты обновления также может содержать ограниченное число запрошенных изменений структуры и функции.|
|Инструменты|Служебные программы или функции, помогающие выполнять задачи или набора задач.|
|Накопительные пакеты обновления|Совокупное набор исправлений, обновления для системы безопасности, критические обновления и другие обновления, объединенных в один пакет для более удобного развертывания. Накопительный пакет обычно предназначен для определенной области, например безопасности или определенный компонент, такие как Internet Information Services (IIS).|
|Обновления|Исправления, выпускаемые для устранения некритические, не связанный с безопасностью связанные ошибки.|
