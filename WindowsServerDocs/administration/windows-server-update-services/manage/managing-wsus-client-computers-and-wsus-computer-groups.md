# <a name="managing-wsus-client-computers-and-wsus-computer-groups"></a>Управление клиентскими компьютерами и группами компьютеров служб WSUS

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Узел компьютеры — это центральная точка доступа в консоли администрирования WSUS для управления клиентскими компьютерами и устройствами. В этом узле вы найдете различные группы, которые вы настроили (плюс группу по умолчанию неназначенные компьютеры).

## <a name="managing-client-computers"></a>Управление клиентскими компьютерами
Выбрав один из группы компьютеров в **компьютеров** узле **параметры** приводит к компьютерам в этой группе, для отображения в области сведений. Если компьютеру назначен нескольким группам, он будет отображаться в списке обе группы. Если в списке выберите компьютер, вы увидите его свойства, в том числе Общие сведения о компьютере и состояния обновлений, таких как установки или обнаружения состояния обновления для конкретного компьютера. Список компьютеров в группе данного компьютера можно фильтровать по состоянию. Значение по умолчанию показаны только компьютеры для обновлений, необходимых или который были сбои установки; Тем не менее можно отфильтровать отображения с любым состоянием. Нажмите кнопку **обновить** после изменения состояния фильтра.

Можно также управлять группы компьютеров, на странице компьютеров, которая включает в себя создание групп и назначения им компьютеров. Дополнительные сведения об управлении группами компьютеров, см. в разделе Управление группами компьютеров в далее разделе данного руководства, а раздел [1.5. Планирование групп компьютеров WSUS](../plan/plan-your-wsus-deployment.md#15-plan-wsus-computer-groups) на шаге 1: Подготовка к развертыванию WSUS руководства по развертыванию WSUS.

> [!NOTE]
> Во-первых, необходимо настроить клиентские компьютеры к серверу WSUS, прежде чем их можно будет управлять с этого сервера. До выполнения этой задачи, сервер WSUS не распознает клиентских компьютеров, и они не отображаются в списке на странице "компьютеры". Дополнительные сведения о настройке клиентских компьютеров см. в разделе [1.5. Планирование групп компьютеров WSUS](../plan/plan-your-wsus-deployment.md#15-plan-wsus-computer-groups) шаг 1: Подготовка к развертыванию службы WSUS чтобы шаг 3: Настройка служб WSUS, в руководстве по развертыванию WSUS.

## <a name="controlling-when-wsus-client-computers-install-updates"></a>Управление, когда клиентских компьютеров WSUS устанавливают обновления
Существует два способа управления клиентских компьютеров WSUS установки обновлений:

-   Утверждения с крайними сроками: Крайние сроки строго ограничивает при установке обновления

-   WSUS групповых политик: Групповые политики управления, когда агент обновления Windows проверяет и устанавливает обновления

    Дополнительные сведения см. в разделе: [Шаг 5. Настройка параметров групповой политики для автоматического обновления](../deploy/4-configure-group-policy-settings-for-automatic-updates.md), в руководстве по развертыванию WSUS.

## <a name="managing-computer-groups"></a>Управление группами компьютеров
Служба WSUS позволяет направлять обновления на группы клиентских компьютеров, таким образом обеспечивается получение определенными компьютерами нужных обновлений в самое подходящее время. Например, если компьютеры в одном отделе (например, бухгалтерии) имеют особую конфигурацию, то можно настроить группу для данного отдела, определить, какие обновления нужны на их компьютерах и в какое время их нужно устанавливать, а затем использовать отчеты WSUS для оценки данных обновлений.

Компьютеры всегда входят в **все компьютеры** группирование и остаются назначенными **неназначенные компьютеры** группе, пока вы не добавите их в другую группу. Компьютеры могут входить в несколько групп.

Можно создать иерархические группы компьютеров (например, создать группу "Зарплата" и группу "Расчеты с поставщиками" в группе "Бухгалтерия"). Обновления, одобренные для высшей группы будут автоматически развернуты на группы нижнего уровня, а также к самой группе, более поздней версии. Таким образом Если утвердить обновление 1 для этой группы, обновление будет развертываться на все компьютеры этой группы, все компьютеры в группе заработной платы и все компьютеры в группе Расчеты с поставщиками.

Поскольку компьютеры могут состоять в нескольких группах, то возможна ситуация, когда одно обновление будет одобрено несколько раз на одном и том же компьютере. Тем не менее само обновление будет развернуто только один раз, а любые конфликты будут разрешены сервером WSUS. Чтобы продолжить в примере выше, если компьютер a назначается заработной платы и Расчеты с поставщиками группы, а обновление 1 одобрено для обеих групп, он будет развернут только один раз.

Для добавления компьютеров в группы можно использовать один из двух методов: указание на стороне сервера и указание на стороне клиентов. При указании на стороне сервера вручную переместить один или несколько клиентских компьютеров одной группы компьютеров одновременно. При указании на стороне клиента используется групповая политика или изменяются настройки реестра на клиентских компьютерах, чтобы эти компьютеры для автоматического добавления клиентских компьютеров в ранее созданную в группы. Этот процесс можно в скрипт и развернуть на нескольких компьютерах, за один раз. Укажите целевой метод, используемый на сервере WSUS, выбрав один из двух вариантов на **компьютеров** раздел **параметры** страницы.

> [!NOTE]
> Если сервер WSUS работает в режиме реплики, группы компьютеров нельзя создать на сервере. Все группы компьютеров, необходимые для клиентов сервера-реплики должны создаваться на сервере WSUS, который является корневым элементом иерархии сервера WSUS. Дополнительные сведения о режиме реплики см. в разделе [режим реплики WSUS под управлением](running-wsus-replica-mode.md) Дополнительные сведения о серверных и клиентских целевой настройке см. в разделе [1.5. Планирование групп компьютеров WSUS](../plan/plan-your-wsus-deployment.md#15-plan-wsus-computer-groups) шаг 1: Подготовка к развертыванию WSUS в руководстве по развертыванию WSUS.
