# <a name="chcp"></a>chcp



Изменение текущей кодовой страницы. При использовании без параметров, **chcp** номер кодовой страницы.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
chcp [<NNN>]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|\<NNN>|Задает кодовую страницу.|
|/?|Отображение справки в командной строке.|

В следующей таблице перечислены каждого Поддерживаемые кодовые страницы и страны или региона или языка:

|Кодовая страница|Страны или региона или языка|
|---------|--------------------------|
|437|США|
|850|Многоязыковой (Latin I)|
|852|Восточная и Центральная Европа (латиница II)|
|855|Кириллица (русская)|
|857|Турецкий|
|860|Португальский|
|861|Исландский|
|863|Французский (Канада)|
|865|Nordic|
|866|Русский|
|869|Греческая современная|
|936|Китайский|

## <a name="remarks"></a>Примечания

-   Только изготовителя оборудования (OEM) кодовую страницу, входящую в состав Windows правильно отображается в окне командной строки, который использует растровые шрифты. В полноэкранном режиме или в командной строке windows, использующих шрифты TrueType, другие кодовые страницы отображаются правильно.
-   Необходимо подготовить кодовые страницы (как в MS-DOS).
-   Программы, которые запускаются после того как вы назначить новый вариант использования страницы код новую страницу кода. Однако программы (за исключением Cmd.exe), прежде чем начать назначать новый код страницы ключевое исходная кодовая страница.

## <a name="BKMK_examples"></a>Примеры

Для просмотра текущей кодовой страницы, введите:
```
chcp
```
Появится сообщение следующего вида:

`Active code page: 437`

Чтобы изменить активную кодовую страницу 850 (латиница), введите следующую команду:
```
chcp 850
```
Если указанная кодовая страница является недопустимым, появится следующее сообщение об ошибке:

`Invalid code page`
