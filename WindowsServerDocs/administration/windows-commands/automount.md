# <a name="automount"></a>automount

Область применения. Windows Server (Semi-Annual Channel), Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

- [Условные обозначения синтаксиса команд командной строки](command-line-syntax-key.md)

> [!IMPORTANT]
> В конфигурациях сети хранения данных (SAN) отключение автоматического подключения запрещает Windows автоматически подключать или назначать буквы дисков для любых новых базовых томов, видимых системе.

## <a name="syntax"></a>Синтаксис

Автоподключение [{Enable | Disable | очистки}] [Noerr]

### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| enable | Позволяет Windows автоматически подключать новые базовые и динамические тома, добавленные в систему, и назначать им буквы диска. |
| disable | Не позволяет Windows автоматически подключать новые базовые и динамические тома, добавленные в систему.<p>**Примечание**. отключение автоподключения может привести к сбою отказоустойчивых кластеров в части хранилища мастера проверки конфигурации. |
| Ведите | Удаляет каталоги точек подключения томов и параметры реестра для томов, которые больше не находятся в системе. Это позволяет предотвратить автоматическую присоединение томов, которые ранее входили в систему, и присвоить их старым точкам подключения тома при их добавлении в систему. |
| Noerr | Только для сценариев. При возникновении ошибки DiskPart продолжит обрабатывать команды, как если бы ошибка не возникала. Без этого параметра ошибка приводит к выходу из программы DiskPart с кодом ошибки. |

## <a name="examples"></a>Примеры

Чтобы узнать, включена ли функция автоматического подключения, введите в команду diskpart следующие команды:

```
automount
```

Чтобы включить функцию автоматического подключения, введите:

```
automount enable
```

Чтобы отключить функцию автоматического подключения, введите:

```
automount disable
```
