# <a name="wbadmin-start-recovery"></a>Wbadmin start восстановления



Запускает операцию восстановления на основе указанных параметров.

Для выполнения восстановления с помощью этой команды, необходимо быть членом **операторы архива** группы или **Администраторы** группа, или должна была быть делегированы соответствующие разрешения. Кроме того, необходимо запустить **wbadmin** из командной строки с повышенными правами. (Чтобы открыть командную строку с повышенными правами, нажмите кнопку **запустить**, щелкните правой кнопкой мыши **командной строки**, а затем нажмите кнопку **Запуск от имени администратора**.)

Примеры по использованию этого, см. в разделе [примеры](#BKMK_Examples).

## <a name="syntax"></a>Синтаксис

```
wbadmin start recovery
-version:<VersionIdentifier>
-items:{<VolumesToRecover> | <AppsToRecover> | <FilesOrFoldersToRecover>}
-itemtype:{Volume | App | File}
[-backupTarget:{<VolumeHostingBackup> | <NetworkShareHostingBackup>}]
[-machine:<BackupMachineName>]
[-recoveryTarget:{<TargetVolumeForRecovery> | <TargetPathForRecovery>}]
[-recursive]
[-overwrite:{Overwrite | CreateCopy | Skip}]
[-notRestoreAcl]
[-skipBadClusterCheck]
[-noRollForward]
[-quiet]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|-версии|Указывает идентификатор версии для восстановления резервной копии в мм/дд/гггг-формате чч: мм. Если вы не знаете идентификатор версии, введите **wbadmin получить версии**.|
|-элементы|Задает разделенный запятыми список томов, приложений, файлы или папки для восстановления.</br>Если **- itemtype** — **тома**, можно указать только один том, предоставляя букву диска тома, точка подключения тома или имя тома на основе GUID.</br>Если **- itemtype** — **приложение**, можно указать только одно приложение. Для восстановления, необходимо зарегистрировать приложение с помощью архивации данных Windows Server. Можно также использовать значение **ADIFM** для восстановления установки Active Directory. Дополнительные сведения см. в примечаниях.</br>Если **- itemtype** — **файл**, можно указать файлы или папки, но они должны быть частью тот же том, и они должны быть в одной родительской папке.|
|-itemtype|Задает тип элементов для восстановления. Должно быть **тома**, **приложения**, или **файл**.|
|-backupTarget|Указывает расположение хранилища, содержащий нужную резервную копию для восстановления. Этот параметр полезен в тех случаях, когда расположение отличается от обычно хранения резервных копий этого компьютера.|
|-machine|Задает имя компьютера, на котором необходимо выполнить восстановление резервной копии. Этот параметр полезен в тех случаях, когда в том же расположении будут заархивированы нескольких компьютеров. Следует использовать, когда **- backupTarget** указан параметр.|
|-recoveryTarget|Указывает расположение для восстановления. Этот параметр полезен, если это расположение находится отличается от расположения архивов, созданных ранее. Он также используется для восстановления томов, файлов или приложений. При восстановлении тома, можно указать альтернативный том букву диска тома. При восстановлении файлов или приложений, можно указать альтернативное расположение восстановления.|
|рекурсивные-|Данный параметр является допустимым, только в том случае, при восстановлении файлов. Восстанавливает файлы в папках и всех файлов, подчиненных в указанные папки. По умолчанию восстанавливаются только те файлы, которые находятся непосредственно в указанные папки.|
|-Перезаписать|Данный параметр является допустимым, только в том случае, при восстановлении файлов. Указывает действие, выполняемое, когда файл, в которой выполняется восстановление уже существует в том же расположении.</br>-   **Пропустить** вызывает систему архивации данных Windows Server пропустить его и продолжить восстановление следующего файла.</br>-   **CreateCopy** вызывает систему архивации данных Windows Server создать копию существующего файла, чтобы его не изменяется.</br>-   **Перезаписать** приводит к архивации данных Windows Server для перезаписи существующего файла с файлом из резервной копии.|
|-notRestoreAcl|Данный параметр является допустимым, только в том случае, при восстановлении файлов. Указывает не восстановление безопасности списки управления доступом (ACL) файлов, выполняется восстановление из резервной копии. По умолчанию будут восстановлены списки управления доступом системы безопасности (значение по умолчанию — **true)**. Если этот параметр используется, списки ACL для восстановленных файлов будут наследоваться из расположения, к которой восстанавливаются файлы.|
|-skipBadClusterCheck|Данный параметр является допустимым, только в том случае, при восстановлении тома. Пропуск проверки диски, на которые производится восстановление поврежденного кластера сведения. Если выполняется восстановление на другой сервер или оборудование, мы рекомендуем этот параметр не используется. Можно вручную выполнить команду **chkdsk /b** на этих дисках в любое время для проверки их поврежденные кластеры и затем соответствующим образом обновить сведений о файловой системе.</br>Внимание! Пока вы не выполните **chkdsk** как было сказано, поврежденные кластеры, указанное в отчетах для восстановленного системы может оказаться неточным.|
|-noRollForward|Данный параметр является допустимым, только в том случае, при восстановлении приложения. Если выбирается самая поздняя версия из резервных копий позволяет предыдущей операции восстановления на момент времени приложения. Для других версий приложения, которые не являются восстановления на момент последней и предыдущий, выполняется по умолчанию.|
|-quiet|Запускает подкоманды без вывода сообщений для пользователя.|

## <a name="remarks"></a>Примечания

-   Чтобы просмотреть список элементов, доступных для восстановления из определенной резервной копии версии, используйте **wbadmin получить элементы**. Если том не было подключения точки или буквы диска во время резервного копирования, подкоманды возвратит имя тома на основе GUID, который должен использоваться для восстановления тома.
-   Когда **- itemtype** — **приложения**, можно использовать значение **ADIFM** для **-элемент** для выполнения установки из эксплуатации мультимедиа, чтобы восстановить все связанные данные, необходимые для доменных служб Active Directory. **ADIFM** создает копию базы данных Active Directory, реестра и состояние SYSVOL, а затем сохраняет эту информацию в расположении, заданном параметром **- recoveryTarget**. Используйте этот параметр, только если **- recoveryTarget** указан.

>     [!NOTE]
>     Before using **wbadmin** to perform an install from media operation, you should consider using the **ntdsutil** command because **ntdsutil** only copies the minimum amount of data needed, and it uses a more secure data transport method.

## <a name="BKMK_Examples"></a>Примеры

Чтобы выполнить восстановление резервной копии из 31 марта 2013 г., сделанной в 9:00 утра, из тома d:, введите:
```
wbadmin start recovery -version:03/31/2013-09:00 -itemType:Volume -items:d:
```
Для выполнения восстановления на диск d резервной копии из 31 марта 2013 г., сделанной в 9:00 утра, реестра, введите:
```
wbadmin start recovery -version:03/31/2013-09:00 -itemType:App -items:Registry -recoverytarget:d:\
```
Для запуска восстановления резервной копии из 31 марта 2013 г., сделанной в 9:00 утра, d:\folder и папок, подчиненных d:\folder, введите следующую команду:
```
wbadmin start recovery -version:03/31/2013-09:00 -itemType:File -items:d:\folder -recursive
```
Для запуска восстановления резервной копии из 31 марта 2013 г, сделанной в 9:00 утра, тома \\ \\? \Volume{cc566d14-44a0-11d9-9d93-806e6f6e6963}\, типа:
```
wbadmin start recovery -version:03/31/2013-09:00 -itemType:Volume 
-items:\\?\Volume{cc566d14-44a0-11d9-9d93-806e6f6e6963}\
```
Для запуска восстановления резервной копии с 30 апреля 2013 г, сделанной в 9:00 утра, общей папки \\ \\servername\share из server01, введите:
```
wbadmin start recovery -version:04/30/2013-09:00 -backupTarget:\\servername\share -machine:server01
```
