# <a name="macfile"></a>macfile

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Управляет файлового сервера для Macintosh серверы, тома, каталоги и файлы. Можно автоматизировать задачи администрирования, включая ряд команд в пакетных файлах и запустив их вручную или в заранее определенное время. 
-   [Чтобы изменить каталоги в томами](#BKMK_Moddirs)
-   [Объединение данных файла Macintosh и компонентов ресурсов](#BKMK_Joinforks)
-   [Изменение сообщения при входе и ограничение сеансов](#BKMK_LogonLimit)
-   [Чтобы добавить, изменить или удалить тома](#BKMK_addvol)

## <a name="BKMK_Moddirs"></a>Чтобы изменить каталоги в томами

### <a name="syntax"></a>Синтаксис
```
macfile directory[/server:\\<computerName>] /path:<directory> [/owner:<OwnerName>] [/group:<GroupName>] [/permissions:<Permissions>]
```

### <a name="parameters"></a>Параметры
-   / Server:\\\\<computerName> Сервер, на котором изменяется в каталог. Если этот параметр опущен, операция выполняется на локальном компьютере.
-   / PATH:<directory> Обязательный. Указывает путь к каталогу, который вы хотите изменить. Этот каталог должен существовать. **каталог MacFile** не создает каталоги.
-   / Owner:<OwnerName> изменяет владельца каталога. Если этот параметр опущен, владелец не изменяется.
-   / Group:<GroupName> Задает или изменяет основную группу Macintosh, которая связана с каталогом. Если этот параметр опущен, основная группа остается неизменным.
-   /Permissions:<Permissions> Задает разрешения на папку для владельца, основную группу и всех остальных. 11 цифр используется для задания разрешений. Предоставляется разрешение номер 1 и 0 отменяет разрешение (например, 11111011000). Если этот параметр опущен, разрешения не изменяются.
    Положение цифры определяет набор разрешений, как описано в следующей таблице.

    |Положение|Задает разрешение для|
    |------|------------|
    |Первая|OwnerSeeFiles|
    |Вторая|OwnerSeeFolders|
    |Третья|OwnerMakechanges|
    |Четвертая|GroupSeeFiles|
    |Пятый|GroupSeeFolders|
    |Шестой|GroupMakechanges|
    |Седьмой|WorldSeeFiles|
    |Восьмой|WorldSeeFolders|
    |Девятый|WorldMakechanges|
    |Десятый|Каталог нельзя переименовать, переместить или удалить.|
    |Одиннадцатый|Изменения применяются в текущем каталоге и всех подкаталогах.|

-   /?
    Отображение справки в командной строке.

### <a name="remarks"></a>Примечания
- Если сведения, которые вы предоставляете содержит пробелы или специальные символы, используйте кавычки вокруг текста (например, **"** <em>компьютера имя</em> **"** ).
- Используйте **macfiledirectory** для предоставления существующий каталог в томе, доступном для Macintosh для пользователей Macintosh. **Macfiledirectory** команда не создает каталоги. Используйте диспетчер файлов, в командной строке или **новой папки macintosh** команду, чтобы создать каталог в томе, доступном для Macintosh, прежде чем использовать **macfile directory** команды.
  ### <a name="BKMK_Examples"></a>Примеры
  В следующем примере изменяется разрешения продаж подкаталог мая, в том Macintosh статистики, на диске E локального сервера. В примере назначается см. в разделе файлы, папки, см. в разделе и внесите изменения разрешения владельца и см. в разделе файлов и папок, см. в разделе разрешений для всех других пользователей, позволять каталоге переименованы, перемещены или удалены.
  ```
  macfile directory /path:"e:\statistics\may sales" /permissions:11111011000
  ```

## <a name="BKMK_Joinforks"></a>Объединение данных файла Macintosh и компонентов ресурсов

### <a name="syntax"></a>Синтаксис
```
macfile forkize[/server:\\<computerName>] [/creator:<CreatorName>] [/type:<typeName>]  [/datafork:<Filepath>] [/resourcefork:<Filepath>] /targetfile:<Filepath>
```

### <a name="parameters"></a>Параметры

|         Параметр          |                                                                                                           Описание                                                                                                            |
|----------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| / Server:\\\\<computerName> |                                                            Указывает сервер, на котором будут объединены файлы. Если этот параметр опущен, операция выполняется на локальном компьютере.                                                            |
|   / Creator:<CreatorName>   |                                      Задает источник файла. Macintosh finder использует **/Creator** параметр командной строки для определения приложения, создавшего файл.                                       |
|      / Type:<typeName>      |                                 Указывает тип файла. Macintosh finder использует **/type** параметр командной строки, чтобы определить тип файла в приложении, созданном файле.                                 |
|    /datafork:<Filepath>    |                                                                   Указывает расположение данных, предназначенного для объединения. Можно указать удаленный путь.                                                                   |
|  /resourcefork:<Filepath>  |                                                                 Указывает расположение ресурсов, предназначенного для объединения. Можно указать удаленный путь.                                                                 |
|   /targetfile:<Filepath>   | Обязательный. Указывает расположение файла, который создается путем объединения данных и файла ресурсов, или задает расположение файла, тип и вы изменяете источник. Файл должен находиться на указанном сервере. |
|             /?             |                                                                                               Отображение справки в командной строке.                                                                                               |

### <a name="remarks"></a>Примечания
- Если сведения, которые вы предоставляете содержит пробелы или специальные символы, используйте кавычки вокруг текста (например, **"** <em>компьютера имя</em> **"** ).

### <a name="examples"></a>Примеры
Чтобы создать файл treeapp на том, доступный для Macintosh d:\Release» из файла ресурсов C:\Cross\Mac\Appcode и сделать этот новый файл, который отображается для клиентов Macintosh, как приложение (приложений Macintosh используют тип РИМЕНИТЬ) с автором (подпись ) присвоено MAGNOLIA, тип:
```
macfile forkize /resourcefork:c:\cross\mac\appcode /type:APPL /creator:MAGNOLIA /targetfile:D:\Release\treeapp
```
Можно переключиться Microsoft Word 5.1, создатель файла для файла WOrd.txt в файлах documents\Group D:\Word каталога, на сервере \\\SERverA, введите:
```
macfile forkize /server:\\servera /creator:MSWD /type:TEXT /targetfile:"d:\Word documents\Group files\Word.txt"
```

## <a name="BKMK_LogonLimit"></a>Изменение сообщения при входе и ограничение сеансов
### <a name="syntax"></a>Синтаксис
```
macfile server [/server:\\<computerName>] [/maxsessions:{Number | unlimited}] [/loginmessage:<Message>]
```

### <a name="parameters"></a>Параметры

|               Параметр                |                                                                                                                                                                           Описание                                                                                                                                                                            |
|----------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|       / Server:\\\\<computerName>       |                                                                                                                        Сервер, на котором будут изменены параметры. Если этот параметр опущен, операция выполняется на локальном компьютере.                                                                                                                         |
| /maxsessions: {номер &#124; неограниченное} |                                                                                         Указывает максимальное число пользователей, которые можно одновременно использовать файл и серверы печати для Macintosh. Если этот параметр опущен, **maxsessions** параметр для сервера остается неизменным.                                                                                         |
|        / loginmessage:<Message>         | изменения пользователей Macintosh сообщений см. в разделе при входе в систему в файловый сервер для Macintosh. Максимальное количество символов для сообщения является 199. Если этот параметр опущен, **прежним** сообщения для сервера остается неизменным. Чтобы удалить существующее сообщение, включите **/loginmessage** параметра, но оставьте *сообщение* переменной пустое. |
|                   /?                   |                                                                                                                                                               Отображение справки в командной строке.                                                                                                                                                               |

### <a name="remarks"></a>Примечания
- Если сведения, которые вы предоставляете содержит пробелы или специальные символы, используйте кавычки вокруг текста (например, **"** <em>компьютера имя</em> **"** ).

### <a name="examples"></a>Примеры
Чтобы изменить количество файлов и сервер печати для Macintosh сеансов, которые разрешены на локальном сервере с текущего значения пяти сеансов и добавления сообщения «Выйдите из сервера для Macintosh завершении», введите:
```
macfile server /maxsessions:5 /loginmessage:"Log off from Server for Macintosh when you are finished."
```

## <a name="BKMK_addvol"></a>Чтобы добавить, изменить или удалить тома
### <a name="syntax"></a>Синтаксис
```
macfile volume {/add|/set} [/server:\\<computerName>] /name:<volumeName>/path:<directory>[/readonly:{true | false}] [/guestsallowed:{true | false}] [/password:<Password>] [/maxusers:{<Number>>|unlimited}]
macfile volume /remove[/server:\\<computerName>] /name:<volumeName>
```

### <a name="parameters"></a>Параметры

|              Параметр               |                                                                                                                                                                       Описание                                                                                                                                                                        |
|--------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|          {/add &#124; /set}          |                                                                                                                      Требуется, если при добавлении или изменении тома Macintosh. Добавляет или изменяет указанного тома.                                                                                                                       |
|      / Server:\\\\<computerName>      |                                                                                                             Сервер, на котором следует добавить, изменить или удалить том. Если этот параметр опущен, операция выполняется на локальном компьютере.                                                                                                              |
|          / Name:<volumeName>          |                                                                                                                                          Обязательный. Указывает имя тома добавлены, изменены или удалены.                                                                                                                                           |
|          / PATH:<directory>           |                                                                                                                Требуется и используется только в том случае, при добавлении тома. Указывает путь к корневой каталог тома, который требуется добавить.                                                                                                                 |
|    /readonly:{true &#124; false}     | Указывает, могут ли пользователи изменять файлы в томе. Введите значение true, чтобы указать, что пользователи не могут изменять файлы в томе. Введите значение false, чтобы указать, что пользователи могут изменять файлы в томе. Если не указано, при добавлении тома, допускаются изменения файлов. При изменении тома, **readonly** для тома не изменяется. |
|  /guestsallowed:{true &#124; false}  |      Указывает, может ли входе как гостевые пользователи использовать тома. Введите значение true, чтобы указать, что гости могут использовать тома. Введите значение false, чтобы указать, что Гости не может использовать тома. Если не указано, при добавлении тома, гости могут использовать том. При изменении тома, **guestsallowed** для тома не изменяется.       |
|         / Password:<Password>         |                                                                               Указывает пароль, которые потребуются для доступа к тому. Если не указано, при добавлении тома, пароль не создается. Если не указано, при изменении тома, пароль не изменяется.                                                                               |
| /maxusers:{<Number>>&#124;unlimited} |                                                 Указывает максимальное число пользователей, которые могут одновременно использовать файлы в томе. Если не указано, при добавлении тома, неограниченное количество пользователей можно использовать тома. При изменении тома, **maxusers** значение остается неизменным.                                                 |
|               /remove                |                                                                                                                                Требуется при удалении тома Макинтоша. Удаляет указанного тома.                                                                                                                                |
|                  /?                  |                                                                                                                                                           Отображение справки в командной строке.                                                                                                                                                           |

### <a name="remarks"></a>Примечания
- Если сведения, которые вы предоставляете содержит пробелы или специальные символы, используйте кавычки вокруг текста (например, **"** <em>компьютера имя</em> **"** ).

### <a name="examples"></a>Примеры
Чтобы создать том с именем US Marketing Statistics на локальном сервере, используя папку Stats на диске E и чтобы указать, что том не может получить доступ гостей, введите:
```
macfile volume /add /name:"US Marketing Statistics" /guestsallowed:false /path:e:\Stats
```
Изменить объем созданную только для чтения и требуется пароль и задать максимальное количество пользователей пятью, введите:
```
macfile volume /set /name:"US Marketing Statistics" /readonly:true /password:saturn /maxusers:5
```
Чтобы добавить том вызывается на сервере разработки Альбомная \\\Magnolia, с помощью деревьев каталог на диске E, а также указать, что тома возможен Гости, тип:
```
macfile volume /add /server:\\Magnolia /name:"Landscape Design" /path:e:\trees
```
Чтобы удалить том с именем отчеты о продажах на локальном сервере, введите следующую команду:
```
macfile volume /remove /name:"Sales Reports"
```
