# <a name="break"></a>break



Отменяет связь на томе теневого копирования VSS и делает его доступным как обычным томом. Том может осуществляться с помощью буквы диска (Если назначена) или имя тома. При использовании без параметров, **break** отображает справку в командной строке.

> [!NOTE]
> Эта команда относится только к аппаратных теневых копий после импорта.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
break [writable] <SetID>
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|[для записи]|Включает чтения и записи на томе.|
|\<SetID >|Указывает идентификатор набор теневого копирования.|

## <a name="remarks"></a>Примечания

-   Предоставленный тома, как теневые копии, которые происходят из, являются только для чтения по умолчанию.
-   Псевдоним идентификатор теневой копии, который хранится в виде переменной среды с **загрузить метаданные** команды, можно использовать в *SetID* параметра.

## <a name="BKMK_examples"></a>Примеры

Чтобы сделать тень копирования с именем псевдонима псевдоним1 уровень доступности, как доступный для записи тома в операционной системе, введите следующую команду:
```
break writable %Alias1%
```

> [!NOTE]
> Доступ к тому устанавливается напрямую к поставщику оборудования без записи была теневого копирования тома.
