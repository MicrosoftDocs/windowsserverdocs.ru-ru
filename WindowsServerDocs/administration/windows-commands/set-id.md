# <a name="set-id"></a>Идентификатор набора

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Команда Diskpart задать идентификатор изменяет поле типа раздела для раздела, имеющего фокус.  
  
> [!IMPORTANT]  
> Эта команда предназначен для использования вычислительной техники \(сборщикам систем\) только. Изменение поля типа секции с помощью этого параметра может привести к компьютер к сбою или не удалось выполнить загрузку. Если вы используете изготовителей Оборудования или с таблицей, не изменяйте поля типа секции на GPT-дисках с помощью этого параметра. Всегда используйте [Создание раздела efi](create-partition-efi.md) команду, чтобы создать системных разделов EFI, [создать секции msr](create-partition-msr.md) команду, чтобы создать такие разделы и [Создание первичный раздел](create-partition-primary.md) команду без параметра ID для создания основных разделов на GPT-дисках.  
  
  
  
## <a name="syntax"></a>Синтаксис  
  
```  
set id={ <byte> | <GUID> } [override] [noerr]  
```  
  
## <a name="parameters"></a>Параметры  
  
| Параметр |                                                                                                                                                                                                                                                                                                                                                                   Описание                                                                                                                                                                                                                                                                                                                                                                   |
|-----------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|  <byte>   |                                                                                                                                                                                                       для основной загрузочной записью \(MBR\) дисков, указывает новое значение для поля типа в виде шестнадцатеричного числа, для секции. С помощью этого параметра, за исключением типа 0x42, который указывает раздел LDM можно указать любой байт типа раздела. Обратите внимание на то, что начальные 0 x опущен, при указании типа шестнадцатеричное секции.                                                                                                                                                                                                       |
|  <GUID>   | для таблицы разделов GUID \(gpt\) дисков, указывает новое значение GUID для типа поля для секции. Идентификаторы GUID распознанного включают:<br /><br />-Системный раздел EFI: c12a7328\-f81f\-11d 2\-ba4b\-00a0c93ec93b<br />-Базовый раздел данных: ebd0a0a2\-b9e5\-4433\-87c 0\-68b6b72699c7<br /><br />Любой тип GUID можно задать с помощью этого параметра, за исключением следующих:<br /><br />-Такие: e3c9e316\-0b5c\-4db8\-817d\-f92df00215ae<br />-Раздел метаданных LDM на динамический диск: 5808c8aa\-7e8f\-42e0\-85d 2\-e1e90434cfb3<br />-Раздел данных LDM на динамическом диске: af9b60a0\-1431\-4f62\-bc68\-3311714a69ad<br />-Кластера раздела метаданных: db97dba9\-0840\-4bae\-97f0\-ffb9a327c7e1 |
| Переопределение  |                                                                заставляет в файловой системе в том, чтобы отключить перед изменением типа раздела. При запуске **идентификатор набора** команды DiskPart пытается заблокировать и отключить файловой системы на томе. Если **переопределить** не указан, и происходит сбой вызова для блокировки файловой системы \(к примеру, так как имеется открытый дескриптор\), операция завершится ошибкой. Когда **переопределить** указан, программа DiskPart отключение даже в том случае, если происходит сбой вызова для блокировки файловой системы, и все открытые дескрипторы тома станут недействительными.<br /><br />Эта команда доступна только для Windows 7 и Windows Server 2008 R2.                                                                 |
|   успешного   |                                                                                                                                                                                                                                                                    Используется только для сценариев. При обнаружении ошибки, DiskPart продолжает обрабатывать команды, как если бы ошибки не было. Без этого параметра по ошибке будет возобновлено DiskPart завершить работу с кодом ошибки.                                                                                                                                                                                                                                                                    |
  
## <a name="remarks"></a>Примечания  
  
-   Кроме ограничений, упомянутых выше, DiskPart не проверяет допустимость вами значением \(за исключением того, чтобы удостовериться в наличии байт в шестнадцатеричной форме или GUID\).  
  
-   Эта команда не работает на динамических дисках или на резервный раздел Майкрософт.  
  
## <a name="BKMK_examples"></a>Примеры  
Значение поля типа 0x07 и принудительно файловой системы, чтобы отключить, введите:  
  
```  
set id=0x07 override  
```  
  
Чтобы задать поле типа основного раздела диска, введите следующую команду:  
  
```  
set id=ebd0a0a2-b9e5-4433-87c0-68b6b72699c7  
```
