# <a name="bcdedit"></a>bcdedit



Файлы данных конфигурации загрузки обеспечивают хранилище, которое используется для описания приложений загрузки и их параметров. Объекты и элементы хранилища успешно заменяют Boot.ini.

BCDEdit является средством командной строки для управления хранилища данных конфигурации загрузки. Он может использоваться для различных целей, включая создание новых хранилищ, изменение существующих хранилищ, добавив параметры меню загрузки и т. д. Программа BCDEdit выполняет те же функции, как Bootcfg.exe в более ранних версиях Windows, но имеет два существенных преимущества:
-   Предоставляет более широкий диапазон параметров загрузки, чем Bootcfg.exe.
-   Улучшена поддержка сценариев.

> [!NOTE]
> Использование BCDEdit для изменения BCD требуются права администратора.

BCDEdit является основным средством для изменения конфигурации загрузки операционной системы Windows Vista и более поздних версиях Windows. Он входит в состав Windows Vista распределение в папке %WINDIR%\System32.

BCDEdit ограничена стандартных типов данных и предназначен для выполнения обычных изменений в файл данных конфигурации. Для более сложных операций или нестандартные данные типов рассмотрите возможность использования BCD инструментария управления Windows (WMI) интерфейс программирования (API) для создания более мощные и гибкие средства.

## <a name="syntax"></a>Синтаксис

```
BCDEdit /Command [<Argument1>] [<Argument2>] ...
```

## <a name="parameters"></a>Параметры

### <a name="general-bcdedit-command-line-option"></a>Общие параметры командной строки BCDEdit

|Параметр|Описание|
|------|-----------|
|/?|Отображает список команд BCDEdit. Выполните следующую команду без аргумента Сводка доступных команд. Чтобы отобразить подробную справку для определенной команды, запустите **bcdedit /?** \<Команда >, где <command> является имя команды, нужно найти дополнительные сведения о. Например **bcdedit /? createstore** выводит подробную справку по команде Createstore.|

### <a name="parameters-that-operate-on-a-store"></a>Параметры, которые оперируют Store

|Параметр|Описание|
|------|-----------|
|/createstore|Создает новое хранилище данных конфигурации загрузки пустой. Созданное хранилище не является системным.|
|/ export|Экспортирует содержимое системы хранилища в файл. Этот файл может использоваться для восстановления состояния системного хранилища. Эта команда может применяться только для системного хранилища.|
|/ import|Восстанавливает состояние системного хранилища с помощью файла резервной копии данных, созданного с помощью **/export** параметр. Эта команда удаляет существующие записи в хранилище системы перед выполнением операции импорта. Эта команда может применяться только для системного хранилища.|
|/ Store|Этот параметр можно использовать с большинством команд BCDedit для указания хранилища для использования. Если этот параметр не указан, программа BCDEdit работает системного хранилища. Под управлением **bcdedit/Store** команда сама по себе эквивалентно запуску команды **bcdedit/enum active** команды.|

### <a name="parameters-that-operate-on-entries-in-a-store"></a>Параметры для работы с записями в Store

|Параметр|Описание|
|---------|-----------|
|/ Copy|Создает копию указанной записи в том же хранилище системы.|
|/ create|Создает новую запись в хранилище данных конфигурации загрузки. Если известный идентификатор указан, то **приложении/**, **/ наследовать**, и **/Device** нельзя указывать параметры. Если идентификатор не указан или не хорошо известен, **приложении/**, **/ наследовать**, или **/Device** необходимо указать параметр.|
|/delete|Удаляет элемент из указанной записи.|

### <a name="parameters-that-operate-on-entry-options"></a>Параметры, которые работают с параметрами записей

|Параметр|Описание|
|---------|-----------|
|/deletevalue|Удаляет указанный элемент из загрузочную запись.|
|/ set|Задает значение параметра записи.|

### <a name="parameters-that-control-output"></a>Параметры, управляющие выходных данных

|Параметр|Описание|
|---------|-----------|
|/ Enum|Перечисляет записи в хранилище. **/Enum** параметр значение по умолчанию для команды BCEdit, поэтому выполнение команды **bcdedit** команду без параметров эквивалентен запуску **bcdedit/enum active** команды.|
|/v|Режим подробного протоколирования. Как правило все известные идентификаторы записей представлены понятными краткими формами. Указание **/v** как параметр командной строки отображает все идентификаторы в полном объеме. Под управлением **bcdedit /v** команда сама по себе эквивалентно запуску команды **bcdedit/enum active /v** команды.|

### <a name="parameters-that-control-the-boot-manager"></a>Параметры, управляющие диспетчера загрузки

|Параметр|Описание|
|---------|-----------|
|/ bootsequence|Указывает одноразовую последовательность отображения для использования при следующей перезагрузке. Эта команда аналогична **/DISPLAYORDER** запускает параметр, за исключением того, что он используется только при следующем запуске компьютера. После этого компьютер возвращается к последовательность отображения.|
|/ Default|Указывает запись, диспетчером загрузки истечения времени ожидания по умолчанию.|
|/ DISPLAYORDER|Указывает порядок отображения, используемый диспетчером загрузки при отображении параметров загрузки для пользователя.|
|/ timeout|Указывает время ожидания в секундах, перед загрузкой ОС запись по умолчанию.|
|/TOOLSDISPLAYORDER|Указывает порядок отображения для диспетчера загрузки, используемый при отображении **средства** меню.|

### <a name="parameters-that-control-emergency-management-services"></a>Параметры, которые управляют службы аварийного управления

|Параметр|Описание|
|---------|-----------|
|/ bootems|Включает или отключает службы аварийного управления (EMS) для указанной записи.|
|/ EMS|Включает или отключает EMS для указанной операционной системой.|
|/emssettings|Задает глобальные параметры EMS для компьютера. **/emssettings** не включить или отключить EMS для определенной записи загрузки.|

### <a name="parameters-that-control-debugging"></a>Параметры, управляющие отладки

|Параметр|Описание|
|---------|-----------|
|/bootdebug|Включает или отключает отладку загрузки для указанной записи. Несмотря на то, что эта команда работает для любой записи загрузки, он действует только для загрузки приложений.|
|/dbgsettings|Задает или отображает глобальные параметры для системы. Эта команда не включает и отключает отладку ядра; Используйте **/debug** параметр для этой цели. Чтобы задать параметры отдельных глобального отладчика, используйте **bcdedit/set** \<dbgsettings > <type> <value> команда.|
|/debug|Включает или отключает отладчик ядра для указанной записи.|
