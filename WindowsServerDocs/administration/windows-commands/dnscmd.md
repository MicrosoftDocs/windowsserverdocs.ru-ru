# <a name="dnscmd"></a>Dnscmd

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Интерфейс командной строки для управления DNS-серверами. Эта служебная программа используется в пакетных файлах для автоматизации повседневных задач управления DNS или для выполнения простой автоматической установки и настройки DNS-серверов в сети сценариев.  
## <a name="syntax"></a>Синтаксис  
```  
dnscmd <ServerName> <command> [<command parameters>]  
```  
## <a name="parameters"></a>Параметры  

|  Параметр   |                         Описание                          |
|--------------|--------------------------------------------------------------|
| <ServerName> | IP-адрес или имя удаленного или локального DNS-сервер. |

### <a name="BKMK_1"></a>dnscmd /ageallrecords  
Задает текущее время на отметку времени для записей ресурсов в указанной зоны или узел на DNS-сервере.  
#### <a name="syntax"></a>Синтаксис  
```  
dnscmd [<ServerName>] /ageallrecords <ZoneName>[<NodeName>] | [/tree]|[/f]  
```  
#### <a name="parameters"></a>Параметры  
<ServerName><br/>Указывает DNS-сервер, который администратор планирует управления, представленный IP-адрес, полное доменное имя (FQDN) или имя узла. Если этот параметр опущен, используется локальный сервер.<br/><ZoneName><br/>Указывает полное ДОМЕННОЕ имя зоны.<br/><NodeName><br/>Указывает конкретный узел или поддерево в зоне. *NodeName* указывает узел или поддерево в зону с помощью следующих:<br/>-@ корневая зона или полное доменное имя<br/>— Полное ДОМЕННОЕ имя узла (имя с точки (.) в конце)<br/>-Одну метку для имени относительно корня зоны<br/>/Tree<br/>Указывает, что все дочерние узлы также получать метку времени.<br/>/f<br/>Выполняет команду без запроса на подтверждение.<br/>### "Примечания"<br/>- **Ageallrecords** команда является для обеспечения обратной совместимости между текущей версии службы DNS и предыдущих версиях DNS, в котором очистки не поддерживались. Он добавляет метку времени, с текущим временем записей ресурсов, которые имеют отметку времени, и задает текущее время на записи ресурсов, которые имеют отметку времени.<br/>-Очистку записей не появляется, если записи являются отметкой времени. Сервера имен (NS) записей ресурсов, в начале записей ресурсов начальной записи и записи ресурсов службы имя Windows Internet (WINS) не включаются в процесс очистки и они не являются отметкой времени, даже когда **ageallrecords**выполняется команда.<br/>— Эта команда завершается ошибкой, если очистка включена для DNS-сервер и зону. Сведения о том, как включить очистку для зоны, см. в разделе **срокам** параметр синтаксисе уровня зоны в <a href="#BKMK_3" data-raw-source="[config](#BKMK_3)">config</a> команды.<br/>— Добавление отметки времени для записей ресурсов DNS делает их несовместимы с DNS-серверов, работающих в операционных системах, отличных от Windows 2000, Windows XP или Windows Server 2003. Метка времени, добавленные с помощью **ageallrecords** команду нельзя отменить.<br/>— Если не указан ни один из необязательных параметров, то команда возвращает все записи в указанном узле. Если задано значение хотя бы для одного из необязательных параметров, **dnscmd** перечисляет только записей ресурсов, которые соответствуют значению или значения, заданные в параметрах или необязательный параметр.<br/>### Пример<br/>См. в разделе <a href="https://technet.microsoft.com/library/cc784399(v=ws.10).aspx" data-raw-source="[Example 1: Set the current time on a time stamp to resource records](https://technet.microsoft.com/library/cc784399(v=ws.10).aspx)">пример 1: Присваивает текущее время на отметку времени для записей ресурсов</a>.<br/>### <a name="BKMK_2"></a>dnscmd/ClearCache<br/>Очищает кэш-памяти DNS записей ресурсов на указанном сервере DNS.<br/>### Синтаксис<br/><code>dnscmd [&lt;ServerName&gt;] /clearcache</code><br/>### Параметров<br/><ServerName><br/>Указывает DNS-сервер для управления, представленный IP-адрес, полное доменное имя или имя узла. Если этот параметр указан, все серверы, на которых размещены эту зону можно очистить его.<br/>### "Примечания"<br/>— По умолчанию все серверы, на которых размещаются зоны можно очистить этой зоны.<br/>-Если зоны размещается на нескольких DNS-сервер, эта команда используется для уменьшения количества раз в сутки зоны.<br/>-Очистки необходимо включить на DNS-сервер и зону, затронутых этой командой.<br/>### Пример использования<br/><code>dnscmd dnssvr1.contoso.com /zoneresetscavengeservers test.contoso.com 10.0.0.1 10.0.0.2</code><br/>### <a name="BKMK_34"></a>dnscmd /zoneresetsecondaries<br/>Указывает список IP-адреса дополнительных серверов, на которые главный сервер отвечает, ответ на запрос для передачи зоны.<br/>### Синтаксис<br/><code>dnscmd [&lt;ServerName&gt;] /zoneresetsecondaries &lt;ZoneName&gt; {/noxfr | /nonsecure | /securens | /securelist &lt;SecurityIPaddresses&gt;} {/nonotify | /notify | /notifylist &lt;NotifyIPaddresses&gt;}</code><br/>### Параметров<br/>**<ServerName>**<br/>Указывает DNS-сервер для управления, представленный IP-адрес, полное доменное имя или имя узла. Если параметр опущен, используется локальный сервер.<br/>**<ZoneName>**<br/>Указывает имя зоны, будет иметь его серверы-получатели сброса.<br/>**/noxfr | /nonsecure | /securens | /securelist <SecurityIPaddresses>**<br/>Задает ли все или только некоторые из серверов-получателей, запрос обновления получения обновления.<br/>**/noxfr**<br/>Указывает, что разрешены без передачи зоны.<br/>**/ небезопасные**<br/>Указывает, что предоставлены всех запросов на передачу зон.<br/>**/securens**<br/>Указывает, что перенос предоставлены только на сервере, который указан в записи ресурса имени сервера (NS) для зоны.<br/>**/securelist**<br/>Указывает, что предоставлены передачи зоны только список серверов. Этот параметр следует указать IP-адрес или адреса, используемые главным сервером.<br/>**<SecurityIPaddresses>**<br/>Список IP-адресов, которые получать передачи зон с главного сервера. Этот параметр используется только с **/SecureList** параметра.<br/>**/nonotify | / Уведомлять | /notifylist <NotifyIPaddresses>**<br/>Указывает, что уведомление об изменениях отправляется только для определенных серверов-получателей:<br/>**/nonotify**<br/>Указывает, что уведомления не изменения отправляются с серверами-получателями.<br/>**/ уведомления**<br/>Отправка уведомлений об изменениях на всех серверах-получателях.<br/>**/notifylist**<br/>Отправка уведомлений об изменениях в список серверов. Эта команда следует указать IP-адрес или адреса, используемые главным сервером.<br/>**<NotifyIPaddresses>**<br/>Указывает IP-адреса сервера-получателя или нескольких серверов, какие изменения будут отправлены. Этот список используется только с **/notifylist** параметра.<br/>### "Примечания"<br/>— Используйте **zoneresetsecondaries** команду на главном сервере, чтобы указать, как оно ответит запросов на передачу зон из серверов-получателей.<br/>### Пример использования<br/><pre>dnscmd dnssvr1.contoso.com /zoneresetsecondaries test.contoso.com /noxfr /nonotify

Если этот параметр опущен, используется локальный сервер.<br/>**<ZoneName>**<br/>Определяет зону, на котором изменяется тип.<br/>**<Zonetype>**<br/>Указывает тип зоны, для создания. Каждый тип имеет различные необходимые параметры:<br/>**/ DsPrimary**<br/>Создает зоны, интегрированной в active directory.<br/>**/ primary/file <FileName>**<br/>Создает стандартной основной зоны.<br/>**или вторичных <MasterIPaddress> [,<MasterIPaddress>...]**<br/>Создает стандартный дополнительной зоны.<br/>**/ stub <MasterIPaddress>[,<MasterIPaddress>...] / file <FileName>**<br/>Создает зона-заглушка с файловой поддержкой.<br/>**/dsstub <MasterIPaddress>[,<MasterIPaddress>...]**<br/>Создает зоны заглушки интегрированной с active directory.<br/>**/forwarder &lt;главный_IP-адрес [,<MasterIPaddress>]... / file<FileName>**<br/>Указывает, что созданный зоны перенаправляет неразрешенные запросы на другой сервер DNS.<br/>**/dsforwarder**<br/>Указывает, что созданный интегрированной зоне active directory перенаправляет неразрешенные запросы на другой сервер DNS.<br/>**/overwrite_mem | /overwrite_ds**<br/>Указывает, как перезаписать существующие данные:<br/>**/overwrite_mem**<br/>Перезаписывает данные DNS из данных в Доменных службах Active Directory.<br/>**/overwrite_ds**<br/>Перезаписывает существующие данные в Доменных службах Active Directory.<br/>### "Примечания"<br/>— Задание типа зоны как **/dsforwarder** создается зона, которая выполняет условное перенаправление.<br/>### Пример использования<br/><pre>dnscmd dnssvr1.contoso.com /zoneresettype test.contoso.com /primary /file test.contoso.com.dns  
dnscmd dnssvr1.contoso.com /zoneresettype second.contoso.com /secondary 10.0.0.2</pre><br/>### <a name="BKMK_35"></a>dnscmd /zoneresume<br/>запускает указанной зоны, которая ранее была приостановлена.<br/>### Синтаксис<br/><code>dnscmd &lt;ServerName&gt; /zoneresume &lt;ZoneName&gt;</code><br/>### Параметров<br/>**<ServerName>**<br/>Указывает DNS-сервер для управления, представленный IP-адрес, полное доменное имя или имя узла.
