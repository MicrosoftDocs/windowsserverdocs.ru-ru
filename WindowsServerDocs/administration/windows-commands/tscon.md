# <a name="tscon"></a>tscon

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Подключается к другому сеансу на сервере узла сеансов удаленных рабочих столов (rd узла сеансов).  
Примеры для использования этой команды, см. в разделе [примеры](#BKMK_examples).  

> [!NOTE]  
> В Windows Server 2008 R2 службы терминалов были переименованы на службы удаленных рабочих столов. Чтобы найти новые возможности в последней версии, см. в разделе [какие возможности служб удаленных рабочих столов в Windows Server 2012](https://technet.microsoft.com/library/hh831527) в технической библиотеке Windows Server.  

## <a name="syntax"></a>Синтаксис  
```  
tscon {<SessionID> | <SessionName>} [/dest:<SessionName>] [/password:<pw> | /password:*] [/v]  
```  
## <a name="parameters"></a>Параметры  
|Параметр|Описание|  
|-------|--------|  
|\<SessionID >|Указывает идентификатор сеанса, к которому необходимо подключиться. Если вы используете необязательный **/dest:**<*SessionName*> параметра, это идентификатор сеанса, к которому необходимо подключиться.|  
|\<Имя сеанса >|Указывает имя сеанса, к которому необходимо подключиться.|  
|/dest:\<SessionName>|Задает имя текущего сеанса. Этот сеанс будет отключен при подключении к новому сеансу.|  
|/password:\<pw>|Указывает пароль для пользователя, владеющего сеанса, к которому необходимо подключиться. Этот пароль необходим, если подключающийся пользователь не является владельцем сеанса.|  
|/ Password: *|запрашивает пароль пользователя, владеющего сеанса, к которому необходимо подключиться.|  
|/v|Отображает сведения о выполненных действиях.|  
|/?|Отображение справки в командной строке.|  

## <a name="remarks"></a>Примечания  
-   Необходимо иметь разрешение на полный доступ или подключения специального разрешения для подключения к другому сеансу.  
-   **/Dest:**<*SessionName*> позволяет подключить сеанс другого пользователя к другому сеансу.  
-   Если вы не укажете пароль в <*пароль*> параметра и нужном сеансе принадлежит пользователю, отличный от текущего, **tscon** завершается ошибкой.  
-   Не удается подключиться к сеансу консоли.  

## <a name="BKMK_examples"></a>Примеры  
-   Чтобы подключиться к сеансу 12 на текущем сервере узла сеансов удаленных рабочих столов и отключиться от текущего сеанса, введите следующую команду:  
    ```  
    tscon 12  
    ```  
-   Чтобы подключиться к сеансу 23 на текущем сервере узла сеансов удаленных рабочих столов с помощью пароль mypass и отключиться от текущего сеанса, введите следующую команду:  
    ```  
    tscon 23 /password:mypass  
    ```  
-   Чтобы подключиться с именем TERM03 сеанс с именем TERM05 сеанс, а затем отключить сеанс TERM05, если он подключен, введите:  
    ```  
    tscon TERM03 /v /dest:TERM05  
    ```
