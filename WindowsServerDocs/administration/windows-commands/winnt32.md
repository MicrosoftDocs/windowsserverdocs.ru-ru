# <a name="winnt32"></a>winnt32

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Выполняет установку или обновление до продукта в Windows Server 2003. Можно запустить **winnt32** в командной строке на компьютере под управлением Windows 95, Windows 98, Windows Millennium edition, Windows NT, Windows 2000, Windows XP или продукт в Windows Server 2003. При запуске **winnt32** на компьютере под управлением Windows NT версии 4.0, необходимо установить пакет обновления 5 или более поздней версии.
## <a name="syntax"></a>Синтаксис
```
winnt32 [/checkupgradeonly] [/cmd: <CommandLine>] [/cmdcons] [/copydir:{i386|ia64}\<FolderName>] [/copysource: <FolderName>] [/debug[<Level>]:[ <FileName>]] [/dudisable] [/duprepare: <pathName>] [/dushare: <pathName>] [/emsport:{com1|com2|usebiossettings|off}] [/emsbaudrate: <BaudRate>] [/m: <FolderName>]  [/makelocalsource] [/noreboot] [/s: <Sourcepath>] [/syspart: <DriveLetter>] [/tempdrive: <DriveLetter>] [/udf: <ID>[,<UDB_File>]] [/unattend[<Num>]:[ <AnswerFile>]]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|/ checkupgradeonly|Проверяет компьютер на предмет совместимости обновления с продуктами в Windows Server 2003.<br /><br />Если вы используете этот параметр вместе с **/ unattend**, вмешательство пользователя не требуется.  В противном случае результаты отображаются на экране, и их можно сохранить в указанную вами имя файла. Имя файла по умолчанию — **upgrade.txt** в корневом каталоге.|
|/ cmd|Программа установки для выполнения определенной команды до окончательного этапа установки. Это происходит после перезагрузки компьютера и после сбора необходимых сведений о конфигурации программы установки, но до завершения установки.|
|\<CommandLine >|Указывает, должно быть выполнено до последней фазы установки команд в командной строке.|
|/ cmdcons|На компьютере x86 устанавливает консоль восстановления как вариант загрузки.  Консоль восстановления — это интерфейс командной строки, с помощью которого можно выполнять задачи, такие как запуск и остановка служб, а также доступ к локальному диску (включая с файловой системой NTFS). Можно использовать только **/cmdcons** параметр после завершения установки.|
|/ copydir|Создает дополнительную папку внутри папки, в которой установлены файлы операционной системы.  Например, для x86 и x64-разрядных компьютерах, можно создать папку с именем *Private_drivers* в исходной папке i386 вашей установки, а также место файлы драйверов в папке. Тип **/copydir:i386\\*** Private_drivers* программа установки скопируйте эту папку на новый компьютер, что новое расположение папки **systemroot** \\*Private_drivers *.<br /><br />-   **i386** указывает i386<br />-   **IA64** указывает ia64<br /><br />Можно использовать **/copydir** можно создать столько дополнительных папок.|
|\<Имя папки >|Указывает папку, созданную для хранения изменений веб-узла.|
|/ copysource|создает временную дополнительную папку внутри папки, в которой установлены файлы операционной системы. Можно использовать **/copysource** можно создать столько дополнительных папок.<br /><br />В отличие от папки **/copydir** создает, **/copysource** папки удаляются после завершения установки.|
|/debug|Создание журнала отладки на заданном уровне, например, **/debug4:Debug.log**.  Этот файл журнала по умолчанию является **C:\ systemroot\winnt32.log**, и|
|\<уровень >|Уровень значения и описания<br /><br />-   0: Серьезные ошибки<br />-   1: Ошибки<br />-   2: Уровень по умолчанию. Предупреждения<br />-   3: Информация<br />-4: подробные сведения для отладки<br /><br />Каждый уровень включает все уровни под ним.|
|/ dudisable|Предотвращает запуск динамического обновления. Без динамического обновления программа установки выполняется только с исходными файлами установки. Этот параметр будет отключить динамическое обновление, даже в том случае, если использовать файл ответов и указать параметры динамического обновления в этом файле.|
|/ duprepare|Выполнение операций по подготовке в общей папке установки, чтобы его можно использовать с файлами динамического обновления, загруженные из центра обновления Windows, веб-сайта. Затем этого общего ресурса можно использовать для установки Windows XP для нескольких клиентов.|
|\<pathName>|Указывает полное имя пути.|
|/dushare|Указывает общий ресурс, на котором ранее загруженные файлы динамического обновления (обновленные файлы, используемые программой установки), центра обновления Windows, и которая ранее была **/duprepare: *** < путь >*. При запуске на клиентском компьютере, указывает, что сделает установку клиента используйте обновленных файлов в общей папке, указанной в <pathName>.|
|/ emsport|Включает или отключает службы аварийного управления во время установки, и после установки операционной системы. С помощью службы удаленного управления можно удаленно управлять сервером в аварийных ситуациях, в которых обычно необходимы клавиатура, мышь и монитор, например когда сеть недоступна или сервер не работает должным образом. Службы аварийного управления имеет собственные требования к оборудованию и доступна только для продуктов в Windows Server 2003.<br /><br />-   **COM1** применимо только для x86 компьютеров (не Itanium на базе архитектуры).<br />-   **COM2**применимо только для x86 компьютеров (не Itanium на базе архитектуры).<br />-По умолчанию. Использует параметры, указанные в таблице BIOS последовательный порт консоли перенаправления (SPCR), или, в архитектуре-на основе Itanium, с помощью пути устройства консоли EFI. Если указать **usebiossettings** и нет таблицы SPCR или подходящий путь устройства консоли EFI, Serices аварийного управления не будет включен.<br />-   **Отключение** отключает службы аварийного управления. Позже его можно включить, изменив параметры загрузки.|
|/EMSBAUDRATE|для компьютеров с x86 указывает скорость в бодах служб аварийного управления. (Параметр не применимы для компьютеров на базе архитектуры Itanium.) Необходимо использовать с **/emsport:com1** или **/emsport:com2** (в противном случае **/emsbaudrate** игнорируется).|
|\<Значение BaudRate >|Указывает скорость 9600, 19200, 57600 или 115200. 9600 используется по умолчанию.|
|/m|Указывает, что программа установки копирует файлы замены из альтернативного источника.  Программа установки для поиска в другом месте, во-первых и если файлы находятся представления, чтобы использовать их вместо файлы из расположения по умолчанию.|
|/ makelocalsource|Программа установки, чтобы скопировать все исходные файлы установки на локальный жесткий диск.  Используйте **/makelocalsource** при установке с компакт-диска, чтобы предоставить установочные файлы в том случае, если недоступен компакт-диск позднее в ходе установки.|
|/noreboot|Программа установки не перезагружать компьютер после завершения фазы копирования файлов программы установки для запуска другой команды.|
|/s|Задает исходное расположение файлов для установки. Для копирования файлов с нескольких серверов, введите **/s:**\<Sourcepath > параметр несколько раз (не более восьми). Если ввести параметр несколько раз, первый из указанных серверов должен быть доступен, иначе установка будет невозможна.|
|\<SourcePath >|Указывает полный исходный путь.|
|/ Syspart|На компьютере с x86 указывает, что можно скопировать файлы на жесткий диск, пометить диск как активный и затем установить диск на другом компьютере. При запуске компьютера, он автоматически переходит к следующему этапу установки.<br /><br />Необходимо всегда использовать **/tempdrive** параметр с **/Syspart** параметра.<br /><br />Можно запустить **winnt32** с **/Syspart** параметр на x86 компьютере под управлением Windows NT 4.0, Windows 2000, Windows XP или продукт в Windows Server 2003. Если компьютер работает под управлением Windows NT версии 4.0, необходим пакет обновления 5 или более поздней версии. На компьютере не может выполняться в Windows 95, Windows 98 или Windows Millennium edition.|
|\<Буква диска >|Указывает букву диска.|
|/ tempdrive|Указывает для размещения временных файлов в указанной секции.<br /><br />для новой установки операционной системы также будет устанавливаться в указанный раздел.<br /><br />При обновлении **/tempdrive** параметр влияет на размещение только временные файлы; операционная система обновляется в секции, из которой запускается **winnt32**.|
|/udf|Указывает идентификатор (\<идентификатор >), программа установки использует для указания способа изменения файла ответов файлом базы данных уникальности (UDB) (см. в разделе **/ unattend** параметр).  UDB изменяет значения в файле ответов, а идентификатор определяет используемые значения в файле UDB. Например **команда /udf:Пользователь_RAS,Наша_организация.udb** переопределяет параметры, указанные для Пользователь_RAS в файле Наша_организация.udb. Если не \<UDB-файл > указан, программа установки предложит пользователю вставить диск, содержащий **$Unique$ .udb** файл.|
|\<ИДЕНТИФИКАТОР &GT;|Указывает идентификатор, используемый для указания способа изменения файла ответов файлом базы данных уникальности (UDB).|
|\<UDB_file>|Указывает на файл базы данных уникальности (UDB).|
|/unattend|На компьютере x86 обновление предыдущей версии Windows NT 4.0 Server (с пакетом обновления 5 или более поздней версии) или Windows 2000 в автоматическом режиме. Все пользовательские параметры берутся из предыдущей установки, поэтому вмешательство пользователя не требуется во время установки.|
|\<num >|Указывает количество секунд с момента окончания копирования файлов до момента перезагрузки компьютера. Можно использовать \<Num > на любом компьютере под управлением Windows 98, Windows Millennium edition, Windows NT, Windows 2000, Windows XP или продукт в Windows Server 2003. Если компьютер работает под управлением Windows NT версии 4.0, необходим пакет обновления 5 или более поздней версии.|
|\<Файл_ответов >|Содержит особые спецификации|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания
Если вы развертываете Windows XP на клиентских компьютерах, можно использовать версию winnt32.exe, входящий в состав Windows XP. Другой способ развертывания Windows XP является использование winnt32.msi, в котором работает через установщик Windows, часть IntelliMirror набор технологий. Дополнительные сведения о развертывании клиентов см. в разделе Windows Server 2003 Deployment Kit, который описан в [с помощью развертывания Windows и наборы ресурсов](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx).

На компьютере с процессором Itanium **winnt32** можно запускать из Extensible Firmware Interface (EFI) или Windows Server 2003 Enterprise, Windows Server 2003 R2 Enterprise, Windows Server 2003 R2 Datacenter или Windows Server 2003 Центр обработки данных. Кроме того, на компьютерах с процессором Itanium **/cmdcons** и **/Syspart** недоступны и параметры обновления будут недоступны.
Дополнительные сведения о совместимости оборудования см. в разделе [совместимость оборудования](https://technet.microsoft.com/library/cc757927(v=ws.10).aspx).
Дополнительные сведения об использовании динамического обновления и установке нескольких клиентов, см. в разделе Windows Server 2003 Deployment Kit, который описан в [с помощью развертывания Windows и наборы ресурсов](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx).
сведения об изменении параметров загрузки см. в разделе Развертывание Windows и ресурсов комплекты для Windows Server 2003. Дополнительные сведения см. в разделе [с помощью развертывания Windows и наборы ресурсов](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx).
С помощью **/ unattend** параметр командной строки для автоматизации установки подтверждает, что вы прочитал и принял Microsoft лицензионного соглашения для Windows Server 2003. Прежде чем использовать этот параметр командной строки для установки Windows Server 2003, от имени другой организации, необходимо подтвердить, что конечный пользователь (ли отдельного пользователя или одну сущность) получил, чтение и приняли условия лицензии Microsoft Соглашения для этого продукта.  Изготовители оборудования могут не указать этот ключ на компьютерах, продаваемых конечным пользователям.
