# <a name="prncnfg"></a>prncnfg

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Задает или отображает сведения о конфигурации принтера.

## <a name="syntax"></a>Синтаксис
```
cscript Prncnfg {-g | -t | -x | -?} [-S <ServerName>] [-P <printerName>] [-z <NewprinterName>] [-u <UserName>] [-w <Password>] [-r <PortName>] [-l <Location>] [-h <Sharename>] [-m <Comment>] [-f <SeparatorFileName>] [-y <Datatype>] [-st <starttime>] [-ut <Untiltime>] [-i <DefaultPriority>] [-o <Priority>] [<+|->shared] [<+|->direct] [<+|->hidden] [<+|->published] [<+|->rawonly] [<+|->queued] [<+|->enablebidi] [<+|->keepprintedjobs] [<+|->workoffline] [<+|->enabledevq] [<+|->docompletefirst]
```

## <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|-g|Отображает сведения о конфигурации принтера.|
|-t|Настраивает принтер.|
|-x|Переименовывает принтера.|
|-S \<ServerName\>|Задает имя удаленного компьютера, на котором размещена принтера, которое вы хотите управлять. Если компьютер не указан, используется локальный компьютер.|
|-P \<Имя_принтера\>|Задает имя используемого принтера, которое вы хотите управлять. Обязательный.|
|-z \<NewprinterName\>|Указывает новое имя принтера. Требуется **- x** и **-P** параметров.|
|-u \<UserName\> -w \<Password\>|Задает учетную запись с разрешениями на подключение к компьютеру, на котором размещена принтера, которое вы хотите управлять. Эти разрешения всех членов локальной группы администраторов на конечном компьютере, но можно также выделить разрешения другим пользователям. Если не указать учетную запись, вы должны войти в систему с учетной записью с этими разрешениями для команды для работы.|
|-r \<имя порта\>|Указывает порт, где он подключен. Если это параллельный или последовательный порт, используйте идентификатор порта (например, "LPT1" или "COM1"). Если это порт TCP/IP, используйте имя порта, который был указан при добавлении порта.|
|-l \<расположение\>|Указывает расположение принтера, например «Копировать комнаты.»|
|-h \<Sharename\>|Указывает имя общего принтера.|
|-m \<комментарий\>|Указывает строку комментария принтера.|
|-f \<SeparatorFileName\>|Указывает файл, содержащий текст, отображаемый на странице "Разделитель".|
|-y \<тип данных\>|Указывает типы данных, которые могут принимать принтера.|
|Сент - \<starttime\>|Настройка принтера для ограниченной доступности. Указывает время дня, когда принтер доступен. Если отправить документ на принтер, когда она недоступна, документ является хранится в очереди пока принтер не станет доступен. Необходимо указать время в 24-часовом формате. Например, чтобы указать 23:00:00, введите **2300**.|
|-ut \<Endtime\>|Настройка принтера для ограниченной доступности. Указывает время дня, когда принтер больше не доступен. Если отправить документ на принтер, когда она недоступна, документ является хранится в очереди пока принтер не станет доступен. Необходимо указать время в 24-часовом формате. Например, чтобы указать 23:00:00, введите **2300**.|
|-o \<приоритет\>|Указывает приоритет, который использует диспетчер очереди печати для маршрутизации заданий печати в очереди печати. Очередь печати с более высоким приоритетом получает все задания перед любой очереди с более низким приоритетом.|
|-i \<DefaultPriority\>|Указывает приоритет по умолчанию, назначенный для каждого задания печати.|
|{+&#124;-} общий|Указывает, является ли общим данному принтеру в сети.|
|{+&#124;-}direct|Указывает, следует ли отправлять непосредственно на принтер по документа, минуя.|
|{+&#124;-} опубликованы|Указывает, следует ли опубликовывать принтером в active directory. Если опубликовать принтер, другие пользователи могут искать в зависимости от его расположения и возможности (такие как цвет, печать и сшивание).|
|{+&#124;-} скрытые|Зарезервированные функция.|
|{+&#124;-}rawonly|Указывает, можно ли буферизовано в этой очереди только необработанные данные задания на печать.|
|{+ &#124; -}queued|Указывает, что принтер не должно начинаться печати после последней страницы документа помещаются в очередь. Выполнение программы, недоступна, до окончания печати документа. Тем не менее использование этого параметра гарантирует, что весь документ доступен на принтер.|
|{+ &#124; -}keepprintedjobs|Указывает, должен ли диспетчер очереди сохранить документы после печати. Включение этого параметра пользователь может повторно отправить документ на принтер из очереди печати, а не из программы печати.|
|{+ &#124; -}workoffline|Указывает, является ли пользователь для отправки заданий печати в очередь печати, если компьютер не подключен к сети.|
|{+ &#124; -}enabledevq|Указывает, нужно ли хранить в очереди заданий печати, которые не соответствуют настройках принтера (например, PostScript-файлы не PostScript) вместо того, чтобы печать файла.|
|{+ &#124; -}docompletefirst|Указывает, должна ли диспетчер очереди отправлять задания печати с более низким приоритетом, поступающих перед отправкой задания печати с более высоким приоритетом, которые не были завершены постановки в очередь. Если этот параметр включен, документы не закончивших диспетчер очереди печати будет отправлять документы большего раньше. Этот параметр следует включать, если вы хотите ускорить печать за счет приоритет задания. Если этот параметр отключен, диспетчер очереди всегда отправляет задания с более высоким приоритетом соответствующие очереди сначала.|
|{+ &#124; -}enablebidi|Указывает, отправляет ли принтер сведения о состоянии диспетчер очереди печати.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания
-   **Prncnfg** команда — это сценарий Visual Basic, расположенный в %WINdir%\System32\printing_Admin_Scripts\\ <language> каталога. Чтобы использовать эту команду в командной строке, введите **cscript** следуют полный путь к файлу prncnfg, или изменить каталоги в соответствующую папку. Пример:
    ```
    cscript %WINdir%\System32\printing_Admin_Scripts\en-US\prncnfg
    ```
-   Если сведения, которые вы предоставляете содержит пробелы, используйте кавычки вокруг текста (например, `"computer Name"`).

## <a name="BKMK_examples"></a>Примеры
Чтобы отобразить сведения о конфигурации принтера с именем colorprinter_2 с очередью печати, размещенной на удаленном компьютере с именем HRServer, введите:
```
cscript prncnfg -g -S HRServer -P colorprinter_2 
```

Чтобы настроить принтер с именем colorprinter_2 таким образом, чтобы диспетчер очереди печати на удаленном компьютере с именем HRServer оставляющая задания печати, после их печати, введите:
```
cscript prncnfg -t -S HRServer -P colorprinter_2 +keepprintedjobs 
```

Чтобы изменить имя принтера на удаленном компьютере с именем HRServer из colorprinter_2 colorprinter 3, тип:
```
cscript prncnfg -x -S HRServer -P colorprinter_2 -z "colorprinter 3" 
```
