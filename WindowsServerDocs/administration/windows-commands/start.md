# <a name="start"></a>start



Запускает заданную программу или команду в отдельном окне.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
start ["<Title>"] [/d <Path>] [/i] [{/min | /max}] [{/separate | /shared}] [{/low | /normal | /high | /realtime | /abovenormal | belownormal}] [/affinity <HexAffinity>] [/wait] [/b {<Command> | <Program>} [<Parameters>]]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|"\<Title >»|Указывает заголовок, отображаемый в строке заголовка окна командной строки.|
|/d \<путь >|Указывает каталог запуска.|
|/i|Передает среду загрузки Cmd.exe в новое окно командной строки. Если **/i** не задан, используется текущая среда.|
|/Min  \| /Макс.|Указывает, чтобы свести к минимуму ( **/min**) или полностью развернуть ( **/max**) новое окно командной строки.|
|/ разделения \| / shared|Запуск 16-разрядных программ в отдельной области памяти ( **/разделения**) или в общей памяти ( **/ shared**). Эти параметры не поддерживаются на 64-разрядных платформах.|
|/ низкой \| или обычного \| , высокая \| /realtime \| /abovenormal \| /belownormal|Запускает приложение с указанным приоритетом. Класс значения допустимым приоритетом: **/низкой**, **или обычного**, **, высокая**, **/realtime**, **/abovenormal**, и **/belownormal**.|
|/affinity \<HexAffinity >|Применяет указанный маска соответствия процессоров (выражено в шестнадцатеричном формате) для нового приложения.|
|/ wait|Запускает приложение и ожидает его завершения.|
|/b|Запускает приложение без необходимости открывать новое окно командной строки. CTRL + C обработки игнорируется в том случае, если приложение включает обработку CTRL + C. Используйте сочетание клавиш CTRL + BREAK для остановки приложения.|
|/b \<команда > \| \<программы >|Указывает команду или программу для запуска.|
|\<Параметры >|Указывает параметры для передачи в вызываемой программе.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания

- Неисполняемые файлы можно запустить через сопоставление файла, введя имя файла как команду.
- При выполнении команды, которая содержит строку «CMD» как первый токен без квалификатора расширения или пути, значение переменной COMSPEC заменяется «CMD». Благодаря этому пользователи комплектация **cmd** из текущего каталога.
- При запуске приложения 32-разрядных графический пользовательский интерфейс (GUI), **cmd** не ожидает приложение выйти из программы перед возвратом к командной строке. Это происходит, если приложение запускается из командной строки.
- При выполнении команды, которая использует первый токен, который не содержит расширение, Cmd.exe использует значение переменной среды PATHEXT для определения расширений и в каком порядке. Значение по умолчанию для переменной PATHEXT —:  
  ```
  .COM;.EXE;.BAT;.CMD 
  ```  
  Обратите внимание, что синтаксис совпадает со значением переменной PATH, Start запускает каждое расширение.
- При поиске исполняемого файла, если нет совпадений на любое расширение **запустить** проверяет, соответствует ли имя каталога. В этом случае **запустить** открывает Explorer.exe в той же папке.

## <a name="BKMK_examples"></a>Примеры

Чтобы запустить программу Myapp в командной строке с сохранением использования текущего окна командной строки, введите следующую команду:
```
start myapp 
```
Чтобы просмотреть **запустить** командной строки раздела справки в отдельном в развернутом состоянии окна командной строки, введите:
```
start /max start /?
```
