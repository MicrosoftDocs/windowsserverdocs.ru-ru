# <a name="manage-bde-on"></a>готов: на



Шифрует диск и включает BitLocker. Примеры использования этой команды см. в разделе [примеры](#BKMK_Examples).

## <a name="syntax"></a>Синтаксис

```
manage-bde –on <Drive> {[-recoveryPassword <NumericalPassword>]|[-recoverykey <PathToExternalDirectory>]|[-startupkey <PathToExternalKeyDirectory>]|[-certificate]|
[-tpmandpin]|[-tpmandpinandstartupkey <PathToExternalKeyDirectory>]|[-tpmandstartupkey <PathToExternalKeyDirectory>]|[-password]|[-ADAccountOrGroup <Domain\Account>]}
[-UsedSpaceOnly][-encryptionmethod {aes128_diffuser|aes256_diffuser|aes128|aes256}] [-skiphardwaretest] [-discoveryvolumetype <FileSystemType>] [-ForceEncryptionType <type>] [-RemoveVolumeShadowCopies][-computername <Name>] 
[{-?|/?}] [{-help|-h}]
```

### <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|\<Диск >|Представляет букву диска, за которым следует двоеточие.|
|-recoverypassword|Добавляет предохранитель числового пароля. Можно также использовать **- rp** как сокращенную версию этой команды.|
|\<NumericalPassword >|Представляет пароль восстановления.|
|-recoverykey|Добавляет предохранителя внешнего ключа для восстановления. Можно также использовать **- ринтерах** как сокращенную версию этой команды.|
|\<PathToExternalDirectory >|Представляет путь к каталогу для ключа восстановления.|
|-startupkey|Добавляет предохранителя внешнего ключа для запуска. Можно также использовать **-sk** как сокращенную версию этой команды.|
|\<PathToExternalKeyDirectory >|Представляет путь к каталогу для ключа запуска.|
|-сертификата|Добавляет общедоступный предохранитель для диска с данными. Можно также использовать **-cert** как сокращенную версию этой команды.|
|-tpmandpin|Добавление доверенного платформенного модуля (TPM) и персональный идентификационный номер (ПИН-код) предохранитель для диска операционной системы. Можно также использовать **- tp** как сокращенную версию этой команды.|
|-tpmandstartupkey|Добавление предохранителя ключа TPM и запуска для диска операционной системы. Можно также использовать **-tsk** как сокращенную версию этой команды.|
|-tpmandpinandstartupkey|Добавление доверенного платформенного МОДУЛЯ, ПИН-код и предохранитель ключа запуска для диска операционной системы. Можно также использовать **- tpsk** как сокращенную версию этой команды.|
|-пароль|Добавляет предохранитель ключа пароля для диска с данными. Можно также использовать **- pw** как сокращенную версию этой команды.|
|-ADAccountOrGroup|Добавляет предохранитель удостоверений на основе SID для тома. Том автоматическое снятие блокировки, если пользователь или компьютер имеет правильные учетные данные. При указании учетной записи компьютера, добавьте **$** к компьютеру имя и указать **— службы** для указания снятие блокировки должны быть выполнены в содержимом сервере BitLocker, а не пользователь. Можно также использовать **-sid** как сокращенную версию этой команды.|
|-UsedSpaceOnly|Задает режим шифрования для шифрования только используемого пространства. Разделы тома, содержащего используемого пространства будут зашифрованы, но свободное пространство не будет. Если этот параметр не указан, все используемые пространства и свободного места на томе шифруются... Можно также использовать **-использовать** как сокращенную версию этой команды.|
|-encryptionMethod|Настраивает размер алгоритма и ключа шифрования. Можно также использовать **-em** как сокращенную версию этой команды.|
|-skiphardwaretest|Начинает шифрование без проверки оборудования. Можно также использовать **-s** как сокращенную версию этой команды.|
|-discoveryvolumetype|Указывает файловую систему для использования для диска с данными обнаружения. Диск с данными обнаружения — это скрытый диск, добавляемый в формате FAT, защищенных с помощью BitLocker со съемного диска, содержащий чтения BitLocker To Go, чтобы ОС Windows Vista или Windows XP можно использовать для просмотра дисков, защищенных BitLocker.|
|-ForceEncryptionType|Заставляет BitLocker для шифрования оборудования или программного обеспечения. Можно указать либо **оборудования** или **программного обеспечения** как тип шифрования. Если **оборудования** был выбран параметр, но диск не поддерживает шифрование оборудования, готов возвращает сообщение об ошибке. Если параметры групповой политики запрещает заданным типом шифрования, готов возвращает ошибку. Можно также использовать **-транзистор** как сокращенную версию этой команды.|
|-RemoveVolumeShadowCopies|Принудительно deletikon тома теневых копий для тома. Вы не сможете восстановить этот том, с помощью точки восстановления системы после выполнения этой команды. Можно также использовать **- rvsc** как сокращенную версию этой команды.|
|\<FileSystemType >|Указывает, какие файловые системы, которые могут использоваться с дисков с данными обнаружения: FAT32, по умолчанию или нет.|
|-computername|Указывает, что Manage-bde используется для изменения защиты BitLocker на другом компьютере. Можно также использовать **- cn** как сокращенную версию этой команды.|
|\<Имя >|Представляет имя компьютера, на которой требуется изменить защиту BitLocker. Допустимые значения включают имя NetBIOS компьютера и IP-адрес компьютера.|
|-? или /?|Отображение кратких справки в командной строке.|
|-help или -h|Отображает полную справку в командной строке.|

## <a name="BKMK_Examples"></a>Примеры

В следующем примере демонстрируется использование **-на** команду, чтобы включить BitLocker для диска C и добавьте пароль восстановления на диск.
```
manage-bde –on C: -recoverypassword
```
В следующем примере демонстрируется использование **-на** команду, чтобы включить BitLocker для диска C, добавьте пароль восстановления на диск и сохранить ключ восстановления на диске E.
```
manage-bde –on C: -recoverykey E:\ -recoverypassword
```
В следующем примере демонстрируется использование **-на** команду, чтобы включить BitLocker для диска C с использованием предохранителя внешнего ключа (например, USB-ключ) для разблокировки диска с операционной системой. Этот метод является обязательным, если вы используете BitLocker с компьютерами, не имеющие доверенного платформенного МОДУЛЯ.
```
manage-bde -on C: -startupkey E:\
```
В следующем примере демонстрируется использование **-на** команду, чтобы включить BitLocker для диска с данными E и добавления предохранителя ключа пароля. Готов предложит вам ввести пароль, после ввода этой команды.
```
manage-bde –on E: -pw
```
В следующем примере демонстрируется использование **-на** команду, чтобы включить BitLocker для диска операционной системы и использовать шифрование на основе оборудования.
```
manage-bde –on C: -fet Hardware
```
