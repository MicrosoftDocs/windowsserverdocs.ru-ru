# <a name="driverquery"></a>driverquery



Позволяет администратору для отображения списка установленных драйверов устройств и их свойства. При использовании без параметров, **неправильный** выполняется на локальном компьютере.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
driverquery [/s <System> [/u [<Domain>\]<Username> [/p <Password>]]] [/fo {table | list | csv}] [/nh] [/v | /si]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|/s \<system >|Указывает имя или IP-адрес удаленного компьютера. Не используйте символы обратной косой черты. По умолчанию используется локальный компьютер.|
|/u [\<домена >\]<Username>|Выполняет команду с учетными данными учетной записи пользователя в соответствии с *пользователя* или *домена*\*пользователя *. По умолчанию **/s** использует учетные данные пользователя, вошедшего в систему на компьютер, который выполняет команду. **/u** невозможно использовать, если **/s** указан.|
|/p \<пароль >|Указывает пароль для учетной записи пользователя, который указан в **/u** параметра. **/p** невозможно использовать, если **/u** указан.|
|/FO {таблицы | list | CSV}|Задает формат для отображения сведений о драйвере. Допустимые значения: **таблицы**, **списка**, и **csv**. Формат по умолчанию для выходных данных — **таблицы**.|
|/NH|Исключение столбца заголовков из отображаемых сведений о драйверах. Не является допустимым, если **/fo** параметр имеет значение **списка**.|
|/v|Отображает подробные выходные данные. **/v** не является допустимым для подписанных драйверов.|
|/SI|Сведения о подписанных драйверах.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_examples"></a>Примеры

Чтобы отобразить список установленных драйверов устройств на локальном компьютере, введите:
```
driverquery 
```
Чтобы отобразить выходные данные в формате с разделителями-запятыми (CSV), введите:
```
driverquery /fo csv 
```
Чтобы скрыть в строке заголовков в выходных данных, введите следующую команду:
```
driverquery /nh 
```
Чтобы использовать **неправильный** команду на удаленном сервере с именем **server1** используя текущие учетные данные на локальном компьютере, введите:
```
driverquery /s server1
```
Для использования **неправильный** команду на удаленном сервере с именем **server1** с учетными данными для **user1** в домене **maindom**, тип:
```
driverquery /s server1 /u maindom\user1 /p p@ssw3d
```
