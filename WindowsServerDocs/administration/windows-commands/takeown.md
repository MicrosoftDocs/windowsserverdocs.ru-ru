# <a name="takeown"></a>takeown

Позволяет администратору восстанавливать доступ к файлу, который ранее был отклонен, путем назначения администратора владельца файла.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
takeown [/s <Computer> [/u [<Domain>\]<User name> [/p [<Password>]]]] /f <File name> [/a] [/r [/d {Y|N}]]
```

### <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|/s \<компьютер >|Указывает имя или IP-адрес удаленного компьютера (не используйте символы обратной косой черты). По умолчанию используется локальный компьютер. Этот параметр применяется ко всем файлы и папки, указанные в команде.|
|/u [\<домена >\]<User name>|Выполняет сценарий с разрешениями указанной учетной записи. Значение по умолчанию используются системные разрешения.|
|/p [\<пароль >]|Указывает пароль для учетной записи пользователя, который указан в **/u** параметра.|
|/f \<имя файла >|Указывает имя файла или имя каталога шаблона. Можно использовать подстановочный знак * при указании шаблона. Можно также использовать синтаксис *ShareName*\*имя файла *.|
|/a|Пользователь владельцем в группу администраторов, а не текущего пользователя.|
|/r|Выполняет операцию рекурсивного на все файлы в указанном каталоге и подкаталогах.|
|/d {Y \| N}|Отключает запрос подтверждения, который отображается, если текущий пользователь не имеет разрешения «Список содержимого папки» в указанном каталоге и вместо этого использует указанное значение по умолчанию. Допустимые значения для **/d** параметр таковы:</br>-   Y: Станьте владельцем каталога.</br>-N: Перейдите в каталог.</br>Обратите внимание, что этот параметр необходимо использовать в сочетании с **/r** параметр.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания

-   Эта команда обычно используется в пакетных файлах.
-   Если **/a** параметр не указан, файл владельцем назначается пользователя, вошедшего в систему на компьютере.
-   Смешанные шаблоны с использованием (**?** и **&#42;**) не поддерживаются **takeown** команды.
-   После удаления блокировки с **takeown**, возможно, с помощью проводника Windows или **cacls** команду, чтобы предоставить себе полные права доступа к файлам и каталогам, прежде чем их можно удалить. Дополнительные сведения о **cacls**, см. в разделе «Дополнительные ссылки» в конце этого раздела.

## <a name="BKMK_examples"></a>Примеры

Чтобы стать владельцем файла с именем Lostfile, введите:
```
takeown /f lostfile
```
