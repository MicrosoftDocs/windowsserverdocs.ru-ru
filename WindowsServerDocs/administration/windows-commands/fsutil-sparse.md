# <a name="fsutil-sparse"></a>разреженных файлах fsutil
>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows 10, Windows Server 2012 R2, Windows 8.1, Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows 7

Управляет разреженные файлы.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
fsutil sparse [queryflag] <FileName>
fsutil sparse [queryrange] <FileName>
fsutil sparse [setflag] <FileName>
fsutil sparse [setrange] <FileName> <BeginningOffset> <Length>
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|-------------|---------------|
|queryflag|Запрашивает разреженные файлы.|
|queryrange|Ищет в файле и поиск диапазоны, которые может содержать ненулевое значение данных.|
|setflag|Отмечает указанный файл как разреженный.|
|SetRange|Заполняет указанный диапазон файл нули.|
|<FileName>|Указывает полный путь к файлу, включая имя файла и расширение, например C:\documents\filename.txt.|
|<BeginningOffset>|Задает смещение в файле, чтобы пометить разреженной.|
|<Length>|Указывает длину области в файле были помечены как разреженные (в байтах).|

## <a name="remarks"></a>Примечания

-   Разреженный файл является файлом с одного или нескольких регионов незанятые данными. Программа будет рассматривать эти незанятые области как содержащие байты с нулевым значением, но нет места на диске используется для представления этих нулей. Все значимые или ненулевое значение данных выделяется, тогда как все данные nonmeaningful (больших строк данных, состоящий из нулей) не выделяется. При чтении разреженного файла выделенные данные возвращаются как хранятся, и невыделенные данные возвращаются по умолчанию виде нулей, согласно требованиям спецификации безопасности C2. Поддержка разреженных файлов позволяет данных, который необходимо освободить из в любом месте в файле.

-   В разреженный файл большие диапазоны нули может не потребоваться выделение места на диске. При необходимости при записи в файл, выделяется пространство для ненулевое значение данных.

-   Только сжатые и разреженные файлы можно zeroed диапазоны, имеющиеся в операционной системе.

-   Если файл является разреженным или сжатые, NTFS может освободить дисковое пространство в файле. Это задает диапазон байтов в нули без увеличения размера файла.

## <a name="BKMK_examples"></a>Примеры
Чтобы пометить файл с именем "Sample.txt" в каталоге C:\Temp как разреженный, введите следующую команду:

```
fsutil sparse setflag c:\temp\sample.txt 
```
