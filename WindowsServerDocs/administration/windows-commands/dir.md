# <a name="dir"></a>dir



Список файлов и подкаталогов каталога. При использовании без параметров, **dir** Отображает метку тома диска и серийный номер, за которым следует список каталогов и файлов на диске (включая их имена и дату и время последнего изменения каждого). Для файлов **dir** отображает расширение имени и размер в байтах. **Dir** также отображается общее число файлов и каталогов, их общий размер и свободного места (в байтах), оставшегося на диске.

В разделе [Примеры](#examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
dir [<Drive>:][<Path>][<FileName>] [...] [/p] [/q] [/w] [/d] [/a[[:]<Attributes>]][/o[[:]<SortOrder>]] [/t[[:]<TimeField>]] [/s] [/b] [/l] [/n] [/x] [/c] [/4]
```

## <a name="remarks"></a>Примечания

- Для выполнения нескольких *FileName* параметров, разделяйте каждое имя файлового пространства, запятой или точкой с запятой.
- Можно использовать подстановочные знаки ( **&#42;** или **?** ), чтобы представить один или несколько символов имени файла и отобразить подмножество файлы или подкаталоги.

  **Звездочка (\*):** Используйте символ звездочки в качестве замены для любой строки символов, например:  
  - **dir \*.txt** перечислены все файлы в текущем каталоге с расширениями, которые начинаются с расширением txt, например .txt, .txt1, .txt_old.
  - **чтение dir\*.txt** перечислены все файлы в текущем каталоге, которые начинаются с «чтение» и с расширениями, которые начинаются с расширением txt, например txt, .txt1 или .txt_old.
  - **чтение dir\*.\\** * перечислены все файлы в текущем каталоге, которые начинаются с «read» с любым расширением.

  Шаблон «звездочка» всегда использует короткие сопоставления имен, поэтому можно получить непредвиденные результаты. Например следующий каталог содержит два файла (t.txt2 и t97.txt): 
 
  ```
  C:\test>dir /x
  Volume in drive C has no label.
  Volume Serial Number is B86A-EF32
    
  Directory of C:\test
    
  11/30/2004  01:40 PM <DIR>  .
  11/30/2004  01:40 PM <DIR> ..
  11/30/2004  11:05 AM 0 T97B4~1.TXT t.txt2
  11/30/2004  01:16 PM 0 t97.txt
  ```  

  Можно предположить, что нажатие **dir t97\\** * вернет t97.txt файл. Тем не менее, введя **dir t97\\** * возвращает оба файла, так как шаблон «звездочка» соответствует t.txt2 файл для t97.txt с помощью своего короткое имя сопоставления T97B4~1.TXT. Аналогичным образом, введя **del t97\\** * удалит оба файла.

  **Вопросительный знак (?):** Используйте вопросительный знак в качестве замены одного символа в имени. Например, при вводе **dir прочитать код. txt** перечислены все файлы в текущем каталоге с расширением .txt, которые начинаются с «чтение» и следуют до трех символов. Сюда входят Read.txt, Read1.txt, Read12.txt, Read123.txt и Readme1.txt, но не Readme12.txt.
- Указание атрибутов отображения файла

  Если вы используете **/a** с более чем одним значением в *атрибуты*, **dir** имена только те файлы, все указанные атрибуты. Например, если вы используете **/a** с **r** и **-h** как атрибуты (либо при помощи **/ a: r-h** или **/ar-h** ), **dir** будет отображаемое только имена файлов только для чтения, которые не скрыты.
- Указание порядка сортировки имен файлов

  Если указать более одного *SortOrder* значение, **dir** сортирует имена файлов, первый критерий, а затем по второму и т. д. Например, если вы используете **/o** с **e** и **-s** значений в параметре *SortOrder* (либо при помощи **/o:e-s**или **указан**), **dir** сортирует имена каталогов и файлов по расширению, с наибольшим первым, а затем отображает результат. Сортировки по алфавиту результате имена файлов без расширений будут отображаться первыми, а затем имена каталогов, а затем имена файлов с расширениями.
- С помощью перенаправления и каналов

  При использовании символ перенаправления ( **>** ) для отправки **dir** выходные данные в файл или вертикальной чертой ( **|** ) для отправки **dir**выходные данные в другую команду, используйте **/a:-d** и **/b** Чтобы вывести список только имена файлов. Можно использовать *FileName* с **/b** и **/s** позволяет указать, что **dir** следует искать в текущем каталоге и его подкаталогах, для всех файлов имена, соответствующие *FileName*. **Dir** перечислены только букву диска, имя каталога, имя файла и расширение имени файла (один путь к каждой строке), для каждого файла имени, он находит. Прежде чем использовать канал для отправки **dir** выходные данные в другую команду, вы должны задайте переменной среды TEMP в файле Autoexec.nt.
- **Dir** команда, с различными параметрами доступна в консоли восстановления.

## <a name="examples"></a>Примеры

Для отображения всех каталогов, один за другим, в алфавитном порядке, в широком формате и приостановка после каждого экрана, убедитесь, что корневой каталог является текущий каталог и введите:

```
dir /s/w/o/p
```

**Dir** перечислены корневой каталог, подкаталоги и файлы в корневом каталоге, включая расширения. Затем **dir** перечислены имена подкаталогов и имена файлов в каждом дереве.

Для изменения в предыдущем примере, чтобы **dir** отображает имена файлов и расширения, но пропускает имена каталогов, тип:

```
dir /s/w/o/p/a:-d
```

Чтобы распечатать список каталогов, введите следующую команду:

```
dir > prn
```

При указании **prn**, принтер, подключенный к порту LPT1 отправляется списка каталогов. Если принтер подключен к другому порту, необходимо заменить **prn** именем правильный порт.

Можно также выполнить перенаправление выходных данных **dir** в файл, заменив **prn** с именем файла. Можно также ввести путь. Например, для прямого **dir** вывода dir.doc файл в папке записи, введите:

```
dir > \records\dir.doc
```

Если dir.doc не существует, **dir** создает ее, если только нет записей каталога. В этом случае появится следующее сообщение:

`File creation error`

Чтобы вывести список всех имен файлов с расширением .txt во всех каталогах на диске C, введите следующую команду:

```
dir c:\*.txt /w/o/s/p
```

**Dir** отображаются, в расширенном формате алфавитный список соответствующего файла имена в каждом каталоге, и он останавливается каждый раз, когда экран заполняет, пока не нажать любую клавишу для продолжения.
