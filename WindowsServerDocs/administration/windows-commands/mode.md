# <a name="mode"></a>mode

Отображает состояние системы, изменяет параметры системы или перестраивает порты или устройства. Если используется без параметров, в **режиме** отображаются все управляемые атрибуты консоли и доступные устройства com.

## <a name="serial-port"></a>Последовательный порт

Настраивает порт последовательного подключения и задает подтверждение вывода.

### <a name="syntax"></a>Синтаксис

```
mode com<m>[:] [baud=<b>] [parity=<p>] [data=<d>] [stop=<s>] [to={on|off}] [xon={on|off}] [odsr={on|off}] [octs={on|off}] [dtr={on|off|hs}] [rts={on|off|hs|tg}] [idsr={on|off}]
```

#### <a name="parameters"></a>Параметры

| Параметр  | Описание |
| ---------- | ----------- |
| `com<m>[:]` | Указывает номер порта связи Async Прнкнфг. вбшронаус. |
| `baud=<b>`  | Указывает скорость передачи в битах в секунду. Допустимые значения:<ul><li>**11** – 110 бод</li><li>**15** – 150 бод</li><li>**30** – 300 бод</li><li>**60** -600 бод</li><li>**12** – 1200 бод</li><li>**24** – 2400 бод</li><li>**48** -4800 бод</li><li>**96** -9600 бод</li><li>**19** – 19 200 бод</li></ul> |
| `parity=<p>` | Указывает, как система использует бит четности для проверки на наличие ошибок передачи. Допустимые значения:<ul><li>**n** — нет</li><li>**e** — даже (значение по умолчанию)</li><li>**o** -нечетный</li><li>**m** -Mark</li><li>**s** -пространство</li></ul>Не все устройства поддерживают использование параметров **m** или **s** . |
| `data=<d>` | Указывает число битов данных в символе. Допустимы значения в диапазоне от **5** до **8**. Значение по умолчанию — **7**. Не все устройства поддерживают значения **5** и **6**. |
| `stop=<s>` | Указывает число битов прерываний, определяющих конец символа: **1**, **1,5**или **2**. Если скорость передачи составляет **110**, значение по умолчанию — **2**. В противном случае значение по умолчанию равно **1**. Не все устройства поддерживают значение **1,5**. |
| `to={on | off}` | Указывает, использует ли устройство бесконечную обработку времени ожидания. Значение по умолчанию — **Off**. Включение **этого параметра означает, что устройство** никогда не будет ожидать получения ответа от узла или клиентского компьютера. |
| `xon={on | off}` | Указывает, разрешает ли система протокол XON/XOFF. Этот протокол обеспечивает управление потоком для последовательной связи, повышение надежности, но снижает производительность. |
| `odsr={on | off}` | Указывает, включает ли система подтверждение вывода готовности к набору данных (DSR). |
| `octs={on | off}` | Указывает, включает ли система подтверждение вывода CTS для отправки. |
| `dtr={on | off | hs}` | Указывает, включает ли система подтверждение вывода данных о готовности терминала (DTR). Если установить это значение **в** режим «в», то предоставляет постоянный сигнал, показывающий, что терминал готов отправить данные. Установка этого значения в режиме **HS** обеспечивает сигнал подтверждения между двумя терминалами. |
| `rts={on | off | hs | tg}` | Указывает, включает ли система запрос на отправку запроса (RTS). Если установить это значение **в** режим «в», то предоставляет постоянный сигнал, показывающий, что терминал готов отправить данные. Установка этого значения в режиме **HS** обеспечивает сигнал подтверждения между двумя терминалами. Установка этого значения в режиме **TG** обеспечивает возможность переключения между состояниями "Готово" и "не готово". |
| `idsr={on | off}` | Указывает, включает ли система чувствительность DSR. Этот параметр необходимо включить для использования подтверждения DSR.
| /? | Отображение справки в командной строке. |

## <a name="device-status"></a>состояние устройства;

Отображает состояние указанного устройства. Если используется без параметров, в поле **режим** отображается состояние всех устройств, установленных в системе.

### <a name="syntax"></a>Синтаксис

```
mode [<device>] [/status]
```

#### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| `<device>` | Указывает имя устройства, для которого необходимо отобразить состояние. К стандартным именам относятся, LPT1: до LPT3:, COM1: до COM9: и CON. |
| /status | Запрашивает состояние всех перенаправленных параллельных принтеров. Также можно использовать **/STA** в качестве сокращенной версии этой команды. |
| /? | Отображение справки в командной строке. |

## <a name="redirect-printing"></a>Перенаправление печати

Перенаправляет выходные данные принтера. Для перенаправления печати необходимо быть членом группы администраторов.

> [!NOTE]
Чтобы настроить систему таким образом, чтобы она отправляла параллельный вывод принтера на последовательный принтер, необходимо дважды использовать команду **mode** . В первый раз для настройки последовательного порта необходимо использовать **режим** . Во второй раз необходимо использовать **режим** , чтобы перенаправить вывод параллельного принтера на последовательный порт, указанный в команде первого **режима** .

### <a name="syntax"></a>Синтаксис

```
mode LPT<n>[:]=COM<m>[:]
```

#### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| LPT `<n>` [:] | Указывает номер настраиваемого LPT. Как правило, это означает предоставление значения из **LTP1: до LTP3:**, если в системе не включена поддержка специальных параллельных портов. Это обязательный параметр.|
| COM `<m>` [:] | Указывает COM-порт для настройки. Как правило, это означает предоставление значения из **порта COM1: через COM9:**, если система не имеет специального оборудования для дополнительных COM-портов. Это обязательный параметр. |
| /? | Отображение справки в командной строке.|

#### <a name="examples"></a>Примеры

Чтобы перенаправить последовательный принтер, который работает с частотой 4800 бод с четностью и подключен к порту COM1 (первое последовательное подключение на компьютере), введите:

```
mode com1 48,e,,,b
mode lpt1=com1
```

Чтобы перенаправить вывод параллельного принтера с порта LPT1 на порт COM1, а затем распечатать файл с помощью порта LPT1, введите следующую команду перед печатью файла:

```
mode lpt1
```

Эта команда предотвращает перенаправление файла с порта LPT1 на порт COM1.

## <a name="select-code-page"></a>Выбор кодовой страницы

Настраивает или запрашивает сведения о кодовой странице для выбранного устройства.

### <a name="syntax"></a>Синтаксис

```
mode <device> codepage select=<yyy>
mode <device> codepage [/status]
```

#### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| `<device>` |Указывает устройство, для которого необходимо выбрать кодовую страницу. CON — это единственное допустимое имя устройства. Это обязательный параметр. |
| codepage | Указывает кодовую страницу, используемую с указанным устройством. Также можно использовать **CP** в качестве сокращенной версии этой команды. Это обязательный параметр. |
| SELECT =`<yyy>` | Указывает номер кодовой страницы для использования с устройством. Поддерживаемые кодовые страницы, по странам, регионам или языкам, включают:<ul><li>**437:** США</li><li>**850:** Многоязычная (латиница I)</li><li>**852:** Славянские (латиница II)</li><li>**855:** Кириллица (Русский)</li><li>**857:** турецкий</li><li>**860:** португальский</li><li>**861:** исландский</li><li>**863:** Канада — французский</li><li>**865:** Скандинавская</li><li>**866:** русский</li><li>**869:** Современный греческий</li></ul> Это обязательный параметр. |
| /status | Отображает номера текущей кодовой страницы, выбранной для указанного устройства. Также можно использовать **/STA** в качестве сокращенной версии этой команды. Независимо от того, задано ли значение **/Status**, команда в **режиме кодовой** страницы будет отображать номера кодовых страниц, выбранных для указанного устройства. |
| /? | Отображение справки в командной строке. |

## <a name="display-mode"></a>Режим отображения

Изменение размера буфера экрана командной строки

### <a name="syntax"></a>Синтаксис

```
mode con[:] [cols=<c>] [lines=<n>]
```

#### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| Con [:] | Указывает, что изменение применяется к окну командной строки. Это обязательный параметр. |
| Колс =`<c>` | Указывает число столбцов в буфере экрана командной строки. Значение по умолчанию — 80 столбцов, но можно задать для этого параметра любое из значений. Если не используется значение по умолчанию, то типичными значениями являются 40 и 135 столбцов. Использование нестандартных значений может привести к проблемам с приложением командной строки. |
| строк =`<n>` | Указывает число строк в буфере экрана командной строки. Значение по умолчанию — 25, но можно задать любое значение. Если вы не используете значение по умолчанию, то другое стандартный параметр — 50 строк. |
| /? | Отображение справки в командной строке. |

## <a name="typematic-rate"></a>Частота повторения

Задает скорость ввода с клавиатуры. Скорость повторения — это скорость, с которой Windows может повторять символ при нажатии клавиши на клавиатуре.

> [!NOTE]
> Некоторые клавиатуры не распознают эту команду.

### <a name="syntax"></a>Синтаксис

```
mode con[:] [rate=<r> delay=<d>]
```

#### <a name="parameters"></a>Параметры

| Параметр | Описание |
| --------- | ----------- |
| Con [:] | Задает клавиатуру. Это обязательный параметр. |
| ставка =`<r>` | Указывает скорость, с которой символ повторяется на экране при удержании клавиши. Значение по умолчанию — 20 символов в секунду для клавиатуры, совместимых с IBM, и 21 для IBM PS/2-совместимых клавиатур, но можно использовать любое значение от 1 до 32. Если этот параметр задан, необходимо также задать параметр **delay** .|
| задержка =`<d>` | Указывает время, которое будет продолжено после нажатия и удерживания ключа перед повторным выводом символа. Значение по умолчанию — 2 (. 50 секунд), но можно также использовать 1 (. 25 секунд), 3 (75 секунд) или 4 (1 секунда). Если этот параметр задан, необходимо также задать параметр **Rate** . |
| /? | Отображение справки в командной строке. |
