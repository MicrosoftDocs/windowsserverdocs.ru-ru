# <a name="wbadmin-delete-systemstatebackup"></a>WBADMIN delete systemstatebackup



Удаление резервных копий состояния системы, указываемые. Если указанный том содержит резервные копии, отличный от резервных копий состояния системы из локального сервера, эти резервные копии не удаляется.

> [!NOTE]
> Система архивации данных Windows Server не архивировать или восстановить кустов реестра пользователя (HKEY_CURRENT_USER) в процессе резервного копирования состояния системы или восстановления состояния системы.

Чтобы удалить резервную копию состояния системы с помощью подкоманды, необходимо быть членом **операторы архива** группы или **Администраторы** группа, или должна была быть делегированы соответствующие разрешения. Кроме того, необходимо запустить **wbadmin** из командной строки с повышенными правами. (Чтобы открыть командную строку, щелкните правой кнопкой мыши **командной**, а затем нажмите кнопку **Запуск от имени администратора**.)

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
wbadmin delete systemstatebackup
{-keepVersions:<NumberofCopies> | -version:<VersionIdentifier> | -deleteOldest}
[-backupTarget:<VolumeName>]
[-machine:<BackupMachineName>]
[-quiet]
```

> [!IMPORTANT]
> Должен быть указан один и только один из этих параметров: **- keepVersions**, **-версии**, или **- deleteOldest**.

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|-keepVersions|Указывает количество последних резервных копий состояния системы нужно хранить. Значение должно быть положительным целым числом. Значение параметра **- keepVersions: 0** удаляет все резервные копии состояния системы.|
|-версии|Задает идентификатор версии для резервного копирования в мм/дд/гггг-формате чч: мм. Если вы не знаете идентификатор версии, введите **wbadmin получить версии**.</br>Версии, которые являются исключительно резервные копии состояния системы можно удалить с помощью следующей команды. Используйте **wbadmin получить элементы** для просмотра версии типа.|
|-deleteOldest|Удаляет старой резервной копии состояния системы.|
|-backupTarget|Задает место хранения для резервного копирования, который требуется удалить. Место хранения для резервных копий дисков может быть буквой диска, точку подключения или путь к основе GUID тома. Это значение должно быть указано для обнаружения резервных копий, которые не на локальном компьютере. Сведения о резервном копировании для локального компьютера, будут доступны в каталоге резервных копий на локальном компьютере.|
|-machine|Указывает компьютер, для которого вы хотите удалить резервной копии состояния системы. Это удобно при нескольких компьютеров создавались резервные копии в том же расположении. Когда следует использовать **- backupTarget** указан параметр.|
|-quiet|Запускает подкоманды без вывода сообщений для пользователя.|

## <a name="BKMK_examples"></a>Примеры

Чтобы удалить резервного копирования состояния системы, создан на 31 марта 2013 г., 10:00, введите следующую команду:
```
wbadmin delete systemstatebackup -version:03/31/2013-10:00
```
Чтобы удалить все резервные копии состояния системы, за исключением три последних, введите:
```
wbadmin delete systemstatebackup -keepVersions:3
```
Для удаления старой резервной копии состояния системы, хранящиеся на диске f, введите следующую команду:
```
wbadmin delete systemstatebackup -backupTarget:f -deleteOldest
```
