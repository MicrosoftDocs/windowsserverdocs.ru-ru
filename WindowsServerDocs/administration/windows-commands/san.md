# <a name="san"></a>по сети SAN

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Отображает или задает политику по сети SAN область хранения для операционной системы.
> [!NOTE]
> Эта команда применяется только к Windows 7 и Windows Server 2008 R2.

## <a name="syntax"></a>Синтаксис
```
san [policy={onlineAll | offlineAll | offlineShared}] [noerr]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|policy={ onlineAll &#124; offlineAll &#124; offlineShared }]|Задает политику san для операционной системы в данный момент загружена. Политики сети san определяет ли вновь обнаруженных диск будет подключен к сети или находится в автономном режиме, и он становится чтения и записи или остается доступной только для чтения. Если диск находится в автономном режиме, разметка диска могут считываться, но регистрируются нет тома устройств Plug and Play. Это означает, что на диске могут быть подключены файловая система. Если диск подключен к сети, одно или несколько томов устройств устанавливаются для диска. Ниже приведено описание каждого параметра.<br /><br />-   **onlineAll**. Указывает, что все вновь обнаруженные диски переместится online они становятся чтения и записи. **ВАЖНО:**     Указание **onlineAll** на сервере, который использует дисков может привести к повреждению данных. Таким образом эта политика не следует задавать, если диски являются общими для серверов, не является частью кластера.<br />-   **offlineAll**. Указывает, что все обнаруженные диски, за исключением загрузочного диска будет находиться в автономном режиме только для andread по умолчанию.<br />-   **offlineShared**. Указывает, что все недавно обнаружили, что диски, которые не находятся на общей шине (например, SCSI и iSCSI) переводятся в оперативный режим и внесенные чтения и записи. Диски, которые находятся в автономном режиме будут доступны только для чтения по умолчанию.<br /><br />Дополнительные сведения см. в разделе [перечисления VDS_san_POLICY](https://go.microsoft.com/fwlink/?LinkId=203815) (https://go.microsoft.com/fwlink/?LinkId=203815).|
|успешного|Используется только для сценариев. При обнаружении ошибки, DiskPart продолжает обрабатывать команды, как если бы ошибки не было. Без этого параметра по ошибке будет возобновлено DiskPart завершить работу с кодом ошибки.|
## <a name="remarks"></a>Примечания
-   Если команда имеет без параметров, отображается текущей политики по сети san.
## <a name="BKMK_Examples"></a>Примеры
Чтобы просмотреть текущую политику, введите следующую команду:
```
san
```
Чтобы сделать все обнаруженные диски, за исключением загрузочного диска вне сети "и" только для чтения по умолчанию, введите:
```
san policy=offlineAll
```
