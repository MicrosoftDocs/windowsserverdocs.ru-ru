# <a name="auditpol-set"></a>Auditpol set

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Политика аудита наборов на пользователя, политика аудита системы или параметры аудита.

## <a name="syntax"></a>Синтаксис
```
auditpol /set
[/user[:<username>|<{sid}>][/include][/exclude]]
[/category:<name>|<{guid}>[,:<name|<{guid}> ]]
[/success:<enable>|<disable>][/failure:<enable>|<disable>]
[/subcategory:<name>|<{guid}>[,:<name|<{guid}> ]]
[/success:<enable>|<disable>][/failure:<enable>|<disable>]
[/option:<option name> /value: <enable>|<disable>]
```
## <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|/ User|Субъект безопасности, для которого политики, заданные категорию или подкатегорию аудита на пользователя имеет значение. Параметр категории или подкатегории необходимо указать, как имя или идентификатор безопасности (SID).|
|/ include|Указан параметр/User; Указывает, что в политике пользователя будет приводит аудита будет создан, даже если он не указан, политики аудита системы. Этот параметр используется по умолчанию и применяется автоматически в том случае, если ни один из / учтено, и явным образом указаны параметры/EXCLUDE.|
|/ Exclude|Указан параметр/User; Указывает, что в политике пользователя будет приводит аудита для подавления независимо от политики аудита системы. Этот параметр учитывается для пользователей, которые являются членами локальной группы администраторов.|
|/ Category|Один или несколько категорий аудита, определяемого глобальный уникальный идентификатор (GUID) или именем. Если пользователь не указан, устанавливается политика системы.|
|/SubCategory|Один или несколько подкатегории аудита, указанного GUID или имя. Если пользователь не указан, устанавливается политика системы.|
|/Success|Указывает аудит успешных попыток. Этот параметр используется по умолчанию и применяется автоматически в том случае, если ни /success, ни /failure параметры указаны явным образом. Этот параметр должен использоваться с параметром, указывающим, следует ли включить или отключить этот параметр.|
|/FAILURE|Указывает аудита отказов. Этот параметр должен использоваться с параметром, указывающим, следует ли включить или отключить этот параметр.|
|/ параметр|Задает политику аудита для параметров CrashOnAuditFail, FullprivilegeAuditing, AuditBaseObjects или AuditBasedirectories.|
|/SD|Задает дескриптор безопасности, используемые для делегирования доступа к политике аудита. Дескриптор безопасности необходимо указать с помощью языка определения дескрипторов безопасности (SDDL). Дескриптор безопасности должен иметь список управления доступом на уровне пользователей (DACL).|
|/?|Отображение справки в командной строке.|
## <a name="remarks"></a>Примечания
для всех операций для политики пользователя задать и изменить системную политику, необходимо написать или набор разрешений полного доступа для этого объекта в дескрипторе безопасности. Позволяет выполнять операции с множествами, исходя из **Управление аудитом и журналом безопасности** право пользователя (SeSecurityPrivilege). Тем не менее это право позволяет доступ к дополнительным возможностям, нет необходимости выполнить операцию set.
## <a name="BKMK_examples"></a>Примеры
### <a name="examples-for-the-per-user-audit-policy"></a>Примеры для политики пользователя
Для задания политики для всех подкатегорий в подробное отслеживание категории mikedan пользователя аудита на пользователя, таким образом, чтобы все, что будет выполняться аудит успешных попыток пользователя, введите:
```
auditpol /set /user:mikedan /category:"detailed Tracking" /include /success:enable
```
Чтобы настроить политику аудита пользователя для категорий, указанный по имени и идентификатора GUID и подкатегории, указанные по GUID, чтобы отключить аудит успешных или неудачных попыток, введите:
```
auditpol /set /user:mikedan /exclude /category:"Object Access","System",{6997984b-797a-11d9-bed3-505054503030} 
/subcategory:{0ccee9210-69ae-11d9-bed3-505054503030},:{0ccee9211-69ae-11d9-bed3-505054503030}, /success:enable /failure:enable
```
Чтобы настроить политику аудита на пользователя для указанного пользователя для всех категорий для подавления аудита из всех, кроме успешных попыток, введите:
```
auditpol /set /user:mikedan /exclude /category:* /success:enable
```
### <a name="examples-for-the-system-audit-policy"></a>Примеры для политики аудита системы
Чтобы настроить политику аудита системы для всех подкатегорий в подробные категории отслеживания, включать аудит только успешных попыток, введите:
```
auditpol /set /category:"detailed Tracking" /success:enable
```
> [!NOTE]
> Настройки сбоя не изменяется.
Чтобы настроить политику аудита системы для доступа к объектам и системы категорий (которые подразумевается, так как подкатегории указаны) и подкатегории, заданные идентификаторы GUID для подавления неудачных попыток входа и аудит успешных попыток, введите:
```
auditpol /set /subcategory:{0ccee9210-69ae-11d9-bed3-505054503030},{0ccee9211-69ae-11d9-bed3-505054503030}, /failure:disable /success:enable
```
### <a name="example-for-auditing-options"></a>Пример для параметров аудита
Чтобы задать параметры аудита включенное состояние для параметра CrashOnAuditFail, введите следующую команду:
```
auditpol /set /option:CrashOnAuditFail /value:enable
```
