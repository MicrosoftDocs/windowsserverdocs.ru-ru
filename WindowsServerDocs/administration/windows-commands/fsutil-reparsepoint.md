# <a name="fsutil-reparsepoint"></a>fsutil reparsepoint
>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows 10, Windows Server 2012 R2, Windows 8.1, Windows Server 2012, Windows 8, Windows Server 2008 R2, Windows 7, Windows 2008, Windows Vista

Запросы или операции удаления точки повторного анализа.  **Fsutil reparsepoint** команда обычно используется специалистами службы поддержки.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
fsutil reparsepoint [query] <FileName>
fsutil reparsepoint [delete] <FileName>
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|-------------|---------------|
|запрос|Извлекает данные точки повторной обработки, который связан с файлом или каталогом, определенным указанным дескриптором.|
|"Удалить"|Удаляет точку повторной обработки из файла или каталога, который определяется указанным дескриптором, но не удаляет файл или каталог.|
|<FileName>|Указывает полный путь к файлу, включая имя файла и расширение, например C:\documents\filename.txt.|

## <a name="remarks"></a>Примечания

-   Точки повторного анализа, системные объекты, которые имеют определяемый атрибут, содержащий определенные пользователем данные файла NTFS, и они используются для расширения функциональных возможностей в подсистеме ввода вывода (I/O).

-   Повторной обработки точки используются для точки соединения каталогов и точки подключения тома. Они также используются драйверы фильтра файловой системы, чтобы пометить определенные файлы как специфические для этого драйвера.

-   Когда программа устанавливает точку повторной обработки, он сохраняет эти данные, а также тег повторной обработки, который однозначно определяет данные, которые он хранит. В файловой системе Открытие файла с точкой повторного анализа, предпринимается попытка найти фильтр связан файловой системы. Если найден фильтр файловой системы, фильтр обрабатывает файл, как указано выше, данные повторного анализа. При обнаружении не фильтр файловой системы, происходит сбой операции открытия файла.

## <a name="BKMK_examples"></a>Примеры
Для получения данных точку повторной обработки, связанные с C:\Server, введите следующую команду:

```
fsutil reparsepoint query c:\server
```

Чтобы удалить точку повторного анализа из указанного файла или каталога, используйте следующий формат:

```
fsutil reparsepoint delete c:\server
```
