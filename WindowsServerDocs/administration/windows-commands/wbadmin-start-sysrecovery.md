# <a name="wbadmin-start-sysrecovery"></a>Wbadmin start sysrecovery



Выполняет восстановление системы (восстановление) с использованием указанных параметров.

> [!NOTE]
> Подкоманды можно запустить только из среды восстановления Windows, и он не отображается по умолчанию в тексте использования **Wbadmin**. Дополнительные сведения см. в разделе [Общие сведения о среде восстановления Windows (Windows RE)](https://technet.microsoft.com/library/hh825173.aspx).

Для выполнения восстановления системы с помощью подкоманды, необходимо быть членом **операторы архива** группы или **Администраторы** группа, или должна была быть делегированы соответствующие разрешения.

Примеры по использованию этого, см. в разделе [примеры](#BKMK_examples).

## <a name="syntax"></a>Синтаксис

```
wbadmin start sysrecovery
-version:<VersionIdentifier>
-backupTarget:{<BackupDestinationVolume> | <NetworkShareHostingBackup>}
[-machine:<BackupMachineName>]
[-restoreAllVolumes]
[-recreateDisks]
[-excludeDisks]
[-skipBadClusterCheck]
[-quiet]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|-версии|Задает идентификатор версии для резервного копирования для восстановления в мм/дд/гггг-формате чч: мм. Если вы не знаете идентификатор версии, введите **wbadmin получить версии**.|
|-backupTarget|Задает место хранения, содержащий резервные копии, которые вы хотите восстановить. Этот параметр полезен в тех случаях, когда место хранения отличается от обычно хранения резервных копий этого компьютера.|
|-machine|Задает имя компьютера, на котором вы хотите восстановить. Этот параметр полезен в тех случаях, когда в том же расположении будут заархивированы нескольких компьютеров. Когда следует использовать **- backupTarget** указан параметр.|
|-restoreAllVolumes|Восстанавливает все тома из выбранной резервной копии. Если этот параметр не указан, восстанавливаются только критические тома (тома, содержащие компоненты ОС и состояние системы). Этот параметр полезен в тех случаях, когда требуется восстановление некритические томов во время восстановления системы.|
|-recreateDisks|Восстанавливает конфигурацию дисков к состоянию, существовавшему при создании резервной копии.</br>Предупреждение: Этот параметр удаляет все данные на томах, компоненты операционной системы узла. Он также может удалить данные из томов данных.|
|-excludeDisks|Допустимым только в том случае, при указании **- recreateDisks** параметр и следует ввести в виде списка с разделителями запятыми идентификаторов диска (как указано в выходных данных **wbadmin получить диски**). Исключенные из репликации диски не секционирована и не в формате. Этот параметр помогает сохранить данные на дисках, которые не должны изменяться во время операции восстановления.|
|-skipBadClusterCheck|Пропуск проверки дисков восстановления поврежденного кластера сведения. При восстановлении на другой сервер или оборудование, мы рекомендуем этот параметр не используется. Вы можете вручную запустить **chkdsk /b** восстановления дисков в любое время для проверки их поврежденные кластеры и затем соответствующим образом обновить сведений о файловой системе.</br>Предупреждение: Пока вы не выполните **chkdsk** как было сказано, поврежденные кластеры, указанное в отчетах для восстановленного системы может оказаться неточным.|
|-quiet|Выполняет команду без вывода сообщений для пользователя.|

## <a name="BKMK_examples"></a>Примеры

Чтобы начать восстановление из резервной копии, выполненного на 31 марта 2013 в 9:00, данные находятся на диске d:, тип:
```
wbadmin start sysrecovery -version:03/31/2013-09:00 -backupTarget:d:
```
Чтобы начать восстановление из резервной копии, который выполнялся 30 апреля 2013 г. в 9:00, данные находятся в общей папке \\ \\servername\shared: server01, введите:
```
wbadmin start sysrecovery -version:04/30/2013-09:00 -backupTarget:\\servername\share -machine:server01
```
