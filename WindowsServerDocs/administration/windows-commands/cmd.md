# <a name="cmd"></a>Cmd



Запускает новый экземпляр интерпретатор команд Cmd.exe. При использовании без параметров, **cmd** сведения о версии и авторских правах операционной системы.

## <a name="syntax"></a>Синтаксис

```
cmd [/c|/k] [/s] [/q] [/d] [/a|/u] [/t:{<B><F>|<F>}] [/e:{on|off}] [/f:{on|off}] [/v:{on|off}] [<String>]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|/c|Выполняет команду, заданную параметром *строка* , а затем останавливается.|
|/k|Выполняет команду, заданную параметром *строка* и продолжает работу.|
|/s|Изменение обработки *строка* после **/c** или **/k**.|
|/q|Отключает echo.|
|/d|Отключает выполнение команд автозапуска.|
|/a|Форматирование вывода внутренней команды в канал или файл как Американский национальный институт стандартов (ANSI).|
|/u|Форматирование вывода внутренней команды в канал или файл в Юникоде.|
|/ t: {\<B\>\<F\>\|\<F\>}|В качестве фона (*B*) и переднего плана (*F*) цветов.|
|/e:on|Включает расширения команд.|
|/e:off|Отключает команды расширения.|
|/f: на|Обеспечивает завершение имен файлов и каталогов.|
|/f: off|Отключает завершение имен файлов и каталогов.|
|/v:on|Включение отложенного расширение переменных среды.|
|/v:off|Отключение отложенного расширение переменных среды.|
|\<строка >|Указывает команду, которую вы хотите выполнить.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания

-   Использование нескольких команд

    Использование нескольких команд для \<строка >, разделите их точками разделитель команд **&&** и заключите в кавычки. Пример:  
    ```
    "<Command>&&<Command>&&<Command>"
    ```  
-   Обработка кавычек

    Если указать **/c** или **/k**, **cmd** обрабатывает остальная часть *строку,* и кавычки сохраняются только в том случае, если все из следующих условий:  
    -   Вы не используете **/s**.
    -   Используется только один набор кавычки.
    -   Вы не используете специальные символы в кавычках (например: & () < > @ ^ |).
    -   Можно использовать один или несколько символов пробела в кавычках.
    -   *Строка* в кавычках — имя исполняемого файла.

    Если предыдущие условия не выполняются, *строка* обрабатывается, первый символ, чтобы проверить, является ли открывающей кавычкой. Если первый символ является открывающей кавычкой, он связывается с закрывающей кавычкой. Любой текст, следующий закрывающие кавычки сохраняется.
-   Выполнение подразделов реестра

    Если вы не укажете **/d** в *строка*, Cmd.exe ищет следующие подразделы:

    **HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\AutoRun\REG_SZ**

    **HKEY_CURRENT_USER\Software\Microsoft\Command Processor\AutoRun\REG_EXPAND_SZ**

    Если один или оба подраздела реестра, они выполняются до всех остальных переменных.

> [!CAUTION]
> Неправильное изменение реестра может серьезно повредить систему. Перед внесением изменений следует сделать резервную копию всех ценных данных на компьютере.
-   Включение и отключение расширения команд

    Расширения команд включены по умолчанию в Windows XP. Можно отключить для определенного процесса с помощью **/e: off**. Можно включить или отключить расширения для всех **cmd** параметры командной строки в сеансе компьютера или пользователя, задав следующие **REG_DWORD** значения:

    **HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\EnableExtensions\REG_DWORD**

    **HKEY_CURRENT_USER\Software\Microsoft\Command Processor\EnableExtensions\REG_DWORD**

    Задайте **REG_DWORD** значение либо **0 × 1** (включено) или **0 × 0** (отключен) в реестре с помощью Regedit.exe. Пользовательские настройки имеют приоритет над параметрами компьютера и параметры командной строки имеют приоритет над параметрами реестра.

>     [!CAUTION]
>     Incorrectly editing the registry may severely damage your system. Before making changes to the registry, you should back up any valued data on the computer.

    When you enable command extensions, the following commands are affected:  
    -   **assoc**
    -   **call**
    -   **chdir (cd)**
    -   **color**
    -   **del (erase)**
    -   **endlocal**
    -   **for**
    -   **ftype**
    -   **goto**
    -   **if**
    -   **mkdir (md)**
    -   **popd**
    -   **prompt**
    -   **pushd**
    -   **set**
    -   **setlocal**
    -   **shift**
    -   **start** (also includes changes to external command processes)
-   Включение расширения переменной среды с задержкой

    Если вы включаете расширение переменной среды с задержкой, восклицательного знака можно использовать для замены текущего значения переменной среды во время выполнения.
-   Включение завершения имен файлов и каталогов

    Завершение имен файлов и каталогов не включена по умолчанию. Можно включить или отключить завершения имени файла для конкретного процесса **cmd** с **/f:**{**на**|**off**}. Можно включить или отключить завершение имен файлов и каталогов для всех процессов **cmd** команду на компьютере, или для сеанса входа в систему пользователя, задав следующие **REG_DWORD** значения:

    **HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\CompletionChar\REG_DWORD**

    **HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\PathCompletionChar\REG_DWORD**

    **HKEY_CURRENT_USER\Software\Microsoft\Command Processor\CompletionChar\REG_DWORD**

    **HKEY_CURRENT_USER\Software\Microsoft\Command Processor\PathCompletionChar\REG_DWORD**

    Чтобы задать **REG_DWORD** значение, запустите Regedit.exe и использовать шестнадцатеричное значение управляющего символа для определенной функции (например, **0 × 9** — ВКЛАДКА и **0 × 08** — BACKSPACE). Пользовательские настройки имеют приоритет над параметрами компьютера и параметры командной строки имеют приоритет над параметрами реестра.

> [!CAUTION]
> Неправильное изменение реестра может серьезно повредить систему. Перед внесением изменений следует сделать резервную копию всех ценных данных на компьютере.

Если включить завершение имен файлов и каталогов с помощью **/f: на**, используйте сочетание клавиш CTRL + D для завершения имен папок и CTRL + F для завершения имени файла. Чтобы отключить определенный символ завершения в реестре, используйте значение для пробелов [**0 x 20**], так как он не является допустимым элементом управления символом.

При нажатии клавиши CTRL + D или CTRL + F, **cmd** обрабатывает завершение имен файлов и каталогов. Эти функции сочетание клавиш Добавление подстановочных знаков для *строка* (если оно не доступно), построить список путей, которые соответствуют, а затем отобразите первый соответствующий путь. Если ни один из путей, файлов и каталогов функция завершения имя подает сигнал и не приводит к изменению отображения. Для перемещения по списку сопоставления пути, нажимайте клавиши CTRL + D или CTRL + F. Для перемещения по списку в обратном направлении, одновременно нажмите клавиши SHIFT и CTRL + D или CTRL + F. Чтобы отменить сохраненные список соответствующих путей и создать новый список, измените *строка* и нажмите клавиши CTRL + D или CTRL + F. При переключении между CTRL + D и CTRL + F, сохраненный список соответствующих путей удаляется и создается новый список. Единственное различие между клавиши CTRL + D или CTRL + F является сочетание клавиш CTRL + D соответствует только имена каталогов и CTRL + F соответствует имена файлов и каталогов. При использовании завершение имен файлов и каталогов на любой из встроенных команд каталогов (то есть **компакт-диска**, **MD**, или **к удаленному рабочему Столу**), предполагается завершение папок.

Завершение имен файлов и каталогов правильно обрабатывает имена файлов, содержащие пробелы или специальные символы, если поместить соответствующий путь в кавычки.

Кавычки необходимы следующие специальные символы: & < > [] {} ^ =;! '+' ~ [пробел].

Если сведения, которые вы предоставляете содержит пробелы, используйте кавычки вокруг текста (например, «имя компьютера»).

При обработке завершения имен файлов и каталогов внутри *строка*, то любой частью *путь* удаляются справа от курсора (в том месте *строка* где Завершение обработки).
