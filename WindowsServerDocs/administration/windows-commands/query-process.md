# <a name="query-process"></a>процесс запроса

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Отображает сведения о процессах, выполняемых на сервере узла сеансов удаленный рабочий стол.
С помощью этой команды можно узнать, какие программы запущены конкретным пользователем, а также какие пользователи используют определенную программу.
Примеры использования этой команды см. в разделе [примеры](#BKMK_examples).
> [!NOTE]
> В Windows Server 2008 R2 службы терминалов называются службами удаленных рабочих столов. Чтобы узнать о новых возможностях последней версии, см. статью [новые возможности службы удаленных рабочих столов в Windows server 2012](https://technet.microsoft.com/library/hh831527) в библиотеке TechNet по Windows Server.
> ## <a name="syntax"></a>Синтаксис
> ```
> query process [* | <ProcessID> | <UserName> | <SessionName> | /id:<nn> | <ProgramName>] [/server:<ServerName>]
> ```
> ### <a name="parameters"></a>Параметры
> 
> |      Параметр       |                                                                 Описание                                                                  |
> |----------------------|----------------------------------------------------------------------------------------------------------------------------------------------|
> |          \*          |                                                    список процессов для всех сеансов.                                                     |
> |     <ProcessID>      |                                   Указывает числовой идентификатор, определяющий процесс, который необходимо запросить.                                   |
> |      <UserName>      |                                       Указывает имя пользователя, процессы которого необходимо перечислить.                                       |
> |    <SessionName>     |                                     Указывает имя сеанса, процессы которого необходимо вывести в список.                                      |
> |       /ID:<nn>       |                                      Указывает идентификатор сеанса, процессы которого необходимо вывести в список.                                       |
> |    <ProgramName>     |                     Указывает имя программы, процессы которой необходимо запросить. Требуется расширение EXE.                     |
> | /server:<ServerName> | Указывает сервер узла сеансов удаленных рабочих столов, для которого необходимо выполнить перечисление процессов. Если не указано, используется сервер, на котором вы выполнили вход в систему. |
> |          /?          |                                                     Отображает справку в командной строке.                                                     |
> 
> ## <a name="remarks"></a>Примечания
> - Администраторы имеют полный доступ ко всем функциям **обработки запросов** .
> - Если не указать <*UserName*>, <*SessionName*>, **/ID:** <*nn*>, <*ProgramName*> или **\\** * Parameters, в **процессе запроса** отображаются только процессы, принадлежащие текущему пользователю.
> - Если сеанс указан, он должен указать активный сеанс.
> - **процесс запроса** возвращает следующие сведения:
>   -   Пользователь, владеющий процессом
>   -   Сеанс, владеющий процессом
>   -   Идентификатор сеанса
>   -   Имя процесса
>   -   Идентификатор процесса
> - Когда **процесс запроса** возвращает сведения, перед каждым процессом, принадлежащим текущему сеансу, отображается символ "больше чем" (>).
>   ## <a name="examples"></a><a name=BKMK_examples></a>Примеров
> - Чтобы отобразить сведения о процессах, используемых всеми сеансами, введите:
>   ```
>   query process *
>   ```
> - Чтобы отобразить сведения о процессах, используемых сеансом с ИДЕНТИФИКАТОРом 2, введите:
>   ```
>   query process /ID:2
>   ```
