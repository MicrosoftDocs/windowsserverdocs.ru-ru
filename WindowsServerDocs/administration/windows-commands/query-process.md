# <a name="query-process"></a>обработка запросов

Область применения: Windows Server (половина ежегодного канала), Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Отображает сведения о процессах, выполняемых на сервере узла удаленный рабочий стол сеансов. С помощью этой команды можно узнать, какие программы запущены конкретным пользователем, а также какие пользователи используют определенную программу. Эта команда возвращает приведенные ниже сведения.

- Пользователь, владеющий процессом

- Сеанс, владеющий процессом

- Идентификатор сеанса

- Имя процесса

- Идентификатор процесса

> [!NOTE]
> Чтобы узнать о новых возможностях последней версии, см. статью [новые возможности службы удаленных рабочих столов в Windows Server](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-R2-and-2012/dn283323(v=ws.11)).

## <a name="syntax"></a>Синтаксис

```
query process [*|<processID>|<username>|<sessionname>|/id:<nn>|<programname>] [/server:<servername>]
```

### <a name="parameters"></a>Параметры

| Параметр | Описание: |
|--|--|
| * | Список процессов для всех сеансов. |
| `<processID>` | Указывает числовой идентификатор, определяющий процесс, который необходимо запросить. |
| `<username>` | Указывает имя пользователя, процессы которого необходимо перечислить. |
| `<sessionname>` | Указывает имя активного сеанса, процессы которого необходимо вывести в список. |
| /ID`<nn>` | Указывает идентификатор сеанса, процессы которого необходимо вывести в список. |
| `<programname>` | Указывает имя программы, процессы которой необходимо запросить. Требуется расширение EXE. |
| /server:`<servername>` | Указывает удаленный рабочий стол сервер узла сеансов, процессы которого необходимо перечислить. Если не указано, используется сервер, на котором вы выполнили вход в систему. |
| /? | Отображение справки в командной строке. |

#### <a name="remarks"></a>Примечания

- Администраторы имеют полный доступ ко всем функциям **обработки запросов** .

- Если не указать <*username*>, <*sessionname*>, */id `<nn>` :*, <*programname*> или *&#42;* Parameters, в этом запросе отображаются только процессы, принадлежащие текущему пользователю.

- Когда **процесс запроса** возвращает сведения, `(>)` перед каждым процессом, принадлежащим текущему сеансу, отображается символ "больше чем".

## <a name="examples"></a>Примеры

Чтобы отобразить сведения о процессах, используемых всеми сеансами, введите:

```
query process *
```

Чтобы отобразить сведения о процессах, используемых *сеансом с идентификатором 2*, введите:

```
query process /ID:2
```
