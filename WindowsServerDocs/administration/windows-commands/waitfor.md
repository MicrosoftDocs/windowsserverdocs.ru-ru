# <a name="waitfor"></a>waitfor



Отправляет или ожидает сигнала в системе. **WAITFOR** используется для синхронизации компьютеров в сети.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
waitfor [/s <Computer> [/u [<Domain>\]<User> [/p [<Password>]]]] /si <SignalName>
waitfor [/t <Timeout>] <SignalName>
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|/s \<компьютер >|Указывает имя или IP-адрес удаленного компьютера (не используйте символы обратной косой черты). По умолчанию используется локальный компьютер. Этот параметр применяется ко всем файлам и папкам, указанным в команде.|
|/u [\<домена >\]<User>|Запускает сценарий, используя учетные данные учетной записи указанного пользователя. По умолчанию **waitfor** использует учетные данные текущего пользователя.|
|/p [\<пароль >]|Указывает пароль для учетной записи пользователя, который указан в **/u** параметра.|
|/SI|Отправляет указанный сигнал по сети.|
|/t \<время ожидания >|Указывает количество секунд ожидания сигнала. По умолчанию **waitfor** ждет неопределенно долгое время.|
|\<Имя_сигнала >|Сигнал, который **waitfor** ожидает или отправки. *Имя_сигнала* не учитывает регистр.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания

-   Имена сигнала, не может превышать 225 знаков. Допустимые символы: a – z, A-Z, 0-9 и ASCII расширенный набор символов (128-255).
-   Если вы не используете **/s**, сигнал передается ко всем системам в домене. Если вы используете **/s**, отправляется сигнал только к указанной системе.
-   Можно запустить несколько экземпляров **waitfor** на одном компьютере, но каждый экземпляр **waitfor** необходимо дождаться разных сигналов. Только один экземпляр **waitfor** может ожидать сигнала на данном компьютере.
-   Вы можете активировать сигнала вручную с помощью **/si** параметр командной строки.
-   **WAITFOR** выполняется только в Windows XP и серверы под управлением ОС Windows Server 2003, но можно отправлять сигналы на любом компьютере под управлением ОС Windows.
-   Компьютеры могут получать сигналы только в том случае, если они находятся в том же домене, что и компьютер, отправке сигнала.
-   Можно использовать **waitfor** при тестировании сборок программного обеспечения. Например, компиляция компьютере отправляют сигнал на нескольких компьютерах под управлением **waitfor** после успешного завершения компиляции. После получения сигнала пакетный файл, включает в себя **waitfor** можно указать компьютеры, которые будут немедленно начать установку программного обеспечения или выполнение тестов откомпилированную сборку.

## <a name="BKMK_examples"></a>Примеры

Чтобы Подождите, пока не будет получен сигнал «espresso\build007», введите:
```
waitfor espresso\build007
```
По умолчанию **waitfor** бесконечно ожидает сигнала.

В течение 10 секунд для сигнала «espresso\compile007» должно быть получено до истечения времени ожидания, введите следующую команду:
```
waitfor /t 10 espresso\build007
```
Чтобы вручную активировать сигнала «espresso\build007», введите следующую команду:
```
waitfor /si espresso\build007
```
