# <a name="reg-query"></a>reg query



Возвращает список следующего уровня подразделы и параметры, которые находятся в указанном в реестре.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
reg query <KeyName> [{/v <ValueName> | /ve}] [/s] [/se <Separator>] [/f <Data>] [{/k | /d}] [/c] [/e] [/t <Type>] [/z]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|\<KeyName>|Указывает полный путь к разделу. Для указания удаленных компьютеров включают имя компьютера (в формате \\ \\ComputerName\) как часть *KeyName*. Пропуск \\ \\ComputerName\ вызывает операцию по умолчанию на локальном компьютере. *KeyName* должен содержать допустимый корневой раздел. Приведены допустимые корневые ключи для локального компьютера. HKLM, HKCU, HKCR, HKU и HKCC. Если указан удаленный компьютер, являются допустимым корневые ключи: HKLM и HKU.|
|/v \<ValueName>|Указывает имя значения реестра, который должен запрашиваться. Если не указано, все значение имен для *KeyName* возвращаются. *ValueName* для этого параметра является необязательным при **/f** также используется параметр.|
|/ ve|Выполняет запрос для имена значений, которые являются пустыми.|
|/s|Указывает, чтобы запросить все подразделы и значения имен рекурсивно.|
|/SE \<разделителя >|Указывает разделитель одно значение для поиска в качестве значения имя REG_MULTI_SZ. Если *разделителя* не указан, **\0** используется.|
|/f \<данных >|Указывает значение или шаблон для поиска. Используйте двойные кавычки, если строка содержит пробелы. Если не указан подстановочный знак (**&#42;**) используется в качестве шаблона поиска.|
|/k|Задает поиск только имена ключей.|
|/d|Задает поиск только данные.|
|/c|Указывает, что запрос чувствительно к регистру. По умолчанию запросы не учитывают регистр.|
|/e|Указывает, чтобы вернуть только точные совпадения. По умолчанию возвращаются все совпадения.|
|/t \<тип >|Указывает типы реестра для поиска. Допустимые типы: REG_SZ, REG_MULTI_SZ, REG_EXPAND_SZ, REG_DWORD, REG_BINARY, REG_NONE. Если не указан, производится поиск всех типов.|
|/ z|Задает включение числового эквивалента тип реестра в результатах поиска.|
|/?|Отображает справку по **reg query** в командной строке.|

## <a name="remarks-optional-section"></a>"Примечания" \<дополнительный раздел >

В следующей таблице перечислены возвращаемые значения для **reg query** операции.

|Значение|Описание|
|-----|-----------|
|0|Success|
|1|Отказ|

## <a name="BKMK_examples"></a>Примеры

Чтобы отобразить имя значение версии в ключе HKLM\Software\Microsoft\ResKit, введите следующую команду:
```
REG QUERY HKLM\Software\Microsoft\ResKit /v Version
```
Чтобы отобразить все подразделы и значения в разделе HKLM\Software\Microsoft\ResKit\Nt\Setup на удаленном компьютере с именем ABC, введите следующую команду:
```
REG QUERY \\ABC\HKLM\Software\Microsoft\ResKit\Nt\Setup /s
```
Чтобы отобразить все подразделы и значения REG_MULTI_SZ типа с помощью **#** в качестве разделителя, введите:
```
REG QUERY HKLM\Software\Microsoft\ResKit\Nt\Setup /se #
```
Чтобы отобразить ключ, значение и с учетом регистра и соответствует системы в корневом разделе HKLM данных типа REG_SZ, введите:
```
REG QUERY HKLM /f SYSTEM /t REG_SZ /c /e
```
Чтобы отобразить ключ, значение и данные, соответствующие **0F** в данных в разделе HKCU корневой раздел данных типа REG_BINARY.
```
REG QUERY HKCU /f 0F /d /t REG_BINARY
```
Чтобы отобразить значение и данных для имен значение null (значение по умолчанию) в разделе HKLM\SOFTWARE, введите:
```
REG QUERY HKLM\SOFTWARE /ve
```
