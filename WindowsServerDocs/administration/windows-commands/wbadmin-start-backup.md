# <a name="wbadmin-start-backup"></a>wbadmin start backup

Создает резервную копию с использованием указанных параметров. Если параметры не указаны и вы создали запланированную ежедневную архивацию, эта команда создает резервную копию с использованием параметров запланированной архивации. Если указаны параметры, создается резервная копия копии служба теневого копирования томов (VSS) и не обновляется журнал файлов, для которых выполняется резервное копирование.

Чтобы создать однократную резервную копию с помощью этой команды, необходимо быть членом группы " **Операторы резервного копирования** " или " **Администраторы** ", либо вам должны быть делегированы соответствующие разрешения. Кроме того, необходимо запустить программу **Wbadmin** из командной строки с повышенными привилегиями, щелкнув правой кнопкой мыши **командную строку** и выбрав команду **Запуск от имени администратора**.

## <a name="syntax"></a>Синтаксис

```com
wbadmin start backup [-backupTarget:{<BackupTargetLocation> | <TargetNetworkShare>}] [-include:<ItemsToInclude>] [-nonRecurseInclude:<ItemsToInclude>] [-exclude:<ItemsToExclude>] [-nonRecurseExclude:<ItemsToExclude>] [-allCritical] [-systemState] [-noVerify] [-user:<UserName>] [-password:<Password>] [-noInheritAcl] [-vssFull | -vssCopy] [-quiet]
```

### <a name="parameters"></a>Параметры

| Параметр | Описание |
|--|--|
| -backupTarget | Указывает место хранения для этой резервной копии. Требуется буква жесткого диска (f:), путь на основе GUID тома в формате `\\?\Volume{GUID}` или UNC-путь к удаленной общей папке `(\\<servername>\<sharename>\)` . По умолчанию резервная копия будет сохранена в: `\\<servername>\<sharename>\WindowsImageBackup\<ComputerBackedUp>\` . |
| -include | Задает разделенный запятыми список элементов, включаемых в резервную копию. Можно включить несколько файлов, папок или томов. Пути к томам можно указать с помощью букв диска тома, точек подключения томов или имен томов на основе GUID. Если используется имя тома на основе GUID, оно должно заканчиваться обратной косой чертой ( `\` ). `*`При указании пути к файлу можно использовать подстановочный знак () в имени файла. Параметр **-include** следует использовать только в сочетании с параметром **-backupTarget** . |
| — исключить | Задает разделенный запятыми список элементов, исключаемых из резервной копии. Можно исключить файлы, папки или тома. Пути к томам можно указать с помощью букв диска тома, точек подключения томов или имен томов на основе GUID. Если используется имя тома на основе GUID, оно должно заканчиваться обратной косой чертой ( `\` ). `*`При указании пути к файлу можно использовать подстановочный знак () в имени файла. Параметр **-Exclude** следует использовать только совместно с параметром **-backupTarget** . |
| -Нонрекурсеинклуде | Указывает нерекурсивный, разделенный запятыми список элементов, включаемых в резервную копию. Можно включить несколько файлов, папок или томов. Пути к томам можно указать с помощью букв диска тома, точек подключения томов или имен томов на основе GUID. Если используется имя тома на основе GUID, оно должно заканчиваться обратной косой чертой ( `\` ). `*`При указании пути к файлу можно использовать подстановочный знак () в имени файла. Параметр **-нонрекурсеинклуде** следует использовать только в сочетании с параметром **-backupTarget** . |
| -Нонрекурсиксклуде | Указывает нерекурсивный, разделенный запятыми список элементов, исключаемых из резервной копии. Можно исключить файлы, папки или тома. Пути к томам можно указать с помощью букв диска тома, точек подключения томов или имен томов на основе GUID. Если используется имя тома на основе GUID, оно должно заканчиваться обратной косой чертой ( `\` ). `*`При указании пути к файлу можно использовать подстановочный знак () в имени файла. Параметр **-нонрекурсиксклуде** следует использовать только в сочетании с параметром **-backupTarget** . |
| -allCritical | Указывает, что все критические тома (тома, содержащие состояние операционной системы) будут включены в резервные копии. Этот параметр полезен при создании резервной копии для восстановления исходного состояния системы. Его следует использовать, только если указан параметр **-backupTarget** , в противном случае команда завершается ошибкой. Можно использовать с параметром **-include** .<p>**Совет.** Целевой том для резервного копирования критических томов может быть локальным диском, но не может быть любым из томов, входящих в резервную копию. |
| -systemState | Создает резервную копию, которая включает в себя состояние системы в дополнение к любым другим элементам, указанным с помощью параметра **-include** . Состояние системы содержит файлы загрузки (Boot.ini, НДТЛДР, NTDetect.com), реестр Windows, включая параметры COM, SYSVOL (групповые политики и сценарии входа в систему), Active Directory и NTDS. DIT на контроллерах домена и, если установлена служба сертификатов, хранилище сертификатов. Если на сервере установлена роль веб-сервера, будет включена служба IIS метакаталога. Если сервер входит в состав кластера, будут также включены сведения о службе кластеров. |
| -не проверять | Указывает, что резервные копии, сохраненные на съемном носителе (например, на DVD-диске), не проверяются на наличие ошибок. Если этот параметр не используется, резервные копии, сохраненные на съемном носителе, проверяются на наличие ошибок. |
| пользователь | Если резервная копия сохранена в удаленной общей папке, указывает имя пользователя с разрешением на запись в папку. |
| -password | Указывает пароль для имени пользователя, предоставленного параметром **-User**. |
| -Ноинхеритакл | Применяет разрешения списка управления доступом (ACL), соответствующие учетным данным, предоставленным параметрами **-User** и **-Password** `\\<servername>\<sharename>\WindowsImageBackup\<ComputerBackedUp>\` (папка, содержащая резервную копию). Чтобы получить доступ к резервной копии позже, необходимо использовать эти учетные данные или быть членом группы "Администраторы" или "Операторы архива" на компьютере с общей папкой. Если параметр **-ноинхеритакл** не используется, разрешения ACL из удаленной общей папки применяются к `\<ComputerBackedUp>` папке по умолчанию, чтобы любой пользователь, имеющий доступ к удаленной общей папке, мог получить доступ к резервной копии. |
| -Вссфулл | Выполняет полное резервное копирование с помощью служба теневого копирования томов (VSS). Выполняется резервное копирование всех файлов. журнал каждого файла обновляется с учетом того, что была создана резервная копия, а журналы предыдущих резервных копий могут быть усечены. Если этот параметр не используется, программа **Wbadmin start backup** создает резервную копию копии, но журнал резервных копий файлов не обновляется.<p>**Внимание!** Не используйте этот параметр, если вы используете продукт, отличный от cистема архивации данных Windows Server, для резервного копирования приложений, которые находятся на томах, включенных в текущую резервную копию. Это может привести к нарушению добавочного, разностного или другого типа резервных копий, создаваемых другим продуктом резервного копирования, так как журнал, на котором они основаны, определяет, сколько данных может отсутствовать в резервной копии, и может выполнить полное резервное копирование необязательно. |
| -Всскопи | Выполняет резервное копирование с помощью VSS. Выполняется резервное копирование всех файлов, но журнал резервных копий файлов не обновляется, поэтому вы сохраняете все данные, на которые были внесены изменения, удалены и т. д., а также любые файлы журнала приложений. Использование этого типа резервного копирования не влияет на последовательность добавочных и разностных резервных копий, которые могут происходить независимо от этой резервной копии. Это значение по умолчанию.<p>**Предупреждение:** Резервную копию нельзя использовать для добавочных или разностных резервных копий или восстановления. |
| -quiet | Выполняет команду без запроса пользователю. |

#### <a name="remarks"></a>Комментарии

- Если сохранить резервную копию в удаленной общей папке, а затем выполнить еще одну резервную копию на том же компьютере и в той же удаленной общей папке, то предыдущая резервная копия будет перезаписана.

- Если операция резервного копирования завершится неудачей, резервное копирование можно завершить без необходимости, поскольку старая резервная копия перезаписывается, но новая резервная копия не может использоваться. Чтобы избежать этого, рекомендуется создавать вложенные папки в удаленной общей папке для Организации резервных копий. Тем не менее, в связи с этой организацией у вас должно быть вдвое больше места, чем в родительской папке.

## <a name="examples"></a>Примеры

Чтобы создать резервную копию томов *e:*, *d: \\ mountpoint* и `\\?\Volume{cc566d14-4410-11d9-9d93-806e6f6e6963}\` to Volume *f:*, введите:

```
wbadmin start backup -backupTarget:f: -include:e:,d:\mountpoint,\\?\Volume{cc566d14-44a0-11d9-9d93-806e6f6e6963}\
```

Чтобы выполнить однократную архивацию *f: \\ папка1* и *h: \\ folder2* на том *d:*, чтобы создать резервную копию состояния системы и создать резервную копию, чтобы обычно запланированное разностное резервное копирование не влияло, введите:

```
wbadmin start backup –backupTarget:d: -include:g\folder1,h:\folder2 –systemstate -vsscopy
```

Чтобы выполнить однократную, нерекурсивную резервную копию *d: \\ папка1* в `\\backupshare\backup1*` сетевую папку и ограничить доступ к членам группы " **Администраторы** " или " **Операторы резервного копирования** ", введите:

```
wbadmin start backup –backupTarget: \\backupshare\backup1 -noinheritacl -nonrecurseinclude:d:\folder1
```
