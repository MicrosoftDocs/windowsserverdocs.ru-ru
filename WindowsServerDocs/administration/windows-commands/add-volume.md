# <a name="add-volume"></a>Добавление тома



Добавляет теневого копирования набора, который представляет собой набор томов для теневого копирования томов. Эта команда необходима для создания теневых копий. При использовании без параметров, **добавить том** отображает справку в командной строке.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
add volume <Volume> [provider <ProviderID>]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|\<Тома >|Указывает том, который требуется добавить в набор теневого копирования. По крайней мере один том является обязательным для теневой копии.|
|[поставщика \<ProviderID >]|Указывает идентификатор поставщика зарегистрированного поставщика, который используется для создания теневой копии. Если **поставщика** не указан, используется поставщик по умолчанию.|

## <a name="remarks"></a>Примечания

-   Добавляются тома по одному.
-   Каждый раз при добавлении тома, он проверяется, чтобы убедиться, что VSS поддерживает создание теневой копии этого тома. Эта проверка первичной может стать недействительной, тем не менее, с помощью более поздней версии **контекст набора** команды.
-   При создании теневого копирования, переменной среды связывает псевдоним идентификатор теневой, чтобы затем псевдоним можно использовать для создания сценариев.

## <a name="BKMK_examples"></a>Примеры

Чтобы просмотреть текущий список зарегистрированных поставщиков в `DISKSHADOW>` , введите:
```
list providers
```
Следующие выходные данные демонстрируют один поставщик, который будет использоваться по умолчанию:
```
* ProviderID: {b5946137-7b9f-4925-af80-51abd60b20d5}
        Type: [1] VSS_PROV_SYSTEM
        Name: Microsoft Software Shadow Copy provider 1.0
        Version: 1.0.0.7
        CLSID: {65ee1dba-8ff4-4a58-ac1c-3470ee2f376a}
1 provider registered.
```
Чтобы добавить в набор теневого копирования диска C и назначить псевдоним с именем System1, тип:
```
add volume c: alias System1
```
