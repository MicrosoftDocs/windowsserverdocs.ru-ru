# <a name="add-volume"></a>Добавить том



Добавляет тома в набор теневых копий, который представляет собой набор томов для теневого копирования. Эта команда необходима для создания теневых копий. Если используется без параметров, команда **Добавить том** отображает справку в командной строке.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
add volume <Volume> [provider <ProviderID>]
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|@no__t 0Volume >|Указывает том, добавляемый в набор теневых копий. Для создания теневой копии требуется по крайней мере один том.|
|[Provider @no__t — 0ProviderID >]|Указывает идентификатор поставщика зарегистрированного поставщика, который будет использоваться для создания теневой копии. Если **поставщик** не указан, используется поставщик по умолчанию.|

## <a name="remarks"></a>Примечания

-   Тома добавляются по одному за раз.
-   При каждом добавлении тома он проверяется, чтобы служба VSS поддерживала создание теневых копий этого тома. Однако эта основная проверка может стать недействительной при использовании команды **Set context** в дальнейшем.
-   При создании теневой копии переменная среды связывает псевдоним с идентификатором теневого номера, поэтому псевдоним можно использовать для создания скриптов.

## <a name="BKMK_examples"></a>Примеров

Чтобы просмотреть текущий список зарегистрированных поставщиков, в командной строке `DISKSHADOW>` введите:
```
list providers
```
В следующем выводе отображается один поставщик, который будет использоваться по умолчанию:
```
* ProviderID: {b5946137-7b9f-4925-af80-51abd60b20d5}
        Type: [1] VSS_PROV_SYSTEM
        Name: Microsoft Software Shadow Copy provider 1.0
        Version: 1.0.0.7
        CLSID: {65ee1dba-8ff4-4a58-ac1c-3470ee2f376a}
1 provider registered.
```
Чтобы добавить диск C в набор теневых копий и назначить псевдоним с именем System1, введите:
```
add volume c: alias System1
```
