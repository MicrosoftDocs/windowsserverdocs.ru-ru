# <a name="query-user"></a>запрос пользователя

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Отображает сведения о пользовательских сеансах на сервере узла сеансов удаленных рабочих столов (rd узла сеансов).
Примеры для использования этой команды, см. в разделе [примеры](#BKMK_examples).
> [!NOTE]
> В Windows Server 2008 R2 службы терминалов были переименованы на службы удаленных рабочих столов. Чтобы найти новые возможности в последней версии, см. в разделе [какие возможности служб удаленных рабочих столов в Windows Server 2012](https://technet.microsoft.com/library/hh831527) в технической библиотеке Windows Server.
## <a name="syntax"></a>Синтаксис
```
query user [<UserName> | <SessionName> | <SessionID>] [/server:<ServerName>]
```
## <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|<UserName>|Указывает имя входа пользователя, который требуется запросить.|
|<SessionName>|Задает имя сеанса, который требуется запросить.|
|<SessionID>|Указывает идентификатор сеанса, который требуется запросить.|
|/server:<ServerName>|Указывает сервер узла сеансов удаленных рабочих столов, который требуется запросить. В противном случае используется текущий сервер узла сеансов удаленных рабочих столов.|
|/?|Отображение справки в командной строке.|
## <a name="remarks"></a>Примечания
-   Эта команда позволяет узнать, если пользователь вошел в систему для конкретных к удаленному рабочему столу сервера узла сеансов. **запрос пользователя** возвращает следующую информацию:
    -   Имя пользователя
    -   Имя сеанса на сервере узла сеансов удаленных рабочих столов
    -   Идентификатор сеанса
    -   Состояние сеанса ("Активно" или "отключено")
    -   Время простоя (количество минут с момента последнего нажатия клавиши клавиатуры или мыши в сеансе)
    -   Дата и время входа пользователя в систему
-   Чтобы использовать **запроса пользователя**, должен иметь разрешение на полный доступ или запрашивать сведения специального разрешения.
-   Если вы используете **запроса пользователя** без указания <*UserName*>, <*SessionName*>, или <*SessionID*>, список всех Пользователи, которые выполнили вход на сервер возвращается. Кроме того, можно также использовать **запрос сеанса** для отображения списка всех сеансов на сервере.
-   Когда **запроса пользователя** возвращает сведения, символ больше (>) будет отображаться перед текущего сеанса.
-   **/Server** параметр является обязательным только в том случае, если вы используете **запроса пользователя** с удаленного сервера.
## <a name="BKMK_examples"></a>Примеры
-   Чтобы отобразить сведения обо всех пользователях, подключенных к системе, введите следующую команду:
    ```
    query user
    ```
-   Чтобы отобразить сведения о пользователе USER1 на сервере SERver1, введите следующую команду:
    ```
    query user USER1 /server:SERver1
    ```
