# <a name="where"></a>, где



Расположение файлов, соответствующих заданному шаблону поиска.

В разделе [Примеры](#BKMK_examples) показан принцип использования этой команды.

## <a name="syntax"></a>Синтаксис

```
where [/r <Dir>] [/q] [/f] [/t] [$<ENV>:|<Path>:]<Pattern>[ ...] 
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|/r \<Dir >|Указывает, рекурсивный поиск, начиная с указанного каталога.|
|/q|Возвращает код выхода (**0** для достижения успеха, **1** для сбоя) без отображения списка найденных файлов.|
|/f|Отображает результаты **где** команду в кавычки.|
|/t|Отображает размер файла, Дата последнего изменения и время каждого соответствующего файла.|
|[$\<ENV >:\|\<путь >:]\<шаблон > [...]|Задает шаблон поиска для искомых файлов. Необходим по крайней мере один шаблон и шаблон может включать символы-шаблоны (**&#42;** и **?**). По умолчанию **где** выполняет поиск в текущем каталоге и пути, указанные в переменной среды PATH. Можно указать другой путь для поиска с помощью формате $*ENV*:*шаблон* (где *ENV* значение существующей переменной среды, содержащий один или несколько путей) или с помощью формат *путь*:*шаблон* (где *путь* — это путь к каталогу для поиска). Эти дополнительные форматы не следует использовать с **/r** параметр командной строки.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания

-   Если вы не укажете расширение имени файла, расширениях, указанных в переменной среды PATHEXT добавляются в шаблон по умолчанию.
-   **Где** можно выполнить рекурсивный поиск, отображать сведения о файле, такие как даты или размер и примите переменные среды вместо пути на локальных компьютерах.

## <a name="BKMK_examples"></a>Примеры

Чтобы найти все файлы с именем Test на диске C локального компьютера и его подкаталогах, введите:
```
where /r c:\ test 
```
Чтобы получить список всех файлов в общем каталоге, введите следующую команду:
```
where $public:*.*
```
Чтобы найти все файлы с именем Notepad на диске C удаленного компьютера, Computer1 и его подкаталогах, введите:
```
where /r \\computer1\c notepad.*
```
