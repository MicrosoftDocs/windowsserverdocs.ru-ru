# <a name="bootcfg-ems"></a>bootcfg ems

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Позволяет пользователю добавить или изменить параметры для перенаправления консоли служб аварийного управления к удаленному компьютеру. Включение служб аварийного управления, добавьте «перенаправления = порт #» строки в раздел [загрузчик] файл Boot.ini и параметр/Redirect, чтобы в строке записи определенной операционной системы. Функция служб аварийного управления включена только на серверах.

## <a name="syntax"></a>Синтаксис
```
bootcfg /ems {ON | OFF | edit} [/s <computer> [/u <Domain>\<User> /p <Password>]] [/port {COM1 | COM2 | COM3 | COM4 | BIOSSET}] [/baud {9600 | 19200 | 38400 | 57600 | 115200}] [/id <OSEntryLineNum>]
```
## <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|{ON &#124; OFF&#124; edit}|Указывает значение для перенаправления EMS.<br /><br />**ON** -позволяет удаленных выходных данных для указанного <OSEntryLineNum>. Добавляет указанный параметр/redirect <OSEntryLineNum> и перенаправления = com<X> параметр в разделе [загрузчик]. Значение com<X> задается **/port** параметра.<br /><br />**ОТКЛЮЧЕНИЕ** -отключает вывод на удаленный компьютер. Удаляет из указанного параметра/redirect <OSEntryLineNum> и перенаправления = com<X> параметр из раздела [загрузчик].<br /><br />**изменить** -позволяет вносить изменения параметров портов, изменив перенаправления = com<X> параметр в разделе [загрузчик]. Значение com<X> сбрасывается до значения, указанного в **/port** параметра.|
|/s <computer>|Указывает имя или IP-адрес удаленного компьютера (не используйте символы обратной косой черты). По умолчанию используется локальный компьютер.|
|/u <Domain>\\<User>|Выполняет команду с разрешениями учетной записи пользователя, указанного по <User> или <Domain> \\ <User>. По умолчанию используется разрешения текущего, вошедшего в систему пользователя на компьютере, используя следующую команду.|
|/p <Password>|Указывает пароль для учетной записи пользователя, который указан в **/u** параметра.|
|/port {COM1 &#124; COM2 &#124; COM3 &#124; COM4 &#124; BIOSSET}|Указывает COM-порт, используемый для перенаправления. **BIOSSET** направляет служб аварийного управления, чтобы получить параметры BIOS, чтобы определить, какой порт следует использовать для перенаправления. Не используйте **/port** отключается параметр, если выходные данные для администрирования.|
|/ Baud {9600 &#124; 19200 &#124; 38400&#124; 57600 &#124; 115200}|Скорость передачи, используемый для перенаправления. Не используйте **/BAUD** отключается параметр, если выходные данные для администрирования.|
|/ID <OSEntryLineNum>|Указывает номер строки записи операционной системы, к которой добавляется параметр EMS в разделе [operating systems] файла Boot.ini. Первая строка после заголовка раздела [операционные системы]-1. Этот параметр является обязательным, если присвоено значение служб аварийного управления **ON** или **OFF**.|
|/?|Отображение справки в командной строке.|
## <a name="BKMK_examples"></a>Примеры
В следующих примерах показано, как можно использовать **неправильный** команды:
```
bootcfg /ems on /port com1 /baud 19200 /id 2 
bootcfg /ems on /port biosset /id 3 
bootcfg /s srvmain /ems off /id 2 
bootcfg /ems edit /port com2 /baud 115200 
bootcfg /s srvmain /u maindom\hiropln /p p@ssW23 /ems off /id 2
```
