# <a name="logman-update-trace"></a>Logman update trace

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Обновление свойств сбора данных трассировки событий.  
  
## <a name="syntax"></a>Синтаксис  
```  
logman update trace <[-n] <name>> [options]  
```  
## <a name="parameters"></a>Параметры  
|Параметр|Описание|  
|-------|--------|  
|/?|Отображение контекстной справки.|  
|-s <computer name>|Выполнение команды на удаленный компьютер.|  
|-config <value>|Указывает файл параметров, содержащий параметры команды.|  
|-ets|Отправьте команды сеансам трассировки событий напрямую без сохранения или планирования.|  
|[-n]. <name>|Имя целевого объекта.|  
|-f <bin&#124;bincirc&#124;csv&#124;tsv&#124;sql>|Указывает формат журнала сборщика данных.|  
|-[-]u <user [password]>|Указывает пользователя для запуска от имени. Введя * для пароля выводит приглашение для ввода пароля. Пароль не отображается при вводе пароля.|  
|-m < [start] [stop] [[start] [stop] [...]] >|изменить на ручной запуск или остановка вместо запланированное время begin или end.|  
|-rf < [[чч:] мм:] сс >|Запуск сбора данных в течение указанного периода времени.|  
|-b < м/ГГГГ ч: мин: [AM&#124;PM] >|Начните сбор данных в указанное время.|  
|-e < м/ГГГГ ч: мин: [AM&#124;PM] >|Окончания сбора данных в указанное время.|  
|-o <path&#124;dsn!log>|Указывает выходной файл журнала или имя источника данных и журнала, задайте имя базы данных SQL.|  
|-[-]r|Повторите сборщика данных ежедневно в указанный начала и окончания.|  
|-[-]|Добавьте в существующий файл журнала.|  
|-[-] как|Перезаписать существующий файл журнала.|  
|-v [-] < nnnnnn&#124;mmddhhmm >|Присоедините файл сведений об управлении версиями в конец имени файла журнала.|  
|-[-] версия-кандидат <task>|Запуск указанной команды при каждом закрытии журнала.|  
|-max [-] <value>|Максимальный размер файла журнала в МБ или максимальное количество записей в журнале SQL.|  
|-cnf [-] < [[чч:] мм:] сс >|При указании времени создания файла, по истечении указанного времени. Если времени не указан, создайте новый файл, при превышении максимального размера.|  
|-y|Да, ответьте на все вопросы без запроса подтверждения.|  
|-ct <perf&#124;system&#124;cycle>|Указывает тип clock сеанс трассировки событий.|  
|-ln <logger_name>|Задает имя средства ведения журнала для сеансов трассировки событий.|  
|-ft <[[hh:]mm:]ss>|Задает таймер сброса сеанса трассировки событий.|  
|-p [-] < поставщика [flags [level]] >|Указывает одного поставщика трассировки событий для включения.|  
|-pf <filename>|Указывает файл, содержащий несколько поставщиков трассировки событий для включения. Файл должен быть текстовый файл, содержащий один поставщик каждой строке.|  
|-[-] rt|Запустите сеанс трассировки событий в режиме реального времени.|  
|-ul [-]|Запустите сеанс трассировки событий в пользовательском режиме.|  
|-bs <value>|Указывает размер буфера сеанса трассировки событий, в КБ.|  
|-nb <min max>|Указывает число буферов сеанса трассировки событий.|  
|-mode <globalsequence&#124;localsequence&#124;pagedmemory>|Задает режим ведения журнала сеанса трассировки событий.<br /><br />**Globalsequence** трассировщик событий добавляет порядковый номер для каждого события, он получает независимо от трассировки сеанс получил событие.<br /><br />**Localsequence** трассировщик событий добавляет порядковых номеров для событий, полученных по конкретному сеансу трассировки. Когда **localsequence** параметр используется, может существовать во всех сеансах повторяющихся порядковых номеров, но будет уникальным в пределах каждого сеанса трассировки.<br /><br />**Pagedmemory** использование трассировщик событий выгружаемой системной памяти, а не в памяти невыгружаемого пула по умолчанию для внутренних выделений буфера.|  
## <a name="remarks"></a>Примечания  
Там, где [-] присутствует в списке, отключает параметр.  
## <a name="BKMK_examples"></a>Примеры  
Следующая команда обновляет существующие perf_log сборщика данных, изменение максимального размера журнала до 10 МБ, обновление формата файла журнала в CSV-ФАЙЛ и добавления файлов управления версиями в формате mmddhhmm.  
```  
logman update perf_log -max 10 -f csv -v mmddhhmm  
```
