# <a name="bitsadmin-setsecurityflags"></a>битсадмин сетсекуритифлагс

Устанавливает флаги для HTTP, которые определяют, должен ли BITS проверять список отзыва сертификатов, игнорировать определенные ошибки сертификата и определять политику, используемую, когда сервер перенаправляет HTTP-запрос. Значением является целое число без знака.

## <a name="syntax"></a>Синтаксис

```
bitsadmin /SetSecurityFlags <Job> <Value>
```

### <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|Job|Отображаемое имя задания или идентификатор GUID|
|Значение|См. примечания|

## <a name="remarks"></a>Примечания

Параметр **value** может содержать один или несколько следующих флагов уведомления.

|Действие|Двоичное представление|
|------|---------------------|
|Включить проверку списка отзыва сертификатов|Задать наименьший значащий бит|
|Пропуск недопустимого общего имени в сертификате сервера|Установить 2-й бит справа|
|Пропуск недопустимой даты в сертификате сервера|Установить 3-й бит справа|
|Пропуск недопустимого центра сертификации в сертификате сервера|Установка четвертого бита справа|
|Игнорировать недопустимое использование сертификата|Установка 5 бита справа|
|Политика перенаправления|Управляется от 9 до 11 разрядов справа</br>0, 0, 0 — перенаправления будут разрешены автоматически.</br>0, 0, 1 — удаленное имя в интерфейсе Ибаккграундкопифиле будет обновлено при перенаправлении.</br>0, 1, 0 — если перенаправление произойдет, задание завершится ошибкой.|
|Разрешить перенаправление с HTTPS на HTTP|Установка 12 бита справа|

## <a name="examples"></a><a name=BKMK_examples></a>Примеров

В следующем примере устанавливаются флаги безопасности для включения проверки списка отзыва сертификатов для задания с именем *myJob*.
```
C:\>bitsadmin /SetSecurityFlags myJob 0x0001
```
