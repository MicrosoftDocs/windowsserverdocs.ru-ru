# <a name="bitsadmin-setsecurityflags"></a>bitsadmin setsecurityflags



Задает флаги для протокола HTTP, определить, если BITS должен проверять список отзыва сертификатов, пропускают некоторые ошибки сертификатов и определить политику для использования при сервер перенаправляет HTTP-запроса. Значение — целое число без знака.

## <a name="syntax"></a>Синтаксис

```
bitsadmin /SetSecurityFlags <Job> <Value>
```

## <a name="parameters"></a>Параметры

|Параметр|Описание|
|---------|-----------|
|Job|Отображаемое имя или идентификатор GUID задания|
|Значение|См. в разделе "Примечания"|

## <a name="remarks"></a>Примечания

**Значение** параметр может содержать один или несколько из следующих флагов уведомлений.

|Действие|Двоичное представление|
|------|---------------------|
|Включить проверку списка отзыва Сертификатов|Задать младший значащий бит|
|Игнорировать недопустимое общее имя в сертификате сервера|Установить 2-й бит справа|
|Игнорировать недопустимую дату в сертификате сервера|Установить бит 3-й справа|
|Пропускать недопустимые сертификации в сертификате сервера|Установить 4-й бит справа|
|Игнорировать недопустимое использование сертификата|Установить 5-й бит справа|
|Политику перенаправления|Управляет биты 9-е число в 11-й справа</br>0,0,0 - перенаправления будет автоматически разрешено.</br>0,0,1 - удаленное имя в интерфейсе IBackgroundCopyFile будет обновляться в случае перенаправления.</br>0,1,0 - бит удастся задания, если перенаправление происходит.|
|Разрешить функцию перенаправления с HTTPS-HTTP|Установить 12-й бит справа|

## <a name="BKMK_examples"></a>Примеры

Следующий пример устанавливает флаги безопасности, чтобы включить проверку списка отзыва Сертификатов для задания с именем *myJob*.
```
C:\>bitsadmin /SetSecurityFlags myJob 0x0001
```
