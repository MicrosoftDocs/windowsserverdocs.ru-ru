# <a name="pathping"></a>pathping

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Сведения о задержки в сети и потери пакетов в сети в промежуточные Прыжки между источником и назначением. **Pathping** отправляет несколько echo сообщений-запросов на все маршрутизаторы между источником и назначением за период времени, а затем вычисляет результаты в зависимости от пакетов, полученных от каждого из них. Так как **pathping** отображает уровень потери пакетов каждого маршрутизатора или связи, можно определить, маршрутизаторы или подсети возникли проблемы с сетью. 

**Pathping** выполняет эквивалент **tracert** команду, указав, какие маршрутизаторы расположены по пути. Затем периодически отправляет пакеты проверки связи для все маршрутизаторы в течение указанного времени и вычисляет статистику на основании, возвращенному из каждого. При использовании без параметров, **pathping** отображает справку. 

## <a name="syntax"></a>Синтаксис
```
pathping [/n] [/h] [/g <Hostlist>] [/p <Period>] [/q <NumQueries> [/w <timeout>] [/i <IPaddress>] [/4 <IPv4>] [/6 <IPv6>][<TargetName>]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|/n|Предотвращает **pathping** из пытается разрешить IP-адреса промежуточные маршрутизаторы к их именам. Это может ускорить вывод **pathping** результаты.|
|/h \<MaximumHops >|Указывает максимальное число прыжков в путь для поиска для целевого объекта (назначение). Значение по умолчанию равно 30.|
|/g \<Hostlist >|Указывает, что в IP-заголовке с набором промежуточных мест назначения, указанный в диалоговом окне echo использование сообщений запроса свободной маршрутизации *Hostlist*. При свободной маршрутизации последовательные промежуточные места назначения могут быть разделены одним или несколькими маршрутизаторами. Максимальное число адресов или имен в списке узлов равно 9. *Hostlist* представляет собой ряд IP-адреса (в точечно-десятичной нотации), разделенных пробелами.|
|/p \<период >|Указывает количество миллисекунд ожидания между последовательными проверками связи. Значение по умолчанию — 250 миллисекунд (1/4 секунды).|
|/q \<NumQueries>|Указывает количество echo запрос сообщения, отправленные на каждый маршрутизатор в пути. По умолчанию — 100.|
|/w \<время ожидания >|Указывает количество миллисекунд ожидания для каждого ответа. Значение по умолчанию равно 3000 миллисекунд (3 секунды).|
|/i \<IPaddress>|Указывает адрес источника.|
|/ 4 \<IPv4 >|Указывает, что команда pathping использует только IPv4.|
|/ 6 \<IPv6 >|Указывает, что команда pathping использует только IPv6.|
|\<TargetName >|Указывает целевой объект, который определяется IP-адресом или именем.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания
-   **Pathping** параметры зависят от регистра.
-   Чтобы избежать перегрузки сети, пакеты должны передаваться в довольно большие темпе.
-   Чтобы свести к минимуму последствия потери пакетов, не отправлять запросы проверки связи слишком часто.
-   При использовании **/p** параметр, пакеты передаются по отдельности каждого промежуточных прыжок. По этой причине является интервал между двумя пакетами одному узлу *период* умноженное на количество прыжков.
-   При использовании **/w** параметр, пакеты могут отправляться в параллельном режиме. По этой причине количество времени, указанного в *время ожидания* параметр не ограничивается количество времени, указанного в *период* параметр промежутком.
-   Эта команда доступна только в том случае, если протокол Интернета (TCP/IP) устанавливается как компонент в свойствах сетевого адаптера в сетевых подключений.

## <a name="BKMK_Examples"></a>Примеры

В следующем примере показан **pathping** выходные данные команды:

```
D:\>pathping /n corp1
Tracing route to corp1 [10.54.1.196]
over a maximum of 30 hops:
  0  172.16.87.35
  1  172.16.87.218
  2  192.168.52.1
  3  192.168.80.1
  4  10.54.247.14
  5  10.54.1.196
computing statistics for 125 seconds...
            Source to Here   This Node/Link
Hop  RTT    Lost/Sent = Pct  Lost/Sent = Pct  address
  0                                           172.16.87.35
                                0/ 100 =  0%   |
  1   41ms     0/ 100 =  0%     0/ 100 =  0%  172.16.87.218
                               13/ 100 = 13%   |
  2   22ms    16/ 100 = 16%     3/ 100 =  3%  192.168.52.1
                                0/ 100 =  0%   |
  3   24ms    13/ 100 = 13%     0/ 100 =  0%  192.168.80.1
                                0/ 100 =  0%   |
  4   21ms    14/ 100 = 14%     1/ 100 =  1%  10.54.247.14
                                0/ 100 =  0%   |
  5   24ms    13/ 100 = 13%     0/ 100 =  0%  10.54.1.196
Trace complete.
```
Когда **pathping** — запустить, сначала выводится путь. Это тот же путь, который приводится с использованием **tracert** команды. Далее занят сообщение отображается для около 90 секунд (время зависит от число прыжков). В это время сбора информации из всех маршрутизаторов, перечисленных выше и ссылки между ними. в конце этого периода отображаются результаты теста.

В образце отчета выше **This Node/Link**, **Lost/Sent = Pct** и **адрес** столбцы показывают, что связь между узлами 172.16.87.218 и 192.168.52.1 13 Процент пакетов. Маршрутизаторы в прыжков 2 и 4 также пропускают пакеты, направляемые к ним, но такая потеря не влияет на их возможность перенаправлять трафик, не предназначенный им.

Оценки потерь для связи, обозначаются символом вертикальной черты (**|**) в **адрес** столбца, показывают перегрузку, которая вызывает потерю пакетов, перенаправляемых на путь. Оценки потерь для маршрутизаторов (определяется по их IP-адреса) показывает, что эти маршрутизаторы перегружены.
