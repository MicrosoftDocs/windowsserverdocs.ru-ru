# <a name="tapicfg"></a>tapicfg

> Область применения: Windows Server (половина ежегодного канала), Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Создает, удаляет или отображает раздел каталога приложений TAPI или задает раздел каталога приложений TAPI по умолчанию. Клиенты TAPI 3,1 могут использовать сведения из этого раздела каталога приложений со службой локатора службы каталогов для поиска и взаимодействия с каталогами TAPI. Кроме того, с помощью программы **Tapicfg** можно создавать и удалять точки подключения службы, которые позволяют клиентам TAPI эффективно размещать разделы каталога приложений TAPI в домене. Дополнительные сведения см. в разделе Примечания. Чтобы просмотреть синтаксис команды, щелкните команду.
-   [Установка Tapicfg](#BKMK_install)

-   [Tapicfg ремовескп](#BKMK_removescp)
-   [вывод Tapicfg](#BKMK_show)
-   [Tapicfg македефаулт](#BKMK_makedefault)

## <a name="tapicfg-install"></a><a name="BKMK_install"></a>Установка Tapicfg
Создает раздел каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg install /directory:<PartitionName> [/server:<DCName>] [/forcedefault]
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|Установка/Directory:\<PartitionName>|Обязательный. Указывает DNS-имя создаваемого раздела каталога приложений TAPI. Это имя должно быть полным доменным именем.|
|/Server \<DCName>|Указывает DNS-имя контроллера домена, на котором создан раздел каталога приложений TAPI. Если имя контроллера домена не указано, используется имя локального компьютера.|
|/форцедефаулт|Указывает, что этот каталог является разделом каталога приложений TAPI по умолчанию для домена. В домене может быть несколько разделов каталога приложений TAPI.<p>Если этот каталог является первым разделом каталога приложений TAPI, созданным в домене, он автоматически задается по умолчанию независимо от того, используется ли параметр **/форцедефаулт** .|
|/?|Отображение справки в командной строке.|

## <a name="tapicfg-remove"></a><a name="BKMK_remove"></a>Удаление Tapicfg
Удаляет раздел каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg remove /directory:<PartitionName>
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|удалить/Directory:\<PartitionName>|Обязательный. Указывает DNS-имя удаляемого раздела каталога приложений TAPI. Обратите внимание, что это имя должно быть полным доменным именем.|
|/?|Отображение справки в командной строке.|

## <a name="tapicfg-publishscp"></a><a name="BKMK_publishscp"></a>Tapicfg публишскп
Создает точку подключения службы для публикации раздела каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg publishscp /directory:<PartitionName> [/domain:<DomainName>] [/forcedefault]
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|публишскп/Directory:\<PartitionName>|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI, который будет опубликован точкой подключения службы.|
|/Domain\<DomainName>|Указывает DNS-имя домена, в котором создана точка подключения службы. Если имя домена не указано, используется имя локального домена.|
|/форцедефаулт|Указывает, что этот каталог является разделом каталога приложений TAPI по умолчанию для домена. В домене может быть несколько разделов каталога приложений TAPI.|
|/?|Отображение справки в командной строке.|

## <a name="tapicfg-removescp"></a><a name="BKMK_removescp"></a>Tapicfg ремовескп
Удаляет точку подключения службы для раздела каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg removescp /directory:<PartitionName> [/domain:<DomainName>]
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|ремовескп/Directory:\<PartitionName>|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI, для которого удаляется точка подключения службы.|
|/Domain \<DomainName>|Указывает DNS-имя домена, из которого удаляется точка подключения службы. Если имя домена не указано, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="tapicfg-show"></a><a name="BKMK_show"></a>вывод Tapicfg
Отображает имена и расположение разделов каталога приложений TAPI в домене.

### <a name="syntax"></a>Синтаксис
```
tapicfg show [/defaultonly][ /domain:<DomainName>]
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|/дефаултонли|Отображает имена и расположения только раздела каталога приложений TAPI по умолчанию в домене.|
|/Domain \<DomainName>|Указывает DNS-имя домена, для которого отображаются разделы каталога приложений TAPI. Если имя домена не указано, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="tapicfg-makedefault"></a><a name="BKMK_makedefault"></a>Tapicfg македефаулт
Задает раздел каталога приложений TAPI по умолчанию для домена.

### <a name="syntax"></a>Синтаксис
```
tapicfg makedefault /directory:<PartitionName> [/domain:<DomainName>]
```
#### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|македефаулт/Directory:\<PartitionName>|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI, заданное в качестве раздела по умолчанию для домена. Обратите внимание, что это имя должно быть полным доменным именем. Указывает DNS-имя домена, для которого в качестве раздела каталога приложений TAPI задано значение по умолчанию. Если имя домена не указано, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Remarks
Чтобы выполнить команду **Tapicfg Install** (для создания раздела каталога приложений TAPI) или команду **Tapicfg Remove** (удалить раздел каталога приложений TAPI), необходимо быть членом группы "Администраторы предприятия" в Active Directory.

Это средство командной строки можно запустить на любом компьютере, входящем в домен.

Введенный пользователем текст (например, имена разделов каталога приложений TAPI, серверов и доменов) с международными или Юникод-символами отображается правильно, только если установлены соответствующие шрифты и языковая поддержка.

Вы по-прежнему можете использовать в организации серверы службы локатора Интернета (ILS), если для поддержки определенных приложений требуется ILS, так как клиенты TAPI, работающие под управлением Windows XP или ОС Windows Server 2003, могут запрашивать серверы ILS или разделы каталога приложений TAPI.

Для создания или удаления точек подключения службы можно использовать **Tapicfg** . Если раздел каталога приложений TAPI переименовывается по какой-либо причине (например, при переименовании домена, в котором он находится), необходимо удалить существующую точку подключения службы и создать новую, содержащую новое DNS-имя публикуемого раздела каталога приложений TAPI. В противном случае клиенты TAPI не смогут выполнить обнаружение раздела каталога приложений TAPI и получить к нему доступ. Точку подключения службы также можно удалить в целях обслуживания или безопасности (например, если нежелательно предоставлять доступ к данным TAPI в конкретном разделе каталога приложений TAPI).

## <a name="examples"></a>Примеры
Чтобы создать раздел каталога приложений TAPI с именем tapifiction.testdom.microsoft.com на сервере с именем testdc.testdom.microsoft.com, а затем задать его в качестве раздела каталога приложений TAPI по умолчанию для нового домена, введите:
```
tapicfg install /directory:tapifiction.testdom.microsoft.com /server:testdc.testdom.microsoft.com /forcedefault
```
Чтобы отобразить имя раздела каталога приложений TAPI по умолчанию для нового домена, введите:
```
tapicfg show /defaultonly
```
