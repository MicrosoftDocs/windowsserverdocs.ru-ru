# <a name="tapicfg"></a>tapicfg

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

Создает, удаляет, отображает раздел каталога приложений TAPI или задает раздел каталога приложений по умолчанию TAPI. TAPI 3.1 клиенты могут использовать сведения раздела каталога приложений со службой локатора службы каталогов, чтобы поиск и обмен данными с каталогами TAPI. Можно также использовать **tapicfg** для создания или удаления точек подключения службы, позволяющих клиентам TAPI эффективно находить разделов каталога приложений TAPI в домене. Дополнительные сведения см. в разделе "Примечания". Чтобы просмотреть синтаксис команды, щелкните команду. 
-   [Установка Tapicfg](#BKMK_install)
-   [Удаление Tapicfg](#BKMK_remove)
-   [Tapicfg publishscp](#BKMK_publishscp)
-   [Tapicfg removescp](#BKMK_removescp)
-   [Показать Tapicfg](#BKMK_show)
-   [Tapicfg makedefault](#BKMK_makedefault)

## <a name="BKMK_install"></a>Установка Tapicfg
Создает раздел каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg install /directory:<PartitionName> [/server:<DCName>] [/forcedefault]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|Установка /directory:\<PartitionName >|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI должен быть создан. Это имя должно быть полное доменное имя.|
|/ Server: \<DCName>|Указывает DNS-имя контроллера домена, на котором создается раздел каталога приложений TAPI. Если имя контроллера домена не указан, используется имя локального компьютера.|
|/ forcedefault|Указывает, что этот каталог является раздела каталога приложений по умолчанию TAPI для домена. Может существовать несколько разделов каталога приложений TAPI в домене.<br /><br />Если этот каталог является первый раздел каталога приложений TAPI, созданный в домене, он автоматически устанавливается по умолчанию, независимо от того, используете ли вы **/forcedefault** параметр.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_remove"></a>Удаление Tapicfg
Удаляет раздел каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg remove /directory:<PartitionName>
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|удалить /directory:\<PartitionName >|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI для удаления. Обратите внимание на то, что это имя должно быть полное доменное имя.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_publishscp"></a>Tapicfg publishscp
Создает точку подключения службы для публикации раздела каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg publishscp /directory:<PartitionName> [/domain:<DomainName>] [/forcedefault]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|publishscp /directory:\<PartitionName >|Обязательный. Указывает, что DNS-имя раздела каталога приложений TAPI, точка подключения службы будет публиковать.|
|/ domain:\<DomainName >|Указывает, создается DNS-имя домена, в котором точка подключения службы. Если имя домена не указан, используется имя локального домена.|
|/ forcedefault|Указывает, что этот каталог является раздела каталога приложений по умолчанию TAPI для домена. Может существовать несколько разделов каталога приложений TAPI в домене.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_removescp"></a>tapicfg removescp
Удаляет точку подключения службы для раздела каталога приложений TAPI.

### <a name="syntax"></a>Синтаксис
```
tapicfg removescp /directory:<PartitionName> [/domain:<DomainName>]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|removescp /directory:\<PartitionName>|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI, для которого удаляется точка подключения службы.|
|/ domain: \<Имя домена >|Указывает DNS-имя домена, из которого удаляется точка подключения службы. Если имя домена не указан, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_show"></a>Показать Tapicfg
Отображает имена и расположение разделов каталога приложений TAPI в домене.

### <a name="syntax"></a>Синтаксис
```
tapicfg show [/defaultonly][ /domain:<DomainName>]
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|/defaultonly|Отображает имена и расположение только по умолчанию TAPI раздела каталога приложений в домене.|
|/ domain: \<Имя домена >|Указывает DNS-имя домена, для которого отображаются разделы каталога приложений TAPI. Если имя домена не указан, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="BKMK_makedefault"></a>Tapicfg makedefault
Задает раздела каталога приложений по умолчанию TAPI для домена.

### <a name="syntax"></a>Синтаксис
```
tapicfg makedefault /directory:<PartitionName> [/domain:<DomainName>]  
```
### <a name="parameters"></a>Параметры
|Параметр|Описание|
|-------|--------|
|makedefault /directory:\<PartitionName >|Обязательный. Указывает DNS-имя раздела каталога приложений TAPI раздел по умолчанию для домена. Обратите внимание на то, что это имя должно быть полное доменное имя. Указывает DNS-имя домена, для которой задается TAPI раздела каталога приложений по умолчанию. Если имя домена не указан, используется имя локального домена.|
|/?|Отображение справки в командной строке.|

## <a name="remarks"></a>Примечания
Необходимо быть членом группы "Администраторы предприятия" в active directory для выполнения либо **установить tapicfg** (для создания раздела каталога приложений TAPI) или **удалить tapicfg** (Чтобы удалить приложение TAPI раздел каталога).

Это средство командной строки может выполняться на любом компьютере, который является членом домена.

Передаваемый пользователем текст (например, имена разделов каталога приложений, серверов и доменов TAPI), международных или Юникода символов отображаются только правильно установлены соответствующие шрифты и поддержка языков.

По-прежнему можно использовать серверы службы ILS (Internet Locator Service) в вашей организации, если необходима для поддержки определенных приложений, поскольку TAPI клиентов, работающих под управлением Windows Server 2003 или Windows XP можно запросить ILS серверы или приложения TAPI разделы каталога.

Можно использовать **tapicfg** для создания или удаления точек подключения службы. Если раздел каталога приложений TAPI переименовывается по любой причине (например, при переименовании домена, в котором он находится), необходимо удалить существующую точку подключения службы и создать новый, который содержит новый DNS-имя каталога приложения TAPI раздел для публикации. В противном случае клиенты TAPI не поиска и получения доступа TAPI раздела каталога приложений. Можно также удалить точку подключения службы в целях обслуживания или безопасности (например, если вы не хотите предоставления доступа к данным TAPI конкретного раздела каталога приложений TAPI).

## <a name="examples"></a>Примеры
Чтобы создать раздел каталога приложения TAPI именованный tapifiction.testdom.microsoft.com на сервере с именем testdc.testdom.microsoft.com и задать его в качестве раздела каталога приложений по умолчанию TAPI для нового домена, введите следующую команду:
```
tapicfg install /directory:tapifiction.testdom.microsoft.com /server:testdc.testdom.microsoft.com /forcedefault
```
Чтобы отобразить имя раздела каталога приложений по умолчанию TAPI для нового домена, введите следующую команду:
```
tapicfg show /defaultonly
```
