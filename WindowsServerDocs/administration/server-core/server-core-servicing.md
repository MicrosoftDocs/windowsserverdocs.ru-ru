# <a name="patch-a-server-core-installation"></a>Исправление установки основных серверных компонентов

> Применимо к: Windows Server (Channel точками годовая) и Windows Server 2016

Можно применить исправление сервер, выполняющий установку Server Core следующими способами:

- **Использование Windows Update автоматически или с помощью Windows Server Update Services (WSUS)**. С помощью центра обновления Windows, либо автоматически или с помощью средства командной строки или Windows Server Update Services (WSUS), может обслуживать серверы установки основных серверных компонентов.

- **Вручную**. Даже в организации, которые не использовать Центр обновления Windows или WSUS можно вручную применить обновления.

## <a name="view-the-updates-installed-on-your-server-core-server"></a>Просмотр обновлений, установленных на сервере основных серверных компонентов
Перед добавлением нового обновления ядра сервера рекомендуется просмотреть, какие обновления уже установлены.

Чтобы просмотреть обновлений с помощью Windows PowerShell, выполните **Get-исправлений**.

Чтобы просмотреть обновления, выполнив следующую команду, выполните **systeminfo.exe**. Хотя средство проверяет в системе возможно короткой паузы.

Также можно запустить **wmic qfe списка** с помощью командной строки. 

## <a name="patch-server-core-automatically-with-windows-update"></a>Исправление Server Core автоматически с помощью центра обновления Windows

Применить исправление server автоматически с помощью центра обновления Windows, выполните следующие действия:

1. Проверьте текущее значение центра обновления Windows:
   ```
   %systemroot%\system32\Cscript scregedit.wsf /AU /v 
   ```

2. Для включения автоматического обновления:

   ```
   Net stop wuauserv 
   %systemroot%\system32\Cscript scregedit.wsf /AU 4 
   Net start wuauserv
   ```  

3. Чтобы отключить автоматическое обновление, выполните следующую команду:

   ```
   Net stop wuauserv 
   %systemroot%\system32\Cscript scregedit.wsf /AU 1 
   Net start wuauserv 
   ```

Если сервер является членом домена, также можно настроить с помощью групповой политики центра обновления Windows. Дополнительные сведения можно https://go.microsoft.com/fwlink/?LinkId=192470. Тем не менее при использовании этого метода, единственно возможный вариант 4 («автоматической загрузки и установки расписания») относится к установки Server Core из-за нехватки графического интерфейса. Для управления процессом которого установленных обновлений и когда, можно использовать скрипт, который предоставляет командной строки эквивалент большая часть графического интерфейса центра обновления Windows. Сведения о скрипте можно https://go.microsoft.com/fwlink/?LinkId=192471.

Для принудительного обновления, чтобы немедленно обнаруживать и установить все доступные обновления, выполните следующую команду:

```
Wuauclt /detectnow 
```

В зависимости от установленных обновлений может потребоваться перезагрузка компьютера, несмотря на то, что система не будет уведомлять об этом. Чтобы определить, если процесс установки, используйте диспетчер задач для проверки, что **Wuauclt** или **Надежные установщика** процессы не запущена. Можно также использовать методы в [представлении Список обновлений, установленную на сервере ядра сервера](#view-the-updates-installed-on-your-Server-Core-server) для проверки список установленных обновлений.

## <a name="patch-the-server-with-wsus"></a>Исправления сервера с помощью WSUS 

Если сервер основных серверных компонентов является членом домена, его можно настроить для использования сервера WSUS с помощью групповой политики. Для получения дополнительных сведений загрузите [справочную информацию групповой политики](https://www.microsoft.com/download/details.aspx?id=25250). Также можно обратиться к [Настройка параметров групповой политики для автоматического обновления](../windows-server-update-services/deploy/4-configure-group-policy-settings-for-automatic-updates.md)

## <a name="patch-the-server-manually"></a>Вручную применить исправление сервера

Загрузите обновление и сделать его доступным для установки основных серверных компонентов.
В командной строке выполните следующую команду:

```
Wusa <update>.msu /quiet 
```

В зависимости от установленных обновлений может потребоваться перезагрузка компьютера, несмотря на то, что система не будет уведомлять об этом.

Чтобы удалить обновление вручную, выполните следующую команду:

```
Wusa /uninstall <update>.msu /quiet 
```
