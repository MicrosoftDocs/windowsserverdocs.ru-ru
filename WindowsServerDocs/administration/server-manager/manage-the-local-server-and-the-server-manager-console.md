# <a name="manage-the-local-server-and-the-server-manager-console"></a>Управление локальным сервером и консолью диспетчера серверов

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В Windows Server, диспетчер серверов позволяет управлять как локальным сервером (Если вы используете Диспетчер серверов в Windows Server, а не в операционной системе клиента на основе Windows) и удаленных серверах, работающих под управлением Windows Server 2008 и более поздние выпуски Windows Серверная операционная система.

**Локального сервера** страницу в диспетчере сервера отображает данные счетчиков свойства, события, службы и производительности сервера и приводит рекомендации анализатора соответствия рекомендациям (BPA) для локального сервера. Плитки событий, служб, плитки анализатора соответствия рекомендациям (BPA) и плитки производительности работают так, как на страницах групп серверов и ролей. Дополнительные сведения о настройке данных, которые отображаются на этих плитках, см. в разделах [View и Configure Performance, Event, и Service Data и [Run Best Practices Analyzer Scans и Manage Scan Results.

Команды меню и параметры на панелях заголовка консоли диспетчера сервера применяются глобально ко всем серверам в пуле серверов и позволяют с помощью диспетчера серверов для управления всем пулом серверов.

## <a name="BKMK_shutdown"></a>Завершить работу локального сервера
**Задачи** меню на локальном сервере **свойства** позволяет плитку, вы запустите сеанс Windows PowerShell на локальном сервере, откройте **компьютера управления** mmc оснастку либо открыть оснастки MMC для ролей или компонентов, установленных на локальном сервере. Также завершить работу локального сервера можно с помощью команды **Завершить работу локального сервера** в меню **Задачи**. Команда **Завершить работу локального сервера** также доступна для локального сервера на плитке **Серверы** на странице **Все серверы** либо на любой странице роли или группы, на которой представлен локальный сервер.

Завершение работы локального сервера с помощью этого метода, в отличие от завершения работы Windows Server 2016 из **запустить** откроется экран **завершение работы Windows** диалоговое окно, в котором можно указать причины завершения работы в **регистрации событий завершения работы** области.

> [!NOTE]
> Завершить работу сервера или перезагрузить его могут только члены группы "Администраторы". Обычные пользователи не могут завершить работу сервера или перезагрузить его. В результате выполнения команды **Завершить работу локального сервера** сеансы обычных пользователей на сервере закрываются. При этом происходит то же самое, что и при выполнении команды завершения работы **Alt+F4** обычным пользователем на рабочем столе сервера.

## <a name="BKMK_props"></a>Настройка свойств диспетчера серверов
На плитке **Свойства** на странице **Локальный сервер** можно просмотреть или изменить следующие параметры. Чтобы изменить значение параметра, щелкните гиперссылку со значением параметра.

> [!NOTE]
> Обычно свойства, отображаемые на плитке **Свойства** локального сервера, можно изменить только на локальном сервере. Свойства локального сервера невозможно изменить с удаленного компьютера с помощью диспетчера серверов, так как **свойства** плитки можно получать сведения только о локальном компьютере, а не об удаленных компьютерах.

## <a name="BKMK_managesm"></a>Управление консолью диспетчера серверов
Глобальные параметры, применяемые ко всей консоли диспетчера серверов и всем удаленным серверам, которые были добавлены в пул серверов диспетчера серверов, находятся на панелях заголовка в верхней части окна консоли диспетчера серверов.

### <a name="add-servers-to-server-manager"></a>Добавление серверов в диспетчер сервера
Команда, которая открывает **добавить серверы** диалоговое окно и позволяет добавить удаленные физические или виртуальные серверы в пул серверов диспетчера серверов, находится в **управление** меню консоли Server Manager. Подробные сведения о способах добавления серверов см. в разделе [добавить серверы в диспетчер серверов](add-servers-to-server-manager.md).

### <a name="refresh-data-that-is-displayed-in-server-manager"></a>Обновление данных, отображающихся в диспетчере серверов
Можно настроить интервал обновления для данных, который отображается в диспетчере серверов на **свойства диспетчера серверов** диалоговое окно, которое открывается из **управление** меню.

##### <a name="to-configure-the-refresh-interval-in-server-manager"></a>Настройка интервала обновления в диспетчере серверов

1.  На **управление** меню в консоли диспетчера серверов щелкните **свойства диспетчера серверов**.

2.  В **свойства диспетчера серверов** диалоговом окне укажите период времени в минутах, который должен проходить между операциями обновления данных, которые отображаются в диспетчере сервера. Значение по умолчанию составляет 10 минут. По завершении нажмите кнопку ОК.

#### <a name="refresh-limitations"></a>Ограничения обновления
Обновление применяется глобально к данным всех серверов, которые были добавлены в пул серверов диспетчера серверов. Невозможно обновить данные или настроить различные интервалы обновления для отдельных серверов, ролей или групп.

При серверов, входящих в кластер добавляются в диспетчер серверов, являются ли они физические компьютеры или виртуальные машины, Первое обновление данных может завершиться ошибкой или отображения данных только для сервера узла для кластерных объектов. При последующих обновлениях будут отображаться точные данные для физических или виртуальных серверов в кластере серверов.

Данные, которые отображаются на домашних страницах ролей в диспетчере сервера для служб удаленных рабочих столов, IP-адрес управления и файловые службы и службы хранилища не обновляется автоматически. Обновить данные, отображающиеся на этих страницах вручную, нажав **F5** или щелкнув **обновить** в диспетчере сервера заголовке консоли на этих страницах.

### <a name="add-or-remove-roles-or-features"></a>Добавление или удаление ролей и компонентов
Команды, откройте мастер компонентов и добавить роли и удаление ролей и мастер компонентов и позволяют, добавление или удаление ролей, служб ролей и компонентов на серверы в пул серверов, находятся в **управление** меню диспетчера серверов консоль и **задачи** меню **роли и компоненты** плитку на страницах ролей или групп. Подробные сведения о добавлении и удалении ролей и компонентов см. в разделе [Install or Uninstall Roles, Role Services, or Features](install-or-uninstall-roles-role-services-or-features.md).

В диспетчере серверов, ролей и компонентов данные отображаются на базовом языке системы, что также называется языком графического интерфейса пользователя системы по умолчанию на языке, выбранном во время установки операционной системы.

### <a name="create-server-groups"></a>Создание групп серверов
Команда, которая открывает **создать группу серверов** диалоговое окно и позволяет создавать настраиваемые группы серверов, находится в **управление** меню консоли Server Manager. Подробные сведения о создании групп серверов см. в разделе [создать and Manage Server Groups](create-and-manage-server-groups.md).

### <a name="prevent-server-manager-from-opening-automatically-at-logon"></a>Запрет автоматического открытия диспетчера серверов при входе в систему
**Не запускать диспетчер серверов автоматически при входе в систему** флажок в **свойства диспетчера серверов** элементы управления диалогового окна ли диспетчер сервера открывается автоматически при входе в систему для членов Группы администраторов на локальном сервере. Этот параметр не влияет на поведение диспетчера сервера при работе в Windows 10 как часть средств удаленного администрирования сервера. Дополнительные сведения о настройке этого параметра см. в разделе [диспетчера серверов](server-manager.md).

### <a name="zoom-in-or-out"></a>Увеличение или уменьшение масштаба
Чтобы увеличить или уменьшить масштаб представления консоли диспетчера серверов, вы можете использовать **масштаб** команды **представление** меню или нажмите клавишу **Ctrl + плюс (+)** для увеличения и **Ctrl + знак минус (-)** для уменьшения масштаба.

## <a name="BKMK_tools"></a>Настройка средств, которые отображаются в меню "Сервис"
**Средства** в диспетчере серверов откройте меню содержит гибкие ссылки на ярлыки в **Администрирование** папку в **панель управления/система и безопасность**. **Администрирование** папка содержит список ярлыков или LNK-файлов для доступных средств управления, таких как оснастки mmc. Заполняет Диспетчер серверов **средства** меню со ссылками на эти ярлыки и копирует структуру папки **Администрирование** папку **средства** меню. По умолчанию средства в папке "Администрирование" организованы в виде неструктурированного списка, отсортированного по типу и имени. В диспетчере серверов**средства** меню, элементы сортируются только по имени, а не по типу.

Чтобы настроить меню **Сервис** , скопируйте ярлыки средств или сценариев, которые требуется использовать, в папку **Администрирование** . Также ярлыки можно разместить в папках, в результате чего будут созданы вложенные меню в меню **Сервис** . Кроме того Если вы хотите ограничить доступ к настраиваемым средствам в **средства** меню, можно задать права доступа пользователя к папкам пользовательский инструмент «Администрирование» или непосредственно в исходные файлы средств или сценариев.

Не рекомендуется перемещать системные и административные средства, а также любые средства управления, связанные с ролями и компонентами, установленными на локальном сервере. Перемещение средств управлениями ролями и компонентами может помешать успешному удалению этих средств управления, когда это будет необходимо. После удаления роли или компонента нерабочая ссылка на средство, ярлык которого был удален, может остаться в меню **Сервис**. Если переустановить роль, в меню **Сервис** создается дублирующаяся ссылка на то же самое средство, но одна из ссылок не будет работать.

Однако средства ролей и компонентов, установленные в составе средств удаленного администрирования сервера на компьютере под управлением клиента Windows, можно разместить в настраиваемых папках. Удаление родительской роли или компонента не влияет на ярлыки средств, доступных на удаленном компьютере под управлением Windows 10.

Следующая процедура описывает создание примера папки с именем *MyTools*, и переместите ярлыков для двух сценариев Windows PowerShell в папку, затем будут доступны из меню средства диспетчера серверов.

#### <a name="to-customize-the-tools-menu-by-adding-shortcuts-in-administrative-tools"></a>Настройка меню "Сервис" путем добавления ярлыков в папку "Администрирование"

1.  Создайте новую папку с именем *MyTools* в удобном расположении.

    > [!NOTE]
    > Из-за ограниченных прав доступа к папке **Администрирование** невозможно создать новую папку непосредственно в папке **Администрирование** ; новую папку необходимо создать в другом расположении (например, на рабочем столе), а затем скопировать ее в папку **Администрирование** .

2.  Перемещение или копирование *MyTools* для **панель управления/система и безопасность/Администрирование**. По умолчанию вы должны быть членом группы "Администраторы" на компьютере, чтобы вносить изменения в папку **Администрирование** .

3.  Если вам не нужно ограничивать права доступа пользователя к ярлыкам настраиваемых средств, перейдите к шагу 6. В противном случае щелкните правой кнопкой мыши файл средства (или папку *MyTools*), затем щелкните **Свойства**.

4.  На **безопасности** вкладка файла **свойства** диалоговом окне щелкните **изменить**.

5.  для пользователей, которым требуется ограничить доступ к средству, снимите флажки для **чтения & выполнение**, **чтения**, и **записи** разрешения. Эти разрешения наследуются ярлыком средства в папке **Администрирование**.

    Если изменить права доступа для пользователя, пока пользователь использует диспетчер серверов (или при диспетчера сервера откройте), то изменения не отобразятся в **средства** меню, пока пользователь не перезапустит диспетчера сервера.

    > [!NOTE]
    > При ограничении доступа ко всей папке, скопированной Администрирование, пользователи с ограниченным доступом не могут видеть папку и ее содержимое в диспетчере серверов**средства** меню.
    > 
    > Измените разрешения для папки в **Администрирование** папки. Поскольку скрытые файлы и папки «Администрирование» всегда отображаются в диспетчере серверов**средства** меню, не используйте **Hidden** установки для файла или папки **свойства** диалоговое окно, чтобы ограничить доступ пользователя к ярлыкам настраиваемых средств.
    > 
    > Разрешения **Отказать** всегда перезаписывают разрешения **Разрешить**.

6.  Щелкните правой кнопкой мыши исходное средство, сценарий или исполняемый файл, для которого требуется добавить записи в **средства** меню, а затем щелкните **создать ярлык**.

7.  Переместите ярлык в *MyTools* папки «Администрирование».

8.  Обновите или перезапустите диспетчер серверов, при необходимости, чтобы ярлык настраиваемого средства в см. в разделе **средства** меню.

## <a name="BKMK_roles"></a>Управление ролями на домашних страницах ролей
После добавления серверов в пул серверов диспетчера серверов и Server Manager собирает данные инвентаризации о серверах в пуле, диспетчер сервера добавит страницы в область навигации для ролей, обнаруженных на управляемых серверах. На плитке **Серверы** на страницах ролей перечисляются управляемые серверы, на которых выполняется данная роль. По умолчанию на плитках **События**, **Анализатор соответствия рекомендациям**, **Службы**и **Производительность** отображаются данные для всех серверов, на которых выполняется роль. При выборе конкретных серверов на плитке **Серверы** отображаются результаты событий, служб, счетчиков производительности и анализатора соответствия рекомендациям только для выбранных серверов. Средства управления обычно доступны в консоли диспетчера серверов **средства** меню после роль или компонент установки или обнаружения на управляемом сервере. Также можно щелкнуть правой кнопкой мыши записи сервера на плитке **Серверы** для роли или группы, а затем запустить средство управления, которое требуется использовать.

В Windows Server 2016 следующие роли и компоненты имеют средства управления, которые интегрированы в консоль диспетчера сервера в виде страниц.

-   **Файловые службы и службы хранилища.** Страницы файловых служб и служб хранилища содержат настраиваемые плитки и команды для управления томами, общими ресурсами, виртуальными дисками iSCSI и пулами носителей. При открытии файла и открывает главной страницы роли служб хранилища в диспетчере серверов отображается скрываемая панель, которая отображает страницы настраиваемых средств управления для файловых служб и служб хранилища. Дополнительные сведения о развертывании файловых служб и служб хранилища и управлении ими см. в разделе [Файловые службы и службы хранилища](https://go.microsoft.com/fwlink/p/?LinkId=241530).

-   **Службы удаленных рабочих столов.** Страницы служб удаленных рабочих столов содержат настраиваемые плитки и команды для управления сеансами, лицензиями, шлюзами и виртуальными рабочими столами. Дополнительные сведения о развертывании и управлении служб удаленных рабочих столов см. в разделе [служб удаленных рабочих столов (rdS)](https://go.microsoft.com/fwlink/p/?LinkId=241532).

-   **IP-адрес Management (IPAM).** Страница роли IPAM содержит настраиваемую плитку **приветствия** со ссылками на общие задачи конфигурации и управления IPAM, включая мастер для подготовки IPAM-сервера. Домашняя страница IPAM также содержит плитки для просмотра управляемой сети, сводки конфигурации и назначенных заданий.

    Существуют некоторые ограничения управления IPAM в диспетчере сервера. В отличие от обычных страниц ролей и групп IPAM не имеет плиток **Серверы**, **События**, **Производительность**, **Анализатор соответствия рекомендациям** и **Службы**. Не существует анализатор соответствия рекомендациям для модели для IPAM; Наиболее проверок анализатором соответствия рекомендациям в IPAM не поддерживаются. Чтобы получить доступ к серверам в пуле серверов, на которых выполняется IPAM, создайте настраиваемую группу серверов, на которых выполняется IPAM, и откройте список серверов на плитке **Серверы** на странице настраиваемой группы. Кроме того, доступ к серверам IPAM можно получить с помощью плитки **Серверы** на странице группы **Все серверы**.

    Также на эскизах панели мониторинга отображаются ограниченные строки для IPAM по сравнению с эскизами для других ролей и групп. Щелкнув строки эскизов IPAM, можно просмотреть события, данные производительности и оповещения о статусе управляемости для серверов, на которых выполняется IPAM. Службами, связанными с IPAM, можно управлять с помощью страниц для групп серверов, содержащих IPAM-серверы, например с помощью страницы для группы **Все серверы** .

    Дополнительные сведения о развертывании IPAM и управлении ею см. в разделе [IP-адрес Management (IPAM)](https://go.microsoft.com/fwlink/p/?LinkId=241533).
