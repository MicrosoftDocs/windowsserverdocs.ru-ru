# <a name="view-task-details-and-notifications"></a>Просмотр сведений о задачах и уведомлений

>Область применения. Windows Server 2016

В диспетчере серверов в Windows Server 2012 R2 или Windows Server 2012, при выполнении задач управления, таких как добавление ролей и компонентов, запуск служб, обновление данных, который отображается в консоли диспетчера сервера или создание настраиваемой группы серверов, отображается уведомление **уведомления** области заголовка консоли диспетчера сервера. Уведомления и **задач сведения** диалоговое окно, которое можно открыть из **уведомления** меню, щелкнув значок флага, отображают состояние задач или запросов пользователя, покажут, если не удалось выполнить задачу и помочь вам Устранение неполадок, указывая на подробные сообщения об сбои задач.

## <a name="the-notifications-area"></a>Область уведомлений
**Уведомления** области в строке меню диспетчера серверов, отмеченной значком флага, отображаются результаты задач, которые начинаются в диспетчере сервера. Уведомления информируют о том, были ли задачи, которые запущены в диспетчере серверов успешные и завершившиеся сбоем. Если уведомления доступны для просмотра, количество доступных уведомлений отображается рядом со значком флага. Если задача не завершена, может быть завершена только частично (например, задача не может быть выполнена на всех указанных вами удаленных серверах) или завершена с предупреждениями, флаг **Уведомления**  становится красным. Ниже представлены задачи, для которых отображаются уведомления.

-   Обновление данных, отображаемых в диспетчере серверов (уведомления отображаются для автоматических обновлений только в том случае, если обновление завершится неудачно) вручную

-   Запуск или остановка служб

-   Установка и удаление ролей, служб ролей и компонентов

-   Запуск сканирования рекомендации анализатора соответствия рекомендациям (BPA)

-   Добавление удаленных серверов для управления (уведомления отображаются при сбоях доступа или обновления данных, отображаемых для удаленных серверов)

Элементы в меню **Уведомления** включают индикатор выполнения, краткое описание задачи, имя целевого сервера задачи (или одного из целевых серверов, если их было выбрано несколько), ссылку на связанный элемент управления или диалоговое окно при их наличии и меню **Задачи** . В меню **Задачи** находятся команды, применимые к активному уведомлению (уведомлению, на которое наведен указатель мыши). Например, при остановке службы можно щелкнуть команду **Перезапустить** в меню уведомления **Задачи** , чтобы перезапустить службу.

Уведомления особенно полезны для установки и удаления ролей, служб ролей и компонентов. Например, при запуске установки компонента на удаленном сервере, можно закрыть Добавить роли и функции мастера установки все еще выполняется, но при этом активная задача остается в **уведомления** списка. **Уведомления** отображает индикатор выполнения установки и позволяет повторно открыть Добавить роли и функции мастера, при необходимости, щелкнув **функции мастер добавления ролей и**. Элементы этого списка оповещают, если произошел сбой установки или если для окончания развертывания компонента требуется выполнить дополнительные действия по настройке.

Уведомления также играют важную роль в устранении неполадок с задачами или процессами в диспетчере сервера. Дополнительные сведения об использовании сообщений в **уведомления** области и **задач сведения** диалоговое окно для устранения неполадок неудачных задач или процессов, см. в разделе [диспетчера серверов Руководство по устранению неполадок](https://social.technet.microsoft.com/wiki/contents/articles/13443.windows-server-2012-server-manager-troubleshooting-guide-part-i-overview.aspx).

Чтобы удалить уведомление, больше не требуется просматривать из **уведомления** , наведите курсор мыши на уведомление и нажмите **удалите задачу** (**X**).

## <a name="viewing-and-troubleshooting-tasks-by-using-task-details"></a>Просмотр и диагностика задач с помощью сведения о задаче
**Задач сведения** команды в нижней части **уведомления** откроется меню **задач сведения** диалоговое окно, в котором располагается полное описание событий задачи (запуск, Остановка, предупреждения, успехов или сбоев). Как и другие элементы в диспетчере серверов как **события**, **служб**, и **анализатор соответствия рекомендациям** плитки, можно фильтровать и создавать запросы, чтобы запускать задачи, отображаются в **задач сведения** диалоговое окно. (Дополнительные сведения о фильтрации и создании запросов к элементам управления списками, см. в разделе [фильтрация, сортировка и запрос данных в плитках диспетчера серверов](filter-sort-and-query-data-in-server-manager-tiles.md).) В верхней области можно просмотреть уведомления в порядке их появления в меню **Уведомления** и узнать, сколько уведомлений было создано для определенной задачи. выбрав уведомления в верхней области отображается подробная информация об уведомлении в нижней области.

Нижняя область особенно полезна для диагностики неудачно завершившихся задач. Если диспетчер сервера не может подключиться к или получить данные для сервера, который является членом пула серверов, записи в этой области часто содержат подробные сообщения, включая полный текст базового удаленного управления Windows (WinRM), сети или проблем безопасности, препятствовать связи с целевым сервером Server Manager.
