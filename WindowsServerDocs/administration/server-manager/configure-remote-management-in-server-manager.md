# <a name="configure-remote-management-in-server-manager"></a>Настройка удаленного управления в диспетчере сервера

>Область применения. Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В Windows Server можно использовать диспетчер серверов для выполнения задач управления на удаленных серверах. удаленное управление включено по умолчанию на серверах, работающих под управлением Windows Server 2016. Чтобы удаленно управлять сервером с помощью диспетчера сервера, добавить сервер в пул серверов диспетчера серверов.

Можно использовать диспетчер серверов для управления удаленными серверами, работающих под управлением более ранних версий Windows Server, но для полноценного управления этими предшествующими версиями операционных систем необходимы следующие обновления.

Для управления серверами, работающими под управлением Windows Server версий старше Windows Server 2016, установите следующее программное обеспечение и обновления, чтобы сделать более старых версиях Windows Server управляемыми с помощью диспетчера серверов в Windows Server 2016.

## <a name="BKMK_remote"></a>Включение и отключение удаленного управления
В Windows Server 2016 удаленное управление включено по умолчанию. Перед подключением к компьютеру, на котором работает Windows Server 2016 удаленно с помощью диспетчера серверов, удаленного управления диспетчера серверов необходимо включить на конечном компьютере, если она была отключена. В процедурах данного раздела описано, как отключить удаленное управление и как его вновь включить, если оно было отключено. В консоли диспетчера серверов, отображается состояние удаленного управления для локального сервера в **свойства** область **локального сервера** страницы.

Локальные учетные записи с правами администратора, за исключением встроенной учетной записи администратора, могут не иметь права на удаленное управление сервером, даже если удаленное управление включено. Удаленного контроля учетных записей (UAC) **LocalAccountTokenFilterPolicy** необходимо настроить параметр реестра, чтобы разрешить локальным учетным записям из группы администраторов, отличные от встроенной учетной записи администратора для удаленного управления сервер.

В Windows Server 2016 Диспетчер серверов зависит от удаленного управления Windows (WinRM) и распределенная модель COM (DCOM) для удаленной связи. Параметры, которые управляются **Настройка удаленного управления** диалоговое окно только влияют на компоненты диспетчера серверов и Windows PowerShell, которые используют для удаленной связи WinRM. Они не влияют на те компоненты диспетчера серверов, которые используют для удаленной связи DCOM. Например диспетчер сервера использует WinRM для взаимодействия с удаленными серверами, работающих под управлением Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012, но использует DCOM для взаимодействия с серверов, работающих под управлением Windows Server 2008 и Windows Server 2008 R2 но не имеют [Windows Management Framework 4.0](https://go.microsoft.com/fwlink/?LinkId=293881) или [Windows Management Framework 3.0](https://go.microsoft.com/fwlink/p/?LinkID=229019) применены обновления. Консоль управления (mmc) и другими традиционными средствами управления используют DCOM. Дополнительные сведения о способах изменения этих параметров см. в разделе [Настройка mmc или другого средства удаленного управления через DCOM](#BKMK_dcom) в этом разделе.

> [!NOTE]
> Процедуры этого раздела можно выполнить только на компьютерах, работающих под управлением Windows Server. Нельзя включить или отключить удаленное управление на компьютере под управлением Windows 10 с помощью этих процедур, так как этой клиентской операционной системой невозможно управлять с помощью диспетчера сервера.

-   Чтобы включить удаленное управление WinRM, выберите одну из следующих процедур.

    -   [Включение удаленного управления диспетчера серверов с помощью интерфейса Windows](#BKMK_windows)

    -   [Включение удаленного управления диспетчера серверов с помощью Windows PowerShell](#BKMK_ps)

    -   [Включение удаленного управления диспетчера серверов с помощью командной строки](#BKMK_cmdline)

    -   [Чтобы включить диспетчер серверов и Windows PowerShell удаленного управления в более ранних выпусках Windows Server](#BKMK_old)

-   Чтобы отключить удаленное управление WinRM и диспетчер сервера, выберите один из следующих процедур.

    -   [Чтобы отключить удаленное управление с помощью групповой политики](#BKMK_disableGP)

    -   [Чтобы отключить удаленное управление с помощью файла ответов при автоматической установке](#BKMK_unattend)

-   Сведения о настройке удаленного управления с помощью DCOM см. в разделе [Настройка удаленного управления с помощью DCOM](#BKMK_dcom).

### <a name="BKMK_windows"></a>Включение удаленного управления диспетчера серверов с помощью интерфейса Windows

1.  > [!NOTE]
    > Параметры, которые управляются **Настройка удаленного управления** диалоговое окно не влияют на части диспетчера серверов, которые используют для удаленной связи DCOM.

    Откройте диспетчер сервера, на компьютере, предназначенном для удаленного управления, если он еще не открыт. На панели задач Windows щелкните **Диспетчер серверов**. На **запустить** щелкните **диспетчера серверов** плитку.

2.  В **свойства** область **локальные серверы** щелкните значение с гиперссылкой для **удаленного управления** свойство.

3.  Выполните одно из указанных ниже действий, а затем нажмите кнопку **ОК**.

    -   Чтобы предотвратить удаленное управление с помощью диспетчера сервера (или Windows PowerShell, если он установлен) этого компьютера, снимите **разрешить удаленное управление этим сервером с других компьютеров** "флажок".

    -   Чтобы разрешить этот компьютер удаленное управление с помощью диспетчера серверов или Windows PowerShell, выберите **разрешить удаленное управление этим сервером с других компьютеров**.

### <a name="BKMK_ps"></a>Включение удаленного управления диспетчера серверов с помощью Windows PowerShell

1.  На компьютере, предназначенном для удаленного управления выполните одно из следующих действий, чтобы открыть сеанс Windows PowerShell с повышенными правами пользователя.

    -   На рабочем столе Windows щелкните правой кнопкой мыши **Windows PowerShell** на панели задач и выберите команду **Запустить от имени администратора**.

    -   В Windows **запустить** щелкните правой кнопкой мыши **Windows PowerShell**, а затем на панели приложения щелкните **Запуск от имени администратора**.

2.  Введите следующую команду и нажмите клавишу **ввод** Чтобы включить все необходимые исключения правил брандмауэра.

    **Настройка SMremoting.exe-включить**

### <a name="BKMK_cmdline"></a>Включение удаленного управления диспетчера серверов с помощью командной строки

1.  На компьютере, которым нужно удаленно управлять, откройте сеанс командной строки с повышенными правами пользователя. Для этого на **запустить** введите **cmd**, щелкните правой кнопкой мыши **командной** плитку, когда она появится в **приложений** результатов, и затем на панели приложения щелкните **Запуск от имени администратора**.

2.  Запустите следующий исполняемый файл.

    **%windir%\system32\Configure-SMremoting.exe**

3.  Выполните одно из следующих действий.

    -   Чтобы отключить удаленное управление, введите **Configure-SMremoting.exe-отключить**, а затем нажмите клавишу **ввод**.

    -   Чтобы включить удаленное управление, введите **Configure-SMremoting.exe-включить**, а затем нажмите клавишу **ввод**.

    -   Чтобы просмотреть текущее значение удаленного управления, введите **Configure-SMremoting.exe-get**, и нажмите клавишу ВВОД.

### <a name="BKMK_dcom"></a>Настройка mmc или другого средства удаленного управления через DCOM

1.  Выполните одно из следующих действий, чтобы открыть оснастку "Брандмауэр Windows в режиме повышенной безопасности".

    -   В **свойства** область **локального сервера** в диспетчере серверов щелкните гиперссылку со значением для **брандмауэра Windows** , а затем щелкните  **Дополнительные параметры**.

    -   На **запустить** введите **WF.msc**и затем выберите плитку оснастки, когда она появится в **приложений** результаты.

2.  В древовидном представлении выберите **Правила для входящих подключений**.

3.  Убедитесь, что исключения для приведенных ниже правил брандмауэра включены и не были переопределены параметрами групповой политики. Если хотя бы одно исключение не включено, перейдите к следующему шагу.

    -   Доступ к сети COM+ (DCOM — входящий трафик)

    -   удаленное управление журналом событий (именованные каналы — входящий трафик)

    -   удаленного управления журнала событий (RPC)

    -   удаленное управление журналом событий (RPC-EPMAP)

4.  Щелкните правой кнопкой мыши отключенные правила, затем в контекстном меню выберите **Включить правило** .

5.  Закройте оснастку "Брандмауэр Windows в режиме повышенной безопасности".

### <a name="BKMK_disableGP"></a>Чтобы отключить удаленное управление с помощью групповой политики

1.  Выполните одно из следующих действий, чтобы открыть редактор локальных групповых политик.

    -   На сервере под управлением Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 с **запустить** введите **gpedit.msc**, а затем нажмите кнопку **gpedit** плитку При отображении.

    -   На сервере под управлением Windows Server 2008 R2 или Windows Server 2008 в **запуска** диалоговом окне **gpedit.msc**, а затем нажмите клавишу **ввод**.

2.  Откройте **компьютера Конфигурация компьютера\Административные шаблоны\Компоненты components\Windows удаленного управления (WinRM) \WinRM службы**.

3.  Дважды щелкните **Разрешить удаленное управление сервером с помощью WinRM** в области содержимого.

4.  В диалоговом окне параметра политики **Разрешить удаленное управление сервером с помощью WinRM** выберите **Отключено** , чтобы отключить удаленное управление. Нажмите кнопку **ОК** , чтобы сохранить изменения и закрыть диалоговое окно параметра политики.

### <a name="BKMK_unattend"></a>Чтобы отключить удаленное управление с помощью файла ответов при автоматической установке

1.  Создайте файл ответов автоматической установки для установок Windows Server 2016 с помощью диспетчера образов системы Windows (Windows SIM). Дополнительные сведения о том, как создать файл ответов и использовании диспетчера установки Windows см. в разделе [что такое диспетчер образов систем Windows?](https://technet.microsoft.com/library/cc766347.aspx) и [пошаговые: Развертывание основных Windows для ИТ-специалистов](https://technet.microsoft.com/library/dd349348.aspx).

2.  В файле ответов, найдите параметр **Microsoft-Windows-Web-Services-for-Management-Core\EnableServerremoteManagement**.

3.  Чтобы отключить удаленного управления диспетчера серверов по умолчанию на всех серверах, к которым вы хотите применить файл ответов, задайте **Microsoft-Windows-Web-Services-for-Management-Core \EnableServerremoteManagement** для **False** .

    > [!NOTE]
    > Этот параметр отключает удаленное управление при установке операционной системы. Настройка этого параметра не мешает администратору Включение удаленного управления диспетчера серверов на сервере после завершения установки операционной системы. Администраторы могут включить удаленное управление еще раз с помощью шагов в диспетчере сервера [для настройки удаленного управления диспетчера серверов с помощью интерфейса Windows](#BKMK_windows) или [Включение удаленного управления диспетчера серверов с помощью Windows PowerShell](#BKMK_ps) в этом разделе.
    > 
    > Если отключить удаленное управление по умолчанию в рамках автоматической установки и не включить удаленное управление на сервере еще раз после установки, серверы, к которым применяется этот файл ответов нельзя полностью управлять с помощью диспетчера сервера. Серверы под управлением Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 (и которых удаленное управление отключено по умолчанию) создания сообщения об ошибках состояния управляемости в консоли диспетчера серверов, после их добавления на сервер диспетчера серверов пул.

## <a name="windows-remote-management-winrm-listener-settings"></a>Параметры прослушивателя удаленного управления (WinRM) Windows
Диспетчер сервера использует параметры прослушивателя WinRM по умолчанию на удаленных серверах, которыми вы хотите управлять. Если механизм проверки подлинности по умолчанию или номер порта прослушивателя WinRM на удаленном сервере было изменено с параметрами по умолчанию, диспетчер серверов не могут взаимодействовать с удаленным сервером.

Ниже приведены параметры прослушивателя WinRM по умолчанию для управления с помощью диспетчера сервера.

-   Служба WinRM запущена.

-   Создан прослушиватель WinRM для принятия HTTP-запросов через порт номер 5985.

-   В параметрах брандмауэра Windows порт номер 5985 включен для разрешения запросов через WinRM.

-   Включены типы проверки подлинности **Kerberos** и **Negotiate**.

Номер порта по умолчанию для связи WinRM с удаленным компьютером — 5985.

Дополнительные сведения о том, как настроить параметры прослушивателя WinRM, в командной строке введите **winrm help config**, и нажмите клавишу ВВОД.
