# <a name="desktop-hosting-service"></a>Служба размещения рабочих столов

>Относится к: Windows Server (полугодовой канал), Windows Server 2016

В этой статье будет подробнее о рабочем столе, в котором размещены компоненты службы.

## <a name="tenant-environment"></a>Среда клиента

Как описано в разделе [ролей службы удаленного рабочего стола](rds-roles.md), каждая роль играет роль distinct в среды клиента.

Служба размещения рабочих столов поставщика реализован как набор сред, изолированному клиенту. Каждый клиент среда состоит из контейнера хранилища, набор виртуальных машин и сочетания служб Azure, все связи по изолированной виртуальной сети. Каждая виртуальная машина содержит один или несколько компонентов, составляющих размещенной среде рабочего стола клиента. В следующих подразделах описываются компоненты, входящие в состав каждого клиента среде рабочего стола.

## <a name="active-directory-domain-services"></a>Доменные службы Active Directory

Доменных служб Active Directory (AD DS) предоставляет сведения о домене и лесу, таким образом, чтобы пользователи клиента могут входить к рабочим столам и приложениям для выполнения своих рабочих нагрузок. Это также позволяет настроить или подключиться к необходимые общие папки и баз данных, которые могут потребоваться для приложений Windows.

Все отношения доверия с лесом управления поставщика леса клиента не требуется. Учетную запись администратора домена может быть настроен в домене клиента позволяет техническими специалистами поставщика для выполнения административных задач в среде клиента (например, мониторинг состояния системы и установки обновлений программного обеспечения) и для помощи с Устранение неполадок и конфигурации.

Существует несколько способов для развертывания AD DS:

1. Включение доменных служб Azure Active Directory в виртуальной сети среде клиента. Это будет создание управляемого экземпляра AD DS для клиента, на основе пользователей и групп, которые существуют в Azure AD.
2. Настройка изолированного сервера AD DS в виртуальной сети среды клиента. Это обеспечивает получение всех полный контроль над AD DS, работающий на виртуальных машинах.
3. Создайте сеть сеть VPN-подключение к серверу AD DS, расположенный в локальной среде клиента. Это позволяет клиенту соединиться с их существующего экземпляра AD DS и сократить дублирование пользователей, групп, подразделений и т. д.

Дополнительные сведения см. в следующих статьях:

## <a name="sql-database"></a>база данных SQL

Высокой доступности базы данных SQL используется посредника подключений к удаленному рабочему столу, чтобы хранить сведения о развертывании, такие как сопоставление текущих пользователей подключений к серверам узла.

Существует несколько способов развертывания базы данных SQL:

1. Создание базы данных SQL Azure в среде клиента. Это дает функции избыточную базу данных SQL без необходимости управлять серверами, сами. Это также позволяет вам платить только за используемые вместо вложениях в инфраструктуру.
2. Создание кластера SQL Server AlwaysOn. Это позволяет использовать существующую инфраструктуру SQL Server и дает полный контроль над экземплярами SQL Server.

Дополнительные сведения о настройке инфраструктуры высокой доступности базы данных SQL см. в следующих статьях:

## <a name="file-server"></a>Файловый сервер

Файловый сервер использует протокол Server Message Block (SMB) 3.0 для предоставления общих папок. Эти общие папки позволяют создать и сохранить файлы диска профиля пользователя (.vhdx) для резервного копирования данных и позволяют пользователям обмениваться данными друг с другом в пределах клиента облачной службы.

Виртуальной машины, которая развертывает файловый сервер должен иметь диском данных Azure подключена и настроена с общими папками. Диски данных Azure используйте сквозной записи кэширование, гарантируя, что операции записи на диск, не удаляются при перезапуске виртуальной машины.

Маленькие клиенты могут снизить затраты на путем объединения файлового сервера и [роли лицензирования удаленных рабочих Столов](rds-roles.md#remote-desktop-licensing) на одной виртуальной машины в среде клиента.

Дополнительные сведения см. в следующих статьях:

### <a name="user-profile-disks"></a>Диски профилей пользователей

Диски профилей пользователей позволяет пользователю сохранить личные настройки и файлы, если они выполнили вход сеанс на сервере узла сеансов удаленных рабочих Столов в одной коллекции, а затем получить доступ к параметрам и файлы при входе на другой [узла сеансов удаленных рабочих Столов](rds-roles.md#remote-desktop-session-host) сервер в коллекции. При первом входе пользователя в, файловый сервер клиента создает диск профиля пользователя, получает, подключенных к серверу узла сеансов удаленных рабочих Столов, который в данный момент подключен пользователь. Для каждого последующего входа, подключен диск профиля пользователя на соответствующий сервер узла сеансов удаленных рабочих Столов, и он не подключен с каждым выхода. Только пользователь может получить доступ к содержимое диска профиля.
