# <a name="get-started-with-remote-desktop-on-windows"></a>Начало работы с удаленного рабочего стола для Windows

>Относится к: Windows 10, Windows 8.1, Windows Server 2019 г., Windows Server 2016, Windows Server 2012 R2

Клиент удаленного рабочего стола для Windows можно использовать для работы с приложениями Windows и настольных компьютеров удаленно с другого устройства Windows.

Используйте следующие сведения для начала работы. Обязательно ознакомьтесь с [часто задаваемые вопросы о](remote-desktop-client-faq.md) Если у вас возникнут вопросы.

> [!NOTE]
> - Узнать о новых выпусках для клиента Windows? Ознакомьтесь с [новые возможности удаленного рабочего стола в Windows?](windows-whatsnew.md)
> - В любой версии Windows 10 может функционировать клиент.

## <a name="get-the-rd-client-and-start-using-it"></a>Получение клиента к удаленному рабочему Столу и начать с ним

Выполните следующие действия, чтобы начать работу с удаленного рабочего стола на устройстве Windows 10.

1. Скачивание клиента удаленного рабочего стола из [Microsoft Store](https://www.microsoft.com/store/p/microsoft-remote-desktop/9wzdncrfj3ps). 
2. [Настройка компьютера для удаленных соединений](remote-desktop-allow-access.md).
3. Добавьте к удаленному рабочему столу или удаленному ресурсу. Подключение можно использовать для прямого подключения к ПК Windows и удаленным ресурсом, чтобы использовать программы RemoteApp, рабочего стола на основе сеанса или виртуальному рабочему столу, опубликованных вашим администратором. 
4. Закрепление элементов, чтобы быстро приступить к удаленному рабочему столу.

### <a name="add-a-remote-desktop-connection"></a>Добавление подключения к удаленному рабочему столу

Чтобы создать подключение удаленного рабочего стола:

1. В tap Центр подключений **+ добавить**, а затем коснитесь **Desktop**.
2. Введите следующие сведения для компьютера, что вы хотите подключиться к:
   - **Имя компьютера** — имя компьютера. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно добавить к имени компьютера (например, **MyDesktop:3389** или **10.0.0.1:3389**).
   - **Учетная запись пользователя** — учетная запись пользователя для доступа к удаленным ПК. Коснитесь **+** для добавления новой учетной записи, или выберите существующую учетную запись. Для имени пользователя можно использовать следующие форматы: *user_name*, *"Домен\имя_пользователя"* , или <em>user_name@domain.com</em>. Можно также указать, следует ли запрашивать имя пользователя и пароль во время соединения, выбрав **запрашивать каждый раз**.
3. Можно также задать дополнительные параметры, коснувшись **Показать больше**:
   - **Отображаемое имя** — легко запоминаемых име при подключении к ПК. Можно использовать любой строкой, но если понятное имя не указано, отображается имя компьютера.
   - **Группа** — укажите группу, чтобы было проще найти подключения позже. Можно добавить новую группу, коснувшись **+** или выберите его из списка.
   - **Шлюз** — шлюз удаленных рабочих столов, который вы хотите использовать для подключения к виртуальным рабочим столам, программам RemoteApp и рабочим столам на основе сеансов во внутренней корпоративной сети. Получите сведения о шлюзе у системного администратора.
   - **Подключиться к сеансу администрирования** -используйте этот параметр для подключения к сеансу консоли для администрирования сервера Windows.
   - **Кнопок мыши** — используйте этот параметр для замены функции кнопки мыши правой кнопкой мыши. (Это особенно полезно, если удаленный компьютер настроен для левши пользователя, но используется для правши мышь.)
   - **Значение моей разрешение удаленного сеанса:** — выберите разрешение, вы хотите использовать в сеансе. **Выберите для меня** установит разрешение, в зависимости от размера клиента.
   - **Изменить размер экрана:** — при выборе статического высоким разрешением для сеанса, у вас есть возможность сделать элементы на экране отображалось большего размера для повышения удобочитаемости. Примечание. Это применяется только при подключении к Windows 8.1 или более поздней версии.
   - **Обновить разрешение удаленного сеанса при изменении размера** — Если этот параметр включен, клиент будет динамически обновляться сеанса разрешения, в зависимости от размера клиента. Примечание. Это применяется только при подключении к Windows 8.1 или более поздней версии.
   - **Буфер обмена** — Если этот параметр включен, вы получите возможность копировать текст и изображения и с удаленного компьютера.
   - **Воспроизведение аудиозаписей** — выберите устройство, используемый для аудио во время удаленного сеанса. Вы можете для воспроизведения звука на локальных устройствах, удаленном ПК или вообще не.
   - **Запись звука** — Если этот параметр включен, позволяет использовать локального микрофона с приложениями на удаленном ПК.
4. Коснитесь **Сохранить**.

Необходимо изменить эти параметры? Коснитесь меню переполнения ( **...** ) рядом с именем рабочего стола, а затем коснитесь **изменить**.

Удалить подключение? Опять же, коснитесь меню переполнения ( **...** ), а затем коснитесь **удалить**.

### <a name="add-a-remote-resource"></a>Добавить удаленный ресурс
Удаленные ресурсы являются программы RemoteApp, рабочие столы на основе сеансов и виртуальных рабочих столов, опубликованных вашим администратором, с помощью служб удаленных рабочих столов.

Чтобы добавить удаленный ресурс:

1. На экране Центр подключений, нажмите кнопку **+ добавить**, а затем коснитесь **удаленные ресурсы**. 
2. Введите **URL-адрес канала** предоставляемые системному администратору и tap **найти веб-каналы**.
3. Ответ на приглашение введите учетные данные, используемые для подписки на веб-канала.

Удаленные ресурсы будут отображаться в центре подключений.


Чтобы удалить удаленные ресурсы:

1. В центре подключений, выберите в меню переполнения ( **...** ) рядом с удаленного ресурса.
2. Коснитесь **удалить**.

### <a name="pin-a-saved-desktop-to-your-start-menu"></a>ПИН-код сохраненного рабочего стола в меню «Пуск»

Чтобы закрепить подключение к меню «Пуск», коснитесь меню переполнения ( **...** ) рядом с именем рабочего стола, а затем коснитесь **закрепить на начальном экране**.

Теперь можно начать удаленного рабочего стола непосредственно из меню «Пуск», коснувшись его.

## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>Подключение к шлюзу удаленных рабочих Столов для доступа к внутренними ресурсами

Шлюз удаленных рабочих столов (шлюз RD) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять шлюзами с помощью клиента удаленного рабочего стола.

Чтобы настроить новый шлюз:

1. В центре подключений, коснитесь **параметры**.
2. Рядом с каждым шлюзов, коснитесь **+** для добавления нового шлюза. Примечание. Шлюз можно также добавить при добавлении нового соединения.
3. Введите следующие сведения:
   - **Имя сервера** — имя компьютера, который вы хотите использовать в качестве шлюза. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно также добавить к имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
   - **Учетная запись пользователя** — выберите или добавьте учетную запись пользователя для использования шлюза удаленных рабочих столов, вы подключаетесь. Можно также выбрать **использовать учетную запись пользователя рабочего стола** использовать те же учетные данные, которые используются для удаленного рабочего стола.
4. Коснитесь **Сохранить**.  

## <a name="global-app-settings"></a>Параметры глобального приложения

Можно задать следующие глобальные параметры в клиентском приложении можно просмотреть, выбрав **параметры**:

УПРАВЛЯЕМЫЕ ЭЛЕМЕНТЫ
- **Учетная запись пользователя** — позволяет добавить, изменить и удалить учетные записи пользователей, сохраненные в клиенте. Это хороший способ обновлять пароль для учетной записи, после его изменения.
- **Шлюз** — позволяет добавить, изменить и удалить серверы шлюза, сохраненные в клиенте.
- **Группа** — позволяет добавить, изменить и удалить группы, сохраненные в клиенте. Они позволяют легко подключения к группе.

ПАРАМЕТРЫ СЕАНСА
- **Запустить в полноэкранном режиме соединения** — Если этот параметр включен, каждый раз, когда запускается соединение, клиент будет использовать весь экран текущего монитора.
- **Запуск каждого соединения в новом окне** -при включении каждое подключение запускается в отдельном окне, что позволяет помещать их на разных мониторах и переключаться между ними с помощью панели задач.
- **При изменении размера приложения:** -позволяет контролировать что происходит при изменении размера окна клиента. По умолчанию используется **растянуть содержимое, сохраняя пропорции**.
- **Использовать сочетания клавиш с:** -укажите которых команд клавиатуры, например *ВЫИГРАТЬ* или *ALT + TAB* используются. По умолчанию разрешено отправлять их в сеанс, когда соединение находится в во весь экран.
- **Избежать экрана** — вы можете хранить на экране по истечении времени ожидания, если сеанс активен. Это полезно в том случае, если соединение не требует вмешательства для длительных периодов времени.

ПАРАМЕТРЫ ПРИЛОЖЕНИЯ
- **Показать предварительные версии Desktop** -позволяет просмотреть рабочий стол в центре подключений перед подключением к нему. По умолчанию, это имеет значение **на**.
- **Помогите нам улучшить удаленного рабочего стола** -отправляет в корпорацию Майкрософт анонимные данные. Мы используем эти данные для повышения клиент. Дополнительные сведения о том, как мы обращаемся этот анонимный, закрытых данных, см. в разделе [заявление о конфиденциальности Microsoft](https://privacy.microsoft.com/en-us/privacystatement). По умолчанию этому параметру задано **на**.

### <a name="manage-your-user-accounts"></a>Управление учетными записями пользователей

При подключении к рабочего стола или удаленных ресурсов, можно сохранить учетные записи пользователей, чтобы выбрать еще раз. Можно также определить учетные записи пользователей в сам клиент, в отличие от сохранение данных пользователя при подключении к рабочему столу.

Чтобы создать новую учетную запись пользователя:

1. В центре подключений, коснитесь **параметры**.
2. Рядом с учетной записью пользователя, коснитесь **+** для добавления учетной записи пользователя.
3. Введите следующие сведения:
   - **Имя пользователя** -имя пользователя, чтобы сохранить для использования с удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name, "Домен\имя_пользователя", или user_name@domain.com.
   - **Пароль** -пароль для пользователя, который вы указали. Это можно оставить пустым, чтобы быть предложено ввести пароль во время соединения.
4. Коснитесь **Сохранить**.

Чтобы удалить учетную запись пользователя:

1. В центре подключений, коснитесь **параметры**.
2. Выберите учетную запись для удаления из списка под учетной записью пользователя.
3. Рядом с учетной записью пользователя нажмите значок редактирования.
4. Коснитесь **удалить эту учетную запись** внизу, чтобы удалить учетную запись пользователя.
5. Можно также изменить учетную запись пользователя и коснитесь **Сохранить**.

## <a name="navigate-the-remote-desktop-session"></a>Навигация по сеансу удаленного рабочего стола
При запуске удаленного рабочего стола, существуют средства, которое можно использовать для Навигация по сеансу.

### <a name="start-a-remote-desktop-connection"></a>Запустить подключение к удаленному рабочему столу

1. Выберите подключение удаленного рабочего стола для запуска сеанса.
2. Если вы еще не сохранены учетные данные для подключения, вам будет предложено предоставить **Username** и **пароль**.
3. В ответ на запрос для проверки сертификата для удаленного рабочего стола, просмотрите сведения и убедитесь, это ПК, вы доверяете перед утверждением введите **Connect**. Можно также выбрать **больше не спрашивать об этом сертификате** всегда принимать этот сертификат.

### <a name="connection-bar"></a>Панель подключений

Подключение панель обеспечивает доступ к элементам управления дополнительных переходов. По умолчанию в строке подключения помещается в середине в верхней части экрана. Коснитесь и перетащите ползунок влево или вправо, чтобы переместить его.

- **Сдвиг управления** -элемент управления pan позволяет экрана, чтобы быть увеличено и в дальнейшем двигался. Обратите внимание на то, что Панорама управления доступен только на устройствах с поддержкой сенсорного ввода и в режиме прямого касания.
   - Включение и отключение управления pan: Коснитесь значка панорамирования в строке подключения, чтобы отобразить элемент управления сдвиг и масштабирование экрана. Коснитесь значка панорамирования в строке подключения, еще раз, чтобы скрыть элемент управления и вернуться к его исходным разрешением экрана.
   - Использование элемента управления pan - Tap и удерживайте клавишу CTRL pan, а затем перетащите в направлении, чтобы переместить на экране.
   - Перемещение элемента управления pan - двойного касания и удерживайте элемент управления pan, чтобы переместить элемент управления на экране.
- **Дополнительные параметры** -значок Дополнительные параметры, чтобы отобразить выделение сеанса панели и командной строки (см. ниже).
- **Клавиатуры** -значок клавиатуры, чтобы отобразить или скрыть экранную клавиатуру. Элемент управления pan завершения отображается автоматически при отображении клавиатуры.

### <a name="command-bar"></a>Панель команд

Коснитесь **...**  в строке подключения для отображения на панели команд в правой части экрана.

- **Главная** -использовать кнопки домашней страницы для возврата Центр подключений из панели команд.
  - Кроме того можно использовать кнопки «Назад» для одного действия.
  - Ваш сеанс не будет отключен.
  - Это позволяет запускать дополнительные подключения.
- **Отключить** -используйте кнопки "Отключить" для завершения подключения.
  - Ваши приложения будет оставаться активным до тех пор, пока сеанс не завершается на удаленном ПК.
- **Весь экран** - входит или выходит из полноэкранного режима.
- **Touch / мыши** — вы можете переключаться между режимами мыши (прямого касания и указатель мыши).

### <a name="use-direct-touch-gestures-and-mouse-modes-in-a-remote-session"></a>Используйте прямой touch жестов и режимы мыши в удаленном сеансе

Для взаимодействия с сеансом, доступны два режима мыши.
- **Прямой touch**: Передает все сенсорные контакты для сеанса, чтобы интерпретироваться удаленно.
  - Использовать так же, можно использовать Windows с сенсорным экраном.
- **Указатель мыши**: Преобразует локальный сенсорного экрана в больших сенсорной панели, позволяя перемещать указатель мыши в сеансе.
  - Использовать так же, используется Winodws с сенсорной панели.

> [!NOTE]
> Взаимодействие с Windows 8 или более поздней версии собственного сенсорные жесты поддерживаются в режиме прямого касания. 

| Режим мыши    | Операции мышью      | Жест                                                               |
|---------------|----------------------|-----------------------------------------------------------------------|
| Прямое касание  | Щелчок левой кнопкой           | коснитесь пальцем 1                                                          |
| Прямое касание  | Щелчок правой кнопкой мыши          | 1 нажмите пальца и удержание                                                 |
| Указатель мыши | Щелчок левой кнопкой           | коснитесь пальцем 1                                                          |
| Указатель мыши | Щелкните левой кнопкой мыши и перетащите  | 1 палец двойного касания и удержание, затем перетащите                               |
| Указатель мыши | Щелчок правой кнопкой мыши          | коснитесь пальцем 2                                                          |
| Указатель мыши | Щелкните правой кнопкой мыши и перетащите | 2 двойного касания пальцем и удержание, затем перетащите                               |
| Указатель мыши | Колесико мыши          | 2 палец, коснитесь и удерживайте, затем перетащите вверх или вниз                           |
| Указатель мыши | Zoom                 | Используйте 2 пальцев и сжатие для увеличения и уменьшения перемещения пальцы друг от друга, чтобы уменьшить масштаб. |

> [!TIP]
