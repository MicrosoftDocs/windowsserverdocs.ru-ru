# <a name="get-started-with-remote-desktop-on-mac"></a>Начало работы с удаленного рабочего стола на компьютере Mac

>Область применения. Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

Клиент удаленного рабочего стола для Mac можно использовать для работы с приложениями Windows, ресурсы и рабочим столам на компьютере Mac. Используйте следующие сведения, чтобы начать работу, [часто задаваемые вопросы о](remote-desktop-client-faq.md) при наличии вопросов.

>[!Note]
> - Узнать о новых выпусках для клиента macOS? Ознакомьтесь с [новые возможности удаленного рабочего стола на компьютере Mac?](mac-whatsnew.md)
> - Клиент Mac выполняется на компьютерах под управлением macOS 10.10 и более поздних версиях.
> - Сведения в этой статье, относятся в основном для полной версии клиента Mac - версии, доступной в Mac AppStore. Испытайте новые функции, скачав наше приложение предварительной версии: [заметки о выпуске бета-версии клиента](https://go.microsoft.com/fwlink/?LinkID=619698&clcid=0x409).

## <a name="get-the-remote-desktop-client"></a>Получение клиента удаленного рабочего стола
Выполните следующие действия, чтобы приступить к работе с помощью удаленного рабочего стола на компьютере Mac:

1. Скачивание клиента удаленного рабочего стола Майкрософт из [Mac App Store](https://itunes.apple.com/us/app/microsoft-remote-desktop/id1295203466?mt=12).
2. [Настройка компьютера для удаленных соединений](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop). (Можно пропустить этот шаг, вы не сможете подключиться к Компьютеру.)
3. Добавьте к удаленному рабочему столу или удаленному ресурсу. Подключение можно использовать для прямого подключения к ПК Windows и удаленный ресурс, для использования удаленного приложения RemoteApp, рабочего стола на основе сеанса или виртуальному рабочему столу опубликовали в локальной с помощью подключения к удаленным рабочим столам и приложениям RemoteApp. Как правило, эта функция доступна в корпоративных средах.

## <a name="add-a-remote-desktop-connection"></a>Добавление подключения к удаленному рабочему столу
Чтобы создать удаленный рабочий стол:

1. В центре подключений, нажмите кнопку **+**, а затем нажмите кнопку **Desktop**.
2. Введите следующие сведения:
   - **Имя компьютера** -имя компьютера.
      - Это может быть именем компьютера Windows (в **системы** параметры), доменное имя или IP-адресом.
      - Сведения о порте можно также добавить в конец это имя, например *MyDesktop:3389*.
   - **Учетная запись пользователя** -добавить учетную запись для доступа к удаленным ПК.
      - Для компьютеров или локальных учетных записей, присоединенных к Active Directory (AD), используйте один из следующих форматов: *user_name*, *"Домен\имя_пользователя"*, или *user_name@domain.com*.
      - Для Azure Active Directory (AAD), присоединенные к компьютерам, используйте один из следующих форматов: *AzureAD\user_name* или *AzureAD\user_name@domain.com*.
      - Можно также выбрать, следует ли запрашивать пароль.
      - При управлении несколько учетных записей пользователей с тем же именем пользователя, задайте понятное имя, позволяющее отличать учетные записи.
      - Управлять учетными записями пользователя, сохраненного в настройках приложения. 

3. Можно также задать эти необязательные параметры для подключения:
   - Указание понятного имени 
   - Добавление шлюза
   - Задайте звуковые выходные данные
   - Кнопок мыши
   - Включить режим администратора
   - Локальные папки перенаправления в удаленном сеансе
   - Прямой локальных принтерах
   - Прямой смарт-карт
4. Нажмите кнопку **Сохранить**.

Чтобы установить подключение, просто дважды щелкните его. То же самое касается и удаленным ресурсам.

## <a name="add-a-remote-resource"></a>Добавить удаленный ресурс
Удаленные ресурсы являются программы RemoteApp, рабочие столы на основе сеансов и виртуальных рабочих столов, опубликованной с использованием подключения к удаленным рабочим столам и приложениям RemoteApp.

- URL-адрес отображает ссылку на сервер RD Web Access, который предоставляет доступ к рабочим столам и приложениям RemoteApp.
- Настроенное подключение к рабочему столу и приложениям RemoteApp, перечислены.

Чтобы добавить удаленный ресурс:

1. В щелкните центр подключений **+**, а затем нажмите кнопку **добавить удаленные ресурсы**. 
2. Введите сведения для удаленного ресурса:
   - **URL-адрес канала** -URL-адрес сервера веб-доступа к удаленным рабочим Столам. Можно также ввести учетную запись корпоративной электронной почты в этом поле — при этом клиент для поиска сервера веб-доступа к удаленному рабочему Столу, связанные с вашим адресом электронной почты.
   - **Имя пользователя** -имя пользователя для использования сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
   - **Пароль** -пароль, используемый для сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
3. Нажмите кнопку **Сохранить**.


Удаленные ресурсы будут отображаться в центре подключений.


## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>Подключение к шлюзу удаленных рабочих Столов для доступа к внутренними ресурсами

Шлюз удаленных рабочих столов (шлюз RD) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять шлюзами в настройках приложения или при настройке нового подключения к рабочему столу.

Чтобы настроить новый шлюз в настройках:

1. В центре подключений, нажмите кнопку **предпочтения > шлюзы**. 
2. Нажмите кнопку **+** кнопки в нижней части таблицы введите следующие сведения:
  - **Имя сервера** — имя компьютера, который вы хотите использовать в качестве шлюза. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно также добавить к имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
  - **Имя пользователя** -имя пользователя и пароль, используемый для шлюза удаленного рабочего стола, необходимо соединиться. Можно также выбрать **использовать учетные данные подключения** для использования тем же именем пользователя и пароль, которые используются для удаленного рабочего стола.


## <a name="manage-your-user-accounts"></a>Управление учетными записями пользователей

При подключении к рабочего стола или удаленных ресурсов, можно сохранить учетные записи пользователей, чтобы выбрать еще раз. Учетные записи пользователей можно управлять с помощью клиента удаленного рабочего стола.

Чтобы создать новую учетную запись пользователя:

1. В центре подключений, нажмите кнопку **параметры** > **учетные записи**.
2. Нажмите кнопку **добавьте учетную запись пользователя**.
3. Введите следующие сведения:
   - **Имя пользователя** -имя пользователя, чтобы сохранить для использования с удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name, "Домен\имя_пользователя", или user_name@domain.com.
   - **Пароль** -пароль для пользователя, который вы указали. Каждой учетной записи пользователя, который вы хотите сохранить для использования для удаленных соединений должен иметь пароль, связанный с ним.
   - **Понятное имя** — Если вы используете той же учетной записью с различных паролей, задайте понятное имя, чтобы различать эти учетные записи пользователей.
4. Коснитесь **Сохранить**, а затем коснитесь **параметры**.

## <a name="customize-your-display-resolution"></a>Настроить разрешение экрана
Разрешение экрана можно указать для сеанса удаленного рабочего стола.

1. В центре подключений, нажмите кнопку **предпочтения**.
2. Нажмите кнопку **разрешение**. 
3. Нажмите кнопку **+**.
4. Введите разрешение высотой и шириной и нажмите кнопку **ОК.**

Чтобы удалить разрешение, выберите его и нажмите кнопку **-**.

**Отображает быть пробелов в отдельном** Если вы используете Mac OS X 10.9 и отключить **отображает быть пробелов в отдельном** в Mavericks (**системные настройки > пакет управления**), необходимо настроить Этот параметр в клиент удаленного рабочего стола, используя тот же параметр.

### <a name="drive-redirection-for-remote-resources"></a>Перенаправление дисков для удаленных ресурсов
Перенаправление дисков поддерживается для удаленных ресурсов, чтобы можно было сохранить файлов, созданных с помощью удаленного приложения локально на компьютере Mac. Перенаправленной папки всегда является вашим домашним каталогом, отображаются как сетевой диск в удаленном сеансе.

> [!NOTE]
> Чтобы использовать эту функцию, администратор должен задать соответствующие параметры на сервере.


## <a name="use-a-keyboard-in-a-remote-session"></a>Использование клавиатуры в удаленном сеансе

Раскладки клавиатуры Mac отличаются от раскладок клавиатуры Windows. 

- Управляющая клавиша на клавиатуре Mac разделу Windows.
- Для выполнения действий, использующих кнопки на компьютере Mac, необходимо использовать элемент управления кнопки в Windows (например: Copy = Ctrl + C).
- Функциональные клавиши могут активироваться в сеансе, кроме того нажав клавишу FN (например: FN + F1).
- Клавиша Alt справа от пробела на клавиатуре Mac равно Alt Gr или правая клавиша Alt в Windows.

По умолчанию удаленный сеанс будет использовать один и тот же язык клавиатуры как вы используете клиент на платформе операционной системы. (Если компьютер Mac под управлением какой-либо разделы en-us операционной системы, который будет использоваться для удаленных сеансов, а также. Если используется не тот язык клавиатуры операционной системы, проверьте клавиатуры задание на удаленном ПК и изменив параметр вручную. См. в разделе [вопросы и ответы клиента удаленного рабочего стола](remote-desktop-client-faq.md) Дополнительные сведения о языковых стандартов и клавиатуры.
