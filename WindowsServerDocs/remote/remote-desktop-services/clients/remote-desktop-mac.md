# Начало работы с удаленного рабочего стола на компьютере Mac

>Область применения: Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

Можно использовать клиент удаленного рабочего стола для Mac в работе приложения Windows, ресурсы и рабочие столы на компьютере Mac. Начало работы - и изучите [вопросы и ответы](remote-desktop-client-faq.md) , если у вас возникли вопросы, используйте следующие сведения.

>[!Note]
> - Хотите узнать, новые выпуски для клиент macOS? Ознакомьтесь с [новые возможности для удаленного рабочего стола на компьютере Mac?](mac-whatsnew.md)
> - Клиент Mac выполняется на компьютерах под управлением macOS 10.10 и более поздних версиях.
> - Сведения в этой статье применяется в первую очередь до полной версии клиента Mac - версия, доступная в Mac AppStore. Оцените новые возможности, скачайте наше приложение предварительного просмотра: [заметки о выпуске бета-версии клиента](https://go.microsoft.com/fwlink/?LinkID=619698&clcid=0x409).

## Получение клиентом удаленного рабочего стола
Выполните следующие действия, чтобы начать работу с помощью удаленного рабочего стола на Mac.

1. Скачайте клиент удаленного рабочего стола Майкрософт в [Хранилище приложения Mac](https://itunes.apple.com/us/app/microsoft-remote-desktop/id1295203466?mt=12).
2. [Настройка компьютера для удаленных подключений](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop). (Если вы пропустите этот шаг, вам не удается подключиться к Компьютеру.)
3. Добавление подключения удаленного рабочего стола или удаленных ресурсов. Использовать подключение к подключаться напрямую к ПК с Windows и удаленный ресурс для использования программы RemoteApp, на основе сеанса рабочего стола или виртуального рабочего стола опубликованы в локальной с помощью RemoteApp и подключений к рабочему столу. Как правило, эта функция доступна в корпоративных средах.

## Добавьте к удаленному рабочему столу
Создание удаленного рабочего стола:

1. В центре подключения нажмите кнопку **+** и нажмите кнопку **рабочего стола**.
2. Введите следующие сведения:
   - **Имя компьютера** - имя компьютера.
      - Это может быть имя компьютера Windows (находится в параметрах **системы** ), доменное имя или IP-адреса.
      - Вы также можете добавить сведения о портах в конце этого имени, например *MyDesktop:3389*.
   - **Учетная запись пользователя** - добавить учетную запись пользователя, можно использовать для доступа к удаленному Компьютеру.
      - Для компьютеров или локальные учетные записи, присоединенными к Active Directory (AD), используйте один из следующих форматов: *user_name*, *домен\имя_пользователя*или *user_name@domain.com*.
      - Для компьютеров, присоединенными к Azure Active Directory (AAD), используйте один из следующих форматов: *AzureAD\user_name* или *AzureAD\ user_name@domain.com*.
      - Можно также выбрать, следует ли требовать пароль.
      - При управлении несколько учетных записей пользователей с тем же именем пользователя, указать понятное имя для учетных записей.
      - Управление сохраненных учетных записей пользователей в параметрах приложения. 

3. Вы также можете задать следующие необязательные параметры для подключения:
   - Указать понятное имя 
   - Добавление шлюза
   - Задайте вывода звука
   - Кнопок мыши
   - Включите режим администратора
   - Перенаправление локальным папкам в удаленном сеансе
   - Вперед локальных принтеров
   - Вперед смарт-карт
4. Нажмите кнопку **Сохранить**.

Чтобы начать подключение, просто дважды щелкните его. То же справедливо для удаленных ресурсов.

## Добавление удаленных ресурса
Удаленные ресурсы являются программ RemoteApp, на основе сеанса настольных компьютеров и виртуальных рабочих столов публикации с помощью RemoteApp и подключений к рабочему столу.

- URL-адрес отображает ссылку на веб-доступа к удаленному рабочему Столу сервера, который предоставляет доступ к рабочему столу и RemoteApp.
- Перечислены настроенное RemoteApp и подключений к рабочему столу.

Добавление удаленных ресурсов:

1. В центре подключения, выберите **+**, а затем нажмите кнопку **Добавить удаленным ресурсам**. 
2. Введите сведения для удаленного ресурса.
   - **URL-адрес веб-каналов** - URL-адрес сервера веб-доступа к удаленному рабочему Столу. Можно также ввести учетной записи корпоративной электронной почте, в этом поле — это клиенту о том, для поиска веб-сервера к удаленному рабочему Столу, связанные с свой адрес электронной почты.
   - **Имя пользователя** - имя пользователя для веб-доступа к удаленному рабочему Столу сервер, которым вы подключаетесь.
   - **Пароль** - пароль, используемый для веб-доступа к удаленному рабочему Столу сервер, которым вы подключаетесь.
3. Нажмите кнопку **Сохранить**.


Удаленные ресурсы отображаются в центре подключения.


## Подключение для доступа к ресурсам внутренней шлюза удаленных рабочих Столов

Шлюз удаленных рабочих столов (шлюз удаленных рабочих Столов) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять вашей шлюзов в параметрах приложения или при настройке нового подключения рабочего стола.

Настройка нового шлюза в установках:

1. В центре подключения нажмите кнопку **предпочтения > шлюзов**. 
2. Нажмите кнопку **+** кнопка в нижней части таблицы введите следующие сведения:
  - **Имя сервера** — имя компьютера, который необходимо использовать в качестве шлюза. Это может быть Windows имя компьютера, имя домена в Интернете или IP-адреса. Вы также можете добавить сведения о портах для имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
  - **Имя пользователя** - имя пользователя и пароль для шлюзов удаленного рабочего стола, которые вы подключаетесь. Также можно выбрать **использовать учетные данные для подключения** , чтобы использовать одинаковые имя пользователя и пароль, которые используются для подключения удаленного рабочего стола.


## Управление учетными записями

При подключении к рабочего стола или удаленным ресурсам, вы можете сохранить учетные записи пользователей для выбора из еще раз. Учетные записи пользователей можно управлять с помощью клиентом удаленного рабочего стола.

Для создания новой учетной записи:

1. В центре подключения, выберите **Параметры** > **учетные записи**.
2. Нажмите кнопку **Добавить учетную запись пользователя**.
3. Введите следующие сведения:
   - **Имя пользователя** - имя пользователя для сохранения для использования с помощью удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name домен\имя_пользователя, или user_name@domain.com.
   - **Пароль** - пароль для пользователя, указанные вами. Каждой учетной записи пользователя, которое вы хотите сохранить для удаленных подключений требуется пароль, связанный с ним.
   - **Понятное имя** - при использовании одной учетной записи пользователя с помощью различных паролей, задайте понятное имя для разделения этих учетных записей пользователей.
4. Нажмите **Сохранить**, а затем коснитесь **параметров**.

## Настройка разрешения экрана
Вы можете задать разрешение экрана для сеансов удаленных рабочих столов.

1. В центре подключения нажмите кнопку " **Параметры**".
2. Выберите **разрешение**. 
3. Нажмите кнопку **+**.
4. Введите разрешение высота и ширина и нажмите кнопку **OK.**

Чтобы удалить разрешения, выберите его и нажмите кнопку **-**.

**Отображает имеют отдельные пробелы** Если работают под управлением Mac OS X 10,9 и отключить **Отображение имеют отдельные пробелы** в Mavericks (**настройки системы > управления критически**), необходимо настроить этот параметр в клиента удаленного рабочего стола, используя тот же параметр.

### Перенаправление дисков для удаленных ресурсов
Перенаправление дисков поддерживается удаленным ресурсам, таким образом, вы можете сохранить файлы, созданные с помощью удаленного приложения локально на вашем Mac. Перенаправленные папки всегда является к домашнему каталогу, отображается как сетевой диск в удаленном сеансе.

> [!NOTE]
> Чтобы использовать эту функцию, администратор должен настроить соответствующие параметры на сервере.


## Использование клавиатуры в удаленном сеансе

Раскладки клавиатуры Mac отличаются от раскладки клавиатуры Windows. 

- Клавиша для команды на клавиатуре Mac равно клавиши Windows.
- Для выполнения действий, использующих кнопки на Mac, необходимо использовать кнопки управления в Windows (например: Копировать Ctrl + C).
- Функциональные клавиши можно активировать сеанса, кроме того нажав клавишу FN (например: FN + F1).
- Клавиша Alt справа от пробела на клавиатуре Mac равно клавиши Alt Alt Gr вправо в Windows.

По умолчанию удаленный сеанс будет использовать тот же самый язык клавиатуры как вы используете клиент на ОС. (Если Mac работает под управлением en-us операционной системы, который будет использоваться для удаленных сеансов, а также. Если языковой стандарт операционной системы клавиатуры не используется, проверьте клавиатуры на удаленном Компьютере и изменив параметр вручную. См.
