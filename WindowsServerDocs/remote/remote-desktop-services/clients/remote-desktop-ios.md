# Начало работы с удаленного рабочего стола в iOS

>Область применения: Windows 10, Windows 8.1, Windows Server 2019 г., Windows Server 2016, Windows Server 2012 R2

Можно использовать клиент удаленного рабочего стола для iOS в работе приложения Windows, ресурсы и настольными компьютерами с вашего устройства iOS (iPhone и iPad).

Используйте следующие сведения для начала работы. Обязательно изучите [вопросы и ответы](remote-desktop-client-faq.md) , если у вас возникают вопросы.

> [!NOTE]
> - Хотите узнать, новые выпуски для клиента iOS? Ознакомьтесь с [новые возможности для удаленного рабочего стола в iOS?](ios-whatsnew.md)
> - Клиент iOS поддерживает устройств под управлением iOS 6.x и более поздних версиях.

## Получение клиентом удаленного рабочего стола и начать использовать это решение

### Скачайте клиент удаленного рабочего стола из хранилища iOS
Выполните следующие действия, чтобы начать работу с помощью удаленного рабочего стола на устройстве iOS.

1. Для загрузки клиента удаленного рабочего стола Майкрософт [iTunes](https://itunes.apple.com/us/app/microsoft-remote-desktop/id714464092?mt=8).
2. [Настройка компьютера на прием удаленных подключений](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop).
3. Добавьте [подключения удаленного рабочего стола](#add-a-remote-desktop-connection) или [удаленных ресурсов](#add-a-remote-resource). Использовать подключение к непосредственно к ПК с Windows и удаленного ресурса с помощью программы RemoteApp, доставлять сеанса или виртуального рабочего стола публикации локального с помощью подключения к удаленным рабочим столам и RemoteApp. Как правило, эта функция доступна в корпоративных средах.

### Скачайте клиент бета-версии iOS удаленного рабочего стола
На устройстве iOS следуйте [этим инструкциям](https://aka.ms/rdiosbeta) для скачивания клиентом удаленного рабочего стола iOS бета-версии.

### Добавьте к удаленному рабочему столу

Создание удаленного рабочего стола: 
1. В центре подключения tap **+** и нажмите кнопку **Добавить Компьютере или сервере**.
2. Введите следующие сведения для удаленного рабочего стола подключения:
  - **Имя компьютера** — имя компьютера. Это может быть имя компьютера Windows, имя домена в Интернете или IP-адреса. Сведения о портах также можно добавить к имени компьютера (например, **MyDesktop:3389** или **10.0.0.1:3389**).
  - **Имя пользователя** — имя пользователя, чтобы использовать для доступа к удаленному Компьютеру. Можно использовать следующие форматы: *user_name*, *домен\имя_пользователя*или *user_name@domain.com*. Можно также задать запрашивает имя пользователя и пароль.
3. Вы также можете задать следующие дополнительные параметры:
  - **Понятное имя (необязательно)** — это удобного для запоминания имени на компьютере, вы подключаетесь. Можно использовать любую строку, но если не указать понятное имя, имя компьютера отображается.
  - **Шлюз (необязательно)** — шлюза удаленного рабочего стола, который вы хотите использовать для подключения к виртуальным рабочим столам программ RemoteApp и рабочие столы на основе сеансов во внутренней корпоративной сети. Получение сведений о шлюз из к системному администратору.
  - **Звук** : выберите устройства для использования для воспроизведения звука во время удаленного сеанса. Вы можете воспроизводить звук на локальном удаленного устройства, или не во всех.
  - **Кнопок мыши** — всякий раз, когда жест мыши будет отправлять команды с левой кнопки мыши, отправляет той же команды с правой кнопкой мыши вместо. Это необходимо, если удаленный компьютер настроен для режима левосторонней мыши.
  - **Режим администратора** - подключения к сеансу администрирования на сервере под управлением Windows Server 2003 или более поздней версии.
4. Коснитесь кнопки **Сохранить**.

Необходимо изменить эти параметры? Нажмите и удерживайте рабочем столе, который требуется изменить, а затем нажмите значок «параметры». 

### Добавление удаленных ресурса
Удаленные ресурсы являются программ RemoteApp, настольные компьютеры на основе сеансов и публикации с помощью подключения к удаленным рабочим столам и RemoteApp виртуальных рабочих столов.

- URL-адрес отображает ссылку на сервер веб-доступа к удаленному рабочему Столу, который предоставляет доступ к удаленным рабочим столам и RemoteApp.
- Указаны настроенное RemoteApp и удаленным рабочим столам.

Добавление удаленных ресурса:

1. На экране Center подключения нажмите **+**, а затем коснитесь **Добавление удаленными ресурсами**. 
2. Введите сведения для удаленного ресурса.
   - **URL-адрес веб-каналов** - URL-адрес сервера веб-доступа к удаленному рабочему Столу. Учетной записи корпоративной электронной почте можно также ввести в это поле — это сообщает клиенту для поиска веб-сервера к удаленному рабочему Столу, связанные с свой адрес электронной почты.
   - **Имя пользователя** - имя пользователя для веб-доступа к удаленному рабочему Столу сервер, которым вы подключаетесь.
   - **Пароль** - пароль, используемый для веб-доступа к удаленному рабочему Столу сервер, которым вы подключаетесь.
3. Коснитесь кнопки **Сохранить**.

Удаленными ресурсами отображается в центре подключения.


## Подключение для доступа к ресурсам внутренней шлюза удаленных рабочих Столов

Шлюз удаленных рабочих столов (шлюз удаленных рабочих Столов) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять вашей шлюзов с помощью клиента удаленного рабочего стола.

Настройка нового шлюза:

1. В центре подключения нажмите **Параметры > шлюзов**. 
2. Коснитесь **шлюза добавить удаленного рабочего стола**.
3. Введите следующие сведения:
  - **Имя сервера** — имя компьютера, который необходимо использовать в качестве шлюза. Это может быть имя компьютера Windows, имя домена в Интернете или IP-адреса. Вы также можете добавить сведения о портах для имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
  - **Имя пользователя** - имя пользователя и пароль для шлюзов удаленных рабочих столов, которые вы подключаетесь. Также можно выбрать **использовать учетные данные для подключения** , чтобы использовать одинаковые имена пользователя и пароль, которые используются для подключения удаленного рабочего стола.


## Управление учетными записями 

При подключении к рабочего стола или удаленным ресурсам, вы можете сохранить учетные записи пользователей для выбора из еще раз. Вы можете управлять учетные записи пользователей с помощью клиента удаленного рабочего стола.

Создание новой учетной записи пользователя:

1. В центре подключения нажмите кнопку **Параметры**и выберите **Имена пользователей**.
2. Коснитесь, **Добавить учетную запись пользователя**.
3. Введите следующие сведения:
   - **Имя пользователя** - имя пользователя для сохранения для использования с удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name, домен\имя_пользователя, или user_name@domain.com.
   - **Пароль** — пароль пользователя, указанные вами. Каждой учетной записи пользователя, который вы хотите сохранить для удаленных подключений необходимо иметь пароль, связанный с ним.
4. Нажмите **Сохранить**, а затем коснитесь **Параметры**.
5. Коснитесь **сделать** , чтобы сохранить новую конфигурацию.

Чтобы удалить учетную запись пользователя:

1. В центре подключения нажмите **Параметры > имена пользователей**.
2. Прокрутите строки справа налево, выберите пользователя.
3. Коснитесь кнопки **Удалить**.



## Перейдите сеансов удаленных рабочих столов
При запуске сеансов удаленных рабочих столов, существует средств можно использовать для перехода к сеансу.

### Запуск удаленного рабочего стола

1. Коснитесь удаленный рабочий стол для запуска сеансов удаленных рабочих столов. 
2. Если появится запрос для проверки сертификата для удаленного рабочего стола, коснитесь **принять**. Вы можете всегда принимать переведите переключатель **не спрашивать снова для подключения к этому компьютеру** для **д**. 

### Панель подключения

Подключение панель обеспечивает доступ к элементам управления дополнительных навигации. 

- **Сдвиг управления**: элемент управления сдвиг позволяет экрана, чтобы увеличить и перемещены. Обратите внимание, что сдвиг элемент управления доступен только с помощью прямой сенсорного ввода.
   - Включить / отключить сдвиг управления: нажмите значок сдвиг в строке подключения для отображения элемента управления сдвигать и масштабировать экрана. Коснитесь значка сдвиг в строке подключения еще раз, чтобы скрыть элемент управления и вернуть его исходное разрешение экрана.
   - Используйте элемент управления сдвиг: касание и удерживание сдвига элементов управления, а затем перетащите в направлении, нужно переместить на экране.
   - Перемещение элемента управления сдвиг: двойное касание и удерживание элемента управления сдвиг для перемещения элемента управления на экране.
- **Имя подключения**: отображается имя текущего подключения. Коснитесь имени подключения для отображения панели выбора сеанса.
- **Клавиатура**: нажмите значок клавиатуры, чтобы отобразить или скрыть клавиатуры. Элемент управления сдвиг автоматически отображается при отображении клавиатуры.
- **Переместить в строке подключения**: касание и удерживание в строке подключения и затем перетаскивания в новое место в верхней части экрана.

### Выбор сеанса
Может иметь несколько открытых на разных компьютерах в то же время подключений. Коснитесь в строке подключения для отображения панели выбора сеанс в левой части экрана. Панель выбора сеанса позволяет просматривать откройте подключений и переключаться между ними. 

- Переключение между приложениями во время сеанса открытые удаленного ресурса.

    При подключении к удаленным ресурсам, вы можете переключаться между открытые приложения в течение этого сеанса, коснитесь меню разворачивания и выбрав команду из списка доступных элементов.
- Меню "Пуск" новый сеанс

  Вы можете начать новых приложений или сеансов рабочего стола с внутри текущего подключения: выберите **Начать новый**, а затем выберите из списка доступных элементов.

- Отключение сеанса

  Чтобы отключить сеанс касание X в левой части плитки, сеанс.

### Панель команд

Панель команд заменен служебную программу панели начиная с версии 8.0.1. Можно переключаться между режимами мыши и вернуться в центр подключения из командной строки.

## Используйте сенсорные жесты и режима мыши в удаленном сеансе

Клиент использует стандартные сенсорные жесты. Сенсорные жесты можно также использовать для выполнения действий с мышью на удаленный рабочий стол. Режимы мыши определяются в следующей таблице.

> [!NOTE]
> Взаимодействия с Windows 8 или более поздней версии собственные сенсорные жесты поддерживаются в режиме прямого сенсорного ввода. Дополнительные сведения о Windows 8 жесты см. в разделе [Touch: прокрутки, касание и более](https://windows.microsoft.com/en-US/windows-8/touch-swipe-tap-beyond).

| Режим мыши    | Операции мыши      | Жест                                                    |
|---------------|----------------------|------------------------------------------------------------|
| Прямой сенсорного ввода  | Щелкайте левой кнопкой мыши           | касание пальцем 1                                               |
| Прямой сенсорного ввода  | Щелчок правой кнопкой мыши          | 1 палец касание и удерживание                                      |
| Указатель мыши | Щелкайте левой кнопкой мыши           | касание пальцем 1                                               |
| Указатель мыши | Щелкните левой кнопкой мыши и перетащите  | 1 палец двойное касание и удерживание, затем перетащите                    |
| Указатель мыши | Щелчок правой кнопкой мыши          | касание пальцем 2                                               |
| Указатель мыши | Щелкните правой кнопкой мыши и перетащите | 2 палец двойное касание и удерживание, затем перетащите                    |
| Указатель мыши | Колесика мыши          | 2 палец нажмите и удерживайте, а затем перетащите вверх или вниз                |
| Указатель мыши | Масштаб                 | Сжатие 2 пальцы для увеличения и уменьшения распределены 2 пальцев для уменьшения масштаба |

## При помощи клавиатуры в удаленном сеансе

Вы можете использовать экранную клавиатуру или физической клавиатуры в удаленном сеансе.

Для экранной клавиатуры, с помощью кнопки по правому краю панели над клавиатурой переключаться между клавиатуры стандартных и дополнительных.

Если Bluetooth включен для вашего устройства iOS, клиент автоматически обнаруживает клавиатуры с поддержкой Bluetooth.

Имейте в виду, что, из-за ограничений на ОС, специальные клавиши, например Ctrl, параметр и функции не будет работать правильно с помощью клавиатуры Bluetooth. Работают следующие разделы:

- Буквенно-цифровых клавиш
- Клавиши управления курсором
- Вкладка: Вкладка работает, но клавиши Shift + Tab не работает
- Домашняя / Pos1: Alt + Стрелка влево = Домашняя страница
- Конец: Alt + Стрелка вправо = End
- Page Up: Alt + Стрелка вверх = Page Up
- Page Down: Alt + Стрелка вниз = Page Down
- Выберите все: Команды + A = Ctrl + A (все содержимое в большинстве программ)
- "Вырезать": Команда + X = Ctrl + X ("Вырезать" в большинстве программ)
- Copy: Команда + C = сочетание клавиш Ctrl + C (Копировать в большинстве программ)
- Вставка: Команды + V = сочетание клавиш Ctrl + V (вставить в большинстве программ)
- Символы: Alt + алфавитно-цифровых клавиш создаст различные символы в зависимости от языка настроены
