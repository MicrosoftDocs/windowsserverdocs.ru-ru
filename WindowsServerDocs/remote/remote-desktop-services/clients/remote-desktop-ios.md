# <a name="get-started-with-remote-desktop-on-ios"></a>Начало работы с удаленного рабочего стола на устройстве iOS

>Область применения. Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

Клиент удаленного рабочего стола для iOS можно использовать для работы с приложениями Windows, ресурсы и рабочие столы, с устройства iOS (iPhone и iPad).

Используйте следующие сведения для начала работы. Обязательно ознакомьтесь с [часто задаваемые вопросы о](remote-desktop-client-faq.md) Если у вас возникнут вопросы.

> [!NOTE]
> - Узнать о новых выпусках для клиента iOS? Ознакомьтесь с [новые возможности удаленного рабочего стола на устройстве iOS?](ios-whatsnew.md)
> - Клиент iOS поддерживает устройства под управлением iOS 6.x и более поздних версиях.

## <a name="get-the-remote-desktop-client-and-start-using-it"></a>Получение клиента удаленного рабочего стола и начать с ним

### <a name="download-the-remote-desktop-client-from-the-ios-store"></a>Скачивание клиента удаленного рабочего стола из магазина iOS
Выполните следующие действия, чтобы начать работу с удаленного рабочего стола на устройстве iOS.

1. Скачивание клиента удаленного рабочего стола Майкрософт из [iTunes](https://itunes.apple.com/us/app/microsoft-remote-desktop/id714464092?mt=8).
2. [Настройка компьютера для удаленных соединений](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop).
3. Добавить [подключения удаленного рабочего стола](#add-a-remote-desktop-connection) или [удаленный ресурс](#add-a-remote-resource). Подключение можно использовать для подключения к непосредственно на ПК Windows и удаленный ресурс с помощью программы RemoteApp, рабочего стола на основе сеанса или виртуальному рабочему столу, опубликованной в локальной с помощью подключения к удаленным рабочим столам и приложениям RemoteApp. Как правило, эта функция доступна в корпоративных средах.

### <a name="download-the-remote-desktop-ios-beta-client"></a>Скачайте бета-версии клиента удаленного рабочего стола iOS
На устройстве iOS, выполните [эти инструкции](https://aka.ms/rdiosbeta) для скачивания клиента удаленного рабочего стола iOS бета-версии.

### <a name="add-a-remote-desktop-connection"></a>Добавление подключения к удаленному рабочему столу

Чтобы создать удаленный рабочий стол: 
1. В tap Центр подключений **+**, а затем нажмите **добавить ПК или сервер**.
2. Введите следующие сведения для удаленного рабочего стола:
  - **Имя компьютера** — имя компьютера. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно добавить к имени компьютера (например, **MyDesktop:3389** или **10.0.0.1:3389**).
  - **Имя пользователя** — имя пользователя для доступа к удаленным ПК. Можно использовать следующие форматы: *user_name*, *"Домен\имя_пользователя"*, или *user_name@domain.com*. Можно также указать, следует ли запрашивать имя пользователя и пароль.
3. Также можно задать следующие дополнительные параметры:
  - **Понятное имя (необязательно)** — легко запоминаемых име при подключении к ПК. Можно использовать любой строкой, но если понятное имя не указано, отображается имя компьютера.
  - **Шлюз (необязательно)** — шлюз удаленных рабочих столов, который вы хотите использовать для подключения к виртуальным рабочим столам, программам RemoteApp и рабочим столам на основе сеансов во внутренней корпоративной сети. Получите сведения о шлюзе у системного администратора.
  - **Звук** — выберите устройство, используемый для аудио во время удаленного сеанса. Вы можете для воспроизведения звука на локальных устройствах, удаленное устройство, или вообще не.
  - **Кнопок мыши** — всякий раз, когда жест мыши следует отправить команду с левой кнопки мыши, он отправляет ту же команду с помощью правой кнопки мыши. Это необходимо, если удаленный компьютер настроен для режима для левши мыши.
  - **Режим администратора** -подключиться к сеансу администрирования на сервере под управлением Windows Server 2003 или более поздней версии.
4. Коснитесь **Сохранить**.

Необходимо изменить эти параметры? Нажмите и удерживайте рабочего стола, который вы хотите изменить, а затем нажмите значок "Параметры". 

### <a name="add-a-remote-resource"></a>Добавить удаленный ресурс
Удаленные ресурсы являются программы RemoteApp, рабочие столы на основе сеансов и виртуальных рабочих столов, опубликованной с использованием подключения к удаленным рабочим столам и приложениям RemoteApp.

- URL-адрес отображает ссылку на сервер RD Web Access, который предоставляет доступ к рабочим столам и приложениям RemoteApp.
- Настроенное подключение к рабочему столу и приложениям RemoteApp, перечислены.

Чтобы добавить удаленный ресурс:

1. На экране Центр подключений, нажмите кнопку **+**, а затем коснитесь **добавить удаленные ресурсы**. 
2. Введите сведения для удаленного ресурса:
   - **URL-адрес канала** -URL-адрес сервера веб-доступа к удаленным рабочим Столам. Можно также ввести учетную запись корпоративной электронной почты в этом поле — при этом клиент для поиска сервера веб-доступа к удаленному рабочему Столу, связанные с вашим адресом электронной почты.
   - **Имя пользователя** -имя пользователя для использования сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
   - **Пароль** -пароль, используемый для сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
3. Коснитесь **Сохранить**.

Удаленные ресурсы будут отображаться в центре подключений.


## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>Подключение к шлюзу удаленных рабочих Столов для доступа к внутренними ресурсами

Шлюз удаленных рабочих столов (шлюз RD) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять шлюзами с помощью клиента удаленного рабочего стола.

Чтобы настроить новый шлюз:

1. В центре подключений, коснитесь **параметры > шлюзы**. 
2. Коснитесь **шлюз удаленных рабочих столов на добавление**.
3. Введите следующие сведения:
  - **Имя сервера** — имя компьютера, который вы хотите использовать в качестве шлюза. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно также добавить к имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
  - **Имя пользователя** -имя пользователя и пароль, используемый для шлюза удаленного рабочего стола, необходимо соединиться. Можно также выбрать **использовать учетные данные подключения** для использования тем же именем пользователя и пароль, которые используются для удаленного рабочего стола.


## <a name="manage-your-user-accounts"></a>Управление учетными записями пользователей 

При подключении к рабочего стола или удаленных ресурсов, можно сохранить учетные записи пользователей, чтобы выбрать еще раз. Учетные записи пользователей можно управлять с помощью клиента удаленного рабочего стола.

Чтобы создать новую учетную запись пользователя:

1. В центре подключений, коснитесь **параметры**, а затем коснитесь **имена пользователей**.
2. Коснитесь **добавьте учетную запись пользователя**.
3. Введите следующие сведения:
   - **Имя пользователя** -имя пользователя, чтобы сохранить для использования с удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name, "Домен\имя_пользователя", или user_name@domain.com.
   - **Пароль** -пароль для пользователя, который вы указали. Каждой учетной записи пользователя, который вы хотите сохранить для использования для удаленных соединений должен иметь пароль, связанный с ним.
4. Коснитесь **Сохранить**, а затем коснитесь **параметры**.
5. Коснитесь **сделать** чтобы сохранить новую конфигурацию.

Чтобы удалить учетную запись пользователя:

1. В центре подключений, коснитесь **параметры > имена пользователей**.
2. Проведите по строке справа налево, выберите пользователя.
3. Коснитесь **удалить**.



## <a name="navigate-the-remote-desktop-session"></a>Навигация по сеансу удаленного рабочего стола
При запуске сеанса удаленного рабочего стола, существуют средства, которое можно использовать для Навигация по сеансу.

### <a name="start-a-remote-desktop-connection"></a>Запуск удаленного рабочего стола

1. Коснитесь удаленного рабочего стола для запуска сеанса удаленного рабочего стола. 
2. Если будет предложено проверить сертификат для удаленного рабочего стола, коснитесь **Accept**. Вы можете всегда принимать, перемещая **больше не спрашивать для подключения к этому компьютеру** переключатель **ON**. 

### <a name="connection-bar"></a>Панель подключений

Подключение панель обеспечивает доступ к элементам управления дополнительных переходов. 

- **Сдвиг управления**: Элемент управления pan позволяет экрана, чтобы быть увеличено и в дальнейшем двигался. Обратите внимание на то, что Панорама элемент управления доступен только с помощью прямого касания.
   - Включение и отключение управления pan: Коснитесь значка панорамирования в строке подключения, чтобы отобразить элемент управления сдвиг и масштабирование экрана. Коснитесь значка панорамирования в строке подключения, еще раз, чтобы скрыть элемент управления и вернуться к его исходным разрешением экрана.
   - Использование элемента управления pan: Нажмите и удерживайте клавишу CTRL pan, а затем перетащите в направлении, чтобы переместить на экране.
   - Перемещение элемента управления pan: Double коснитесь и удерживайте pan, чтобы переместить элемент управления на экране.
- **Имя подключения**: Отображается имя текущего соединения. Выберите имя подключения для отображения панели выбора сеанса.
- **Клавиатуры**: Нажмите значок клавиатуры, чтобы отобразить или скрыть клавиатуры. Элемент управления pan завершения отображается автоматически при отображении клавиатуры.
- **Переместить линию соединения**: Нажмите и удерживайте панель подключений затем и перетащите в новое расположение, в верхней части экрана.

### <a name="session-selection"></a>Выбор сеанса
Может иметь несколько подключений открыть на разных компьютерах, в то же время. Выберите в строке подключения для отображения панели Выбор сеанса в левой части экрана. Панель выбора сеанса позволяет увидеть ваши открытые подключения и переключаться между ними. 

- Переключение между приложениями в сеансе открыть удаленный ресурс.

    При подключении к удаленным ресурсам, можно переключаться между открытых приложений, в течение этого сеанса, коснитесь меню расширитель и выбрав из списка доступных элементов.
- Запустите новый сеанс

  В текущее соединение можно инициировать новые приложения или сеансы рабочего стола из: коснитесь **начать новый**, а затем выберите из списка доступных элементов.

- Отключение сеанса

  Для отключения отвод сеанса X в левой части сеанса плитки.

### <a name="command-bar"></a>Панель команд

На панели команд заменить программу панели начиная с версии 8.0.1. Можно переключаться между режимами мыши и вернуться в центр соединения на панели команд.

## <a name="use-touch-gestures-and-mouse-modes-in-a-remote-session"></a>Используйте сенсорный ввод и режимы мыши в удаленном сеансе

Клиент использует стандартный сенсорных жестов. Также можно сенсорные жесты воспроизводящие действия мыши удаленный рабочий стол. Доступные режимы мыши определяются в следующей таблице.

> [!NOTE]
> Взаимодействие с Windows 8 или более поздней версии собственного сенсорные жесты поддерживаются в режиме прямого касания. Дополнительные сведения о Windows 8 жестов см. в разделе [Touch: Проведите по экрану, касание и более](https://windows.microsoft.com/en-US/windows-8/touch-swipe-tap-beyond).

| Режим мыши    | Операции мышью      | Жест                                                    |
|---------------|----------------------|------------------------------------------------------------|
| Прямое касание  | Щелчок левой кнопкой           | коснитесь пальцем 1                                               |
| Прямое касание  | Щелчок правой кнопкой мыши          | 1 нажмите пальца и удержание                                      |
| Указатель мыши | Щелчок левой кнопкой           | коснитесь пальцем 1                                               |
| Указатель мыши | Щелкните левой кнопкой мыши и перетащите  | 1 палец двойного касания и удержание, затем перетащите                    |
| Указатель мыши | Щелчок правой кнопкой мыши          | коснитесь пальцем 2                                               |
| Указатель мыши | Щелкните правой кнопкой мыши и перетащите | 2 двойного касания пальцем и удержание, затем перетащите                    |
| Указатель мыши | Колесико мыши          | 2 палец, коснитесь и удерживайте, затем перетащите вверх или вниз                |
| Указатель мыши | Zoom                 | Сжатие 2 пальцы, чтобы увеличить или распределить 2 пальцы, чтобы уменьшить масштаб |

## <a name="use-a-keyboard-in-a-remote-session"></a>Использование клавиатуры в удаленном сеансе

Можно использовать либо экранной клавиатуры или физическую клавишу клавиатуры во время удаленного сеанса.

На экране клавиатуры, использовать кнопки на правого края панели над клавиатуры для переключения между стандартных и дополнительных клавиатуры.

Если Bluetooth включен для вашего устройства iOS, клиент автоматически обнаруживает клавиатуры Bluetooth.

Имейте в виду, что, из-за ограничений в операционной системе, специальные клавиши Ctrl, параметр и функции не будут работать должным образом с клавиатуры Bluetooth. Работают следующие ключи:

- Буквенно-цифровые ключи
- Клавиши управления курсором
- Вкладка ": Вкладке работает, но Shift + Tab не работает
- Главная / Pos1: Alt + Стрелка влево = Home
- Окончание: Alt + Стрелка вправо = End
- Page Up: Alt + Стрелка вверх = Page Up
- Page Down: Alt + Стрелка вниз = Page Down
- Выберите все: Команда + A = Ctrl + A (выберите все в большинстве программ)
- Вырежьте: Команда + X = Ctrl + X (вырезания в большинстве программ)
- Копировать: Command + C = Ctrl + C (Копировать в большинстве программ)
- Вставить: Команда + V = Ctrl + V (вставить в большинстве программ)
- Символы: Клавиши ALT + буквенно-цифровой создаст различными символами в зависимости от настройки языка
