# <a name="supported-remote-desktop-rdp-file-settings"></a>Поддерживаемые параметры RDP-файла удаленного рабочего стола

Следующая таблица содержит список поддерживаемых параметров RDP-файла, которые можно использовать с клиентами удаленных рабочих столов.

В таблице также показано, какие параметры поддерживаются в качестве пользовательских свойств для Виртуального рабочего стола Windows. См. сведения об [использовании PowerShell для настройки свойств протокола RDP для пулов узлов Виртуальных рабочих столов Windows](https://go.microsoft.com/fwlink/?linkid=2098243&clcid=0x409).

## <a name="connection-information"></a>Сведения о подключении

| Параметры RDP                        | Описание            | Значения                 | Значение по умолчанию          | Виртуальный рабочий стол Windows |
|------------------------------------|------------------------|------------------------|:----------------------:|:-----------------------:|
| full address:s:value | Имя компьютера.</br>Этот параметр указывает имя или IP-адрес удаленного компьютера, к которому вы хотите подключиться.</br></br>Это единственный обязательный параметр в RDP-файле. | Допустимое имя, IPv4- или IPv6-адрес. | | |
| alternate full address:s:value | Указывает альтернативное имя или IP-адрес удаленного компьютера. | Допустимое имя, IPv4- или IPv6-адрес. | | |
| username:s:value | Указывает имя учетной записи пользователя для входа на удаленный компьютер. | Любое допустимое имя пользователя. | | |
| domain:s:value | Указывает имя домена, в котором будет использоваться учетная запись пользователя для входа на удаленный компьютер. | Допустимое доменное имя, например CONTOSO. | | |
| gatewayhostname:s:value | Указывает имя узла шлюза удаленных рабочих столов. | Допустимое имя, IPv4- или IPv6-адрес. | | |
| gatewaycredentialssource:i:value | Задает метод проверки подлинности шлюза удаленных рабочих столов. | - 0: запрос пароля (NTLM).</br>- 1: использование смарт-карты.</br>- 4: пользователь может сделать выбор позже. | 0 | |
| gatewayprofileusagemethod:i:value | Указывает, следует ли использовать параметры по умолчанию шлюза удаленных рабочих столов. | - 0: используется режим профиля по умолчанию, как указано администратором.</br>- 1: используются явные параметры, указанные пользователем. | 0 | |
| gatewayusagemethod:i:value | Указывает, в каких случаях следует использовать шлюз удаленных рабочих столов для подключения. | - 0: шлюз удаленных рабочих столов не используется.</br>- 1: шлюз удаленных рабочих столов используется всегда.</br>- 2: шлюз удаленных рабочих столов используется, если не удается установить прямое подключение к узлу сеансов удаленных рабочих столов.</br>- 3: используются параметры по умолчанию шлюза удаленных рабочих столов.</br>- 4: шлюз удаленных рабочих столов не используется, подключение обходит шлюз для локальных адресов.</br>Значения 0 или 4 этого свойства действуют практически одинаково, но значение 4 позволяет обходить локальные адреса. | 0 | |
| promptcredentialonce:i:value | Определяет, сохраняются ли учетные данные пользователя и используются ли для шлюза удаленных рабочих столов и на удаленном компьютере. | - 0: удаленный сеанс не будет использовать те же учетные данные.</br>- 1: удаленный сеанс будет использовать те же учетные данные. | 1 | |
| authentication level:i:value | Определяет параметры уровня аутентификации на сервере. | - 0: если аутентификация на сервере не пройдена, можно подключиться к компьютеру без предупреждения (Подключаться без предупреждения).</br>- 1: если аутентификация на сервере не пройдена, не устанавливать подключение (Не подключаться).</br>- 2: если аутентификация на сервере не пройдена, выводится предупреждение и разрешается подключиться или отклонить подключение (предупреждение).</br>- 3: обязательная аутентификация не указана. | 3 | |
| enablecredsspsupport:i:value | Определяет, используется ли клиент для аутентификации поставщика поддержки безопасности учетных данных (CredSSP), если он доступен. | - 0: RDP не будет использовать CredSSP, даже если операционная система поддерживает CredSSP.</br>- 1: RDP будет использовать CredSSP, если операционная система поддерживает CredSSP. | 1 | X |
| disableconnectionsharing:i:value | Определяет, выполняет ли клиент повторное подключение к какому-либо существующему отключенному сеансу или инициирует новое подключение. | - 0: повторное подключение к имеющемуся сеансу.</br>- 1: инициация нового подключения. | 0 | X |
| alternate shell:s:value | Определяет программу, запускаемую автоматически в удаленном сеансе в качестве оболочки вместо Проводника. | Допустимый путь к исполняемому файлу, например C:\ProgramFiles\Office\word.exe. | | X |

## <a name="session-behavior"></a>Поведение сеанса

| Параметры RDP                        | Описание            | Значения                 | Значение по умолчанию          | Виртуальный рабочий стол Windows |
|------------------------------------|------------------------|------------------------|:----------------------:|:-----------------------:|
| autoreconnection enabled:i:value | Определяет, будет ли клиент автоматически пытаться повторно подключиться к удаленному компьютеру, если подключение прервано, например, при нарушении сетевого подключения. | - 0: клиент не пытается автоматически восстановить подключение.</br>- 1: клиент пытается автоматически восстановить подключение. | 1 | X |
| bandwidthautodetect:i:value | Определяет, включено ли автоматическое определение типа сети. | - 0: отключение автоматического определения типа сети.</br>- 1: включение автоматического определения типа сети. | 1 | X |
| networkautodetect:i:value | Определяет, следует ли использовать автоматическое определение пропускной способности сети. Требует, чтобы для параметра bandwidthautodetect было установлено значение 1. | - 0: автоматическое определение пропускной способности сети не используется.</br> - 1: автоматическое определение пропускной способности сети используется. | 1 | X |
| compression:i:value | Определяет, включено ли массовое сжатие при передаче данных по протоколу RDP на локальный компьютер.|- 0: отключение массового сжатия при передаче по протоколу RDP.</br>- 1: включение массового сжатия при передаче по протоколу RDP. | 1 | X |
| videoplaybackmode:i:value| Определяет, будет ли подключение использовать оптимизированную для RDP потоковую передачу мультимедиа для воспроизведения видео.|- 0: оптимизированная для RDP потоковая передача мультимедиа не используется для воспроизведения видео.</br>- 1: оптимизированная для RDP потоковая передача мультимедиа используется для воспроизведения видео. | 1 | X |

## <a name="device-redirection"></a>Перенаправление устройств

|------------------------------------|------------------------|------------------------|:----------------------:|:-----------------------:|

| encode redirected video capture:i:value | Включает или отключает кодирование перенаправленного видео. | - 0: отключить кодирование перенаправленного видео</br>- 1: включить кодирование перенаправленного видео | 1 | X |
| redirected video capture encoding quality:i:value | Управляет качеством закодированного видео. | - 0: видео с высокой степенью сжатия. Качество может снизиться, если в кадре часто присутствует движение. </br>- 1: средняя степень сжатия.</br>- 2: видео с низкой степенью сжатия и высоким качеством изображения. | 0 | X |
| audiomode:i:value | Расположение аудиовыхода.</br>Определяет, воспроизводит ли звук локальный или удаленный компьютер. | - 0: воспроизведение звуков на локальном компьютере ("Воспроизвести на этом компьютере").</br>- 1: воспроизведение звуков на удаленном компьютере ("Воспроизвести на удаленном компьютере").</br>- 2: отключение звука (Не воспроизводить). | 0 | X |
| camerastoredirect:s:value | Перенаправление камеры.</br>Позволяет настроить перенаправляемые камеры. | devicestoredirect:s:value | Перенаправление USB-устройств.</br>Определяет, какие устройства на локальном компьютере будут перенаправлены в удаленный сеанс. | – *: перенаправлять все поддерживаемые устройства, включая подключенные позже;</br> — допустимый идентификатор оборудования для одного или нескольких устройств. | Не перенаправлять устройства. | X |
| drivestoredirect:s:value | Перенаправление диска или хранилища.</br>Определяет, какие диски на локальном компьютере будут перенаправлены и доступны в удаленном сеансе. | Значение не указано: перенаправление дисков не используется.</br>- * : перенаправление всех дисков, включая диски, которые будут подключены позже.</br>DynamicDrives: перенаправление всех дисков, которые будут подключены позже.</br>Диск и метки для одного или нескольких дисков, например "drivestoredirect:s:C:;E:;": перенаправление указанных дисков. | Не перенаправлять диски. | X |
| redirectclipboard:i:value | Перенаправление буфера обмена.</br>Определяет, включено ли перенаправление буфера обмена. | - 0: буфер обмена на локальном компьютере недоступен в удаленном сеансе.</br>- 1: буфер обмена на локальном компьютере доступен в удаленном сеансе. | 1 | X |
| redirectprinters:i:value | Перенаправление принтеров.</br>Определяет, будут ли принтеры, настроенные на локальном компьютере, перенаправлены и доступны в удаленном сеансе. | - 0: принтеры на локальном компьютере недоступны в удаленном сеансе.</br>- 1: принтеры на локальном компьютере доступны в удаленном сеансе. | 1 | X |
| redirectsmartcards:i:value | Перенаправление смарт-карт.</br>Определяет, будут ли устройства чтения смарт-карт, настроенные на локальном компьютере, перенаправлены и доступны в удаленном сеансе. |- 0: устройства смарт-карт на локальном компьютере недоступны в удаленном сеансе.</br>- 1: устройства смарт-карт на локальном компьютере доступны в удаленном сеансе. | 1 | X |

|------------------------------------|------------------------|------------------------|:----------------------:|:-----------------------:|

Для параметра multimon нужно установить значение 1.</br></br>Доступно только на клиентах Windows Inbox (MSTSC) и Windows Desktop (MSRDC). | Разделенный запятыми список идентификаторов дисплеев для определенного компьютера. Идентификаторы можно получить с помощью команды mstsc.exe/l. Первый идентификатор в списке будет установлен в качестве основного дисплея для сеанса. | Все дисплеи | X |

| singlemoninwindowedmode:i:value | Определяет, будет ли удаленный сеанс с несколькими дисплеями автоматически переключаться на один дисплей при выходе из полноэкранного режима. Для параметра multimon нужно установить значение 1.</br></br>Доступно только на клиенте Windows Desktop (MSRDC). | - 0: сеанс остается на всех дисплеях при выходе из полноэкранного режима.</br>- 1: сеанс переключается на один дисплей при выходе из полноэкранного режима. | 0 | X |
| screen mode id:i:value | Определяет, будет ли отображаться окно удаленного сеанса на весь экран при запуске подключения. | - 1: удаленный сеанс будет отображаться в окне.</br>- 2: удаленный сеанс будет отображаться на весь экран. | 2 | X |

| desktopwidth:i:value | Задает ширину разрешения удаленного сеанса в пикселях. | Числовое значение в диапазоне от 200 до 8192. | В соответствии с параметрами локального компьютера. | X |
