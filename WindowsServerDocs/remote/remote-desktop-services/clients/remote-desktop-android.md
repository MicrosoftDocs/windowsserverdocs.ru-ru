# <a name="get-started-with-remote-desktop-on-android"></a>Начало работы с удаленного рабочего стола на устройстве Android

>Область применения. Windows 10, Windows 8.1, Windows Server 2012 R2, Windows Server 2016

Клиент удаленного рабочего стола для Android можно использовать для работы с приложениями Windows и настольных компьютеров непосредственно с устройства Android.

Используйте следующие сведения для начала работы. Обязательно ознакомьтесь с [часто задаваемые вопросы о](remote-desktop-client-faq.md) Если у вас возникнут вопросы.

> [!NOTE]
> - Узнать о новых выпусках для клиента Android? Ознакомьтесь с [новые возможности удаленного рабочего стола на устройстве Android?](android-whatsnew.md)
> На Android 4.1 и более новых устройствах, а также на ли Chromebooks с 53 ChromeOS установки можно запустить клиент. Дополнительные сведения о приложениях Android на Chrome [здесь](https://sites.google.com/a/chromium.org/dev/chromium-os/chrome-os-systems-supporting-android-apps).

## <a name="get-the-rd-client-and-start-using-it"></a>Получение клиента к удаленному рабочему Столу и начать с ним

Выполните следующие действия, чтобы начать работу с удаленного рабочего стола на устройстве Android.

1. Скачивание клиента удаленного рабочего стола из [Google Play](https://play.google.com/store/apps/details?id=com.microsoft.rdc.android). 
2. [Настройка компьютера для удаленных соединений](remote-desktop-allow-access.md).
3. Добавьте к удаленному рабочему столу или удаленному ресурсу. Виртуальный рабочий стол публикации в локальной, или использовать подключение для прямого подключения к ПК Windows и удаленный ресурс, для использования удаленного приложения RemoteApp, рабочего стола на основе сеанса. 
4. Создайте мини-приложение, чтобы быстро приступить к удаленному рабочему столу.

> [!NOTE]
> Если вы хотите ранее черный новых функций мы рекомендуем скачать наш [Microsoft удаленного рабочего стола бета-версии](https://play.google.com/store/apps/details?id=com.microsoft.rdc.android.beta) приложения из магазина Google Play. 

### <a name="add-a-remote-desktop-connection"></a>Добавление подключения к удаленному рабочему столу

Чтобы создать подключение удаленного рабочего стола:

1. В tap Центр подключений **+**, а затем коснитесь **Desktop**.
2. Введите следующие сведения для компьютера, что вы хотите подключиться к:
  - **Имя компьютера** — имя компьютера. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно добавить к имени компьютера (например, **MyDesktop:3389** или **10.0.0.1:3389**).
  - **Имя пользователя** — имя пользователя для доступа к удаленным ПК. Можно использовать следующие форматы: *user_name*, *"Домен\имя_пользователя"*, или *user_name@domain.com*. Можно также указать, следует ли запрашивать имя пользователя и пароль.
3. Также можно задать следующие дополнительные параметры:
  - **Понятное имя** — легко запоминаемых име при подключении к ПК. Можно использовать любой строкой, но если понятное имя не указано, отображается имя компьютера.
  - **Шлюз** — шлюз удаленных рабочих столов, который вы хотите использовать для подключения к виртуальным рабочим столам, программам RemoteApp и рабочим столам на основе сеансов во внутренней корпоративной сети. Получите сведения о шлюзе у системного администратора.
    Необходимость в настройке шлюза удаленных рабочих столов?
  - **Звук** — выберите устройство, используемый для аудио во время удаленного сеанса. Вы можете для воспроизведения звука на локальных устройствах, удаленное устройство, или вообще не.
  - **Настраивать разрешающую способность** -задать пользовательское разрешение для подключения при включении этого параметра. При применении off разрешения которого вы определили в глобальных параметрах приложения.
  - **Кнопок мыши** — используйте этот параметр для замены функции кнопки мыши правой кнопкой мыши. (Это особенно полезно, если удаленный компьютер настроен для левши пользователя, но используется для правши мышь.)
  - **Подключиться к сеансу администрирования** -используйте этот параметр для подключения к сеансу консоли для администрирования сервера Windows.
  - **Перенаправление в локальное хранилище** — подключает в локальное хранилище в качестве удаленной файловой системы на удаленном ПК.
4. Коснитесь **Сохранить**.

Необходимо изменить эти параметры? Коснитесь меню переполнения (**...** ) рядом с именем рабочего стола, а затем коснитесь **изменить**.

Удалить подключение? Опять же, коснитесь меню переполнения (**...** ), а затем коснитесь **удалить**.

>[!TIP]
> Если возникнет ошибка 0xf07 неверный пароль («не удалось подключиться к удаленным ПК, так как пароль, связанный с учетной записи пользователя истек»), изменить пароль и повторите попытку.

### <a name="add-a-remote-resource"></a>Добавить удаленный ресурс
Удаленные ресурсы являются программы RemoteApp, рабочие столы на основе сеансов и виртуальных рабочих столов, опубликованной с использованием подключения к удаленным рабочим столам и приложениям RemoteApp.

Чтобы добавить удаленный ресурс:

1. На экране Центр подключений, нажмите кнопку **+**, а затем коснитесь **удаленных ресурсов веб-канала**. 
2. Введите сведения для удаленного ресурса:
   - **Адрес электронной почты или URL-адрес** -URL-адрес сервера веб-доступа к удаленным рабочим Столам. Можно также ввести учетную запись корпоративной электронной почты в этом поле — при этом клиент для поиска сервера веб-доступа к удаленному рабочему Столу, связанные с вашим адресом электронной почты.
   - **Имя пользователя** -имя пользователя для использования сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
   - **Пароль** -пароль, используемый для сервера веб-доступа к удаленным рабочим Столам, вы подключаетесь.
3. Коснитесь **Сохранить**.

Удаленные ресурсы будут отображаться в центре подключений.


Чтобы удалить удаленные ресурсы:

1. В центре подключений, выберите в меню переполнения (**...** ) рядом с удаленного ресурса.
2. Коснитесь **удалить**.
3. Подтвердите удаление.

### <a name="widgets--pin-a-saved-desktop-to-your-home-screen"></a>Мини-приложения — ПИН-код сохраненного рабочего стола на начальный экран

Удаленный рабочий стол приложения поддерживают закрепляющий подключений на начальный экран с помощью функции Android мини-приложения. Способ добавления мини-приложение зависит от типа вашего устройства Android вы используете, и его операционной системы. Вот наиболее распространенный способ создания мини-приложения: 

1. Коснитесь **приложений** для запуска в меню приложения.
2. Коснитесь **мини-приложения**.
3. Проведите пальцем по мини-приложений и найдите значок удаленного рабочего стола с описанием «ПИН-код удаленного рабочего стола.»
4. Нажмите и удерживайте мини-приложения удаленного рабочего стола и переместите его на начальном экране.
5. Если отпустить значок, вы увидите, что сохраненный удаленные рабочие столы. Выберите подключение, которое вы хотите сохранить на начальном экране.

Теперь можно начать удаленного рабочего стола непосредственно с начального экрана, коснувшись его.

> [!NOTE]
> При переименовании подключения к рабочему столу в приложение удаленного рабочего стола, метки этого закрепленные удаленного рабочего стола не будут обновляться.

## <a name="connect-to-an-rd-gateway-to-access-internal-assets"></a>Подключение к шлюзу удаленных рабочих Столов для доступа к внутренними ресурсами

Шлюз удаленных рабочих столов (шлюз RD) позволяет подключаться к удаленному компьютеру в корпоративной сети из любого места в Интернете. Можно создать и управлять шлюзами с помощью клиента удаленного рабочего стола.

Чтобы настроить новый шлюз:

1. В центре подключений, коснитесь **параметры > шлюзы**. Коснитесь **+** для добавления нового шлюза.
2. Введите следующие сведения:
  - **Имя сервера** — имя компьютера, который вы хотите использовать в качестве шлюза. Это может быть именем компьютера Windows, имени домена Интернета или IP-адресом. Сведения о порте можно также добавить к имени сервера (например: **RDGateway:443** или **10.0.0.1:443**).
  - **Имя пользователя** -имя пользователя и пароль, используемый для шлюза удаленных рабочих столов, вы подключаетесь. Можно также выбрать **использовать учетную запись пользователя рабочего стола** использовать те же учетные данные, которые используются для удаленного рабочего стола.

## <a name="manage-your-user-accounts"></a>Управление учетными записями пользователей

При подключении к рабочего стола или удаленных ресурсов, можно сохранить учетные записи пользователей, чтобы выбрать еще раз. Можно также определить учетные записи пользователей в сам клиент, в отличие от сохранение данных пользователя при подключении к рабочему столу.

Чтобы создать новую учетную запись пользователя:

1. В центре подключений, коснитесь **параметры**, а затем коснитесь **учетные записи пользователей**.
2. Коснитесь **+** для добавления учетной записи пользователя.
3. Введите следующие сведения:
   - **Имя пользователя** -имя пользователя, чтобы сохранить для использования с удаленного подключения. Можно ввести имя пользователя в любом из следующих форматов: user_name, "Домен\имя_пользователя", или user_name@domain.com.
   - **Пароль** -пароль для пользователя, который вы указали. Каждой учетной записи пользователя, который вы хотите сохранить для использования для удаленных соединений должен иметь пароль, связанный с ним.
4. Коснитесь **Сохранить**.

Чтобы удалить учетную запись пользователя:

1. В центре подключений, коснитесь **параметры > учетные записи пользователей**.
2. Нажмите и удерживайте учетной записи пользователя в списке, чтобы выбрать его. Можно выбрать несколько пользователей.
3. Коснитесь Корзина для удаления выбранного пользователя.

## <a name="navigate-the-remote-desktop-session"></a>Навигация по сеансу удаленного рабочего стола
При запуске удаленного рабочего стола, существуют средства, которое можно использовать для Навигация по сеансу.

### <a name="start-a-remote-desktop-connection"></a>Запустить подключение к удаленному рабочему столу

1. Выберите подключение удаленного рабочего стола для запуска сеанса. 
2. Если будет предложено проверить сертификат для удаленного рабочего стола, коснитесь **Connect**. Можно также выбрать **больше не спрашивать для подключения к этому компьютеру** всегда принять сертификат.

### <a name="manage-global-app-settings"></a>Управление параметрами глобального приложения

Следующие глобальные параметры можно задать в клиентском приложении Android:

- **Показать предварительные версии Desktop** -позволяет просмотреть рабочий стол в центре подключений перед подключением к нему. По умолчанию, это имеет значение **на**.
- **Сжатие для увеличения** -позволяет использовать жесты масштаба жестом сжатия. Если вы используете через удаленный рабочий стол приложение поддерживает мультисенсорные (появился в Windows 8), включите этот параметр **off**.
- **Справка для улучшения удаленного рабочего стола** -отправляет в корпорацию Майкрософт анонимные данные. Мы используем эти данные для повышения клиент. Можно узнать больше о том, как мы обращаемся этот анонимный, закрытых данных, см. в разделе [заявление о конфиденциальности клиента удаленного рабочего стола](https://www.microsoft.com/privacystatement/RemoteApp/Default.aspx). По умолчанию этому параметру задано **на**.
- **Отобразить** -есть две глобальные параметры для монитора:
   - **Ориентация** -задает предпочтительный ориентации (книжная или альбомная) для сеанса. 
   >[!NOTE]
   > При подключении к ПК под управлением Windows 8 или более ранней версии Windows, сеанс не будет правильно масштабировать. Лучшим решением является отключиться от компьютера и снова установите ориентацию, которую вы хотите использовать. Еще лучшим вариантом является по крайней мере обновить ПК до Windows 8.1.

   - **Разрешение** -задает разрешение, вы хотите использовать для подключения к рабочему столу, глобально. Если вы уже задали пользовательское разрешение для отдельные приложения или подключения, этот параметр не будет заменить его.
   >[!NOTE]
   >При изменении одного из параметров отображения, они применяются только для создания подключений с этого момента на. Чтобы увидеть изменения в сеансе вы уже подключены к отключиться и повторите попытку подключения.

### <a name="connection-bar"></a>Панель подключений

Подключение панель обеспечивает доступ к элементам управления дополнительных переходов. По умолчанию в строке подключения помещается в середине в верхней части экрана. Дважды щелкните и перетащите ползунок влево или вправо, чтобы переместить его.

- **Сдвиг управления**: Элемент управления pan позволяет экрана, чтобы быть увеличено и в дальнейшем двигался. Обратите внимание на то, что Панорама элемент управления доступен только с помощью прямого касания.
   - Включение и отключение управления pan: Коснитесь значка панорамирования в строке подключения, чтобы отобразить элемент управления сдвиг и масштабирование экрана. Коснитесь значка панорамирования в строке подключения, еще раз, чтобы скрыть элемент управления и вернуться к его исходным разрешением экрана.
   - Использование элемента управления pan: Нажмите и удерживайте клавишу CTRL pan, а затем перетащите в направлении, чтобы переместить на экране.
   - Перемещение элемента управления pan: Double коснитесь и удерживайте pan, чтобы переместить элемент управления на экране.
- **Дополнительные параметры**: Значок Дополнительные параметры, чтобы отобразить выделение сеанса панели и командной строки (см. ниже).
- **Клавиатуры**: Нажмите значок клавиатуры, чтобы отобразить или скрыть клавиатуры. Элемент управления pan завершения отображается автоматически при отображении клавиатуры.
- **Переместить линию соединения**: Нажмите и удерживайте панель подключений затем и перетащите в новое расположение, в верхней части экрана.


### <a name="command-bar"></a>Панель команд

Выберите в строке подключения для отображения на панели команд в правой части экрана. Вы можете переключаться между режимами мыши (прямого касания и указатель мыши). Используйте кнопка "Главная", чтобы вернуться в центр подключений из панели команд. Кроме того можно использовать кнопки «Назад» для одного действия. Ваш сеанс не будет отключен. 


### <a name="use-direct-touch-gestures-and-mouse-modes-in-a-remote-session"></a>Используйте прямой touch жестов и режимы мыши в удаленном сеансе

Клиент использует стандартный сенсорных жестов. Также можно сенсорные жесты воспроизводящие действия мыши удаленный рабочий стол. Доступные режимы мыши определяются в следующей таблице.

> [!NOTE]
> Взаимодействие с Windows 8 или более поздней версии собственного сенсорные жесты поддерживаются в режиме прямого касания. 

| Режим мыши    | Операции мышью      | Жест                                                               |
|---------------|----------------------|-----------------------------------------------------------------------|
| Прямое касание  | Щелчок левой кнопкой           | коснитесь пальцем 1                                                          |
| Прямое касание  | Щелчок правой кнопкой мыши          | 1 нажмите пальца и удержание                                                 |
| Указатель мыши | Zoom                 | Используйте 2 пальцев и сжатие для увеличения и уменьшения перемещения пальцы друг от друга, чтобы уменьшить масштаб. |
| Указатель мыши | Щелчок левой кнопкой           | коснитесь пальцем 1                                                          |
| Указатель мыши | Щелкните левой кнопкой мыши и перетащите  | 1 палец двойного касания и удержание, затем перетащите                               |
| Указатель мыши | Щелчок правой кнопкой мыши          | коснитесь пальцем 2                                                          |
| Указатель мыши | Щелкните правой кнопкой мыши и перетащите | 2 двойного касания пальцем и удержание, затем перетащите                               |
| Указатель мыши | Колесико мыши          | 2 палец, коснитесь и удерживайте, затем перетащите вверх или вниз                           |

Вместо этого перейдите к [форум клиента удаленного рабочего стола](https://social.technet.microsoft.com/forums/windowsserver/en-us/home?forum=winrdc) и запустить новый поток.
