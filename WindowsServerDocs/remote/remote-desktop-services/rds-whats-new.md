# <a name="whats-new-in-remote-desktop-services"></a>Новые возможности служб удаленных рабочих столов

Remote Desktop Services (RDS) на Windows Server 2016 — это платформа виртуализации, включение широкий спектр сценариев клиентов. Усовершенствования в RDS решения в целом включает работы, выполненной команды удаленного рабочего стола и другие партнеры по разработке технологий в корпорации Microsoft. Следующие сценарии и технологии, обновлены или улучшены в Windows Server 2016.

Также обязательно ознакомьтесь с нашей сеанса на конференции Ignite 2016: [Использовать оптимизированные возможности RDS в Windows Server 2016](https://channel9.msdn.com/Events/Ignite/2016/BRK3098). В этом видео рассматриваются группы разработчиков продукта все новые и улучшенные возможности служб удаленных рабочих столов, включая поддержку vGPU. 

## <a name="app-compatibility---windows-server-2016-and-windows-10"></a>Совместимость приложений — Windows Server 2016 и Windows 10
Основанная на ту же основу, Windows 10, Windows Server 2016 не имеет только с оформлением ожидать за пределы рабочего стола, но также могут запускать множество функциональных те же приложения. Выполнить связывание Windows Server 2016 с помощью графических средств (см. ниже) предоставляет среду для всех пользователей для обеспечения производительности. 

## <a name="azure-sql-database---the-new-database-for-your-highly-available-environment"></a>База данных SQL Azure — новая база данных для среды высокой доступности
Посредник подключений к удаленному рабочему Столу имеет возможность сохранять сведения о развертывании (например, состояния подключения и сопоставления пользователя и узла) в общую базу данных SQL, например к базе данных Azure SQL. Выделить ручного развертывания SQL Server группы доступности AlwaysOn, получить строку подключения к базе данных Azure SQL и начните использовать среду высокой доступности.

Дополнительные сведения: [Использовать базу данных SQL Azure для высокодоступной среды посредника подключений к удаленному рабочему столу](https://blogs.technet.microsoft.com/enterprisemobility/2016/05/03/new-windows-server-2016-capability-use-azure-sql-db-for-your-remote-desktop-connection-broker-high-availability-environment/)

## <a name="graphics---solving-graphics-needs-across-various-scenarios"></a>Графика - удовлетворения потребностей в графики для различных сценариев
Благодаря Hyper-V дискретных назначение устройств теперь можно сопоставить графических процессоров на хост-компьютере непосредственно к виртуальной Машине для обработки в свои приложения, требующие GPU. Также были внесены усовершенствования в vGPU RemoteFX, включая поддержку OpenGL 4.4, OpenCL 1.1, разрешением k 4 и виртуальные машины Windows Server.

Дополнительные сведения: [Дискретное назначение устройств](https://blogs.technet.microsoft.com/virtualization/2015/11/)

## <a name="rd-connection-broker---improved-connection-handling-during-logon-storms"></a>Посредник подключений к удаленному рабочему Столу - улучшенного подключения, обработка во время входа в систему лавины
С обработкой улучшенного подключения, посредника подключений к удаленному рабочему Столу теперь имеет возможность обработки более 10 000 запросов число одновременных подключений, иногда происходит во время «лавины входа в систему». Улучшенная посредника подключений к удаленному рабочему Столу также упрощает обслуживание развертывания проще возможность более быстро добавить серверы обратно в среду.

Дополнительные сведения: [Улучшенная производительность Broker подключения удаленного рабочего стола](https://blogs.technet.microsoft.com/enterprisemobility/2015/12/15/improved-remote-desktop-connection-broker-performance-with-windows-server-2016-and-windows-server-2012-r2-hotfix-kb3091411/)

## <a name="rdp-10---new-capabilities-built-into-the-protocol"></a>RDP 10 — новые возможности, встроенные в протокол
10 протокола удаленного рабочего СТОЛА теперь использует кодек H.264/AVC 444, соответствующим образом оптимизация как видео, так и текст. В этом выпуске также поддерживается удаленное взаимодействие пера. Благодаря этим возможностям сеансов работы удаленного запуска чувствовать даже больше похожи на локальный сеанс.  

Дополнительные сведения: [Улучшения кодеков AVC/H.264 10 протокола удаленного рабочего СТОЛА в Windows 10 и Windows Server 2016](https://blogs.technet.microsoft.com/enterprisemobility/2016/01/11/remote-desktop-protocol-rdp-10-avch-264-improvements-in-windows-10-and-windows-server-2016-technical-preview/)

## <a name="personal-session-desktops---providing-individual-desktops-to-any-end-user"></a>Личные сеансовых рабочих столов — предоставление отдельных рабочих столах любой конечному пользователю.
Личные сеансовых рабочих столов — это новый способ имеет свои собственные личного рабочего стола, размещенных для вас в облаке. Права администратора и удаляет узлы выделенного сеанса сложности, на котором размещается средах, где пользователи хотят управления настольной системой, как и свои собственные.

Дополнительные сведения: [Личные сеансовых рабочих столов](rds-personal-session-desktops.md)
