# <a name="migrate-your-remote-desktop-services-client-access-licenses-rds-cals"></a>Перенос клиентских лицензий (CAL) служб удаленных рабочих столов (RDS)

У вас есть три варианта миграции Клиентские лицензии:
1. Метод автоматического подключения: Этот рекомендуемый метод обменивается данными через Интернет непосредственно к расчетной палате корпорации Майкрософт исходящего трафика через TCP-порт 443.  
2. С помощью веб-браузер: Этот метод позволяет миграции, когда сервер, запустив средство Диспетчер лицензирования удаленных рабочих столов не имеет подключения к Интернету, но администратор имеет подключение к Интернету на отдельном устройстве. URL-адрес для веб-метода миграции отображается в мастере управления клиентскими лицензиями служб удаленных рабочих СТОЛОВ. 
3. С помощью телефона: Этот метод позволяет администратору завершить процесс миграции по телефону с представителем Майкрософт. Соответствующий номер телефона определяется по странам и регионам, выбранных в мастере активации сервера и отображается в мастере управления клиентскими лицензиями служб удаленных рабочих СТОЛОВ.

В этой статье [метод установления RDS CAL миграции](#establish-rds-cal-migration-method) выделяет распространенные общие шаги по любой метод миграции клиентских лицензий служб удаленных рабочих СТОЛОВ, хотя [перенос клиентских лицензий служб удаленных рабочих СТОЛОВ](#migrate-rds-cals) выделяет конкретные инструкции каждую операцию миграции метод.

Независимо от метода миграции как минимум, будьте является членом локальной группы администраторов для выполнения шагов миграции.

## <a name="establish-rds-cal-migration-method"></a>Установить метод миграции клиентских лицензий служб удаленных рабочих СТОЛОВ

1. На сервере лицензирования откройте **Диспетчер лицензирования удаленных рабочих столов**. (Щелкните **Пуск > Администрирование**. Введите **служб удаленных рабочих столов** каталог и запустите **Диспетчер лицензирования удаленных рабочих столов**.)
2. Проверить метод подключения для сервера лицензирования удаленных рабочих столов: щелкните правой кнопкой мыши сервер лицензий, к которому вы хотите рабочих СТОЛОВ, а затем нажмите кнопку **свойства**. На **метод подключения** Убедитесь **метод подключения** -его можно изменить в раскрывающемся меню. Нажмите кнопку **ОК**.
3. Щелкните правой кнопкой мыши сервер лицензий, к которому вы хотите рабочих СТОЛОВ, а затем нажмите кнопку **управление клиентскими лицензиями служб удаленных рабочих СТОЛОВ**.
4. Следуйте указаниям мастера, чтобы **Выбор действия** страницы. Нажмите кнопку **перенести CAL RDS с другого сервера лицензирования на этот сервер лицензирования**.
6. Выберите причину перенос клиентских лицензий служб удаленных рабочих СТОЛОВ и нажмите кнопку **Далее**. Можно выбрать один из следующих вариантов:
    - Исходный сервер лицензирования заменяется на этот сервер лицензирования.
    - Исходный сервер лицензирования перестал функционировать.
7. На следующую страницу мастера зависит от причины миграции, вы выбрали.
    - Если вы выбрали **исходный сервер лицензирования заменяется этот сервер лицензирования** как причина перенос клиентских лицензий служб удаленных рабочих СТОЛОВ, **сведения о сервере лицензирования источника** откроется страница.
    
       На странице "сведения о сервере лицензирования источник" введите имя или IP-адрес исходного сервера лицензирования.

       Если исходный сервер лицензирования в сети, нажмите кнопку **Далее**. Мастер обращается исходный сервер лицензирования. Если исходный сервер лицензирования работает операционной системы более ранних, чем Windows Server 2008 R2 или деактивирован исходный сервер лицензирования, получают уведомление, что вы вручную удалить клиентских лицензий служб удаленных рабочих СТОЛОВ с исходного сервера лицензирования по после завершения работы мастера. Убедившись, что вы понимаете это требование **получить ключевой пакет клиентской лицензии** появится страница.

       Если исходный сервер лицензирования недоступен в сети, выберите **указанный исходный сервер лицензирования недоступен в сети**. Укажите операционную систему, на котором выполняется исходный сервер лицензирования и укажите идентификатор сервера лицензирования для этого сервера. После того как вы щелкнете **Далее**, получают уведомление, что вы вручную удалить клиентских лицензий служб удаленных рабочих СТОЛОВ с исходного сервера лицензирования по после завершения работы мастера. Убедившись, что вы понимаете это требование **получить ключевой пакет клиентской лицензии** появится страница.

    - Если вы выбрали **исходный сервер лицензирования перестал функционировать** как причина перенос клиентских лицензий служб удаленных рабочих СТОЛОВ, получают уведомление, что вы вручную удалить клиентских лицензий служб удаленных рабочих СТОЛОВ с исходного сервера лицензирования по после завершения работы мастера. Убедившись, что вы понимаете это требование **получить ключевой пакет клиентской лицензии** появится страница.

Следующим шагом является перенос клиентских лицензий - используйте приведенные ниже сведения для завершения работы мастера. Обратите внимание на то, что вы видите в мастере зависит от метода подключения, определенного в шаге 2 выше.

## <a name="migrate-rds-cals"></a>Перенос клиентских лицензий служб удаленных рабочих СТОЛОВ

Существует три механизма для переноса лицензий на сервере назначения лицензии; Выполните шаги, соответствующий **метод подключения** проверен на шаге 2:
  - [Метод автоматического подключения](#automatic-connection-method)
  - [В веб-браузере](#using-a-web-browser)
  - [По телефону](#using-a-telephone)

### <a name="automatic-connection-method"></a>Метод автоматического подключения

1. На **программе лицензирования** странице, выберите подходящую программу, по которой приобретены Клиентские лицензии, а затем нажмите кнопку **Далее**.
2. Введите необходимые сведения (обычно код лицензии или номер соглашения, в зависимости от **программе лицензирования**), а затем нажмите кнопку **Далее**. См. в документации, предоставленной вам при покупке лицензии служб удаленных рабочих СТОЛОВ.
4. Выберите подходящую версию продукта, тип лицензии и количество клиентских лицензий служб удаленных рабочих СТОЛОВ для вашей среды, в зависимости от соглашения приобретения клиентских лицензий служб удаленных рабочих СТОЛОВ и нажмите кнопку **Далее**.
5. Мастер автоматически свяжется с расчетной палатой корпорации Майкрософт, и она обработает запрос. Клиентские лицензии служб удаленных рабочих СТОЛОВ, затем переносятся на сервере лицензирования.
6. Нажмите кнопку **Готово** для завершения процесса миграции клиентских лицензий служб удаленных рабочих СТОЛОВ.

### <a name="using-a-web-browser"></a>В веб-браузере
1. На **получить ключевой пакет клиентской лицензии** щелкните гиперссылку, чтобы соединиться с удаленного рабочего стола служб лицензирования веб-сайта.
Если вы используете Диспетчер лицензирования удаленных рабочих столов на компьютере, который не поддерживает подключение к Интернету, запишите адрес, удаленный рабочий стол служб лицензирования веб-сайта и затем подключитесь к веб-сайта на компьютере, подключенном к Интернету. 
2. Удаленный рабочий стол служб лицензирования веб-страницы в разделе **Выбор параметра**выберите **управление клиентскими лицензиями**и нажмите кнопку **Далее**.
3. Укажите необходимые сведения, а затем нажмите кнопку **Далее**:
    - **Целевой идентификатор сервера лицензирования**: 35 цифр, в группы по 5 цифр, которые отображаются на **получить ключевой пакет клиентской лицензии** страницы в мастере управления клиентскими лицензиями служб удаленных рабочих СТОЛОВ.
    - **Причина для восстановления**: Выберите причину перенос клиентских лицензий служб удаленных рабочих СТОЛОВ.
    - **Лицензии программы**: Выберите программу, по которой приобретены Клиентские лицензии.
4. Укажите необходимые сведения, а затем нажмите кнопку **Далее**:
    - Фамилия
    - Имя или заданное имя
    - Название организации
    - Страна или регион

    Также можно предоставить дополнительные сведения, например адрес компании, адрес электронной почты и номер телефона. В поле «Подразделение» можно описать единицы в вашей организации, этот сервер лицензирования.

5. Программа лицензирования, выбранная на предыдущей странице определяет, какие сведения необходимо предоставить на следующей странице. В большинстве случаев необходимо предоставить код лицензии или номер соглашения. См. в документации, предоставленной вам при покупке лицензии служб удаленных рабочих СТОЛОВ. Кроме того необходимо указать, какой тип клиентских лицензий служб удаленных рабочих СТОЛОВ и количество, которое вы хотите перейти на сервере лицензирования.
6. После ввода необходимой информации нажмите кнопку **Далее**.
7. Убедитесь, что все сведения, которые вы ввели правильный, а затем щелкните **Далее** Чтобы отправить запрос в расчетную палату корпорации Майкрософт. Веб-страницы, затем отображается код ключевого пакета лицензий, создаваемые в информационную службу Майкрософт.

   > [!IMPORTANT] 
   > Сохраните копию идентификатор ключевого пакета лицензий. Наличие этой информации вы облегчает обмен данными с расчетной палатой корпорации Майкрософт, вам необходима помощь в восстановлении клиентских лицензий служб удаленных рабочих СТОЛОВ.

8. На том же **получить ключевой пакет клиентской лицензии** странице, введите идентификатор пакета лицензионных ключей и затем нажмите кнопку **Далее** для рабочих СТОЛОВ на сервере лицензирования.
9. Нажмите кнопку **Готово** для завершения процесса миграции клиентских лицензий служб удаленных рабочих СТОЛОВ.

### <a name="using-a-telephone"></a>По телефону
1. На **получить ключевой пакет клиентской лицензии** странице, использовать номер телефона отображается для вызова в информационную службу Майкрософт. Сообщите представителю идентификатор сервера лицензирования на удаленный рабочий стол и необходимые сведения о программе лицензирования, по которой приобретены Клиентские лицензии. Представитель обрабатывает ваш запрос на рабочих СТОЛОВ и содержат уникальный идентификатор для клиентских лицензий служб удаленных рабочих СТОЛОВ. Этот уникальный идентификатор называется **код ключевого пакета лицензии**.

   > [!IMPORTANT]
   > Сохраните копию идентификатор ключевого пакета лицензий. Наличие этой информации вы облегчает обмен данными с расчетной палатой корпорации Майкрософт, вам необходима помощь в восстановлении клиентских лицензий служб удаленных рабочих СТОЛОВ.

2. На том же **получить ключевой пакет клиентской лицензии** странице, введите идентификатор пакета лицензионных ключей и затем нажмите кнопку **Далее** для рабочих СТОЛОВ на сервере лицензирования.
3. Нажмите кнопку **Готово** для завершения процесса миграции клиентских лицензий служб удаленных рабочих СТОЛОВ.
