# <a name="configure-disaster-recovery-for-remote-desktop-services"></a>Настройка аварийного восстановления для служб удаленных рабочих столов

При развертывании служб удаленных рабочих столов в вашей среде, она становится важной частью инфраструктуры, особенно приложения и ресурсы, которые совместно с пользователями. Если развертывание служб удаленных рабочих СТОЛОВ выходит из строя из-за ничего после сбоя сети для стихийного бедствия, недоступна для пользователей этих приложений и ресурсов и отрицательно влияет на вашего бизнеса. Чтобы избежать этого, можно настроить решение аварийного восстановления, позволяющий осуществлять переход на другой ресурс развертывания — Если развертывание служб удаленных рабочих СТОЛОВ будет недоступна, для какой-либо причине, нет резервной копии автоматически смены.

Для обеспечения развертывания служб удаленных рабочих СТОЛОВ под управлением в случае компонент и машины, которые выходят из строя, рекомендуется Настройка развертывания служб удаленных рабочих СТОЛОВ для обеспечения высокой доступности. Это можно сделать, настроив [фермы RDSH](rds-scale-rdsh-farm.md) и проверку того, ваши [посредников подключений к удаленному кластеризованы для обеспечения высокой доступности](rds-connection-broker-cluster.md). 

Решения аварийного восстановления, мы рекомендуем здесь являются обеспечивает защиту развертывания после катастрофических сбоев - то, что принимает работу всего развертывания служб удаленных рабочих СТОЛОВ (включая избыточные ролей, настроенных для обеспечения высокой доступности). Если такое случается, решение аварийного восстановления, встроенные в развертывании будет можно выполнить отработку отказа всего развертывания и быстрое приложения и ресурсы и выполнение для пользователей.

Используйте следующие сведения для развертывания решений для аварийного восстановления в служб удаленных рабочих СТОЛОВ.

- [Использовать несколько центров обработки данных Azure, чтобы убедиться, что пользователи могут обращаться к развертывания служб удаленных рабочих СТОЛОВ, даже если один центр обработки данных Azure выходит из строя (геоизбыточность)](rds-multi-datacenter-deployment.md)
- [Развернуть Azure Site Recovery, чтобы обеспечить отработку отказа для компонентов служб удаленных рабочих СТОЛОВ в сайт сайт или сайт в Azure отработки отказа](rds-disaster-recovery-with-azure.md)
