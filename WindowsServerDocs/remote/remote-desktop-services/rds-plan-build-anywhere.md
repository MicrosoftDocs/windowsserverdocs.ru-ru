# <a name="remote-desktop-services---build-anywhere"></a>Службы удаленных рабочих столов - сборки в любом месте

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

Развертывание локально, в облаке или в гибридных условиях. Измените развертывание, при изменении потребностей вашего бизнеса.

Независимо от местонахождения, базовый [архитектура](desktop-hosting-logical-architecture.md) служб удаленных рабочих столов в среде остается неизменным:
- По-прежнему необходимо иметь сервер выходом в Интернет, использовать веб-доступа к удаленным рабочим Столам и шлюза удаленных рабочих Столов для внешних пользователей
- По-прежнему необходимо иметь Active Directory и--для высокой доступности сред--базы данных SQL для пользователя дома и свойства удаленного рабочего стола
- Необходимо по-прежнему иметь доступ обмен данными между ролей инфраструктуры удаленных рабочих Столов (RD Connection Broker, шлюз удаленных рабочих Столов, Лицензирование удаленных рабочих Столов и веб-доступа к удаленному рабочему Столу) и завершением RDSH или RDVH узлов, чтобы иметь возможность подключения конечных пользователей в свои рабочие столы и приложения.

Такая гибкость позволяет получить преимущества обоих миров:
- Простота и оплаты по мере использования методы, связанные с облаком и мир Интернета.
- Знакомство и беспроблемный путь от использования ресурсов, которые уже существуют локально.

Дополнительные сведения, рассмотрим получение [построение и развертывание служб удаленных рабочих столов развертывания](rds-build-and-deploy.md).
