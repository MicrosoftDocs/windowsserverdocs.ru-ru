# <a name="optimizing-windows-10-version-1803-for-a-virtual-desktop-infrastructure-vdi-role"></a>Оптимизация Windows 10 версии 1803, для роли инфраструктуры виртуальных рабочих столов (VDI)

Эта статья поможет вам выбрать параметры для Windows 10 версии 1803 (сборка 17134), которые должны привести к оптимальной производительности в среде инфраструктуры виртуализированных рабочих столов (VDI). Все параметры в этом руководстве будут *рекомендации следует учитывать* и находятся в требования не так.

В среде VDI основные способы оптимизировать производительность Windows 10 должны свести к минимуму графических приложений будет обновлен, фоновых действий, которые имеют не Главное преимущество к среде VDI, и обычно уменьшить запущенных процессов на основе компьютера. Вторичный цель — уменьшить использование дискового пространства в базовый образ на основе компьютера. С помощью VDI реализаций, наименьшее возможных базы или размер «золотого» образа, можно немного сократить использование памяти на низкоуровневой оболочки, а также небольших снижение общей сетевых операций, необходимых для предоставления образа настольной системы к потребителю.

> [!NOTE]  
> Рекомендуемые здесь параметры могут применяться к другой установке Windows 10 версии 1803, в том числе на физических или виртуальных устройств. Нет рекомендаций в этом разделе должна влиять на поддержки Windows 10 версии 1803.

> [!TIP]  
> Скрипт, который реализует оптимизации, описанные в этой статье--, а также в файл экспорта объекта групповой Политики, который можно импортировать с помощью **LGPO.exe**--доступен по [TheVDIGuys](https://github.com//TheVDIGuys) на сайте GitHub.

## <a name="vdi-optimization-principles"></a>Оптимизация принципов VDI

В среде VDI предоставляет полного настольного сеанса, включая приложения, для пользователя компьютера в сети. Среды VDI обычно используют базового образа операционной системы, которая становится основой для настольных компьютеров, которые позднее будут представлены пользователям для работы. Существуют варианты реализации VDI, например «постоянный», «временный» и «сеанса рабочего стола». Тип постоянного сохраняет изменения для VDI настольной операционной системы из одного сеанса к следующему. Непостоянный тип не сохраняет изменения в VDI рабочего стола операционную систему из одного сеанса к следующему. Для пользователя этот рабочий стол несколько отличается от других виртуальных или физических устройств, отличные от осуществляется по сети.

Параметры оптимизации будет выполняться на устройстве ссылку. Виртуальная машина — это идеальное место для создания образа, так как вы можете сохранить состояние, сделать контрольные точки и резервное копирование можно выполнить, а также ряд других полезных задач. Начните с установки операционной системы по умолчанию на базовой виртуальной Машины, а затем оптимизировать базовой виртуальной Машины для использования VDI, удалив ненужные приложения, установки обновлений Windows, установка других обновлений, удаление временных файлов, применения параметров и т. д.

Существуют другие виды VDI, такие как постоянные и удаленных столов (RDS). Подробное обсуждение относительно этих технологий выходит за рамки этого раздела, Windows, которые рассматриваются базовые параметры изображения со ссылкой на других факторов, в среде, например оптимизация узла.

### <a name="persistent-vdi"></a>Постоянные VDI

На базовом уровне, постоянные VDI является виртуальную Машину, которая сохраняет состояние операционной системы между перезапусков. Другие слои программного обеспечения, решения VDI предоставляют пользователям простой и удобный доступ к своим назначенные виртуальным машинам, часто с помощью одного решения единого входа.

Существует несколько разных реализаций постоянных VDI:

-   Традиционные виртуальной машины, где виртуальная машина имеет свой собственный файл виртуального диска, запускается обычным образом и сохраняет изменения в одном сеансе к следующему по существу просто обычная виртуальная машина. Разница заключается в том, как пользователь обращается к этой виртуальной Машины. Возможно, веб-портал, к которому подключается пользователь, автоматически направляет пользователя для их один или несколько назначенных виртуальных машин VDI.

-   Изображение постоянных виртуальную машину под управлением, с помощью личных виртуальных дисков. В этом типе реализации есть изображение базового уровня gold или один или несколько серверов узла. Создается виртуальная машина, и один или несколько виртуальных дисков создаются и назначаются на этот диск для получения постоянного хранилища.

    -   При запуске виртуальной Машины копию базового образа считывается в память виртуальной машины. В то же время постоянный виртуальный диск, назначенные этой виртуальной Машины, любые предыдущие изменения операционной системы, объединенные с помощью сложный процесс.

    -   Изменения, такие как записи в журнал событий, операций записи в журнал, и т.д., перенаправляются к виртуальному диску чтения и записи, назначенные этой виртуальной Машины.

    -   В этом случае операционная система и обслуживание приложений может работать в обычном режиме, с помощью традиционных обслуживания программного обеспечения, например Windows Server Update Services или другие технологии управления.

### <a name="non-persistent-vdi"></a>Непостоянные VDI

Непостоянные реализации VDI на основании основание или «золотого» образа, оптимизации главным образом выполняются в базовый образ, а затем через локальные параметры и локальные политики.

При использовании образа непостоянный VDI базовый образ только для чтения. При запуске непостоянный ВМ VDI копию базового образа передаются на компьютер к виртуальной Машине. Действие, которое происходит во время запуска, а затем до следующей перезагрузки перенаправляется во временную папку. Обычно пользователям предоставляются сетевые расположения для хранения своих данных. В некоторых случаях профиль пользователя объединяется со стандартной виртуальной Машине, чтобы предоставить пользователю возможность их параметры.

Одним важным аспектом VDI не persitent, основанный на один образ обслуживания. Обновления для операционной системы предоставляются обычно один раз в месяц.
С помощью VDI на основе образа есть набор процессов для выполнения для получения обновлений к образу:

-   На конкретном узле все виртуальные машины на этом узле, являются производными от базового образа необходимо завершить работу или отключена. Это означает, что пользователи будут перенаправлены к другим виртуальным машинам.

-   Базовый образ открывается и запуска. Затем, выполнить все действия по обслуживанию, например обновления операционной системы, обновлений платформы .NET, обновления приложений, и т.д.

-   В настоящее время применяются все новые параметры, которые должны применяться.

-   Любое другое обслуживание выполняется в данный момент.

-   Базовый образ, затем завершите работу.

-   Базовый образ является запечатанным и задать, чтобы вернуться в рабочую среду.

-   Пользователи могут войти в систему.

> [!NOTE]  
> Windows 10 автоматически выполняет ряд задач по обслуживанию на периодической основе. Нет запланированную задачу, которая по умолчанию имеет значение для запуска в 3:00 по местному времени каждый день. Это запланированное задание выполняет список задач, включая Очистка обновлений Windows. Вы можете просмотреть все категории обслуживания, которые выполняются автоматически с помощью следующей команды PowerShell:

`Get-ScheduledTask | ? {$_.Settings.MaintenanceSettings}`



Одной из проблем, с непостоянными VDI является, что при выходе пользователя удаляется почти все действия операционной системы. Может быть сохранен или пользователя профиль и состояние, но почти все изменения, внесенные с момента последней загрузки отбрасывает самой виртуальной машины. Таким образом оптимизации, предназначенные для компьютера Windows, который сохраняет состояние из одного сеанса к следующему не применяются.

В зависимости от архитектуры виртуальных Машин VDI перезапустите вещи, like предварительной выборки и SuperFetch не помогут в одном сеансе к другому, как все оптимизации, удаляются на виртуальной Машине. Индексирование может быть частичной тратой ресурсов, как если бы любой оптимизаций дисков, например традиционные дефрагментации.

#### <a name="managing-optional-features-with-powershell"></a>Управление дополнительных компонентов с помощью PowerShell

 Чтобы перечислить установленные компоненты Windows, выполните следующую команду PowerShell:

`Get-WindowsOptionalFeature -Online`


Можно включить или отключить определенные дополнительный компонент Windows, как в следующем примере:

`Enable-WindowsOptionalFeature -Online -FeatureName "DirectPlay" -All`

Дополнительные сведения об этом см. в разделе [Windows 10: Управление дополнительных компонентов с помощью PowerShell](https://social.technet.microsoft.com/wiki/contents/articles/39386.windows-10-managing-optional-features-with-powershell.aspx).

#### <a name="enable-or-disable-windows-features-by-using-dism"></a>Включение и отключение компонентов Windows с помощью DISM

Можно воспользоваться встроенным **Dism.exe** средство для перечисления и управления дополнительные компоненты Windows. Сценарий Dism.exe можно настроить для запуска во время последовательности задач, которая устанавливает операционную систему.

### <a name="tuning-windows-10-network-performance-by-using-registry-settings"></a>Настройка производительности сети в Windows 10 с помощью параметров реестра

Это особенно важно в средах, где имеет рабочей нагрузке, главным образом сети на основе VDI или физического компьютера. Параметры защиты производительности этот раздел предпочитать сети, путем настройки дополнительных буферизации и кэширование объектов, такие как записи каталога и т. д.

Обратите внимание, что некоторые параметры в этом разделе *системного реестра только* и должны быть включены в базовый образ, до развертывания образа для использования в рабочей среде.

#### <a name="disablebandwidththrottling"></a>DisableBandwidthThrottling

HKLM\\системы\\CurrentControlSet\\служб\\LanmanWorkstation\\параметры\\DisableBandwidthThrottling

Применяется к Windows 10. По умолчанию используется **0**. По умолчанию перенаправитель SMB регулирует пропускную способность для сетевых подключений с высокой задержкой, что позволяет в некоторых случаях избежать превышения времени ожидания, связанного с сетью. Установка этого значения реестра в **1** отключает повтор, включение более высокую пропускную способность для передачи файла через соединения с высокой задержкой сети, поэтому используйте этот параметр.

#### <a name="fileinfocacheentriesmax"></a>FileInfoCacheEntriesMax

HKLM\\системы\\CurrentControlSet\\служб\\LanmanWorkstation\\параметры\\FileInfoCacheEntriesMax

Применяется к Windows 10. По умолчанию используется **64**, находиться в диапазоне 1 к 65536. Это значение используется для определения объема метаданных файла, которые могут кэшироваться клиентом. Увеличение значения можно сократить сетевой трафик и повысить производительность при доступе к много файлов. Попробуйте увеличить это значение, чтобы **1024**.

#### <a name="directorycacheentriesmax"></a>DirectoryCacheEntriesMax

HKLM\\системы\\CurrentControlSet\\служб\\LanmanWorkstation\\параметры\\DirectoryCacheEntriesMax

Применяется к Windows 10. По умолчанию используется **16**, находиться в диапазоне от 1 к 4096. Это значение используется для определения объема сведений о каталогах, которые могут кэшироваться клиентом. Увеличение значения позволяет сократить сетевой трафик и повысить производительность, если осуществляется доступ к большому числу каталогов. Попробуйте увеличить это значение, чтобы **1024**.

#### <a name="filenotfoundcacheentriesmax"></a>FileNotFoundCacheEntriesMax

HKLM\\системы\\CurrentControlSet\\служб\\LanmanWorkstation\\параметры\\FileNotFoundCacheEntriesMax

HKLM\\системы\\CurrentControlSet\\служб\\LanmanWorkstation\\параметры\\DormantFileLimit

### <a name="disk-cleanup-including-using-the-disk-cleanup-wizard"></a>Очистка диска (в том числе с помощью мастера очистки диска)

Очистка диска может быть особенно полезным с реализациями VDI главного образа. После подготовки, обновления и настройки главного образа, одним из последней задачи, выполняемые является очистки диска. Мастер очистки диска, встроенные в Windows может помочь очистки большинство возможных областей Экономия дискового пространства.

> [!NOTE]  
> Мастер очистки диска больше не находится в стадии разработки. Windows будет использовать другие методы для обеспечения функции очистки диска.



Ниже приведены рекомендации для различных задач очистки диска. Следует проверить их перед реализацией любого из них:

1.  Запустите мастер очистки диска (с повышенными правами), после применения всех обновлений. Включать категории **оптимизации доставки** и **Очистка обновлений Windows**. Вы можете автоматизировать этот процесс с **Cleanmgr.exe** с **/sageset: 11** параметр. Этот параметр задает значения реестра, которые могут использоваться позже для автоматизации очистки диска, используя все доступные параметры в мастере очистки диска.

    1.  На тестовой виртуальной Машины, из чистую установку, под управлением **Cleanmgr.exe/sageset: 11** показывает, что только двумя способами автоматических дисков очистки включен по умолчанию:

    - Загруженные файлы программ

    - Временные файлы Интернета

    2.  Если задать дополнительные параметры, или все параметры, эти параметры записываются в реестре, в соответствии с значением индекса, предоставленные в предыдущей команде (**Cleanmgr.exe/sageset: 11**). В этом примере мы используем значение *11* как наш индекс, для процедуры очистки последующих автоматических диска.

    3.  После выполнения команды **Cleanmgr.exe/sageset: 11** вы увидите несколько категорий параметров очистки диска. Выберите каждый из вариантов, а затем выберите **ОК**. Обратите внимание, что мастер очистки диска не исчезнет. Тем не менее, выбранные параметры сохраняются в реестре и его можно вызывать, выполнив **Cleanmgr.exe /SAGERUN:11**.

2.  Очистка хранилища теневого копирования томов, если тип используется. Чтобы сделать это, выполните следующие команды в строке с повышенными привилегиями:

    - **vssadmin list shadows**

    - **Vssadmin list местонахождения**

        Если выходные данные с помощью этих команд *элементы не найдены, который удовлетворяет запрос.* , то хранилище VSS не используется.

3.  Очистить временные файлы и журналы. Из командной строки с повышенными привилегиями выполните следующие команды:

    - **DEL C:\\\*.tmp /s**

    - **DEL C:\\Windows\\Temp\\.**

    - **Del % temp %\\.**

### <a name="remove-onedrive"></a>Удалить OneDrive

Удаление OneDrive включает в себя удаление пакета, удаления служб \*lnk-файлами. Для помощи в удалении OneDrive из образа, можно использовать следующий пример кода PowerShell:

```azurecli

Taskkill.exe /F /IM "OneDrive.exe"
Taskkill.exe /F /IM "Explorer.exe"` 
    if (Test-Path "C:\\Windows\\System32\\OneDriveSetup.exe")`
     { Start-Process "C:\\Windows\\System32\\OneDriveSetup.exe"`
         -ArgumentList "/uninstall"`
         -Wait }
    if (Test-Path "C:\\Windows\\SysWOW64\\OneDriveSetup.exe")`
     { Start-Process "C:\\Windows\\SysWOW64\\OneDriveSetup.exe"`
         -ArgumentList "/uninstall"`
         -Wait }
Remove-Item -Path
"C:\\Windows\\ServiceProfiles\\LocalService\\AppData\\Roaming\\Microsoft\\Windows\\Start Menu\\Programs\\OneDrive.lnk" -Force
Remove-Item -Path "C:\\Windows\\ServiceProfiles\\NetworkService\\AppData\\Roaming\\Microsoft\\Windows\\Start Menu\\Programs\\OneDrive.lnk" -Force \# Remove the automatic start item for OneDrive from the default user profile registry hive
Start-Process C:\\Windows\\System32\\Reg.exe -ArgumentList "Load HKLM\\Temp C:\\Users\\Default\\NTUSER.DAT" -Wait
Start-Process C:\\Windows\\System32\\Reg.exe -ArgumentList "Delete HKLM\\Temp\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run /v OneDriveSetup /f" -Wait
Start-Process C:\\Windows\\System32\\Reg.exe -ArgumentList "Unload HKLM\\Temp" -Wait Start-Process -FilePath C:\\Windows\\Explorer.exe -Wait
```
