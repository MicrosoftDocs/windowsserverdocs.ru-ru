# <a name="supported-configurations-for-remote-desktop-services-in-windows-server-2016"></a>Поддерживаемые конфигурации для служб удаленных рабочих столов в Windows Server 2016

> Область применения. Windows Server 2016

Когда дело доходит до поддерживаемой конфигурации для среды служб удаленных рабочих столов, наибольшее тем, что обычно версии взаимодействия. Большинство сред включать несколько версий Windows Server — например, может существующего развертывания Windows Server 2012 R2 RDS, но необходимо обновить до Windows Server 2016, чтобы воспользоваться преимуществами новых функций (таких как поддержка OpenGL\OpenCL, дискретные Назначение устройств или дисковые пространства). Тогда встает вопрос, какие компоненты служб удаленных рабочих СТОЛОВ может работать с различными версиями и который должны совпадать?

Поэтому с учетом этого вот основные рекомендации по поддерживаемым конфигурациям служб удаленных рабочих столов в Windows Server 2016.

> [!NOTE]
> Не забудьте проверить [требования к системе для Windows Server 2016](../../get-started/system-requirements.md).

## <a name="best-practices"></a>Рекомендации
- Используйте Windows Server 2016 для инфраструктуры удаленного рабочего стола — веб-доступа, шлюза, посредника подключений и сервер лицензий. Windows Server 2016 является обратную совместимость для этих компонентов — поэтому узла сеансов удаленных рабочих Столов 2012 R2 можно подключаться к посредника подключений к удаленному рабочему Столу 2016, но не наоборот.

- Для узлов сеансов удаленных рабочих Столов — все узлы сеансов в коллекции должны быть на том же уровне, но может иметь множество коллекций. Может иметь коллекции с узлов Windows Server 2012 R2 сеанс и один с узлах сеансов Windows Server 2016.

- При обновлении вашего узла сеансов удаленных рабочих Столов на Windows Server 2016, также следует обновите сервер лицензий. Помните, что сервер лицензий 2016 может обрабатывать клиентские лицензии из всех предыдущих версиях Windows Server до Windows Server 2003.

- Выполните порядок обновления, рекомендуется в [обновлении среды служб удаленных рабочих столов](upgrade-to-rds.md#flow-for-deployment-upgrades). 

- При создании среды высокой доступности, все ваши посредников подключений должны быть на том же уровне операционной системы.

## <a name="single-sign-on-sso"></a>Единый вход (SSO)
Служба удаленных рабочих СТОЛОВ для Windows Server 2016 поддерживает два основных варианта единого входа:

 - Приложение (приложение удаленного рабочего стола в Windows, iOS, Android и Mac)
 - Веб-служба SSO
 
С помощью приложения удаленного рабочего стола, можно хранить учетные данные либо как часть сведений о соединении ([Mac](clients\remote-desktop-mac.md)) или как часть управляемых учетных записей ([iOS](clients\remote-desktop-ios.md#manage-your-user-accounts), [Android](clients\remote-desktop-android.md#manage-your-user-accounts), Windows) безопасно через механизмы, уникальные для каждой операционной системы.

Чтобы подключиться к рабочими столами и RemoteApp с единым ВХОДОМ через клиент удаленного рабочего стола папки "Входящие" на Windows, необходимо подключиться к удаленным рабочим Столам веб-страницы через обозреватель Internet Explorer. На стороне сервера требуются следующие параметры конфигурации. Другие конфигурации не поддерживаются для единого входа через Интернет:

 - RD Web значение проверки подлинности на основе форм (по умолчанию)
 - Шлюз удаленных рабочих Столов, задайте для проверки пароля (по умолчанию)
 - Значение развертывания служб удаленных рабочих СТОЛОВ «Шлюз удаленных рабочих Столов используйте учетные данные для удаленных компьютеров» (по умолчанию), в свойствах шлюза удаленных рабочих Столов

> [!NOTE]
> Из-за параметров необходимые конфигурации единого входа через Интернет не поддерживается с помощью смарт-карт. Пользователи, которым имя входа с помощью смарт-карт могут возникнуть вывода нескольких запросов для входа в систему.

Дополнительные сведения о создании VDI развертывания служб удаленных рабочих столов, ознакомьтесь с [конфигурации безопасности поддерживается Windows 10 для удаленных рабочих столов VDI служб](rds-vdi-supported-config.md).

## <a name="using-remote-desktop-services-with-application-proxy-services"></a>С помощью служб удаленных рабочих столов с помощью служб прокси-сервера приложений

Службы удаленных рабочих столов, за исключением веб-клиента, можно использовать с [прокси приложения Azure AD](https://docs.microsoft.com/azure/active-directory/application-proxy-publish-remote-desktop). Службы удаленных рабочих столов не поддерживает использование [прокси веб-приложения](https://docs.microsoft.com/windows-server/remote/remote-access/web-application-proxy/web-application-proxy-windows-server), которая включена в Windows Server 2016 и более ранних версий.
