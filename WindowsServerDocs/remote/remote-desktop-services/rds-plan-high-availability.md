# <a name="remote-desktop-services---high-availability"></a>Службы удаленных рабочих столов — высокий уровень доступности

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В крупномасштабных системах неизбежны сбоев и регулирования. Просто настроить удаленный рабочий стол ролей инфраструктуры для поддержки высокой доступности и разрешить конечным пользователям для подключения без проблем, каждый раз.

В службы удаленных рабочих столов следующие элементы представляют ролями инфраструктуры удаленного рабочего стола, их соответствующие рекомендации для обеспечения высокого уровня доступности:
- [Посредник подключений к удаленному рабочего стола](Deploy-a-Remote-Desktop-Connection-Broker-cluster.md)
- [Шлюз удаленных рабочих столов](Deploy-a-RD-Web-Access-and-Gateway-farm.md)
- Лицензирование удаленных рабочих столов
- [Доступа к удаленному рабочему столу](Deploy-a-RD-Web-Access-and-Gateway-farm.md)

Высокий уровень доступности устанавливается путем дублирования всех служб ролей на второй машинах. В Azure, вы можете получать гарантирована времени, поместив набор двух виртуальных машин (при размещении той же роли) в группе доступности задает.

Вместе с группы доступности теперь вы можете использовать возможность базы данных SQL Azure и его соглашение об уровне Azure убедитесь, что вы всегда содержат сведения о соединении и перенаправлять пользователей на своих настольных компьютеров и приложений.

Рекомендации по созданию среды служб удаленных рабочих СТОЛОВ, см. в разделе [архитектуры по размещению рабочих столов](desktop-hosting-reference-architecture.md).
