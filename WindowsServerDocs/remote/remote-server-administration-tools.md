# <a name="remote-server-administration-tools"></a>Средства удаленного администрирования сервера

>Относится к: Windows Server 2019 г., Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

В этом разделе поддерживает удаленного сервера администрирования средств для Windows 10.

> [!IMPORTANT]
> Начиная с Windows 10 октября 2018 Update, RSAT входит в состав набора **компоненты по требованию** в Windows 10, сам. См. в разделе **когда следует использовать версию RSAT** ниже инструкции по установке.

RSAT позволяет ИТ-администраторам управлять ролями Windows Server и компонентами на Компьютере с Windows 10.

Средства удаленного администрирования сервера включают диспетчера серверов, оснастки Microsoft Management Console (mmc), консоли, командлеты Windows PowerShell и поставщики и некоторые программы командной строки для управления ролями и компонентами, которые выполняются на сервере Windows.

Средства удаленного администрирования сервера включают модули командлетов Windows PowerShell, которые могут использоваться для управления ролями и функциями, выполняющимися на удаленных серверах. Несмотря на то, что удаленное управление Windows PowerShell включено по умолчанию в Windows Server 2016, не включена по умолчанию в Windows 10. Чтобы выполнять командлеты, которые являются частью средств удаленного администрирования сервера для удаленного сервера, запустите `Enable-PSremoting` в сеансе Windows PowerShell, который был открыт с повышенными правами пользователя (которые Запуск от имени администратора) на клиентском компьютере Windows, после Установка средств удаленного администрирования сервера.

## <a name="BKMK_Thresh"></a>Средства удаленного администрирования сервера для Windows 10
Используйте удаленный сервер администрирования средств для Windows 10, чтобы управлять специальными технологиями на компьютерах, работающих под управлением Windows Server 2016, Windows Server 2012 R2, и в ограниченных случаях, Windows Server 2012 или Windows Server 2008 R2.

Удаленный сервер администрирования средств для Windows 10 включает в себя поддержку удаленного управления компьютерами под управлением параметра установки основных серверных компонентов или конфигурация минимального серверного интерфейса Windows Server 2016, Windows Server 2012 R2 и ограничено случаях — варианты установки основных серверных компонентов Windows Server 2012. Тем не менее удаленного сервера администрирования средств для Windows 10 нельзя установить на любой версии операционной системы Windows Server.

### <a name="tools-available-in-this-release"></a>Служебные программы, доступные в данной версии
список инструментов, доступных в удаленный сервер администрирования средств для Windows 10, см. в таблице в [удаленного администрирования сервера средства (RSAT) для операционных систем Windows](https://support.microsoft.com/help/2693643/remote-server-administration-tools-rsat-for-windows-operating-systems).

### <a name="system-requirements"></a>Системные требования
Удаленный сервер администрирования средств для Windows 10 можно установить только на компьютерах, работающих под управлением Windows 10. Средства удаленного администрирования сервера нельзя установить на компьютерах, работающих под управлением Windows RT 8.1 или другие устройства на микросхемах.

Удаленный сервер администрирования средств для Windows 10 работает на x86 и x64-разрядных выпусков Windows 10.

> [!IMPORTANT]
> Удаленный сервер администрирования средств для Windows 10 не должен быть установлен на компьютере, использующих пакеты средств администрирования для Windows 8.1, Windows 8, Windows Server 2008 R2, Windows Server 2008, Windows Server 2003 или Windows 2000 Server. Удалите все прежние версии средства администрирования или средства удаленного администрирования сервера, включая более ранних предварительных версий и выпусков для разных языков и раскладок средств с компьютера перед установкой удаленного администрирования сервера Средства для Windows 10.

Чтобы использовать этот выпуск диспетчера серверов для доступа и управления удаленными серверами, работающих под управлением Windows Server 2012 R2, Windows Server 2012 или Windows Server 2008 R2, необходимо установить некоторые обновления, чтобы сделать старые операционные системы Windows Server управляемыми с помощью Se на сервере диспетчера. Подробные сведения о том, как подготовка Windows Server 2012 R2, Windows Server 2012 и Windows Server 2008 R2 для управления с помощью диспетчера серверов в удаленный сервер администрирования средств для Windows 10 см. в разделе [Manage Multiple, Remote Servers с помощью диспетчера серверов](https://technet.microsoft.com/library/hh831456.aspx).

Удаленное управление Windows PowerShell и диспетчера серверов необходимо включить на удаленных серверах, чтобы управлять ими с помощью средств, которые являются частью удаленного сервера администрирования средств для Windows 10. Удаленное управление включено по умолчанию на серверах, работающих под управлением Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012. Дополнительные сведения о том, как включить удаленное управление, если оно отключено, см. в разделе [Управление несколькими удаленными серверами с помощью диспетчера серверов](https://go.microsoft.com/fwlink/p/?LinkId=241358).

## <a name="install-uninstall-and-turn-offon-rsat-tools"></a>Установить, удалите, а также включить и отключить на средства удаленного администрирования сервера

### <a name="use-features-on-demand-fod-to-install-specific-rsat-tools-on-windows-10-october-2018-update-or-later"></a>Обновить использование функций по запросу (FoD), чтобы установить конкретные средства RSAT на Windows 10 октября 2018 года, или более поздней версии

Начиная с Windows 10 октября 2018 Update, RSAT входит в состав набора **компоненты по требованию** прямо из Windows 10. Теперь, вместо загрузки пакет RSAT можно просто перейти к **управление дополнительными компонентами** в **параметры** и нажмите кнопку **добавить компонент** для просмотра списка доступных средств RSAT. Выберите и установите необходимые определенного средства RSAT. Чтобы просмотреть ход выполнения установки, нажмите кнопку **обратно** кнопку, чтобы просмотреть состояние на **управление дополнительными компонентами** страницы.

См. в разделе [список средств удаленного администрирования сервера инструментальных средств, доступных через **компоненты по требованию**](https://docs.microsoft.com/windows-hardware/manufacture/desktop/features-on-demand-non-language-fod#remote-server-administration-tools-rsat). Помимо установки с помощью графических **параметры** приложение, вы также можете установить средства RSAT командной строке или с помощью автоматизации [ **DISM /Add-Capability**](https://docs.microsoft.com/windows-hardware/manufacture/desktop/features-on-demand-v2--capabilities#using-dism-add-capability-to-add-or-remove-fods).

Одним из преимуществ компоненты по требованию является то, что установленные компоненты сохраняются обновлений Windows 10 версии!

#### <a name="to-uninstall-specific-rsat-tools-on-windows-10-october-2018-update-or-later-after-installing-with-fod"></a>Для удаления конкретных RSAT инструменты отладки в Windows 10 октября 2018 с обновлением или более поздней версии (после установки с FoD)

В Windows 10 откройте **параметры** приложение, перейдите к **управление дополнительными компонентами**выберите и удалите конкретные средства RSAT, нужно удалить. Обратите внимание, что в некоторых случаях вам потребуется вручную удалить зависимости. В частности средства RSAT объект при необходимости средством RSAT B, выбрав удаление объект средства RSAT завершится ошибкой, если установлены средства RSAT B. В этом случае сначала удалить средства RSAT B, а затем удалите а RSAT средство. Обратите внимание на то, что в некоторых случаях удаление средства RSAT может отображаться для успешного выполнения, несмотря на то, что по-прежнему установлено средство. В этом случае перезапуск компьютера завершит удаления этого средства.

См. в разделе [список средств RSAT, включая зависимости](https://docs.microsoft.com/windows-hardware/manufacture/desktop/features-on-demand-non-language-fod#remote-server-administration-tools-rsat). Помимо удаления с помощью графического приложения параметры также можно удалить конкретные средства RSAT, с помощью командной строки или с помощью автоматизации [ **DISM /Remove-Capability**](https://docs.microsoft.com/windows-hardware/manufacture/desktop/features-on-demand-v2--capabilities#using-dism-add-capability-to-add-or-remove-fods).

### <a name="when-to-use-which-rsat-version"></a>Когда следует использовать версию RSAT

При наличии версии Windows 10 до октября 2018 обновление (1809), вы не сможете использовать **компоненты по требованию**. Необходимо загрузить и установить средства удаленного администрирования сервера.

- **Установить средства удаленного администрирования сервера FODs непосредственно из Windows 10, как описано выше**: При установке на Windows 10 октября 2018 обновление (1809) или более поздней версии, для управления Windows Server 2019 или более ранних версий.

- **Скачайте и установите пакет WS_1803 средства удаленного администрирования сервера, как описано ниже**: При установке на Windows 10 апреля 2018 г. обновление (1803) или более ранней версии, для управления Windows Server, версии 1803 или Windows Server версии 1709.

- **Скачайте и установите пакет WS2016 средства удаленного администрирования сервера, как описано ниже**: При установке на Windows 10 апреля 2018 г. обновление (1803) или более ранней версии, для управления Windows Server 2016 или более ранних версий.

#### <a name="BKMK_installthresh"></a>Загрузить пакет средства удаленного администрирования сервера для установки удаленного сервера администрирования средств для Windows 10

Вы можете запустить программу установки непосредственно с веб-узла Центра загрузки или сохранить загружаемый пакет на локальный компьютер или общий ресурс.

Средства удаленного администрирования сервера нельзя установить на компьютерах, работающих под управлением Windows RT 8.1 или другие устройства на микросхемах.

2.  Если вы сохранили загружаемый пакет на локальный компьютер или общий ресурс, двойным щелчком запустите программу установки **WindowsTH-KB2693643-x64.msu** или **WindowsTH-KB2693643-x86.msu**в зависимости от архитектуры компьютера, на который вы хотите установить служебные программы.

3.  Когда откроется диалоговое окно **Автономный установщик центра обновлений Windows**, нажмите кнопку **Да** для установки обновления.

5. Когда система запросит подтверждение удаления, нажмите кнопку **Да**.

### <a name="run-remote-server-administration-tools"></a>Запуск средств администрирования удаленного сервера

> [!NOTE]
> После установки удаленного сервера администрирования средств для Windows 10, **Администрирование** появляется папка **запустить** меню. Вы можете получить доступ к этим средствам из следующих расположений.
>
> -   **Средства** меню в консоли диспетчера сервера.
> -   **Панель управления\Система и безопасность\Администрирование**.
> -   Ярлык папки **Администрирование** , сохраненный на рабочем столе (для этого щелкните правой кнопкой мыши ссылку **Панель управления\Система и безопасность\Администрирование** и выберите пункт **Создать ярлык**).

Средства, установленные как часть удаленного сервера администрирования средств для Windows 10 не может использоваться для управления на локальном клиентском компьютере. Независимо от средства запуска необходимо указать удаленный сервер или несколько удаленных серверов, на котором запускается средство. Так как большинство средств объединено с помощью диспетчера серверов, добавить удаленные серверы, которые вы хотите управлять в пул серверов диспетчера серверов, прежде чем управление сервером с помощью средств в **средства** меню. Дополнительные сведения о добавлении серверов в пул серверов и о создании настраиваемых групп серверов см. в разделах [Добавление серверов в диспетчер серверов](https://go.microsoft.com/fwlink/p/?LinkId=241353) и [Создание групп серверов и управление ими](https://go.microsoft.com/fwlink/?LinkId=247328).

В средствах удаленного администрирования сервера для Windows 10, все средства администрирования сервера на основе графического пользовательского интерфейса, такие как оснастки mmc и диалоговые окна, доступны из **средства** меню консоли Server Manager. Несмотря на то, что на компьютере под управлением удаленного сервера администрирования средств для Windows 10 работает на базе клиента операционной системы, после установки этих средств, диспетчер сервера, в состав удаленного сервера администрирования средств для Windows 10, автоматически открывается по умолчанию на клиентском компьютере. Следует отметить, что не **локального сервера** страницы в консоли диспетчера сервера, который выполняется на клиентском компьютере.

##### <a name="to-start-server-manager-on-a-client-computer"></a>Чтобы запустить диспетчер серверов на клиентском компьютере:

1.  В меню **Пуск** щелкните **Все приложения**, а затем выберите **Администрирование**.

2.  В папке **Администрирование** щелкните **Диспетчер серверов**.

Несмотря на то, что они не перечислены в консоли диспетчера серверов **средства** меню, командлеты Windows PowerShell и средства администрирования командной они также установлены для ролей и компонентов в составе средств удаленного администрирования сервера. Например, если открыть сеанс Windows PowerShell с повышенными правами пользователя (Запуск от имени администратора) и выполните командлет `Get-Command -Module RDManagement`, результаты включают список командлеты удаленного рабочего стола служб, которые теперь доступны для запуска на локальном компьютере после Установка средств удаленного администрирования сервера, до тех пор, пока командлеты нацелены на удаленный сервер, на котором выполняется полностью или частично удаленного рабочего стола служб роли.

##### <a name="to-start-windows-powershell-with-elevated-user-rights-run-as-administrator"></a>Запуск Windows PowerShell с повышенными правами пользователя (запуск от имени администратора)

1.  В меню **Пуск** щелкните **Все приложения**, выберите раздел **Система Windows**и щелкните пункт **Windows PowerShell**.

2.  Чтобы запустить Windows PowerShell с правами администратора с рабочего стола, щелкните правой кнопкой мыши **Windows PowerShell** ярлык, а затем щелкните **Запуск от имени администратора**.

> [!NOTE]
> Можно также запустить сеанс Windows PowerShell, который нацелен на конкретном сервере, щелкнув правой кнопкой мыши управляемый сервер на странице роли или группы в диспетчере сервера и нажмите кнопку **Windows PowerShell**.


## <a name="known-issues"></a>Известные проблемы

### <a name="issue-rsat-fod-installation-fails-with-error-code-0x800f0954"></a>**Проблема**: Средства удаленного администрирования сервера FOD установка завершается сбоем с кодом ошибки 0x800f0954

> **Влияние**: FODs RSAT для Windows 10 1809 (обновление октября 2018 г.) в средах WSUS или SCCM
> 
> **Разрешение**: Чтобы установить FODs на ПК, присоединенных к домену, который получает обновления с помощью служб WSUS или SCCM, необходимо изменить параметр групповой политики, чтобы включить загрузку FODs непосредственно из центра обновления Windows или локальный общий ресурс. Дополнительные сведения и инструкции о том, как изменить эту настройку, см. в разделе [как доступность компонентов по требованию и языковые пакеты при работе с WSUS или SCCM](https://docs.microsoft.com/windows/deployment/update/fod-and-lang-packs).

> **Влияние**: FODs RSAT для Windows 10 1809 (октября 2018 с обновлением)
> 
> **Разрешение**: Чтобы просмотреть ход выполнения установки, нажмите кнопку **обратно** кнопку, чтобы просмотреть состояние на **управление дополнительными компонентами** страницы.

### <a name="issue-rsat-fod-uninstallation-via-settings-app-may-fail"></a>**Проблема**: Удаление FOD средства удаленного администрирования сервера с помощью параметров приложения может завершиться ошибкой

> **Влияние**: FODs RSAT для Windows 10 1809 (октября 2018 с обновлением)
> 
> **Разрешение**: В некоторых случаях удаление сбои, из-за необходимости вручную удалять зависимости. В частности средства RSAT объект при необходимости средством RSAT B, выбрав удаление объект средства RSAT завершится ошибкой, если установлены средства RSAT B. В этом случае сначала удалить средства RSAT B, а затем удалите а RSAT средство. Просмотреть список FODs средства удаленного администрирования сервера, включая зависимости.

### <a name="issue-rsat-fod-uninstallation-appears-to-succeed-but-the-tool-is-still-installed"></a>**Проблема**: Удаление RSAT FOD кажется успешно, но по-прежнему установлено средство

> **Влияние**: FODs RSAT для Windows 10 1809 (октября 2018 с обновлением)
> 
> **Разрешение**: Перезапуск компьютера завершит удаления этого средства.

### <a name="issue-rsat-missing-after-windows-10-upgrade"></a>**Проблема**: Средства удаленного администрирования сервера отсутствует, после обновления Windows 10

> **Влияние**: Все средства удаленного администрирования сервера. Установка пакета по MSU (до RSAT FODs) не автоматическую переустановку
> 
> **Разрешение**: Установка средства удаленного администрирования сервера не могут быть сохранены для обновления операционной системы из-за RSAT. В качестве пакета обновления Windows, который доставляется MSU. Установите средства удаленного администрирования сервера еще раз, после обновления Windows 10. Обратите внимание на то, что это ограничение является одной из причин, почему мы повысили FODs, начиная с Windows 10 1809. FODs средства удаленного администрирования сервера, которые устанавливаются будет сохранено в будущих обновлениях версий Windows 10.
