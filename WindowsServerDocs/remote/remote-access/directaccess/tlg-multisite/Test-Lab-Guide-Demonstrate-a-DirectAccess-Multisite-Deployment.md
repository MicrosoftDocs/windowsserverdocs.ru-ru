# <a name="test-lab-guide-demonstrate-a-directaccess-multisite-deployment"></a>Руководство по лаборатории тестирования: Демонстрация многосайтового развертывания DirectAccess

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

Удаленный доступ — это роль сервера в операционных системах Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012, позволяющая удаленным пользователям безопасный доступ к внутренним сетевым ресурсам с помощью DirectAccess или RRAS VPN. В этом руководстве содержатся пошаговые инструкции по расширению [руководство по лаборатории тестирования: Демонстрация настройки одного сервера DirectAccess в смешанной IPv4 и IPv6](https://go.microsoft.com/fwlink/p/?LinkId=237004) для демонстрации удаленного доступа в сценарий с несколькими сайтами.  
  
Развертывание удаленного доступа в сценарий с несколькими сайтами можно настроить серверы удаленного доступа в географически разнесенных местоположений. Ранее удаленные пользователи были обязаны проходить всегда подключается к корпоративной сети через определенный сервер DirectAccess. С помощью Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 и Windows 10 или Windows 8 можно настроить точки входа для каждого региона в развертывании. Каждой точки входа может быть одного сервера удаленного доступа или кластера серверов удаленного доступа. Удаленные пользователи имеют возможность подключения к любому из точки входа удаленного доступа в организации. Например если удаленный пользователь обычно подключается к точке входа удаленного доступа, расположенных в Азии, но затем переключается на командировки в Европе, клиентский компьютер автоматически подключается к ближайшей точке входа удаленного доступа.  
  
## <a name="about-this-guide"></a>Об этом руководстве  
Это руководство содержит инструкции по настройке и демонстрации удаленного доступа с использованием девяти серверов и трех клиентских компьютеров. Завершенные лаборатории тестирования многосайтового удаленного доступа имитирует интрасеть, Интернет и домашнюю сеть и демонстрирует функциональные возможности удаленного доступа в различных сценариях подключения к Интернету.  
  
> [!IMPORTANT]  
> Эта лаборатория является экспериментом, в котором используется минимальное количество компьютеров. Конфигурация, используемая в этом руководстве, приводится только для целей лаборатории тестирования, она не предназначена для использования в рабочей среде.
