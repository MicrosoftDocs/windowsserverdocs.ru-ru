# <a name="test-lab-guide-demonstrate-a-directaccess-multisite-deployment"></a>Руководство по лаборатории тестирования: демонстрация многосайтового развертывания DirectAccess

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016

Удаленный доступ — это роль сервера в операционных системах Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012, которая позволяет удаленным пользователям безопасно получать доступ к внутренним сетевым ресурсам с помощью DirectAccess или VPN RRAS. Это руководство содержит пошаговые инструкции по расширению [лабораторного руководства по тестированию: Демонстрация установки одиночного сервера DirectAccess с смешанными IPv4 и IPv6](https://go.microsoft.com/fwlink/p/?LinkId=237004) для демонстрации удаленного доступа в сценарии с несколькими сайтами.  
  
Развертывание удаленного доступа в многосайтовых сценариях позволяет настраивать серверы удаленного доступа в географически различных расположениях. Ранее удаленным пользователям требовалось всегда подключаться к корпоративной сети через конкретный сервер DirectAccess. В Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 и Windows 10 или Windows 8 можно настроить точки входа для каждого географического расположения в развертывании. Каждая точка входа может быть одним сервером удаленного доступа или кластером серверов удаленного доступа. Удаленные пользователи могут подключаться к любой точке входа удаленного доступа Организации. Например, если удаленный пользователь обычно подключается к точке входа удаленного доступа, расположенной в Азии, а затем переходит в командировку, клиентский компьютер автоматически подключается к ближайшей точке входа удаленного доступа.  
  
## <a name="about-this-guide"></a>Об этом руководстве  
Это краткое справочное по содержит инструкции по настройке и демонстрации удаленного доступа с помощью девяти серверов и трех клиентских компьютеров. Среда выполнения многосайтового тестирования с удаленным доступом имитирует интрасеть, Интернет и домашнюю сеть и демонстрирует функции удаленного доступа в различных сценариях подключения к Интернету.  
  
> [!IMPORTANT]  
> Эта лаборатория является экспериментом, в котором используется минимальное количество компьютеров. Конфигурация, используемая в этом руководстве, приводится только для целей лаборатории тестирования, она не предназначена для использования в рабочей среде.
