# <a name="step-6-install-and-configure-2-dc1"></a>Шаг 6 Установка и настройка 2 DC1

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

2-DC1 предоставляет следующие службы:  
  
-   Контроллер домена для домена corp2.corp.contoso.com доменных служб Active Directory (AD DS).  
  
-   A DNS server for the corp2.corp.contoso.com DNS domain.  
  
2-DC1 конфигурации состоит из следующих элементов:  
  
- Установите операционную систему на 2-DC1
  
- Настройка свойств TCP/IP

- Настройте 2-DC1 в качестве контроллера домена и DNS-сервера

- Предоставление разрешений групповой политики для CORP\User1
  
- Разрешить CORP2 компьютеры на получение сертификатов компьютера
  
- Принудительно выполнить репликацию между DC1 и 2-DC1
  
## <a name="install-the-operating-system-on-2-dc1"></a>Установите операционную систему на 2-DC1  
Во-первых установите Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.  
  
### <a name="to-install-the-operating-system-on-2-dc1"></a>Для установки операционной системы на 2-DC1  
  
1.  Начните установку Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.  
  
2.  Следуйте инструкциям для завершения установки, указав Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 (полная установка) и надежный пароль для учетной записи локального администратора. Войдите, используя учетную запись локального администратора.  
  
3.  Подключение к сети, которая имеет доступ к Интернету 2-DC1 и запустите Центр обновления Windows Чтобы установить последние обновления для Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012, а затем отключиться от Интернета.  
  
4.  2-DC1 подключитесь к подсети 2 Corpnet.  
  
## <a name="configure-tcpip-properties"></a>Настройка свойств TCP/IP  
Настройка протокола TCP/IP статических IP-адресов.  
  
### <a name="to-configure-tcpip-on-2-dc1"></a>Настройка TCP/IP на 2 DC1  
  
1.  В консоли диспетчера серверов щелкните **локального сервера**, а затем в **свойства** область, рядом с полем **проводное Ethernet-подключение**, щелкните ссылку.  
  
2.  На панели управления **Сетевые подключения** щелкните правой кнопкой мыши **Проводное Ethernet-подключение** и затем выберите пункт **Свойства**.  
  
3.  Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
4.  Выберите вариант **Использовать следующий IP-адрес**. В **IP-адрес**, тип **10.2.0.1**. В поле **Маска подсети**введите **255.255.255.0**. В **шлюз по умолчанию**, тип **10.2.0.254**. Нажмите кнопку **использовать следующие адреса DNS-серверов**в **предпочитаемый DNS-сервер**, тип **10.2.0.1**, а затем в **Альтернативный DNS-сервер**, тип **10.0.0.1**.  
  
5.  Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"**.  
  
6.  В **DNS-суффикс этого подключения**, тип **corp2.corp.contoso.com**, а затем нажмите кнопку **ОК** дважды.  
  
7.  Нажмите кнопку **Internet Protocol версии 6 (TCP/IPv6)**, а затем нажмите кнопку **свойства**.  
  
8.  Нажмите кнопку **использовать следующий IPv6-адрес**. В **IPv6-адрес**, тип **2001:db8:2::1**. В **длина префикса подсети**, тип **64**. В **шлюз по умолчанию**, тип **2001:db8:2::fe**. Нажмите кнопку **использовать следующие адреса DNS-серверов**в **предпочитаемый DNS-сервер**, тип **2001:db8:2::1**, а затем в **Альтернативный DNS-сервер**, Тип **2001:db8:1::1**.  
  
9. Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"**.  
  
10. В **DNS-суффикс этого подключения**, тип **corp2.corp.contoso.com**, а затем нажмите кнопку **ОК** дважды.  
  
11. На **свойства проводного Ethernet-подключения** диалоговом окне щелкните **закрыть**.  
  
12. Закройте окно **Сетевые подключения** .  
  
13. В консоли диспетчера серверов в **локального сервера**в **свойства** область, рядом с полем **имя_компьютера**, щелкните ссылку.  
  
14. В диалоговом окне **"Свойства системы"** на вкладке **"Имя компьютера"** щелкните **"Изменить"**.  
  
15. На **изменение имени компьютера или домена** отображаемое в диалоговом окне **имя_компьютера**, тип **2-DC1**, а затем нажмите кнопку **ОК**.  
  
16. При появлении запроса на перезагрузку компьютера нажмите кнопку **ОК**.  
  
17. В диалоговом окне **Свойства системы** нажмите **Закрыть**.  
  
18. При появлении запроса на перезагрузку компьютера нажмите кнопку **Перезагрузить сейчас**.  
  
19. После перезапуска, выполните вход с помощью учетной записи локального администратора.  
  
## <a name="configure-2-dc1-as-a-domain-controller-and-dns-server"></a>Настройте 2-DC1 в качестве контроллера домена и DNS-сервера  
Настройте 2 DC1 как контроллер домена для домена corp2.corp.contoso.com и в качестве DNS-сервер для домена DNS corp2.corp.contoso.com.  
  
### <a name="to-configure-2-dc1-as-a-domain-controller-and-dns-server"></a>Настройка 2 DC1 в качестве контроллера домена и DNS-сервера  
  
1.  В консоли диспетчера серверов на **панели мониторинга**, нажмите кнопку **Добавить роли и компоненты**.  
  
2.  Нажмите кнопку **Далее** три раза, чтобы получить экран выбора ролей сервера  
  
3.  На **Выбор ролей сервера** выберите **доменных служб Active Directory**. Нажмите кнопку **добавить компоненты** при появлении запроса, а затем нажмите кнопку **Далее** три раза.  
  
4.  На странице **Подтверждение выбранных элементов для установки** нажмите кнопку **Установить**.  
  
5.  По завершении установки нажмите кнопку **повысить уровень этого сервера до контроллера домена**.  
  
6.  В Active Directory мастер настройки доменных служб на **конфигурация развертывания** щелкните **добавить новый домен в существующий лес**.  
  
7.  В **имя родительского домена**, тип **corp.contoso.com**в **новое имя домена**, тип **corp2**.  
  
8.  В разделе **передать учетные данные для этой операции**, нажмите кнопку **изменение**. На **безопасности Windows** отображаемое в диалоговом окне **имя пользователя**, тип **corp.contoso.com\Administrator**, а затем в **пароль**, введите corp\ Пароль администратора, нажмите кнопку **ОК**, а затем нажмите кнопку **Далее**.  
  
9. На **параметры контроллера домена** странице, убедитесь, что **имя_сайта** — **второго сайта**. В разделе **введите пароль режима восстановления служб каталогов (DSRM)** в **пароль** и **подтверждение пароля**дважды введите надежный пароль и нажмите кнопку  **Далее** пять раз.  
  
10. На **проверка предварительных требований** странице, после проверки необходимых компонентов нажмите кнопку **установить**.  
  
11. Подождите завершения работы мастера конфигурации служб Active Directory и DNS и нажмите кнопку **закрыть**.  
  
12. После перезагрузки компьютера войдите в домен CORP2, с помощью учетной записи администратора.  
  
## <a name="provide-group-policy-permissions-to-corpuser1"></a>Предоставление разрешений групповой политики для CORP\User1  
Эта процедура позволяет предоставить пользователю CORP\User1 полный набор разрешений на создание и изменение объектов групповой политики corp2.  
  
### <a name="to-provide-group-policy-permissions"></a>Чтобы предоставить разрешения групповой политики  
  
1.  На **запустить** введите**gpmc.msc**, и нажмите клавишу ВВОД.  
  
2.  В консоли управления групповыми политиками откройте **леса: corp.contoso.com/Domains/corp2.corp.contoso.com**.  
  
3.  В области сведений щелкните **делегирования** вкладки. В **разрешение** стрелку раскрывающегося списка, щелкните **связанные объекты GPO**.  
  
4.  Нажмите кнопку **добавить**и на новом **Выбор пользователя, компьютер или группу** диалоговом окне щелкните **расположения**.  
  
5.  На **расположения** отображаемое в диалоговом окне **расположение** дерева, щелкните **corp.contoso.com**и нажмите кнопку **ОК**.  
  
6.  В **введите имена выбираемых объектов** тип **User1**, нажмите кнопку **ОК**, а затем на **добавить группу или пользователя** диалоговом окне нажмите кнопку **ОК** .  
  
7.  В консоли управления групповыми политиками, в дереве выберите **объектов групповой политики**, и в области сведений щелкните панель **делегирования** вкладки.  
  
8.  Нажмите кнопку **добавить**и на новом **Выбор пользователя, компьютер или группу** диалоговом окне щелкните **расположения**.  
  
9. На **расположения** отображаемое в диалоговом окне **расположение** дерева, щелкните **corp.contoso.com**и нажмите кнопку **ОК**.  
  
10. В **введите имена выбираемых объектов** тип **User1**, нажмите кнопку **ОК**.  
  
11. В консоли управления групповыми политиками, в дереве выберите **фильтры WMI**, и в области сведений щелкните панель **делегирования** вкладки.  
  
12. Нажмите кнопку **добавить**и на новом **Выбор пользователя, компьютер или группу** диалоговом окне щелкните **расположения**.  
  
13. На **расположения** отображаемое в диалоговом окне **расположение** дерева, щелкните **corp.contoso.com**и нажмите кнопку **ОК**.  
  
14. В **введите имена выбираемых объектов** тип **User1**, нажмите кнопку **ОК**. На **добавить группу или пользователя** диалогового окна поле, убедитесь, что **разрешения** присваивается **полный доступ**и нажмите кнопку **ОК**.  
  
15. Закройте консоль управления групповыми политиками.  
  
## <a name="allow-corp2-computers-to-obtain-computer-certificates"></a>Разрешить CORP2 компьютеры на получение сертификатов компьютера 

Компьютеры в домене CORP2 необходимо получить сертификаты из центра сертификации на APP1. Выполните эту процедуру на APP1.  
  
### <a name="to-allow-corp2-computers-to-automatically-obtain-computer-certificates"></a>Чтобы разрешить компьютерам CORP2 автоматического получения сертификатов компьютера  
  
1.  На компьютере APP1, нажмите кнопку **запустить**, тип **certtmpl.msc**, и нажмите клавишу ВВОД.  
  
2.  В **консоль шаблонов сертификатов**, в средней области дважды щелкните **проверки подлинности между клиентом и сервером**.  
  
3.  На **свойства проверки подлинности между клиентом и сервером** диалоговом окне щелкните **безопасности** вкладки.  
  
4.  Нажмите кнопку **добавить**, а затем на **Выбор пользователей, компьютеров, учетных записей служб или групп** диалоговом окне щелкните **расположения**.  
  
5.  На **расположения** отображаемое в диалоговом окне **расположение**, разверните **corp.contoso.com**, нажмите кнопку **corp2.corp.contoso.com**, а затем нажмите кнопку  **ОК**.  
  
6.  В **введите имена выбираемых объектов**, тип **"Администраторы домена"; Компьютеры домена** и нажмите кнопку **ОК**.  
  
7.  На **свойства проверки подлинности между клиентом и сервером** отображаемое в диалоговом окне **имена групп или пользователей**, нажмите кнопку **(CORP2\Domain "Администраторы") "Администраторы домена"** и в  **Разрешения для "Администраторы домена"** в **Разрешить** столбец, выберите **записи** и **регистрация**.  
  
8.  В **имена групп или пользователей**, нажмите кнопку **домена компьютеров (CORP2\Domain)**, а затем в **разрешения для компьютеров домена**в **Разрешить**столбец, выберите **регистрация** и **автоматическая подача заявок**, а затем нажмите кнопку **ОК**.  
  
9. Закройте консоль шаблонов сертификатов.  
  
## <a name="replication"></a>Принудительно выполнить репликацию между DC1 и 2-DC1  
Перед регистрацией сертификатов для 2 EDGE1, необходимо принудительно реплицировать параметры из DC1 к 2-DC1. Эта операция должна выполняться на DC1.  
  
### <a name="to-force-replication"></a>Для принудительной репликации  
  
1.  На DC1 щелкните **запустить**, а затем нажмите кнопку **Active Directory — сайты и службы**.  
  
2.  В консоли "Active Directory — сайты и службы" в дереве разверните **межузловыми транспортами**, а затем нажмите кнопку **IP-адрес**.  
  
3.  В области сведений дважды щелкните **DEFAULTIPSITELINK**.  
  
4.  На **свойства DEFAULTIPSITELINK** отображаемое в диалоговом окне **стоимость**, тип **1**в **Реплицировать каждые**, тип **15**, а затем нажмите кнопку **ОК**. Подождите 15 минут для выполнения репликации.  
  
5.  Для принудительной репликации теперь в дереве консоли, разверните **параметры папке Sites\Default-первый-Site-name\Servers\DC1\NTDS**, в области сведений щелкните правой кнопкой мыши **<automatically generated>**, нажмите кнопку  **Теперь репликация**, а затем на **Реплицировать сейчас** диалоговом окне нажмите кнопку **ОК**.  
  
6.  Для обеспечения успешного завершения репликации выполните следующие действия:  
  
    1.  На **запустить** введите**cmd.exe**, и нажмите клавишу ВВОД.  
  
    2.  Введите следующую команду и нажмите клавишу ВВОД.  
  
        ```  
        repadmin /syncall /e /A /P /d /q  
        ```  
  
    3.  Убедитесь, что все секции будут синхронизированы без ошибок. Если нет, затем повторно выполните команду, пока ошибки не обнаружены, прежде чем продолжить.  
  
7.  Закройте окно командной строки.
