# <a name="step-10-install-and-configure-2-edge1"></a>Шаг 10. Установка и настройка 2-EDGE1

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016

2 — EDGE1 конфигурация состоит из следующих компонентов:  
  
- Установите операционную систему на 2-EDGE1. Установите Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 на 2-EDGE1.  
  
- настроить свойства TCP/IP; Настройте 2-EDGE1 со статическими адресами в обоих сетевых интерфейсах.  
  
- Настройте маршрутизацию между подсетями. Чтобы обеспечить обмен данными между корпоративной и многокорпоративной сетями, необходимо настроить маршрутизацию.  
  
- Присоединение 2 — EDGE1 к домену CORP2. Присоединение 2 — EDGE1 к домену corp2.corp.contoso.com.  
  
- Получите сертификаты на 2-EDGE1. Сертификаты необходимы для подключения IPsec между клиентами DirectAccess и сервером удаленного доступа, а также для проверки подлинности прослушивателя IP-HTTPS при подключении клиентов по протоколу HTTPS.  
  
- Предоставление доступа к CORP\User1. Пользователь CORP\User1 является администратором удаленного доступа. Чтобы разрешить этому пользователю вносить изменения в 2-EDGE1 из EDGE1, необходимо предоставить доступ пользователю.  
  
- Установите роль удаленного доступа на 2-EDGE1. Чтобы включить многосайтовое развертывание, необходимо установить роль удаленного доступа на 2-EDGE1.  
  
2-EDGE1 должны быть установлены два сетевых адаптера.  
  
## <a name="install-the-operating-system-on-2-edge1"></a><a name="installOS"></a>Установка операционной системы на 2-EDGE1  
  
1.  Запустите установку Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.  
  
2.  Следуйте инструкциям по завершению установки, указав Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 (полная установка) и надежный пароль для учетной записи локального администратора. Войдите в систему с помощью учетной записи локального администратора.  
  
3.  Подключите 2 — EDGE1 к сети с доступом к Интернету и запустите Центр обновления Windows, чтобы установить последние обновления для Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012, а затем отключитесь от Интернета.  
  
4.  Подключите один сетевой адаптер к подсети с 2 сетями, а другой — к имитируемому Интернету.  
  
## <a name="configure-tcpip-properties"></a><a name="tcpip"></a>Настройка свойств TCP/IP  
  
1.  В консоли диспетчер сервера щелкните **локальный сервер**, а затем в области **свойства** рядом с **проводным Ethernet подключением**щелкните ссылку.  
  
2.  В окне " **Сетевые подключения**" щелкните правой кнопкой мыши сетевое подключение, подключенное к подсети с 2 сетями, выберите команду **Переименовать**, введите **2-** Subnet и нажмите клавишу ВВОД.  
  
3.  Щелкните правой кнопкой мыши **2-** Корпоративная сеть и выберите пункт **свойства**.  
  
4.  Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
5.  Выберите вариант **Использовать следующий IP-адрес**. В списке **IP-адрес**введите **10.2.0.20**в **маске подсети**, введите **255.255.255.0**.  
  
6.  Выберите параметр **Использовать следующие адреса DNS-серверов**. В качестве **предпочитаемого DNS-сервера**введите **10.2.0.1**, а в **качестве альтернативного DNS-сервера**введите **10.0.0.1**.  
  
7.  Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"** .  
  
8.  В **качестве DNS-суффикса для этого подключения**введите **corp2.Corp.contoso.com**, а затем дважды нажмите кнопку **ОК** .  
  
9. Выберите **Протокол IP версии 6 (TCP/IPv6)** , а затем **Свойства**.  
  
10. Щелкните **использовать следующий IPv6-адрес**. В качестве **адреса IPv6**введите **2001: db8:2:: 20**, в качестве **длины префикса подсети**введите **64**. Щелкните **использовать следующие адреса DNS-серверов**и в поле **Предпочитаемый DNS-сервер**введите **2001: db8:2:: 1**в **альтернативном DNS-сервере**введите **2001: db8:1:: 1**.  
  
11. Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"** .  
  
12. В **качестве DNS-суффикса для этого подключения**введите **corp2.Corp.contoso.com**, а затем дважды нажмите кнопку **ОК** .  
  
13. В диалоговом окне **свойства 2 корпоративной** сети нажмите кнопку **Закрыть**.  
  
14. В окне " **Сетевые подключения** " щелкните правой кнопкой мыши сетевое подключение, подключенное к подсети Интернета, выберите команду **Переименовать**, введите **Интернет**и нажмите клавишу ВВОД.  
  
15. Правой кнопкой мыши щелкните **"Интернет"** и выберите в контекстном меню команду **"Свойства"** .  
  
16. Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
17. Выберите вариант **Использовать следующий IP-адрес**. В списке **IP-адрес**введите **131.107.0.20**. В поле **Маска подсети** введите **255.255.255.0**.  
  
18. Нажмите кнопку **Дополнительно**. На вкладке **"Параметры IP"** в области **"IP-адреса"** щелкните **"Добавить"** . В диалоговом окне **адрес TCP/IP** в поле **IP-адрес** введите **131.107.0.21**в качестве **маски подсети** введите **255.255.255.0**и нажмите кнопку **Добавить**.  
  
19. Перейдите на вкладку **DNS**.  
  
20. В **качестве DNS-суффикса для этого подключения**введите **ISP.example.com**, дважды нажмите кнопку **ОК** , а затем нажмите кнопку **Закрыть**.  
  
21. Закройте окно **Сетевые подключения**.  
  
## <a name="configure-routing-between-subnets"></a><a name="routing"></a>Настройка маршрутизации между подсетями  
  
1.  На **начальном** экране введите**cmd. exe**и нажмите клавишу ВВОД.  
  
2.  В окне командной строки введите следующие команды. После ввода каждой команды нажмите клавишу ВВОД.  
  
    ```  
    netsh interface IPv4 add route 10.0.0.0/24 2-Corpnet 10.2.0.254  
    netsh interface IPv6 add route 2001:db8:1::/64 2-Corpnet 2001:db8:2::fe  
    ```  
  
3.  Чтобы проверить сетевое взаимодействие между 2-EDGE1 и DC1, введите **ping dc1.Corp.contoso.com**.  
  
4.  Убедитесь в наличии четырех ответов с IPv4-адреса, 10.0.0.1 или адреса IPv6, 2001: db8:1:: 1.  
  
5.  Закройте окно командной строки.  
  
## <a name="join-2-edge1-to-the-corp2-domain"></a><a name="Join"></a>Присоединение 2 — EDGE1 к домену CORP2  
  
1.  В консоли диспетчер сервера в области **Свойства** на **локальном сервере**щелкните ссылку рядом с полем **имя компьютера**.  
  
2.  В диалоговом окне **Свойства системы** на вкладке  **Имя компьютера** щелкните **Изменить**.  
  
3.  В диалоговом окне **изменения имени компьютера или домена** в поле **имя компьютера**введите **2-EDGE1**. В списке **член**выберите **домен**, введите **corp2.Corp.contoso.com**и нажмите кнопку **ОК**.  
  
4.  Если появится запрос на ввод имени пользователя и пароля, введите имя **администратора** и пароль, а затем нажмите кнопку **ОК**.  
  
5.  Когда появится диалоговое окно, дружелюбны вас в домен corp2.corp.contoso.com, нажмите кнопку **ОК**.  
  
6.  При появлении запроса на перезагрузку компьютера нажмите кнопку **ОК**.  
  
7.  В диалоговом окне **Свойства системы** нажмите **Закрыть**.  
  
8.  При появлении запроса на перезагрузку компьютера нажмите кнопку **Перезагрузить сейчас**.  
  
9. После перезагрузки компьютера щелкните **Сменить пользователя**, а затем выберите **другой пользователь** и войдите в домен CORP2 с учетной записью администратора.  
  
## <a name="obtain-certificates-on-2-edge1"></a><a name="certs"></a>Получение сертификатов на 2-EDGE1  
  
1.  На **начальном** экране введите**MMC. exe**и нажмите клавишу ВВОД.  
  
2.  В консоли MMC в меню **Файл** выберите пункт **Добавить или удалить оснастку**.  
  
3.  В диалоговом окне **Добавить или удалить оснастку** щелкните **Сертификаты**, после чего нажмите **Добавить**, выберите **Учетная запись компьютера**, щелкните **Далее**, нажмите **Локальный компьютер**, щелкните **Готово**, а затем — **OK**.  
  
4.  В дереве консоли оснастки «Сертификаты» откройте окно **Сертификаты (локальный компьютер) \персонал**.  
  
5.  Щелкните правой кнопкой мыши **личное**, укажите **все задачи**, а затем выберите **запросить новый сертификат**.  
  
6.  Щелкните **Далее** дважды.  
  
7.  На странице **запрос сертификатов** установите флажки **Проверка подлинности клиент-сервер** и **веб-сервер** , а затем щелкните **Дополнительные сведения, чтобы зарегистрировать этот сертификат**.  
  
8.  В диалоговом окне **Свойства сертификата** на вкладке **Тема** в области **имя субъекта** в поле **тип**выберите **Общее имя**.  
  
9. В списке **значение**введите **2-EDGE1.contoso.com**и нажмите кнопку **добавить**.  
  
10. В области **Альтернативное имя** в поле **Тип** выберите **DNS**.  
  
11. В списке **значение**введите **2-EDGE1.contoso.com**и нажмите кнопку **добавить**.  
  
12. На вкладке **Общие** в поле **понятное имя**введите **IP-HTTPS Certificate**.  
  
13. Нажмите кнопку **OK**, щелкните **Зарегистрировать** и нажмите кнопку **Готово**.  
  
14. В области сведений оснастки "сертификаты" убедитесь, что новый сертификат с именем 2-edge1.contoso.com был зарегистрирован в целях проверки подлинности сервера, а новый сертификат с именем 2-edge1.corp2.corp.contoso.com был зарегистрирован в Предназначено для аутентификации клиента и проверки подлинности сервера.  
  
15. Закройте окно консоли. Если будет предложено сохранить параметры, нажмите кнопку **нет**.  
  
## <a name="provide-access-to-corpuser1"></a><a name="Access"></a>Предоставление доступа к CORP\User1  
  
1.  На **начальном** экране введите**compmgmt. msc**и нажмите клавишу ВВОД.  
  
2.  В левой области щелкните **Локальные пользователи и группы**.  
  
3.  Дважды щелкните **группы**, а затем дважды щелкните **Администраторы**.  
  
4.  В диалоговом окне **Свойства администратора** нажмите кнопку **Добавить**, а затем в диалоговом окне **Выбор пользователей, компьютеров, учетных записей служб или групп** нажмите кнопку **расположения**.  
  
5.  В диалоговом окне **расположения** в дереве **Расположение** щелкните **Corp.contoso.com**, а затем нажмите кнопку **ОК**.  
  
6.  В поле **Введите имена объектов для выбора** введите **User1**и нажмите кнопку **ОК**.  
  
7.  В диалоговом окне **Свойства администратора** нажмите кнопку **ОК**.  
  
8.  Закройте окно Управление компьютером.  
  
## <a name="install-the-remote-access-role-on-2-edge1"></a><a name="InstallDA"></a>Установка роли удаленного доступа на 2-EDGE1  
  
1.  В консоли диспетчер сервера на **панели мониторинга**щелкните **Добавить роли и компоненты**.  
  
2.  Нажмите кнопку **Далее** трижды, чтобы перейти на страницу выбора роли сервера.  
  
3.  В диалоговом окне **Выбор ролей сервера** выберите **Удаленный доступ**, щелкните **Добавить компоненты**, а затем нажмите кнопку **Далее**.  
  
4.  Нажмите кнопку **Далее** пять раз.  
  
5.  В диалоге **Подтверждение установки выбранных компонентов** щелкните **Установить**.  
  
6.  В диалоге **Ход выполнения установки**, проверьте, что установка прошла успешно, затем щелкните **Закрыть**.
