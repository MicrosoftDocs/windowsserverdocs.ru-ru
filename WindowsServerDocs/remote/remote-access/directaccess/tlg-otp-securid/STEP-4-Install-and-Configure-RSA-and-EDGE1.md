# <a name="step-4-install-and-configure-rsa-and-edge1"></a>Шаг 4 Установка и настройка RSA и EDGE1

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

RSA — это сервер, RADIUS и OTP и устанавливается перед настройкой RADIUS и OTP.  
  
Будет выполните следующие действия, чтобы настроить развертывание RSA.  
  
1. Установка операционной системы на сервере RSA. Установите Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 на сервере RSA.  
  
2. Настройте параметры TCP/IP на RSA. Настройте параметры TCP/IP на сервере RSA.  
  
3. Скопируйте файлы установки диспетчер проверки подлинности на сервере RSA. После установки операционной системы на RSA, скопируйте файлы диспетчер проверки подлинности RSA компьютер.  
  
4. Присоедините сервер RSA в домен CORP. RSA присоединения к домену CORP.  
  
5. Отключение брандмауэра Windows на RSA. Отключите брандмауэр Windows на сервере RSA.  
  
6. Установите диспетчер проверки подлинности RSA на сервере RSA. Установите диспетчер проверки подлинности RSA.  
  
7. Настройте диспетчер проверки подлинности RSA. Настройте диспетчер проверки подлинности.  
  
8. Создайте DAProbeUser. Создайте учетную запись пользователя для целей проверки.  
  
9. Установите маркер RSA SecurID программного обеспечения на компьютере CLIENT1. Установите маркер RSA SecurID программного обеспечения на компьютере CLIENT1.  
  
10. Настройте EDGE1 как агент аутентификации RSA. Настройте агент проверки подлинности RSA EDGE1.  
  
11. Настройка EDGE1 для поддержки проверки подлинности OTP. Настройка OTP для DirectAccess и проверьте конфигурацию.  
  
## <a name="InstallOS"></a>Установка операционной системы на сервере RSA  
  
1.  В RSA начните установку Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.  
  
2.  Следуйте инструкциям для завершения установки, указав Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 (полная установка) и надежный пароль для учетной записи локального администратора. Войдите, используя учетную запись локального администратора.  
  
3.  Подключение к сети, которая имеет доступ к Интернету RSA и запустите Центр обновления Windows Чтобы установить последние обновления для Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012, а затем отключиться от Интернета.  
  
4.  RSA подключиться к подсети Corpnet.  
  
## <a name="TCP"></a>Настройка протокола TCP/IP на RSA  
  
1.  Задачи начальной настройки, нажмите кнопку **Настройка сети**.  
  
2.  В **сетевых подключений**, щелкните правой кнопкой мыши **подключение по локальной сети**, а затем нажмите кнопку **свойства**.  
  
3.  Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
4.  Выберите вариант **Использовать следующий IP-адрес**. В поле **IP-адрес**введите **10.0.0.5**. В поле **Маска подсети**введите **255.255.255.0**. В **шлюза по умолчанию**, тип **10.0.0.2**. Нажмите кнопку **использовать следующие адреса DNS-серверов**в **предпочитаемый DNS-сервер**, тип **10.0.0.1**.  
  
5.  Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"**.  
  
6.  В **DNS-суффикс этого подключения**, тип **corp.contoso.com**, а затем нажмите кнопку **ОК** дважды.  
  
7.  На **свойства подключения по локальной сети** диалоговом окне щелкните **закрыть**.  
  
8.  Закройте окно **Сетевые подключения** .  
  
## <a name="copyinstfiles"></a>Скопируйте файлы установки диспетчер проверки подлинности RSA сервера  
  
1.  Создайте папку C:\RSA установки на сервере RSA.  
  
2.  Скопируйте содержимое носителя SP4 7.1 диспетчер проверки подлинности RSA в папку установки C:\RSA.  
  
3.  Создайте вложенную папку C:\RSA Installation\License и маркер.  
  
4.  Скопируйте файлы лицензий RSA, чтобы C:\RSA Installation\License и маркеров.  
  
## <a name="JoinDomain"></a>Присоедините RSA сервера к домену CORP  
  
1.  Щелкните правой кнопкой мыши **Мой компьютер**и нажмите кнопку **свойства**.  
  
2.  В диалоговом окне **Свойства системы** на вкладке **Имя компьютера** щелкните **Изменить**.  
  
3.  В **имя_компьютера**, тип **RSA**. В **членом**, нажмите кнопку **домена**, тип **corp.contoso.com**и нажмите кнопку **ОК**.  
  
4.  При появлении запроса имени пользователя и пароля введите **User1** и его пароль и нажмите кнопку **ОК**.  
  
5.  Диалоговое окно приветствия домена щелкните **ОК**.  
  
6.  При появлении запроса на перезагрузку компьютера нажмите кнопку **ОК**.  
  
7.  В диалоговом окне **Свойства системы** нажмите **Закрыть**.  
  
8.  При появлении запроса на перезагрузку компьютера нажмите кнопку **Перезагрузить сейчас**.  
  
9. После перезагрузки компьютера, введите **User1** и пароль, выберите CORP в **входят в систему:** раскрывающемся списке и нажмите кнопку **ОК**.  
  
## <a name="BKMK_Firewall"></a>Отключение брандмауэра Windows на RSA  
  
1.  Нажмите кнопку **запустить**, нажмите кнопку **панели управления**, нажмите кнопку **система и безопасность**и нажмите кнопку **брандмауэра Windows**.  
  
2.  Нажмите кнопку **Включение и отключение брандмауэра Windows**.  
  
3.  **Отключите брандмауэр Windows** для всех параметров.  
  
4.  Нажмите кнопку **ОК** и закройте брандмауэр Windows.

2.  Откройте папку установки C:\RSA и дважды щелкните **autorun.exe**.  
  
3.  Нажмите кнопку **установить сейчас**, нажмите кнопку **Далее**, выберите параметр top для Америке и нажмите кнопку **Далее**.  
  
4.  Выберите **я принимаю условия лицензионного соглашения**и нажмите кнопку **Далее**.  
  
5.  Выберите **экземпляра сервера-источника**и нажмите кнопку **Далее**.  
  
6.  В **имя каталога:** тип поля **C:\RSA**и нажмите кнопку **Далее**.  
  
7.  Проверьте правильность имени сервера (RSA.corp.contoso.com) и IP-адрес и нажмите кнопку **Далее**.  
  
8.  Перейдите к C:\RSA Installation\License и токен и нажмите кнопку **Далее**.  
  
9. На **файл лицензии проверьте** щелкните **Далее**.  
  
10. В **идентификатор пользователя** тип поля **администратора**, а затем в **пароль** и **подтверждение пароля** поля введите надежный пароль. Нажмите кнопку **Далее**.  
  
11. На экране выбора журнала, примите значения по умолчанию и нажмите кнопку **Далее**.  
  
12. В окне сводки нажмите кнопку **установить**.  
  
13. После завершения установки нажмите кнопку **Готово**.  
  
## <a name="confiauthmgr"></a>Настроить диспетчер проверки подлинности RSA  
  
1.  Если консоль безопасности RSA не открывается автоматически, на рабочем столе компьютера RSA дважды щелкните «Консоли RSA Security».  
  
2.  Если безопасность сертификатов предупреждение, и появляется предупреждение системы безопасности, нажмите кнопку **Продолжить открытие этого веб-сайта** или нажмите кнопку **Да** чтобы продолжить и добавить этот сайт в список надежных сайтов, при запросе.  
  
3.  В **идентификатор пользователя** тип поля **администратора** и нажмите кнопку **ОК**.  
  
4.  В **пароль** введите пароль для учетной записи администратора и нажмите кнопку **вход в систему**.  
  
5.  Вставьте данные токена.  
  
    1.  В **консоли RSA Security** щелкните **проверки подлинности** и нажмите кнопку **маркеры SecurID**.  
  
    2.  Нажмите кнопку **задания импорта маркеры**, а затем нажмите кнопку **Add New**.  
  
    3.  В **параметры импорта** щелкните **Обзор**. Найдите и выберите XML-файле маркеры в C:\ RSA Installation\License и токен папку и нажмите кнопку **откройте**.  
  
    4.  Нажмите кнопку **отправить задание** в нижней части страницы.  
  
6.  Создание нового пользователя OTP.  
  
    1.  В **консоли RSA Security** щелкните **удостоверений** щелкните **пользователей**и нажмите кнопку **Add New**.  
  
    2.  В **Last Name:** тип секции **пользователя**, а затем в **идентификатор пользователя:** тип секции **User1** (идентификатор пользователя должен быть таким же, как имя пользователя AD, используемое для Это занятие).  В **пароль:** и **подтверждение пароля:** разделов введите надежный пароль. Очистить **«Пользователь должен сменить пароль при следующем входе в систему»** флажок и нажмите кнопку **Сохранить**.  
  
7.  Назначьте User1 один из импортированных маркеров.  
  
    1.  На **пользователей** странице щелкните **User1** и нажмите кнопку **маркеры SecurID**.  
  
    2.  Нажмите кнопку **маркеры SecurID** и нажмите кнопку **назначить маркера**.  
  
    3.  В разделе **серийный номер** заголовок щелкните первое число в списке и нажмите кнопку **назначить**.  
  
    4.  Щелкните маркер назначенные и нажмите кнопку **изменить**. В **SecurID ПИН-код управления** раздел **требования проверки подлинности пользователя**выберите **не требовать ПИН-код (только такие)**.  
  
    5.  Нажмите кнопку **сохранять и распространять токен**.  
  
    6.  На **маркера распространения программного обеспечения** странице в **основы** щелкните **проблема маркера файл (SDTID)**.  
  
    7.  На **маркера распространения программного обеспечения** странице в **маркеров параметров файла** снимите **Включение защиты от копирования** "флажок". Нажмите кнопку **без пароля** и **Далее**.  
  
    8.  На **маркера распространения программного обеспечения** странице в **загрузить файл** щелкните **Download Now**. Нажмите кнопку **Сохранить**. Перейдите к C:\RSA установки и нажмите кнопку **Сохранить** и **закрыть**.  
  
    9. Свести к минимуму **консоли RSA Security** для последующего использования.  
  
8.  Настройте диспетчер проверки подлинности как сервер RADIUS.  
  
    1.  По двойному щелчку рабочего стола компьютера RSA **«Консоли управления безопасности RSA»**.  
  
    2.  Если безопасность сертификатов предупреждение, и появляется предупреждение системы безопасности, нажмите кнопку **Продолжить открытие этого веб-сайта** или нажмите кнопку **Да** чтобы продолжить и добавить этот сайт в список надежных сайтов, при запросе.  
  
    3.  Введите идентификатор пользователя и пароль и нажмите кнопку **вход в систему**.  
  
    4.  Нажмите кнопку **конфигурации развертывания — RADIUS - настроить сервер**.  
  
    5.  На **требуются дополнительные учетные данные** странице введите имя администратора идентификатор пользователя и пароль и нажмите кнопку **ОК**.  
  
    6.  На **Настройка сервера RADIUS** странице введите тот же пароль, используемый для администратора для **секреты** и **основной пароль**. Введите идентификатор пользователя администратора и пароль и нажмите кнопку **Настройка**.  
  
    7.  Убедитесь, что сообщение **«успешно настроен сервер RADIUS»** отображается. Нажмите кнопку **Готово**. Закрыть **консоли операций RSA**.  
  
    8.  Вернитесь в **«Консоли RSA Security»**.  
  
    9. На **RADIUS** вкладке **серверы RADIUS**. Убедитесь, что указан, rsa.corp.contoso.com.  
  
9. Настройка сервера RSA в качестве клиент проверки подлинности RSA.  
  
    1.  На **RADIUS** щелкните **RADIUS-клиенты** и **Add New**.  
  
    2.  Нажмите кнопку **любой клиент RADIUS** "флажок".  
  
    3.  Введите надежный пароль по своему усмотрению **общий секрет** поля. Этот пароль будет использоваться позднее при настройке EDGE1 для OTP.  
  
    4.  Оставьте **IP-адрес** пустым и **сделать / модели** записи в виде **Standard RADIUS**.  
  
    5.  Нажмите кнопку **сохранить без агента RSA**.  
  
10. Создайте файлы, необходимые для настройки EDGE1 как агент проверки подлинности RSA.  
  
    1.  На **доступа** вкладке, выделите **агентов аутентификации**и нажмите кнопку **Add New**.  
  
    2.  Тип **EDGE1** в **Hostname** и нажмите кнопку **разрешить IP-адрес**.  
  
    3.  Обратите внимание, что IP-адрес для EDGE1 теперь отображается в **IP-адрес** поля. Нажмите кнопку **Сохранить**.  
  
11. Создайте файл конфигурации для сервера EDGE1 (AM_Config.zip).  
  
    1.  На **доступа** вкладке, выделите **агентов аутентификации**и нажмите кнопку **создать файл конфигурации**.  
  
    2.  На **создать файл конфигурации** странице щелкните **создать файл конфигурации**, а затем нажмите кнопку **Download Now**.  
  
    3.  Нажмите кнопку **Сохранить**, перейдите к C:\ Установкой RSA и нажмите кнопку **Сохранить**.  
  
    4.  Нажмите кнопку **закрыть** на **Загрузка завершена** диалоговое окно.  
  
12. Создайте файл секрета узла для сервера EDGE1 (EDGE1_NodeSecret.zip).  
  
    1.  На **доступа** вкладке, выделите **агентов аутентификации**и нажмите кнопку **управление существующих**.  
  
    2.  Выберите текущий настроенный узел EDGE1 и кнопку **управлять секрет узел**.  
  
    3.  Проверьте **Создание секрета случайный узел и экспортировать в файл секрета узел** "флажок".  
  
    4.  Введите тот же пароль, используемый для администратора в **пароль шифрования** и **подтверждение пароля шифрования** поля и нажмите кнопку **Сохранить**.  
  
    5.  На **узел секрет файл создан** странице щелкните **Download Now**.  
  
    6.  На **Загрузка файла** диалоговом окне щелкните **Сохранить**, перейдите к C:\RSA установки и нажмите кнопку **Сохранить**. Нажмите кнопку **закрыть** на **Загрузка завершена** диалоговое окно.  
  
    7.  Из \auth_mgr\windows-x86_64\am\rsa-ace_nsload\win32-5.0-x86\agent_nsload.exe копирования мультимедиа диспетчер проверки подлинности RSA для C:\RSA установки.

2.  Доступ к источнику токена носителя программное обеспечение RSA SecurID и дважды щелкните RSASECURIDTOKEN410 в **SecurID SoftwareToken клиентское приложение** папки, чтобы начать установку RSA SecurID. Если **открыть файл - предупреждение системы безопасности** появится сообщение, нажмите кнопку **запуска**.  
  
3.  На **RSA SecurID программного обеспечения маркера - Мастер InstallShield** диалоговом окне щелкните **Далее** дважды.

6.  Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
7.  Выберите **RSA запуска программного обеспечения SecurID маркера** флажок и нажмите кнопку **Готово**.  
  
8.  Нажмите кнопку **импорта из файла**.  
  
9. Нажмите кнопку **Обзор**C:\RSA Files\User1_000031701832.SDTID выберите и щелкните **откройте**.  
  
10. Щелкните дважды **ОК** .  
  
## <a name="configAuthAgt"></a>Настройка EDGE1 как агент аутентификации RSA  
Эта процедура позволяет настроить EDGE1 для проверки подлинности RSA.  
  
#### <a name="configure-the-rsa-authentication-agent"></a>Настройка агента проверки подлинности RSA  
  
1.  На EDGE1 откройте проводник Windows и создайте папку C:\RSA файлов. Перейдите к RSA ACE установочного носителя.  
  
2.  Копия agent_nsload.exe файлы, AM_Config.zip и EDGE1_NodeSecret.zip с носителя RSA C:\RSA файлы.  
  
3.  Извлеките содержимое ZIP-файлы, как в следующих расположениях:  
  
    1.  C:\Windows\system32\  
  
    2.  C:\Windows\SysWOW64\  
  
4.  Скопируйте agent_nsload.exe C:\Windows\SysWOW64\\.  
  
5.  Откройте командную строку и перейдите к C:\Windows\SysWOW64.  
  
6.  Тип **nodesecret.rec agent_nsload.exe -f -p <password>**  где <password> — это надежный пароль, созданный во время начальной настройки RSA. Нажмите клавишу ВВОД.  
  
7.  Скопируйте C:\Windows\SysWOW64\securid C:\Windows\System32.  
  
## <a name="configOTP"></a>Настройка EDGE1 для поддержки проверки подлинности OTP  
Эта процедура позволяет настроить OTP для DirectAccess и проверьте конфигурацию.  
  
#### <a name="configure-otp-for-directaccess"></a>Настройка OTP для DirectAccess  
  
1.  На EDGE1, откройте диспетчер сервера и нажмите кнопку **УДАЛЕННОГО доступа** в левой области.  
  
2.  Щелкните правой кнопкой мыши **EDGE1** в области СЕРВЕРОВ и выберите **управления удаленным доступом**.  
  
3.  Нажмите кнопку **конфигурации**.  
  
4.  В **установки DirectAccess** окна в разделе **шаг 2 - сервер удаленного доступа**, нажмите кнопку **изменить**.  
  
5.  Нажмите кнопку **Далее** три раза и в **проверки подлинности** разделе выберите **двухфакторной проверки подлинности** и **использовать OTP**и убедитесь, что **Использовать сертификаты компьютеров** проверяется. Убедитесь, что корневой ЦС **CN = corp-APP1-CA**. Нажмите кнопку **Далее**.  
  
6.  В **сервер RADIUS OTP** дважды щелкните пустую **имя_сервера** поля.  
  
7.  В **Добавление RADIUS-сервера** диалоговое окно, введите **RSA** в **имя_сервера** поля. Нажмите кнопку **изменение** рядом с полем **общий секрет** поле и введите тот же пароль, который использовался при настройке RADIUS-клиенты на сервере RSA в **новый секрет** и  **Подтверждение** поля. Нажмите кнопку **ОК** дважды и нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > Если сервер RADIUS находится в домене, который отличается от сервера удаленного доступа, а затем **имя_сервера** поле необходимо указать полное ДОМЕННОЕ имя сервера RADIUS.  
  
8.  В **серверы ЦС OTP** выберите APP1.corp.contoso.com, при этом щелкните **добавить**. Нажмите кнопку **Далее**.  
  
9. На **шаблонов сертификатов для OTP** странице щелкните **Обзор** для выбора шаблона сертификата, используемого для подачи заявок на сертификаты, выданные для проверки подлинности OTP и на  **Шаблоны сертификатов** диалогового окна выберите **DAOTPLogon**. Нажмите кнопку **ОК**. Нажмите кнопку **Обзор** для выбора шаблона сертификата, используемый для регистрации сертификата, используемый сервером удаленного доступа для запросов на регистрацию сертификата OTP входа, а на **шаблонов сертификатов** диалоговое окно Выберите **DAOTPRA**. Нажмите кнопку **ОК**. Нажмите кнопку **Далее**.  
  
10. На **установки сервера удаленного доступа** странице щелкните **Готово**и нажмите кнопку **Готово** на **мастер установки DirectAccess Expert**.  
  
11. На **удаленном доступе** диалоговом окне **применить**, подождите для политики DirectAccess, чтобы обновить и нажмите кнопку **закрыть**.  
  
12. На **запустить** введите**powershell.exe**, щелкните правой кнопкой мыши **powershell**, нажмите кнопку **Дополнительно**и нажмите кнопку **Запуск от имени Администратор**. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
13. В окне Windows PowerShell введите **gpupdate/force** и нажмите клавишу ВВОД.  
  
14. Закройте и снова откройте консоль управления удаленным доступом и проверьте правильность всех параметров проверки подлинности OTP.
