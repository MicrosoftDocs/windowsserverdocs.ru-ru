# <a name="step-4-create-the-network-load-balanced-remote-access-cluster"></a>Шаг 4, создайте кластер удаленного доступа, с балансировкой сетевой нагрузки

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

 Windows Server 2016, Windows Server 2012 R2 и Windows Server 2012 позволяют создавать кластеры серверов удаленного доступа. Кластер выступает в качестве одного логического сервера и обеспечивает централизованную настройку и управление для серверов в кластере. При использовании балансировки сетевой нагрузки (NLB) поддерживается до 8 удаленного доступа к элементам в одном кластере. Удаленный доступ кластеры обеспечивают высокий уровень доступности и балансировку нагрузки для подключений от клиентов DirectAccess к внутренней сети.  
  
Следующие процедуры позволяют создавать и тестировать кластера удаленного доступа:  
  
1. Установите компонент балансировки сетевой нагрузки на EDGE1 и EDGE2. Прежде чем включить балансировку нагрузки, необходимо установить функцию балансировки сетевой нагрузки на EDGE1 и EDGE2.
  
2. Включение балансировки нагрузки на EDGE1. EDGE1 изначально был установлен в режиме одного сервера. Чтобы включить балансировку нагрузки, следует настроить EDGE1 новый внешний и внутренний выделенный IP-адреса (DIP). Предыдущий частные интерфейсы на EDGE1 автоматически настраиваются как виртуальные IP-адреса (VIP) для кластера. Новый внешний DIP-адрес является 131.107.0.10, нового внутреннего DIP IPv4 10.0.0.10, нового внутреннего DIP IPv6 является 2001:db8:1::10. VIP-адресов кластера — 131.107.0.2 и 131.107.0.3 (внешний) и — 10.0.0.2 и 2001:db8:1::2 (внутренний).
  
3. Добавьте EDGE2 кластере с балансировкой нагрузки. После включения балансировки нагрузки, теперь можно добавить в кластер для обеспечения балансировки нагрузки и высокий уровень доступности для клиентских подключений DirectAccess EDGE2.

## <a name="prerequisites"></a>Предварительные требования

При создании этой лаборатории тестирования на виртуальных машинах, необходимо включить спуфинг на EDGE1 и EDGE2 MAC-адресов.  
  
### <a name="enable-mac-address-spoofing-on-edge1-and-edge2"></a>Включить на EDGE1 и EDGE2 спуфинг MAC-адресов  
  
1.  Выполните нормальное завершение работы на EDGE1 и EDGE2.  
  
2.  На компьютере, где размещен на виртуальных машинах, в **диспетчера Hyper-V**, щелкните правой кнопкой мыши EDGE1 и нажмите кнопку **параметры**.  
  
3.  На **параметры** отображаемое в диалоговом окне **оборудования** списке, щелкните сетевой адаптер, подключенный к сети corpnet и затем в области сведений выберите **разрешить спуфинг MAC-адресов**  "флажок".  
  
4.  В **оборудования** списке, щелкните сетевой адаптер, подключенный к сети Интернет и в области сведений выберите **разрешить спуфинг MAC-адресов** "флажок".  
  
5.  На **параметры** диалоговом окне щелкните **ОК**.  
  
6.  Повторите эту процедуру из шага 2 в EDGE2.  
  
## <a name="install-the-network-load-balancing-feature-on-edge1-and-edge2"></a>Установите компонент балансировки сетевой нагрузки на EDGE1 и EDGE2  
Чтобы настроить EDGE1 и EDGE2 в кластере, необходимо установить компонент балансировки сетевой нагрузки на EDGE1 и EDGE2.  
  
### <a name="to-install-network-load-balancing"></a>Установка балансировки сетевой нагрузки  
  
1.  На EDGE1, в консоли диспетчера серверов в **панели мониторинга**, нажмите кнопку **Добавить роли и компоненты**.  
  
2.  Нажмите кнопку **Далее** четыре раза, чтобы перейти на страницу выбора компонентов сервера.  
  
3.  На **Выбор компонентов** диалоговое окно, выберите **балансировки сетевой нагрузки**, нажмите кнопку **добавить компоненты**, нажмите кнопку **Далее**, а затем нажмите кнопку **Установить**.  
  
4.  В диалоговом окне **Ход установки** убедитесь в успешном завершении установки, а затем нажмите кнопку **Закрыть**.  
  
5.  Повторите эту процедуру на EDGE2.  
  
## <a name="enable-load-balancing-on-edge1"></a>Включение балансировки нагрузки на EDGE1  
Эта процедура позволяет включить балансировку нагрузки и настроить новый частные интерфейсы на EDGE1.  
  
### <a name="enable-load-balancing"></a>Включение балансировки нагрузки  
  
1.  На EDGE1, нажмите кнопку **запустить**, тип **RAMgmtUI.exe**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом в области слева щелкните **конфигурации**, а затем в **задачи** панели щелкните **Включение балансировки нагрузки**.  
  
3.  Включить нагрузки балансировки, нажмите кнопку в мастере **Далее**.  
  
4.  На **метода балансировки нагрузки** щелкните **использования Windows сетевой балансировки нагрузки (NLB)** , а затем нажмите кнопку **Далее**.  
  
5.  На **внешних IP-адресов, выделенных** странице **IPv4-адрес** введите **131.107.0.10**в **маска подсети** убедитесь, что префикс подсети **255.255.255.0**, а затем нажмите кнопку **Далее**.  
  
6.  На **внутренних IP-адресов, выделенных** странице, выполните следующие действия и нажмите кнопку **Далее**:  
  
    1.  В **IPv4-адрес** введите **10.0.0.10** и в **маска подсети** убедитесь в том, используется префикс подсети **255.255.255.0**.  
  
    2.  В **IPv6-адрес** введите **2001:db8:1::10** в Длина префикса подсети, убедитесь, значение равно **64**.  
  
7.  На **Сводка** щелкните **зафиксировать**.  
  
8.  На **Включение балансировки нагрузки** диалоговом окне щелкните **закрыть**.  
  
9. Включить нагрузки балансировки, нажмите кнопку в мастере **закрыть**.  
  
## <a name="add-edge2-to-the-load-balanced-cluster"></a>Добавить EDGE2 в кластере с балансировкой нагрузки  
Эта процедура позволяет добавить EDGE2 в кластер балансировки сетевой Нагрузки.  
  
> [!NOTE]  
> По истечении двух минут после завершения предыдущих шагов, прежде чем продолжить. После включения балансировки сетевой Нагрузки, RAConfigTask выполняется и настраивает параметры балансировки сетевой Нагрузки на компьютере. Это может занять несколько минут, и если администратор выполняет другую конфигурацию связанных балансировки сетевой Нагрузки до завершения задачи, что конфигурация завершится ошибкой.  
  
### <a name="add-edge2-to-the-cluster"></a>Добавление EDGE2 в кластер  
  
1.  На компьютере EDGE1 или виртуальной машины, в консоли управления удаленным доступом в **задачи** панели в разделе **кластера с балансировкой нагрузки**, нажмите кнопку **добавить или удалить серверы**.  
  
2.  На **добавить или удалить серверы** диалоговом окне щелкните **добавить сервер**.  
  
3.  На **добавить сервер** мастера **Выбор сервера** введите **EDGE2**, а затем нажмите кнопку **Далее**.  
  
4.  На **сетевых адаптеров** странице **внешнего адаптера**, убедитесь, что **Internet** выбран и в **внутренний адаптер**, убедитесь, что **Corpnet** выбран. Нажмите кнопку **Обзор**на **безопасности Windows** диалогового окна поле, убедитесь, что **сертификата IP-HTTPS** — флажок установлен, нажмите кнопку **ОК**и нажмите кнопку **Далее**.  
  
5.  На **Сводка** щелкните **добавить**.  
  
6.  На странице **Завершение** нажмите кнопку **Закрыть**.  
  
7.  На **добавить или удалить серверы** диалоговом окне щелкните **зафиксировать**.  
  
8.  На **Добавление и удаление серверов** диалоговом окне щелкните **закрыть**.  
  
9. На **запустить** введите**nlbmgr.exe**и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
10. В **диспетчер балансировки сетевой нагрузки**, нажмите кнопку **кластера внутренней DA**. В области сведений убедитесь, что оба **EDGE1(Corpnet)** и **EDGE2(Corpnet)** со статусом **схождение выполнено**.  
  
11. Если сервер не **схождение выполнено**, в дереве консоли щелкните правой кнопкой мыши сервер, укажите **узел управления**, а затем нажмите кнопку **запустить**.  
  
12. В **диспетчер балансировки сетевой нагрузки**, нажмите кнопку **кластера Internet DA**. Убедитесь, что в области сведений щелкните оба **EDGE1(Internet)** и **EDGE2(Internet)** со статусом **схождение выполнено**.  
  
13. Если сервер не **схождение выполнено**, в дереве консоли щелкните правой кнопкой мыши сервер, укажите **узел управления**, а затем нажмите кнопку **запустить**.
