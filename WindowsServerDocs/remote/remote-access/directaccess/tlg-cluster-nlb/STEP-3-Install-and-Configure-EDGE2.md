# <a name="step-3-install-and-configure-edge2"></a>Шаг 3 Установка и настройка EDGE2

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

EDGE2 является второй членом кластера удаленного доступа. EDGE2 установлен и настроен перед включением конфигурации кластера.

Выполните следующие действия, чтобы настроить EDGE2.

## <a name="installOS"></a>Установите операционную систему на EDGE2  
  
1.  На EDGE2 начните установку Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012.  
  
2.  Следуйте инструкциям для завершения установки, указав Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012 (полная установка) и надежный пароль для учетной записи локального администратора. Войдите, используя учетную запись локального администратора.  
  
3.  Подключение к сети, которая имеет доступ к Интернету EDGE2 и запустите Центр обновления Windows Чтобы установить последние обновления для Windows Server 2016, Windows Server 2012 R2 или Windows Server 2012, а затем отключиться от Интернета.  
  
4.  Один сетевой адаптер подключите к подсети Corpnet или виртуального коммутатора, представляющая подсети corpnet, а другой к подсети Интернет или виртуального коммутатора, представляющий подсети Интернет.  
  
## <a name="TCP"></a>Настройте свойства TCP/IP  
  
1.  В консоли диспетчера серверов щелкните **локального сервера**, а затем в **свойства** область, рядом с полем **проводное Ethernet-подключение**, щелкните ссылку.  
  
2.  В **сетевых подключений** окно, щелкните правой кнопкой мыши сетевое подключение, которое подключается к подсети Corpnet или виртуальный коммутатор и нажмите кнопку **Переименовать**.  
  
3.  Тип **Corpnet**, и нажмите клавишу ВВОД.  
  
4.  Щелкните правой кнопкой мыши **Corpnet**, а затем нажмите кнопку **свойства**.  
  
5.  Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
6.  Выберите вариант **Использовать следующий IP-адрес**. В **IP-адрес**, тип **10.0.0.8**. В поле **Маска подсети**введите **255.255.255.0**.  
  
7.  Выберите вариант **Использовать следующие адреса DNS-серверов**. В поле **Предпочитаемый DNS-сервер** введите **10.0.0.1**.  
  
8.  Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"**.  
  
9. В **DNS-суффикс этого подключения**, тип **corp.contoso.com**, нажмите кнопку **ОК** дважды.  
  
10. Нажмите кнопку **Internet Protocol версии 6 (TCP/IPv6)**, а затем нажмите кнопку **свойства**.  
  
11. Нажмите кнопку **использовать следующий IPv6-адрес**. В **IPv6-адрес**, тип **2001:db8:1::8**. В **длина префикса подсети**, тип **64**.  
  
12. Выберите вариант **Использовать следующие адреса DNS-серверов**. В **предпочитаемый DNS-сервер**, тип **2001:db8:1::1**.  
  
13. Нажмите кнопку **"Дополнительно"** и перейдите на вкладку **"DNS"**.  
  
14. В **DNS-суффикс этого подключения**, тип **corp.contoso.com**, нажмите кнопку **ОК** дважды, а затем нажмите кнопку **закрыть**.  
  
15. В **сетевых подключений** окно, щелкните правой кнопкой мыши сетевое подключение, подключенный к подсети Интернет и нажмите кнопку **Переименовать**.  
  
16. Тип **Internet**, и нажмите клавишу ВВОД.  
  
17. Правой кнопкой мыши щелкните **"Интернет"** и выберите в контекстном меню команду **"Свойства"**.  
  
18. Щелкните пункт **IP версия 4 (TCP/IPv4)** и нажмите кнопку **Свойства**.  
  
19. Выберите вариант **Использовать следующий IP-адрес**. В **IP-адрес**, введите **131.107.0.8**. В **маска подсети**, введите **255.255.255.0**.  
  
20. Нажмите кнопку **DNS** вкладку  
  
21. В **DNS-суффикс этого подключения**, тип **isp.example.com**и нажмите кнопку **ОК** дважды, а затем нажмите кнопку **закрыть**.  
  
22. Закройте окно **Сетевые подключения** .  
  
23. Чтобы проверить взаимодействие между EDGE2 и DC1, нажмите кнопку **запустить**, тип **cmd**, и нажмите клавишу ВВОД.  
  
24. В окне командной строки введите **ping dc1.corp.contoso.com** и нажмите клавишу ВВОД. Убедитесь, что существует четыре ответы от 10.0.0.1 или 2001:db8:1::1 адрес IPv6  
  
25. Закройте окно командной строки.  
  
## <a name="rename"></a>Переименуйте EDGE2 и присоединить ее к домену  
  
1.  В консоли диспетчера серверов в **локального сервера**в **свойства** область, рядом с полем **имя_компьютера**, щелкните ссылку.  
  
2.  В диалоговом окне **"Свойства системы"** на вкладке **"Имя компьютера"** щелкните **"Изменить"**.  
  
3.  На **изменение имени компьютера или домена** отображаемое в диалоговом окне **имя_компьютера** введите **EDGE2**. В **членом** область, нажмите кнопку **домена**и в текстовом поле введите **corp.contoso.com**, а затем нажмите кнопку **ОК**.  
  
4.  При появлении запроса на ввод имени пользователя и пароля введите **User1** и пароль для этого пользователя, а затем нажмите кнопку **ОК**.  
  
5.  Когда откроется диалоговое окно приветствия домена corp.contoso.com, нажмите кнопку **ОК**.  
  
6.  При появлении запроса на перезагрузку компьютера нажмите кнопку **ОК**.  
  
7.  В диалоговом окне **Свойства системы** нажмите **Закрыть**.  
  
8.  При появлении запроса на перезагрузку компьютера нажмите кнопку **Перезагрузить сейчас**.  
  
9. После перезагрузки войдите в систему как CORP\User1.  
  
## <a name="IPHTTPSCert"></a>Установка сертификата IP-HTTPS  
  
1.  На **запустить** введите**mmc.exe**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли MMC в меню **Файл** выберите пункт **Добавить или удалить оснастку**.  
  
3.  На **Добавление или удаление оснасток** диалоговом окне щелкните **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку  **Далее**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
4.  В левой области консоли, перейдите к **\Personal\Certificates сертификаты (локальный компьютер)**. Щелкните правой кнопкой мыши **сертификаты** узел **все задачи**, а затем нажмите кнопку **запросить новый сертификат**.  
  
5.  В мастере регистрации сертификатов, щелкните **Далее** дважды.  
  
6.  На **запрос сертификатов** выберите **веб-сервере** флажок и нажмите кнопку **требуется больше данных для подачи заявки на этот сертификат**.  
  
7.  На **свойства сертификата** диалоговом окне **субъекта** на вкладке **имя субъекта** область в **тип** выберите **Общее имя**.  
  
8.  В **значение**, тип **edge1.contoso.com**, а затем нажмите кнопку **добавить**.  
  
9. В **альтернативное имя** область в **тип** выберите **DNS**.  
  
10. В **значение**, тип **edge1.contoso.com**, а затем нажмите кнопку **добавить**.  
  
11. На **Общие** на вкладке **понятное имя**, тип **сертификата IP-HTTPS**.  
  
12. Нажмите кнопку **OK**, щелкните **Зарегистрировать** и нажмите кнопку **Готово**.  
  
13. В области сведений оснастки сертификатов убедитесь, что новый сертификат с именем edge1.contoso.com был зарегистрирован с целью из сервера проверки подлинности.  
  
14. Закройте окно консоли. Если вам будет предложено сохранить параметры, нажмите кнопку **нет**.  
  
## <a name="InstallDA"></a>Установите роль удаленного доступа на EDGE2  
  
1.  В консоли диспетчера серверов в **панели мониторинга**, нажмите кнопку **Добавить роли и компоненты**.  
  
2.  Нажмите кнопку **Далее** трижды, чтобы перейти на страницу выбора роли сервера.  
  
3.  В диалоговом окне **Выбор ролей сервера** выберите **Удаленный доступ**, щелкните **Добавить компоненты**, а затем нажмите кнопку **Далее**.  
  
4.  Нажмите кнопку **Далее** пять раз.  
  
5.  В диалоговом окне **Подтверждение выбранных элементов для установки** нажмите кнопку **Установить**.  
  
6.  В диалоговом окне **Ход установки** убедитесь в успешном завершении установки, а затем нажмите кнопку **Закрыть**.
