# <a name="step-3-configure-a-load-balanced-cluster"></a>Шаг 3 настройка кластера с балансировкой нагрузки

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

После подготовки серверов для кластера, Настройка балансировки нагрузки на одном сервере, настройки необходимых сертификатов и развертывания кластера.  
  
|Задача|Описание|  
|----|--------|  
|[3.1 Настройте префикс IPv6](#BKMK_Prefix)|Если корпоративной среде — IPv4 + IPv6, или только для IPv6, на одном сервере удаленного доступа, убедитесь, что префикс IPv6, назначенный клиентскому компьютеру DirectAccess является достаточно большим, чтобы охватить все серверы в кластере.|  
|[3.2 работы подсистемы балансировки нагрузки](#BKMK_NLB)|Включение балансировки нагрузки на сервер удаленного доступа.|  
|[3.3 установите сертификат IP-HTTPS](#BKMK_InstallIPHTTP)|Каждый сервер в кластере требуется сертификат сервера для проверки подлинности подключения IP-HTTPS.  Экспорт сертификата IP-HTTPS с одного сервера удаленного доступа и развернуть его на каждом сервере, которые будут добавлены в кластер. Этот параметр обязателен, только в том случае, если с помощью сертификатов, не являются самозаверяющими.|  
|[3.4 установите сертификат сервера сетевых расположений](#BKMK_NLS)|Если один сервер имеет сервер сетевых расположений, развернутых локально, затем необходимо будет развернуть сертификат сервера сетевых расположений на каждом сервере в кластере. Если сервер сетевых расположений размещен на внешнем сервере, сертификат на каждом сервере не требуется. Этот параметр обязателен, только в том случае, если с помощью сертификатов, не являются самозаверяющими.|  
|[3.5 добавить серверы в кластер](#BKMK_Add)|Добавьте все серверы в кластер. Удаленный доступ не должен быть настроен на серверах должен быть добавлен.|  
|[3.6 удаление сервера из кластера](#BKMK_remove)|Инструкции для удаления сервера из кластера.|  
|[3.7 отключение балансировки нагрузки](#BKBK_disable)|Инструкции по отключение балансировки нагрузки.|  
  
> [!NOTE]  
> IP-адрес, выбранный для DIP-адрес не должен использоваться в на сетевых адаптерах первого сервера удаленного доступа в кластере. Начала развертывания DirectAccess с VIP и DIP, добавляемый сетевой адаптер приведет к сбою.  
  
> [!NOTE]  
> Убедитесь в том, что не следует использовать DIP-адрес, который уже присутствует на другом компьютере в сети.  
  
## <a name="BKMK_Prefix"></a>3.1 Настройте префикс IPv6  
  
### <a name="configDA"></a>Чтобы настроить префикс  
  
1.  На сервере удаленного доступа, нажмите кнопку **запустить**, а затем нажмите кнопку **управления удаленным доступом**. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом щелкните **Конфигурация**.  
  
3.  В средней области консоли в **шаг 2. сервер DirectAccess** область, нажмите кнопку **изменить**.  
  
4.  Нажмите кнопку **префикса конфигурации**. На **Конфигурация префикса** странице **префикс IPv6, назначенный клиентскому компьютеру DirectAccess**, введите префикс IPv6, используемый для клиентских компьютеров DirectAccess с длиной 59, например, подсеть **2001:db8:1:1000:: / 59**. Если VPN также были включены с поддержкой IPv6, затем будет отображаться префикс IPv6 и длина подсети потребуется изменить до 59. Нажмите кнопку **Далее**.  
  
5.  В средней области консоли щелкните **Готово**.  
  
6.  На **удаленного доступа просмотрите** диалоговое окно, просмотрите параметры конфигурации и нажмите кнопку **применить**. В диалоговом окне **Применение параметров мастера настройки удаленного доступа** нажмите кнопку **Закрыть**.  
  
## <a name="BKMK_NLB"></a>3.2 работы подсистемы балансировки нагрузки  
  
#### <a name="to-enable-load-balancing"></a>Для включения балансировки нагрузки  
  
1.  На настроенный сервер DirectAccess, нажмите кнопку **запустить**, а затем нажмите кнопку **управления удаленным доступом**. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом в области слева щелкните **конфигурации**, а затем в **задачи** панели щелкните **Включение балансировки нагрузки**.  
  
3.  Включить нагрузки балансировки, нажмите кнопку в мастере **Далее**.  
  
4.  В зависимости от выбора в этапы планирования:  
  
    1.  Windows NLB: На **метода балансировки нагрузки** щелкните **использования Windows сетевой балансировки нагрузки (NLB)**, а затем нажмите кнопку **Далее**.  
  
    2.  Внешняя Подсистема балансировки нагрузки: На **метод балансировки нагрузки** щелкните **использовать внешний балансировщик нагрузки**, а затем нажмите кнопку **Далее**.  
  
5.  При развертывании одного сетевого адаптера на **выделенного IP-адреса** странице, выполните следующие действия и нажмите кнопку **Далее**:  
  
    1.  В **IPv4-адрес** », вкладка «введите новый адрес IPv4 для этого сервера удаленного доступа; текущий виртуальный IP-адрес (VIP) кластера с балансировкой нагрузки будет адрес IPv4. В **маска подсети** введите маску подсети.  
  
    2.  Если корпоративной среде является собственной сети IPv6, то в **IPv6-адрес** », вкладка «введите новый адрес IPv6 для этого сервера удаленного доступа; текущий IPv6 адрес может быть виртуальный IP-адрес кластера с балансировкой нагрузки. В **длина префикса подсети** введите длину префикса подсети.  
  
6.  В двух сетевого адаптера развертывания, на **внешних IP-адресов, выделенных** странице, выполните следующие действия и нажмите кнопку **Далее**:  
  
    1.  В **IPv4-адрес** », вкладка «введите новый внешний адрес IPv4 для этого сервера удаленного доступа; текущий IPv4 адрес может быть кластер балансировки нагрузки виртуальный IP-адрес (VIP). В **маска подсети** введите маску подсети.  
  
    2.  Есть ли в настоящее время собственный IPv6-адресов, настроенных на Интернет-адаптера сервера удаленного доступа, в **IPv6-адрес** введите новый внешний адрес IPv6 для этого удаленного доступа сервер; текущий IPv6 адрес будет виртуальный IP-адрес кластера балансировки нагрузки. В **длина префикса подсети** введите длину префикса подсети.  
  
7.  В двух сетевого адаптера развертывания, на **внутренних IP-адресов, выделенных** странице, выполните следующие действия и нажмите кнопку **Далее**:  
  
    1.  В **IPv4-адрес** », вкладка «введите новый внутренний адрес IPv4 для этого сервера удаленного доступа; текущий IPv4 адрес может быть балансировки нагрузки виртуальный IP-адрес кластера. В **маска подсети** введите маску подсети.  
  
    2.  Если корпоративной среде является собственной сети IPv6, то в **IPv6-адрес** », вкладка «введите новый внутренний адрес IPv6 для этого сервера удаленного доступа; текущий IPv6 адрес может быть балансировки нагрузки виртуальный IP-адрес кластера. В **длина префикса подсети** введите длину префикса подсети.  
  
8.  На **Сводка** щелкните **зафиксировать**.  
  
9. На **Включение балансировки нагрузки** диалоговом окне щелкните **закрыть**.  
  
10. Включить нагрузки балансировки, нажмите кнопку в мастере **закрыть**.  
  
    > [!NOTE]  
    > Если используется внешняя Балансировка нагрузки, обратите внимание, виртуальные IP-адреса и сообщите на внешние подсистемы балансировки нагрузки.  
  
![Windows PowerShell](../../../../media/Step-3-Configure-a-Load-Balanced-Cluster/PowerShellLogoSmall.gif)Windows PowerShell эквивалентные команды ***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
Если вы выбрали для использования балансировки сетевой Нагрузки Windows в этапы планирования, а затем выполните следующее:  
  
```  
Set-RemoteAccessLoadBalancer -InternetDedicatedIPAddress "2.1.1.20/255.255.255.0" -InternalDedicatedIPAddress @("10.1.1.30/255.255.255.0","3ffe::20/64") -InternetVirtualIPAddress @("2.1.1.1/255.255.255.0","2.1.1.2/255.255.255.0") -InternalVirtualIPAddress @("10.1.1.2/255.255.255.0","3ffe::2/64")  
```  
  
Если вы решили использовать внешний балансировщик нагрузки в шаги планирования: затем выполните следующее:  
  
```  
Set-RemoteAccessLoadBalancer -InternetDedicatedIPAddress "2.1.1.20/255.255.255.0" -InternalDedicatedIPAddress @("10.1.1.30/255.255.255.0","3ffe::20/64") -UseThirdPrtyLoadBalancer  
```  
  
> [!NOTE]  
> Рекомендуется не включать изменения в параметры подсистемы балансировки нагрузки с изменения других параметров, при использовании промежуточных объектах групповой политики. Сначала необходимо применить все изменения параметров подсистемы балансировки нагрузки и затем следует учитывать другие изменения в конфигурацию. Кроме того после настройки подсистемы балансировки нагрузки на новом сервере DirectAccess, подождите некоторое время применять и репликации для DNS-серверов в предприятии, прежде чем изменять другие параметры DirectAccess, относящиеся к новому кластеру изменения IP-адрес.  
  
## <a name="BKMK_InstallIPHTTP"></a>3.3 установите сертификат IP-HTTPS  
Чтобы выполнить эту процедуру, вы должны быть членом локальной группы **Администраторы** или аналогичной.  
  
### <a name="IPHTTPSCert"></a>Чтобы установить сертификат IP-HTTPS  
  
1.  На настроенный сервер удаленного доступа, нажмите кнопку **запустить**, тип **mmc** и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли MMC в меню **Файл** выберите пункт **Добавить или удалить оснастку**.  
  
3.  На **Добавление или удаление оснасток** диалоговом окне щелкните **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку  **Далее**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
4.  В левой области консоли, перейдите к **\Personal\Certificates сертификаты (локальный компьютер)**. Щелкните правой кнопкой мыши сертификат IP-HTTPS, укажите **все задачи** и нажмите кнопку **Экспорт**.  
  
5.  На **Добро пожаловать в мастер экспорта сертификатов** щелкните **Далее**.  
  
6.  На странице **Экспорт закрытого ключа** выберите вариант **Да, экспортировать закрытый ключ**, а затем нажмите кнопку **Далее**.  
  
7.  На **Формат экспортируемого файла** щелкните **файл обмена личной информацией - PKCS #12 (. PFX-ФАЙЛ)**, а затем нажмите кнопку **Далее**.  
  
8.  На **безопасности** выберите **пароль** , введите пароль в **пароль** и подтвердите пароль и нажмите кнопку **Далее**.  
  
9. На **экспортируемого файла** странице, введите имя для файла сертификата и сохраните его на рабочем столе и нажмите кнопку **Далее**.  
  
10. На **завершение работы мастера экспорта сертификатов** щелкните **Готово**.  
  
11. На **мастер экспорта сертификатов** диалоговом окне щелкните **ОК**.  
  
12. Скопируйте сертификат на все серверы, которые должны входить в кластер.  
  
13. На новом сервере DirectAccess, нажмите кнопку **запустить**, тип **mmc** и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
14. В консоли MMC в меню **Файл** выберите пункт **Добавить или удалить оснастку**.  
  
15. На **Добавление или удаление оснасток** диалоговом окне щелкните **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку  **Далее**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
16. В левой области консоли, перейдите к **\Personal\Certificates сертификаты (локальный компьютер)**. Щелкните правой кнопкой мыши **сертификаты** узел **все задачи**, а затем нажмите кнопку **импорта**.  
  
17. На **Добро пожаловать в мастер импорта сертификатов** щелкните **Далее**.  
  
18. На **импортируемый файл** щелкните **Обзор** для поиска сертификата. Выберите сертификат и нажмите кнопку **Далее**.  
  
19. На **защиту закрытого ключа** странице **пароль** введите пароль и нажмите кнопку **Далее**.  
  
20. На странице **Хранилище сертификатов** нажмите кнопку **Далее**.  
  
21. На **завершение работы мастера импорта сертификатов** щелкните **Готово**.  
  
22. На **мастер импорта сертификатов** диалоговом окне щелкните **ОК**.  
  
23. Повторите шаги 13 – 22 на всех серверах, которые вы хотите быть членов кластера.  
  
## <a name="BKMK_NLS"></a>3.4 установите сертификат сервера сетевых расположений  
Чтобы выполнить эту процедуру, вы должны быть членом локальной группы **Администраторы** или аналогичной.  
  
#### <a name="to-install-a-certificate-for-network-location"></a>Чтобы установить сертификат для сетевого расположения  
  
1.  На сервере удаленного доступа, нажмите кнопку **запустить**, тип **mmc**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  Нажмите кнопку **файл**, а затем нажмите кнопку **Add/Remove Snap in**.  
  
3.  Нажмите кнопку **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку **Далее**, нажмите кнопку **локального компьютера**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
4.  В дереве консоли оснастки "Сертификаты" откройте раздел **Сертификаты (локальный компьютер)\Личные\Сертификаты**.  
  
5.  Щелкните правой кнопкой **Сертификаты**, наведите указатель на элемент **Все задачи**, а затем щелкните **Запрос нового сертификата**.  
  
6.  Щелкните дважды **Далее** .  
  
7.  На **запрос сертификатов** странице, выберите шаблон сертификата веб-сервера и нажмите кнопку **требуется больше данных для подачи заявки на этот сертификат**.  
  
    Если шаблон сертификата веб-сервер не отображается, убедитесь, что учетная запись компьютера сервера удаленного доступа имеет регистрируют разрешения для шаблона сертификата веб-сервера. Дополнительные сведения см. в разделе [настроить разрешения для шаблона сертификата веб-сервера](https://msdn.microsoft.com/library/ee649249.aspx).  
  
8.  На **субъекта** вкладке **свойства сертификата** отображаемое в диалоговом окне **имя субъекта**, для **тип**выберите **Common имя**.  
  
9. В **значение**, введите полное доменное имя (FQDN) для сетевого расположения сервера веб-сайта (например, nls.corp.contoso.com) имя в интрасети и нажмите кнопку **добавить**.  
  
10. Нажмите кнопку **OK**, щелкните **Зарегистрировать** и нажмите кнопку **Готово**.  
  
11. В области сведений оснастки сертификатов, убедитесь, что новый сертификат с полным доменным ИМЕНЕМ был зарегистрирован с **назначения** из **проверки подлинности сервера**.  
  
12. Щелкните правой кнопкой мыши сертификат, а затем нажмите кнопку **свойства**.  
  
13. В **понятное имя**, тип **сертификата сетевых расположений**, а затем нажмите кнопку **ОК**.  
  
    > [!TIP]  
    > Шаги 12 и 13 необязательны, но также упрощают процесс можно было выбрать сертификат для сетевого расположения, при настройке удаленного доступа.  
  
14. Повторите эту процедуру на всех серверах, которые вы хотите быть членов кластера.  
  
## <a name="BKMK_Add"></a>3.5 добавить серверы в кластер  
 
  
#### <a name="to-add-servers-to-the-cluster"></a>Чтобы добавить серверы в кластер  
  
1.  На настроенный сервер DirectAccess, нажмите кнопку **запустить**, а затем нажмите кнопку **управления удаленным доступом**. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом щелкните **Конфигурация**. В **задачи** панели в разделе **кластера с балансировкой нагрузки**, нажмите кнопку **добавить или удалить серверы**.  
  
3.  На **добавить или удалить серверы** диалоговом окне щелкните **добавить сервер**.  
  
4.  На **добавить сервер** диалоговом окне **Выбор сервера** странице, введите имя дополнительного сервера удаленного доступа и нажмите кнопку **Далее**.  
  
5.  На **сетевых адаптеров** выполните одно из следующих:  
  
    -   Если вы собираетесь выполнять развертывание топологии с двумя сетевыми адаптерами в **внешнего адаптера**, выберите адаптер, подключенный к внешней сети. В **внутренний адаптер**, выберите адаптер, подключенный к внутренней сети.  
  
    -   Если вы собираетесь выполнять развертывание топологии с одним сетевым адаптером в **сетевой адаптер**, выберите адаптер, подключенный к внутренней сети.  
  
6.  На **сетевых адаптеров** странице **выберите сертификат, используемый для проверки подлинности подключений IP-HTTPS**, нажмите кнопку **Обзор** чтобы найти и выбрать сертификат IP-HTTPS, а затем нажмите кнопку **Далее**.  
  
7.  На **сервера сетевых расположений** щелкните **Обзор** выберите сертификат для сетевого расположения сервера веб-сайта на сервере удаленного доступа, а затем нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > **Сервера сетевых расположений** страница появляется только при выполнении веб-сайта сервера сетевых расположение на сервере удаленного доступа.  
  
    > [!NOTE]  
    > Если VPN также были настроены на сервере удаленного доступа, вы бы запрос на добавление IP-адрес VPN пула информации об адресе на этом этапе.  
  
8.  На **Сводка** щелкните **добавить**.  
  
9. На странице **Завершение** нажмите кнопку **Закрыть**.  
  
10. Повторите эту процедуру для всех серверов удаленного доступа для добавления в кластер.  
  
11. На **добавить или удалить серверы** диалоговом окне щелкните **зафиксировать**.  
  
12. На **Добавление и удаление серверов** диалоговом окне щелкните **закрыть**.

## <a name="BKBK_disable"></a>3.7 отключение балансировки нагрузки  
[Этот шаг, с помощью Windows PowerShell](assetId:///7a817ca0-2b4a-4476-9d28-9a63ff2453f9)  
  
#### <a name="to-disable-load-balancing"></a>Отключение балансировки нагрузки  
  
1.  На настроенный сервер DirectAccess, нажмите кнопку **запустить**, а затем нажмите кнопку **управления удаленным доступом**. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом щелкните **Конфигурация**. В **задачи** панели в разделе **кластера с балансировкой нагрузки**, нажмите кнопку **отключение балансировки нагрузки**.  
  
3.  На **отключение балансировки нагрузки** диалоговом окне щелкните **ОК**.  
  
4.  На **отключение балансировки нагрузки** диалоговом окне щелкните **закрыть**.  
  
![Windows PowerShell](../../../../media/Step-3-Configure-a-Load-Balanced-Cluster/PowerShellLogoSmall.gif)Windows PowerShell эквивалентные команды ***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
```  
set-RemoteAccessLoadBalancer -disable  
```  
  
Отключение балансировки нагрузки приведет к удалению параметров удаленного доступа и параметры балансировки сетевой Нагрузки (если настроено) от всех серверов, за исключением сервера, из которого она выполняется. На этом сервере удаленного доступа будут удалены параметры балансировки сетевой Нагрузки (если он был настроен), но останется параметры удаленного доступа.  
  
Щелкнув **удалить параметры конфигурации** приведет к удалению удаленного доступа и балансировки сетевой Нагрузки (если настроено) от всех серверов в развертывании.  
  
> [!NOTE]  
> -   При развертывании балансировки нагрузки удаленного доступа в случае удаления всех серверов, остаются с частные интерфейсы. VIP-адресов, будут удалены. В результате все маршруты в корпоративной сети, которые предназначены для адресов виртуальные IP-адреса, переход на другой. Это также влияет на DNS-записей, которые были разрешены для VIP-адресов, такие как имя субъекта сертификата сервера расположение сети. Чтобы избежать этой проблемы, отключите балансировки нагрузки, оставляет VIP-адресов на последний сервер удаленного доступа и затем удалить компонент удаленного доступа.  
> -   После использования **RemoteAccessLoadBalancer набора** командлет, чтобы отключить балансировки нагрузки, подождите 2 минуты, прежде чем запускать любой другой командлет. Это следует сделать в сценариев, запускаемых другого командлета после **Set RemoteAccessLoadBalancer-отключить** командлета.  
> -   Отключение изменений балансировки нагрузки виртуальный IP-адрес кластера в выделенный IP-адрес. Таким образом любая операция, которая запрашивает имя сервера не удастся до истечения срока действия кэшированной записи DNS на сервере. Убедитесь, что не выполнять все командлеты PowerShell удаленного доступа после отключения балансировки нагрузки, пока не истечет срок действия кэша на сервере. Эта проблема чаще, если попытаться отключить балансировки нагрузки в диск с другого компьютера, который находится в другом домене. Это также происходит, если отключить из консоли управления удаленным доступом балансировки нагрузки и может запретить загрузку. Конфигурация будет загружаться после истечения срока действия кэша, или был очищен.
