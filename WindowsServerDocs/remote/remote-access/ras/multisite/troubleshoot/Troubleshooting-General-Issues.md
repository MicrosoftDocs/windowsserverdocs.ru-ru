# <a name="troubleshooting-general-issues"></a>Диагностика проблем общего характера

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

Этот раздел содержит сведения об устранении неполадок для общие проблемы, связанные с удаленного доступа.  
  
## <a name="gpo-retrieval-error"></a>Ошибка при получении объекта групповой Политики  
**Сообщение об ошибке**. Не удалось получить параметры объекта групповой Политики сервера DirectAccess. Убедитесь, что у вас есть разрешения на изменение объекта групповой политики.  
  
Консоли управления удаленным доступом не отвечает после возникновения данной ошибки.  
  
**Причина**  
  
DirectAccess не может получить доступ к GPO из одной из точек входа в развертывании, и в результате конфигурации не удается загрузить.  
  
**Решение**  
  
Убедитесь, что каждой точки входа в развертывание имеет соответствующий объект групповой Политики на контроллер домена и убедитесь, что вошедший пользователь разрешения чтение и запись для всех объектов групповой политики, настроенных в развертывании удаленного доступа.  
  
Обойти это ограничение используйте командлеты конфигурации вместо использования консоли управления удаленным доступом; Например, с помощью `Get-RemoteAccess` и `Get-DAEntryPoint`.  
  
> [!NOTE]  
> Этот сценарий не возникает, когда GPO сервера текущей точки входа недоступна.  
  
Можно использовать `Get-DAEntryPointDC` командлет, чтобы получить список всех контроллеров домена, в которых хранятся объекты групповой политики сервера и `Get-DAMultiSite` в сочетании с `Get-RemoteAccess` для получения полного списка объектов групповой политики сервера в развертывании. Пример:  
  
```  
$ServerGpos = Get-DAEntryPointDC | ForEach-Object {   
   @{   
       GpoName = (Get-RemoteAccess -EntryPoint $_.EntryPointName).ServerGpoName;   
        DC = $_.DomainControllerName }   
}  
$ServerGpos | ForEach-Object { $GpoName = $_['GpoName'] ; $DC = $_['DC'] ; Write-Host "Server GPO '$GpoName' on DC '$DC'" }  
```  
  
## <a name="windows-7-to-windows-8-or-10-client-upgrade"></a>Windows 7 и Windows 8 или 10 обновления клиента  
**Симптом**. После обновления клиента Windows 7 для Windows 10 или Windows 8 при многосайтовом развертывании подключение DirectAccess не отображается в списке сетей.  
  
**Причина**  
  
Помощник по подключению к Windows 8 сетевой конфигурации не содержат объекты групповой политики Windows 7 при многосайтовом развертывании.  
  
 Клиенты Windows 7 следует использовать помощник по подключению к DirectAccess мониторинга состояния подключения DirectAccess которого требуется отдельная настройка вручную, в Windows 7 клиентских объектов групповой политики. При обновлении клиенты Windows 7 до Windows 10 или Windows 8, помощник по подключению к сети не будет работать, если по-прежнему применяется объект групповой Политики клиента Windows 7.  
  
**Решение**  
  
Если помощник по подключению к DirectAccess параметры настраиваются в объекты групповой политики Windows 7, можно устранить эту проблему, перед обновлением клиентские компьютеры путем изменения объектов групповой политики Windows 7, используя следующие командлеты PowerShell:  
  
```  
Set-GPRegistryValue -Name <Windows7GpoName> -Domain <DomainName> -Key "HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\NetworkConnectivityAssistant" -ValueName "TemporaryValue" -Type Dword -Value 1  
Remove-GPRegistryValue -Name <Windows7GpoName> -Domain <DomainName> -Key "HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\NetworkConnectivityAssistant"  
```  
  
Если клиент уже была обновлена или не настроен DCA, переместите клиентский компьютер в группу безопасности Windows 10 или Windows 8.  
  
## <a name="general-cmdlet-errors"></a>Командлет Общие ошибки  
  
-   **Причина 1**  
  
    **Сообщение об ошибке**. Контроллер домена < контроллер_домена > для < имя_сервера или имя_точки_входа > становится недоступной.  
  
    **Причина**  
  
    Для поддержания согласованности конфигурации в многосайтовом развертывании важно, чтобы каждым объектом групповой политики управлял один контроллер домена. Когда контроллер домена, который управляет точку входа объекта групповой Политики сервера не станет доступным, параметров конфигурации удаленного доступа невозможно прочитать или изменить.  
  
    **Решение**  
  
    Выполните процедуру «Изменение контроллера домена, который управляет объектами групповой политики сервера» в [2.4. Настройка объектов групповой политики](assetId:///b1960686-a81e-4f48-83f1-cc4ea484df43#ConfigGPOs).  
  
-   **Причина 2**  
  
    **Сообщение об ошибке**. Основной контроллер домена в домене < имя_домена > становится недоступной.  
  
    **Причина**  
  
    Для поддержания согласованности конфигурации в многосайтовом развертывании важно, чтобы каждым объектом групповой политики управлял один контроллер домена. Управление клиентскими объектами групповой политики осуществляется на основном контроллере домена. Если основной контроллер домена недоступен, чтение или изменение параметров конфигурации удаленного доступа невозможно.  
  
    **Решение**  
  
    Выполните процедуру «передача роли эмулятора основного контроллера домена» в [2.4. Настройка объектов групповой политики](assetId:///b1960686-a81e-4f48-83f1-cc4ea484df43#ConfigGPOs).
