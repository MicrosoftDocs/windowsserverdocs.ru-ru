# <a name="step-3-configure-the-multisite-deployment"></a>Шаг 3 Настройка многосайтового развертывания

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

После настройки инфраструктуры многосайтового выполните следующие действия для настройки мультисайтовое развертывание удаленного доступа.  
  
|Задача|Описание|  
|----|--------|  
|3.1. Настроить серверы удаленного доступа|Настройка дополнительных серверов удаленного доступа, Настройка IP-адресов, их присоединения к домену и установка роли удаленного доступа.|  
|3.2. Предоставьте доступ с правами администратора|Предоставление права на дополнительные серверы удаленного доступа, администратору DirectAccess.|  
|3.3. Настройка IP-HTTPS для многосайтового развертывания|Настройте сертификат IP-HTTPS, используемый при многосайтовом развертывании.|  
|3.4. Настройка сервера сетевых расположений для многосайтового развертывания|Настройте сертификат сервера сетевых расположений, используемый при многосайтовом развертывании.|  
|3.5. Настройка клиентов DirectAccess для многосайтового развертывания|Удаление клиентских компьютеров Windows 7 из групп безопасности Windows 8.|  
|3.6. Включение многосайтового развертывания|Включение многосайтового развертывания на первом сервере удаленного доступа.|  
|3.7. Добавление точек входа для многосайтового развертывания|Добавьте дополнительные точки входа для многосайтового развертывания.|  
  
> [!NOTE]  
> В этом разделе приводятся примеры командлетов Windows PowerShell, которые можно использовать для автоматизации некоторых описанных процедур. Дополнительные сведения см. в разделе [Командлеты](https://go.microsoft.com/fwlink/p/?linkid=230693).  
  
## <a name="BKMK_ConfigServer"></a>3.1. Настроить серверы удаленного доступа  

  
### <a name="to-install-the-remote-access-role"></a>Установка роли удаленного доступа  
  
1.  Убедитесь, что каждый сервер удаленного доступа настроен с топологией правильного развертывания (за NAT, одному сетевому интерфейсу Edge) и соответствующие маршруты.  
  
2.  Настройка IP-адреса на каждом сервере удаленного доступа в соответствии с топологии сайтов и схему IP-адресов в вашей организации.  
  
3.  Присоединение каждого сервера удаленного доступа к домену Active Directory.  
  
4.  В консоли диспетчера серверов в **панели мониторинга**, нажмите кнопку **Добавить роли и компоненты**.  
  
5.   Нажмите кнопку **Далее** трижды, чтобы перейти на страницу выбора роли сервера.  
  
6.  На **Выбор ролей сервера** диалоговом окне выберите **удаленного доступа**, а затем нажмите кнопку **Далее**.  
  
7.  Нажмите кнопку **Далее** три раза.  
  
8.  На **Выбор служб ролей** диалоговом окне выберите **DirectAccess и VPN (RAS)** и нажмите кнопку **добавить компоненты**.  
  
9.  Выберите **маршрутизации**выберите **прокси веб-приложения**, нажмите кнопку **добавить компоненты**, а затем нажмите кнопку **Далее**.  
  
10. Нажмите кнопку **Далее**, а затем щелкните **Установить**.  
  
11.  В диалоговом окне **Ход установки** убедитесь в успешном завершении установки, а затем нажмите кнопку **Закрыть**.  
  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>эквивалентные команды Windows PowerShell</em>***  

  
Шаги 1 – 3 должна выполняться вручную, а не выполняются с помощью этого командлета Windows PowerShell.  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
```  
Install-WindowsFeature RemoteAccess -IncludeManagementTools  
```  
  
## <a name="BKMK_Admin"></a>3.2. Предоставьте доступ с правами администратора  
  
#### <a name="to-grant-administrator-permissions"></a>Чтобы предоставить разрешения администратора  
  
1.  На сервере удаленного доступа в точке входа в дополнительных: На **запустить** введите **Управление компьютером**, и нажмите клавишу ВВОД.  
  
2.  В левой области щелкните **локальные пользователи и группы**.  
  
3.  Дважды щелкните **группы**, а затем дважды щелкните **Администраторы**.  
  
4.  На **свойства администратора** диалоговом окне щелкните **добавить**и на **Выбор пользователей, компьютеров, учетных записей служб или групп** диалоговое окно, нажмите кнопку  **Расположения**.  
  
5.  На **расположения** отображаемое в диалоговом окне **расположение** дерева, щелкните место, содержащий учетную запись администратора, DirectAccess и нажмите кнопку **ОК**.  
  
6.  В **введите имена выбираемых объектов**, введите имя пользователя администратора DirectAccess и нажмите кнопку **ОК** дважды.  
  
7.  На **свойства администратора** диалоговом окне щелкните **ОК**.  
  
8.  Закрытие окна управления компьютером.  
  
9. Повторите эту процедуру на всех серверах удаленного доступа, которые будут входить в многосайтовое развертывание.  
  
## <a name="BKMK_IPHTTPS"></a>3.3. Настройка IP-HTTPS для многосайтового развертывания  
На каждом сервере удаленного доступа, которая будет добавлена в многосайтовое развертывание проверить подключение HTTPS к веб-сервера IP-HTTPS необходим сертификат SSL. Чтобы выполнить эту процедуру, вы должны быть членом локальной группы **Администраторы** или аналогичной.  
  
#### <a name="to-obtain-an-ip-https-certificate"></a>Для получения сертификата IP-HTTPS  
  
1.  На каждом сервере удаленного доступа: На **запустить** введите **mmc**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  Нажмите кнопку **файл**, а затем нажмите кнопку **Add/Remove Snap in**.  
  
3.  Нажмите кнопку **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку **Далее**выберите **локального компьютера**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
4.  В дереве консоли оснастки "Сертификаты" откройте раздел **Сертификаты (локальный компьютер)\Личные\Сертификаты**.  
  
5.  Щелкните правой кнопкой **Сертификаты**, наведите указатель на элемент **Все задачи**, а затем щелкните **Запрос нового сертификата**.  
  
6.  Щелкните дважды **Далее** .  
  
7.  На **запрос сертификатов** странице, выберите шаблон сертификата веб-сервера и нажмите кнопку **требуется больше данных для подачи заявки на этот сертификат**.  
  
    Если шаблон сертификата веб-сервер не отображается, убедитесь, что учетная запись компьютера сервера удаленного доступа имеет регистрируют разрешения для шаблона сертификата веб-сервера. Дополнительные сведения см. в разделе [настроить разрешения для шаблона сертификата веб-сервера](https://technet.microsoft.com/library/ee649249(v=ws.10).aspx).  
  
8.  На **субъекта** вкладке **свойства сертификата** отображаемое в диалоговом окне **имя субъекта**, для **тип**выберите **Common имя**.  
  
9. В **значение**, введите полное доменное имя (FQDN) Internet имени сервера удаленного доступа (например, Europe.contoso.com) и нажмите кнопку **добавить**.  
  
10. Нажмите кнопку **OK**, щелкните **Зарегистрировать** и нажмите кнопку **Готово**.  
  
11. В области сведений оснастки сертификатов, убедитесь, что новый сертификат с полным доменным ИМЕНЕМ был зарегистрирован с **назначения** из **проверки подлинности сервера**.  
  
12. Щелкните правой кнопкой мыши сертификат, а затем нажмите кнопку **свойства**.  
  
13. В **понятное имя**, тип **сертификата IP-HTTPS**, а затем нажмите кнопку **ОК**.  
  
    > [!TIP]  
    > Шаги 12 и 13 необязательны, но также упрощают процесс можно было выбрать сертификат для IP-HTTPS, при настройке удаленного доступа.  
  
14. Повторите эту процедуру на всех серверах удаленного доступа в развертывании.  
  
## <a name="BKMK_NLS"></a>3.4. Настройка сервера сетевых расположений для многосайтового развертывания  
Если вы выбрали для настройки сетевого расположения сервера веб-сайта на сервере удаленного доступа при настройке первого сервера, каждый новый сервер удаленного доступа, что добавляемые необходимо настроить с помощью сертификата веб-сервера, который имеет то же имя субъекта, который был выбран для t он сервера сетевых расположений для первого сервера. Каждому серверу требуется сертификат для проверки подлинности подключения для сервера сетевых расположений и клиентскими компьютерами, расположенными за пределами внутренней сети должен иметь возможность разрешить имя веб-сайта в DNS.  
  
#### <a name="to-install-a-certificate-for-network-location"></a>Чтобы установить сертификат для сетевого расположения  
  
1.  На сервере удаленного доступа: На **запустить** введите **mmc**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  Нажмите кнопку **файл**, а затем нажмите кнопку **Add/Remove Snap in**.  
  
3.  Нажмите кнопку **сертификаты**, нажмите кнопку **добавить**, нажмите кнопку **учетная запись компьютера**, нажмите кнопку **Далее**выберите **локального компьютера**, нажмите кнопку **Готово**, а затем нажмите кнопку **ОК**.  
  
4.  В дереве консоли оснастки "Сертификаты" откройте раздел **Сертификаты (локальный компьютер)\Личные\Сертификаты**.  
  
5.  Щелкните правой кнопкой **Сертификаты**, наведите указатель на элемент **Все задачи**, а затем щелкните **Запрос нового сертификата**.  
  
    > [!NOTE]  
    > Можно также импортировать тот же сертификат, который использовался для сервера сетевых расположений для первого сервера удаленного доступа.  
  
6.  Щелкните дважды **Далее** .  
  
7.  На **запрос сертификатов** странице, выберите шаблон сертификата веб-сервера и нажмите кнопку **требуется больше данных для подачи заявки на этот сертификат**.  
  
    Если шаблон сертификата веб-сервер не отображается, убедитесь, что учетная запись компьютера сервера удаленного доступа имеет регистрируют разрешения для шаблона сертификата веб-сервера. Дополнительные сведения см. в разделе [настроить разрешения для шаблона сертификата веб-сервера](https://technet.microsoft.com/library/ee649249(v=ws.10).aspx).  
  
8.  На **субъекта** вкладке **свойства сертификата** отображаемое в диалоговом окне **имя субъекта**, для **тип**выберите **Common имя**.  
  
9. В **значение**, введите полное доменное имя (FQDN), настроенного для сертификата сервера сетевых расположений первого сервера удаленного доступа (например, nls.corp.contoso.com) и нажмите кнопку **добавить**.  
  
10. Нажмите кнопку **OK**, щелкните **Зарегистрировать** и нажмите кнопку **Готово**.  
  
11. В области сведений оснастки сертификатов, убедитесь, что новый сертификат с полным доменным ИМЕНЕМ был зарегистрирован с **назначения** из **проверки подлинности сервера**.  
  
12. Щелкните правой кнопкой мыши сертификат, а затем нажмите кнопку **свойства**.  
  
13. В **понятное имя**, тип **сертификата сетевых расположений**, а затем нажмите кнопку **ОК**.  
  
    > [!TIP]  
    > Шаги 12 и 13 необязательны, но также упрощают процесс можно было выбрать сертификат для сетевого расположения, при настройке удаленного доступа.  
  
14. Повторите эту процедуру на всех серверах удаленного доступа в развертывании.  
  
### <a name="NLS"></a>Создание записей DNS для сервера сетевых расположений  
  
1.  На DNS-сервер: На **запустить** введите **dnsmgmt.msc**, и нажмите клавишу ВВОД.  
  
2.  В левой части **диспетчера DNS** консоли, откройте зону прямого просмотра для внутренней сети. Щелкните правой кнопкой мыши соответствующую зону и выберите **новый узел (A или AAAA)** .  
  
3.  На **новый узел** отображаемое в диалоговом окне **имя (используется имя родительского домена Если оставить пустым)** введите имя, которое было использовано для сервера сетевых расположений для первого сервера удаленного доступа. В **IP-адрес** введите IPv4-адрес сервера удаленного доступа для интрасети и нажмите кнопку **добавить узел**. В диалоговом окне **DNS** щелкните **OK**.  
  
4.  На **новый узел** отображаемое в диалоговом окне **имя (используется имя родительского домена Если оставить пустым)** введите имя, которое было использовано для сервера сетевых расположений для первого сервера удаленного доступа. В **IP-адрес** введите для интрасети IPv6-адрес сервера удаленного доступа и нажмите кнопку **добавить узел**. В диалоговом окне **DNS** щелкните **OK**.  
  
5.  Повторите шаги 3 и 4 для каждого сервера удаленного доступа в развертывании.  
  
6.  Нажмите кнопку **Готово**.  
  
7.  Повторите эту процедуру, прежде чем добавлять серверы как дополнительные точки входа в развертывании.  
  
## <a name="BKMK_Client"></a>3.5. Настройка клиентов DirectAccess для многосайтового развертывания  
Клиентские компьютеры DirectAccess Windows должны быть членами группы безопасности, который определять их связи DirectAccess. Прежде, чем разрешено многосайтовое развертывание, эти группы безопасности может содержать клиентов Windows 8 и клиенты Windows 7 (если был выбран режим соответствующие «нижнего уровня»). После включения Мультисайт с помощью существующей группы безопасности клиента, в режиме одного сервера, преобразуются в группы безопасности для Windows 8 только. После включения многосайтового клиентским компьютерам DirectAccess Windows 7 необходимо переместить соответствующий выделенной Windows 7 клиентских групп безопасности (которые связаны с определенной точки входа), или они не смогут подключиться через DirectAccess. Клиенты Windows 7 необходимо сначала удалить из существующих групп безопасности, которые теперь являются группами безопасности Windows 8. Осторожно.  Клиентские компьютеры Windows 7, которые являются членами клиентских групп безопасности в Windows 7 и Windows 8 потеряет возможности удаленного подключения, и клиенты Windows 7 без SP1 установлены потеряют также корпоративного подключения. Таким образом все клиентские компьютеры Windows 7 необходимо удалить из группы безопасности Windows 8.  
  
#### <a name="remove--windows-7--clients-from-windows-8-security-groups"></a>Удаление клиентов Windows 7 из групп безопасности Windows 8  
  
1.  На основном контроллере домена, нажмите кнопку **запустить**, а затем нажмите кнопку **Active Directory — пользователи и компьютеры**.  
  
2.  Удаление компьютеров из группы безопасности, дважды щелкните группу безопасности и на **< имя_группы > свойства** диалоговом окне щелкните **члены** вкладки.  
  
3.  Выберите клиентский компьютер Windows 7 и нажмите кнопку **удалить**.  
  
4.  Повторите эту процедуру для удаления клиентских компьютеров Windows 7 из групп безопасности Windows 8.  
  
> [!IMPORTANT]  
> При включении конфигурации удаленного доступа с несколькими узлами всех клиентских компьютеров (Windows 7 и Windows 8) будут потеряны возможности удаленного подключения, пока они не может подключиться к корпоративной сети напрямую или через подключение VPN, чтобы обновить их групповых политик. Это является истинным при включении функции Мультисайта в первый раз, а также при отключении несколькими сайтами.  
  
## <a name="BKMK_Enable"></a>3.6. Включение многосайтового развертывания  
Чтобы настроить многосайтового развертывания, включите функцию многосайтового на существующий сервер удаленного доступа. Перед включением многосайтового в развертывании, убедитесь, что у вас есть следующие сведения:  
  
1.  Параметры подсистемы балансировки нагрузки глобального и IP-адреса, если вы хотите загрузить балансировки подключения клиентов DirectAccess на всех точек входа в вашем развертывании.  
  
2.  Безопасность группы (групп) содержащего клиентские компьютеры Windows 7 для первой точки входа в развертывании, если вы хотите включить удаленный доступ для 7 Windows клиентские компьютеры.  
  
3.  Группировать имена объектов политики, если требуется использовать нестандартные объекты групповой политики, которой применяются на клиентских компьютерах Windows 7 для первой точки входа в развертывании, если требуется поддержка для клиентских компьютеров Windows 7.  
  
### <a name="EnabledMultisite"></a>Чтобы включить конфигурации с несколькими узлами  
  
1.  На существующий сервер удаленного доступа: На **запустить** введите **RAMgmtUI.exe**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом, нажмите кнопку **конфигурации**, а затем в **задачи** области нажмите кнопку **Включение многосайтового**.  
  
3.  В **включения многосайтового развертывания** мастера **перед началом работы** щелкните **Далее**.  
  
4.  На **имя развертывания** странице **многосайтового развертывания имя**, введите имя для развертывания. В **Знакомство с именем**, введите имя для идентификации первую точку входа, который является текущим сервером удаленного доступа и нажмите кнопку **Далее**.  
  
5.  На **выбирать точки входа** выполните одно из следующих:  
  
    -   Нажмите кнопку **назначения точки входа автоматически, и позволяет клиентам выбирать вручную** автоматически направлять клиентские компьютеры в наиболее подходящем точку входа, позволяя клиентских компьютеров, чтобы вручную выбрать точку входа. Выбор точки ручной ввод доступен только для компьютеров под управлением Windows 8. Нажмите кнопку **Далее**.  
  
    -   Нажмите кнопку **автоматически назначить точек входа** автоматически направлять клиентские компьютеры в наиболее подходящем точку входа, а затем нажмите кнопку **Далее**.  
  
6.  На **глобальной балансировки нагрузки** выполните одно из следующих:  
  
    -   Нажмите кнопку **нет, не используйте глобальная Балансировка нагрузки** Если вы не хотите использовать глобальная Балансировка нагрузки и нажмите кнопку **Далее**.  
  
        > [!NOTE]  
        > При выборе этого параметра клиентские компьютеры автоматически подключаться к ближайшей точки входа.  
  
    -   Нажмите кнопку **Да, использование глобальной балансировки нагрузки** Если вы хотите загрузить сбалансировать трафик глобально между всех точек входа. В **введите полное доменное имя для использования всех точек входа глобальная Балансировка**, введите полное доменное имя, глобальная Балансировка и в **введите IP-адрес для этой точки глобальная Балансировка** , содержащий первый Введите IP-адрес для этой точки глобальная Балансировка сервера удаленного доступа и нажмите кнопку **Далее**.  
  
7.  На **поддержка клиента** выполните одно из следующих:  
  
    -   Чтобы ограничить доступ для клиентских компьютеров под управлением Windows 8 или более поздних операционных системах, щелкните **ограничить доступ для клиентских компьютеров под управлением Windows 8 или более поздней версии операционной системы**, а затем нажмите кнопку **Далее**.  
  
    -   Чтобы разрешить клиентским компьютерам под управлением Windows 7 для доступа к этой точке входа, щелкните **разрешить клиентским компьютерам под управлением Windows 7 для доступа к этой точке входа**и нажмите кнопку **добавить**. На **Выбор групп** диалоговое окно, выберите группы безопасности, содержащих клиентские компьютеры Windows 7, нажмите кнопку **ОК**, а затем нажмите кнопку **Далее**.  
  
8.  На **параметров объекта групповой Политики клиента** странице, не изменяя объект групповой Политики для Windows 7 по умолчанию клиентские компьютеры для данной точки входа, введите имя объекта групповой Политики удаленного доступа автоматически создавать, или нажмите кнопку **Обзор** для Найдите клиентским компьютерам объект групповой Политики для Windows 7 и нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > -   **Параметров объекта групповой Политики клиента** страница появляется только при настройке точки входа, чтобы компьютеры могли получить доступ к точке входа клиент Windows 7.  
    > -   Можно нажать кнопку **проверить объекты групповой политики** обеспечить соответствующие разрешения для выбранного объекта групповой Политики или объекты групповой политики для данной точки входа. Если объект групповой Политики не существует и будет автоматически создан, затем создать и связать разрешения не требуются. В случае, когда объекты групповой политики были созданы вручную, затем измените, изменения уровня безопасности и разрешения delete необходимы.  
  
9. На **Сводка** щелкните **зафиксировать**.  
  
10. На **Включение многосайтового развертывания** диалоговом окне щелкните **закрыть** и мастер включения многосайтового развертывания, щелкните **закрыть**.  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>эквивалентные команды Windows PowerShell</em>***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
Чтобы включить многосайтовое развертывание с именем «Contoso» на первую точку входа с именем «Edge1-US». Развертывание позволяет клиентам, чтобы вручную выбрать точку входа и не использует глобальной подсистемы балансировки нагрузки.  
  
```  
Enable-DAMultiSite -Name 'Contoso' -EntryPointName 'Edge1-US' -ManualEntryPointSelectionAllowed 'Enabled'  
```  
  
Чтобы разрешить Windows 7 клиентские компьютеры получают доступ через первую точку входа через группу безопасности DA_Clients_US и с помощью DA_W7_Clients_GPO_US объекта групповой Политики.  
  
```  
Add-DAClient -EntrypointName 'Edge1-US' -DownlevelSecurityGroupNameList @('corp.contoso.com\DA_Clients_US') -DownlevelGpoName @('corp.contoso.com\DA_W7_Clients_GPO_US)  
```  
  
## <a name="BKMK_EntryPoint"></a>3.7. Добавление точек входа для многосайтового развертывания  
После включения многосайтового в развертывании, можно добавить дополнительные точки входа с помощью точки входа мастера добавления. Прежде чем добавлять точки входа, убедитесь, что у вас есть следующие сведения:  
  
-   Глобальная нагрузки IP-адреса для каждой новой записи точки, если вы используете глобальная Балансировка нагрузки.  
  
-   Группы безопасности, входят компьютеры клиентов Windows 7 для каждой точки входа, которые будут добавлены, чтобы настройка удаленного доступа для 7 Windows на клиентских компьютерах.  
  
-   Имена объектов политики, группы, в случае необходимости для использования не по умолчанию объекты групповой политики, которой применяются на клиентских компьютерах Windows 7 для каждой точки входа, которые будут добавлены, если требуется поддержка для клиентских компьютеров Windows 7.  
  
-   В случае, где развернут IPv6 в сети организации необходимо подготовить IP-HTTPS-префикс для новой точки входа.  
  
### <a name="AddEP"></a>Добавление точек входа в многосайтовое развертывание  
  
1.  На существующий сервер удаленного доступа: На **запустить** введите **RAMgmtUI.exe**, и нажмите клавишу ВВОД. Если появится диалоговое окно **контроля учетных записей**, подтвердите, что отображаемое в нем действие именно то, которое требуется, и нажмите кнопку **Да**.  
  
2.  В консоли управления удаленным доступом щелкните **конфигурации**, а затем в **задачи** панели щелкните **добавьте точку входа**.  
  
3.  В окне добавления точку входа мастера, на **сведения о точках входа** странице **сервера удаленного доступа**, укажите полное доменное имя (FQDN) сервера, чтобы добавить. В **имя точки входа**, введите имя точки входа и нажмите кнопку **Далее**.  
  
4.  На **глобальной балансировки параметрах нагрузочного** странице, введите глобальная Балансировка IP-адрес этой точки входа и нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > **Глобальной балансировки параметрах нагрузочного** страница появляется только в том случае, если конфигурации многосайтового развертывания использует глобальной подсистемы балансировки нагрузки.  
  
5.  На **топологии сети** щелкните топологии, которая соответствует с топологией сети сервера удаленного доступа, который добавляется и нажмите кнопку **Далее**.  
  
6.  На **сетевое имя или IP-адрес** странице **введите общедоступное имя или IP-адрес, используемый клиентами для подключения к серверу удаленного доступа**, введите общедоступное имя или IP-адрес, используемый клиентами для подключения к Сервер удаленного доступа. Общедоступное имя соответствует с именем субъекта сертификата IP-HTTPS. В случае, где была реализована пограничной топологии сети IP-адрес — это внешнего адаптера сервера удаленного доступа. Нажмите кнопку **Далее**.  
  
7.  На **сетевых адаптеров** выполните одно из следующих:  
  
    -   Если вы собираетесь выполнять развертывание топологии с двумя сетевыми адаптерами в **внешнего адаптера**, выберите адаптер, подключенный к внешней сети. В **внутренний адаптер**, выберите адаптер, подключенный к внутренней сети.  
  
    -   Если вы собираетесь выполнять развертывание топологии с одним сетевым адаптером в **сетевой адаптер**, выберите адаптер, подключенный к внутренней сети.  
  
8.  На **сетевых адаптеров** странице **выберите сертификат, используемый для проверки подлинности подключений IP-HTTPS**, нажмите кнопку **Обзор** чтобы найти и выбрать сертификат IP-HTTPS. Нажмите кнопку **Далее**.  
  
9. Если IPv6 в корпоративной сети, настраивается на **Конфигурация префикса** странице **префикс IPv6, назначенный на клиентских компьютерах**, введите IP-HTTPS-префикс для назначения IPv6-адреса клиента DirectAccess компьютеры и нажмите кнопку **Далее**.  
  
10. На **поддержка клиента** выполните одно из следующих:  
  
    -   Чтобы ограничить доступ для клиентских компьютеров под управлением Windows 8 или более поздних операционных системах, щелкните **ограничить доступ для клиентских компьютеров под управлением Windows 8 или более поздней версии операционной системы**, а затем нажмите кнопку **Далее**.  
  
    -   Чтобы разрешить клиентским компьютерам под управлением Windows 7 для доступа к этой точке входа, щелкните **разрешить клиентским компьютерам под управлением Windows 7 для доступа к этой точке входа**и нажмите кнопку **добавить**. На **Выбор групп** диалоговом окне выберите группы безопасности, содержащие клиентские компьютеры Windows 7, которые будут подключаться к этой точке входа, щелкните **ОК**и нажмите кнопку **Далее**.  
  
11. На **параметров объекта групповой Политики клиента** странице, не изменяя объект групповой Политики для Windows 7 по умолчанию клиентские компьютеры для данной точки входа, введите имя объекта групповой Политики удаленного доступа автоматически создавать, или нажмите кнопку **Обзор** для Найдите клиентским компьютерам объект групповой Политики для Windows 7 и нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > -   **Параметров объекта групповой Политики клиента** страница появляется только при настройке точки входа, чтобы компьютеры могли получить доступ к точке входа клиент Windows 7.  
    > -   Можно нажать кнопку **проверить объекты групповой политики** обеспечить соответствующие разрешения для выбранного объекта групповой Политики или объекты групповой политики для данной точки входа. Если объект групповой Политики не существует и будет автоматически создан, затем создать и связать разрешения не требуются. В случае, когда объекты групповой политики были созданы вручную, затем измените, изменения уровня безопасности и разрешения delete необходимы.  
  
12. На **параметры объекта групповой Политики сервера** странице, не изменяя объект групповой Политики по умолчанию для этого сервера удаленного доступа, введите имя объекта групповой Политики удаленного доступа автоматически создавать, или нажмите кнопку **Обзор** искомый объект групповой Политики для Этот сервер, а затем щелкните **Далее**.  
  
13. На **сервера сетевых расположений** щелкните **Обзор** выберите сертификат для сетевого расположения сервера веб-сайта на сервере удаленного доступа, а затем нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > **Сервера сетевых расположений** страница появляется только при выполнении веб-сайта сервера сетевых расположение на сервере удаленного доступа.  
  
14. На **Сводка** странице просмотрите параметры точки входа и нажмите кнопку **зафиксировать**.  
  
15. На **точки входа добавление** диалоговом окне щелкните **закрыть** и точки входа мастера добавления, щелкните **закрыть**.  
  
    > [!NOTE]  
    > Если точка входа, который был добавлен в лесу, отличном от существующей точки входа или клиентских компьютеров, то она необходима для щелкните **обновить серверы управления** в **задачи** области для обнаружения контроллеры домена и System Center Configuration Manager в новом лесу.  
  
16. Повторите эту процедуру шаг 2 для каждой точки входа, который требуется добавить в многосайтовое развертывание.  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>эквивалентные команды Windows PowerShell</em>***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
Чтобы добавить edge2 компьютера из домена corp2, что второй точки входа с именем Edge2 Европе. Конфигурация точки входа: префикс IPv6 клиента "2001:db8:2:2000:: / 64", подключение к адрес (IP-HTTPS сертификата на компьютере edge2) «edge2.contoso.com», объект групповой Политики сервера с именем «DirectAccess Server параметры – Edge2-Europe» и внутренний и внешние интерфейсы, с именем Интернета и Corpnet2:  
  
```  
Add-DAEntryPoint -RemoteAccessServer 'edge2.corp2.corp.contoso.com' -Name 'Edge2-Europe' -ClientIPv6Prefix '2001:db8:2:2000::/64' -ConnectToAddress 'Europe.contoso.com' -ServerGpoName 'corp2.corp.contoso.com\DirectAccess Server Settings - Edge2-Europe' -InternetInterface 'Internet' -InternalInterface 'Corpnet2'  
```  
  
Чтобы разрешить Windows 7 клиентские компьютеры получают доступ через второй точки входа через группу безопасности DA_Clients_Europe и с помощью DA_W7_Clients_GPO_Europe объекта групповой Политики.  
  
```  
Add-DAClient -EntrypointName 'Edge2-Europe' -DownlevelGpoName @('corp.contoso.com\ DA_W7_Clients_GPO_Europe') -DownlevelSecurityGroupNameList @('corp.contoso.com\DA_Clients_Europe')  
```
