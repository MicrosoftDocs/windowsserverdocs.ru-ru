# <a name="step-2-configure-the-multisite-infrastructure"></a>Шаг 2 Настройка многосайтового инфраструктуры

>Область применения. Windows Server 2012 R2, Windows Server 2012

Чтобы настроить в многосайтовое развертывание, существует ряд действий, необходимых, чтобы изменить параметры инфраструктуры сети, включая: Настройка дополнительных сайтов Active Directory и контроллеры домена, Настройка дополнительной защиты групп и настройка Объекты групповой политики (GPO), если вы не используете автоматически настроить объекты групповой политики.  
  
|Задача|Описание|  
|----|--------|  
|2.1. Настройте дополнительные сайты Active Directory|Настройте дополнительные сайты Active Directory для развертывания.|  
|2.2. Настройка дополнительных контроллеров домена|Настройка дополнительных контроллеров домена Active Directory при необходимости.|  
|2.3. Настройка групп безопасности|Настройка групп безопасности для клиентских компьютеров Windows 7.|  
|2.4. Настройка объектов групповой политики|Настройте дополнительные объекты групповой политики, согласно требованиям.|  
  
> [!NOTE]  
> В этом разделе приводятся примеры командлетов Windows PowerShell, которые можно использовать для автоматизации некоторых описанных процедур. Дополнительные сведения см. в разделе [Командлеты](https://go.microsoft.com/fwlink/p/?linkid=230693).  
  
## <a name="BKMK_ConfigAD"></a>2.1. Настройте дополнительные сайты Active Directory  
Все точки входа может находиться на одном сайте Active Directory. Поэтому хотя бы один сайт Active Directory не требуется для реализации серверов удаленного доступа в конфигурации с несколькими узлами. Эта процедура используется в том случае, если вам нужно создать первый сайт Active Directory, или в том случае, если вы желаете использовать дополнительные сайты Active Directory для многосайтового развертывания. Используйте оснастку «Active Directory — сайты и службы» для создания сайтов в вашей организации «"сети s".  

Членство в группе **"Администраторы предприятия"** группу в лесу или **"Администраторы домена"** группы в корневом домене леса или эквивалент, как минимум необходим для выполнения этой процедуры. Просмотрите сведения об использовании соответствующих учетных записей и членства в группах в [локальные и доменные группы по умолчанию](https://go.microsoft.com/fwlink/?LinkId=83477).  

Дополнительные сведения см. в разделе [Добавление сайта в лесу](https://technet.microsoft.com/library/cc732761.aspx).  

### <a name="to-configure-additional-active-directory-sites"></a>Чтобы настроить дополнительные сайты Active Directory  
  
1.  На основном контроллере домена, нажмите кнопку **запустить**, а затем нажмите кнопку **Active Directory — сайты и службы**.  
  
2.  В консоли "Active Directory — сайты и службы" в дереве консоли щелкните правой кнопкой мыши **сайты**, а затем нажмите кнопку **новый сайт**.  
  
3.  На **новый объект — сайт** отображаемое в диалоговом окне **имя** введите имя для нового сайта.  
  
4.  В **имя ссылки**, щелкните объект связи сайтов и нажмите кнопку **ОК** дважды.  
  
5.  В дереве консоли разверните **сайты**, щелкните правой кнопкой мыши **подсетей**, а затем нажмите кнопку **новую подсеть**.  
  
6.  На **новый объект — подсеть** диалогового **префикс**, введите префикс подсети IPv4 или IPv6, в **выберите объект сайта для этого префикса** выберите сайт, который требуется связать с этой подсетью и нажмите кнопку **ОК**.  
  
7.  Повторите шаги 5 и 6, пока не будут созданы все подсети, необходимых для развертывания.  
  
8.  Закройте Active Directory — сайты и службы.  
  
![Windows PowerShell](../../../../media/Step-2-Configure-the-Multisite-Infrastructure/PowerShellLogoSmall.gif)***<em>эквивалентные команды Windows PowerShell</em>***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
Чтобы установить компонент Windows «Модуль Active Directory для Windows PowerShell»:  
  
```  
Install-WindowsFeature "Name RSAT-AD-PowerShell  
```  
  
или добавьте «Active Directory оснастка PowerShell» через OptionalFeatures.  
  
Если приведенные ниже командлеты под управлением Windows 7" или Windows Server 2008 R2, необходимо импортировать модуль PowerShell для Active Directory:  
  
```  
Import-Module ActiveDirectory  
```  
  
Чтобы настроить сайт Active Directory с именем «Секунду-Site», с помощью встроенных DEFAULTIPSITELINK:  
  
```  
New-ADReplicationSite -Name "Second-Site"  
Set-ADReplicationSiteLink -Identity "DEFAULTIPSITELINK" -sitesIncluded @{Add="Second-Site"}  
```  
  
Настройка подсетей IPv4 и IPv6 для второго сайта:  
  
```  
New-ADReplicationSubnet -Name "10.2.0.0/24" -Site "Second-Site"  
New-ADReplicationSubnet -Name "2001:db8:2::/64" -Site "Second-Site"  
```  
  
## <a name="BKMK_AddDC"></a>2.2. Настройка дополнительных контроллеров домена  
Чтобы настроить многосайтового развертывания в одном домене, рекомендуется наличие по крайней мере один контроллер домена с возможностью записи для каждого узла в развертывании.  
  
Для выполнения этой процедуры, как минимум, необходимо быть членом группы "Администраторы домена" в домене, в котором установлен контроллер домена.  
  
Дополнительные сведения см. в разделе [установка дополнительного контроллера домена](https://technet.microsoft.com/library/cc733027.aspx).
  
### <a name="to-configure-additional-domain-controllers"></a>Чтобы настроить дополнительные контроллеры домена  
  
1.  На сервере, который будет выступать в качестве контроллера домена в **диспетчера серверов**на **панели мониторинга**, нажмите кнопку **Добавить роли и компоненты**.  
  
2.  Нажмите кнопку **Далее** три раза, чтобы получить экран выбора ролей сервера  
  
3.  На **Выбор ролей сервера** выберите **доменных служб Active Directory**. Нажмите кнопку **добавить компоненты** при появлении запроса, а затем нажмите кнопку **Далее** три раза.  
  
4.  На странице **подтверждения** нажмите кнопку **Установить**.  
  
5.  По завершении установки нажмите кнопку **повысить уровень этого сервера до контроллера домена**.  
  
6.  В Active Directory мастер настройки доменных служб на **конфигурация развертывания** щелкните **добавить контроллер домена в существующий домен**.  
  
7.  В **домена**, введите имя домена, имя, например corp.contoso.com.  
  
8.  В разделе **передать учетные данные для этой операции**, нажмите кнопку **изменение**. На **безопасности Windows** диалоговом окне укажите имя пользователя и пароль для учетной записи, можно установить дополнительный контроллер домена. Чтобы установить дополнительный контроллер домена, необходимо быть членом группы "Администраторы предприятия" или "Администраторы домена". По завершении ввода учетных данных нажмите кнопку **Далее**.  
  
9. На **параметры контроллера домена** выполните следующее:  
  
    1.  Задайте следующие параметры:  
  
        -   **Сервер доменных имен (DNS)** «этот параметр выбран по умолчанию, чтобы контроллер домена может выступать в качестве сервера доменных имен (DNS). Если контроллер домена не должен работать в качестве DNS-сервера, снимите флажок.  
  
            Если роль DNS-сервера не установлен на эмулятор основного контроллера домена (PDC) в корневом домене леса, параметр для установки DNS-сервера на дополнительный контроллер домена недоступен. В качестве решения в этой ситуации можно установить роль DNS-сервера до или после установки AD DS.  
  
            > [!NOTE]  
            > Если выбран вариант установки DNS-сервер, может появиться сообщение о том, что не удалось создать делегирование DNS для DNS-сервера и что следует вручную создать делегирование DNS на DNS-сервер, чтобы обеспечить надежное разрешение имен. Если вы устанавливаете дополнительный контроллер домена в корневом домене леса или корневой домен дерева, у вас нет создания делегирования DNS. В этом случае щелкните **Да** и пропускают сообщения.  
  
        -   **Глобальный каталог (GC)** «этот параметр выбран по умолчанию. Он добавляет глобальный каталог (доступные только для чтения разделы) в контроллер домена и позволяет использовать поиск по глобальному каталогу.  
  
        -   **Контроллер домена только для чтения (RODC)** «этот параметр не выбран по умолчанию. Он выполняет дополнительный контроллер домена только для чтения; то есть становится контроллером домена RODC.  
  
    2.  В **имя_сайта**, выберите сайт из списка.  
  
    3.  В разделе **введите пароль режима восстановления служб каталогов (DSRM)** в **пароль** и **подтверждение пароля**дважды введите надежный пароль и нажмите кнопку  **Далее**. Этот пароль необходимо использовать для запуска AD DS в режиме DSRM для задач, которые должны выполняться в автономном режиме.  
  
10. На **параметры DNS** выберите **обновить DNS-делегирование** флажок, если вы хотите обновить делегирование DNS во время установки роли и нажмите кнопку **Далее**.  
  
11. На **Дополнительные параметры** странице введите или перейдите к папкам тома и папки для файла базы данных, файлы журналов службы каталогов и системные файлы том (SYSVOL). Укажите параметры репликации при необходимости и нажмите кнопку **Далее**.  
  
12. На **просмотреть параметры** странице просмотрите параметры установки и нажмите кнопку **Далее**.  
  
13. На **проверка предварительных требований** странице, после проверки необходимых компонентов нажмите кнопку **установить**.  
  
14. Дождитесь завершения работы мастера конфигурации и нажмите кнопку **закрыть**.  
  
15. Перезагрузите компьютер, если его не удалось перезапустить автоматически.  
  
## <a name="BKMK_ConfigSG"></a>2.3. Настройка групп безопасности  
Многосайтовое развертывание требует группу дополнительной защиты для клиентских компьютеров Windows 7 для каждой точки входа в развертывание, которое разрешает доступ к клиентским компьютерам Windows 7. Если имеется несколько доменов, которые входят компьютеры клиентов Windows 7, рекомендуется создать группу безопасности в каждом домене, для одной точки входа. Кроме того можно использовать одну универсальную группу безопасности входят компьютеры клиентов обоих доменов. Например, в среде с двумя доменами, если вы хотите разрешить доступ к клиентским компьютерам Windows 7 в точек входа, 1 и 3, но не в записи указывают 2, создайте две новые группы безопасности должен содержать клиентские компьютеры Windows 7 для каждой точки входа в каждом из  домены.  
  
### <a name="to-configure-additional-security-groups"></a>Чтобы настроить дополнительную защиту группы  
  
1.  На основном контроллере домена, нажмите кнопку **запустить**, а затем нажмите кнопку **Active Directory — пользователи и компьютеры**.  
  
2.  В дереве консоли щелкните правой кнопкой мыши папку, в которой вы хотите добавить новую группу, к примеру, corp.contoso.com/Users. Подведите курсор к пункту **Создать** и щелкните **Группа**.  
  
3.  На **новый объект - группа** диалогового **имя группы**, введите имя новой группы, к примеру, Win7_Clients_Entrypoint1.  
  
4.  В разделе **область действия группы**, нажмите кнопку **универсальной**в разделе **тип группы**, нажмите кнопку **безопасности**, а затем нажмите кнопку **ОК**.  
  
5.  Чтобы добавить компьютеры в новую группу безопасности, дважды щелкните группу безопасности и на **< имя_группы > свойства** диалоговом окне щелкните **члены** вкладки.  
  
6.  На вкладке **Члены группы** щелкните **Добавить**.  
  
7.  Выберите компьютеры с Windows 7 необходимо добавить в эту группу безопасности, а затем нажмите кнопку **ОК**.  
  
8.  Повторите эту процедуру, чтобы создать группу безопасности для каждой точки входа, при необходимости.  
  
![Windows PowerShell](../../../../media/Step-2-Configure-the-Multisite-Infrastructure/PowerShellLogoSmall.gif)***<em>эквивалентные команды Windows PowerShell</em>***  
  
Следующие командлеты Windows PowerShell выполняют ту же функцию, что и предыдущая процедура. Вводите каждый командлет в одной строке, несмотря на то, что здесь они могут отображаться разбитыми на несколько строк из-за ограничений форматирования.  
  
Чтобы установить компонент Windows «Модуль Active Directory для Windows PowerShell»:  
  
```  
Install-WindowsFeature "Name RSAT-AD-PowerShell  
```  
  
или добавьте «Active Directory оснастка PowerShell» через OptionalFeatures.  
  
Если приведенные ниже командлеты под управлением Windows 7" или Windows Server 2008 R2, необходимо импортировать модуль PowerShell для Active Directory:  
  
```  
Import-Module ActiveDirectory  
```  
  
Чтобы настроить группу безопасности с именем Win7_Clients_Entrypoint1 и добавить клиентский компьютер с именем CLIENT2:  
  
```  
New-ADGroup -GroupScope universal -Name Win7_Clients_Entrypoint1  
Add-ADGroupMember -Identity Win7_Clients_Entrypoint1 -Members CLIENT2$  
```  
  
## <a name="ConfigGPOs"></a>2.4. Настройка объектов групповой политики  
Многосайтового развертывания удаленного доступа требуются следующие объекты групповой политики:  
  
-   Объект групповой Политики для каждой точки входа для сервера удаленного доступа.  
  
-   Объект групповой Политики для всех клиентских компьютерах Windows 8 для каждого домена.  
  
-   Объект групповой Политики в каждом домене, содержащем клиентские компьютеры Windows 7 для каждой точки входа настроен для поддержки Windows 7 клиентов.  
  
    > [!NOTE]  
    > Если у вас все клиентские компьютеры Windows 7, вам не обязательно для создания объектов групповой политики для Windows 7 для компьютеров.  
  
При настройке удаленного доступа мастер автоматически создаст необходимые объекты групповой политики, если они не «t уже существует. Если у вас необходимые разрешения для создания объектов групповой политики, они должны быть созданы до настройки удаленного доступа. Администратору DirectAccess должны иметь полные права доступа объекты групповой политики (редактирование + изменения уровня безопасности + delete).  
  
> [!IMPORTANT]  
> После ручного создания объектов групповой политики для удаленного доступа необходимо разрешить достаточно времени для Active Directory и репликации DFS на контроллер домена в сайте Active Directory, которая связана с сервера удаленного доступа. Если удаленного доступа автоматически созданные объекты групповой политики, не нужно ждать не требуется.  
  
Чтобы создать объекты групповой политики, см. в разделе [Создание и изменение объекта групповой политики](https://technet.microsoft.com/library/cc754740.aspx).  
  
### <a name="DCMaintandDowntime"></a>Обслуживание контроллеров домена и время простоя  
Когда контроллер домена под управлением в роли эмулятора основного контроллера домена или контроллеров домена, управление объектами групповой политики сервера времени простоя, не существует возможность загружать или изменять параметры удаленного доступа. Это не влияет на возможность подключения клиентов Если доступны другие контроллеры домена.  
  
Чтобы загрузить или изменить настройки удаленного доступа, можно перенести роль эмулятора основного контроллера домена для другого контроллера домена для клиента или приложения сервера объектов групповой политики; для объектов групповой политики сервера измените контроллеры домена, управление GPO сервера.  
  
> [!IMPORTANT]  
> Эта операция может выполняться только администратором домена. Влияние изменения на основном контроллере домена не сводится к удаленного доступа; Таким образом Будьте осторожны при передачи роли эмулятора основного контроллера домена.  
  
> [!NOTE]  
> Прежде чем изменять сопоставления контроллера домена, убедитесь, что все объекты групповой политики в развертывании удаленного доступа были реплицированы на все контроллеры домена в домене. Если объект групповой Политики не синхронизированы, последние изменения конфигурации могут быть потеряны после изменения сопоставления контроллера домена, что может привести к повреждению конфигурации. Чтобы проверить синхронизацию объекта групповой Политики, см. в разделе [проверка статуса инфраструктуры групповой политики](https://technet.microsoft.com/library/jj134176.aspx).  
  
#### <a name="TransferPDC"></a>Для передачи роли эмулятора основного контроллера домена  
  
1.  На **запустить** введите**dsa.msc**, и нажмите клавишу ВВОД.  
  
2.  В левой области консоли Active Directory — пользователи и компьютеры, щелкните правой кнопкой мыши **Active Directory — пользователи и компьютеры**, а затем нажмите кнопку **Сменить контроллер домена**. В диалоговом окне изменения сервера каталогов щелкните **этот контроллер домена или экземпляр AD LDS**в списке щелкните контроллер домена, который будет передана роль, а затем нажмите кнопку **ОК**.  
  
    > [!NOTE]  
    > Этот шаг необходимо выполнить, если вы не на контроллере домена, к которому вы хотите передать роль. Не выполняйте этот шаг, если вы уже подключены к контроллеру домена, к которому вы хотите передать роль.  
  
3.  В дереве консоли щелкните правой кнопкой мыши **Active Directory — пользователи и компьютеры**, пункты **все задачи**, а затем нажмите кнопку **Хозяева операций**.  
  
4.  В диалоговом окне Хозяева операций щелкните **PDC** , а затем щелкните **изменение**.  
  
5.  Нажмите кнопку **Да** подтверждение передачи роли, а затем нажмите кнопку **закрыть**.  
  
#### <a name="ChangeDC"></a>Чтобы изменить контроллер домена, управляющий объектами групповой политики сервера  
  
-   Выполните командлет Windows PowerShell `HYPERLINK "https://technet.microsoft.com/library/hh918412.aspx" Set-DAEntryPointDC` на сервере удаленного доступа и укажите имя контроллера домена недоступны для *ExistingDC* параметра. Эта команда изменяет связь контроллера домена для объектов групповой политики сервера точки входа, которые в настоящее время управляются контролере домена.  
  
    -   Чтобы заменить контроллер домена недоступен «dc1.corp.contoso.com» с контроллера домена «dc2.corp.contoso.com», сделайте следующее:  
  
        ```  
        Set-DAEntryPointDC "ExistingDC 'dc1.corp.contoso.com' "NewDC 'dc2.corp.contoso.com' "ErrorAction Inquire  
        ```  
  
    -   Чтобы заменить контроллер домена недоступен «dc1.corp.contoso.com» с контроллером домена в ближайшем узле Active Directory на сервер удаленного доступа «DA1.corp.contoso.com», сделайте следующее:  
  
        ```  
        Set-DAEntryPointDC "ExistingDC 'dc1.corp.contoso.com' "ComputerName 'DA1.corp.contoso.com' "ErrorAction Inquire  
        ```  
  
### <a name="ChangeTwoDCs"></a>Измените два или несколько контроллеров домена, которые управляют объектами групповой политики сервера  
Минимальное количество вариантов два или несколько контроллеров домена, которые управляют объектами групповой политики сервера будут недоступны. В этом случае для изменения сопоставления контроллера домена для объектов групповой политики сервера необходимы дополнительные действия.  
  
Сведения о сопоставлении контроллера домена хранится в реестре на серверах удаленного доступа и в все GPO сервера. В следующем примере существуют две точки входа с двумя серверами удаленного доступа, «DA1» в «1" и «DA2» точки входа в «2 точки входа». На контроллере домена «DC1», пока сервер объекта групповой Политики из управляется групповой Политики сервера «Точки входа 1» «в контроллере домена «DC2» управляется точки входа 2». «DC1» и «DC2» недоступны. Третий контроллер домена по-прежнему доступен в домене «DC3», и данные из «DC1» и «DC2» уже было реплицировано «DC3».  
  
![Настройка инфраструктуры Мультисайт](../../../../media/Step-2-Configure-the-Multisite-Infrastructure/DCAssoc1.png)  
  
##### <a name="to-change-two-or-more-domain-controllers-that-manage-server-gpos"></a>Чтобы изменить два или несколько контроллеров домена, которые управляют объектами групповой политики сервера  
  
1.  Чтобы заменить контроллер домена недоступен «DC2» с контроллера домена «DC3», выполните следующую команду:  
  
    ```  
    Set-DAEntryPointDC "ExistingDC 'DC2' "NewDC 'DC3' "ComputerName 'DA2' "ErrorAction Continue  
    ```  
  
    Эта команда обновляет домена контроллера ассоциации для групповой Политики в реестре сервера «Точка входа 2» DA2 и на сервере «2 точки входа» объекта групповой Политики Тем не менее он не обновляет групповой Политики сервера «1 точки входа», так как недоступен контроллер домена, который управляет его.  
  
    > [!TIP]  
    > Эта команда использует значение Continue *ErrorAction* параметр, который обновляет групповой Политики сервера «Точка входа 2» несмотря на сбой при обновлении объекта групповой Политики «1 точки входа» сервера.  
  
    Итоговая конфигурация показан на следующей схеме.  
  
    ![Windows PowerShell](../../../../media/Step-2-Configure-the-Multisite-Infrastructure/DCAssoc2.png)  
  
2.  Чтобы заменить контроллер домена недоступен «DC1» с контроллера домена «DC3», выполните следующую команду:  
  
    ```  
    Set-DAEntryPointDC "ExistingDC 'DC1' "NewDC 'DC3' "ComputerName 'DA2' "ErrorAction Continue  
    ```  
  
    Эта команда обновляет ассоциации контроллера домена для объектов групповой политики для сервера «Точка входа 1» объект групповой Политики сервера в реестре DA1, а также в «1 точки входа» и «2 точки входа». Итоговая конфигурация показан на следующей схеме.  
  
    ![Windows PowerShell](../../../../media/Step-2-Configure-the-Multisite-Infrastructure/DCAssoc3.png)  
  
3.  Для синхронизации ассоциации контроллера домена для групповой Политики на сервере «1 точки входа», выполните команду, чтобы заменить «DC2» с «DC3» и укажите сервер удаленного доступа, объект групповой Политики сервера не синхронизирован, в этом случае объект групповой Политики сервера «Точка входа 2» «DA1», для  *ComputerName* параметра.  
  
    ```  
    Set-DAEntryPointDC "ExistingDC 'DC2' "NewDC 'DC3' "ComputerName 'DA1' "ErrorAction Continue  
    ```  
  
    На следующей схеме показан окончательной конфигурации.

### <a name="ConfigDistOptimization"></a>Оптимизация конфигурации распространения  
При внесении изменений конфигурации, изменения применяются только после распространения объектов групповой политики сервера на серверах удаленного доступа. Для сокращения времени распространения конфигурации удаленного доступа автоматически выбирает контроллер домена с возможностью записи, который представляет ГИПЕРССЫЛКУ "<https://technet.microsoft.com/library/cc978016.aspx>" ближайший сервер удаленного доступа при создании своего сервера объекта групповой Политики.  
  
В некоторых случаях он может потребоваться вручную изменить контроллер домена, который управляет объекта групповой Политики сервера для оптимизации времени на настройку распространения:  
  
-   Во время добавления в качестве точки входа было не для записи контроллеров домена в сайте Active Directory сервера удаленного доступа. Контроллер домена с возможностью записи теперь добавлена на сайт Active Directory сервера удаленного доступа.  
  
-   Изменение IP-адреса, или произошло изменение Active Directory — сайты и подсети был перемещен сервера удаленного доступа к другому сайту Active Directory.  
  
-   Связь контроллера домена для точки входа вручную был изменен из-за работ по обслуживанию на контроллере домена, и теперь контроллер домена будет восстановлена.  
  
В этих сценариях, выполните командлет PowerShell `Set-DAEntryPointDC` на сервере удаленного доступа и укажите имя точки входа, можно оптимизировать с помощью параметра *EntryPointName*. Это следует сделать только после данными объекта групповой Политики с контроллера домена, в настоящее время хранения сервера, который объект групповой Политики уже полностью были реплицированы на нужный новый контроллер домена.  
  
> [!NOTE]  
> Прежде чем изменять сопоставления контроллера домена, убедитесь, что все объекты групповой политики в развертывании удаленного доступа были реплицированы на все контроллеры домена в домене. Если объект групповой Политики не синхронизированы, последние изменения конфигурации могут быть потеряны после изменения сопоставления контроллера домена, что может привести к повреждению конфигурации. Чтобы проверить синхронизацию объекта групповой Политики, см. в разделе [проверка статуса инфраструктуры групповой политики](https://technet.microsoft.com/library/jj134176.aspx).  
  
Чтобы оптимизировать время настройки распространения, выполните одно из следующих действий.  
  
-   Для управления сервером GPO записи пункты «1 точки входа» на контроллере домена в ближайшем узле Active Directory сервера удаленного доступа «DA1.corp.contoso.com», выполните следующую команду:  
  
    ```  
    Set-DAEntryPointDC "EntryPointName 'Entry point 1' "ComputerName 'DA1.corp.contoso.com' "ErrorAction Inquire  
    ```  
  
-   Для управления сервером GPO записи пункт «1 точки входа» домена контроллера «dc2.corp.contoso.com», выполните следующую команду:  
  
    ```  
    Set-DAEntryPointDC "EntryPointName 'Entry point 1' "NewDC 'dc2.corp.contoso.com' "ComputerName 'DA1.corp.contoso.com' "ErrorAction Inquire  
    ```  
  
    > [!NOTE]  
    > При изменении контроллер домена, связанный с конкретной точкой входа, необходимо указать сервер удаленного доступа, который является членом этой точки входа для *ComputerName* параметра.
