# <a name="step-4-verify-directaccess-with-otp"></a>Шаг 4 Проверка DirectAccess с помощью OTP

>Область применения. Windows Server (полугодовой канал), Windows Server 2016

В этом разделе описывается убедитесь, что вы правильно настроили вашей DirectAccess с развертыванием OTP.
  
### <a name="to-verify-otp-health-on-the-remote-access-server"></a>Чтобы проверить работоспособность OTP на сервере удаленного доступа

1. На сервере удаленного доступа откройте **управления удаленным доступом** консоли.  

2. В разделе **серверы удаленного доступа** щелкните сервер удаленного доступа, который был настроен для поддержки OTP.  

3. Нажмите кнопку **состояние операций**.  

4. Убедитесь, что состояние OTP отображается зеленый значок и приведен в рабочее состояние.  
  
    > [!NOTE]  
    > Интервал обновления состояния работоспособности будет превышать сумму значений из раздела реестра HKLM\SYSTEM\CCS\Services\Ramgmtsvc\parameters\HealthRefreshTimeout и **интервал времени для активности сервера** , было установлено Конфигурации удаленного доступа.  
  
### <a name="to-verify-access-to-internal-resources-using-otp-authentication"></a>Проверить доступ к внутренним ресурсам при помощи проверки подлинности OTP  
  
1.  Подключении клиентского компьютера DirectAccess к корпоративной сети и запустите **gpupdate/force** из командной строки для получения групповой политики.  
  
2.  Отключить клиентский компьютер от корпоративной сети, подключиться к внешней сети и попытка получить доступ к внутренним ресурсам. Следует, у вас нет доступа к внутренним ресурсам.  
  
3.  В случае программного токена доступа также маркера одноразового ПАРОЛЯ клиента, следуя инструкциям поставщика и запишите текущий код токена. При использовании аппаратного токена, следуйте инструкциям поставщика для проверки подлинности.  
  
4.  В области уведомлений щелкните значок **Сетевые подключения** , чтобы получить доступ к диспетчеру мультимедиа DA.  
  
5.  Нажмите кнопку **подключения DirectAccess**и нажмите кнопку **Продолжить**.  
  
6.  Введите код маркера, записанным ранее и нажмите кнопку **ОК**. Дождитесь завершения операции проверки подлинности. Подключение к рабочему месту DirectAccess состояние теперь будет **подключено**.  
  
7.  Попытка получить доступ к внутренним ресурсам. Вам должны быть доступны все корпоративные ресурсы.
