# Обзор технологии VPN
>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **Предыдущая:** сведения об усовершенствованиях VPN](always-on-vpn-enhancements.md)<br>
& #187;  [ **Далее:** сведения о расширенных возможностях VPN](deploy/always-on-vpn-adv-options.md)

Для развертывания вам необходимо установить новый сервер удаленного доступа под управлением Windows Server 2016, а также изменить некоторые из существующей инфраструктуре для развертывания.

На следующем рисунке показано инфраструктуру, которую требуется развернуть VPN.

![Всегда в инфраструктуре VPN](../../../media/Always-On-Vpn/Ao-Vpn-Overview.jpg)

В процессе подключения, приведенные на этой иллюстрации состоит из следующих действий:

1. С помощью открытого DNS-серверы, клиент VPN в Windows 10 выполняет запрос разрешения имен для IP-адрес шлюза VPN.

2. Использование IP-адреса, возвращенный DNS, VPN-клиент отправляет запрос на подключение VPN-шлюза.

3. VPN-шлюза также настроен как \(RADIUS\) удаленной службы проверки подлинности удаленного пользователя клиента; VPN-клиент RADIUS отправляет запрос на сервере организации или корпоративной политики сети для обработки запросов на подключение.

4. Сервер политики сети обрабатывает запрос, включая выполнение авторизации и проверки подлинности и определяет, следует ли разрешить или запретить запрос.

5. Сервер политики сети перенаправляет ответом предоставления доступа или запретить доступ к VPN-шлюза.

6. Оно инициировано или завершается в зависимости от ответа, VPN-сервер получен от сервера политики сети.

Дополнительные сведения о всех компонентов инфраструктуры, показано на рисунке выше см. следующие разделы.

## Служба доменных имен (DNS)

Внутренние и внешние зон доменных имен (DNS) являются обязательными, который предполагается, что внутренний зоны делегированного поддомена внешних зоны (например, corp.contoso.com и contoso.com).

Узнайте больше о [Доменных имен (DNS)](../../../../networking/dns/dns-top.md) или [Руководство по основной сети](../../../../networking/core-network-guide/core-network-guide.md).




>[!NOTE] 
>Другие DNS макеты пользовательского интерфейса, таких как разделение вычислительных мощностей DNS (имя совпадает с именем домена внутренних и внешних в отдельные зоны DNS) или которые не связаны с внутренних и внешних доменов (например, contoso.local и contoso.com) можно также использовать. Дополнительные сведения о развертывании разделенной системы DNS см. в разделе [Использование политики DNS для разделения/мозг DNS развертывания](../../../../networking/dns/deploy/split-brain-DNS-deployment.md).

## Как шлюз RAS-сервер VPN удаленного доступа

В Windows Server 2016 роли сервера удаленного доступа предназначена для выполнения, а также маршрутизатор и сервера удаленного доступа; Таким образом она поддерживает широкий набор функций. Это руководство по развертыванию, вам требуется только небольшое подмножество этих функций: поддержка VPN-подключений IKEv2 и маршрутизации локальной сети.

IKEv2 — это описано в Internet Engineering Task Force запроса для 7296 комментарии протокол туннелирования VPN. Главное преимущество IKEv2 — что он допускает перерывов в основное сетевое подключение. Например, если подключение временно недоступно или пользователь перемещает клиентского компьютера из одной сети на другой, IKEv2 автоматически восстанавливает VPN-подключение при сетевое подключение восстанавливается — все это без вмешательства пользователя.

С помощью шлюза RAS-сервера, вы можете развернуть VPN-подключений для предоставления пользователям с помощью удаленного доступа к сети и ресурсам организации. Развертывание VPN обеспечивает постоянное подключение между клиентами и сети вашей организации, каждый раз, когда удаленных компьютеров, подключенных к Интернету. С помощью шлюза RAS-сервера можно также создать сеть сеть VPN-подключение между двумя серверами в разных местах, например между основного офиса и филиале и использовать \(NAT\) преобразование сетевых адресов, чтобы пользователи в сети можно получать доступ к внешним ресурсы, например в Интернете. Кроме того шлюз RAS-сервера поддерживает протокол шлюза границу (BGP), предоставляющий службы маршрутизации для динамического при вашего удаленных офисов также имеют шлюзов edge поддержкой протокола BGP.

Вы можете управлять шлюзов удаленного доступа (RAS) с помощью команды Windows PowerShell и удаленный доступ к консоли управления (MMC).



## Сервер политики сети

Сервер политики сети позволяет создавать и применять политики доступа к сети организации для проверки подлинности и авторизации. При использовании сервера политики сети в качестве сервера службы проверки подлинности удаленных пользователей (RADIUS) в качестве RADIUS-клиентов в NPS настраивается серверы доступа к сети, такие как VPN-серверов.

Можно также настроить политики сети, используемые сервером политики сети для авторизации запросов на подключение и учет RADIUS можно настроить таким образом, чтобы NPS записывает сведения об учета журнал файлов на локальном жестком диске или в базе данных Microsoft SQL Server.

Дополнительные сведения см. в разделе [Сервера политики сети (NPS)](../../../../networking/technologies/nps/nps-top.md).


## Службы сертификатов Active Directory

Сервер центра сертификации (ЦС) является центром сертификации, на котором выполняется служб сертификатов Active Directory. Конфигурация VPN требует используется Active Directory под управлением инфраструктура открытых ключей (PKI).

Организации могут использовать AD CS, чтобы повысить безопасность путем привязки удостоверение пользователя, устройства или службы для соответствующего открытого ключа. AD CS также включает в себя функции, которые позволяют управлять регистрации сертификатов и отзыва в различных масштабируемых средах. Дополнительные сведения см. в разделе [Обзор служб сертификатов Active Directory](https://technet.microsoft.com/library/hh831740.aspx) и [Руководство по проектированию для открытого ключа инфраструктуры](https://social.technet.microsoft.com/wiki/contents/articles/2901.public-key-infrastructure-design-guidance.aspx).

Во время завершения развертывания необходимо настроить следующие типы шаблонов сертификатов в центре сертификации.

-   Шаблон сертификата проверки подлинности пользователей

-   Шаблон сертификата проверки подлинности сервера VPN

-   Шаблон сертификата проверки подлинности сервера политики сети

### Шаблоны сертификатов

Шаблоны сертификатов могут значительно упростить задачу администрирования центра сертификации (ЦС), позволяя для выдачи сертификатов, подготовленных для выбранных задач. Оснастки MMC шаблоны сертификатов позволяет выполнять следующие задачи.

-   Просмотреть свойства каждого шаблона сертификата.

-   Скопировать и изменить шаблоны сертификатов.

-   Элемент управления, какие пользователи и компьютеры могут считывать шаблоны и регистрировать сертификаты.

-   Выполните другие задачи администрирования, связанные с шаблонов сертификатов.

Шаблоны сертификатов являются неотъемлемой частью центра сертификации предприятия (ЦС). Они являются важным элементом политики сертификата для среды, которая представляет собой набор правил и форматов для регистрации сертификатов, использования и управления.

Дополнительные сведения см. в разделе [Шаблонов сертификатов](https://technet.microsoft.com/library/cc730705.aspx).

### Цифровой серверных сертификатов

Это руководство содержит инструкции по регистрации и автоматической регистрации сертификатов для удаленного доступа и серверы инфраструктуры сервера политики сети с помощью служб сертификатов Active Directory (AD CS). AD CS позволяет вам создавать инфраструктура открытых ключей (PKI) и предоставляет шифрованием с открытым ключом, цифровые сертификаты и возможности цифровой подписи для вашей организации.

При использовании цифровых серверных сертификатов для проверки подлинности между компьютерами сети, содержат сертификаты.

1.  Конфиденциальность через шифрования.

2.  Целостности с помощью цифровых подписей.

3.  Проверка подлинности, связав ключи сертификатов с учетными записями компьютеров, пользователя или устройства в компьютерной сети.

Дополнительные сведения см. в разделе [AD CS Пошаговое руководство: развертывание иерархии инфраструктуры открытых КЛЮЧЕЙ два уровня](https://social.technet.microsoft.com/wiki/contents/articles/15037.ad-cs-step-by-step-guide-two-tier-pki-hierarchy-deployment.aspx).

## Доменные службы Active Directory (AD DS)

Службы AD DS предоставляют распределенных базу данных, которая хранит и управляет сведения о сетевых ресурсов и данных приложения из приложения с поддержкой каталога. Администраторы могут использовать AD DS для упорядочения элементов в сети, например пользователей, компьютеров и других устройств в иерархическую вложенную структуру. Иерархическая вложенная структура включает в себя леса Active Directory, доменах в лесу и подразделений (OU) в каждом домене. Сервер, на котором выполняется AD DS называется контроллера домена.

AD DS содержит учетные записи пользователей, учетные записи компьютеров и параметры учетной записи, необходимые, защищены расширяемый проверки подлинности протокол PEAP для проверки подлинности учетных данных пользователя и оценить авторизации для запросов на подключение VPN. Сведения о развертывании AD DS см. в Windows Server 2016 [Руководство по основной сети](../../../../networking/core-network-guide/Core-Network-Guide.md).



Во время завершения действия, описанные в этом развертывания необходимо настроить параметры на контроллере домена.

-   Включение автоматической регистрации сертификатов в групповой политике для компьютеров и пользователей

-   Создание группы пользователей VPN

-   Создание группы серверов VPN

-   Создание группы серверов NPS

### Пользователи и компьютеры Active Directory

Active Directory — пользователи и компьютеры — это компонент AD DS, содержащего учетные записи, которые представляют физические объекты, такие как компьютера, пользователя или группу безопасности. Группа безопасности — это коллекция учетных записей пользователя или компьютера, которые администраторы могут управлять как единое целое. Учетных записей пользователей и компьютеров, которые относятся к определенной группы называются члены группы.

Учетные записи пользователей в Active Directory — пользователи и компьютеры имеют свойства входном, которые оценивает сервера политики сети в процессе авторизации -, только если свойство **Права доступа к сети** учетной записи пользователя задано значение **Управление доступом с помощью политики сети **. Это параметр по умолчанию для всех учетных записей пользователей. Однако в некоторых случаях этот параметр могут иметь различные конфигурации, которую пользователь теряет подключение с помощью VPN. Чтобы избежать этого, можно настроить игнорирование свойства учетной записи пользователя dial сервера политики сети.

Дополнительные сведения см. в разделе [Настройка сервера политики сети, чтобы игнорировать учетной записи пользователя в свойствах удаленного доступа](../../../../networking/technologies/nps/nps-np-configure.md#configure-nps-to-ignore-user-account-dial-in-properties).



### Управление групповой политикой

Управление групповыми политиками позволяет directory изменения и настройки управления на основе параметров пользователя и компьютера, включая сведения о безопасности и пользователя. Использование групповой политики для определения конфигурации групп пользователей и компьютеров.

С помощью групповой политики можно задать параметры для записи реестра, безопасности, установка программного обеспечения, сценарии, перенаправление папок, служб удаленной установки и настройки Internet Explorer. Параметры групповой политики, которые вы создаете содержатся в объекте групповой политики (GPO). За счет связывания объекта групповой Политики с выбранной контейнерами Active Directory — сайты, домены и подразделения, можно применить параметры групповой Политики для пользователей и компьютеров в этих контейнеров Active Directory. Для управления объектами групповой политики на предприятии, можно использовать групповую политику управления редактора консоли управления (MMC).

### Узлы VPNv2 CSP

В это развертывание узел ProfileXML VPNv2 CSP используется для создания профиля VPN, который доставляется Windows10 клиентские компьютеры. Поставщики служб конфигурации (CSP), интерфейсы, которые предоставляют различные возможности управления в клиент Windows; по сути поставщики служб конфигурации работают аналогично работу групповой политики. Каждый поставщик служб Шифрования содержит настройки узлов, представляющих отдельных параметров. Также как параметры групповой политики, можно привязать параметры CSP для разделов реестра, файлов, разрешения и т. д. Как и как использовать редактор управления групповыми политиками для настройки объектов групповой политики (GPO), настройки узлов CSP с помощью решения мобильными устройствами (MDM) например Microsoft Intune. MDM продукты как Intune предлагают использовать параметр понятное конфигурации, настраивает CSP в операционную систему.

![Поставщик служб конфигурации конфигурации управления мобильными устройствами](../../../media/Always-On-Vpn/Vpn-Mdm.jpg)

Тем не менее нельзя настроить некоторые узлы CSP непосредственно с помощью пользовательского интерфейса (UI), например консоль администрирования Intune. В этих случаях необходимо настроить параметры Open Mobile Alliance универсального кода ресурса (OMA-URI) вручную. Настройка OMA-URI, используя протокол управления устройствами OMA (OMA-DM), спецификацию управления универсальных устройств, поддерживающих большинство современных устройств Apple, Android и Windows. До тех пор, пока они соответствуют спецификации OMA DM, все продукты MDM по взаимодействию с этих операционных систем аналогичным образом.

Windows 10 предлагает множество CSP, но это развертывание основное внимание уделяется использованию VPNv2 CSP для настройки VPN-клиент. VPNv2 CSP позволяет настраивать каждого параметра профиля VPN в Windows10 через уникальный узла CSP. Также содержатся в VPNv2 CSP представляет собой узел под названием *ProfileXML*, который позволяет настроить все параметры в одном узле, а не по отдельности. Дополнительные сведения о ProfileXML см. раздел «Обзор ProfileXML» далее в этом развертывания. Сведения о каждом узле VPNv2 CSP см. в разделе [VPNv2 CSP](https://msdn.microsoft.com/windows/hardware/commercialize/customize/mdm/vpnv2-csp).

- [Удаленный доступ](../../Remote-Access.md): в этом разделе представлен обзор роли сервера удаленного доступа в Windows Server 2016.

- [Технический справочник по VPN Windows10](https://docs.microsoft.com/windows/access-protection/vpn/vpn-guide): в этом руководстве описана решения необходимо принять для клиентов Windows 10 в свое решение VPN предприятия и способы настройки развертывания. В этом руководстве представлены ссылки на поставщик службы конфигурации (CSP) VPNv2 и инструкции для настройки решения управления мобильными устройствами (MDM) с помощью Microsoft Intune и шаблона профиля VPN для Windows 10.

- [Руководство по основной сети](../../../../networking/core-network-guide/Core-Network-Guide.md): в этом руководстве приведены инструкции о том, как планирование и развертывание основные компоненты, необходимые для полнофункциональное сети и новый домен в новом лесу Active Directory.

- [Доменных имен (DNS)](../../../../networking/dns/dns-top.md): в этом разделе представлен обзор системы доменных имен (DNS). В Windows Server 2016 DNS-это роль сервера, которую можно установить с помощью диспетчера сервера или Windows PowerShell команд. При установке нового леса Active Directory и домен, DNS устанавливается автоматически с помощью Active Directory сервером глобального каталогу для леса и домена. 

- [Обзор служб сертификатов Active Directory](https://technet.microsoft.com/library/hh831740.aspx): этот документ содержит общие сведения о сертификате служб Active Directory (AD CS) в Windows Server® 2012. AD CS — это роль сервера, который позволяет вам создавать инфраструктура открытых ключей (PKI) и предоставляет шифрованием с открытым ключом, цифровые сертификаты и возможности цифровой подписи для вашей организации.

- [Руководство по проектированию для открытого ключа инфраструктуры](https://social.technet.microsoft.com/wiki/contents/articles/2901.public-key-infrastructure-design-guidance.aspx): этом вики-сайте приведены рекомендации по разработке инфраструктурах открытого ключа (PKI). Прежде чем настраивать центр сертификации (ЦС) иерархии инфраструктуры открытых КЛЮЧЕЙ и сертификация, следует иметь в виду безопасности вашей организации политики и сертификата инструкции практического занятия (CPS).

- [AD CS Пошаговое руководство: развертывание иерархии инфраструктуры открытых КЛЮЧЕЙ два уровня](https://social.technet.microsoft.com/wiki/contents/articles/15037.ad-cs-step-by-step-guide-two-tier-pki-hierarchy-deployment.aspx): в этом пошаговом руководстве описаны шаги, необходимые для настройки основных конфигурацию служб сертификатов Active Directory® (AD CS) в лабораторной среде. AD CS в Windows Server® 2008 R2 предоставляет настраиваемые службы для создания и управления сертификатами открытого ключа, используемые в системах безопасности программного обеспечения, применением технологии открытых ключей.

- [Сервер политики сети (NPS)](../../../../networking/technologies/nps/nps-top.md): в этом разделе представлен обзор сервера политики сети в Windows Server 2016. Сервер сетевых политик (NPS) позволяет создавать и применять политики доступа к сети на уровне организации для проверки подлинности и авторизации сетевых подключений.
