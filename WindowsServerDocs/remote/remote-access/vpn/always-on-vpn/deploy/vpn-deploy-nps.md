# Шаг 4. Установка и настройка сервера политики сети (NPS)

>   Область применения: Windows Server 2019, Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows 10


& #171;  [ **Далее:** шаг 3. Настройка сервера удаленного доступа для всегда на VPN](vpn-deploy-ras.md)<br>
& #187;  [ **Далее:** шаг 5. Настройка параметров брандмауэра и DNS](vpn-deploy-dns-firewall.md)


На этом этапе установкой сервера политики сети (NPS) для обработки запросов на подключение, отправляемых по VPN-сервер:

- Для выполнения авторизации, чтобы убедиться, что пользователь имеет разрешение на подключение.
- Выполнение проверки подлинности для проверки удостоверения пользователя.
- Выполнение учета журнал аспекты, выбранные при настройке RADIUS-учет в NPS запрос.

Действия, описанные в этом разделе, позволяют выполнять следующие элементы.

1.  На компьютере или виртуальной Машине, который планируется сервера политики сети, установленных в вашей организации или корпоративной сети вы можете установить сервера политики сети.

   >[!TIP] 
   >Если уже имеется один или несколько серверов сервера политики сети в вашей сети, необходимо выполнить установку сервера политики сети — вместо этого можно использовать для обновления конфигурации существующего сервера политики сети в этом разделе.

>[!NOTE]  
Не может установить службу сервера политики сети в Windows Server Core.

2.  На сервере организации или корпоративной политики сети можно настроить для выполнения в качестве RADIUS-сервера, обрабатывающий запросы соединения, полученных от VPN-сервер.

## Установка сервера политики сети

В этой процедуре установки сервера политики сети с помощью либо Windows PowerShell или Server Manager добавить мастер ролей и компонентов. Сервер политики сети является службой роли роль сервера политики сети и доступа к службам.

>[!TIP] 
>По умолчанию сервер политики сети прослушивает RADIUS-трафика на портах 1812, 1813, 1645 и 1646 на все установленные сетевые адаптеры. При установке сервера политики сети и включение брандмауэра Windows в режиме повышенной безопасности, исключения брандмауэра для эти порты автоматически создаются для трафика IPv4 и IPv6. Если сервер удаленного доступа настроены для отправки RADIUS-трафика через порты, отличных от эти значения по умолчанию, удалить исключения, созданные в брандмауэра Windows в режиме повышенной безопасности при установке сервера политики сети и создавать исключения для портов, используемые для RADIUS-трафика.

**Процедура Windows PowerShell:**

Для выполнения этой процедуры с помощью Windows PowerShell, запустите Windows PowerShell от имени администратора, введите следующую команду и нажмите клавишу ВВОД.

`Install-WindowsFeature NPAS -IncludeManagementTools`

**Процедура для диспетчера сервера:**

1.  В диспетчере сервера щелкните **Управление**и нажмите кнопку **добавления ролей и компонентов**. <p>Откроется мастер добавления ролей и компонентов.

2.  В перед началом нажмите кнопку " **Далее**".

    >[!NOTE] 
    >Если ранее вы выбрали **Пропустить эту страницу по умолчанию** при запуске функции мастера добавления ролей и странице **Перед началом** функции мастера добавления ролей и не отображается.

1.  В окне Выбор типа установки убедитесь, что **Установка ролей или компонентов** при нажатии и нажмите кнопку **Далее**.

2.  В Выбор целевого сервера убедитесь, что выбран **Выберите сервер из пула серверов** .

3.  В пул серверов убедитесь, что выбран локального компьютера и нажмите кнопку **Далее**.

4.  На Выбор ролей сервера, в **роли**выберите **службы политики сети и доступа**. Откроется диалоговое окно с запросом, если он должен добавить компоненты, необходимые для службы политики сети и доступа.

5.  Нажмите кнопку **Добавить компоненты**и нажмите кнопку **Далее**

6.  Выберите функции нажмите кнопку **Далее**и в политике сети и доступ к службам, просмотрите сведения, предоставляемые и нажмите кнопку **Далее**.

7.  Выберите роль службы нажмите кнопку **Сервера политики сети**.

8.  Для компонентов, необходимых для сервера политики сети нажмите кнопку **Добавить компоненты** и нажмите кнопку **Далее**.

9.  В выбранным подтвердить установку нажмите кнопку **автоматически в том случае, при необходимости перезапустите сервер назначения**.

10. Щелкните **Да** для подтверждения выбранного и нажмите кнопку **установить**. <p>На странице Ход выполнения установки отображается состояние во время установки. По завершении процесса сообщение «Установка успешно на *ComputerName*» отображается, где *имя_компьютера* — имя компьютера, на котором установлен сервера политики сети.

11. Нажмите кнопку **Закрыть**.

## Настройка сервера политики сети

После установки сервера политики сети, настройки NPS для обработки всех проверки подлинности, авторизации и учета обязанностей для подключения к запросу получает от VPN-сервер.

### Регистрация сервера политики сети в Active Directory

В этой процедуре зарегистрировать сервер в Active Directory, чтобы он имеет разрешение на доступ к учетной записи пользователя во время обработки запросов на подключение.

**Процедура:**

1.  В диспетчере сервера щелкните **средства**и нажмите кнопку **Сервера политики сети**. Откроется консоль сервера политики сети.

2.  В консоли сервера политики сети щелкните правой кнопкой мыши **сервер сетевых Политик (локальный)** и щелкнуть **зарегистрировать сервер в Active Directory** , чтобы выбрать его.<p>Откроется диалоговое окно сервера политики сети.

3.  В диалоговом окне сервера политики сети и дважды нажмите кнопку " **ОК** ".

Альтернативные регистрации сервера политики сети см. в разделе [Регистрация сервера политики сети в домене Active Directory](../../../../../networking/technologies/nps/nps-manage-register.md).

### Настройка учета сервера политики сети

В этой процедуре настройки сети политики сервера учета с помощью одного из следующих типов ведения журнала:

-   В **журнал событий**. Используется в основном для аудита и устранения неполадок попытки подключения. Вы можете настроить ведение журнала, получая свойств сервера политики сети в консоли NPS событий сервера политики сети.

-   **Ведение журнала проверок подлинности пользователя и запросов учета в локальный файл**. Используется главным образом для подключения анализа и выставления счетов. Также используется как средство анализа безопасности, так как он предоставляет метод отслеживания действий злоумышленник после атаки. Можно настроить с помощью мастера настройки учета локальный файл журнала.

-   **Ведение журнала проверки подлинности пользователей и учета запросов к базе данных Microsoft SQL Server XML-стандартами**. Позволяет разрешить несколько серверов NPS иметь один источник данных. Также предоставляет преимущества использования реляционной базе данных. Ведение журнала SQL Server можно настроить с помощью мастера настройки учета.

Настройка учета сервера политики сети, см. в разделе [Настройка сети политики сервера учета](../../../../../networking/technologies/nps/nps-accounting-configure.md).

### Добавить VPN-сервер в качестве RADIUS-клиента

В разделе [Настройка сервера удаленного доступа для VPN](vpn-deploy-ras.md) установить и настроить VPN-сервер. Во время настройки сервера VPN добавленные вами общего секрета RADIUS на VPN-сервер. 

В этой процедуре используйте ту же общего секретного текстовую строку для настройки VPN-сервер в качестве RADIUS-клиента в NPS. Используйте ту же строку текста, который вы указали на VPN-сервер или обмен данными между сервером политики сети и VPN-сервер не удается.

>[!IMPORTANT] 
>При добавлении нового сервера сетевого доступа (VPN сервера, беспроводной точки доступа, коммутатор с проверкой подлинности или сервера удаленного доступа) к сети, необходимо добавить сервер в качестве RADIUS-клиента в NPS визуализация NPS известно, могут связываться с сервером доступа к сети.

**Процедура:**

1.  На сервере политики сети в консоли сервера политики сети дважды щелкните **RADIUS-клиенты и серверы**.

2.  Щелкните правой кнопкой мыши **RADIUS-клиентов** и нажмите кнопку **Создать**. Откроется диалоговое окно нового RADIUS-клиента.

3.  Убедитесь, что установлен флажок **Включить RADIUS-клиент** .

4.  **Понятное имя**введите отображаемое имя VPN-сервера.

5.  В **адрес (IP-адрес или DNS)** введите NAS IP-адрес или полное доменное имя.<p>Если введено полное доменное имя, щелкните **Проверить** , если вы хотите проверить правильность имени и сопоставляет адрес.

6.  В **общий секрет**выполните:

    1.  Убедитесь, что выбран параметр **вручную** .

    2.  Введите надежный текстовая строка, введенные также на VPN-сервер.

    3.  Затем введите подтверждение общего секрета в подтверждение.

7.  Нажмите кнопку **ОК**. VPN-сервер отображается в списке RADIUS-клиенты, настроенные на сервер политики сети.

## Настройка сервера политики сети радиуса для VPN-подключений

В этой процедуре Настройка сервера политики сети в качестве RADIUS-сервера в сети вашей организации. На сервер политики сети, необходимо определить политику, возможен только в определенной группы для доступа к организации или в корпоративной сети через VPN-сервер - и затем только при использовании действительный сертификат пользователя в запрос проверки подлинности PEAP.

**Процедура:**

1.  В консоли сервера политики сети в стандартной конфигурации должен быть установлен **RADIUS-сервера для удаленного доступа или VPN-подключений** .

2.  Выберите **настроить VPN или удаленного доступа**.<p>Откроется мастер настройки VPN или удаленного доступа.

3.  Щелкните **подключений виртуальной частной сети (VPN)** и нажмите кнопку **Далее**.

4.  Укажите удаленного доступа или VPN-сервер, в RADIUS-клиентов выберите имя VPN-сервера, который был добавлен в предыдущем шаге.<p>Например если имя NetBIOS сервера VPN RAS1, нажмите кнопку **RAS1**.

5.  Нажмите кнопку **Далее**.

6.  В Настройка методов проверки подлинности выполните следующие действия:

    1.  Снимите флажок **Шифрованная проверка подлинности Microsoft версии 2 (MS-CHAPv2)** .

    2.  Установите флажок **Расширяемый протокол проверки подлинности** , чтобы выделить его.

    3.  В типе (в зависимости от метода доступа и сетевых конфигураций), нажмите кнопку **Майкрософт: защищенный EAP (PEAP)** и нажмите кнопку **настроить**.<p>Откроется диалоговое окно Изменение свойства защищенного EAP.

    4.  Нажмите кнопку **удаления** для удаления типа EAP защищенный пароль (EAP-MSCHAP версии 2).

    5.  Нажмите кнопку **Добавить**. Откроется диалоговое окно добавления EAP.

    6.  Щелкните **смарт-карта или другой сертификат**и нажмите кнопку **ОК**.

    7.  Нажмите кнопку **ОК** , чтобы закрыть изменить свойства защищенного EAP.

7.  Нажмите кнопку **Далее**.

8.  Укажите групп пользователей выполните следующие действия:

    1.  Нажмите кнопку **Добавить**. Откроется диалоговое окно Выбор пользователей, компьютеров, учетных записей служб или групп.

    2.  Введите **Пользователей VPN** и нажмите кнопку **ОК**.

    3.  Нажмите кнопку **Далее**.

9.  Укажите фильтры IP-адрес нажмите кнопку " **Далее**".

10. В параметрах указать шифрования нажмите кнопку " **Далее**". Не вносить никаких изменений.<p>Эти параметры применяются только для шифрования точка-точка MPPE подключений, которые не поддерживают этот сценарий.

11. В поле Укажите имя сферы нажмите кнопку " **Далее**".

12. Нажмите кнопку **Готово,** чтобы закрыть мастер.

## Автоматическая регистрация сертификатов сервера политики сети

В этой процедуре можно обновить групповую политику для локального сервера политики сети вручную. Когда обновление групповой политики, если настроена автоматической регистрации сертификатов и работает неправильно, локальный компьютер является auto зарегистрировала сертификат центром сертификации (ЦС).

>[!NOTE]  
>Групповая политика обновляется автоматически при перезагрузке членом домена или при входе пользователя в систему членом домена. Кроме того Групповая политика периодически обновляет. По умолчанию это периодическое обновление происходит каждые 90 минут со смещением случайно до 30 минут.

Членство в группе **администраторов**или аналогичной является минимальным требованием для выполнения этой процедуры.

**Процедура:**

1.  Откройте Windows PowerShell на сервер политики сети.

2.  В командной строке Windows PowerShell введите **gpupdate**и нажмите клавишу ВВОД.
