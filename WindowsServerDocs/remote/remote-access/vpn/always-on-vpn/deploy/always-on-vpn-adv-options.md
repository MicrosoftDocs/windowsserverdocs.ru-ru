# Расширенные функции VPN

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **Предыдущая:** сведения о технологии VPN](../always-on-vpn-technology-overview.md)<br>
& #187; [ **Далее:** начать планировать развертывание VPN](always-on-vpn-deploy-planning.md)

Помимо сценария развертывания, представленные вы можете добавить другие расширенные функции VPN, чтобы повысить безопасность и доступность VPN-подключение. Например такие компоненты могут помочь убедитесь, что подключаемый клиент работоспособном прежде чем разрешить подключение.


## Высокая доступность

Ниже приведены дополнительные параметры для обеспечения высокой доступности.


|Параметр  |Описание  |
|---------|---------|
|Устойчивость сервера и балансировки нагрузки     |В средах, требующих высокой доступности или поддержки большого количества запросов может повысить производительность и устойчивости удаленного доступа с помощью балансировки нагрузки между несколькими серверами под управлением сервера политики сети (NPS) и разрешение удаленного Сервер доступа кластеризации.<p>Связанные документы:<ul><li>[Балансировка нагрузки сервера прокси-сервера политики сети](../../../../../networking/technologies/nps/nps-manage-proxy-lb.md)</li><li>[Развертывание удаленного доступа в кластере](https://docs.microsoft.com/windows-server/remote/remote-access/ras/cluster/deploy-remote-access-in-cluster)</li></ul>        |
|Географическое устойчивость     |Для географического расположения на основе IP-адрес можно использовать глобальный диспетчер трафика с DNS в Windows Server 2016. Для более надежного географическое балансировки нагрузки, можно использовать глобальный сервер балансировки нагрузки решения, такие как диспетчер трафика Microsoft Azure.<p>Связанные документы:<ul><li>[Обзор диспетчера трафика](https://docs.microsoft.com/azure/traffic-manager/traffic-manager-overview)</li><li>[Диспетчер трафика Microsoft Azure](https://azure.microsoft.com/services/traffic-manager)</li></ul>         |

## Расширенная проверка подлинности

Ниже приведены дополнительные параметры для проверки подлинности.


|Параметр  |Описание  |
|---------|---------|
|Windows Hello для бизнеса     |В Windows 10 служба Windows Hello для бизнеса позволяет заменить пароли строгой двухфакторной проверкой подлинности на компьютерах и мобильных устройствах. Эта проверка подлинности заключается в новом типе учетных данных пользователя, привязанных к устройству и использует биометрические данные или идентификационный номер (PIN).<p>Клиент VPN в Windows 10 не совместимы с Windows Hello для бизнеса. После входа пользователя в систему с помощью жеста, VPN-подключение использует Windows Hello для бизнеса сертификат для проверки подлинности на основе сертификатов.<p>Связанные документы:<ul><li>[Windows Hello для бизнеса](https://docs.microsoft.com/windows/access-protection/hello-for-business/hello-identity-verification)</li><li>Практический пример: [Включение удаленного доступа с помощью Windows Hello для бизнеса в Windows 10](https://msdn.microsoft.com/library/mt728163.aspx)</li></ul>         |
|Многофакторная Идентификация Azure (MFA)     |Предоставление Azure MFA имеет облачных и локальных версий, которые можно интегрировать с помощью механизма проверки подлинности VPN в Windows.<p>Дополнительные сведения о том, как работает этот механизм см. в разделе [интеграции РАДИУС проверки подлинности с помощью сервера многофакторной идентификации Azure](https://docs.microsoft.com/azure/multi-factor-authentication/multi-factor-authentication-get-started-server-radius).         |

## Функции расширенных VPN

Ниже приведены дополнительные параметры для дополнительных возможностей.


|Параметр  |Описание  |
|---------|---------|
|Фильтры трафика     |Если необходимо применить VPN приложения, которые клиенты могут получать доступ к, вы можете включить фильтры трафика VPN.<p>Дополнительные сведения см. в разделе [функции безопасности VPN](https://docs.microsoft.com/windows/access-protection/vpn/vpn-security-features).         |
|VPN подключения, инициируемого приложением     |Вы можете настроить VPN-профили автоматически подключаться при запуске некоторых приложений или типов приложений.<p>Дополнительные сведения об этом и других активирующее параметров см. в разделе [VPN автоматически инициируемые параметры профиля](https://docs.microsoft.com/windows/access-protection/vpn/vpn-auto-trigger-profile).         |
|Условный доступ через VPN   |Соответствие условного доступа и устройства может потребоваться управляемых устройств на соответствие стандартам, прежде чем они смогут подключиться к VPN. Один из расширенные функции для условного доступа VPN позволяет ограничить VPN-подключений только те, где сертификат проверки подлинности клиента содержит «AAD условного доступа OID из "1.3.6.1.4.1.311.87».<p>Чтобы ограничить VPN-подключений, необходимо:<ol><li>Откройте оснастку **Сервера политики сети** на сервера политики сети.</li><li>Разверните **политики** > **политики сети**.</li><li>Щелкните правой кнопкой мыши политики сетевых **подключений виртуальной частной сети (VPN)** и выберите **Свойства**.</li><li>Выберите вкладку " **Параметры** ".</li><li>Выберите **Определенного поставщика** и нажмите кнопку **Добавить**.</li><li>Выберите вариант **Разрешенных сертификатов OID** и нажмите кнопку **Добавить**.</li><li>Вставьте AAD условного доступа OID **1.3.6.1.4.1.311.87** в качестве значения атрибута и дважды нажмите кнопку **ОК** .</li><li>Нажмите кнопку **закрытия** , а затем **Применить**.<p>Теперь при попытке подключения с помощью любого сертификата, сертификат кратковременные облачных клиентами VPN, подключение не удастся.</li></ol>Дополнительные сведения о условного доступа см. в разделе [VPN и условный доступ](https://docs.microsoft.com/windows/access-protection/vpn/vpn-conditional-access).   |

## Дополнительный уровень защиты

**Аттестации ключей доверенного платформенного модуля (TPM)**<p>
Сертификат пользователя с помощью ключа аттестованы TPM обеспечивает более безопасность при правильном, резервные копии, не являющихся возможность экспорта, противодействовать и изоляции ключей, предоставляемых доверенного платформенного МОДУЛЯ.

Дополнительные сведения о Аттестация ключа TPM в Windows 10 см. в разделе [Аттестация ключа TPM](https://docs.microsoft.com/windows-server/identity/ad-ds/manage/component-updates/tpm-key-attestation).

После правильного планирования можно развертывать VPN и при необходимости настройте условного доступа для подключения к VPN с помощью Azure AD.

## Смежные разделы
- [Балансировка нагрузки сервера NPS прокси-сервера](../../../../../networking/technologies/nps/nps-manage-proxy-lb.md): клиенты службы проверки подлинности удаленных пользователей (RADIUS), которые являются серверы доступа к сети, такие как серверы виртуальной частной сети (VPN) и точек доступа, создание запросов на подключение и отправлять их RADIUS серверы, такие как сервера политики сети. В некоторых случаях сервера политики сети может возникнуть слишком большое количество запросов на подключение за один раз, что приводит к низкой производительности или перегрузки.

- [Обзор диспетчера трафика](https://docs.microsoft.com/azure/traffic-manager/traffic-manager-overview): в этом разделе представлен обзор из Azure диспетчер трафика, который позволяет управлять распространением пользователь трафика для конечных точек службы. Диспетчер трафика использует доменных имен (DNS) для прямых клиентских запросов наиболее подходящего конечной точки, на основе метода маршрутизации трафика и работоспособности конечных точек. 

- [Windows Hello для бизнеса](https://docs.microsoft.com/windows/access-protection/hello-for-business/hello-identity-verification): в этом разделе представлен необходимые компоненты, такие как развертывание только облачные и гибридные развертывания.  В этом разделе также перечислены часто задаваемые вопросы о Windows Hello для бизнеса.

- [Практическим: Включение удаленного доступа с помощью Windows Hello для бизнеса в Windows 10](https://msdn.microsoft.com/library/mt728163.aspx): это практическим вы узнаете, как корпорация Майкрософт реализует удаленного доступа с помощью Windows Hello для бизнеса.  Windows Hello для бизнеса — это открытого и закрытого ключа или проверка подлинности на основе сертификатов подход организации и отдельные пользователи, который выходит за рамки пароли. Эта форма проверки подлинности полагается на учетные данные пары ключей, которые можно заменить пароли и устойчивы взломов, краж и фишинга. 

- [Интеграция RADIUS-проверки подлинности с помощью сервера многофакторной идентификации Azure](https://docs.microsoft.com/azure/multi-factor-authentication/multi-factor-authentication-get-started-server-radius): в этом разделе рассматривается добавление и настройка проверки подлинности клиента RADIUS с сервера многофакторной идентификации Azure. RADIUS — это стандартный протокол принимать запросы на проверку подлинности и обрабатывать эти запросы. Сервер многофакторной идентификации Azure может выступать в качестве RADIUS-сервера. 

- [Функции безопасности VPN](https://docs.microsoft.com/windows/access-protection/vpn/vpn-security-features): в этом разделе приведены рекомендации по безопасности VPN для блокировки VPN, интеграция Windows Information Protection (WIP) с VPN, и фильтры трафика. 

- [VPN автоматически инициируемые параметры профиля](https://docs.microsoft.com/windows/access-protection/vpn/vpn-auto-trigger-profile): в этом разделе представлен VPN автоматически инициируемые параметры профиля, такие как триггер приложений, триггер на основе имени и всегда включен.

- [VPN и условный доступ](https://docs.microsoft.com/windows/access-protection/vpn/vpn-conditional-access): в этом разделе представлен обзор облачной платформой условного доступа для предоставления удаленным клиентам возможности обеспечения соответствия устройства. Условный доступ — это модуль оценки на основе политик, который позволяет создавать правила доступа для любого приложения, подключенного к Azure Active Directory (Azure AD). 

- [Аттестация ключа TPM](https://docs.microsoft.com/windows-server/identity/ad-ds/manage/component-updates/tpm-key-attestation): в этом разделе представлен обзор из доверенного платформенного модуля (TPM) и основные действия по развертыванию доверенного платформенного МОДУЛЯ аттестации. Можно также найти сведения и решить проблемы устранения неполадок.
