# Шаг 3. Настройка сервера удаленного доступа для постоянно подключенного VPN-профиля

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **Предыдущих:** шаг 2. Настройка инфраструктуры сервера](vpn-deploy-server-infrastructure.md)<br>
& #187;  [ **Предыдущих:** шаг 4. Установка и настройка сервера политики сети (NPS)](vpn-deploy-nps.md)


RRAS предназначена для выполнения хорошо как маршрутизатор и сервера удаленного доступа, поскольку она поддерживает широкий набор функций. В целях данного развертывания требуется только небольшое подмножество этих функций: поддержка VPN-подключений IKEv2 и маршрутизации локальной сети.

IKEv2 является VPN, описанные в Internet Engineering Task Force запроса для 7296 комментарии протокол туннелирования. Основным преимуществом IKEv2 является, что он допускает перерывов в основное сетевое подключение. Например, если подключение временно недоступно или пользователь перемещает клиентского компьютера из одной сети на другой, IKEv2 автоматически восстанавливает VPN-подключение при сетевое подключение восстанавливается — все это без вмешательства пользователя.

Настройка сервера RRAS на поддержку подключений IKEv2 во время отключение неиспользуемые протоколы, что сокращает объем памяти системы безопасности сервера. Кроме того настройте назначение адреса VPN-клиентам из пула статических адресов сервером. Вы можете назначить периодичностью адреса из пула или DHCP-сервера; Тем не менее DHCP-сервер добавляет сложность дизайна и обеспечивает преимущества в минимальной.


>[!Important]
>Важно:
>- Установка двух сетевых адаптеров Ethernet в физическом сервере. Если вы устанавливаете VPN-сервер на виртуальной Машине, необходимо создать два внешних виртуальных коммутаторов, один для каждого физическому сетевому адаптеру; а затем создать два виртуальных сетевых адаптеров для виртуальной Машины, с каждого сетевого адаптера, подключенных к одной виртуального коммутатора.
>
>- Установка сервера в сети периметра между edge и внутренние брандмауэры, один сетевой адаптер, подключенный к внешней сети периметра, и один сетевой адаптер, подключенный к внутренней сети периметра.


>[!Warning]
>Перед началом работы убедитесь, что включить IPv6 на VPN-сервер. В противном случае не удается установить подключение и отображает сообщение об ошибке.

## Установка удаленного доступа в качестве VPN-сервера шлюза RAS-сервера

В ходе данной процедуры необходимо установить роль удаленного доступа как один сервер VPN шлюза RAS-сервера. Дополнительные сведения см. в разделе [Удаленный доступ](../../../Remote-Access.md).


### Установите роль удаленного доступа с помощью Windows PowerShell

1. Откройте Windows PowerShell от имени **администратора**.

2. Введите следующую команду и нажмите клавишу **Ввод**:

   `Install-WindowsFeature DirectAccess-VPN -IncludeManagementTools`

   После завершения установки в Windows PowerShell появится следующее сообщение.
    
   | Выполнено | Требуется перезагрузка | Код выхода | Результат функции                             |
   |---------|----------------|-----------|--------------------------------------------|
   | True    | Нет             | Выполнено   | {RAS администрирования диспетчера подключений |
   ---

### Установите роль удаленного доступа с помощью диспетчера сервера

Чтобы установить роль удаленного доступа с помощью диспетчера сервера можно использовать следующую процедуру.

1.  На сервере VPN, в диспетчере сервера щелкните **Управление** , а затем щелкните **добавления ролей и компонентов**. <p>Откроется мастер добавления ролей и компонентов.

2.  На до начала страницы, нажмите кнопку**Далее**.

3.  На странице "Выбор типа установки" выберите вариант **установки ролей или компонентов** и нажмите кнопку **Далее**.

4.  На странице "Выбор целевого сервера" выберите вариант **Выберите сервер из пула серверов** .

5.  В разделе пул серверов выберите локальный компьютер и нажмите кнопку **Далее**.

6.  На странице роли выберите сервер в **роли**щелкните **Удаленного доступа**, а затем **Далее**.

7.  На странице "Выбор компонентов" нажмите кнопку " **Далее**".

8.  На странице "удаленный доступ" нажмите кнопку " **Далее**".

9.  На странице службы установите роль **службы роли**щелкните**DirectAccess и VPN (RAS)**.<p>Откроется диалоговое окно **мастера добавления ролей и компонентов** .

10. На добавления ролей и компонентов диалоговое окно нажмите кнопку **Добавить компоненты** и нажмите кнопку **Далее**.

11. На странице роли веб-сервер (IIS) нажмите кнопку " **Далее**".

12. На странице выберите роль службы нажмите кнопку " **Далее**".

13. На странице подтверждения выбранные параметры установки просмотрите выбранные параметры и нажмите кнопку **Установка**.

14. После завершения установки нажмите кнопку **Закрыть**.

## Настройка удаленного доступа в качестве сервера VPN

В этом разделе можно настроить VPN удаленного доступа разрешить VPN-подключений IKEv2, запрет подключения из других VPN-протоколы и назначить пул статических IP-адресов для выдачи IP-адреса для подключения авторизованных VPN-клиентов.

1.  На сервере VPN, в диспетчере сервера щелкните флаг **уведомления** .

2.  В меню " **задачи** " выберите **Открыть мастер начало работы**.<p>Откроется мастер настройки удаленного доступа. 

    >[!NOTE] 
    >Мастер настройки удаленного доступа может открыть программной части диспетчера серверов. Если вы считаете, что мастер выполняется слишком долго для открытия, переместить или свести к минимуму диспетчера серверов, чтобы узнать, является ли мастер за ним. В противном случае подождите, пока мастер для инициализации.

3.  Щелкните **развертывание только VPN**.<p>Откроется Маршрутизация и удаленный доступ к консоли управления (MMC).

4.  Щелкните правой кнопкой мыши VPN-сервер и нажмите кнопку **настроить и включить маршрутизацию и удаленный доступ**.<p>Откроется мастер настройки сервера удаленного доступа и маршрутизации.

5.  В окне Вас приветствует маршрутизации и мастер настройки сервера удаленного доступа нажмите кнопку " **Далее**".

6.  В **конфигурации**нажмите кнопку **Настраиваемая конфигурация**и нажмите кнопку **Далее**.

7.  **Настраиваемая конфигурация**щелкните **доступ к VPN**и нажмите кнопку **Далее**.<p>Завершение работы откроется мастер настройки сервера удаленного доступа и маршрутизации.

8.  Нажмите кнопку **Готово,** чтобы закрыть мастер и нажмите кнопку **ОК** , чтобы закрыть диалоговое окно Маршрутизация и удаленный доступ.

9.  Нажмите кнопку, **запустите службу** для запуска удаленного доступа.

10. В окне удаленного доступа к Консоли щелкните правой кнопкой мыши VPN-сервер и нажмите кнопку **Свойства**.

11. В диалоговом окне Свойства перейдите на вкладку **Безопасность** и выполните:

    А. Выберите **поставщика проверки подлинности** и кнопку **RADIUS-проверки подлинности**.
    
    Б. Щелкните **Настроить**.<p>Откроется диалоговое окно RADIUS-проверки подлинности.
    
    в. Нажмите кнопку **Добавить**.<p>Откроется диалоговое окно Добавление RADIUS-сервера.
    
    г. **Имя сервера**введите полное доменное имя (FQDN) сервера политики сети в вашей организации в корпоративной сети.<p>Например если имя NetBIOS сервера политики сети — NPS1 и corp.contoso.com — это имя вашего домена, введите **NPS1.corp.contoso.com**.
    
    д. В **общий секрет**нажмите кнопку " **Изменить**".<p>Откроется диалоговое окно Изменение секрет.
    
    е. В **Новый секрет**введите строку текста.
    
    ж. В **Подтверждение**введите одну и ту же строку текста и нажмите кнопку **ОК**.

    >[!IMPORTANT] 
    >Сохраните эту строку текста. При настройке сервера политики сети в вашей организации в корпоративной сети, будут добавлены это VPN-сервера как RADIUS-клиента. Во время этой конфигурации вы будете использовать этот же общего секрета таким образом, чтобы сервер политики сети и VPN-серверов могут взаимодействовать.

12. **Добавление RADIUS-сервера**просмотрите параметры по умолчанию для:

    - **Время ожидания**
    
    - **Начальная оценка**
    
    - **Port**

13. При необходимости измените значения в соответствии с требованиями для вашей среды и нажмите кнопку **ОК**.<p>NAS является устройством, которое обеспечивает определенный уровень доступа к сети большего размера. С помощью инфраструктуры RADIUS NAS также является RADIUS-клиента, отправка запросов на подключение и сообщения учета RADIUS-сервера для проверки подлинности, авторизации и учета.

14. Просмотрите параметр для **учета поставщика**:

    | Если вы хотите, чтобы …  | То...             |
    |---------------------|-------------------|
    | Удаленный доступ активность, войти в систему сервера удаленного доступа | Убедитесь, что выбран **Учет Windows** .      |
    | Сервер политики сети, для обеспечения работы служб учета для VPN   | Изменить **поставщика учета** **RADIUS-учета** и настройте сервер политики сети в качестве поставщика службы учетных записей. |
    ---

15. Откройте вкладку **IPv4** и выполните:

    А. Щелкните **статический пул адресов**.
    
    Б. Нажмите кнопку **Добавить** для настройки пул IP-адресов.<p>Статический пул адресов должен содержать адреса из внутреннего периметра сети. Эти адреса находятся в общении сетевых подключений VPN-сервера, а не в корпоративной сети.
    
    в. В **Начальный IP-адрес**введите начальный IP-адрес в диапазоне, который вы хотите назначить VPN-клиентов.
    
    г. В **конечный IP-адрес**введите конечный IP-адрес в диапазоне, который вы хотите назначить VPN-клиентов или в **число адресов**, число адрес, который вы хотите сделать доступными. Если вы используете DHCP для этой подсети, убедитесь, настроить соответствующие исключения адреса DHCP-серверы.
    
    д. (Необязательно) Если вы используете протокол DHCP, нажмите кнопку **адаптер**затем в списке результатов, адаптер Ethernet, подключенных к внутренней локальной сети.

16. (Необязательно) *Если вы настраиваете условного доступа для подключения VPN*, из раскрывающегося списка **сертификатов** , в разделе **Привязки сертификата SSL**, выберите подлинности VPN-сервера.

17. (Необязательно) *Если вы настраиваете условного доступа для подключения VPN*, в окне Консоли NPS разверните **Политики Policies\\Network** , выполните: 

    А. Справа политики сетевого **подключения к серверу маршрутизации и сервера удаленного доступа** и выберите **Свойства**.
    
    Б. Выберите **предоставить доступ. Предоставление доступа, если запрос на подключение соответствует этой политики** вариант.
    
    в. В разделе тип сервера доступа к сети выберите **Сервер удаленного доступа (VPN-удаленного доступа)** из раскрывающегося списка.

3.  В Маршрутизация и удаленный доступ к MMC щелкните правой кнопкой мыши **портов** и нажмите кнопку **Свойства**. <p>Откроется диалоговое окно Свойства портов.

4.  Щелкните **WAN мини-портов (SSTP)** и нажмите кнопку **настроить**. Настройка устройства - WAN мини-портов (SSTP) диалоговое окно.

    А. Снимите флажки **подключения удаленного доступа (только входящие)** и **подключений по требованию (входящий и исходящий)** .
    
    Б. Нажмите кнопку **ОК**.

5.  Щелкните **Мини-порта глобальной сети (L2TP)** и нажмите кнопку **настроить**. Настройка устройства - WAN мини-портов (L2TP) диалоговое окно.

    А. **Максимальное число портов**введите число портов в соответствии с максимальное число параллельных VPN-подключений, которые вы хотите поддерживать.
    
    Б. Нажмите кнопку **ОК**.

6.  Щелкните **WAN мини-портов (PPTP)** и нажмите кнопку **настроить**. Настройка устройства - WAN мини-портов (PPTP) диалоговое окно.

    А. **Максимальное число портов**введите число портов в соответствии с максимальное число параллельных VPN-подключений, которые вы хотите поддерживать.
    
    Б. Нажмите кнопку **ОК**.
    
7. Щелкните **Мини-порта глобальной сети (IKEv2)** и нажмите кнопку **настроить**. Настройка устройства - WAN мини-портов (IKEv2) диалоговое окно.

    А. **Максимальное число портов**введите число портов в соответствии с максимальное число параллельных VPN-подключений, которые вы хотите поддерживать.
    
    Б. Нажмите кнопку **ОК**.

7.  При необходимости щелкните **Да** для подтверждения перезапуском сервера и нажмите кнопку **Закрыть** перезапускать сервер.
