# <a name="step-2-configure-the-server-infrastructure"></a>Шаг 2. Настройка инфраструктуры сервера

>Относится к: Windows Server (полугодовой канал), Windows Server 2016, Windows Server 2012 R2, Windows 10

- [**Предыдущих:** Шаг 1. Планирование развертывания постоянно подключенного VPN-профиля](always-on-vpn-deploy-planning.md)
- [**Далее:** Шаг 3. Настройка сервера удаленного доступа для постоянно подключенного VPN-профиля](vpn-deploy-ras.md)

На этом шаге вы установите и настроите серверные компоненты, необходимые для поддержки VPN-Подключение. Серверные компоненты включают настройке PKI для распространения сертификатов, которые используются пользователями, VPN-сервер и сервер политики сети.  Можно также настроить RRAS для поддержки подключения IKEv2 и NPS-сервере для выполнения авторизации для VPN-подключений.

## <a name="configure-certificate-autoenrollment-in-group-policy"></a>Настройка автоматической регистрации сертификатов в групповой политике
В этой процедуре настроить групповую политику на контроллере домена, чтобы члены домена автоматически запрашивать сертификаты пользователей и компьютеров. Это позволит пользователям VPN для запроса и получения сертификатов пользователей, выполняющих проверку подлинности VPN-подключений, автоматически. Аналогичным образом, эта политика позволяет серверам политики сети для запроса сервера сертификатов проверки подлинности автоматически. 

Вы вручную регистрировать сертификаты на VPN-серверов.

>[!TIP]
>Для компьютеров, объединенных-domained, см. в разделе [конфигурации ЦС для домена компьютеров, присоединенных к](#ca-configuration-for-non-domain-joined-computers). Поскольку сервер RRAS не присоединен к домену, автоматической регистрации не может использоваться для регистрации сертификата шлюза VPN.  Таким образом используйте процедуру запроса сертификата в автономном режиме.

1. На контроллере домена откройте Управление групповой политикой.

2. В области навигации щелкните правой кнопкой мыши домен (например, corp.contoso.com), а затем выберите **создать объект групповой Политики в этом домене и связать его**.

3. В диалоговом окне новый объект групповой Политики, введите **политики автоматической подачи заявок**, а затем выберите **ОК**.

4. В области навигации щелкните правой кнопкой мыши **политики автоматической подачи заявок**, а затем выберите **изменить**.

5. В редакторе управления групповой политики, выполните следующие действия по настройке автоматической регистрации сертификатов компьютера.

    1. В области навигации перейдите к **Конфигурация компьютера** > **политики** > **параметры Windows**  >   **Параметры безопасности** > **политики открытого ключа**.

    2. В области сведений щелкните правой кнопкой мыши **клиент служб сертификации: автоматическая регистрация**, а затем выберите **свойства**.

    3. На клиент служб сертификации: автоматическая регистрация свойства отображаемое в диалоговом окне **модель конфигурации**выберите **включено**.

    4. Выберите параметры **Обновлять сертификаты с истекшим сроком действия или в состоянии ожидания и удалять отозванные сертификаты** и **Обновлять сертификаты, использующие шаблоны сертификатов**.

    5. Нажмите кнопку **ОК**.

6. В редакторе управления групповой политики, выполните следующие действия, чтобы Настройка автоматической регистрации сертификатов пользователя.

    1. В области навигации перейдите к **Конфигурация пользователя** > **политики** > **параметры Windows**  >   **Параметры безопасности** > **политики открытого ключа**.

    2. В области сведений щелкните правой кнопкой мыши **Клиент служб сертификации: автоматическая регистрация** и выберите **Свойства**.

    3. На клиент служб сертификации: автоматическая регистрация свойства отображаемое в диалоговом окне **модель конфигурации**выберите **включено**.

    4. Выберите параметры **Обновлять сертификаты с истекшим сроком действия или в состоянии ожидания и удалять отозванные сертификаты** и **Обновлять сертификаты, использующие шаблоны сертификатов**.

    5. Нажмите кнопку **ОК**.

    6. Закройте редактор управления групповыми политиками.

7. Закройте оснастку "Управление групповой политикой".

### <a name="ca-configuration-for-non-domain-joined-computers"></a>Настройки ЦС для компьютеров присоединены к домену

Поскольку сервер RRAS не присоединен к домену, автоматической регистрации не может использоваться для регистрации сертификата шлюза VPN.  Таким образом используйте процедуру запроса сертификата в автономном режиме.

1. На RRAS-сервере, создайте файл с именем **VPNGateway.inf** на основании пример запроса политики сертификат, предоставленный в приложении (раздел 0) и настроить следующие записи:

   - В разделе [NewRequest] замените vpn.contoso.com, используемый в качестве имени субъекта с выбранным [_клиента_] конечная точка VPN полное доменное имя.

   - В разделе [расширений] замените vpn.contoso.com, используемый для альтернативного имени субъекта с выбранным [_клиента_] конечная точка VPN полное доменное имя.

2. Сохранить или скопировать **VPNGateway.inf** файл в выбранном местоположении.

3. Из командной строки с повышенными перейдите к папке, содержащей **VPNGateway.inf** файл и введите:

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

4. Скопируйте созданный **VPNGateway.req** выходной файл на сервере центра сертификации, или доступа к рабочей станции с ПРИВИЛЕГИРОВАННЫМ доступом.

5. Сохранить или скопировать **VPNGateway.req** файл в выбранном местоположении на сервере центра сертификации, или доступа к рабочей станции с ПРИВИЛЕГИРОВАННЫМ доступом.

6. Из командной строки с повышенными правами перейдите к папке, содержащей файл VPNGateway.req, созданный на предыдущем шаге и тип:

   ```
   certreq -attrib “CertificateTemplate:[Customer]VPNGateway” -submit VPNgateway.req VPNgateway.cer
   ```

7. При появлении соответствующего запроса в окне Список центров сертификации, выберите соответствующий ЦС предприятия для обслуживания запроса на сертификат.

8. Скопируйте созданный **VPNGateway.cer** выходной файл для сервера RRAS. 

9. Сохранить или скопировать **VPNGateway.cer** файл в выбранном местоположении на RRAS-сервере.

10. Из командной строки с повышенными правами перейдите к папке, содержащей файл VPNGateway.cer, созданный на предыдущем шаге и тип:

    ```
    certreq -accept VPNGateway.cer
    ```

11. Запустите оснастку MMC для сертификатов, как описано [здесь](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in) выбора **учетная запись компьютера** параметр.

12. Убедитесь в наличии действительный сертификат для сервера RRAS со следующими свойствами:

    - **Назначения:** Проверки подлинности сервера, IP-безопасности IKE-посредник 

    - **Шаблон сертификата:** [_клиента_] VPN-сервера

#### <a name="example-vpngatewayinf-script"></a>Пример. Сценарий VPNGateway.inf

Здесь вы видите пример сценария политики запросов на сертификат, используемый для запроса сертификата шлюза VPN с помощью процесса out of band.

>[!TIP]
>Вы найдете копию скрипта VPNGateway.inf VPN предложения IP-адрес пакета в папке политики запросов на сертификат. Только обновить «Тема» и "\_по-прежнему\_" с помощью значений, зависящих от клиента.

```
[Version] 

Signature="$Windows NT$"

[NewRequest]
Subject = "CN=vpn.contoso.com"
Exportable = FALSE   
KeyLength = 2048     
KeySpec = 1          
KeyUsage = 0xA0      
MachineKeySet = True
ProviderName = "Microsoft RSA SChannel Cryptographic Provider"
RequestType = PKCS10 

[Extensions]
2.5.29.17 = "{text}"
_continue_ = "dns=vpn.contoso.com&"
```

## <a name="create-the-vpn-users-vpn-servers-and-nps-servers-groups"></a>Создание пользователей VPN, VPN-серверов и групп серверов политики сети

В этой процедуре можно добавить новую группу Active Directory (AD), содержащий пользователей, которым разрешено использовать VPN-Подключение для подключения к сети организации.

Эта группа служит двум целям:

- Он определяет, какие пользователи могут для автоматической регистрации для сертификатов пользователей, которые требуются VPN-Подключение.

- Он определяет пользователей, которые разрешает сервер политики сети для доступа через VPN.

Используя пользовательскую группу, если потребуется отозвать доступ пользователя VPN, можно удалить этого пользователя из группы.

Можно также добавить группа, содержащая VPN-серверов и другую группу, содержащую серверы политики сети. Эти группы используются для ограничения запросов на сертификаты своих членов.

>[!NOTE]
>Мы рекомендуем VPN серверами, находящимися в DMA/периметра не входить в домен. Тем не менее если требуется, чтобы VPN-серверов, присоединенных к домену для более эффективного управления (групповые политики, агент резервного копирования и мониторинга нет локальных пользователей, для управления и так далее), затем добавить в группу AD к шаблону сертификата сервера VPN.

### <a name="configure-the-vpn-users-group"></a>Настройка группы пользователей VPN

1. На контроллере домена откройте Active Directory — пользователи и компьютеры.

2. Щелкните правой кнопкой мыши контейнер или подразделение, выберите **New**, а затем выберите **группы**.

3. В **имя группы**, введите **пользователей VPN**, а затем выберите **ОК**.

4. Щелкните правой кнопкой мыши **пользователей VPN** и выберите **свойства**.

5. На **члены** вкладке диалогового окна Свойства пользователей VPN, выберите **добавить**.

6. В диалоговом окне Выбор пользователей, добавить всех пользователей, которым требуется доступ к VPN и выберите **ОК**.

7. Закройте окно "Пользователи и компьютеры Active Directory".

### <a name="configure-the-vpn-servers-and-nps-servers-groups"></a>Настройка групп VPN-серверов и серверов политики сети

1. На контроллере домена откройте Active Directory — пользователи и компьютеры.

2. Щелкните правой кнопкой мыши контейнер или подразделение, выберите **New**, а затем выберите **группы**.

3. В **имя группы**, введите **VPN-серверов**, а затем выберите **ОК**.

4. Щелкните правой кнопкой мыши **VPN-серверов** и выберите **свойства**.

5. На **члены** вкладке диалогового окна Свойства VPN-серверов, выберите **добавить**.

6. Выберите **типы объектов**выберите **компьютеров** флажок, а затем выберите **ОК**.

7. В **введите имена выбираемых объектов**, введите имена серверов VPN, а затем выберите **ОК**.

8. Выберите **ОК** чтобы закрыть диалоговое окно Свойства VPN-серверов.

9. Повторите предыдущие шаги для группы серверов политики сети.

10. Закройте окно "Пользователи и компьютеры Active Directory".

## <a name="create-the-user-authentication-template"></a>Создание шаблона проверки подлинности пользователя

В этой процедуре настройки шаблона пользовательского между клиентом и сервером проверки подлинности. Этот шаблон является обязательным, поскольку для повышения общей безопасности сертификата, выбрав уровни совместимости обновленной и выбрав поставщик службы шифрования платформы Майкрософт. Это последнее изменение позволяет использовать доверенный платформенный модуль на клиентских компьютерах для защиты сертификата. Обзор доверенного платформенного МОДУЛЯ, см. в разделе [доверенные Обзор технологии платформы модуля](https://docs.microsoft.com/windows/device-security/tpm/trusted-platform-module-overview).

**Процедура.**

1. В центре сертификации откройте центр сертификации.

2. В области навигации щелкните правой кнопкой мыши **шаблонов сертификатов** и выберите **управление**.

3. В консоли шаблонов сертификатов, щелкните правой кнопкой мыши **пользователя** и выберите **скопировать шаблон**.

   >[!WARNING]
   >Не устанавливайте **применить** или **ОК** в любое время до шаг 10.  Если вы выбираете эти кнопки до ввода всех параметров, множество вариантов становятся основных и больше не может изменять. Например, на **криптографии** вкладку, если _поставщика служб шифрования хранилища прежних версий_ показан в поле "Категория поставщика", она становится недоступной, предотвращая какого-либо изменений. Единственная альтернатива — удалить шаблон, а затем создать ее снова.  

4. В диалоговом окне Свойства нового шаблона на **Общие** вкладку, выполните следующие действия:

   1. В **отображаемое имя шаблона**, тип **проверки подлинности пользователя VPN**.

   2. Очистить **опубликовать сертификат в Active Directory** "флажок".

5. На **безопасности** вкладку, выполните следующие действия:

   1. Нажмите кнопку **Добавить**.

   2. На Выбор пользователей, компьютеров, учетных записей служб или групп диалоговое окно, введите **пользователей VPN**, а затем выберите **ОК**.

   3. В **имена групп или пользователей**выберите **пользователей VPN**.

   4. В **разрешения для пользователей VPN**выберите **регистрация** и **Авторегистрация** флажки в **Разрешить** столбца.

      >[!TIP]
      >Не забудьте оставить флажок чтения. Другими словами необходимо разрешения на чтение для регистрации. 

   5. В **имена групп или пользователей**выберите **пользователей домена**, а затем выберите **удалить**.

6. На **совместимости** вкладку, выполните следующие действия:

   1. В **сертификации**выберите **Windows Server 2012 R2**.

   2. На **последующие изменения** выберите **ОК**.

   3. В **получатель сертификата**выберите **Windows 8.1 и Windows Server 2012 R2**.

   4. На **последующие изменения** выберите **ОК**.

7. На **обработка запроса** вкладке Очистить **Разрешить экспортировать закрытый ключ** "флажок".

8. На **криптографии** вкладку, выполните следующие действия:

   1. В **категория поставщика**выберите **поставщик хранилища ключей**.

   2. Выберите **запросы должны использовать один из следующих поставщиков**.

   3. Выберите **поставщик службы шифрования платформы Майкрософт** "флажок".

9. На **имя субъекта** вкладку, если у вас нет адреса электронной почты, в списке на все учетные записи пользователей, снимите флажок **включить имя электронной почты в имя субъекта** и **имя электронной почты** флажки.

10. Выберите **ОК** для сохранения шаблона сертификата проверки подлинности пользователя VPN.

11. Закройте консоль шаблонов сертификатов.

12. В области навигации оснастки центра сертификации щелкните правой кнопкой мыши **шаблонов сертификатов**выберите **New** , а затем выберите **Выдаваемый шаблон сертификата**.

13. Выберите **проверки подлинности пользователя VPN**, а затем выберите **ОК**.

14. Закройте оснастку центра сертификации.

## <a name="create-the-vpn-server-authentication-template"></a>Создание шаблона проверки подлинности сервера VPN

В этой процедуре можно настроить нового шаблона проверки подлинности сервера для VPN-серверу. Добавление политики приложения промежуточного IKE IP-безопасность (IPsec) позволяет серверу сертификаты фильтр, если доступен более одного сертификата с помощью проверки подлинности сервера расширенное использование ключа.

>[!IMPORTANT]
>Так как VPN-клиентам доступ к этому серверу из Интернета, тему и альтернативные имена отличаются от внутреннее имя сервера. Таким образом вы не можете автоматическая подача заявок этот сертификат на VPN-серверов.

**Предварительные требования:**

VPN-серверы, присоединенные к домену

**Процедура.**

1. В центре сертификации откройте центр сертификации.

2. В области навигации щелкните правой кнопкой мыши **шаблонов сертификатов** и выберите **управление**.

3. В консоли шаблонов сертификатов, щелкните правой кнопкой мыши **RAS и IAS-сервер** и выберите **скопировать шаблон**.

4. В диалоговом окне Свойства нового шаблона на **Общие** на вкладке **отображаемое имя шаблона**, например, введите описательное имя для VPN-сервера, **проверки подлинности сервера VPN** или **сервер RADIUS**.

5. На **расширения** вкладку, выполните следующие действия:

    1. Выберите **политики приложений**, а затем выберите **изменить**.

    2. В **Изменение расширения политик применения** выберите **добавить**.

    3. На **Добавление политики применения** выберите **IP-безопасности IKE-посредник**, а затем выберите **ОК**.
   
        Добавляется IP-адрес IKE EKU безопасности помогает в сценариях, где существует более одного сертификата проверки подлинности сервера на сервере VPN. Если IP-безопасности IKE-посредник, IPSec использует только сертификат с параметрами и EKU. Без этого проверки подлинности IKEv2 может завершиться ошибкой 13801: Неприемлемые учетные данные проверки подлинности IKE.

    4. Выберите **ОК** для возврата **свойства нового шаблона** диалоговое окно.

6. На **безопасности** вкладку, выполните следующие действия:

    1. Нажмите кнопку **Добавить**.

    2. На **Выбор пользователей, компьютеров, учетных записей служб или групп** диалогового окна введите **VPN-серверов**, а затем выберите **ОК**.

    3. В **имена групп или пользователей**выберите **VPN-серверов**.

    4. В **разрешения для VPN-серверов**выберите **регистрация** флажок в **Разрешить** столбца.

    5. В **имена групп или пользователей**выберите **серверы RAS и IAS**, а затем выберите **удалить**.

7. На **имя субъекта** вкладку, выполните следующие действия:

    1. Выберите **предоставляется в запросе**.

    2. На **шаблонов сертификатов** диалоговом окне предупреждения установите **ОК**.

8. (Необязательно) При настройке условного доступа для VPN-подключения, выберите **обработка запроса** , а затем установите **Разрешить экспортировать закрытый ключ**.

9. Выберите **ОК** для сохранения шаблона сертификата VPN-сервера.

10. Закройте консоль шаблонов сертификатов.

11. В области навигации оснастки центра сертификации щелкните правой кнопкой мыши **шаблонов сертификатов**выберите **New** , а затем выберите **Выдаваемый шаблон сертификата**.

12. Выберите имя, выбранное на шаге 4 и нажмите кнопку **ОК**.

13. Закройте оснастку центра сертификации.

## <a name="create-the-nps-server-authentication-template"></a>Создание шаблона проверки подлинности сервера политики сети

Третья и последняя Выдаваемый шаблон сертификата создайте является шаблоном проверки подлинности сервера политики сети. Шаблон проверки подлинности сервера политики сети — это простой копия шаблона RAS и IAS-сервер, защищены в группу NPS-сервер, созданный ранее в этом разделе.

Вы настроите этот сертификат для автоматической регистрации.

**Процедура.**

1. В центре сертификации откройте центр сертификации.

2. В области навигации щелкните правой кнопкой мыши **шаблонов сертификатов** и выберите **управление**.

3. В консоли шаблонов сертификатов, щелкните правой кнопкой мыши **RAS и IAS-сервер**и выберите **скопировать шаблон**.

4. В диалоговом окне Свойства нового шаблона на **Общие** на вкладке **отображаемое имя шаблона**, тип **проверки подлинности сервера политики сети**.

5. На **безопасности** вкладку, выполните следующие действия:

    1. Нажмите кнопку **Добавить**.

    2. На **Выбор пользователей, компьютеров, учетных записей служб или групп** диалогового окна введите **NPS-серверы**, а затем выберите **ОК**.

    3. В **имена групп или пользователей**выберите **NPS-серверы**.

    4. В **разрешений для серверов NPS**выберите **регистрация** и **Авторегистрация** флажки в **Разрешить** столбца.

    5. В **имена групп или пользователей**выберите **серверы RAS и IAS**, а затем выберите **удалить**.

6. Выберите **ОК** для сохранения шаблона сертификата сервера политики сети.

7. Закройте консоль шаблонов сертификатов.

8. В области навигации оснастки центра сертификации щелкните правой кнопкой мыши **шаблонов сертификатов**выберите **New** , а затем выберите **Выдаваемый шаблон сертификата**.

9. Выберите **проверки подлинности сервера политики сети**и выберите **ОК**.

10. Закройте оснастку центра сертификации.

## <a name="enroll-and-validate-the-user-certificate"></a>Зарегистрировать и проверить сертификат пользователя

Так как вы используете групповой политики для автоматической регистрации пользовательских сертификатов, нужно только обновить политику и Windows 10 будут автоматически зарегистрированы для правильного сертификата учетной записи пользователя. Затем можно проверить сертификат в консоли сертификатов.

**Процедура.**

1. Войдите на присоединенных к домену клиентский компьютер является членом **пользователей VPN** группы.

2. Нажмите клавиши Windows + R, введите **gpupdate/force**, и нажмите клавишу ВВОД.

3. В меню «Пуск» введите **certmgr.msc**, и нажмите клавишу ВВОД.

4. В оснастке сертификатов в разделе **личные**выберите **сертификаты**. Сертификаты отображаются в области сведений.

5. Щелкните правой кнопкой мыши сертификат, который имеет свое текущее имя пользователя домена, а затем выберите **откройте**.

6. На **Общие** вкладке, убедитесь, что дата, указанная в разделе **действителен с** используется текущая дата. Если это не так, вы выбрали неправильный сертификат.

7. Выберите **ОК**и закройте оснастку «Сертификаты».

## <a name="enroll-and-validate-the-server-certificates"></a>Регистрация и проверять сертификаты сервера

В отличие от сертификат пользователя необходимо вручную зарегистрировать сертификат VPN-сервера. После вы зарегистрировали его, проверьте его, используя тот же процесс, который использовался для сертификата пользователя. Как и сертификат пользователя на NPS-сервер автоматически регистрирует свой сертификат проверки подлинности, вам нужно всего лишь проверить его.

>[!NOTE]
>Может потребоваться перезагрузите серверы политики сети и VPN, чтобы разрешить ему обновлять их членства в группах, чтобы выполнить следующие действия.

### <a name="enroll-and-validate-the-vpn-server-certificate"></a>Зарегистрировать и проверить сертификат сервера VPN

1. В меню "Пуск" VPN-сервера введите **certlm.msc**, и нажмите клавишу ВВОД.

2. Щелкните правой кнопкой мыши **личные**выберите **все задачи** , а затем выберите **запросить новый сертификат** запустить мастер регистрации сертификатов.

3. На странице «перед началом работы» выберите **Далее**.

4. На странице "Выбор политики регистрации сертификатов" выберите **Далее**.

5. На странице "запрос сертификатов" установите флажок рядом с сервером VPN, чтобы выбрать его.

6. Установите флажок сервер VPN **больше данных** чтобы открыть диалоговое окно свойств сертификата и выполните следующие действия:

    1. Выберите **субъекта** выберите **общее имя** под **имя субъекта**в **тип**.

    2. В разделе **имя субъекта**в **значение**, введите имя внешнего домена клиентов, используемый для подключения к VPN, к примеру, vpn.contoso.com, а затем выберите **добавить**.

    3. В разделе **альтернативное имя**в **тип**выберите **DNS**.

    4. В разделе **альтернативное имя**в **значение**, введите все имена серверов, используемый клиентами для подключения к VPN, к примеру, vpn.contoso.com, vpn, 132.64.86.2.

    5. Выберите **добавить** после ввода каждого имени.

    6. Выберите **ОК** после завершения.

7. Выберите **зарегистрировать**.

8. Выберите **Готово**.

9. В оснастке сертификатов в разделе **личные**выберите **сертификаты**.
    
    В области сведений отображаются указанные сертификаты.

10. Щелкните правой кнопкой мыши сертификат, который имеет VPN-серверу по имени, а затем выберите **откройте**.

11. На **Общие** вкладке, убедитесь, что дата, указанная в разделе **действителен с** используется текущая дата. Если это не так, вы выбрали неправильный сертификат.

12. На **сведения** выберите **расширенное использование ключа**и убедитесь, что **IP-безопасности IKE-посредник** и **проверки подлинности сервера** отобразить в списке.

13. Выберите **ОК** закрыть сертификат.

14. Закройте оснастку «Сертификаты».

### <a name="validate-the-nps-server-certificate"></a>Проверить сертификат сервера политики сети

1. Перезапустите сервер политики сети.

2. В меню "Пуск" сервер политики сети введите **certlm.msc**, и нажмите клавишу ВВОД.

3. В оснастке сертификатов в разделе **личные**выберите **сертификаты**.

    В области сведений отображаются указанные сертификаты.

4. Щелкните правой кнопкой мыши сертификат с сервера политики сети по имени, а затем выберите **откройте**.

5. На **Общие** вкладке, убедитесь, что дата, указанная в разделе **действителен с** используется текущая дата. Если это не так, вы выбрали неправильный сертификат.

6. Выберите **ОК** закрыть сертификат.

7. Закройте оснастку «Сертификаты».
