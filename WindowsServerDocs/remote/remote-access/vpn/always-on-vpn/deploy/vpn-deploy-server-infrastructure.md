# Шаг 2. Настройка инфраструктуры сервера

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows Server 2012 R2, Windows 10

& #171;  [ **Предыдущих:** шаг 1. Планирование развертывания постоянно включенной VPN](always-on-vpn-deploy-planning.md)<br>
& #187;  [ **Далее:** шаг 3. Настройка сервера удаленного доступа для всегда к VPN](vpn-deploy-ras.md)


На этом этапе можно установить и настроить серверные компоненты, необходимые для поддержки VPN-Подключение. Серверные компоненты включают Настройка инфраструктуры открытых КЛЮЧЕЙ для распространения сертификатов, используемых пользователей, VPN-сервера и сервера политики сети.  Можно также настроить RRAS для поддержки подключений IKEv2 и сервера политики сети для выполнения авторизации для VPN-подключений.

## Настройка автоматической регистрации сертификатов в групповой политике
В этой процедуре настройки групповой политики на контроллере домена таким образом, входящих в состав домена автоматически запрашивают сертификаты пользователей и компьютеров. Это позволит пользователям VPN для запроса и получения сертификатов пользователей, которые автоматически проверки подлинности VPN-подключений. Аналогичным образом, эта политика позволяет серверами NPS для запроса сервера сертификатов проверки подлинности автоматически. 

Можно вручную регистрировать сертификаты на VPN-серверов.

>[!TIP]
>Не domained к домену компьютеров см. в разделе ниже [Конфигурация ЦС для недоменных к домену компьютеры](#ca-configuration-for-non-domain-joined-computers) . Так как сервер RRAS не присоединение к домену, автоматической регистрации не может использоваться для регистрации сертификата шлюза VPN.  Таким образом используйте процедуру запроса автономного ЦС. 


1.  На контроллере домена откройте Управление групповой политикой.

2.  В области навигации щелкните правой кнопкой мыши домен (например, corp.contoso.com) и нажмите кнопку**создать объект групповой Политики в этом домене и связать его здесь**.

3.  В диалоговом окне новый объект групповой Политики введите **Политики автоматической регистрации**и нажмите кнопку **ОК**.

4.  В области навигации щелкните правой кнопкой мыши **Политику автоматической регистрации**и нажмите кнопку **Изменить**.

5.  В редакторе групповой политики управления, выполните следующие действия для настройки автоматической регистрации сертификатов компьютера:

    1.  В области навигации щелкните **Политики компьютера Configuration\\Policies\\Windows Settings\\Security Settings\\Public ключа**.

    2.  В области сведений щелкните правой кнопкой мыши**Службы клиента — автоматической регистрации сертификатов**и нажмите кнопку **Свойства**.

    3.  На клиент служб сертификации: автоматическая регистрация свойства, в **Модель конфигурации**, нажмите кнопку " **включено**".

    4.  Выберите**обновлять сертификаты с истекшим сроком действия или в состоянии ожидания и удалять отозванные сертификаты**и**обновлять сертификаты, использующие шаблоны сертификатов**.

    5.  Нажмите кнопку**ОК**.

6.  В редакторе групповой политики управления, выполните следующие действия, чтобы настроить автоматической регистрации сертификатов пользователя.

    1.  В области навигации щелкните **Политики пользователя Configuration\\Policies\\Windows Settings\\Security Settings\\Public ключа**.

    2.  В области сведений щелкните правой кнопкой мыши**Службы клиента — автоматической регистрации сертификатов**и нажмите кнопку **Свойства**.

    3.  На клиент служб сертификации: автоматическая регистрация свойства, в **Модель конфигурации**, нажмите кнопку " **включено**".

    4.  Выберите**обновлять сертификаты с истекшим сроком действия или в состоянии ожидания и удалять отозванные сертификаты**и**обновлять сертификаты, использующие шаблоны сертификатов**.

    5.  Нажмите кнопку**ОК**.

    6.  Закройте редактор управления групповыми политиками.

7.  Закрыть управления групповыми политиками.

### Конфигурация ЦС для компьютеров, не присоединенными к домену

Так как сервер RRAS не присоединение к домену, автоматической регистрации не может использоваться для регистрации сертификата шлюза VPN.  Таким образом используйте процедуру запроса автономного ЦС. 

1. На сервере RRAS создайте файл под названием **VPNGateway.inf** на основе политики сертификатов пример запроса, представленным в приложении (раздел 0) и настроить следующие записи: 

   - В разделе [NewRequest] замените vpn.contoso.com, используемый в качестве имени субъекта с выбранного конечной точкой [_клиент_] VPN полное доменное имя.

   - В разделе [расширения] замените vpn.contoso.com, используемый для альтернативное имя субъекта с выбранного конечной точкой [_клиент_] VPN полное доменное имя.

2. Сохраните или скопировать файл **VPNGateway.inf** в выбранном расположении.

3. В командной строке с повышенными привилегиями перейдите в папку, содержащую файл **VPNGateway.inf** и введите:

   ```
   certreq -new VPNGateway.inf VPNGateway.req
   ```

4. Скопируйте созданный выходной файл **VPNGateway.req** сервера центра сертификации или на рабочую станцию привилегированным доступом (ЛАПЕ). 

5. Сохраните или скопируйте файл **VPNGateway.req** в выбранном расположении на сервере центра сертификации или на рабочую станцию привилегированным доступом (ЛАПЕ).

6. В командной строке с повышенными привилегиями перейдите в папку, содержащую файл VPNGateway.req, созданный в предыдущем шаге и тип: 

   ```
   certreq -attrib “CertificateTemplate:[Customer]VPNGateway” -submit VPNgateway.req VPNgateway.cer
   ```

7. Если запрос с помощью окна Список центр сертификации, выберите соответствующий ЦС предприятия для обслуживания запроса сертификата.

8. Скопируйте созданный выходной файл **VPNGateway.cer** сервер RRAS. 

9. Сохраните или скопировать файл **VPNGateway.cer** в выбранном расположении на сервер RRAS.

10. В командной строке с повышенными привилегиями перейдите в папку, содержащую файл VPNGateway.cer, созданный в предыдущем шаге и тип:
   
   ```
   certreq -accept VPNGateway.cer
   ```

11. Запустите оснастку MMC Сертификаты как описано [здесь](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in) при выборе параметра **учетной записи компьютера** .

12. Убедитесь, что существует действительный сертификат для сервера RRAS со следующими свойствами:

   - **Назначения:** Проверки подлинности сервера, промежуточных IKE IP-безопасности 

   - **Шаблон сертификата:** [_Клиент_] VPN-сервер

#### Пример: VPNGateway.inf сценария

Здесь можно просмотреть пример сценария политики запроса сертификата, используемое для запроса сертификата шлюз VPN с помощью процесса резервному.

>[!TIP]
>Копия VPNGateway.inf сценария можно найти в VPN предлагать IP-адрес комплекта в папке политики запроса сертификата. Обновите только «Субъект» и «\_continue\_» с помощью клиента определенные значения.

```

[NewRequest]

Exportable = FALSE

KeySpec = 1          
KeyUsage = 0xA0

ProviderName = "Microsoft RSA SChannel Cryptographic Provider"
RequestType = PKCS10 

[Extensions]
2.5.29.17 = "{text}"
_continue_ = "dns=vpn.contoso.com&"

```

## Создайте пользователей VPN, VPN-серверов и группы серверов NPS

В этой процедуре можно добавить группу Active Directory (AD), содержащий пользователи, которым разрешено использовать VPN для подключения к сети вашей организации.

Эта группа служит двум целям:

-   Он определяет, какие пользователи имеют право для автоматической регистрации сертификатов пользователя, требуемые для VPN-Подключение.

-   Он определяет, какие пользователи авторизует NPS для доступа через VPN.

С помощью пользовательской группы, если позже потребуется отозвать доступ к VPN пользователя, можно удалить этого пользователя из группы.

Можно также добавить группу, содержащую VPN-серверов и другую группу, содержащую серверами NPS. Используйте эти группы для ограничения запросы сертификатов к их членам.

>[!NOTE] 
>Мы рекомендуем использовать VPN серверы, находящиеся в DMA/периметра не присоединенный к домену. Тем не менее если вы хотите, чтобы VPN-серверов, присоединенный к домену для лучшей управляемости (групповых политик, агент резервного копирования/мониторинг нет локальных пользователей и т. д), добавьте группы AD для шаблона сертификата сервера VPN.

### Настройка группы пользователей VPN

1.  На контроллере домена откройте Active Directory — пользователи и компьютеры.

2.  Щелкните правой кнопкой мыши контейнер или подразделение, нажмите кнопку **Создать** и выберите **группу**.

3.  **Имя группы**введите **Пользователей VPN**и нажмите кнопку **ОК**.

4.  Щелкните правой кнопкой мыши **Пользователей VPN**и нажмите кнопку **Свойства**.

5.  На вкладку " **члены** " диалогового окна свойств пользователей VPN нажмите кнопку " **Добавить**".

6.  В диалоговом окне Выбор пользователей, добавьте всех пользователей, которым требуется VPN доступ и нажмите кнопку **ОК**.

7.  Закройте пользователи и компьютеры Active Directory.

### Настройка групп VPN-серверов и серверами NPS

1.  На контроллере домена откройте Active Directory — пользователи и компьютеры.

2.  Щелкните правой кнопкой мыши контейнер или подразделение, нажмите кнопку **Создать** и выберите **группу**.

3.  **Имя группы**введите **VPN-серверов**и нажмите кнопку **ОК**.

4.  Щелкните правой кнопкой мыши **VPN-серверов**и нажмите кнопку **Свойства**.

5.  На вкладке **члены** диалогового окна Свойства серверов VPN нажмите кнопку " **Добавить**".

6.  Нажмите кнопку **Типы объектов**, установите флажок " **компьютеры** " и нажмите кнопку **ОК**.

7.  В поле **Введите имена выбираемых объектов**введите имена VPN-серверов и нажмите кнопку **ОК**.

8.  Нажмите кнопку **ОК** , чтобы закрыть диалоговое окно Свойства серверов VPN.

9.  Повторите предыдущие действия для группы серверов политики сети.

10. Закройте пользователи и компьютеры Active Directory.

## Создание шаблона для проверки подлинности пользователей

В этой процедуре настройки шаблон проверки подлинности пользовательских клиент сервер. Этот шаблон не требуется, так как для повышения общей безопасности сертификата, выбрав уровни нужного совместимости и выбрать поставщик службы шифрования платформы Microsoft. Это последнее изменение позволяет использовать доверенный платформенный модуль на клиентских компьютерах для защиты сертификата. Обзор доверенного платформенного МОДУЛЯ см. в разделе [Модуль Обзор технологии доверенного платформенного](https://docs.microsoft.com/windows/device-security/tpm/trusted-platform-module-overview).

**Процедура:**

1.  В центре сертификации откройте центр сертификации.

2.  В области навигации щелкните правой кнопкой мыши**Шаблонов сертификатов**и выберите команду**Управление**.

3.  В консоли шаблонов сертификатов щелкните правой кнопкой мыши**пользователя**и нажмите кнопку**Скопировать шаблон**.

    >[!WARNING]
    >Не нажимайте кнопку **ОК** или **Применить** в любое время до шаг 10.  Если щелкнуть эти кнопки перед вводом все параметры, множество вариантов становятся основных и больше не редактируется. Например, на вкладке " **шифрование** " Если _Поставщик служб шифрования хранилища устаревшие_ отображается в поле Категория поставщика становится будет отключен, тем самым предотвратив дальнейших изменений. Единственной альтернативой является удалить шаблон и повторно создать ее.  

4.  В диалоговом окне свойств нового шаблона на**Общие**вкладке, выполните следующие действия:

    1.  В **отображаемое имя шаблона**тип **Проверки подлинности пользователей VPN**.

    2.  Снимите флажок **Опубликовать сертификат в Active Directory** .

5.  Средство обеспечения**безопасности**навкладке, выполните следующие действия:

    1.  Нажмите кнопку **Добавить**.

    2.  На Выбор пользователей, компьютеров, учетных записей служб или групп диалоговое окно введите **Пользователей VPN**и нажмите кнопку **ОК**.

    3.  В **имена пользователей или групп**нажмите кнопку **Пользователей VPN**.

    4.  В списке **разрешений для пользователей VPN**установите флажки **Регистрация** и **Автоматическая регистрация** в столбце " **Разрешить** ".

       >[!TIP]
       >Убедитесь, что для чтения установлен флажок. Другими словами необходимы разрешения чтения для регистрации. 

    5.  В **имена пользователей или групп**выберите **Пользователей домена**и кнопку **Удалить**.

6.  На вкладке " **совместимости** " выполните следующие действия:

    1.  В **Центр сертификации**выберите **Windows Server2012R2**.

    2.  В диалоговом окне **Resulting изменений** нажмите кнопку " **ОК**".

    3.  В **Получатель сертификата**щелкните **Windows8.1 или Windows Server2012R2**.

    4.  В диалоговом окне **Resulting изменений** нажмите кнопку " **ОК**".

7.  На вкладке **Обработка запроса** снимите флажок « **Разрешить экспортировать закрытый ключ** ».

8.  На вкладке " **шифрование** " выполните следующие действия:

    1.  **Категория поставщика**щелкните **Поставщик хранилища ключей**.

    2.  Щелкните **запросы должны использовать один из следующих поставщиков**.

    3.  Установите флажок **Поставщик службы шифрования платформы Microsoft** .

9.  На вкладке **Имя субъекта** Если у вас нет адреса электронной почты для всех учетных записей пользователей в списке, снимите флажки **Включить имя электронной почты в имя субъекта** и **имя электронной почты** .

10. Нажмите кнопку**ОК** , чтобы сохранить шаблон сертификата проверки подлинности пользователей VPN.

11. Консоль шаблонов закрыть сертификата.

12. В области навигации theCertification Authoritysnap-in, щелкните правой кнопкой мыши**Шаблонов сертификатов**, щелкните **Создать** и нажмите кнопку**Выдаваемый шаблон сертификата**.

13. Выберите**VPN проверка подлинности пользователя**и нажмите кнопку**ОК**.

14. Закройте оснастку theCertification центр сертификации.

## Создание шаблона для проверки подлинности сервера VPN

В этой процедуре можно настроить новый шаблон проверки подлинности сервера для VPN-сервера. Добавлять политику приложений промежуточных IKE IP-безопасности (IPsec) позволяет серверу фильтра сертификаты, если доступно несколько сертификатов с проверкой подлинности сервера расширенного использования ключей.

>[!IMPORTANT] 
>Так как VPN-клиентам доступ к этому серверу через Интернет, тему и альтернативные названия отличаются от имени внутреннего сервера. В результате вы не можете Автоматическая регистрация этот сертификат на VPN-серверов.

**Предварительные требования:**<p>
Присоединенный к домену VPN-серверов

**Процедура:** 

1.  В центре сертификации откройте центр сертификации.

2.  В области навигации щелкните правой кнопкой мыши **Шаблонов сертификатов**и выберите команду**Управление**.

3.  В консоль шаблонов сертификатов щелкните правой кнопкой мыши**RAS и IAS-сервер**и нажмите кнопку**Скопировать шаблон**.

4.  В диалоговом окне свойств нового шаблона на**Общие**вкладке в **отображаемое имя шаблона**, введите описательное имя для VPN-сервера, например, **Подлинности VPN-сервера** или **Сервера RADIUS**.

5.  **Расширения**вкладке, выполните следующие действия:

    1.  Щелкните**Политики приложений**и нажмите кнопку**Изменить**.

    2.  В диалоговом окне **Изменение расширения политик применения** нажмите кнопку "**Добавить**".

    3.  В диалоговом окне **Добавление политики применения** щелкните **IP-безопасности IKE промежуточных**и нажмите кнопку **ОК**.<p>Добавление IP-адрес безопасности IKE промежуточных EKU помогает в сценариях, где существует несколько сертификат проверки подлинности сервера на сервере VPN. При наличии промежуточных IKE IP-безопасности IPSec используются только сертификат с оба этих способа EKU. Без этой проверки подлинности IKEv2 может завершиться ошибкой 13801: Неприемлемые учетные данные проверки подлинности IKE.

    4.  Нажмите кнопку **ОК** , чтобы вернуться к**Свойства нового шаблона**диалоговое окно.

6.  Средство обеспечения**безопасности**навкладке, выполните следующие действия:

    1.  Нажмите кнопку **Добавить**.

    2.  В диалоговом окне **Выбор пользователей, компьютеров, учетных записей служб или групп** введите **VPN-серверов**и нажмите кнопку **ОК**.

    3.  В **имена пользователей или групп**выберите **VPN-серверов**.

    4.  В списке **разрешений для VPN-серверов**установите флажок **Регистрация** в столбце " **Разрешить** ".

    5.  В **имена пользователей или групп**выберите **Серверы RAS и IAS**и кнопку **Удалить**.

7.  На вкладке " **Имя субъекта** " выполните следующие действия:

    1.  Нажмите кнопку, **предоставляется в запросе**.

    2.  В диалоговом окне предупреждения **Шаблонов сертификатов** нажмите кнопку " **ОК**".

8.  (Необязательно) *Если вы настраиваете условного доступа для подключения VPN*, перейдите на вкладку **Обработка запроса** и нажмите кнопку **Разрешить экспортировать закрытый ключ** , чтобы выделить его.

9.  Нажмите кнопку**ОК** , чтобы сохранить шаблон сертификата VPN-сервера.

10. Консоль шаблонов закрыть сертификата.

11. В области thenavigation сертификации в Authoritysnap щелкните правой кнопкой мыши**Шаблонов сертификатов**, нажмите кнопку **Создать** и нажмите кнопку**Выдаваемый шаблон сертификата**.

12. Перезапустите службы сертификации.

13. Выберите имя, которое вы выбрали на шаге 4 и нажмите кнопку**ОК**.

14. Закройте оснастку theCertification центр сертификации.

## Создание шаблона для проверки подлинности сервера политики сети

Шаблон сертификата третья и последняя создать стандартный шаблон проверки подлинности сервера политики сети. Шаблон проверки подлинности сервера политики сети — простой копию шаблона RAS и IAS-сервер, защищенными в группе сервера политики сети, которую вы создали ранее в этом разделе. 

Необходимо настроить этот сертификат для автоматической регистрации.

**Процедура:**

1.  В центре сертификации откройте центр сертификации.

2.  В области навигации щелкните правой кнопкой мыши **Шаблонов сертификатов**и выберите команду**Управление**.

3.  В консоль шаблонов сертификатов щелкните правой кнопкой мыши**RAS и IAS-сервер**и выберите **Скопировать шаблон**.

4.  В диалоговом окне свойств нового шаблона на**Общие**вкладке **отображаемое имя шаблона**, тип **Проверки подлинности сервера политики сети**.

5.  Средство обеспечения**безопасности**навкладке, выполните следующие действия:

    1.  Нажмите кнопку **Добавить**.

    2.  В диалоговом окне **Выбор пользователей, компьютеров, учетных записей служб или групп** введите **Серверами NPS**и нажмите кнопку **ОК**.

    3.  В **имена пользователей или групп**нажмите кнопку **Серверами NPS**.

    4.  В списке **разрешений для серверов NPS**установите флажки **Регистрация** и **Автоматическая регистрация** в столбце " **Разрешить** ".

    5.  В **имена пользователей или групп**выберите **Серверы RAS и IAS**и кнопку **Удалить**.

6.  Нажмите кнопку**ОК** , чтобы сохранить шаблон сертификата сервера политики сети.

7.  Консоль шаблонов закрыть сертификата.

8.  В области thenavigation сертификации в Authoritysnap щелкните правой кнопкой мыши**Шаблонов сертификатов**, нажмите кнопку **Создать** и нажмите кнопку**Выдаваемый шаблон сертификата**.

9.  Щелкните**Проверки подлинности сервера политики сети**и нажмите кнопку**ОК**.

10. Закройте оснастку theCertification центр сертификации.

## Регистрация и проверить сертификат пользователя

Так как вы используете групповую политику для автоматической регистрации сертификатов пользователей, необходимо только обновить политику и Windows10 будет автоматической регистрации в учетной записи пользователя для нужного сертификата. Затем можно проверить сертификат в хранилище сертификатов консоли.

**Процедура:**

1.  Вход на присоединенный к домену клиентский компьютер является членом группы **Пользователей VPN** .

2.  Нажатие клавиши Windows + R, введите **gpupdate/force**и нажмите клавишу ВВОД.

3.  В меню "Пуск" введите **certmgr.msc**и нажмите клавишу ВВОД.

4.  В оснастке сертификатов щелкните в папке **личных** **сертификатов**. Сертификаты отображаются в области сведений.

5.  Щелкните правой кнопкой мыши сертификат, который имеет свое текущее имя пользователя домена и нажмите кнопку **Открыть**.

6.  На вкладке " **Общие** " Убедитесь, что даты, указанной в столбце **Срок действия с** используется текущая дата. Если это не так, вы выбрали неверный сертификат.

7.  Нажмите кнопку **ОК**и закройте оснастки сертификатов.

## Регистрация и проверка сертификатов сервера

В отличие от сертификат пользователя необходимо вручную зарегистрировать сертификат сервера VPN. После того, как регистрации его, проверьте его, используя том же процессе, которая использовалась для сертификат пользователя. Как сертификат пользователя сервера политики сети автоматически регистрирует его сертификат проверки подлинности, поэтому все, что нужно сделать, — это проверить его.

>[!NOTE] 
>Может потребоваться перезапустить серверы VPN и NPS позволить им для обновления членства в группах, прежде чем выполнять следующие действия.

### Регистрация и проверить сертификат сервера VPN

1.  На VPN-сервер меню "Пуск" введите **certlm.msc**и нажмите клавишу ВВОД.

2.  Щелкните правой кнопкой мыши **Личные**, выберите **Все задачи** и нажмите кнопку **Запросить новый сертификат** , чтобы открыть мастер регистрации сертификатов.

3.  На странице Перед началом работы нажмите кнопку **Далее**.

4.  На странице "Выбор политики регистрации сертификатов" нажмите кнопку " **Далее**".

5.  На странице запрос сертификатов установите флажок рядом с VPN-сервер, чтобы выделить его.

6.  В группе флажок сервер VPN выберите **Дополнительные сведения, необходимые** для отображения диалогового окна Свойства сертификата и выполните следующие действия:

    1.  Щелкните вкладку " **субъект** ", щелкните **Общее имя** в поле **имени субъекта**в **типе**.

    2.  В группе **имя субъекта**, в **значение**введите имя внешнего домена клиентов, которые используются для подключения к VPN, например, vpn.contoso.com и нажмите кнопку **Добавить**.

    3.  **Имя субъекта**, в **Тип**установите для **DNS**.

    4.  В поле **Имени субъекта**в **значение**, введите все сервера, который имена клиенты используют для подключения к виртуальной частной сети, например, vpn.contoso.com, vpn, 132.64.86.2.

    5.  После ввода каждого имени нажмите кнопку " **Добавить** ".

    6.  По завершении нажмите кнопку **ОК**.

7.  Нажмите кнопку **Регистрация**.

8.  Нажмите кнопку **Готово**.

9.  В оснастке сертификатов щелкните в папке **личных** **сертификатов**.<p>Список сертификатов отображаются в области сведений.

10. Щелкните правой кнопкой мыши сертификат, который имеет имя VPN-сервер и нажмите кнопку **Открыть**.

11. На вкладке " **Общие** " Убедитесь, что даты, указанной в столбце **Срок действия с** используется текущая дата. Если это не так, вы выбрали неверный сертификат.

12. На вкладке " **сведения** " щелкните **Улучшенного ключа**и убедиться, что **IP-безопасности IKE промежуточных** и **Проверки подлинности сервера** отображаются в списке.

13. Нажмите кнопку **ОК** , чтобы закрыть сертификата.

14. Закройте оснастку «Сертификаты».

### Проверки сертификата сервера политики сети

1.  Перезапустите сервер политики сети.

2.  На сервер политики сети меню "Пуск" введите **certlm.msc**и нажмите клавишу ВВОД.

3.  В оснастке сертификатов щелкните в папке **личных** **сертификатов**.<p>Список сертификатов отображаются в области сведений.

4.  Щелкните правой кнопкой мыши сертификат, который имеет имя сервера политики сети и нажмите кнопку **Открыть**.

5.  На вкладке " **Общие** " Убедитесь, что даты, указанной в столбце **Срок действия с** используется текущая дата. Если это не так, вы выбрали неверный сертификат.

6.  Нажмите кнопку **ОК** , чтобы закрыть сертификата.

7.  Закройте оснастку «Сертификаты».
