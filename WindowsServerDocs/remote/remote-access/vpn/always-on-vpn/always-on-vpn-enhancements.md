# Усовершенствования постоянно подключенного VPN-профиля

>Область применения: Windows Server (Semi-Annual Channel), Windows Server 2016, Windows 10

& #171; [ **Предыдущая:** сведения о возможностях VPN](../vpn-map-da.md)<br>
& #187; [ **Далее:** сведения о технологии VPN](always-on-vpn-technology-overview.md)

VPN имеет много преимуществ перед решения VPN в Windows в предыдущих. Следующие важные усовершенствования соответствуют VPN с помощью облачных, мобильные видение корпорации Майкрософт:

- **Интеграция платформы:** VPN имеет Улучшенная интеграция с операционной системой Windows и сторонними решениями для обеспечения надежной платформы бесчисленные сценарии расширенного подключения.

- **Безопасности:** VPN обладает возможностями, новые, улучшенные средства безопасности для ограничения типа трафика, какие приложения могут использовать VPN-подключение, и какие методы проверки подлинности, его можно использовать для запуска подключения. Когда подключение активно в большинстве случаев, особенно важно для защиты подключения. Дополнительные сведения см. в разделе [Параметры проверки подлинности VPN](https://docs.microsoft.com/en-us/windows/security/identity-protection/vpn/vpn-authentication).

- **Подключения к виртуальной частной сети:** VPN, независимо от туннеля устройства предоставляет возможность автоматически инициируемых. Прежде чем VPN не удалось возможность запуска автоматического подключения через пользователя или проверка подлинности устройства.  

- **Управления сеть:** VPN позволяет администраторам определять политики маршрутизации на более детальном уровне — даже до отдельного приложения, который идеально подходит для приложений из бизнес-(LOB), которым требуется особый удаленного доступа.  VPN полностью совместима с протокол Интернета версии 4 (IPv4) и версии 6 (IPv6). В отличие от DirectAccess нет никакой определенной зависимости на IPv6.

  >[!Note]
  >Перед началом работы убедитесь, что включить IPv6 на VPN-сервер. В противном случае не удается установить подключение и отображает сообщение об ошибке.
  
- **Конфигурации и совместимость:** VPN можно развернуть и управлять несколькими способами, что дает несколько преимуществ над другим программным обеспечением клиент VPN VPN.

## Интеграция платформы

Microsoft появились или улучшены следующие возможности интеграции в VPN:

| Основные улучшения | Описание |
| ---- | ---- |
| **[Windows Information Protection (WIP)](https://docs.microsoft.com/windows/threat-protection/windows-information-protection/protect-enterprise-data-using-wip)** | Интеграция с WIP позволяет применение сетевой политики определить, является ли трафик разрешен проходить через VPN-Подключение. Если имеется активный профиль пользователя и применяются политики WIP, VPN автоматически запускается для подключения. Кроме того при использовании WIP, нет необходимости указание правила требуется и TrafficFilterList отдельно в профиле VPN (Если вы хотите более сложная конфигурация), так как политики WIP и списки приложений автоматически вступают в силу. |
| **[Windows Hello для бизнеса](https://docs.microsoft.com/windows/access-protection/hello-for-business/hello-overview)** | VPN по умолчанию поддерживает Windows Hello для бизнеса (в режиме проверки подлинности на основе сертификатов) для предоставления непрерывное единого входа в систему для обоих вход в систему на компьютере и подключение к виртуальной частной сети. Таким образом не вторичной проверки подлинности (учетные данные пользователя) не требуется для VPN-подключения, что позволяет использовать постоянное подключение с помощью Windows Hello для бизнеса проверки подлинности. |
| **[Условного доступа в Microsoft Azure](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-controls)** | Всегда на VPN-клиент можно интегрировать с помощью платформы Azure условного доступа для принудительного применения многофакторную проверку подлинности (MFA), соответствия устройств или сочетание этих двух. Когда политикам условного доступа, Azure Active Directory (Azure AD) выдает кратковременные (по умолчанию 60 минут) сертификат проверки подлинности IP-безопасности (IPsec), который можно использовать для проверки подлинности VPN-шлюза. Соответствия устройств использует политики соответствия System Center Configuration Configuration Manager/Intune, которые могут включать состояние работоспособности устройства как часть проверки соответствия подключения. |
| **Предоставление Azure MFA** | В сочетании с помощью службы службы проверки подлинности удаленных пользователей (RADIUS) и расширение сервера политики сети (NPS) для Azure MFA, проверки подлинности для VPN можно использовать строгую MFA. |
| **Подключаемого модуля VPN сторонних поставщиков** | С помощью универсальной платформы Windows (UWP), сторонние поставщики VPN можно создать одно приложение для полный диапазон Windows 10 устройства. UWP обеспечивает гарантированный уровень базового API на разных устройствах, исключив сложность и устраняет проблемы, связанные с написанием драйверов на уровне ядра. В настоящее время существуют подключаемых модулей VPN в Windows 10 UWP для [Pulse Secure](https://www.microsoft.com/p/pulse-secure/9nblggh3b0bp), [F5 Access](https://www.microsoft.com/p/f5-access/9wzdncrdsfn0), [Check Point Capsule VPN](https://www.microsoft.com/p/check-point-capsule-vpn/9wzdncrdjxtj), [FortiClient](https://www.microsoft.com/p/forticlient/9wzdncrdh6mc), [SonicWall Mobile Connect](https://www.microsoft.com/p/sonicwall-mobile-connect/9wzdncrdsfkz)и [GlobalProtect](https://www.microsoft.com/p/globalprotect/9nblggh6bzl3); без сомнения других пользователей будет отображаться в будущем. |

## Безопасность

Основные улучшения безопасности приведены в следующих областях:

| Основные улучшения | Описание |
| ---- | ---- |
| **Фильтры трафика** | Фильтры трафика вы можете указать клиентского политик, которые определяют, какой трафик, допустимый в корпоративной сети. Таким образом администраторы могут применять ограничения приложения или трафика для интерфейса VPN, ограничить его использование отдельных источников, конечного портов и IP-адреса. Доступны два типа правил фильтрации:<ul><li>**Правила на основе приложений.** Правила брандмауэра на основе приложений зависят от указанного приложения из списка, таким образом, чтобы только трафик, исходящий из этих приложений запрещено проходить через интерфейс VPN.</li><li>**Правила на основе трафика.** Правила на основе трафика брандмауэра основаны на требования к сети, такие как порты, адреса и протоколы. Используйте эти правила только для трафика, который соответствует этим определенным условиям разрешены проходить через интерфейс VPN.<p><p>_**Примечание.**_<br>Эти правила применяются только к исходящий трафик от устройства. Использование трафика фильтрует блоки входящий трафик от корпоративной сети клиенту. </li></ul> |
| **VPN для каждого приложения** | VPN для отдельного приложения — это похоже на фильтра трафика на основе приложений, но оно пройдет дальше для объединения триггеры приложения с помощью фильтра трафика на основе приложения таким образом, чтобы подключаться к сети VPN ограничивается для определенного приложения, в отличие от всех приложений на VPN-клиент. Функция автоматически инициирует при запуске приложения. |
| **Поддержка настраиваемый алгоритмов шифрования IPsec** | VPN поддерживает использование RSA и эллиптических кривых на основе шифрования пользовательских криптографических алгоритмов, в соответствии с кэшированием государственных организаций и политиками безопасности организации. |
| **Встроенная поддержка расширяемый протокол проверки подлинности (EAP)** | VPN по умолчанию поддерживает EAP, который позволяет использовать широкого набора корпорации Майкрософт и сторонних поставщиков EAP типов в рамках процесса проверки подлинности. Протокол EAP обеспечивает безопасную проверку подлинности на основе следующих типов проверки подлинности:<ul><li>Имя пользователя и пароль</li><li>Смарт-карты (физических и виртуальных)</li><li>Сертификаты пользователей</li><li>Windows Hello для бизнеса</li><li>Поддержка MFA через интеграции РАДИУС EAP</li></ul>Поставщик приложения управляет методам подключаемого модуля проверки подлинности VPN платформы UWP третьих лиц, несмотря на то, что у них есть массив из доступных вариантов, включая типы пользовательских учетных данных "и" Поддержка OTP. |

## VPN-подключения

Ниже приведены основные улучшения подключения к VPN.

| Основные улучшения | Описание |
| ---- | ---- |
| **Режим "Всегда включен"** | Постоянная — это функция Windows 10, которая позволяет активному профилю VPN автоматически подключаться и остается подключенным в зависимости от триггеры — а именно, вход пользователя в систему, изменения состояния сети или активного экрана устройства. Всегда включен также участвует в подключенной текущего режима ожидания возможности для повышения уровня заряда батареи. |
| **Запуск приложения** | Можно настроить профили VPN в Windows 10 для автоматического подключения при запуске указанного набора приложений. Можно настроить рабочего стола и приложений UWP для активации VPN-подключение. |
| **На основе имени активации** | С помощью VPN можно определить правила, таким образом, чтобы запросы имени указанного домена активировать VPN-подключение. Windows 10 теперь поддерживает на основе имени активирующего присоединенный к домену и неподключенные к домену компьютеров (ранее поддерживались только неподключенные к домену компьютеров). |
| **Обнаружение доверенной сети** | Эту функцию, чтобы убедиться, что подключение VPN не происходит, если пользователь подключен к доверенной сети корпоративных границах включает в себя VPN. Эту функцию можно объединить с любой из методов активирующее говорилось плавные взаимодействия пользователя «подключаться, только если». |
| **[Туннеля устройства](../vpn-device-tunnel-config.md)** | VPN дает возможность создания специальной профиль VPN для устройства или компьютера. В отличие от _Туннеля пользователя_, который подключается только после входа пользователя в систему на устройстве или компьютере, _Туннеля устройства_ позволяет VPN для установления подключения, прежде чем вход пользователя в систему. Туннеля устройства и пользователя туннеля работать независимо друг от друга, с помощью профилей VPN, можно подключить одновременно и могут использовать различные методы проверки подлинности и другие параметры конфигурации VPN соответствующим образом. |

## Сеть

Ниже приведены некоторые сетевые усовершенствования в VPN:

| Основные улучшения | Описание |
| ---- | ---- |
| **Поддержка двойной стек IPv4 и IPv6** | VPN по умолчанию поддерживает использование IPv4 и IPv6-адресов в стек двумя подход. Он имеет независимо от определенных на одного протокола над другим, что обеспечивает максимальную совместимость приложений IPv4 или IPv6, в сочетании с поддержкой для будущих IPv6 сетевых потребностей.<p>**_Примечание._** Перед началом работы убедитесь, что включить IPv6 на VPN-сервер. В противном случае не удается установить подключение и отображает сообщение об ошибке.|
| **Политики маршрутизации конкретного приложения** | Помимо определения глобального VPN подключения политики маршрутизации для разделения трафика Интернета и интрасети, можно добавить маршрутизации политики для управления использованием разделение туннеля или принудительного туннеля конфигураций на основе каждого приложения. Этот параметр дает более тщательный контроль над тем, какие приложения могут взаимодействовать с каким ресурсам через туннель VPN. |
| **Маршруты исключения** | VPN поддерживает возможность указать маршруты исключения, определяющих, в частности маршрутизации поведение для определения трафик должен проходить только VPN и не проходить через физический сетевой интерфейс.<p><p>_**Примечания.**_<br>-Маршруты исключения в настоящее время не работает трафика в той же подсети, что и клиент, например, LinkLocal.<br>-Маршруты исключения, работают только при настройке разделение туннеля. |

## Совместимость и конфигурации 

Ниже приведены некоторые улучшения совместимость и конфигурации VPN:

| Основные улучшения | Описание |
| ---- | ---- |
| **Совместимость шлюз VPN сторонних поставщиков** | Клиент VPN не требует использования на основе Microsoft VPN-шлюз для работы. Благодаря поддержке протоколу IKEv2 клиент облегчает взаимодействие с шлюзов VPN третьих лиц, которые поддерживает этот тип туннелирования стандартные для отрасли. Также можно добиться взаимодействия со шлюзами сторонних поставщиков VPN с помощью подключаемого модуля UWP VPN в сочетании с пользовательским типом туннелирования без ущерба для возможностей платформы VPN и преимущества.<p><p>_**Примечание.**_<br>Обратитесь к поставщику устройства серверное шлюза или стороннее решение о конфигурации и совместимость с VPN и туннеля устройства с помощью IKEv2. |
| **Поддержка протокола IKEv2 VPN стандартные для отрасли** | Клиент VPN поддерживает IKEv2, один из современных наиболее широко используется протоколов тунеллирования стандартные для отрасли. Эта совместимость обеспечивает максимальную совместимость с шлюзами VPN сторонних. |
| **Поддержка платформы** | Поддерживает AAlways VPN-профиль присоединенный к домену, неподключенные к домену (рабочей группы) или Azure AD — устройств для разрешить для обоих предприятий и собственных устройств (BYOD) сценариях. Кроме того VPN доступны во всех выпусках Windows. |
| **Различные механизмы управления и развертывания** | Многие механизмы управления и развертывания можно использовать для управления параметрами виртуальной частной сети (называемые _профиля VPN_), включая Windows PowerShell, System Center Configuration Manager, Intune (или мобильных устройств сторонних поставщиков [средства управления MDM]) и Windows Конструктор конфигураций. Эти параметры упрощения конфигурации VPN независимо от того, средства управления клиентами, которые можно использовать. |
| **Определение стандартных профиля VPN** | VPN поддерживает конфигурации с помощью стандартный профиль XML (ProfileXML), предоставляя формат шаблона стандартной конфигурации, используемую большинство наборов средств развертывания и управления. |
