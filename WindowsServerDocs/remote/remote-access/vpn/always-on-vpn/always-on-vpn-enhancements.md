# <a name="always-on-vpn-enhancements"></a>Усовершенствования постоянно подключенного VPN-профиля

>Область применения: Windows Server (половина ежегодного канала), Windows Server 2016, Windows 10

- [**Назад:** Дополнительные сведения о функциях VPN Always On](../vpn-map-da.md)
- [**Далее:** Дополнительные сведения о технологии VPN Always On](always-on-vpn-technology-overview.md)

Always On VPN имеет много преимуществ по сравнению с решениями VPN Windows в прошлом. Следующие основные улучшения выключают Always On VPN с помощью облачных и предварительных концепций Майкрософт.

- **Интеграция платформы:** Always On VPN улучшила интеграцию с операционной системой Windows и сторонними решениями, чтобы обеспечить надежную платформу для бесчисленных расширенных сценариев подключения.

- **Безопасность:** Always On VPN имеет новые, расширенные возможности безопасности для ограничения типа трафика, приложения могут использовать VPN-подключение, а также методы проверки подлинности, которые можно использовать для инициации подключения. Если подключение активно в большую часть времени, особенно важно защитить подключение. Дополнительные сведения см. в разделе [Параметры проверки подлинности VPN](/windows/security/identity-protection/vpn/vpn-authentication).

- **VPN-подключение:** Always On VPN с туннелированием устройства или без него предоставляет возможность автоматического триггера. До Always On VPN возможность запуска автоматического подключения через проверку подлинности пользователя или устройства не поддерживается.  

- **Управление сетью:** Always On VPN позволяет администраторам более детально задавать политики маршрутизации — даже для отдельных приложений, которые идеально подходят для бизнес-приложений, требующих специального удаленного доступа.  Always On VPN также полностью совместима с протоколом IP версии 4 (IPv4) и версией 6 (IPv6). В отличие от DirectAccess, нет особой зависимости от IPv6.

  >[!NOTE]
  >Прежде чем приступить к работе, обязательно включите IPv6 на VPN-сервере. В противном случае соединение не может быть установлено, и отображается сообщение об ошибке.

- **Конфигурация и совместимость:** Always On VPN можно развертывать и управлять несколькими способами, что дает Always On VPN несколько преимуществ по сравнению с другим программным обеспечением VPN-клиента.

## <a name="platform-integration"></a>Интеграция платформы

Корпорация Майкрософт предоставила или улучшила следующие возможности интеграции Always On VPN:


| Улучшение ключей   |  Описание  |
|----------------|---|
| **[Windows Information Protection (WIP)](/windows/threat-protection/windows-information-protection/protect-enterprise-data-using-wip)** | Интеграция с WIP позволяет применять политику сети, чтобы определить, разрешается ли трафик проходить через VPN. Если профиль пользователя активен и применяются политики WIP, Always On VPN автоматически активируется для подключения. Кроме того, при использовании WIP нет необходимости указывать правила Апптригжерлист и Траффикфилтерлист отдельно в профиле VPN (если требуется более сложная настройка), так как политики НЗП и списки приложений вступают в силу автоматически. |
|**[Windows Hello для бизнеса](/windows/access-protection/hello-for-business/hello-overview)** |Always On VPN изначально поддерживает Windows Hello для бизнеса (в режиме проверки подлинности на основе сертификатов), чтобы обеспечить простой единый вход для входа на компьютер и подключение к VPN. Поэтому для VPN-подключения не требуется дополнительная проверка подлинности (учетные данные пользователя), что позволяет использовать подключение Always On с проверкой подлинности Windows Hello для бизнеса. |
| **[Условный доступ Microsoft Azure](/azure/active-directory/active-directory-conditional-access-controls)**  |Always On VPN-клиент может интегрироваться с платформой условного доступа Azure, чтобы обеспечить многофакторную проверку подлинности (MFA), соответствие устройства или сочетание этих двух параметров. Если в соответствии с политиками условного доступа Azure Active Directory (Azure AD) выдается кратковременный сертификат проверки подлинности IP-безопасности (по умолчанию 60 минут), который затем может использоваться для проверки подлинности в VPN-шлюзе. Для соответствия устройств используются политики соответствия Configuration Manager/Intune, которые могут включать состояние аттестации работоспособности устройства в рамках проверки соответствия подключения.|
|  **Многофакторная идентификация Azure** |При совместном использовании служб протокол RADIUS (RADIUS) и расширения сервера политики сети (NPS) для Azure MFA проверка подлинности VPN может использовать надежный MFA. | **Сторонний подключаемый модуль VPN**  | С универсальная платформа Windows (UWP) сторонние сторонние поставщики услуг VPN могут создавать единое приложение для всего спектра устройств Windows 10. UWP предоставляет гарантированный базовый уровень API на разных устройствах, устраняя сложность и проблемы, часто связанные с написанием драйверов уровня ядра. В настоящее время подключаемые модули VPN Windows 10 UWP существуют для [Pulse Secure](https://www.microsoft.com/p/pulse-secure/9nblggh3b0bp), [F5 доступ](https://www.microsoft.com/p/f5-access/9wzdncrdsfn0), [Check Point капсула VPN](https://www.microsoft.com/p/check-point-capsule-vpn/9wzdncrdjxtj), [фортиклиент](https://www.microsoft.com/p/forticlient/9wzdncrdh6mc), [SonicWALL Mobile Connect](https://www.microsoft.com/p/sonicwall-mobile-connect/9wzdncrdsfkz)и [GlobalProtect](https://www.microsoft.com/p/globalprotect/9nblggh6bzl3); нет сомнений, другие будут отображаться в будущем. |

## <a name="security"></a>Безопасность

Основные улучшения безопасности приведены в следующих областях:

| Улучшение ключей | Описание  |
|---|---|
| **Фильтры трафика** | С помощью фильтров трафика можно указать политики на стороне клиента, которые определяют, какой трафик разрешен в корпоративной сети. Таким образом, администраторы могут применять ограничения на приложения или трафик для интерфейса VPN, ограничивая его использование конкретными источниками, портами назначения и IP-адресами. Доступны два типа правил фильтрации:<ul><li>**Правила на основе приложений.** Правила брандмауэра на основе приложений основаны на списке указанных приложений, поэтому только трафик, исходящий из этих приложений, может проходить через интерфейс VPN.</li><li>**Правила на основе трафика.** Правила брандмауэра на основе трафика основываются на сетевых требованиях, таких как порты, адреса и протоколы. Используйте эти правила только для трафика, соответствующего указанным условиям, которые могут проходить через VPN-интерфейс.<p><p>***Примечание***.<br>Эти правила применяются только к исходящему трафику с устройства. Использование фильтров трафика блокирует входящий трафик от корпоративной сети к клиенту. </li></ul> |**VPN для каждого приложения**|VPN для каждого приложения похожа на фильтр трафика на основе приложений, но он состоит в том, чтобы объединить триггеры приложений с фильтром трафика на основе приложений, чтобы VPN-подключение было ограничено конкретным приложением, а не всеми приложениями на VPN-клиенте. Эта функция автоматически инициируется при запуске приложения.|
|  **Поддержка настраиваемых алгоритмов шифрования IPsec**   |  Always On VPN поддерживает использование криптографических криптографических алгоритмов RSA и эллиптических кривых для удовлетворения строгих политик безопасности государственных организаций или организации.|
| **Поддержка протокола EAP в собственном режиме** |Always On VPN изначально поддерживает протокол EAP, который позволяет использовать различные типы EAP корпорации Майкрософт и сторонних производителей в рамках рабочего процесса проверки подлинности. EAP обеспечивает безопасную проверку подлинности на основе следующих типов проверки подлинности:<ul><li>Имя пользователя и пароль</li><li>Смарт-карта (как физических, так и виртуальных)</li><li>Пользовательские сертификаты</li><li>Windows Hello для бизнеса</li><li>Поддержка MFA с помощью интеграции EAP RADIUS</li></ul>Поставщик приложения управляет методами проверки подлинности подключаемых модулей VPN UWP сторонних разработчиков, хотя они имеют массив доступных параметров, включая пользовательские типы учетных данных и поддержку OTP.|

## <a name="vpn-connectivity"></a>VPN-подключение

Ниже приведены основные улучшения Always On VPN-подключения.

|  Улучшение ключей  | Описание |
|---|---|
|                    **Всегда включено**                    |                                                                                          Always On является компонентом Windows 10, который позволяет автоматически подключаться к активному профилю VPN и оставаться подключенным на основе триггеров — а именно, для входа пользователя, изменения состояния сети или активного экрана устройства. Always On также интегрированы в режим ожидания подключения, чтобы максимально увеличить время работы от аккумулятора.                                                                                           |
|             **Запуск приложений**              |                                                                                                                                          Можно настроить профили VPN в Windows 10 для автоматического подключения при запуске указанного набора приложений. Для настольных и приложений UWP можно настроить активацию VPN-подключения.                                                                                                                                          |
|              **Триггеры на основе имени**              |                                                                                                            С помощью Always On VPN можно определить правила, чтобы определенные запросы доменных имен активировали VPN-подключение. Windows 10 теперь поддерживает триггеры на основе имен для компьютеров, присоединенных к домену и не присоединенных к домену (ранее поддерживались только компьютеры, не присоединенные к домену).                                                                                                            |
|            **Обнаружение доверенной сети**            |                                                                                    Always On VPN включает эту функцию, чтобы гарантировать, что VPN-подключение не срабатывает, если пользователь подключен к доверенной сети в пределах корпоративной границы. Эту функцию можно объединить с любым из упомянутых выше методов запуска, чтобы обеспечить простое "подключение только при необходимости".                                                                                     |
| **[Туннель устройства](../vpn-device-tunnel-config.md)** | Always On VPN предоставляет возможность создания выделенного профиля VPN для устройства или компьютера. В отличие от *пользовательского туннеля*, которое подключается только после входа пользователя на устройство или компьютер, *туннель устройства* позволяет VPN устанавливать подключение перед входом пользователя в систему. Туннель туннеля устройства и пользователь взаимодействуют независимо с их профилями VPN. они могут быть подключены одновременно и могут использовать разные методы проверки подлинности и другие параметры конфигурации VPN. |

## <a name="networking"></a>Сеть

Ниже приведены некоторые улучшения сети в Always On VPN.


|              Улучшение ключей              |                                                                                                                                                                                                                Описание                                                                                                                                                                                                                |
|-------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Поддержка двойного стека для IPv4 и IPv6**  | Always On VPN изначально поддерживает использование IPv4 и IPv6 в подходе с двойным стеком. Он не имеет особой зависимости от одного протокола в другом, что позволяет обеспечить максимальную совместимость приложений IPv4/IPv6 в сочетании с поддержкой будущих потребностей в сети IPv6.<p>***Примечание.*** Прежде чем приступить к работе, обязательно включите IPv6 на VPN-сервере. В противном случае соединение не может быть установлено, и отображается сообщение об ошибке. |
| **Политики маршрутизации для конкретных приложений** |                            Помимо определения глобальных политик маршрутизации VPN-подключений для разделения трафика в Интернете и интрасети, можно добавить политики маршрутизации для управления использованием раздельного туннеля или конфигурации принудительного туннелирования на уровне приложения. Этот параметр обеспечивает более детальный контроль над тем, какие приложения могут взаимодействовать с ресурсами через VPN-туннель.                             |
|           **Исключаемые маршруты**            |                 Always On VPN поддерживает возможность указывать маршруты исключения, которые специально управляют поведением маршрутизации, чтобы определить, какой трафик должен проходить через VPN, а не через физический сетевой интерфейс.<p><p>***Примечания.***<br>— Маршруты исключения в настоящее время работают для трафика в той же подсети, что и клиент, например LinkLocal.<br>— Маршруты исключения работают только в настройке раздельного туннеля.                  |

|                 Улучшение ключей                  |                                                                                                                                                                                                                                                                                                                       Описание                                                                                                                                                                                                                                                                                                                       |
|--------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|    **Совместимость сторонних VPN-шлюзов**     | Always On VPN-клиент не требует использования VPN-шлюза на основе Майкрософт для работы. Благодаря поддержке протокола IKEv2 клиент упрощает взаимодействие с VPN-шлюзами сторонних производителей, которые поддерживают этот промышленный тип туннелирования. Вы также можете обеспечить взаимодействие со сторонними VPN-шлюзами с помощью подключаемого модуля универсального ключа пользователя UWP, объединенного с пользовательским типом туннелирования без ущерба для Always On функций и преимуществ платформы VPN.<p><p>***Примечание***.<br>Обратитесь к своему шлюзу или поставщику серверного устройства сторонних производителей по конфигурациям и совместимости с Always On VPN и туннелем устройств с помощью IKEv2. |
| **Стандартная промышленная поддержка протокола VPN IKEv2** |                                                                                                                                                                                                                              Always On VPN-клиент поддерживает IKEv2, один из современных наиболее широко используемых отраслевых протоколов туннелирования. Такая совместимость позволяет максимально повысить совместимость с VPN-шлюзами сторонних производителей.                                                                                                                                                                                                                               |
|               **Поддержка платформы**               |                                                                                                                                                                                                           Аалвайс в VPN поддерживает присоединенные к домену, не присоединенные к домену (workgroup) или присоединенные к Azure AD устройства, которые позволяют выполнять сценарии как для предприятий, так и для собственных устройств (BYOD). Кроме того, Always On VPN доступна во всех выпусках Windows.                                                                                                                                                                                                           |
| **Различные механизмы управления и развертывания** |                                                                                                                                 Вы можете использовать различные механизмы управления и развертывания для управления параметрами VPN (называемыми *профилем VPN*), включая Windows PowerShell, Microsoft Endpoint Configuration Manager, Intune или стороннего средства управления мобильными устройствами (MDM) и конструктор конфигураций Windows. Эти параметры упрощают настройку Always On VPN независимо от используемых вами средств управления клиентами.                                                                                                                                 |
|     **Стандартизированное определение профиля VPN**      |                                                                                                                                                                                                                                  Always On VPN поддерживает конфигурацию с помощью стандартного профиля XML (Профилексмл), предоставляя стандартный формат шаблона конфигурации, используемый большинством наборов средств управления и развертывания.                                                                                                                                                                                                                                   |
