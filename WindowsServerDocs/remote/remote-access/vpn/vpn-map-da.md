# VPN компоненты и возможности

>Область применения: Windows Server \(Semi-Annual Channel\), Windows Server 2016, Windows 10

& #171;  [ **Предыдущих:** всегда в развертывании VPN для Windows Server и Windows 10](always-on-vpn/deploy/always-on-vpn-deploy.md)<br>
& #187; [ **Далее:** сведения об усовершенствованиях постоянно включенной VPN](../vpn/always-on-vpn/always-on-vpn-enhancements.md)

В этом разделе вы узнаете о функциях и возможностях VPN.  В таблице ниже, не исчерпывающий список, тем не менее, он включает некоторые из наиболее распространенных компоненты и возможности, используемые в решений удаленного доступа. 

>[!TIP]
>Если вы сейчас используете DirectAccess, рекомендуется изучить VPN возможность тщательно проверьте Если решены всех требований удаленного доступа перед выполнением миграции форме DirectAccess к VPN.  

| Функциональные области | Постоянно подключенный VPN-профиль  |
| ---- | ---- |
| Непрерывное, прозрачный подключения к корпоративной сети. | Можно настроить VPN для поддержки автоматической активации зависимости от запросов на разрешение запуска или пространство имен приложения.<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/AlwaysOn**<br>**VPNv2/Имя_профиля/требуется**<br>**VPNv2/Имя_профиля/DomainNameInformationList/AutoTrigger** |
| Использование специальной туннеля инфраструктурой для предоставления подключения для пользователей, не входили в корпоративной сети. | Эти функции можно добиться с помощью компонента туннеля устройства в профиле VPN.<p><p>_**Примечание.**_<br>Туннеля устройства можно настроить только на устройствах присоединенный к домену, с помощью IKEv2 с проверкой подлинности сертификатов компьютера.<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/DeviceTunnel** |
| Использование управления масштабированием разрешить удаленного подключения для клиентов с системами управления находится в корпоративной сети. | Эти функции можно добиться с помощью компонента туннеля устройства в профиле VPN, в сочетании с настройки VPN-подключение на динамическую регистрацию IP-адресов, назначенных VPN-интерфейсу с использованием внутренних служб DNS.<p><p>_**Примечание.**_<br>Если включить фильтры трафика в профиле туннеля устройства туннеля устройства запрещает входящий трафик (из корпоративной сети на клиенте).<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/DeviceTunnel**<br>**VPNv2/Имя_профиля/RegisterDNS** |
| Относятся к возвращается в том случае, когда клиенты находятся за брандмауэрами и прокси-серверы. | Обратиться к SSTP (из IKEv2) можно настроить с помощью автоматического туннеля и протокол типа в профиле VPN.<p><p>_**Примечание.**_<br>Пользователь туннеля поддерживает SSTP и IKEv2, а туннеля устройства поддерживает IKEv2 только с поддержка SSTP резервный.<p>Определение с помощью:<br>**VPNv2/Имя_профиля/NativeProfile/NativeProtocolType** |
| Поддержка режима доступа end до края. | VPN предоставляет возможность подключения к корпоративным ресурсам с помощью политики туннеля, требующих проверки подлинности и шифрования, пока они не достигнут VPN-шлюза. По умолчанию сеансы туннеля завершить работу в шлюз VPN, который также выполняет функции шлюз IKEv2, предоставляя безопасности конечных до края. |
| Поддержка проверки подлинности сертификата компьютера. | IKEv2 тип протокола доступен как часть платформы VPN специально поддерживает использование сертификатов компьютера или компьютера для проверки подлинности VPN.<p><p>_**Примечание.**_<br>IKEv2 — это единственный поддерживаемый протокол для туннеля устройства и нет возможности поддержки для резервной проверки подлинности SSTP. <p>Определение с помощью:<br>**VPNv2/Имя_профиля/NativeProfile/проверки подлинности или MachineMethod** |
| Группы безопасности можно используйте для ограничения возможностей удаленного доступа для отдельных клиентов. | Можно настроить VPN для детального авторизации при использовании RADIUS, который включает использование групп безопасности для управления доступа через VPN. |
| Поддержка серверов позади edge брандмауэра или устройства NAT. | VPN дает возможность использовать такие протоколы, как SSTP и IKEv2, которые полностью поддерживают использование VPN-шлюз, расположенным за брандмауэром NAT устройства или edge.<p><p>_**Примечание.**_<br>Пользователь туннеля поддерживает SSTP и IKEv2, а туннеля устройства поддерживает IKEv2 только с поддержка SSTP резервный. |
| Возможность определить подключения к интрасети при подключении к корпоративной сети. | Обнаружение доверенной сети предоставляет возможность определять подключения корпоративной сети, и она основана на оценке подключения DNS-суффикс назначается сетевых интерфейсов и сетевого профиля.<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/TrustedNetworkDetection** |
| Соответствие требованиям, с помощью защиты доступа к сети (NAP). | Всегда на VPN-клиент можно интегрировать с Azure условного доступа для принудительного применения MFA, соответствия устройств или их комбинация. Когда политикам условного доступа, Azure AD выдает сертификат проверки подлинности IPsec кратковременные (по умолчанию 60 минут), клиент может затем использовать для проверки подлинности VPN-шлюза. Соответствия устройств использует политики соответствия System Center Configuration Configuration Manager/Intune, которые могут включать состояние работоспособности устройства. В настоящее время условным доступом Azure VPN предоставляет замены ближе всего к существующему решению NAP, несмотря на то, что существует не форма исправления возможностей сетевой службой или помещение. Дополнительные сведения см. в разделе [VPN и условный доступ](https://docs.microsoft.com/en-us/windows/security/identity-protection/vpn/vpn-conditional-access).<p>Определение с помощью:<br>**VPNv2/Имя_профиля/devicecompliance** |
| Возможность определять, какие серверы управления должны быть доступны, прежде чем вход пользователя в систему. | Эта функция VPN можно добиться с помощью компонента туннеля устройства (доступна в версии 1709 — для только IKEv2) в сочетании с фильтры трафика для управления системам управления в корпоративной сети через VPN-профиля Туннеля устройства.<p><p>_**Примечание.**_<br>Если включить фильтры трафика в профиле туннеля устройства туннеля устройства запрещает входящий трафик (из корпоративной сети на клиенте).<p>Определение с помощью:<br>**VPNv2/Имя_профиля/DeviceTunnel**<br>**VPNv2/Имя_профиля/TrafficFilterList** |

## Дополнительные возможности

Каждый элемент в этом разделе — это случай использования или часто используемых удаленного доступа к функциям, для которого VPN улучшила функциональные возможности — либо с помощью расширения функциональности или исключить их из предыдущего ограничение.


| Функциональные области | Постоянно подключенный VPN-профиль  |
| ---- | ---- |
| Присоединенный к домену устройств с требованием Enterprise SKU. | Поддержка VPN присоединенный к домену, неподключенные к домену (рабочей группы) или Azure AD — устройств для корпоративных и BYOD-сценариев. VPN доступны во всех выпусках Windows, и возможностей платформы доступны третьим лицам через поддержка подключаемого модуля UWP VPN.<p><p>_**Примечание.**_<br>Туннеля устройства можно настроить только на присоединенный к домену устройства с Windows 10 Корпоративная или для образовательных учреждений версии 1709 или более поздней версии. Не поддерживается для элемента управления стороннего туннеля устройства. |
| Поддержка IPv4 и IPv6. | С помощью VPN пользователи получают доступ к ресурсам корпоративной сети IPv4 и IPv6. Всегда на VPN-клиент использует двумя стека подход, который не зависит от специально IPv6 требуется или VPN-шлюза для предоставления NAT64 или DNS64 службы перевода. |
| Поддержка двухфакторной или проверкой подлинности OTP. |Платформа VPN по умолчанию поддерживает EAP, который позволяет использовать типы EAP сторонних поставщиков и разнообразных Майкрософт в рамках процесса проверки подлинности. VPN поддерживает, в частности смарт-карт (физических и виртуальных) и Windows Hello для бизнеса сертификаты для соблюдения требований двухфакторную проверку подлинности. Кроме того VPN поддерживает OTP через MFA (не поддерживается изначально, только для подключаемых модулей сторонних) через интеграции РАДИУС EAP.<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/NativeProfile/проверки подлинности** |
| Поддержка несколькими доменами и лесами. | Платформа VPN имеет независимо от на доменных служб Active Directory (AD DS) леса или домена топологию (или связанных функциональных схемы и уровни), так как не требуется VPN-клиент может быть должны быть присоединены к функции. Групповой политики таким образом, не зависимостей для определения параметров профиля VPN, так как вы ее не следует использовать во время настройки клиента. Там, где интеграция авторизации Active Directory, его можно добиться через RADIUS в рамках процесса проверки подлинности и авторизации EAP. |
| Поддержка обоих разделение и принудительного туннеля для разделения трафика Интернета или интрасети. | Можно настроить VPN для поддержки обоих для принудительного туннеля (по умолчанию режим работы) и изначально разделение туннеля. VPN предоставляет дополнительные детализации для маршрутизации политик по конкретному приложению.<p><p>_**Примечание.**_<br>Поддерживается только туннеля пользователя.<p><p>Определение с помощью:<p> **VPNv2/Имя_профиля/NativeProfile/RoutingPolicyType**<br>**VPNv2/Имя_профиля/TrafficFilterList/App/RoutingPolicyType** |
| Поддержка нескольких протоколов. | Можно настроить VPN для встроенной поддержки SSTP, если требуется Secure Sockets Layer резервный из IKEv2.<p><p>_**Примечание.**_<br>Пользователь туннеля поддерживает SSTP и IKEv2, а туннеля устройства поддерживает IKEv2 только с поддержка SSTP резервный.  |
| Помощник по подключения для предоставления состояния подключения к корпоративной. | VPN полностью интегрирована с собственным сетевого подключения помощник по и предоставляет состояние подключения от всех сетей представления интерфейса. С появлением Windows 10 Creators Update (версия 1703), состояние подключения VPN и управления подключение VPN для пользователя туннеля теперь доступны через всплывающее меню сети (для Windows встроенный клиент VPN), а также. |
| Разрешение имен для корпоративных ресурсов, с помощью короткое имя, полное доменное имя (FQDN) и DNS-суффикс. | Изначально VPN можно определить один или несколько DNS-суффиксы как часть VPN-подключение и процесса назначения адрес IP-адрес, включая разрешение имен корпоративных ресурсов для короткие имена, полных доменных имен или всей пространств имен DNS. VPN также поддерживает использование таблиц политики разрешения имен для разрешения имен конкретных детализации.<p><p>_**Примечание.**_<br>Избегайте использования глобальной суффиксов, так как они мешают разрешение короткое имя, при использовании таблицы политики разрешения имен.<p><p>Определение с помощью:<br>**VPNv2/Имя_профиля/DnsSuffix**<br>**VPNv2/Имя_профиля/DomainNameInformationList** |
