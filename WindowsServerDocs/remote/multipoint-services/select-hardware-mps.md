# <a name="selecting-hardware-for-your-multipoint-services-system"></a>Выбор оборудования для системы служб MultiPoint
При построении системы MultiPoint Services, следует выбрать компьютер, отвечающий требованиям системы Windows Server 2016. При выборе компонентов для выбора следует учитывайте следующее.  
  
-   Целевой диапазон цен полного решения.  
  
-   Типы сценариев использования, которые можно ожидать для системы MultiPoint Services, например ли пользователей работе с приложениями мультимедиа, используя текстовые редакторы и программы для повышения производительности или поиск в Интернете.  
  
-   Требует ли сценарию свойственны обработки больших или памяти.  
  
-   Число пользователей, которые может использовать система, в то же время. Если вы планируете использовать много пользователей в вашей системе, в то же время, или пользователей, использующих программы высокие требования к системе, необходимо запланировать большие вычислительные мощности для вашей системы.  
  
-   Тип станции. Сколько портов USB или видео порты требуется?  
  
-   Планы расширения в будущем. Вы планируете добавить станций в системе MultiPoint Services позднее? Будут у вас есть достаточно слотов видеоадаптер, USB-порты, или касается его сети? Для поддержки сколько дополнительных пользователей потребуется оборудование?  
  
-   Физическая структура. Дополнительные сведения см. в разделе [планирование сайта служб MultiPoint](MultiPoint-services-Site-Planning.md).  
  
Обычно в системе MultiPoint Services включает следующие компоненты:  
  
-   Один компьютер, на котором выполняется MultiPoint Services, в том числе ЦП, ОЗУ, жестких дисков и видеокарты.  
  
-   Монитор, концентратор станции, клавиатуры и мыши для каждой станции.  
  
-   Дополнительные периферийные устройства для станции MultiPoint Services, включая докладчиков, наушники, микрофонов или запоминающих устройств, которые доступны только пользователю станции.  
  
-   Необязательный периферийных устройств, которые доступны всем пользователям системы MultiPoint Services, подключенных непосредственно к главному компьютеру, таких как принтеры, внешние жесткие диски и запоминающих устройств USB.  
  
Используйте следующие сведения для принятия решений оборудования:  
  
-   [Выбрав ЦП](#selecting-a-cpu)  
-   [Выбор компонентов оборудования](#selecting-hardware-components)  
  
## <a name="selecting-a-cpu"></a>Выбрав ЦП  
Система MultiPoint Services представляет собой результат умножения\-среде пользователя, всем пользователям, подключенным к компьютеру один узел. Это увеличивает степень использования ЦП, так как все пользователи совместно используют тот же компьютер. Некоторые задачи, например мультимедийными программами \(к примеру, мультимедиа-проигрывателей или видео\-редактирования программного обеспечения\), большего количества обработки. Таким образом не забудьте выбрать ЦП, который может обрабатывать требования к обработке, количество пользователей и типов пользовательских сценариев, которые требуется поддерживать.  
  
MultiPoint Services требует x x64\-на основе ЦП, а система должна отвечать требованиям для компьютера, как описано в разделе [требования к оборудованию и рекомендации по повышению производительности](Hardware-Requirements-and-Performance-Recommendations.md).  
  
Для использования в системе MultiPoint Services с высоким, протестированы следующие типы процессоров\-требуют обработки программ, например мультимедийными программами:  
  
-   **Двойной\-процессор core:** Может поддерживать до 8 станции.  
-   **Четырехъядерный\-процессор core:** Может поддерживать до 16 станции.
-   **Четырехъядерный\-ядро процессора с многопоточностью:** Может поддерживать до 20 станции.      
-   **Шесть\-ядро процессора с многопоточностью:** Может поддерживать до 24 станции.  
  
На основе этой информации выберите ЦП, который соответствует требованиям обработки для системы MultiPoint Services. 
> [!NOTE] 
> При запуске приложений с большим объемом видео рекомендуется по крайней мере одного ядра в каждой станции.

## <a name="selecting-video-hardware"></a>Выбор видео оборудования
Видеоустройства выбранного должен поддерживать число мониторов, которые необходимы для числа пользователей, которые вы собираетесь использовать рабочий на станции MultiPoint Services. Кроме того, различных типах видео оборудования может предоставить более высокий\-решение для графических производительностью\-с большим объемом программ, таких как мультимедийного содержимого.  
  
Выберите видео оборудования, который может поддерживать максимальное количество мониторов для типа производительности, требуемый системой MultiPoint Services. Убедитесь, что проверить производительность оборудования видео, вы решили обеспечить соответствие требований к производительности.  
  
> [!NOTE]  
> Необходимо установить драйвер видеоадаптера, который поддерживает расширение рабочего стола на нескольких мониторах.  
  
Параметры оборудования:  
  
-   Внутренняя видеоадаптеров, использующих PCI или интерфейсу шине PCIe  
  
-   Внешние контроллерах видео, подключенные по USB  
  
Следующие разделы описывают возможности каждого из этих типов оборудования. Можно объединить внутренней видеоадаптеров и создать систему, которая должна управляться внешними контроллерами видео.  
  
### <a name="internal-video-cards"></a>Внутренняя видеоадаптеров  
Внутренняя видеоадаптер подключен\-в системной платы на компьютере. Внутренняя видеоадаптер — это решение, это может повысить производительность графики\-с большим объемом мультимедийными программами. Однако внутренняя видеоадаптер требует разъем PCI или PCI, доступных для подключения\-в системной платы. Многие высокой\-видеоадаптеров производительности требуется слот PCIe, но существует ограниченное число слотов PCIe системной платы. Вы должны знать, какие видеокарты слоты доступны на компьютере, таким образом, вы можете приобрести правильный тип видеоадаптеров.  
  
Число мониторов, которые могут подключаться к каждой видеоадаптер зависит от GPU, который будет использоваться для карты и количество портов, которые он поддерживает, обычно диапазоне от 2 до 6.  
  
При выборе внутреннего видеоадаптеров, выберите видеокарты, которые поддерживают количество мониторы, необходимые для создания требуемое число прямых видео подключенных станций. Максимальное число мониторов, которые могут поддерживаться равно числу внутренних видеокарт, подключены\-в системной платы, умноженное на количество портов монитора на каждом из этих видео карточек. Например если имеется два внутренних видеоадаптеров и каждая карта есть два монитора порта, может поддерживать до четырех мониторов.    
  
### <a name="external-video-controllers"></a>Внешними контроллерами видео  
Клиенты USB ноль содержат внешних видеоконтроллер для монитора подключения к клиенту. Клиент USB zero может также включать подключения для наушников, динамиков, микрофона или других периферийных устройств.  
  
Выберите USB ноль клиента, если вы хотите включить поддержку дополнительных мониторов без необходимости открывать на компьютере, или если требуется поддерживать несколько станций, чем доступных видеосигналы. Например, если ранее вы подключены четырех мониторов\-внутренней видеоадаптеров, и требуется добавить две дополнительные мониторы, можно подключить\-в два внешних контроллерах видео, чтобы компьютер и достаточно места для двух дополнительных мониторов. Таким образом можно объединять клиент USB zero с видеоконтроллер и не использовать дополнительные слоты PCI или PCIe на материнской плате.  
  
## <a name="BKMK_Selectingdirect-video-connectedorUSBzeroclientstationdevices"></a>Выбор direct\-видео\-подключен или USB нуль станции клиентских устройств  
Станции MultiPoint Services состоит из концентратора станции или USB-ноль клиента с помощью клавиатуры и мыши подключать\-in и монитор, подключенный\-в на главном компьютере или в клиент USB zero. Можно подключить другие периферийные устройства\-в станцию концентратора или USB-нуля клиента, но они не требуются для создания станции MultiPoint. Эти периферийные устройства описаны в [выбора других периферийных устройств станции](#selecting-other-station-peripheral-devices).  
  
Устройства, которые выбраны для создания станции MultiPoint Services должен удовлетворять минимальным требованиям для работы с системой MultiPoint Services. В этом разделе предоставляются сведения о требованиях для следующих устройств станции MultiPoint Services:  
  
-   [Выбрав USB-концентраторы](#selecting-usb-hubs)  
-   [Выбрав USB ноль клиентов](#selecting-usb-zero-clients)

-   [Выбрав мониторов](#selecting-monitors)  
  
### <a name="selecting-usb-hubs"></a>Выбрав USB-концентраторы  
USB-концентраторы, которые используются в системе MultiPoint Services может быть универсальным USB-концентратор. Обычно такие концентраторы имеют четыре или более USB-портов, и они дают несколько USB-устройства должен быть подключен к одному порту USB компьютера. Некоторые устройства, такие как клавиатуры или видеомониторы, могут включать USB-концентратор в своем проекте.  
  
Дополнительная рекомендация заключается в использовании *возможностью внешнего питания* центра, а не *шины\-на платформе* концентратора. С шиной\-концентратор электрического тока, которое предоставляется узлом компьютер должен быть достаточно для предоставления возможности для периферийных устройств, подключенных\-в концентратор, без снижения производительности системы. Концентратор с внешним питанием позволяет подключить более периферийных устройств и достаточную мощность к каждому из них. Использование концентраторов внешним питанием может помочь предотвратить проблемы с производительностью, порт сбоев и другие временные неполадки.  
  
При выборе USB-концентратор для системы MultiPoint Services, можно его использовать. Концентратор может использоваться как *концентратор станции*, *промежуточным концентратором*, или *подчиненный центр*. См. в следующей таблице, Дополнительные сведения по каждому типу концентратора. Мы рекомендуем всем устройствам USB, чтобы быть USB 2.0 или более поздней.
  
||На платформе|  
|-|-----------|  
|Концентратор станции|Может быть шины\-на платформе, если не высокой\-будет подключать устройства\-в его или подчиненного центра будет подключена к нему|  
|Промежуточным концентратором |Должен быть возможностью внешнего питания|  
|Подчиненный центр|Возможностью внешнего питания или шины на платформе, в зависимости от устройства, которые подключены\-в к концентратору|  
|Расширитель Active USB-кабеля|Active USB-кабели, включающие USB-концентратор, обычно являются шины на платформе; Таким образом они не рекомендуется для подключения к компьютеру концентраторами станций.|  
  
### <a name="selecting-usb-zero-clients"></a>Выбрав USB ноль клиентов  
Клиент USB zero — USB-концентратор, который содержит выходные видеоданные. Таким образом он позволяет монитора должен быть подключен к компьютеру через USB-подключение. Дополнительные сведения об использовании клиентов USB zero для видео, см. в разделе [Выбор видео оборудования](#selecting-video-hardware) в этом документе. Клиент USB zero также можно включить подключение различных USB и не\-USB-устройства к центру. Клиентов USB нуля создаются производителями оборудования, и требуют установки устройства\-конкретного драйвера.  
  
### <a name="selecting-keyboards-and-mouse-devices"></a>Выбрав клавиатурой и мышью  
Клавиатура и мышь устройства, подключаемые\-в к станции обычно будет USB-устройства. Некоторые USB ноль клиенты предоставляют PS\/2 порты, в этом случае клавиатуры и мыши следует использовать PS\/2 для подключения к концентратору станции. Можно также использовать PS\/2, клавиатуры и мыши, если вы настраиваете PS\/2 direct\-видео\-подключенный.  
  
Можно использовать клавиатуры с помощью внутреннего центра концентратора станции. Тем не менее все прочие устройства станции необходимо подключиться к внутреннему концентратору с использованием портов на клавиатуре. Если такой клавиатуры подключен к компьютеру через другой концентратор, этого концентратора будет рассматриваться как промежуточным концентратором.  
  
Если вы используете разделение\-станции экрана, вы можете рассмотреть возможность использования мини-клавиатура не поддерживает числовую таким образом, чтобы можно было загрузить две клавиатуры перед монитор.  
  
### <a name="selecting-monitors"></a>Выбрав мониторов  
Должен быть один монитор, предоставляемые для каждой станции MultiPoint Services, если разбиение\-планируется экрана. Монитор подключены к видеоадаптера на компьютере, клиент USB zero или локальной сети\-на основе клиента. Любой тип монитора, поддерживаемый видеоадаптер, клиент USB zero client или по локальной сети\-зависимости можно использовать клиент, включая мониторы CRT.  
  
Некоторые специальные мониторы включают внутренней сети\-на основе клиента или клиент USB zero client. Аудио вход как правило, включают такие мониторы\/вывода камешков и внутренней USB-концентраторы, для подключения клавиатуры и мыши. Они подключаются к серверу через USB или подключения по локальной сети.  
  
#### <a name="display-resolution"></a>Разрешение экрана  
Минимальное разрешение поддерживаемых область отображения станции — 512 x 768 пикселей. Если в системе MultiPoint Services запускает и обнаруживает, что область отображения станции меньше, чем минимальное разрешение, пустой экран будет отображаться на этой станции и станция будет непригодным для использования.  
  
Если монитором будет совместно использоваться две станции как разделенный\-экрана станции, минимальным требованием для отображения — 1024 x 768, таким образом, полученный области экрана отдельных станции менее 512 x 768. Лучшее разделение\-экрана взаимодействие с пользователем, рекомендуется ךנאםםו с как минимум разрешение 1600 x 900.  
  
## <a name="selecting-other-station-peripheral-devices"></a>Выбор других периферийных устройств станции  
Система multiPoint Services поддерживает периферийных устройств, подключенных к концентратору станции, клиент USB zero, или непосредственно к компьютеру. Устройства, подключенные к концентратору станции будет связан с этой конкретной станции. Другие устройства доступны для каждой станции при подключении непосредственно к компьютеру. Клиенты по локальной сети, также поддерживают периферийные устройства.  
  
> [!IMPORTANT]  
> Клавиатура невозможно подключить к концентратору подчиненных \(к примеру, к концентратору, который подключен к концентратору станции\). При подключении клавиатуры к концентратору подчиненных периферийных устройств, которые подключены\-в подчиненный центр будет больше не будут доступны для этой станции. Это поведение обеспечивает поддержку последовательно\-связанными концентраторами станций.  
  
**Для всех станций** устройства, подключенные к компьютеру \(для примера, не через концентратор станции\) доступна для всех станций. В зависимости от устройства он может использоваться несколькими пользователями одновременно, или только один пользователь может доступ к нему одновременно. В следующей таблице описаны способ обращения к USB-устройства.  
  
> [!NOTE]  
> Столбец «Подключен на главном компьютере» в таблице относится к поведение, когда компьютер, на котором работает MultiPoint Services работает в режиме станции с станции. При запуске в режиме консоли, периферийные устройства, подключенные в любом месте ведут себя так же, как стандартный сервер в сеансе консоли.

## <a name="BKMK_SelectingRDP-over-LAN-connectedstationhardware"></a>Выбор протокола удаленного рабочего СТОЛА\-над\-LAN\-подключенных оборудованием станции  
Любой клиент по локальной сети, который может подключиться к служб удаленных рабочих столов с помощью протокола удаленного рабочего стола, может стать станции MultiPoint Services.  
  
Если требуется, чтобы клиент по локальной сети можно использовать только как станцию MultiPoint, можно «блокировать» по локальной сети клиента. Например настройте тонкий клиент, чтобы его можно только подключения к сеансу MultiPoint Services или настроить настольных компьютерах, что доступ к значков и меню "Пуск", а также элементы, такие как удаления веб-браузер, чтобы предотвратить прямой доступ к Интернету. Эти настройки, с помощью средства настройки локальной сети клиента или группы или локальные политики, можно изменять.  
  
## <a name="selecting-audio-devices"></a>Выбор звуковых устройств  
Важно, чтобы убедиться в том, что при выборе аудиоустройства, их можно подключить к концентратору станции, клиент USB zero client или клиента по локальной сети. Некоторые USB-концентраторы, USB zero клиентов и клиентов локальной сети имеют аналоговый Джек аудио, который может использоваться с традиционной компания analog devices аудио \(например наушники или earbuds\). Концентраторы станций, у которых нет аналогового камешков можно использовать звуковые устройства USB.  
  
Если вы настроили PS\/2 direct\-видео\-соединенных станции с помощью PS\/2 порта на системной плате для клавиатуры и мыши, необходимо использовать аналоговое воспроизведение звука на материнскую плату в порядок для звукового устройства необходимо обеспечение доступа к этой станции, когда в системе MultiPoint Services работает в режиме станции.  
  
Если у вас нет PS\/2 direct\-видео\-подключенных станции, аудиоустройства узла на материнской плате компьютера будут доступны только в том случае, когда в системе MultiPoint Services выполняется в консольном режиме.
