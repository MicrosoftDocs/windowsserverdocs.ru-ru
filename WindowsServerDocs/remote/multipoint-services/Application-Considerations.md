# <a name="application-considerations"></a>Рекомендации по приложениям
  
## <a name="application-compatibility"></a>Совместимость приложений

Любое приложение, которое вы хотите запустить в системе MultiPoint Services необходимо выполнить следующие требования:
  
- Он должен установить и запустить в Windows Server 2016 
- Необходимо учитывать сеанса таким образом, каждый пользователь может выполнять экземпляр приложения в системе MultiPoint.
  
Если приложение указывает это требование, мы рекомендуем попытайтесь установить приложения и использовать его в сеанс удаленного рабочего стола. 

## <a name="addressing-application-compatibility-problems"></a>Проблемы совместимости приложений  
MultiPoint Services предоставляет возможность связывать станции с полной экземпляров выпусков Windows 10 Корпоративная, практического запуска на одном главном компьютере. Для важных приложений, которые не будет работать несколько экземпляров для нескольких пользователей, или не будет устанавливать на 64-разрядной операционной системе это может быть решение. Развертывание настольных систем, таким образом, необходимо использовать вкладку виртуальных рабочих столов в MultiPoint Manager, чтобы:  
  
-   Включить виртуальные рабочие столы  
-   Создание шаблона рабочего стола  
-   Настройка шаблона в приложении проблема  
-   Связывать станции с настроенным шаблоном  

Каждая станция начинается с помощью одного шаблона, поэтому любые изменения будут отменены при каждом запуске компьютера.  
  
>[!NOTE] 
>Очень важно проверить условия лицензионного соглашения для приложений, что вы хотите запустить на MultiPoint. Несмотря на то, что вы устанавливаете одну копию приложений может потребоваться Лицензирование на пользователя.
