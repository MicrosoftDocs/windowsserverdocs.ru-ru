# <a name="multipoint--stations"></a>Станции multiPoint
В системе MultiPoint Services *станции* являются конечными точками пользователя для подключения к компьютеру с запущенной MultiPoint Services. Каждая станция предоставляет пользователю независимых возможностей Windows 10. Поддерживаются следующие типы станции:  
  
-   Станций, подключенных напрямую видео  
  
-   Ноль клиента-подключаемом к USB-станции (включая клиенты USB через Ethernet ноль)  
  
-   Станции RDP-over локальной сети (для клиентских или тонким клиентом компьютеров)  
  
Полный ПК которых установлен соединитель MultiPoint можно отслеживать и контролировать с помощью MultiPoint Dashboard. В Windows 10 можно включить соединитель MultiPoint с помощью панели управления для функции Windows. 

MultiPoint Services поддерживает любое сочетание этих типов станции, но его рекомендуется одной станции станции подключенных напрямую видео, который можно использовать в качестве основного станции. Эта рекомендация связано сможет ожидается поддержка сценариев. Например чтобы взаимодействовать с системой по BIOS, до запуска MultiPoint Services.  
  
## <a name="primary-stations-and-standard-stations"></a>Основной станции и стандартный станций  
Одной станции на подключенных напрямую видео определяется как *Главная станция*. Остальные станции называются *стандартный станций*.  
  
Главная станция отображает экраны запуска при включении компьютера. Он предоставляет доступ к конфигурации системы и сведения об устранении, доступна только во время запуска. Первичное устройство должно быть станции подключенных напрямую видео. После запуска можно использовать как любой другой станции MultiPoint Главная станция.  
  
## <a name="direct-video-connected-stations"></a>Станций, подключенных напрямую видео  
Компьютер, на котором работает MultiPoint Services может содержать несколько видеоадаптеров, каждый из которых может содержать один или несколько портов на видео. Это позволяет подключать следит за несколько станций прямо в компьютер. Клавиатуры и мыши соединены через USB-концентраторы, которые связаны с каждого монитора. Эти центры называются *станции концентраторов*. Другие периферийные устройства, например динамики, наушники или запоминающих устройств USB, также может подключаться к концентратору станции, и они становятся доступны только пользователю станции.  
  
> [!IMPORTANT]  
> Должен быть по крайней мере один *подключенных напрямую видео станции* на каждый сервер в качестве основного станции для отображения процесса загрузки, когда компьютер включен.

### <a name="BKMK_PS2stations"></a>Станции PS/2  
С системой MultiPoint Services можно сопоставить PS/2 клавиатуру и мышь на материнской плате прямой подключенных монитор для создания станции PS/2. Аналоговые аудио высокой четкости на материнской плате является звук, связанный с этим типом станции. Это не относится к компьютерам при наличии не камешков PS/2 на материнской плате.  
  
## <a name="usb-zero-client-connected-stations"></a>Ноль клиента-подключаемом к USB-станции  
Использовать ноль клиента-подключаемом к USB-станции *клиент USB zero client* концентратора станции. Клиенты USB ноль иногда называют многофункционального концентратора с видео. Они являются центр, который подключается к компьютеру через USB-кабеля и этих центров, как правило, поддерживает видео монитор, мышь и клавиатура (PS/2 или USB), аудио- и дополнительных USB-устройства. В этом руководстве ссылается на эти специализированные концентраторов как USB ноль клиентов.

### <a name="usb-over-ethernet-zero-clients"></a>Клиенты USB через Ethernet нулю  
Клиенты USB через Ethernet нулевой — разновидность USB zero клиенты отправляют USB по локальной сети в систему MultiPoint Services. Эти типы клиентов USB zero работают так же, чтобы другие USB ноль клиентов, но не ограничивается максимальные значения длина кабеля USB. Клиенты USB через Ethernet нуля не традиционных тонкие клиенты, и они отображаются в виде виртуальных устройств USB в системе MultiPoint Services. При использовании этих устройств, обратитесь к производителю устройства для производительности и рекомендации по планированию сайта. Большинства устройств есть подключаемый модуль стороннего MultiPoint Manager, который позволяет связать и подключения устройства к системе MultiPoint Services.  
  
## <a name="rdp-over-lan-connected-stations"></a>Станции, подключенные к RDP через Локальную  
Тонкие клиенты и традиционными настольными, ноутбука или планшетных компьютеров, можно подключиться к компьютеру с MultiPoint Services через локальную сеть (LAN) с помощью протокола удаленного рабочего стола (RDP) или собственный протокол и протокол удаленного рабочего стола Поставщик. RDP-подключения обеспечивают взаимодействие с пользователем, которая очень похожа на другой станции MultiPoint, но использует оборудования на локальном клиентском компьютере. Дополнительные сведения о наших удаленного рабочего стола приложений доступны для Android, iOS, Mac и Windows в [клиенты удаленного рабочего стола](../remote-desktop-services/clients/remote-desktop-clients.md). 
  
Клиентов и устройств, работающих под управлением Microsoft RemoteFX может предоставляют широкие возможности мультимедиа, используя преимущества процессора и видеопамяти аппаратные возможности локального тонкого клиента или компьютера для предоставления видео высокой четкости по сети.  
  
При наличии существующих клиентов локальной сети MultiPoint Services можно указать быстрый и экономичный способ одновременного обновления всех пользователей для работы в Windows 10.  
  
С точки зрения развертывания и администрирования при используется RDP-over локальной сети станций существуют следующие отличия:  
  
-   Не ограничивается физической расстояния подключение USB  
  
-   Возможность повторно использовать старое компьютерное оборудование как станций  
  
-   Проще масштабировать более высокое значение станций. Любой клиент в сети можно потенциально использовать в качестве удаленной станции  
  
-   Устранение неполадок с помощью консоли диспетчера MultiPoint оборудования  
  
-   Никакая функциональность разделенным экраном.  
  
    Дополнительные сведения см. в разделе [станции с разделенным экраном](#a-namebkmksplitscreenstationsasplit-screen-stations) далее в этом разделе  
  
-   Настройка автоматического входа в систему через консоль MultiPoint Manager ни переименования станции

### <a name="BKMK_SplitscreenStations"></a>Станции с разделенным экраном  
MultiPoint Services предоставляет возможность разделить экран на компьютерах с станций, подключенных напрямую видео или станции, подключенного через USB ноль клиент. С разделенным экраном предоставляет возможность создания дополнительных станции каждого монитора. Не требует двух мониторов, чтобы создать две станции с одного монитора можно использовать один монитор с двумя настройками концентратора станции. Можно быстро увеличить число доступных станций без приобретения дополнительных мониторов, USB-нулю клиентов или видеоадаптеры.  
  
Преимущества использования станции с разделенным экраном могут включать:  
  
-   Сокращение расходов и занимаемого места за дополнительных пользователей в системе MultiPoint Services счет.  
  
-   Что позволяет двум пользователям совместно работать side-by-side, над проектом.  
  
-   Для демонстрации процедуры на одной станции, то время как учащийся на другие станции действий преподавателем.  
  
Монитор с разрешением 1024 x 768 или выше можно разделить на две станции экраны станцию любой служб MultiPoint. Для разделения экран пользовательского интерфейса рекомендуется ךנאםםו с минимальное разрешение 1600 x 900. Мини-клавиатуры без числовую также рекомендуется иметь две клавиатуры в соответствии с перед монитор.  
  
Создание станции с разделенным экраном, Настройка одной станции видео подключенных напрямую, или ноль клиента подключенные по USB. Затем добавляются концентратору станции дополнительных путем подключения в клавиатуру и мышь к USB-концентратору, который подключен к серверу. Затем можно преобразовать станции на две станции с помощью MultiPoint Manager позволяет разделить экран и сопоставить новый центр половины монитор. В левой части экрана становится одной рабочей станции, а правую половину второй станции.  
  
После разделения станции, один пользователь может войти станцию слева, а другой — на станцию справа.
