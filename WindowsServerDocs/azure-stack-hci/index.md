# <a name="azure-stack-hci-overview"></a>Обзор Azure Stack HCI

>Относится к: Windows Server 2019

Azure Stack HCI представляет собой гиперконвергентный кластер Windows Server 2019, в котором используется проверенное оборудование для локального выполнения виртуализированных рабочих нагрузок с возможностью подключения к службам Azure для облачного резервного копирования, восстановления сайтов и т. д. В решениях Azure Stack HCI используется оборудование, проверенное корпорацией Майкрософт, для обеспечения оптимальной производительности и надежности. Кроме того, они поддерживают такие технологии, как диски NVMe, энергонезависимая память и удаленный доступ к памяти (RDMA).

Решение Azure Stack HCI сочетает в себе несколько продуктов:

- оборудование нашего OEM-партнера;

- Windows Server 2019 Datacenter Edition;

- Windows Admin Center

- службы Azure (необязательно).

![Azure Stack HCI представляет собой гиперконвергентное решение Майкрософт, предоставляемое большим количеством партнеров — производителей оборудования.](media/AS_HCI_solution.png)

Azure Stack HCI представляет собой гиперконвергентное решение Майкрософт, предоставляемое большим количеством партнеров — производителей оборудования. Изучите следующие сценарии гиперконвергентных решений, которые помогут вам определить, будет ли Azure Stack HCI оптимальным для вас решением.

- **Обновление устаревающего оборудования.** Заменяйте старые серверы и инфраструктуру хранения, запускайте виртуальные машины Windows и Linux в локальной среде и в пограничной зоне, используя существующие навыки и средства работы с ИТ.

- **Консолидация виртуализированных рабочих нагрузок.** Консолидируйте устаревшие приложения в эффективной гиперконвергентной инфраструктуре. В вашем распоряжении та же облачная эффективность, которая используется для Microsoft Azure и других гипермасштабируемых центров обработки данных.

- **Подключение к Azure для создания гибридных облачных служб.** Упростите доступ к службам управления облаком и обеспечения безопасности в Azure, таким как внешнее резервное копирование, восстановление сайтов, облачный мониторинг и многое другое.

## <a name="the-azure-stack-family"></a>Семейство Azure Stack

Azure Stack HCI является частью Azure и семейства Azure Stack, то есть в нем используются те же программно определяемые ресурсы вычислений, хранения и сети, что и в Azure Stack. Вот краткая сводка по различным решениям:

- [Azure](https://azure.microsoft.com) — общедоступные облачные службы;
- [Azure Stack](https://azure.microsoft.com/overview/azure-stack) — локальное выполнение облачных служб;
- [Azure Stack HCI](https://azure.microsoft.com/overview/azure-stack/hci) — локальное выполнение виртуализированных приложений с возможностью подключения к Azure.

## <a name="hyperconverged-efficiencies"></a>Эффективность гиперконвергентной среды

Решения Azure Stack HCI объединяют ресурсы вычислений, хранения и сети с высокой степенью виртуализации на серверах и компонентах с архитектурой x86, соответствующих отраслевым стандартам. Объединение ресурсов в одном кластере упрощает для их развертывание, администрирование и масштабирование. Используйте для управления любимые средства автоматизации из командной строки или Windows Admin Center.

Обеспечьте для своих серверных приложений лучшую в отрасли производительность виртуальных машин с помощью Hyper-V — фундаментальной технологии низкоуровневой оболочки в облаке Майкрософт и технологии локальных дисковых пространств со встроенной поддержкой NVMe, энергонезависимой памяти и удаленного доступа к памяти (RDMA).

Безопасность приложений и данных можно поддерживать с помощью экранированных виртуальных машин, микросегментации сети и встроенных механизмов шифрования при хранении и передаче данных.

## <a name="hybrid-capabilities"></a>Гибридные возможности

Платформа с гиперконвергентной инфраструктурой в общедоступном облаке позволяет настроить совместную работу облачных и локальных решений. Ваша команда может нарабатывать навыки использования облачных технологий благодаря встроенной интеграции со службами управления инфраструктурой Azure:

- Azure Site Recovery для обеспечения высокой доступности и аварийного восстановления как услуги (DRaaS).

- Azure Monitor — централизованный узел для отслеживания всего, что происходит в приложениях, сети и инфраструктуре. Также поддерживает возможности расширенной аналитики на базе искусственного интеллекта.

- Облако-свидетель для использования Azure в качестве нетребовательного к ресурсам прерывателя соединений при поиске кворума в кластере.

- Служба Azure Backup для обеспечения безопасности данных во внешнем хранилище и для защиты от программ-шантажистов.

- Служба "Управление обновлениями" Azure для оценки и развертывания обновлений на виртуальных машинах Windows, которые работают в Azure или в локальной среде.

- Сетевой адаптер Azure для подключения локальных ресурсов к виртуальным машинам в Azure через VPN-подключение типа "точка — сеть".

- Синхронизация файлового сервера с облаком при помощи функции "Синхронизация файлов Azure".

Дополнительные сведения см. в разделе [о подключении Windows Server к гибридным службам Azure](..\manage\windows-admin-center\azure\index.md).

## <a name="management-tools-and-system-center"></a>Средства управления и System Center

В Azure Stack HCI используется такое же программное обеспечение, что и в Azure Stack, для поддержки виртуализации, программно определяемого хранилища и сети. Azure Stack HCI предоставляет вам полные административные права в кластере, то есть вы можете использовать [Windows Admin Center](..\manage\windows-admin-center\overview.md), [System Center](https://www.microsoft.com/cloud-platform/system-center) и любую функцию [Hyper-V](../virtualization/hyper-v/hyper-v-on-windows-server.md), [локальных дисковых пространств](..\storage\storage-spaces\storage-spaces-direct-overview.md), PowerShell и средств сторонних производителей, например диспетчера 5Nine.

Microsoft Azure работает на платформах Windows Server и Hyper-V, входящих в состав Windows Server. Windows Server и System Center содержат усовершенствования и реализации рекомендаций, разработанных на основе опыта Майкрософт в сфере управления глобальными сетями ЦОД, такими как Microsoft Azure, что позволяет вам развертывать те же самые программные сетевые технологии для обеспечения гибкости, автоматизации и управляемости.

Развертывайте инфраструктуру и управляйте ею с помощью System Center Virtual Machine Manager (VMM) и System Center Operations Manager. С помощью VMM вы можете подготавливать и контролировать ресурсы, необходимые для создания и развертывания виртуальных машин и служб в частных облаках. С помощью Operations Manager можно отслеживать службы, устройства и операции по всему предприятию для выявления проблем и немедленного их исправления.

## <a name="faq"></a>Вопросы и ответы

### <a name="what-do-azure-stack-and-azure-stack-hci-solutions-have-in-common"></a>Что общего между решениями Azure Stack и Azure Stack HCI?

В решениях Azure Stack HCI используются те же программно определяемые технологии вычислений, хранилища и сети на основе Hyper-V, что и в Azure Stack. Оба предложения соответствуют строгим критериям тестирования и проверок, что позволяет гарантировать надежность и совместимость с базовой платформой оборудования.

### <a name="how-are-they-different"></a>Чем они отличаются?

В Azure Stack локально выполняются облачные службы. Вы можете запускать службы Azure IaaS и PaaS в локальной среде, чтобы последовательно создавать и запускать облачные приложения в любом месте, а также управлять ими с помощью локального портала Azure.

В Azure Stack HCI вы локально выполняете виртуализированные рабочие нагрузки, управляя ими с помощью Windows Admin Center и хорошо знакомых средств Windows Server. При необходимости вы можете подключиться к Azure для реализации гибридных сценариев, таких как восстановление сайтов с использованием облака, облачный мониторинг и т. п.

### <a name="why-is-microsoft-bringing-its-hci-offering-to-the-azure-stack-family"></a>Почему корпорация Майкрософт добавила в семейство Azure Stack предложение HCI?

Гиперконвергентные технологии корпорации Майкрософт уже сейчас используются как основа для Azure Stack.

Многие клиенты корпорации Майкрософт используют сложные ИТ-среды, и мы стремимся предоставить им решения, которые соответствуют их ситуации и позволяют использовать нужные технологии для тех или иных бизнес-требований. Azure Stack HCI развивается на основе программно-определяемых решений Windows Server (WSSD) на базе Windows Server 2016, которые ранее предоставлялись нашими партнерами по оборудованию. Мы включили это предложение в семейство Azure Stack, так как подготовили для него новые варианты подключения к Azure для использования служб управления инфраструктурой.

### <a name="does-azure-stack-hci-need-to-be-connected-to-azure"></a>Нужно ли подключать Azure Stack HCI к Azure?

Нет, это совершенно необязательно. Вы можете воспользоваться преимуществами интеграции с Azure для гибридных сценариев, таких как удаленное резервное копирование и аварийное восстановление, облачный мониторинг и управление обновлениями, но все они являются необязательными. Мы полностью понимаем и поддерживаем ваше желание или потребность не использовать подключение к Интернету.

### <a name="how-does-azure-stack-hci-relate-to-windows-server"></a>Как связаны между собой Azure Stack HCI и Windows Server?

Windows Server 2019 — основа практически всех продуктов Azure. Все важные для вас функции будут и далее предоставляться и поддерживаться в Windows Server. Azure Stack HCI станет рекомендуемым способом развертывания локальной гиперконвергентной инфраструктуры на оборудовании наших партнеров, проверенном корпорацией Майкрософт.

### <a name="will-i-be-able-to-upgrade-from-azure-stack-hci-to-azure-stack"></a>Смогу ли я обновить Azure Stack HCI до Azure Stack? 

Нет, но клиенты смогут перенести свои рабочие нагрузки из Azure Stack HCI в Azure Stack или Azure.

### <a name="what-azure-services-can-i-connect-to-azure-stack-hci"></a>Какие службы Azure я могу подключить к Azure Stack HCI?

Обновляемый список служб Azure, которые можно подключить к Azure Stack HCI, вы найдете в статье [о подключении Windows Server к гибридным службам Azure](../azure-hybrid-services/index.md).

### <a name="how-does-the-cost-of-azure-stack-hci-compare-to-azure-stack"></a>Как соотносятся затраты на Azure Stack HCI и Azure Stack? 

Azure Stack продается как полностью интегрированная система, включающая все вспомогательные службы и поддержку. Вы можете приобрести Azure Stack как самостоятельно управляемую систему или как полностью управляемую службу от наших партнеров. В дополнение к базовой системе предоставляются службы Azure с оплатой по мере использования, которые выполняются в Azure Stack или Azure.

Решения Azure Stack HCI соответствуют традиционной модели приобретения. Вы можете приобрести прошедшее проверку оборудование у наших партнеров по Azure Stack HCI и программное обеспечение (Windows Server 2019 Datacenter Edition с программно-определяемыми возможностями центра обработки данных и Windows Admin Center) из любых существующих каналов. Все службы Azure, которые можно использовать с Windows Admin Center, оплачиваются через подписку Azure.

### <a name="how-do-i-buy-azure-stack-hci-solutions"></a>Как приобрести решения Azure Stack HCI?

Выполните следующие действия.

1. Приобретите проверенную корпорацией Майкрософт аппаратную систему у любого из наших партнеров по производству оборудования.
1. Установите Windows Server 2019 Datacenter Edition и Windows Admin Center, чтобы получить возможность управления и подключения к Azure для использования облачных служб.
1. Вы также можете применить учетную запись Azure для подключения облачных служб управления и обеспечения безопасности к своим рабочим нагрузкам.

|Azure Stack ограничивает возможности настройки Hyper-V и набор доступных функций, чтобы поддерживать совместимость с Azure.|Используйте Azure Stack HCI для беспроблемной виртуализации Hyper-V для классических корпоративных приложений, например Exchange, SharePoint и SQL Server, а также для виртуализации ролей Windows Server, таких как файловый сервер, DNS, DHCP, IIS и AD. Неограниченный доступ к использованию всех функций Hyper-V, например экранированных виртуальных машин.|
|Azure Stack не предоставляет такие инфраструктурные технологии.|Используйте Azure Stack HCI для создания программно-определяемой инфраструктуры взамен устаревающих массивов хранения данных или сетевых устройств, не изменяя архитектуру существенным образом. Встроенные функции локальных дисковых пространств и программно-определяемой сети обеспечивают беспроблемную интеграцию со средами Hyper-V.|
